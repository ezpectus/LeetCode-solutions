# 2663. Lexicographically Smallest Beautiful String
## O(n ¬∑ k) ‚ÄĒ Rightmost Increment + Greedy Forward Fill

---

## Problem Statement

- A string is **beautiful** if it uses only the first `k` lowercase letters and contains no palindrome substring of length ‚Č• 2.
- Given a beautiful string `s` (1 ‚Č§ n ‚Č§ 10‚ĀĶ, 4 ‚Č§ k ‚Č§ 26), return the lexicographically smallest beautiful string strictly greater than `s`, or `""` if none exists.

---

## Core Insight ‚ÄĒ No Palindrome ‚üļ Two Local Rules

A string with no palindrome substring of length ‚Č• 2 must satisfy exactly two conditions:
- `s[i] != s[i-1]` ‚ÄĒ no length-2 palindrome
- `s[i] != s[i-2]` ‚ÄĒ no length-3 palindrome (middle char is free)

Length ‚Č• 4 palindromes always contain a length-2 or length-3 palindrome as a substring, so these two rules are **sufficient**.

This makes validity checking O(1) per position ‚ÄĒ only look at the two previous characters.

---

## Algorithm ‚ÄĒ Next Permutation Style

1. Scan from **right to left** trying to increment a position
2. At position `i`: try chars from `s[i]+1` up to the `k`-th letter
   - Pick the smallest that doesn't conflict with `s[i-1]` or `s[i-2]`
   - If found: stop and fill the rest greedily forward
   - If none found: backtrack to `i-1`
3. Fill positions `i+1..n-1` with the smallest valid char (‚Č† previous two)
4. If we exhaust all positions: return `""`

**Why k ‚Č• 4 guarantees forward fill always succeeds:** at any position, at most 2 chars are banned (previous and two-back). With k ‚Č• 4 choices, we always have at least 2 valid options.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string SmallestBeautifulString(string s, int k) {
        char[] arr = s.ToCharArray();
        int n = arr.Length;
        char limit = (char)('a' + k); // exclusive upper bound

        int i = n - 1;

        while (i >= 0) {
            arr[i]++;

            // Advance until we find a valid char or exceed limit
            while (arr[i] < limit) {
                bool valid = true;
                if (i > 0 && arr[i] == arr[i - 1]) valid = false;
                if (i > 1 && arr[i] == arr[i - 2]) valid = false;
                if (valid) break;
                arr[i]++;
            }

            if (arr[i] < limit) {
                // Found a valid increment ‚ÄĒ fill rest greedily
                for (int j = i + 1; j < n; j++) {
                    for (char c = 'a'; c < limit; c++) {
                        bool valid = true;
                        if (j > 0 && c == arr[j - 1]) valid = false;
                        if (j > 1 && c == arr[j - 2]) valid = false;
                        if (valid) { arr[j] = c; break; }
                    }
                }
                return new string(arr);
            }

            // All chars at position i exhausted, backtrack
            i--;
        }

        return "";
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n ¬∑ k) | At most k increments per position, n positions |
| Space Complexity | O(n) | Character array for in-place editing |

In practice O(n) ‚ÄĒ the backtrack loop rarely goes far, and k ‚Č§ 26 is constant.

## Why This Works ‚ÄĒ Example Walkthrough

**Example 1:** `s = "abcz"`, `k = 26` (limit = '{')

- `i = 3`: `'z'++` = `'{'` = limit ‚Üí no valid char, `i--`
- `i = 2`: `'c'++` = `'d'`
  - Check: `'d' != arr[1]='b'` ‚úď and `'d' != arr[0]='a'` ‚úď ‚Üí valid!
  - Fill `j = 3`: try `'a'`: `'a' != 'd'` ‚úď and `'a' != 'b'` ‚úď ‚Üí `arr[3] = 'a'`
- Return `"abda"` ‚úď

---

**Example 2:** `s = "dc"`, `k = 4` (limit = 'e', chars: a,b,c,d)

- `i = 1`: `'c'++` = `'d'`
  - Check: `'d' == arr[0]='d'` ‚úó ‚Üí `'d'++` = `'e'` = limit ‚Üí no valid char, `i--`
- `i = 0`: `'d'++` = `'e'` = limit ‚Üí no valid char, `i--`
- `i < 0` ‚Üí return `""` ‚úď

---

**Trickier case:** `s = "aba"`, `k = 4`

- `i = 2`: `'a'++` = `'b'`
  - `'b' == arr[1]='b'` ‚úó ‚Üí `'b'++` = `'c'`
  - `'c' != 'b'` ‚úď and `'c' != 'a'` ‚úď ‚Üí valid! `arr[2] = 'c'`
- No fill needed (i = n-1)
- Return `"abc"` ‚úď (larger than "aba", no palindromes: a‚Č†b, b‚Č†c, a‚Č†c)

---

## Pitfalls & Edge Cases

- **Both local rules must pass** ‚ÄĒ It's easy to forget `s[i] != s[i-2]`. For example, `"aba"` is invalid (palindrome "aba"), even though adjacent chars differ.

- **Backtrack resets forward chars** ‚ÄĒ When backtracking, positions after `i` get filled fresh in the forward pass. Don't rely on their old values during the increment search.

- **`arr[i]++` before checking** ‚ÄĒ We increment first (we need strictly greater than current), then validate. The while loop handles the case where we skip multiple invalid chars.

- **Last char is max** ‚ÄĒ If `s = "abdc"` and the last char is `'d'` (= k-th letter), increment gives limit ‚Üí must backtrack.

- **Empty return** ‚ÄĒ Only when every position has been exhausted (e.g., `s` is the largest beautiful string for given `k`, like `"dc"` with k=4).

---

## Key Takeaway

This is **next permutation** adapted with a validity constraint:
- Find rightmost position that can be incremented to a valid char
- Fill the suffix with the greedy minimum

The constraint `s[i] ‚Č† s[i-1]` and `s[i] ‚Č† s[i-2]` is the beautiful string condition reduced to two local checks ‚ÄĒ no global scanning needed.

**Pattern recognition:** "Find next lexicographic string satisfying local constraints" ‚Üí rightmost increment + greedy fill. The local nature of the constraint (only depends on the previous 2 chars) makes this O(n) amortized with at most k constant-work attempts per position.
