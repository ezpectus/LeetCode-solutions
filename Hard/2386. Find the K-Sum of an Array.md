# 2386. Find the K‑Sum of an Array  
## O(n log n + k log n) — Max Subsequence Sum via Min‑Heap of “Excluded Negatives”

---

## Problem Statement

You are given:

- an integer array `nums`
- a positive integer `k`

A **subsequence** is any subset of elements in original order (possibly empty).  
Each subsequence has a **sum**.

Define:

> **K‑Sum** = the k‑th largest subsequence sum (not necessarily distinct).

Return that value.

Constraints:

- `1 ≤ n ≤ 100000`
- `−10⁹ ≤ nums[i] ≤ 10⁹`
- `1 ≤ k ≤ min(2000, 2ⁿ)`

---

## Core Idea — Transform to “Subtract Smallest Negative Contributions”

### Step 1 — Compute the maximum possible subsequence sum

Take all **positive** numbers:

```
maxSum = sum(x for x in nums if x > 0)
```

This is the **largest** subsequence sum.

### Step 2 — Convert the problem

Let’s define:

- Let `absNums = sorted(|nums[i]|)` in **ascending** order.

Every subsequence sum can be represented as:

```
maxSum − (sum of some subset of absNums)
```

Thus:

> Finding the k‑th largest subsequence sum  
> is equivalent to  
> finding the k‑th smallest “penalty sum” of excluded absolute values.

This is now a classical **k‑th smallest subset sum** problem.

### Step 3 — Use a min‑heap to generate the k smallest penalties

We push into a min‑heap:

```
( penalty = absNums[0], index = 0 )
```

Then repeatedly:

1. Pop the smallest penalty `(p, i)`
2. This corresponds to subsequence sum: `maxSum − p`
3. Push next candidates:
   - `(p + absNums[i+1], i+1)` — include next element
   - `(p − absNums[i] + absNums[i+1], i+1)` — replace current element with next

We repeat **k−1** times, because:

- the 1st largest sum is `maxSum` (penalty = 0)
- the next k−1 penalties give the k‑th largest sum

This is identical to the classical “k smallest sums from sorted list” algorithm.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long KSum(int[] nums, int k) {
        long maxSum = 0;
        List<long> arr = new List<long>();

        foreach (int x in nums) {
            if (x > 0) maxSum += x;
            arr.Add(Math.Abs((long)x));
        }

        arr.Sort();

        // If k == 1, the answer is simply the maximum subsequence sum
        if (k == 1) return maxSum;

        // Min-heap of (penalty, index)
        var pq = new PriorityQueue<(long sum, int idx), long>();
        pq.Enqueue((arr[0], 0), arr[0]);

        long penalty = 0;

        // We already counted the first subsequence (penalty = 0)
        for (int count = 1; count < k; count++) {
            var top = pq.Dequeue();
            penalty = top.sum;
            int i = top.idx;

            if (i + 1 < arr.Count) {
                // Option 1: extend subset with arr[i+1]
                pq.Enqueue((penalty + arr[i + 1], i + 1), penalty + arr[i + 1]);

                // Option 2: replace arr[i] with arr[i+1]
                pq.Enqueue((penalty - arr[i] + arr[i + 1], i + 1),
                           penalty - arr[i] + arr[i + 1]);
            }
        }

        return maxSum - penalty;
    }
}
```

## Example Walkthrough

### Example 1  
`nums = [2,4,-2], k = 5`

Positive sum:

```
maxSum = 2 + 4 = 6
absNums = [2,2,4]
```

Penalties generated in order:

```
0, 2, 2, 4, 4, 6, ...
```

Corresponding subsequence sums:

```
6, 4, 4, 2, 2, 0, ...
```

5‑th largest = **2**.

---

## Pitfalls & Edge Cases

- Must use `long` — sums can exceed 32‑bit.
- Negative numbers are handled via absolute values.
- The empty subsequence is valid (sum = 0).
- k = 1 → answer is simply `maxSum`.
- Sorting absolute values is essential for heap correctness.

---

## Key Takeaway

This problem reduces to:

> **K‑th largest subsequence sum**  
> =  
> **maxSum − k‑th smallest penalty**,  
> where penalties are subset sums of sorted absolute values.

A min‑heap best‑first search over penalties yields an elegant and efficient solution.
