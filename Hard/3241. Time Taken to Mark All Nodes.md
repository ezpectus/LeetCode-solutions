# LeetCode 3241. Time Taken to Mark All Nodes

## Problem Description
There exists an undirected tree with `n` nodes numbered `0` to `n - 1`. You are given a 2D integer array `edges` of length `n - 1`.
Initially, all nodes are unmarked. For each node `i`:
* If `i` is **odd**, it takes `1` unit of time to mark it from an adjacent marked node.
* If `i` is **even**, it takes `2` units of time to mark it from an adjacent marked node.

Return an array `times` where `times[i]` is the minimum time when all nodes get marked in the tree if you start marking node `i` at time `t = 0`.

---

## Method Explanation: Tree Rerooting DP (Two-Pass DFS)

If we were to compute the maximum time for each node individually as a root, it would take $\mathcal{O}(N^2)$ time, leading to a Time Limit Exceeded (TLE) error. Instead, we use a **Two-Pass Rerooting Dynamic Programming** strategy to solve it in linear time.

### 1. First Pass (Bottom-Up DFS)
We choose an arbitrary node (e.g., node `0`) as the fixed root. For each node `u`, we compute the maximum time required to mark all nodes within its subtree. 
* To seamlessly reroot later, we must store both the **primary maximum time** (`max_depth1`) and the **secondary maximum time** (`max_depth2`) coming from different child branches, along with the identifier of the child that yielded the absolute maximum (`best_child`).

### 2. Second Pass (Top-Down Rerooting DFS)
We propagate down from the root to compute the time needed to reach parts of the tree *outside* of the current node's subtree (coming from its parent's direction). 
* When moving from parent `u` to child `v`, the maximum time coming from the parent's side is determined by picking either the parent's own outer carry-over or the best sibling branch of `u` that does not include `v`. We then append the cost of moving into `u` (`1` if `u` is odd, `2` if even).

## C++ Source Code

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
private:
    vector<vector<int>> adj;
    vector<int> max_depth1, max_depth2, best_child;
    vector<int> ans;

    // Transition cost depends entirely on the target node's parity
    inline int get_cost(int node) {
        return (node % 2 == 1) ? 1 : 2;
    }

    // Pass 1: Compute subtree maximums
    void dfs_subtree(int u, int p) {
        max_depth1[u] = 0;
        max_depth2[u] = 0;
        best_child[u] = -1;

        for (int v : adj[u]) {
            if (v == p) continue;
            dfs_subtree(v, u);
            
            int cost = get_cost(v) + max_depth1[v];
            if (cost > max_depth1[u]) {
                max_depth2[u] = max_depth1[u];
                max_depth1[u] = cost;
                best_child[u] = v;
            } else if (cost > max_depth2[u]) {
                max_depth2[u] = cost;
            }
        }
    }

    // Pass 2: Reroot top-down and accumulate global answers
    void dfs_reroot(int u, int p, int partial_max) {
        ans[u] = max(max_depth1[u], partial_max);

        for (int v : adj[u]) {
            if (v == p) continue;
            
            // Choose the max path from parent side avoiding the branch of v itself
            int sibling_max = (best_child[u] == v) ? max_depth2[u] : max_depth1[u];
            int from_parent = max(partial_max, sibling_max) + get_cost(u);
            
            dfs_reroot(v, u, from_parent);
        }
    }

public:
    vector<int> timeTaken(vector<vector<int>>& edges) {
        int n = edges.size() + 1;
        adj.assign(n, vector<int>());
        for (const auto& edge : edges) {
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]);
        }

        max_depth1.assign(n, 0);
        max_depth2.assign(n, 0);
        best_child.assign(n, -1);
        ans.assign(n, 0);

        dfs_subtree(0, -1);
        dfs_reroot(0, -1, 0);

        return ans;
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N)$
  * **Two-Pass DP Traversal:** The tree structure is scanned exactly twice—once bottom-up via subtree aggregation and once top-down using explicit parent-to-child rerooting.
  * **Linear Bounds:** Each directional edge and vertex transition evaluates a set of primitive conditional statements in strict $\mathcal{O}(1)$ time.
  * **Total Time:** $\mathcal{O}(N)$, scaling linearly with the number of nodes up to the $10^5$ boundary constraint.

* **Space Complexity:** $\mathcal{O}(N)$
  * **Graph Representation:** Allocations match tree constraints line items exactly, building a clean adjacency mapping requiring $\mathcal{O}(N)$ space.
  * **State Vectors:** Auxiliary memory consumption remains bounded to a small cluster of tracking arrays (`max_depth1`, `max_depth2`, `best_child`, and `ans`) storing data arrays of size $n$, resulting in an operational footprint of $\mathcal{O}(N)$.
