# 1192. Critical Connections in a Network
## O(V + E) — Tarjan's Bridge-Finding Algorithm

---

## Problem Statement

- Find all "bridges" in an undirected graph — edges whose removal disconnects the graph.

---

## Core Idea — Tarjan's Algorithm (disc/low arrays)

**DFS** assigns each node a discovery time `disc[u]`. Also compute `low[u]` = minimum discovery time reachable from the subtree rooted at `u` (via back edges).

**Bridge condition**: edge `(u, v)` is a bridge iff `low[v] > disc[u]` — meaning the subtree rooted at `v` has no back edge to `u` or its ancestors. Removing `(u,v)` would disconnect `v`'s subtree.

**Transitions**:
- Tree edge to unvisited `v`: recurse, then `low[u] = min(low[u], low[v])`.
- Back edge to visited `v` (not parent): `low[u] = min(low[u], disc[v])`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int[] disc, low;
    int timer = 0;
    List<int>[] graph;
    List<IList<int>> res = new();

    public IList<IList<int>> CriticalConnections(int n, IList<IList<int>> connections) {
        disc = new int[n]; low = new int[n];
        Array.Fill(disc, -1);
        graph = new List<int>[n];
        for (int i = 0; i < n; i++) graph[i] = new();
        foreach (var e in connections) {
            graph[e[0]].Add(e[1]);
            graph[e[1]].Add(e[0]);
        }
        Dfs(0, -1);
        return res;
    }

    void Dfs(int u, int parent) {
        disc[u] = low[u] = timer++;
        foreach (int v in graph[u]) {
            if (v == parent) continue;
            if (disc[v] == -1) {
                Dfs(v, u);
                low[u] = Math.Min(low[u], low[v]);
                if (low[v] > disc[u]) res.Add(new List<int>{u, v});
            } else {
                low[u] = Math.Min(low[u], disc[v]);
            }
        }
    }
}
```

## Example Walkthrough

**`n=4, connections=[[0,1],[1,2],[2,0],[1,3]]`**

DFS from 0: disc=[0,1,2,3], low=[0,0,0,3].
- Edge (1,3): low[3]=3 > disc[1]=1 → bridge [1,3] ✓

---

## Pitfalls & Edge Cases

- **Skip parent**: `if (v == parent) continue` — avoids treating the edge we came from as a back edge.
- **Multigraph**: if there are multiple edges between same nodes, track by edge index not parent node. (Not an issue here per constraints.)
- **`disc[v] == -1`**: unvisited node check.

---

## Key Takeaway

"Find bridges in undirected graph" → **Tarjan's: DFS + disc/low arrays. Bridge iff `low[v] > disc[u]`**.
