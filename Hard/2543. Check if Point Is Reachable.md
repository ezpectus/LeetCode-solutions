# 2543. Check if Point Is Reachable
## O(log(min(x,y))) — GCD Must Be a Power of Two

---

## Problem Statement

- Start at (1,1). Moves: `(x,y-x)`, `(x-y,y)`, `(2x,y)`, `(x,2y)`.
- Determine if `(targetX, targetY)` is reachable.

---

## Core Insight — Invariant: gcd(x, y) Is Always a Power of 2

**Claim:** A point (x, y) is reachable from (1, 1) if and only if `gcd(x, y)` is a power of 2 (including 2⁰ = 1).

### Why the GCD Stays a Power of 2

Examine each move's effect on `g = gcd(x, y)`:

- **`(x, y-x)`:** `gcd(x, y-x) = gcd(x, y) = g` (standard subtraction property of gcd). **Unchanged.**
- **`(x-y, y)`:** Symmetric — `gcd(x-y, y) = gcd(x, y) = g`. **Unchanged.**
- **`(2x, y)`:**
  - If `y` is odd: `gcd(2x, y) = gcd(x, y) = g` (2 and y are coprime). **Unchanged.**
  - If `y` is even: `gcd(2x, y)` could be `2g` (doubling x can introduce a new factor of 2 shared with y). **At most doubles.**
- **`(x, 2y)`:** Symmetric to above. **At most doubles.**

So `g` either stays the same or **doubles** at each step. Since the starting gcd is `gcd(1,1) = 1 = 2⁰`, the gcd is **always a power of 2** along any forward path.

**Necessity:** If `gcd(targetX, targetY)` is NOT a power of 2 (has an odd prime factor), it's unreachable.

**Sufficiency:** If `gcd(targetX, targetY) = 2^k`, we can construct a reverse path: work backwards from (x,y) toward (1,1) using:
- If `x == y`: only reachable if `x == y == 1` (otherwise stuck, but this case implies gcd=x=y, a power of 2 > 1 means we'd halve repeatedly first).
- If both even: halve both is NOT a valid reverse move directly, but we use reverse-subtraction (Euclidean-style) which works because the gcd being a power of 2 means the Euclidean algorithm's steps correspond to valid reverse-moves (reverse of `(2x,y)`→`(x,y)` requires x even, which the Euclidean reduction naturally provides when gcd is even).

The standard accepted proof shows this construction always terminates at (1,1) when gcd is a power of 2, by induction on `x+y` using a Euclidean-like reduction (subtract the smaller from the larger repeatedly = reverse moves 1&2, then halve when both even = reverse moves 3&4).

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool IsReachable(int targetX, int targetY) {
        int g = Gcd(targetX, targetY);
        return (g & (g - 1)) == 0; // g is a power of 2 (including 1)
    }

    private int Gcd(int a, int b) => b == 0 ? a : Gcd(b, a % b);
}
```

## Example Walkthrough

**Example 1:** `targetX = 6, targetY = 9`

gcd(6,9) = 3. `3 & 2 = 2 ≠ 0` → not a power of 2 → **false** ✓

**Example 2:** `targetX = 4, targetY = 7`

gcd(4,7) = 1. `1 & 0 = 0` → power of 2 (2⁰=1) → **true** ✓

Path given: (1,1)→(1,2)→(1,4)→(1,8)→(1,7)→(2,7)→(4,7). Check gcd throughout: gcd(1,1)=1, gcd(1,2)=1, gcd(1,4)=1, gcd(1,8)=1, gcd(1,7)=1, gcd(2,7)=1, gcd(4,7)=1. All stay 1=2⁰. ✓

## Pitfalls & Edge Cases

- **`g & (g-1) == 0`** correctly identifies powers of 2 INCLUDING 1 (since `1 & 0 = 0`), and correctly rejects 0 (but gcd is never 0 for positive inputs). ✓

- **targetX = targetY = 1:** gcd = 1, power of 2 → true (already at start). ✓

- **Large values up to 10⁹:** GCD computation is O(log) — no overflow issues with int. ✓

---

## Key Takeaway

This is a classic **invariant-based reachability** problem: instead of BFS/DFS on an astronomically large state space, find a quantity (here, `gcd(x,y)`) that's preserved or constrained in a simple way by every move, then check that constraint directly.

**Pattern recognition:** "Reachability with multiply/subtract operations" → look for a GCD or modular invariant. Doubling operations often correspond to "powers of 2" constraints on the gcd; subtraction operations preserve gcd (Euclidean algorithm connection).
