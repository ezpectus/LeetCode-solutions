# 2488. Count Subarrays With Median K

## O(n) — Balance Counting with HashMap

---

## Problem Statement

- Given array `nums` of distinct integers from 1 to n and positive integer `k`.
- Count subarrays where **median equals k**.
- **Median definition**: 
  - Odd length: middle element after sorting
  - Even length: left middle element (smaller of two middle elements)
- **Constraints**: 1 ≤ n ≤ 10⁵, distinct integers

---

## Core Idea — Transform to Balance Problem

**Key observation**: For subarray to have median k, k must be present AND positioned correctly after sorting.

**Transform elements**:
- Elements > k → +1 (larger than median)
- Element = k → 0 (the median itself)
- Elements < k → -1 (smaller than median)

**After transformation, median k exists if**:
- **Odd length subarray**: balance = 0 (equal count of larger/smaller)
- **Even length subarray**: balance = 1 (one more larger than smaller, k is left-middle)

**Example**: `nums = [3,2,1,4,5], k = 4`
- Transform: [−1, −1, −1, 0, +1]
- Subarray [4]: balance = 0, length = 1 (odd) ✓
- Subarray [4,5]: balance = 1, length = 2 (even) ✓
- Subarray [1,4,5]: balance = 0, length = 3 (odd) ✓

---

## Algorithm Breakdown

### Phase 1: Find Position of k

```
Find index pos where nums[pos] == k
```

### Phase 2: Count Valid Subarrays

**Strategy**: Iterate through all subarrays containing k (starting or ending at pos).

**Two passes**:
1. **Left side**: compute balance for prefixes ending at pos-1
2. **Right side**: compute balance for suffixes starting at pos+1
3. **Combine**: count pairs (left, right) where combined balance = 0 or 1

**Balance calculation**:
- Start from k (balance = 0)
- Extend left/right, updating balance (+1 for >k, -1 for <k)
- Store balance frequencies in hashmap

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CountSubarrays(int[] nums, int k) {
        int n = nums.Length;
        
        // Find position of k
        int pos = -1;
        for (int i = 0; i < n; i++) {
            if (nums[i] == k) {
                pos = i;
                break;
            }
        }
        
        // Count balances on left side
        var leftBalances = new Dictionary<int, int>();
        int balance = 0;
        
        // Go from k leftward
        for (int i = pos - 1; i >= 0; i--) {
            balance += nums[i] > k ? 1 : -1;
            leftBalances[balance] = leftBalances.GetValueOrDefault(balance, 0) + 1;
        }
        
        // Count valid subarrays
        int count = 0;
        balance = 0;
        
        // Count subarrays that include k
        // 1. Just k itself (odd length, balance = 0)
        count++;
        
        // 2. k + left side only
        // Odd length: need balance = 0
        count += leftBalances.GetValueOrDefault(0, 0);
        // Even length: need balance = 1 (more larger elements)
        count += leftBalances.GetValueOrDefault(1, 0);
        
        // 3. k + right side (and possibly left)
        for (int i = pos + 1; i < n; i++) {
            balance += nums[i] > k ? 1 : -1;
            
            // Just k + right (no left)
            // Odd length: need balance = 0
            if (balance == 0) count++;
            // Even length: need balance = 1
            if (balance == 1) count++;
            
            // k + left + right
            // Odd length total: need left + right = 0
            count += leftBalances.GetValueOrDefault(-balance, 0);
            // Even length total: need left + right = 1
            count += leftBalances.GetValueOrDefault(1 - balance, 0);
        }
        
        return count;
    }
}
```

## Why This Works — Example Walkthrough

### Example 1: `nums = [3,2,1,4,5], k = 4`

**Find k**: pos = 3

**Transform**: [−1, −1, −1, 0, +1]

**Left balances** (from pos-1 down to 0):
- i=2 (nums[2]=1<4): balance = -1 → leftBalances[-1] = 1
- i=1 (nums[1]=2<4): balance = -2 → leftBalances[-2] = 1
- i=0 (nums[0]=3<4): balance = -3 → leftBalances[-3] = 1
- Include k itself: leftBalances[0] = 1

**leftBalances**: {0:1, -1:1, -2:1, -3:1}

**Count from left only** (subarrays ending at pos):
- Odd (balance=0): leftBalances[0] = 1 → subarray [4]
- Even (balance=-1): leftBalances[-1] = 1 → (none, this is for left+right=-1 which gives total 0)

Wait, let me recalculate...

**Actually**, for even length we need **total balance = 1** (k is left-middle).

**Corrected logic**:
- Odd length: left + right = 0
- Even length: left + right = 1

**Count from left only**:
- Odd: leftBalances[0] = 1 → [4] ✓
- Even: leftBalances[-1] = ? (need left=-1 to get total 0... but wait)

Let me trace more carefully:

**Right side** (from pos+1):
- i=4 (nums[4]=5>4): balance = +1

**Combine**:
- Right balance = +1
- For odd length (total=0): need left = -1 → leftBalances[-1] = 1 → subarray [1,4,5] ✓
- For even length (total=1): need left = 0 → leftBalances[0] = 1 → subarray [4,5] ✓

**Total**: 1 (left only [4]) + 1 (odd [1,4,5]) + 1 (even [4,5]) = 3 ✓

---

## Edge Cases & Pitfalls

- **k at start**: No left side, only process right extensions
- **k at end**: No right side, only count left balances
- **Single element**: [k] always valid (balance=0, odd length)
- **Balance overflow**: Not possible (balance range: [-n, n])
- **HashMap access**: Use GetValueOrDefault to handle missing keys

---

## Key Takeaway

"Count subarrays with median k" → **Transform to balance problem**. **Algorithm**:
1. **Find k position**
2. **Transform**: >k → +1, =k → 0, <k → -1
3. **Count left balances**: prefixes ending before k
4. **Extend right**: for each right balance, find matching left balances
5. **Valid pairs**: left + right = 0 (odd) or left + right = 1 (even)

O(n) time, O(n) space. **Critical insight**: Median constraint becomes balance constraint after transformation. Use hashmap to count matching balance pairs.
