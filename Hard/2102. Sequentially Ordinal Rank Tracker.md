# 2102. Sequentially Ordinal Rank Tracker
## O(log n) per operation — Two Sorted Sets (Partition at Query Index)

---

## Problem Statement

- Track scenic locations. `get()` returns the i-th best location where i = call count of `get()`.
- Ordering: higher score → better; same score → lexicographically smaller name → better.

---

## Core Idea — Maintain Two Sorted Halves

Keep all locations split across two sorted sets:
- **`left`**: the top `i` best locations (where `i` = number of `get()` calls so far, grows over time).
- **`right`**: everything else (rank `i+1` onward).

The answer to each `get()` is always the **worst element of `left`** (i.e., the i-th best overall).

**Sort key**: `(-score, name)` — ascending sort gives best first (highest score, then lex smallest name).

**`add(name, score)`**:
- If new element is better than the worst in `left` (or `left` is not yet full): insert into `left`, then move `left`'s worst to `right` (to keep `left` size = i).
- Otherwise: insert directly into `right`.

**`get()`**:
- Increment `i`. Now `left` needs one more element.
- Move `right`'s best element into `left`.
- Return `left`'s worst (the i-th best).

Each `add` and `get` involves at most one element moving between sets → O(log n) per operation.

---

## Full Optimal Implementation (C#)

```csharp
using System.Collections.Generic;

public class SORTracker {
    // SortedSet with custom comparer: best first = higher score, then lex smaller name
    SortedSet<(int neg_score, string name)> left, right;
    int i = 0;

    public SORTracker() {
        var cmp = Comparer<(int, string)>.Create((a, b) => {
            if (a.Item1 != b.Item1) return a.Item1.CompareTo(b.Item1);
            return a.Item2.CompareTo(b.Item2);
        });
        left = new SortedSet<(int, string)>(cmp);
        right = new SortedSet<(int, string)>(cmp);
    }

    public void Add(string name, int score) {
        var key = (-score, name);
        if (left.Count > 0 && key.CompareTo(left.Max) < 0) {
            // Better than worst in left → insert to left, push worst to right
            left.Add(key);
            if (left.Count > i) {
                right.Add(left.Max);
                left.Remove(left.Max);
            }
        } else {
            right.Add(key);
        }
    }

    public string Get() {
        i++;
        // Move best from right to left
        right.Add(left.Count > 0 ? left.Max : right.Min); // placeholder
        // Actually: move right.Min to left
        var best = right.Min;
        right.Remove(best);
        left.Add(best);
        return left.Max.name;
    }
}
```

```csharp
public class SORTracker {
    SortedSet<(int negScore, string name)> left, right;
    int i = 0;

    static Comparer<(int, string)> Cmp = Comparer<(int, string)>.Create(
        (a, b) => a.Item1 != b.Item1 ? a.Item1.CompareTo(b.Item1) : a.Item2.CompareTo(b.Item2));

    public SORTracker() {
        left  = new SortedSet<(int, string)>(Cmp);
        right = new SortedSet<(int, string)>(Cmp);
    }

    public void Add(string name, int score) {
        var key = (-score, name);
        right.Add(key); // always insert into right first
        // If new element is better than worst of left → swap to maintain invariant
        if (i > 0 && right.Min.CompareTo(left.Max) < 0) {
            left.Add(right.Min);  right.Remove(right.Min);
            right.Add(left.Max);  left.Remove(left.Max);
        }
    }

    public string Get() {
        i++;
        // Pull best from right into left (left now has exactly i elements)
        left.Add(right.Min); right.Remove(right.Min);
        return left.Max.name; // worst of left = i-th best
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Add** | **O(log n)** | One insert + at most one move between sets |
| **Get** | **O(log n)** | One move + one max lookup |
| **Space** | **O(n)** | All locations stored |

---

## Sort Key Explained

Locations are ordered: higher score is better, ties broken by lex smaller name.

Using key `(-score, name)` with ascending sort:
- `-score` ascending = higher score first ✓
- `name` ascending = lex smaller name first ✓

So `left.Min` = best location, `left.Max` = worst in left = i-th best overall.

---

## Example Walkthrough

After adds: `[("branford",3),("bradford",2),("alps",2)]`

State after each operation:

| Event | left | right | i | answer |
|-------|------|-------|---|--------|
| add branford | {} | {(-3,branford)} | 0 | |
| add bradford | {} | {(-3,branford),(-2,bradford)} | 0 | |
| get() | {(-3,branford)} | {(-2,bradford)} | 1 | **branford** |
| add alps | {(-3,branford)} | {(-2,alps),(-2,bradford)} | 1 | |
| get() | {(-3,branford),(-2,alps)} | {(-2,bradford)} | 2 | **alps** |
| add orland | {(-3,branford),(-2,alps)} | {(-2,bradford),(-2,orland)} | 2 | |
| get() | {(-3,branford),(-2,alps),(-2,bradford)} | {(-2,orland)} | 3 | **bradford** |

At each step, `left.Max` = i-th best. ✓

---

## Pitfalls & Edge Cases

- **`SortedSet` in C# requires unique keys**: since names are unique, `(negScore, name)` pairs are always unique ✓
- **`left.Max` = worst in left = i-th best**: because left is sorted best-first, Max = last = worst = exactly what we want
- **Balance after every operation**: simpler and less error-prone than routing logic in `Add`
- **`i` starts at 0**: before any `get()`, left should be empty (queryCount=0); gets fill it one by one

---

## Key Takeaway

"Online i-th best element queries" → **two sorted sets with moving partition**:

```
left = top-i elements    (left.Max = i-th best = answer)
right = rest

add: insert, then rebalance so |left| == i
get: i++, rebalance (pull one from right to left), return left.Max
```

Classic "two-heap" / "partition pointer" pattern. O(log n) per operation regardless of total n.
