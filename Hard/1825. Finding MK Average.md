# 825. Finding MK Average
## O(log m) per operation — Three Sorted Sets + Running Sum

---

## Problem Statement

- Maintain a stream, track last `m` elements.
- `calculateMKAverage()`: take last `m` elements, remove `k` smallest and `k` largest, return floor average of remaining.
- If fewer than `m` elements in stream → return -1.

---

## Core Idea — Three Sorted Sections

Divide the last `m` elements into three sorted groups:
- **`lo`**: the `k` smallest elements
- **`mid`**: the middle `m-2k` elements (these contribute to the average)
- **`hi`**: the `k` largest elements

Maintain `mid_sum` = sum of `mid` elements. When an element is added or removed:
1. Insert/delete from the appropriate group.
2. **Rebalance** to restore invariant: `|lo| == k`, `|hi| == k`, `|mid| == m-2k`.

**Rebalancing** moves elements between groups:
- `lo` too large → move its maximum to `mid`
- `lo` too small → move `mid`'s minimum to `lo`
- Same logic for `hi`

**Adding to stream**: once `m` elements accumulated, initialize the three groups. For each subsequent element: remove the outgoing element, add the incoming element, rebalance.

## Full Optimal Implementation (C#)

```csharp
using System.Collections.Generic;

public class MKAverage {
    int m, k;
    Queue<int> stream = new();
    SortedList<int,int> lo = new(), mid = new(), hi = new();
    // SortedList doesn't support duplicates → use counts
    // Use a frequency-based sorted multiset via SortedDictionary
    
    // Simpler: implement multiset with SortedDictionary<val, count>
    SortedDictionary<int,int> loS = new(), midS = new(), hiS = new();
    int loCount, midCount, hiCount;
    long midSum;

    public MKAverage(int m, int k) { this.m = m; this.k = k; }

    void Add(SortedDictionary<int,int> s, ref int cnt, int v) {
        s[v] = s.GetValueOrDefault(v) + 1; cnt++;
    }
    void Remove(SortedDictionary<int,int> s, ref int cnt, int v) {
        s[v]--; if (s[v] == 0) s.Remove(v); cnt--;
    }
    int Min(SortedDictionary<int,int> s) => s.Keys.Min();
    int Max(SortedDictionary<int,int> s) => s.Keys.Max();

    void Rebalance() {
        while (loCount > k) {
            int v = Max(loS); Remove(loS, ref loCount, v);
            Add(midS, ref midCount, v); midSum += v;
        }
        while (loCount < k && midCount > 0) {
            int v = Min(midS); Remove(midS, ref midCount, v); midSum -= v;
            Add(loS, ref loCount, v);
        }
        while (hiCount > k) {
            int v = Min(hiS); Remove(hiS, ref hiCount, v);
            Add(midS, ref midCount, v); midSum += v;
        }
        while (hiCount < k && midCount > 0) {
            int v = Max(midS); Remove(midS, ref midCount, v); midSum -= v;
            Add(hiS, ref hiCount, v);
        }
    }

    void Insert(int num) {
        if (loCount < k || (loCount > 0 && num <= Max(loS))) {
            Add(loS, ref loCount, num);
        } else if (hiCount < k || (hiCount > 0 && num >= Min(hiS))) {
            Add(hiS, ref hiCount, num);
        } else {
            Add(midS, ref midCount, num); midSum += num;
        }
        Rebalance();
    }

    void Delete(int num) {
        if (loCount > 0 && loS.ContainsKey(num) && (midCount == 0 || num <= Max(loS))) {
            Remove(loS, ref loCount, num);
        } else if (hiCount > 0 && hiS.ContainsKey(num) && (midCount == 0 || num >= Min(hiS))) {
            Remove(hiS, ref hiCount, num);
        } else {
            Remove(midS, ref midCount, num); midSum -= num;
        }
        Rebalance();
    }

    public void AddElement(int num) {
        stream.Enqueue(num);
        if (stream.Count == m) {
            foreach (int x in stream) Insert(x);
        } else if (stream.Count > m) {
            int old = stream.Dequeue();
            Delete(old);
            Insert(num);
        }
    }

    public int CalculateMKAverage() {
        if (stream.Count < m) return -1;
        return (int)(midSum / (m - 2*k));
    }
}
```

**Note**: The insertion/deletion routing logic (`Insert`/`Delete`) is delicate. A cleaner approach always adds to `mid` and rebalances, or uses a simpler routing based on boundary values. The rebalance step handles all cases correctly.

---

## Cleaner Insert/Delete Approach

```csharp
void Insert(int num) {
    // Always add to mid, then rebalance
    Add(midS, ref midCount, num); midSum += num;
    Rebalance(); // will move to lo/hi if needed
}
// But this requires rebalance to handle lo/hi being underfull
```

This is cleaner but requires careful rebalance ordering (fill lo first, then hi, then let excess flow to mid).

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **AddElement** | **O(log m)** | SortedDictionary insert/remove + O(1) rebalance steps |
| **CalculateMKAverage** | **O(1)** | `midSum / (m-2k)` |
| **Space** | **O(m)** | At most m elements across three groups |

## Example Walkthrough

**`m=3, k=1`**. After adding [3,1,10]:

Initialize: lo={1}, mid={3}, hi={10}. midSum=3.
`calculateMKAverage()` = 3 / (3-2) = **3** ✓

After adding 5 (stream=[1,10,5]):
- Remove 3 (old): from mid → midSum=0. Rebalance: lo short → pull 1 from mid? mid empty → lo empty. lo={1} still? Wait: after removing 3 from mid={3}: mid={}. Rebalance: lo={1}, hi={10}, mid={} → loCount=1=k✓, hiCount=1=k✓. OK.
- Add 5: goes to mid (between lo.max=1 and hi.min=10). mid={5}, midSum=5.

`calculateMKAverage()` = 5 / 1 = **5** ✓

---

## Pitfalls & Edge Cases

- **Duplicates**: use `SortedDictionary<int,int>` (value → count) as a multiset, not `SortedList` (which requires unique keys)
- **Routing on insert**: always safe to insert to mid and rebalance; routing directly saves one rebalance step but adds complexity
- **Routing on delete**: must identify which group the element belongs to — compare with boundary values of lo (max) and hi (min)
- **`long` for midSum**: up to 10^5 elements × 10^5 value = 10^10, overflows `int`

---

## Key Takeaway

"Sliding window MK-average" → **three sorted multisets (lo, mid, hi) + running sum of mid**:

```
lo:  k smallest  → never contributes to average
mid: middle m-2k → sum / (m-2k) = answer
hi:  k largest   → never contributes to average

rebalance after each insert/delete:
  lo overflow → move max(lo) to mid
  lo underflow → move min(mid) to lo
  hi overflow → move min(hi) to mid
  hi underflow → move max(mid) to hi
```
