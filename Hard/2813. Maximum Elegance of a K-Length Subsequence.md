## 2813. Maximum Elegance of a K-Length Subsequence

### Problem Statement
You are given a 0-indexed 2D integer array `items` of length `n` and an integer `k`. Each item is defined as `items[i] = [profit_i, category_i]`.

The **elegance** of a subsequence of items is calculated as:
`elegance = total_profit + (distinct_categories * distinct_categories)`

Return the maximum elegance possible among all subsequences of size exactly `k`.

---

### Core Insight — Greedy Strategy with Category Replacement
Because the score depends heavily on both individual profits and the number of distinct categories squared, we can approach this with a greedy replacement strategy:

1. **Start with the highest profits:** Sort all items in descending order of their profit. Select the first `k` items. This gives us the maximum possible baseline `total_profit` for a subsequence of size `k`.
2. **Identify duplicates:** Among these initial `k` items, some categories might appear more than once. The items causing a duplicate category are candidates to be removed because removing them does not decrease our `distinct_categories` count. We store the profits of these duplicate items in a stack (ordered from smallest profit to largest).
3. **Try expanding distinct categories:** Iterate through the remaining items (from index `k` to `n - 1`). If we encounter an item belonging to a category we haven't seen yet, we can potentially increase our `distinct_categories` count by 1.
4. **Perform the swap:** To make room for this new distinct category item, we pop the item with the smallest profit from our duplicate stack and replace it. We recalculate the elegance score at each valid step and track the maximum value found.

Since the items are sorted by profit, any swap will decrease or keep `total_profit` equal, but it increases `distinct_categories`. We stop early if we run out of duplicate elements to replace.

---

### Algorithm
1. Sort `items` in descending order based on `profit`.
2. Maintain a set `seenCategories` to track categories included in our current selection.
3. Maintain a stack `duplicateProfits` to store profits of selected items whose categories have already been seen.
4. Fill the initial subsequence with the first `k` items, updating `totalProfit`, `seenCategories`, and `duplicateProfits`.
5. Compute the initial elegance.
6. Iterate through the remaining items from index `k` to `n - 1`:
   * If the item's category is not in `seenCategories` and we have duplicates available in our stack:
     * Pop the smallest duplicate profit from `duplicateProfits`.
     * Add the current item's profit to `totalProfit` and subtract the popped duplicate profit.
     * Add the current category to `seenCategories`.
     * Update the maximum elegance.
7. Return the maximum elegance found.

---

### Full Optimal Implementation (C#)
```csharp
using System;
using System.Collections.Generic;

public class Solution {
    public long FindMaximumElegance(int[][] items, int k) {
        // Sort items by profit in descending order
        Array.Sort(items, (a, b) => b[0].CompareTo(a[0]));
        
        HashSet<int> seenCategories = new HashSet<int>();
        Stack<int> duplicateProfits = new Stack<int>();
        
        long totalProfit = 0;
        
        // Step 1: Select the top 'k' items with highest profits
        for (int i = 0; i < k; i++) {
            int profit = items[i][0];
            int category = items[i][1];
            
            totalProfit += profit;
            if (seenCategories.Contains(category)) {
                // Duplicate category item; potential candidate for replacement
                duplicateProfits.Push(profit);
            } else {
                seenCategories.Add(category);
            }
        }
        
        long distinctCount = seenCategories.Count;
        long maxElegance = totalProfit + distinctCount * distinctCount;
        
        // Step 2: Try replacing duplicate items with unseen categories
        for (int i = k; i < items.Length; i++) {
            int profit = items[i][0];
            int category = items[i][1];
            
            // If we run out of duplicate items to replace, no further improvements can be made
            if (duplicateProfits.Count == 0) {
                break;
            }
            
            // We only consider the item if it introduces a brand new category
            if (!seenCategories.Contains(category)) {
                seenCategories.Add(category);
                
                // Replace the duplicate item that contributed the least profit
                long lowestDuplicateProfit = duplicateProfits.Pop();
                totalProfit = totalProfit - lowestDuplicateProfit + profit;
                
                distinctCount++;
                long currentElegance = totalProfit + distinctCount * distinctCount;
                maxElegance = Math.Max(maxElegance, currentElegance);
            }
        }
        
        return maxElegance;
    }
}
```
### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N log N) | Sorting the `items` array dominates the time complexity. The subsequent linear scans and stack operations take O(N) time. |
| **Space** | O(N) | The hash set and stack take up to O(N) auxiliary space to store categories and duplicate item profits. |

### Pitfalls & Edge Cases
* **No Duplicates to Replace:** If all initial `k` elements belong to completely distinct categories, `duplicateProfits.Count` will be 0. The second loop will immediately break via the safety check, returning the initial baseline calculation correctly.
* **Large Integer Values:** Profits can be up to 10^9. Summing them together up to `k = 10^5` times can easily overflow standard 32-bit signed integers. Ensure `totalProfit` and `maxElegance` are declared as `long`.

### Key Takeaway
When optimization metrics conflict (e.g., maximizing absolute values versus maximizing element diversity), anchor your initial solution to one constraint entirely. Starting with the absolute greedy maximum allows you to incrementally trade down value for structural variance, guaranteeing optimal evaluations at every inflection boundary.
