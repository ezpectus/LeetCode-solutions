# LeetCode 3311. Construct 2D Grid Matching Graph Layout

## Method Explanation

The problem asks us to reconstruct a 2D grid of size $N$ from its undirected graph representation. Since the input is guaranteed to form a valid 2D grid, we can deduce the layout using the **degrees of the vertices** (number of neighbors).

In a standard 2D grid:
* **Corners** have a degree of 2 (or 1 if the grid is a single line $1 \times N$).
* **Boundaries (edges)** have a degree of 3 (or 2 if $1 \times N$).
* **Internal nodes** have a degree of 4.

### 1. Identifying the Structural Dimensions
First, we compute the degree of every node and find a node with the absolute minimum degree to serve as our starting corner (`grid[0][0]`).
* **Case 1 (1D Row):** If the minimum degree is 1, the grid is a linear chain of size $1 \times N$. We can reconstruct it completely using a simple traversal from one leaf to the other.
* **Case 2 (2D Grid):** If the minimum degree is 2, we track along the boundary to reconstruct the very first row. Starting from the corner, we repeatedly transition to an unvisited neighbor that belongs to the boundary (degree $\le 3$) until we hit another corner (degree 2). The number of nodes visited during this path gives us the exact width $W$ of the grid. Consequently, the height is $H = N / W$.

### 2. Layer-by-Layer Row Reconstruction
Once the first row is fully established, we can reconstruct the remaining rows deterministically. For any cell `grid[r][c]`, its vertical neighbor directly below it (`grid[r+1][c]`) must be an unvisited neighbor of `grid[r][c]`. To ensure we match the grid adjacencies correctly, we can safely use a BFS-like layer transition or cross-reference shared neighbors with `grid[r][c+1]`.

## C++ Source Code

```cpp
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<vector<int>> constructGridLayout(int n, vector<vector<int>>& edges) {
        vector<vector<int>> adj(n);
        vector<int> degree(n, 0);
        
        for (const auto& edge : edges) {
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]);
            degree[edge[0]]++;
            degree[edge[1]]++;
        }
        
        // Find a node with the minimum degree to act as our starting corner
        int start_node = 0;
        for (int i = 1; i < n; ++i) {
            if (degree[i] < degree[start_node]) {
                start_node = i;
            }
        }
        
        vector<int> visited(n, 0);
        vector<vector<int>> grid;
        
        // Case 1: The grid is a 1D line (1 x n)
        if (degree[start_node] == 1) {
            vector<int> row;
            int curr = start_node;
            while (curr != -1) {
                row.push_back(curr);
                visited[curr] = 1;
                int next_node = -1;
                for (int neighbor : adj[curr]) {
                    if (!visited[neighbor]) {
                        next_node = neighbor;
                        break;
                    }
                }
                curr = next_node;
            }
            grid.push_back(row);
            return grid;
        }
        
        // Case 2: The grid is a standard 2D layout
        // Reconstruct the first row by moving along the boundary (degree <= 3)
        vector<int> first_row;
        int curr = start_node;
        first_row.push_back(curr);
        visited[curr] = 1;
        
        // Choose the first step neighbor with the smallest degree to stay on the boundary
        int next_node = -1;
        for (int neighbor : adj[curr]) {
            if (next_node == -1 || degree[neighbor] < degree[next_node]) {
                next_node = neighbor;
            }
        }
        
        curr = next_node;
        while (true) {
            first_row.push_back(curr);
            visited[curr] = 1;
            if (degree[curr] == 2) {
                // Reached the second corner, row 1 complete
                break; 
            }
            
            int tracking_next = -1;
            for (int neighbor : adj[curr]) {
                if (!visited[neighbor] && degree[neighbor] <= 3) {
                    tracking_next = neighbor;
                    break;
                }
            }
            curr = tracking_next;
        }
        
        int width = first_row.size();
        int height = n / width;
        grid.assign(height, vector<int>(width, 0));
        grid[0] = first_row;
        
        // Reconstruct the remaining rows row by row
        for (int r = 0; r < height - 1; ++r) {
            for (int c = 0; c < width; ++c) {
                int u = grid[r][c];
                for (int neighbor : adj[u]) {
                    if (!visited[neighbor]) {
                        // If it's the first column, any unvisited neighbor works
                        if (c == 0) {
                            grid[r + 1][c] = neighbor;
                            visited[neighbor] = 1;
                            break;
                        } else {
                            // Verify alignment by checking if this node connects to the left neighbor's bottom node
                            int left_bottom = grid[r + 1][c - 1];
                            if (find(adj[neighbor].begin(), adj[neighbor].end(), left_bottom) != adj[neighbor].end()) {
                                grid[r + 1][c] = neighbor;
                                visited[neighbor] = 1;
                                break;
                            }
                        }
                    }
                }
            }
        }
        
        return grid;
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N + E)$
  * **Graph Generation & Degree Counting:** Constructing the adjacency list from the input `edges` array takes linear time proportional to the number of connections, $\mathcal{O}(E)$. Concurrently, tracking the degree of each node requires incrementing counters for both endpoints of an edge, which maps directly to $\mathcal{O}(E)$ operations.
  * **Corner Detection:** Finding the starting corner node involves scanning the `degree` array of size $N$ exactly once to identify the absolute minimum value. This requires a strict linear scan of $\mathcal{O}(N)$ time.
  * **Boundary Row Reconstruction:** Building the first row takes time proportional to the width of the grid $W$. In the worst-case scenario (a single 1D row where $W = N$), it traverses all $N$ nodes. Because each step transitions only through unvisited neighbors with a degree $\le 3$, this step runs in $\mathcal{O}(W)$ steps, which is bounded by $\mathcal{O}(N)$.
  * **Layered Matrix Mapping:** Populating the remaining rows of the matrix requires a nested traversal over the dimensions $H \times W = N$. For each cell `grid[r][c]`, we iterate over its list of neighbors in the adjacency list. Crucially, because the graph is guaranteed to form a valid 2D grid, the degree of any vertex is strictly bounded by a constant factor:
    $$\text{deg}(u) \le 4$$
    As a result, operations like `std::find` over the neighbors list or checking neighbor availability run in strict $\mathcal{O}(1)$ constant time per cell.
  * **Total Time Bound:** Summing up the sequential phases ($\mathcal{O}(E) + \mathcal{O}(N) + \mathcal{O}(W) + \mathcal{O}(N \times 4)$), the overall execution time scales linearly as $\mathcal{O}(N + E)$. Given the constraints $N \le 5 \cdot 10^4$ and $E \le 10^5$, this optimal approach avoids nested combinatorial lookups and completes well within the runtime limits.

* **Space Complexity:** $\mathcal{O}(N + E)$
  * **Adjacency Matrix Buffers:** The primary memory overhead comes from the adjacency list `adj`. It stores $N$ vector buckets, and across all buckets, exactly $2 \times E$ node entries are kept (since the graph is undirected). This consumes $\mathcal{O}(N + E)$ auxiliary space.
  * **Tracking and Sizing States:** The explicit degree-tracking vector `degree` and the boolean flag array `visited` each require exactly $N$ elements, taking $\mathcal{O}(N)$ space.
  * **Grid Recomposition Layout:** The final structured `grid` matrix contains exactly $H$ rows and $W$ columns, storing precisely $H \times W = N$ elements. 
  * **Total Memory Footprint:** Combining the structural representations and validation buffers, the auxiliary memory scales linearly and is bounded by $\mathcal{O}(N + E)$ space allocations.
