# 2846. Minimum Edge Weight Equilibrium Queries in a Tree
## O((n + m) · W) — LCA with Per-Weight Prefix Counts on Root–Node Paths

---

## Problem Statement

Undirected tree, `n` nodes, edge weights ∈ `[1, 26]`. For each query `(a, b)`: find the minimum number of edge weight changes to make **all edges on the path from `a` to `b` equal**.

---

## Core Insight #1 — Optimum = Keep the Most Frequent Weight, Change Everything Else

On any path with `L` edges, the minimum operations to equalize all weights is:
```
L - max_frequency_of_any_single_weight_on_the_path
```

We keep all edges of the most common weight intact and change all others to it. This is always optimal because we can change each edge to any value in one operation — no "cascades" or dependencies.

---

## Core Insight #2 — Path Decomposition via LCA

Any tree path `a → b` decomposes at the Lowest Common Ancestor (LCA) `l`:
```
path(a, b) = path(root → a) ⊕ path(root → b), minus path(root → l) counted twice
```

For each weight `w ∈ [1..26]`, define:
```
cnt[v][w] = number of edges with weight w on the root-to-v path
```

Then on path `(a, b)` with LCA `l`:
```
freq(w)  = cnt[a][w] + cnt[b][w] - 2 · cnt[l][w]
pathLen  = depth[a] + depth[b] - 2 · depth[l]
answer   = pathLen - max_w(freq(w))
```

Each `freq(w)` and `pathLen` computes in O(1) given `cnt[]` and `depth[]`. Finding `max_w` is O(26).

---

## Core Insight #3 — Binary Lifting for O(log n) LCA Per Query

Root the tree at node 0. Build:
- `depth[v]` = tree depth of node `v`
- `up[v][j]` = 2^j-th ancestor of `v` (binary lifting table)
- `cnt[v]` = 26-element count array, built in one DFS pass by copying parent's array and incrementing the arriving edge's weight

LCA query:
1. Bring the deeper node up to match depths (bit-by-bit via `up[]`).
2. If equal, that's the LCA.
3. Otherwise lift both simultaneously until they converge; return `up[a][0]`.

---

## State Diagram

```
Root (node 0)
  cnt[0][w] = 0 for all w
  depth[0] = 0

  ↓  edge weight ew
Child v:
  cnt[v] = copy of cnt[parent]
  cnt[v][ew]++
  depth[v] = depth[parent] + 1
  up[v][0] = parent
  up[v][j] = up[up[v][j-1]][j-1]

Query (a, b):
  lca = LCA(a, b)
  pathLen = depth[a] + depth[b] - 2·depth[lca]
  freq(w) = cnt[a][w] + cnt[b][w] - 2·cnt[lca][w]
  answer  = pathLen - max_w(freq(w))
```

## Full Implementation (C#)

```csharp
public class Solution {
    public int[] MinOperationsQueries(int n, int[][] edges, int[][] queries) {
        const int W = 26;
        const int LOG = 14; // 2^14 = 16384 > n_max = 10^4

        var adj = new List<(int to, int w)>[n];
        for (int i = 0; i < n; i++) adj[i] = new List<(int, int)>();
        foreach (var e in edges) {
            adj[e[0]].Add((e[1], e[2]));
            adj[e[1]].Add((e[0], e[2]));
        }

        int[] depth = new int[n];
        int[][] cnt = new int[n][];  // cnt[v][w] = #edges weight w on root-to-v path
        int[][] up  = new int[n][];  // binary lifting table
        for (int i = 0; i < n; i++) {
            cnt[i] = new int[W + 1];
            up[i]  = new int[LOG];
        }

        // Iterative DFS to fill depth, cnt, up
        bool[] visited = new bool[n];
        var stack = new Stack<(int v, int par, int ew)>();
        stack.Push((0, 0, 0)); // root: parent = self (sentinel), no incoming edge
        visited[0] = true;

        while (stack.Count > 0) {
            var (v, par, ew) = stack.Pop();

            if (v != par) { // non-root: copy parent's counts and add edge
                Array.Copy(cnt[par], cnt[v], W + 1);
                cnt[v][ew]++;
                depth[v] = depth[par] + 1;
            }

            up[v][0] = par;
            for (int j = 1; j < LOG; j++)
                up[v][j] = up[up[v][j - 1]][j - 1];

            foreach (var (u, w) in adj[v]) {
                if (!visited[u]) {
                    visited[u] = true;
                    stack.Push((u, v, w));
                }
            }
        }

        // LCA via binary lifting
        int Lca(int a, int b) {
            if (depth[a] < depth[b]) (a, b) = (b, a);
            int diff = depth[a] - depth[b];
            for (int j = 0; j < LOG; j++)
                if (((diff >> j) & 1) == 1) a = up[a][j];
            if (a == b) return a;
            for (int j = LOG - 1; j >= 0; j--)
                if (up[a][j] != up[b][j]) { a = up[a][j]; b = up[b][j]; }
            return up[a][0];
        }

        int[] ans = new int[queries.Length];
        for (int i = 0; i < queries.Length; i++) {
            int a = queries[i][0], b = queries[i][1];
            if (a == b) { ans[i] = 0; continue; }

            int lca    = Lca(a, b);
            int pathLen = depth[a] + depth[b] - 2 * depth[lca];

            int maxFreq = 0;
            for (int w = 1; w <= W; w++) {
                int freq = cnt[a][w] + cnt[b][w] - 2 * cnt[lca][w];
                if (freq > maxFreq) maxFreq = freq;
            }

            ans[i] = pathLen - maxFreq;
        }

        return ans;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| DFS preprocessing | O(n · W) | Copy 26-element array per node |
| Binary lifting build | O(n · LOG) | 14 ancestors per node |
| Per query | O(LOG + W) | LCA in O(14) + max freq in O(26) |
| **Total** | **O((n + m) · W)** | ≤ (10⁴ + 2×10⁴) × 26 ≈ 780K ops |
| Space | O(n · (W + LOG)) | `cnt` table + `up` table |

## Pitfalls & Edge Cases

- **Root's binary lifting sentinel:** Set `up[0][j] = 0` for all `j` (root points to itself). This lets the LCA equalization step work without bounds checking — lifting past the root just stays at the root.

- **`Array.Copy` for `cnt[]`, not reference copy.** Each node needs its own independent array. Sharing references would corrupt all ancestor counts when a child's array is modified.

- **Iterative DFS, not recursive:** n ≤ 10⁴ with a linear tree (worst case) means recursion depth up to 10⁴. C# default stack limit is ~1MB → stack overflow for deep trees. Iterative DFS avoids this entirely.

- **`a == b` edge case:** Path length is 0, answer is 0. Short-circuit before computing LCA to avoid `pathLen = 0` with a useless `max_w` search.

- **Weight 0 does not exist:** Weights are `∈ [1, 26]`, so `cnt[v][0]` is always 0. The loop from `w=1` to `w=26` is correct and complete.

- **LOG = 14:** `2^14 = 16384 > 10^4 = n_max`. Sufficient for all depths. Using 15 or 16 wastes a small constant but doesn't affect correctness.

---

## Why This Beats Naive per-Query Path Traversal

**Naive:** For each query, walk the path (O(n) in worst case), count weight frequencies (O(n)), find max. Total: O(m × n) = O(2×10⁴ × 10⁴) = 2×10⁸ — tight or too slow.

**This approach:** All path info is encoded in `cnt[]` differences; LCA takes O(log n). Each query costs O(26) after O(log n) LCA — negligible. The W=26 bounded weight set is the key enabler: it turns "count frequencies on an arbitrary path" into "26 array lookups."

---

## Key Takeaway

This is the **"tree path queries via root-to-node prefix arrays + LCA"** pattern. Whenever an additive quantity over tree paths needs to be computed (count of something, sum of weights, etc.), storing it as root-to-node prefix values and subtracting at the LCA gives O(1) per query after O(n) preprocessing.

**Pattern recognition:** "Aggregate over path (a, b) in a tree" → precompute root-to-node prefix for the aggregate → query = `prefix[a] + prefix[b] - 2·prefix[LCA(a,b)]`. Combine with binary lifting for O(log n) LCA. When the aggregate has bounded dimension W (here W=26), the entire query cost is O(log n + W).
