# 2862. Maximum Element-Sum of a Complete Subset of Indices
## O(n√n) — Square-Free Grouping: Reduce i·j ∈ □ to a Shared Invariant

---

## Problem Statement

1-indexed array `nums`. Select a "complete subset" of indices where **every pair** `i·j` is a perfect square. Return the maximum sum of `nums` values in such a subset.

---

## Core Insight #1 — i·j Is a Perfect Square Iff i and j Share the Same Square-Free Part

Every positive integer `i` can be **uniquely** written as:
```
i = squarefree(i) · k²
```
where `squarefree(i)` has no prime factor appearing more than once.

**Claim:** `i · j` is a perfect square **if and only if** `squarefree(i) == squarefree(j)`.

**Proof:**
- Write `i = a·s²`, `j = a·t²` (same squarefree part `a`).
- Then `i·j = a²·s²·t² = (a·s·t)²` ✓
- Conversely, if `i = a·s²`, `j = b·t²` with `a ≠ b` (both squarefree), then `i·j = ab·(st)²`. Since `a` and `b` are squarefree and `a ≠ b`, `ab` is not a perfect square → `i·j` is not a perfect square.

**Consequence:** Two indices belong to the same valid complete subset **exactly when** they have the same squarefree part. The maximum complete subset is the group with the highest sum.

---

## Core Insight #2 — Computing squarefree(i) in O(√i)

Remove all squared prime factors iteratively:
```
squarefree(i): for each prime p ≤ √i, while p² | i, divide by p²
```

No prime factorization needed — just trial division by squares:

```
squarefree(12) = 12 → 12/4=3 (p=2, remove 2²) → 3 (p=3: 9>3, stop) → 3
squarefree(18) = 18 → 18/9=2 (p=3? No wait: p=2: 4 doesn't divide 18; p=3: 9 divides 18) → 2
squarefree(36) = 36 → 36/4=9 → 9/9=1 → 1
squarefree(50) = 50 → 50/4=12.5? No. p=2: 4 doesn't divide 50. p=5: 25 divides 50 → 2 → 2
```

Wait: squarefree(50): 50 = 2 × 5². Remove 5² → 50/25 = 2. squarefree(50) = 2. ✓ (2 × 25 = 50, 50 × 2 = 100 = 10² ✓)

---

## Algorithm

```
1. For each index i from 1 to n:
     sf = squarefree(i)
     groupSum[sf] += nums[i]   (1-indexed access)

2. Return max(groupSum.Values)
```

That's it. One pass, O(n·√n).

---

## Full Implementation (C#)

```csharp
public class Solution {
    public long MaximumSum(IList<int> nums) {
        int n = nums.Count;
        var groupSum = new Dictionary<int, long>();

        for (int i = 1; i <= n; i++) {
            int sf = SquareFree(i);
            if (!groupSum.ContainsKey(sf)) groupSum[sf] = 0;
            groupSum[sf] += nums[i - 1]; // nums is 0-indexed, i is 1-indexed
        }

        long ans = 0;
        foreach (long s in groupSum.Values)
            ans = Math.Max(ans, s);

        return ans;
    }

    private int SquareFree(int x) {
        for (int p = 2; p * p <= x; p++)
            while (x % (p * p) == 0)
                x /= p * p;
        return x;
    }
}
```

## Pitfalls & Edge Cases

- **1-indexing:** The problem says indices are 1-based. `squarefree(i)` uses `i` (1-indexed), but `nums[i-1]` accesses the 0-indexed array.

- **squarefree(1) = 1:** The loop `p=2; p*p<=1` never executes. Returns 1 immediately. ✓

- **Overflow:** `nums[i] ≤ 10⁹` and at most `n ≤ 10⁴` elements in a group → max sum ≤ 10¹³. Fits in `long` (max ~9.2×10¹⁸). The accumulator and return type must be `long`.

- **p*p overflow in SquareFree:** p is at most √(10⁴) = 100, so `p*p ≤ 10000` — safe for `int`.

- **Single-element subsets:** Any single index is a valid complete subset (no pair to check). Always contributes positively, so the answer ≥ max(nums).

- **All indices in one group:** e.g., n=4, all squarefree(i)=1 (indices 1,4). The entire array is the optimal subset. Handled naturally.

---

## Why Not Check All Pairs?

The naive approach: try all `2^n` subsets, verify all pairs, sum. Even for n=20, 2^20 = 10^6 subsets with O(n²) pair-checks each → too slow. With the grouping insight, it's O(n) after computing squarefree values.

**The square-free equivalence class is the key invariant** — it reduces a "check all pairs" problem to a "assign each element to a bucket" problem, without needing any quadratic verification at all.

---

## Key Takeaway

This is a **number-theory grouping** problem disguised as a set/subset question. The multiplicative condition `i·j ∈ □` defines an equivalence relation via the squarefree kernel — and once you see that, the algorithm is trivial. The hard part is recognizing that "every pair" translates to a single per-element invariant.

**Pattern recognition:** "i·j is a perfect square for all pairs in a set" → compute squarefree(i) for each i → group by squarefree part → take max group sum. Appears in problems about "multiplicative perfect squares", "rational ratios of square roots", or "common squarefree kernel".
