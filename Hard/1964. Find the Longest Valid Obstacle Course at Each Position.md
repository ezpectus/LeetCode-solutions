# 1964. Find the Longest Valid Obstacle Course at Each Position
## O(n log n) — Non-Decreasing LIS via Binary Search (bisect_right)

---

## Problem Statement

- For each index `i`, find the length of the longest non-decreasing subsequence ending at `obstacles[i]`.
- Return this length for every position.

---

## Core Idea — LIS with Non-Decreasing Constraint

This is the **Longest Non-Decreasing Subsequence** (LNDS) problem — a variant of LIS where equal elements are allowed (`≤` instead of `<`).

**Standard LIS** uses `bisect_left` to maintain strictly increasing tails. For **non-decreasing**, use `bisect_right` instead — this allows placing equal elements after existing ones.

**`tails` array**: `tails[i]` = smallest possible ending value of a non-decreasing subsequence of length `i+1`.

For each element `x`:
- Find position `pos = bisect_right(tails, x)` = first index where `tails[pos] > x`.
- Length of LNDS ending at `x` = `pos + 1`.
- Update `tails[pos] = x` (replace or extend).

**Why `bisect_right`?** It finds the insertion point AFTER any existing equal values, allowing `x` to extend a sequence that ends with `x` (non-decreasing).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] LongestObstacleCourseAtEachPosition(int[] obstacles) {
        int n = obstacles.Length;
        int[] res = new int[n];
        var tails = new List<int>();

        for (int i = 0; i < n; i++) {
            int x = obstacles[i];
            // Find first position where tails[pos] > x
            int pos = UpperBound(tails, x);
            res[i] = pos + 1;
            if (pos == tails.Count) tails.Add(x);
            else tails[pos] = x;
        }

        return res;
    }

    int UpperBound(List<int> list, int target) {
        int lo = 0, hi = list.Count;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (list[mid] <= target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
```

## bisect_right vs bisect_left

| Operation | Use case | Behavior |
|-----------|----------|----------|
| `bisect_left` | Strictly increasing LIS | Find first pos where `tails[pos] >= x` → equal elements can't extend |
| `bisect_right` | Non-decreasing LNDS | Find first pos where `tails[pos] > x` → equal elements CAN extend |

**Example**: tails=[1,2], x=2. `bisect_left`→ pos=1 (replaces 2, length 2). `bisect_right`→ pos=2 (extends to length 3 with [1,2,2]).

---

## Example Walkthrough

**`obstacles=[1,2,3,2]`**

| i | x | tails before | pos=bisect_right | res[i] | tails after |
|---|---|-------------|-----------------|--------|-------------|
| 0 | 1 | [] | 0 | 1 | [1] |
| 1 | 2 | [1] | 1 | 2 | [1,2] |
| 2 | 3 | [1,2] | 2 | 3 | [1,2,3] |
| 3 | 2 | [1,2,3] | 2 | 3 | [1,2,2] |

Output: [1,2,3,3] ✓ (last 2 extends [1,2,2])

---

## Pitfalls & Edge Cases

- **Equal elements**: `obstacles=[2,2]`. `bisect_right([2],2)=1` → second 2 extends to length 2. ✓
- **Strictly decreasing**: each element finds pos=0 → all lengths = 1.
- **`tails` doesn't represent an actual subsequence**: it's a bookkeeping structure for optimal end-values. The actual subsequence is reconstructed differently (not needed here).

---

## Key Takeaway

"Longest non-decreasing subsequence at each position" → **LIS algorithm with `bisect_right` instead of `bisect_left`**:

```
for each x:
    pos = bisect_right(tails, x)   // upper bound: allows equal
    res[i] = pos + 1
    tails[pos] = x                 // extend or replace
```

The single change from `bisect_left` to `bisect_right` converts strictly-increasing LIS to non-decreasing LNDS.
