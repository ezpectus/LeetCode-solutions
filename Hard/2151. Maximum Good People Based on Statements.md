# 2151. Maximum Good People Based on Statements
## O(2ⁿ × n²) — Bitmask Enumeration of "Good" Subsets

---

## Problem Statement

- `n` people, each either "good" (always truthful) or "bad" (may lie).
- `statements[i][j]`: person `i` says person `j` is good (1), bad (0), or says nothing (2).
- Find the maximum number of good people in any **consistent** assignment.

---

## Core Idea — Enumerate All Possible "Good" Sets

With `n ≤ 15`, there are at most `2^15 = 32768` subsets. For each subset (bitmask representing "which people are good"), check if it's **consistent**:

A subset is consistent if every person marked as **good** makes only true statements:
- If `i` is good and `statements[i][j] == 1`, then `j` must also be good.
- If `i` is good and `statements[i][j] == 0`, then `j` must be bad.
- If `i` is good and `statements[i][j] == 2`, no constraint.

**Bad people** can say anything — no constraints from their statements.

Return the size of the largest consistent subset.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumGood(int[][] statements) {
        int n = statements.Length;
        int res = 0;

        for (int mask = 0; mask < (1 << n); mask++) {
            bool valid = true;
            for (int i = 0; i < n && valid; i++) {
                if ((mask >> i & 1) == 0) continue; // i is bad → skip
                for (int j = 0; j < n && valid; j++) {
                    if (statements[i][j] == 2) continue;
                    bool jIsGood = (mask >> j & 1) == 1;
                    if (statements[i][j] == 1 && !jIsGood) valid = false;
                    if (statements[i][j] == 0 && jIsGood)  valid = false;
                }
            }
            if (valid)
                res = Math.Max(res, System.Numerics.BitOperations.PopCount((uint)mask));
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(2ⁿ × n²)** | 32768 subsets × 15² = ~7.4M checks |
| **Space** | **O(1)** | No extra space |

With `n ≤ 15`: trivially fast.

---

## Consistency Check Logic

For a given `mask` (subset of good people), person `i` is good iff bit `i` is set:

```
for each good person i:
    for each statement statements[i][j]:
        if statements[i][j] == 1: j must be in mask
        if statements[i][j] == 0: j must NOT be in mask
        if statements[i][j] == 2: no constraint
```

Bad people have no constraints — they may say anything whether true or false.

---

## Example Walkthrough

**`statements=[[2,1,2],[1,2,2],[2,0,2]]`**, n=3

Check mask=`011` (persons 0 and 1 are good):
- Person 0 (good): `statements[0][1]=1` → person 1 must be good → ✓ (bit 1 set)
- Person 1 (good): `statements[1][0]=1` → person 0 must be good → ✓ (bit 0 set)
- Consistent! PopCount = 2.

Check mask=`111` (all good):
- Person 2 (good): `statements[2][1]=0` → person 1 must be bad → ✗ (bit 1 is set)
- Inconsistent.

Max consistent = **2** ✓

---

## Pitfalls & Edge Cases

- **`mask=0`** (no good people): trivially consistent (no good person makes any constraint) → always valid as a baseline
- **Self-statements**: `statements[i][i]=2` always → no constraint on self
- **Bad people don't constrain**: a statement from a bad person is irrelevant regardless of truth value
- **`PopCount`**: use `BitOperations.PopCount` in C# for bit counting; alternatively iterate and count set bits

---

## Key Takeaway

"Maximum consistent assignment with binary good/bad labels" → **bitmask enumeration**:

- Try all `2ⁿ` subsets of "good" people
- For each subset: verify good people's statements are all satisfied
- Return size of largest consistent subset

`n ≤ 15` makes this tractable. Pattern: "find maximum valid subset" with small n → bitmask brute force with consistency check.
