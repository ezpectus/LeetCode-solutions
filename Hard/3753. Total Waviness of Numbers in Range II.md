# 3753. Total Waviness of Numbers in Range II
## O(16 · 11² · 4) — Digit DP with Lazy Contribution Tracking

---

## Problem Statement

- Given `[num1, num2]` where `num2` can reach 10¹⁵ (up to 16 digits).
- Waviness = count of interior digits that are peaks or valleys.
- Return total waviness summed over all numbers in the range.

---

## Core Idea — Digit DP with Prefix Complement (f(num2) − f(num1−1))

Compute `f(n)` = total waviness for [1, n], then answer = `f(num2) − f(num1−1)`.

**Key observation:** When placing digit `d` at position `pos`, we can determine if the *previous significant digit* `last1` is an interior peak or valley:
- `last1` is interior if there's a significant digit before it (`last2 ≠ −1`) and there's a digit after it (we're placing one now).
- peak: `last2 < last1 > d`
- valley: `last2 > last1 < d`

This "lazy evaluation" — determining the contribution of position `pos−1` when processing position `pos` — keeps the DP clean.

**DP returns (count, total_sum):**
- `count` = number of valid completions from this state
- `total_sum` = sum of all waviness contributions from this state onward

**Key transition:** when placing digit `d` and `last2 ≠ −1`:
```
total_sum += sub_sum + contrib × sub_count
```
where `contrib` (0 or 1) is the peak/valley determination for `last1`.

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    string s;
    long long dp_cnt[16][11][11][2][2];
    long long dp_sum[16][11][11][2][2];
    bool dp_vis[16][11][11][2][2];

    // last2, last1 in {-1, 0..9}; stored as last2+1, last1+1 (0-indexed)
    pair<long long, long long> dp(int pos, int last2, int last1, int tight, int started) {
        if (pos == (int)s.size()) return {1LL, 0LL};

        int l2 = last2 + 1, l1 = last1 + 1;
        if (dp_vis[pos][l2][l1][tight][started])
            return {dp_cnt[pos][l2][l1][tight][started],
                    dp_sum[pos][l2][l1][tight][started]};
        dp_vis[pos][l2][l1][tight][started] = true;

        int limit = tight ? (s[pos] - '0') : 9;
        long long total_count = 0, total_sum = 0;

        for (int d = 0; d <= limit; d++) {
            int new_tight   = (tight && d == limit) ? 1 : 0;
            int new_started = (started || d != 0)  ? 1 : 0;

            int  new_last2 = last2, new_last1 = last1;
            long contrib   = 0;

            if (!started && d == 0) {
                // Still leading zeros — no significant digit added
                new_last2 = -1; new_last1 = -1;
            } else if (!started) {
                // First significant digit — cannot be interior
                new_last2 = -1; new_last1 = d;
            } else {
                // Another significant digit — check if last1 is interior
                if (last2 != -1) {
                    bool peak   = (last2 < last1) && (last1 > d);
                    bool valley = (last2 > last1) && (last1 < d);
                    contrib = (peak || valley) ? 1 : 0;
                }
                new_last2 = last1;
                new_last1 = d;
            }

            auto [cnt, sm] = dp(pos + 1, new_last2, new_last1, new_tight, new_started);
            total_count += cnt;
            total_sum   += sm + contrib * cnt;
        }

        dp_cnt[pos][l2][l1][tight][started] = total_count;
        dp_sum[pos][l2][l1][tight][started] = total_sum;
        return {total_count, total_sum};
    }

    long long f(long long n) {
        if (n <= 0) return 0;
        s = to_string(n);
        memset(dp_vis, 0, sizeof(dp_vis));
        return dp(0, -1, -1, 1, 0).second;
    }

    long long totalWaviness(long long num1, long long num2) {
        return f(num2) - f(num1 - 1);
    }
};
```

## State Design Explained

| State | Range | Purpose |
|-------|-------|---------|
| `pos` | 0..15 | Current digit position |
| `last2` | −1..9 | Second-to-last significant digit (−1 = none) |
| `last1` | −1..9 | Last significant digit (−1 = none) |
| `tight` | 0,1 | Whether still bounded by n |
| `started` | 0,1 | Whether first non-zero digit has been placed |

`last2 = −1` means fewer than 2 significant digits seen → current significant digit can't be interior.

---

## Example Walkthrough — Why `contrib × sub_count`

When at position `pos` placing digit `d`:
- `last1` (at position `pos−1`) is determined to be a peak/valley → `contrib = 1`.
- Every one of the `sub_count` numbers that can be formed from `pos+1` onward will share this same peak/valley at `pos−1`.
- So the total added to waviness from this decision: `1 × sub_count`.

**Example 3:** num1 = num2 = 4848.

Trace through 4848 = [4,8,4,8]:

| pos | d | last2 | last1 | last1 check | contrib |
|-----|---|-------|-------|-------------|---------|
| 0   | 4 | −     | −     | first sig digit | 0 |
| 1   | 8 | −1    | 4     | last2=−1, skip | 0 |
| 2   | 4 | 4     | 8     | 4<8>4? **peak!** | 1 |
| 3   | 8 | 8     | 4     | 8>4<8? **valley!** | 1 |

Total waviness = 2 ✓

## Leading Zero Handling

A number like 31 in a 4-position DP is represented as [0, 0, 3, 1]:
- Positions 0,1: leading zeros. `started = false`, `last2 = last1 = −1`.
- Position 2: d=3. First significant digit. `last2=−1, last1=3`. `started=true`.
- Position 3: d=1. `last2=−1` → no interior check. `last2=3, last1=1`.

No contributions → waviness(31) = 0 ✓ (2-digit number has no interior digits).

---

## Pitfalls & Edge Cases

- **`num1 = 1`:** f(num1−1) = f(0). The `n ≤ 0` guard returns 0. ✓

- **1 and 2-digit numbers:** `last2` stays −1 throughout → no interior check → waviness = 0. ✓

- **Trailing zeros:** 3010 = [3,0,1,0]. All digits are significant. The 0 at position 1 and 1 at position 2 are interior. Correctly detected. ✓

- **`tight` propagation:** only stays tight when the chosen digit equals the bound digit. ✓

- **`started` trailing zeros:** once `started = true`, zeros are significant (e.g., the 0 in 301 is interior and correctly detected as a valley). ✓

---

## Key Takeaway

This is a classic **digit DP** where we compute a sum (not just a count) over a range.

The trick: **lazy contribution evaluation** — the contribution of digit at position `i` is determined when we place digit `i+1`. This avoids needing to know the "last digit" of the number in advance.

The return value `(count, total_sum)` pattern enables summing waviness over all valid completions:
```
total_sum += sub_sum + contrib × sub_count
```

**Pattern recognition:** "Sum a property over [num1, num2] for large ranges" → digit DP with f(n) − f(num1−1), returning (count, sum) pairs, evaluating local contributions lazily.
