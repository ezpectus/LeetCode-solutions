# LeetCode 3357. Minimize the Maximum Adjacent Element Difference

## Method Explanation

The problem challenges us to find a pair of integers $(x, y)$ to replace all occurrences of `-1` inside an array such that the maximum absolute difference between any two adjacent elements is minimized. 

Given the nature of the optimization function ("minimize the maximum"), **Binary Search on the Answer** is the core structural choice. We binary search for the target minimum maximum difference, $mid$, within the range $[0, 10^9]$. For each $mid$, we run a predicate function `f(mid)` to check whether a valid pair $(x, y)$ can be constructed to satisfy all constraints.

### 1. Mathematical Range Translation
Any fixed element $v$ dictates a hard constraint on any adjacent replaced element. The missing element must fall within a distance of $mid$ from $v$. This defines a valid closed mathematical interval for that specific neighbor:
$$I(v) = [\max(1, v - mid), v + mid]$$

When analyzing segments of consecutive `-1`s, the constraints depend entirely on the length of the missing block and the existence of boundary elements (neighbors):

* **Single Missing Element (`L, -1, R`):**
  A single gap has two immediate constraints. The chosen number must satisfy both sides simultaneously. Thus, the value must fall into the intersection of both intervals:
  $$I_{both} = I(L) \cap I(R) = [\max(L - mid, R - mid), \min(L + mid, R + mid)]$$
  If at any point the intersection bounds cross ($\max > \min$), it is structurally impossible for a single number to satisfy both neighbors, making this $mid$ immediately invalid.

* **Multiple Missing Elements (`L, -1, -1, ..., R`):**
  When two or more `-1`s are adjacent, we gain structural flexibility. We no longer need a single number to bridge the entire gap because we have *two* numbers $(x, y)$ at our disposal. We can place $x$ near $L$ and $y$ near $R$. This opens up two distinct topological scenarios for our pair:
  1. **The Close Scenario ($|x - y| \le mid$):** If the distance between our chosen $x$ and $y$ is within the acceptable threshold $mid$, we can place them right next to each other inside the gap. This means $x$ can safely cover $L$'s interval $I(L)$, and $y$ can cover $R$'s interval $I(R)$.
  2. **The Far Scenario ($|x - y| > mid$):** If $x$ and $y$ are placed further apart than $mid$, they *cannot* be adjacent inside the gap. To prevent a violation, another valid element would have to step between them, which isn't possible in a pure block of $-1$s unless one of the values can independently bridge the gap. Thus, a single value must be capable of satisfying both $L$ and $R$ simultaneously, falling back to the intersection $I(L) \cap I(R)$.

* **One-Sided Edge Blocks (`-1, -1, R` or `L, -1, -1`):**
  If a block of `-1`s touches the absolute boundaries of the array, it only has one real neighbor $v$. It only generates a single loose interval $I(v)$, which can be satisfied by either $x$ or $y$.

### 2. Double-Track Interval Accumulation
To evaluate whether a pair $(x, y)$ can satisfy the whole array, the predicate tracks two sets of interval buckets independently:
* `closeRanges`: Collects all required intervals assuming we are allowed to place $x$ and $y$ directly next to each other ($|x - y| \le mid$).
* `farRanges`: Collects all required intervals under the restriction that $x$ and $y$ cannot be placed adjacently, forcing single-value intersection coverage across multi-gaps.

### 3. The Interval Cover Algorithm (`cover`)
To check if a collection of intervals can be completely covered by just two points $x$ and $y$:
1. **Sort by Endpoints:** We sort the incoming intervals ascending by their upper bounds (`second`).
2. **Greedy Placement of $x$:** To maximize our coverage area, we greedily place our first point $x$ at the absolute earliest ending boundary: `x = ranges[0].second`.
3. **Filter and Delegate to $y$:** We iterate through all remaining intervals. If an interval already contains $x$ ($l \le x \le r$), it is fully satisfied and skipped. If an interval does not contain $x$, then **$y$ must cover it**.
4. **Compute $y$'s Valid Domain:** We intersect the boundaries of all intervals that missed $x$. This computes a rigid bounding range `[left, right]` where $y$ is forced to exist.
5. **Enforce Proximity Constraints:** If the track is evaluating the `close` state, $y$ cannot be anywhere; it must also be within $mid$ distance of our greedily chosen $x$. We dynamically clamp $y$'s target domain:
   $$\text{left} = \max(\text{left}, x - mid), \quad \text{right} = \min(\text{right}, x + mid)$$
6. **Final Validation:** If `left <= right`, a valid coordinate for $y$ exists, confirming that the two points $x$ and $y$ can successfully cover the entire set of constraints.

## C++ Source Code

```cpp
#include <vector>
#include <cmath>
#include <algorithm>
#include <utility>

using namespace std;

class Solution {
public:
    using ll = long long;

    // Generates the valid safe boundary interval around a neighbor value
    pair<ll, ll> getRange(ll val, ll mid){
        return {max(1LL, val - mid), val + mid};
    }

    // Intersects two intervals to find their overlapping region
    pair<ll, ll> intersect(const pair<ll, ll>& a, const pair<ll, ll>& b){
        return {max(a.first, b.first), min(a.second, b.second)};
    }
    
    // Greedy sweep checking if two points (x, y) can satisfy a collection of intervals
    bool cover(vector<pair<ll, ll>>& ranges, ll mid, bool close){
        if(ranges.empty()) return true;

        // Sort intervals by their upper bounds to optimize greedy coordination
        sort(begin(ranges), end(ranges), [](auto& a, auto& b){
            return a.second < b.second;
        });

        // Greedily position the first coordinate 'x' at the earliest closing interval wall
        ll x = ranges[0].second;
        
        // Tracking boundaries for where our second coordinate 'y' must land
        ll left = 1;
        ll right = 1e18;

        for(auto& [l, r] : ranges){
            // If current interval already wraps around 'x', it is covered
            if(l <= x && x <= r) continue;

            // Otherwise, 'y' is forced to step in and cover this specific interval
            left = max(left, l);
            right = min(right, r);
        }

        // If x and y must be placed adjacent inside a gap, enforce proximity limits
        if(close){
            left = max(left, x - mid);
            right = min(right, x + mid);
        }

        // Returns true if the remaining target space for 'y' is structurally valid
        return left <= right;
    }

    // Predicate check to verify if a candidate max difference 'mid' is achievable
    bool f(int mid, vector<int>& nums){
        int n = nums.size();

        vector<pair<ll, ll>> closeRanges;
        vector<pair<ll, ll>> farRanges;

        bool farPossible = true;

        for(int i = 0; i < n; ){
            // Skip fixed values while checking if any pre-existing adjacencies violate mid
            if(nums[i] != -1){
                if(i + 1 < n && nums[i + 1] != -1){
                    if(abs((ll)nums[i] - nums[i + 1]) > mid) return false;
                }
                i++;
                continue;
            }

            int start = i;
            while(i < n && nums[i] == -1) ++i;
            int end = i - 1;

            // Corner case: if the entire array consists of -1s, we can make all items identical
            if(start == 0 && i == n) continue;

            // Internal missing blocks bounded by both left and right neighbors
            if(start >= 1 && i < n){
                auto leftRange = getRange(nums[start - 1], mid);
                auto rightRange = getRange(nums[i], mid);

                if(end == start){
                    // A single missing slot requires one number to satisfy both sides
                    auto both = intersect(leftRange, rightRange);
                    if(both.first > both.second) return false;

                    closeRanges.push_back(both);
                    farRanges.push_back(both);
                }
                else{
                    // Multiple consecutive missing slots
                    // Close track: one number anchors left, the other anchors right
                    closeRanges.push_back(leftRange);
                    closeRanges.push_back(rightRange);

                    // Far track: one number must be capable of single-handedly spanning both sides
                    auto both = intersect(leftRange, rightRange);
                    if(both.first > both.second) farPossible = false;
                    else farRanges.push_back(both);
                }
            }
            else{
                // Edge missing blocks touching the boundary of the array (only one neighbor)
                int neigh = (start != 0) ? nums[start - 1] : nums[i];
                auto range = getRange(neigh, mid);

                closeRanges.push_back(range);
                farRanges.push_back(range);
            }
        }

        // The state is valid if either the close configuration or far configuration works
        bool close = cover(closeRanges, mid, true);
        bool far = farPossible && cover(farRanges, mid, false);

        return close || far;
    }

public:
    int minDifference(vector<int>& nums) {
        int n = nums.size();

        int low = 0;
        int high = 1e9;
        int ans = high;

        // Binary Search on the minimum acceptable maximum difference
        while(low <= high){
            int mid = low + (high - low) / 2;

            if(f(mid, nums)){
                ans = mid;
                high = mid - 1; // Try to minimize further
            }
            else low = mid + 1; // Increase target boundary threshold
        }

        return ans;
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N \log(\text{MAX}) + N \log N)$
  * **Binary Search Envelope:** The binary search operates on a fixed numeric domain range $[0, 10^9]$, running for exactly $\log_2(10^9) \approx 30$ iterations.
  * **Linear Segment Scan:** Inside `f(mid)`, we perform a single linear traversal of the array of size $N$ to partition elements and compute interval ranges, taking $\mathcal{O}(N)$ time.
  * **Sorting Constraints:** The `cover` function calls `std::sort` on the vectors of generated ranges. In the worst-case scenario (e.g., alternating elements), the number of ranges scales proportionally to $N/2$. Sorting these states takes $\mathcal{O}(N \log N)$ time.
  * **Total Time Bound:** The algorithm evaluates 30 iterations, each processing a sort and sweep. With $N = 10^5$, $N \log N \approx 1.7 \cdot 10^6$ operations, completing well within roughly 25-30 milliseconds, execution speed is exceptionally optimal.

* **Space Complexity:** $\mathcal{O}(N)$
  * **Dynamic Range Accumulators:** The vectors `closeRanges` and `farRanges` explicitly log intervals generated across missing sub-segments. Under max fragmentation layouts, these structures scale linearly to hold up to $\mathcal{O}(N)$ pairs.
  * **Total Memory Footprint:** Allocations scale linearly to accommodate tracking boundaries, remaining strictly within $\mathcal{O}(N)$ auxiliary space.
