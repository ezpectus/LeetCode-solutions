# 3229. Minimum Operations to Make Array Equal to Target
## O(n) — Difference Array "Sum of Upward Steps"

---

## Problem Statement

Each operation selects any subarray and increments or decrements all its elements by 1. Minimize operations to transform `nums` into `target`.

---

## Core Insight #1 — Reduce to Zeroing a Difference Array

Define `d[i] = target[i] - nums[i]`. Our goal is to make all `d[i] = 0`.

Each operation "+1 to nums[l..r]" **decreases** `d[l..r]` by 1.  
Each operation "-1 to nums[l..r]" **increases** `d[l..r]` by 1.

We need the minimum number of "add ±1 to a contiguous subarray of d" operations to zero out `d[]`.

---

## Core Insight #2 — Think in Terms of "Rectangular Pulses"

Visualize `d[]` as a signed histogram. Each operation lays a horizontal stripe of height +1 or -1 across a contiguous range. We need to build `d` from zero using such stripes and then remove them — but more directly: we need to cover the entire shape with the minimum number of non-overlapping (or overlapping) horizontal stripes.

The classical insight: the number of stripes needed = the number of **upward steps** in the sequence.

Define the extended sequence with sentinel values: `d[-1] = 0` and `d[n] = 0`. Consider consecutive differences:
```
D[i] = d[i] - d[i-1]   for i = 0..n   (with d[-1] = d[n] = 0)
```

**Each positive D[i] (upward step) requires starting a new stripe.** Each stripe can extend rightward until the sequence drops back, so we only "pay" at the starts of stripes.

```
Answer = Σ max(0, D[i])   for i = 0..n
```

Since `Σ D[i] = d[n] - d[-1] = 0`, the sum of positive D equals the sum of |negative D|. So equivalently: `Answer = Σ max(0, D[i])` = total upward variation.

---

## Why This Is Correct

**Lower bound:** Each positive upward step D[i] > 0 at position i means the number of "active positive-direction stripes" must increase by D[i] units at position i. Each such new stripe is a distinct operation. So we need at least Σ max(0, D[i]) operations.

**Achievability:** Greedily extend each active stripe as far right as possible (until d drops or we hit the boundary). This matches the exact count of upward steps. Each stripe is one operation.

---

## Full Implementation (C#)

```csharp
public class Solution {
    public long MinimumOperations(int[] nums, int[] target) {
        int n = nums.Length;
        long ans = 0;
        long prev = 0; // d[-1] = 0

        for (int i = 0; i < n; i++) {
            long d = (long)target[i] - nums[i];
            long diff = d - prev;          // D[i] = d[i] - d[i-1]
            if (diff > 0) ans += diff;     // count upward steps only
            prev = d;
        }
        // Boundary: D[n] = d[n] - d[n-1] = 0 - prev = -prev
        if (prev < 0) ans -= prev;         // max(0, -prev)

        return ans;
    }
}
```

## Example Walkthrough

### Example 1 — `nums=[3,5,1,2]`, `target=[4,6,2,4]`

`d = [1, 1, 1, 2]`. Extended: `0 → 1 → 1 → 1 → 2 → 0`.

| Transition | D[i] | max(0,D[i]) |
|---|---|---|
| 0 → 1 | +1 | 1 |
| 1 → 1 | 0 | 0 |
| 1 → 1 | 0 | 0 |
| 1 → 2 | +1 | 1 |
| 2 → 0 | -2 | 0 |

**Answer = 2** ✓

```
Code trace:
i=0: d=1, diff=1>0, ans=1, prev=1
i=1: d=1, diff=0,   ans=1, prev=1
i=2: d=1, diff=0,   ans=1, prev=1
i=3: d=2, diff=1>0, ans=2, prev=2
After: prev=2>0, no boundary add. Return 2.
```

---

### Example 2 — `nums=[1,3,2]`, `target=[2,1,4]`

`d = [1, -2, 2]`. Extended: `0 → 1 → -2 → 2 → 0`.

| Transition | D[i] | max(0,D[i]) |
|---|---|---|
| 0 → 1 | +1 | 1 |
| 1 → -2 | -3 | 0 |
| -2 → 2 | +4 | 4 |
| 2 → 0 | -2 | 0 |

**Answer = 5** ✓

The +4 step at i=2 reflects that d[1]=-2 pushed stripes far negative, and d[2]=2 requires 4 entirely new (positive) stripes.

---

## Sign Separation — Positive and Negative Stripes Don't Mix

A +1 operation (decrements d) and a -1 operation (increments d) can't cancel each other since they go in opposite directions. The formula correctly treats them independently by only summing the upward transitions.

If d has only positives: we only need "+1 to nums" stripes, covering the positive histogram. If d has only negatives: only "-1 to nums" stripes (upward steps happen at the NEGATIVE-to-less-negative transitions). Mixed signs increase cost because at a sign change, all stripes from one side must close and new ones of the opposite type must open.

---

## Pitfalls & Edge Cases

- **Boundary term `max(0, -prev)`:** Required when `d[n-1] < 0`. Without it, we'd miss the cost of "stepping up from negative to 0" at the right boundary. Example: `d=[-2]` requires 2 ops; boundary adds `-(-2)=2`.

- **Overflow:** `target[i], nums[i] ≤ 10⁸`. The difference fits in `int`, but the cumulative sum can reach O(n × max_value) = 10^13. Use `long` for `d`, `ans`, and arithmetic.

- **All equal (nums == target):** Every `d[i] = 0`, all diffs = 0. `ans = 0`. ✓

- **Negative-only diff:** e.g., `d = [-3, -1, -2]`. Steps: 0→-3 (D=-3, skip), -3→-1 (D=+2, add 2), -1→-2 (D=-1, skip), -2→0 (D=+2, add 2). Answer = 4. Each element takes decrement-stripes; the "upward" steps at transitions and the boundary count how many new stripes must start.

---

## Key Takeaway

"Min operations with ±1 range updates to reach a target" → reduce to "sum of upward steps in the difference sequence." This is the signed generalization of the histogram-painting problem: instead of counting how many new horizontal brush strokes start at each column, we count how many new strokes-of-either-sign start at each position's upward transition.

**Pattern recognition:** "Count minimum ±1 range operations to achieve array" → compute `d[i] = target[i] - source[i]`, then answer = `Σ max(0, d[i] - d[i-1])` with sentinels `d[-1] = d[n] = 0`.
