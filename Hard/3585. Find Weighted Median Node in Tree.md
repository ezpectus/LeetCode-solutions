# 3585. Find Weighted Median Node in Tree

## 1. Algorithmic Core & Intuition

The problem asks us to find the "weighted median node" on the unique path between two nodes `u` and `v` in a tree. The total weight of the path can be calculated quickly using node depths from the root (node 0). The weighted median is the first node `x` on the path from `u` to `v` such that the distance from `u` to `x` satisfies:
`2 * dist(u, x) >= total_path_weight`

Given the constraints where `N, Q <= 10^5`, a linear scan of the path for each query is too slow (`O(N)` per query). We need a logarithmic approach (`O(log N)` per query). Since a path in a rooted tree goes up from `u` to the Lowest Common Ancestor (`LCA`) and then goes down to `v`, we can analyze the path as two vertical segments: `u -> LCA` and `LCA -> v`.

By precomputing the standard binary lifting tables (ancestors at powers of 2) along with the cumulative weights to those ancestors, we can binary search the exact node vertically in `O(log N)` time.

---

## 2. Dynamic Programming & Transition Steps

### 2.1 State Definitions
We choose node `0` as the root of the tree and run a Depth First Search (DFS) to compute:
- `depth[i]`: The number of edges on the path from root `0` to node `i`.
- `dist_from_root[i]`: The cumulative sum of edge weights from root `0` to node `i`.
- `up[i][j]`: The `2^j`-th ancestor of node `i`.
- `weight_up[i][j]`: The total weight of edges between node `i` and its `2^j`-th ancestor.

### 2.2 Table Transitions (Precomputation)
The tables are filled using dynamic programming. The base cases (`j = 0`) are populated directly from the tree edges during the initial DFS. For `j > 0`, the relations are:
`up[i][j] = up[up[i][j - 1]][j - 1]`
`weight_up[i][j] = weight_up[i][j - 1] + weight_up[up[i][j - 1]][j - 1]`

### 2.3 Mathematical Breakdown by Cases
For each query `[u, v]`, let `L` be the Lowest Common Ancestor of `u` and `v`. 
The total path weight is calculated in `O(1)` as:
`total_weight = (dist_from_root[u] - dist_from_root[L]) + (dist_from_root[v] - dist_from_root[L])`

The target distance we need to cover starting from node `u` is `target = (total_weight + 1) / 2` (using integer ceiling division for `half`).

We can locate the median node by splitting the path into two primary cases:

#### Case 1: The median lies on the upward segment `u -> LCA`
This case occurs if the total weight from `u` up to `L` is greater than or equal to the `target`:
`(dist_from_root[u] - dist_from_root[L]) >= target`

To find the first node `x` where the distance from `u` is `>= target`, we can instead find the **last** node where the distance from `u` is **strictly less** than `target`, and then take its immediate parent (`up[x][0]`).
- We loop `j` from `19` down to `0`.
- If `up[u][j]` exists (is not root's parent) and its depth is `>= depth[L]`, we check if moving to it keeps the accumulated weight strictly less than `target`.
- If `accumulated_weight + weight_up[u][j] < target`, we make the jump: `accumulated_weight += weight_up[u][j]` and `u = up[u][j]`.
- After the loop, `u` is the highest node whose distance from the initial position is `< target`. The final answer is its direct parent `up[u][0]`. If the initial `u` already satisfied the condition (`target == 0`), the answer is simply the initial `u`.

#### Case 2: The median lies on the downward segment `LCA -> v`
This occurs if the upward segment does not have enough weight to satisfy the `target`. 
If the median lies on the segment from `LCA` down to `v`, it means it is located at a specific distance from node `v` when looking *upwards* from `v` to `L`.
- The distance from the median node to `v` is equal to the remaining part of the total path:
  `target_from_v = total_weight - target`
- Now, the problem transforms into finding a node `x` on the upward path from `v` to `L` such that the distance from `v` to `x` is **exactly** or **at most** `target_from_v`, but since we want the first node from `u`'s perspective, we need the highest node from `v`'s perspective that satisfies the condition.
- We loop `j` from `19` down to `0`. If `up[v][j]` exists and its depth `>= depth[L]`, we check if jumping up exceeds `target_from_v`.
- If `weight_up[v][j] <= target_from_v`, we can safely lift `v` up because it means that node is still on the path segment containing the median or closer to `LCA`. We subtract the weight from the target: `target_from_v -= weight_up[v][j]` and update `v = up[v][j]`.
- After the loop terminates, `v` will land precisely on the target median node.

## 3. Full Optimal Implementation (C++)
```
// Binary Lifting + Lowest Common Ancestor + Tree DP
#include <vector>
#include <cmath>
#include <algorithm>

class Solution {
private:
    int max_power;
    std::vector<int> depth;
    std::vector<long long> dist_from_root;
    std::vector<std::vector<int>> up;
    std::vector<std::vector<long long>> weight_up;
    std::vector<std::vector<std::pair<int, long long>>> adj;

    void dfs(int node, int parent, int current_depth, long long current_dist) {
        depth[node] = current_depth;
        dist_from_root[node] = current_dist;
        
        up[node][0] = parent;
        for (int j = 1; j < max_power; ++j) {
            if (up[node][j - 1] != -1) {
                up[node][j] = up[up[node][j - 1]][j - 1];
                weight_up[node][j] = weight_up[node][j - 1] + weight_up[up[node][j - 1]][j - 1];
            } else {
                up[node][j] = -1;
            }
        }

        for (const auto& edge : adj[node]) {
            int neighbor = edge.first;
            long long weight = edge.second;
            if (neighbor != parent) {
                weight_up[neighbor][0] = weight;
                dfs(neighbor, node, current_depth + 1, current_dist + weight);
            }
        }
    }

    int get_lca(int u, int v) {
        if (depth[u] < depth[v]) {
            std::swap(u, v);
        }
        
        for (int j = max_power - 1; j >= 0; --j) {
            if (up[u][j] != -1 && depth[up[u][j]] >= depth[v]) {
                u = up[u][j];
            }
        }
        
        if (u == v) return u;
        
        for (int j = max_power - 1; j >= 0; --j) {
            if (up[u][j] != -1 && up[u][j] != up[v][j]) {
                u = up[u][j];
                v = up[v][j];
            }
        }
        return up[u][0];
    }

public:
    std::vector<int> findMedian(int n, std::vector<std::vector<int>>& edges, std::vector<std::vector<int>>& queries) {
        max_power = std::ceil(std::log2(n)) + 1;
        if (max_power < 1) max_power = 1;

        depth.assign(n, 0);
        dist_from_root.assign(n, 0LL);
        up.assign(n, std::vector<int>(max_power, -1));
        weight_up.assign(n, std::vector<long long>(max_power, 0LL));
        adj.assign(n, std::vector<std::pair<int, long long>>());

        for (const auto& edge : edges) {
            adj[edge[0]].push_back({edge[1], edge[2]});
            adj[edge[1]].push_back({edge[0], edge[2]});
        }

        dfs(0, -1, 0, 0LL);

        std::vector<int> ans;
        ans.reserve(queries.size());

        for (const auto& query : queries) {
            int u = query[0];
            int v = query[1];
            
            if (u == v) {
                ans.push_back(u);
                continue;
            }

            int lca_node = get_lca(u, v);
            long long weight_u_to_lca = dist_from_root[u] - dist_from_root[lca_node];
            long long weight_v_to_lca = dist_from_root[v] - dist_from_root[lca_node];
            long long total_weight = weight_u_to_lca + weight_v_to_lca;
            long long target = (total_weight + 1) / 2;

            if (weight_u_to_lca >= target) {
                long long accumulated = 0;
                for (int j = max_power - 1; j >= 0; --j) {
                    if (up[u][j] != -1 && depth[up[u][j]] >= depth[lca_node]) {
                        if (accumulated + weight_up[u][j] < target) {
                            accumulated += weight_up[u][j];
                            u = up[u][j];
                        }
                    }
                }
                ans.push_back(accumulated >= target ? u : up[u][0]);
            } else {
                long long target_from_v = total_weight - target;
                for (int j = max_power - 1; j >= 0; --j) {
                    if (up[v][j] != -1 && depth[up[v][j]] >= depth[lca_node]) {
                        if (weight_up[v][j] <= target_from_v) {
                            target_from_v -= weight_up[v][j];
                            v = up[v][j];
                        }
                    }
                }
                ans.push_back(v);
            }
        }

        return ans;
    }
};
```
## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The total execution time can be broken down into the graph initialization, tree traversal precomputation, and query processing phases:
T_total = T_init + T_dfs + T_queries

1. **Graph Initialization (T_init):**
   - Building the adjacency list takes a single pass over the edges array of size N - 1. This requires O(N) time.

2. **DFS Tree Precomputation (T_dfs):**
   - The DFS visits each of the N nodes exactly once.
   - At each node, we populate the binary lifting rows up to max_power (which is log2(N)).
   - This takes O(N * log N) operations. For N = 10^5, log2(N) is approximately 17, leading to around 1.7 * 10^6 primitive operations.

3. **Query Processing (T_queries):**
   - For each of the Q queries, we find the Lowest Common Ancestor (LCA) which uses binary lifting and takes O(log N) time.
   - After locating the LCA, we perform another descending binary lifting loop of O(log N) steps to jump directly to the target node.
   - Thus, each query takes O(log N) time. For Q queries, the total processing phase takes O(Q * log N).

Combining all parts, the total time complexity is bounded by:
T_total = O((N + Q) * log N)

### 4.2 Detailed Space Complexity Decomposition
- **Adjacency List:** Stores 2 * (N - 1) directed structural items containing target nodes and weight attributes. This takes O(N) space.
- **Binary Lifting Tables (up and weight_up):** Both tables are size N x log2(N). Storing these values requires O(N * log N) memory.
- **Depth and Distance Arrays:** Simple linear arrays tracking metrics per node require O(N) space.
- **Recursion Stack:** In the worst-case scenario of a completely skewed line tree, the call stack depth reaches O(N).

Therefore, the final space complexity is:
Space Complexity = O(N * log N)

### 4.3 Complexity Summary
| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O((N + Q) * log N) | Dominated by the O(N log N) tree preparation and O(log N) operations per query. |
| **Space Complexity** | O(N * log N) | Required to hold the 2D sparse tables for parents and path weights. |

## Pitfalls & Edge Cases
- **Integer Overflow on Distances:** Edge weights can be up to 10^9. Accumulating these over a path can easily exceed the max limit of a standard 32-bit signed integer (2 * 10^9). To avoid overflow, path lengths and binary lifting weight accumulations must use long long (64-bit integers).
- **Ceiling Division for Paths:** When computing half of the path weight, the condition says "greater than or equal to half". For odd total weights, integer division like total / 2 truncates downwards. We use (total_weight + 1) / 2 to properly handle fractional values.
- **Queries on the Same Node:** If u == v, the path length is 0. The code short-circuits this immediately by verifying equality and pushing u directly to results to avoid faulty LCA jumps.
- **LCA Corner Case:** When u is an ancestor of v (or vice-versa), get_lca correctly returns the ancestor early without executing the secondary loop. The code structure accounts for this naturally.

## Key Takeaway
When you need to find a specific threshold node or accumulate values along paths between arbitrary nodes in a tree, combine Lowest Common Ancestor (LCA) queries with an extended Binary Lifting sparse table containing prefix aggregate states.

**Pattern recognition:** "path from ui to vi such that the sum of edge weights... minimum number... tree queries" -> Tree DP + Binary Lifting + Lowest Common Ancestor (LCA)
