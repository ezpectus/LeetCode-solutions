# 3098. Find the Sum of Subsequence Powers

## O(N² * K * Unique_Diffs) — Top-Down DP via State Compression Mask

### Core Idea — Value Sorting and State Packing
The problem asks for the sum of "powers" (minimum absolute differences) of all subsequences of length `k`. Sorting the array initially simplifies the math: the minimum difference in any subsequence will always exist between two adjacent elements in its sorted form. This completely eliminates the need for absolute value operations.

We can model this as a classic picking problem using Top-Down Dynamic Programming with Memoization. At each step `i`, we make a binary choice: either `skip` the current element or `pick` it. When we pick it, we update the running minimum difference (`mini`) using the distance to the previous picked element (`nums[prev]`).

To avoid allocating a complex, sparse multi-dimensional DP table due to the unpredictable floating nature of `mini`, we pack all four state variables (`i`, `prev`, `k`, `mini`) into a single 64-bit integer (`long long key`) using mathematical offsets. We then store and look up these states in a Hash Map (`unordered_map`).

---

### Full Optimal Implementation (C++)

```cpp
#include <vector>
#include <algorithm>
#include <unordered_map>

using namespace std;

class Solution {
private:
    int n;
    int mod = 1e9 + 7;
    unordered_map<long long, int> dp;

    int sim(int i, int prev, int k, int mini, vector<int>& nums) {
        // Base Cases
        if (!k) return mini;
        if (i == n and k) return 0;

        // Compress multi-dimensional state parameters into a single unique 64-bit key
        long long key = i + 1e2 * prev + 1e4 * k + 1e6 * mini;
        if (dp.find(key) != dp.end()) return dp[key];

        // Choice 1: Skip the current element
        int skip = sim(i + 1, prev, k, mini, nums);

        // Choice 2: Pick the current element and recalculate the minimum delta bounds
        int currentDiff = (prev == -1) ? mod : nums[i] - nums[prev];
        int pick = sim(i + 1, i, k - 1, min(mini, currentDiff), nums);

        return dp[key] = (pick + skip) % mod;
    }

public:
    int sumOfPowers(vector<int>& nums, int k) {
        n = nums.size();
        sort(nums.begin(), nums.end());
        dp.clear();
        return sim(0, -1, k, 1e9, nums);
    }
};
```
### Complexity

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | $O(N^2 \cdot K \cdot U)$ | $U$ represents the number of unique minimum differences possible. State transitions take $O(1)$ due to Hash Map lookups. |
| **Space Complexity** | $O(N^2 \cdot K \cdot U)$ | Memory scales dynamically based on the total number of unique valid state masks saved inside the `dp` hash map cache. |

---

### Pitfalls & Edge Cases

* **Negative Value Modulo Boundaries:** Since original constraints permit negative inputs down to $-10^8$, sorting the elements safely aligns them sequentially so that `nums[i] - nums[prev]` remains strictly non-negative.
* **Hash Collisions & Lookups:** Compressing keys using static factors (like `1e2`, `1e4`) works flawlessly here because the constraints on $N$ and $K$ are small ($\le 50$), ensuring values never overflow or bleed into each other's bit boundaries.

---

### Key Takeaway

> When faced with a multi-dimensional DP state where one or more parameters are highly dynamic or sparse (like min/max tracking ranges), discard rigid multi-dimensional arrays. Compress the active parameters into a single composite key integer and cache transitions dynamically inside a Hash Table.

* **Pattern recognition:** "Sum of min/max structural attributes over combinations of length K" -> Sorted Array Recursion + Composite Key State Memoization.
