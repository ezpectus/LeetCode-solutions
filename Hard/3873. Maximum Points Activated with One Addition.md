# 3873. Maximum Points Activated with One Addition

## 1. Idea & Architectural Choice

### 1.1 Core Philosophy
The problem asks for the maximum number of activated points after adding one new point at an empty coordinate $(x, y)$. Two points are connected if they share the same x-coordinate or the same y-coordinate. Activation propagates transitively across all connected points.

This equivalence relation (sharing an x or y coordinate) defines a grid graph where components consist of points transitively sharing coordinates. 

- Each original point $(x_i, y_i)$ connects the x-line $x_i$ and the y-line $y_i$.
- Placing a new point $(x_{new}, y_{new})$ can bridge two distinct connected components $C_A$ and $C_B$ if we pick $x_{new}$ from a line belonging to $C_A$ and $y_{new}$ from a line belonging to $C_B$.
- The total number of activated points will be:
  `activated = size(C_A) + size(C_B) + 1` (where +1 accounts for the newly placed point).

If there is only 1 connected component in the entire graph, adding a new point anywhere sharing a coordinate with that component will activate all $N$ original points plus the 1 new point, yielding $N + 1$.

To maximize the total activated points, our goal simplifies to finding the sizes of the **two largest connected components** formed by the initial set of points.

### 1.2 Mathematical Reduction & DSU Coordinate Mapping
Coordinates $x_i$ and $y_i$ can be up to $10^9$ in absolute value. Since $x$ and $y$ axes are orthogonal, we must prevent identifier collisions between $x = c$ and $y = c$. We map:
- $x \to x$
- $y \to y + \text{OFFSET}$, where $\text{OFFSET} = 3 \times 10^9$

Using Disjoint Set Union (DSU) with a dynamic hash map (or coordinate compression), we group all connected lines and points.

---

## 2. Component Enumeration & Maximization Strategy

### 2.1 Algorithm Steps
1. **Initialize DSU:** Map parent pointers and component sizes dynamically using hash tables.
2. **Build Components:** For each point $(x_i, y_i)$, perform `union(x_i, y_i + OFFSET)`.
3. **Count Point Densities:** Iterate over all original points and accumulate the size of each component by querying `find(x_i)`.
4. **Find Top Two Sizes:** Extract component sizes, track the largest $S_1$ and second largest $S_2$.
5. **Compute Final Answer:**
   - If number of components $\ge 2$: `return S_1 + S_2 + 1`.
   - If number of components $== 1$: `return S_1 + 1` (which equals $N + 1$).

## 3. Full Optimal Implementation (C++)
```

// Hash Table + Disjoint Set Union (DSU) + Greedy
#include <vector>
#include <unordered_map>
#include <algorithm>

class DynamicDSU {
private:
    std::unordered_map<long long, long long> parent;

public:
    long long find(long long i) {
        if (parent.find(i) == parent.end()) {
            return parent[i] = i;
        }
        if (parent[i] == i) {
            return i;
        }
        return parent[i] = find(parent[i]);
    }

    void unite(long long i, long long j) {
        long long root_i = find(i);
        long long root_j = find(j);
        if (root_i != root_j) {
            parent[root_i] = root_j;
        }
    }
};

class Solution {
public:
    int maxActivated(std::vector<std::vector<int>>& points) {
        DynamicDSU dsu;
        const long long OFFSET = 3000000000LL; // 3 * 10^9 to avoid coordinate collision

        // Step 1: Union x and y coordinates for each point
        for (const auto& point : points) {
            long long x = point[0];
            long long y = point[1] + OFFSET;
            dsu.unite(x, y);
        }

        // Step 2: Count how many original points belong to each connected component
        std::unordered_map<long long, int> component_size;
        for (const auto& point : points) {
            long long root = dsu.find(point[0]);
            component_size[root]++;
        }

        // Step 3: Find the two largest component sizes
        int max1 = 0;
        int max2 = 0;

        for (const auto& [root, size] : component_size) {
            if (size > max1) {
                max2 = max1;
                max1 = size;
            } else if (size > max2) {
                max2 = size;
            }
        }

        // Step 4: Return maximum activated points including the new point
        return max1 + max2 + 1;
    }
};
```

## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
Let N be the number of points (N <= 10^5).

1. **DSU Building Phase**:
   - Iterates N times.
   - Each `unite` operation performs `find` with path compression on hash map lookup.
   - Operations: T_dsu = O(N * alpha(N)), where alpha is the inverse Ackermann function.

2. **Component Sizing Phase**:
   - Iterates over N points, performing 1 `find` query per point.
   - Operations: T_size = O(N * alpha(N)).

3. **Top 2 Extraction Phase**:
   - Iterates through unique component roots (at most N).
   - Operations: T_top = O(N).

**Total Time Complexity:**
T_total = T_dsu + T_size + T_top = O(N * alpha(N))

For N = 10^5, alpha(N) <= 4, total operations stay well under 5 * 10^5, running in less than 25 milliseconds.

### 4.2 Detailed Space Complexity Decomposition
- **parent Hash Map**: Stores at most 2N coordinate nodes (N distinct x and N distinct y).
- **component_size Hash Map**: Stores at most N unique root entries.

**Total Space Complexity:**
S_total = O(N)

Memory consumed is under 10 MB for N = 10^5, well within standard platform limits (256 MB).

### 4.3 Complexity Summary

| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(N * alpha(N)) | N = points.length <= 10^5. Dynamic DSU with path compression. |
| **Space Complexity** | O(N) | Hash maps storing coordinate nodes and component sizes. |

## 5. Mathematical Formulations & Foundational Algorithms

### Component Merging & Maximum Connectivity Theorem
Let G = (V, E) be a bipartite-like graph where V_X are x-lines, V_Y are y-lines, and edges E are given points (x_i, y_i).
A newly added point (x, y) creates a new edge between line node x and line node y.

1. If x belongs to component C_A and y belongs to component C_B where C_A != C_B, adding (x, y) merges components C_A and C_B into a single connected component of total points size(C_A) + size(C_B).
2. The added point itself contributes +1 to the total activated count.
3. By the greedy principle, choosing C_A and C_B as the two largest components maximizes size(C_A) + size(C_B) + 1.

## Pitfalls & Edge Cases

* **Coordinate Collisions:** x_i and y_i can have identical values (e.g., x = 2, y = 2). Adding an offset OFFSET = 3 * 10^9 prevents x and y from accidentally sharing DSU parent pointers.
* **Single Component Case:** If all points are already connected in 1 component, max2 remains 0. The formula max1 + max2 + 1 seamlessly yields N + 0 + 1 = N + 1, which is correct.
* **Large Coordinates Overflow:** Coordinates range up to 10^9. Adding 3 * 10^9 exceeds 32-bit signed integer capacity (2 * 10^9). Using `long long` for keys avoids overflow.

---

## Key Takeaway

When activation or connection spreads along shared coordinates, model x-lines and y-lines as graph nodes and points as edges. Use DSU to find connected component sizes, then solve modification queries greedily by bridging the largest components.

**Pattern recognition:** "points sharing x or y coordinate activated" -> Disjoint Set Union (DSU) + Coordinate Offset + Greedy Component Merging
