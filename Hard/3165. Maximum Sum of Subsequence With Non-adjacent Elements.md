## 3165. Maximum Sum of Subsequence With Non-adjacent Elements

### Problem Statement
You are given an array `nums` consisting of integers and a 2D array `queries`, where `queries[i] = [posi, xi]`.

For each query `i`:
1. Update `nums[posi] = xi`.
2. Calculate the maximum sum of a subsequence of `nums` where no two selected elements are adjacent.

Return the sum of the answers to all queries modulo **10^9 + 7**.

*Note: An empty subsequence has a sum of 0.*

---

### Core Insight — Segment Tree + Node-Level DP
A naive Dynamic Programming approach (like the classic House Robber) takes `O(N)` time per query. With `Q` queries, the total time would be `O(N * Q)`, which easily triggers a Time Limit Exceeded (TLE) when `N, Q <= 5 * 10^4`. 

To handle point updates and dynamic queries in `O(log N)` time, we must use a **Segment Tree** where each node stores localized DP states for its range `[L, R]`.

For any interval, we maintain **4 distinct DP sub-states** depending on whether the boundary elements are included:
* `f00`: Max sum excluding both `L` and `R`.
* `f01`: Max sum excluding `L`, including `R`.
* `f10`: Max sum including `L`, excluding `R`.
* `f11`: Max sum including both `L` and `R`.

**Merging States:**
When combining a left child `[L, Mid]` and a right child `[Mid + 1, R]`, the elements at `Mid` and `Mid + 1` are adjacent. Therefore, we can never combine states where both the right boundary of the left child and the left boundary of the right child are active simultaneously.

---

### Algorithm
1. Allocate a Segment Tree array of size `4 * N`.
2. **Base Case (Leaf Nodes where L == R):**
   * `f11 = max(0, nums[L])` (choosing the single available element if it is positive).
   * `f00 = 0` (choosing nothing).
   * `f01 = f10 = -infinity` (or `0` under these constraints since invalid single-element configurations fall back to empty choices).
3. **Point Update:** Traverse down to the leaf representing `pos`, update its value, and systematically re-merge states up to the root node.
4. **Answer Accumulation:** The global maximum for the entire array is always located at the root node (`tree[1]`), calculated as `max({f00, f01, f10, f11})`.

---

### Full Optimal Implementation (C++)
```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
    struct Node {
        long long f00 = 0;
        long long f01 = 0;
        long long f10 = 0;
        long long f11 = 0;
    };

    vector<Node> tree;
    int n;
    const int MOD = 1e9 + 7;

    void merge(Node& res, const Node& left, const Node& right) {
        // Prevent picking both the right side of the left child and the left side of the right child
        res.f00 = max({left.f00 + right.f00, left.f00 + right.f10, left.f01 + right.f00});
        res.f01 = max({left.f00 + right.f01, left.f00 + right.f11, left.f01 + right.f01});
        res.f10 = max({left.f10 + right.f00, left.f10 + right.f10, left.f11 + right.f00});
        res.f11 = max({left.f10 + right.f01, left.f10 + right.f11, left.f11 + right.f01});
    }

    void build(const vector<int>& nums, int node, int start, int end) {
        if (start == end) {
            tree[node].f11 = max(0, nums[start]);
            tree[node].f00 = 0;
            tree[node].f01 = 0;
            tree[node].f10 = 0;
            return;
        }
        int mid = start + (end - start) / 2;
        build(nums, 2 * node, start, mid);
        build(nums, 2 * node + 1, mid + 1, end);
        merge(tree[node], tree[2 * node], tree[2 * node + 1]);
    }

    void update(int node, int start, int end, int idx, int val) {
        if (start == end) {
            tree[node].f11 = max(0, val);
            return;
        }
        int mid = start + (end - start) / 2;
        if (idx <= mid) {
            update(2 * node, start, mid, idx, val);
        } else {
            update(2 * node + 1, mid + 1, end, idx, val);
        }
        merge(tree[node], tree[2 * node], tree[2 * node + 1]);
    }

public:
    int maximumSumSubsequence(vector<int>& nums, vector<vector<int>>& queries) {
        n = nums.size();
        tree.resize(4 * n);
        
        build(nums, 1, 0, n - 1);
        
        long long totalSum = 0;
        for (const auto& q : queries) {
            update(1, 0, n - 1, q[0], q[1]);
            
            long long currentMax = max({tree[1].f00, tree[1].f01, tree[1].f10, tree[1].f11});
            totalSum = (totalSum + currentMax) % MOD;
        }
        
        return totalSum;
    }
};
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N + Q log N) | Tree initialization takes `O(N)`. Each of the `Q` queries processes down a single tree branch path, running in `O(log N)` time. |
| **Space** | O(N) | The segment tree linear array storage structure allocates exactly `4 * N` element nodes. |

### Pitfalls & Edge Cases
* **Negative Value Inclusions:** Elements can drop down to `-10^5`. An empty subsequence yields a sum of `0`. Always anchor your baseline states with `max(0, val)` to avoid accumulating negative elements.
* **State Mapping Matrix:** Keep transitions clean during the node merge step. Ensure you don't map overlapping combinations (like `left.f01` with `right.f11`) that breach the adjacency restriction.

### Connection to Classic Problems
| Problem | Core Target | Processing Strategy |
|---|---|---|
| **House Robber** | Find maximum non-adjacent subarray sums on a static array | Simple linear 1D dynamic programming |
| **Range Sum Query - Mutable** | Track sum intervals over changing point states | Standard single-value tracking segment tree |
| **This Problem** | Maintain maximum non-adjacent sums across real-time edits | Matrix-state DP transitions mapped inside a segment tree |

### Key Takeaway
Whenever you see a dynamic array query problem that requires recalculating a complex DP state after every single point update, consider embedding the DP states directly into the nodes of a segment tree. This allows you to scale state updates from linear time `O(N)` down to highly performant log time `O(log N)`.
