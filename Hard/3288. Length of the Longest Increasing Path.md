# LeetCode 3288. Length of the Longest Increasing Path

## Problem Description
You are given a 2D array of integers `coordinates` of length `n` and an integer `k`, where `0 <= k < n`. `coordinates[i] = [xi, yi]` indicates the point `(xi, yi)` in a 2D plane.

An increasing path of length `m` is defined as a list of points `(x1, y1), (x2, y2), (x3, y3), ..., (xm, ym)` such that:
* `xi < xi + 1` and `yi < yi + 1` for all `i` where `1 <= i < m`.
* `(xi, yi)` is in the given coordinates for all `i` where `1 <= i <= m`.

Return the *maximum length* of an increasing path that **contains** `coordinates[k]`.

---

## Method Explanation: Two-Sided 2D LIS (Longest Increasing Subsequence)

The core constraint of the problem is that the path **must include** the pivot point $P = \text{coordinates}[k]$. A path is strictly increasing if both $x$ and $y$ coordinates strictly increase ($x_i < x_{i+1}$ and $y_i < y_{i+1}$).

This constraint naturally splits the 2D plane into two isolated valid zones relative to $P = (targetX, targetY)$:
1. **Left (Before) Zone:** Points that can legally appear *before* $P$. For these points, it must hold that $x < targetX$ and $y < targetY$.
2. **Right (After) Zone:** Points that can legally appear *after* $P$. For these points, it must hold that $x > targetX$ and $y > targetY$.

Any point that does not fit into either category can never be part of a valid strictly increasing path that passes through $P$. Therefore, we can filter the coordinates into two separate groups, find the 2D Longest Increasing Subsequence (LIS) for each group independently, and sum their lengths up plus `1` (accounting for the pivot point $P$ itself).

### 2D LIS Optimization ($O(N \log N)$)
To find the LIS in a 2D space efficiently:
1. Sort the points by their $x$-coordinate in **ascending** order.
2. If two points have the same $x$-coordinate, sort them by their $y$-coordinate in **descending** order. This prevents the 1D LIS algorithm from picking multiple points with the same $x$-coordinate (since $y$ decreases, a standard LIS on $y$ won't allow a smaller or equal value to extend the subsequence).
3. Apply the classic 1D LIS algorithm using binary search (`std::lower_bound` / Patience Sorting) on the $y$-coordinates of the sorted points.

## C++ Source Code

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
private:
    // Standard O(N log N) 2D LIS using Patience Sorting / Binary Search
    int getLIS(vector<vector<int>>& points) {
        if (points.empty()) return 0;

        // Sort by X ascending. If X is equal, sort by Y descending.
        sort(points.begin(), points.end(), [](const vector<int>& a, const vector<int>& b) {
            if (a[0] == b[0]) return a[1] > b[1];
            return a[0] < b[0];
        });

        vector<int> lis_tails;
        for (const auto& point : points) {
            int y = point[1];
            auto it = lower_bound(lis_tails.begin(), lis_tails.end(), y);
            if (it == lis_tails.end()) {
                lis_tails.push_back(y);
            } else {
                *it = y;
            }
        }
        return lis_tails.size();
    }

public:
    int maxPathLength(vector<vector<int>>& coordinates, int k) {
        int targetX = coordinates[k][0];
        int targetY = coordinates[k][1];

        vector<vector<int>> leftGroup;
        vector<vector<int>> rightGroup;

        // Partition coordinates into two independent sets relative to coordinates[k]
        for (const auto& point : coordinates) {
            if (point[0] < targetX && point[1] < targetY) {
                leftGroup.push_back(point);
            } else if (point[0] > targetX && point[1] > targetY) {
                rightGroup.push_back(point);
            }
        }

        // Total Max Path Length = LIS(left) + Pivot Node (1) + LIS(right)
        return getLIS(leftGroup) + 1 + getLIS(rightGroup);
    }
};
```
## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N \log N)$
  * **Filtering:** Partitioning the initial array into `leftGroup` and `rightGroup` takes linear time, $\mathcal{O}(N)$.
  * **Sorting:** Sorting both filtered coordinate groups takes $\mathcal{O}(N \log N)$ time.
  * **LIS Processing:** The binary search LIS loop (`std::lower_bound`) processes elements in $\mathcal{O}(N \log N)$ time.
  * **Total Time:** $\mathcal{O}(N \log N)$, which scales optimally and comfortably runs within the $10^5$ constraints boundary.

* **Space Complexity:** $\mathcal{O}(N)$
  * **Auxiliary Storage:** Linear space is required to store the partitioned elements in `leftGroup` and `rightGroup`.
  * **LIS Tracking:** The internal dynamic array (`lis_tails`) consumes up to $\mathcal{O}(N)$ space to maintain the active tails of the increasing subsequences.
