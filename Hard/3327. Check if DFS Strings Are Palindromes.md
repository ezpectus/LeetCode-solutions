# LeetCode 3327. Check if DFS Strings Are Palindromes

## Problem Description
You are given a tree rooted at node `0` with `n` nodes. Each node has a character assigned to it from a string `s`. A recursive post-order DFS traversal forms a string `dfsStr` for any node by first visiting its children in increasing order of their IDs and then appending its own character. 

Return a boolean array `answer` where `answer[i]` is `true` if the string generated by starting the DFS at node `i` is a palindrome.

---

## Method Explanation: Post-Order DFS Flattening & Manacher's Algorithm

Generating the DFS string independently for each node requires $\mathcal{O}(N^2)$ time, causing a Time Limit Exceeded error. We can achieve an optimal $\mathcal{O}(N)$ linear time bound by decoupling the tree structure from the palindrome verification:

1. **Tree Flattening via Post-Order DFS:** We execute a single global DFS traversal starting at the root (node `0`). By visiting children in increasing order and appending the current node's character *after* visiting its subtrees, the DFS string for any node $x$ maps perfectly to a contiguous subarray range `[start[x], end[x]]` inside one single flattened global string.
2. **Linear Palindrome Substring Querying (Manacher's Algorithm):** Once the global string is fully assembled, we compute palindrome radii centered at every character using **Manacher's Algorithm** in $\mathcal{O}(N)$ time. This maps character symmetries across the entire string length.
3. **Constant Time Validation:** For each node $x$, we check if its corresponding segment `[start[x], end[x]]` is a palindrome by querying the precomputed Manacher values in $\mathcal{O}(1)$ time.

## C++ Source Code

```cpp
#include <vector>
#include <string>
#include <algorithm>

using namespace std;

class Solution {
private:
    vector<vector<int>> adj;
    string global_str = "";
    vector<int> start_idx, end_idx;

    // Flatten the tree into a single post-order traversal string
    void dfs_flatten(int u, const string& s) {
        start_idx[u] = global_str.length();
        
        for (int v : adj[u]) {
            dfs_flatten(v, s);
        }
        
        global_str += s[u];
        end_idx[u] = global_str.length() - 1;
    }

public:
    vector<bool> findAnswer(vector<int>& parent, string s) {
        int n = parent.size();
        adj.assign(n, vector<int>());
        start_idx.assign(n, 0);
        end_idx.assign(n, 0);

        // Build adjacency tree ensuring children are in sorted order
        for (int i = 1; i < n; ++i) {
            adj[parent[i]].push_back(i);
        }
        for (int i = 0; i < n; ++i) {
            sort(adj[i].begin(), adj[i].end());
        }

        // 1. Flatten the tree post-order
        dfs_flatten(0, s);

        // 2. Transform the string for Manacher's Algorithm to handle even/odd lengths smoothly
        // e.g., "aba" -> "^#a#b#a#$"
        string T = "^";
        for (char c : global_str) {
            T += "#";
            T += c;
        }
        T += "#$";

        int m = T.length();
        vector<int> P(m, 0);
        int C = 0, R = 0;

        for (int i = 1; i < m - 1; i++) {
            int i_mirror = 2 * C - i;
            if (R > i) {
                P[i] = min(R - i, P[i_mirror]);
            }
            
            while (T[i + 1 + P[i]] == T[i - 1 - P[i]]) {
                P[i]++;
            }

            if (i + P[i] > R) {
                C = i;
                R = i + P[i];
            }
        }

        // 3. Process the results for each node in O(1) using the precomputed radii
        vector<bool> ans(n);
        for (int i = 0; i < n; ++i) {
            int L = start_idx[i];
            int r = end_idx[i];
            
            // Map the structural indices [L, r] into Manacher's transformed string index space
            int manacher_center = (L + r) + 2; 
            int segment_len = r - L + 1;
            
            ans[i] = (P[manacher_center] >= segment_len);
        }

        return ans;
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N)$
  * **Tree Linearization:** Distributing and ordering child edges takes $\mathcal{O}(N \log D)$ time due to the explicit identifier sorting step (where $D$ is the maximum node degree). The subsequent post-order DFS scan processes each vertex exactly once, building the flattened structure in linear time.
  * **Manacher Processing (Palindromic Core):** The core string algorithm evaluates the transformed string array in exactly $\mathcal{O}(N)$ operations. It achieves this linear bound by leveraging symmetric mirrors around a moving center $C$ bounded by a rightmost boundary $R$. Since the right boundary $R$ moves strictly forward and never backtracks, the inner loop transitions execute an aggregate total of $\mathcal{O}(N)$ times over the entire execution span.
  * **Range Queries:** Validating whether a specific segment is a palindrome reduces to an index mapping arithmetic verification, running in strict $\mathcal{O}(1)$ constant time for each of the $N$ nodes.
  * **Total Time:** $\mathcal{O}(N)$, which scales optimally and comfortably satisfies the $10^5$ constraints.

* **Space Complexity:** $\mathcal{O}(N)$
  * **Graph Structure:** The adjacency map requires $\mathcal{O}(N)$ memory allocations to store parent-child tree layouts.
  * **Transform String Arrays:** Memory consumption for the structural tracking arrays (`start_idx`, `end_idx`), accumulated `global_str`, padded Manacher string `T`, and symmetry radius tracking array `P` scales strictly linearly with input length, maintaining a clean $\mathcal{O}(N)$ memory environment.
