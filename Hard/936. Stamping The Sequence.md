# 936. Stamping The Sequence  
*O(n²) — Greedy Reverse Simulation*

---

## Problem Statement

You are given two strings `stamp` and `target`.  
Initially, there is a string `s` of length `target.length` with all characters `'?'`.

In one turn, you can:
- Choose an index `i` such that stamp fully fits in `s` starting at `i` (i.e., `i + stamp.length - 1 < s.length`)
- Replace every position in that range with the corresponding character from `stamp`  
  (overwriting whatever was there, including '?')

Goal: convert `s` to `target` using **at most 10 * target.length** turns.  
Return an array of the **starting indices** used in each turn (in the order performed).  
If impossible, return empty array.

**Examples**:

**Example 1**  
Input: stamp = "abc", target = "ababc"  
Output: **[0,2]**  
Sequence:  
"?????" → place at 0 → "abc??" → place at 2 → "ababc"

**Example 2**  
Input: stamp = "abca", target = "aabcaca"  
Output: **[3,0,1]**  
Sequence:  
"???????" → place at 3 → "???abca" → place at 0 → "abcabca" → place at 1 → "aabcaca"

**Constraints**:
- 1 ≤ stamp.length ≤ target.length ≤ 1000
- stamp and target consist of lowercase English letters

---

## Core Idea — Greedy Reverse Simulation

**Beautiful observation**:
- Forward stamping is hard to plan (many choices, overlapping)
- But **reverse** process is easier:
  - Start from target
  - Repeatedly **find** a position where the current string matches stamp exactly (no '?')
  - Replace that segment with '?' (undo the stamp)
  - Record the position (but in reverse order)
- The last stamp applied is the first one we undo
- We need to reduce target to all '?' using exactly the stamps we applied

**Greedy strategy**:
- At each step, find **any** position where the current string matches stamp exactly
- Replace it with '?' and record the position
- Repeat until all '?' or no more possible
- If we reach all '?' → success → reverse the order of positions
- If stuck with non-'?' and cannot stamp → impossible

**Key optimization**:
- To avoid TLE, we don't check every position every time
- But since n ≤ 1000, even O(n²) is acceptable (1000² = 10⁶)

**Time**: O(n²) — in worst case, we scan the whole string each time we try to find a match  
**Space**: O(n) — working string

## Clean Implementation (C#)

```csharp
public class Solution {
    public int[] MovesToStamp(string stamp, string target) {
        int n = target.Length;
        int m = stamp.Length;
        char[] s = target.ToCharArray();
        List<int> ans = new List<int>();
        int stars = n;
        
        // Repeat until all positions are stamped (turned to '?')
        while (stars > 0) {
            bool stamped = false;
            
            // Try to find a position to stamp
            for (int i = 0; i <= n - m; i++) {
                bool canStamp = true;
                int nonStarCount = 0;
                
                // Check if we can stamp at position i
                for (int j = 0; j < m; j++) {
                    if (s[i + j] != '?' && s[i + j] != stamp[j]) {
                        canStamp = false;
                        break;
                    }
                    if (s[i + j] != '?') nonStarCount++;
                }
                
                // Only stamp if it covers at least one non-'?' character
                if (canStamp && nonStarCount > 0) {
                    // Apply stamp (turn to '?')
                    for (int j = 0; j < m; j++) {
                        s[i + j] = '?';
                    }
                    ans.Add(i);
                    stars -= nonStarCount;
                    stamped = true;
                    break;
                }
            }
            
            // If we couldn't stamp anything but still have non-'?' → impossible
            if (!stamped) {
                return new int[0];
            }
        }
        
        // Reverse the order (last stamped was first undone)
        ans.Reverse();
        return ans.ToArray();
    }
}
```

## Complexity

| **Metric**            | **Value**     | **Notes**                                      |
|-----------------------|---------------|------------------------------------------------|
| **Time Complexity**   | **O(n²)**     | Up to n steps (each step removes at least 1 non-'?' character)<br>Each step scans up to (n - m + 1) positions, and each position check takes O(m) time → worst-case O(n × n) = O(n²) |
| **Space Complexity**  | **O(n)**      | Working character array: O(n)<br>List to store answer indices: O(n) |

**Optimal** — quadratic time is perfectly acceptable for n ≤ 1000 (maximum ~10⁶ operations, runs very fast in practice).

## Why This Works — Example Walkthrough

**Example 1**: stamp = "abc", target = "ababc"

**Initial state**: s = "ababc"

**Reverse simulation** (working backwards from target to "?????"):

1. Find exact match of stamp "abc" at position 2 → "ab**abc**"  
   → Replace with "???" → s becomes "ab???"  
   → Record position **2**

2. Find exact match of stamp "abc" at position 0 → "**ab?**??" (the '?' are ignored during matching)  
   → Replace with "???" → s becomes "???"  
   → Record position **0**

**Reverse the recorded order** (because we simulated backwards): [0, 2] → correct

**Correct** —  
Working **backwards** (from target to all '?') is much easier than building forward.  
At each step, we greedily erase (stamp over) any position where the current string **exactly matches** stamp (ignoring already stamped '?' positions).  
We only perform a stamp if it covers **at least one non-'?'** character to ensure progress.  
The order in which we erase matches is the **reverse** of the forward stamping sequence — so we reverse the list at the end.

---

## Key Takeaway

This is a **clever reverse greedy simulation** problem:

* Instead of trying to build target from '?' (hard to decide order), simulate **undoing** stamps from target back to all '?'
* At each step: scan for any position where the current string **exactly matches** stamp (treating '?' as wildcard)
* If found → replace that segment with '?' and record the starting index
* Continue until the string is all '?' or no more stamps possible
* If successful → **reverse** the recorded indices → this is a valid forward stamping sequence
* If stuck with non-'?' characters and cannot stamp → impossible → return empty array

**Pure, clean, optimal** — O(n²) time, O(n) space, very fast for n ≤ 1000, elegant reverse thinking, and the greedy choice usually produces a near-minimal number of stamps (well within 10×n limit).
