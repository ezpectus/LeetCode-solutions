# 3093. Longest Common Suffix Queries
## O((N + Q) · L) — Reversed Trie with Precomputed Best Index per Node

---

## Problem Statement

- Given `wordsContainer` (N strings) and `wordsQuery` (Q strings).
- For each query, find the index of the string in `wordsContainer` with the **longest common suffix**.
- Tiebreak: shortest length → earliest index in `wordsContainer`.
- Return array of indices.

---

## Core Idea — Suffix Trie (Reversed Strings)

Key insight: Longest common **suffix** = longest common **prefix** of reversed strings. Build a trie on reversed `wordsContainer` words. At each trie node, precompute the best candidate index among all words that pass through that node.

This way, each query just traverses the trie as far as possible (while the path exists) and reads the precomputed answer at the deepest reachable node — no need to scan all container words per query.

Trie node stores:
- `children[26]` — next character links
- `bestIdx` — index of the best `wordsContainer` word passing through this node (shortest length, earliest in case of tie)

Build phase: Insert each reversed `wordsContainer[i]` into the trie, updating `bestIdx` at every node along the path.

Query phase: Walk the reversed query string down the trie as far as possible, return `bestIdx` at the last reachable node.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    private class TrieNode {
        public int[] children = new int[26];
        public int bestIdx = -1;

        public TrieNode() {
            Array.Fill(children, -1);
        }
    }

    public int[] StringIndices(string[] wordsContainer, string[] wordsQuery) {
        var trie = new List<TrieNode>();
        trie.Add(new TrieNode()); // root

        // Helper: compare which index is better (shorter length, earlier index)
        bool IsBetter(int newIdx, int curIdx) {
            if (curIdx == -1) return true;
            int newLen = wordsContainer[newIdx].Length;
            int curLen = wordsContainer[curIdx].Length;
            return newLen < curLen || (newLen == curLen && newIdx < curIdx);
        }

        // Build trie on reversed wordsContainer
        for (int i = 0; i < wordsContainer.Length; i++) {
            string word = wordsContainer[i];
            int node = 0;

            // Update root's bestIdx
            if (IsBetter(i, trie[node].bestIdx))
                trie[node].bestIdx = i;

            // Insert reversed word character by character
            for (int j = word.Length - 1; j >= 0; j--) {
                int c = word[j] - 'a';

                if (trie[node].children[c] == -1) {
                    trie[node].children[c] = trie.Count;
                    trie.Add(new TrieNode());
                }

                node = trie[node].children[c];

                if (IsBetter(i, trie[node].bestIdx))
                    trie[node].bestIdx = i;
            }
        }

        // Answer each query
        int[] res = new int[wordsQuery.Length];

        for (int i = 0; i < wordsQuery.Length; i++) {
            string query = wordsQuery[i];
            int node = 0;

            for (int j = query.Length - 1; j >= 0; j--) {
                int c = query[j] - 'a';

                if (trie[node].children[c] == -1) break;

                node = trie[node].children[c];
            }

            res[i] = trie[node].bestIdx;
        }

        return res;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(N·L + Q·L) | N = container size, Q = query size, L = max word length |
| Space Complexity | O(N·L·26) | Trie nodes × alphabet size |

Optimal — each character processed once during insert, once during query.

## Why This Works — Example Walkthrough

**Example 1:** `wordsContainer = ["abcd","bcd","xbcd"]`, `wordsQuery = ["cd","bcd","xyz"]`

Reversed container: `["dcba", "dcb", "dcbx"]`

Build trie (inserting reversed words):
- Root `bestIdx = 1` (shortest is "bcd" at length 3)
- After 'd': node bestIdx = 1
- After 'd'→'c': node bestIdx = 1
- After 'd'→'c'→'b': node bestIdx = 1 (all three pass here; "bcd" is shortest)
- After 'd'→'c'→'b'→'a': only "abcd" → bestIdx = 0
- After 'd'→'c'→'b'→'x': only "xbcd" → bestIdx = 2

Query `"cd"` reversed = `"dc"`:
- Root → 'd' → 'c', stops here → bestIdx = 1 ✓

Query `"bcd"` reversed = `"dcb"`:
- Root → 'd' → 'c' → 'b', stops here → bestIdx = 1 ✓

Query `"xyz"` reversed = `"zyx"`:
- Root → 'z' not found, stays at root → bestIdx = 1 ✓

**Output:** [1, 1, 1] ✓

---

**Example 2:** `wordsContainer = ["abcdefgh","poiuygh","ghghgh"]`

Reversed: `["hgfedcba", "hgyuiop", "hghghg"]`

Lengths: 8, 7, 6 → root bestIdx = 2 (shortest = "ghghgh")

Query `"gh"` reversed = `"hg"`:
- 'h' → 'g', both exist, walk stops → bestIdx = 2 ✓

Query `"acbfgh"` reversed = `"hgfbca"`:
- 'h'→'g'→'f' — only "abcdefgh" has 'f' here → bestIdx = 0 ✓

Query `"acbfegh"` reversed = `"hgefbca"`:
- 'h'→'g', then 'e' not found (only 'f' and 'g' branches) → stays at 'hg' node → bestIdx = 2 ✓

**Output:** [2, 0, 2] ✓

## Pitfalls & Edge Cases

- **Root always has a valid bestIdx** — Since every `wordsContainer` word inserts through the root, root always holds the globally best candidate. This handles queries with zero common suffix (no trie path matches).

- **Update bestIdx on every node, not just leaves** — A word passing through a node means that node's suffix is a suffix of that word. All nodes on the path must track the best candidate.

- **Tiebreak order matters** — Update only when strictly better (shorter, then earlier index). Do not overwrite with equal-length later-indexed words.

- **List-based trie vs array** — Using `List<TrieNode>` with integer child indices avoids recursive class references and is more cache-friendly than pointer-based tries.

- **Reversed traversal** — Insert and query both traverse from the last character backward. Common suffix in original = common prefix in reversed.

---

## Key Takeaway

"Longest common suffix" problems → **reverse strings and use a prefix Trie**.

The key optimization here is **precomputing the best answer at every trie node** during the build phase. Without this, each query would need to scan all container words that share a suffix — defeating the purpose of the trie. With precomputation, each query is answered in O(L) by walking down and reading the stored `bestIdx`.

**Pattern recognition:**
- "Longest common prefix/suffix among many strings" → Trie
- "Best candidate among all strings sharing a prefix/suffix" → Store best candidate at each trie node during insert
