# 2276. Count Integers in Intervals
## O(n log n) amortized — Sorted Set of Merged Intervals

---

## Problem Statement

- Maintain a set of intervals. Support two operations:
  - `add(left, right)`: add interval `[left, right]`.
  - `count()`: return number of distinct integers covered by at least one interval.

---

## Core Idea — Merge Overlapping Intervals, Track Total Coverage

Keep intervals in a **sorted set** ordered by left endpoint. Maintain a running `count` of total covered integers.

**`add(left, right)`**:
1. Find all existing intervals that **overlap or touch** the new interval `[left, right]`. An interval `[l, r]` overlaps/touches if `l ≤ right+1` AND `r ≥ left-1`.
2. Merge all overlapping intervals into one: `new_left = min(left, all l's)`, `new_right = max(right, all r's)`.
3. Subtract their individual lengths from `count`, remove them from the set.
4. Insert the merged interval, add its length to `count`.

**`count()`**: O(1), return the running total.

**Why amortized O(n log n)?** Each interval is inserted once and removed at most once. Total removals across all `add` calls = total insertions = at most `n`. Each removal is O(log n). So total time is O(n log n).

---

## Full Optimal Implementation (C#)

```csharp
public class CountIntervals {
    SortedList<int, int> ivals = new(); // left -> right, no overlaps maintained
    int total = 0;

    public void Add(int left, int right) {
        int l = left, r = right;

        // Find index of first interval with left > right+1 (cannot overlap)
        int hi = UpperBound(right + 1);

        // Scan backwards from hi-1, merging all overlapping/touching intervals
        int i = hi - 1;
        while (i >= 0 && ivals.Values[i] >= left - 1) {
            l = Math.Min(l, ivals.Keys[i]);
            r = Math.Max(r, ivals.Values[i]);
            total -= ivals.Values[i] - ivals.Keys[i] + 1;
            ivals.RemoveAt(i);
            i--;
        }

        ivals[l] = r;
        total += r - l + 1;
    }

    int UpperBound(int val) {
        int lo = 0, hi = ivals.Count;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (ivals.Keys[mid] <= val) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }

    public int Count() => total;
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Add** | **O(n log n) amortized** | Each interval inserted/removed once; n total operations |
| **Count** | **O(1)** | Running total maintained |
| **Space** | **O(n)** | At most n non-overlapping intervals stored |

## Example Walkthrough

**Operations**: add(2,3), add(7,10), count, add(5,8), count.

After add(2,3): intervals={(2,3)}, count=2.

After add(7,10): no overlap with (2,3). intervals={(2,3),(7,10)}, count=6.

count() → **6** ✓

add(5,8): overlaps (7,10) since 5 ≤ 10+1 and 10 ≥ 5-1. Also (2,3): 2 ≤ 8+1 but 3 < 5-1=4 → no overlap.
- Only (7,10) merges: newL=min(5,7)=5, newR=max(8,10)=10. Remove (7,10), count-=4. Add (5,10), count+=6. count=2+6=8.
- intervals={(2,3),(5,10)}, count=8.

count() → **8** ✓

---

## Pitfalls & Edge Cases

- **Touching intervals** (`left-1` / `right+1`): `[2,3]` and `[4,5]` should merge since they cover all integers 2-5. Use `r >= left-1` (not `r >= left`) and `l <= right+1` for touch detection.
- **Running count update**: subtract old intervals' lengths BEFORE removing, add merged interval's length AFTER inserting — always keep `count` consistent.
- **`count` field name collision in C#**: rename to `_count` or `total` to avoid conflict with `Count()` method name.

---

## Key Takeaway

"Online interval union with integer count" → **sorted map of merged intervals + running sum**:

```
on add(l,r): find all overlapping intervals, merge into one, update count
on count(): return running total O(1)
```

Amortized O(n log n) because each interval is added and removed at most once. The key invariant: no two intervals in the set overlap or touch at any time.
