# 2267. Check if There Is a Valid Parentheses String Path
## O(m × n × (m+n)) — 3D DP on (row, col, balance)

---

## Problem Statement

- Grid of `(` and `)`. Move only right or down from (0,0) to (m-1,n-1).
- Return true if any path forms a valid parentheses string.

---

## Core Idea — DP Tracking Balance at Each Cell

A valid parentheses string has balance (open minus close) always ≥ 0 and ends at 0.

**State**: `dp[r][c][bal]` = can we reach `(r,c)` with balance `bal`?

**Transition**: to reach `(r,c)` with balance `bal`, we must come from `(r-1,c)` or `(r,c-1)` with balance `bal - delta`, where `delta = +1` for `(` and `-1` for `)`.

**Key observations**:
- Maximum useful balance = `(m+n)/2` (path length = m+n-1, at most half can be `(`).
- Any balance > `(m+n)/2` is useless (can't close enough before the end).
- Early pruning: if `grid[0][0] == ')'` or `grid[m-1][n-1] == '('` → immediately false.

**Answer**: `dp[m-1][n-1][0]`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool HasValidPath(char[][] grid) {
        int m = grid.Length, n = grid[0].Length;
        if (grid[0][0] == ')' || grid[m-1][n-1] == '(') return false;

        int maxBal = (m + n) / 2;
        bool[,,] dp = new bool[m, n, maxBal + 1];
        dp[0, 0, 1] = true; // grid[0][0] must be '(' (checked above)

        for (int r = 0; r < m; r++)
        for (int c = 0; c < n; c++) {
            if (r == 0 && c == 0) continue;
            int delta = grid[r][c] == '(' ? 1 : -1;
            for (int bal = 0; bal <= maxBal; bal++) {
                int prev = bal - delta;
                if (prev < 0 || prev > maxBal) continue;
                if (r > 0 && dp[r-1, c, prev]) dp[r, c, bal] = true;
                if (c > 0 && dp[r, c-1, prev]) dp[r, c, bal] = true;
            }
        }

        return dp[m-1, n-1, 0];
    }
}
```

Actual valid path gives "(((())))" balanced. dp[3][2][0] = True → **true** ✓

---

## Pitfalls & Edge Cases

- **`maxBal = (m+n)/2`**: path length = m+n-1, at most `(m+n-1)/2` opening brackets → balance never exceeds this.
- **`prev = bal - delta`**: going backwards — if current char is `(` (delta=+1), previous balance was `bal-1`. If `)` (delta=-1), previous was `bal+1`.
- **`grid[m-1][n-1] == '('`**: last cell must be `)` for balance to reach 0. Early return false.
- **Path length parity**: valid string requires even length, so m+n-1 must be even → m+n must be odd. Otherwise return false immediately (though the DP handles this naturally via dp[m-1][n-1][0] being unreachable).

---

## Key Takeaway

"Valid parentheses path in grid" → **3D DP: state = (row, col, current balance). Prune balances > (m+n)/2. Answer = dp[m-1][n-1][0]**.
