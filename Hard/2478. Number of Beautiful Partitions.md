# 2478. Number of Beautiful Partitions
## O(n · k) — DP with Prefix Sum Optimization

---

## Problem Statement

- Partition string `s` into exactly `k` parts where each part:
  1. Has length ≥ `minLength`
  2. Starts with a prime digit ('2','3','5','7')
  3. Ends with a non-prime digit ('1','4','6','8','9')
- Return count mod 10⁹ + 7.

---

## Core Idea — Rolling DP with Prefix Sums

Define `dp[i]` = number of ways to partition `s[0..i-1]` into exactly `j` valid parts.

**Transition:** For a new part ending at position `i-1`:
- `s[i-1]` must be non-prime (end condition).
- Part starts at some position `m` where `s[m]` is prime and `i - m ≥ minLength`.
- `dp_new[i] = Σ dp_prev[m]` for all valid m.

**Naïve:** O(n²·k) — for each (i, j), scan all m.

**Optimized with prefix sums:** Precompute `prefSum[i] = Σ dp_prev[m]` for m in `[0, i-1]` where `s[m]` is prime. Then each transition is O(1).

**Total:** O(n·k) with rolling arrays.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int BeautifulPartitions(string s, int k, int minLength) {
        const int MOD = 1_000_000_007;
        int n = s.Length;

        bool IsPrime(char c) => c is '2' or '3' or '5' or '7';

        // Quick pruning
        if (!IsPrime(s[0]) || IsPrime(s[n - 1])) return 0;
        if ((long)k * minLength > n) return 0;

        // dp[i] = ways to partition s[0..i-1] into the current number of parts
        long[] dp = new long[n + 1];
        dp[0] = 1; // base: 0 parts, 0 characters processed

        for (int j = 0; j < k; j++) {
            // Build prefix sums: prefSum[i] = Σ dp[m] for m in [0, i-1] where s[m] is prime
            long[] prefSum = new long[n + 2];
            for (int m = 0; m < n; m++) {
                prefSum[m + 1] = (prefSum[m] + (IsPrime(s[m]) ? dp[m] : 0)) % MOD;
            }

            long[] newDp = new long[n + 1];
            for (int i = 1; i <= n; i++) {
                if (IsPrime(s[i - 1])) continue; // end must be non-prime

                int limit = i - minLength; // start position m ≤ limit
                if (limit < 0) continue;

                newDp[i] = prefSum[limit + 1]; // sum of dp[m] for m in [0, limit], s[m] prime
            }

            dp = newDp;
        }

        return (int)dp[n];
    }
}
```

## Example Walkthrough

**Example 1:** `s = "23542185131"`, `k = 3`, `minLength = 2`

s-indices: 0(2),1(3),2(5),3(4),4(2),5(1),6(8),7(5),8(1),9(3),10(1). Prime at: 0,1,2,4,7,9. Non-prime at: 3,5,6,8,10.

**After j=0 (1st part):** Starts at m=0 (only prime dp[0]=1). Valid ends: positions i where s[i-1] non-prime and i≥2.
- dp[4]=1 ("2354"), dp[6]=1 ("235421"), dp[7]=1 ("2354218"), dp[9]=1 ("235421851"), dp[11]=1.

**After j=1 (2nd part):** Starting positions with dp>0 and prime: m=4 (dp[4]=1,'2'), m=7 (dp[7]=1,'5').
- dp[6]=1 (part "21", dp[4]), dp[7]=1 (part "218", dp[4]), dp[9]=2 (part "21851" from m=4, or "51" from m=7), dp[11]=3.

**After j=2 (3rd part):** Starting positions: m=7 (dp[7]=1,'5'), m=9 (dp[9]=2,'3').
- dp[9]: i=9, limit=7. prefSum covers m=7 only → dp[9]=1.
- dp[11]: i=11, limit=9. prefSum[10] = dp[7]+dp[9] = 1+2 = 3 → dp[11]=**3**.

**Output:** 3 ✓

## Pitfalls & Edge Cases

- **Quick pruning:** s[0] must be prime (first part must start there) and s[n-1] must be non-prime (last part must end there). Otherwise return 0 immediately. ✓

- **Single-character parts:** A part needs `s[m]` prime (start) AND `s[i-1]` non-prime (end). A single character can't be both, so single-char parts are always invalid regardless of `minLength`. ✓

- **prefSum includes only prime-starting positions:** Non-prime `s[m]` values contribute 0 to the prefix sum, so they're automatically excluded from valid start positions. ✓

- **Rolling array:** After each of the k iterations, `dp` represents the count for exactly `j` parts. Only O(n) space needed. ✓

---

## Key Takeaway

Classic **DP with prefix sum optimization** for "partition into k groups with window constraints":
- State: `dp[i][j]` = ways to use first i chars in j parts.
- Transition: sum over valid starting positions → prefix sum on valid (prime) positions.
- Rolling array reduces space from O(n·k) to O(n).

**Pattern recognition:** Any "partition into exactly k groups, each with start/end conditions, minimum length" → DP with two conditions (end condition on current cell, start condition on prefix sum) where the prefix sum only aggregates over cells satisfying the start condition.
