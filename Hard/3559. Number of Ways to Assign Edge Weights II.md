# 3559. Number of Ways to Assign Edge Weights II
## O((n + q) log n) — Path Length via LCA + 2^(d-1) Formula

---

## Problem Statement

- Tree with n nodes. Each edge gets weight 1 or 2.
- For query (u, v): count assignments where path cost is odd, mod 10⁹+7.

---

## Core Insight — Count Path Edges, Not Individual Assignments

Weights 1 (odd) and 2 (even). For the path sum to be odd, we need an **odd number of weight-1 edges**.

For a path with **d edges**, the total assignments = 2^d. By symmetry (swapping all 1↔2 maps every odd-sum assignment to an even-sum one), exactly half are odd. So:

- d = 0: answer = **0** (sum = 0, always even).
- d > 0: answer = **2^(d-1) mod (10⁹+7)**.

**Problem reduces to:** for each query (u, v), find d = number of edges on the u→v path.

Using LCA: `d = depth[u] + depth[v] - 2 × depth[LCA(u, v)]`.

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    static const int MOD = 1e9 + 7;
    static const int LOG = 17; // 2^17 > 10^5

    vector<int> assignEdgeWeights(vector<vector<int>>& edges,
                                   vector<vector<int>>& queries) {
        int n = (int)edges.size() + 1;

        vector<vector<int>> adj(n + 1);
        for (auto& e : edges) {
            adj[e[0]].push_back(e[1]);
            adj[e[1]].push_back(e[0]);
        }

        vector<int> depth(n + 1, 0);
        vector<array<int, 17>> par(n + 1);
        for (auto& a : par) a.fill(0);

        // BFS for depth + binary lifting table
        vector<bool> visited(n + 1, false);
        queue<int> bfsQ;
        bfsQ.push(1);
        visited[1] = true;
        while (!bfsQ.empty()) {
            int u = bfsQ.front(); bfsQ.pop();
            for (int i = 1; i < LOG; i++)
                par[u][i] = par[par[u][i-1]][i-1];
            for (int v : adj[u]) {
                if (!visited[v]) {
                    visited[v] = true;
                    depth[v] = depth[u] + 1;
                    par[v][0] = u;
                    bfsQ.push(v);
                }
            }
        }

        // LCA via binary lifting
        auto lca = [&](int u, int v) {
            if (depth[u] < depth[v]) swap(u, v);
            int diff = depth[u] - depth[v];
            for (int i = 0; i < LOG; i++)
                if ((diff >> i) & 1) u = par[u][i];
            if (u == v) return u;
            for (int i = LOG - 1; i >= 0; i--)
                if (par[u][i] != par[v][i]) {
                    u = par[u][i];
                    v = par[v][i];
                }
            return par[u][0];
        };

        // Precompute powers of 2
        vector<long long> pow2(n + 1);
        pow2[0] = 1;
        for (int i = 1; i <= n; i++)
            pow2[i] = pow2[i-1] * 2 % MOD;

        vector<int> ans;
        ans.reserve(queries.size());
        for (auto& q : queries) {
            int u = q[0], v = q[1];
            int l = lca(u, v);
            int d = depth[u] + depth[v] - 2 * depth[l];
            ans.push_back(d == 0 ? 0 : (int)pow2[d - 1]);
        }

        return ans;
    }
};
```

## Why 2^(d-1) Is Always Correct

For d edges, each takes value 1 or 2. Total = 2^d assignments. Map each assignment to its "complement" (swap all 1↔2). This flips all edge weights: 1→2 (odd→even) and 2→1 (even→odd). So the total parity flips. This is a bijection between odd-sum and even-sum assignments. Therefore, exactly 2^d / 2 = **2^(d-1)** assignments are odd. ✓

## Example Walkthrough

**Example 2:** `edges = [[1,2],[1,3],[3,4],[3,5]]`

Tree (root=1): depths: 1→0, 2→1, 3→1, 4→2, 5→2.

| Query | u | v | LCA | d = dep[u]+dep[v]-2×dep[LCA] | Answer = 2^(d-1) |
|-------|---|---|-----|------------------------------|-----------------|
| [1,4] | 1 | 4 | 1   | 0+2-0=2                      | 2^1=**2** ✓     |
| [3,4] | 3 | 4 | 3   | 1+2-2=1                      | 2^0=**1** ✓     |
| [2,5] | 2 | 5 | 1   | 1+2-0=3                      | 2^2=**4** ✓     |

---

## Pitfalls & Edge Cases

- **u == v:** LCA = u = v, d = 0 → answer = 0 (sum is 0, even). ✓
- **Root's parent:** `par[1][0] = 0` (node 0 = sentinel). All `par[0][i] = 0` by fill, so lifting past the root is stable. ✓
- **BFS order ensures parent rows are filled before children use them** — ancestors processed before descendants in BFS, so binary lifting is computed correctly. ✓

---

## Key Takeaway

"Count weighted path assignments mod p" → reduce to path length via LCA, then apply combinatorial formula. The symmetry argument (bijection between odd/even assignments via complement) gives the 2^(d-1) formula without computing any actual assignments.

**Pattern recognition:** Whenever edge weights are {1, 2} or {odd, even} and you need parity counts, the answer is always 2^(path_length - 1) (half the space) — confirmed by the complement bijection.
