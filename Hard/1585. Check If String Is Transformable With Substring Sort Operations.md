# 1585. Check If String Is Transformable With Substring Sort Operations
## O(n × 10) — Greedy with Per-Digit Position Queues

---

## Problem Statement

- Transform `s` into `t` by repeatedly sorting any substring in place.
- Return true if `t` is achievable from `s`.

---

## Core Idea — Sorting Substrings = Selective Bubble Sort

**Key insight**: sorting a substring is equivalent to performing a sequence of adjacent swaps. Specifically, a digit `d` can move **left** past another digit `e` only if `d < e` (we can sort a substring containing both, putting `d` before `e`). A digit can **never** move left past a smaller digit.

This means: we can only move digits leftward past larger ones, never past smaller ones. This is exactly bubble sort behavior.

**Greedy algorithm**:
For each character `t[i]` from left to right:
1. Find the **leftmost** remaining occurrence of `t[i]` in `s` (use per-digit queues)
2. Check: does any digit **smaller than `t[i]`** appear **before** that position in `s`? If yes → impossible (we can't move `t[i]` past that smaller digit)
3. If no smaller digit blocks the path → "consume" that position (pop from queue)

**Why leftmost occurrence?** Using a later occurrence would leave smaller occurrences of `t[i]` that might block future placements.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool IsTransformable(string s, string t) {
        var pos = new Queue<int>[10];
        for (int d = 0; d < 10; d++) pos[d] = new Queue<int>();
        for (int i = 0; i < s.Length; i++) pos[s[i]-'0'].Enqueue(i);

        foreach (char c in t) {
            int d = c - '0';
            if (pos[d].Count == 0) return false;

            int idx = pos[d].Peek(); // leftmost position of digit d

            // Check: no smaller digit appears before idx
            for (int smaller = 0; smaller < d; smaller++)
                if (pos[smaller].Count > 0 && pos[smaller].Peek() < idx)
                    return false;

            pos[d].Dequeue();
        }

        return true;
    }
}
```

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × 10)** = O(n) | For each of n chars in t, check 10 digit queues |
| **Space** | **O(n)** | Position queues total n entries |

---

## Why This Works — The Blocking Condition

A digit `d` at position `idx` in `s` can be moved to the current "front" of processing iff no digit smaller than `d` exists to its left in the remaining string.

If a smaller digit `e < d` is at position `j < idx`, we'd need `d` to pass `e` going left — but sorting any substring containing both `e` and `d` in order would place `e` before `d` (since `e < d`), blocking `d`'s leftward movement.

**Once we process position `idx`, it's "consumed"** — only future positions matter for subsequent checks.

## Pitfalls & Edge Cases

- **Use queues not sets**: we must use occurrences in order (leftmost first), so a deque/queue per digit is essential
- **Check strictly smaller** (not smaller-or-equal): equal digits are interchangeable and don't block each other
- **`pos[d].Count == 0`**: digit `d` doesn't appear in remaining `s` → immediately false (character counts must match)

---

## Key Takeaway

"Sort substrings to transform s to t" → **greedy: for each t[i], find leftmost occurrence in s; blocked if any smaller digit precedes it**:

```
for each t[i] = d:
    idx = pos[d].front
    if any e < d with pos[e].front < idx: return false
    pos[d].dequeue()
```

Sorting substrings = bubble sort = can only move digits left past larger ones. Per-digit queues + O(10) check per character gives O(n) total.
