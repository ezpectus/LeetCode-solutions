# 2973. Find Number of Coins to Place in Tree Nodes

## O(N) — Post-Order DFS with Subtree Top-K Elements Merging

### Problem Statement
You are given an undirected tree with `n` nodes labeled from `0` to `n - 1`, rooted at node `0`. You are given a 2D integer array `edges` representing a valid tree structure and a 0-indexed integer array `cost` representing the cost of each node.

You need to place some coins on every node calculated as follows:
* If the size of the subtree of node `i` is less than 3, place `1` coin.
* Otherwise, place an amount of coins equal to the **maximum product of 3 distinct nodes** in the subtree of node `i`. If this product is negative, place `0` coins.

Return an array denoting the number of coins placed at each node.

---

### Core Idea — Subtree Local States and Optimal Triplet Selections
The core sub-problem is finding the maximum product of three numbers in a dynamically changing collection. In an array containing both positive and negative integers, the maximum product of 3 elements can only come from two combinations:
1. The product of the **3 largest elements** (all positive, or mixed).
2. The product of the **2 smallest elements** (potentially highly negative numbers whose product becomes positive) multiplied by the **largest element**.

Thus, to fully represent any subtree's state, a parent node does not need to collect all costs from its descendants. It only needs to track at most **3 maximums** and **2 minimums** from across its entire sub-hierarchy.

#### DFS Bottom-Up Merging Protocol
We execute a bottom-up Post-Order DFS traversal starting from the root (node `0`):
1. **Base Leaf State:** Every leaf node creates a list containing its own `cost[u]`, sets its subtree size to 1, and assigns 1 coin to itself.
2. **Post-Order Accumulation:** A parent node waits for its children to return their tracked state lists. It pools all of their maximums and minimums together with its own `cost[parent]`.
3. **State Compression:** The parent sorts the accumulated elements. To prevent memory scaling, it trims the pool down to keep only the top 3 maximums and bottom 2 minimums before passing the state further up the recursion stack.
4. **Coin Allocation:** If the combined subtree size at the current node is $\ge 3$, we evaluate the two candidate products using our trimmed list. If the result is negative, we drop it to `0`.

### Full Optimal Implementation (C++)

```cpp
#include <vector>
#include <algorithm>
#include <cmath>

using namespace std;

class Solution {
private:
    struct SubtreeState {
        vector<long long> values;
        int size;
    };

    vector<long long> ans;
    vector<vector<int>> adj;

    SubtreeState dfs(int u, int p, const vector<int>& cost) {
        SubtreeState currentState;
        currentState.size = 1;
        currentState.values.push_back(cost[u]);

        // Step 1: Recurse into children and pool tracking states
        for (int v : adj[u]) {
            if (v == p) continue;
            
            SubtreeState childState = dfs(v, u, cost);
            currentState.size += childState.size;
            
            for (long long val : childState.values) {
                currentState.values.push_back(val);
            }
        }

        // Step 2: Sort and compress the list to retain only top-3 and bottom-2 elements
        sort(currentState.values.begin(), currentState.values.end());
        
        if (currentState.values.size() > 5) {
            vector<long long> trimmed;
            int sz = currentState.values.size();
            // Keep 2 smallest elements
            trimmed.push_back(currentState.values[0]);
            trimmed.push_back(currentState.values[1]);
            // Keep 3 largest elements
            trimmed.push_back(currentState.values[sz - 3]);
            trimmed.push_back(currentState.values[sz - 2]);
            trimmed.push_back(currentState.values[sz - 1]);
            currentState.values = move(trimmed);
        }

        // Step 3: Evaluate coin allocation boundaries
        if (currentState.size < 3) {
            ans[u] = 1;
        } else {
            int sz = currentState.values.size();
            long long product1 = currentState.values[sz - 1] * currentState.values[sz - 2] * currentState.values[sz - 3];
            long long product2 = currentState.values[0] * currentState.values[1] * currentState.values[sz - 1];
            
            long long maxProduct = max(product1, product2);
            ans[u] = maxProduct < 0 ? 0 : maxProduct;
        }

        return currentState;
    }

public:
    vector<long long> placedCoins(vector<vector<int>>& edges, vector<int>& cost) {
        int n = cost.size();
        ans.resize(n, 0);
        adj.resize(n);

        for (const auto& edge : edges) {
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]);
        }

        dfs(0, -1, cost);
        return ans;
    }
};
```

### Complexity

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | $O(N)$ | Each node is visited exactly once via DFS. At each step, a node merges lists of size at most 5 from its children. Sorting a list of max size $5 \times \text{deg}(u)$ takes constant time $O(1)$ relative to $N$. Thus, overall execution scales linearly. |
| **Space Complexity** | $O(N)$ | Driven by the recursion stack depth during skewed tree traversals and the graph representation adjacency vectors. |

---

### Pitfalls & Edge Cases

* **64-bit Integer Overflow:** The product of three node costs can reach up to $10^4 \times 10^4 \times 10^4 = 10^{12}$, which easily overflows standard 32-bit signed integers. Ensure all multiplication variables and tracking vectors utilize `long long` data types.
* **Insufficient Unique Nodes:** The problem states that the 3 nodes picked within a subtree must be distinct. Our bottom-up merge naturally keeps individual nodes distinct because we pass up actual node values collected via the tree topology.
* **Negative Product Drop:** If the optimal triplet multiplication yields a negative number (e.g., all values in subtree are negative), the problem strictly expects `0` coins to be placed instead of the negative amount.

---

### Key Takeaway

> When calculating sliding or sub-structural aggregates that depend on maximum products or extreme offsets, you can avoid transferring whole collections up a tree hierarchy. Deduce the exact number of boundary states required to compute that condition globally (like top-3 and bottom-2) and maintain only that micro-state during node merging.

* **Pattern recognition:** "Find maximum triplet product for every subtree in a graph" -> Bottom-Up DFS State Merging + Top-3 Max / Bottom-2 Min Tracking.
