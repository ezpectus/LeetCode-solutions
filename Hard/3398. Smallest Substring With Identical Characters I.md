# 3398. Smallest Substring With Identical Characters I
## O(n log n) — Binary Search on Max Run Length + Two-Case Check (len=1 Special, len≥2 Greedy)

---

## Problem Statement

Binary string s of length n, integer numOps. Flip at most numOps characters. Minimize the length of the longest run of identical characters.

---

## Core Insight — Binary Search on the Answer

The answer is monotone: if we can achieve max run ≤ len with numOps flips, we can certainly achieve max run ≤ len+1 (using the same or fewer flips). So binary search on len ∈ [1, n]:

> **check(len):** Can we make all runs of identical characters have length ≤ len using ≤ numOps flips?

Find the smallest len where check(len) = true.

---

## Check Function — Two Distinct Cases

### Case 1: len = 1 (Alternating String)

If len=1, every character must differ from its neighbors. The only binary strings with no run >1 are exactly the two alternating patterns:
- Pattern0 = "010101…"
- Pattern1 = "101010…"

Since these are bitwise complements, any position contributes to exactly one pattern's mismatch count. If `ops0` = positions where s[i] ≠ Pattern0[i], then ops1 = n − ops0.

**Check:** `min(ops0, n − ops0) ≤ numOps`

**Why the general run formula fails for len=1:** Flipping a character inside a '1' run creates a '0' that may be adjacent to a neighboring '0' run, creating a new "00" clash. Example: "011010" — the "11" run needs 1 flip, but any flip of "11" yields a '0' adjacent to an existing '0', creating a run of 2. The global alternating-pattern analysis is needed to account for boundary interactions.

### Case 2: len ≥ 2 (Per-Run Greedy)

For a run of k identical characters, to make all sub-runs ≤ len, place flips every (len+1) positions within the run:

```
aaa…a  (k copies)
flip at positions len, 2·(len+1)-1, 3·(len+1)-1, …
→ sub-runs of: len, 1, len, 1, …, ≤ len
```

Minimum flips for a run of k = **⌊k / (len+1)⌋**.

**Why this is safe for len≥2:** Flipped characters land deep inside the original run (≥2 positions from the boundary), so the newly created opposite-character runs don't merge with adjacent external runs.

**Check:** `Σ over all runs i of ⌊k_i / (len+1)⌋ ≤ numOps`

---

## Algorithm

```
lo = 1, hi = n
Binary search: smallest len where check(len) = true.
  if check(mid): hi = mid
  else:          lo = mid + 1
Return lo.
```

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int minLength(string s, int numOps) {
        int n = s.size();

        auto check = [&](int len) -> bool {
            if (len == 1) {
                // Count mismatches against "010101..." pattern
                int ops0 = 0;
                for (int i = 0; i < n; i++) {
                    char expected = '0' + (char)(i % 2);  // '0' or '1' alternating
                    if (s[i] != expected) ops0++;
                }
                return min(ops0, n - ops0) <= numOps;  // ops1 = n - ops0
            }

            // len >= 2: sum floor(k / (len+1)) over all runs
            int ops = 0, i = 0;
            while (i < n) {
                char c = s[i];
                int j = i;
                while (j < n && s[j] == c) j++;
                ops += (j - i) / (len + 1);
                i = j;
            }
            return ops <= numOps;
        };

        int lo = 1, hi = n;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (check(mid)) hi = mid;
            else            lo = mid + 1;
        }
        return lo;
    }
};
```

## Example Walkthroughs

### Example 1 — `s = "000001"`, `numOps = 1`, Expected: 2

Binary search on [1, 6]:

**check(3):** runs = '0'(5), '1'(1). ops = ⌊5/4⌋ + ⌊1/4⌋ = 1+0 = 1 ≤ 1. ✓ → hi=3.
**check(2):** runs = '0'(5), '1'(1). ops = ⌊5/3⌋ + ⌊1/3⌋ = 1+0 = 1 ≤ 1. ✓ → hi=2.
**check(1):** Pattern "010101": mismatches at pos 1 ('0'≠'1'), 3 ('0'≠'1') → ops0=2. ops1=4. min=2 > 1. ✗ → lo=2.

**Return 2** ✓

*Solution: flip s[2] to '1' → "001001". Longest runs: '0'(2). ✓*

---

### Example 2 — `s = "0000"`, `numOps = 2`, Expected: 1

**check(2):** runs = '0'(4). ops = ⌊4/3⌋ = 1 ≤ 2. ✓ → hi=2.
**check(1):** Pattern "0101": mismatches at pos 1 ('0'≠'1'), pos 3 ('0'≠'1') → ops0=2. ops1=2. min=2 ≤ 2. ✓ → hi=1.
lo=hi=1. **Return 1** ✓

*Solution: flip s[0] and s[2] → "1010". All runs length 1. ✓*

---

### Example 3 — `s = "0101"`, `numOps = 0`, Expected: 1

**check(2):** runs = '0','1','0','1' (all length 1). ops = 0 ≤ 0. ✓ → hi=2.
**check(1):** Pattern "0101": 0 mismatches (already matches). ops0=0. min=0 ≤ 0. ✓ → hi=1.
lo=hi=1. **Return 1** ✓

### Why the Formula Fails for len=1 — Concrete Trace

s = "011010". Runs: '0'(1), '1'(2), '0'(1), '1'(1), '0'(1). Formula gives: 0+⌊2/2⌋+0+0+0 = **1 flip**.

All 6 possible single flips:
| Flip pos | Result | Max run |
|---|---|---|
| 0 | "111010" | 3 ✗ |
| 1 | "001010" | 2 ✗ |
| 2 | "010010" | 2 ✗ |
| 3 | "011110" | 4 ✗ |
| 4 | "011000" | 3 ✗ |
| 5 | "011011" | 2 ✗ |

None achieve max run 1. Formula was **wrong** — it says 1 flip suffices, but 1 flip does NOT.

Alternating check: Pattern "010101" → 4 mismatches. Pattern "101010" → 6. min=4 flips needed. ✓

The issue: flipping inside the '11' run creates a '0' adjacent to the neighboring '0' run, forming "00". The per-run formula cannot detect cross-boundary clashes; the global alternating pattern check can.

---

### Verification of the len≥2 Formula

**s = "111000111", numOps = 3, len=2:**

Runs: '1'(3), '0'(3), '1'(3). ops = ⌊3/3⌋ × 3 = 1+1+1 = 3.

Strategy: flip middle of each run:
- "111" → "101" (flip position 1 within run)
- "000" → "010"
- "111" → "101"
- Result: "101 010 101" = "101010101". Max run = 1 ≤ 2. ✓

With only 2 flips: fix 2 of the 3 runs. The unfixed run still has length 3 > 2. ✗. So 3 is tight. ✓

## Pitfalls & Edge Cases

- **len=1 needs global treatment, NOT per-run:** The per-run formula systematically underestimates the required flips when runs of the same character are separated by single opposite characters (as shown above). Always use the alternating pattern count for len=1. ✓

- **ops1 = n − ops0 (complementary patterns):** Since pattern0 and pattern1 are bitwise complements, every position's s[i] matches exactly one pattern. Computing `ops0` alone suffices; `ops1 = n − ops0`. ✓

- **Binary search: lo=1, hi=n, find FIRST true:** The invariant is: answer ∈ [lo, hi]. If check(mid) = true, answer ≤ mid → hi=mid. Else answer > mid → lo=mid+1. Returns lo=hi = minimum achievable len. ✓

- **numOps = 0:** check(len) passes only for len ≥ current max run length. Binary search finds it correctly. ✓

- **numOps ≥ n:** We can always reach the alternating pattern (n/2 flips at most). check(1) returns true. Binary search returns 1. ✓

- **Single character (n=1):** Max run = 1. check(1) with the alternating formula: ops0=0 or 1, min ≤ numOps ≥ 0 = true. Returns 1. ✓

- **Already optimal string:** If s is already alternating, check(1) needs 0 flips → returns 1 immediately. ✓

---

## Key Takeaway

**"Minimize the maximum under a flip budget"** → binary search on the maximum + feasibility check. The feasibility check depends critically on the target:

- **Target=1 (alternating):** Only two valid global patterns; count mismatches to the closer one. O(n).
- **Target≥2 (no long runs):** Greedy per-run analysis: each run of k needs ⌊k/(len+1)⌋ independent flips. Runs don't interact for len≥2. O(n).

**Pattern recognition:** Binary search on "maximum allowed X" + O(n) check is the canonical approach for minimization problems where the feasibility is monotone. The two-case split (len=1 vs len≥2) reflects a structural phase transition: for len=1, runs interact at boundaries (global constraint); for len≥2, runs are decoupled (local greedy applies). Recognizing this transition is the key to getting the check function right.
