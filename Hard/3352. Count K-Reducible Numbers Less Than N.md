# LeetCode 3352. Count K-Reducible Numbers Less Than N

## Method Explanation

The problem asks us to count positive integers strictly less than a giant binary number $N$ (given as a string $s$ up to 800 characters) that can be reduced to 1 in at most $k$ operations. One operation replaces a number with its set bit count.

### 1. The Core Insight
While $N$ can be as large as $2^{800}$, after exactly **one** operation, any number $x < N$ collapses into an integer representing its set bit count. This count cannot exceed the length of the string, which is at most 800. 

Therefore, we can split the problem into two phases:
1. **Precompute Bit Reduction Steps:** For every integer $b \in [1, 800]$, compute how many operations it takes to reduce $b$ to 1. If it takes $\le k - 1$ operations, then any number $x < N$ with exactly $b$ set bits is valid (since the first operation costs 1 step to convert $x \to b$).
2. **Combinatorial Digit Counting:** Find how many positive integers $x < N$ have exactly $b$ set bits using combinatorics (combinations $\binom{n}{m}$).

### 2. Scanning Bounded Subsets
To find numbers strictly less than $N$, we iterate through the binary string $s$ from left to right. Whenever we encounter a `'1'` at index $i$, we can safely simulate placing a `'0'` at this position. This frees up the remaining positions to the right to be filled arbitrarily. If we need to place $R$ remaining set bits into $S$ available slots, the number of ways is given by the binomial coefficient $\binom{S}{R}$.

## C++ Source Code

```cpp
#include <vector>
#include <string>
#include <algorithm>

using namespace std;

class Solution {
public:
    int countKReducibleNumbers(string s, int k) {
        int n = s.length();
        const int MOD = 1e9 + 7;

        // Precompute binomial coefficients (Pascal's Triangle) up to N
        vector<vector<long long>> C(n + 1, vector<long long>(n + 1, 0));
        for (int i = 0; i <= n; ++i) {
            C[i][0] = 1;
            for (int j = 1; j <= i; ++j) {
                C[i][j] = (C[i - 1][j - 1] + C[i - 1][j]) % MOD;
            }
        }

        // Precompute the number of operations to reduce each bit-count in [1, n] to 1
        vector<int> steps(n + 1, 0);
        vector<bool> is_valid_bit_count(n + 1, false);
        
        // 1 is already reduced (0 operations), so steps[1] = 0.
        is_valid_bit_count[1] = (0 <= k - 1); 
        
        for (int i = 2; i <= n; ++i) {
            int set_bits = 0;
            int temp = i;
            while (temp > 0) {
                if (temp & 1) set_bits++;
                temp >>= 1;
            }
            steps[i] = steps[set_bits] + 1;
            // Valid if total steps from x -> i -> 1 is <= k
            if (steps[i] <= k - 1) {
                is_valid_bit_count[i] = true;
            }
        }

        long long total_ans = 0;

        // For each valid target bit count 'b', count how many numbers x < N have exactly 'b' bits
        for (int b = 1; b <= n; ++b) {
            if (!is_valid_bit_count[b]) continue;

            long long ways = 0;
            int current_set_bits = 0;

            for (int i = 0; i < n; ++i) {
                if (s[i] == '1') {
                    int remaining_slots = n - 1 - i;
                    int needed_bits = b - current_set_bits;
                    
                    if (needed_bits >= 0 && needed_bits <= remaining_slots) {
                        ways = (ways + C[remaining_slots][needed_bits]) % MOD;
                    }
                    current_set_bits++;
                }
            }
            
            total_ans = (total_ans + ways) % MOD;
        }

        // Base Case Handling: The loops include x = 0 (0 set bits), but problem specifies positive integers.
        // However, since b starts at 1, x = 0 is naturally excluded from counting towards any b >= 1.
        // But if s itself is valid for some b, it's not counted since we only count strictly less than N.
        
        return total_ans;
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N^2)$
  * **Pascal Triangle Generation:** Computing combination states up to $N \times N$ runs inside a nested grid structure bounded by $\mathcal{O}(N^2)$ steps.
  * **Bit Count Precomputation:** We scan indices from $1$ up to $N$. Finding set bits via bitwise shifts takes $\mathcal{O}(\log N)$ per integer, resulting in a tiny $\mathcal{O}(N \log N)$ total step footprint.
  * **Combinatorial Traversal:** We iterate through each target bit count $b \in [1, N]$. For each target, we scan the string $s$ of length $N$ once. This double loop executes exactly $N \times N$ times, bounded by $\mathcal{O}(N^2)$.
  * **Total Time Bound:** The combined phases scale within $\mathcal{O}(N^2)$. With $N \le 800$, $N^2 \approx 6.4 \cdot 10^5$ operations, which runs instantly under 5 milliseconds.

* **Space Complexity:** $\mathcal{O}(N^2)$
  * **Combinations Table Allocation:** Storing the calculated binomial coefficients inside the $N \times N$ grid `C` dominates the memory profile, requiring an explicit $\mathcal{O}(N^2)$ space footprint. All other auxiliary tracking vectors (`steps`, `is_valid_bit_count`) are strictly linear $\mathcal{O}(N)$.
