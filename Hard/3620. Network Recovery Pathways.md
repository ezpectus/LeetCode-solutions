## 3620. Network Recovery Pathways

### Problem Statement
You are given a directed acyclic graph (DAG) of `n` nodes numbered from `0` to `n - 1`. This is represented by a 2D array `edges`, where `edges[i] = [ui, vi, costi]` indicates a one-way communication from node `ui` to node `vi` with a recovery cost of `costi`.

Some nodes may be offline. You are given a boolean array `online` where `online[i] = true` means node `i` is online. Nodes `0` and `n - 1` are always online.

A path from `0` to `n - 1` is valid if:
* All intermediate nodes on the path are online.
* The total recovery cost of all edges on the path does not exceed `k`.

For each valid path, its score is the minimum edge-cost along that path. Return the maximum path score among all valid paths. If no valid path exists, return -1.

---

### Core Insight — Binary Search on Answer + Heap-Optimized Dijkstra
We are looking to maximize the *minimum* edge cost on a valid path from `0` to `n - 1`. This standard maximin property can be solved cleanly using **Binary Search on the Answer**:

1. **Monotonicity:** If a path exists where every edge has a cost of at least `X` and the total path cost is `<= k`, then we can try to find a larger minimum threshold. If no such path exists even for `X`, then our threshold `X` is too high.
2. **Graph Filtering:** For a guessed value `mid`, we filter the graph. We ignore any edge whose `costi` is strictly less than `mid`. We also completely skip any edge connected to a node that is offline (`online[i] == false`).
3. **Shortest Path Verification:** After filtering, we use Dijkstra's algorithm to check if the shortest path from `0` to `n - 1` has a total cost `<= k`. 

By binary searching over the distinct edge weights present in the graph, we find the maximum valid threshold efficiently.

---

### Algorithm
1. Collect all distinct edge costs and sort them to form our binary search range boundaries.
2. Filter nodes: build the graph adjacency list using only edges where both the source and destination nodes are online.
3. Perform binary search with `low = 0` and `high = distinct_costs.size() - 1`:
   * Set `mid` as the middle index and pick the candidate threshold `target_cost = distinct_costs[mid]`.
   * Run a heap-optimized Dijkstra from node `0`:
     * When relaxing edges, only traverse an edge if its weight is `>= target_cost`.
     * Stop early or check at the end if `dist[n - 1] <= k`.
   * If the target destination is reachable within cost `k`, update our maximum valid score and try a larger threshold (`low = mid + 1`).
   * Otherwise, lower our expectations (`high = mid - 1`).
4. If no valid threshold works, return `-1`.

---

### Full Optimal Implementation (C++)
```cpp
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

class Solution {
public:
    int findMaxPathScore(vector<vector<int>>& edges, vector<bool>& online, long long k) {
        int n = online.size();
        
        // Step 1: Collect and sort distinct edge weights for binary search boundaries
        vector<int> weights;
        for (const auto& edge : edges) {
            weights.push_back(edge[2]);
        }
        sort(weights.begin(), weights.end());
        weights.erase(unique(weights.begin(), weights.end()), weights.end());
        
        // Step 2: Build adjacency list ignoring edges hitting offline nodes
        vector<vector<pair<int, int>>> adj(n);
        for (const auto& edge : edges) {
            int u = edge[0];
            int v = edge[1];
            int cost = edge[2];
            
            if (online[u] && online[v]) {
                adj[u].push_back({v, cost});
            }
        }
        
        // Helper lambda for Dijkstra verification
        auto check = [&](int min_allowed_cost) -> bool {
            vector<long long> dist(n, -1); 
            // Min-heap to store pairs of (total_path_cost, current_node)
            priority_queue<pair<long long, int>, vector<pair<long long, int>>, greater<pair<long long, int>>> pq;
            
            dist[0] = 0;
            pq.push({0, 0});
            
            while (!pq.empty()) {
                auto [curr_cost, u] = pq.top();
                pq.pop();
                
                if (curr_cost > dist[u]) continue;
                if (u == n - 1) return curr_cost <= k;
                
                for (const auto& [v, weight] : adj[u]) {
                    // Filter step: Skip edges that don't meet our current binary search threshold
                    if (weight < min_allowed_cost) continue;
                    
                    long long next_cost = curr_cost + weight;
                    if (dist[v] == -1 || next_cost < dist[v]) {
                        dist[v] = next_cost;
                        pq.push({next_cost, v});
                    }
                }
            }
            return dist[n - 1] != -1 && dist[n - 1] <= k;
        };
        
        // Step 3: Binary search over the valid sorted unique weights
        int low = 0, high = (int)weights.size() - 1;
        int ans = -1;
        
        while (low <= high) {
            int mid = low + (high - low) / 2;
            if (check(weights[mid])) {
                ans = weights[mid];
                low = mid + 1; // Try to maximize the minimum edge-cost score
            } else {
                high = mid - 1; // Threshold is too demanding
            }
        }
        
        return ans;
    }
};
```
### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(M log M + M log N log W) | Sorting distinct weights takes O(M log M). The binary search performs O(log W) checks, where W is the number of unique weights. Each check runs Dijkstra using a min-heap in O(M log N) time. |
| **Space** | O(N + M) | The adjacency list consumes O(N + M) memory. The priority queue and distance arrays within the Dijkstra tracking utilize up to O(N) space. |

### Pitfalls & Edge Cases
* **Large Cost Sums:** Path totals can accumulate up to 5 * 10^13, which completely overflows standard 32-bit signed integers. Ensure that your Dijkstra distance state mutations and accumulation variables use `long long`.
* **Completely Unreachable States:** If node `n - 1` cannot be reached from node `0` under any cost constraint (e.g., critical bridge components are `online == false`), the binary search loop will fail every check, accurately falling through to return `-1`.

### Key Takeaway
When you encounter a graph optimization task asking you to maximize a minimum threshold value (maximin) or minimize a maximum threshold value (minimax), check for monotonicity. Decoupling the threshold constraint using a binary search lets you rewrite a complex single-pass formulation into a simpler verification loop.
