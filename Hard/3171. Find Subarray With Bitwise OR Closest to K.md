## 3171. Find Subarray With Bitwise OR Closest to K

### Problem Statement
You are given an array `nums` and an integer `k`. You need to find a contiguous subarray `nums[l..r]` such that the absolute difference between `k` and the bitwise OR sum of the elements in that subarray is as small as possible.

Return the minimum possible value of this absolute difference: `|k - (nums[l] OR nums[l+1] ... OR nums[r])|`.

---

### Core Insight — Bitwise Monotonicity & In-Place Relaxation
As we extend a subarray to the right, the bitwise OR sum can only increase or stay the same (i.e., more bits get turned on, never turned off). Furthermore, since elements are bounded up to 10^9, there are at most 30 bits. This means that for a fixed starting or ending position, there are at most 30 distinct bitwise OR values possible.

Instead of a brute-force O(N^2) evaluation, we can maintain a set of all unique bitwise OR results ending at the current index `i`. For each new element `nums[i]`:
* We compute the bitwise OR of `nums[i]` with all unique OR results from the previous step `i-1`.
* This generates a new, small set of unique OR values ending at `i`.
* We update our minimum absolute difference with `|k - value|` for every value in this set.

Since the size of the set is bounded by 30 (the number of bits), each transition step takes O(30) constant time, reducing the overall complexity to linear.

---

### Algorithm
1. Initialize `minDiff = int.MaxValue`.
2. Create a set `prev_ors` to hold the unique bitwise OR values of subarrays ending at the previous index.
3. Iterate through each element `x` in `nums`:
   * Create a new set `curr_ors`.
   * Insert `x` into `curr_ors` (representing the single-element subarray `nums[i..i]`).
   * For each value `prev` in `prev_ors`, compute `prev | x` and insert it into `curr_ors`.
   * For each value `val` in `curr_ors`, update `minDiff = min(minDiff, abs(k - val))`.
   * Assign `prev_ors = curr_ors` for the next iteration.
4. Return `minDiff`.

---

### Full Optimal Implementation (C++)
```cpp
#include <vector>
#include <unordered_set>
#include <cmath>
#include <algorithm>
#include <climits>

using namespace std;

class Solution {
public:
    int minimumDifference(vector<int>& nums, int k) {
        int minDiff = INT_MAX;
        minDiff = abs(k - nums[0]);
        
        // Tracks unique bitwise OR results ending at the previous element
        unordered_set<int> prev_ors;
        
        for (int x : nums) {
            unordered_set<int> curr_ors;
            curr_ors.insert(x);
            
            // Perform bitwise OR transitions from previous subarrays
            for (int prev : prev_ors) {
                curr_ors.insert(prev | x);
            }
            
            // Evaluate absolute differences against target k
            for (int val : curr_ors) {
                minDiff = min(minDiff, abs(k - val));
            }
            
            prev_ors = move(curr_ors);
        }
        
        return minDiff;
    }
};
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N * log(MAX_VAL)) | The outer loop runs `N` times. The inner loops process at most 30 elements because a number can accumulate at most 30 set bits before saturating. |
| **Space** | O(log(MAX_VAL)) | The space complexity is bounded by the size of our sets, which hold at most 30 distinct integer values at any given point. |

### Pitfalls & Edge Cases
* **Saturating Bits Early:** If a subarray OR value becomes larger than `k` and turns on all bits, subsequent operations will keep it high. The absolute difference might increase, but our algorithm gracefully captures the closest inflection point before saturation occurs.
* **Single Element Subarrays:** Remember that a single element is a valid subarray. Explicitly inserting `x` directly into `curr_ors` ensures single-element arrays are always checked.

### Key Takeaway
When working with bitwise operators like OR, AND, or GCD across contiguous segments, remember that properties can only change monotonically and a limited number of times (bounded by bit width). Tracking only the distinct running values allows you to compress what looks like a quadratic search space into an optimized linear runtime.
