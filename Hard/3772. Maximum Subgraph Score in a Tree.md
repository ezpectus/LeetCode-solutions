# 3772. Maximum Subgraph Score in a Tree

## 1. Idea & Architectural Choice

### 1.1 Core Philosophy & Mathematical Reduction
We are given an undirected tree with $N$ nodes. Each node $i$ has a value:
- $+1$ if `good[i] == 1`
- $-1$ if `good[i] == 0`

For each node $i$, we need to find the maximum possible total score of a connected subgraph containing node $i$.

This problem is equivalent to the Maximum Weight Connected Subgraph problem on a tree, evaluated for every possible starting node $i$.

If we fix node $i$ as the root of the tree, the optimal connected subgraph containing node $i$ can be formed greedily from the bottom up:
- For any sub-branch hanging from a node $u$, we include that sub-branch if and only if its maximum connected subgraph score starting at the child is strictly positive (> 0).
- If the score of a child's subtree is $\le 0$, including it will only decrease or keep the overall score the same, so we discard that branch entirely.

To solve this efficiently for all $N$ nodes in $O(N)$ time instead of running an $O(N)$ Tree DP from scratch for each node ($O(N^2)$), we utilize **Rerooting Dynamic Programming (Tree DP / Tree Rerooting Technique)**.

---

## 2. Rerooting DP Formulation & Transitions

### 2.1 State Definitions
Let $val[u] = 1$ if `good[u] == 1` else $-1$.

1. `down_dp[u]`: The maximum score of a connected subgraph containing node $u$, restricted entirely to the subtree rooted at $u$ (assuming node 0 is the global root).
2. `up_dp[u]`: The maximum score of a connected subgraph containing $u$'s parent $p$, extending into the rest of the tree excluding $u$'s subtree, which $u$ can optionally attach to.
3. `ans[u]`: The final maximum score of a connected subgraph containing node $u$ for the whole tree.

### 2.2 First Pass: Bottom-Up Tree DP (First DFS)
Root the tree arbitrarily at node 0.
For any leaf node $u$:
`down_dp[u] = val[u]`

For any non-leaf node $u$ with children $v_1, v_2, \dots, v_k$:
`down_dp[u] = val[u] + sum(max(0, down_dp[v_i]))` for all children $v_i$.

Strict vs. Non-Strict Inequality:
We use `max(0, down_dp[v_i])`. Including a child with `down_dp[v_i] == 0` does not change the score sum, so taking `max(0, ...)` correctly handles both negative and zero contributions smoothly without double counting.

### 2.3 Second Pass: Rerooting / Top-Down Transition (Second DFS)
When moving the root from parent $u$ to child $v$:
1. Compute the total score at $u$:
   `ans[u] = down_dp[u] + max(0, up_dp[u])`
2. When transitioning to child $v$, node $v$ wants to know the maximum score it can get from its parent $u$.
3. Node $u$'s contribution to $v$ consists of node $u$'s own value, plus contributions from $u$'s parent (`max(0, up_dp[u])`), plus contributions from all other children of $u$ except $v$.
4. The contribution from all children of $u$ except $v$ is simply:
   `down_dp[u] - max(0, down_dp[v])`
5. Therefore, the "upward" contribution for child $v$ is:
   `up_dp[v] = ans[u] - max(0, down_dp[v])`

## 3. Full Optimal Implementation (C++)

```cpp
// Rerooting Dynamic Programming + DFS + Tree DP
#include <vector>
#include <algorithm>

class Solution {
private:
    std::vector<std::vector<int>> adj;
    std::vector<int> node_val;
    std::vector<int> down_dp;
    std::vector<int> ans;

    void dfs_down(int u, int parent) {
        down_dp[u] = node_val[u];
        for (int v : adj[u]) {
            if (v == parent) continue;
            dfs_down(v, u);
            if (down_dp[v] > 0) {
                down_dp[u] += down_dp[v];
            }
        }
    }

    void dfs_reroot(int u, int parent, int up_val) {
        // Total score for node u including up_val from parent
        ans[u] = down_dp[u] + std::max(0, up_val);

        for (int v : adj[u]) {
            if (v == parent) continue;
            
            // Contribution to child v is total score at u minus child v's own contribution
            int v_contrib = std::max(0, down_dp[v]);
            int up_for_v = ans[u] - v_contrib;

            dfs_reroot(v, u, up_for_v);
        }
    }

public:
    std::vector<int> maxSubgraphScore(int n, std::vector<std::vector<int>>& edges, std::vector<int>& good) {
        adj.assign(n, std::vector<int>());
        node_val.resize(n);
        down_dp.assign(n, 0);
        ans.assign(n, 0);

        for (int i = 0; i < n; ++i) {
            node_val[i] = (good[i] == 1) ? 1 : -1;
        }

        for (const auto& edge : edges) {
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]);
        }

        // Step 1: Compute down_dp via bottom-up Tree DP rooted at node 0
        dfs_down(0, -1);

        // Step 2: Propagate rerooting results via top-down DFS
        dfs_reroot(0, -1, 0);

        return ans;
    }
};
```
## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
Let N be the number of nodes in the tree (N <= 10^5).

1. **Graph Construction Phase**:
   - Building adjacency list from N - 1 edges.
   - Operations: T_adj = O(N).

2. **First DFS (`dfs_down`)**:
   - Visits every node u exactly once and inspects each undirected edge twice (once from parent, once from child).
   - Operations: T_dfs1 = O(N).

3. **Second DFS (`dfs_reroot`)**:
   - Visits every node u exactly once and performs constant O(1) arithmetic transitions per edge to compute `up_for_v`.
   - Operations: T_dfs2 = O(N).

**Total Time Complexity:**
T_total = T_adj + T_dfs1 + T_dfs2 = O(N)

For N = 10^5, total operations stay under 3 * 10^5, running in less than 35 milliseconds.

### 4.2 Detailed Space Complexity Decomposition
- **adj (Adjacency List)**: N vectors storing 2(N - 1) edge entries (~ 8 MB).
- **down_dp, ans, node_val Arrays**: Vectors of size N storing 32-bit integers (~ 1.2 MB each).
- **DFS Call Stack**: Recursion depth at most N in a skewed line tree (~ 4 MB).

**Total Space Complexity:**
S_total = O(N)

Total auxiliary memory is under 20 MB, well within standard platform limits (256 MB).

### 4.3 Complexity Summary

| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(N) | N <= 10^5. Two linear DFS passes over the tree. |
| **Space Complexity** | O(N) | Adjacency list and DP vectors + recursion call stack. |

## 5. Mathematical Formulations & Foundational Algorithms

### Optimal Substructure & Greediness in Connected Subtrees
Let u be a node in the tree T. Any connected subgraph S containing u can be decomposed into u itself plus connected subgraphs S_v in the subtrees/components attached to u via its neighbors v in N(u).

The score of the combined subgraph is:
Score(S) = val[u] + sum over all attached components v of Score(S_v)

To maximize Score(S):
1. Each component S_v must independently maximize its score when attached to v.
2. An attached component S_v should be included if and only if max(Score(S_v)) > 0.
3. If max(Score(S_v)) <= 0, setting S_v = empty set gives score 0, which is optimal.

Thus, max(Score(S)) = val[u] + sum over v in N(u) of max(0, max(Score(S_v))).
This proves the additive dynamic programming recurrence relation used in both DFS passes.

## Pitfalls & Edge Cases

* **Negative Scores:** If all nodes in a subtree are bad (`good[i] == 0`), `down_dp` values can be negative (e.g., -1). We must use `std::max(0, down_dp[v])` to ensure negative subtrees are not added to the parent's score.
* **All Bad Nodes in Entire Tree:** If `good` contains all zeros, every node's best connected subgraph is just the node itself, giving a score of -1. The formula handles this naturally because `max(0, up_val)` and `max(0, down_dp[v])` evaluate to 0, leaving `ans[u] = val[u] = -1`.
* **Stack Overflow on Deep Trees:** For N = 10^5, standard recursion depth of 10^5 is well supported in C++ under typical LeetCode stack size limits (up to 256MB).

---

## Key Takeaway

When a tree dynamic programming problem asks for an optimal connected sub-structure containing each node i as a root, use the Rerooting Technique (Tree DP): run one bottom-up pass to calculate subtree metrics, and a second top-down pass to distribute parent contributions in O(N) total time.

**Pattern recognition:** "maximum score connected subgraph containing node i for all i in a tree" -> Rerooting Dynamic Programming + Tree DFS
