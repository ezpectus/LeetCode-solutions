# 3261. Count Substrings That Satisfy K-Constraint II
## O(n + q log n) — Sliding Window + Prefix Sums + Binary Search

---

## Problem Statement

Binary string `s`, integer `k`. A substring is valid if `min(count0, count1) ≤ k` (either zeros or ones number at most k). For each query `[L, R]`, count valid substrings of `s[L..R]`.

---

## Core Insight #1 — Compute the "Minimum Valid Left" Array

Define `l[j]` = smallest left index such that `s[l[j]..j]` satisfies the k-constraint. Substrings ending at `j` are valid iff their left endpoint is ≥ `l[j]`.

**Key property: `l[j]` is non-decreasing** — as `j` grows by 1, the constraint can only get tighter.

Compute with a sliding window (shrink left when BOTH counts exceed k):
```
while (cnt0 > k AND cnt1 > k): remove s[left], left++
l[j] = left
```

---

## Core Insight #2 — Count for a Query Using a Split Point

For query `[L, R]`, valid substrings ending at `j` start in `[max(L, l[j]), j]`, contributing `j - max(L, l[j]) + 1` substrings.

**Define `T`** = first `j` in `[L, R]` where `l[j] > L` (i.e., `s[L..j]` does NOT satisfy k-constraint). Found by binary search since `l` is non-decreasing.

**Part 1 (j in [L, T-1]):** `l[j] ≤ L`, so contribution = `j - L + 1`.
```
Sum₁ = Σ_{j=L}^{T-1} (j - L + 1) = (T-L)(T-L+1)/2
```

**Part 2 (j in [T, R]):** `l[j] > L`, so contribution = `j - l[j] + 1`.
```
Sum₂ = Σ_{j=T}^{R} (j - l[j] + 1)
     = [Σ(j+1) from T to R] - [Σ l[j] from T to R]
     = [(R+1)(R+2)/2 - T(T+1)/2] - [psum_l[R+1] - psum_l[T]]
```

where `psum_l[i] = Σ_{j=0}^{i-1} l[j]` (prefix sum of `l`).

## Full Implementation (C++)

```cpp
class Solution {
public:
    vector<long long> countKConstraintSubstrings(string s, int k,
                                                  vector<vector<int>>& queries) {
        int n = s.size();

        // Sliding window to compute l[j]
        vector<int> l(n);
        int cnt0 = 0, cnt1 = 0, left = 0;
        for (int j = 0; j < n; j++) {
            (s[j] == '0' ? cnt0 : cnt1)++;
            while (cnt0 > k && cnt1 > k) {
                (s[left] == '0' ? cnt0 : cnt1)--;
                left++;
            }
            l[j] = left;
        }

        // Prefix sums of l[]
        vector<long long> psum_l(n + 1, 0);
        for (int j = 0; j < n; j++) psum_l[j + 1] = psum_l[j] + l[j];

        vector<long long> ans(queries.size());
        for (int q = 0; q < (int)queries.size(); q++) {
            int L = queries[q][0], R = queries[q][1];

            // T = first j in [L, R] with l[j] > L (binary search)
            int T = (int)(upper_bound(l.begin() + L, l.begin() + R + 1, L)
                          - l.begin());

            // Part 1: j in [L, T-1]
            long long span = T - L;
            long long total = span * (span + 1) / 2;

            // Part 2: j in [T, R]
            if (T <= R) {
                long long sum_jp1 = (long long)(R + 1) * (R + 2) / 2
                                  - (long long)T * (T + 1) / 2;
                long long sum_lj  = psum_l[R + 1] - psum_l[T];
                total += sum_jp1 - sum_lj;
            }

            ans[q] = total;
        }
        return ans;
    }
};
```

## Complexity

| Step | Time | Notes |
|---|---|---|
| Sliding window (`l[]`) | O(n) | Each index enters/exits window once |
| Prefix sums | O(n) | |
| Per query (binary search) | O(log n) | `l` is sorted |
| **Total** | **O(n + q log n)** | n=q=10⁵ → ~1.7M ops |
| Space | O(n) | `l[]` + `psum_l[]` |

## Example Walkthrough

### Example 1 — `s="0001111"`, `k=2`, `queries=[[0,6]]`

**Sliding window for `l[]`:**
| j | char | cnt0 | cnt1 | both>2? | l[j] |
|---|---|---|---|---|---|
| 0 | 0 | 1 | 0 | no | 0 |
| 1 | 0 | 2 | 0 | no | 0 |
| 2 | 0 | 3 | 0 | no (cnt1=0) | 0 |
| 3 | 1 | 3 | 1 | no (cnt1=1) | 0 |
| 4 | 1 | 3 | 2 | no (cnt1=2=k) | 0 |
| 5 | 1 | 3 | 3 | YES → shrink: remove s[0]='0' → cnt0=2 | 1 |
| 6 | 1 | 2 | 4 | no (cnt0=2=k) | 1 |

`l = [0,0,0,0,0,1,1]`. `psum_l = [0,0,0,0,0,0,1,2]`.

**Query [L=0, R=6]:**
Binary search for first j with `l[j] > 0` in `[0..6]`: `l[5]=1>0` → **T=5**.

Part 1 (j in [0,4]): span=5-0=5 → `5×6/2 = 15`.
Part 2 (j in [5,6]):
- `sum_jp1 = 7×8/2 - 5×6/2 = 28-15 = 13`
- `sum_lj = psum_l[7]-psum_l[5] = 2-0 = 2`
- contributes `13-2 = 11`

**Total = 15+11 = 26** ✓

*(The 2 invalid substrings "000111" and "0001111" are correctly excluded.)*

## Why Binary Search Works

`l[j]` is non-decreasing (adding characters to the right can only make the left boundary move rightward or stay). So for fixed `L`, the condition `l[j] > L` is a monotone predicate on `j`, allowing `upper_bound`.

---

## Pitfalls & Edge Cases

- **`l[j] < L`:** The constraint is satisfied "further left" than our query window. These j values fall in Part 1 (l[j] ≤ L), contributing `j - L + 1`. ✓

- **T > R (all j in [L,R] have l[j] ≤ L):** Part 2 is empty. Total = `(R-L+1)(R-L+2)/2` — all substrings in the query range are valid. ✓

- **T = L (l[L] > L is impossible):** `l[L] ≤ L` always since the single character at L always satisfies k-constraint (length 1 means min count ≤ 1 ≤ k). So Part 1 always has at least j=L. ✓

- **Overflow:** `span × (span+1) / 2` with span up to 10⁵: max value ~5×10⁹, fits in `long long`. ✓

---

## Key Takeaway

This is the **"sliding-window minimum left + prefix sum decomposition"** pattern. For "count valid subarrays/substrings within a query range", precompute the minimum valid left per right endpoint, then split the query at the first j where the left endpoint "enters the query window". The split enables arithmetic formulas for both halves, reducing each query to O(log n) (binary search) + O(1) (formula evaluation).

**Pattern recognition:** "Count substrings satisfying a monotone sliding-window condition over a query range [L,R]" → precompute `l[j]` (non-decreasing), binary-search for the split point `T`, apply prefix-sum formulas for both halves.
