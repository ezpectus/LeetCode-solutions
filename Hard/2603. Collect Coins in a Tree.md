# 2603. Collect Coins in a Tree
## O(n) — Three-Phase Pruning: Zero-Coin Leaves → Two Leaf Rings → 2× Core Edges

---

## Problem Statement

Unrooted tree, `n` nodes, each either has a coin (`coins[i]=1`) or not. From any node, one operation collects all coins within distance ≤ 2. Another operation moves to an adjacent node. Return the **minimum number of edge traversals** to collect all coins and return to start (each traversal in each direction counts separately).

---

## Core Insight #1 — Zero-Coin Leaves Are Completely Irrelevant

A leaf node with `coins[i] = 0` contributes nothing: it has no coin of its own, and visiting it can only give access to distance-2 neighbors — all of which are also reachable from the leaf's parent. We never gain anything by going there.

**Prune** all such leaves. But pruning a leaf may expose its parent as a new zero-coin leaf. Repeat until no zero-coin leaves remain.

This is a standard **topological-sort trim**: put all degree-1 zero-coin nodes in a queue, process each by removing it and decrementing neighbor degrees, and enqueue any newly-created zero-coin leaves.

After this phase: **every remaining leaf has a coin.**

---

## Core Insight #2 — Radius-2 Collection Eliminates Two Leaf Rings

After phase 1, consider any remaining leaf L with parent P and grandparent G (all in the remaining tree). Standing at G, the collection operation gathers all coins within distance 2 — including L (at distance 2 from G) and P (at distance 1 from G). There is **zero benefit** to physically walking to P or L, because G already covers them.

This reasoning applies to the entire leaf ring: all current leaves can be served from their grandparents. So prune the leaves. This exposes a new ring of leaves. The same argument applies once more: those new leaves can be served from *their* grandparents (which are in the core). Prune again.

**Two rounds of leaf removal.** After both rounds, what remains is the **core**: the minimal subgraph such that every pruned coin is within distance 2 of some core node.

---

## Core Insight #3 — Answer = 2 × Core Edge Count

The core must be **fully traversed** — no edge can be skipped:
- Any edge `u—v` in the core splits the core into two components.
- Coins in each component's hanging subtrees (distance 1–2 from that component's nodes) cannot be collected from the other side — they're too deep.
- So we must cross `u—v` going forward and again coming back. Each edge is traversed at least twice.

Since an **Euler tour** on a tree achieves exactly 2 traversals per edge (one each way), the minimum is precisely:

```
answer = 2 × (number of edges remaining in the core)
```

Special case: if the core has ≤ 1 node (no edges), answer = 0 — all coins are collectible from a single standing position.

---

## Three-Phase Visual

```
Original tree
     │
     ▼
[Phase 1] Remove zero-coin leaves (topological sort, repeated)
     │         → every remaining leaf now has a coin
     ▼
[Phase 2a] Remove all current leaves (round 1 of 2)
     │         → coins in removed ring ≤ dist 2 from their grandparents
     ▼
[Phase 2b] Remove all current leaves (round 2 of 2)
     │         → coins in removed ring ≤ dist 2 from new core leaves
     ▼
[Core] Count remaining edges → answer = 2 × count
```

## Full Implementation (C#)

```csharp
public class Solution {
    public int CollectTheCoins(int[] coins, int[][] edges) {
        int n = coins.Length;
        var adj = new List<int>[n];
        for (int i = 0; i < n; i++) adj[i] = new List<int>();
        int[] deg = new int[n];

        foreach (var e in edges) {
            adj[e[0]].Add(e[1]);
            adj[e[1]].Add(e[0]);
            deg[e[0]]++;
            deg[e[1]]++;
        }

        bool[] alive = new bool[n];
        Array.Fill(alive, true);

        // Phase 1: Remove all zero-coin leaves (topological trim)
        var q = new Queue<int>();
        for (int i = 0; i < n; i++)
            if (deg[i] == 1 && coins[i] == 0) q.Enqueue(i);

        while (q.Count > 0) {
            int u = q.Dequeue();
            alive[u] = false;
            foreach (int v in adj[u]) {
                if (alive[v]) {
                    deg[v]--;
                    if (deg[v] == 1 && coins[v] == 0) q.Enqueue(v);
                }
            }
        }

        // Phase 2: Remove two full rings of leaves (batch, one ring at a time)
        for (int round = 0; round < 2; round++) {
            var leaves = new List<int>();
            for (int i = 0; i < n; i++)
                if (alive[i] && deg[i] == 1) leaves.Add(i);

            foreach (int u in leaves) {
                alive[u] = false;
                foreach (int v in adj[u])
                    if (alive[v]) deg[v]--;
            }
        }

        // Count remaining (core) edges
        int ans = 0;
        foreach (var e in edges)
            if (alive[e[0]] && alive[e[1]]) ans++;

        return 2 * ans;
    }
}
```

| Metric | Value | Notes |

| Space Complexity | O(n) | Adjacency list, `deg`, `alive` arrays |

## Example Walkthrough

### Example 1 — `coins = [1,0,0,0,0,1]`, path: 0–1–2–3–4–5

**Phase 1:** Leaves are 0 (coin=1) and 5 (coin=1). Neither is zero-coin. No removal.

**Phase 2, Round 1:** Remove leaves 0 and 5. Node 1: deg 1→ stays alive. Node 4: deg 1 → stays alive.

**Phase 2, Round 2:** Remove leaves 1 and 4. Remaining: nodes 2 and 3, edge 2–3.

```
Before:  0—1—2—3—4—5
Round 1: (0)—1—2—3—4—(5)   ← (x) = removed
Round 2: (1)—2—3—(4)
Core:        2—3              ← 1 edge
```

**Answer = 2 × 1 = 2** ✓

---

### Example 2 — `coins = [0,0,0,1,1,0,0,1]`

Tree: 0–1, 0–2, 1–3, 1–4, 2–5, 5–6, 5–7

**Phase 1:** Find zero-coin leaves: node 6 (coin=0, deg=1). Remove 6. Node 5 becomes deg=2 (still not a zero-coin leaf since coin[5]=0 but deg≠1). No more zero-coin leaves.

**Phase 2, Round 1:** Current leaves: 3, 4, 7. Remove all three.
- Node 1: deg 3→1 (lost neighbors 3 and 4). Becomes a leaf.
- Node 5: deg 2→1 (lost neighbor 7). Becomes a leaf.

**Phase 2, Round 2:** Current leaves: 1 and 5. Remove both.
- Node 0: deg 2→1. Node 2: deg 2→1. Now 0 and 2 are new leaves.
- But this is round 2 — we only run the snapshot of leaves found at the start of this round (nodes 1 and 5), not the newly created ones.

Remaining alive: 0 and 2, connected by edge 0–2. 1 edge.

```
Before:  3   4           6   7
          \ /             \ /
      0—1           2—5
Phase1:  (6) removed
Round 1: (3),(4),(7) removed
Round 2: (1),(5) removed
Core:    0—2              ← 1 edge
```

**Answer = 2 × 1 = 2** ✓

## Pitfalls & Edge Cases

- **Phase 2 must batch, not stream.** Each round of leaf removal must snapshot ALL current leaves first, then remove them together. If you remove leaves one by one, the degree changes mid-round can incorrectly cause a new leaf to be removed in the same round (when it should survive to be removed in round 2, or not at all). The snapshot ensures exactly one ring is removed per round.

- **Phase 1 can stream (queue BFS), Phase 2 cannot.** In Phase 1, order doesn't matter — we're eliminating all zero-coin leaves regardless of order. In Phase 2, we must peel exactly 2 rings in sequence.

- **Degree 0 after pruning ≠ leaf.** After Phase 1 removes all neighbors of some node, that node may have `deg == 0`. A degree-0 alive node is an isolated survivor (e.g., a coin-1 center of a star whose all coin-0 arms were trimmed). It does not get removed in Phase 2 (not a leaf). It contributes 0 edges to the answer. Correct: you just stand there and collect. ✓

- **All coins are 0:** Phase 1 removes everything. No alive nodes. Phase 2 is a no-op. Answer = 0. ✓ (Nothing to collect.)

- **n = 1:** No edges. Answer = 0 immediately. ✓

- **All coins reachable from one spot (small diameter trees):** After pruning, the core collapses to 0 or 1 node. Answer = 0. ✓ (Example: path of 4 nodes all with coins — collecting from the middle node covers everything.)

---

## Key Takeaway

This is a **"tree reduction to essential core"** pattern. Three sequential reductions each eliminate exactly the nodes that require no physical traversal:
1. Zero-coin leaves: useless forever.
2. Coin leaves: their coins are reachable from grandparents (radius-2 shortcut).
3. Post-removal leaves: same reasoning one level up.

What remains after 1+2 reductions is the minimal subgraph that must be physically walked. Every edge in it must be crossed twice (forward and back), giving the tight 2× formula.

**Pattern recognition:** "Minimum traversal in a tree with radius-k collection" → prune useless leaves, then prune k rings of remaining leaves, then answer = 2 × edges remaining.
