# 3614. Process String with Special Operations II
O(n) — Reverse Engineering via Backward Coordinate Tracking

---

## Problem Statement
You are given a string s consisting of lowercase English letters and the special characters: '*', '#', and '%', and an integer k.
Build a new string result by processing s according to the following rules from left to right:
- If the letter is a lowercase English letter append it to result.
- A '*' removes the last character from result, if it exists.
- A '#' duplicates the current result and appends it to itself.
- A '%' reverses the current result.
Return the kth character of the final string result. If k is out of the bounds of result, return '.'.

## Core Idea — Coordinate Mapping via Simulation Rollback
Since the final string result can grow exponentially up to 10¹⁵ characters long, simulating the string generation directly is impossible. Instead of building the string, we track the total length forward, check bounds, and then reverse the timeline to map index k back to its source character.

## Algorithm:
- 1. Forward pass: compute the final length of result. A letter adds 1, '*' subtracts 1 (min 0), '#' doubles the length, and '%' keeps it the same.
- 2. If k >= final_length, return '.' immediately.
- 3. Backward pass: iterate through s from right to left. Undo each operation's effect on the length and map k back to its previous position. When k matches the position of a newly inserted letter, return that letter.

## Full Optimal Implementation (C++)\
```
#include <string>
#include <algorithm>

using namespace std;

class Solution {
public:
    char processStr(string s, long long k) {
        long long m = 0;
        
        // Step 1: Forward Pass to calculate the length of the string at each step
        for (char c : s) {
            if (c == '*') {
                m = max(0LL, m - 1);
            } else if (c == '#') {
                m <<= 1; // Double the length
            } else if (c != '%') {
                m += 1;  // Normal lowercase letter
            }
            // '%' does not change the length of the string
        }
        
        // Out of bounds check
        if (k >= m) {
            return '.';
        }
        
        // Step 2: Backward Pass to track index k back to its origin
        for (int i = s.length() - 1; i >= 0; --i) {
            char c = s[i];
            
            if (c == '*') {
                m += 1; // Undo the deletion
            } else if (c == '#') {
                m /= 2; // Undo the duplication
                if (k >= m) {
                    k -= m; // Shift k back into the original first half
                }
            } else if (c == '%') {
                k = m - 1 - k; // Undo the reversal
            } else {
                m -= 1; // Undo the letter insertion
                if (k == m) {
                    return c; // The index matches the inserted character
                }
            }
        }
        
        return '.';
    }
};
```

## Complexity
- Time Complexity | O(n) | Two independent linear scans over s, each character takes O(1) time. n ≤ 10⁵.
- Space Complexity | O(1) | Only a few long long primitive variables are used; no strings are stored.

Why "Backward Tracking" Always Resolves the Correct Character
String modifications like duplication ('#') and reversal ('%') produce completely predictable coordinate transformations. When a string of length m is duplicated via '#', the second half is an exact copy of the first half; hence, any index k ≥ m/2 can be mapped back to the first half by subtracting the half-length (k -= m/2). When a string is reversed via '%', the character at index k originally came from index m - 1 - k. By reversing the timeline step-by-step, we accurately reconstruct the index trajectory without ever materializing the massive string.

## Example Walkthrough
Example 1: s = "a#b%*", k = 1
Forward Pass (Length Tracking):
- 'a' -> m = 1
- '#' -> m = 2
- 'b' -> m = 3
- '%' -> m = 3
- '*' -> m = 2
Final length m = 2. k = 1 < 2, within bounds.

Backward Pass (Index Tracking):
- i = 4, c = '*': Undo deletion → m = m + 1 = 3. k = 1.
- i = 3, c = '%': Undo reversal → k = m - 1 - k = 3 - 1 - 1 = 1.
- i = 2, c = 'b': Undo letter insertion → m = m - 1 = 2. Since k (1) != m (2), skip.
- i = 1, c = '#': Undo duplication → m = m / 2 = 1. Since k (1) >= m (1), k = k - m = 0.
- i = 0, c = 'a': Undo letter insertion → m = m - 1 = 0. Since k (0) == m (0), match found. Return 'a' ✓
Output: "a" ✓ matches expected.

Example 2: s = "cd%#*#", k = 3
Forward Pass (Length Tracking):
- 'c' -> m = 1
- 'd' -> m = 2
- '%' -> m = 2
- '#' -> m = 4
- '*' -> m = 3
- '#' -> m = 6
Final length m = 6. k = 3 < 6, within bounds.

## Backward Pass (Index Tracking):
- i = 5, c = '#': Undo duplication → m = 3. Since k (3) >= m (3), k = k - m = 0.
- i = 4, c = '*': Undo deletion → m = 4. k = 0.
- i = 3, c = '#': Undo duplication → m = 2. Since k (0) < m (2), k stays 0.
- i = 2, c = '%': Undo reversal → k = m - 1 - k = 2 - 1 - 0 = 1.
- i = 1, c = 'd': Undo letter insertion → m = 1. Since k (1) == m (1), match found. Return 'd' ✓
Output: "d" ✓ matches expected.

## Pitfalls & Edge Cases
- Use of long long for m and k — values up to 10¹⁵ will overflow a standard 32-bit int, so using a wider type is required. ✓
- Reversal off-by-one — mapping the index back during a '%' operation requires using m - 1 - k due to 0-based indexing. ✓
- Backwards handling of '*' — when rolling back a deletion, we must increment the current length tracker m to correctly restore the historical size context. ✓

## Key Takeaway
- When a problem asks for a single index from an object generated by exponential modifications, do not attempt to construct it. Compute sizes going forward to validate bounds, then roll back the transformations to find the exact origin.
- Pattern recognition: "Exponential growth via structural operations + single index query" → tracking lengths forward and mapping indices backward avoids simulation entirely.
