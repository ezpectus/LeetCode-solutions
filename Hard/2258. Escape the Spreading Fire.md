# 2258. Escape the Spreading Fire
## O(m×n × log(m×n)) — Fire BFS + Binary Search + Player BFS

---

## Problem Statement

- Grid with grass (0), fire (1), walls (2). Fire spreads each minute. Find max minutes you can wait at (0,0) before moving to reach (m-1,n-1) safely. Return -1 if impossible, 10⁹ if always possible.

---

## Core Idea — Precompute Fire Times, Binary Search on Wait

**Phase 1**: Multi-source BFS from all initial fire cells → `fire_time[r][c]` = earliest minute fire reaches `(r,c)`. O(m×n).

**Phase 2**: Binary search on wait time `w`. For each `w`, BFS the player starting at time `w`:
- Player can step on `(nr,nc)` at time `t+1` iff `fire_time[nr][nc] > t+1` (not yet on fire).
- Exception: the safehouse `(m-1,n-1)` can be entered iff `fire_time[m-1][n-1] >= t+1` (fire arrives same time is OK).

If `can_escape(0)` is false → return -1.
If `can_escape(m×n)` is true → return 10⁹ (fire can't reach player no matter how long they wait).
Otherwise binary search finds the maximum valid `w`.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int m, n;
    int[][] grid;
    int[,] fireTime;
    int[] dr = {0,0,1,-1}, dc = {1,-1,0,0};

    public int MaximumMinutes(int[][] grid) {
        this.grid = grid;
        m = grid.Length; n = grid[0].Length;
        BfsFire();
        
        if (!CanEscape(0)) return -1;
        if (CanEscape(m*n)) return 1_000_000_000;

        int lo = 0, hi = m*n;
        while (lo < hi) {
            int mid = (lo+hi+1)/2;
            if (CanEscape(mid)) lo = mid; else hi = mid-1;
        }
        return lo;
    }

    void BfsFire() {
        fireTime = new int[m, n];
        for (int i = 0; i < m; i++) for (int j = 0; j < n; j++) fireTime[i,j] = int.MaxValue;
        var q = new Queue<(int,int)>();
        for (int r = 0; r < m; r++) for (int c = 0; c < n; c++)
            if (grid[r][c] == 1) { fireTime[r,c] = 0; q.Enqueue((r,c)); }
        while (q.Count > 0) {
            var (r,c) = q.Dequeue();
            for (int d = 0; d < 4; d++) {
                int nr = r+dr[d], nc = c+dc[d];
                if (nr>=0&&nr<m&&nc>=0&&nc<n&&grid[nr][nc]==0&&fireTime[nr,nc]==int.MaxValue) {
                    fireTime[nr,nc] = fireTime[r,c]+1;
                    q.Enqueue((nr,nc));
                }
            }
        }
    }

    bool CanEscape(int wait) {
        if (fireTime[0,0] <= wait) return false;
        var visited = new bool[m,n];
        visited[0,0] = true;
        var q = new Queue<(int r, int c, int t)>();
        q.Enqueue((0,0,wait));
        while (q.Count > 0) {
            var (r,c,t) = q.Dequeue();
            for (int d = 0; d < 4; d++) {
                int nr = r+dr[d], nc = c+dc[d];
                if (nr<0||nr>=m||nc<0||nc>=n||grid[nr][nc]!=0||visited[nr,nc]) continue;
                int nt = t+1;
                if (nr==m-1&&nc==n-1) { if (fireTime[nr,nc]>=nt) return true; continue; }
                if (fireTime[nr,nc] > nt) { visited[nr,nc]=true; q.Enqueue((nr,nc,nt)); }
            }
        }
        return false;
    }
}
```

## Key Rules for Player Movement

- Regular cell `(nr,nc)` at time `nt`: valid iff `fireTime[nr][nc] > nt` (fire hasn't arrived yet).
- Safehouse `(m-1,n-1)` at time `nt`: valid iff `fireTime[m-1][n-1] >= nt` (arriving same time as fire is OK per problem statement).

---

## Example Walkthrough

**Grid example 1**: fire starts at (1,5). fire_time BFS computes earliest fire arrival everywhere. Binary search finds w=3: player can wait 3 minutes and still reach safehouse before fire.

---

## Pitfalls & Edge Cases

- **Safehouse condition `>=`**: player arriving same minute as fire counts as safe — use `>=` not `>`.
- **Other cells `>`**: player cannot step on a cell that fire has already reached.
- **`10⁹` response**: check `can_escape(m×n)` (max meaningful wait) as a proxy for "always escapable".
- **`fireTime[0][0] <= wait`**: starting cell already on fire before player starts moving.

---

## Key Takeaway

"Max wait before moving to escape spreading fire" → **fire BFS for precomputed arrival times + binary search on wait + player BFS with fire-time constraints**:

```
fire_time = multi-source BFS from fire cells
binary search w in [0, m*n]:
    can_escape(w) = player BFS from (0,0) at time w
                    can step on (r,c) at time t iff fire_time[r][c] > t (or >= for safehouse)
```
