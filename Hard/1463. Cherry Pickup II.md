# 1463. Cherry Pickup II
## O(rows × cols²) — DP on (row, c1, c2)

---

## Problem Statement

- Two robots start at `(0, 0)` and `(0, cols-1)`. Each moves one row down per step, shifting column by -1, 0, or +1.
- Both collect cherries; if on same cell, only one copy counted.
- Maximize total cherries collected.

---

## Core Idea — Synchronous Row-by-Row DP

Since both robots move down together one row at a time, we can model the state as their **current column positions** after processing each row.

**State**: `dp[c1][c2]` = maximum cherries collected when robot 1 is at column `c1` and robot 2 is at column `c2` after the current row.

**Transition**: for each valid `(c1, c2)` pair, try all 9 combinations of moves `(dc1, dc2) ∈ {-1,0,1}²`. For each valid new pair `(nc1, nc2)`:
```
cherries = grid[row][nc1] + (nc1 != nc2 ? grid[row][nc2] : 0)
new_dp[nc1][nc2] = max(..., dp[c1][c2] + cherries)
```

**Initialization**: `dp[0][cols-1] = grid[0][0] + grid[0][cols-1]` (robot 1 starts at col 0, robot 2 at col cols-1).

**Answer**: max value in `dp` after the last row.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CherryPickup(int[][] grid) {
        int rows = grid.Length, cols = grid[0].Length;
        const int NEG_INF = int.MinValue / 2;

        int[,] dp = new int[cols, cols];
        for (int a = 0; a < cols; a++)
            for (int b = 0; b < cols; b++)
                dp[a,b] = NEG_INF;
        dp[0, cols-1] = grid[0][0] + grid[0][cols-1];

        for (int row = 1; row < rows; row++) {
            int[,] newDp = new int[cols, cols];
            for (int a = 0; a < cols; a++)
                for (int b = 0; b < cols; b++)
                    newDp[a,b] = NEG_INF;

            for (int c1 = 0; c1 < cols; c1++) {
                for (int c2 = 0; c2 < cols; c2++) {
                    if (dp[c1,c2] == NEG_INF) continue;
                    for (int dc1 = -1; dc1 <= 1; dc1++) {
                        for (int dc2 = -1; dc2 <= 1; dc2++) {
                            int nc1 = c1+dc1, nc2 = c2+dc2;
                            if (nc1 < 0 || nc1 >= cols || nc2 < 0 || nc2 >= cols) continue;
                            int cherries = grid[row][nc1];
                            if (nc1 != nc2) cherries += grid[row][nc2];
                            newDp[nc1,nc2] = Math.Max(newDp[nc1,nc2], dp[c1,c2] + cherries);
                        }
                    }
                }
            }
            dp = newDp;
        }

        int res = 0;
        for (int c1 = 0; c1 < cols; c1++)
            for (int c2 = 0; c2 < cols; c2++)
                if (dp[c1,c2] != NEG_INF)
                    res = Math.Max(res, dp[c1,c2]);
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(rows × cols² × 9)** = **O(rows × cols²)** | rows × cols² states × 9 transitions |
| **Space** | **O(cols²)** | Rolling DP, only two rows needed |

With rows=cols=70: 70 × 70² × 9 ≈ 3M operations — instant.

---

## Example Walkthrough

**`grid=[[3,1,1],[2,5,1],[1,5,5],[2,1,1]]`**, cols=3.

Row 0: dp[0][2] = 3+1 = 4. All other entries = NEG_INF.

Row 1 (transition from row 0 to row 1):
From (c1=0, c2=2), all 9 moves to (nc1∈{0,1}, nc2∈{1,2}):
- (0,1): grid[1][0]+grid[1][1] = 2+5 = 7 → newDp[0][1] = 4+7=11
- (0,2): 2+1=3 → newDp[0][2] = 4+3=7
- (1,1): 5 (same cell) → newDp[1][1] = 4+5=9
- (1,2): 5+1=6 → newDp[1][2] = 4+6=10
... best: newDp[0][1]=11

Continuing through all rows: final max = **24** ✓

## Pitfalls & Edge Cases

- **Same cell (`nc1 == nc2`)**: count cherries only once — `cherries = grid[row][nc1]` without adding again.
- **`NEG_INF = int.MinValue/2`**: prevent overflow when adding `cherries` to `NEG_INF`.
- **Rolling DP**: create `newDp` each row and replace `dp` — O(cols²) space instead of O(rows×cols²).
- **Robots can start at the same column** if `cols == 1`: `dp[0][0] = grid[0][0]` — handled by the same-cell check.

---

## Key Takeaway

"Two entities moving synchronously through a grid" → **joint DP on both positions at each row**:

```
state: (c1, c2) = column positions of both robots
transition: try 9 move combinations, accumulate cherries (dedup same cell)
answer: max dp[c1][c2] after last row
```

O(rows × cols²) — efficient because both robots' states are combined in one DP table, processed row by row.
