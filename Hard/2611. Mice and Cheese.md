# 2611. Mice and Cheese
## O(n log n) — Greedy Marginal Gain (Upgrade Trick)

---

## Problem Statement

- Two mice, `n` types of cheese. Each cheese eaten by exactly one mouse.
- Cheese `i` gives `reward1[i]` if eaten by mouse 1, `reward2[i]` if eaten by mouse 2.
- Mouse 1 eats **exactly k** cheeses, mouse 2 eats the rest.
- Return the **maximum total points**.

---

## Core Idea — Greedy Marginal Gain

**Key insight**: Start by giving **all cheese to mouse 2** (base score = `Σ reward2`). Then "upgrade" exactly `k` cheeses to mouse 1. Each upgrade of cheese `i` changes the score by:

```
gain[i] = reward1[i] - reward2[i]
```

To maximize total points, greedily pick the **top k upgrades** (largest `gain[i]`, even if negative when forced).

**Why this works?**

```
Total = Σ reward2[i]  +  Σ (reward1[i] - reward2[i])  for the k chosen cheeses
      = base           +  sum of top-k gains
```

All cheese must be assigned; mouse 1 must eat exactly `k`. So we must pick exactly `k` upgrades — choosing the largest gains maximizes the total.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MiceAndCheese(int[] reward1, int[] reward2, int k) {
        int n = reward1.Length;
        int res = 0;

        // Gain from upgrading cheese i from mouse 2 → mouse 1
        int[] diff = new int[n];
        for (int i = 0; i < n; i++) {
            diff[i] = reward1[i] - reward2[i];
            res += reward2[i]; // base: all to mouse 2
        }

        // Pick top k gains
        Array.Sort(diff);
        for (int i = n - 1; i >= n - k; i--)
            res += diff[i];

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n log n)** | Sorting `diff`; sum passes are O(n) |
| **Space Complexity** | **O(n)** | `diff` array of size n |

**Note**: Can be O(n) using `nth_element` / partial sort, but O(n log n) is clean and sufficient for n ≤ 10⁵.

---

## Formula Breakdown

```
Total = Σ_{all i} reward2[i]           ← give everything to mouse 2
      + Σ_{top k} (reward1[i] - reward2[i])   ← upgrade k cheeses to mouse 1
```

Maximized by sorting `diff = reward1[i] - reward2[i]` descending and summing the first `k` entries.

---

## Example Walkthrough

**Example 1**: `reward1=[1,1,3,4]`, `reward2=[4,4,1,1]`, `k=2`

| i | reward1 | reward2 | diff |
|---|---------|---------|------|
| 0 | 1 | 4 | -3 |
| 1 | 1 | 4 | -3 |
| 2 | 3 | 1 | +2 |
| 3 | 4 | 1 | +3 |

`base = 4+4+1+1 = 10`

Sorted diff desc: `[3, 2, -3, -3]`

Top 2 gains: `3 + 2 = 5`

`res = 10 + 5 = 15` ✓

**Example 2**: `reward1=[1,1]`, `reward2=[1,1]`, `k=2`

`base = 2`, all diffs = 0, top 2 = 0 → `res = 2` ✓

---

## Pitfalls & Edge Cases

- **k = 0** — no upgrades, `res = Σ reward2` ✓ (loop `i >= n - 0 = n` never executes)
- **k = n** — all upgrades, `res = Σ reward1` ✓
- **Negative gains** — when `reward1[i] < reward2[i]`, upgrading that cheese hurts; but if forced (k is large), we still pick the least harmful ones last — sort handles this naturally
- **All equal rewards** — diff = 0 for all, any assignment gives the same total ✓

---

## Key Takeaway

"Assign items to two groups with fixed group sizes, maximize total" → **greedy marginal gain**:

- Assign everything to one group as a baseline
- Define `gain[i]` = score change from switching item `i` to the other group
- Sort by `gain` descending, take top `k`

Pattern: fixed-size partition optimization → reduce to "pick top k from a gain array".
