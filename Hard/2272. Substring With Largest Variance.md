# 2272. Substring With Largest Variance
## O(26² × n) — Kadane's Algorithm for Each Character Pair

---

## Problem Statement

- Variance of a string = (max frequency of any char) − (min frequency of any char present).
- Find the maximum variance over all substrings of `s`.

---

## Core Idea — Fix the Two Characters, Apply Kadane's

For a fixed pair `(a, b)`, the variance contribution from `a` and `b` in a substring = `count(a) - count(b)`. We want to maximize this over all substrings where **both `a` and `b` appear**.

Use **Kadane's algorithm** adapted with two variables:
- `diff` = running `count(a) - count(b)` (like standard Kadane, reset to 0 when negative)
- `diff_with_b` = same but constrained to substrings containing at least one `b`

**At each character `c`**:
- If `c == a`: `diff++`, `diff_with_b++` (if b has been seen)
- If `c == b`: `diff_with_b = diff - 1` (extend to include this new b), then `diff--`. If `diff < 0`: reset `diff = 0`, keep `diff_with_b = -1` (a fresh start including this b)

Update `res = max(res, diff_with_b)` whenever `diff_with_b` is valid (b has been seen).

Try all 26×25 = 650 pairs.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int LargestVariance(string s) {
        int res = 0;
        bool[] present = new bool[26];
        foreach (char c in s) present[c-'a'] = true;

        for (int ai = 0; ai < 26; ai++) {
            if (!present[ai]) continue;
            for (int bi = 0; bi < 26; bi++) {
                if (ai == bi || !present[bi]) continue;
                char a = (char)('a'+ai), b = (char)('a'+bi);
                int diff = 0, diffWithB = int.MinValue;

                foreach (char c in s) {
                    if (c == a) {
                        diff++;
                        if (diffWithB != int.MinValue) diffWithB++;
                    } else if (c == b) {
                        diffWithB = diff - 1; // include this b
                        diff--;
                        if (diff < 0) {
                            diff = 0;
                            diffWithB = -1; // restart but keep this b
                        }
                    }
                    if (diffWithB != int.MinValue)
                        res = Math.Max(res, diffWithB);
                }
            }
        }
        return res;
    }
}
```

| **Space** | **O(1)** | Only a few variables |

## Why `diff_with_b = diff - 1` When Seeing b?

When we encounter `b`, we want to extend the current segment to include this `b`. The current `diff` is `count(a) - count(b)` **before** adding this `b`. After adding: `diff - 1`. This becomes the new `diff_with_b` because the segment now contains at least one `b`.

**Reset case** (`diff < 0` after encountering `b`): the current segment has more `b`s than `a`s — not useful. Restart from this `b`: `diff = 0`, `diff_with_b = -1` (0 a's, 1 b = -1 balance, but b is present).

## Example Walkthrough

**`s = "aababbb"`, pair (a='b', b='a')**

| c | diff | diff_with_b | res |
|---|------|------------|-----|
| a | 0 | -1 (reset) | 0 |
| a | 0 | -1 | 0 |
| b | 1 | — | — |

Better pair is (a='b', b='a') where we count b's minus a's:

| c | diff (b-a) | diff_with_a | res |
|---|-----------|------------|-----|
| a | -1→0 | -1 | 0 |
| a | -1→0 | -1 | 0 |
| b | 1 | — | — |
| ...

For pair (a='b', b='a') in "aababbb": the substring "babbb" has 4 b's and 1 a → variance = 4-1 = **3** ✓

---

## Pitfalls & Edge Cases

- **`diff_with_b` initialization**: use `int.MinValue` (not 0) to indicate "no b seen yet". Updating with 0 would incorrectly count substrings without b.
- **`diff_with_b++` only if valid**: skip update when `diff_with_b == int.MinValue`.
- **Skip absent characters**: check `present[]` to avoid unnecessary pairs.

---

## Key Takeaway

"Max variance over all substrings" → **fix 2 chars (a,b), run Kadane to maximize count(a)-count(b) in substrings containing both**. Two-variable Kadane: `diff` tracks balance, `diff_with_b` tracks best balance in segments containing b. O(26² × n).
