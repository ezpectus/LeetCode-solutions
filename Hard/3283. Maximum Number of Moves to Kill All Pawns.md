# 3283. Maximum Number of Moves to Kill All Pawns
## O(n² · 2^n + n · B²) — BFS Preprocessing + Bitmask Minimax DP

---

## Problem Statement

Knight at `(kx, ky)`, `n ≤ 15` pawns on a 50×50 board. Alice and Bob alternate (Alice first). Each turn the player picks any remaining pawn and captures it with the knight in minimum moves. Alice **maximizes** total moves; Bob **minimizes**. Return the optimal total.

---

## Core Insight #1 — State = (Remaining Pawns Bitmask, Current Knight Position)

After each capture, the knight sits on the captured pawn's square. So the game state is fully described by:
- `mask`: bitmask of remaining (uncaptured) pawns
- `cur`: index of the current knight position (0..n-1 = pawn indices; n = starting position)

Whose turn it is is **implicitly determined** by `mask`:
```
captured = n - popcount(mask)
Alice's turn iff captured % 2 == 0
```

---

## Core Insight #2 — Precompute All Pairwise Knight Distances via BFS

Before the DP, run BFS from each of the n+1 "anchor" positions (n pawns + starting square) to compute `dist[i][j]` = minimum knight moves from position `i` to position `j`.

Knight BFS on a 50×50 board: O(50²) per source = O(n × 50²) total. With n=15, this is 37,500 operations — negligible.

---

## Core Insight #3 — Top-Down Minimax DP

```
Solve(mask, cur):
  if mask == 0: return 0

  Alice's turn if (n - popcount(mask)) % 2 == 0

  For each pawn i in mask:
    val = dist[cur][i] + Solve(mask ^ (1<<i), i)

  return max(val) if Alice, else min(val)
```

**State count:** O(2^n × n) = O(2^15 × 16) ≈ 524K states.
**Per state:** O(n) = O(15) transitions.
**Total DP work:** O(n² × 2^n) ≈ 7.4M operations.

## Full Implementation (C#)

```csharp
public class Solution {
    public int MaxMoves(int kx, int ky, int[][] positions) {
        int n = positions.Length;

        // pts[0..n-1] = pawns, pts[n] = starting position
        int[][] pts = new int[n + 1][];
        for (int i = 0; i < n; i++) pts[i] = positions[i];
        pts[n] = new int[] { kx, ky };

        int[] dx = { -2, -2, -1, -1,  1,  1,  2,  2 };
        int[] dy = { -1,  1, -2,  2, -2,  2, -1,  1 };

        // BFS from each anchor position → min knight moves to every other anchor
        int[,] dist = new int[n + 1, n + 1];
        for (int src = 0; src <= n; src++) {
            int[,] d = new int[50, 50];
            for (int i = 0; i < 50; i++)
                for (int j = 0; j < 50; j++) d[i, j] = -1;

            int sx = pts[src][0], sy = pts[src][1];
            d[sx, sy] = 0;
            var q = new Queue<(int, int)>();
            q.Enqueue((sx, sy));
            while (q.Count > 0) {
                var (x, y) = q.Dequeue();
                for (int k = 0; k < 8; k++) {
                    int nx = x + dx[k], ny = y + dy[k];
                    if (nx >= 0 && nx < 50 && ny >= 0 && ny < 50 && d[nx, ny] == -1) {
                        d[nx, ny] = d[x, y] + 1;
                        q.Enqueue((nx, ny));
                    }
                }
            }
            for (int dst = 0; dst <= n; dst++)
                dist[src, dst] = d[pts[dst][0], pts[dst][1]];
        }

        // dp[mask][cur] = optimal total moves; -1 = uncomputed
        int[,] dp = new int[1 << n, n + 1];
        for (int i = 0; i < (1 << n); i++)
            for (int j = 0; j <= n; j++) dp[i, j] = -1;

        int Solve(int mask, int cur) {
            if (mask == 0) return 0;
            if (dp[mask, cur] >= 0) return dp[mask, cur];

            int remaining = System.Numerics.BitOperations.PopCount((uint)mask);
            bool aliceTurn = (n - remaining) % 2 == 0;
            int best = aliceTurn ? int.MinValue : int.MaxValue;

            for (int i = 0; i < n; i++) {
                if ((mask & (1 << i)) == 0) continue;
                int val = dist[cur, i] + Solve(mask ^ (1 << i), i);
                best = aliceTurn ? Math.Max(best, val) : Math.Min(best, val);
            }

            return dp[mask, cur] = best;
        }

        return Solve((1 << n) - 1, n);
    }
}
```

## Complexity

| Phase | Time | Notes |
|---|---|---|
| BFS preprocessing | O(n · B²) | n+1 sources × B=50 board size |
| DP states | O(n · 2^n) | n+1 positions × 2^n masks |
| DP transitions | O(n² · 2^n) | n choices per state |
| **Total** | **O(n² · 2^n + n · B²)** | ≈ 7.4M + 37.5K = trivially fast |
| Space | O(n · 2^n + B²) | dp table + per-BFS grid |

## State Machine Diagram

```
State: (mask, cur)
  mask = bitmask of remaining pawns
  cur  = current knight position (index into pts[])

  popcount(mask) = remaining pawns
  captured = n - remaining
  Turn: Alice if captured even, Bob if captured odd

  Transition: pick pawn i from mask
    → cost = dist[cur][i]
    → new state = (mask ^ (1<<i), i)
    → total = cost + Solve(mask ^ (1<<i), i)

  Alice: max over all i
  Bob:   min over all i

  Base: mask = 0 → return 0
```

| (0,2) | (3,3) | 2 | (0,2)→(2,1)→(3,3) |

| (3,3) | (1,1) | 4 | (3,3)→(4,1)→(2,2)→(0,3)→(1,1) |

**DP trace for `Solve(111, 3)` — Alice's turn:**

| Option | Cost formula | Value |
|---|---|---|
| Pick pawn 0 (1,1) | dist[3][0] + Solve(110, 0) | 2 + 4 = 6 |
| Pick pawn 1 (2,2) | dist[3][1] + Solve(101, 1) | 2 + 6 = **8** |
| Pick pawn 2 (3,3) | dist[3][2] + Solve(011, 2) | 2 + 4 = 6 |

Alice chooses max = **8** ✓

**Sub-state `Solve(101, 1)` — Bob's turn, knight at (2,2), pawns 0 and 2 remain:**

| Option | Cost | Value |
|---|---|---|
| Pick pawn 0 (1,1) | 2 + Solve(100, 0) | 2 + dist[0][2] = 2 + 4 = 6 |
| Pick pawn 2 (3,3) | 2 + Solve(001, 2) | 2 + dist[2][0] = 2 + 4 = 6 |

Bob minimizes = 6. Both choices give 6 (symmetric).

**Sub-state `Solve(011, 2)` — Bob's turn, knight at (3,3), pawns 0 and 1 remain:**

| Option | Cost | Value |
|---|---|---|
| Pick pawn 0 (1,1) | 4 + Solve(010, 0) = 4+2 | 6 |
| Pick pawn 1 (2,2) | 2 + Solve(001, 1) = 2+2 | **4** |

Bob minimizes = 4.

## Pitfalls & Edge Cases

- **`cur = n` is the starting position**, not a pawn index. The `pts[n]` entry is `(kx, ky)`. BFS is run from it exactly like the pawns. `dist[n][i]` gives the knight's initial distance to each pawn.

- **Turn determination from mask, not explicit state:** `aliceTurn = (n - popcount(mask)) % 2 == 0`. This works because pawns are captured strictly one at a time, so `n - popcount(mask)` = number of turns elapsed.

- **Memo check `dp[mask, cur] >= 0`:** The sentinel is -1 (uncomputed). Actual values are always ≥ 0 (sum of non-negative moves). This check correctly distinguishes uncomputed from computed states.

- **BFS completeness:** The 50×50 knight graph is connected — every square can reach every other. So `dist[i][j]` is always well-defined (never -1). The maximum knight distance between any two squares on a 50×50 board is at most ~10 moves.

- **Position exclusivity:** The problem guarantees all pawn positions are distinct and none coincides with the starting knight position. No special handling needed.

- **n=1:** Single pawn. Alice picks it in `dist[n][0]` moves. Bob has no turn. Answer = `dist[n][0]`. Handled naturally (Solve(1, n) = dist[n][0] + Solve(0, 0) = dist[n][0]).

---

## Key Takeaway

This is a **bitmask minimax game DP** with preprocessed distances. The structure: precompute all pairwise move costs (BFS), then do top-down DP where each state is (which pawns remain, current position) and the player alternates based on how many pawns have been captured.

**Pattern recognition:** "Two players alternately pick from n items (n ≤ 15), one player maximizes, other minimizes, item cost depends on current state" → bitmask DP with turn derived from popcount; precompute pairwise costs if needed. The 2^n state space makes this feasible for n ≤ 20.
