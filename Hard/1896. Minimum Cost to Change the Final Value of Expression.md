# 1896. Minimum Cost to Change the Final Value of Expression
## O(n) — Stack-Based Evaluation with (value, flip_cost) Pairs

---

## Problem Statement

- Boolean expression with `0`, `1`, `&`, `|`, `(`, `)`. No operator precedence — evaluate left-to-right, parentheses first.
- Each operation costs 1: flip `0↔1` or flip `&↔|`.
- Return minimum operations to make expression evaluate to the opposite value.

---

## Core Idea — Evaluate with (value, flip_cost) Pairs

Each sub-expression carries two values:
- `val` = its current boolean value
- `flip_cost` = minimum operations to make it evaluate to `1 - val`

**Leaf**: `'0'` → `(0, 1)`, `'1'` → `(1, 1)` (one flip to change the literal).

**Combine**: `A op B` with `A=(a, ca)` and `B=(b, cb)`:
- Current result `r = a op b`.
- To get `1-r`, try all `2^3 = 8` combinations of {flip A, flip B, flip op}:
  - Cost = `ca * flip_A + cb * flip_B + 1 * flip_op`
  - Check if resulting expression evaluates to `1-r`.
- Take the minimum valid cost as the new `flip_cost`.

**Parsing**: use two stacks — `val_stack` for `(value, flip_cost)` pairs, `op_stack` for operators and `(`. Apply pending operators after each complete sub-expression (leaf or closed parenthesis).

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperationsToFlip(string expression) {
        var vals = new Stack<(int v, int c)>();
        var ops  = new Stack<char>();

        void ApplyOp() {
            var (bv, bc) = vals.Pop();
            char op = ops.Pop();
            var (av, ac) = vals.Pop();
            int r = op == '&' ? av & bv : av | bv;
            char nop = op == '&' ? '|' : '&';
            int best = int.MaxValue;
            for (int fa = 0; fa < 2; fa++)
            for (int fb = 0; fb < 2; fb++)
            for (int fo = 0; fo < 2; fo++) {
                int a2 = fa == 1 ? 1-av : av;
                int b2 = fb == 1 ? 1-bv : bv;
                char o2 = fo == 1 ? nop : op;
                int cost = (fa==1?ac:0) + (fb==1?bc:0) + fo;
                int res = o2 == '&' ? a2&b2 : a2|b2;
                if (res != r) best = Math.Min(best, cost);
            }
            vals.Push((r, best));
        }

        foreach (char c in expression) {
            if (c == '0' || c == '1') {
                vals.Push((c-'0', 1));
                while (ops.Count > 0 && (ops.Peek()=='&'||ops.Peek()=='|') && vals.Count >= 2)
                    ApplyOp();
            } else if (c == '&' || c == '|') {
                ops.Push(c);
            } else if (c == '(') {
                ops.Push('(');
            } else { // ')'
                ops.Pop(); // remove '('
                if (ops.Count > 0 && (ops.Peek()=='&'||ops.Peek()=='|') && vals.Count >= 2)
                    ApplyOp();
            }
        }
        return vals.Peek().c;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Each character processed once; ApplyOp is O(8) = O(1) |
| **Space** | **O(n)** | Stack depth proportional to nesting |

## The 8-Combination Combine Logic

For `A op B` = `r`, to get `1-r`:

| flip_A | flip_B | flip_op | conditions | typical cases |
|--------|--------|---------|------------|---------------|
| 0 | 0 | 1 | `a (~op) b ≠ r` | change operator |
| 1 | 0 | 0 | `(1-a) op b ≠ r` | flip A only |
| 0 | 1 | 0 | `a op (1-b) ≠ r` | flip B only |
| 1 | 0 | 1 | combined | less common |
| 0 | 1 | 1 | combined | less common |
| 1 | 1 | 0 | `(1-a) op (1-b) ≠ r` | flip both |
| 1 | 1 | 1 | combined | rarely optimal |
| 0 | 0 | 0 | same as original → `r`, not `1-r` | skip |

We enumerate all 8, filter to those giving `1-r`, take minimum cost.

---

## Example Walkthrough

**`expression="1&(0|1)"`**

Parse: push `'1'`=(1,1). Push `'&'`. Push `'('`. Push `'0'`=(0,1). Push `'|'`. Push `'1'`=(1,1) → apply `|`: combine (0,1)|(1,1).

Combine `0|1` = 1. To get 0:
- flip A (0→1): 1|1=1 ≠ 0? No.
- flip B (1→0): 0|0=0 ✓. Cost=1.
- flip op (|→&): 0&1=0 ✓. Cost=1.
→ flip_cost=1. Push (1,1).

Pop `'('`. Now apply `'&'`: combine (1,1)&(1,1).

Combine `1&1` = 1. To get 0:
- flip A: 0&1=0 ✓. Cost=1(ca).
- flip B: 1&0=0 ✓. Cost=1(cb).
- flip op: 1|1=1 ≠ 0. Cost=1 but doesn't work.
→ flip_cost=1. Answer: **1** ✓

## Pitfalls & Edge Cases

- **Left-to-right evaluation**: apply operators eagerly as we encounter operands — no precedence rules.
- **After `)`**: remove matching `(` from op_stack, then apply ONE pending operator (not loop), since the group just became one operand.
- **After leaf**: apply ALL pending operators (loop), since a leaf may close multiple pending ops in a fully-right-associated expression like `a&b&c`.
- **`int.MaxValue` for initial best**: ensures we always take a valid option when one exists.

---

## Key Takeaway

"Min cost to flip boolean expression" → **evaluate with `(value, flip_cost)` pairs, combine via 8-combination search**:

```
leaf: (0, 1) or (1, 1)
combine A op B:
    r = a op b
    flip_cost = min cost over all (fa,fb,fo) ∈ {0,1}³ where (fa?1-a:a) (fo?~op:op) (fb?1-b:b) ≠ r
```

Stack-based left-to-right parsing handles parentheses and operator application correctly. O(n) total.
