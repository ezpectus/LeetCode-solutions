## 2827. Number of Beautiful Integers in the Range

### Problem Statement
You are given positive integers `low`, `high`, and `k`. A number is defined as **beautiful** if it meets both of the following conditions:  
* The count of even digits in the number is equal to the count of odd digits.  
* The number is divisible by `k`.  

Return the number of beautiful integers in the range `[low, high]`.  

---

### Core Insight — Digit DP with Remainder & Parity Balance
Counting numbers matching specific digit characteristics within a massive numerical range (10^9) is a classic indicator for **Digit Dynamic Programming (Digit DP)**. Instead of querying the bounded range `[low, high]` directly, we calculate `Solve(high) - Solve(low - 1)`.

#### DP State Variables
To define a complete state for our recursive traversal, we track five specific parameters:
* `idx`: The current digit position being placed (from left to right).
* `rem`: The running remainder of the constructed prefix modulo `k`.
* `balance`: The difference between the number of even digits and odd digits. We offset this value by 10 to avoid negative array indices (`balance = even_count - odd_count + 10`).
* `isLess`: A boolean flag checking if the prefix is already strictly less than the corresponding prefix of the bounding limit. If `false`, our choice for the current digit is bounded up to `limit[idx]`.
* `isStarted`: A boolean flag checking if we have placed any non-zero leading digits yet. This prevents leading zeros from being counted as valid even digits.

---

### Algorithm
1. Convert the upper limit into a string or digit array representation.
2. Initialize a 5-dimensional memoization table `memo[idx][rem][balance][isLess][isStarted]` with `-1`.
3. Recursively place digits from `0` to `9`:
   * If `isStarted` is `false` and the current digit is `0`, we are still placing leading zeros; the `balance` does not change, and the `rem` remains `0`.
   * If a valid digit is placed, we transition the remainder: `next_rem = (rem * 10 + digit) % k`.
   * We transition the balance: increment if the digit is even, decrement if it is odd.
4. **Base Case:** When `idx` reaches the length of the string, return `1` if `isStarted == true`, `rem == 0`, and `balance == 10` (meaning `even_count == odd_count`). Otherwise, return `0`.
5. Subtract `Solve(low - 1)` from `Solve(high)` to get the final count.

---

### Full Optimal Implementation (C#)
```csharp
public class Solution {
    // 4D array inside a jagged array: [rem, balance, isLess, isStarted]
    private int[][,,,] memo;
    private int K;

    public int NumberOfBeautifulIntegers(int low, int high, int k) {
        this.K = k;
        return Solve(high) - Solve(low - 1);
    }

    private int Solve(int num) {
        if (num <= 0) return 0;
        
        string s = num.ToString();
        int n = s.Length;
        
        // 3 commas inside the brackets defines a 4D array
        memo = new int[n][,,,];
        for (int i = 0; i < n; i++) {
            memo[i] = new int[K, 21, 2, 2];
            for (int r = 0; r < K; r++) {
                for (int b = 0; b <= 20; b++) {
                    memo[i][r, b, 0, 0] = -1; memo[i][r, b, 0, 1] = -1;
                    memo[i][r, b, 1, 0] = -1; memo[i][r, b, 1, 1] = -1;
                }
            }
        }

        return Dfs(0, 0, 10, 0, 0, s);
    }

    private int Dfs(int idx, int rem, int balance, int isLess, int isStarted, string s) {
        if (idx == s.Length) {
            return (isStarted == 1 && rem == 0 && balance == 10) ? 1 : 0;
        }

        if (memo[idx][rem, balance, isLess, isStarted] != -1) {
            return memo[idx][rem, balance, isLess, isStarted];
        }

        int limit = (isLess == 1) ? 9 : (s[idx] - '0');
        int count = 0;

        for (int d = 0; d <= limit; d++) {
            int nextIsLess = (isLess == 1 || d < limit) ? 1 : 0;
            int nextIsStarted = (isStarted == 1 || d > 0) ? 1 : 0;
            
            int nextBalance = balance;
            if (nextIsStarted == 1) {
                nextBalance += (d % 2 == 0) ? 1 : -1;
            }

            // Pruning: if remaining positions cannot bridge the balance gap back to 10, skip
            int remainingDigits = s.Length - 1 - idx;
            if (Math.Abs(nextBalance - 10) > remainingDigits) {
                continue;
            }

            int nextRem = (rem * 10 + d) % K;
            count += Dfs(idx + 1, nextRem, nextBalance, nextIsLess, nextIsStarted, s);
        }

        return memo[idx][rem, balance, isLess, isStarted] = count;
    }
}
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(log10(High) * K * 20 * 10) | The number of digits `D` is at most 10. The table parameters are bounded by `D * K * 21 * 2 * 2`. Since `K <= 20`, the state space contains around 16,800 states, each taking 10 transitions. Execution takes under a millisecond. |
| **Space** | O(log10(High) * K) | Memory scale matches the size of the 5D states table allocation plus call stack depths capped at 10 layers. |

### Example Walkthrough
#### `low = 10, high = 20, k = 3`
* `Solve(20)` triggers string exploration for maximum value `"20"`.
* **Digit 1 Options (idx = 0):**
  - `d = 0`: Treated as a leading zero, transitions to state `(idx=1, rem=0, balance=10, isLess=1, isStarted=0)`.
  - `d = 1`: Odd digit, `balance = 10 - 1 = 9`. Transitions to state `(idx=1, rem=1, balance=9, isLess=1, isStarted=1)`.
  - `d = 2`: Even digit, `balance = 10 + 1 = 11`. Transitions to state `(idx=1, rem=2, balance=11, isLess=0, isStarted=1)`.
* **Tracing down `d = 1` branch to `idx = 1`:**
  - `d = 2` (forms 12): Even digit updates balance back to `9 + 1 = 10`. `rem = (1 * 10 + 2) % 3 = 0`. Base case matched! Valid count increases.
  - `d = 8` (forms 18): Even digit updates balance back to `9 + 1 = 10`. `rem = (1 * 10 + 8) % 3 = 0`. Base case matched! Valid count increases.

**Total Output:** `2`. ✓

### Pitfalls & Edge Cases
* **Leading Zero Parity Trap:** A naive implementation might count a leading `0` as an even digit. For example, the number `5` represented with padding as `05` would appear to have one even and one odd digit. You must use the `isStarted` flag to skip parity changes until the first non-zero digit is placed.
* **Balance Bounds Mapping:** The total balance can swing from `-10` (all odd digits) to `+10` (all even digits). Offset the indexing layer by adding `10` to ensure your arrays don't throw out-of-bounds exceptions.

### Key Takeaway
Whenever you need to count integers up to a large threshold based on structural rules (like digit counts, sum of digits, or divisibility properties), use **Digit DP**. Treat the placement of digits as processing path steps in a tree, and look for invariant properties (like keeping an active mod or balance offset) to minimize your memoization tracking space.
