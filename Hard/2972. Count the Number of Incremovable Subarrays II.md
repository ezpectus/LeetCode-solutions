## 2972. Count the Number of Incremovable Subarrays II

### Problem Statement
You are given a 0-indexed array of positive integers `nums`. A subarray of `nums` is called **incremovable** if `nums` becomes **strictly increasing** after removing that subarray. 

Return the total number of incremovable subarrays of `nums`. Note that an empty array is considered strictly increasing, and you must select a non-empty subarray to remove.

---

### Core Insight — Two Pointers on Prefix and Suffix
When we remove a contiguous subarray from `nums`, the remaining elements consist of a (possibly empty) prefix and a (possibly empty) suffix. For the remaining array to be strictly increasing, three conditions must hold:
1. The remaining prefix must be strictly increasing.
2. The remaining suffix must be strictly increasing.
3. The last element of the prefix must be strictly less than the first element of the suffix: `nums[prefix_end] < nums[suffix_start]`.

Because the constraints are large (N <= 10^5), an O(N^2) solution will Time Out. We can optimize this to O(N) using a **Two-Pointer** approach.

#### Structural Strategy
1. **Find the longest valid prefix:** Move a pointer `left` from `0` onwards as long as `nums[left] < nums[left + 1]`. The longest valid prefix goes from index `0` to `left`.
2. **Handle a fully sorted array:** If `left == n - 1`, the entire array is already strictly increasing. Any subarray removed will leave a strictly increasing sequence. The total number of non-empty subarrays is `N * (N + 1) / 2`.
3. **Find the longest valid suffix:** Move a pointer `right` from `n - 1` backwards as long as `nums[right - 1] < nums[right]`. The longest valid suffix goes from index `right` to `n - 1`.
4. **Combine combinations:** We can iterate through every possible prefix end index `i` (from `-1` up to `left`, where `-1` represents an empty prefix). For each chosen prefix end `i`, we use a moving pointer `right` to find the first element in the suffix that satisfies `nums[i] < nums[right]`. Every suffix starting from this `right` up to `n` (where `n` represents an empty suffix) forms a valid pair.

---

### Algorithm
1. Check if N = 1. If so, return `1`.
2. Find the maximum index `left` such that the prefix `nums[0...left]` is strictly increasing.
3. If `left == n - 1`, return `(long)n * (n + 1) / 2`.
4. Find the minimum index `right` such that the suffix `nums[right...n-1]` is strictly increasing.
5. Initialize `ans = 0`.
6. Use a two-pointer combination loop:
   * Iterate `i` from `-1` to `left`.
   * While `right < n` and `i >= 0` and `nums[i] >= nums[right]`, increment `right`.
   * Add the number of valid suffixes to `ans`: `ans += (n - right + 1)`.
7. Return `ans`.

---

### Full Optimal Implementation (C#)
```csharp
using System;

public class Solution {
    public long IncremovableSubarrayCount(int[] nums) {
        int n = nums.Length;
        
        // Step 1: Find the longest strictly increasing prefix
        int left = 0;
        while (left < n - 1 && nums[left] < nums[left + 1]) {
            left++;
        }
        
        // If the entire array is already strictly increasing
        if (left == n - 1) {
            return (long)n * (n + 1) / 2;
        }
        
        // Step 2: Find the longest strictly increasing suffix
        int right = n - 1;
        while (right > 0 && nums[right - 1] < nums[right]) {
            right--;
        }
        
        long ans = 0;
        
        // Step 3: Combine valid prefixes and suffixes
        // i = -1 represents taking an empty prefix
        for (int i = -1; i <= left; i++) {
            while (right < n && i >= 0 && nums[i] >= nums[right]) {
                right++;
            }
            // All suffixes starting from 'right' up to 'n' (empty suffix) are valid
            ans += (n - right + 1);
        }
        
        return ans;
    }
}
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N) | Finding the maximum `left` and minimum `right` takes linear time. In the combination phase, `i` increases monotonically and `right` only moves forward, meaning each element is processed at most a constant number of times. |
| **Space** | O(1) | No extra memory or dynamic arrays are allocated. The search operates entirely using primitive pointers and in-place comparisons. |

### Example Walkthrough
#### `nums = [6, 5, 7, 8]`
* **Find Prefix (`left`):** `nums[0] = 6`, `nums[1] = 5`. Since 6 >= 5, the loop stops immediately. `left = 0`. Valid prefix choices end at indices: `[-1, 0]`.
* **Find Suffix (`right`):** Traversing from right to left, `nums[2] < nums[3]` (7 < 8) and `nums[1] < nums[2]` (5 < 7). But `nums[0] >= nums[1]` (6 >= 5). The loop stops. `right = 1`. Valid suffix bounds start at indices: `[1, 2, 3, 4]`.
* **Iteration Trace:**
  - **`i = -1` (Empty Prefix):** No restriction on suffix elements. `right` stays at `1`. Valid suffix options are starting at `1, 2, 3, 4`. Count = `4 - 1 + 1 = 4`. (`ans = 4`).
  - **`i = 0` (`nums[0] = 6`):** We need `nums[right] > 6`.
    - Check `right = 1` (`nums[1] = 5`): 6 >= 5, so increment `right` to `2`.
    - Check `right = 2` (`nums[2] = 7`): 6 < 7, condition satisfied. Loop breaks.
    - Valid suffix options start at `2, 3, 4`. Count = `4 - 2 + 1 = 3`. (`ans = 4 + 3 = 7`).

**Total Output:** `7`. ✓

### Pitfalls & Edge Cases
* **Fully Sorted Arrays:** If the input is already sorted (e.g., `[1, 2, 3, 4]`), `left` will reach `n - 1`. If not caught early, the two-pointer combining loops will double-count indices or break bounds. Guard this early via `if (left == n - 1)`.
* **Integer Overflows:** The count of subarrays can be up to `(10^5 * (10^5 + 1)) / 2` which is around `5 * 10^9`. This value completely exceeds the limit of a signed 32-bit integer. Ensure the storage accumulator variables and return type values are marked as `long`.

### Key Takeaway
When a problem calls for counting contiguous segments whose removal leaves a globally valid property, look at what remains. Dividing the remaining state into an independent **Prefix** and **Suffix** decomposes the problem. Monotonicity allows you to replace a brute-force search with an optimized linear two-pointer search sweep.
