# 2514. Count Anagrams

## O(n) — Combinatorics with Modular Arithmetic

---

## Problem Statement

- Given string `s` with words separated by spaces.
- String `t` is an anagram of `s` if **each word** in `t` is a permutation of the corresponding word in `s`.
- Return count of distinct anagrams modulo 10⁹ + 7.
- **Constraints**: 1 ≤ s.length ≤ 10⁵, lowercase letters + spaces only.

---

## Core Idea — Permutations with Repetitions

**Key observation**: Anagrams of `s` = permutations of each word **independently** multiplied together.

**For a single word**: Count of distinct permutations with repeated characters:
```
permutations = n! / (count[char1]! × count[char2]! × ... × count[chark]!)
```
Where `n` = word length, `count[char]` = frequency of each character.

**For entire string**: 
```
total = product of permutations for each word
```

**Example**: `"too hot"`
- Word "too": 3 chars, 'o' appears 2 times → 3! / 2! = 6 / 2 = 3
- Word "hot": 3 chars, all unique → 3! / 1! = 6
- Total: 3 × 6 = 18 ✓

---

## Algorithm Breakdown

### Phase 1: Precompute Factorials and Modular Inverses

```
Precompute factorial[i] = i! mod MOD for i from 0 to n
Precompute invFactorial[i] = (i!)^(-1) mod MOD using Fermat's Little Theorem
```

**Fermat's Little Theorem**: For prime `p`, `a^(p-1) ≡ 1 (mod p)` → `a^(-1) ≡ a^(p-2) (mod p)`

**Modular inverse of k!**: `invFactorial[k] = factorial[k]^(MOD-2) mod MOD`

**Complexity**: O(n) for factorials + O(log MOD) for each inverse = O(n log MOD)

### Phase 2: Count Permutations Per Word

```
For each word in s:
  1. Count frequency of each character
  2. Calculate permutations = len! / (freq[a]! × freq[b]! × ...)
  3. Multiply to result

Using modular arithmetic:
  permutations = factorial[len] × invFactorial[freq[a]] × ... mod MOD
```

**Complexity**: O(n) for processing all characters

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    private const int MOD = 1_000_000_007;
    
    public int CountAnagrams(string s) {
        int maxLen = s.Length;
        
        // Precompute factorials and inverse factorials
        long[] factorial = new long[maxLen + 1];
        long[] invFactorial = new long[maxLen + 1];
        
        factorial[0] = 1;
        for (int i = 1; i <= maxLen; i++) {
            factorial[i] = factorial[i - 1] * i % MOD;
        }
        
        invFactorial[maxLen] = ModPow(factorial[maxLen], MOD - 2, MOD);
        for (int i = maxLen - 1; i >= 0; i--) {
            invFactorial[i] = invFactorial[i + 1] * (i + 1) % MOD;
        }
        
        // Process each word
        long result = 1;
        string[] words = s.Split(' ');
        
        foreach (string word in words) {
            int len = word.Length;
            
            // Count character frequencies
            int[] freq = new int[26];
            foreach (char c in word) {
                freq[c - 'a']++;
            }
            
            // Calculate permutations for this word
            long wordPerms = factorial[len];
            foreach (int count in freq) {
                if (count > 0) {
                    wordPerms = wordPerms * invFactorial[count] % MOD;
                }
            }
            
            result = result * wordPerms % MOD;
        }
        
        return (int)result;
    }
    
    private long ModPow(long baseNum, long exp, long mod) {
        long result = 1;
        baseNum %= mod;
        
        while (exp > 0) {
            if (exp % 2 == 1) {
                result = result * baseNum % mod;
            }
            baseNum = baseNum * baseNum % mod;
            exp /= 2;
        }
        
        return result;
    }
}
```

### Example 1: `s = "too hot"`

**Word "too"**:
- Length = 3
- Frequencies: t=1, o=2
- Permutations = 3! / (1! × 2!) = 6 / 2 = 3

**Word "hot"**:
- Length = 3
- Frequencies: h=1, o=1, t=1
- Permutations = 3! / (1! × 1! × 1!) = 6 / 1 = 6

**Total**: 3 × 6 = 18 ✓

### Example 2: `s = "aa"`

**Word "aa"**:
- Length = 2
- Frequencies: a=2
- Permutations = 2! / 2! = 2 / 2 = 1

**Total**: 1 ✓

## Mathematical Formula

For word with length `n` and character frequencies `f1, f2, ..., fk`:

```
Distinct permutations = n! / (f1! × f2! × ... × fk!)
```

**Intuition**: 
- Total permutations of n characters = n!
- Divide by fi! for each repeated character to remove duplicates

**Example**: "aab"
- Total arrangements = 3!
- But "aab" and "aab" are same (swapping the two 'a's)
- Distinct = 3! / 2! = 6 / 2 = 3 ("aab", "aba", "baa")

## Modular Arithmetic Details

### Why Modular Inverse?

Division in modular arithmetic: `(a / b) mod p ≠ (a mod p) / (b mod p)`

Instead: `(a / b) mod p = (a × b^(-1)) mod p`

Where `b^(-1)` is the modular inverse of b.

### Fermat's Little Theorem

For prime `p` and `a` not divisible by `p`:
```
a^(p-1) ≡ 1 (mod p)
a × a^(p-2) ≡ 1 (mod p)
Therefore: a^(-1) ≡ a^(p-2) (mod p)
```

### Precomputing Inverse Factorials

```
invFactorial[n] = (n!)^(-1) mod MOD
invFactorial[n] = factorial[n]^(MOD-2) mod MOD  // Using Fermat

For i < n:
invFactorial[i] = invFactorial[i+1] × (i+1) mod MOD
```

**Why this works**:
```
invFactorial[i+1] = 1 / (i+1)!
invFactorial[i+1] × (i+1) = 1 / i! = invFactorial[i]
```

---

## Pitfalls & Edge Cases

- **Modular arithmetic**: Every multiplication must include `% MOD` to prevent overflow.
- **Inverse factorials**: Must precompute using Fermat's theorem (division doesn't work in modular arithmetic).
- **Frequency counting**: Use array of size 26 for lowercase letters (not dictionary for speed).
- **Single character words**: Frequency = 1 → permutations = 1! / 1! = 1.
- **Empty words**: Not possible per constraints.
- **Overflow**: Use `long` for intermediate calculations before modulo.

---

## Alternative Approach — Computing Inverses On-the-Fly

Instead of precomputing all inverses, compute for each frequency:

```csharp
foreach (int count in freq) {
    if (count > 0) {
        long inv = ModPow(factorial[count], MOD - 2, MOD);
        wordPerms = wordPerms * inv % MOD;
    }
}
```

**Trade-off**: 
- Saves O(n) space for invFactorial array
- Slower: O(k log MOD) per word where k = unique characters (up to 26)
- For this problem: precomputing is better (one inverse computation vs many)

---

## Key Takeaway

"Count anagrams with word boundaries" → **Permutations with repetitions per word, multiply results**. **Formula**:
```
Total = ∏(word_i! / ∏(freq_char_j!)) for each word
```

**Implementation**: Precompute factorials and modular inverses, then:
1. For each word: count character frequencies
2. Calculate permutations using formula in modular arithmetic
3. Multiply all word results

O(n log MOD) time, O(n) space. **Critical insight**: Use Fermat's Little Theorem for modular division, precompute inverses for efficiency.
