# 1036. Escape a Large Maze
## O(b²) — Limited BFS from Both Endpoints

---

## Problem Statement

- 10⁶×10⁶ grid. At most 200 blocked cells. Can source reach target?

---

## Core Idea — Maximum Enclosed Area Bound

**Key insight**: with at most `b` blocked cells, the maximum area they can enclose is `b²/2` (a right-triangle-shaped enclosure along a grid corner). With `b=200`, max enclosed area ≈ 20,000.

**Strategy**: do BFS from `source` (and separately from `target`) but **stop early** once we've visited more than `b²/2` cells without being stopped. If we visit that many cells, we've escaped any possible enclosure — the region is open to the infinite grid.

**Two BFS checks**:
1. BFS from `source`: either reaches `target` → True, OR visits > maxArea → source not enclosed → proceed. If terminates with ≤ maxArea cells → source **is enclosed** → False.
2. BFS from `target`: same. If target is also not enclosed → True (both are in open space and can meet).

If either is enclosed (and we didn't reach the other directly) → False.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool IsEscapePossible(int[][] blocked, int[] source, int[] target) {
        if (blocked.Length == 0) return true;

        var blockedSet = new HashSet<(int,int)>(blocked.Select(b => (b[0], b[1])));
        int N = 1_000_000;
        int maxArea = blocked.Length * blocked.Length / 2 + 1;
        int[] dx = {0,0,1,-1}, dy = {1,-1,0,0};

        bool Bfs(int[] start, int[] goal) {
            var visited = new HashSet<(int,int)> { (start[0], start[1]) };
            var q = new Queue<(int,int)>();
            q.Enqueue((start[0], start[1]));

            while (q.Count > 0) {
                var (x, y) = q.Dequeue();
                if (x == goal[0] && y == goal[1]) return true;
                if (visited.Count > maxArea) return true; // escaped enclosure

                for (int d = 0; d < 4; d++) {
                    int nx = x+dx[d], ny = y+dy[d];
                    if (nx>=0&&nx<N&&ny>=0&&ny<N
                        &&!blockedSet.Contains((nx,ny))
                        &&visited.Add((nx,ny)))
                        q.Enqueue((nx,ny));
                }
            }
            return false; // enclosed
        }

        return Bfs(source, target) && Bfs(target, source);
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(b²)** | Each BFS visits at most b²/2 cells; b ≤ 200 → ~20K cells |
| **Space** | **O(b²)** | Visited set |

The 10⁶×10⁶ grid is never explicitly traversed.

---

## Why b²/2 is the Maximum Enclosed Area

The maximum area enclosed by `b` cells along a grid forms a right isosceles triangle against a corner:

```
. . . B         B = blocked cells forming the hypotenuse
. . B .         The corner and triangle interior = enclosed area
. B . .
B . . .
```

A triangle with `b` cells on the hypotenuse encloses roughly `b²/2` interior cells. With `b=200`: max enclosed area ≈ 20,000.

If BFS visits more than `b²/2` cells, it has definitively escaped any possible enclosure by `b` cells → the region is unbounded.

---

## Why Two BFS Checks?

BFS from source reaching > maxArea cells means source is not enclosed. But source and target might still be in separate unenclosed regions if:

```
[source region] --- blocked wall --- [target region]
```

Both regions are "unenclosed" (each can expand infinitely) but they can't reach each other. By running BFS from both endpoints:
- If either fails (gets enclosed) → False
- If both escape → they're both in the infinite open region → can reach each other → True

Actually: if both succeed (escaped), they're both in the connected infinite region (which is one connected component) → True. The only way they can't reach each other is if one is enclosed.

## Example Walkthrough

**`blocked=[[0,1],[1,0]], source=[0,0], target=[0,2]`**

BFS from `[0,0]`: can go... nowhere (N=[0,1] blocked, E=[1,0] blocked, S/W out of bounds). Visits only 1 cell < maxArea=3. Returns False.

`Bfs(source,target) = False` → overall **False** ✓

---

## Pitfalls & Edge Cases

- **`blocked` empty**: return True immediately (no enclosure possible)
- **Source or target directly adjacent to blocked**: BFS handles naturally
- **`maxArea = b²/2 + 1`**: the `+1` ensures we correctly identify escaping (visited > maxArea means we're safe)
- **`visited.Add` returning bool**: C# HashSet.Add returns true if newly added — use as combined add+check

---

## Key Takeaway

"Escape on huge grid with few blocked cells" → **limited BFS from both endpoints**:

- With ≤ b blocked cells, max enclosed area ≤ b²/2
- If BFS visits > b²/2 cells → escaped → not enclosed
- Two BFS checks (source→target and target→source) handle the case where both are unenclosed but separated

The geometric bound on enclosed area is the key insight — makes an otherwise intractable problem (10¹² cells) solvable in O(b²).
