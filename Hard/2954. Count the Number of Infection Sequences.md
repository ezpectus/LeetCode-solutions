# 2954. Count the Number of Infection Sequences

## O(N) — Combinatorics with Factorial Inverses and Segment Analysis

### Problem Statement
You are given an integer `n` and an array `sick` sorted in increasing order, representing positions of infected people in a line of `n` people. At each step, one uninfected person adjacent to an infected person gets infected. This process continues until everyone is infected.

An infection sequence is the order in which uninfected people become infected, excluding those initially infected. Return the number of different infection sequences possible, modulo $10^9 + 7$.

---

### Core Idea — Segment Independence and Multi-Set Permutations
The initially infected individuals partition the remaining uninfected people into independent continuous segments. The total number of uninfected positions that need to be filled is $total = n - sick.length$.

We analyze three categories of uninfected segments:
1. **The Left-Most Border Segment:** Length $L = sick[0]$. Since there is no infected person to the left of this block, the virus can only propagate unidirectionally from right to left. There is only **$1$** unique way to infect this entire block.
2. **The Right-Most Border Segment:** Length $R = n - 1 - sick[sick.length - 1]$. Similarly, the virus propagates unidirectionally from left to right. There is only **$1$** unique way to infect this block.
3. **Internal Segments:** For any segment trapped between two infected individuals $sick[i]$ and $sick[i+1]$, the length is $len = sick[i+1] - sick[i] - 1$. At any step before the segment is fully consumed, the infection can advance from either the left bound or the right bound. This grants $2$ choices for each slot except the final remaining person, yielding $2^{len - 1}$ distinct internal sequences.

#### Merging Across Segments
While each internal segment has $2^{len - 1}$ options, the individual infection steps across all segments can be interleaved in any order. If we have segments of lengths $len_1, len_2, \dots, len_m$, the number of ways to interleave their steps is given by the multinomial coefficient:
$$\text{Interleave Ways} = \frac{total!}{len_1! \times len_2! \times \dots \times len_m!}$$

The absolute answer is the product of the interleaving configurations and the internal choice permutations:
$$\text{Total Sequences} = \frac{total!}{\prod (len_i!)} \times \prod_{internal} 2^{len_i - 1} \pmod{10^9 + 7}$$

Since we need to perform divisions modulo $10^9 + 7$, we compute the modular multiplicative inverse of the factorials using Fermat's Little Theorem ($a^{-1} \equiv a^{p-2} \pmod p$).

---

### Full Optimal Implementation (C#)

```csharp
using System;

public class Solution {
    private const int MOD = 1000000007;

    public int NumberOfSequence(int n, int[] sick) {
        int m = sick.Length;
        int totalUninfected = n - m;
        
        // Step 1: Precompute factorials, inverse factorials, and powers of 2 up to N
        long[] fact = new long[n + 1];
        long[] invFact = new long[n + 1];
        long[] pow2 = new long[n + 1];
        
        fact[0] = 1;
        invFact[0] = 1;
        pow2[0] = 1;
        
        for (int i = 1; i <= n; i++) {
            fact[i] = (fact[i - 1] * i) % MOD;
            pow2[i] = (pow2[i - 1] * 2) % MOD;
        }
        
        invFact[n] = Power(fact[n], MOD - 2);
        for (int i = n - 1; i >= 1; i--) {
            invFact[i] = (invFact[i + 1] * (i + 1)) % MOD;
        }
        
        long totalWays = fact[totalUninfected];
        long internalCombinations = 1;
        
        // Step 2: Process the left-most edge segment
        int leftEdge = sick[0];
        if (leftEdge > 0) {
            totalWays = (totalWays * invFact[leftEdge]) % MOD;
        }
        
        // Step 3: Process the right-most edge segment
        int rightEdge = n - 1 - sick[m - 1];
        if (rightEdge > 0) {
            totalWays = (totalWays * invFact[rightEdge]) % MOD;
        }
        
        // Step 4: Process all internal segments between infected pairs
        for (int i = 0; i < m - 1; i++) {
            int len = sick[i + 1] - sick[i] - 1;
            if (len > 0) {
                totalWays = (totalWays * invFact[len]) % MOD;
                internalCombinations = (internalCombinations * pow2[len - 1]) % MOD;
            }
        }
        
        // Step 5: Merge interleaving options with internal selections
        long ans = (totalWays * internalCombinations) % MOD;
        return (int)ans;
    }
    
    private long Power(long baseVal, long exp) {
        long res = 1;
        baseVal %= MOD;
        while (exp > 0) {
            if ((exp & 1) == 1) res = (res * baseVal) % MOD;
            baseVal = (baseVal * baseVal) % MOD;
            exp >>= 1;
        }
        return res;
    }
}
```

### Complexity

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | $O(N)$ | Precomputing the factorial arrays takes linear $O(N)$ time. Iterating through the `sick` segments checks each division boundary exactly once in constant mathematical processing steps. |
| **Space Complexity** | $O(N)$ | Allocated to store the `fact`, `invFact`, and `pow2` caches up to size $N + 1$. Easily passes within maximum execution limits. |

---

### Pitfalls & Edge Cases

* **Edge vs. Internal Segment Logic:** Do not multiply by $2^{len - 1}$ for the left-most or right-most segments! Since they touch the physical array boundaries, there is only $1$ fixed way to process them, not a bi-directional choice.
* **Modular Multiplicative Inverse:** Division under modulo constraints cannot be done directly. Forgetting to multiply by `invFact[len]` instead of dividing by `fact[len]` will corrupt the combinatorial calculation state.

---

### Key Takeaway

> When an interleaving problem can be broken down into isolated pockets of work, separate the local permutations of each pocket from the macro-level multi-set combination. Use precomputed combinations/factorial arrays to compute the macro-interleaving states in $O(1)$ per segment.

* **Pattern recognition:** "Interleave combinations of dynamic localized sequences with binary directional choices" -> Segment Partition Analysis + Multinomial Factorial Inverses.
