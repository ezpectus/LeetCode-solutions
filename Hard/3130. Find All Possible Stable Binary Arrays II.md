# 3130. Find All Possible Stable Binary Arrays II
## O(zero × one) — Same DP as 3129, Larger Constraints

---

## Problem Statement

Identical to **3129** — same stability condition, same algorithm.  
Only difference: `zero, one, limit ≤ 1000` instead of 200.

---

## Solution

Identical to [3129. Find All Possible Stable Binary Arrays I](./3129_stable_arrays.md). Copy the solution verbatim.

```csharp
public class Solution {
    const int MOD = 1_000_000_007;

    public int NumberOfStableArrays(int zero, int one, int limit) {
        long[,,] dp = new long[zero + 1, one + 1, 2];

        for (int i = 1; i <= Math.Min(zero, limit); i++)
            dp[i, 0, 0] = 1;
        for (int j = 1; j <= Math.Min(one, limit); j++)
            dp[0, j, 1] = 1;

        for (int i = 0; i <= zero; i++) {
            for (int j = 0; j <= one; j++) {
                if (i == 0 || j == 0) continue;

                dp[i, j, 0] = (dp[i-1, j, 0] + dp[i-1, j, 1]) % MOD;
                if (i - limit - 1 >= 0)
                    dp[i, j, 0] = (dp[i, j, 0] - dp[i-limit-1, j, 1] + MOD) % MOD;

                dp[i, j, 1] = (dp[i, j-1, 0] + dp[i, j-1, 1]) % MOD;
                if (j - limit - 1 >= 0)
                    dp[i, j, 1] = (dp[i, j, 1] - dp[i, j-limit-1, 0] + MOD) % MOD;
            }
        }

        return (int)((dp[zero, one, 0] + dp[zero, one, 1]) % MOD);
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(zero × one)** | Up to 10⁶ cells, O(1) per cell |
| **Space Complexity** | **O(zero × one)** | ~8 MB for `long[1001,1001,2]` — fine |

`1000 × 1000 = 10⁶` operations → well within time limits.

---

## Why No Changes Needed

The DP transition is O(1) per cell — no inner loop over `limit`. The subtraction `dp[i-limit-1][j][1]` is a single array access. Scaling from 200 to 1000 is linear: 5× more cells, 5× more time, still fast.
