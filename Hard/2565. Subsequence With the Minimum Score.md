# 2565. Subsequence With the Minimum Score
## O(n) — Prefix/Suffix Greedy Match + Single Pass Minimum

---

## Problem Statement

- You are given strings `s` and `t`. Remove any characters from `t` forming a contiguous range `[left, right]`. Score = `right − left + 1` (or 0 if nothing removed).
- Find the minimum score such that the remaining `t` is a subsequence of `s`.

---

## Core Insight — Split s at Every Position, Take the Cheapest Middle Removal

For a fixed "split point" `i` in `s` (between index `i−1` and `i`):
- Match `t`'s prefix greedily using `s[0..i−1]` → match `pre[i]` characters of `t`
- Match `t`'s suffix greedily using `s[i..n−1]` → `t[suf[i]..m−1]` is a valid suffix

The required deletion window in `t` is `[pre[i], suf[i] − 1]`, which has length `max(0, suf[i] − pre[i])`.

Minimize this over all split points `i = 0` to `n`.

**Two arrays:**
- `pre[i]` = number of `t` chars greedily matched in `s[0..i−1]` (left-to-right scan)
- `suf[i]` = leftmost start position in `t` such that `t[suf[i]..m−1]` is a subsequence of `s[i..n−1]` (right-to-left scan)

**Answer:** `min over i of max(0, suf[i] − pre[i])`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumScore(string s, string t) {
        int n = s.Length, m = t.Length;

        // pre[i] = chars of t matched greedily in s[0..i-1]
        int[] pre = new int[n + 1];
        int ti = 0;
        for (int i = 0; i < n; i++) {
            if (ti < m && s[i] == t[ti]) ti++;
            pre[i + 1] = ti;
        }

        // suf[i] = start j in t such that t[j..m-1] is subsequence of s[i..n-1]
        int[] suf = new int[n + 1];
        suf[n] = m;
        int tj = m - 1;
        for (int i = n - 1; i >= 0; i--) {
            if (tj >= 0 && s[i] == t[tj]) tj--;
            suf[i] = tj + 1;
        }

        // For each split point, the minimum deletion window length
        int res = m;
        for (int i = 0; i <= n; i++) {
            res = Math.Min(res, Math.Max(0, suf[i] - pre[i]));
        }

        return res;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n + m) | One left scan, one right scan, one pass to compute answer |
| Space Complexity | O(n) | Two arrays of size n+1 |

## Why This Works — Proof

**For any valid solution:** We remove some `t[left..right]`. The kept parts are `t[0..left−1]` (matched in some prefix of `s`) and `t[right+1..m−1]` (matched in the remaining suffix of `s`). This defines a natural split point `i` in `s`.

**For any split point `i`:** The best we can do is match as many t-prefix chars as possible (greedy = `pre[i]`), and as far-left in t's suffix as possible (greedy = `suf[i]`). The middle `t[pre[i]..suf[i]−1]` must be deleted.

If `pre[i] ≥ suf[i]`, there's an overlap or exact meeting → score 0.

The minimum over all split points is optimal.

## Pitfalls & Edge Cases

- **Score 0 (no deletion):** occurs when `suf[i] ≤ pre[i]` for some `i`, meaning the matched prefix and suffix already cover all of `t`. min clips to 0 via `Math.Max(0, ...)`.

- **`suf[n] = m`** — base case: using all of `s`, the suffix of `t` we still need to match starts at position `m` (nothing remaining), which is correct.

- **Entire t unmatchable:** if `s` has none of `t`'s characters, `pre[n] = 0` and `suf[0] = 0`, so `suf[0] − pre[0] = 0`. Wait — if suf[0] = 0, we need to delete t[0..-1] = nothing? But suf[0]=0 means t[0..m-1] is a subsequence of s[0..n-1]... which contradicts the premise. Let me re-check example 2: suf[0]=3, pre[0]=0. diff=3=m. ✓ And since suf[i]=m=3 for all i, and pre[i]=0 for all i, the answer is 3. ✓

- **No left segment:** `i=0` means no prefix used. Deletion = t[0..suf[0]−1].
- **No right segment:** `i=n` means no suffix used. Deletion = t[pre[n]..m−1], length = m − pre[n] = suf[n] − pre[n] = m − pre[n].

---

## Key Takeaway

"Minimum contiguous deletion from `t` to make it a subsequence of `s`" → **scan both directions, define split at every position in `s`, take min gap.**

The key is seeing that the problem decomposes cleanly: for any valid deletion `[left, right]`, there's a split point in `s` such that the left part of `t` is matched before it and the right part after. Pre/suffix greedy arrays let us find the cheapest such split in O(n).

**Pattern recognition:** "Minimum-cost deletion of a contiguous window" + "subsequence matching" → greedy prefix from left + greedy suffix from right + single pass over all split points.
