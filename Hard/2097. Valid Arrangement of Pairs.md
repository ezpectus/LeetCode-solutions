# 2097. Valid Arrangement of Pairs  
## O(n) — Eulerian Path in a Directed Graph (Hierholzer’s Algorithm)

---

## Problem Statement

You are given:

- A 2D array `pairs`, where each element is `[start, end]`
- Each pair represents a directed edge: `start → end`
- You must reorder all pairs so that:```end[i - 1] == start[i]```
- for every `i` from `1` to `pairs.length - 1`.
- You must return **any valid arrangement**.
- The input is guaranteed to have **at least one valid arrangement**.

---

## Core Idea — This Is Exactly an Eulerian Path

Each pair `[u, v]` is a directed edge `u → v`.

A valid arrangement is a sequence of edges such that:
```
u1 → v1
v1 → v2
v2 → v3
...
```

This is precisely the definition of an **Eulerian path** in a directed graph:

> A path that uses every edge exactly once.

Since the problem guarantees a valid arrangement exists, the graph satisfies Eulerian path conditions:

- All edges must be used exactly once
- At most one node has `outdegree = indegree + 1` (start)
- At most one node has `indegree = outdegree + 1` (end)
- All other nodes have equal in/out degrees

Thus we simply:

1. Build adjacency lists
2. Track indegree/outdegree
3. Find the Eulerian path start node
4. Run **Hierholzer’s algorithm**
5. Reverse the result (because Hierholzer builds it backwards)

---

## Algorithm

1. Build a graph:
   - `adj[u]` = stack/list of all outgoing edges (their endpoints)
   - Track `out[u]++` and `in[v]++`
2. Determine the start node:
   - If a node has `out[u] = in[u] + 1`, it must be the start
   - Otherwise, start from any node with outgoing edges
3. Run Hierholzer’s DFS:
   - While `adj[u]` is not empty:
     - Pop next edge `u → v`
     - DFS on `v`
   - Append `[u, last_v]` to result
4. Reverse the result
5. Return the sequence of pairs

This guarantees a valid arrangement.

## Full Implementation (C#)

```csharp
public class Solution {
    public int[][] ValidArrangement(int[][] pairs) {
        int n = pairs.Length;

        // adjacency list
        var adj = new Dictionary<int, Stack<int>>();
        var indeg = new Dictionary<int, int>();
        var outdeg = new Dictionary<int, int>();

        foreach (var p in pairs) {
            int u = p[0], v = p[1];

            if (!adj.ContainsKey(u)) adj[u] = new Stack<int>();
            adj[u].Push(v);

            if (!indeg.ContainsKey(v)) indeg[v] = 0;
            if (!indeg.ContainsKey(u)) indeg[u] = 0;
            if (!outdeg.ContainsKey(u)) outdeg[u] = 0;
            if (!outdeg.ContainsKey(v)) outdeg[v] = 0;

            outdeg[u]++;
            indeg[v]++;
        }

        // find start node
        int start = pairs[0][0];
        foreach (var kv in outdeg) {
            int node = kv.Key;
            int outd = kv.Value;
            int ind = indeg[node];

            if (outd == ind + 1) {
                start = node;
                break;
            }
        }

        // Hierholzer
        var res = new List<int[]>();
        DFS(start, adj, res);

        res.Reverse();
        return res.ToArray();
    }

    private void DFS(int u, Dictionary<int, Stack<int>> adj, List<int[]> res) {
        var stack = adj.ContainsKey(u) ? adj[u] : null;

        while (stack != null && stack.Count > 0) {
            int v = stack.Pop();
            DFS(v, adj, res);
            res.Add(new int[] { u, v });
        }
    }
}
```
## Complexity

- Time Complexity	O(n) — each edge processed exactly once
- Space Complexity	O(n) — adjacency lists + recursion stack
- Graph Type	Directed multigraph

  ## Pitfalls & Edge Cases

- Graph may have **multiple edges** between the same nodes → must use **stacks/lists**, not sets  
- **Start node must be chosen correctly** according to Eulerian path rules  
- **Hierholzer builds the path in reverse**, so the final result must be reversed  
- Node values may be **up to 1e9**, so we must use **dictionaries**, not arrays  
- Input **guarantees** a valid arrangement → no need to check connectivity or existence of Eulerian path

## Example

### Example 1

**Input:**
```[[5,1],[4,5],[11,9],[9,4]]```
**Graph:** ```11 → 9 → 4 → 5 → 1```
**Output:** ```[[11,9],[9,4],[4,5],[5,1]]```

A perfect Eulerian path.

---

## Key Takeaway

This problem is a **direct Eulerian path reconstruction** in a directed graph:

- Build the graph  
- Track in/out degrees  
- Find the correct start node  
- Run **Hierholzer’s algorithm**  
- Reverse the resulting path  

This yields a clean **O(n)** solution that handles up to **100k edges** efficiently.
