# 2088. Count Fertile Pyramids in a Land
## O(m×n) — Bottom-Up DP for Pyramid Height

---

## Problem Statement

- Count all pyramidal and inverse pyramidal plots (height ≥ 2) in a binary grid.

---

## Core Idea — DP on Max Pyramid Height at Each Apex

For **normal pyramids** (apex = topmost cell, growing downward):

`dp[r][c]` = height of the largest pyramid with apex at `(r,c)`.

Process **bottom-up** (last row to first):
- Last row: `dp[r][c] = grid[r][c]` (height 1 if fertile, 0 otherwise).
- Earlier rows: `dp[r][c] = min(dp[r+1][c-1], dp[r+1][c], dp[r+1][c+1]) + 1` if `grid[r][c]==1`.

Each `dp[r][c]` contributes `dp[r][c]-1` pyramids (heights 2, 3, ..., `dp[r][c]`).

For **inverse pyramids**: flip the grid upside down and apply the same algorithm.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CountPyramids(int[][] grid) {
        return Count(grid) + Count(grid.Reverse().ToArray());
    }

    int Count(int[][] g) {
        int rows = g.Length, cols = g[0].Length, res = 0;
        int[,] dp = new int[rows, cols];
        for (int c = 0; c < cols; c++) dp[rows-1, c] = g[rows-1][c];

        for (int r = rows-2; r >= 0; r--) {
            for (int c = 0; c < cols; c++) {
                if (g[r][c] == 0) { dp[r,c] = 0; continue; }
                int left  = c > 0       ? dp[r+1, c-1] : 0;
                int mid   = dp[r+1, c];
                int right = c < cols-1  ? dp[r+1, c+1] : 0;
                dp[r,c] = Math.Min(left, Math.Min(mid, right)) + 1;
                if (dp[r,c] > 1) res += dp[r,c] - 1;
            }
        }
        return res;
    }
}
```

| **Metric** | **Value** |

## Why dp[r][c]-1 Pyramids?

A cell with `dp[r][c] = h` is the apex of pyramids with heights 2, 3, ..., h — that's `h-1` distinct pyramids. Each additional height level defines a unique valid pyramid.

---

## Example Walkthrough

**`grid=[[0,1,1,0],[1,1,1,1]]`**

Normal pyramids (process bottom-up):

Last row dp = [1,1,1,1].

Row 0:
- c=0: g=0 → dp=0.
- c=1: g=1, min(0, dp[1][0]=1, dp[1][1]=1)+1=2. res+=1.
- c=2: g=1, min(dp[1][1]=1, dp[1][2]=1, dp[1][3]=1)+1=2. res+=1.
- c=3: g=0 → dp=0.

Normal count = **2** ✓. Inverse count = 0. Total = **2** ✓

---

## Pitfalls & Edge Cases

- **Boundary columns**: `c=0` has no left neighbor → treat as 0. Same for `c=cols-1`.
- **Only count `dp > 1`**: height 1 = single cell, not a valid pyramid (size > 1 required).
- **Inverse pyramids**: reversing rows transforms inverse pyramids into normal ones — same DP applies.

---

## Key Takeaway

"Count pyramidal plots" → **bottom-up DP: dp[r][c] = min of 3 neighbors below + 1. Each cell contributes dp-1 pyramids. Run twice (normal + flipped) for inverse**:

```
dp[r][c] = min(dp[r+1][c-1], dp[r+1][c], dp[r+1][c+1]) + 1
res += max(0, dp[r][c] - 1)
```
