# 2719. Count of Integers
## O(L · maxSum) — Digit DP with Prefix Complement

---

## Problem Statement

- Count integers `x` in `[num1, num2]` (huge numbers, up to 10²² as strings) where `min_sum ≤ digit_sum(x) ≤ max_sum`.
- Return count mod 10⁹+7.

---

## Core Idea — f(n) = Count in [0, n], Answer = f(num2) − f(num1 − 1)

Define `f(numStr)` = count of integers in `[0, numStr]` with digit sum in `[min_sum, max_sum]`.

Answer = `f(num2) − f(decrement(num1))`.

Since `num1` is a huge numeric string, decrementing requires string-level borrow logic (no native bignum needed — just simulate "subtract 1 with carry/borrow").

**Digit DP for f(numStr):**
- State: `(pos, sumSoFar, tight)`.
- `tight=1` means we're still bound by the digit at this position in `numStr`; `tight=0` means we're free to place 0-9.
- Prune early: if `sumSoFar > max_sum`, return 0 immediately (no need to continue).
- At the end (`pos == n`): valid iff `sumSoFar ≥ min_sum`.
- Memoize only non-tight states (tight states are rare — at most one per digit position).

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    const int MOD = 1_000_000_007;
    int n;
    string s;
    long[,,] memo;
    int minSum, maxSum;

    public int Count(string num1, string num2, int min_sum, int max_sum) {
        minSum = min_sum;
        maxSum = max_sum;

        int len = num2.Length;
        num1 = num1.PadLeft(len, '0');

        long count2 = CountUpTo(num2);
        string num1Minus1 = Decrement(num1);
        long count1 = CountUpTo(num1Minus1);

        long ans = (count2 - count1) % MOD;
        if (ans < 0) ans += MOD;
        return (int)ans;
    }

    long CountUpTo(string numStr) {
        n = numStr.Length;
        s = numStr;
        memo = new long[n + 1, maxSum + 1, 2];
        for (int i = 0; i <= n; i++)
            for (int j = 0; j <= maxSum; j++)
                for (int t = 0; t < 2; t++)
                    memo[i, j, t] = -1;

        return Dp(0, 0, 1);
    }

    long Dp(int pos, int sumSoFar, int tight) {
        if (sumSoFar > maxSum) return 0;
        if (pos == n) return sumSoFar >= minSum ? 1 : 0;

        if (tight == 0 && memo[pos, sumSoFar, tight] != -1)
            return memo[pos, sumSoFar, tight];

        int limit = tight == 1 ? s[pos] - '0' : 9;
        long total = 0;

        for (int d = 0; d <= limit; d++) {
            int newTight = (tight == 1 && d == limit) ? 1 : 0;
            total += Dp(pos + 1, sumSoFar + d, newTight);
        }
        total %= MOD;

        if (tight == 0) memo[pos, sumSoFar, tight] = total;
        return total;
    }

    string Decrement(string numStr) {
        char[] arr = numStr.ToCharArray();
        int i = arr.Length - 1;
        while (i >= 0 && arr[i] == '0') {
            arr[i] = '9';
            i--;
        }
        if (i >= 0) arr[i]--;
        return new string(arr);
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(L · maxSum) | L ≤ 23 digits, maxSum ≤ 400; ~9200 states × 10 transitions |
| Space Complexity | O(L · maxSum) | Memoization table |

Two calls to `CountUpTo` total. Effectively O(1) for given constraints.

## Example Walkthrough

**Example 1:** `num1="1", num2="12", min_sum=1, max_sum=8`

`len(num2)=2`. `num1` padded to `"01"`.

**f("12")** — count x in [0,12] with digitsum ∈ [1,8]:
- 0:sum0(excl), 1-8:sum1-8(incl, 8 nums), 9:sum9(excl), 10:sum1(incl), 11:sum2(incl), 12:sum3(incl)
- Total = 8 + 3 = **11**

**Decrement("01") = "00"**. **f("00")** — count x in [0,0]: sum0, excluded (min_sum=1) → **0**.

**Answer = 11 − 0 = 11** ✓

---

**Example 2:** `num1="1", num2="5", min_sum=1, max_sum=5`

**f("5")** — x in [0,5]: sums 0,1,2,3,4,5. Exclude sum0 → values 1,2,3,4,5 → **5**.

**Decrement("1") = "0"**. **f("0")** — x=0, sum0, excluded → **0**.

**Answer = 5 − 0 = 5** ✓

## Pitfalls & Edge Cases

- **Padding num1 to match num2's length** — needed so both digit DP calls operate on equal-length strings; leading zeros don't change digit sum, so this is safe. ✓

- **Decrementing across multiple leading zeros** — e.g., `"100"` → `"099"`. The borrow loop correctly cascades through zeros. ✓

- **num1 = "1" (smallest possible):** after padding and decrementing, becomes all zeros, representing integer 0 — correctly excluded since `min_sum ≥ 1` always (per constraints). ✓

- **Early pruning `sumSoFar > maxSum`** — critical for performance; without it, the DP could explore many wasted branches, though even unpruned it would still fit since maxSum ≤ 400. Still good practice. ✓

- **Memoizing only `tight=0` states** — tight states are few (at most one tight path through the entire recursion), so they're cheap to recompute and don't need caching. ✓

---

## Key Takeaway

Classic **digit DP with prefix complement**: f(n) − f(m−1) pattern for range queries, where decrementing a big numeric string requires manual borrow simulation (no built-in bignum subtraction needed for strings).

**Pattern recognition:** "Count integers in [a, b] satisfying a digit-based property" → digit DP computing f(b) − f(a−1), with string-level decrement for the lower bound.
