# 🏔️ 3691. Maximum Total Subarray Value II — Architectural Solution

## Problem Statement

You are given an integer array `nums` of length `n` and an integer `k`. 
You need to select exactly `k` distinct subarrays `nums[l..r]`. Subarrays can overlap, but the exact same pair of indices (l, r) cannot be chosen more than once.

The value of a subarray is defined as:
Value = max(nums[l..r]) - min(nums[l..r])

The total value is the sum of the values of all `k` chosen subarrays. Return the maximum possible total value you can achieve.

### Constraints
- 1 <= n <= 5 * 10^4
- 0 <= nums[i] <= 10^9
- 1 <= k <= min(10^5, n * (n + 1) / 2)

---

## 💡 Core Idea

To find the top `k` largest subarray values efficiently without checking all O(n^2) pairs, the solution combines two powerful concepts:
1. **Sparse Table (RMQ)**: Precomputes range minimum and maximum queries in O(n log n) time, allowing us to calculate any subarray's value (max - min) in O(1) constant time.
2. **Max-Heap (Priority Queue)**: Efficiently tracks and extracts the largest available subarray values. 

Instead of pushing all possible subarrays into the heap, we initialize it with the largest possible right-bound intervals for each starting position `l`. When an interval [l, r] is extracted as a maximum, its next reduced sub-interval [l, r-1] is pushed back into the heap as a candidate. This keeps the heap size bounded by O(n) while guaranteed to find the top `k` values.

## 💻 Implementation (C++)

```cpp
#include <vector>
#include <cmath>
#include <queue>
#include <tuple>
#include <algorithm>
#include <bit>

using namespace std;

class SparseTable {
    vector<vector<int>> Min, Max;

public:
    SparseTable(const vector<int>& num) {
        size_t n = num.size();
        int w = bit_width(n);
        Min.resize(w, vector<int>(n));
        Max.resize(w, vector<int>(n));

        for (int i = 0; i < n; i++)
            Min[0][i] = Max[0][i] = num[i];

        for (int i = 1; i < w; i++)
            for (int j = 0; j + (1 << i) <= n; j++) {
                Min[i][j] = min(Min[i - 1][j], Min[i - 1][j + (1 << (i - 1))]);
                Max[i][j] = max(Max[i - 1][j], Max[i - 1][j + (1 << (i - 1))]);
            }
    }

    int query(int left, int right) {
        int k = bit_width((uint32_t)right - left) - 1;
        return max(Max[k][left], Max[k][right - (1 << k)]) -
               min(Min[k][left], Min[k][right - (1 << k)]);
    }
};

class Solution {
public:
    long long maxTotalValue(vector<int>& nums, int k) {
        int n = nums.size();
        long long res = 0;
        SparseTable LUT(nums);

        priority_queue<tuple<int, int, int>> pq;
        for (int i = 0; i < n; i++)
            pq.emplace(LUT.query(i, n), i, n);

        while (get<0>(pq.top()) && k--) {
            auto [val, l, r] = pq.top();
            pq.pop();
            res += val;
            pq.emplace(LUT.query(l, r - 1), l, r - 1);
        }

        return res;
    }
};
```

## 📐 Complexity Analysis

| Metric | Value | Justification |
| :--- | :--- | :--- |
| **Time Complexity** | O((n + k) * log n) | **Sparse Table Preprocessing**: Takes O(n log n) time.<br>**Heap Initialization**: Takes O(n log n) or O(n) depending on heapification method.<br>**Processing**: We pop and push from the Max-Heap k times. Each heap operation takes O(log n) time because the heap size never exceeds n. |
| **Space Complexity** | O(n * log n) | **Sparse Table Matrix**: Requires O(n log n) space to store min/max values for power-of-two intervals.<br>**Priority Queue**: Stores at most n states simultaneously, requiring O(n) space. |

## ⚠️ Pitfalls & Edge Cases

- **Zero-Value Subarrays**: Subarrays with a length of 1 or containing identical elements have a value of 0 (max - min = 0). The `while (get<0>(pq.top()) && k--)` condition breaks early if remaining values are 0, preventing useless additions.
- **Off-by-One in Ranges**: The `SparseTable` uses an exclusive right bound (`right`). This requires precise tracking when reducing intervals from `r` to `r - 1`.
- **Integer Overflow**: The accumulated result can easily exceed the 32-bit integer limit since values can be up to 10^9 and `k` up to 10^5. Using `long long` for `res` avoids truncation.

## 🔄 Alternative Approaches

- **Segment Tree**: Replacing the Sparse Table with a Segment Tree would reduce space complexity to O(n) but increase the range query time from O(1) to O(log n), driving the total runtime up to O(n + k * log n). Since n <= 5 * 10^4, the Sparse Table is a better tradeoff.
- **Brute Force**: Constructing all possible subarrays, computing their values, and sorting them takes O(n^2) space and time. This approach will crash with a Time Limit Exceeded (TLE) error for n = 5 * 10^4.
