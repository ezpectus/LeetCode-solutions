## 3113. Find the Number of Subarrays Where Boundary Elements Are Maximum

### Problem Statement
Given an array of positive integers `nums`, return the number of subarrays where the first and the last elements are equal to each other, and this boundary value is strictly the maximum element within that entire subarray.

### Core Insight — Monotonic Stack & Boundary Segments
For a subarray $[l, r]$ to satisfy the condition, three requirements must be met simultaneously:
1. `nums[l] == nums[r]` (Boundary elements must be equal).
2. `nums[l]` must be the maximum value in $[l, r]$.
3. Therefore, no element between $l$ and $r$ can be strictly greater than `nums[l]`. If any element $x \in (l, r)$ has $x > \text{nums[l]}$, then `nums[l]` is no longer the maximum of the subarray, invalidating it.

> **Architectural Constraint:** Instead of scanning pairs of identical elements, which would degrade the runtime to $\mathcal{O}(N^2)$, we can process the array sequentially using a **Monotonic Decreasing Stack**. The stack maintains elements alongside their valid contiguous frequency count.

Whenever we encounter a new element `nums[i]`:
- We must eject all elements from the stack that are strictly smaller than `nums[i]`. Why? Because `nums[i]` acts as a structural barrier: no element smaller than `nums[i]` located to its left can ever form a valid subarray extending to the right of $i$, as `nums[i]` violates the maximum condition.
- After clearing out smaller elements, if the top of the stack matches `nums[i]`, it means we have found a valid chain of identical values that haven't been blocked by a larger element. The number of valid subarrays ending at index $i$ is exactly equal to the existing count of this element plus 1 (for the single element subarray `[nums[i]]`).

---

### Algorithm
1. Initialize a stack to store pairs of integers: `{element_value, continuous_count}`.
2. Maintain a running accumulator variable `ans` typed as `long long` to prevent scalar overflow.
3. Iterate through `nums` from left to right. For each element `x`:
   - Pop elements from the stack while the stack is not empty and the top element's value is **strictly less** than `x`.
   - If the stack is not empty and the top element's value is **exactly equal** to `x`:
     - Increment its frequency counter: `top.count++`.
     - Add this updated counter to `ans`.
   - If the stack is empty or the top element's value is **strictly greater** than `x`:
     - Push a new pair `{x, 1}` onto the stack.
     - Add `1` to `ans`.
4. Return `ans`.

---

### Full Optimal Implementation (C++)
```cpp
#include <vector>
#include <stack>

using namespace std;

class Solution {
public:
    long long numberOfSubarrays(vector<int>& nums) {
        long long ans = 0;
        // Stack stores pairs: {element_value, valid_current_frequency}
        stack<pair<int, int>> st;

        for (int x : nums) {
            // Eject smaller elements that are blocked by x from expanding right
            while (!st.empty() && st.top().first < x) {
                st.pop();
            }

            // If an unblocked identical element is found at the top
            if (!st.empty() && st.top().first == x) {
                st.top().second++;
                ans += st.top().second;
            } else {
                // First occurrence or a fresh start after a larger element
                st.push({x, 1});
                ans += 1;
            }
        }

        return ans;
    }
};
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | $\mathcal{O}(N)$ | Amortized analysis: Each element is pushed onto the stack exactly once and popped at most once across the entire runtime loop. |
| **Space** | $\mathcal{O}(N)$ | Worst-case storage allocation when `nums` is strictly decreasing, requiring the stack to hold up to $N$ entries. |

### Example Walkthrough
#### `nums = [1,4,3,3,2]`
* **$x = 1$:** Stack is empty. Push `{1, 1}`. `ans += 1` $\rightarrow$ `ans = 1`.
* **$x = 4$:** `st.top().first (1) < 4`. Pop `{1, 1}`. Stack empty. Push `{4, 1}`. `ans += 1` $\rightarrow$ `ans = 2`.
* **$x = 3$:** `st.top().first (4) > 3`. No pop. Push `{3, 1}`. `ans += 1` $\rightarrow$ `ans = 3`.
* **$x = 3$:** `st.top().first == 3`. Match found! Increment count: `{3, 2}`. `ans += 2` $\rightarrow$ `ans = 5`. *(Subarrays counted: `[3]` at index 3, and `[3,3]` from index 2 to 3)*.
* **$x = 2$:** `st.top().first (3) > 2`. No pop. Push `{2, 1}`. `ans += 1` $\rightarrow$ `ans = 6`.

**Total Output:** `6`. ✓

### Pitfalls & Edge Cases
* **Clearing Non-Strict Maximums:** Ensure the pop condition is strictly `st.top().first < x` and **not** `<=`. If you pop equal values, you destroy the coordinate history needed to link pairs of equal boundaries.
* **Large Monotonic Duplicates:** If the array consists entirely of identical numbers (e.g., $10^5$ copies of `3`), the frequency tracking resolves values matching the combinatoric arithmetic $N \times (N+1) / 2$. Accumulating this sequentially requires `ans` to be a 64-bit integer type (`long long`) to completely avoid integer overflow bounds.

### Key Takeaway
When tasked with finding subarrays bound by equal elements where those boundaries are local maximums, any element strictly larger than the current item represents a permanent invalidation barrier for smaller numbers trailing behind it. A Monotonic Stack lets you dynamically wipe out dead historical elements while tracking active groups of equal numbers in $\mathcal{O}(1)$ updates per step.
