# 3579. Minimum Steps to Convert String with Operations

## 1. Constraints & Complexity Bottleneck Analysis

### Constraints:
- Length of strings: 1 <= word1.length == word2.length <= 100
- Character set: Lowercase English letters only ('a' - 'z').

### The Complexity Bottleneck:
The maximum length of the strings is small (N = 100), which heavily hints that a polynomial time complexity like O(N^2) or O(N^3) is acceptable. 
The main difficulty comes from the rule that we can partition the string into any number of contiguous, disjoint substrings, and perform operations within each partition independently. Within a single partition, characters can be reversed (at most once for the whole substring), swapped in pairs (each index can participate in at most one swap), and replaced (each index can be replaced at most once). 

Because the choice of partitioning affects the availability of operations (e.g., reversing or swapping across partition boundaries is forbidden), we must explore all possible valid partitions. This structural breakdown naturally lends itself to Dynamic Programming over intervals.

## 2. Algorithm Design & Mathematical Invariants

### 2.1 Core Concept
We can solve this problem using a 1D Dynamic Programming array where `dp[i]` represents the minimum number of operations required to transform the prefix of length `i` of `word1` into the prefix of length `i` of `word2`.

The base case is:
`dp[0] = 0`

For each position `j` from 1 to N, we can consider every possible start index `i` (from 0 to `j-1`) of the last substring partition. The transition is defined as:
`dp[j] = min(dp[j], dp[i] + calculate_cost(i, j - 1))`

Where `calculate_cost(start, end)` is a helper function that determines the minimum operations to transform the substring `word1[start..end]` into `word2[start..end]`.

### 2.2 Substring Cost Invariants (The Matching Theory)
For a fixed substring partition source `S` and target `T`:
1. **Reverse Option:** We can either evaluate the cost of transforming `S` directly to `T`, or pay a cost of `1` operation to reverse `S` first and then evaluate the cost of transforming the reversed `S` to `T`. We take the minimum of these two scenarios.
2. **Optimal Swaps vs. Replaces:** Once the orientation of `S` is fixed, we look at the character mismatches. A character at index `k` needs to change if `S[k] != T[k]`. 
   - A `replace` operation fixes 1 mismatched character at the cost of 1 operation.
   - A `swap` operation exchanges two characters at indices `u` and `v`. It costs 1 operation. Since each index can participate in at most one swap, a swap can fix at most 2 mismatched characters. 
   - To fix 2 characters and be optimal, a swap must satisfy: `S[u] == T[v]` and `S[v] == T[u]` where `S[u] != T[u]` and `S[v] != T[v]`. This converts a cost of 2 replaces into 1 swap, saving exactly 1 operation.
   - Swaps that only fix 1 character (e.g., `S[u] == T[v]` but `S[v] != T[u]`) cost 1 swap + 1 replace = 2 operations, yielding zero net savings compared to just replacing both characters.

Therefore, the maximum number of beneficial swaps is found by matching pairs of type `(X, Y)` with pairs of type `(Y, X)` where `X` and `Y` are distinct characters. Let `C(X, Y)` be the count of indices `k` where `S[k] == X` and `T[k] == Y`. The maximum number of profitable swaps for a specific unordered pair of characters `{X, Y}` is exactly:
`Savings = min(C(X, Y), C(Y, X))`

The total cost to transform `S` to `T` without a reverse operation is:
`Cost = Total_Mismatches - Total_Savings`

## 3. Full Optimal Implementation (C++)

```
// Dynamic Programming + String Partitioning + String Matching
#include <string>
#include <vector>
#include <algorithm>

class Solution {
private:
    int get_transformation_cost(const std::string& s, const std::string& t) {
        int mismatches = 0;
        int count[26][26] = {0};
        int n = s.length();

        for (int i = 0; i < n; ++i) {
            if (s[i] != t[i]) {
                mismatches++;
                count[s[i] - 'a'][t[i] - 'a']++;
            }
        }

        int savings = 0;
        for (int i = 0; i < 26; ++i) {
            for (int j = i + 1; j < 26; ++j) {
                if (count[i][j] > 0 && count[j][i] > 0) {
                    savings += std::min(count[i][j], count[j][i]);
                }
            }
        }

        return mismatches - savings;
    }

    int calculate_substring_cost(const std::string& w1, const std::string& w2, int start, int end) {
        std::string s = w1.substr(start, end - start + 1);
        std::string t = w2.substr(start, end - start + 1);

        // Case 1: Transform without reversing the substring
        int cost_without_reverse = get_transformation_cost(s, t);

        // Case 2: Transform with reversing the substring (costs 1 extra operation)
        std::reverse(s.begin(), s.end());
        int cost_with_reverse = 1 + get_transformation_cost(s, t);

        return std::min(cost_without_reverse, cost_with_reverse);
    }

public:
    int minOperations(std::string word1, std::string word2) {
        int n = word1.length();
        std::vector<int> dp(n + 1, 1e9);
        dp[0] = 0;

        for (int j = 1; j <= n; ++j) {
            for (int i = 0; i < j; ++i) {
                int current_cost = calculate_substring_cost(word1, word2, i, j - 1);
                dp[j] = std::min(dp[j], dp[i] + current_cost);
            }
        }

        return dp[n];
    }
};
```

## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The overall execution time is determined by the nested DP loops and the cost computation helper function:
T_total = T_dp_loops * T_cost_computation

1. **DP State Space & Loops:**
   - The outer loop runs N times (from 1 to N) to define the end of the current partition.
   - The inner loop runs j times for each outer loop iteration to test all possible starting positions of the partition.
   - The total number of state pairs (i, j) processed is the sum from 1 to N of j, which equals N * (N + 1) / 2 iterations. For N = 100, this evaluates to exactly 5,050 state transitions.

2. **Cost Computation Matrix (T_cost_computation):**
   - Inside each state, calculate_substring_cost creates a substring slice of length L = j - i.
   - It performs character-by-character scans of length L to count mismatches and populate a 26x26 frequency array, taking O(L) time.
   - It then performs a fixed-size nested loop over the alphabet matrix: 26 * 25 / 2 = 325 constant steps to compute maximum savings. This is a strict O(1) operation because the alphabet size is bounded and does not scale with N.
   - Reversing the substring takes O(L) time.
   - Therefore, computing the exact transformation cost for an interval of length L takes O(L) time.

Summing O(L) over all valid pairs (i, j) bounds the total time complexity by:
T_total = O(N * N * N) = O(N^3)

Given N = 100, N^3 = 1,000,000 operations, which effortlessly completes within the standard 2-second time limit (typically taking less than 5 milliseconds in C++).

### 4.2 Detailed Space Complexity Decomposition
- **DP Array:** A single 1D array of size N + 1 integers is allocated to store optimal prefix answers. This requires O(N) space.
- **Substring Storage:** Temporary string copies of size at most L (where L <= N) are created during the cost evaluation phase, utilizing O(N) space.
- **Frequency Matrix:** The count matrix has a fixed size of 26x26 integers, which takes O(1) auxiliary space.

Thus:
Space Complexity = O(N) auxiliary space.

### 4.3 Complexity Summary
| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(N^3) | Arises from N^2 loop states evaluating substring costs of length up to N. |
| **Space Complexity** | O(N) | Requires linear auxiliary space to store the DP table and substring slices. |

## Pitfalls & Edge Cases
- **Strict Single-Swap Rule:** A naive greedy approach might try to chain multiple swaps on the same index to fix cyclic character shifts (e.g., rotating 3 elements). The problem constraints strictly forbid this because each index can participate in at most one swap. The code correctly handles this by only matching disjoint, mutually beneficial pairs (X, Y) and (Y, X).
- **Mandatory Reverse Cost:** When evaluating the reversed substring option, we must explicitly add 1 to the operation count. Forgetting this would treat the structural reversal as free.
- **Base Case Initialization:** The dp array must be initialized to a sufficiently large infinity value (e.g., 1e9) to ensure that std::min transitions correctly discover the true minimal path without encountering integer overflow bugs (which would happen if using INT_MAX).

---

## Key Takeaway
When an array or string transformation allows independent operations on contiguous disjoint partitions, apply interval-based or prefix-based 1D Dynamic Programming to test all valid boundary allocations.

**Pattern recognition:** "divide word1 into one or more contiguous substrings ... minimum number of operations" -> Dynamic Programming + String Partitioning + String Matching
