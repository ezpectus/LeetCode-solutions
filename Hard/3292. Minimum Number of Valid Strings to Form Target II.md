# LeetCode 3292. Minimum Number of Valid Strings to Form Target II

## Problem Description
You are given an array of strings `words` and a string `target`. A string `x` is called **valid** if `x` is a **prefix** of any string in `words`.

Return the **minimum number** of valid strings that can be concatenated to form `target`. If it is not possible to form `target`, return `-1`.

---

## Method Explanation: Aho-Corasick / Z-Function + Greedy DP

The problem can be modeled as finding the minimum number of jumps to reach the end of the `target` string. From any position `i` in `target`, we can jump forward by `len` characters if the substring `target[i ... i + len - 1]` is a valid prefix of at least one string in `words`.

To solve this efficiently for **Target II** constraints:
1. **Find Maximum Prefix Matches:** For each position `i` in `target`, we need to know the *longest* prefix of any word in `words` that matches the substring starting at `target[i]`. We can calculate this using the **Z-algorithm** or **KMP (Prefix Function)** by concatenating each `word` with `target` (e.g., `word + "#" + target`).
2. **Greedy / BFS / DP Jump Array:** Once we have an array `max_match[i]` representing the maximum valid jump length from index `i`, the problem reduces to the classic "Jump Game" (LeetCode 45). We can use a greedy approach (or BFS tracking the furthest reachable index) to count the minimum number of steps to reach the end of `target`.

## C++ Source Code

```cpp
#include <vector>
#include <string>
#include <algorithm>

using namespace std;

class Solution {
public:
    int minValidStrings(vector<string>& words, string target) {
        int n = target.length();
        // max_match[i] stores the longest prefix of any word matching target starting at index i
        vector<int> max_match(n, 0);

        // Process each word using the Z-algorithm concept or KMP to find prefix matches against target
        for (const string& word : words) {
            // Form the temporary string for Z-algorithm pattern matching
            string s = word + "#" + target;
            int m = s.length();
            vector<int> z(m, 0);
            
            int l = 0, r = 0;
            for (int i = 1; i < m; ++i) {
                if (i <= r) {
                    z[i] = min(r - i + 1, z[i - l]);
                }
                while (i + z[i] < m && s[z[i]] == s[i + z[i]]) {
                    z[i]++;
                }
                if (i + z[i] - 1 > r) {
                    l = i;
                    r = i + z[i] - 1;
                }
            }

            // Map the Z-values back onto the target indices
            int word_len = word.length();
            for (int i = 0; i < n; ++i) {
                int z_idx = word_len + 1 + i;
                max_match[i] = max(max_match[i], z[z_idx]);
            }
        }

        // Greedy Jump Game phase to find the minimum segments required
        int ans = 0;
        int current_end = 0;
        int furthest = 0;

        for (int i = 0; i < n; ++i) {
            furthest = max(furthest, i + max_match[i]);
            
            // If we can't move forward from the current point
            if (i == furthest) {
                return -1;
            }

            if (i == current_end) {
                ans++;
                current_end = furthest;
                if (current_end >= n) {
                    return ans;
                }
            }
        }

        return current_end >= n ? ans : -1;
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(\sum |words[i]| + |words| \cdot |target|)$
  * **Z-Algorithm Processing:** For each word in the input array, we construct a temporary matching string of size $|words[i]| + 1 + |target|$ and execute the linear-time Z-algorithm on it.
  * **Constraint Scaling:** Given the problem constraints ($\sum |words[i]| \le 10^5$ and $|target| \le 5 \cdot 10^4$), the total operations scale efficiently and comfortably execute well within the standard time limit.

* **Space Complexity:** $\mathcal{O}(\max |words[i]| + |target|)$
  * **Z-Array Workspace:** The dynamic memory allocation is dominated by the tracking vectors and temporary concatenated strings built inside the loop workspace, which requires space proportional to the longest word plus the target length.
  * **State Tracking:** The `max_match` array requires fixed linear space of $\mathcal{O}(|target|)$ to store the lookahead jump indicators.
