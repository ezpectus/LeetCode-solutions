# 2251. Number of Flowers in Full Bloom
## O((n + m) log n) — Binary Search on Sorted Start/End Arrays

---

## Problem Statement

- Flowers bloom from `start` to `end` inclusive. For each person arriving at time `t`, count how many flowers are in full bloom.

---

## Core Idea — Count Started minus Count Ended

A flower is blooming at time `t` iff `start ≤ t AND end ≥ t`.

**Total blooming at t** = (flowers that started by t) − (flowers that ended before t):
```
blooming(t) = |{i : start[i] <= t}| - |{i : end[i] < t}|
            = |{i : start[i] <= t}| - |{i : end[i] <= t-1}|
```

Sort `starts` and `ends` separately. For each person, two binary searches (upper bound) give both counts in O(log n).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] FullBloomFlowers(int[][] flowers, int[] people) {
        int[] starts = flowers.Select(f => f[0]).OrderBy(x => x).ToArray();
        int[] ends   = flowers.Select(f => f[1]).OrderBy(x => x).ToArray();

        int[] res = new int[people.Length];
        for (int i = 0; i < people.Length; i++) {
            int t = people[i];
            res[i] = UpperBound(starts, t) - UpperBound(ends, t-1);
        }
        return res;
    }

    int UpperBound(int[] arr, int val) {
        int lo = 0, hi = arr.Length;
        while (lo < hi) {
            int mid = (lo+hi)/2;
            if (arr[mid] <= val) lo = mid+1; else hi = mid;
        }
        return lo;
    }
}
```

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O((n+m) log n)** | n = flowers, m = people; sort O(n log n) + m × O(log n) |
| **Space** | **O(n)** | Two sorted arrays |

## Example Walkthrough

**`flowers=[[1,6],[3,7],[9,12],[4,13]], people=[2,3,7,11]`**

starts sorted: [1,3,4,9], ends sorted: [6,7,12,13].

Person at t=2: started=UpperBound([1,3,4,9],2)=1, ended=UpperBound([6,7,12,13],1)=0. Blooming=**1** ✓
Person at t=7: started=UpperBound([1,3,4,9],7)=3, ended=UpperBound([6,7,12,13],6)=1. Blooming=**2** ✓

---

## Pitfalls & Edge Cases

- **`ends <= t-1` not `ends < t`**: use `UpperBound(ends, t-1)` to count flowers whose end is strictly before `t`. Inclusive bloom means `end = t` is still blooming.
- **Separate sorted arrays**: starts and ends are sorted independently — they don't correspond to the same flower anymore, which is fine since we only count.
- **`t-1` overflow**: `t ≥ 1` per constraints, so `t-1 ≥ 0` — safe.

---

## Key Takeaway

"Count intervals containing point t" → **sort starts and ends separately. `blooming(t) = upperBound(starts, t) - upperBound(ends, t-1)`**. O(log n) per query after O(n log n) preprocessing.
