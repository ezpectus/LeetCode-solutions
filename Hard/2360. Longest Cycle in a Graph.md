# 2360. Longest Cycle in a Graph
## O(n) — Iterative Walk with Visit Timestamps

---

## Problem Statement

- Each node has at most one outgoing edge (`edges[i]` or -1). Find the longest cycle. Return -1 if none.

---

## Core Idea — Visit Timestamps for Cycle Detection

For each unvisited node, walk along the chain stamping each node with a `timer` value. When we hit:
- A node with `visited = -1`: already fully processed, no cycle in this direction.
- A node with `visited >= start` (visited in this current walk): **cycle found!** Length = `timer - visited[cur]`.
- A node with `visited > 0` but from a previous walk (< start): no new cycle.

After each walk, mark all nodes in it as `-1` (done) to avoid re-processing.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int LongestCycle(int[] edges) {
        int n = edges.Length;
        int[] visited = new int[n]; // 0=unvisited, -1=done, >0=timestamp
        int res = -1, timer = 1;

        for (int i = 0; i < n; i++) {
            if (visited[i] != 0) continue;
            int start = timer, cur = i;

            // Walk and stamp
            while (cur != -1 && visited[cur] == 0) {
                visited[cur] = timer++;
                cur = edges[cur];
            }

            // Check if we found a cycle in this walk
            if (cur != -1 && visited[cur] >= start)
                res = Math.Max(res, timer - visited[cur]);

            // Mark all nodes from this walk as done
            cur = i;
            while (cur != -1 && visited[cur] > 0) {
                int next = edges[cur];
                visited[cur] = -1;
                cur = next;
            }
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — each node visited at most twice |
| **Space** | **O(n)** — visited array |

---

## Example Walkthrough

**`edges=[3,3,4,2,3]`**

Walk from 0: visited[0]=1→visited[3]=2→visited[2]=3→visited[4]=4→cur=3, visited[3]=2 >= start=1.
Cycle length = timer(5) - visited[3](2) = **3** ✓ (cycle: 2→4→3→2).

---

## Pitfalls & Edge Cases

- **`visited[cur] >= start`**: cycle must be from **this walk** (not a previous one). Use `start` as the lower bound for timestamps.
- **Cleanup pass**: mark all nodes in current walk as `-1` to prevent re-walking.
- **No cycle**: if `cur == -1` or `visited[cur] < start`, no cycle found in this walk.

---

## Key Takeaway

"Longest cycle in functional graph (each node has ≤ 1 outgoing edge)" → **iterate from each unvisited node, stamp with timer, detect cycle when revisiting current-walk node. Cycle length = timer - timestamp\_of\_cycle\_entry**. O(n).
