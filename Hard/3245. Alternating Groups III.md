# LeetCode 3245. Alternating Groups III

## Method Explanation

The problem requires us to dynamically maintain contiguous alternating groups in a circular array under single-element color updates. 

### 1. Representing Blocks via Violations
An alternating group is disrupted wherever two adjacent elements have the same color: `colors[i] == colors[(i + 1) % n]`. We call these indices **violations** or **splits**. 
If we keep track of all split indices in a sorted configuration (`std::set`), any two consecutive splits in the set define a maximal alternating block. For a maximal alternating block of length $L$, the number of alternating sub-groups of size $S$ it contains is given by:
$$\max(0, L - S + 1)$$

### 2. Querying Counts with a Fenwick Tree / Segment Tree
To efficiently answer size queries, we maintain a Fenwick tree (or Binary Indexed Tree) that tracks two frequencies for block lengths:
1. `cnt_tree`: Stores the number of blocks of length $L$.
2. `sum_tree`: Stores the total combined length of all blocks of length $L$.

When a query of type 1 asks for groups of size $S$, we can count the total contributions from all blocks with length $L \ge S$:
$$\text{Total} = \sum_{L \ge S} (L - S + 1) \cdot \text{cnt}(L) = \sum_{L \ge S} L \cdot \text{cnt}(L) - (S - 1) \sum_{L \ge S} \text{cnt}(L)$$
Using range sum queries on our Fenwick tree, this calculation takes $\mathcal{O}(\log N)$ time.

### 3. Dynamic Updates
When `colors[index]` changes, it affects at most two adjacency relations: with `index - 1` and `index + 1`. We look up adjacent splits in our `std::set`, remove the old blocks from our Fenwick tree tracking structures, mutate the split states, and insert the newly formed blocks.

## C++ Source Code

```cpp
#include <vector>
#include <set>
#include <numeric>
#include <algorithm>

using namespace std;

class Fenwick {
    int n;
    vector<long long> tree_cnt;
    vector<long long> tree_sum;

public:
    Fenwick(int n) : n(n), tree_cnt(n + 2, 0), tree_sum(n + 2, 0) {}

    void add(int idx, int val) {
        if (idx <= 0 || idx > n) return;
        long long delta_cnt = val;
        long long delta_sum = (long long)idx * val;
        for (; idx <= n; idx += idx & -idx) {
            tree_cnt[idx] += delta_cnt;
            tree_sum[idx] += delta_sum;
        }
    }

    pair<long long, long long> query_range(int left) {
        long long total_cnt = 0, total_sum = 0;
        auto query_prefix = [&](int idx) {
            long long c = 0, s = 0;
            for (; idx > 0; idx -= idx & -idx) {
                c += tree_cnt[idx];
                s += tree_sum[idx];
            }
            return make_pair(c, s);
        };
        auto right_res = query_prefix(n);
        auto left_res = query_prefix(left - 1);
        return {right_res.first - left_res.first, right_res.second - left_res.second};
    }
};

class Solution {
public:
    vector<int> numberOfAlternatingGroups(vector<int>& colors, vector<vector<int>>& queries) {
        int n = colors.size();
        Fenwick bit(n);
        set<int> splits;

        auto get_block_len = [&](int left_split, int right_split) {
            if (left_split < right_split) return right_split - left_split;
            return right_split + n - left_split;
        };

        auto add_split = [&](int idx) {
            if (splits.empty()) {
                // Transitioning from 0 splits to 1 split: 
                // Remove the virtual full circular block of length N
                bit.add(n, -1);
                splits.insert(idx);
                // The single split forms a block of length N with itself circularly
                bit.add(n, 1);
                return;
            }
            
            auto it = splits.lower_bound(idx);
            int nxt = (it == splits.end()) ? *splits.begin() : *it;
            int prv = (it == splits.begin()) ? *splits.rbegin() : *prev(it);

            bit.add(get_block_len(prv, nxt), -1);
            bit.add(get_block_len(prv, idx), 1);
            bit.add(get_block_len(idx, nxt), 1);
            splits.insert(idx);
        };

        auto remove_split = [&](int idx) {
            if (splits.size() == 1 && splits.count(idx)) {
                // Transitioning from 1 split to 0 splits:
                bit.add(n, -1);
                splits.erase(idx);
                // Array becomes completely alternating
                bit.add(n, 1);
                return;
            }

            auto it = splits.find(idx);
            auto nxt_it = next(it) == splits.end() ? splits.begin() : next(it);
            auto prv_it = it == splits.begin() ? prev(splits.end()) : prev(it);
            
            int nxt = *nxt_it;
            int prv = *prv_it;

            bit.add(get_block_len(prv, idx), -1);
            bit.add(get_block_len(idx, nxt), -1);
            bit.add(get_block_len(prv, nxt), 1);
            splits.erase(it);
        };

        // Seed initial split conditions
        for (int i = 0; i < n; ++i) {
            if (colors[i] == colors[(i + 1) % n]) {
                splits.insert(i);
            }
        }

        if (splits.empty()) {
            bit.add(n, 1);
        } else {
            int first = *splits.begin();
            int prev_split = first;
            for (auto it = next(splits.begin()); it != splits.end(); ++it) {
                bit.add(*it - prev_split, 1);
                prev_split = *it;
            }
            bit.add(first + n - prev_split, 1);
        }

        vector<int> ans;
        for (const auto& q : queries) {
            if (q[0] == 1) {
                int sz = q[1];
                if (splits.empty()) {
                    ans.push_back(n); 
                } else {
                    auto [cnt, sum] = bit.query_range(sz);
                    long long res = sum - (sz - 1) * cnt;
                    ans.push_back(res);
                }
            } else {
                int idx = q[1];
                int col = q[2];
                if (colors[idx] == col) continue;

                int left_neighbor = (idx - 1 + n) % n;
                int right_neighbor = (idx + 1) % n;

                if (colors[left_neighbor] == colors[idx]) remove_split(left_neighbor);
                if (colors[idx] == colors[right_neighbor]) remove_split(idx);

                colors[idx] = col;

                if (colors[left_neighbor] == colors[idx]) add_split(left_neighbor);
                if (colors[idx] == colors[right_neighbor]) add_split(idx);
            }
        }
        return ans;
    }
};
```


## Complexity Analysis

* **Time Complexity:** $\mathcal{O}((N + Q) \log N)$
  * **Fenwick Tree Initialization:** Building the initial state takes $\mathcal{O}(N \log N)$ time, as we process up to $N$ structural split intervals and insert their lengths into the tracking arrays.
  * **Type-1 Queries (Sizing Fetch):** Takes $\mathcal{O}(\log N)$ time. The Fenwick tree dynamically computes prefix counts and length sums. By executing a suffix range query via two standard prefix lookups, we instantly extract the aggregate group contributions for all blocks of length $L \ge S$.
  * **Type-2 Updates (Color Mutations):** Takes $\mathcal{O}(\log N)$ time. Changing a tile's color alters at most two adjacency relations. Inserting or removing boundary indices from `std::set` takes logarithmic time, and updating the corresponding frequencies in the Fenwick tree scales precisely with the bitwise tree traversal bounds.

* **Space Complexity:** $\mathcal{O}(N)$
  * **Static Linear Allocation:** The space complexity is strictly bounded by the size of the input. The `std::set` stores at most $N$ active split coordinates, and both internal Fenwick tree arrays (`tree_cnt` and `tree_sum`) scale up to $N + 2$ entries to account for maximum possible block lengths. This keeps the total auxiliary memory locked inside $\mathcal{O}(N)$.
