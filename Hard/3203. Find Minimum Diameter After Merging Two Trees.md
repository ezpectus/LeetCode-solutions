# 3203. Find Minimum Diameter After Merging Two Trees
## O(n + m) — Double-BFS Diameter + ⌈d/2⌉ Center Argument

---

## Problem Statement

Two trees with diameters `d1` and `d2`. Connect one node from each with a single edge. Return the minimum possible diameter of the resulting tree.

---

## Core Insight — Three Candidate Diameters

After merging, any path in the resulting tree falls into one of three categories:

1. **Entirely within tree 1:** max length = `d1`.
2. **Entirely within tree 2:** max length = `d2`.
3. **Crosses the bridge:** path goes from some node in tree 1, through the new edge, to some node in tree 2.

For case 3, the bridge crosses at the **connection nodes** `u ∈ tree1` and `v ∈ tree2`. The longest crossing path has length `maxDist(u, tree1) + 1 + maxDist(v, tree2)`. We choose `u` and `v` to minimize the worst case — the optimal choice is the **center** of each tree, which has eccentricity `⌈d/2⌉`.

```
min over (u,v) of (max crossing path) = ⌈d1/2⌉ + 1 + ⌈d2/2⌉
```

**Answer = max(d1, d2, ⌈d1/2⌉ + 1 + ⌈d2/2⌉)**

---

## Why the Center Minimizes Eccentricity

The **center** of a tree lies on the diameter path, at the midpoint. Every node in the tree is within distance `⌈d/2⌉` from the center. No connection point achieves a smaller maximum distance to the rest of the tree. (Proof: moving off the midpoint increases the distance to the far end of the diameter.)

In integer arithmetic: `⌈d/2⌉ = (d + 1) / 2`.

---

## Computing Tree Diameter in O(n): Double BFS

Standard two-BFS approach:
1. BFS from any node → find the farthest node `u`.
2. BFS from `u` → find the farthest node `v`. Distance `u→v` = diameter.

**Why it works:** The farthest node from any starting point is always a leaf endpoint of some diameter path. Two BFS runs provably find the diameter.

## Full Implementation (C#)

```csharp
public class Solution {
    public int MinimumDiameterAfterMerge(int[][] edges1, int[][] edges2) {
        int d1 = GetDiameter(edges1);
        int d2 = GetDiameter(edges2);
        int combined = (d1 + 1) / 2 + (d2 + 1) / 2 + 1; // ⌈d1/2⌉ + ⌈d2/2⌉ + 1 (bridge)
        return Math.Max(Math.Max(d1, d2), combined);
    }

    private int GetDiameter(int[][] edges) {
        if (edges.Length == 0) return 0; // single-node tree
        int n = edges.Length + 1;
        var adj = new List<int>[n];
        for (int i = 0; i < n; i++) adj[i] = new List<int>();
        foreach (var e in edges) {
            adj[e[0]].Add(e[1]);
            adj[e[1]].Add(e[0]);
        }
        int u = BFS(adj, 0, n).farthest;     // BFS 1: find a diameter endpoint
        return BFS(adj, u, n).maxDist;         // BFS 2: diameter = farthest distance from u
    }

    private (int farthest, int maxDist) BFS(List<int>[] adj, int start, int n) {
        var dist = new int[n];
        Array.Fill(dist, -1);
        var q = new Queue<int>();
        dist[start] = 0;
        q.Enqueue(start);
        int farthest = start, maxDist = 0;
        while (q.Count > 0) {
            int u = q.Dequeue();
            foreach (int v in adj[u]) {
                if (dist[v] == -1) {
                    dist[v] = dist[u] + 1;
                    if (dist[v] > maxDist) { maxDist = dist[v]; farthest = v; }
                    q.Enqueue(v);
                }
            }
        }
        return (farthest, maxDist);
    }
}
```

## Example Walkthrough

### Example 1 — `edges1=[[0,1],[0,2],[0,3]], edges2=[[0,1]]`

**Tree 1 (star, n=4):** Path 1→0→2 (or any pair of leaves) → `d1 = 2`.

**Tree 2 (path, n=2):** 0→1 → `d2 = 1`.

```
combined = ⌈2/2⌉ + ⌈1/2⌉ + 1 = 1 + 1 + 1 = 3
answer   = max(2, 1, 3) = 3 ✓
```

---

### Example 2 — Both trees identical, edges = [[0,1],[0,2],[0,3],[2,4],[2,5],[3,6],[2,7]]

**Tree (n=8):** Path `6→3→0→2→4` gives length 4. → `d1 = d2 = 4`.

```
combined = ⌈4/2⌉ + ⌈4/2⌉ + 1 = 2 + 2 + 1 = 5
answer   = max(4, 4, 5) = 5 ✓
```

The bridge connects the center (midpoint of diameter) of each tree.

---

## Pitfalls & Edge Cases

- **Single-node tree (n=1, edges=[]):** Diameter = 0. `GetDiameter` returns 0 immediately. The combined path = ⌈0/2⌉ + ⌈d2/2⌉ + 1 = 0 + ⌈d2/2⌉ + 1. Correct: connecting to a single node adds at most 1 to the bridge leg.

- **Both single-node trees:** d1=d2=0, combined=1. Answer=1. The merged tree is just one edge. ✓

- **`⌈d/2⌉ = (d+1)/2` in integer arithmetic.** Don't write `(d/2)` (floor), which underestimates for odd diameters. Example: d=3, center eccentricity=2, not 1.

- **Do not return `d1 + d2 + 1`** (the naive sum of diameters plus the bridge). This ignores that we're optimally placing the bridge at the tree centers.

---

## Key Takeaway

When merging two trees with one edge, the minimum achievable diameter is `max(d1, d2, ⌈d1/2⌉ + 1 + ⌈d2/2⌉)`. The three terms cover the three possible longest-path scenarios: fully in tree 1, fully in tree 2, or crossing the bridge. The `⌈d/2⌉` term comes from placing the bridge at the tree's center, which minimizes the farthest reach within that tree.

**Pattern recognition:** "Merge two structures with an edge, minimize resulting diameter" → compute individual diameters, use `⌈d/2⌉` as the optimal center-eccentricity contribution, take max of three candidates.
