# 1210. Minimum Moves to Reach Target with Rotations
## O(n²) — BFS on Snake State Space with 4D Position Tracking

---

## Problem Statement

Given an `n × n` grid where cells are either `0` (empty) or `1` (blocked).

A snake spans **2 cells**, starting horizontally at `(0,0)` and `(0,1)`.  
The goal is to reach `(n-1, n-2)` and `(n-1, n-1)` — the bottom-right corner.

In one move, the snake can:
- **Move right** — shift both cells one column to the right (if both destination cells are empty)
- **Move down** — shift both cells one row down (if both destination cells are empty)
- **Rotate clockwise** — only if horizontal; tail stays at `(r, c)`, head moves to `(r+1, c)` (requires both cells below to be empty)
- **Rotate counterclockwise** — only if vertical; tail stays at `(r, c)`, head moves to `(r, c+1)` (requires both cells to the right to be empty)

Return the **minimum number of moves** to reach the target, or `-1` if impossible.

---

## Core Idea — BFS on 4D State Space

The snake's position is fully described by **4 coordinates**: `(r1, c1, r2, c2)` — the tail and head cells.

We run a standard **BFS** (Breadth-First Search) from the initial state `(0, 0, 0, 1)`, exploring all valid transitions at each step.

We keep:
- A **Queue** of states `(r1, c1, r2, c2, moves)` — the BFS frontier
- A **HashSet** of visited state strings — to avoid re-processing the same position

At each state we check:
- Whether we reached the target → return current move count
- Whether the snake is **horizontal** (`r1 == r2`) — try moving right, down, and rotating clockwise
- Whether the snake is **vertical** (`c1 == c2`) — try moving right, down, and rotating counterclockwise

All transitions are guarded by boundary checks and blocked-cell checks.

## Clean Implementation (C#)

```csharp
public class Solution {
    public int MinimumMoves(int[][] grid) {
        int n = grid.Length;
        Queue<(int r1, int c1, int r2, int c2, int move)> q = 
            new Queue<(int, int, int, int, int)>();
        q.Enqueue((0, 0, 0, 1, 0));
        HashSet<string> map = new HashSet<string>();

        while (q.Count > 0) {
            var (r1, c1, r2, c2, move) = q.Dequeue();

            if (r1 == n-1 && c1 == n-2 && r2 == n-1 && c2 == n-1) return move;

            string key = $"R1C1-{r1}-{c1}-R2C2-{r2}-{c2}";
            if (map.Contains(key)) continue;
            map.Add(key);

            // Snake is horizontal (r1 == r2)
            if (r1 == r2) {
                // Move right
                if (c2+1 < n && grid[r1][c1+1] == 0 && grid[r2][c2+1] == 0)
                    q.Enqueue((r1, c1+1, r2, c2+1, move+1));

                // Move down + rotate clockwise
                if (r1+1 < n && r2+1 < n && grid[r1+1][c1] == 0 && grid[r2+1][c2] == 0) {
                    q.Enqueue((r1+1, c1, r2+1, c2, move+1));        // move down
                    q.Enqueue((r1, c1, r2+1, c1, move+1));          // rotate clockwise
                }
            }

            // Snake is vertical (c1 == c2)
            if (c1 == c2) {
                // Move right + rotate counterclockwise
                if (c2+1 < n && c1+1 < n && grid[r1][c1+1] == 0 && grid[r2][c2+1] == 0) {
                    q.Enqueue((r1, c1+1, r2, c2+1, move+1));        // move right
                    q.Enqueue((r1, c1, r1, c2+1, move+1));          // rotate counterclockwise
                }

                // Move down
                if (r1+1 < n && r2+1 < n && grid[r1+1][c1] == 0 && grid[r2+1][c2] == 0)
                    q.Enqueue((r1+1, c1, r2+1, c2, move+1));
            }
        }

        return -1;
    }
}
```

## Complexity

| Metric | Value | Notes |
|--------|-------|-------|
| **Time Complexity** | O(n⁴) | At most O(n²) horizontal + O(n²) vertical states, each processed once |
| **Space Complexity** | O(n⁴) | HashSet stores all visited `(r1, c1, r2, c2)` states |

Practical — for `n ≤ 100` the state space is at most `2 × 100² = 20,000` states, well within limits.

---

## Why This Works — Example Walkthrough

**Example 1:** `grid` is 6×6, output = `11`

One solution path: `[right, right, rotate clockwise, right, down, down, down, down, rotate counterclockwise, right, down]`

- BFS explores all reachable states level by level
- At each level the move count increases by 1
- The **first time** BFS reaches `(5, 4, 5, 5)` is guaranteed to be the **minimum** (BFS property)
- Both horizontal and vertical rotation candidates are enqueued from a single state when the surrounding cells allow it — this correctly captures that rotate + move share the same precondition check

**Key insight on rotations:**

When the snake is horizontal and the two cells directly below **both** tail and head are empty:
- It can move **down** (stays horizontal)
- It can also **rotate clockwise** (becomes vertical, head swings down)

Both transitions share the same boundary/blocked-cell check — so they are both enqueued in the same branch.

---

## Pitfalls & Edge Cases

- **Rotation preconditions** — clockwise rotation requires BOTH `grid[r1+1][c1]` and `grid[r2+1][c2]` to be clear, not just one
- **Counterclockwise rotation** — requires BOTH cells to the right of the vertical snake to be free
- **Visited state deduplication** — without a HashSet, BFS will loop indefinitely cycling between positions
- **n = 2** — minimum grid; snake starts already adjacent to the goal, may return 0 moves in trivial cases
- **Blocked target** — grid cells at `(n-1, n-2)` and `(n-1, n-1)` are guaranteed empty by problem constraints
- **No path** — return `-1` when queue empties without reaching the target

All handled correctly by standard BFS termination.

---

## Key Takeaway

This is a clean BFS state-space traversal problem disguised by the snake mechanics:

- Model the snake as **4 coordinates** `(r1, c1, r2, c2)` — tail and head
- Use a **HashSet** to avoid revisiting the same configuration
- At each state, enumerate all **valid transitions** based on snake orientation (horizontal vs vertical)
- Horizontal snake → try: move right, move down, rotate clockwise
- Vertical snake → try: move right, move down, rotate counterclockwise
- Each rotation is just an alternate enqueue in the same branch where the move-down or move-right check already passes
- BFS guarantees the **first time we reach the target** is the minimum number of moves

Straightforward, clean, and optimal — BFS naturally finds the shortest path in an unweighted state graph.
