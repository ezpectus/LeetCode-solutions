# 1467. Probability of a Two Boxes Having The Same Number of Distinct Balls
## DFS over Color Splits + Multinomial Counting

---

## Problem Statement

- `2n` balls of `k` colors. Shuffle randomly, split first `n` to box1, last `n` to box2.
- Return P(box1 and box2 have equal number of distinct colors).
- `k ≤ 8`, `balls[i] ≤ 6`.

---

## Core Idea — DFS over How Many Balls of Each Color Go to Box1

**Key insight**: For each color `i`, we choose `j` balls (out of `balls[i]`) to go to box1, and `balls[i]-j` to box2. The number of ways to make this split = `C(balls[i], j)` (choosing which specific balls go to box1).

**Total ways** for a complete split = product of `C(balls[i], j_i)` over all colors (multinomial coefficient).

**DFS state**: `(color_idx, balls_in_box1, distinct_in_box1, distinct_in_box2, accumulated_ways)`.

At the end (`color_idx == k`): if `balls_in_box1 == n`, it's a valid split → add to `total`. If additionally `distinct1 == distinct2` → add to `valid`.

**Probability** = `valid / total`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int k, n;
    int[] balls;
    double total, valid;

    public double GetProbability(int[] balls) {
        this.balls = balls;
        k = balls.Length;
        n = balls.Sum() / 2;
        total = 0; valid = 0;
        Dfs(0, 0, 0, 0, 1.0);
        return valid / total;
    }

    void Dfs(int i, int box1Count, int box1Dist, int box2Dist, double ways) {
        if (i == k) {
            if (box1Count == n) {
                total += ways;
                if (box1Dist == box2Dist) valid += ways;
            }
            return;
        }
        for (int j = 0; j <= balls[i]; j++) {
            int box2j = balls[i] - j;
            Dfs(i+1,
                box1Count + j,
                box1Dist + (j > 0 ? 1 : 0),
                box2Dist + (box2j > 0 ? 1 : 0),
                ways * Comb(balls[i], j));
        }
    }

    double Comb(int n, int r) {
        if (r > n) return 0;
        double res = 1;
        for (int i = 0; i < r; i++)
            res = res * (n - i) / (i + 1);
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(Π(balls[i]+1))** | At most 7^8 ≈ 6M leaves for k=8, balls[i]=6 |
| **Space** | **O(k)** | Recursion depth = k ≤ 8 |

In practice much faster — most branches are pruned implicitly when `box1Count > n`.

---

## Why Multinomial Coefficient

Consider color `i` with `balls[i] = 3` and we put `j=2` in box1. The 3 balls of this color are distinguishable (they have unique positions in the shuffle). The number of ways to assign 2 of them to box1 = `C(3,2) = 3`.

The total number of valid arrangements = `(2n)! / (balls[0]! × balls[1]! × ...)` (multinomial). But we can work with relative weights using products of `C(balls[i], j_i)` — the denominator cancels when computing the probability.

---

## Example Walkthrough

**`balls=[2,1,1]`**, n=2

DFS explores: for color 0 (2 balls): j=0,1,2; for color 1 (1 ball): j=0,1; for color 2 (1 ball): j=0,1.

Valid split example: j=[1,0,1] → box1 gets 2 balls, distinct1=2; box2 gets [1,1,0] → distinct2=2.
Ways = C(2,1)×C(1,0)×C(1,1) = 2×1×1 = 2.

Summing all valid/total → 8/12 = **0.66667** ✓

## Pitfalls & Edge Cases

- **Only count when `box1Count == n`** — splits where box1 doesn't get exactly n balls are invalid (they can't occur with uniform random shuffle)
- **Floating point** — use `double`; results within 10⁻⁵ are accepted
- **`C(balls[i], j)` is small** — `balls[i] ≤ 6` → `C(6,3)=20` max; no overflow

---

## Key Takeaway

"Probability over multinomial splits" → **DFS over per-color splits, weight by C(balls[i], j)**:

```
ways(split) = Π C(balls[i], j_i)
P(event) = Σ ways(valid splits) / Σ ways(all splits with box1Count == n)
```

The multinomial weights correctly represent the probability of each arrangement. DFS with k≤8 makes this tractable without memoization.
