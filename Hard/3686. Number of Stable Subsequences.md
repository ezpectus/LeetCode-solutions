# 3686. Number of Stable Subsequences

## 1. Idea & Architectural Choice

### 1.1 Core Philosophy
A subsequence is defined as **stable** if it does not contain three consecutive elements with the same parity (i.e., three consecutive even numbers or three consecutive odd numbers). 

Instead of generating subsequences explicitly or filtering invalid ones using inclusion-exclusion, we can process the array linearly using **Dynamic Programming (DP)**. As we iterate through `nums`, each element `x` can either:
1. Start a new single-element subsequence.
2. Append to any previously formed valid subsequence, provided appending `x` does not create three consecutive elements of the same parity.

### 1.2 State Space & Dynamic Transitions
To enforce the constraint, the DP state only needs to track two properties of any valid subsequence built so far:
- The **parity** of the last element: `0` (even) or `1` (odd).
- The **streak** of consecutive elements with this same parity at the end of the subsequence: `1` or `2`.

We maintain four primary DP counters:
- `dp[0][1]`: Count of valid subsequences ending with exactly **one** even number.
- `dp[0][2]`: Count of valid subsequences ending with exactly **two** even numbers.
- `dp[1][1]`: Count of valid subsequences ending with exactly **one** odd number.
- `dp[1][2]`: Count of valid subsequences ending with exactly **two** odd numbers.

When processing an element `x` with parity `p = x % 2` (and opposite parity `opp = 1 - p`):
1. **Extending same parity (Streak 1 -> Streak 2):**
   Appending `x` to subsequences that previously ended with a streak of 1 of parity `p` turns their streak into 2.
   `new_dp[p][2] = dp[p][2] + dp[p][1]`

2. **Creating new Streak 1 (New Subsequence or Parity Switch):**
   Appending `x` to subsequences ending in the opposite parity `opp` (streaks 1 or 2) resets the streak of parity `p` to 1. Additionally, `x` can start a standalone new subsequence of length 1.
   `new_dp[p][1] = dp[p][1] + (1 + dp[opp][1] + dp[opp][2])`

3. **Preventing Streak 3:**
   We never transition from `dp[p][2]` using parity `p`. Any transition that would produce three consecutive elements of parity `p` is implicitly dropped (added by `0`).

### 1.3 Architectural Choice: State Machine DP vs Combinatorics / Backtracking
- **Alternative 1: Backtracking / Brute Force ($O(2^N)$)**:
  Generates all $2^N$ subsequences and checks each for stability. Completely unfeasible for $N = 10^5$.
- **Alternative 2: Combinatorics with Segment Tree ($O(N \log N)$)**:
  Overkill. Tracking transitions between parities does not depend on values or ranges, only on immediate state changes.
- **Selected Choice: Constant-Space State Machine DP ($O(N)$ Time, $O(1)$ Space)**:
  Updates 4 variables in $O(1)$ time per element. This provides optimal efficiency under $N = 10^5$.

## 2. Algorithm Design & Mathematical Invariants

### 2.1 Variables & State Transitions
- `MOD = 1,000,000,007`
- `dp[p][c]`: Stores the number of valid non-empty subsequences ending with parity `p` and streak count `c` ($\in \{1, 2\}$).

#### Transition Step for `x` with Parity `p`:
Let `opp = 1 - p`.
- `add_to_streak_1 = (1 + dp[opp][1] + dp[opp][2]) % MOD`
- `add_to_streak_2 = dp[p][1] % MOD`
- Update:
  - `dp[p][1] = (dp[p][1] + add_to_streak_1) % MOD`
  - `dp[p][2] = (dp[p][2] + add_to_streak_2) % MOD`

The total count of stable subsequences after processing all elements is:
`Total = (dp[0][1] + dp[0][2] + dp[1][1] + dp[1][2]) % MOD`

## 3. Full Optimal Implementation (C++)

```cpp
// Dynamic Programming + State Machine
#include <vector>

class Solution {
public:
    int countStableSubsequences(std::vector<int>& nums) {
        const int MOD = 1000000007;

        // dp[p][c]: count of subsequences ending in parity p with streak c (1 or 2)
        // dp[0] -> even, dp[1] -> odd
        long long dp[2][3] = {0};

        for (int x : nums) {
            int p = x % 2;
            int opp = 1 - p;

            // Ways to create a new streak of 1 with parity p:
            // 1 (start new subsequence) + ending in opposite parity (streak 1 or 2)
            long long new_streak_1 = (1 + dp[opp][1] + dp[opp][2]) % MOD;

            // Ways to extend existing streak of 1 of parity p into streak of 2:
            long long new_streak_2 = dp[p][1];

            // Update states in place
            dp[p][1] = (dp[p][1] + new_streak_1) % MOD;
            dp[p][2] = (dp[p][2] + new_streak_2) % MOD;
        }

        long long total_stable = (dp[0][1] + dp[0][2] + dp[1][1] + dp[1][2]) % MOD;
        return static_cast<int>(total_stable);
    }
};
```
## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The overall execution time is derived from a single linear pass over the input array `nums`:
- **Phase 1: Array Traversal**: Loop runs N times, where N = nums.length.
- **Phase 2: State Transitions**: Inside the loop, parity derivation, basic arithmetic operations, and modulo calculations take constant O(1) time.

**Total Time Complexity:**
T_total = O(N) * O(1) = O(N)

For N = 10^5, total operations are approximately 10^5, executing in under 5 milliseconds.

### 4.2 Detailed Space Complexity Decomposition
- **dp Array**: A fixed-size 2D array of size 2 * 3 storing long long values.
- **Auxiliary Primitive Variables**: Constant memory for iteration indices and modulo constants.

**Total Space Complexity:**
S_total = O(1)

The memory footprint is independent of input size N.

### 4.3 Complexity Summary

| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(N) | Single pass through the array with O(1) state updates per element. |
| **Space Complexity** | O(1) | Requires only 4 constant DP counter variables. |

## 5. Mathematical Formulations & Foundational Algorithms

### Parity Streak Invariant
Any valid stable subsequence S can be represented as a sequence of parity block lengths (L_1, L_2, ..., L_k) where each L_m represents the number of consecutive elements of the same parity.

**Invariant:** A subsequence S is stable if and only if L_m <= 2 for all 1 <= m <= k.

Our dynamic programming transition guarantees this invariant inductively:
1. **Base Case:** An empty set of subsequences satisfies the invariant.
2. **Inductive Step:** Adding element x with parity p:
   - It appends to subsequences ending in parity 1 - p, establishing L_(k+1) = 1 <= 2.
   - It appends to subsequences ending in parity p with L_k = 1, updating L_k = 2 <= 2.
   - It refuses to append to subsequences ending in parity p with L_k = 2, preventing L_k = 3.

Thus, by construction, invalid subsequences with L_m >= 3 are never formed.

## Pitfalls & Edge Cases

* **Integer Overflow during Addition:** Intermediate DP sums can exceed 2^31 - 1. Using `long long` for DP states and applying modulo 10^9 + 7 after additions prevents overflow.
* **Negative Modulo Values:** Negative numbers in C++ `x % 2` can evaluate to -1. Input constraints guarantee 1 <= nums[i] <= 10^5, ensuring `x % 2` is strictly 0 or 1.
* **Single / Two-Element Subsequences:** Naturally supported without special handling as single elements initiate new subsequences (+ 1), and pairs automatically transition to streak 2.

---

## Key Takeaway

When counting constrained subsequences based on consecutive property streaks, maintain state variables tracking the last property value and current streak length, ignoring updates that violate the maximum allowed streak limit.

**Pattern recognition:** "number of subsequences without 3 consecutive same property" -> DP + State Machine / Parity Streak Tracking
