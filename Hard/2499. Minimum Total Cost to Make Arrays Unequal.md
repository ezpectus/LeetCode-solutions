# 2499. Minimum Total Cost to Make Arrays Unequal  
## O(n) â€” Fix Conflict Positions and Handle Majority Value

---

## Problem Statement

You are given two integer arrays `nums1` and `nums2` of equal length `n`.

You may perform the following operation any number of times:
* Swap `nums1[i]` and `nums1[j]`, cost = `i + j`

Your goal is to make:
`nums1[i] != nums2[i]` for all `i`.

Return the **minimum total cost**, or `-1` if it is impossible.

---

## Core Idea â€” Fix Equal Positions, Watch the Majority Value

We only care about indices where:
`nums1[i] == nums2[i]`

Call these **conflict indices** â€” they must be â€śbrokenâ€ť.

Let:
* `conflictIndices` = all `i` where `nums1[i] == nums2[i]`
* `totalConflicts = conflictIndices.Count`

If `totalConflicts == 0`, the arrays are already valid â†’ return `0`.

### Key Observation
Among conflict indices, some value `v` may appear very frequently:
`nums1[i] == nums2[i] == v`

If one value `v` appears **too many times**, swapping only among conflict indices will keep recreating `v` against `v`. Then we must use **extra indices** outside conflicts.

We need to:
1. Count how many times each value appears on conflict indices.
2. Find the value `major` with the highest frequency `cntMajor`.
3. **Check Condition:**
   * If `cntMajor * 2 <= totalConflicts`: conflicts can be resolved internally.
   * Otherwise:
     * We must add extra indices where:
       ```
       nums1[i] != nums2[i]
       nums1[i] != major
       nums2[i] != major
       ```
     * Let `need = cntMajor * 2 - totalConflicts`.
     * Collect `need` such indices with minimal cost.

### Why sum of indices?
Each index participates in swaps exactly once in the optimal construction. Thus the minimal total cost is simply:
`sum(conflictIndices) + sum(extraIndices)`

---

## Algorithm

1. **Traverse all indices `i`**:
   * If `nums1[i] == nums2[i]`:
     * Add `i` to `conflictIndices`.
     * Increment `freq[nums1[i]]`.
2. If `conflictIndices` is empty â†’ return `0`.
3. Find `major` with maximum `freq[major]`.
4. **If `cntMajor * 2 <= totalConflicts`**:
   * Return `sum(conflictIndices)`.
5. **Otherwise**:
   * Compute `need = cntMajor * 2 - totalConflicts`.
   * Traverse all indices where `nums1[i] != nums2[i]`:
     * If both `nums1[i]` and `nums2[i]` are not equal to `major`, use this index.
     * Add `i` to `extraCost`.
     * Stop when `need == 0`.
6. If `need > 0` â†’ return `-1`.
7. Return `sum(conflictIndices) + sum(extraIndices)`.

## Full Implementation (C#)

```csharp
public class Solution {
    public long MinimumTotalCost(int[] nums1, int[] nums2) {
        int n = nums1.Length;
        var freq = new Dictionary<int, int>();
        long baseCost = 0;
        int totalConflicts = 0;

        for (int i = 0; i < n; i++) {
            if (nums1[i] == nums2[i]) {
                totalConflicts++;
                baseCost += i;

                int v = nums1[i];
                if (!freq.ContainsKey(v)) freq[v] = 0;
                freq[v]++;
            }
        }

        if (totalConflicts == 0)
            return 0;

        int major = -1;
        int cntMajor = 0;
        foreach (var kv in freq) {
            if (kv.Value > cntMajor) {
                cntMajor = kv.Value;
                major = kv.Key;
            }
        }

        if (cntMajor * 2 <= totalConflicts)
            return baseCost;

        int need = cntMajor * 2 - totalConflicts;
        long extraCost = 0;

        for (int i = 0; i < n && need > 0; i++) {
            if (nums1[i] == nums2[i]) continue; 
            if (nums1[i] == major || nums2[i] == major) continue;

            extraCost += i;
            need--;
        }

        return need == 0 ? baseCost + extraCost : -1;
    }
}
```

## Pitfalls & Edge Cases

* **No conflicts:** If all `nums1[i] != nums2[i]`, the answer is `0`.
* **Majority Rule:** If a value dominates conflicts (`cntMajor * 2 > totalConflicts`), extra indices are required.
* **Valid Extra Indices:** Extra indices must satisfy `nums1[i] != major` and `nums2[i] != major`.
* **Overflow:** Use `long` for cost â€” the sum of indices can exceed $2^{31}-1$.

## Example Trace

### Example 1
`nums1 = [1,2,3,4,5]`, `nums2 = [1,2,3,4,5]`

1.  **Conflicts at all indices:** `[0, 1, 2, 3, 4]`
2.  **Frequency:** Each value appears once â†’ `cntMajor = 1`, `totalConflicts = 5`.
3.  **Check:** $1 * 2 \le 5$ (True).
4.  **Cost:** $0 + 1 + 2 + 3 + 4 = 10$.

---

## Key Takeaway

This problem is solved by fixing positions where `nums1[i] == nums2[i]` 
while carefully managing any value that appears in more than half of the conflict cases. It's a clean **O(n)** greedy approach.
