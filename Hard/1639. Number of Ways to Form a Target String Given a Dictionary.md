# 1639. Number of Ways to Form a Target String Given a Dictionary
## O(W × T) — Frequency Precomputation + 1D DP

---

## Problem Statement

- Build `target` left-to-right by picking characters from word columns (left-to-right, no reuse of a column or anything to its left).
- Count total ways mod 10⁹+7.

---

## Core Idea — Two Phases

### Phase 1: Frequency Table

Since all words have the same length `W`, precompute:
```
freq[k][c] = number of words that have character c at column k
```
This collapses all `words.length` strings into a single 2D array of size `W × 26`. Any character we pick from column `k` contributes `freq[k][c]` choices simultaneously (across all words).

### Phase 2: 1D DP

**State**: `dp[i]` = number of ways to form `target[0..i-1]` using columns seen so far.

**Transition** (sweeping column `k` left to right):

```
dp[i] += dp[i-1] * freq[k][target[i-1]]
```

"To form `i` characters of target, form `i-1` characters using columns before `k`, then use column `k` for the `i`-th character."

**Why iterate `i` backwards?** Same as 0/1 knapsack — prevents using column `k` more than once in the same target formation. If we iterated forward, `dp[i]` updated at step `k` would be used again for `dp[i+1]` in the same `k` iteration.

**Base case**: `dp[0] = 1` — one way to form empty target.

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int numWays(vector<string>& words, string target) {
        const int MOD = 1e9 + 7;
        int W = words[0].size(), T = target.size();

        // freq[k][c] = count of words with char c at position k
        vector<array<int,26>> freq(W, array<int,26>{});
        for (auto& word : words)
            for (int k = 0; k < W; k++)
                freq[k][word[k]-'a']++;

        // 1D DP over target length
        vector<long long> dp(T + 1, 0);
        dp[0] = 1;

        for (int k = 0; k < W; k++) {
            int c = target[...]; // handled per i below
            for (int i = min(k+1, T); i >= 1; i--) {
                int ch = target[i-1] - 'a';
                dp[i] = (dp[i] + dp[i-1] * freq[k][ch]) % MOD;
            }
        }

        return dp[T];
    }
};
```

```cpp
class Solution {
public:
    int numWays(vector<string>& words, string target) {
        const int MOD = 1e9 + 7;
        int W = words[0].size(), T = target.size();

        vector<array<int,26>> freq(W);
        for (auto& f : freq) f.fill(0);
        for (auto& word : words)
            for (int k = 0; k < W; k++)
                freq[k][word[k]-'a']++;

        vector<long long> dp(T + 1, 0);
        dp[0] = 1;

        for (int k = 0; k < W; k++)
            for (int i = min(k + 1, T); i >= 1; i--)
                dp[i] = (dp[i] + dp[i-1] * freq[k][target[i-1]-'a']) % MOD;

        return dp[T];
    }
};
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(W×T + N×W)** | Freq build O(N×W); DP O(W×T) |
| **Space** | **O(W×26 + T)** | Freq table + dp array |

With W=T=1000: `10⁶` DP operations — fast.

---

## Why Backwards Iteration (Knapsack Pattern)

Consider column `k` and two target positions `i=1` and `i=2`.

**Forward** (wrong):
- Update `dp[1]` using `dp[0]` at column `k`
- Update `dp[2]` using `dp[1]` (already updated at column `k`) → uses column `k` twice!

**Backward** (correct):
- Update `dp[2]` using `dp[1]` (not yet touched at column `k`) ✓
- Update `dp[1]` using `dp[0]` ✓

Each column `k` is used at most once per target position — same logic as 0/1 knapsack.

## Example Walkthrough

**`words=["acca","bbbb","caca"], target="aba"`**

```
freq[0]: a=2,b=1,c=0...  (words[0][0]='a', words[1][0]='b', words[2][0]='c' → a:1,b:1,c:1)
```

Actually: words[0]="acca", words[1]="bbbb", words[2]="caca"
```
freq[0]: a:1('a'), b:1('b'), c:1('c')
freq[1]: c:1('c'), b:1('b'), a:1('a')
freq[2]: c:1('c'), b:1('b'), c:1('c') → c:2, b:1
freq[3]: a:1('a'), b:1('b'), a:1('a') → a:2, b:1
```

target="aba": target[0]='a', target[1]='b', target[2]='a'

dp starts: [1,0,0,0]

k=0: i=1: dp[1] += dp[0]*freq[0]['a']=1*1=1 → dp=[1,1,0,0]
k=1: i=1: dp[1] += dp[0]*freq[1]['a']=1*1=1 → dp[1]=2
     i=2: dp[2] += dp[1_old=1]*freq[1]['b']=1*1=1 → dp=[1,2,1,0]
k=2: i=1: dp[1] += dp[0]*freq[2]['a']=1*0=0 (no 'a' at col 2)
     i=2: dp[2] += dp[1]*freq[2]['b']=2*1=2 → dp[2]=3
     (i=3 not yet reachable: min(3,3)=3, target[2]='a', freq[2]['a']=0 → no change)
k=3: i=3: dp[3] += dp[2]*freq[3]['a']=3*2=6 → dp[3]=6
     i=2: dp[2] += dp[1]*freq[3]['b']=2*1=2 → dp[2]=5 (doesn't matter)
     i=1: dp[1] += dp[0]*freq[3]['a']=1*2=2 → dp[1]=4 (doesn't matter)

dp[3] = **6** ✓

## Pitfalls & Edge Cases

- **`i` upper bound = `min(k+1, T)`** — can't have formed more target characters than columns processed; prevents accessing negative `dp[i-1]` implicitly and saves iterations
- **`long long` for dp** — `dp[i-1] * freq[k][c]` can be up to `10⁹ × 1000` before mod
- **Backward loop is essential** — forward loop double-counts same column

---

## Key Takeaway

"Count ways to form string from column-constrained dictionary" → **frequency table + 0/1 knapsack DP**:

```
freq[k][c] = words contributing char c at column k
dp[i] += dp[i-1] * freq[k][target[i-1]]   (iterate i backwards)
```

Collapsing `words` into `freq` reduces N×W word data to W×26 — the key optimization. Backward iteration prevents reusing the same column, identical to 0/1 knapsack.
