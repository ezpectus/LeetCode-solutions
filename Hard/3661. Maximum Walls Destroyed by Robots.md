# 3661. Maximum Walls Destroyed by Robots
## O(n log n + w log w) — Sort + DP with Overlap Correction

---

## Problem Statement

- Each robot fires ONE bullet (left or right) up to `distance[i]` meters.
- Bullet stops if it hits another robot before reaching max distance.
- Maximize the number of **unique** walls destroyed across all robots.

---

## Core Idea — Structural Decomposition + DP

### Key Observations

1. **Walls at robot positions** are always destroyed (bullet starts there regardless of direction) → count separately as `fixed`.

2. **After sorting robots by position**, the line is divided into gaps. Walls in a gap between `pos[i]` and `pos[i+1]` can **only** be reached by:
   - Robot `i` firing **right** (blocked by `pos[i+1]`)
   - Robot `i+1` firing **left** (blocked by `pos[i]`)

3. This creates a **chain structure**: each robot's choice (left/right) affects which adjacent gaps it covers, and adjacent robots share a gap.

### DP Formulation

Sort robots by position. Define:
- `right_walls[i]` = walls in `(pos[i], min(pos[i]+dist[i], pos[i+1]-1)]`
- `left_walls[i]` = walls in `[max(pos[i]-dist[i], pos[i-1]+1), pos[i])`
- `overlap_gap(i)` = walls in the gap between `pos[i]` and `pos[i+1]` reachable by BOTH robots

**State**: `dp[dir]` = max extra walls (beyond `fixed`) considering robots `0..i`, robot `i` fires in direction `dir` (0=left, 1=right).

**Transitions** for robot `i`:
```
dp[1] = max(dp_prev[0], dp_prev[1]) + right_walls[i]
// firing right: previous state doesn't affect gap[i] (robot i+1 handles that)

dp[0] = max(
    dp_prev[0] + left_walls[i],            // prev fired left: no overlap possible
    dp_prev[1] + left_walls[i] - overlap    // prev fired right: subtract overlap to avoid double-count
)
```

The overlap correction handles walls counted in both `right_walls[i-1]` (prev fired right) and `left_walls[i]` (current fires left).

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxWalls(int[] robots, int[] distance, int[] walls) {
        int n = robots.Length;
        var sorted = robots.Select((r,i) => (r, distance[i]))
                           .OrderBy(x => x.r).ToArray();
        int[] pos = sorted.Select(x => x.r).ToArray();
        int[] dist = sorted.Select(x => x.Item2).ToArray();

        Array.Sort(walls);
        int w = walls.Length;

        int CountIn(int lo, int hi) {
            if (lo > hi) return 0;
            int l = Array.BinarySearch(walls, lo);
            if (l < 0) l = ~l;
            int r2 = Array.BinarySearch(walls, hi);
            if (r2 < 0) r2 = ~r2 - 1;
            return Math.Max(0, r2 - l + 1);
        }

        var posSet = new HashSet<int>(pos);
        int fixed_ = walls.Count(ww => posSet.Contains(ww));

        int[] rightW = new int[n], leftW = new int[n];
        for (int i = 0; i < n; i++) {
            int r = pos[i], d = dist[i];
            int rHi = i+1<n ? Math.Min(r+d, pos[i+1]-1) : r+d;
            rightW[i] = CountIn(r+1, rHi);
            int lLo = i>0 ? Math.Max(r-d, pos[i-1]+1) : r-d;
            leftW[i] = CountIn(lLo, r-1);
        }

        int Overlap(int i) {
            int rHi = Math.Min(pos[i]+dist[i], pos[i+1]-1);
            int lLo = Math.Max(pos[i+1]-dist[i+1], pos[i]+1);
            return CountIn(Math.Max(pos[i]+1, lLo), Math.Min(rHi, pos[i+1]-1));
        }

        int dpL = leftW[0], dpR = rightW[0];
        for (int i = 1; i < n; i++) {
            int ovlp = Overlap(i-1);
            int newL = Math.Max(dpL + leftW[i], dpR + leftW[i] - ovlp);
            int newR = Math.Max(dpL, dpR) + rightW[i];
            dpL = newL; dpR = newR;
        }

        return fixed_ + Math.Max(dpL, dpR);
    }
}
```

## Example Walkthrough

**`robots=[10,2], distance=[5,1], walls=[5,2,7]`**

Sorted robots: `[(2,1),(10,5)]` → pos=[2,10], dist=[1,5]

Walls sorted: [2,5,7]

- `fixed` = walls at {2,10}: only wall `2` → fixed=1
- `right_walls[0]` = walls in (2, min(3, 9)] = (2,3] → none → 0
- `left_walls[0]` = walls in [max(1,1), 1] = [1,1] → none → 0
- `right_walls[1]` = walls in (10, 15] → none → 0
- `left_walls[1]` = walls in [max(5,3), 9] = [5,9] → {5,7} → 2
- `overlap_gap(0)` = walls in [max(3,5), min(3,9)] = [5,3] → empty (lo>hi) → 0

DP start: dpL=0, dpR=0

i=1: ovlp=0
- newL = max(0+2, 0+2-0) = 2
- newR = max(0,0)+0 = 0

Result = fixed(1) + max(2,0) = **3** ✓

Robot[1] fires left covering walls 5 and 7. Robot[0] position has wall 2 (fixed). Total=3.

## Pitfalls & Edge Cases

- **`fixed` count**: walls exactly at robot positions are always destroyed → separate from DP
- **Overlap subtraction**: when prev robot fired right AND current fires left, walls in the overlapping region are already counted in `dpR` → subtract `overlap` to avoid double-counting
- **Single robot**: dp initialized directly from left_walls[0] and right_walls[0]
- **No walls in a gap**: binary search handles gracefully → 0

---

## Key Takeaway

"Maximize unique walls from binary (left/right) choices across chain of robots" → **sort robots, DP on direction choice**:

```
dp[LEFT]  = max(prev_LEFT + leftW, prev_RIGHT + leftW - overlap)
dp[RIGHT] = max(prev_LEFT, prev_RIGHT) + rightW
```

The overlap correction prevents double-counting walls reachable by two adjacent robots. Structural decomposition into gaps is the key insight that makes the DP tractable.
