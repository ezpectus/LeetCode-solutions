# 3225. Maximum Score From Grid Operations
## O(n³) — DP on (prev_height, curr_height) + Prefix Sums

---

## Problem Statement

- Color columns: operation colors column `j` from row 0 to row `i` (height = i+1). 
- Score: sum of `grid[i][j]` where cell is white AND has a horizontally adjacent black cell.
- Maximize score.

---

## Core Idea — DP on Column Heights

Each column `j` gets a height `h[j] ∈ [0..n]` (0 = not colored). A cell `(i,j)` is white iff `i >= h[j]`, black iff `i < h[j]`.

**Contribution of column `j`**: white cells with left or right black neighbor.
- White rows: `i >= h[j]`
- Has left black at row i: `i < h[j-1]`
- Has right black at row i: `i < h[j+1]`
- Combined (OR): `i < max(h[j-1], h[j+1])`
- Contribution = `sum(grid[i][j])` for `i ∈ [h[j], max(h[j-1], h[j+1]))`

**DP state** `dp[h_prev][h_curr]` = max score after deciding columns `0..j`, where column `j-1` has height `h_prev` and column `j` has height `h_curr`.

**Transition** to `h_next` for column `j+1`:
- Adds contribution of column `j`: `col_sum(j, h_curr, max(h_prev, h_next))`

**Boundary handling**:
- Column 0 has no left neighbor (`h_prev = sentinel`): only right neighbor `h_next` matters.
- Column `n-1` has no right neighbor: contribution = `col_sum(n-1, h, h_prev)` if `h_prev > h`.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumScore(int[][] grid) {
        int n = grid.Length;
        long[][] psum = new long[n][];
        for (int j = 0; j < n; j++) {
            psum[j] = new long[n+1];
            for (int i = 0; i < n; i++) psum[j][i+1] = psum[j][i] + grid[i][j];
        }

        long ColSum(int j, int lo, int hi) =>
            lo >= hi ? 0 : psum[j][hi] - psum[j][lo];

        const long NEG_INF = long.MinValue / 2;
        // dp[hp][h]: hp=-1 (sentinel=n) for no left column
        long[,] dp = new long[n+1, n+1];
        for (int i = 0; i <= n; i++) for (int j = 0; j <= n; j++) dp[i,j] = NEG_INF;
        // Initialize: col 0 decided, no left (hp=n sentinel), any height h0
        // Store in a separate row of dp using hp=n as sentinel won't work easily
        // Use a 1D array for col 0
        long[] prev = new long[n+1];
        // prev[h0] = 0 (no score yet, col 0 just decided)

        for (int j = 1; j < n; j++) {
            long[,] newDp = new long[n+1, n+1];
            for (int i = 0; i <= n; i++) for (int k = 0; k <= n; k++) newDp[i,k] = NEG_INF;

            if (j == 1) {
                // Previous state: sentinel (no left col), col 0 has height h0
                for (int h0 = 0; h0 <= n; h0++) {
                    for (int h1 = 0; h1 <= n; h1++) {
                        // Contribution of col 0: only right (h1) matters
                        long contrib = ColSum(0, h0, h1);
                        if (contrib > newDp[h0, h1]) newDp[h0, h1] = contrib;
                    }
                }
            } else {
                for (int hp = 0; hp <= n; hp++)
                    for (int h = 0; h <= n; h++) {
                        if (dp[hp,h] == NEG_INF) continue;
                        for (int hn = 0; hn <= n; hn++) {
                            int maxH = Math.Max(hp, hn);
                            long contrib = ColSum(j-1, h, maxH);
                            long ns = dp[hp,h] + contrib;
                            if (ns > newDp[h,hn]) newDp[h,hn] = ns;
                        }
                    }
            }
            dp = newDp;
        }

        long res = 0;
        for (int hp = 0; hp <= n; hp++)
            for (int h = 0; h <= n; h++) {
                if (dp[hp,h] == NEG_INF) continue;
                long contrib = hp > h ? ColSum(n-1, h, hp) : 0;
                long total = dp[hp,h] + contrib;
                if (total > res) res = total;
            }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n⁴)** worst case | n columns × O(n²) states × O(n) transitions |
| **Optimized** | **O(n³)** | Precompute max over h_prev and h_next separately |
| **Space** | **O(n²)** | DP table |

With n=100: O(n⁴) = 10⁸ in C# is borderline. Optimization: for each (h, hn), maximize over hp using a running max.

---

## Score Contribution Formula

For column `j` with height `h`, between neighbors `h_prev` and `h_next`:

```
white rows: i ∈ [h, n-1]
black left at i: i < h_prev
black right at i: i < h_next
black on either side: i < max(h_prev, h_next)
contribution = col_sum(j, h, max(h_prev, h_next))
```

This correctly handles the OR condition (a cell with both neighbors black counts only once).

---

## Key Takeaway

"Score depends on whether white cell has adjacent black column" → **DP on (prev_height, curr_height) per column. Contribution of middle column = col_sum(j, h, max(h_prev, h_next))**. Prefix sums for O(1) range queries.
