# 2911. Minimum Changes to Make K Semi-palindromes
## O(n³ · D) — Precompute Semi-palindrome Costs + 2D Partition DP

---

## Problem Statement

Partition string `s` into `k` substrings. For each substring, count the minimum character changes needed to make it a **semi-palindrome**. Return the total minimum changes.

A string of length `n` is a **semi-palindrome** if there exists a divisor `d` of `n` (with `1 ≤ d < n`) such that grouping characters by their index modulo `d` produces `d` palindromic groups.

---

## Semi-palindrome Unpacked

For divisor `d` of length `n` (so `n = d × m`):
- Group `g` (0-indexed, `g ∈ [0..d-1]`) contains characters at positions `g, g+d, g+2d, ..., g+(m-1)d`.
- Each group has exactly `m = n/d` characters.
- The string is semi-palindrome with divisor `d` iff every group string is a palindrome.

Palindrome condition on group `g`: character at offset `g + p·d` equals character at offset `g + (m-1-p)·d`, for each `p ∈ [0..⌊m/2⌋-1]`.

**Key fact:** `d=1` is always a valid divisor candidate for any length ≥ 2 (1 divides everything, 1 < 2 ≤ len). So the "changes for d=1" baseline = minimum changes to make the full string a palindrome.

---

## Algorithm Overview

### Phase 1 — Precompute `cost[i][j]`

For every substring `s[i..j]` (length ≥ 2):
```
cost[i][j] = min over all valid d (d | len, 1 ≤ d < len) of:
             Σ_g Σ_p [ s[i+g+p·d] ≠ s[i+g+(m-1-p)·d] ]
```

Each mismatched palindrome pair costs 1 change (change either character to match the other).

### Phase 2 — Partition DP

```
dp[t][j] = min changes to partition s[0..j-1] into t semi-palindromes
dp[0][0] = 0
dp[t][j] = min over i ∈ [0..j-2] of (dp[t-1][i] + cost[i][j-1])
answer   = dp[k][n]
```

The constraint `i ≤ j-2` ensures every partition part has length ≥ 2 (single-character strings have no valid semi-palindrome divisor).

## Full Implementation (C#)

```csharp
public class Solution {
    public int MinimumChanges(string s, int k) {
        int n = s.Length;

        // Phase 1: cost[i][j] = min changes to make s[i..j] a semi-palindrome
        int[,] cost = new int[n, n];
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                int len = j - i + 1;
                int best = len; // safe upper bound

                // Try every valid divisor d of len (1 ≤ d < len)
                for (int d = 1; d < len; d++) {
                    if (len % d != 0) continue;
                    int m = len / d; // characters per group
                    int changes = 0;
                    for (int g = 0; g < d; g++) {
                        for (int p = 0; p < m / 2; p++) {
                            int left  = i + g + p * d;
                            int right = i + g + (m - 1 - p) * d;
                            if (s[left] != s[right]) changes++;
                        }
                    }
                    if (changes < best) best = changes;
                }
                cost[i, j] = best;
            }
        }

        // Phase 2: dp[t][j] = min changes to split s[0..j-1] into t semi-palindromes
        const int INF = int.MaxValue / 2;
        int[,] dp = new int[k + 1, n + 1];
        for (int t = 0; t <= k; t++)
            for (int j = 0; j <= n; j++)
                dp[t, j] = INF;
        dp[0, 0] = 0;

        for (int t = 1; t <= k; t++)
            for (int j = 2; j <= n; j++)
                for (int i = 0; i <= j - 2; i++)
                    if (dp[t - 1, i] < INF)
                        dp[t, j] = Math.Min(dp[t, j], dp[t - 1, i] + cost[i, j - 1]);

        return dp[k, n];
    }
}
```

## Complexity

| Phase | Time | Notes |
|---|---|---|
| cost precomputation | O(n² · D · n) | n² pairs; D ≤ 18 divisors; O(n) work per divisor |
| partition DP | O(k · n²) | k layers × n² transitions |
| **Total** | **O(n³ · D + k · n²)** | n ≤ 200, D ≤ 18, k ≤ 100 → ~100M ops |
| Space | O(n² + k·n) | cost table + dp table |

For n=200: cost precomputation ≈ 40K pairs × 9 avg divisors × 100 avg work = 36M. DP ≈ 100 × 40K = 4M. Both fine.

## Cost Precomputation — Worked Example

**`cost["abcabc"]` = cost[0][5]** (`s = "abcabc"`, len=6)

Divisors of 6 with d < 6: `d ∈ {1, 2, 3}`.

**d=1** (m=6, one group = entire string "abcabc"):
| Pair p | left | right | match? |
|---|---|---|---|
| p=0 | 'a' | 'c' | ✗ |
| p=1 | 'b' | 'b' | ✓ |
| p=2 | 'c' | 'a' | ✗ |

cost(d=1) = 2.

**d=2** (m=3, 2 groups):
- Group 0: positions 0,2,4 → "aca": (a,a) ✓. cost=0.
- Group 1: positions 1,3,5 → "bcb": (b,b) ✓. cost=0.

cost(d=2) = 0 ← **minimum!**

**d=3** (m=2, 3 groups):
- Group 0: positions 0,3 → "aa": ✓. cost=0.
- Group 1: positions 1,4 → "bb": ✓. cost=0.
- Group 2: positions 2,5 → "cc": ✓. cost=0.

cost(d=3) = 0 also. `cost[0][5] = 0`.

## Pitfalls & Edge Cases

- **d must divide len exactly:** `len % d != 0` → skip. Otherwise group sizes are non-uniform and the palindrome pairing breaks.

- **d < len strictly:** d = len gives groups of 1 character each (trivially palindromes, but d = len is excluded by definition). Don't include it.

- **Length-1 substrings have no valid d:** Never arise in valid partitions since k ≤ n/2 guarantees we can always partition with each part having length ≥ 2. The DP enforces this via `i ≤ j-2`.

- **"changes" not "replacements":** Each mismatched pair costs exactly 1 change (change one char to equal the other). Don't double-count.

- **Even vs odd group length m:** For odd m, the middle element (at p = m/2) is self-paired and always satisfies the palindrome condition — automatically excluded by the loop `p < m/2`.

- **Upper bound for `best`:** Initialize `best = len` (not `best = len/2`). The true maximum is `⌊len/2⌋`, but using `len` as a safe upper bound and letting the `d=1` trial correct it is simpler and still correct.

---

## Key Takeaway

This is a **2D interval DP** problem with a non-trivial interval cost function. The cost `cost[i][j]` (min changes for a single semi-palindrome) is itself a minimization over multiple palindrome structures. The two-phase approach — precompute all O(n²) costs, then run standard partition DP — cleanly separates concerns.

**Pattern recognition:** "Partition string into k parts minimizing a per-part cost" → partition DP: `dp[t][j] = min over split points i of dp[t-1][i] + cost[i][j-1]`. The hard part is computing `cost[i][j]` efficiently. For semi-palindrome, try all divisors with a sum-of-mismatches computation per divisor.
