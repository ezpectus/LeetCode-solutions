# 3260. Find the Largest Palindrome Divisible by K
## O(n ¬∑ k) ‚ÄĒ Suffix Achievable Sets + Greedy Digit Selection

---

## Problem Statement

Find the largest n-digit palindrome divisible by k (1 ‚Č§ k ‚Č§ 9). No leading zeros.

---

## Core Insight #1 ‚ÄĒ A Palindrome Is Determined by Its First Half

An n-digit palindrome `d‚āÄd‚āĀ...d_{half-1}...d‚āĀd‚āÄ` (where `half = ‚Ćąn/2‚ĆČ`) is fully determined by its first `half` digits. The value mod k equals:

```
ő£ŠĶĘ dŠĶĘ ¬∑ wŠĶĘ   (mod k)
```

where the **weight** `wŠĶĘ` for position `i` (0-indexed from left) is:
```
wŠĶĘ = (10^(n-1-i) + 10^i) mod k     if i ‚Č† n-1-i  (symmetric pair)
wŠĶĘ = 10^i mod k                      if i = n-1-i  (middle digit, odd n)
```

The key: `10^a mod k` has very small periods for k ‚Č§ 9 (period ‚Č§ 6 for k=7, period 1 or 2 for others). Weights are precomputable in O(n).

---

## Core Insight #2 ‚ÄĒ Suffix Achievable Residue Sets

Define `S[i]` = the set of residues mod k achievable by choosing digits at positions `i, i+1, ..., half-1`:
```
S[half] = {0}                          (no digits left, only 0 is achievable)
S[i] = { (d¬∑wŠĶĘ + r) mod k : d ‚ąą 0..9, r ‚ąą S[i+1] }
```

Since k ‚Č§ 9, `S[i] ‚äÜ {0,...,k-1}` ‚ÄĒ stored as a **k-bit bitmask** (an int). Computation: O(half ¬∑ 10 ¬∑ k) = O(n ¬∑ k).

---

## Core Insight #3 ‚ÄĒ Greedy: Largest Digit That Keeps Goal Achievable

At each position `i` (left to right), choose the **largest** digit `d` (9 down to 0; ‚Č•1 at position 0) such that:
```
needed = (k - (current_sum + d¬∑wŠĶĘ) % k) % k
needed ‚ąą S[i+1]
```

This ensures the remaining positions can still reach the target residue 0.

Since we always scan from 9 downward, the result is the lexicographically largest (and numerically largest) valid palindrome.

## Full Implementation (C#)

```csharp
public class Solution {
    public string LargestPalindrome(int n, int k) {
        int half = (n + 1) / 2;

        // Precompute 10^a mod k for a = 0..n-1
        int[] pow10 = new int[n];
        pow10[0] = 1 % k;
        for (int a = 1; a < n; a++)
            pow10[a] = pow10[a - 1] * 10 % k;

        // Weight for each half-position i
        int[] w = new int[half];
        for (int i = 0; i < half; i++) {
            int mirror = n - 1 - i;
            w[i] = (mirror == i) ? pow10[i] : (pow10[mirror] + pow10[i]) % k;
        }

        // S[i] = bitmask of residues achievable from positions i..half-1
        int[] S = new int[half + 1];
        S[half] = 1; // bit 0 set = only residue 0 reachable
        for (int i = half - 1; i >= 0; i--) {
            int mask = 0;
            for (int d = 0; d <= 9; d++) {
                int shift = d * w[i] % k;
                for (int r = 0; r < k; r++)
                    if ((S[i + 1] >> r & 1) == 1)
                        mask |= 1 << ((r + shift) % k);
            }
            S[i] = mask;
        }

        // Greedily pick largest digit at each half-position
        char[] half_d = new char[half];
        int cur = 0;
        for (int i = 0; i < half; i++) {
            for (int d = 9; d >= (i == 0 ? 1 : 0); d--) {
                int next = (cur + d * w[i]) % k;
                int needed = next == 0 ? 0 : k - next;
                if ((S[i + 1] >> needed & 1) == 1) {
                    half_d[i] = (char)('0' + d);
                    cur = next;
                    break;
                }
            }
        }

        // Mirror first half to build palindrome
        char[] ans = new char[n];
        for (int i = 0; i < half; i++) {
            ans[i] = half_d[i];
            ans[n - 1 - i] = half_d[i];
        }
        return new string(ans);
    }
}
```

## Example Walkthroughs

### Example 1 ‚ÄĒ `n=3, k=5` ‚Üí `"595"`

`pow10 = [1, 0, 0]` (mod 5). Half=2.

| Position | Mirror | Weight `w[i]` |
|---|---|---|
| 0 | 2 | (pow10[2]+pow10[0])%5 = (0+1)%5 = 1 |
| 1 | 1 (middle) | pow10[1] = 0 |

**Suffix sets:**
- `S[2] = {0}`
- `S[1]` (w=0): any digit contributes 0 ‚Üí `S[1] = {0}`
- `S[0]` (w=1): digit d shifts {0} by d%5 ‚Üí `S[0] = {0,1,2,3,4}`

**Greedy:**
- i=0: d=9‚Üíneeded=1‚ąČ{0}. d=8‚Üíneeded=2‚ąČ{0}. ... d=5‚Üíneeded=0‚ąą{0}. ‚úď cur=0.
- i=1: any d ‚Üí next=0, needed=0‚ąą{0}. d=9. ‚úď

**Palindrome: "595"** ‚úď

---

### Example 2 ‚ÄĒ `n=1, k=4` ‚Üí `"8"`

Half=1. `w[0]=pow10[0]=1%4=1`. `S[1]={0}`.

**`S[0]`:** `{0,1,2,3}` (d*1 mod 4 for d=0..9).

**Greedy:** i=0 (d‚Č•1): d=9‚Üínext=1,needed=3‚ąČ{0}. d=8‚Üínext=0,needed=0‚ąą{0}. ‚úď

**Answer: "8"** ‚úď

---

### Example 3 ‚ÄĒ `n=5, k=6` ‚Üí `"89898"`

`pow10 = [1,4,4,4,4]` (10^a mod 6=4 for a‚Č•1). Half=3.

| i | mirror | w[i] |
|---|---|---|
| 0 | 4 | (4+1)%6=5 |
| 1 | 3 | (4+4)%6=2 |
| 2 | 2 (mid) | 4 |

**Suffix sets:** S[3]={0}, S[2]={0,2,4}, S[1]={0,2,4}, S[0]={0,1,2,3,4,5}.

**Greedy:**
- i=0: d=9‚Üíneeded=3‚ąČ{0,2,4}. **d=8**: next=4, needed=2‚ąą{0,2,4}. ‚úď
- i=1: d=9‚Üínext=(4+18)%6=4, needed=2‚ąą{0,2,4}. ‚úď
- i=2: d=9‚Üíneeded=2‚ąČ{0}. d=8‚Üínext=0, needed=0‚ąą{0}. ‚úď

**"89898"** ‚úď (89898 / 6 = 14983)

## Key Patterns by K

| k | Pattern | Logic |
|---|---|---|
| 1 | "999...9" | Everything divisible; pick max |
| 2 | "8999...998" | Last digit even, max=8 |
| 3 | "999...9" | Digit sum always divisible (9‚Č°0 mod 3) |
| 4 | Last 2 digits: d‚āād‚āĀ‚Č°0 (mod 4) | Often "8888...88" |
| 5 | "5999...995" | Last digit 5; max non-zero palindrome starter |
| 6 | Both k=2 and k=3 satisfied | First digit even, digit sum ‚Č°0 mod 3 |
| 7 | Period-6 weights, greedy | w=(10^a+10^b) mod 7, set varies |
| 8 | Last 3 digits determine divisibility | Inner digits usually 9 |
| 9 | "999...9" | Digit sum=9n‚Č°0 mod 9 |

The algorithm handles all k=1..9 uniformly without special cases.

---

## Pitfalls

- **No leading zeros:** At position `i=0`, only digits 1..9 are tried.
- **Middle digit (odd n):** Its weight is `pow10[(n-1)/2] mod k`, not the sum with a mirror.
- **k=1 edge case:** `pow10[a]=0`, all weights 0 ‚Üí `S[i]={0}` for all i ‚Üí any digit works ‚Üí "999...9".
- **Large n with small periods:** `pow10[a]` for k=2,3,4,5,6,8,9 repeats within 3 steps; for k=7 within 6 steps. Precomputing all n values is O(n) and correct.
