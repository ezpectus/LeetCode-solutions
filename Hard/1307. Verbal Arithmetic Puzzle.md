# 1307. Verbal Arithmetic Puzzle
## Backtracking + Coefficient Pruning

---

## Problem Statement

- Assign digits 0-9 to unique letters (bijection, no leading zeros).
- Check if the resulting arithmetic equation holds: `sum(words) == result`.
- At most 10 unique characters.

---

## Core Idea — Three Phases

### Phase 1: Column Coefficients

Instead of evaluating the full equation each time, precompute each character's **total positional contribution**:

```
coeff[c] = Σ (10^k for each position k where c appears in a word)
         - Σ (10^k for each position k where c appears in result)
```

The equation holds iff `Σ coeff[c] * digit[c] == 0`.

Example: in `SEND + MORE = MONEY`, `D` at position 0 in `SEND` → `coeff['D'] += 1`. `Y` at position 0 in `MONEY` → `coeff['Y'] -= 1`.

### Phase 2: Sort by |coeff| Descending

Assign digits to high-impact characters first. This means early branches in the backtracking tree eliminate large chunks of the search space.

### Phase 3: Suffix Bounds Pruning

For characters not yet assigned (indices `idx..n-1`), compute the **minimum and maximum possible contribution** to the total. If the current partial `total` can't possibly reach 0 given remaining freedom, prune immediately:

```
min_suf[idx] ≤ -total ≤ max_suf[idx]  →  continue
else → prune
```

Without pruning: up to `10! = 3.6M` assignments. With pruning: typically < 1000 nodes visited.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int n;
    long[] coeffs;
    bool[] isLeading, used = new bool[10];
    long[] minSuf, maxSuf;

    public bool IsSolvable(string[] words, string result) {
        var charSet = new HashSet<char>();
        foreach (var w in words) foreach (var c in w) charSet.Add(c);
        foreach (var c in result) charSet.Add(c);
        var chars = charSet.ToList();
        n = chars.Count;

        var leading = new HashSet<char>();
        foreach (var w in words.Append(result)) if (w.Length > 1) leading.Add(w[0]);

        var coeff = new Dictionary<char, long>();
        foreach (var c in chars) coeff[c] = 0;

        int maxLen = words.Append(result).Max(w => w.Length);
        for (int col = 0; col < maxLen; col++) {
            long place = 1;
            for (int i = 0; i < col; i++) place *= 10;
            foreach (var w in words)
                if (col < w.Length) coeff[w[w.Length-1-col]] += place;
            if (col < result.Length) coeff[result[result.Length-1-col]] -= place;
        }

        // Sort by |coeff| descending
        chars.Sort((a, b) => Math.Abs(coeff[b]).CompareTo(Math.Abs(coeff[a])));
        coeffs    = chars.Select(c => coeff[c]).ToArray();
        isLeading = chars.Select(c => leading.Contains(c)).ToArray();

        // Suffix min/max contribution
        minSuf = new long[n+1]; maxSuf = new long[n+1];
        for (int i = n-1; i >= 0; i--) {
            int dMin = isLeading[i] ? 1 : 0;
            long lo = Math.Min(coeffs[i]*dMin, coeffs[i]*9);
            long hi = Math.Max(coeffs[i]*dMin, coeffs[i]*9);
            minSuf[i] = minSuf[i+1] + lo;
            maxSuf[i] = maxSuf[i+1] + hi;
        }

        return Bt(0, 0);
    }

    bool Bt(int idx, long total) {
        if (idx == n) return total == 0;
        if (!(-total >= minSuf[idx] && -total <= maxSuf[idx])) return false;

        for (int d = 0; d <= 9; d++) {
            if (used[d] || (d == 0 && isLeading[idx])) continue;
            used[d] = true;
            if (Bt(idx+1, total + coeffs[idx]*d)) return true;
            used[d] = false;
        }
        return false;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time (worst)** | **O(10!)** | 10 unique chars × 10 digit choices |
| **Time (practical)** | **Much less** | Bounds pruning eliminates most branches early |
| **Space** | **O(n)** | Recursion depth ≤ 10 |

With pruning + sorted coefficients: worst-case inputs run in ~10ms.

---

## Why Coefficient Approach

Direct simulation: for each assignment, recompute the full sum → O(word_len) per leaf node. Coefficient approach: precompute once, then evaluation is O(n) — and crucially, the **partial sum** at depth `idx` already reflects all assigned characters, enabling tight bounds checking.

---

## Suffix Bounds Pruning — Key Insight

At depth `idx`, `total` accumulates contributions from assigned chars 0..idx-1. For the equation to hold, remaining chars must contribute exactly `-total`.

If `-total < minSuf[idx]` or `-total > maxSuf[idx]`, no assignment of remaining digits can satisfy the equation → prune the entire subtree immediately.

This turns the search from `10! = 3.6M` nodes to typically a few thousand.

---

## Example: SEND + MORE = MONEY

Unique chars: `{S,E,N,D,M,O,R,Y}` = 8 chars.

Coefficients (computed from positions):
- `S`: `+1000` (col 3 in SEND)
- `E`: `+10 + 100 - 10` = `+100` (col 1 in SEND, col 2 in MORE, col 1 in MONEY... etc.)
- Sorted by |coeff| → high positional chars assigned first

Backtracking assigns digits, prunes when partial total is already out of reachable range.

Solution found: `S=9,E=5,N=6,D=7,M=1,O=0,R=8,Y=2` → `9567+1085=10652` ✓

---

## Pitfalls & Edge Cases

- **Leading zero**: a word of length > 1 can't start with 0 → collect all such first chars into `leading` set
- **Single-char words**: no leading zero constraint for single chars (e.g. word `"A"` — `A` can be 0)
- **`long` for coefficients**: `10^6` positional value × up to `words.length=5` words → max coeff ~5×10⁶; stays in `int` range but use `long` for `total` which accumulates all
- **Result char coefficients are negative**: subtracted because `result` is on the RHS of the equation

---

## Key Takeaway

Verbal arithmetic → **backtracking with three optimizations**:
1. **Coefficient precomputation** — reduces equation check to a running sum
2. **Sort by |coeff|** — high-impact chars assigned first for faster pruning
3. **Suffix min/max bounds** — prune when partial total can't reach 0 with remaining digits

Without pruning: TLE. With all three: runs in milliseconds even for 10-char puzzles.
