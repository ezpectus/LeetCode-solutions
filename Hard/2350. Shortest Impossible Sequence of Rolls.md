# 2350. Shortest Impossible Sequence of Rolls

## O(n) — Greedy Window Counting

---

## Problem Statement

- Given array `rolls` of length `n` and integer `k` (k-sided dice).
- Find length of shortest sequence that **cannot** be formed as subsequence from `rolls`.
- **Constraints**: 1 ≤ n ≤ 10⁵, 1 ≤ rolls[i] ≤ k ≤ 10⁵

---

## Core Idea — Count Complete Windows

**Key observation**: A sequence of length `L` cannot be formed if we cannot find all `k^L` possible sequences of length `L` as subsequences.

**Greedy insight**: 
- For length 1: need all k values [1, 2, ..., k] to appear
- For length 2: need all k² combinations as subsequences
- For length L: need all k^L combinations

**Window approach**:
- Scan `rolls` left to right
- Track how many "complete windows" we can form
- A complete window = segment containing all k distinct values
- Answer = (number of complete windows) + 1

**Why +1?** If we have `m` complete windows:
- All sequences of length ≤ m can be formed
- But NOT all sequences of length m+1
- So shortest impossible = m+1

---

## Algorithm

```
completeWindows = 0
seen = empty set

for roll in rolls:
  seen.add(roll)
  
  if seen.size() == k:
    // Found complete window
    completeWindows++
    seen.clear()

return completeWindows + 1
```

---

## Full Implementation (C#)

```csharp
public class Solution {
    public int ShortestSequence(int[] rolls, int k) {
        int completeWindows = 0;
        var seen = new HashSet<int>();
        
        foreach (int roll in rolls) {
            seen.Add(roll);
            
            // Complete window: all k values seen
            if (seen.Count == k) {
                completeWindows++;
                seen.Clear();
            }
        }
        
        return completeWindows + 1;
    }
}
```

## Why This Works — Example Walkthrough

### Example 1: `rolls = [4,2,1,2,3,3,2,4,1], k = 4`

**Scan**:
- i=0: roll=4, seen={4}
- i=1: roll=2, seen={4,2}
- i=2: roll=1, seen={4,2,1}
- i=3: roll=2, seen={4,2,1}
- i=4: roll=3, seen={4,2,1,3} → **complete window 1**, clear
- i=5: roll=3, seen={3}
- i=6: roll=2, seen={3,2}
- i=7: roll=4, seen={3,2,4}
- i=8: roll=1, seen={3,2,4,1} → **complete window 2**, clear

**Complete windows**: 2  
**Answer**: 2 + 1 = 3 ✓

**Explanation**:
- All length-1 sequences [1], [2], [3], [4] exist (window 1 has all)
- All length-2 sequences exist (can combine windows 1 and 2)
- Not all length-3 sequences exist (e.g., [1,4,2] missing)

### Example 2: `rolls = [1,1,2,2], k = 2`

**Scan**:
- i=0: roll=1, seen={1}
- i=1: roll=1, seen={1}
- i=2: roll=2, seen={1,2} → **complete window 1**, clear
- i=3: roll=2, seen={2}

**Complete windows**: 1  
**Answer**: 1 + 1 = 2 ✓

### Example 3: `rolls = [1,1,3,2,2,2,3,3], k = 4`

**Scan**:
- All rolls processed, but never see all 4 values together
- seen never reaches size 4
- No complete windows

**Complete windows**: 0  
**Answer**: 0 + 1 = 1 ✓

**Explanation**: Value 4 never appears, so [4] cannot be formed.

## Mathematical Proof

**Claim**: Answer = (complete windows) + 1

**Proof**:
- Let `m` = number of complete windows
- Each complete window contributes ability to form one more position in sequences
- With `m` windows, we can form all sequences of length ≤ m:
  - Length 1: use any value from window 1
  - Length 2: use any value from window 1, then any from window 2
  - Length m: pick one value from each of m windows
- For length m+1: need m+1 windows, but we only have m
- Therefore, some sequence of length m+1 cannot be formed

**Example with k=2**:
- 1 window: can form [1], [2], [1,1], [1,2], [2,1], [2,2]
- Cannot form all length-3 sequences (need 2³=8, but only have 2²×2=8... wait)

Actually let me reconsider...

**Correct reasoning**:
- With m complete windows, we can pick one value from each window to form length-m sequence
- Since each window has all k values, we can form all k^m sequences of length m
- But for length m+1, we'd need m+1 independent choices, which requires m+1 windows
- So shortest impossible = m+1

---

## Edge Cases

- **No complete windows** (some value never appears): answer = 1 (that missing value is shortest impossible)
- **All same value**: answer = 1 (any other value is impossible)
- **Perfectly covers all k values once**: 1 window, answer = 2

---

## Key Takeaway

**Count complete windows (segments with all k distinct values)**. Each window allows forming one additional position in sequences. Answer = windows + 1.

O(n) time, O(k) space. **Critical insight**: Greedy window counting directly computes maximum achievable sequence length.
