# 3534. Path Existence Queries in a Graph II
## O((n + q) log n) — Sort + Monotone Far-Pointer + Binary Lifting

---

## Problem Statement

Undirected graph where nodes `i` and `j` are connected iff `|nums[i] - nums[j]| <= maxDiff`. For each query `[u, v]`, find the shortest path length (BFS distance), or -1 if no path exists.

---

## Core Insight #1 — Sort and Collapse to a 1D Range Problem

Sort nodes by their `nums` values. In sorted order, node at position `k` connects to all nodes in some contiguous range `[leftk, rightk]` around it (those within maxDiff of `nums[k]`).

Define `far[k]` = rightmost sorted position reachable from `k` in one hop:
```
far[k] = last position j such that sn[j] <= sn[k] + maxDiff
```

Since `sn` (sorted nums) is non-decreasing, `far[k]` is also **non-decreasing** — key property for everything below.

---

## Core Insight #2 — BFS Expands Like a Jump Game

For a query from sorted position `a` to `b` (a ≤ b), BFS "waves" expand rightward:

```
f₀ = a
f₁ = far[a]                 (frontier after 1 hop)
f₂ = far[far[a]] = far[f₁]  (frontier after 2 hops)
fₖ = far[fₖ₋₁]             (frontier after k hops)
```

**Why is leftward expansion irrelevant?**  
From sorted position `a`, we can also jump LEFT (to positions < a with small difference). But since `far` is non-decreasing, any leftward neighbor `j < a` has `far[j] ≤ far[a]` — it can't extend the rightward frontier beyond what position `a` already provides. So we only track the rightward frontier.

**The shortest path** from `a` to `b` = minimum `k` such that `farᵏ(a) ≥ b`.

---

## Core Insight #3 — Binary Lifting for Fast Queries

Define `jump[k][t]` = position reached after applying `far` exactly `2^t` times from `k`:
```
jump[k][0] = far[k]
jump[k][t] = jump[jump[k][t-1]][t-1]    (apply 2^(t-1) steps, then 2^(t-1) more)
```

**Minimum hops from a to b (binary descent):**
```
pos = a, steps = 0
For t from LOG-1 down to 0:
    if jump[pos][t] < b:
        pos = jump[pos][t]
        steps += 2^t
return steps + 1
```

After the loop, `far[pos] ≥ b` (one more hop covers `b`). Total: `steps + 1`.

---

## Connectivity

Two sorted positions `a` and `b` are in **different components** iff there exists a position `k` with `a ≤ k < b` where `far[k] < k+1` (a "gap" in the sorted order that no hop can bridge).

Precompute component IDs with a single pass:
```
comp[0] = 0
comp[k] = comp[k-1] + (far[k-1] < k ? 1 : 0)
```
Then `a` and `b` are connected iff `comp[a] == comp[b]`.

## Full Implementation (C++)

```cpp
class Solution {
public:
    vector<int> pathExistenceQueries(int n, vector<int>& nums, int maxDiff,
                                      vector<vector<int>>& queries) {
        const int LOG = 17; // 2^17 > 10^5

        // Sort node indices by nums value
        vector<int> order(n);
        iota(order.begin(), order.end(), 0);
        sort(order.begin(), order.end(), [&](int i, int j){ return nums[i] < nums[j]; });

        vector<int> sn(n);   // sorted nums
        vector<int> rk(n);   // rank[v] = sorted position of node v
        for (int i = 0; i < n; i++) { sn[i] = nums[order[i]]; rk[order[i]] = i; }

        // far[k]: rightmost position reachable from k in one hop (two-pointer, O(n))
        vector<int> far(n);
        for (int k = 0, r = 0; k < n; k++) {
            r = max(r, k);
            while (r + 1 < n && sn[r + 1] <= sn[k] + maxDiff) r++;
            far[k] = r;
        }

        // Component IDs: new component starts when far[k-1] < k
        vector<int> comp(n, 0);
        for (int k = 1; k < n; k++)
            comp[k] = comp[k-1] + (far[k-1] < k ? 1 : 0);

        // Binary lifting: jump[k][t] = far applied 2^t times from position k
        vector<vector<int>> jump(n, vector<int>(LOG));
        for (int k = 0; k < n; k++) jump[k][0] = far[k];
        for (int t = 1; t < LOG; t++)
            for (int k = 0; k < n; k++)
                jump[k][t] = jump[jump[k][t-1]][t-1];

        // Answer each query
        vector<int> ans(queries.size());
        for (int i = 0; i < (int)queries.size(); i++) {
            int a = rk[queries[i][0]], b = rk[queries[i][1]];
            if (a > b) swap(a, b);
            if (a == b)              { ans[i] = 0;  continue; }
            if (comp[a] != comp[b]) { ans[i] = -1; continue; }
            // Binary descent: find smallest steps such that far^steps(a) >= b
            int pos = a, steps = 0;
            for (int t = LOG - 1; t >= 0; t--)
                if (jump[pos][t] < b) { pos = jump[pos][t]; steps += (1 << t); }
            ans[i] = steps + 1;
        }
        return ans;
    }
};
```

## Complexity

| Step | Time | Notes |
|---|---|---|
| Sort + rank | O(n log n) | |
| `far[]` (two pointers) | O(n) | `r` moves right at most n times total |
| `comp[]` | O(n) | single pass |
| Binary lifting | O(n log n) | n × LOG table |
| Per query | O(log n) | LOG-bit descent |
| **Total** | **O((n + q) log n)** | ~3.4M ops for n=q=10⁵ |
| Space | O(n log n) | jump table |

## Example Walkthrough

### Example 1 — `nums=[1,8,3,4,2]`, `maxDiff=3`, `queries=[[0,3],[2,4]]`

**Sorted order:** node0(1), node4(2), node2(3), node3(4), node1(8).
`sn=[1,2,3,4,8]`, `rk=[0,4,2,3,1]`.

**`far[]` (two-pointer):**
- k=0: extend r to 3 (sn[3]=4≤1+3, sn[4]=8>4). `far[0]=3`.
- k=1,2,3: r stays at 3. `far[1]=far[2]=far[3]=3`.
- k=4: r=4. `far[4]=4`.

**Components:** far[3]=3 < 4 → gap. `comp=[0,0,0,0,1]`.

**Query [0,3]:** a=rk[0]=0, b=rk[3]=3. Same comp. jump[0][0]=3≥3. No bits applied. `ans = 0+1 = 1` ✓

**Query [2,4]:** a=rk[4]=1, b=rk[2]=2. Same comp. jump[1][0]=3≥2. No bits. `ans = 1` ✓

## Pitfalls & Edge Cases

- **`a == b` (same sorted position):** This happens both when `u == v` AND when two distinct nodes with the same `nums` value are in the same "slot"... but wait, sorting is deterministic (one position per node). If `u == v`, `rk[u] == rk[v]`, distance = 0. ✓

- **Multiple nodes with the same `nums[i]`:** They're all within maxDiff=0 of each other (always connected). After sorting, consecutive equal values have `sn[k+1] - sn[k] = 0 ≤ maxDiff`, so they're in the same component. ✓

- **`maxDiff = 0`:** Each node only connects to nodes with identical `nums` values. Components = groups of equal values. ✓

- **`far` two-pointer correctness:** `r` is initialized to 0 and only moves forward. At each `k`, `r = max(r, k)` prevents `r` from being behind `k`. Total moves: O(n). ✓

- **Binary lifting fixed point:** `jump[n-1][t] = n-1` for all `t` (the rightmost position can't go further). All indices stay within `[0, n-1]`. ✓

---

## Key Takeaway

This is the **"sorted-window-adjacency + jump-game binary lifting"** pattern. When edges connect nodes within a sorted range, the BFS expansion from position `a` to `b` is exactly the "minimum jumps" problem on 1D positions where `far[k]` is the reach of position `k`. The monotonicity of `far` allows O(n) preprocessing and O(log n) queries via binary lifting.

**Pattern recognition:** "Shortest path in a graph where adjacency is determined by a value proximity threshold" → sort by value, compute rightward reach `far[k]`, build binary lifting on `far`, answer queries with binary descent. Connectivity is detected by gaps in `far`.
