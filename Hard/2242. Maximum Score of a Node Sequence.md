# 2242. Maximum Score of a Node Sequence
## O(E × 9) — Top-3 Neighbors per Node + Edge Enumeration

---

## Problem Statement

- Find a path of 4 nodes `a - u - v - b` (all connected, all distinct) maximizing sum of scores.

---

## Core Idea — For Each Edge, Try Top-3 Neighbors

For a valid 4-node sequence `a - u - v - b`:
- Edge `(u,v)` is the middle edge.
- `a` is a neighbor of `u` (not `v`).
- `b` is a neighbor of `v` (not `u`, not `a`).

To maximize, we want `a` = best neighbor of `u` and `b` = best neighbor of `v`. But we need to avoid conflicts (`a ≠ v`, `b ≠ u`, `b ≠ a`).

**Key insight**: keep only the **top 3 neighbors** (by score) for each node. Among 3 choices for `a` and 3 for `b`, at least one valid pair `(a,b)` exists that avoids all conflicts (9 combinations to check).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumScore(int[] scores, int[][] edges) {
        int n = scores.Length;
        var top3 = new List<int>[n];
        for (int i = 0; i < n; i++) top3[i] = new();

        foreach (var e in edges) {
            foreach (var (node, nb) in new[]{(e[0],e[1]),(e[1],e[0])}) {
                top3[node].Add(nb);
                top3[node] = top3[node].OrderByDescending(x => scores[x]).Take(3).ToList();
            }
        }

        int res = -1;
        foreach (var e in edges) {
            int u = e[0], v = e[1];
            foreach (int a in top3[u]) {
                if (a == v) continue;
                foreach (int b in top3[v]) {
                    if (b == u || b == a) continue;
                    res = Math.Max(res, scores[a]+scores[u]+scores[v]+scores[b]);
                }
            }
        }
        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(E × 9)** = **O(E)** | Each edge checks 3×3=9 pairs |
| **Space** | **O(n)** | Top-3 lists |

---

## Why Top-3 Suffices

For a given edge `(u,v)`, we need `a ∈ top_neighbors(u)` and `b ∈  top_neighbors(v)` such that `a ≠ v`, `b ≠ u`, `b ≠ a`. With only 3 nodes to exclude (v, u, and one another), and 3 candidates each, we're guaranteed at least one valid pair unless the graph is too sparse.

---

## Key Takeaway

"Max 4-node path score" → **top-3 neighbors per node + enumerate middle edge + try 9 outer pairs**. O(E) after O(E log 3) preprocessing.
