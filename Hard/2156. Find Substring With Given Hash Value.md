# 2156. Find Substring With Given Hash Value
## O(n) — Rolling Hash Right-to-Left

---

## Problem Statement

- `hash(s[i..i+k-1]) = Σ val(s[i+j]) × power^j mod modulo` for j = 0..k-1.
- Find the **first** substring of length k with the given hash value.

---

## Core Idea — Scan Right to Left, Avoid Modular Inverse

**Why right to left?** The rolling hash recurrence when moving left requires subtracting the rightmost element. Moving left from window `[i+1, i+k]` to `[i, i+k-1]`:

```
h[i] = val(s[i]) + power × h[i+1] − power^k × val(s[i+k])   (mod modulo)
```

This only requires **multiplication** (no division/modular inverse), since we're shifting the polynomial left by adding a lower-degree term and removing the highest-degree term via subtraction.

**Algorithm**:
1. Compute `h[n-k]` (rightmost window) directly in O(k).
2. Scan right to left: `h[i] = val(s[i]) + power × h[i+1] - power^k × val(s[i+k])` (mod modulo).
3. Track the **last** matching position (since we scan right to left, the last match = leftmost = first in original string).
4. Return `s[res..res+k-1]`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string SubStrHash(string s, int power, int modulo, int k, int hashValue) {
        int n = s.Length;
        long mod = modulo;
        long pw = power;

        // Precompute power^k mod modulo
        long pk = 1;
        for (int i = 0; i < k; i++) pk = pk * pw % mod;

        // Compute hash of rightmost window [n-k, n-1]
        long h = 0, p = 1;
        for (int j = 0; j < k; j++) {
            h = (h + (s[n-k+j] - 'a' + 1) * p) % mod;
            if (j < k-1) p = p * pw % mod;
        }

        int res = -1;
        if (h == hashValue) res = n-k;

        // Slide right to left
        for (int i = n-k-1; i >= 0; i--) {
            h = ((s[i] - 'a' + 1) + pw * h - pk * (s[i+k] - 'a' + 1)) % mod;
            if (h < 0) h += mod;
            if (h == hashValue) res = i;
        }

        return s.Substring(res, k);
    }
}
```

## Recurrence Derivation

Window `[i+1, i+k]` has hash:
```
h[i+1] = val(s[i+1])·p⁰ + val(s[i+2])·p¹ + ... + val(s[i+k])·p^(k-1)
```

Window `[i, i+k-1]` has hash:
```
h[i] = val(s[i])·p⁰ + val(s[i+1])·p¹ + ... + val(s[i+k-1])·p^(k-1)
     = val(s[i]) + p · (val(s[i+1])·p⁰ + ... + val(s[i+k-1])·p^(k-2))
     = val(s[i]) + p · h[i+1] - p^k · val(s[i+k])
```

The last term removes `val(s[i+k])·p^(k-1)` from `p·h[i+1]` (which had it at `p^k` weight).

## Pitfalls & Edge Cases

- **Negative modulo**: `h` can go negative due to the subtraction. Add `mod` if `h < 0`: `if (h < 0) h += mod`.
- **`long` arithmetic**: `pw × h` can reach up to `10⁹ × (10⁹-1)` ≈ `10¹⁸` — must use `long`.
- **Track last match, not first**: since we scan right to left, the last position that matches (smallest `i`) is the answer — overwrite `res` every time to keep the leftmost.
- **`power^k` precomputed**: used in every sliding step, compute once.

---

## Key Takeaway

"Find first substring with given rolling hash" → **scan right to left using `h[i] = val(s[i]) + power × h[i+1] - power^k × val(s[i+k])`**. No modular inverse needed. Track last matching index = leftmost (first) occurrence.
