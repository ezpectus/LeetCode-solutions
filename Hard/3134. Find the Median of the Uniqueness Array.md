## 3134. Find the Median of the Uniqueness Array

### Problem Statement
You are given an integer array `nums`. The **uniqueness array** of `nums` is a sorted array containing the number of distinct elements of all possible subarrays of `nums`.

Return the **median** of the uniqueness array. If there are two choices for a median, take the smaller value.

---

### Core Insight — Binary Search on Answer + Sliding Window
The total number of subarrays for an array of length `N` is given by the formula:
`totalSubarrays = N * (N + 1) / 2`

The uniqueness value (number of distinct elements) for any subarray ranges from 1 to `N`. Generating the uniqueness array explicitly is impossible due to the O(N^2) total count of subarrays (since `N <= 10^5`). However, the count property is monotonic: if we guess a uniqueness value `mid`, we can easily count how many subarrays have a uniqueness value less than or equal to `mid`.

This monotonic property allows us to use **Binary Search** on the answer over the range `[1, N]`. For a chosen target `mid`:
* We find the number of subarrays with `<= mid` distinct elements using a **Sliding Window (Two Pointers)** approach.
* If the total number of subarrays matching this condition is `>= (totalSubarrays + 1) / 2`, then `mid` is a valid candidate for the median, and we try to find a smaller valid value by shrinking our upper bound.

---

### Algorithm
1. Calculate the total number of subarrays, and determine the target threshold required to reach the median index: `targetCount = (totalSubarrays + 1) / 2`.
2. Set up binary search pointers: `low = 1`, `high = N`, and `ans = N`.
3. While `low <= high`:
   * Set `mid = low + (high - low) / 2`.
   * Count how many subarrays have at most `mid` distinct elements using a sliding window:
     * Expand the window by adding `nums[right]`. Track element frequencies in a hash map or frequency array.
     * While the count of unique elements in the map exceeds `mid`, shrink the window from the left by removing `nums[left]`.
     * The number of valid subarrays ending at `right` is `right - left + 1`. Add this to a running total.
   * If the running total of subarrays is `>= targetCount`, then `mid` can be the median. Record `ans = mid` and move `high = mid - 1`.
   * Otherwise, move `low = mid + 1`.
4. Return `ans`.

---

### Full Optimal Implementation (C#)
```csharp
using System;
using System.Collections.Generic;

public class Solution {
    public int MedianOfUniquenessArray(int[] nums) {
        int n = nums.Length;
        long totalSubarrays = (long)n * (n + 1) / 2;
        long targetCount = (totalSubarrays + 1) / 2;

        int low = 1;
        int high = n;
        int ans = n;

        // Binary Search on the uniqueness value
        while (low <= high) {
            int mid = low + (high - low) / 2;

            if (CountSubarraysWithMaxDistinct(nums, mid) >= targetCount) {
                ans = mid;
                high = mid - 1; // Try to find a smaller valid uniqueness score
            } else {
                low = mid + 1;  // Uniqueness score is too low
            }
        }

        return ans;
    }

    // Sliding window helper to count subarrays with <= maxDistinct unique elements
    private long CountSubarraysWithMaxDistinct(int[] nums, int maxDistinct) {
        long count = 0;
        int left = 0;
        Dictionary<int, int> freqMap = new Dictionary<int, int>();

        for (int right = 0; right < nums.Length; right++) {
            if (!freqMap.ContainsKey(nums[right])) {
                freqMap[nums[right]] = 0;
            }
            freqMap[nums[right]]++;

            // Shrink the window if unique element count exceeds our target constraint
            while (freqMap.Count > maxDistinct) {
                freqMap[nums[left]]--;
                if (freqMap[nums[left]] == 0) {
                    freqMap.Remove(nums[left]);
                }
                left++;
            }

            // All subarrays starting from index 'left' up to 'right' are valid
            count += (right - left + 1);
        }

        return count;
    }
}
```
### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N log N) | The binary search takes O(log N) iterations. Inside each iteration, the sliding window passes through the array elements in O(N) linear time using hash map operations. |
| **Space** | O(N) | The tracking dictionary allocates unique entries up to the total number of distinct values present inside `nums`. |

### Pitfalls & Edge Cases
* **Integer Overflow on Counts:** The total number of subarrays for `N = 10^5` is roughly 5 * 10^9, which exceeds the max capacity of a standard 32-bit signed integer. Ensure `totalSubarrays`, `targetCount`, and window accumulations utilize `long` types.
* **Median Ceiling Definition:** The problem statement specifies that if two choices exist, pick the smaller element. This corresponds to the structural formula `(totalSubarrays + 1) / 2`, matching up perfectly with our standard binary search upper-bound reduction branch.

When asked to locate an ordered rank value (like a median or Kth smallest element) from a collection too large to fit in memory or compute explicitly, see if the condition allows binary searching on the *result space* instead of the *index space*.
