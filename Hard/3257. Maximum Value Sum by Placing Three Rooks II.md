# 3257. Maximum Value Sum by Placing Three Rooks II
## O(mn) — Fix the Middle Row, Prefix/Suffix Top-3 Column Maxima

---

## Problem Statement

Place 3 non-attacking rooks on an `m × n` board (no two in the same row or column). Maximize the sum of the 3 selected cell values.

---

## Core Insight #1 — Fix the "Middle" Row and Enumerate

Any valid triple of rows `(r1, r2, r3)` with `r1 < r2 < r3` has a unique **middle row** `r2`. For each possible `r2 = j` (ranging from 1 to m-2):
- The **top rook** comes from some row `< j`.
- The **bottom rook** comes from some row `> j`.
- All three must use **distinct columns**.

This decomposes the problem into: for each middle row `j`, find the best (value, column) from rows above + best from rows below + best from row j itself, with all columns distinct.

---

## Core Insight #2 — Prefix/Suffix Column Maxima

For column `c`, define:
```
prefBest[c] = max value of board[i][c]   for i < j   (best achievable from column c above j)
sufBest[c]  = max value of board[i][c]   for i > j   (best achievable from column c below j)
```

The **top rook's contribution** is `prefBest[c_top]` for whichever column `c_top` it uses. Similarly for bottom. This absorbs row-selection: we don't care which specific row above j the top rook is placed in — just take the maximum for that column.

---

## Core Insight #3 — Top-3 per Layer Is Sufficient

For each layer (prefix, middle row, suffix), we only need the **top 3 (value, column) pairs** — because:
- At most **2 columns** are blocked by the other two rooks.
- The 3rd entry in the top-3 is always unblocked (by pigeonhole: 3 entries, 2 blocked → at least 1 valid).

So for each middle row `j`: enumerate 3 × 3 × 3 = **27 combinations** from (prefix top-3) × (row j top-3) × (suffix top-3), reject those with column conflicts, take the maximum. O(27) per `j`.

---

## Full Implementation (C++)

```cpp
class Solution {
public:
    long long maximumValueSum(vector<vector<int>>& board) {
        int m = board.size(), n = board[0].size();
        const long long NEG = LLONG_MIN / 2;

        // Linear scan: return top 3 (val, col) from arr[], descending
        auto getTop3 = [&](vector<long long>& arr) {
            vector<pair<long long, int>> t = {{NEG,-1},{NEG,-1},{NEG,-1}};
            for (int c = 0; c < n; c++) {
                if      (arr[c] > t[0].first) { t[2]=t[1]; t[1]=t[0]; t[0]={arr[c],c}; }
                else if (arr[c] > t[1].first) { t[2]=t[1]; t[1]={arr[c],c}; }
                else if (arr[c] > t[2].first) { t[2]={arr[c],c}; }
            }
            return t;
        };

        // Top 3 per row
        vector<vector<pair<long long,int>>> rowT3(m);
        for (int i = 0; i < m; i++) {
            vector<long long> row(n);
            for (int c = 0; c < n; c++) row[c] = board[i][c];
            rowT3[i] = getTop3(row);
        }

        // Prefix: prefT3[j] = top 3 cols by max(board[i][c] for i < j)
        vector<vector<pair<long long,int>>> prefT3(m);
        vector<long long> prefBest(n, NEG);
        for (int j = 1; j < m; j++) {
            for (int c = 0; c < n; c++)
                prefBest[c] = max(prefBest[c], (long long)board[j-1][c]);
            prefT3[j] = getTop3(prefBest);
        }

        // Suffix: sufT3[j] = top 3 cols by max(board[i][c] for i > j)
        vector<vector<pair<long long,int>>> sufT3(m);
        vector<long long> sufBest(n, NEG);
        for (int j = m-2; j >= 0; j--) {
            for (int c = 0; c < n; c++)
                sufBest[c] = max(sufBest[c], (long long)board[j+1][c]);
            sufT3[j] = getTop3(sufBest);
        }

        // Main: fix middle row j, try all valid 27 triples
        long long ans = NEG;
        for (int j = 1; j < m-1; j++) {
            for (auto& [vj, cj] : rowT3[j]) {
                for (auto& [vp, cp] : prefT3[j]) {
                    if (cp == cj) continue;
                    for (auto& [vs, cs] : sufT3[j]) {
                        if (cs == cj || cs == cp) continue;
                        ans = max(ans, vj + vp + vs);
                    }
                }
            }
        }

        return ans;
    }
};
```

## Complexity

| Step | Time | Notes |
|---|---|---|
| Row top-3 | O(mn) | O(n) per row |
| Prefix prefBest update | O(mn) | O(n) per j |
| Prefix top-3 | O(mn) | O(n) linear scan per j |
| Suffix (same) | O(mn) | symmetric |
| Main triple loop | O(27m) | 27 combos × m rows |
| **Total** | **O(mn)** | m=n=500 → 250K ops |
| Space | O(m + n) | rolling arrays + top-3 tables |

---

## Why 3 Entries Per Layer Is Always Sufficient

For a given middle row `j`, once we choose:
- Middle column `c_j` (from rowT3[j])
- Prefix column `c_p` (must ≠ c_j)
- Suffix column `c_s` (must ≠ c_j, must ≠ c_p)

**Proof:** The prefix top-3 contains 3 distinct columns. At most 1 (c_j) is blocked → at least 2 prefix options exist. For each valid prefix column `c_p`, the suffix has 2 blocked columns (c_j, c_p). The suffix top-3 has 3 distinct columns → at least 1 unblocked. By exhausting 3 × 3 × 3 = 27 combinations (and skipping conflicts), we always hit the valid optimum.

If `n = 3` exactly: all 3 columns must be used (1 per rook). The top-3 arrays contain all 3 columns; the enumeration tests all assignments.

## Example Walkthrough

### Example 1 — `board = [[-3,1,1,1],[-3,1,-3,1],[-3,2,1,1]]`, m=3, n=4

**Row top-3:** Row 0: (1,1),(1,2),(1,3). Row 1: (1,1),(1,3),(-3,0). Row 2: (2,1),(1,2),(1,3).

**prefT3[j=1]** (from row 0): (1,1),(1,2),(1,3).

**sufT3[j=1]** (from row 2): (2,1),(1,2),(1,3).

**Main loop, j=1:**

| cj (row 1) | cp (prefix, ≠cj) | cs (suffix, ≠cj,cp) | Sum |
|---|---|---|---|
| 3, val=1 | 2, val=1 | 1, val=2 | 1+1+2=**4** ← |
| 3, val=1 | 1, val=1 | 2, val=1 | 1+1+1=3 |
| 1, val=1 | 2, val=1 | 3, val=1 | 3 |
| ... | | | ≤4 |

**Output: 4** ✓ (rooks at row0,col2 + row1,col3 + row2,col1)

---

### Example 2 — `board = [[1,2,3],[4,5,6],[7,8,9]]`

**j=1:** prefT3 = (3,2),(2,1),(1,0). sufT3 = (9,2),(8,1),(7,0). rowT3[1] = (6,2),(5,1),(4,0).

Best triple with distinct cols: cj=2, cp=1, cs=0 → 6+2+7=15. Or cj=2, cp=0, cs=1 → 6+1+8=15. All combinations give **15** ✓

## Pitfalls & Edge Cases

- **Why `prefBest[c]` (column max over all prefix rows) rather than single-row values?** When we say "place the top rook in column c", we want the absolute best value in column c from ANY row above j. Taking column-wise maximums handles this.

- **Prefix top-3 must be recomputed for each j** (as j increases, new rows enter the prefix). Done incrementally: update prefBest with row j-1, then compute top-3 in O(n). Not a sort — a single O(n) linear scan with 3-way insertion.

- **Handling ties in top-3:** If two columns have the same value, both are included independently. The distinctness check is on column index, not value. So (val=1, col=2) and (val=1, col=3) are separate valid entries.

- **NEG_INF initialization:** `prefBest` and `sufBest` start at `LLONG_MIN/2` (not `LLONG_MIN`) to avoid overflow when adding 3 such values together.

- **m=3 minimum:** Only j=1 is tested as middle row. Both prefix (row 0) and suffix (row 2) contribute exactly 1 row each. Still works correctly.

---

## Key Takeaway

This is the **"fix the middle, compress the rest"** pattern. When placing k items with ordering constraints, fix the "median" item and independently precompute the best options above and below. For "no shared columns" constraints, keeping only the top-3 per layer guarantees finding the global optimum (by pigeonhole: 2 possible conflicts, 3 entries available → always one valid).

**Pattern recognition:** "Choose k items from different rows and columns, no two sharing a column" + large m, n → fix one row as anchor, precompute prefix/suffix column-wise maxima, enumerate top-k triples with column-conflict rejection.
