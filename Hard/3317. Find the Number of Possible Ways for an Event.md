# LeetCode 3317. Find the Number of Possible Ways for an Event

## Method Explanation

The problem asks us to find the number of ways to distribute $n$ performers into $x$ distinct stages and then assign a score from $1$ to $y$ to each non-empty stage. This can be modeled purely using **combinatorics** and basic dynamic programming to precompute combinations and partitions.

Let $k$ be the exact number of stages that end up being **active** (non-empty). The value of $k$ can range from $1$ to $\min(n, x)$. For a fixed number of active stages $k$, the total number of configurations is the product of three independent choices:

1. **Stage Selection:** Out of $x$ available distinct stages, we must choose exactly $k$ stages to be active. The number of ways to do this is given by the binomial coefficient:
   $$\binom{x}{k}$$

2. **Performer Partitioning:** We need to distribute $n$ distinct performers into $k$ distinct non-empty groups. The number of ways to partition $n$ distinct elements into $k$ *identical* non-empty subsets is defined by the **Stirling Numbers of the Second Kind**, denoted as $S(n, k)$. Because our stages are distinct (ordered), we multiply this by $k!$:
   $$\text{Ways} = S(n, k) \times k!$$

3. **Score Assignment:** Each of the $k$ active stages is independently awarded a score between $1$ and $y$. The number of ways to distribute these scores is:
   $$y^k$$

### Total Combination Formula
By iterating over all possible values of $k$, the total number of valid event arrangements is:
$$\text{Total Ways} = \sum_{k=1}^{\min(n, x)} \left( \binom{x}{k} \times S(n, k) \times k! \times y^k \right) \pmod{10^9 + 7}$$

We precompute both the combinations $\binom{n}{k}$ and Stirling numbers $S(n, k)$ in $\mathcal{O}(N^2)$ using their standard recurrence relations.

## C++ Source Code

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    int numberOfWays(int n, int x, int y) {
        long long MOD = 1e9 + 7;
        int max_limit = max({n, x, y});
        
        // 1. Precompute Binomial Coefficients (Combinations) using Pascal's Identity
        vector<vector<long long>> C(max_limit + 1, vector<long long>(max_limit + 1, 0));
        for (int i = 0; i <= max_limit; ++i) {
            C[i][0] = 1;
            for (int j = 1; j <= i; ++j) {
                C[i][j] = (C[i - 1][j - 1] + C[i - 1][j]) % MOD;
            }
        }
        
        // 2. Precompute Stirling Numbers of the Second Kind S(i, j)
        vector<vector<long long>> S(n + 1, vector<long long>(n + 1, 0));
        S[0][0] = 1;
        for (int i = 1; i <= n; ++i) {
            for (int j = 1; j <= i; ++j) {
                // S(i, j) = S(i-1, j-1) + j * S(i-1, j)
                S[i][j] = (S[i - 1][j - 1] + j * S[i - 1][j]) % MOD;
            }
        }
        
        // 3. Precompute Factorials for ordering the distinct active stages
        vector<long long> fact(max_limit + 1, 1);
        for (int i = 1; i <= max_limit; ++i) {
            fact[i] = (fact[i - 1] * i) % MOD;
        }
        
        // 4. Accumulate ways over all possible counts of active stages 'k'
        long long total_ways = 0;
        long long current_y_pow = 1; // Tracks y^k dynamically
        
        int limit = min(n, x);
        for (int k = 1; k <= limit; ++k) {
            current_y_pow = (current_y_pow * y) % MOD;
            
            // Ways = C(x, k) * S(n, k) * k! * y^k
            long long term1 = C[x][k];
            long long term2 = (S[n][k] * fact[k]) % MOD;
            long long combined = (term1 * term2) % MOD;
            long long final_term = (combined * current_y_pow) % MOD;
            
            total_ways = (total_ways + final_term) % MOD;
        }
        
        return total_ways;
    }
};
```
## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(\max(N, X, Y)^2)$
  * **Binomial Coefficient Precomputation:** Filling out the combination grid using Pascal's identity ($\binom{i}{j} = \binom{i-1}{j-1} + \binom{i-1}{j}$) requires updating every cell in a triangular layout. This takes exactly $\frac{\max(N, X, Y)^2}{2}$ structural iterations, operating at solid $\mathcal{O}(\max(N, X, Y)^2)$ efficiency.
  * **Stirling Matrix Partitioning:** Generating the Stirling numbers of the second kind via the recurrence structure $S(i, j) = S(i-1, j-1) + j \cdot S(i-1, j)$ requires processing an $N \times N$ tabular area. This adds an explicit $\mathcal{O}(N^2)$ runtime cost.
  * **Combinatorial Aggregation Loop:** Sweeping linearly from $k = 1$ to $\min(n, x)$ calculates the aggregated product terms. Each step executes basic arithmetic mutations (including dynamic tracking of modular power increments $y^k$) in strict $\mathcal{O}(1)$ time. This phase scales within a negligible $\mathcal{O}(\min(N, X))$ boundary.
  * **Total Time Bound:** The system execution limits are bound entirely by the nested generation loops for the auxiliary dynamic programming lookup tables. Given the extreme edge constraints where $N, X, Y \le 1000$, the calculation hits at most $10^6$ basic operations. This processes instantly on modern environments, wrapping up within 3–5 milliseconds.

* **Space Complexity:** $\mathcal{O}(\max(N, X, Y)^2)$
  * **Dynamic Programming Matrix Layouts:** The algorithm instantiates two multi-dimensional heap arrays: matrix `C` tracking combinations up to $\max(N, X, Y) \times \max(N, X, Y)$ entries, and matrix `S` containing Stirling data partitions sized at $N \times N$.
  * **Auxiliary Factorial Buffers:** A linear 1D vector `fact` is mapped out up to the maximum constraints to cache pre-calculated factorials in $\mathcal{O}(\max(N, X, Y))$ space.
  * **Total Memory Footprint:** The absolute static and dynamic allocations scale quadratically to $\mathcal{O}(\max(N, X, Y)^2)$. Storing two $1000 \times 1000$ grids of 64-bit integer values (`long long`) claims a lightweight, predictable footprint of roughly 16 MB, passing execution bounds safely without running into memory thrashing issues.
