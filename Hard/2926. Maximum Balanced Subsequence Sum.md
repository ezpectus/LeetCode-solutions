## 2926. Maximum Balanced Subsequence Sum

### Problem Statement
You are given a 0-indexed integer array `nums`. A subsequence consisting of indices `i_0 < i_1 < ... < i_{k-1}` is **balanced** if the following condition holds for every `j` in the range `[1, k - 1]`:
`nums[i_j] - nums[i_{j-1}] >= i_j - i_{j-1}`

Return the maximum possible sum of elements in a balanced subsequence of `nums`.

---

### Core Insight — Coordinate Transformation & Fenwick Tree (BIT)
The core mathematical constraint can be rewritten by rearranging the terms to group index and value together:
`nums[i_j] - nums[i_{j-1}] >= i_j - i_{j-1}`
`nums[i_j] - i_j >= nums[i_{j-1}] - i_{j-1}`

Let's define a transformed value array `A` where:
`A[i] = nums[i] - i`

The structural balancing condition simplifies to:
`A[i_j] >= A[i_{j-1}]`

This transforms our problem into finding a subsequence that maximizes the sum of the original `nums[i]` elements while ensuring that their corresponding transformed values `A[i]` are **non-decreasing**. This is a variation of the Maximum Weight Increasing Subsequence problem.

#### Dynamic Programming Definition
Let `dp[v]` be the maximum sequence sum ending at a transformed coordinate value `v`.
When processing `nums[i]`, its transformed value is `v = nums[i] - i`. We can extend any valid subsequence ending at a value less than or equal to `v`.
`dp[v] = nums[i] + max(0, max_{u <= v} dp[u])`

Because the range of `nums[i] - i` can span from `-10^9` to `10^9`, we cannot use a direct array mapping. We must **coordinate compress** all unique values of `nums[i] - i` to a compact ranking range `[1, M]` where `M <= N`. 

To maintain and query the running prefixes efficiently in `O(log N)` time, we use a **Binary Indexed Tree (Fenwick Tree)** or a Segment Tree configured to store maximum values instead of sums.

---

### Algorithm
1. Create an auxiliary array `A` where `A[i] = nums[i] - i`.
2. Extract all unique values from `A`, sort them, and map each value to its 1-indexed rank to perform coordinate compression.
3. Initialize a Fenwick Tree (Binary Indexed Tree) that supports prefix maximum queries and point updates.
4. Track a global variable `maxSum` initialized to the minimum possible integer value (or the maximum single element) to handle arrays containing only negative numbers.
5. Loop through the array from `0` to `N - 1`:
   * Compute `v = nums[i] - i`.
   * Find the compressed index/rank of `v` in our sorted map.
   * Query the Fenwick tree for the maximum prefix sum available up to that rank.
   * Calculate `current_dp = nums[i] + Math.Max(0, prefix_max)`.
   * Update the global `maxSum` with `current_dp`.
   * Update the Fenwick tree at the item's compressed rank with `current_dp`.
6. Return `maxSum`.

---

### Full Optimal Implementation (C#)
```csharp
using System;
using System.Collections.Generic;
using System.Linq;

public class Solution {
    private class FenwickTree {
        private long[] tree;
        private int size;

        public FenwickTree(int n) {
            size = n;
            tree = new long[n + 1];
            Array.Fill(tree, long.MinValue);
        }

        public void Update(int idx, long val) {
            while (idx <= size) {
                if (val > tree[idx]) {
                    tree[idx] = val;
                }
                idx += idx & -idx;
            }
        }

        public long Query(int idx) {
            long maxVal = long.MinValue;
            while (idx > 0) {
                if (tree[idx] > maxVal) {
                    maxVal = tree[idx];
                }
                idx -= idx & -idx;
            }
            return maxVal;
        }
    }

    public long MaxBalancedSubsequenceSum(int[] nums) {
        int n = nums.Length;
        long[] transformed = new long[n];
        
        for (int i = 0; i < n; i++) {
            transformed[i] = (long)nums[i] - i;
        }

        // Coordinate Compression
        long[] sortedUnique = transformed.Distinct().ToArray();
        Array.Sort(sortedUnique);

        FenwickTree bit = new FenwickTree(sortedUnique.Length);
        long globalMaxSum = long.MinValue;

        for (int i = 0; i < n; i++) {
            long tVal = transformed[i];
            // Find 1-indexed rank using binary search
            int rank = Array.BinarySearch(sortedUnique, tVal) + 1;

            long prevMax = bit.Query(rank);
            long currentDp = nums[i];
            if (prevMax > 0) {
                currentDp += prevMax;
            }

            globalMaxSum = Math.Max(globalMaxSum, currentDp);
            bit.Update(rank, currentDp);
        }

        return globalMaxSum;
    }
}
```
### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N log N) | Sorting the unique values for coordinate compression takes O(N log N) time. The main loop runs N times, with each binary search rank check and Fenwick Tree prefix query executing in O(log N) time. |
| **Space** | O(N) | Auxiliary space is required to store the transformed array, unique sorted coordinates, and the internal array representation of the Fenwick Tree. |

### Example Walkthrough
#### `nums = [3,3,5,6]`
* **Transformations (`nums[i] - i`):**
  - i = 0: 3 - 0 = 3
  - i = 1: 3 - 1 = 2
  - i = 2: 5 - 2 = 3
  - i = 3: 6 - 3 = 3
* **Sorted Unique Values:** `[2, 3]` (Compressed Ranks: `2 -> 1`, `3 -> 2`)
* **Iteration Trace:**
  - **i = 0 (`nums[0] = 3`, `t = 3`, `rank = 2`):** Query up to rank 2 returns `MinValue`. `currentDp = 3`. Update rank 2 with 3. `globalMaxSum = 3`.
  - **i = 1 (`nums[1] = 3`, `t = 2`, `rank = 1`):** Query up to rank 1 returns `MinValue`. `currentDp = 3`. Update rank 1 with 3. `globalMaxSum = 3`.
  - **i = 2 (`nums[2] = 5`, `t = 3`, `rank = 2`):** Query up to rank 2 returns 3 (from rank 1 or 2). `currentDp = 5 + 3 = 8`. Update rank 2 with 8. `globalMaxSum = 8`.
  - **i = 3 (`nums[3] = 6`, `t = 3`, `rank = 2`):** Query up to rank 2 returns 8. `currentDp = 6 + 8 = 14`. Update rank 2 with 14. `globalMaxSum = 14`.

**Total Output:** `14`. ✓

### Pitfalls & Edge Cases
* **All Negative Values:** If the array contains exclusively negative numbers (e.g., `[-2, -1]`), extending a sequence using a negative `prevMax` decreases the overall total. Our logic uses `if (prevMax > 0)` to guarantee that we isolate the maximum single element instead of compounding negative balances.
* **64-bit Accumulation Fields:** The total output sum can scale up to 10^14, which completely overflows standard 32-bit integers. Ensure the Fenwick Tree payload structures, DP calculations, and return types use 64-bit types (`long`).

### Key Takeaway
Whenever you encounter an inequality relation involving both elements and their positions across a sequence, attempt to isolate the variable terms. Rearranging indices to one side and values to the other often uncovers a hidden monotonic order requirement that can be optimized using coordinate compression and range query structures.
