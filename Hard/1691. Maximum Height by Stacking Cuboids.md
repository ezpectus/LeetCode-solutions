# 1691. Maximum Height by Stacking Cuboids
## O(n²) — Sort Dimensions + LIS-style DP

---

## Problem Statement

- `n` cuboids, each can be rotated (dimensions reordered freely).
- Cuboid `i` can be placed on cuboid `j` if all three dimensions of `i` ≤ corresponding dimensions of `j`.
- Return maximum total height of a valid stack.

---

## Core Idea — Two Sorting Steps + DP

### Step 1: Sort Each Cuboid's Dimensions

For each cuboid, sort its three dimensions so `a ≤ b ≤ c`. This normalizes the rotation — we always orient the cuboid with its **largest dimension as height**.

**Why is this optimal?** Using the largest dimension as height maximizes contribution. And placing it with the smallest face as base makes it easiest to stack on top of other cuboids. Formal proof: if a cuboid is oriented with a non-maximum height `b`, switching to height `c ≥ b` increases stack height and the smaller base only makes it easier (or equally hard) to be accepted on top of cuboids below.

### Step 2: Sort All Cuboids Lexicographically

After normalizing dimensions, sort all cuboids. This ensures that for any valid stacking (cuboid `j` below `i`), `j` appears before `i` in the sorted order — allowing the DP to work forward.

### Step 3: LIS-style DP

```
dp[i] = max height of stack with cuboid i on top
dp[i] = cuboids[i][2]   (at least its own largest dimension)
       + max dp[j]  for all j < i where cuboids[j] ≤ cuboids[i] componentwise
```

**Answer**: `max(dp)`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxHeight(int[][] cuboids) {
        // Step 1: sort each cuboid's dimensions ascending
        foreach (var c in cuboids) Array.Sort(c);

        // Step 2: sort all cuboids lexicographically
        Array.Sort(cuboids, (a, b) =>
            a[0] != b[0] ? a[0]-b[0] : a[1] != b[1] ? a[1]-b[1] : a[2]-b[2]);

        int n = cuboids.Length;
        int[] dp = new int[n];

        for (int i = 0; i < n; i++) {
            dp[i] = cuboids[i][2]; // own height = largest dimension
            for (int j = 0; j < i; j++) {
                if (cuboids[j][0] <= cuboids[i][0] &&
                    cuboids[j][1] <= cuboids[i][1] &&
                    cuboids[j][2] <= cuboids[i][2]) {
                    dp[i] = Math.Max(dp[i], dp[j] + cuboids[i][2]);
                }
            }
        }

        return dp.Max();
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n² + n log n)** | Sort O(n log n), DP O(n²) |
| **Space** | **O(n)** | dp array |

With `n ≤ 100`: at most 10,000 DP operations — instant.

---

## Why Lexicographic Sort Enables Forward DP

After normalizing each cuboid's dimensions, if cuboid `j` can be placed **below** cuboid `i`, then `cuboids[j][0] ≤ cuboids[i][0]`, `cuboids[j][1] ≤ cuboids[i][1]`, `cuboids[j][2] ≤ cuboids[i][2]`. This means `j` sorts before or equal to `i` lexicographically. So in the forward DP sweep, all valid predecessors `j` are already computed when we reach `i`.

This is the same reasoning as why sorting enables O(n²) LIS.

## Example Walkthrough

**Example 1**: `cuboids=[[50,45,20],[95,37,53],[45,23,12]]`

**After sorting each cuboid's dimensions**:
```
[50,45,20] → [20,45,50]
[95,37,53] → [37,53,95]
[45,23,12] → [12,23,45]
```

**After sorting all cuboids**:
```
[12,23,45], [20,45,50], [37,53,95]
```

**DP**:

| i | cuboid | j=0 fits? | j=1 fits? | dp[i] |
|---|--------|-----------|-----------|-------|
| 0 | [12,23,45] | — | — | 45 |
| 1 | [20,45,50] | ✓ → 45+50=95 | — | **95** |
| 2 | [37,53,95] | ✓ → 45+95=140 | ✓ → 95+95=**190** | 190 |

`max(45, 95, 190) = 190` ✓

## Pitfalls & Edge Cases

- **`dp[i]` initialized to `cuboids[i][2]`** — a single cuboid alone is a valid stack of its largest dimension
- **Duplicate cuboids** (example 3): all six variants of `[7,11,17]` normalize to `[7,11,17]` → each stacks on previous → `dp = [17, 34, 51, 68, 85, 102]` → answer 102 ✓
- **No valid stacking** (example 2): cuboids don't fit → each `dp[i]` stays at its own height → answer = max single cuboid height

---

## Key Takeaway

"Maximum height stacking 3D cuboids with rotation" → **normalize orientation (sort dims ascending), sort cuboids, O(n²) DP**:

```
sort each cuboid → largest dim becomes height
sort all cuboids → valid predecessors always come first
dp[i] = max(dp[j] + height[i]) for fitting j < i
```

Sorting dimensions is the non-obvious insight — it reduces a 3D rotation problem to a clean componentwise comparison enabling standard LIS-style DP.
