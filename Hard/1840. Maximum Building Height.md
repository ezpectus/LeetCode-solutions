# 1840. Maximum Building Height
## O(R log R) — Sort + Two-Pass Propagation + Peak Formula

---

## Problem Statement

- Build `n` buildings labeled `1` to `n` in a line.
- Rules: heights are non-negative integers, **building 1 has height 0**, adjacent buildings differ by **at most 1**.
- `restrictions[i] = [id, maxHeight]` caps the height of building `id`.
- Return the **maximum possible height** of the tallest building.

---

## Core Idea — Two-Pass Constraint Propagation + Peak Formula

**Key insight**: Treat only the restricted buildings (plus building 1 and building n as anchors). Between any two adjacent restricted buildings `(id1, h1)` and `(id2, h2)`, the maximum achievable peak height can be computed directly with a formula.

**Three steps:**

**Step 1 — Anchor + sort**:
- Add `[1, 0]` (building 1 forced to 0) and `[n, n-1]` (building n unrestricted → max height `n-1`)
- Sort by building id

**Step 2 — Forward pass** (left → right):
- Each restriction can only be as high as what the previous restriction allows, propagating rightward:
- `r[i].h = min(r[i].h, r[i-1].h + (r[i].id - r[i-1].id))`

**Step 3 — Backward pass** (right → left):
- Symmetrically propagate constraints leftward:
- `r[i].h = min(r[i].h, r[i+1].h + (r[i+1].id - r[i].id))`

After both passes, each restricted building has its **true effective max height** considering all neighbors.

**Peak formula** between adjacent pair `(id1, h1)` and `(id2, h2)`:
- We can ramp up from `h1` and ramp up from `h2` toward the middle
- Peak = `(h1 + h2 + dist) / 2` where `dist = id2 - id1`
- Intuition: total "height budget" is split between the two sides; the peak sits where they meet

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxBuilding(int n, int[][] restrictions) {
        // Anchor: building 1 (h=0) and building n (unrestricted)
        var r = new List<int[]>(restrictions) {
            new[] { 1, 0 },
            new[] { n, n - 1 }
        };
        r.Sort((a, b) => a[0] - b[0]);
        int m = r.Count;

        // Forward pass: propagate left->right
        for (int i = 1; i < m; i++)
            r[i][1] = Math.Min(r[i][1], r[i-1][1] + (r[i][0] - r[i-1][0]));

        // Backward pass: propagate right->left
        for (int i = m - 2; i >= 0; i--)
            r[i][1] = Math.Min(r[i][1], r[i+1][1] + (r[i+1][0] - r[i][0]));

        // Find max peak between each adjacent pair
        int res = 0;
        for (int i = 1; i < m; i++) {
            int h1 = r[i-1][1], h2 = r[i][1];
            int dist = r[i][0] - r[i-1][0];
            int peak = (h1 + h2 + dist) / 2;
            res = Math.Max(res, peak);
        }

        return res;
    }
}
```

**Optimal** — must process all restrictions at least once.

## Peak Formula — Math Breakdown

Between two anchors `(id1, h1)` and `(id2, h2)` with `dist = id2 - id1`:

```
Heights ramp from left:  h1, h1+1, h1+2, ...
Heights ramp from right: h2, h2+1, h2+2, ...
```

The peak is where these two ramps "meet". Total height units available: `h1 + h2 + dist`. Split evenly: **peak = `(h1 + h2 + dist) / 2`** (integer division).

**Verify**: if h1=1, h2=1, dist=3 (positions 2,3,4,5 with heights 1,?,?,1):
- Peak = `(1 + 1 + 3) / 2 = 2` → heights `[1, 2, 1]` between anchors ✓

## Example Walkthrough

**Example 3**: `n=10, restrictions = [[5,3],[2,5],[7,4],[10,3]]`

After adding anchors and sorting:
`[[1,0],[2,5],[5,3],[7,4],[10,3],[10,9]]` → wait, `[10,3]` overrides `[10,9]`

Sorted: `[[1,0],[2,5],[5,3],[7,4],[10,3]]`

**Forward pass** (cap each by left neighbor + dist):

| i | id | h before | prev h + dist | h after |
|---|-----|----------|--------------|---------|
| 0 | 1 | 0 | — | 0 |
| 1 | 2 | 5 | 0+1=1 | **1** |
| 2 | 5 | 3 | 1+3=4 | **3** |
| 3 | 7 | 4 | 3+2=5 | **4** |
| 4 | 10 | 3 | 4+3=7 | **3** |

**Backward pass** (cap each by right neighbor + dist):

| i | id | h before | next h + dist | h after |
|---|-----|----------|--------------|---------|
| 3 | 7 | 4 | 3+3=6 | **4** |
| 2 | 5 | 3 | 4+2=6 | **3** |
| 1 | 2 | 1 | 3+3=6 | **1** |
| 0 | 1 | 0 | 1+1=2 | **0** |

**Peak between each pair:**

| pair | h1 | h2 | dist | peak |
|------|----|----|------|------|
| (1,2) | 0 | 1 | 1 | 1 |
| (2,5) | 1 | 3 | 3 | **3** → wait: (1+3+3)/2=3 |
| (5,7) | 3 | 4 | 2 | (3+4+2)/2=4 |
| (7,10) | 4 | 3 | 3 | (4+3+3)/2=**5** |

`res = 5` ✓

## Pitfalls & Edge Cases

- **Add building n as anchor** with `maxHeight = n-1` — without it, the last segment (from last restriction to n) is ignored; building n can reach up to `n-1` freely
- **Forward before backward** — forward pass tightens right-side constraints; backward then tightens left-side; both are needed
- **Integer division in peak formula** — `(h1 + h2 + dist) / 2` always floors correctly; heights are integers so the peak is always achievable
- **Building n already in restrictions** — adding `[n, n-1]` and sorting still works; `min(restricted, n-1)` is taken naturally in the forward pass
- **No restrictions** — only anchors `[1,0]` and `[n,n-1]`; peak = `(0 + (n-1) + (n-1)) / 2 = n-1` ✓
- **Large n (up to 10⁹)** — dist values can be huge; use `int` carefully; in C# `int` holds up to ~2.1×10⁹ so dist fits, but products might not — use `long` if needed

---

## Key Takeaway

Sparse restrictions on a range → **two-pass propagation to find effective bounds, then closed-form between pairs**:

1. **Sort** restricted points
2. **Forward pass**: cap each point by what the previous allows (left can't ramp faster than +1/step)
3. **Backward pass**: same from right
4. **Peak formula**: between adjacent anchors `(h1, h2, dist)`, max peak = `(h1 + h2 + dist) / 2`

Pattern: whenever constraints propagate in both directions along a line, sort + two-pass propagation resolves all interactions in O(R).
