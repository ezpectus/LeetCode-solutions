# 2681. Power of Heroes
## O(n log n) — Sorting + Prefix Contribution Tracking (Math DP)

### Problem Statement
You are given an array `nums`. The "power" of a group of heroes is defined as:
$$\text{Power} = (\max(\text{group}))^2 \times \min(\text{group})$$
Return the sum of the power of all possible non-empty groups. Since the answer can be massive, return it modulo $10^9 + 7$.

---

### Core Idea — Contribution to Sum (Order Independence)
Generating all subsets directly results in an unmanageable $O(2^n)$ complexity. However, notice that the internal order of elements within a group does not affect its minimum or maximum boundaries. This allows us to **sort the array** first.

Let's evaluate a sorted array $a_0, a_1, a_2, \dots, a_{n-1}$. 
If we fix an element $a_j$ to act as the absolute **maximum** element of a group, then all other chosen members must be picked from indices $i < j$.

For a fixed maximum element $a_j$:
1. **The single-element group $[a_j]$:** Max is $a_j$, Min is $a_j$. $\text{Power} = a_j^2 \times a_j = a_j^3$.
2. **Multi-element groups where $a_i$ ($i < j$) is the minimum boundary:** * The elements strictly between $a_i$ and $a_j$ can either be included or excluded. There are $j - 1 - i$ such elements, yielding $2^{j - 1 - i}$ ways to form these exact boundaries.
   * $\text{Power} = a_j^2 \times (a_i \times 2^{j - 1 - i})$.

Factoring out $a_j^2$ for our fixed maximum, the total sum of minimums for all valid subsets ending at $a_j$ expands to:
$$\text{Total}(j) = a_j^2 \times \left( a_j + \sum_{i=0}^{j-1} a_i \times 2^{j - 1 - i} \right)$$

---

### The State Tracking Optimization
Let $S_j = \sum_{i=0}^{j-1} a_i \times 2^{j - 1 - i}$. Let's expand this progression step-by-step:
* $S_0 = 0$
* $S_1 = a_0$
* $S_2 = 2a_0 + a_1$
* $S_3 = 4a_0 + 2a_1 + a_2$

This reveals a clean mathematical recurrence relation: **$S_j = 2 \times S_{j-1} + a_{j-1}$**
Instead of computing this inner summation with a nested loop (which triggers an $O(n^2)$ Time Limit Exceeded), we can maintain this prefix contribution value dynamically using a single variable during a single linear scan.

---

### Full Optimal Implementation (C#)

```csharp
using System;

public class Solution {
    public int SumOfPower(int[] nums) {
        long mod = 1000000007;
        
        // Step 1: Sort the array to process elements in predictable min/max order
        Array.Sort(nums);
        
        long totalPowerSum = 0;
        long minContributionSum = 0; // Tracks the running prefix state variable (S)
        
        foreach (long x in nums) {
            // Step 2: Calculate max^2 % mod
            long maxSquared = (x * x) % mod;
            
            // Step 3: Combine the current max acting as its own min + previous trailing min combinations
            long combinedMins = (x + minContributionSum) % mod;
            
            // Step 4: Compute power for groups where x is the maximum, then add to global sum
            long currentGroupPower = (maxSquared * combinedMins) % mod;
            totalPowerSum = (totalPowerSum + currentGroupPower) % mod;
            
            // Step 5: Update the rolling contribution window for the next state iteration
            // Recurrence: S_next = (S_curr * 2) + current_element
            minContributionSum = (minContributionSum * 2 + x) % mod;
        }
        
        return (int)totalPowerSum;
    }
}
```

---

### Complexity

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | O(n log n) | Dominated completely by `Array.Sort(nums)`. The subsequent single-pass loop runs in linear $O(n)$ time since all state transitions complete in $O(1)$ constant time operations. Efficiently processes maximum constraints up to $n = 10^5$. |
| **Space Complexity** | O(1) | In-place array modification ensures $O(1)$ auxiliary memory space footprint. Primitive `long` scaling factors remain static regardless of input scale. |

---

### Pitfalls & Edge Cases
* **Preventing Integer Overflow:** Operations like `minContributionSum * 2 + x` can temporarily produce scalar values up to $\approx 3 \times 10^9$. If left inside a standard 32-bit signed `int` context, this will trigger an arithmetic wrap-around into negative numbers. Processing with `long` constraints provides type headroom up to $9 \times 10^{18}$ before resolving with the `% mod` operator.
* **Single Element Subsets:** Single element groups have identical minimum and maximum values. This requirement is gracefully covered by injecting the current element `x` directly into `combinedMins` before multiplying by `maxSquared`.

---

### Key Takeaway
> When required to evaluate aggregate math properties across combinatorial spaces containing asynchronous boundary constraints (like independent tracking of min/max pairs), sort the array to lock down one of the boundaries. Then, determine if the trailing values can be collapsed into a running prefix multiplier state machine.

* **Pattern recognition:** "Sum of values across all subsets dependent on min/max bounds" -> Sort + Rolling Value Prefix State Machine.
