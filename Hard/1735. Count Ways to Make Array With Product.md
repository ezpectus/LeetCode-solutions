# 1735. Count Ways to Make Array With Product
## O(Q · √K + K · log K) — Stars and Bars + Precomputed Binomial Coefficients

---

## Problem Statement

- For each query `[n, k]`: count the number of ways to fill an array of size `n` with **positive integers** whose product equals `k`.
- Return answers modulo 10⁹ + 7.
- Constraints: 1 ≤ n, k ≤ 10⁴, up to 10⁴ queries.

---

## Core Idea — Prime Factorization + Stars and Bars

**Key insight**: Factorize `k = p1^e1 × p2^e2 × ... × pm^em`. The problem of distributing factor `k` among `n` slots **decomposes independently per prime**:

- For prime `p` with exponent `e`: distribute `e` indistinguishable copies of `p` into `n` distinguishable slots (each slot gets ≥ 0 copies)
- This is a classic **stars and bars** problem: `C(n + e - 1, e)` ways
- Total ways = **product** of `C(n + e_i - 1, e_i)` over all prime factors

**Why independent?** Each prime contributes independently to the product. Choices for prime `p` don't affect choices for prime `q`.

**Precompute binomial coefficients** with Pascal's triangle up to row `MAXN = 10^4 + 14`:
- Max `e` for any prime: since `k ≤ 10^4` and `2^13 = 8192 < 10^4 < 2^14`, max exponent is **13**
- We need `C(n + e - 1, e)` with `n ≤ 10^4` and `e ≤ 13` → max row = `10^4 + 13`
- Store only `C[row][0..13]` → memory is `O(MAXN × 14)`

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    const int MOD = 1_000_000_007;
    const int MAXN = 10_014; // 10^4 + 14
    const int MAXE = 14;     // max prime exponent + 1

    // Precompute Pascal's triangle: C[i][j] = C(i, j)
    static long[,] C = Precompute();
    static long[,] Precompute() {
        var c = new long[MAXN, MAXE];
        c[0, 0] = 1;
        for (int i = 1; i < MAXN; i++) {
            c[i, 0] = 1;
            for (int j = 1; j < MAXE; j++)
                c[i, j] = (c[i-1, j-1] + c[i-1, j]) % MOD;
        }
        return c;
    }

    public int[] WaysToFillArray(int[][] queries) {
        int[] res = new int[queries.Length];

        for (int q = 0; q < queries.Length; q++) {
            int n = queries[q][0], k = queries[q][1];
            long ways = 1;

            // Factorize k and apply stars & bars per prime
            for (int p = 2; (long)p * p <= k; p++) {
                if (k % p == 0) {
                    int e = 0;
                    while (k % p == 0) { e++; k /= p; }
                    // C(n + e - 1, e) ways to distribute e copies into n slots
                    ways = ways * C[n + e - 1, e] % MOD;
                }
            }
            if (k > 1) ways = ways * C[n, 1] % MOD; // remaining prime with e=1 → C(n,1) = n

            res[q] = (int)ways;
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(MAXN × MAXE + Q × √K)** | Pascal precompute once; Q queries × O(√K) factorization |
| **Space Complexity** | **O(MAXN × MAXE)** | ~140K entries of `long` ≈ 1.1MB |

**Practical**: MAXN×MAXE = 10014 × 14 ≈ 140K ops to precompute; each query ≈ 100 trial divisions.

## Stars and Bars — Math Breakdown

Distributing `e` identical items into `n` distinct bins (each bin ≥ 0):

```
C(n + e - 1, e)  =  C(n + e - 1, n - 1)
```

**Why?** Imagine `e` stars and `n-1` dividers in a row — total `n + e - 1` positions, choose `e` for stars.

**Examples with n=2:**
- `e=1` (prime appears once): `C(2, 1) = 2` → goes to slot 1 or slot 2
- `e=2`: `C(3, 2) = 3` → (2,0), (1,1), (0,2)

**For query [2,6] = [2, 2¹×3¹]:**
- Prime 2, e=1: `C(2,1) = 2`
- Prime 3, e=1: `C(2,1) = 2`
- Total: `2 × 2 = 4` ✓ → `[1,6],[6,1],[2,3],[3,2]`

## Example Walkthrough

**Query [5, 1]**: k=1 has no prime factors → `ways = 1` (only `[1,1,1,1,1]`) ✓

**Query [73, 660]**: `660 = 2² × 3 × 5 × 11`

| prime | e | C(73+e-1, e) |
|-------|---|--------------|
| 2 | 2 | C(74, 2) = 2701 |
| 3 | 1 | C(73, 1) = 73 |
| 5 | 1 | C(73, 1) = 73 |
| 11 | 1 | C(73, 1) = 73 |

`2701 × 73 × 73 × 73 mod 10⁹+7 = 50734910` ✓

**Query [4, 4]**: `4 = 2²`
- `C(4+2-1, 2) = C(5, 2) = 10` ✓

## Pitfalls & Edge Cases

- **`k = 1`** — no prime factors → `ways = 1` (the loop body never executes, `ways` stays 1) ✓
- **Remaining prime after loop** — when `k > 1` after trial division, `k` is a prime with `e = 1` → `C(n + 0, 1) = n` → handled with `C[n, 1]`
- **`static` precomputation** — Pascal's triangle built once across all test cases via `static` field; avoids re-building per call
- **`e ≤ 13` guaranteed** — `2^14 = 16384 > 10^4`, so no prime can appear more than 13 times; `MAXE = 14` is safe
- **Modify `k` in-place** — the factorization loop divides out primes directly from `k`; use the original `n`, not `k`, for `C[n + e - 1, e]`
- **Long overflow** — intermediate `ways * C[...]` can exceed `int`; use `long` and reduce mod at every step

---

## Key Takeaway

"Distribute a product across `n` slots" = **factorize + stars and bars per prime**:

- `k = p1^e1 × ... × pm^em` → each prime is independent
- Per prime: `C(n + e - 1, e)` ways (stars and bars)
- Total = product of all per-prime binomials
- Precompute Pascal's triangle up to row `n + maxExp` for O(1) per coefficient

Pattern: combinatorics on multiplicative structure → factorize first, then apply counting formula independently per prime.
