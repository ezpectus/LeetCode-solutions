# 3276. Select Cells in Grid With Maximum Score
## O(V · 2^m · m) — Bitmask DP on "Rows Used" with Per-Value Passes

---

## Problem Statement

Select cells from a grid such that:
1. No two selected cells share a row.
2. All selected cell values are distinct.
3. Maximize the sum of selected values.

Grid is at most 10×10, values ∈ [1, 100].

---

## Core Insight #1 — Reduce to "Which Rows Are Used"

Since we pick at most one cell per row, the set of selected rows forms a bitmask over m rows (m ≤ 10 → at most 2^10 = 1024 masks). The distinct-value constraint means we treat each unique value independently: include it from at most one row.

**State:**
```
dp[mask] = maximum sum of selected cells using exactly the rows in mask,
           with all selected values distinct
```

---

## Core Insight #2 — Process Each Value Independently

For each unique value `v` ∈ [1..100]:
- Let `rowsWithValue[v]` = bitmask of rows that contain value `v`.
- For each existing DP state `(mask, dp[mask])`, we may optionally add `v` from one row in `rowsWithValue[v]` that isn't already in `mask`.
- This extends state `mask` to `mask | (1 << r)` with cost `dp[mask] + v`.

Since `v` is processed in one dedicated pass, it's used **at most once** across the entire selection.

---

## Core Insight #3 — Decreasing Mask Order Prevents Double-Use Without Copying

Within the pass for value `v`, we iterate masks from `2^m - 1` down to `0`. When we update `dp[mask | (1<<r)]` from `dp[mask]`:

- `mask | (1<<r) > mask` — the target mask is **larger** than the source.
- In decreasing order, we reach the **larger** mask first — it was already processed.
- Therefore, the update is **deferred**: the target mask won't be visited again in this pass.

This guarantees value `v` contributes **at most once** per DP path — without making any array copies.

**Why this is safe:** `dp[mask]` can only be updated FROM smaller masks (which come later in the decreasing pass). So when we reach `mask`, `dp[mask]` is still its "pre-v" value. ✓

---

## Precomputation: `rowsWithValue[v]`

```
for each row r, for each cell v in grid[r]:
    rowsWithValue[v] |= (1 << r)
```

Then, for each mask, the rows available for value `v` that aren't used:
```
available = rowsWithValue[v] & ~mask
```

Iterate through set bits of `available` using `bit = available & (-available)`.

---

## Full Implementation (C#)

```csharp
public class Solution {
    public int MaxScore(IList<IList<int>> grid) {
        int m = grid.Count;

        // rowsWithValue[v] = bitmask of rows containing value v
        int[] rowsWithValue = new int[101];
        for (int r = 0; r < m; r++)
            foreach (int v in grid[r])
                rowsWithValue[v] |= (1 << r);

        // dp[mask] = max sum using rows in mask, all values distinct
        int[] dp = new int[1 << m]; // initialized to 0 (valid: all values ≥ 1)

        // Process each value from high to low (order doesn't affect correctness)
        for (int v = 100; v >= 1; v--) {
            if (rowsWithValue[v] == 0) continue;

            // Decreasing mask order: prevents using v more than once per path
            for (int mask = (1 << m) - 1; mask >= 0; mask--) {
                int available = rowsWithValue[v] & ~mask; // rows with v, not yet used
                while (available != 0) {
                    int bit = available & (-available); // lowest set bit
                    available &= available - 1;         // remove lowest set bit
                    int newMask = mask | bit;
                    if (dp[mask] + v > dp[newMask])
                        dp[newMask] = dp[mask] + v;
                }
            }
        }

        return dp.Max();
    }
}
```

## Example Walkthrough

### Example 1 — `grid = [[1,2,3],[4,3,2],[1,1,1]]`, m = 3

**`rowsWithValue[]`:**
| Value | Rows | Bitmask |
|---|---|---|
| 1 | 0, 2 | 101 = 5 |
| 2 | 0, 1 | 011 = 3 |
| 3 | 0, 1 | 011 = 3 |
| 4 | 1 | 010 = 2 |

**After v=4** (`rowsWithValue=010`):
- mask=000: add row 1 → dp[010] = 4.

**After v=3** (`rowsWithValue=011`):
- mask=010: available=001 (row 0). dp[011] = dp[010]+3 = **7**.
- mask=000: rows 0,1 available. dp[001]=3, dp[010]=max(4,3)=4.

**After v=2** (`rowsWithValue=011`):
- mask=011: available=0. Skip.
- mask=010: row 0 available. dp[011]=max(7,4+2)=7.
- mask=000: dp[001]=max(3,2)=3.

**After v=1** (`rowsWithValue=101`):
- mask=011: row 2 available (bit=100). dp[111]=dp[011]+1=**8**. ✓
- mask=010: rows 0,2 available. dp[011]=max(7,4+1)=7. dp[110]=4+1=5.

Max dp = **8** ✓ (rows 0,1,2 selected with values 3,4,1)

---

### Example 2 — `grid = [[8,7,6],[8,3,2]]`, m = 2

**`rowsWithValue[]`:**
| Value | Rows | Bitmask |
|---|---|---|
| 8 | 0, 1 | 11 = 3 |
| 7 | 0 | 01 = 1 |
| 6 | 0 | 01 = 1 |
| 3 | 1 | 10 = 2 |
| 2 | 1 | 10 = 2 |

After v=8: dp[01]=8, dp[10]=8.
After v=7: dp[11] = dp[10]+7 = **15** (row 1 used for 8, row 0 for 7).
After v=6: dp[11] = max(15, dp[10]+6) = max(15,14) = 15.
Others: no improvement.

Max dp = **15** ✓ (values 8 and 7 from rows 1 and 0)

## Decreasing-Mask Guarantee — Illustrated

**Processing v=3, mask=010 → updates mask=011:**

```
Iteration order (m=3): 111, 110, 101, 100, 011, 010, 001, 000

Step: mask=011 processed → dp[011] = 4 (initialized from v=4, row 1)
       ↑ already done

Step: mask=010 processed → dp[011] = max(4, dp[010]+3) = max(4,7) = 7
       dp[011] updated AFTER its own processing slot → safe ✓

Step: mask=001 → tries row 1 (in rowsWithValue=011) → dp[011] = max(7, dp[001]+3)
Step: mask=000 → updates dp[001], dp[010]
```

No matter how many times mask=011 is updated in this pass, it was processed ONCE using its pre-v value. ✓

## Pitfalls & Edge Cases

- **Same value in multiple rows:** `rowsWithValue[v]` captures this as a multi-bit mask. In a single pass for `v`, we try each such row independently. Only one is chosen per DP path (guaranteed by decreasing-mask order).

- **Same value in the same row multiple times:** `rowsWithValue[v] |= (1 << r)` (OR, not ADD) deduplicates automatically. A row with two `3`s is treated the same as a row with one `3`.

- **`dp[mask] = 0` for unreachable mask=0:** Since all values ≥ 1, any achievable mask with sum > 0 means at least one cell was selected. `dp[0] = 0` is the correct base case (no cells selected, sum = 0). Unreachable masks with mask > 0 stay 0, which is ≤ any valid sum, so they don't corrupt the answer.

- **`m ≤ 10` means at most 10 rows = 2^10 = 1024 masks.** With V = 100 values, the total ops are ~1M — trivially fast.

- **Values `> 100`:** Impossible per constraints (`grid[i][j] ≤ 100`). The `rowsWithValue` array of size 101 is exactly sufficient.

---

## Key Takeaway

This is a **bitmask DP on "rows used"** problem. The two constraints (distinct rows, distinct values) are handled separately:
- Distinct rows: encoded in the bitmask.
- Distinct values: enforced by processing each value in its own pass with the decreasing-mask trick.

**Pattern recognition:** "Select items from groups with at-most-one-per-group and global-distinctness constraints, maximize sum" → bitmask DP where groups map to bitmask bits, and each item type gets one dedicated pass with decreasing-order iteration to prevent double-use.
