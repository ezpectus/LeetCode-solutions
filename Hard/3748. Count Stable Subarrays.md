# 3748. Count Stable Subarrays
## O(n + q) — Maximal Non-Decreasing Segments + Prefix Sum + Query Split at Boundary

---

## Problem Statement

Array `nums`. A subarray is **stable** if it is non-decreasing (no inversions). For each query `[l, r]`, count all stable subarrays of `nums[l..r]`. Return answers as `long long`.

Constraints: n, q ≤ 10⁵, nums[i] ≤ 10⁵.

---

## Core Insight 1 — Stable Subarrays Are Exactly the Non-Decreasing Contiguous Ranges

A stable subarray `nums[i..j]` (i ≤ j) exists iff `nums[k] ≤ nums[k+1]` for all k ∈ [i, j−1]. This is equivalent to `i ≥ S[j]`, where `S[j]` = start of the maximal non-decreasing segment containing j.

**Definition:** `S[j]` = leftmost index of the contiguous non-decreasing run ending at j.
- `S[0] = 0`
- `S[j] = j` if `nums[j−1] > nums[j]` (a "break point"), else `S[j] = S[j−1]`.

A subarray `[i, j]` is stable ↔ i ≥ S[j]. ✓

---

## Core Insight 2 — Per-Position Contribution and Prefix Sum

For a fixed j (right endpoint) in the full array, the number of stable subarrays ending at j is:
```
cross[j] = j − S[j] + 1
```
(right endpoints j, left endpoints ranging from S[j] to j.)

Define prefix sum: `P[j+1] = P[j] + cross[j]` = total stable subarrays with right endpoint ≤ j.

For a range of right endpoints [a, b]: total stable subarrays (ignoring query left bound) = P[b+1] − P[a].

---

## Core Insight 3 — Query Decomposition at the Boundary

For query [l, r], a stable subarray `[i, j]` must satisfy:
- l ≤ i ≤ j ≤ r
- i ≥ S[j]

For fixed j, count of valid i = j − max(l, S[j]) + 1.

**Two zones**, split at B = min(r, segEnd[l]) where `segEnd[l]` = last index of the maximal non-decreasing segment containing l:

**Zone 1 — Crossing segment** (j ∈ [l, B]): These positions all belong to the same non-decreasing segment as l. Since S[j] ≤ l for all j in this zone (their segment started at or before l), `max(l, S[j]) = l`. Count for each j = j − l + 1.

```
Zone 1 total = Σ_{j=l}^{B} (j−l+1) = len*(len+1)/2   where len = B−l+1
```

**Zone 2 — Full segments after l** (j ∈ [B+1, r], only if B < r): For these positions, S[j] > l (they're in new segments starting strictly after l). `max(l, S[j]) = S[j]`. Count for each j = cross[j].

```
Zone 2 total = Σ_{j=B+1}^{r} cross[j] = P[r+1] − P[B+1]
```

**Total = len*(len+1)/2 + (B < r ? P[r+1] − P[B+1] : 0)**

---

## Algorithm

```
1. Compute S[j] (start of non-decreasing segment at j): left scan, O(n).
2. Compute segEnd[j] (end of non-decreasing segment at j): right scan, O(n).
3. Compute P[j+1] = P[j] + (j − S[j] + 1): prefix sums, O(n).
4. For each query [l, r]:
     B = min(r, segEnd[l])
     len = B − l + 1
     ans = len*(len+1)/2 + (B < r ? P[r+1] − P[B+1] : 0)
```

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    vector<long long> countStableSubarrays(vector<int>& nums, vector<vector<int>>& queries) {
        int n = nums.size();

        // S[j] = start of the maximal non-decreasing segment containing j
        vector<int> S(n);
        S[0] = 0;
        for (int j = 1; j < n; j++)
            S[j] = (nums[j-1] > nums[j]) ? j : S[j-1];

        // segEnd[j] = end of the maximal non-decreasing segment containing j
        vector<int> segEnd(n);
        segEnd[n-1] = n-1;
        for (int j = n-2; j >= 0; j--)
            segEnd[j] = (nums[j] > nums[j+1]) ? j : segEnd[j+1];

        // P[j+1] = Σ cross[i] for i = 0..j, where cross[i] = i − S[i] + 1
        vector<long long> P(n+1, 0);
        for (int j = 0; j < n; j++)
            P[j+1] = P[j] + (j - S[j] + 1);

        int q = queries.size();
        vector<long long> ans(q);
        for (int i = 0; i < q; i++) {
            int l = queries[i][0], r = queries[i][1];
            int B = min(r, segEnd[l]);      // boundary of crossing segment within [l, r]
            long long len = B - l + 1;
            ans[i] = len * (len + 1) / 2;  // Zone 1: counting subarrays [l, B]
            if (B < r)
                ans[i] += P[r+1] - P[B+1]; // Zone 2: counting subarrays [B+1, r]
        }

        return ans;
    }
};
```

## Example Walkthroughs

### Example 1 — `nums = [3, 1, 2]`, Expected: [2, 3, 4]

**Break points:** nums[0]=3 > nums[1]=1 → break at 0. No break at 1.
**Segments:** [0,0] = {3}, [1,2] = {1,2}.

**S:** S[0]=0, S[1]=1, S[2]=1.
**segEnd:** segEnd[0]=0, segEnd[1]=2, segEnd[2]=2.
**cross:** cross[0]=1, cross[1]=1, cross[2]=2.
**P:** P[0]=0, P[1]=1, P[2]=2, P[3]=4.

**Query [0,1]:**
- B = min(1, segEnd[0]) = min(1, 0) = **0**.
- len = 0−0+1 = 1. Zone 1 = 1×2/2 = **1**.
- B=0 < r=1: Zone 2 = P[2] − P[1] = 2−1 = **1**.
- ans = **2** ✓ (stable: [3], [1])

**Query [1,2]:**
- B = min(2, segEnd[1]) = min(2, 2) = **2**.
- len = 2−1+1 = 2. Zone 1 = 2×3/2 = **3**.
- B=2 = r=2: Zone 2 = 0.
- ans = **3** ✓ (stable: [1], [2], [1,2])

**Query [0,2]:**
- B = min(2, segEnd[0]) = min(2, 0) = **0**.
- len = 0−0+1 = 1. Zone 1 = 1×2/2 = **1**.
- B=0 < r=2: Zone 2 = P[3] − P[1] = 4−1 = **3**.
- ans = **4** ✓ (stable: [3], [1], [2], [1,2])

---

### Example 2 — `nums = [2, 2]`, Expected: [3, 1]

**No break points.** One segment [0,1].
**S:** S[0]=0, S[1]=0. **segEnd:** all 1.
**cross:** cross[0]=1, cross[1]=2. **P:** P[0]=0, P[1]=1, P[2]=3.

**Query [0,1]:**
- B = min(1, 1) = 1. len=2. Zone 1 = 2×3/2 = **3**. B=r: Zone 2=0. ans=**3** ✓

**Query [0,0]:**
- B = min(0, 1) = 0. len=1. Zone 1 = **1**. B=r: Zone 2=0. ans=**1** ✓

---

### Why Zone 1 Uses (j−l+1) Not cross[j]

For j in [l, segEnd[l]], position j is in the SAME non-decreasing segment as l. The segment might extend left of l (if l is in the middle of a segment). We can't use left endpoints before l (they're outside the query range), so the effective left boundary is l, not S[j].

Example: `nums = [1, 2, 3, 4, 5]`, one big segment, query `[2, 4]`.
- For j=4: cross[j] = 4−0+1 = 5 (counts starting positions 0..4). But we only allow starting positions 2..4.
- Zone 1 counts j=2,3,4 with (j−2+1) = 1, 2, 3 → total = 6. That's C(4,2) = 6 = 3×4/2. ✓

If we mistakenly used cross[j] for these positions: 3+4+5=12. Wrong.

---

### Derivation of segEnd

```cpp
segEnd[n-1] = n-1;
for (int j = n-2; j >= 0; j--)
    segEnd[j] = (nums[j] > nums[j+1]) ? j : segEnd[j+1];
```

If nums[j] > nums[j+1]: j is the last element of its segment. segEnd[j] = j.
Else: j and j+1 are in the same segment. segEnd[j] = segEnd[j+1].

For `nums = [3, 1, 2]`: segEnd[2]=2. j=1: 1≤2 → segEnd[1]=segEnd[2]=2. j=0: 3>1 → segEnd[0]=0. ✓

## Pitfalls & Edge Cases

- **l = r (single element query):** B = min(r, segEnd[l]) ≥ l = r → B = r. len = 1. Zone 1 = 1. Zone 2 = 0. ans = 1. ✓

- **Entire query within one segment:** B = segEnd[l] = r (segEnd[l] ≥ r). Zone 1 = (r−l+1)×(r−l+2)/2. Zone 2 = 0. This is the classic triangle number for a fully non-decreasing segment. ✓

- **l at a break point (start of new segment):** S[l] = l. For j in [l, segEnd[l]], S[j] = l. max(l, S[j]) = l. Zone 1 formula remains j−l+1. ✓ (Formula is correct whether l is mid-segment or segment-start.)

- **`long long` overflow:** len ≤ n = 10⁵. len×(len+1)/2 ≤ 5×10⁹. P[n] ≤ Σ n ≤ n²/2 ≤ 5×10⁹. Total per query ≤ 10¹⁰. Fits in `long long` (max ≈ 9.2×10¹⁸). ✓

- **B+1 = n (when segEnd[l] = n−1 and B = r < n−1 is false):** If B < r, we access P[B+1] which is valid for B+1 ≤ r ≤ n−1 → B+1 ≤ n. P has size n+1, indices 0..n. ✓

---

## Key Takeaway

**"Count non-decreasing subarrays within query ranges"** decomposes into:
1. Precompute `cross[j]` = stable subarrays ending at j (= j − S[j] + 1, the segment size ending at j).
2. Build prefix sum P over cross[].
3. For each query [l, r]: split at B = end of segment containing l within [l, r]. Zone 1 (crossing l's segment) contributes a triangle number; Zone 2 (full subsequent segments) contributes P[r+1] − P[B+1].

**Pattern recognition:** Any "count subarrays satisfying a local monotonicity condition within range queries" follows this split-at-boundary structure — a boundary in the query range separates positions needing special "adjustment from query's left bound" (Zone 1, triangle formula) from positions using the precomputed full-segment counts (Zone 2, prefix sum lookup). O(1) per query after O(n) preprocessing.
