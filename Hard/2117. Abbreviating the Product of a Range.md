# 2117. Abbreviating the Product of a Range
## O(n log n) — Separate Suffix, Prefix, and Trailing Zero Computation

---

## Problem Statement

- Compute product of `[left, right]`. Remove trailing zeros (count = C). If remaining digits > 10, keep first 5 ("pre") and last 5 ("suf"). Return `"pre...sufeC"` or `"producteC"`.

---

## Core Idea — Three Independent Computations

The product can be astronomically large, so we can't compute it directly. Split into:

### 1. Trailing Zeros (C)
Trailing zeros = `min(count_of_2s, count_of_5s)` in the product. Count prime factors of 2 and 5 across all numbers in `[left, right]`.

### 2. Last 5 Digits (Suffix)
To find the last 5 digits of `product / 10^C`:

Strip all factors of 2 and 5 from each number as we multiply, keeping the product mod `10^5`. Count removed 2s (`rem2 = c2`) and 5s (`rem5 = c5`). After stripping trailing zeros (`C = min(c2,c5)`), re-apply the excess factors: `extra2 = c2 - C` twos and `extra5 = c5 - C` fives, both mod `10^5`.

### 3. First 5 Digits (Prefix) and Total Digit Count
Use **logarithms**: `log10(product/10^C) = Σ log10(i) - C`. The integer part gives digit count, the fractional part gives the leading digits:
```
d = floor(log_sum) + 1
pre = floor(10^(frac + 4))   // 5 leading digits
```

### 4. Short Products (d ≤ 10)
Compute exactly using big integers.

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    string abbreviateProduct(int left, int right) {
        // Step 1: count trailing zeros
        long long c2 = 0, c5 = 0;
        for (int i = left; i <= right; i++) {
            int n = i;
            while (n % 2 == 0) { c2++; n /= 2; }
            n = i;
            while (n % 5 == 0) { c5++; n /= 5; }
        }
        long long C = min(c2, c5);

        // Step 2: last 5 digits of product / 10^C
        long long suf = 1, rem2 = 0, rem5 = 0;
        const long long MOD = 100000;
        for (int i = left; i <= right; i++) {
            suf *= i;
            while (suf % 2 == 0) { suf /= 2; rem2++; }
            while (suf % 5 == 0) { suf /= 5; rem5++; }
            suf %= MOD;
        }
        long long extra2 = rem2 - C, extra5 = rem5 - C;
        for (int i = 0; i < extra2; i++) suf = suf * 2 % MOD;
        for (int i = 0; i < extra5; i++) suf = suf * 5 % MOD;

        // Step 3: prefix — multiply directly in double, normalize to avoid overflow
        double pre = 1.0;
        for (int i = left; i <= right; i++) {
            pre *= i;
            while (pre >= 1e10) pre /= 10;
        }

        double logSum = 0;
        for (int i = left; i <= right; i++) logSum += log10(i);
        logSum -= C;
        long long d = (long long)logSum + 1;

        if (d <= 10) {
            // Compute exactly
            long long prod = 1;
            for (int i = left; i <= right; i++) prod *= i;
            // Remove trailing zeros
            long long p10 = 1;
            for (int i = 0; i < C; i++) p10 *= 10;
            prod /= p10;
            return to_string(prod) + "e" + to_string(C);
        }

        double frac = logSum - floor(logSum);
        // Extract 5 leading digits from pre (already normalized)
        while (pre >= 1e5) pre /= 10;
        while (pre < 1e4) pre *= 10;
        long long preDigits = (long long)pre;
        string sufStr = to_string(suf);
        while (sufStr.size() < 5) sufStr = "0" + sufStr;
        sufStr = sufStr.substr(sufStr.size() - 5);
        return to_string(preDigits) + "..." + sufStr + "e" + to_string(C);
    }
};
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n log n)** | n = right - left + 1; factor counting is O(log i) per i |
| **Space** | **O(1)** | Only running totals |

With `n ≤ 10^4`: fast.

---

## Why Separate Suffix Tracking Works

During multiplication, we strip all 2s and 5s immediately to prevent accumulation. This keeps `suf` small (< `10^5`). After processing all numbers:
- `rem2 = c2` total 2s stripped, `rem5 = c5` total 5s stripped.
- `C` pairs of (2,5) form the trailing zeros of the original product.
- Remaining: `c2 - C` extra 2s and `c5 - C` extra 5s belong in the suffix.
- Re-multiply these extras into `suf` mod `10^5`.

---

## Example Walkthrough

**`left=371, right=375`**

Product = 371×372×373×374×375 = 7219856259000.

Count 2s: 372=4×93→2²×93 (+2), 374=2×187 (+1), 375=3×125 (+0) → c2=3.
Count 5s: 375=3×5³ (+3) → c5=3. C=min(3,3)=3.

log_sum = log10(7219856259000) - 3 = log10(7219856259) = 9.858... → d=10.

d=10 ≤ 10 → compute exactly: 7219856259000 / 10³ = 7219856259. → **"7219856259e3"** ✓

---

## Pitfalls & Edge Cases

- **`suf` might lose precision**: keep `suf %= MOD` after each multiplication to stay bounded.
- **Floating point error in prefix**: `log10` accumulates floating-point error for large ranges. May need to handle edge case where `d` is off by 1.
- **`sufStr` zero-padded**: suffix must be exactly 5 digits; use `zfill(5)` or manual padding.
- **Short product `d ≤ 10`**: avoid any approximation; compute big integer exactly (safe since result fits in `long long` after dividing by `10^C`).

---

## Key Takeaway

"Abbreviate huge range product" → **three parallel computations**:

```
C = min(count_2, count_5)           // trailing zeros
suf = product / 10^C mod 10^5       // last 5 digits (strip 2s/5s during multiply)
pre = leading 5 digits via log10    // first 5 digits
d = total digit count via log10
```

Each computation is independent and avoids ever storing the full product.
