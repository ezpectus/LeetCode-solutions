# 3444. Minimum Increments for Target Multiples in an Array
## O(n · 4^m) — Bitmask DP: Assign Each nums Element to Cover a Subset of Targets

---

## Problem Statement

Array `nums` (length up to 5×10⁴), array `target` (length ≤ 4). Each operation increments one element of `nums` by 1. Goal: every target[i] must divide at least one element of `nums`. Minimize total operations.

Constraints: m = target.length ≤ 4 (key!), nums[i], target[i] ≤ 10⁴.

---

## Core Insight — One nums Element Can Cover Multiple Targets Via LCM

A single nums element v, when incremented to the next multiple of `LCM(S)` (where S is a subset of target indices), simultaneously covers every target in S. The minimum increment for v to cover subset S is:

```
cost(v, S) = (LCM(S) - v % LCM(S)) % LCM(S)
```

The problem is then: assign each element of nums to cover some subset S (possibly empty), paying cost(v, S) for the assignment. Find a minimum-cost assignment where the union of all covered subsets = full target set.

This is a minimum-cost set cover over at most 2^m = 16 subsets — solvable via bitmask DP.

---

## DP Formulation

**State:** `dp[mask]` = minimum total operations to ensure every target in `mask` is covered by at least one nums element.

**Initialization:** `dp[0] = 0`, `dp[mask > 0] = ∞`.

**Transition (for each nums element v):**
- Create `ndp = copy(dp)` (option: don't modify v, contributes nothing).
- For every current state `mask` and every non-empty subset `S`:
  ```
  ndp[mask | S] = min(ndp[mask | S], dp[mask] + cost(v, S))
  ```
- Set `dp = ndp`.

**Answer:** `dp[(1 << m) − 1]` (all targets covered).

**Why copy (not in-place update):** Each element of nums can cover at most one subset S. Updating `dp` in-place would allow re-using the same element multiple times. Using `dp` on the right-hand side and writing to `ndp` ensures each v is assigned once. ✓

**Why enumerate all (mask, S) pairs, not just complementary subsets:** A later element might cover the same targets as an earlier one (redundant but valid). More importantly, covering S when some targets in S are already covered is allowed — the `|` in `mask | S` naturally handles this. The DP finds the globally optimal assignment.

---

## LCM Precomputation for All Subsets

With m ≤ 4, there are at most 15 non-empty subsets. Build LCMs using the recurrence:

```
LCM(S) = LCM(LCM(S \ {j}), target[j])    for any j ∈ S
```

Process subsets in order from 1 to 2^m − 1; extract the lowest set bit j of S to split off.

**Overflow guard:** LCM of four 10⁴-valued elements can be astronomically large (up to ~10¹⁶). Cap at 2×10¹³ — any cost using such a large LCM would be absurdly high and never chosen by the DP (the true optimal cost is ≤ m × max(target) ≤ 4×10⁴).

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int minimumIncrements(vector<int>& nums, vector<int>& target) {
        int m = target.size();
        int full = (1 << m) - 1;
        const long long INF = 1e18;
        const long long CAP = 2e13;  // LCM cap (any cost > this is unreachable in optimum)

        // Precompute LCM for each non-empty subset of target indices
        vector<long long> L(1 << m, 1LL);
        for (int S = 1; S <= full; S++) {
            int j = __builtin_ctz(S);           // index of lowest set bit
            long long base = L[S ^ (1 << j)];  // LCM of S without j
            long long g = __gcd(base, (long long)target[j]);
            long long l = base / g * target[j];
            L[S] = (l > CAP) ? CAP : l;
        }

        // dp[mask] = min operations to cover all targets in mask
        vector<long long> dp(1 << m, INF);
        dp[0] = 0;

        for (int v : nums) {
            vector<long long> ndp = dp;  // not modifying v = cost 0
            for (int mask = 0; mask <= full; mask++) {
                if (dp[mask] == INF) continue;
                for (int S = 1; S <= full; S++) {
                    long long cost = (L[S] - (long long)v % L[S]) % L[S];
                    long long val = dp[mask] + cost;
                    int nMask = mask | S;
                    if (val < ndp[nMask]) ndp[nMask] = val;
                }
            }
            dp = ndp;
        }

        return (int)dp[full];
    }
};
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| LCM precomputation | O(2^m) | 15 subsets, O(1) each |
| Inner DP loops | O(2^m × 2^m) = O(4^m) | 16 × 16 = 256 per element |
| Total | **O(n · 4^m)** | 5×10⁴ × 256 ≈ 1.3×10⁷ ops |
| Space | O(2^m) | dp and ndp arrays: 16 entries each |

Maximum answer: m × (max_target − 1) ≤ 4 × 9999 ≈ 40,000. Fits in `int`. ✓

## Example Walkthroughs

### Example 1 — `nums = [1,2,3]`, `target = [4]`, Expected: 1

m=1, full=1. L[1] = 4.

| v | ndp[1] update | dp after |
|---|---|---|
| 1 | dp[0]+cost(1,{0})=(4-1%4)%4=3 → ndp[1]=3 | [0, 3] |
| 2 | dp[0]+cost=(4-2)%4=2 → ndp[1]=2 | [0, 2] |
| 3 | dp[0]+cost=(4-3)%4=1 → ndp[1]=**1** | [0, 1] |

**Return 1** ✓ — increment nums[2]=3 to 4.

---

### Example 2 — `nums = [8,4]`, `target = [10,5]`, Expected: 2

m=2, full=3. L[01]=10, L[10]=5, L[11]=LCM(10,5)=10.

**Process v=8:**

| S | L[S] | cost(8,S) | ndp[0 | S] |
|---|---|---|---|
| 01 | 10 | (10-8)%10=2 | ndp[1]=2 |
| 10 | 5 | (5-3)%5=2 | ndp[2]=2 |
| 11 | 10 | (10-8)%10=2 | ndp[3]=**2** |

dp=[0,2,2,2].

**Process v=4:**

| mask | S | cost(4,S) | ndp[mask\|S] update |
|---|---|---|---|
| 0 | 01 | (10-4)=6 | ndp[1]=min(2,6)=2 |
| 0 | 10 | (5-4)=1 | ndp[2]=min(2,1)=**1** |
| 0 | 11 | (10-4)=6 | ndp[3]=min(2,6)=2 |
| 1 | 10 | 1 | ndp[3]=min(2,2+1)=2 |
| 2 | 01 | 6 | ndp[3]=min(2,2+6)=2 |
| ... | ... | ... | no improvements |

dp=[0,2,1,2]. **Return 2** ✓

*Optimal: increment v=8 by 2 → 10 (covers both target[0]=10 and target[1]=5). dp[3]=2 was set from processing v=8 with S={0,1} = LCM=10, cost=2. ✓*

---

### Example 3 — `nums = [7,9,10]`, `target = [7]`, Expected: 0

L[1]=7. Processing v=7: cost=(7-7%7)%7=(7-0)%7=0. dp[1]=0. **Return 0** ✓

---

### Why Redundant Coverage Is Handled Correctly

Consider the update: when v=8 covers S={0,1} (LCM=10) with cost 2, dp[3]=2. Later, v=4 tries to improve from mask=1 (dp[1]=2) by covering S={1} (cost 1): dp[1|10=3]=min(2, 2+1)=2. No improvement — correctly, because the single-element coverage by v=8 was already optimal.

## Pitfalls & Edge Cases

- **Multiple targets sharing a divisor:** e.g., target=[10,5]. LCM({10,5})=10, same as LCM({10}). So cost(v,{0,1}) = cost(v,{0}). The DP correctly identifies that covering {10,5} together costs no more than covering {10} alone when 5|10. ✓

- **LCM overflow:** For target=[9999, 9998, 9997, 9996], LCM ≈ 10¹⁵. Cap at 2×10¹³ — the corresponding cost would be ≈10¹³ per element, never chosen when optimal cost ≤ 4×10⁴. ✓

- **Same element covering multiple independent target elements:** e.g., target=[3,7]. If v=21: v%LCM(3,7)=21%21=0, cost=0. The DP with S={0,1} captures this free coverage. ✓

- **copy dp to ndp before inner loop:** Critical. If we updated dp in-place, we could use the same nums[i] element to cover multiple subsets across iterations of the mask loop, leading to wrong answers. ✓

- **Iterating S from 1 to full (not 0):** S=0 (empty subset) means "don't use v", which is already captured by ndp = copy(dp). Including S=0 would add cost=0 for no benefit but is harmless. Excluding it is clean. ✓

- **Skipping dp[mask] == INF:** Early exit avoids adding ∞ + cost, which could overflow. ✓

## Key Takeaway

**"Minimum cost to cover all elements of a small set"** with n independent items each able to cover subsets at variable cost → bitmask DP over target subsets. Each item is processed in O(4^m) → total O(n · 4^m).

The critical decision: one item can cover MULTIPLE targets simultaneously (at the LCM cost) rather than separately. This joint-coverage benefit is exactly what the bitmask DP captures: trying all 2^m possible subsets S for each item, and transitioning `mask → mask | S`.

**Pattern recognition:** Whenever we have ≤ ~20 "requirements" to satisfy (here m ≤ 4 targets), n independent "items" each of which can satisfy subsets of requirements at different costs, and we want minimum total cost — bitmask DP is the canonical approach. The LCM structure here makes cost(v, S) well-defined and efficiently computable.
