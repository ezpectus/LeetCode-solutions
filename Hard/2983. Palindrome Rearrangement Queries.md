## 2983. Palindrome Rearrangement Queries

### Problem Statement
You are given a 0-indexed string `s` of even length `n`. You are also given a 2D integer array `queries`, where `queries[i] = [ai, bi, ci, di]`. For each query, you can independently:
* Rearrange characters within the substring `s[ai...bi]` (where `0 <= ai <= bi < n/2`).
* Rearrange characters within the substring `s[ci...di]` (where `n/2 <= ci <= di < n`).

Return a boolean array `answer` where `answer[i]` is `true` if it is possible to make `s` a palindrome via these rearrangements, and `false` otherwise.

---

### Core Insight — Symmetrization via Prefix Sums
A string is a palindrome if for every index `i`, `s[i] == s[n - 1 - i]`. Instead of dealing with the right half of the string directly with indices `[n/2...n-1]`, we can reflect the right half onto the left half. 

Let's match each index `i` in the second half (`ci` and `di`) to its mirrored counterpart in the first half:
* `reflected_c = n - 1 - di`
* `reflected_d = n - 1 - ci`

Now, instead of two different intervals on opposite sides, we have two overlapping or separate intervals both operating conceptually on the **left half** of the string: `[ai, bi]` and `[reflected_c, reflected_d]`.

For a query to be valid, any index `i` outside both of these rearrangeable intervals **must already match its mirror counterpart** (`s[i] == s[n - 1 - i]`). Within the intervals, we don't need an exact match initially; we just need to ensure that the total character counts available in the rearranged zones can satisfy the required frequencies of the opposite side.

To answer each query in `O(1)` time, we precompute:
1. A prefix sum array tracking the count of mismatched mirrored pairs (`s[i] != s[n - 1 - i]`).
2. A 2D prefix sum array tracking character frequencies for both the left half and the mirrored right half.

---

### Structural Case Checks
For each query, let the two intervals on the left half be `[a, b]` and `[c, d]` (where `c = n - 1 - di` and `d = n - 1 - ci`). 

1. **Total Outside Unchanged Check:** Any index strictly outside both `[a, b]` and `[c, d]` must have no original mismatches.
2. **Interval Frequency Coverage Check:**
   - For parts of `[a, b]` that do not overlap with `[c, d]`, the characters in `s[a...b]` must completely match the needed mirror characters from the right side.
   - For parts of `[c, d]` that do not overlap with `[a, b]`, the mirrored characters in the right half must cover the characters needed by the left side.
   - For the union of both intervals, the total frequency of each character on the left side must exactly equal the total frequency of that character on the mirrored right side.

---

### Full Optimal Implementation (C#)
```csharp
using System;

public class Solution {
    public bool[] CanMakePalindromeQueries(string s, int[][] queries) {
        int n = s.Length;
        int m = n / 2;
        int numQueries = queries.Length;
        bool[] result = new bool[numQueries];

        // diffPrefix[i] stores the number of mismatches between s[0...i-1] and its mirror
        int[] diffPrefix = new int[m + 1];
        // leftFreq[i, ch] stores frequency of character ch in s[0...i-1]
        int[,] leftFreq = new int[m + 1, 26];
        // rightFreq[i, ch] stores frequency of character ch in the mirrored right half s[n-1...n-i]
        int[,] rightFreq = new int[m + 1, 26];

        for (int i = 0; i < m; i++) {
            diffPrefix[i + 1] = diffPrefix[i] + (s[i] != s[n - 1 - i] ? 1 : 0);
            
            for (int c = 0; c < 26; c++) {
                leftFreq[i + 1, c] = leftFreq[i, c];
                rightFreq[i + 1, c] = rightFreq[i, c];
            }
            leftFreq[i + 1, s[i] - 'a']++;
            rightFreq[i + 1, s[n - 1 - i] - 'a']++;
        }

        // Helper function to check if there are any mismatches in a range [l, r]
        Func<int, int, bool> HasNoMismatches = (l, r) => {
            if (l > r) return true;
            return (diffPrefix[r + 1] - diffPrefix[l]) == 0;
        };

        // Helper function to get character count in an interval [l, r]
        Func<int, int, int, int[,], int> GetFreq = (l, r, ch, freqArr) => {
            if (l > r) return 0;
            return freqArr[r + 1, ch] - freqArr[l, ch];
        };

        for (int q = 0; q < numQueries; q++) {
            int a = queries[q][0];
            int b = queries[q][1];
            // Transform right interval [queries[q][2], queries[q][3]] to mirrored left indices
            int c = n - 1 - queries[q][3];
            int d = n - 1 - queries[q][2];

            // 1. Check all regions outside both intervals [a, b] and [c, d]
            int minL = Math.Min(a, c);
            int maxL = Math.Max(a, c);
            int minR = Math.Min(b, d);
            int maxR = Math.Max(b, d);

            if (!HasNoMismatches(0, minL - 1) || 
                (!HasNoMismatches(minR + 1, maxL - 1) && minR < maxL) || 
                !HasNoMismatches(maxR + 1, m - 1)) {
                result[q] = false;
                continue;
            }

            // 2. Check frequency balances
            bool queryPossible = true;
            for (int ch = 0; ch < 26; ch++) {
                int totalLeft = GetFreq(a, b, ch, leftFreq) + GetFreq(c, d, ch, leftFreq);
                int totalRight = GetFreq(a, b, ch, rightFreq) + GetFreq(c, d, ch, rightFreq);
                
                // If intervals overlap, subtract the duplicated intersection area
                int overlapL = Math.Max(a, c);
                int overlapR = Math.Min(b, d);
                if (overlapL <= overlapR) {
                    totalLeft -= GetFreq(overlapL, overlapR, ch, leftFreq);
                    totalRight -= GetFreq(overlapL, overlapR, ch, rightFreq);
                }

                if (totalLeft != totalRight) {
                    queryPossible = false;
                    break;
                }

                // Check characters in [a, b] not covered by [c, d]
                if (a < c) {
                    int neededFromRight = GetFreq(a, Math.Min(b, c - 1), ch, rightFreq);
                    if (GetFreq(a, b, ch, leftFreq) < neededFromRight) { queryPossible = false; break; }
                }
                if (b > d) {
                    int neededFromRight = GetFreq(Math.Max(a, d + 1), b, ch, rightFreq);
                    if (GetFreq(a, b, ch, leftFreq) < neededFromRight) { queryPossible = false; break; }
                }

                // Check characters in [c, d] not covered by [a, b]
                if (c < a) {
                    int neededFromLeft = GetFreq(c, Math.Min(d, a - 1), ch, leftFreq);
                    if (GetFreq(c, d, ch, rightFreq) < neededFromLeft) { queryPossible = false; break; }
                }
                if (d > b) {
                    int neededFromLeft = GetFreq(Math.Max(c, b + 1), d, ch, leftFreq);
                    if (GetFreq(c, d, ch, rightFreq) < neededFromLeft) { queryPossible = false; break; }
                }
            }

            result[q] = queryPossible;
        }

        return result;
    }
}
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N + Q) | Building prefix configurations takes O(N) time. Each of the Q queries checks a constant number of regions and iterates through 26 alphabet characters, yielding O(1) per query. |
| **Space** | O(N) | The tracking matrices require `O(26 * N / 2)` memory allocation to maintain character prefix arrays across the split domains. |

### Pitfalls & Edge Cases
* **Interval Overlap Arithmetic:** When `[a, b]` and `[c, d]` intersect, computing the union frequency by simply adding counts duplicates the intersection window. Ensure you subtract `overlap` values cleanly.
* **Reflected Suffix Bounds:** Remember that mirroring flips indices. Suffix index `ci` transforms into the reflected right boundary `d`, and suffix index `di` transforms into the reflected left boundary `c`.

### Key Takeaway
When working with palindrome matching problems that isolate actions to left and right halves, your first step should be mirroring all operations onto a single shared half. This structural grouping unifies disparate ranges into standard interval overlap checking problems that can be handled using prefix sum structures.
