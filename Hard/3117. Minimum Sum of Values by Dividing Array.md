# 3117. Minimum Sum of Values by Dividing Array
## O(m·n log n) — Segment-Level DP + Compressed AND Groups (O(log V) per position) + Sparse Table RMQ

---

## Problem Statement

Divide `nums` (length n) into exactly `m` contiguous subarrays such that AND of the i-th subarray = `andValues[i]`. The "value" of each subarray is its last element. Minimize the total value sum. Return −1 if impossible.

Constraints: n ≤ 10⁴, **m ≤ min(n, 10)**, nums[i] < 10⁵ (≤ 17 bits).

---

## Core Insight 1 — DP Level by Level Over m Segments

Define `dp[j][r]` = minimum total value sum for the first `j+1` subarrays where the (j+1)-th subarray ends at index `r`.

**Base (j=0):** `dp[0][r] = nums[r]` iff `AND(nums[0..r]) = andValues[0]`, else ∞.
*(The first subarray must start at index 0, so its AND is simply the running prefix AND.)*

**Transition (j ≥ 1):**
```
dp[j][r] = nums[r] + min{ dp[j-1][l-1] : l ∈ [lo, hi], AND(nums[l..r]) = andValues[j] }
```
where `[lo, hi]` is the range of valid left endpoints at which the j-th subarray can start (all giving AND exactly = andValues[j]).

**Answer:** `dp[m-1][n-1]` (all n elements covered, last subarray ends at n−1).

---

## Core Insight 2 — AND Groups: O(log V) Distinct Values Per Right Endpoint

For a fixed right endpoint r, as the left endpoint l decreases from r to 0:
- `AND(nums[l..r])` can only **decrease or stay equal** (adding elements never sets bits).
- Each decrease clears ≥ 1 bit, so at most 17 = log₂(10⁵) changes occur.
- Therefore, there are at most O(log V) distinct AND values, forming a **staircase** of contiguous l-ranges.

Maintain a list `segs = [(andVal, lo), ...]` (sorted by **increasing** lo) such that for l ∈ [segs[k].lo, segs[k+1].lo − 1], `AND(nums[l..r]) = segs[k].andVal`. The last group covers [segs.last.lo, r].

**Update from r to r+1:**
1. AND every `segs[k].andVal` with `nums[r]` (extend all subarrays one step right).
2. Append `(nums[r], r)` (new single-element subarray [r, r]).
3. Merge adjacent groups with identical andVal (keep leftmost lo).

---

## Core Insight 3 — Sparse Table for O(1) Range Minimum on Previous DP Level

For the transition at each (j, r), once we identify the valid l-range `[lo, hi]`, we need:
```
min{ dp[j-1][l-1] : l ∈ [max(lo, 1), hi] }  =  RMQ(dp[j-1], max(lo,1)−1, hi−1)
```

A **sparse table** built on `dp[j-1]` answers range minimum in O(1) per query after O(n log n) build time. With m=10 levels, building 10 sparse tables is fast.

---

## Algorithm

```
1. dp[] = INF everywhere.
2. j=0: compute prefix AND left-to-right; dp[r] = nums[r] if AND(nums[0..r]) = andValues[0].
         Early break when prefix AND loses a required bit.
3. For j = 1 to m-1:
     prev[] = dp[];  dp[] = INF.
     Build sparse table on prev[].
     segs = [] (AND groups, reset per level).
     For r = 0 to n-1:
       a. Update segs: AND all groups with nums[r], append (nums[r], r), merge adjacent.
       b. For each matching group (andVal = andValues[j]):
            lo, hi = that group's l-range.
            qlo = max(lo, 1) - 1;  qhi = hi - 1.
            if qlo <= qhi:  dp[r] = min(dp[r], RMQ(prev, qlo, qhi) + nums[r]).
4. Return dp[n-1] if < INF, else -1.
```

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    const long INF = long.MaxValue / 2;

    public int MinimumValueSum(int[] nums, int[] andValues) {
        int n = nums.Length, m = andValues.Length;

        long[] dp = new long[n];
        Array.Fill(dp, INF);

        // j = 0: first subarray always starts at 0
        int prefAnd = ~0;
        for (int r = 0; r < n; r++) {
            prefAnd &= nums[r];
            if (prefAnd == andValues[0]) dp[r] = nums[r];
            if ((prefAnd & andValues[0]) != andValues[0]) break; // lost a needed bit: impossible hereafter
        }

        // j = 1..m-1
        for (int j = 1; j < m; j++) {
            long[] prev = dp;
            dp = new long[n];
            Array.Fill(dp, INF);

            long[][] sparse = BuildSparse(prev, n);
            var segs = new List<(int a, int lo)>(); // sorted by increasing lo

            for (int r = 0; r < n; r++) {
                // Step 1: extend all subarrays one step right
                for (int k = 0; k < segs.Count; k++)
                    segs[k] = (segs[k].a & nums[r], segs[k].lo);
                // Step 2: new single-element group at r
                segs.Add((nums[r], r));
                // Step 3: merge adjacent groups with same andVal (keep leftmost lo)
                int w = 0;
                for (int k = 0; k < segs.Count; k++) {
                    if (w > 0 && segs[w - 1].a == segs[k].a) continue;
                    segs[w++] = segs[k];
                }
                segs.RemoveRange(w, segs.Count - w);

                // Step 4: find matching group and query RMQ
                for (int k = 0; k < segs.Count; k++) {
                    if (segs[k].a != andValues[j]) continue;
                    int lo = segs[k].lo;
                    int hi = (k + 1 < segs.Count) ? segs[k + 1].lo - 1 : r;
                    int qlo = Math.Max(lo, 1) - 1;
                    int qhi = hi - 1;
                    if (qlo > qhi) continue;
                    long best = QuerySparse(sparse, qlo, qhi);
                    if (best < INF)
                        dp[r] = Math.Min(dp[r], best + nums[r]);
                }
            }
        }

        return dp[n - 1] >= INF ? -1 : (int)dp[n - 1];
    }

    static long[][] BuildSparse(long[] arr, int n) {
        int log = 1; while ((1 << log) <= n) log++;
        var t = new long[log][];
        t[0] = (long[])arr.Clone();
        for (int k = 1; k < log; k++) {
            t[k] = new long[n];
            for (int i = 0; i + (1 << k) <= n; i++)
                t[k][i] = Math.Min(t[k - 1][i], t[k - 1][i + (1 << (k - 1))]);
        }
        return t;
    }

    static long QuerySparse(long[][] t, int l, int r) {
        int k = (int)Math.Log2(r - l + 1);
        return Math.Min(t[k][l], t[k][r - (1 << k) + 1]);
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| j=0 base | O(n) | Simple prefix AND scan |
| BuildSparse per level | O(n log n) | n = 10⁴, log n ≈ 14 |
| segs update per position | O(log V) | V = 10⁵, log V ≈ 17 |
| RMQ per match | O(1) | Sparse table |
| Total | **O(m · n log n)** | m ≤ 10, ≈ 1.4M ops |
| Space | O(n log n + n log V) | Sparse table + segs per level |

### Why the segs [lo, hi] Range Is Contiguous

Since AND(nums[l..r]) is non-increasing as l decreases:
- For a target `T = andValues[j]`, either AND never reaches T (no valid l), or it equals T across a contiguous interval before dropping below.
- This contiguity is why we can represent the valid range as a single [lo, hi] interval per group.

If AND jumps directly from above T to below T (skipping T), then T is simply not in any group and no valid l exists for this (j, r). ✓

## Pitfalls & Edge Cases

- **`max(lo, 1)` guard for j ≥ 1:** The j-th subarray can start at l=0 only if j=0. For j≥1, l must be ≥1 (the previous subarray occupies at least index 0). Clamping lo to max(lo, 1) ensures we don't index into prev[-1]. ✓

- **Early break in j=0:** Once `(prefAnd & andValues[0]) != andValues[0]` (prefix AND has cleared a bit that andValues[0] requires), no future r can make AND(nums[0..r]) = andValues[0]. Breaking saves O(n) work. ✓

- **segs are in INCREASING lo order:** The new single-element (nums[r], r) is appended LAST (largest lo), not prepended. AND values in the list are non-decreasing. Merging left-to-right keeps the leftmost lo for each duplicate group. ✓

- **hi for the last group = r, not the next group's lo−1:** The loop `int hi = (k+1 < segs.Count) ? segs[k+1].lo - 1 : r` correctly sets hi=r for the last group. Forgetting this would miss valid starting positions in the rightmost range. ✓

- **RMQ query on prev[l-1] not prev[l]:** The j-th subarray starts at l; the (j-1)-th subarray ends at l-1. So the cost to look up is `prev[l-1]` for l ∈ [lo, hi], translating to index range `[lo-1, hi-1]`. Using the wrong index (off-by-one) silently produces wrong answers. ✓

- **Answer is only at dp[n-1]:** The subarrays must cover ALL of nums. So the last subarray MUST end at n-1. Return dp[n-1], not the minimum over all indices. ✓

- **`long` for dp values:** max dp = sum of up to m=10 values each up to 10⁴ (max nums[i] ≈ 10⁵, sum over m ≤ 10 subarrays ≤ 10⁶). Fits in `int`, but use `long` for INF sentinel safety. ✓

---

## Key Takeaway

**"Partition array into m segments satisfying per-segment AND constraints"** → DP level by level over m (m ≤ 10), maintaining:
1. **Compressed AND groups** (O(log V) per right endpoint, updated in O(log V) per step) for O(1) lookup of valid l-ranges.
2. **Sparse table RMQ** on the previous DP level for O(1) range minimum query over valid predecessors.

**Pattern recognition:** "Subarray AND = target, minimize cost" always exploits the O(log V) distinct AND values per right endpoint. The sparse table converts the "best predecessor in a range" problem from O(range) to O(1). Combined, the DP transition drops from naive O(n) per (j, r) to O(log V) per step, giving O(m × n log n) total.
