#  LeetCode 3414: Maximum Score of Non-overlapping Intervals

### Mathematical Formulation & DP State
The problem requires selecting a subset of at most 4 non-overlapping intervals to maximize the total weight. If multiple subsets yield the same maximum weight, we must return the lexicographically smallest sequence of original 0-based indices.

Let each interval be defined as a tuple:
$$\text{Interval}_i = (l_i, r_i, w_i, \text{id}_i)$$

Where $l_i$ and $r_i$ are the strict boundaries, $w_i$ is the weight, and $\text{id}_i$ is the original index. 

We sort all intervals primarily by $l_i$ ascending. If $l_i$ values are equal, we sort them by $\text{id}_i$ ascending. This sorting ensures that a linear sweep processing intervals from right to left can naturally evaluate non-overlapping transitions via binary search.

We define the Dynamic Programming state as:
*   $\text{DP}[i][j]$: The optimal state structure containing the maximum score and the sorted index array achieved by selecting at most $j$ intervals from the suffix range $[i, N-1]$.

---

### The State Transition Equation
For any interval $i$ and capacity $j$, we evaluate two discrete choices:
1.  **Skip:** Do not include interval $i$. The optimal value transitions directly from the subsequent suffix: $\text{DP}[i+1][j]$.
2.  **Take:** Include interval $i$. We collect $w_i$, decrement the capacity $j$, and jump to the first valid interval $\text{next}$ that starts strictly after the current interval ends ($l_{\text{next}} > r_i$). We locate this index in $\mathcal{O}(\log N)$ time using `std::upper_bound`.

$$\text{DP}[i][j] = \max \begin{cases} \text{DP}[i+1][j] & \text{(Skip)} \\ w_i + \text{DP}[\text{next}][j-1] & \text{(Take, if } j > 0) \end{cases}$$

To handle the lexicographical tie-breaking rule without sacrificing performance, the comparison operator between the **Skip** and **Take** states follows this strict precedence layout:
1.  **Maximize Score:** Higher total weight sum always takes priority.
2.  **Minimize Indices Lexicographically:** If the scores are identical, the path producing the lexicographically smaller sorted index configuration is chosen.

---

### C++ Optimized Implementation

To guarantee execution within execution time limits, index vectors are capped at a maximum size of 4, avoiding dynamic heap reallocations during state comparisons.

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
private:
    struct Interval {
        int l, r, w, id;
    };

    struct State {
        long long score = 0;
        vector<int> indices;

        // Strict comparison operator for maximum score and lexicographical tie-breaking
        bool isBetterThan(const State& other) const {
            if (this->score != other.score) {
                return this->score > other.score;
            }
            // If scores are equal, the lexicographically smaller index array wins
            if (this->indices.empty()) return false;
            if (other.indices.empty()) return true;
            return this->indices < other.indices;
        }
    };

public:
    vector<int> maximumWeight(vector<vector<int>>& intervals) {
        int n = intervals.size();
        vector<Interval> arr(n);
        for (int i = 0; i < n; ++i) {
            arr[i] = {intervals[i][0], intervals[i][1], intervals[i][2], i};
        }

        // Primary sort by left endpoint, secondary sort by original index
        sort(arr.begin(), arr.end(), [](const Interval& a, const Interval& b) {
            if (a.l != b.l) return a.l < b.l;
            return a.id < b.id;
        });

        // Extract left boundaries into a dense vector for fast binary search execution
        vector<int> left_bounds(n);
        for (int i = 0; i < n; ++i) {
            left_bounds[i] = arr[i].l;
        }

        // dp[i][j] stores the best State from suffix i..n-1 using at most j intervals
        vector<vector<State>> dp(n + 1, vector<State>(5));

        // Bottom-up evaluation from right to left
        for (int i = n - 1; i >= 0; --i) {
            // Find the first interval that doesn't overlap (target_l > current_r)
            int next_idx = upper_bound(left_bounds.begin() + i, left_bounds.end(), arr[i].r) - left_bounds.begin();

            for (int j = 1; j <= 4; ++j) {
                // Choice A: Skip the current interval
                State best_state = dp[i + 1][j];

                // Choice B: Take the current interval
                State take_state;
                take_state.score = arr[i].w + dp[next_idx][j - 1].score;
                take_state.indices = dp[next_idx][j - 1].indices;
                take_state.indices.push_back(arr[i].id);
                sort(take_state.indices.begin(), take_state.indices.end());

                // Select the optimal transition strategy
                if (take_state.isBetterThan(best_state)) {
                    best_state = take_state;
                }

                dp[i][j] = best_state;
            }
        }

        return dp[0][4].indices;
    }
};
```
## Complexity Verification

### 1. Time Complexity
*   **Sorting Constraints:** Preparing the structural array maps to an Introsort bounding phase running in $\mathcal{O}(N \log N)$ operations.
*   **DP Grid Execution:** The subproblem matrix contains exactly $N \times 4$ coordinates. For each cell, the non-overlapping leap boundary is fetched using binary search (`std::upper_bound`), which takes $\mathcal{O}(\log N)$. Merging and sorting the index tracking vector takes $\mathcal{O}(4 \log 4) \equiv \mathcal{O}(1)$ computational steps.
*   **Total Asymptotic Runtime:** $\mathcal{O}(N \log N)$. Given $N = 5 \times 10^4$, total operations cap out well beneath the execution threshold, eliminating TLE risk.

### 2. Space Complexity
*   **Matrix Allocations:** The auxiliary DP lookup tables require $(N + 1) \times 5$ memory entries. Each tracking block scales inside strict constraints by storing vectors restricted to 4 integers.
*   **Total Asymptotic Space:** $\mathcal{O}(N)$ memory overhead, ensuring complete safety against stack overflow bounds.


##  Architectural Implementation & Core Algorithm Breakdown

### High-Level Algorithmic Strategy
The solution uses **Dynamic Programming (DP) combined with Binary Search (Successor Engine)**. 

Since we can pick at most 4 intervals, a standard interval scheduling approach is expanded into a 2D state system where we keep track of how many intervals we are still allowed to pick. 

To process intervals cleanly without worrying about looking back at what we already picked, we sort the intervals by their left boundary ($l_i$). This allows us to process the subproblems from right to left (bottom-up suffix DP). When we are at interval $i$, any valid next interval must start strictly after the right boundary of $i$ ($r_i$). Because the array is sorted by left boundaries, we can find this next valid interval instantly using binary search.

---

### Step-by-Step Logic Breakdown

#### 1. Sorting and Preserving Lexicographical Integrity
The intervals are sorted using a custom rule:
*   Primary key: Start time ($l_i$) ascending.
*   Secondary key: Original 0-based index ($\text{id}_i$) ascending.

Sorting by original index as a tie-breaker means that when our DP looks at intervals from left to right, it naturally encounters smaller indices first. This handles the condition where the problem demands the lexicographically smallest set of indices if scores are equal.

#### 2. The Successor Engine (`std::upper_bound`)
When we decide to include (take) interval $i$, we cannot pick any other interval that overlaps with it. The interval $i$ ends at $r_i$. The next interval we pick must start at a coordinate strictly greater than $r_i$. 

Instead of scanning linearly, we maintain an isolated flat vector `left_bounds` containing only the sorted start times of all intervals. We run:
```cpp
int next_idx = upper_bound(left_bounds.begin() + i, left_bounds.end(), arr[i].r) - left_bounds.begin();
```

`std::upper_bound` efficiently finds the first index where the start time is strictly greater than $r_i$. This leap executes in $\mathcal{O}(\log N)$ time, completely preventing TLE.

### 3. State Management and Safe Copying
Every cell in our `dp[i][j]` table represents an optimal subset state containing:
*   **score:** The maximum accumulated weight sum (`long long` to prevent integer overflow since weights can be up to $10^9$).
*   **indices:** A vector tracking which original indices were picked (capped at a size of 4).

When evaluating the "Take" option, we pull the index vector from `dp[next_idx][j - 1]`, append the current interval's original ID (`arr[i].id`), and sort it. Sorting a vector of maximum size 4 takes a constant number of operations ($\approx 4 \log 4$), keeping the transition at $\mathcal{O}(1)$.

### 4. Strict Lexicographical Tie-Breaking
The custom comparator function `isBetterThan` handles the core logic of the choice selection:

```cpp
bool isBetterThan(const State& other) const {
    if (this->score != other.score) {
        return this->score > other.score;
    }
    if (this->indices.empty()) return false;
    if (other.indices.empty()) return true;
    return this->indices < other.indices;
}
```

If the total score computed by taking the current interval matches the score achieved by skipping it, the standard `vector::operator<` natively compares the elements element-by-element from left to right. The state with the smaller initial indices wins the branch and updates the grid.
