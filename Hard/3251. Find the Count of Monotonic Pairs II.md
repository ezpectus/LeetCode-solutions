# 3251. Find the Count of Monotonic Pairs II
## O(n · maxVal) — Reduce to Non-Decreasing arr1 DP + Sliding Prefix Sum Transition

---

## Problem Statement

Array `nums` of length n. Count pairs (arr1, arr2) where arr1 is non-decreasing, arr2 is non-increasing, and `arr1[i] + arr2[i] = nums[i]` for all i. Both arrays have non-negative integer entries. Return count mod 10⁹+7.

Constraints: n ≤ 2000, nums[i] ≤ 1000.

---

## Core Insight 1 — Eliminate arr2: Only arr1 Matters

Given `arr2[i] = nums[i] − arr1[i]`, the constraints on arr2 translate to constraints on arr1:

**arr2 non-increasing:** `arr2[i] ≥ arr2[i+1]`
```
nums[i] − arr1[i] ≥ nums[i+1] − arr1[i+1]
⟺ arr1[i+1] − arr1[i] ≥ nums[i+1] − nums[i]
```

**Combined constraints on arr1 only:**
```
0 ≤ arr1[i] ≤ nums[i]                              (non-negativity of both)
arr1[i+1] ≥ arr1[i] + max(0, nums[i+1] − nums[i])  (both monotonicity conditions)
```

Let `δᵢ = max(0, nums[i+1] − nums[i])`. The transition requires: `arr1[i+1] − arr1[i] ≥ δᵢ`.

**Counting (arr1, arr2) pairs = counting valid non-decreasing sequences arr1.** ✓

---

## Core Insight 2 — DP with Prefix Sum Transition in O(maxVal) per Step

**State:** `dp[i][v]` = number of valid arr1[0..i] with arr1[i] = v.

**Base case:** `dp[0][v] = 1` for v ∈ [0, nums[0]], else 0. (arr1[0] is any value 0..nums[0].)

**Transition (i → i+1):**
```
dp[i+1][v'] = Σ dp[i][v]   for all v ∈ [0, v' − δᵢ]
            = prefix_sum(dp[i], 0, v' − δᵢ)
```

where `δᵢ = max(0, nums[i+1] − nums[i])`. This is a **shifted prefix sum**: `dp[i+1][v']` = sum of `dp[i]` up through index `v' − δᵢ`.

**Computing all transitions:** Build prefix sums of `dp[i]` once per step (O(maxVal)), then evaluate each `dp[i+1][v']` in O(1). Total per step: O(maxVal). Total: O(n × maxVal).

**Answer:** `Σ dp[n−1][v]` for v ∈ [0, nums[n-1]].

---

## Why the Transition Captures Both Constraints

`dp[i+1][v'] = Σ dp[i][v] for v ≤ v' − δᵢ` captures simultaneously:
- **v ≤ v' − δᵢ ⟺ v' − v ≥ δᵢ:** ensures `arr1[i+1] − arr1[i] ≥ δᵢ` (arr2 non-increasing and arr1 non-decreasing).
- **v ≥ 0 and v' ≤ nums[i+1]:** the bounds of the summation and the outer loop enforce non-negativity.

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int countOfPairs(vector<int>& nums) {
        int n = nums.size();
        const int MOD = 1e9 + 7;
        int maxVal = *max_element(nums.begin(), nums.end());

        vector<long long> dp(maxVal + 1, 0);
        // Base case: arr1[0] can be any value in [0, nums[0]]
        for (int v = 0; v <= nums[0]; v++) dp[v] = 1;

        for (int i = 1; i < n; i++) {
            int delta = max(0, nums[i] - nums[i - 1]);

            // Build prefix sums of current dp
            vector<long long> prefix(maxVal + 2, 0);
            for (int v = 0; v <= maxVal; v++)
                prefix[v + 1] = (prefix[v] + dp[v]) % MOD;

            // Transition: dp[v'] = sum of dp[v] for v in [0, v' - delta]
            fill(dp.begin(), dp.end(), 0);
            for (int v = delta; v <= nums[i]; v++)  // v' - delta >= 0 iff v' >= delta
                dp[v] = prefix[v - delta + 1];      // prefix[v-delta+1] = sum dp[0..v-delta]
        }

        long long ans = 0;
        for (int v = 0; v <= nums[n - 1]; v++)
            ans = (ans + dp[v]) % MOD;
        return (int)ans;
    }
};
```

## Example Walkthroughs

### Example 1 — `nums = [2, 3, 2]`, Expected: 4

**Base (i=0):** dp = [1, 1, 1, 0] (v=0,1,2 each get 1; v=3 is 0).

**i=1:** δ = max(0, 3−2) = **1**. prefix = [0, 1, 2, 3, 3].
```
v'=0: 0 − 1 = −1 < 0. dp[0] = 0.
v'=1: prefix[0−1+1 = 1] = prefix[1] = 1. dp[1] = 1.
v'=2: prefix[2−1+1 = 2] = prefix[2] = 2. dp[2] = 2.
v'=3: prefix[3−1+1 = 3] = prefix[3] = 3. dp[3] = 3.
```
dp = [0, 1, 2, 3].

**i=2:** δ = max(0, 2−3) = **0**. prefix = [0, 0, 1, 3, 6].
```
v'=0: prefix[0−0+1 = 1] = prefix[1] = 0. dp[0] = 0.
v'=1: prefix[1−0+1 = 2] = prefix[2] = 1. dp[1] = 1.
v'=2: prefix[2−0+1 = 3] = prefix[3] = 3. dp[2] = 3.
```
dp = [0, 1, 3] (v ≤ nums[2]=2 only).

**Answer = 0 + 1 + 3 = 4** ✓

**Verification:** Valid arr1 sequences with arr1[i] ∈ [0, nums[i]]:
- [0,1,1]: δ₀=0≥1? No. Wait: arr1[1]-arr1[0]=1-0=1 ≥ δ₀=1 ✓. arr1[2]-arr1[1]=0 ≥ δ₁=0 ✓. arr1=[0,1,1], arr2=[2,2,1] ✓.
- [0,1,2]: diff 1≥1 ✓, diff 1≥0 ✓. arr2=[2,2,0] ✓.
- [0,2,2]: diff 2≥1 ✓, diff 0≥0 ✓. arr2=[2,1,0] ✓.
- [1,2,2]: diff 1≥1 ✓, diff 0≥0 ✓. arr2=[1,1,0] ✓.

4 valid pairs ✓.

### Example 2 — `nums = [5, 5, 5, 5]`, Expected: 126

**All δᵢ = 0** (equal consecutive nums). Transition: `dp[v'] = prefix[v'+1]` = cumulative sum up through v'.

**Base (i=0):** dp = [1,1,1,1,1,1,0,...] (v=0..5).

**i=1 (δ=0):** dp[v'] = Σ dp[v] for v ≤ v': [1,2,3,4,5,6].

**i=2 (δ=0):** dp[v'] = Σ: [1,3,6,10,15,21].

**i=3 (δ=0):** dp[v'] = Σ: [1,4,10,20,35,56].

**Answer = 1+4+10+20+35+56 = 126** ✓

This is C(5+4, 4) = C(9,4) = 126 — the number of ways to choose a non-decreasing sequence of 4 values from {0,1,2,3,4,5}, which is "stars and bars" with repetition: C(6+4-1, 4) = C(9,4) = 126. ✓ The prefix sum transitions implement the "balls in bins" counting.

---

### Prefix Sum Mechanics — Why v' Loop Starts at delta

```cpp
for (int v = delta; v <= nums[i]; v++)
    dp[v] = prefix[v - delta + 1];
```

For v' < delta: `v' − delta < 0`, so no valid old values exist. dp[v'] = 0. ✓

For v' ≥ delta: sum of dp[0..v'−delta] = prefix[v'−delta+1]. ✓

The inner loop bounds also enforce 0 ≤ arr1[i] ≤ nums[i]: we only set dp[v] for v in [delta, nums[i]]. ✓

---

## Pitfalls & Edge Cases

- **δᵢ = 0 (nums non-increasing transition):** When nums[i] ≤ nums[i-1], δ = 0. The transition is just a standard prefix sum (arr1 just needs to be non-decreasing, no extra gap required). ✓

- **δᵢ > nums[i] (impossible step):** If max(0, nums[i]−nums[i-1]) > nums[i], then no valid v' exists (v'−δ < 0 for all v' ≤ nums[i]). dp becomes all zeros, ans = 0. This can only happen if nums[i] < nums[i] - nums[i-1], i.e., nums[i-1] < 0, which is impossible (nums[i] ≥ 1). ✓

- **Indexing into prefix:** `prefix[v'+1]` = sum of dp[0..v']. We use `prefix[v-delta+1]` = sum of dp[0..v-delta]. The +2-length prefix array avoids out-of-bounds when v-delta = maxVal. ✓

- **Reusing dp array vs. new array:** After computing prefix from dp, we reset dp (`fill`) before writing new values. This prevents old i-1 values from interfering with i values. Alternatively, use a separate `newDp` array. ✓

- **maxVal from the full array (not nums[i]):** dp has size maxVal+1. Even if some entries are 0 (for v > nums[i]), the prefix sums correctly skip them. ✓

---

## Key Takeaway

**Eliminate one array via the sum constraint, reduce both monotonicity conditions to a single transition on arr1.** The combined constraint `arr1[i+1] ≥ arr1[i] + δᵢ` is a "shifted non-decreasing" condition where each step has a minimum gap of δᵢ.

The DP transition `dp[v'] = Σ dp[v] for v ≤ v'−δ` is a **shifted prefix sum** — computable in O(maxVal) per step. This pattern ("count sequences where consecutive differences are ≥ some threshold") is exactly solved by prefix sums with a fixed offset.

**Pattern recognition:** "Count arrays where each element chooses from [0, limit_i] and consecutive differences are ≥ threshold_i" → DP with state = current value → transition via prefix-sum with offset δᵢ. Time: O(n × max_val), Space: O(max_val).
