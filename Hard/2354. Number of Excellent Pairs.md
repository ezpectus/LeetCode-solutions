# 2354. Number of Excellent Pairs
## O(n log n) тАФ Bit Identity + Sort + Binary Search

---

## Problem Statement

- Pair (num1, num2) is excellent if `popcount(num1 OR num2) + popcount(num1 AND num2) тЙе k`.
- Both num1 and num2 must exist in `nums`. Only distinct values matter (duplicates count as one).
- Ordered pairs: (a,b) and (b,a) are distinct.
- Return count of excellent pairs.

---

## Core Insight тАФ OR + AND Bit Identity

For any two integers a and b:
```
popcount(a OR b) + popcount(a AND b) = popcount(a) + popcount(b)
```

**Proof by bit:** For each bit position:
- Both set (1,1): contributes 1 to AND, 1 to OR тЖТ total 2 = 1+1 тЬУ
- One set (1,0) or (0,1): contributes 0 to AND, 1 to OR тЖТ total 1 = 1+0 тЬУ
- Neither set (0,0): contributes 0 to both тЖТ total 0 = 0+0 тЬУ

The condition simplifies to: `popcount(num1) + popcount(num2) тЙе k`.

---

## Algorithm

1. Deduplicate `nums` (only distinct values matter).
2. Compute popcount of each distinct value.
3. Sort the popcounts.
4. For each value with popcount `p`, binary search for the first index where popcounts `тЙе k тИТ p`. Count all such values.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long CountExcellentPairs(int[] nums, int k) {
        var distinct = new HashSet<int>(nums);

        var bits = new List<int>();
        foreach (int x in distinct)
            bits.Add(BitOperations.PopCount((uint)x));

        bits.Sort();
        int n = bits.Count;
        long res = 0;

        for (int i = 0; i < n; i++) {
            int need = k - bits[i];

            // Binary search: first index where bits[idx] >= need
            int lo = 0, hi = n;
            while (lo < hi) {
                int mid = (lo + hi) / 2;
                if (bits[mid] >= need) hi = mid;
                else lo = mid + 1;
            }

            res += n - lo;
        }

        return res;
    }
}
```

## Pitfalls & Edge Cases

- **Deduplication is essential** тАФ if nums has duplicate values, they don't create extra distinct values for pairing. A pair (a,a) is valid only if value `a` exists at least once (not twice). тЬУ

- **Self-pairs (a,a)** тАФ counted correctly: when i and j point to the same element in the sorted list, `nтИТlo` still counts it if `2 ├Ч bits[i] тЙе k`. тЬУ

- **k up to 60** тАФ max popcount of a 32-bit integer is 32, so max popcount sum is 64. The search range `need` could be 0 (all pairs valid) or > 32 (no pairs valid). Both handled naturally. тЬУ

- **`BitOperations.PopCount`** тАФ use `using System.Numerics;` or the static call. The `(uint)` cast is necessary for the method signature. тЬУ

## Key Takeaway

The problem looks like it requires evaluating OR and AND for every pair тАФ but the identity `popcount(a OR b) + popcount(a AND b) = popcount(a) + popcount(b)` collapses it to a simple sum condition.

Once reduced to "count pairs where popcount sum тЙе k":
- Sort by popcount
- Binary search for the complement threshold per element
- O(n log n) total

**Pattern recognition:** Any time you see `popcount(a OR b) + popcount(a AND b)`, immediately substitute `popcount(a) + popcount(b)`. This identity appears frequently in bit manipulation problems.
