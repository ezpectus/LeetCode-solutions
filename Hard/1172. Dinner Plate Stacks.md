# 1172. Dinner Plate Stacks
## O(log n) per operation — List of Stacks + SortedSet of Non-Full Indices

---

## Problem Statement

- Infinite stacks each with `capacity`. `push` → leftmost non-full stack. `pop` → rightmost non-empty stack. `popAtStack(i)` → pop from stack i.

---

## Core Idea — Track Non-Full Stacks with Sorted Set

Maintain:
- `stacks`: list of stacks (dynamic array).
- `not_full`: sorted set of indices of stacks that have room.

**Push**: take `not_full.Min` → push there → if now full, remove from `not_full`.

**Pop**: find rightmost non-empty stack (trim trailing empty stacks), pop from it → add to `not_full` if not already there. Trim trailing empty stacks again.

**PopAtStack(i)**: pop from `stacks[i]` → add `i` to `not_full`.

---

## Full Optimal Implementation (C#)

```csharp
public class DinnerPlates {
    int cap;
    List<Stack<int>> stacks = new();
    SortedSet<int> notFull = new();

    public DinnerPlates(int capacity) { cap = capacity; }

    public void Push(int val) {
        if (notFull.Count == 0) {
            stacks.Add(new Stack<int>());
            notFull.Add(stacks.Count-1);
        }
        int idx = notFull.Min;
        stacks[idx].Push(val);
        if (stacks[idx].Count == cap) notFull.Remove(idx);
    }

    public int Pop() {
        // Trim trailing empty stacks
        while (stacks.Count > 0 && stacks[^1].Count == 0) {
            notFull.Remove(stacks.Count-1);
            stacks.RemoveAt(stacks.Count-1);
        }
        if (stacks.Count == 0) return -1;
        int idx = stacks.Count-1;
        int val = stacks[idx].Pop();
        notFull.Add(idx);
        // Trim again
        while (stacks.Count > 0 && stacks[^1].Count == 0) {
            notFull.Remove(stacks.Count-1);
            stacks.RemoveAt(stacks.Count-1);
        }
        return val;
    }

    public int PopAtStack(int index) {
        if (index >= stacks.Count || stacks[index].Count == 0) return -1;
        int val = stacks[index].Pop();
        notFull.Add(index);
        return val;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Push** | **O(log n)** |
| **Pop** | **O(log n)** amortized |
| **PopAtStack** | **O(log n)** |

---

## Key Takeaway

`SortedSet` gives O(log n) access to the leftmost non-full stack for `push`, and the rightmost non-empty stack is simply `stacks.Last`. Trim trailing empty stacks after `pop` to keep `stacks.Count` accurate.
