# 1284. Minimum Number of Flips to Convert Binary Matrix to Zero Matrix
## O(2^(m×n) × m×n) — BFS on Bitmask State Space

---

## Problem Statement

- Flip any cell: it and its 4 neighbors toggle (0↔1).
- Find minimum flips to reach all-zeros, or -1 if impossible.
- `m,n ≤ 3` → at most 9 cells → 2⁹ = 512 states.

---

## Core Idea — BFS on State Space

**State** = bitmask of the matrix (bit `r*n+c` = value of cell `(r,c)`).

**Each flip** = XOR with a precomputed flip mask for that cell (the cell + its neighbors).

**BFS** from the initial state to state `0` gives the shortest path (minimum flips). Since flipping the same cell twice is a no-op, and each flip is reversible, BFS on the 2^(m×n) state space finds the optimal solution.

**Precompute flip masks**: for each cell `i`, `flipMask[i]` has bits set for cell `i` and all its valid neighbors.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinFlips(int[][] mat) {
        int m = mat.Length, n = mat[0].Length;
        int cells = m * n;

        // Encode initial state
        int start = 0;
        for (int r = 0; r < m; r++)
            for (int c = 0; c < n; c++)
                if (mat[r][c] == 1) start |= 1 << (r*n + c);

        if (start == 0) return 0;

        // Precompute flip masks
        int[] flip = new int[cells];
        int[] dr = {0,0,1,-1}, dc = {1,-1,0,0};
        for (int r = 0; r < m; r++)
            for (int c = 0; c < n; c++) {
                int i = r*n+c;
                flip[i] = 1 << i;
                for (int d = 0; d < 4; d++) {
                    int nr = r+dr[d], nc = c+dc[d];
                    if (nr>=0&&nr<m&&nc>=0&&nc<n)
                        flip[i] |= 1 << (nr*n+nc);
                }
            }

        // BFS
        var visited = new HashSet<int> { start };
        var q = new Queue<(int state, int steps)>();
        q.Enqueue((start, 0));

        while (q.Count > 0) {
            var (state, steps) = q.Dequeue();
            for (int i = 0; i < cells; i++) {
                int next = state ^ flip[i];
                if (next == 0) return steps + 1;
                if (visited.Add(next))
                    q.Enqueue((next, steps + 1));
            }
        }

        return -1;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(2^(mn) × mn)** | At most 512 states × 9 transitions |
| **Space** | **O(2^(mn))** | Visited set |

With m,n≤3: 512 × 9 = 4608 operations — instant.

---

## Why BFS (not DFS)

BFS guarantees the **shortest path** in an unweighted graph. Each flip = one edge. BFS finds minimum flips.

Also key: flipping the same cell twice cancels out → no need to revisit states → `visited` set prevents cycles.

---

## Example Walkthrough

**`mat=[[0,0],[0,1]]`** → start = `0b1000` = 8 (bit 3 set, cell (1,1))

BFS from state 8:
- Flip (1,0): mask=`1101`=13, state=8^13=5 (`0101`)
- Flip (0,1): mask=`1011`=11, state=8^11=3 (`0011`)
- Flip (1,1): mask=`1110`=14, state=8^14=6 (`0110`)
- ... continue until state=0 reached in 3 steps ✓

---

## Pitfalls & Edge Cases

- **`start==0`**: already zero matrix → return 0 immediately
- **Unreachable**: some configurations can't reach all-zeros → BFS exhausts all states → return -1
- **Bit ordering**: cell `(r,c)` → bit `r*n+c`; consistent encoding is crucial

---

## Key Takeaway

"Minimum operations on small binary state" → **BFS on bitmask state space**:

- State = bitmask of the grid (feasible since m,n≤3 → ≤9 bits → 512 states)
- Each operation = XOR with precomputed flip mask
- BFS = minimum number of operations

Pattern: whenever m×n ≤ ~20 cells in a binary grid problem asking for minimum operations → bitmask BFS.
