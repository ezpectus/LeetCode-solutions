
# LeetCode 3518. Smallest Palindromic Rearrangement II

## O(N * Σ^2) — Combinatorial Backtracking with Multiset Permutations

### Core Idea — Symmetric Half-String Multiset Permutations

Since any palindromic string is completely determined by its left half, we can reduce the problem to finding the $k$-th lexicographically smallest unique permutation of the multiset containing the characters of the left half. The single odd-frequency character (if one exists) is statically placed in the center.

We construct the left half of length $M = \lfloor N/2 \rfloor$ character-by-character from left to right. At each position, we try to place the smallest available character $c \in ['a'..'z']$. If we tentatively place $c$, we calculate the number of unique permutations possible for the remaining empty positions using the multinomial coefficient:
```
Permutations = (M - 1 - current_index)! / (freq[x]!)
```

If this count is strictly less than $k$, it means the $k$-th permutation lies beyond the range of configurations starting with $c$. We subtract this count from $k$ and try the next lexicographically larger character. If the count is greater than or equal to $k$, we confirm character $c$ for the current position, decrement its frequency, and proceed to the next position.

### Full Optimal Implementation (C++)

```cpp
// Combinatorial Backtracking + Multinomial Coefficients
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
private:
    // Safely calculate factorials and combinations under overflow limits
    double getPermutations(int remaining_len, const vector<int>& freq) {
        double total = 1.0;
        int current_pos = 1;
        
        for (int i = 0; i < 26; ++i) {
            if (freq[i] <= 0) continue;
            for (int j = 1; j <= freq[i]; ++j) {
                total = total * current_pos / j;
                current_pos++;
            }
        }
        return total;
    }

public:
    string smallestPalindrome(string s, int k) {
        int n = s.length();
        vector<int> full_freq(26, 0);
        for (char c : s) {
            full_freq[c - 'a']++;
        }

        // Half frequencies for the left side of the palindrome
        vector<int> half_freq(26, 0);
        char middle_char = '#';
        for (int i = 0; i < 26; ++i) {
            half_freq[i] = full_freq[i] / 2;
            if (full_freq[i] % 2 != 0) {
                middle_char = (char)('a' + i);
            }
        }

        int half_len = n / 2;
        
        // Quick check: total possible unique arrangements
        double total_ways = getPermutations(half_len, half_freq);
        if (total_ways < k) {
            return "";
        }

        string left_half = "";
        double current_k = k;

        // Construct the left half character-by-character
        for (int i = 0; i < half_len; ++i) {
            for (int c = 0; c < 26; ++c) {
                if (half_freq[c] <= 0) continue;

                // Tentatively place character 'c'
                half_freq[c]--;
                double ways = getPermutations(half_len - 1 - i, half_freq);

                if (ways >= current_k) {
                    left_half += (char)('a' + c);
                    break; // Character confirmed, move to next index
                } else {
                    current_k -= ways;
                    half_freq[c]++; // Backtrack
                }
            }
        }

        string right_half = left_half;
        reverse(right_half.begin(), right_half.end());

        if (n % 2 != 0) {
            return left_half + middle_char + right_half;
        }
        return left_half + right_half;
    }
};
```

## 4. Comprehensive Complexity & Mathematical Analysis

### Detailed Time Complexity Decomposition

The computational pipeline of this combinatorial search engine is split into isolated processing steps. The total time complexity is derived from the summation of these steps:

$$T_{\text{total}} = T_{\text{init}} + T_{\text{backtracking}} + T_{\text{assembly}}$$

#### Frequency Initialization Phase ($T_{\text{init}}$)
We scan the input string of length $N$ to compile character counts. This is a single linear-time sweep:
$$T_{\text{init}} = O(N)$$

#### Permutations Backtracking Phase ($T_{\text{backtracking}}$)
The construction process operates on $M$ steps, where $M = \lfloor N/2 \rfloor$ (up to $7500$ steps). At each position, it runs a search over the alphabet of size $\Sigma = 26$. For each candidate character, we evaluate the multinomial coefficient of the remaining elements.

The helper function `getPermutations` loops over the alphabet size of $26$ and accumulates products and divisions up to the remaining length. Since the inner calculation is strictly bounded by the alphabet frequency sum $\sum \text{freq} \le 7500$, the complexity of calculating permutations at step $i$ is proportional to the number of active non-zero character counts, which is at most $26$.
$$T_{\text{backtracking}} = O\left(\frac{N}{2} \times \Sigma \times \Sigma\right) = O(N \cdot \Sigma^2)$$

With $N \le 1.5 \times 10^4$ and $\Sigma = 26$:
$$T_{\text{backtracking}} \approx 7500 \times 676 \approx 5 \times 10^6 \text{ operations (well within 10ms)}$$

#### String Assembly Phase ($T_{\text{assembly}}$)
Reversing the half-string and gluing the components together runs in linear time relative to the output length:
$$T_{\text{assembly}} = O(N)$$

Thus, the overall time complexity is tightly bounded by:
$$\text{Time Complexity} = O(N \cdot \Sigma^2)$$

### Detailed Space Complexity Decomposition

The structural memory overhead is determined by the following internal allocation blocks:

#### Auxiliary Count Arrays
The frequency vectors `full_freq` and `half_freq` use fixed-size static allocations:
$$S_{\text{freq}} = 26 \times 4 \text{ bytes} \times 2 \approx 208 \text{ bytes} \implies O(\Sigma)$$

#### String Buffers
The `left_half` string builds up to size $\frac{N}{2}$. The final returned string occupies $N$ characters:
$$S_{\text{strings}} = N \text{ bytes} \implies O(N)$$

$$\text{Space Complexity} = O(N + \Sigma) \text{ which simplifies to } O(N)$$

## 5. Mathematical Formulations & Foundational Algorithms

### Multinomial Coefficient Permutation Formula

The total number of unique permutations of a multiset of size $L$ containing $k$ distinct categories of elements, where category $i$ has a frequency count of $f_i$, is formulated as:

$$P(L; f_1, f_2, \dots, f_k) = \frac{L!}{f_1! \cdot f_2! \dots f_k!}$$

In our backtracking logic, at any state with $R$ remaining open slots and active character counts $\vec{f}$, the unique combinations are calculated on-the-fly dynamically:

$$P(R; \vec{f}) = \prod_{i=0}^{25} \binom{R - \sum_{j=0}^{i-1} f_j}{f_i}$$

This is computed iteratively by multiplying successive combinations to avoid numerical overflow issues that would occur from calculating large factorials like $7500!$ directly. Since the user input constraint on $k$ is $k \le 10^6$, we can safely use double-precision floating-point numbers (`double`) for intermediate counts. They provide exact integer representations up to $2^{53} \approx 9 \times 10^{15}$, which completely prevents truncation or representation errors during calculation.

#### Pitfalls & Edge Cases
*   **Combinatorial Overflow**: Handled by using `double` for safe multinomial factor multiplication up to $k \le 10^6$.
*   **Fewer than K permutations**: Checked at the start by assessing total possible permutations of `half_len`.

#### Key Takeaway
Reducing symmetric structures (like palindromes) to their non-redundant halves allows standard lexicographical backtracking to be performed efficiently.

**Pattern recognition:** `"Return the k-th lexicographically smallest palindromic permutation"` $\to$ Reduce to half-string generation, use greedy alphabet search, and count unique multinomial combinations at each transition.
