# LeetCode 3519. Count Numbers with Non-Decreasing Digits

## 1. Constraints & Complexity Bottleneck Analysis

### Constraints:
- **Length of strings l and r:** 1 <= l.length <= r.length <= 100
  - The numbers are massive, containing up to 100 decimal digits. This instantly eliminates any approach using standard integer types (`long`, `double`, or even 64-bit bounds). We must use arbitrary-precision arithmetic (Big Integer operations or custom array manipulation).
- **Base b:** 2 <= b <= 10
  - The base is small, which simplifies state management in our dynamic programming because the maximum digit value is at most b - 1.
- **Value of l is less than or equal to r.**
- **Output:** Modulo 10^9 + 7.

### The Complexity Bottleneck:
The core bottleneck is twofold:
1. **Base Conversion of Huge Decimals:** We are given $l$ and $r$ in base 10, but the non-decreasing property is defined in base $b$. We need to convert a 100-digit decimal string to its equivalent representation in base $b$. Since $b \ge 2$, the length of the base-$b$ representation can be up to ~333 digits (for $b=2$). A naive conversion must be done efficiently without losing precision.
2. **State Space Exploration:** A brute-force traversal of all numbers in the range is impossible. We must use Digit Dynamic Programming to count valid configurations in $O(\text{length} \times b)$ states.

## 2. Algorithm Design & Mathematical Invariants

### 2.1 Core Concept
To find the count of valid numbers in the range [l, r], we transform the range query into two prefix queries:
`Answer = Solve(r) - Solve(l - 1)`

To implement this, we design the following pipeline:
1. **String Subtraction:** Decrement the decimal string $l$ by 1 to get $l - 1$.
2. **Base Conversion:** Convert both decimal strings ($r$ and $l - 1$) into arrays of digits in base $b$. We achieve this using a "divide-by-base" simulation on the decimal string representation.
3. **Digit DP:** Solve the prefix count for a base-$b$ digit array $X$ using memoized depth-first search (DFS).

### 2.2 Key Invariants & Digit DP State
Our DP state is defined as `dp[idx][prev_digit][is_less][is_started]`:
- `idx`: The current digit position we are placing (from most significant to least significant, 0 to length - 1).
- `prev_digit`: The digit selected in the previous step (0 to $b - 1$).
- `is_less`: A boolean flag. If true, the prefix we have placed is already strictly less than the prefix of $X$, meaning we can place any valid digit up to $b - 1$ at the current position. If false, we are bound by the digit $X[idx]$.
- `is_started`: A boolean flag indicating whether we have placed at least one non-zero digit. This is crucial because leading zeros do not count toward the "non-decreasing" constraint. For example, the number "035" in base 8 is conceptually "35", which is valid, even though $0 < 3$.

#### Transitions:
At state `(idx, prev_digit, is_less, is_started)`:
- Determine the upper bound for the current digit: `limit = is_less ? (b - 1) : X[idx]`.
- Loop through each candidate digit `d` from 0 to `limit`:
  - If `is_started` is false and `d == 0`: We transition to `(idx + 1, 0, is_less || (d < X[idx]), false)`.
  - If we decide to place a non-zero digit `d` (or if `is_started` is already true): We must satisfy the invariant `d >= prev_digit`. If this holds, we transition to `(idx + 1, d, is_less || (d < X[idx]), true)`.

## 3. Full Optimal Implementation (C#)
```
// Digit DP + Arbitrary Precision Base Conversion + String Arithmetic
using System;
using System.Collections.Generic;
using System.Text;

public class Solution {
    private const int MOD = 1000000007;

    public int CountNumbers(string l, string r, int b) {
        // Calculate Solve(r)
        int[] rBaseB = ConvertToBase(r, b);
        long ansR = Solve(rBaseB, b);

        // Calculate Solve(l - 1)
        string lMinusOne = DecrementDecimalString(l);
        long ansLMinusOne = 0;
        if (lMinusOne != "0") {
            int[] lBaseB = ConvertToBase(lMinusOne, b);
            ansLMinusOne = Solve(lBaseB, b);
        }

        long result = (ansR - ansLMinusOne + MOD) % MOD;
        return (int)result;
    }

    // Standard high-precision subtraction: string - 1
    private string DecrementDecimalString(string s) {
        char[] digits = s.ToCharArray();
        int i = digits.Length - 1;
        while (i >= 0) {
            if (digits[i] > '0') {
                digits[i]--;
                break;
            } else {
                digits[i] = '9';
                i--;
            }
        }
        
        // Remove leading zeros if they occur
        int start = 0;
        while (start < digits.Length - 1 && digits[start] == '0') {
            start++;
        }
        return new string(digits, start, digits.Length - start);
    }

    // Converts decimal string representation to base B digit array
    private int[] ConvertToBase(string decimalStr, int baseVal) {
        List<int> decimalDigits = new List<int>();
        foreach (char c in decimalStr) {
            decimalDigits.Add(c - '0');
        }

        List<int> resultDigits = new List<int>();
        while (decimalDigits.Count > 0) {
            List<int> nextDecimalDigits = new List<int>();
            int remainder = 0;
            bool lead = true;

            for (int i = 0; i < decimalDigits.Count; i++) {
                int current = remainder * 10 + decimalDigits[i];
                int quotient = current / baseVal;
                remainder = current % baseVal;

                if (quotient > 0 || !lead) {
                    nextDecimalDigits.Add(quotient);
                    lead = false;
                }
            }

            resultDigits.Add(remainder);
            decimalDigits = nextDecimalDigits;
        }

        resultDigits.Reverse();
        return resultDigits.ToArray();
    }

    private long Solve(int[] num, int b) {
        int n = num.Length;
        // memoization table: dp[idx][prev_digit][is_less][is_started]
        long[,,,] dp = new long[n, b, 2, 2];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < b; j++) {
                for (int k = 0; k < 2; k++) {
                    for (int l = 0; l < 2; l++) {
                        dp[i, j, k, l] = -1;
                    }
                }
            }
        }

        return GetCount(0, 0, 0, 0, num, b, dp);
    }

    private long GetCount(int idx, int prevDigit, int isLess, int isStarted, int[] num, int b, long[,,,] dp) {
        if (idx == num.Length) {
            return isStarted == 1 ? 1 : 0; // Exclude empty number
        }

        if (dp[idx, prevDigit, isLess, isStarted] != -1) {
            return dp[idx, prevDigit, isLess, isStarted];
        }

        int limit = (isLess == 1) ? (b - 1) : num[idx];
        long count = 0;

        for (int d = 0; d <= limit; d++) {
            int nextIsLess = (isLess == 1 || d < num[idx]) ? 1 : 0;

            if (isStarted == 0) {
                if (d == 0) {
                    // Keep placing leading zeros
                    count = (count + GetCount(idx + 1, 0, nextIsLess, 0, num, b, dp)) % MOD;
                } else {
                    // Start placing digits
                    count = (count + GetCount(idx + 1, d, nextIsLess, 1, num, b, dp)) % MOD;
                }
            } else {
                // Must maintain non-decreasing order of digits
                if (d >= prevDigit) {
                    count = (count + GetCount(idx + 1, d, nextIsLess, 1, num, b, dp)) % MOD;
                }
            }
        }

        return dp[idx, prevDigit, isLess, isStarted] = count;
    }
}
```

## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The algorithm's time complexity comes from two distinct parts:

1. **Base Conversion:**
   To convert a decimal number of length $L$ to base $b$, we repeatedly divide by $b$.
   - In each step, we perform a division pass over the current decimal representation of length $L$.
   - The number of base-$b$ digits generated is proportional to $L \times \log_b(10)$.
   - Thus, the conversion takes:
     `T_convert = O(L^2 * log_b(10))` operations.
   - Given $L \le 100$, this takes at most ~33,300 operations, which is highly efficient.

2. **Digit DP Solver:**
   The number of states in our dynamic programming is bounded by:
   `States = n * b * 2 * 2`
   where $n \le 333$ (the max length in base 2) and $b \le 10$.
   - For each state, we transition by looping up to $b$ times.
   - Thus, the DP traversal runs in:
     `T_dp = O(n * b^2)` operations.
   - For worst-case inputs ($n = 333, b = 10$), this requires around $333 \times 100 = 33,300$ operations.

**Total Time Complexity:** O(L^2 * log_b(10) + L * b^2 * log_b(10)), which processes within ~5ms.

### 4.2 Detailed Space Complexity Decomposition
- **Memoization Table:**
  The `dp` array has dimensions of size `[n, b, 2, 2]`. For maximum bounds ($333 \times 10 \times 2 \times 2$), it stores $13,320$ 64-bit integer values (approx 106 KB).
- **Converted Digit Arrays:**
  Storing the base-$b$ numbers requires a digit array of size up to $333$ integers.

**Total Space Complexity:** O(L * b * log_b(10)) auxiliary space.

### 4.3 Complexity Summary
| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(L^2 * log_b(10) + L * b^2 * log_b(10)) | High-precision base conversion is the scaling bottleneck. |
| **Space Complexity** | O(L * b * log_b(10)) | Dominated by the dynamic programming memoization table. |

## 5. Mathematical Formulations & Foundational Algorithms

### Base Conversion Logic
Let $D$ be a massive base-10 number represented as an array of decimal digits. To represent $D$ in a target base $b$, we iteratively compute:
`Remainder = D % b`
`D = floor(D / b)`
where division and modulo are simulated digit-by-digit from left to right. The sequence of remainders in reverse order yields the exact digit representation of $D$ in base $b$.

### Non-Decreasing Verification Invariant
Let the base-$b$ string be $d_1 d_2 \dots d_k$. The non-decreasing property holds if and only if:
`d_i <= d_{i+1} for all i in [1, k-1]`
To correctly enforce this while traversing prefix digits, we must parameterize our states with the active predecessor digit `prev_digit` once our number starts (`is_started = 1`), pruning all search branches that attempt to place any digit $d < prev\_digit$.

## Pitfalls & Edge Cases
- **Skipping Leading Zeros Treatment:** If leading zeros are treated as part of the non-decreasing validation, numbers like "25" (represented with padding as "025") would be classified as valid ($0 \le 2 \le 5$). However, a number like "10" (represented as "010") would be incorrectly invalidated because the initial transition $0 \to 1 \to 0$ violates the non-decreasing rule. Tracking `is_started` prevents this.
- **Off-By-One boundary on $l$:** Since $l$ can have 100 digits, subtracting 1 cannot be done with standard integer arithmetic. Implementing a robust string-subtraction helper `DecrementDecimalString` is required.
- **Modulo Arithmetic:** Subtraction in modular arithmetic can lead to negative results. Applying `(ansR - ansLMinusOne + MOD) % MOD` safely handles underflow.

---

## Key Takeaway
When constraints require evaluating digit properties under a different base than given, decouplement is key: first perform a long-division base conversion, then apply standard Digit DP on the converted state array.

**Pattern recognition:** "count of integers in the range [l, r] whose digits are non-decreasing in base b" -> High-Precision Base Conversion + Digit DP with Lead-Zero Flag
