# 2547. Minimum Cost to Split an Array
## O(n²) — Dynamic Programming with Frequency Tracking

---

## Problem Statement

- You are given an integer array `nums` (1 ≤ n ≤ 1000) where 0 ≤ nums[i] < n.
- You need to split the array into some number of non-empty subarrays.
- The cost of a split is the sum of the importance values of each subarray.
- For a subarray, `trimmed(subarray)` is the version where all numbers appearing only once are removed.
- The importance value of a subarray is `k + trimmed(subarray).length`.
- Return the minimum possible cost of a split.

---

## Core Idea — DP with Rolling Frequency Count

Key insight: For each position i, we need to decide where to end the current subarray. The importance of a subarray [j, i] depends on how many elements appear more than once.

DP definition:
- `dp[i]` = minimum cost to split `nums[0...i-1]`
- For each position i, try all possible starting positions j of the last subarray
- Compute the importance of subarray [j, i) using a frequency map
- Transition: `dp[i] = min(dp[j] + importance[j, i))` for all valid j

Importance calculation:
- Track frequency of each element in the current window [j, i)
- `trimmed_length` = count of elements that appear ≥ 2 times (counted with their full frequency)
- `importance = k + trimmed_length`

Optimization: For each ending position i, expand the window backwards from i, maintaining frequencies incrementally.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinCost(int[] nums, int k) {
        int n = nums.Length;
        
        // dp[i] = minimum cost to split nums[0..i-1]
        int[] dp = new int[n + 1];
        
        // Base case: cost of splitting nothing is 0
        dp[0] = 0;
        
        // For each ending position
        for (int i = 1; i <= n; i++) {
            dp[i] = int.MaxValue;
            
            // freq[x] = frequency of element x in current subarray [j, i)
            int[] freq = new int[n];
            int trimmedLen = 0;
            
            // Try all possible starting positions j for the last subarray
            // Expand backwards from i
            for (int j = i - 1; j >= 0; j--) {
                int num = nums[j];
                freq[num]++;
                
                // Update trimmed length
                // When freq changes from 1 to 2, we add 2 (both copies count)
                // When freq goes from k to k+1 (k >= 2), we add 1
                if (freq[num] == 2) {
                    trimmedLen += 2; // Both elements now count
                } else if (freq[num] > 2) {
                    trimmedLen += 1; // One more occurrence
                }
                
                // Importance of subarray [j, i)
                int importance = k + trimmedLen;
                
                // Update dp[i]
                dp[i] = Math.Min(dp[i], dp[j] + importance);
            }
        }
        
        return dp[n];
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n²) | Nested loops: outer O(n), inner O(n) for each position |
| Space Complexity | O(n) | DP array + frequency array (size n) |

Optimal — Cannot do better than O(n²) since we need to consider all possible subarray boundaries and compute their importance values.

## Why This Works — Example Walkthrough

**Example 1:** `nums = [1,2,1,2,1,3,3], k = 2`

Let's trace key DP states:

Initial: `dp[0] = 0`

**Computing dp[2] (splitting [1,2]):**
- Subarray [0, 2): [1, 2]
  - freq[1] = 1, freq[2] = 1
  - trimmedLen = 0 (no duplicates)
  - importance = 2 + 0 = 2
  - dp[2] = dp[0] + 2 = 2

**Computing dp[5] (splitting [1,2,1,2,1]):**
- Trying j=2: subarray [2, 5): [1, 2, 1]
  - freq[1] = 2, freq[2] = 1
  - trimmedLen = 2 (two 1's)
  - importance = 2 + 2 = 4
  - dp[5] = min(dp[5], dp[2] + 4) = min(∞, 2 + 4) = 6

**Computing dp[7] (full array):**
- Trying j=2: subarray [2, 7): [1, 2, 1, 3, 3]
  - freq[1] = 2, freq[2] = 1, freq[3] = 2
  - trimmedLen = 2 + 2 = 4
  - importance = 2 + 4 = 6
  - dp[7] = min(dp[7], dp[2] + 6) = 2 + 6 = 8 ✓

**Output:** 8

---

**Example 2:** `nums = [1,2,1,2,1], k = 2`

- Best split: [1,2] | [1,2,1]
- Cost = 2 + (2 + 2) = 6 ✓

**Example 3:** `nums = [1,2,1,2,1], k = 5`

- Best split: entire array [1,2,1,2,1]
- freq[1] = 3, freq[2] = 2
- trimmedLen = 3 + 2 = 5
- Cost = 5 + 5 = 10 ✓

## Pitfalls & Edge Cases

- **Frequency tracking logic** — When an element's frequency goes from 1 to 2, we must add 2 to trimmedLen (both occurrences now count). For frequency k → k+1 where k ≥ 2, we add only 1.
  
- **Empty subarrays** — The problem guarantees non-empty subarrays. The inner loop correctly starts from `j = i - 1` (at least one element).

- **Large k values** — When k is very large (up to 10⁹), it might be optimal to have just one subarray to minimize the number of times we pay k.

- **All unique elements** — If all elements in a subarray are unique, trimmedLen = 0, and importance = k.

- **Integer overflow** — Not an issue here since max cost ≈ n × k ≤ 1000 × 10⁹ which fits in int.

- **Backwards expansion optimization** — Expanding backwards from position i and maintaining frequency counts incrementally is more efficient than recalculating for each [j, i) pair.

---

## Key Takeaway

This is a classic interval DP problem where the optimal substructure is:
- The minimum cost to split nums[0...i-1] depends on where we place the last split
- For each position i, we try all possible starting positions j and pick the one that minimizes `dp[j] + importance[j, i)`

The crucial optimization is computing importance values incrementally:
- Expand the window backwards from i
- Maintain frequencies as we go
- Update trimmedLen based on frequency transitions (1→2 adds 2, k→k+1 adds 1 for k≥2)

**Pattern recognition:** Whenever you see "split array into subarrays to minimize/maximize cost" → think interval DP with O(n²) transitions.

The constraint `nums[i] < n` is the key — it allows us to use a simple array for frequency counting instead of a hashmap, and guarantees no overflow in the frequency array indexing.
