# 2458. Height of Binary Tree After Subtree Removal Queries
## O(n) — Precomputed Heights + DFS Propagating Best External Height

---

## Problem Statement

- Given a binary tree with n nodes (2 ≤ n ≤ 10⁵), each node has a unique value 1..n.
- m independent queries: for each query, remove the subtree rooted at `queries[i]` and return the resulting tree height.
- Height = number of edges on the longest root-to-leaf path.

---

## Core Idea — Propagate "Best External Height" Top-Down

Key insight: For each node `u`, we want the maximum height the tree can achieve using **only nodes outside u's subtree**. Call this `ans[u]`.

We compute `ans[u]` for every node in a single DFS pass, propagating a `best` value downward:

**`best` = the best height achievable using nodes strictly outside the current subtree**

When at node `p` (depth `d`) deciding which `best` to pass to each child:
- To left child: if we remove left's subtree, we can use the right subtree → `depth + 1 + height[right]`, or just `depth` (node p itself) if no right child. Take max with inherited `best`.
- To right child: symmetric, using left subtree height.

This requires `height[u]` precomputed for all nodes (one DFS upward), then `ans[u]` (one DFS downward).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    private int[] ans = new int[100001];
    private int[] height = new int[100001];

    public int[] TreeQueries(TreeNode root, int[] queries) {
        ComputeHeight(root);
        DFS(root, depth: 0, best: 0);

        int[] res = new int[queries.Length];
        for (int i = 0; i < queries.Length; i++) {
            res[i] = ans[queries[i]];
        }
        return res;
    }

    // Bottom-up: compute height of each node's subtree
    private int ComputeHeight(TreeNode node) {
        if (node == null) return -1;
        height[node.val] = 1 + Math.Max(
            ComputeHeight(node.left),
            ComputeHeight(node.right)
        );
        return height[node.val];
    }

    // Top-down: propagate best external height to each node
    private void DFS(TreeNode node, int depth, int best) {
        if (node == null) return;

        ans[node.val] = best; // if this subtree is removed, this is the answer

        int hLeft  = node.left  != null ? height[node.left.val]  : -1;
        int hRight = node.right != null ? height[node.right.val] : -1;

        // Best external for left child: use right subtree or just current node
        int leftBest = node.right != null
            ? Math.Max(best, depth + 1 + hRight)
            : Math.Max(best, depth);

        // Best external for right child: use left subtree or just current node
        int rightBest = node.left != null
            ? Math.Max(best, depth + 1 + hLeft)
            : Math.Max(best, depth);

        DFS(node.left,  depth + 1, leftBest);
        DFS(node.right, depth + 1, rightBest);
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n + m) | Two O(n) DFS passes + O(m) query lookups |
| Space Complexity | O(n) | Height and answer arrays + recursion stack |

## Why This Works — Proof

For node `u` at depth `d`, when we remove its subtree the surviving nodes are exactly those NOT in `subtree(u)`. The tree height equals the maximum `depth[v] + height[v]` over all surviving nodes.

The DFS propagates this maximum downward:
- At each node `p`, `best` already contains the max depth+height reachable outside `p`'s subtree
- For `p`'s left child `l`: surviving nodes are those outside `p` (captured in `best`) PLUS `p` itself and `p`'s right subtree (best path = `depth[p] + 1 + height[right]`)
- The max of these two is exactly what `l` needs

## Example Walkthrough

**Example 1:** Tree below, `queries = [4]`
```
        1 (h=3)
       / \
    3(h=1) 4(h=2)
     /    / \
   2(h=0) 6  5(h=1)
              \
               7
```

**DFS(1, depth=0, best=0):** ans[1]=0 (unused)
- leftBest  = max(0, 0+1+2) = **3** → DFS(3, 1, 3): ans[3]=3
  - leftBest  = max(3, 1+0) = 3 → DFS(2, 2, 3): ans[2]=3
- rightBest = max(0, 0+1+1) = **2** → DFS(4, 1, 2): **ans[4]=2** ✓
  - leftBest  = max(2, 1+1+1) = 3 → DFS(6, 2, 3): ans[6]=3
  - rightBest = max(2, 1+1+0) = 2 → DFS(5, 2, 2): ans[5]=2
    - rightBest = max(2, 2) = 2 → DFS(7, 3, 2): ans[7]=2

**Output:** [ans[4]] = **[2]** ✓

Remove node 4 → remaining path 1→3→2, height = 2.

**Example 2:** `root=[5,8,9,2,1,3,7,4,6]`, `queries=[3,2,4,8]`

Key heights: height[8]=2, height[9]=1, height[2]=1

**DFS(5, 0, 0):**
- leftBest  = max(0, 0+1+1) = **2** → DFS(8, 1, 2): **ans[8]=2** ✓
  - leftBest  = max(2, 1+1+0) = 2 → DFS(2, 2, 2): **ans[2]=2** ✓
    - leftBest  = max(2, 2+1+0) = **3** → DFS(4, 3, 3): **ans[4]=3** ✓

  - rightBest = max(2, 1+1+1) = **3** → DFS(1, 2, 3): ans[1]=3
- rightBest = max(0, 0+1+2) = **3** → DFS(9, 1, 3): ans[9]=3
  - leftBest  = max(3, 1+1+0) = 3 → DFS(3, 2, 3): **ans[3]=3** ✓
  - rightBest = max(3, 1+1+0) = 3 → DFS(7, 2, 3): ans[7]=3

## Pitfalls & Edge Cases

- **No right child when computing leftBest** — Use `max(best, depth)` where `depth` is the path length to the current node (which remains in the tree). Without this, we'd undercount the current node's contribution. ✓

- **Arrays sized 100001** — Node values are 1..n ≤ 10⁵, so index directly by value. ✓

- **Root is never queried** — `ans[root.val]` gets set to `best=0` (the initial call), but since `queries[i] != root.val`, it's never used.

- **Height of null = -1** — Ensures `1 + max(-1, -1) = 0` for leaf nodes. ✓

- **Stack depth** — Recursion depth up to n=10⁵ (linear tree). For safety in production, use iterative DFS. LeetCode's C# judge handles this within limits.

- **Height vs depth** — `height[u]` = edges downward from u to deepest leaf. `depth` = edges upward from root to u. Combined: `depth + 1 + height[child]` = length of the path root → current node → child → deepest descendant of child. ✓

## Key Takeaway

The problem reduces to: **for each node, precompute the answer assuming it is removed**. This precomputation can be done in O(n) using two DFS passes:

1. **Bottom-up** (`ComputeHeight`): each node learns the height of its subtree
2. **Top-down** (`DFS`): each node receives and stores the best height reachable outside its subtree

The `best` value passed top-down is the maximum of:
- `best` from the parent (heights reachable above)
- `depth + 1 + height[sibling]` (heights reachable through the sibling subtree)

**Pattern recognition:** "For each node in a tree, answer a query about the rest of the tree" → two-pass DFS: one bottom-up to aggregate subtree info, one top-down to propagate external info. Queries become O(1) lookups after O(n) preprocessing.
