# 1955. Count Number of Special Subsequences
## O(n) — DP with 3 States (Phase Tracking)

---

## Problem Statement

- Given array `nums` of only `0`, `1`, `2`.
- A **special** subsequence = positive number of `0`s, then positive number of `1`s, then positive number of `2`s.
- Return the count of different special subsequences modulo `10^9 + 7`.
- Two subsequences are different if their chosen index sets differ.

---

## Core Idea — Track How Many Subsequences Are at Each Phase

**Key insight**: a special subsequence passes through exactly 3 phases:
- **Phase 0**: collecting `0`s (at least one)
- **Phase 1**: collecting `1`s (at least one), after finishing all `0`s
- **Phase 2**: collecting `2`s (at least one), after finishing all `1`s

Define:
- `dp0` = number of subsequences consisting of only `0`s so far (in phase 0)
- `dp1` = number of subsequences of form `0...0 1...1` so far (in phase 1)
- `dp2` = number of **complete** special subsequences so far (in phase 2)

**Transitions** when processing element `x`:

**If `x == 0`**:
- Extend any existing phase-0 subsequence with this `0`: `dp0` ways
- Start a new phase-0 subsequence with just this `0`: `1` way
- → `dp0 = 2*dp0 + 1`

**If `x == 1`**:
- Extend any existing phase-1 subsequence with this `1`: `dp1` ways
- Promote any phase-0 subsequence to phase-1 by appending this `1`: `dp0` ways
- → `dp1 = 2*dp1 + dp0`

**If `x == 2`**:
- Extend any existing phase-2 subsequence with this `2`: `dp2` ways
- Promote any phase-1 subsequence to phase-2 by appending this `2`: `dp1` ways
- → `dp2 = 2*dp2 + dp1`

**Answer**: `dp2` after processing all elements.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CountSpecialSubsequences(int[] nums) {
        const int MOD = 1_000_000_007;
        long dp0 = 0, dp1 = 0, dp2 = 0;

        foreach (int x in nums) {
            if (x == 0) dp0 = (2 * dp0 + 1)   % MOD;
            else if (x == 1) dp1 = (2 * dp1 + dp0) % MOD;
            else dp2 = (2 * dp2 + dp1)  % MOD;
        }

        return (int)dp2;
    }
}
```

## Why This Works — The Phase Transition Argument

Each `dp` variable counts subsequences that are "complete" in their phase — a valid prefix that can still be extended within the same phase or promoted to the next.

**Why `dp0 = 2*dp0 + 1` when seeing a `0`?**

Every existing phase-0 subsequence can either:
- **Include** this `0` → extends it, still in phase 0 (`dp0` new subsequences)
- **Exclude** this `0` → unchanged (`dp0` unchanged subsequences)

So existing ones double to `2*dp0`. Plus we can start a brand new single-`0` subsequence → `+1`.

Combined: `dp0 = 2*dp0 + 1`.

**Why `dp1 = 2*dp1 + dp0` when seeing a `1`?**

- Existing phase-1 subsequences either include or exclude this `1` → `2*dp1`
- Each phase-0 subsequence can be promoted to phase-1 by appending this `1` → `+dp0`

Note: we do **not** add `+1` here because a phase-1 subsequence must have at least one `0` first — we can't start fresh with just a `1`.

**Why `dp2 = 2*dp2 + dp1` when seeing a `2`?**

Same logic: existing complete subsequences double, plus each phase-1 subsequence gets completed by this `2`. No `+1` for the same reason — must have prior `0`s and `1`s.

## Example Walkthrough

**`nums = [0,1,2,2]`**

| Step | x | dp0 | dp1 | dp2 |
|------|---|-----|-----|-----|
| init | — | 0 | 0 | 0 |
| i=0 | 0 | 2·0+1=**1** | 0 | 0 |
| i=1 | 1 | 1 | 2·0+1=**1** | 0 |
| i=2 | 2 | 1 | 1 | 2·0+1=**1** |
| i=3 | 2 | 1 | 1 | 2·1+1=**3** |

Answer: `dp2 = 3` ✓

---

**`nums = [2,2,0,0]`**

| Step | x | dp0 | dp1 | dp2 |
|------|---|-----|-----|-----|
| init | — | 0 | 0 | 0 |
| i=0 | 2 | 0 | 0 | 2·0+0=**0** |
| i=1 | 2 | 0 | 0 | **0** |
| i=2 | 0 | **1** | 0 | 0 |
| i=3 | 0 | **3** | 0 | 0 |

Answer: `dp2 = 0` ✓ (no `1`s → `dp1` never grows → `dp2` stays 0)

---

**`nums = [0,1,2,0,1,2]`**

| Step | x | dp0 | dp1 | dp2 |
|------|---|-----|-----|-----|
| init | — | 0 | 0 | 0 |
| i=0 | 0 | **1** | 0 | 0 |
| i=1 | 1 | 1 | **1** | 0 |
| i=2 | 2 | 1 | 1 | **1** |
| i=3 | 0 | **3** | 1 | 1 |
| i=4 | 1 | 3 | **5** | 1 |
| i=5 | 2 | 3 | 5 | **7** |

Answer: `dp2 = 7` ✓

## Pitfalls & Edge Cases

- **No `+1` for phases 1 and 2**: unlike phase 0, you cannot start a fresh phase-1 or phase-2 subsequence from scratch — prior phases are required
- **Use `long` for intermediate products**: `2 * dp0` can reach up to `2 × (10^9 + 6)` before `% MOD`, overflowing `int`
- **Elements not equal to current phase still matter**: a `0` seen after `1`s contributes to `dp0` for future use, potentially enabling new `dp1` and `dp2` later — the DP handles this naturally
- **Order matters**: the transitions correctly model left-to-right ordering of the original array

---

## Key Takeaway

"Count special subsequences (0...0 1...1 2...2)" → **3-state DP tracking subsequences at each phase**:

```
if x == 0: dp0 = 2*dp0 + 1       // double existing + start new
if x == 1: dp1 = 2*dp1 + dp0     // double existing + promote from phase 0
if x == 2: dp2 = 2*dp2 + dp1     // double existing + promote from phase 1

answer = dp2
```

Each element either extends same-phase subsequences (doubling) or acts as a phase transition. O(n) time, O(1) space.
