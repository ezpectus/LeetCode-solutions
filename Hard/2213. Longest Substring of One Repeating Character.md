# 2213. Longest Substring of One Repeating Character
## O((n + k) log n) — Segment Tree with Run-Length Metadata

---

## Problem Statement

- String `s`, `k` point updates (change one character).
- After each update, return the length of the longest run of a single repeating character.

---

## Core Idea — Segment Tree with Run Metadata

Each node of the segment tree stores metadata for its covered range `[l, r]`:

| Field | Meaning |
|-------|---------|
| `lc` | character at position `l` (leftmost) |
| `rc` | character at position `r` (rightmost) |
| `lpref` | length of left prefix run (all same char starting from `l`) |
| `rpref` | length of right suffix run (all same char ending at `r`) |
| `best` | maximum run length in `[l, r]` |
| `length` | total length of segment (`r - l + 1`) |

**Merge of left and right children**:
The tricky part — when the rightmost char of `left` equals the leftmost char of `right`, the two boundary runs merge:
```
merged_run = left.rpref + right.lpref
best = max(left.best, right.best, merged_run)
```

Also update `lpref` and `rpref`:
- If `left` is entirely one char (`left.lpref == left.length`): `lpref = left.length + right.lpref`
- If `right` is entirely one char (`right.rpref == right.length`): `rpref = right.length + left.rpref`

**Point update**: update position `pos` in O(log n), propagate merges upward.

**Query**: root's `best` = answer after every update.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    record Node(char Lc, char Rc, int Lpref, int Rpref, int Best, int Length);

    Node[] tree;
    int n;

    Node MakeLeaf(char c) => new(c, c, 1, 1, 1, 1);

    Node Merge(Node L, Node R) {
        char lc = L.Lc, rc = R.Rc;
        int lpref = L.Lpref, rpref = R.Rpref;
        int length = L.Length + R.Length;
        int best = Math.Max(L.Best, R.Best);

        if (L.Rc == R.Lc) {
            int merged = L.Rpref + R.Lpref;
            best = Math.Max(best, merged);
            if (L.Lpref == L.Length) lpref = L.Length + R.Lpref;
            if (R.Rpref == R.Length) rpref = R.Length + L.Rpref;
        }

        return new(lc, rc, lpref, rpref, best, length);
    }

    void Build(char[] s, int node, int l, int r) {
        if (l == r) { tree[node] = MakeLeaf(s[l]); return; }
        int mid = (l + r) / 2;
        Build(s, 2*node, l, mid);
        Build(s, 2*node+1, mid+1, r);
        tree[node] = Merge(tree[2*node], tree[2*node+1]);
    }

    void Update(int node, int l, int r, int pos, char c) {
        if (l == r) { tree[node] = MakeLeaf(c); return; }
        int mid = (l + r) / 2;
        if (pos <= mid) Update(2*node, l, mid, pos, c);
        else            Update(2*node+1, mid+1, r, pos, c);
        tree[node] = Merge(tree[2*node], tree[2*node+1]);
    }

    public int[] LongestRepeating(string s, string queryCharacters, int[] queryIndices) {
        n = s.Length;
        tree = new Node[4 * n];
        Build(s.ToCharArray(), 1, 0, n-1);

        int k = queryIndices.Length;
        int[] res = new int[k];
        for (int i = 0; i < k; i++) {
            Update(1, 0, n-1, queryIndices[i], queryCharacters[i]);
            res[i] = tree[1].Best;
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Build** | **O(n)** | One pass over all nodes |
| **Update** | **O(log n)** | Path from leaf to root |
| **Query** | **O(1)** | Root's `best` field |
| **Total** | **O((n + k) log n)** | k queries each O(log n) |
| **Space** | **O(n)** | Segment tree size 4n |

---

## Merge Logic — The Critical Detail

The merge of two adjacent segments handles three cases for `lpref`, `rpref`, and `best`:

```
if left.rc == right.lc:
    best = max(left.best, right.best, left.rpref + right.lpref)
    
    // Can left prefix extend into right?
    if left is entirely one char (lpref == length):
        lpref = left.length + right.lpref
    
    // Can right suffix extend into left?
    if right is entirely one char (rpref == length):
        rpref = right.length + left.rpref
else:
    best = max(left.best, right.best)
    lpref = left.lpref  // unchanged
    rpref = right.rpref // unchanged
```

Without the boundary merge logic, overlapping runs at segment boundaries would be missed.

## Pitfalls & Edge Cases

- **`lpref`/`rpref` extension**: ONLY update if the same-side child is entirely one char. Otherwise the prefix/suffix stops at the boundary — don't extend.
- **Single character string**: leaf node has `lpref = rpref = best = length = 1`. ✓
- **Same char throughout entire string**: `best == length`. Works naturally via prefix extension propagation.
- **No change in character**: still runs the full O(log n) update path — safe but slightly wasteful. Could skip if `s[pos] == c` but not necessary for correctness.

---

## Key Takeaway

"Max run of repeating character with point updates" → **segment tree storing (lc, rc, lpref, rpref, best, length) per node**:

```
merge: if boundary chars match → best = max(best, left.rpref + right.lpref)
       extend lpref/rpref when child segment is uniform
```

O(log n) per update. The 6-field node is the standard approach for "max homogeneous substring" problems on segment trees.
