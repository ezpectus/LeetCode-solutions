# 3277. Maximum XOR Score Subarray Queries
## O(n²) Preprocessing + O(1) per Query — Two Interlocking 2D DPs

---

## Problem Statement

The **XOR score** of array `a` is computed by repeatedly XOR-ing adjacent elements and removing the last until one remains. For each query `[l, r]`, find the maximum XOR score of any subarray of `nums[l..r]`.

---

## Core Insight #1 — XOR Score Has a Closed Form via Pascal's Triangle mod 2

After repeated operations on a[l..r] (length `m = r-l+1`), the surviving element equals:

```
XOR_score(l, r) = XOR of nums[l+i]  for all i ∈ [0, m-1]  where C(m-1, i) ≡ 1 (mod 2)
```

By Lucas' theorem, `C(m-1, i) ≡ 1 (mod 2)` iff the bits of `i` are a **subset** of the bits of `m-1`.

**Verification (length 3, m-1=2=0b10):** only i=0 (0b00) and i=2 (0b10) satisfy the condition → score = a[0] XOR a[2]. ✓

---

## Core Insight #2 — Recurrence: score(l, r) = score(l, r-1) XOR score(l+1, r)

**Proof via Pascal's identity mod 2:**

```
score(l, r-1) = XOR over i of a[l+i] with coefficient C(r-l-1, i) mod 2
score(l+1, r) = XOR over j of a[l+j] with coefficient C(r-l-1, j-1) mod 2

score(l, r-1) XOR score(l+1, r) = XOR over j of a[l+j] with coefficient
    [C(r-l-1, j) + C(r-l-1, j-1)] mod 2 = C(r-l, j) mod 2 = score(l, r). ✓
```

This gives an **O(n²)** DP to precompute all XOR scores.

---

## Core Insight #3 — Max-Score DP via Decomposition

Define `maxDP[l][r]` = max XOR score of any subarray of `nums[l..r]`.

Every subarray `nums[l'..r']` with `l ≤ l' ≤ r' ≤ r` falls into exactly one category:
- **r' < r:** subarray of `nums[l..r-1]` → covered by `maxDP[l][r-1]`
- **l' > l:** subarray of `nums[l+1..r]` → covered by `maxDP[l+1][r]`
- **l' = l and r' = r:** the full subarray → gives `dp[l][r]`

```
maxDP[l][r] = max(maxDP[l][r-1], maxDP[l+1][r], dp[l][r])
```

These three cases partition all subarrays. ✓

## Full Implementation (C++)

```cpp
class Solution {
public:
    vector<int> maximumSubarrayXor(vector<int>& nums, vector<vector<int>>& queries) {
        int n = nums.size();

        // dp[l][r]    = XOR score of nums[l..r]
        // maxDp[l][r] = max XOR score of any subarray of nums[l..r]
        vector<vector<int>> dp(n, vector<int>(n, 0));
        vector<vector<int>> maxDp(n, vector<int>(n, 0));

        // Base case: single elements
        for (int i = 0; i < n; i++) {
            dp[i][i] = nums[i];
            maxDp[i][i] = nums[i];
        }

        // Fill by increasing subarray length
        for (int len = 2; len <= n; len++) {
            for (int l = 0; l + len - 1 < n; l++) {
                int r = l + len - 1;
                dp[l][r] = dp[l][r - 1] ^ dp[l + 1][r];
                maxDp[l][r] = max({dp[l][r], maxDp[l][r - 1], maxDp[l + 1][r]});
            }
        }

        // Answer each query in O(1)
        vector<int> ans(queries.size());
        for (int i = 0; i < (int)queries.size(); i++)
            ans[i] = maxDp[queries[i][0]][queries[i][1]];

        return ans;
    }
};
```

## Worked Example

### Example 1 — `nums = [2, 8, 4, 32, 16, 1]`

**Length-1 dp:** 2, 8, 4, 32, 16, 1.

**Length-2 dp (a XOR b):**
```
dp[0][1]=10, dp[1][2]=12, dp[2][3]=36, dp[3][4]=48, dp[4][5]=17
```

**Length-3 dp = dp[l][r-1] XOR dp[l+1][r]:**
```
dp[0][2] = 10^12 = 6     maxDp[0][2] = max(6, 10, 12) = 12
dp[1][3] = 12^36 = 40    maxDp[1][3] = max(40, 12, 36) = 40
dp[2][4] = 36^48 = 20    maxDp[2][4] = max(20, 36, 48) = 48
dp[3][5] = 48^17 = 33    maxDp[3][5] = max(33, 48, 17) = 48
```

**Length-4:**
```
dp[1][4] = 40^20 = 60    maxDp[1][4] = max(60, 40, 48) = 60 ← query [1,4]
dp[0][3] = 6^40  = 46    maxDp[0][3] = max(46, 12, 40) = 46
```

**Length-5:**
```
dp[0][4] = 46^60 = 18    maxDp[0][4] = max(18, 46, 60) = 60
dp[1][5] = 60^49 = 13    maxDp[1][5] = max(13, 60, 49) = 60
  where dp[2][5] = 20^33 = 49, maxDp[2][5] = max(49,48,48) = 49
```

**Length-6:**
```
dp[0][5] = 18^13 = 31    maxDp[0][5] = max(31, 60, 60) = 60 ← query [0,5]
```

**Query answers:** [0,2]→12, [1,4]→60, [0,5]→60 ✓

## Closed-Form Verification (Example 1)

Subarray [8,4,32,16] = nums[1..4], m-1 = 3 = 0b11. All bits of i ∈ {0,1,2,3} are subsets of 0b11, so all 4 elements contribute:
```
XOR_score = 8 XOR 4 XOR 32 XOR 16 = 60 ✓
```

Subarray [2,8,4] = nums[0..2], m-1 = 2 = 0b10. Only i=0 (0b00) and i=2 (0b10) qualify:
```
XOR_score = 2 XOR 4 = 6 ✓
```

## Pitfalls & Edge Cases

- **The recurrence `dp[l][r] = dp[l][r-1] XOR dp[l+1][r]` requires l+1 ≤ r-1 not to be out of bounds.** When `r = l+1`, both `dp[l][r-1] = dp[l][l]` and `dp[l+1][r] = dp[l+1][l+1]` are single elements — valid base cases. ✓

- **maxDP needs both subproblems AND the full subarray.** A common mistake is `maxDp[l][r] = max(maxDp[l][r-1], maxDp[l+1][r])` without including `dp[l][r]`. But the full subarray itself might score higher than any strict sub-subarray. Always include `dp[l][r]` in the max.

- **Order of DP filling: by increasing length.** When computing `dp[l][r]`, we need `dp[l][r-1]` (shorter, same l) and `dp[l+1][r]` (shorter, same r) — both already computed. ✓

- **n=1:** Only dp[0][0] and maxDp[0][0] exist; every query must be [0,0] (since l ≤ r ≤ n-1 = 0). Returns nums[0]. ✓

- **Memory layout for cache efficiency:** Using `vector<vector<int>>` with row = l, col = r. Accessing `dp[l][r-1]` and `dp[l+1][r]` is cache-friendly for the inner loop over l.

---

## Key Takeaway

This is a **two-DP interlock** problem: one DP (`dp[l][r]`) computes the exact XOR score of each subarray using a Pascal-like recurrence, and a second DP (`maxDp[l][r]`) accumulates the maximum over all nested subarrays using the standard "any subarray of [l..r] is a subarray of [l..r-1] or [l+1..r] or is [l..r] itself" decomposition. The two DPs fill in the same order (by increasing length) and answer all queries in O(1).

**Pattern recognition:** "For each query range [l,r], find the maximum of some function over all sub-ranges" → if the function has a 2D recurrence and the max has the "nested subarray" decomposition, both can be precomputed in O(n²) and answered in O(1).
