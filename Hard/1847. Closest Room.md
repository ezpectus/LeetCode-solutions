# 1847. Closest Room
## O((n + k) log n) — Offline Queries + Sorted Set Binary Search

---

## Problem Statement

- Find for each query `[preferred, minSize]` the room ID closest to `preferred` among rooms with `size >= minSize`. Ties → smallest ID. No valid room → -1.

---

## Core Idea — Offline: Sort by minSize, Add Rooms Incrementally

**Key observation**: if we process queries in decreasing order of `minSize`, we can add rooms to an available set incrementally — rooms once added stay valid for all subsequent (smaller minSize) queries.

**Algorithm**:
1. Sort rooms by size descending.
2. Sort queries by minSize descending (keep original indices).
3. For each query: add all rooms with `size >= minSize` to a sorted set of room IDs.
4. Binary search in the sorted set for the closest ID to `preferred` — check the neighbors at `bisect_left(preferred)` positions ±1.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] ClosestRoom(int[][] rooms, int[][] queries) {
        int k = queries.Length;
        int[] ans = new int[k];
        Array.Fill(ans, -1);

        Array.Sort(rooms, (a, b) => b[1] - a[1]);
        int[] qOrder = Enumerable.Range(0, k).OrderByDescending(i => queries[i][1]).ToArray();

        var available = new SortedList<int, int>(); // roomId -> roomId
        int ri = 0;

        int LowerBound(int val) {
            int lo = 0, hi = available.Count;
            while (lo < hi) { int mid=(lo+hi)/2; if(available.Keys[mid]<val)lo=mid+1; else hi=mid; }
            return lo;
        }

        foreach (int qi in qOrder) {
            int preferred = queries[qi][0], minSize = queries[qi][1];
            while (ri < rooms.Length && rooms[ri][1] >= minSize)
                available[rooms[ri][0]] = rooms[ri++][0];

            if (available.Count == 0) continue;

            int best = -1, bestDiff = int.MaxValue;
            int pos = LowerBound(preferred);

            foreach (int p in new[]{pos-1, pos}) {
                if (p >= 0 && p < available.Count) {
                    int rid = available.Keys[p];
                    int diff = Math.Abs(rid - preferred);
                    if (diff < bestDiff || (diff == bestDiff && rid < best)) {
                        bestDiff = diff; best = rid;
                    }
                }
            }
            ans[qi] = best;
        }
        return ans;
    }
}
```

## Example Walkthrough

**`rooms=[[1,4],[2,3],[3,5],[4,1],[5,2]], queries=[[2,3],[2,4],[2,5]]`**

Sorted queries by minSize desc: [2,5]→qi=2, [2,4]→qi=1, [2,3]→qi=0.

- qi=2, minSize=5: add rooms with size≥5 → {3}. preferred=2. floor=∅, ceiling={3}. best=3.
- qi=1, minSize=4: add size≥4 → {1,3}. preferred=2. floor={1}(diff=1), ceiling={3}(diff=1). Tie → smaller=1. best=1.
- qi=0, minSize=3: add size≥3 → {1,2,3}. preferred=2. ceiling={2}(diff=0). best=2.

ans=[2,1,3] ✓

## Pitfalls & Edge Cases

- **Tie-breaking**: when two IDs equidistant from `preferred`, pick the smaller one — check both neighbors and apply tie-break.
- **`GetViewBetween`**: C#'s `SortedSet` supports this natively; avoids manual binary search.
- **Offline ordering**: queries must be processed in decreasing minSize order so rooms are added exactly once.

---

## Key Takeaway

"Range-filtered closest ID queries" → **offline sort by threshold + sorted set + binary search neighbors**:

```
sort rooms by size desc, queries by minSize desc
for each query: add eligible rooms → binary search for closest ID
```

O((n+k) log n). Offline processing converts a 2D filter into a 1D sweep.
