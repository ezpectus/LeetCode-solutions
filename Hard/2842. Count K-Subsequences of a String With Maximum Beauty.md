## 2842. Count K-Subsequences of a String With Maximum Beauty

### Problem Statement
Given a string `s` and an integer `k`, a **k-subsequence** is a subsequence of length `k` where all characters are unique. Let $f(c)$ be the frequency of character $c$ in the original string `s`. The **beauty** of a k-subsequence is the sum of $f(c)$ for every unique character $c$ in it.

Return the total number of distinct k-subsequences (based on character indices) that achieve the maximum possible beauty, modulo $10^9 + 7$. If no valid k-subsequence of length `k` can be formed, return `0`.

### Core Insight — Greedy Sorting & Combinatorics
Since all characters in a k-subsequence must be unique, a valid k-subsequence can contain at most 26 characters. If $k > \text{number of distinct characters in } s$, it is impossible to choose $k$ unique characters, so we immediately return `0`.

To maximize the total beauty sum, we must pick characters with the highest frequencies. This maps directly to a greedy selection process:
1. Count the frequency $f(c)$ for all characters and sort them in descending order.
2. We must select the top $k$ highest frequencies. 
3. Let the $k$-th highest frequency be $V$. Some chosen frequencies will be strictly greater than $V$, some will be exactly equal to $V$, and some remaining characters not chosen will also have a frequency equal to $V$.

This structural division creates three distinct categories among the unique characters:
* **Strictly Greater ($f(c) > V$):** These characters *must* be included to maximize beauty. If there are $M$ such characters, we take all of them. For each character, we can pick any of its $f(c)$ occurrences in `s`.
* **Exactly Equal ($f(c) == V$):** Let $W$ be the total number of characters in the entire string that have a frequency exactly equal to $V$. We only need to pick a remaining subset of them, say $R = k - M$ characters, to fulfill our requirement of $k$ unique characters.
* **Strictly Less ($f(c) < V$):** These characters are ignored completely.

#### Combinatoric Formulation
- For each character chosen whose frequency is strictly greater than $V$, we multiply our total combinations by its frequency $f(c)$.
- For the characters whose frequency is exactly equal to $V$, we must first select *which* $R$ characters out of the available $W$ characters to include. This is given by the combination formula $\binom{W}{R}$.
- Since each of these $R$ chosen characters has exactly $V$ occurrences in `s`, each provides $V$ independent structural options. This contributes a factor of $V^R$ to our ways.

Thus, the total number of maximum beauty combinations is:
$$\left( \prod_{f(c) > V} f(c) \right) \times \binom{W}{R} \times V^R \pmod{10^9 + 7}$$

---

### Algorithm
1. Collect the frequencies of all 26 lowercase English characters from `s`.
2. If the number of distinct characters with a frequency $> 0$ is less than `k`, return `0`.
3. Group the frequencies or sort them in descending order.
4. Identify the target cutoff frequency $V = \text{frequencies}[k-1]$.
5. Count how many total characters have a frequency equal to $V$ (this is $W$), and how many characters have a frequency strictly greater than $V$ (this is $M$).
6. Calculate $R = k - M$.
7. Compute the answer using modular arithmetic:
   - Multiply the frequencies of all characters where $f(c) > V$.
   - Multiply by $\binom{W}{R} \pmod{10^9+7}$.
   - Multiply by $V^R \pmod{10^9+7}$.
8. Return the result.

### Full Optimal Implementation (C#)
```csharp
using System;
using System.Collections.Generic;
using System.Linq;

public class Solution {
    private const long MOD = 1_000_000_007;

    public int CountKSubsequencesWithMaxBeauty(string s, int k) {
        // Step 1: Compute frequencies of characters
        int[] freq = new int[26];
        foreach (char c in s) {
            freq[c - 'a']++;
        }

        // Extract non-zero frequencies
        List<int> counts = new List<int>();
        foreach (int f in freq) {
            if (f > 0) counts.Add(f);
        }

        // Step 2: Guard check for validity
        if (counts.Count < k) {
            return 0;
        }

        // Step 3: Sort frequencies descending to isolate the top k elements
        counts.Sort((a, b) => b.CompareTo(a));

        int targetV = counts[k - 1];
        long ans = 1;
        int totalW = 0; // Total elements with frequency == targetV
        int selectedM = 0; // Total elements with frequency > targetV

        foreach (int f in counts) {
            if (f > targetV) {
                ans = (ans * f) % MOD;
                selectedM++;
            } else if (f == targetV) {
                totalW++;
            }
        }

        int remR = k - selectedM;

        // Step 4: Compute combinations C(totalW, remR)
        long nCr = GetCombinations(totalW, remR);

        // Step 5: Compute targetV^remR % MOD
        long powerV = Power(targetV, remR);

        // Combine all independent calculation factors
        ans = (ans * nCr) % MOD;
        ans = (ans * powerV) % MOD;

        return (int)ans;
    }

    private long GetCombinations(int n, int r) {
        if (r > n || r < 0) return 0;
        if (r == 0 || r == n) return 1;
        
        // Optimizing calculation size
        if (r > n - r) r = n - r;

        long num = 1;
        long den = 1;

        for (int i = 1; i <= r; i++) {
            num = (num * (n - i + 1)) % MOD;
            den = (den * i) % MOD;
        }

        return (num * ModularInverse(den)) % MOD;
    }

    private long Power(long baseVal, long exp) {
        long res = 1;
        baseVal %= MOD;
        while (exp > 0) {
            if ((exp & 1) == 1) res = (res * baseVal) % MOD;
            baseVal = (baseVal * baseVal) % MOD;
            exp >>= 1;
        }
        return res;
    }

    private long ModularInverse(long n) {
        return Power(n, MOD - 2); // Fermat's Little Theorem
    }
}
```

### Example Walkthrough
#### `s = "bcca"`, `k = 2`
* **Frequencies:** $f('c') = 2$, $f('b') = 1$, $f('a') = 1$. Total unique characters = 3.
* **Sorted Counts:** `[2, 1, 1]`
* **Target Isolation ($k = 2$):** The value at index $k-1$ (index 1) is $V = 1$.
* **Grouping Categories:**
  - Elements $> 1$: `[2]` (Character 'c', $M = 1$)
  - Elements $== 1$: `[1, 1]` (Characters 'b' and 'a', $W = 2$)
* **Factor Accumulation:**
  - From $f(c) > V$: `ans` begins at $2$ (from $f('c') = 2$).
  - Remaining count needed: $R = k - M = 2 - 1 = 1$.
  - Combinations choosing 1 from 2 options: $\binom{2}{1} = 2$.
  - Frequency variations from choices: $V^R = 1^1 = 1$.
* **Final Reduction:** $\text{ans} = 2 \times 2 \times 1 = 4$.

**Total Output:** `4`. ✓

### Pitfalls & Edge Cases
* **Insufficient Unique Elements:** When $k$ exceeds the total pool of unique alphabet characters present in `s`, no combination can satisfy the rule of having all distinct elements. Returning `0` early avoids processing out-of-bounds array indices.
* **Modular Division Errors:** When dividing by a value under a modulo condition (such as evaluating the denominator of combinations $\binom{W}{R}$), simple integer division `/` fails. You must compute the **Modular Multiplicative Inverse** using Fermat's Little Theorem ($\text{denominator}^{\text{MOD}-2} \pmod{\text{MOD}}$).
* **Intermediate Multiplicative Overflow:** While multiplying values up to $10^9+7$, products can reach up to $10^{18}$, which exceeds 32-bit `int` limits. Cast values to 64-bit `long` configurations before applying multiplication steps to avoid sign truncation bugs.

### Key Takeaway
When looking for maximum unique character patterns, sort the underlying frequency metrics to turn the problem into a clear selection challenge. Characters above the cutoff line are required, while characters on the cutoff boundary require combinatorial selection ($\binom{W}{R}$) and power calculations ($V^R$). This avoids expensive subsequence backtracking simulations and simplifies the solution into constant time operations.
