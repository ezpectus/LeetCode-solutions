# LeetCode 3695. Maximize Alternating Sum Using Swaps

## 1. Constraints & Complexity Bottleneck Analysis

### Constraints:
- **Array Length (N):** 2 <= nums.length <= 10^5
  - An O(N^2) solution (such as brute force swaps or Floyd-Warshall connectivity checks) will result in Time Limit Exceeded (TLE). We need an O(N log N) or O(N) solution.
- **Value of Elements (nums[i]):** 1 <= nums[i] <= 10^9
  - Elements are large. The sum can exceed the limits of a standard 32-bit signed integer (up to 10^14). We must use 64-bit integers (`long long` in C++) to prevent integer overflow during summation.
- **Number of Swaps (M):** 0 <= swaps.length <= 10^5
  - The number of allowed swaps is large, so we must find connected components efficiently in nearly linear time.

### The Complexity Bottleneck:
If two indices can be swapped, they belong to the same equivalence class. Any transitive path of swaps (e.g., swapping a with b, then b with c) allows us to move any element within that connected component to any index in that same component. 
The bottleneck is:
1. Grouping the indices into connected components in near-linear time.
2. For each component, identifying how many even positions are available inside it.
3. Extracting and sorting the elements in each component to assign the largest ones to the even indices.

## 2. Algorithm Design & Mathematical Invariants

### 2.1 Core Concept
We can model the indices of the array as vertices in a graph, and the allowable swaps as undirected edges. This splits the array indices into several disconnected components.
We use a Disjoint Set Union (DSU) data structure to group indices.
For any connected component:
- Let the total number of elements in this component be `C`.
- Let `E` be the count of indices in this component that are even.
- Since we can permute the elements of a component in any order, we can choose exactly which `E` elements of this component will land on even indices (and thus be added to the alternating sum), and which `C - E` elements will land on odd indices (and thus be subtracted).

To maximize the alternating sum, we must greedily assign the largest `E` elements of the component to the even positions, and the remaining smaller elements to the odd positions.

### 2.2 Key Invariants & Contribution Formula
Let the sorted elements of a component in ascending order be `v[0], v[1], ..., v[C-1]`.
The optimal strategy places the largest `E` elements, which are `v[C-E], ..., v[C-1]`, at the even positions.

The contribution of this component to the total alternating sum is:
`Contribution = Sum of elements at even positions - Sum of elements at odd positions`

We can rewrite this mathematically to compute it in O(1) after sorting:
`Sum of all elements = Sum of even elements + Sum of odd elements`
`Sum of odd elements = Sum of all elements - Sum of even elements`

Substituting this back into the Contribution formula:
`Contribution = Sum of even elements - (Sum of all elements - Sum of even elements)`
`Contribution = 2 * (Sum of even elements) - Sum of all elements`

Thus, we only need to sum all elements in the component, and add twice the sum of the largest `E` elements to our answer. This invariant holds independently for each connected component.

## 3. Full Optimal Implementation (C++)
```
// Union-Find + Greedy + Sorting
#include <vector>
#include <numeric>
#include <algorithm>

using namespace std;

class DSU {
private:
    vector<int> parent;
public:
    DSU(int n) {
        parent.resize(n);
        iota(parent.begin(), parent.end(), 0);
    }

    int find(int i) {
        if (parent[i] == i)
            return i;
        return parent[i] = find(parent[i]); // Path compression
    }

    void unite(int i, int j) {
        int root_i = find(i);
        int root_j = find(j);
        if (root_i != root_j) {
            parent[root_i] = root_j;
        }
    }
};

class Solution {
public:
    long long maxAlternatingSum(vector<int>& nums, vector<vector<int>>& swaps) {
        int n = nums.size();
        DSU dsu(n);

        // Step 1: Group index positions into connected components
        for (const auto& swap_pair : swaps) {
            dsu.unite(swap_pair[0], swap_pair[1]);
        }

        // Step 2: Organize elements and track available even index counts per component
        vector<vector<int>> comp_elements(n);
        vector<int> comp_even_count(n, 0);

        for (int i = 0; i < n; ++i) {
            int root = dsu.find(i);
            comp_elements[root].push_back(nums[i]);
            if (i % 2 == 0) {
                comp_even_count[root]++;
            }
        }

        // Step 3: Compute the maximum contribution of each component
        long long total_alternating_sum = 0;

        for (int i = 0; i < n; ++i) {
            if (comp_elements[i].empty()) {
                continue;
            }

            // Sort elements of the component in ascending order
            sort(comp_elements[i].begin(), comp_elements[i].end());

            int total_size = comp_elements[i].size();
            int even_slots = comp_even_count[i];

            long long sum_all = 0;
            long long sum_top_even = 0;

            for (int j = 0; j < total_size; ++j) {
                sum_all += comp_elements[i][j];
                // The largest 'even_slots' elements are located at the end of the sorted array
                if (j >= total_size - even_slots) {
                    sum_top_even += comp_elements[i][j];
                }
            }

            total_alternating_sum += (2 * sum_top_even - sum_all);
        }

        return total_alternating_sum;
    }
};
```

## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The overall execution time is the sum of the following sequential phases:

1. **DSU Initialization & Unions:**
   - Initializing parent array: O(N).
   - Performing M unite operations: O(M * alpha(N)), where alpha is the inverse Ackermann function (effectively constant).
   - `T_step1 = O(N + M * alpha(N))`

2. **Component Organization:**
   - Finding root for all N indices and grouping elements/even-counts: O(N * alpha(N)).
   - `T_step2 = O(N * alpha(N))`

3. **Sorting & Summation:**
   - For each component `i` of size `S_i`, we sort its elements which takes O(S_i * log(S_i)) time.
   - Sum of sizes of all components: Sum(S_i) = N.
   - The overall sorting time is bounded by the worst-case single component:
     `T_sort = O(N * log(N))`
   - Accumulating sums within each component takes O(S_i), summing to O(N) overall.
   - `T_step3 = O(N * log(N))`

**Total Time Complexity:** O(N * log(N) + M * alpha(N))
Given N, M <= 10^5, this performs roughly 1.7 * 10^6 operations in the sorting stage, which easily completes within 10-15ms.

### 4.2 Detailed Space Complexity Decomposition
- **DSU Parent Array:** Requires O(N) space.
- **Component Structures:** 
  - `comp_elements` stores all N elements across components, requiring O(N) space.
  - `comp_even_count` requires O(N) space.

**Total Space Complexity:** O(N) auxiliary space.

### 4.3 Complexity Summary
| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(N * log(N) + M * alpha(N)) | Dominated by sorting component arrays of elements. |
| **Space Complexity** | O(N) | Required to store the DSU state and partitioned sub-arrays. |

## 5. Mathematical Formulations & Foundational Algorithms

### Transitive Property of Swaps
We model swaps as equivalence relations. If we can swap index `a` with index `b` and index `b` with index `c`, then the set of reachable configurations is equivalent to any permutation of elements on positions `{a, b, c}`. 
Using Disjoint Set Union with Path Compression allows us to identify the equivalence classes (components) in nearly linear time.

### Greedy Partitioning Formula
Let the multiset of elements in a component be S. We want to partition S into two subsets S_even (size E) and S_odd (size C - E) to maximize:
`Value = Sum(x in S_even) - Sum(y in S_odd)`

Since `Sum(y in S_odd) = Sum(z in S) - Sum(x in S_even)`, the expression simplifies to:
`Value = 2 * Sum(x in S_even) - Sum(z in S)`

Because `Sum(z in S)` is a constant for a given component, maximizing `Value` is mathematically equivalent to maximizing `Sum(x in S_even)`. To maximize the sum of a subset of size E from a set, we must choose the E largest elements. This proves that sorting the component elements and taking the largest ones yields the mathematically optimal alternating sum.

## Pitfalls & Edge Cases
- **64-bit Overflow:** The final alternating sum can exceed the limit of standard 32-bit signed integers because elements are up to 10^9 and N <= 10^5. Using `long long` for `sum_all`, `sum_top_even`, and `total_alternating_sum` prevents overflow.
- **Empty Component Representation:** Iterating over components from index 0 to N - 1 includes indices that are not roots. We must skip non-root component arrays using `comp_elements[i].empty()` checks to prevent redundant work.
- **Components with No Even Indices:** If a component contains only odd indices, `even_slots` will be 0. The code correctly handles this: `sum_top_even` will be 0, and the contribution is `-sum_all` (all elements in the component are subtracted), which is mathematically correct.

---

## Key Takeaway
Whenever index swaps are allowed infinitely, convert the indices into a graph of connected components using DSU. The problem then simplifies to independent optimal greedy assignments within each separate component.

**Pattern recognition:** "allowed to swap elements at indices any number of times" -> Union-Find + Component Grouping + Greedy Sorting
