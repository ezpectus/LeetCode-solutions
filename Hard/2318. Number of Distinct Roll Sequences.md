# 2318. Number of Distinct Roll Sequences
## O(n × 6²) — DP on (last, prev) State

---

## Problem Statement

- Roll a 6-sided die `n` times. Valid sequence: adjacent values coprime AND no equal value within 2 positions. Count sequences mod 10⁹+7.

---

## Core Idea — DP with Last Two Rolls as State

**State**: `dp[last][prev]` = number of valid sequences of current length ending with value `last`, with the roll before that being `prev` (0 = no previous roll).

**Why track `prev`?** The "no equal value within 2 rolls" rule means `next ≠ last` AND `next ≠ prev` (can't repeat either of the last two).

**Transition**: for each `next` from 1 to 6:
- `next ≠ last` (not the immediate previous)
- `next ≠ prev` (not two positions back)
- `gcd(next, last) == 1` (adjacent must be coprime)

**Initialization** (n=2): `dp[b][a] = 1` for all valid pairs (a,b).

**Answer**: sum of all `dp[last][prev]` after n-1 transitions.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int DistinctSequences(int n) {
        const int MOD = 1_000_000_007;
        int Gcd(int a, int b) => b == 0 ? a : Gcd(b, a%b);

        if (n == 1) return 6;

        long[,] dp = new long[7, 7];
        for (int a = 1; a <= 6; a++)
            for (int b = 1; b <= 6; b++)
                if (a != b && Gcd(a, b) == 1) dp[b, a] = 1;

        for (int step = 2; step < n; step++) {
            long[,] ndp = new long[7, 7];
            for (int last = 1; last <= 6; last++)
            for (int prev = 0; prev <= 6; prev++) {
                if (dp[last, prev] == 0) continue;
                for (int next = 1; next <= 6; next++) {
                    if (next == last || next == prev) continue;
                    if (Gcd(next, last) != 1) continue;
                    ndp[next, last] = (ndp[next, last] + dp[last, prev]) % MOD;
                }
            }
            dp = ndp;
        }

        long res = 0;
        for (int i = 1; i <= 6; i++)
            for (int j = 0; j <= 6; j++) res = (res + dp[i, j]) % MOD;
        return (int)res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × 6³)** = **O(n)** | n steps × 6² states × 6 transitions; 6³ is constant |
| **Space** | **O(6²)** = **O(1)** | Rolling DP, 7×7 array |

With n=10⁴: 10⁴ × 216 = 2.16M operations — fast.

---

## GCD Precomputation Optimization

Since dice values are 1-6, there are only 36 pairs. Precompute `coprime[a][b]` once:

```csharp
bool[,] coprime = new bool[7,7];
for (int a=1;a<=6;a++) for (int b=1;b<=6;b++) coprime[a,b] = Gcd(a,b)==1;
```

---

## Example Walkthrough

**n=2**: Initialize pairs where a≠b and gcd=1.
Invalid due to gcd: (2,4),(2,6),(4,6),(3,6) and reverses = 8 pairs.
Invalid due to equal: 6 pairs (i,i).
Valid = 36-6-8 = 22 pairs ✓.

---

## Pitfalls & Edge Cases

- **`prev=0` (no previous roll)** for length-2 initialization: only `last ≠ prev` constraint — but `prev=0` is never a valid die value, so `next ≠ 0` is always true. The constraint naturally works.
- **`n=1`**: return 6 directly (all single rolls valid, no adjacency constraints).
- **Coprimality is symmetric**: `gcd(a,b)==1 iff gcd(b,a)==1` — consistent when rolling forward.

---

## Key Takeaway

"Count valid die roll sequences with coprimality + distance constraints" → **DP(last, prev): track last two rolls. Three conditions for next: ≠last, ≠prev, coprime with last**. O(n × 6³) = O(n).
