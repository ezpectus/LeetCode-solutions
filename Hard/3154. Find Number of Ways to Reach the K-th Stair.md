## 3154. Find Number of Ways to Reach the K-th Stair

### Problem Statement
You are given a non-negative integer `k`. There exists a staircase with an infinite number of stairs, with the lowest stair numbered 0. You start on stair 1 and want to reach stair `k` using any number of operations. 

In one operation, you can:
* **Go down:** Move from stair `i` to `i - 1`. This operation **cannot** be used consecutively or on stair 0.
* **Go up:** Move from stair `i` to `i + 2^jump`. Then, `jump` becomes `jump + 1`. (Initially, `jump = 0`).

Return the total number of ways to reach stair `k`.

---

### Core Insight — Combinatorics & Pascal's Triangle
Let's analyze what happens if we perform exactly `j` up-steps. The total distance covered by jumping up will be:
`1 + 2^0 + 2^1 + 2^2 + ... + 2^(j-1) = 1 + (2^j - 1) = 2^j`

To land exactly on stair `k`, we must use down-steps to adjust our position. Let `d` be the number of down-steps. The final landing position is given by:
`2^j - d = k  =>  d = 2^j - k`

**Constraints on down-steps:**
1. Since we cannot perform two down-steps consecutively, the `d` down-steps must be placed in the slots created by the `j` up-steps. There are exactly `j + 1` available slots (before the first jump, between jumps, or after the last jump).
2. Therefore, a configuration is valid if and only if `0 <= d <= j + 1`.

Since `k <= 10^9`, and `2^30 > 10^9`, we only need to iterate through possible values of `j` from 0 to 30. For each valid `j`, the number of ways to distribute `d` identical down-steps into `j + 1` distinct slots is given by the combinations formula `C(j + 1, d)`, which corresponds directly to values in **Pascal's Triangle**.

---

### Algorithm
1. Precompute Pascal's Triangle combinations `C[n][k]` up to `n = 32`.
2. Initialize `totalWays = 0`.
3. Loop through the number of up-steps `j` from 0 to 30:
   * Calculate the required down-steps: `d = (1 << j) - k`.
   * Check if `d` satisfies the constraints: `d >= 0` and `d <= j + 1`.
   * If valid, add `C[j + 1][d]` to `totalWays`.
4. Return `totalWays`.

---

### Full Optimal Implementation (C++)
```cpp
#include <vector>

using namespace std;

class Solution {
public:
    int waysToReachStair(int k) {
        // Step 1: Precompute Pascal's Triangle for combinations C(n, r)
        long long C[33][33];
        for (int i = 0; i <= 32; i++) {
            C[i][0] = 1;
            for (int j = 1; j <= i; j++) {
                C[i][j] = C[i-1][j-1] + C[i-1][j];
            }
        }

        int totalWays = 0;

        // Step 2: Iterate through all possible numbers of up-jumps (j)
        // Since k <= 10^9, 2^30 is enough to cover the upper bound range
        for (int j = 0; j <= 30; j++) {
            long long rooms = (1LL << j);
            long long d = rooms - k; // Required down-steps

            // Step 3: Check if the required down-steps fit within the available slots
            if (d >= 0 && d <= j + 1) {
                totalWays += C[j + 1][d];
            }
        }

        return totalWays;
    }
};
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(1) | Precomputing Pascal's Triangle takes a fixed 32x32 loop iterations. The main search loop runs exactly 31 times. The execution runtime is virtually instantaneous. |
| **Space** | O(1) | The combinations storage grid uses a minimal, statically bounded footprint of 33x33 elements. |

### Pitfalls & Edge Cases
* **Bit Shift Overflow:** When computing `1 << j`, use `long long` syntax (`1LL << j`) to prevent standard 32-bit signed integer overflow limits when `j = 30` or `31`.
* **Negative Down-Steps:** If `2^j < k`, then `d` becomes negative. Ensure the lower boundary safeguard `d >= 0` filters out these mathematically invalid iterations.

### Key Takeaway
When a sequence problem features an operation that grows exponentially alongside a small, restricted step-back operation, look at the system through a global combinatorial lens rather than simulation. Converting the state transitions into a math formula allows you to use Pascal's Triangle to solve an exponential-looking problem in true constant O(1) time.
