## 2920. Maximum Points After Collecting Coins From All Nodes

### Problem Statement
There exists an undirected tree rooted at node 0 with n nodes labeled from 0 to n - 1. You are given a 2D integer array `edges` representing the tree, a 0-indexed array `coins` where `coins[i]` denotes the number of coins at node i, and an integer k.

You must collect coins from all nodes starting from the root and moving downwards. The coins at a node can only be collected if its ancestors have already been collected. For each node i, you can choose one of two strategies:
* **Method 1:** Collect all coins and get `coins[i] - k` points.
* **Method 2:** Collect all coins and get `floor(coins[i] / 2)` points. However, choosing this method applies a permanent penalty: the coins at all nodes in the subtree of node i are halved (`floor(coins[j] / 2)`).

Return the maximum total points achievable.

---

### Core Insight — Bounded Subtree Halving & Dynamic Programming
This problem can be modeled using Dynamic Programming on Trees (Tree DP) with Memoization.

A naive approach might try to track the exact coin states of the entire tree, which is impossible due to exponential combinations. However, look closely at the constraints: `coins[i] <= 10^4`.

When Method 2 is chosen, all coins in the subtree are divided by 2. If an element is divided by 2 repeatedly, it quickly drops to 0. Since 2^14 > 10^4, any coin value will become exactly 0 after being halved at most 14 times. Once a node's coin count is 0, halving it further yields 0, and Method 1 will always yield `0 - k` points.

Therefore, instead of tracking which specific ancestors were halved, we only need to pass down the **total count of times Method 2 was used** along the path from the root to the current node. This count can be safely capped at 14.

#### DP State Definition
Let `Dfs(u, p, halved)` be a function that returns the maximum points obtainable from the subtree rooted at node `u`, given that its parent is `p` and Method 2 has been used `halved` times by its ancestors.

For each node `u`, we calculate the current coin value after ancestor reductions:
`current_coins = coins[u] >> halved`

Then we choose the maximum between the two available options:
* **Option 1 (Use Method 1):**
  points_1 = (current_coins - k) + sum( Dfs(v, u, halved) ) for all children v
* **Option 2 (Use Method 2):**
  points_2 = floor(current_coins / 2) + sum( Dfs(v, u, min(halved + 1, 14)) ) for all children v

---

### Algorithm
1. Build an adjacency list to represent the tree from the given `edges`.
2. Initialize a 2D memoization array `memo` of size `n x 15` filled with `-1`.
3. Start a recursive DFS from the root node `0` with `p = -1` and `halved = 0`.
4. In the DFS function:
   * If `memo[u, halved]` is already computed, return it.
   * Right-shift `coins[u]` by `halved` to get the current coin count.
   * Compute the total points if Method 1 is picked.
   * Compute the total points if Method 2 is picked (passing down `Math.Min(halved + 1, 14)`).
   * Store the maximum of both options in `memo[u, halved]` and return it.
5. The result of `Dfs(0, -1, 0)` is our final answer.

---

### Full Optimal Implementation (C#)
```csharp
using System;
using System.Collections.Generic;

public class Solution {
    private List<int>[] adj;
    private int[] coinValues;
    private int penalty;
    private int[,] memo;

    public int MaximumPoints(int[][] edges, int[] coins, int k) {
        int n = coins.Length;
        coinValues = coins;
        penalty = k;
        
        // 14 times halving reduces maximum coin value 10^4 to 0
        memo = new int[n, 15]; 
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < 15; j++) {
                memo[i, j] = -1;
            }
        }

        // Build adjacency list
        adj = new List<int>[n];
        for (int i = 0; i < n; i++) {
            adj[i] = new List<int>();
        }
        foreach (var edge in edges) {
            adj[edge[0]].Add(edge[1]);
            adj[edge[1]].Add(edge[0]);
        }

        return Dfs(0, -1, 0);
    }

    private int Dfs(int u, int p, int halved) {
        if (memo[u, halved] != -1) {
            return memo[u, halved];
        }

        // Calculate actual coins left at this node after reductions
        int currentCoins = coinValues[u] >> halved;

        // Choice 1: Use Method 1 (coins[u] - k)
        int choice1 = currentCoins - penalty;
        foreach (int v in adj[u]) {
            if (v != p) {
                choice1 += Dfs(v, u, halved);
            }
        }

        // Choice 2: Use Method 2 (floor(coins[u] / 2))
        int choice2 = currentCoins / 2;
        int nextHalved = Math.Min(halved + 1, 14);
        foreach (int v in adj[u]) {
            if (v != p) {
                choice2 += Dfs(v, u, nextHalved);
            }
        }

        return memo[u, halved] = Math.Max(choice1, choice2);
    }
}
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N) | There are N nodes and at most 15 states per node. Each unique state `(u, halved)` is computed exactly once, making the overall time complexity bounded by 15 * N, which simplifies to O(N). |
| **Space** | O(N) | The memoization table requires 15 * N entries. The recursion stack can go up to depth N in the worst case (for a skewed linear tree tree structure). |

### Example Walkthrough
#### `edges = [[0,1],[1,2],[2,3]]`, `coins = [10,10,3,3]`, `k = 5`
* **Node 0 (`coins = 10`, `halved = 0`):**
  - Current coins = 10.
  - Method 1: (10 - 5) + children = 5 + `Dfs(1, 0, 0)`.
  - Method 2: (10 / 2) + children = 5 + `Dfs(1, 0, 1)`.
* Let's trace down the path where Node 0 and Node 1 choose Method 1, and Node 2 chooses Method 2:
  - **Node 1 (`coins = 10`, `halved = 0`):** Current coins = 10. Method 1 gives 10 - 5 = 5. Total point pool is now 10. Moves to Node 2 with `halved = 0`.
  - **Node 2 (`coins = 3`, `halved = 0`):** Current coins = 3. Method 2 gives 3 / 2 = 1. Total points = 11. Moves to Node 3 with `halved = 1`.
  - **Node 3 (`coins = 3`, `halved = 1`):** Current coins = 3 >> 1 = 1. Method 2 gives 1 / 2 = 0. Total points = 11.
* The algorithm will automatically discover that this combination yields the maximum output of 11.

**Total Output:** `11`. ✓

### Pitfalls & Edge Cases
* **Negative Point Balances:** Note that Method 1 (`currentCoins - k`) can result in a negative point value. Do not clip negative values to 0 prematurely using `Math.Max(0, choice1)`, because losing points at one node can be optimal if it prevents a heavy halving penalty for a massive subtree underneath.
* **Graph Bi-directionality:** The edge definitions do not dictate parent-child pairings explicitly. Always pass down the parent index `p` during recursive traversals to avoid cycles and infinite loops.

### Key Takeaway
When structural actions alter full structural segments dynamically (like modifying an entire subtree), check if the transition process converges rapidly to a base termination state (like division reducing values to 0). Bounding the number of active status modifications lets you turn a seemingly chaotic system state into a compact, easily tracking multi-dimensional DP matrix.
