## 3149. Find the Minimum Cost Array Permutation

### Problem Statement
You are given an array `nums` which is a permutation of `[0, 1, 2, ..., n - 1]`. The score of any permutation `perm` of `[0, 1, 2, ..., n - 1]` is defined as:
`score(perm) = |perm[0] - nums[perm[1]]| + |perm[1] - nums[perm[2]]| + ... + |perm[n - 1] - nums[perm[0]]|`

Return the permutation `perm` which has the minimum possible score. If multiple such permutations exist, return the one that is **lexicographically smallest**.

---

### Core Insight — Cyclic Shift Invariance & Bitmask DP
The formula for `score(perm)` forms a complete **cyclic loop**: the last term connects `perm[n - 1]` back to `nums[perm[0]]`. Because the cost expression is cyclic, shifting the elements of any permutation circularly results in the exact same score. 

To guarantee the **lexicographically smallest** permutation, we can lock the very first element of our sequence to `perm[0] = 0`. 

Given the constraint `n <= 14`, we can frame this as a Travelling Salesperson Problem (TSP) solvable via Dynamic Programming with Bitmasking:
* **State Representation:** `Dfs(last_inserted, mask)` calculates the minimum cost to complete the permutation given the last placed value `last_inserted` and a bitmask `mask` representing all currently visited elements.
* **Lexicographical Recovery:** To reconstruct the path, we iterate through candidate next elements from `1` to `n - 1` in increasing order. We select the first element that satisfies the optimal subproblem cost equation.

---

### Algorithm
1. Initialize a 2D memoization table `memo[last_val][mask]` with `-1`.
2. Start the recursive traversal with `perm[0] = 0`. The initial state is `Dfs(0, 1, n, nums)` since node `0` is visited (`mask = 1`).
3. In the `Dfs` function:
   * **Base Case:** If `mask == (1 << n) - 1` (all elements visited), return the final cyclic cost link: `|last_val - nums[0]|`.
   * Try placing every unvisited element `next_val` (where `(mask & (1 << next_val)) == 0`).
   * The transition cost is `|last_val - nums[next_val]| + Dfs(next_val, mask | (1 << next_val), n, nums)`.
   * Track and return the minimum score achieved.
4. **Reconstruct Path:** Start at `last = 0` and `mask = 1`. Iteratively find the next element `i` from `0` to `n - 1` that matches the precomputed optimal cost. Append `i` to the result, update `mask`, set `last = i`, and repeat until all elements are recovered.

---

### Full Optimal Implementation (C#)
```csharp
using System;

public class Solution {
    private int[,] memo;

    public int[] FindPermutation(int[] nums) {
        int n = nums.Length;
        // memo table size: [last_placed_element][visited_mask]
        memo = new int[n, 1 << n];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < (1 << n); j++) {
                memo[i, j] = -1;
            }
        }

        // Compute the minimum cost starting with perm[0] = 0
        int minCost = Dfs(0, 1, n, nums);

        // Reconstruct the lexicographically smallest optimal permutation
        int[] result = new int[n];
        result[0] = 0;
        int last = 0;
        int mask = 1;

        for (int step = 1; step < n; step++) {
            for (int nextVal = 0; nextVal < n; nextVal++) {
                if ((mask & (1 << nextVal)) == 0) {
                    int takeCost = Math.Abs(last - nums[nextVal]);
                    if (takeCost + Dfs(nextVal, mask | (1 << nextVal), n, nums) == memo[last, mask]) {
                        result[step] = nextVal;
                        mask |= (1 << nextVal);
                        last = nextVal;
                        break; // Guarantees the lexicographically smallest branch is chosen
                    }
                }
            }
        }

        return result;
    }

    private int Dfs(int last, int mask, int n, int[] nums) {
        if (mask == (1 << n) - 1) {
            return Math.Abs(last - nums[0]); // Complete the cyclic loop back to perm[0] = 0
        }

        if (memo[last, mask] != -1) {
            return memo[last, mask];
        }

        int minScore = int.MaxValue;

        for (int nextVal = 0; nextVal < n; nextVal++) {
            if ((mask & (1 << nextVal)) == 0) {
                int cost = Math.Abs(last - nums[nextVal]) + Dfs(nextVal, mask | (1 << nextVal), n, nums);
                if (cost < minScore) {
                    minScore = cost;
                }
            }
        }

        return memo[last, mask] = minScore;
    }
}
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N^2 * 2^N) | There are `N * 2^N` states. Processing each individual state requires looping through `N` potential next elements to visit. With `N <= 14`, execution runs smoothly within a few milliseconds. |
| **Space** | O(N * 2^N) | The size of the memoization array dominates the footprint, allocating storage up to `14 * 16384` integers. |

### Pitfalls & Edge Cases
* **Cyclic Reset Connection:** Do not forget to connect the last element back to `nums[perm[0]]` instead of `nums[perm[last]]`. Since `perm[0]` is locked to 0, the base case simplifies explicitly to `Math.Abs(last - nums[0])`.
* **Path Reconstruction Loops:** When building the path array, ensure the `nextVal` loop always runs from 0 up to `n - 1`. The first match found must immediately break out of the loop to fulfill the lexicographically smallest ordering rule.

### Key Takeaway
Whenever a permutation tracking formula contains a wrapping connection from the last element back to the first, search for cyclic invariance properties. Locking the initial starting node down to a fixed anchor point eliminates redundant pathing evaluations and dramatically simplifies path reconstruction operations.
