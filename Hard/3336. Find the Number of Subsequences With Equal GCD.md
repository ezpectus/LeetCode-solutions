# LeetCode 3336. Find the Number of Subsequences With Equal GCD

## Problem Description
You are given an integer array `nums`. Your task is to find the number of pairs of non-empty subsequences `(seq1, seq2)` of `nums` such that:
* The subsequences `seq1` and `seq2` are **disjoint** (they do not share any indices of `nums`).
* The **GCD** (Greatest Common Divisor) of the elements of `seq1` is equal to the GCD of the elements of `seq2`.

Since the answer can be large, return it modulo $10^9 + 7$.

---

## Method Explanation: 3D Dynamic Programming

The constraints are highly specific: $nums.length \le 200$ and $nums[i] \le 200$. The small upper bound of values means that the maximum possible GCD at any stage can never exceed $200$. This points directly to a **Dynamic Programming** solution with states trackable in a 3D matrix.

### State Representation
Let `dp[i][g1][g2]` represent the number of ways to distribute a subset of the first `i` elements such that:
* The current accumulated GCD of the first subsequence is `g1`.
* The current accumulated GCD of the second subsequence is `g2`.

We use a state value of `0` to denote a subsequence that is currently empty. This is mathematically convenient because $gcd(0, x) = x$.

### Transitions
For each element `nums[i]`, we have three mutually exclusive choices:
1. **Skip the element:** Do not include it in either subsequence. 
   `dp[i][g1][g2] += dp[i-1][g1][g2]`
2. **Add to the first subsequence:** 
   `dp[i][gcd(g1, nums[i])][g2] += dp[i-1][g1][g2]`
3. **Add to the second subsequence:** 
   `dp[i][g1][gcd(g2, nums[i])] += dp[i-1][g1][g2]`

### Space Optimization
Since the transitions for index `i` only depend on the states computed at index `i - 1`, we can optimize space from $\mathcal{O}(N \cdot M^2)$ to $\mathcal{O}(M^2)$ by utilizing a rolling array pattern (maintaining a `prev_dp` array and a `curr_dp` array).

## C++ Source Code

```cpp
#include <vector>
#include <numeric>
#include <algorithm>

using namespace std;

class Solution {
private:
    // Standard Euclidean algorithm for GCD
    int get_gcd(int a, int b) {
        return b == 0 ? a : get_gcd(b, a % b);
    }

public:
    int subsequencePairCount(vector<int>& nums) {
        int n = nums.size();
        int max_val = 0;
        for (int x : nums) max_val = max(max_val, x);

        long long MOD = 1e9 + 7;

        // dp[g1][g2] stores the count of subsequence pairs with respective GCD values
        // Size bounded by max_val (at most 200)
        vector<vector<long long>> dp(max_val + 1, vector<long long>(max_val + 1, 0));
        dp[0][0] = 1; // Base case: both sequences empty

        for (int x : nums) {
            vector<vector<long long>> next_dp = dp; // carry forward the skip option
            
            for (int g1 = 0; g1 <= max_val; ++g1) {
                for (int g2 = 0; g2 <= max_val; ++g2) {
                    if (dp[g1][g2] == 0) continue;

                    long long ways = dp[g1][g2];

                    // Option 2: Add x to the first subsequence
                    int n1 = (g1 == 0) ? x : get_gcd(g1, x);
                    next_dp[n1][g2] = (next_dp[n1][g2] + ways) % MOD;

                    // Option 3: Add x to the second subsequence
                    int n2 = (g2 == 0) ? x : get_gcd(g2, x);
                    next_dp[g1][n2] = (next_dp[g1][n2] + ways) % MOD;
                }
            }
            dp = move(next_dp);
        }

        // Aggregate answers where g1 == g2 and both subseqs are non-empty (g1 > 0)
        long long ans = 0;
        for (int g = 1; g <= max_val; ++g) {
            ans = (ans + dp[g][g]) % MOD;
        }

        return ans;
    }
};
```
## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N \cdot M^2 \cdot \log M)$
  * **Nested Loops:** We iterate over the total array size $N$ ($200$). Inside, we evaluate pairs of GCD states up to $M$ ($200$), running an inner nested structure of $M \times M$ states.
  * **GCD Overhead:** Computing the new state values requires logarithmic transitions $\log M$ via Euclidean divisions.
  * **Total Bounded Operations:** $200 \times 200 \times 200 \times \log(200) \approx 6.4 \times 10^7$ loops max, which runs highly efficiently within execution thresholds.

* **Space Complexity:** $\mathcal{O}(M^2)$
  * **Memory Bounded Optimization:** By maintaining only two 2D grids of size $(M+1) \times (M+1)$ representing the current and adjacent states on-the-fly, we reduce space drastically from three dimensions down to a tight static overhead bounded by $201 \times 201 \approx 40,401$ elements.
