# 2122. Recover the Original Array
## O(n┬▓) тАФ Sort + Enumerate k + Greedy Pairing

---

## Problem Statement

- Alice had array `arr` of `n` positive integers and a positive integer `k`.
- She created `lower[i] = arr[i] - k` and `higher[i] = arr[i] + k`.
- You are given all `2n` values from `lower` and `higher` combined (in any order).
- Recover and return any valid `arr`.

---

## Core Idea тАФ Enumerate k via Minimum Element

**Key formula**: Since `lower[i] = arr[i] - k` and `higher[i] = arr[i] + k`:

```
higher[i] - lower[i] = 2k   тЖТ   k = (higher[i] - lower[i]) / 2
```

**Key observation**: After sorting `nums`, **`nums[0]` is always in `lower`** тАФ it's the global minimum and can only be a lower value (since every lower value < its corresponding higher value).

This means `nums[0]` pairs with some `nums[j]` from higher, giving:
```
2k = nums[j] - nums[0]   тЖТ   k = (nums[j] - nums[0]) / 2
```

So we **enumerate all candidate `j`** (i.e., all candidates for the partner of `nums[0]` in higher), compute `k`, then verify whether this `k` allows a full valid pairing of all `2n` values.

**Verification (greedy)**:
- Use a frequency map of all values
- Sweep sorted `nums` from left to right
- For each unmatched value `x` (treating it as a lower value), check if `x + 2k` exists in the map
- If yes тЖТ pair them, consume both from the map, record `arr[i] = x + k`
- If no тЖТ this `k` is invalid, try next candidate

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] RecoverArray(int[] nums) {
        Array.Sort(nums);
        int total = nums.Length;

        for (int j = 1; j < total; j++) {
            int diff = nums[j] - nums[0];
            if (diff == 0 || diff % 2 != 0) continue; // k must be positive and integer
            int twoK = diff; // 2k

            // Try to greedily pair every x (lower) with x+2k (higher)
            var freq = new Dictionary<int, int>();
            foreach (int x in nums)
                freq[x] = freq.GetValueOrDefault(x, 0) + 1;

            var res = new List<int>();
            bool valid = true;

            foreach (int x in nums) {
                if (freq.GetValueOrDefault(x, 0) == 0) continue; // already consumed
                int higher = x + twoK;
                if (freq.GetValueOrDefault(higher, 0) == 0) { valid = false; break; }

                res.Add(x + twoK / 2); // arr[i] = lower + k = x + k
                freq[x]--;
                freq[higher]--;
            }

            if (valid) return res.ToArray();
        }

        return Array.Empty<int>(); // guaranteed valid answer exists
    }
}
```

## Why nums[0] Is Always in lower тАФ Proof

After sorting, `nums[0]` is the global minimum. Suppose for contradiction `nums[0]` is in `higher`. Then `arr[i] = nums[0] - k` for some `i`. But `arr[i]` must be positive (`arr[i] тЙе 1`) and `lower[i] = arr[i] - k = nums[0] - 2k < nums[0]`, which contradicts `nums[0]` being the minimum. Therefore **`nums[0]` must be in lower**. тЬУ

---

## Why (diff % 2 == 0 && diff > 0) тАФ Proof

```
2k = nums[j] - nums[0]
```

- `diff > 0` because k must be a **positive** integer (k = 0 is invalid per problem statement)
- `diff % 2 == 0` because `2k` must be **even** for `k` to be an integer

Both conditions must hold for a candidate `j` to be viable.

---

## Greedy Pairing тАФ Why It's Correct

After fixing `k`, sweep the sorted array from left to right. When we encounter an unconsumed value `x`:
- `x` **must** be a lower value тАФ since `x` is the smallest remaining, it can't be a higher value (its corresponding lower would be `x - 2k`, which if it existed would still be in the map and would have been processed first)
- So its partner is exactly `x + 2k`
- If `x + 2k` doesn't exist тЖТ contradiction, `k` is invalid

This greedy is deterministic and correct.

## Pitfalls & Edge Cases

- **Skip `diff == 0`** тАФ k must be strictly positive; if `nums[j] == nums[0]`, skip immediately
- **Skip `diff % 2 != 0`** тАФ k must be integer; `2k` must be even
- **Frequency map over sorted array** тАФ iterating sorted `nums` in order ensures the greedy always picks the smallest unmatched element as a lower value
- **Duplicates** тАФ e.g. `[1,1,3,3]`; frequency map handles multiple copies correctly
- **Don't stop at first invalid `j`** тАФ multiple candidates may fail before finding the valid `k`; try all `j` up to `total-1`
- **Result guaranteed** тАФ problem states a valid answer always exists, so no need for a fallback

---

## Key Takeaway

When two arrays are linked by `+k` and `-k`, the key formula is:

```
higher - lower = 2k   тЖТ   k = (higher - lower) / 2
```

**Strategy**:
- The global minimum is always in `lower` тЖТ enumerate its partner to find `k`
- For each candidate `k`, greedily verify in O(n) using a frequency map
- Overall O(n┬▓), efficient for n тЙд 1000

Pattern: unknown offset between two interleaved arrays тЖТ fix one anchor (min), enumerate offset candidates, greedy verify.
