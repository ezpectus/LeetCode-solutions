# 3500. Minimum Cost to Divide Array Into Subarrays

## O(N^2) — Dynamic Programming with Suffix Sum & Index Multiplier Elimination

### Core Idea — Suffix Transformation of Array Order Coefficients

The core difficulty lies in the fact that the cost of the i-th subarray depends explicitly on its sequential order index i:
Cost_i = ( Sum of nums[j] from j=0 to r + k * i ) * Sum of cost[j] from j=l to r

A direct left-to-right Dynamic Programming formulation requires tracking the partition counter i as part of the state, resulting in a state space of O(N^2) and an overall execution complexity of O(N^3) due to O(N) transition steps.

To bypass this dimensionality bottleneck, we process the partitions from right to left (suffixes) and reformulate the sequential multiplier i dynamically. Notice that dividing the prefix into a new subarray shifts the indices of all subsequent subarrays by +1. This index translation increases the contribution of all rightward subarrays exactly by k * Sum of cost[j] for each suffix partition. 

By accumulating the global suffix sum of the `cost` array during each step of our recurrence, we mathematically eliminate the dependency on the partition counter i. This lowers the state complexity to a single dimension DP[i], which denotes the minimum cost to partition the suffix starting at index i.

### Full Optimal Implementation (C++)

```cpp
// Dynamic Programming + Suffix Sum Optimization + Multiplier Elimination
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    long long minimumCost(vector<int>& nums, vector<int>& cost, int k) {
        int n = nums.size();
        
        // Prefix sums of nums for O(1) range sum queries
        vector<long long> prefixNums(n + 1, 0);
        for (int i = 0; i < n; ++i) {
            prefixNums[i + 1] = prefixNums[i] + nums[i];
        }
        
        // Prefix sums of cost for O(1) range cost queries
        vector<long long> prefixCost(n + 1, 0);
        for (int i = 0; i < n; ++i) {
            prefixCost[i + 1] = prefixCost[i] + cost[i];
        }
        
        // dp[i] represents the minimum cost to partition the suffix starting at index i
        vector<long long> dp(n + 1, 0);
        
        // Solve backwards to naturally accumulate suffix weight increments
        for (int i = n - 1; i >= 0; --i) {
            long long min_val = -1;
            for (int j = i; j < n; ++j) {
                // S_nums[j] represents the sum of nums[0...j]
                long long nums_sum = prefixNums[j + 1];
                
                // Cost of current subarray cost[i...j]
                long long current_sub_cost = prefixCost[j + 1] - prefixCost[i];
                
                // The total cost of all elements in the suffix cost[i...n-1]
                long long suffix_cost = prefixCost[n] - prefixCost[i];
                
                // Transition: Current subsegment cost + shift penalty of all subsequent elements + DP of remaining suffix
                long long cost_transition = nums_sum * current_sub_cost + k * suffix_cost + dp[j + 1];
                
                if (min_val == -1 || cost_transition < min_val) {
                    min_val = cost_transition;
                }
            }
            dp[i] = min_val;
        }
        
        return dp[0];
    }
};
```

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | O(N^2) | Two nested loops iterate over all suffix split points. Both prefix sum operations and state transitions are executed in O(1) time. |
| **Space Complexity** | O(N) | Dominated by the 1D DP state table and prefix/suffix sum vectors of size N + 1. |
| **Pitfalls & Edge Cases** | **64-bit Integer Overflow**: Since N, nums[i], cost[i], k <= 1000, intermediate products of range sums can reach up to 10^6 * 10^6 = 10^12, which exceeds the boundaries of standard signed 32-bit integers. We prevent overflow by keeping all DP tables and accumulator variables as `long long`. |
| **Key Takeaway** | Sequentially increasing linear multipliers in a partition cost structure can often be transformed into global suffix or prefix tracking offsets, transforming O(N^3) state machines into highly optimized O(N^2) algorithms. |

## 4. Comprehensive Complexity & Mathematical Analysis

### Detailed Time Complexity Decomposition

The computational pipeline of the dynamic programming model is structured as follows:

T_total = T_prefix_build + T_dp_transitions

#### Prefix Sum Preprocessing (T_prefix_build):
Generating the 1-indexed cumulative tables for both the `nums` and `cost` vectors of size N takes linear time:

T_prefix_build = O(N)

#### DP Matrix Transitions (T_dp_transitions):
The algorithm features a nested loop layout where the outer state variable i goes from N-1 down to 0, and the inner split boundary index j spans from i to N-1:

T_dp_transitions = Sum from i=0 to N-1 of (N - i) = N(N + 1) / 2 = O(N^2)

Each transition relies on three index-based table retrievals and flat basic mathematical operations. Thus, the total operational cost is O(1) per state transition.

Combining these stages, the total runtime is:

Time Complexity = O(N^2)

With N <= 1000, N^2 <= 10^6 operations, which easily runs in less than 5 milliseconds in C++.

### Detailed Space Complexity Decomposition

The memory allocation of this model is determined by linear tracking arrays:

#### Dynamic Programming Cache (dp):
The state vector containing optimal prefix subproblem values is sized N + 1:

S_dp = (N + 1) * 8 bytes => O(N)

#### Prefix Sum Allocations (prefixNums, prefixCost):
Cumulative sum buffers require:

S_prefix = 2 * (N + 1) * 8 bytes => O(N)

#### Total Auxiliary Footprint:

Space Complexity = O(N)

## 5. Mathematical Formulations & Foundational Algorithms

### The Suffix Multiplier Invariant

Let us define a partition of the index space [0, N-1] into m continuous segments. The h-th subarray is bounded by indices [l_h, r_h] where l_1 = 0, r_m = N-1, and l_h = r_(h-1) + 1.

The total cost formula for the partition is:

Total Cost = Sum from h=1 to m of ( S_nums[r_h] + k * h ) * SumCost(l_h, r_h)

We isolate the sequence multiplier component h:

Sum from h=1 to m of h * SumCost(l_h, r_h) = 1 * SumCost(l_1, r_1) + 2 * SumCost(l_2, r_2) + ... + m * SumCost(l_m, r_m)

By grouping terms, we can re-evaluate this summation. A specific sub-segment cost SumCost(l_x, r_x) is added exactly x times. This is equivalent to adding the cumulative suffix of costs starting at each partition index l_h for all h <= x:

Sum from h=1 to m of h * SumCost(l_h, r_h) = Sum from h=1 to m of SuffixCost(l_h)

Where SuffixCost(l_h) = Sum from y=l_h to N-1 of cost[y].

Substituting this back into the total cost formula yields:

Total Cost = Sum from h=1 to m of [ S_nums[r_h] * SumCost(l_h, r_h) + k * SuffixCost(l_h) ]

This formulation decouples the dynamic subproblem from its sequential partition index h, enabling the exact DP state representation:

DP[i] = min over i <= j < N of { S_nums[j] * SumCost(i, j) + k * SuffixCost(i) + DP[j + 1] }

This mathematical reduction allows us to compute the optimal division in O(N^2) time.

**Pattern recognition:** "minimum total weight of a subarray division with an ordinal sequence index multiplier i" -> Segment index reduction -> Suffix transition offset grouping -> Suffix DP.
