# 2290. Minimum Obstacle Removal to Reach Corner
## O(mn) — 0-1 BFS with Deque

---

## Problem Statement

- Grid with 0 (free) and 1 (obstacle). Move up/down/left/right.
- Return minimum obstacles to remove to reach (m-1, n-1) from (0,0).

---

## Core Idea — 0-1 BFS

Each move has cost 0 (enter free cell) or 1 (enter obstacle). This is a single-source shortest path problem with edge weights 0 and 1 — ideal for **0-1 BFS**.

Standard Dijkstra is O(mn log mn). 0-1 BFS is O(mn) because:
- Cost-0 edges (free cells) → push to the **front** of the deque.
- Cost-1 edges (obstacles) → push to the **back** of the deque.

This maintains the deque in non-decreasing cost order without a heap.

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int minimumObstacles(vector<vector<int>>& grid) {
        int m = grid.size(), n = grid[0].size();
        vector<vector<int>> dist(m, vector<int>(n, INT_MAX));
        dist[0][0] = 0;

        deque<pair<int,int>> dq;
        dq.push_front({0, 0});

        int dirs[4][2] = {{0,1},{0,-1},{1,0},{-1,0}};

        while (!dq.empty()) {
            auto [r, c] = dq.front();
            dq.pop_front();

            for (auto& d : dirs) {
                int nr = r + d[0], nc = c + d[1];
                if (nr < 0 || nr >= m || nc < 0 || nc >= n) continue;

                int cost = dist[r][c] + grid[nr][nc]; // 0 or 1
                if (cost < dist[nr][nc]) {
                    dist[nr][nc] = cost;
                    if (grid[nr][nc] == 0) dq.push_front({nr, nc});
                    else                   dq.push_back({nr, nc});
                }
            }
        }

        return dist[m-1][n-1];
    }
};
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(mn) | Each cell processed at most once |
| Space Complexity | O(mn) | dist array + deque |

---

## Why 0-1 BFS Works

The deque always holds states in non-decreasing dist order:
- Front-push (cost 0): new cell has same dist as current → stays at front. ✓
- Back-push (cost 1): new cell has dist+1 → goes after all current dist cells. ✓

This is the key invariant — identical to Dijkstra's monotone processing, but without the log-n overhead of a heap.

---

## Example Walkthrough

**Example 1:** `grid = [[0,1,1],[1,1,0],[1,1,0]]`

| Step | Current | Cost | Queue (front→back) |
|------|---------|------|---------------------|
| Init | (0,0)=0 | 0    | [(0,0)]             |
| Pop (0,0) | → (0,1)=1, (1,0)=1 | 1 | [(0,1) back, (1,0) back] |
| Pop (0,1) | → (0,2)=1 | 2 | [..., (0,2) back] |
| ...continues to find path removing (0,1) and (0,2)... | | | |

**Output:** 2 ✓

**Example 2:** Path exists with 0 removals → **Output:** 0 ✓

---

## Key Takeaway

**0-1 BFS pattern:** Whenever edge weights are exactly 0 or 1, use a deque instead of a heap. Push 0-cost neighbors to front, 1-cost to back. O(V+E) instead of O((V+E)log V).

**Recognition signal:** "Minimum cost path where each step costs 0 or 1" → 0-1 BFS, always.
