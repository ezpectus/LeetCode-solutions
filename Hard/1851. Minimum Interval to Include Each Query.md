# 1851. Minimum Interval to Include Each Query
## O((n + q) log n) — Sweep Line + Min-Heap

---

## Problem Statement

- For each query value `q`, find the smallest interval `[l, r]` that contains `q` (i.e. `l ≤ q ≤ r`).
- Interval size = `r - l + 1`. Return `-1` if none contains `q`.

---

## Core Idea — Offline Sweep with Min-Heap

**Offline processing**: sort queries. Process them in ascending order so we can sweep intervals left-to-right.

**Algorithm**:
1. Sort intervals by left endpoint.
2. Sort queries (keeping original indices for result reconstruction).
3. For each query `q` (in sorted order):
   - **Add** all intervals with `left ≤ q` to a min-heap keyed by size `(r-l+1, r)`.
   - **Remove** from heap top all intervals with `right < q` (they don't contain `q` anymore).
   - **Answer** = `heap[0][0]` (smallest size remaining), or `-1` if heap is empty.

**Why this works**: The heap always contains intervals that have started (`left ≤ q`) but we only answer from those that haven't ended (`right ≥ q`). The lazy deletion (remove from top when `right < q`) is correct because if the smallest interval expired, the next smallest might still be valid — we keep popping until the top is valid or heap is empty.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] MinInterval(int[][] intervals, int[] queries) {
        Array.Sort(intervals, (a, b) => a[0] - b[0]); // sort by left

        int q = queries.Length;
        int[] idx = Enumerable.Range(0, q).OrderBy(i => queries[i]).ToArray();
        int[] res = new int[q];

        // Min-heap: (size, right)
        var heap = new PriorityQueue<(int size, int right), int>();
        int j = 0; // pointer into intervals

        foreach (int i in idx) {
            int query = queries[i];

            // Add intervals starting <= query
            while (j < intervals.Length && intervals[j][0] <= query) {
                int size = intervals[j][1] - intervals[j][0] + 1;
                heap.Enqueue((size, intervals[j][1]), size);
                j++;
            }

            // Remove intervals ending before query
            while (heap.Count > 0 && heap.Peek().right < query)
                heap.Dequeue();

            res[i] = heap.Count > 0 ? heap.Peek().size : -1;
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O((n+q) log n)** | Each interval pushed/popped once; each query does one heap peek |
| **Space Complexity** | **O(n + q)** | Heap + sorted query indices |

---

## Why Offline (Sort Queries)?

If queries were processed in random order, we'd need to re-scan intervals for each query. Sorting queries lets us use a single left-to-right pointer `j` into intervals — each interval is added to the heap at most once. Total heap operations = O(n) insertions + O(n) deletions = O(n log n).

---

## Lazy Deletion Explained

We don't remove intervals when their `right < q` proactively — only when they're at the **top of the heap** (i.e., the current minimum). Why is this safe?

- If a heap element has `right < q`, it can't be the answer to any future query (queries are increasing, right endpoint is fixed).
- But it might be buried deep in the heap. We only care about the minimum, so we pop and discard expired intervals from the top until a valid one appears.
- We never get a wrong answer because an expired interval is never the minimum when a valid one exists (valid intervals are still in the heap).

## Example Walkthrough

**Example 1**: `intervals=[[1,4],[2,4],[3,6],[4,4]]`, `queries=[2,3,4,5]`

Sorted queries: `[2,3,4,5]` (already sorted), intervals sorted: same.

**q=2**: add [1,4](size=4), [2,4](size=3). Heap: {(3,4),(4,4)}. Top right=4≥2 → ans=3 ✓
**q=3**: add [3,6](size=4). Heap: {(3,4),(4,4),(4,6)}. Top right=4≥3 → ans=3 ✓
**q=4**: add [4,4](size=1). Heap: {(1,4),(3,4),(4,4),(4,6)}. Top right=4≥4 → ans=1 ✓
**q=5**: no new intervals. Pop (1,4): right=4<5 ✗. Pop (3,4): right=4<5 ✗. Pop (4,4): right=4<5 ✗. Top=(4,6): right=6≥5 → ans=4 ✓

Output: `[3,3,1,4]` ✓

## Pitfalls & Edge Cases

- **Keep original query indices** — sorting queries loses their position; store `idx = sorted indices` and write `res[idx[i]]` not `res[i]`
- **Lazy deletion only from top** — don't scan whole heap; only pop while `heap.Peek().right < q`
- **`-1` default** — if heap empty after cleanup, no valid interval for this query
- **Duplicate query values** — handled naturally; same `j` pointer state used for all identical queries

---

## Key Takeaway

"Smallest interval containing each query" → **offline sweep + min-heap**:

1. Sort intervals by left, sort queries ascending
2. Sweep: add intervals as their left endpoint is passed, lazy-delete expired ones from heap top
3. Answer = heap minimum (smallest active interval)

Pattern: "active set" problems where elements enter/leave as we sweep → min/max heap with lazy deletion. Same idea as meeting rooms, skyline problem.
