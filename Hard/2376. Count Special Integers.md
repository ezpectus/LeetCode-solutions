# 2376. Count Special Integers

## O(d × 2¹⁰) — Digit DP with Bitmask

---

## Problem Statement

- Given positive integer `n`, count all integers in `[1, n]` where all digits are **distinct** (no repeated digits).
- **Constraints**: 1 ≤ n ≤ 2×10⁹ (up to 10 digits).

---

## Core Idea — Digit DP with Used-Digits Bitmask

**Key observation**: A number has distinct digits iff each digit 0-9 appears **at most once** → track which digits are already used via a **10-bit bitmask** (2¹⁰ = 1024 states).

**Digit DP approach**: Build numbers digit-by-digit from left to right, counting valid constructions while staying ≤ n.

**State**: `dp[pos][mask][tight][leadingZero]`
- `pos` = current digit position (0-indexed from left)
- `mask` = 10-bit bitmask of used digits (bit i set → digit i used)
- `tight` = whether we're still bounded by n's digits (if false, can place any digit)
- `leadingZero` = whether all previous digits were leading zeros (handles numbers shorter than n's length)

**Transition**: At position `pos`, try placing digit `d` from 0 to 9:
- Skip if `d` already used (bit d set in mask)
- If `tight`, can only place `d ≤ digit[pos]` of n
- If placing non-zero digit, clear `leadingZero` flag
- Recurse to `pos+1` with updated mask

**Base case**: Reached end of n's digits → return 1 if not all leading zeros (exclude 0 itself).

**Answer**: `dp[0][0][true][true]` — count all valid numbers from 1 to n.

---

## Implementation Strategy

### Approach 1: Recursive Memoization (Cleaner)

Convert `n` to digit array, use memoization with state dictionary.

### Approach 2: Iterative DP (More Control)

Manually iterate through digit positions, maintaining DP states.

**Choosing recursive**: Easier to handle `tight` and `leadingZero` flags naturally. Memoization handles state space automatically.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    private int[] digits;
    private Dictionary<(int, int, bool, bool), int> memo;
    
    public int CountSpecialNumbers(int n) {
        digits = n.ToString().Select(c => c - '0').ToArray();
        memo = new Dictionary<(int, int, bool, bool), int>();
        
        return Dp(0, 0, true, true);
    }
    
    private int Dp(int pos, int mask, bool tight, bool leadingZero) {
        // Base case: reached end of digits
        if (pos == digits.Length) {
            // Valid number only if we actually placed some digits (not all leading zeros)
            return leadingZero ? 0 : 1;
        }
        
        // Memoization key
        var key = (pos, mask, tight, leadingZero);
        if (memo.ContainsKey(key)) return memo[key];
        
        int limit = tight ? digits[pos] : 9;
        int count = 0;
        
        for (int d = 0; d <= limit; d++) {
            // If we're still in leading zeros and place another 0, continue with leading zeros
            if (leadingZero && d == 0) {
                count += Dp(pos + 1, mask, tight && (d == limit), true);
                continue;
            }
            
            // Skip if digit already used
            if (((mask >> d) & 1) == 1) continue;
            
            int newMask = mask | (1 << d);
            bool newTight = tight && (d == limit);
            
            count += Dp(pos + 1, newMask, newTight, false);
        }
        
        memo[key] = count;
        return count;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(d × 2¹⁰ × 2 × 2)** = **O(d × 2¹⁰)** | d ≤ 10 digits, 1024 masks, 2 tight values, 2 leadingZero values |
| **Space** | **O(d × 2¹⁰)** | Memoization dictionary size |

With d = 10 (max digits for n ≤ 2×10⁹): 10 × 1024 × 4 ≈ 40K states — extremely fast.

## Why This Works — Example Walkthrough

### Example 1: `n = 20`

**Digits**: [2, 0]

**Construction**:

**Position 0** (tens place):
- `d = 0` (leading zero): builds 1-digit numbers [1-9]
  - Position 1: place any unused digit from {1-9} → 9 valid numbers
- `d = 1`: builds [10-19]
  - Position 1: place any unused digit except 1 from {0,2-9} → 9 valid numbers
- `d = 2` (tight limit): builds [20]
  - Position 1: must place 0 (tight), but 0 not used → 1 valid number

**Total**: 9 (single-digit) + 9 ([10-19]) + 1 ([20]) = 19 ✓

### Example 2: `n = 135`

**Digits**: [1, 3, 5]

**High-level breakdown**:
- **1-digit numbers**: [1-9] → 9 valid
- **2-digit numbers**: [10-99]
  - First digit: {1-9} (9 choices)
  - Second digit: any except first (9 choices)
  - Total: 9 × 9 = 81 valid
- **3-digit numbers ≤ 135**:
  - First digit = 1:
    - Second digit ∈ {0,2} (can't use 3 if we want to stay ≤ 135):
      - 1,0,x: x ∈ {2-9} → 8 numbers
      - 1,2,x: x ∈ {0,3-9} → 8 numbers
      - 1,3,x: x ∈ {0-5} \ {1,3} → 4 numbers ([130,132,134,135])
    - Total for first digit 1: 8 + 8 + 4 = 20
  
**Total**: 9 + 81 + 20 = 110 ✓

## Pitfalls & Edge Cases

- **Leading zeros handling**: Numbers like 00123 should be treated as 123 (3-digit number, not 5-digit). The `leadingZero` flag handles this — leading zeros don't consume mask bits and don't contribute to the final count.
- **Excluding 0**: The `leadingZero` check in base case ensures we don't count 0 (all leading zeros = invalid number).
- **Tight bound**: When `tight = true`, we're still bounded by n's digits; once we place a digit < n's digit at that position, `tight` becomes false for all subsequent positions.
- **Mask updates**: Only update mask when placing a **non-leading-zero** digit. Leading zeros don't count as "using" digit 0.
- **Memoization key**: Must include all four state variables (pos, mask, tight, leadingZero) — missing any causes incorrect caching.

## Alternative Approach — Combinatorial Counting

For numbers with **fewer digits** than n, we can count combinatorially:
- k-digit numbers with distinct digits:
  - First digit: 9 choices (1-9, can't be 0)
  - Second digit: 9 choices (0-9 except first)
  - Third digit: 8 choices (0-9 except first two)
  - ... k-th digit: (11-k) choices
  - Total: 9 × 9 × 8 × 7 × ... × (11-k)

Then handle numbers with **same length** as n using digit DP.

**Why digit DP is simpler**: Unified framework handles both cases automatically, avoids off-by-one errors in combinatorial formulas.

---

## Optimization Notes

### Pruning Tight States

When `tight = false`, all remaining digits can be arbitrary (within mask constraints). Can precompute `dp[pos][mask][false][leadingZero]` once and reuse across different n values. Not critical for single query, but useful for batch processing.

### Iterative DP

Can implement bottom-up for slightly better constant factors, but recursive memoization is cleaner for handling tight/leadingZero flags.

## Key Takeaway

"Count numbers ≤ n with distinct digits" → **Digit DP with used-digits bitmask**. **Four-state DP**:
1. **pos**: current digit position
2. **mask**: which digits 0-9 are already used (10-bit bitmask)
3. **tight**: still bounded by n's digits
4. **leadingZero**: whether all previous digits were leading zeros

Build numbers digit-by-digit, skip digits already in mask. O(d × 2¹⁰) where d = number of digits in n.

Classic digit DP pattern — appears in: "count numbers ≤ n with property P" where P involves per-digit constraints. The bitmask tracks "which digits used so far", enabling O(2¹⁰) state space instead of O(10^d).
