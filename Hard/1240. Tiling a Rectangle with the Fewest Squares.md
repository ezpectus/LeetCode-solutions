# 1240. Tiling a Rectangle with the Fewest Squares
## O(m^n) pruned — Backtracking with Skyline

---

## Problem Statement

- Tile an `n × m` rectangle using the minimum number of integer-sided squares.
- `1 ≤ n, m ≤ 13`.

---

## Core Idea — Backtracking on Skyline

Maintain a **skyline** array `sky[0..n-1]` where `sky[j]` = how many rows of column `j` are already filled.

**At each step**:
1. Find the column `j` with minimum `sky[j]` (the "lowest" point = next unfilled position).
2. Determine the maximum square size `s` that fits at `(j, sky[j])`: requires `j+s ≤ n`, `sky[j]+s ≤ m`, and `sky[j..j+s-1]` all equal `sky[j]` (flat base).
3. Try squares from largest to smallest — place one, recurse, undo.

**Pruning**: if `count >= best`, abandon the branch immediately.

The skyline representation ensures we always fill the lowest available spot and never leave gaps, which dramatically prunes the search space.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int best;

    void Bt(int[] sky, int n, int m, int count) {
        if (count >= best) return;
        bool done = true;
        for (int j = 0; j < n; j++) if (sky[j] < m) { done = false; break; }
        if (done) { best = count; return; }

        // Find leftmost minimum
        int minH = sky[0], col = 0;
        for (int j = 1; j < n; j++) if (sky[j] < minH) { minH = sky[j]; col = j; }

        // Max square size with flat base
        int maxS = 1;
        while (col+maxS < n && minH+maxS < m && sky[col+maxS] == minH) maxS++;

        int[] old = (int[])sky.Clone();
        for (int s = maxS; s >= 1; s--) {
            for (int jj = col; jj < col+s; jj++) sky[jj] = minH+s;
            Bt(sky, n, m, count+1);
            Array.Copy(old, col, sky, col, maxS); // restore
        }
    }

    public int TilingRectangle(int n, int m) {
        best = n * m; // worst case all 1×1
        Bt(new int[n], n, m, 0);
        return best;
    }
}
```

## Complexity

With n,m ≤ 13 and aggressive pruning, the search tree is small in practice — all test cases run in milliseconds.

## Example Walkthrough

**`n=2, m=3`**

sky=[0,0]. Min at j=0, maxS=min(2,3)=2.

Try s=2: sky=[2,2]. Min at j=0 (h=2), maxS=1. Place 1×1: sky=[3,2]. Place 1×1: sky=[3,3]. Done! count=3. best=3.
Try s=1: sky=[1,0]. ... eventually also gives 3.

Answer: **3** ✓

---

## Pitfalls & Edge Cases

- **`sky[j..j+s-1]` must be flat**: only extend `maxS` while adjacent columns match the current min height.
- **Try larger squares first**: helps find good solutions early for better pruning.
- **Restore only the modified range** (`col..col+maxS-1`), not the whole array.

---

## Key Takeaway

"Min squares to tile rectangle" → **backtracking on skyline: always fill lowest point, try all valid square sizes with flat-base constraint**:

```
find col = leftmost minimum height
maxS = max square fitting with flat base
for s = maxS down to 1: place, recurse, undo
prune if count >= best
```

Small constraints (n,m ≤ 13) make pruned backtracking feasible.
