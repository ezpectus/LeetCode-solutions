# 3671. Sum of Beautiful Subsequences
## O(n · d̄(maxVal) · log n) — Dirichlet Identity Reduces to φ(g)×h(g) + BIT Counting

---

## Problem Statement

Array `nums`. For every positive integer g, beauty(g) = g × (# strictly increasing subsequences with GCD exactly g). Return Σ beauty(g) mod 10⁹+7.

Constraints: n ≤ 10⁴, nums[i] ≤ 7×10⁴.

---

## Core Insight 1 — Define h(g) as a Dirichlet Relaxation of f(g)

Let:
- `f(g)` = # strictly increasing subsequences with GCD **exactly** g.
- `h(g)` = # strictly increasing subsequences where all elements are **divisible** by g (GCD is g or any multiple).

Then h(g) = Σ_{k: g|k} f(k) (summing f over all multiples of g). By Möbius inversion:
```
f(g) = Σ_{k: g|k} μ(k/g) · h(k)
```

---

## Core Insight 2 — The Euler Totient Shortcut

The full target is:
```
Σ_g g · f(g) = Σ_g g · Σ_{k: g|k} μ(k/g) · h(k)
```

Swap summation, let n = g·m (substitute k = n, g = n/m):
```
= Σ_n h(n) · Σ_{m|n} (n/m) · μ(m)
```

By the classical identity **φ(n) = Σ_{d|n} μ(d) · (n/d)**:
```
Σ_n h(n) · Σ_{m|n} (n/m) · μ(m) = Σ_n h(n) · φ(n)
```

**Result: Total beauty = Σ_{g≥1} φ(g) · h(g)**

This eliminates the Möbius inversion entirely — just compute h(g) for each g, multiply by φ(g), and sum.

---

## Core Insight 3 — Computing h(g) With a Fenwick Tree

h(g) = # strictly increasing subsequences of the sub-sequence `a[]` = {nums[i] : g | nums[i]} (in original order).

**Standard DP:** Let f(i) = # strictly increasing subsequences ending at position i.
```
f(i) = 1 + Σ_{j < i, a[j] < a[i]} f(j)
```

**BIT over coordinate-compressed values:**
1. Compress values of a[] to ranks 1..m.
2. For each element x (in position order): query BIT prefix sum up to rank(x)−1 = count of ways to extend; set f(i) = query + 1; update BIT at rank(x) with f(i).
3. h(g) = Σ f(i).

This counts all strictly increasing subsequences (of any length) in O(|a| log |a|) per g.

---

## Key Efficiency: Precompute Divisor Lists in O(n · d̄)

For each nums[i], find all divisors d of nums[i] and append nums[i] to divList[d] — in position order. Entries in divList[g] are automatically in original array order (since we scan i = 0, 1, …, n−1).

Total entries across all divList[g]: Σ_{i} d(nums[i]) ≈ n × d̄(maxVal) ≈ 10⁴ × 11 ≈ 10⁵.

This replaces the O(n × maxVal) naive approach with O(n × √maxVal) for divisor enumeration.

---

## Algorithm

```
1. Euler's totient sieve: compute φ(g) for all g ≤ maxVal. [O(maxVal log log maxVal)]
2. Build divList[g] = elements of nums divisible by g, in position order. [O(n√maxVal)]
3. For each g from 1 to maxVal:
     a. Coordinate compress divList[g].
     b. BIT over compressed values, compute h(g). [O(|divList[g]| log n)]
     c. ans += φ(g) × h(g) mod MOD.
4. Return ans.
```

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int totalBeauty(vector<int>& nums) {
        const int MOD = 1e9 + 7;
        int n = nums.size();
        int maxVal = *max_element(nums.begin(), nums.end());

        // Euler's totient sieve: phi[i] starts as i; each prime p divides phi[j] by p
        vector<int> phi(maxVal + 1);
        iota(phi.begin(), phi.end(), 0);
        for (int i = 2; i <= maxVal; i++) {
            if (phi[i] == i) {            // i is prime (no factor has updated phi[i] yet)
                for (int j = i; j <= maxVal; j += i)
                    phi[j] -= phi[j] / i; // phi[j] *= (1 - 1/i)
            }
        }

        // divList[g] = nums elements divisible by g, in original position order
        vector<vector<int>> divList(maxVal + 1);
        for (int i = 0; i < n; i++) {
            int v = nums[i];
            for (int d = 1; d * d <= v; d++) {
                if (v % d == 0) {
                    divList[d].push_back(v);
                    if (d != v / d) divList[v / d].push_back(v);
                }
            }
        }

        long long ans = 0;

        for (int g = 1; g <= maxVal; g++) {
            const vector<int>& a = divList[g];
            if (a.empty()) continue;

            // Coordinate compress values in a[]
            vector<int> vals(a.begin(), a.end());
            sort(vals.begin(), vals.end());
            vals.erase(unique(vals.begin(), vals.end()), vals.end());
            int m = (int)vals.size();

            // BIT over compressed values
            vector<long long> bit(m + 1, 0);
            auto upd = [&](int i, long long v) {
                for (; i <= m; i += i & -i) bit[i] = (bit[i] + v) % MOD;
            };
            auto qry = [&](int i) -> long long {
                long long s = 0;
                for (; i > 0; i -= i & -i) s = (s + bit[i]) % MOD;
                return s;
            };

            long long h = 0;
            for (int x : a) {
                int r = (int)(lower_bound(vals.begin(), vals.end(), x) - vals.begin()) + 1;
                long long cnt = (qry(r - 1) + 1) % MOD;  // subseqs ending here
                upd(r, cnt);
                h = (h + cnt) % MOD;
            }

            ans = (ans + (long long)phi[g] * h % MOD) % MOD;
        }

        return (int)ans;
    }
};
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Totient sieve | O(maxVal log log maxVal) | Standard sieve |
| Divisor enumeration | O(n × √maxVal) | Per element: O(√7×10⁴) ≈ 265 ops |
| Total entries in divList | O(n × d̄) ≈ 10⁵ | d̄ ≈ avg divisors ≈ 11 for nums ≤ 7×10⁴ |
| BIT per g | O(|divList[g]| × log n) | |
| Total BIT work | **O(n × d̄ × log n)** | ≈ 10⁵ × 14 ≈ 1.4×10⁶ |
| Overall | **O(n × d̄ × log n + maxVal log log maxVal)** | Effectively O(n log n) |

Σ g·f(g) = 2×1 + 4×1 + 6×1 = **12**.

## Pitfalls & Edge Cases

- **divList built in position order:** By processing i = 0,1,…,n−1 sequentially, each divList[g] accumulates elements in increasing position order. The BIT then correctly counts strictly increasing subsequences (respecting original array order). ✓

- **BIT reset per g:** Each g allocates a fresh `vector<long long> bit(m+1, 0)`. No cross-g contamination. ✓

- **`qry(r-1)` when r=1:** qry(0) returns 0 (loop doesn't execute). The single-element subsequence {x} contributes 0+1=1. ✓

- **Duplicate values within a[]:** Same values map to the same rank. `qry(r-1)` (strictly less than r) correctly excludes equal-valued elements — enforcing STRICT monotonicity. ✓

- **`(long long)phi[g] * h % MOD`:** phi[g] ≤ g ≤ 7×10⁴ and h < MOD ≈ 10⁹. Product ≤ 7×10⁴ × 10⁹ ≈ 7×10¹³ fits in `long long` (max ≈ 9.2×10¹⁸). ✓

- **Why `phi[i] == i` detects primality:** Initially phi[i]=i. A prime i has no proper divisors < i, so no prior iteration has reduced phi[i]. When phi[i]=i, i is prime and we sieve its multiples. ✓

## Key Takeaway

**"Sum over all g of g × count-of-subsequences-with-GCD-g"** → apply the Dirichlet convolution identity:

```
Σ_g g · f(g)   =   Σ_g φ(g) · h(g)
```

where `f(g)` = exact-GCD count, `h(g)` = all-divisible-by-g count. The identity follows from φ(n) = Σ_{d|n} μ(d) × (n/d).

This transforms an "exact GCD" counting problem (requiring Möbius inversion) into a "divisibility" counting problem (directly computed per g). For each g, count strictly increasing subsequences of the g-multiples with a BIT in O(|a| log n). Total: O(n × d̄ × log n) — essentially linear.

**Pattern recognition:** When the answer is Σ g·f(g) and f is defined by "exact arithmetic condition" (GCD, exact divisors, etc.), look for a Dirichlet convolution h = f * 1 where h is easier to compute. The identity Σ g·f(g) = Σ φ(g)·h(g) often appears when the Dirichlet series of g and μ combine to give Euler's totient.
