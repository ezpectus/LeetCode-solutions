# 1627. Graph Connectivity With Threshold
## O(n log n × α(n) + q) — Union-Find with Harmonic Series

---

## Problem Statement

- Cities `1..n`. Cities `x` and `y` are connected if they share a common divisor `z > threshold`.
- For each query `[a, b]`, return whether `a` and `b` are connected (directly or via path).

---

## Core Idea — Union-Find via Sieve-Like Iteration

For each `z > threshold`: all multiples of `z` share `z` as a common divisor, so they should all be in the same connected component. Union `z` with `2z`, `3z`, `4z`, ... up to `n`.

This is O(n log n) total unions (harmonic series: n/z summed for z=threshold+1..n).

Each query is then answered in O(α(n)) with Union-Find path compression.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int[] parent, rank;

    int Find(int x) {
        while (parent[x] != x) { parent[x] = parent[parent[x]]; x = parent[x]; }
        return x;
    }

    void Union(int x, int y) {
        int px = Find(x), py = Find(y);
        if (px == py) return;
        if (rank[px] < rank[py]) (px, py) = (py, px);
        parent[py] = px;
        if (rank[px] == rank[py]) rank[px]++;
    }

    public IList<bool> AreConnected(int n, int threshold, int[][] queries) {
        parent = Enumerable.Range(0, n+1).ToArray();
        rank = new int[n+1];

        // For each z > threshold, union all multiples
        for (int z = threshold+1; z <= n; z++)
            for (int mult = 2*z; mult <= n; mult += z)
                Union(z, mult);

        return queries.Select(q => Find(q[0]) == Find(q[1])).ToList();
    }
}
```

Components: {3,6} and rest isolated.

Queries: [3,6] → same component → True. [1,4], [2,5] → different → False. ✓

---

## Key Takeaway

"Cities connected by shared divisor > threshold" → **for each z > threshold, union all multiples**. Sieve-like O(n log n) build + O(α(n)) per query via Union-Find.
