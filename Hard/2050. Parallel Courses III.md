# 2050. Parallel Courses III
## O(n + E) — Topological Sort + DP (Critical Path)

---

## Problem Statement

- `n` courses with prerequisites. Multiple courses can run in parallel.
- `time[i]` = months to complete course `i`.
- Return the minimum months to complete all courses.

---

## Core Idea — Earliest Start Time via Topological DP

**State**: `earliest[i]` = earliest month course `i` can **start** = max finish time of all its prerequisites.

**Transition**: when course `u` finishes at `earliest[u] + time[u]`, update all its successors:
```
earliest[v] = max(earliest[v], earliest[u] + time[u])
```

Process in **topological order** (Kahn's algorithm / BFS with indegree). Courses with no prerequisites start at month 0.

**Answer**: `max(earliest[i] + time[i])` = latest course completion = critical path length.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumTime(int n, int[][] relations, int[] time) {
        var graph = new List<int>[n];
        for (int i = 0; i < n; i++) graph[i] = new List<int>();
        int[] indegree = new int[n];

        foreach (var r in relations) {
            graph[r[0]-1].Add(r[1]-1);
            indegree[r[1]-1]++;
        }

        int[] earliest = new int[n];
        var q = new Queue<int>();
        for (int i = 0; i < n; i++)
            if (indegree[i] == 0) q.Enqueue(i);

        while (q.Count > 0) {
            int u = q.Dequeue();
            int finish = earliest[u] + time[u];
            foreach (int v in graph[u]) {
                earliest[v] = Math.Max(earliest[v], finish);
                if (--indegree[v] == 0) q.Enqueue(v);
            }
        }

        int res = 0;
        for (int i = 0; i < n; i++)
            res = Math.Max(res, earliest[i] + time[i]);
        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n + E)** | Topological BFS visits each node and edge once |
| **Space** | **O(n + E)** | Adjacency list + DP array |

---

## Why Topological Order is Required

`earliest[v]` depends on `earliest[u]` for all predecessors `u`. Processing in topological order guarantees all predecessors are processed before their successors — so `earliest[v]` is fully computed before `v` is dequeued.

This is the **critical path method (CPM)** — a classic project scheduling algorithm.

---

## Example Walkthrough

**Example 2**: `n=5, relations=[[1,5],[2,5],[3,5],[3,4],[4,5]], time=[1,2,3,4,5]`

Initial: `indegree=[0,0,0,1,4]`, `earliest=[0,0,0,0,0]`

Queue: {0,1,2} (0-indexed)

- Process 0 (time=1): finish=1. Update 4: earliest[4]=max(0,1)=1
- Process 1 (time=2): finish=2. Update 4: earliest[4]=max(1,2)=2
- Process 2 (time=3): finish=3. Update 3: earliest[3]=3, Update 4: earliest[4]=max(2,3)=3
- Process 3 (time=4): finish=3+4=7. Update 4: earliest[4]=max(3,7)=7
- Process 4 (time=5): finish=7+5=12

Answer: max(0+1, 0+2, 0+3, 3+4, 7+5) = **12** ✓

---

## Pitfalls & Edge Cases

- **0-indexed vs 1-indexed**: relations use 1-indexed courses → convert with `r[0]-1`, `r[1]-1`
- **No prerequisites**: starts at 0 → correctly initialized in `earliest`
- **Multiple paths to same node**: `max` correctly takes the latest prerequisite finish

---

## Key Takeaway

"Minimum time with parallel execution and dependencies" → **topological sort + DP on earliest start times**:

```
earliest[v] = max over all u→v of (earliest[u] + time[u])
answer = max(earliest[i] + time[i])
```

This is the **Critical Path Method** — the answer equals the length of the longest path in the DAG (weighted by `time`).
