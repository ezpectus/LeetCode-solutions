# 1028. Recover a Tree From Preorder Traversal
## O(n) — Stack as Depth-Indexed Path

---

## Problem Statement

- A binary tree is serialized as a preorder DFS string where each node is prefixed by **D dashes** indicating its depth (root = depth 0).
- If a node has only one child, it is always the **left child**.
- Given the string `traversal`, reconstruct and return the root of the original tree.

---

## Core Idea — Stack as Depth-Indexed Path

**Key insight**: Use a stack where `stack[i]` = the last node seen at depth `i`. This effectively tracks the **current path from root to the most recent node**.

When we read a new node at depth `d`:
- Pop the stack down to size `d` (discard any nodes deeper than `d-1`)
- The new node's parent is now `stack[d-1]` (top of stack)
- Attach as **left child** if parent has no left child yet, otherwise **right child**
- Push the new node onto the stack

**Why this works?**
- Preorder traversal visits parent before children — so the parent is always already in the stack when we process a child
- Popping to depth `d` "backtracks" up the tree to the correct ancestor
- Stack size after pop = `d`, so `stack[d-1]` is always the direct parent

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public TreeNode RecoverFromPreorder(string traversal) {
        var stack = new List<TreeNode>(); // stack[i] = node at depth i
        int i = 0, n = traversal.Length;

        while (i < n) {
            // Count dashes = depth
            int depth = 0;
            while (i < n && traversal[i] == '-') { depth++; i++; }

            // Parse node value (may be multi-digit)
            int val = 0;
            while (i < n && char.IsDigit(traversal[i]))
                val = val * 10 + (traversal[i++] - '0');

            var node = new TreeNode(val);

            // Trim stack to current depth (backtrack up tree)
            while (stack.Count > depth) stack.RemoveAt(stack.Count - 1);

            // Attach to parent
            if (stack.Count > 0) {
                var parent = stack[stack.Count - 1];
                if (parent.left == null) parent.left = node;
                else parent.right = node;
            }

            stack.Add(node);
        }

        return stack[0]; // root is always first element
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Each character read once; each node pushed/popped at most once |
| **Space Complexity** | **O(h)** | Stack holds at most h nodes where h = tree height |

**Optimal** — single pass over the string.

## Why This Works — Example Walkthrough

**Example 1**: `"1-2--3--4-5--6--7"`

| Token | depth | val | Stack (depths) | Action |
|-------|-------|-----|----------------|--------|
| `1` | 0 | 1 | [] → [1] | root |
| `-2` | 1 | 2 | [1] → [1,2] | 1.left = 2 |
| `--3` | 2 | 3 | [1,2] → [1,2,3] | 2.left = 3 |
| `--4` | 2 | 4 | pop to 2 → [1,2] → [1,2,4] | 2.right = 4 |
| `-5` | 1 | 5 | pop to 1 → [1] → [1,5] | 1.right = 5 |
| `--6` | 2 | 6 | [1,5] → [1,5,6] | 5.left = 6 |
| `--7` | 2 | 7 | pop to 2 → [1,5] → [1,5,7] | 5.right = 7 |

Result: `[1,2,5,3,4,6,7]` ✓

---

## Pitfalls & Edge Cases

- **Multi-digit values** — e.g. `"1-401--349"`: parse full number with `val = val * 10 + digit` loop
- **Pop to exact depth** — trim stack to `Count == depth`, so `stack[depth-1]` is the parent; off-by-one here breaks the whole tree
- **Left before right** — problem guarantees single child = left child; always check `parent.left == null` first
- **Root is always `stack[0]`** — the first node pushed never gets popped (depth 0 means no trimming at the first step)
- **No dashes at start** — root has depth 0, so the dash-counting loop produces `depth = 0` immediately and the stack trim is a no-op

---

## Key Takeaway

Preorder string with depth-encoded dashes → **stack as depth-indexed ancestor path**:

- Stack invariant: `stack[d]` = last node visited at depth `d`
- **Pop to depth `d`** to backtrack up to the right parent
- **Attach** as left or right child depending on what's already filled
- **Push** new node to extend the path

Pattern: any tree traversal encoding with explicit depth info → stack tracking the current root-to-node path, trim on depth decrease.
