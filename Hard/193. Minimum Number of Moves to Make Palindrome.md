# 2193. Minimum Number of Moves to Make Palindrome
## O(n²) — Greedy Two-Pointer with Adjacent Swaps

---

## Problem Statement

- Swap adjacent characters to make `s` a palindrome.
- Return minimum number of swaps.

---

## Core Idea — Greedy: Fix Outermost Characters First

Use two pointers `left` and `right` starting from both ends. At each step:

**Case 1: `s[left] == s[right]`** → already matched, advance both pointers inward.

**Case 2: `s[left] != s[right]`** → need to find a match for `s[left]` from the right side:
- Search rightward from `right` toward `left` for a position `j` where `s[j] == s[left]`.
- If found (`j > left`): move `s[j]` to position `right` by swapping adjacent elements. Cost = `right - j` swaps. Then advance both pointers.
- If not found (`j == left`): `s[left]` is the unique middle character (odd-length palindrome). Swap it one step right (cost 1) — it will eventually settle in the center. Don't advance `left`/`right`.

**Why greedy is optimal?** Fixing the outermost unmatched characters first is always optimal. Delaying an outermost match can only increase cost since the character to move has a longer distance to travel.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinMovesToMakePalindrome(string s) {
        char[] arr = s.ToCharArray();
        int n = arr.Length, res = 0;
        int left = 0, right = n - 1;

        while (left < right) {
            if (arr[left] == arr[right]) {
                left++; right--;
            } else {
                // Find match for arr[left] from right side
                int j = right;
                while (j > left && arr[j] != arr[left]) j--;

                if (j == left) {
                    // Unique char = future middle: swap one step right
                    (arr[left], arr[left+1]) = (arr[left+1], arr[left]);
                    res++;
                    // Don't advance left/right
                } else {
                    // Bring arr[j] to position right
                    while (j < right) {
                        (arr[j], arr[j+1]) = (arr[j+1], arr[j]);
                        j++; res++;
                    }
                    left++; right--;
                }
            }
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | Each outer iteration scans up to n chars; n/2 iterations |
| **Space** | **O(n)** | Mutable character array |

With `n ≤ 2000`: at most 4M operations — fast.

## Example Walkthrough

**`s="letelt"`** → `['l','e','t','e','l','t']`

left=0, right=5: `s[0]='l'`, `s[5]='t'`. Search left from right for 'l': j=4. Move j(4) to right(5): swap (4,5) → `['l','e','t','e','t','l']`. res=1. left=1, right=4.

left=1, right=4: `s[1]='e'`, `s[4]='t'`. Search left for 'e': j=3. Move j(3) to right(4): swap (3,4) → `['l','e','t','t','e','l']`. res=2. left=2, right=3.

left=2, right=3: `s[2]='t'`, `s[3]='t'`. Match! left=3, right=2. Done.

Answer: **2** ✓ → "lettel"

---

## Pitfalls & Edge Cases

- **Unique middle character**: when `j == left`, `s[left]` is the only unpaired character (must go to center). Swapping it right by 1 and **not** advancing pointers is crucial — it will naturally be handled later.
- **Input guarantee**: problem guarantees `s` can always be made palindrome, so every character has a pair (except possibly one odd-count character).
- **Don't search for match on left side**: we always fix `s[left]` by searching rightward. Alternatively could search for `s[right]` from left, but both give the same result.

---

## Key Takeaway

"Min adjacent swaps to make palindrome" → **greedy: fix outermost mismatch first, bring right match to the right end**:

```
while left < right:
    if s[left] == s[right]: advance both
    else: find s[left]'s match at j (searching from right)
          if j == left: unique middle char, swap right by 1
          else: bring j to right (right-j swaps), advance both
```

O(n²) greedy with proven optimality. Each step fixes one outer pair with minimum cost.
