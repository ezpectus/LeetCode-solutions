# 3801. Minimum Cost to Merge Sorted Lists

## 1. Idea & Architectural Choice

### 1.1 Core Philosophy
The problem requires finding the minimum total cost to merge $K$ sorted lists into a single list, where $K = \text{lists.length} \le 12$. Merging any two lists $a$ and $b$ incurs a cost of:
`cost = len(a) + len(b) + abs(median(a) - median(b))`

Notice that the length and median of a merged list depend **solely on the multiset of initial lists** that compose it, regardless of the exact sequence of pairwise merges used to build it. Therefore, any intermediate merged list corresponds to a non-empty subset of the original $K$ lists, uniquely identified by a bitmask `mask` of length $K$.

This structure naturally fits **Dynamic Programming over Subsets (Bitmask DP)**:
- `dp[mask]`: The minimum total cost to merge the subset of lists represented by `mask` into a single combined list.
- For a single list (`popcount(mask) == 1`), `dp[mask] = 0` (no merges required).
- For a mask with multiple lists, we split `mask` into two non-empty disjoint submasks `submask` and `mask ^ submask`:
  `dp[mask] = min(dp[submask] + dp[mask ^ submask] + cost(submask, mask ^ submask))`

### 1.2 Mathematical Reduction & Precomputations
The cost function requires `len(mask)` and `median(mask)` for every valid `mask`:
1. `total_len[mask]`: The sum of lengths of all lists in `mask`.
2. `median[mask]`: The median element of the merged array formed by all lists in `mask`.
   - If `total_len[mask]` is $N$, the median is the 0-indexed element at position `target_k = (N - 1) / 2` in the combined sorted array.
   - Since individual lists are already sorted, we can precompute `median[mask]` for all $2^K - 1$ masks efficiently using binary search over values or K-way merged index searching.

### 1.3 Detailed DP Transition Logic
Let `submask` be a strict non-empty submask of `mask`.
- `len(submask) + len(mask ^ submask)` simplifies to `total_len[mask]`.
- The merge cost between `submask` and `mask ^ submask` is:
  `total_len[mask] + abs(median[submask] - median[mask ^ submask])`
- Transition equation:
  `dp[mask] = min over all submasks s of mask (dp[s] + dp[mask ^ s] + total_len[mask] + abs(median[s] - median[mask ^ s]))`

## 2. Dynamic Programming State & Submask Enumeration

### 2.1 State Variables
- `K`: Number of lists, where $2 \le K \le 12$.
- `total_len[mask]`: Total number of elements in the subset `mask`.
- `median[mask]`: Precomputed median value of the combined sorted elements in `mask`.
- `dp[mask]`: Minimum cost to assemble the combined list for `mask`.

### 2.2 Efficient Submask Iteration
To iterate over all submasks `s` of `mask` in $O(3^K)$ overall time, we use the standard bitwise trick:
`for (int s = (mask - 1) & mask; s > 0; s = (s - 1) & mask)`

To eliminate duplicate unordered pairs `(s, mask ^ s)` and avoid redundant calculations, we only consider submasks `s` that include the lowest set bit of `mask`:
`if ((s & (mask & -mask)) != 0)`

## 3. Full Optimal Implementation (C++)

```cpp
// Dynamic Programming + Bitmask + Binary Search
#include <vector>
#include <cmath>
#include <algorithm>
#include <climits>

class Solution {
private:
    // Helper function to count elements <= val across all lists in mask
    long long countLessOrEqual(long long val, int mask, const std::vector<std::vector<int>>& lists) {
        long long count = 0;
        for (int i = 0; i < lists.size(); ++i) {
            if ((mask & (1 << i)) != 0) {
                count += std::upper_bound(lists[i].begin(), lists[i].end(), val) - lists[i].begin();
            }
        }
        return count;
    }

    // Binary search to find the k-th smallest element (0-indexed) in the combined sorted arrays of mask
    int findKthElement(int mask, long long k, const std::vector<std::vector<int>>& lists) {
        long long low = -2000000000LL;
        long long high = 2000000000LL;
        int ans = high;

        while (low <= high) {
            long long mid = low + (high - low) / 2;
            if (countLessOrEqual(mid, mask, lists) >= k + 1) {
                ans = mid;
                high = mid - 1;
            } else {
                low = mid + 1;
            }
        }
        return ans;
    }

public:
    long long minMergeCost(std::vector<std::vector<int>>& lists) {
        int K = lists.size();
        int num_masks = 1 << K;

        std::vector<long long> total_len(num_masks, 0);
        std::vector<int> median_val(num_masks, 0);

        // Precompute total length and median for every mask
        for (int mask = 1; mask < num_masks; ++mask) {
            long long len_sum = 0;
            for (int i = 0; i < K; ++i) {
                if ((mask & (1 << i)) != 0) {
                    len_sum += lists[i].size();
                }
            }
            total_len[mask] = len_sum;

            long long target_k = (len_sum - 1) / 2;
            median_val[mask] = findKthElement(mask, target_k, lists);
        }

        std::vector<long long> dp(num_masks, LLONG_MAX);

        // Base cases: single lists require 0 cost
        for (int i = 0; i < K; ++i) {
            dp[1 << i] = 0;
        }

        // DP over mask size
        for (int mask = 1; mask < num_masks; ++mask) {
            // Skip base cases with 1 bit
            if ((mask & (mask - 1)) == 0) continue;

            int lowest_bit = mask & -mask;

            // Iterate through non-empty submasks containing lowest_bit
            for (int s = (mask - 1) & mask; s > 0; s = (s - 1) & mask) {
                if ((s & lowest_bit) == 0) continue;

                int other = mask ^ s;
                long long cost = total_len[mask] + std::abs((long long)median_val[s] - median_val[other]);
                
                if (dp[s] != LLONG_MAX && dp[other] != LLONG_MAX) {
                    dp[mask] = std::min(dp[mask], dp[s] + dp[other] + cost);
                }
            }
        }

        return dp[(1 << K) - 1];
    }
};
```

## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The total execution time consists of two distinct components:

1. **Precomputing Medians for all 2^K - 1 Masks**:
   - For each mask, total length calculation takes O(K).
   - Binary search range is [-2 * 10^9, 2 * 10^9], taking approximately 32 iterations.
   - Each binary search iteration counts elements across at most K sorted arrays via `std::upper_bound`, taking O(K log M), where M = max(lists[i].length) <= 500.
   - Precomputation time:
     T_pre = O(2^K * 32 * K log M)
     For K = 12 and M = 500, T_pre is roughly 4096 * 32 * 12 * 9 = 1.4 * 10^7 operations.

2. **Submask DP Transitions**:
   - Iterating through all submasks of all masks takes exactly O(3^K) operations.
   - For K = 12, 3^12 = 531441 submask transitions.
   - Each transition performs constant O(1) arithmetic updates.
   - DP execution time:
     T_dp = O(3^K)

**Total Time Complexity:**
T_total = T_pre + T_dp = O(2^K * K * log(RANGE) * log M + 3^K)

Under constraints (K <= 12, M <= 500), total operations are roughly 1.5 * 10^7, executing well within 50 milliseconds.

### 4.2 Detailed Space Complexity Decomposition
- **total_len Array**: Vector of size 2^K = 4096 storing 64-bit integers (~32 KB).
- **median_val Array**: Vector of size 2^K = 4096 storing 32-bit integers (~16 KB).
- **dp Table**: Vector of size 2^K = 4096 storing 64-bit integers (~32 KB).

**Total Space Complexity:**
S_total = O(2^K)

Total auxiliary memory is under 100 KB, easily satisfying typical memory limits.

### 4.3 Complexity Summary

| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(2^K * K * log(RANGE) * log M + 3^K) | K <= 12, M <= 500. Binary search for medians + Submask DP. |
| **Space Complexity** | O(2^K) | Exponential in K (up to 4096 entries for masks). |

## 5. Mathematical Formulations & Foundational Algorithms

### Submask Enumeration Theorem
The number of submasks across all masks of size K is given by:
Sum over i from 0 to K of (C(K, i) * 2^i) = (1 + 2)^K = 3^K

Proof by Binomial Theorem:
Each element j in {0, 1, ..., K-1} can be in one of three states relative to submask s and parent mask m:
1. j is not in m and j is not in s
2. j is in m and j is not in s
3. j is in m and j is in s

Since there are 3 independent choices for each of the K bits, the total number of valid pairs (m, s) where s is a submask of m is 3^K.

### Median Index Definition
For a sorted array A of size N:
- If N is odd, the median index is (N - 1) / 2.
- If N is even, the left middle element is chosen, which also corresponds to index (N - 1) / 2.

## Pitfalls & Edge Cases

* **Integer Overflow in DP Sums:** Merging large lists repeatedly can yield total costs exceeding 2^31 - 1. Using `long long` for `dp` and `total_len` prevents overflow.
* **Redundant Submask Pairs:** A naive submask loop checks both (s, mask ^ s) and (mask ^ s, s). Enforcing `(s & lowest_bit) != 0` cuts the inner loop iterations in half.
* **Binary Search Range Boundaries:** Array elements range from -10^9 to 10^9. The binary search range must safely cover [-2 * 10^9, 2 * 10^9] to prevent off-by-one errors near boundaries.

## Key Takeaway

When an optimal merge/partition hierarchy over a small set of items (K <= 16) depends on subset properties rather than history, precompute subset properties and solve via Bitmask DP with O(3^K) submask enumeration.

**Pattern recognition:** "minimum cost to merge small number of sorted lists/arrays" -> Bitmask Dynamic Programming + Submask Enumeration + Binary Search
