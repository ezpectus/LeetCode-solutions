# 2035. Partition Array Into Two Arrays to Minimize Sum Difference
## O(2^n × n) — Meet in the Middle

---

## Problem Statement

- Partition `2n` integers into two equal-size (`n` each) arrays.
- Minimize `|sum1 - sum2|`.
- `n ≤ 15` → brute force over all `C(30,15) ≈ 155M` is too slow. Use meet in the middle.

---

## Core Idea — Split, Enumerate, Binary Search

**Split**: divide `nums` into left half and right half, each of size `n`.

**Enumerate**: for each half, generate all possible subset sums grouped by **subset size** (how many elements are chosen from that half). If we pick `k` elements from the left half, we must pick `n-k` from the right half.

For left half: `left_sums[k]` = sorted list of all sums from choosing exactly `k` elements.
For right half: for each mask (subset of right half), let `j` = size of subset, `s_right` = subset sum. We need `n-j` elements from the left with sum `s_left`.

**Objective**: minimize `|2*(s_left + s_right) - total|`. So for fixed `s_right`, we want `s_left ≈ total/2 - s_right`. Binary search in `left_sums[n-j]` for this target.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumDifference(int[] nums) {
        int n = nums.Length / 2;
        long total = nums.Sum(x => (long)x);

        // Build left_sums[k] = sorted sums of k-element subsets from left half
        var leftSums = new List<long>[n+1];
        for (int i = 0; i <= n; i++) leftSums[i] = new List<long>();

        for (int mask = 0; mask < (1 << n); mask++) {
            int k = 0; long s = 0;
            for (int i = 0; i < n; i++)
                if ((mask >> i & 1) == 1) { k++; s += nums[i]; }
            leftSums[k].Add(s);
        }
        for (int i = 0; i <= n; i++) leftSums[i].Sort();

        long res = long.MaxValue;

        // Enumerate right half subsets
        for (int mask = 0; mask < (1 << n); mask++) {
            int j = 0; long sRight = 0;
            for (int i = 0; i < n; i++)
                if ((mask >> i & 1) == 1) { j++; sRight += nums[n+i]; }

            var candidates = leftSums[n-j];
            if (candidates.Count == 0) continue;

            // Binary search for target = total/2 - sRight
            double target = (total - 2.0 * sRight) / 2.0;
            int idx = BinarySearch(candidates, target);
            foreach (int bi in new[]{idx-1, idx}) {
                if (bi >= 0 && bi < candidates.Count) {
                    long sLeft = candidates[bi];
                    res = Math.Min(res, Math.Abs(2*(sLeft+sRight) - total));
                }
            }
        }

        return (int)res;
    }

    int BinarySearch(List<long> list, double target) {
        int lo = 0, hi = list.Count;
        while (lo < hi) {
            int mid = (lo+hi)/2;
            if (list[mid] < target) lo = mid+1; else hi = mid;
        }
        return lo;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(2^n × n + 2^n × log(2^n))** = **O(2^n × n)** | Enumeration + binary search |
| **Space** | **O(2^n)** | Subset sum storage |

With `n=15`: `2^15 = 32768` subsets per half. Total ≈ 2M operations — fast.

---

## Why Meet in the Middle?

Brute force: enumerate all `C(2n, n)` ways to pick `n` elements from `2n`. For `n=15`: `C(30,15) ≈ 155M` — too slow.

Meet in the middle: split into two halves. `2^15 = 32768` subsets per half. For each right half subset (pick `j` elements), binary search in left half subsets for the complementary count (`n-j`). Total work: `32768 × log(32768) ≈ 500K` — very fast.

## Example Walkthrough

**`nums=[3,9,7,3]`**, n=2. total=22. Left=[3,9], Right=[7,3].

left_sums[0]=[0], left_sums[1]=[3,9] (sorted), left_sums[2]=[12].

Right subsets:
- mask=00 (j=0,sRight=0): search left_sums[2]=[12]. target=11. Binary search→idx=0, candidates[0]=12. diff=|2*(12+0)-22|=2.
- mask=01 (j=1,sRight=7): search left_sums[1]=[3,9]. target=(22-14)/2=4. Binary search→idx=1 (9), idx-1=0 (3). min(|2*(3+7)-22|,|2*(9+7)-22|)=min(2,10)=2.
- mask=10 (j=1,sRight=3): search left_sums[1]. target=(22-6)/2=8. Close to 9→diff=|2*(9+3)-22|=2.
- mask=11 (j=2,sRight=10): search left_sums[0]=[0]. diff=|2*(0+10)-22|=2.

Answer: **2** ✓

## Pitfalls & Edge Cases

- **Group by count**: when picking `j` from right, must pair with exactly `n-j` from left. Grouping sums by count is essential.
- **Binary search checks both neighbors**: the target might fall between two sorted values — check `idx` and `idx-1`.
- **`long` for sums**: `n=15`, `nums[i]` up to `10^7` → max sum ≈ `1.5×10^8` fits in int, but use `long` for safety.
- **Empty candidates**: `n-j` might not be achievable if `n-j > n` or `n-j < 0` — handled by size check.

---

## Key Takeaway

"Min partition difference with 2n elements, n ≤ 15" → **meet in the middle**:

```
Split into two n-halves
left_sums[k] = sorted sums of k-element subsets from left
For each right subset (j elements, sum = sRight):
    binary search left_sums[n-j] for target = total/2 - sRight
    minimize |2*(s_left + s_right) - total|
```

Reduces O(C(2n,n)) to O(2^n × n) by splitting and using binary search to combine halves.
