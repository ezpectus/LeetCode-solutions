# 2959. Number of Possible Sets of Closing Branches

## O(2ᴺ * N³) — Bitmask Brute Force + Floyd-Warshall All-Pairs Shortest Path

### Problem Statement
There are `n` company branches connected by undirected roads. The company decides to close down some of these branches (possibly none). However, they want to ensure that all remaining active branches have a shortest path distance of at most `maxDistance` from each other. After closing a branch, any roads connected to it become inaccessible. Multiple roads between the same pair of branches are allowed.

Return the number of possible sets of branches to close such that the remaining active branches satisfy the distance constraint.

---

### Core Idea — Subset Bitmasking with Floyd-Warshall
The problem states that $1 \le n \le 10$. This tiny constraint is a massive hint that an exponential brute-force search combined with an all-pairs shortest path calculation is entirely optimal.

1. **Bitmask Enumeration:** There are $2^n$ possible subsets of active/closed branches. Since $2^{10} = 1024$, we can easily generate every possible state using an integer bitmask from $0$ to $2^n - 1$. A bit set to `1` indicates that the branch remains active, while `0` means it is closed.
2. **Graph Reconstruction per Mask:** For each subset, we initialize a 2D distance adjacency matrix. If multiple edges exist between two active nodes, we greedily keep only the minimum weight.
3. **Floyd-Warshall Verification:** We apply the Floyd-Warshall dynamic programming algorithm ($O(n^3)$) onto the isolated active subgraph to resolve the shortest paths between all active pairs.
4. **Validation:** After running the algorithm, we scan every pair of active nodes. If any pair has a shortest path value strictly greater than `maxDistance`, the current subset is invalid. If all pairs comply, we increment our valid counter.

---

### Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int NumberOfSets(int n, int maxDistance, int[][] roads) {
        int totalSubsets = 1 << n; // 2^n configurations
        int validSetsCount = 0;
        
        // Step 1: Iterate through all possible subsets of active nodes via Bitmask
        for (int mask = 0; mask < totalSubsets; mask++) {
            // Initialize adjacency matrix for the current subgraph configuration
            int[,] dist = new int[n, n];
            const int INF = 1000000000; // Fixed double-to-int compilation error
            
            for (int i = 0; i < n; i++) {
                for (int j = 0; j < n; j++) {
                    if (i == j) dist[i, j] = 0;
                    else dist[i, j] = INF;
                }
            }
            
            // Step 2: Populate the active subgraph boundaries
            foreach (var road in roads) {
                int u = road[0];
                int v = road[1];
                int w = road[2];
                
                // Keep the edge only if both nodes are active in the current bitmask
                if (((mask >> u) & 1) == 1 && ((mask >> v) & 1) == 1) {
                    dist[u, v] = Math.Min(dist[u, v], w);
                    dist[v, u] = Math.Min(dist[v, u], w);
                }
            }
            
            // Step 3: Run Floyd-Warshall DP to find all-pairs shortest paths
            for (int k = 0; k < n; k++) {
                if (((mask >> k) & 1) == 0) continue; // Skip closed pivot nodes
                
                for (int i = 0; i < n; i++) {
                    if (((mask >> i) & 1) == 0) continue;
                    
                    for (int j = 0; j < n; j++) {
                        if (((mask >> j) & 1) == 0) continue;
                        
                        if (dist[i, k] + dist[k, j] < dist[i, j]) {
                            dist[i, j] = dist[i, k] + dist[k, j];
                        }
                    }
                }
            }
            
            // Step 4: Validate shortest path lengths against maxDistance bounds
            bool isValidSubset = true;
            for (int i = 0; i < n; i++) {
                if (((mask >> i) & 1) == 0) continue;
                
                for (int j = i + 1; j < n; j++) {
                    if (((mask >> j) & 1) == 0) continue;
                    
                    if (dist[i, j] > maxDistance) {
                        isValidSubset = false;
                        break;
                    }
                }
                if (!isValidSubset) break;
            }
            
            if (isValidSubset) {
                validSetsCount++;
            }
        }
        
        return validSetsCount;
    }
}
````


### Complexity

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | $O(2^N \cdot (N^3 + R))$ | We evaluate $2^N$ states. In each state, parsing edges takes $O(R)$ and Floyd-Warshall calculations take $O(N^3)$. Max operations $\approx 1024 \times (1000 + 1000) \approx 2 \times 10^6$, executing instantly. |
| **Space Complexity** | $O(N^2)$ | The temporary 2D adjacency matrix scales to $10 \times 10$ cells per mask processing frame, achieving true memory compression. |

---

### Pitfalls & Edge Cases

* **Integer Arithmetic Overflow:** Do not use `int.MaxValue` for graph infinity state indicators. Adding two edge lengths equal to `int.MaxValue` inside the Floyd-Warshall relaxation block (`dist[i, k] + dist[k, j]`) will overflow into negative indices, corrupting your minimum comparisons. Keep `INF` capped safely at $10^9$.
* **Handling Multiple Edges Accurately:** The graph contains duplicate edges. When filtering roads for your subgraph allocations, do not blindly assign weights; always wrap them inside a `Math.Min` filter.
* **Empty Subsets:** If a mask selects $0$ or $1$ active node, it automatically qualifies as valid because no pairs exist to violate the `maxDistance` barrier. The validation loop naturally bypasses these states and tallies them correctly.

---

### Key Takeaway

> When an optimization objective implies altering or reducing graph nodes to fulfill full-grid proximity metrics, look at the constraints. If the structural size limits are minimal ($N \le 10$), combine Bitmask State Enumeration with a dense Floyd-Warshall engine.

* **Pattern recognition:** "Find valid combinations of subgraphs where all-pairs shortest paths must fall below a bound (Small N)" -> Integer Bitmask Loop + Subgraph Floyd-Warshall Engine.
