# 2188. Minimum Time to Finish the Race
## O(n Ã— L + tires Ã— 20) â€” Precompute Best Consecutive Times + DP

---

## Problem Statement

- Each tire `[f, r]` completes lap `x` (1-indexed consecutive) in `f * r^(x-1)` seconds.
- Can change tires (costs `changeTime` seconds) between any laps.
- Find minimum total time to complete `numLaps` laps.

---

## Core Idea â€” Two Phases

### Phase 1: Precompute `best[k]`

`best[k]` = minimum time to run exactly `k` consecutive laps using a single tire (choosing the best tire for this).

**Key bound**: it's never worth running more than ~20 consecutive laps with one tire. The lap time `f * r^(x-1)` grows exponentially. Once `f * r^(x-1) > f + changeTime` (i.e., this lap is more expensive than changing to a fresh tire and doing the first lap), it's better to change. Since `r â‰¥ 2` and `f, changeTime â‰¤ 10^5`, this happens by lap 17-18 at most.

So: for each tire, simulate laps 1, 2, ..., up to 20 (or until lap_time exceeds `f + changeTime`).

### Phase 2: DP on Lap Count

`dp[i]` = minimum time to complete `i` laps.

**Trick**: initialize `dp[0] = -changeTime`. Then the recurrence is:
```
dp[i] = min over k in [1..min(i,20)]:
    dp[i-k] + changeTime + best[k]
```

For the first segment (no preceding change), `dp[0] + changeTime + best[k] = -changeTime + changeTime + best[k] = best[k]` âœ“.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumFinishTime(int[][] tires, int changeTime, int numLaps) {
        const int MAX_CONSEC = 20;
        const int INF = int.MaxValue / 2;

        // Phase 1: best[k] = min time for k consecutive laps with best single tire
        int[] best = new int[MAX_CONSEC + 1];
        Array.Fill(best, INF);

        foreach (var tire in tires) {
            long f = tire[0], r = tire[1];
            long time = 0, lapTime = f;
            for (int k = 1; k <= MAX_CONSEC; k++) {
                time += lapTime;
                if (time > INF) break;
                best[k] = Math.Min(best[k], (int)time);
                lapTime *= r;
                if (lapTime > f + changeTime) break; // next lap worse than fresh start
            }
        }

        // Phase 2: DP
        int[] dp = new int[numLaps + 1];
        Array.Fill(dp, INF);
        dp[0] = -changeTime; // trick: absorb changeTime for first segment

        for (int i = 1; i <= numLaps; i++) {
            for (int k = 1; k <= Math.Min(i, MAX_CONSEC); k++) {
                if (best[k] < INF && dp[i-k] < INF)
                    dp[i] = Math.Min(dp[i], dp[i-k] + changeTime + best[k]);
            }
        }

        return dp[numLaps];
    }
}
```

## Why Max ~20 Consecutive Laps

After `x` consecutive laps, the x-th lap takes `f * r^(x-1)` seconds. It's better to change when this exceeds the cost of a fresh lap: `f * r^(x-1) > f + changeTime`.

With `r=2`, `f=1`, `changeTime=10^5`: `2^(x-1) > 10^5 + 1` â†’ `x > 17`. So max ~17 laps. We use 20 as a safe bound.

## Pitfalls & Edge Cases

- **`dp[0] = -changeTime` trick**: avoids special-casing the first segment. Clean and standard.
- **`lapTime > f + changeTime` break**: prunes tires early when consecutive use becomes suboptimal.
- **`long` for lap time computation**: `f * r^k` can overflow `int` quickly (r up to 10^5, k up to 20).
- **INF guard**: check both `best[k] < INF` and `dp[i-k] < INF` before updating.

---

## Key Takeaway

"Race with exponentially growing lap times and tire changes" â†’ **precompute best consecutive-k times + DP**:

```
best[k] = min single-tire time for k consecutive laps  (k â‰¤ 20)
dp[i] = min(dp[i-k] + changeTime + best[k])  for k âˆˆ [1..20]
dp[0] = -changeTime  (trick for first segment)
```

The ~20 consecutive lap bound makes both precomputation and DP efficient.
