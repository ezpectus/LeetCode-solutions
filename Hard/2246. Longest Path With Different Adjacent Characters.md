# 2246. Longest Path With Different Adjacent Characters
## O(n) — DFS with Top-2 Child Depths

---

## Problem Statement

- Tree rooted at 0. Each node has a character. Find the longest path where no two adjacent nodes share the same character.

---

## Core Idea — DFS Returning Best Downward Depth, Track Top-2 at Each Node

For each node `u`, `Dfs(u)` returns the length of the longest valid downward path starting at `u`.

At node `u`, for each child `c`:
- If `s[c] == s[u]`: this edge is invalid, skip.
- Otherwise: candidate depth = `Dfs(c)`.

Track the **top 2** valid child depths (`best1 ≥ best2`). The longest path **through** `u` = `best1 + best2 + 1` (combining two branches). Update global answer. Return `best1 + 1` (longest downward path from `u`).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int res = 1;
    List<int>[] children;
    string s;

    int Dfs(int node) {
        int best1 = 0, best2 = 0;
        foreach (int child in children[node]) {
            int depth = Dfs(child);
            if (s[child] == s[node]) continue; // invalid edge
            if (depth > best1) { best2 = best1; best1 = depth; }
            else if (depth > best2) best2 = depth;
        }
        res = Math.Max(res, best1 + best2 + 1);
        return best1 + 1;
    }

    public int LongestPath(int[] parent, string s) {
        int n = parent.Length;
        this.s = s;
        children = new List<int>[n];
        for (int i = 0; i < n; i++) children[i] = new();
        for (int i = 1; i < n; i++) children[parent[i]].Add(i);
        Dfs(0);
        return res;
    }
}
```

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — each node visited once |
| **Space** | **O(n)** — children lists + recursion stack |

## Example Walkthrough

**`parent=[-1,0,0,1,1,2], s="abacbe"`**

Tree: 0→{1,2}, 1→{3,4}, 2→{5}.

Dfs(3): no children → return 1.
Dfs(4): no children → return 1.
Dfs(1) ('b'): child 3 ('c'≠'b')→depth 1, child 4 ('b'=='b')→skip. best1=1, best2=0. res=max(1,2)=2. return 2.
Dfs(5): no children → return 1.
Dfs(2) ('a'): child 5 ('e'≠'a')→depth 1. best1=1. res=max(2,2)=2. return 2.
Dfs(0) ('a'): child 1 ('b'≠'a')→depth 2, child 2 ('a'=='a')→skip. best1=2,best2=0. res=max(2,3)=3. return 3.

Answer: **3** ✓ (path 0→1→3: a→b→c)

---

## Pitfalls & Edge Cases

- **Only top 2 depths needed**: the path through a node uses at most 2 branches (left-right).
- **Skip same-char children**: don't include them in best1/best2.
- **Single node**: `res` initialized to 1 handles the edge case where all paths have length 1.
- **Path is upward+downward**: `best1 + best2 + 1` counts both branches plus the node itself.

---

## Key Takeaway

"Longest path with different adjacent chars in tree" → **DFS: at each node, track top-2 valid child depths. Update res = best1+best2+1. Return best1+1**. Classic "diameter of tree" pattern adapted for character constraints.
