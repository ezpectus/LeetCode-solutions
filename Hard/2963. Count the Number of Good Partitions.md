# 2963. Count the Number of Good Partitions

## O(N) — Interval Merging via Last Positions + Bitwise Combinatorics

### Problem Statement
You are given a 0-indexed array `nums` consisting of positive integers. A partition of an array into one or more contiguous subarrays is called **good** if no two subarrays contain the same number.

Return the total number of good partitions of `nums`. Since the answer may be large, return it modulo $10^9 + 7$.

---

### Core Idea — Overlapping Intervals and Split Points
The core constraint dictates that identical values cannot reside in separate partitions. This implies that for any unique number, the entire span from its **first occurrence** to its **last occurrence** must belong to the exact same contiguous subarray. We can think of each unique number as an un-splittable interval.

1. **Mapping Boundaries:** We first scan the array to record the absolute last index position for each unique value.
2. **Greedy Interval Extension:** We iterate through the array using a sliding horizon tracker (`maxLastIndex`). As we move index `i` forward, we dynamically expand our non-splittable window:
   `maxLastIndex = Math.Max(maxLastIndex, lastPositionOf(nums[i]))`
3. **Identifying Independent Blocks:** If our iteration index `i` catches up to `maxLastIndex` (`i == maxLastIndex`), it marks the completion of an independent, self-contained block. Beyond this point, no numbers from inside this window will ever appear again. 
4. **Binary Choice Combination:** If the array is structurally forced into $k$ independent blocks, there are exactly $k - 1$ shared boundaries (or "slots") between them. At each slot, we have a binary choice: either place a partition cut or merge the adjacent blocks together. Thus, the total number of valid configurations is:
   $$\text{Total Partitions} = 2^{k - 1} \pmod{10^9 + 7}$$

### Full Optimal Implementation (C#)

```csharp
using System;
using System.Collections.Generic;

public class Solution {
    public int NumberOfGoodPartitions(int[] nums) {
        int n = nums.Length;
        const int MOD = 1000000007;
        
        // Step 1: Precompute the last occurrence of each unique number
        Dictionary<int, int> lastPositions = new Dictionary<int, int>();
        for (int i = 0; i < n; i++) {
            lastPositions[nums[i]] = i;
        }
        
        int independentBlocksCount = 0;
        int maxLastIndex = 0;
        
        // Step 2: Iterate and merge overlapping spans greedily
        for (int i = 0; i < n; i++) {
            maxLastIndex = Math.Max(maxLastIndex, lastPositions[nums[i]]);
            
            // If the iterator reaches the current bound, an un-splittable block is finalized
            if (i == maxLastIndex) {
                independentBlocksCount++;
            }
        }
        
        // Step 3: Compute 2^(independentBlocksCount - 1) % MOD via modular exponentiation
        return (int)Power(2, independentBlocksCount - 1, MOD);
    }
    
    private long Power(long baseVal, long exp, int mod) {
        long res = 1;
        baseVal %= mod;
        while (exp > 0) {
            if ((exp & 1) == 1) res = (res * baseVal) % mod;
            baseVal = (baseVal * baseVal) % mod;
            exp >>= 1;
        }
        return res;
    }
}
```

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | $O(N)$ | The dictionary population takes linear $O(N)$ time. The greedy window consolidation loop scans the array exactly once. The modular power operation takes logarithmic $O(\log k)$ steps. |
| **Space Complexity** | $O(N)$ | Driven by the storage requirements of the `lastPositions` lookup map containing at most $N$ unique key-value pairs. |

### Pitfalls & Edge Cases

* **Single Element Blocks:** If all elements in the array are identical (e.g., `[1, 1, 1, 1]`), `maxLastIndex` immediately scales to `n - 1` at index `0`. The loop completes with `independentBlocksCount = 1`, and $2^{1 - 1} = 2^0 = 1$ is correctly returned.
* **Modular Multiplicative Thresholds:** While $N \le 10^5$, the power exponents can grow large. Avoid standard integer overflows by computing the binary powering sequence inside a 64-bit `long` framework using standard modular reductions at each step.

---

### Key Takeaway

> When a partition constraint forces elements with shared traits into identical groups, model the element footprints as geometric intervals. Collect their boundary coordinates, compress overlapping zones greedily, and map the remaining components directly to binary subset configurations.

* **Pattern recognition:** "Count subarray partitions where identical elements cannot cross boundaries" -> Last Position Tracking + Greedy Component Isolation + Powers of 2.
