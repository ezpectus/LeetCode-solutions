# 2306. Naming a Company
## O(n · 26²) — Group by Suffix, Count Intersections per Letter-Pair

---

## Problem Statement

Given a list of ideas (strings), count ordered pairs `(ideaA, ideaB)` where swapping their first letters produces two strings that are **both absent** from the original list.

---

## Core Insight #1 — Reformulate: Which Suffix-Letter Swaps Are Valid?

Write each idea as `(first_letter, suffix)`. Swapping ideas `(a, s1)` and `(b, s2)` is valid iff:
- `b + s1` is NOT in ideas (letter b never appears with suffix s1)
- `a + s2` is NOT in ideas (letter a never appears with suffix s2)

Group suffixes by which first letters use them. For suffix `s`, define `letters[s]` = set of first letters appearing with s in ideas.

**Same-suffix pairs are always invalid:** If s1 = s2 = s, then a ∈ letters[s] (since (a, s) is in ideas), so the condition `a ∉ letters[s]` immediately fails. No need to check same-suffix pairs.

---

## Core Insight #2 — Reframe Per Letter-Pair: Count Exclusive Suffixes

For a fixed pair of letters `(c1, c2)`, we want ordered pairs (s1, s2) where:
- s1 appears with c1 but **not** c2 (so swapping gives c2+s1 which is new)
- s2 appears with c2 but **not** c1 (so swapping gives c1+s2 which is new)

Let:
```
A = set of suffixes paired with letter c1
B = set of suffixes paired with letter c2
```

Valid pairs = `(A − A∩B) × (B − A∩B)` = `(|A| − |A∩B|) × (|B| − |A∩B|)`

Since (ideaA, ideaB) and (ideaB, ideaA) are **distinct** ordered pairs:

```
contribution(c1, c2) = 2 × (|A| − |A∩B|) × (|B| − |A∩B|)
```

---

## Core Insight #3 — Compute All Intersections in One Suffix Pass

For each suffix s with its set of first letters (stored as a bitmask):
- Increment `cnt[c]` for each letter c in the mask.
- For each pair (c1, c2) both in the mask, increment `common[c1][c2]`.

Since the mask has at most 26 bits set, each suffix contributes at most C(26,2) = 325 pair increments. With up to 5×10^4 suffixes: at most 16.25M operations — trivially fast.

Then `common[c1][c2]` = `|A ∩ B|`, exactly what the formula needs.

---

## Full Implementation (C#)

```csharp
public class Solution {
    public long DistinctNames(string[] ideas) {
        // Step 1: For each suffix, record which first letters use it (as a bitmask)
        var suffixMask = new Dictionary<string, int>();
        foreach (string idea in ideas) {
            string suffix = idea.Substring(1);
            if (!suffixMask.ContainsKey(suffix)) suffixMask[suffix] = 0;
            suffixMask[suffix] |= 1 << (idea[0] - 'a');
        }

        // Step 2: Count per-letter suffix counts and pairwise intersections
        int[] cnt = new int[26];           // cnt[c] = |suffixes with letter c|
        int[,] common = new int[26, 26];  // common[c1][c2] = |A ∩ B|

        foreach (int mask in suffixMask.Values) {
            for (int c1 = 0; c1 < 26; c1++) {
                if ((mask & (1 << c1)) == 0) continue;
                cnt[c1]++;
                for (int c2 = c1 + 1; c2 < 26; c2++) {
                    if ((mask & (1 << c2)) == 0) continue;
                    common[c1, c2]++;
                    common[c2, c1]++;
                }
            }
        }

        // Step 3: Sum contributions from all 325 letter pairs
        long ans = 0;
        for (int c1 = 0; c1 < 26; c1++)
            for (int c2 = c1 + 1; c2 < 26; c2++) {
                long mut = common[c1, c2];
                ans += 2L * (cnt[c1] - mut) * (cnt[c2] - mut);
            }

        return ans;
    }
}
```

## Complexity

| Step | Time | Notes |
|---|---|---|
| Build suffix map | O(n · L) | n ideas, L ≤ 10 characters |
| Count intersections | O(S · 26²) | S ≤ n suffixes, 325 pairs each |
| Sum contributions | O(26²) | 325 pairs |
| **Total** | **O(n · 26²)** | ≈ 5×10⁴ × 676 = 33.8M ops |
| Space | O(n · L + 26²) | suffix map + common table |

## Example Walkthrough

### Example 1 — `ideas = ["coffee","donuts","time","toffee"]`

**Suffix → mask table:**
| Suffix | Letters present | Mask (bits c=2, d=3, t=19) |
|---|---|---|
| "offee" | c, t | bits 2, 19 |
| "onuts" | d | bit 3 |
| "ime" | t | bit 19 |

**cnt[] after pass:**

| Letter | Count |
|---|---|
| c | 1 (offee) |
| d | 1 (onuts) |
| t | 2 (offee, ime) |

**common[][] from suffix "offee" (has both c and t):**
- `common[c][t] = 1`, `common[t][c] = 1`

**Contributions (only 3 non-empty letter pairs):**

| Pair (c1, c2) | cnt[c1] | cnt[c2] | common | (cnt1−com)×(cnt2−com)×2 |
|---|---|---|---|---|
| (c, d) | 1 | 1 | 0 | 2 × 1 × 1 = **2** |
| (c, t) | 1 | 2 | 1 | 2 × 0 × 1 = **0** |
| (d, t) | 1 | 2 | 0 | 2 × 1 × 2 = **4** |

**Total = 2 + 0 + 4 = 6** ✓

**Verification of pair (d, t):** 4 ordered pairs from suffixes exclusive to d or t:
- d's exclusive: "onuts" (not in t's set)
- t's exclusive: "ime" (not in d's set)
- t also has "offee" but "offee" is NOT exclusive to t (also in c's set, not d's set)

Wait, for (d, t): A = {"onuts"}, B = {"offee", "ime"}, A∩B = {}. Exclusive pairs: 1 × 2 = 2, ×2 = 4.

Valid swaps: ("donuts","time"), ("time","donuts"), ("donuts","toffee"), ("toffee","donuts") = 4 pairs. ✓

## Why Same-Suffix Pairs Are Always Zero

For ideas `(a, s)` and `(b, s)` (same suffix s):
- After swap: (b, s) and (a, s). Both are already in ideas (each original idea has a unique suffix per letter). Contribution = 0.

In the formula: if s appears with both a and b, it's in A∩B, reducing both `(cnt[a]−common)` and `(cnt[b]−common)` by 1 for that overlap. The formula auto-excludes these.

---

## Pitfalls & Edge Cases

- **Order matters** (ideaA ideaB ≠ ideaB ideaA): the factor of 2 handles this. Don't confuse with "choose 2" (which would be C(n,2) without the factor).

- **Both names must be absent:** Missing even one condition makes the pair invalid. The formula requires BOTH s1 ∉ B and s2 ∉ A — both exclusivity conditions simultaneously.

- **All ideas start with the same letter:** All in one `cnt[c]` bucket, no cross-letter pairs, answer = 0.

- **`common[c1][c2] = common[c2][c1]`** must be maintained (symmetric). The inner loop only runs c2 > c1 but updates both directions.

- **`Substring(1)` for suffix:** idea[0] is the first letter, idea.Substring(1) is the suffix. For a single-character idea like "a", the suffix is "" (empty string). This is valid — the empty string can be a suffix key in the dictionary.

---

## Key Takeaway

This is a **group-by-suffix, count-via-letter-pair-intersections** pattern. The critical reduction: instead of comparing O(n²) pairs of ideas directly, group ideas by suffix (reducing to ≤ n suffix sets), then for each of the 325 letter pairs compute how many suffixes are exclusive to each letter. The "mutual" (shared suffix count) is the key quantity.

**Pattern recognition:** "Count swap-pairs with mutual exclusion on two sets" → define the sets (here: suffix sets per letter), compute pairwise intersections via a single scan, then apply inclusion-exclusion: `(|A|−|A∩B|) × (|B|−|A∩B|)`. The factor-2 handles ordered vs unordered pairs.
