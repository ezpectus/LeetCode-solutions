## 2916. Subarrays Distinct Element Sum of Squares II

### Problem Statement
You are given a 0-indexed integer array `nums`. The distinct count of a subarray `nums[i..j]` is the number of distinct values present in that range. Return the sum of the **squares** of distinct counts of all possible subarrays of `nums`.

Since the answer can be very large, return it modulo 10^9 + 7.

---

### Core Insight — Lazy Segment Tree & Incremental DP
A brute-force solution checking every subarray would take O(N^2) time, leading to a Time Limit Exceeded (TLE) due to the constraint N <= 10^5. We need a linearithmic O(N log N) approach.

Instead of processing subarrays independently, we track subarrays ending at a moving index j as we iterate from 0 to N - 1. Let D[i] represent the number of distinct elements in the subarray `nums[i..j]`. When we advance our right pointer from j - 1 to j by adding a new element x = nums[j]:

1. Find the previous occurrence of x, say at index `prev_idx`. If x has not appeared before, `prev_idx = -1`.
2. The presence of x at position j increases the distinct count D[i] by exactly 1 for all subarrays starting anywhere from index `prev_idx + 1` up to j. Subarrays starting at or before `prev_idx` already included x, so their distinct counts do not change.
3. Therefore, expanding the right boundary corresponds to a **range update operation**: incrementing D[i] by 1 for all i in range `[prev_idx + 1, j]`.

#### Managing Squares Dynamically
We need to track the sum of squares of these values: sum(D[i]^2). 
When a value D[i] increases by 1, its squared contribution transitions as follows:
(D[i] + 1)^2 = D[i]^2 + 2 * D[i] + 1

Summing this algebraic identity over a range [L, R] where every element is incremented by 1 gives:
sum((D[i] + 1)^2) = sum(D[i]^2) + 2 * sum(D[i]) + sum(1)

This structure matches the capabilities of a **Segment Tree with Lazy Propagation**. By storing both the sum of elements and the sum of squares inside each segment tree node, we can process range additions and query total global sums in O(log N) time per element.

---

### Algorithm
1. Create a Segment Tree where each node maintains `sumOfSquares` and `sumOfElements`.
2. Use a dictionary or array `lastPos` initialized to `-1` to store the last seen index of every number.
3. Maintain a running `totalGlobalSum` variable.
4. Loop through the array with an index j from 0 to N - 1:
   * Retrieve `prev_idx = lastPos[nums[j]]`.
   * Perform a lazy range addition of +1 over the interval `[prev_idx + 1, j]`.
   * Query the total sum of squares from the root node representing the full `[0, j]` sweep space, adding this value directly to `totalGlobalSum`.
   * Update `lastPos[nums[j]] = j`.
5. Return `totalGlobalSum` modulo 10^9 + 7.

### Full Optimal Implementation (C#)
```csharp
using System;
using System.Collections.Generic;

public class Solution {
    private const long MOD = 1_000_000_007;

    private class SegmentTree {
        private int n;
        private long[] sumSq;
        private long[] sumEl;
        private long[] lazy;

        public SegmentTree(int n) {
            this.n = n;
            sumSq = new long[4 * n];
            sumEl = new long[4 * n];
            lazy = new long[4 * n];
        }

        private void Push(int node, int start, int end) {
            if (lazy[node] == 0) return;

            long val = lazy[node];
            int mid = start + (end - start) / 2;
            int leftChild = 2 * node;
            int rightChild = 2 * node + 1;

            // Apply lazy value to left child
            long leftLen = mid - start + 1;
            sumSq[leftChild] = (sumSq[leftChild] + 2 * val * sumEl[leftChild] + val * val % MOD * leftLen) % MOD;
            sumEl[leftChild] = (sumEl[leftChild] + val * leftLen) % MOD;
            lazy[leftChild] = (lazy[leftChild] + val) % MOD;

            // Apply lazy value to right child
            long rightLen = end - mid;
            sumSq[rightChild] = (sumSq[rightChild] + 2 * val * sumEl[rightChild] + val * val % MOD * rightLen) % MOD;
            sumEl[rightChild] = (sumEl[rightChild] + val * rightLen) % MOD;
            lazy[rightChild] = (lazy[rightChild] + val) % MOD;

            lazy[node] = 0;
        }

        public void Update(int node, int start, int end, int l, int r, long val) {
            if (r < start || end < l) return;

            if (l <= start && end <= r) {
                long len = end - start + 1;
                sumSq[node] = (sumSq[node] + 2 * val * sumEl[node] + val * val % MOD * len) % MOD;
                sumEl[node] = (sumEl[node] + val * len) % MOD;
                lazy[node] = (lazy[node] + val) % MOD;
                return;
            }

            Push(node, start, end);
            int mid = start + (end - start) / 2;
            Update(2 * node, start, mid, l, r, val);
            Update(2 * node + 1, mid + 1, end, l, r, val);

            sumSq[node] = (sumSq[2 * node] + sumSq[2 * node + 1]) % MOD;
            sumEl[node] = (sumEl[2 * node] + sumEl[2 * node + 1]) % MOD;
        }

        public long QueryTotal() {
            return sumSq[1];
        }
    }

    public int SumCounts(int[] nums) {
        int n = nums.Length;
        SegmentTree tree = new SegmentTree(n);
        
        // Tracks the most recent index of each value
        int[] lastPos = new int[100001];
        Array.Fill(lastPos, -1);

        long totalGlobalSum = 0;

        for (int j = 0; j < n; j++) {
            int prevIdx = lastPos[nums[j]];
            
            // Add 1 to distinct counts for all valid starting positions
            tree.Update(1, 0, n - 1, prevIdx + 1, j, 1);
            
            // Collect the total sum of squares for all subarrays ending at j
            totalGlobalSum = (totalGlobalSum + tree.QueryTotal()) % MOD;
            
            lastPos[nums[j]] = j;
        }

        return (int)((totalGlobalSum + MOD) % MOD);
    }
}
```

### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(N log N) | The outer loop runs N times. Inside, each range update down the Segment Tree takes O(log N) operations due to structural split bounds. |
| **Space** | O(N + M) | The internal arrays for the tree structure scale up to 4 * N items. Tracking coordinates via `lastPos` takes constant space bounded by the maximum potential element value M (10^5). |

### Example Walkthrough
#### `nums = [1,2,1]`
* **j = 0 (`nums[0] = 1`):** `prevIdx = -1`. Update range `[0, 0]` with +1.
  - Distinct array state: `D = [1, 0, 0]`
  - Sum of squares: 1^2 = 1. `totalGlobalSum = 1`.
* **j = 1 (`nums[1] = 2`):** `prevIdx = -1`. Update range `[0, 1]` with +1.
  - Distinct array state: `D = [2, 1, 0]`
  - Sum of squares: 2^2 + 1^2 = 5. `totalGlobalSum = 1 + 5 = 6`.
* **j = 2 (`nums[2] = 1`):** `prevIdx = 0`. Update range `[1, 2]` with +1.
  - Distinct array state: `D = [2, 2, 1]`
  - Sum of squares: 2^2 + 2^2 + 1^2 = 9. `totalGlobalSum = 6 + 9 = 15`.

**Total Output:** `15`. ✓

### Pitfalls & Edge Cases
* **Modulo Operations on Reductions:** Subtraction equations during expansion sweeps can yield negative results in languages like C#. Guard your final evaluations using explicit buffer wrappers like `(total + MOD) % MOD`.
* **Lazy Value Scale Expansion:** When updating nodes via `val * val * len`, make sure to evaluate operations inside 64-bit bounds before reducing to avoid unexpected truncation bugs.

### Key Takeaway
When a problem asks for an aggregate property (like a sum of squares) across all subarrays, look for an incremental transformation from index j - 1 to j. If the underlying values update predictably over a continuous range, you can use a segment tree with lazy propagation to optimize the solution from O(N^2) to O(N log N).
