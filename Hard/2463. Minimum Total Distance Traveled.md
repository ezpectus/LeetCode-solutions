# 2463. Minimum Total Distance Traveled
## O(n × M) — Sort + Expand Factory Slots + DP

---

## Problem Statement

- Robots and factories on X-axis. Each factory has a capacity (`limit`).
- Assign each robot to a factory (within capacity) to minimize total distance traveled.
- Robots always move toward their assigned factory.

---

## Core Idea — Sort + Non-Crossing Assignment + DP

**Key theorem**: in any optimal assignment, assignments never "cross". That is, if robot `i < j` (sorted by position), they never go to factories in reversed order. This is provable by exchange argument — crossing assignments can always be uncrossed without increasing total distance.

**Algorithm**:
1. Sort robots and factories by position.
2. **Expand** each factory with limit `L` into `L` individual slots (each slot can take exactly 1 robot).
3. **DP**: `dp[i][j]` = minimum total distance to assign robots `0..i-1` using slots `0..j-1`.

**Transition**:
- **Skip slot j**: `dp[i][j] = dp[i][j-1]` (don't assign robot `i` to slot `j`)
- **Use slot j for robot i**: `dp[i][j] = dp[i-1][j-1] + |robot[i-1] - slots[j-1]|`

Take the minimum of both options.

**Base case**: `dp[0][j] = 0` for all `j` (no robots, no distance).

**Answer**: `dp[n][m]` where `n` = robots count, `m` = total slots.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinimumTotalDistance(IList<int> robot, int[][] factory) {
        var robots = robot.OrderBy(x => x).ToList();
        var facts = factory.OrderBy(f => f[0]).ToList();

        var slots = new List<int>();
        foreach (var f in facts)
            for (int i = 0; i < f[1]; i++)
                slots.Add(f[0]);

        int n = robots.Count, m = slots.Count;
        long[,] dp = new long[n+1, m+1];
        const long INF = long.MaxValue / 2;

        for (int i = 1; i <= n; i++) dp[i, 0] = INF;
        // dp[0, j] = 0 (default)

        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= m; j++) {
                dp[i, j] = dp[i, j-1]; // skip slot j
                if (dp[i-1, j-1] < INF)
                    dp[i, j] = Math.Min(dp[i, j],
                        dp[i-1, j-1] + Math.Abs((long)robots[i-1] - slots[j-1]));
            }
        }

        return dp[n, m];
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × M)** | n = robots, M = total slots = Σ limit_j ≤ 100×100=10000 |
| **Space** | **O(n × M)** | DP table; can be optimized to O(M) with rolling array |

With `n, M ≤ 100`: at most 10,000 states — instant.

---

## Why Non-Crossing is Optimal (Exchange Argument)

Suppose robot `i` (pos `ri`) goes to slot `a` (pos `pa`) and robot `j` (pos `rj`, `rj > ri`) goes to slot `b` (pos `pb < pa`). Then:

Cost = `|ri - pa| + |rj - pb|`

After swap (i→b, j→a): `|ri - pb| + |rj - pa|`

Since `ri ≤ rj` and `pb ≤ pa`, by the rearrangement inequality, the non-crossed version `≤` crossed version. So uncrossing never increases cost.

## Example Walkthrough

**`robot=[0,4,6], factory=[[2,2],[6,2]]`** → sorted same.

Slots = [2, 2, 6, 6] (factory 1 has limit 2 → two slots at pos 2; factory 2 → two at pos 6).
n=3, m=4.

DP (simplified):
- `dp[1][1]` = dp[0][0] + |0-2| = 2. `dp[1][2]` = min(dp[1][1], dp[0][1]+|0-2|) = 2.
- `dp[2][2]` = min(dp[2][1], dp[1][1]+|4-2|) = min(INF, 2+2) = 4.
- `dp[3][4]` = ... ultimately = 4 (robot 0→slot1(pos2), robot 1→slot2(pos2), robot 2→slot3(pos6)).

Answer: **4** ✓

## Pitfalls & Edge Cases

- **`dp[i][0] = INF` for i ≥ 1**: can't assign robots with 0 slots.
- **`long` for distances**: positions up to `10⁹`, sum can reach `10¹¹`, overflows `int`.
- **Slot expansion**: treat each factory slot independently. Slots from the same factory at the same position are identical — order doesn't matter.
- **`INF / 2`**: prevents overflow when adding distances to INF values.

---

## Key Takeaway

"Assign robots to factories minimizing total distance" → **sort both, expand factory slots, DP on (robots, slots)**:

```
dp[i][j] = min(
    dp[i][j-1],          // skip slot j
    dp[i-1][j-1] + dist  // assign robot i to slot j
)
```

Non-crossing property (from exchange argument) enables the O(n×M) DP. Expanding factory capacities into individual slots simplifies the DP to a clean 1-to-1 matching problem.
