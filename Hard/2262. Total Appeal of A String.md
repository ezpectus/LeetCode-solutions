# 2262. Total Appeal of A String
## O(n) — Contribution per Character with Running Sum

---

## Problem Statement

- Appeal of a string = number of distinct characters. Return sum of appeals of all substrings.

---

## Core Idea — Track Each Character's Contribution

For each position `i`, define `cur` = sum of appeals of all substrings ending at `i`.

When we move from `i-1` to `i` by appending `s[i]`:
- Every substring ending at `i` either: already contained `s[i]` (same appeal) or gains `s[i]` as a new character (+1 appeal).
- Substrings that gain +1: those starting from `last[c]+1` to `i` (where `last[c]` = last occurrence of `s[i]` before `i`).

So `cur += i - last[c]` (count of substrings where `s[i]` is first occurrence).

Update `res += cur` at each step.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long AppealSum(string s) {
        int n = s.Length;
        int[] last = new int[26];
        Array.Fill(last, -1);
        long res = 0, cur = 0;

        for (int i = 0; i < n; i++) {
            int c = s[i] - 'a';
            cur += i - last[c]; // new distinct contribution
            res += cur;
            last[c] = i;
        }
        return res;
    }
}
```

**Time**: O(n) — **Space**: O(1).

## Why cur = sum of appeals of substrings ending at i?

At position `i`, all substrings ending here are `s[0..i], s[1..i], ..., s[i..i]`. Their total appeal = `cur`. When we move to `i+1`, each of those substrings gains 1 appeal if `s[i+1]` is new to them. `s[i+1]` is new to `s[j..i+1]` iff `j > last[c]`. Number of such substrings = `i+1 - (last[c]+1) = i - last[c]`. Plus the new single-char substring `s[i+1..i+1]` with appeal 1 (also counted by the formula when `last[c]=-1`).

---

## Key Takeaway

"Total appeal of all substrings" → **running sum: `cur` = total appeal of substrings ending at i. `cur += i - last[c]` when adding character c. `res += cur`**. O(n) with O(26) space.
