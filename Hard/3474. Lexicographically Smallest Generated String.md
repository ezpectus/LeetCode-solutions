# 3474. Lexicographically Smallest Generated String
## O(n×m) — Force T Positions, Fill 'a', Fix F Violations

---

## Problem Statement

- Build string `word` of length `n+m-1`.
- For each `str1[i] == 'T'`: `word[i..i+m-1]` must equal `str2`.
- For each `str1[i] == 'F'`: `word[i..i+m-1]` must NOT equal `str2`.
- Return lexicographically smallest valid word, or `""` if impossible.

---

## Core Idea — Three Phases

### Phase 1: Enforce T Constraints

For each `T` position `i`, write `str2` into `word[i..i+m-1]`. If two `T` windows conflict (require different characters at the same position) → return `""`. Mark these positions as **forced**.

### Phase 2: Fill Remaining with 'a'

Any unset position gets `'a'` — lexicographically smallest possible.

### Phase 3: Fix F Violations

For each `F` position `i`, check if `word[i..i+m-1] == str2`. If yes, we must change one character in that window to break the equality.

**Which character to change?**
- Find the **rightmost non-forced position** in the window (forced positions can't be changed).
- Change it to `'a'` if `str2[j] != 'a'`, else to `'b'`.
- Why rightmost? Minimizes lexicographic impact — earlier characters stay as `'a'` (or whatever was forced), only the last mutable position changes.
- If no mutable position exists → return `""` (all positions forced by T, can't change).

**Why left-to-right F processing works?** When we change a position for F window `i`, it may affect future F windows to the right. But we check each window independently — if a later window becomes equal to `str2` due to our change, we fix it when we reach it. This cascades safely because we always have at least one mutable position per window (unless all forced → impossible).

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string GenerateString(string str1, string str2) {
        int n = str1.Length, m = str2.Length, total = n + m - 1;
        char[] word = new char[total];
        bool[] forced = new bool[total];

        // Phase 1: Force T positions
        for (int i = 0; i < n; i++) {
            if (str1[i] == 'T') {
                for (int j = 0; j < m; j++) {
                    int pos = i + j;
                    if (forced[pos] && word[pos] != str2[j]) return "";
                    word[pos] = str2[j];
                    forced[pos] = true;
                }
            }
        }

        // Phase 2: Fill unforced with 'a'
        for (int i = 0; i < total; i++)
            if (!forced[i]) word[i] = 'a';

        // Phase 3: Fix F violations
        for (int i = 0; i < n; i++) {
            if (str1[i] == 'F') {
                bool equal = true;
                for (int j = 0; j < m && equal; j++)
                    if (word[i+j] != str2[j]) equal = false;

                if (equal) {
                    bool changed = false;
                    for (int j = m - 1; j >= 0; j--) {
                        int pos = i + j;
                        if (!forced[pos]) {
                            word[pos] = str2[j] != 'a' ? 'a' : 'b';
                            changed = true;
                            break;
                        }
                    }
                    if (!changed) return "";
                }
            }
        }

        return new string(word);
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n×m)** | Phase 1: O(n×m) forced writes; Phase 3: O(n×m) checks |
| **Space** | **O(n+m)** | word and forced arrays |

With `n=10⁴`, `m=500`: ~5×10⁶ operations — runs in ~0.1s.

## Example Walkthrough

**`str1="TFTF", str2="ab"`** (n=4, m=2, total=5)

**Phase 1** (T positions: 0, 2):
- i=0: word[0]='a', word[1]='b'; forced=[T,T,F,F,F]
- i=2: word[2]='a', word[3]='b'; forced=[T,T,T,T,F]

**Phase 2**: word[4]='a'; word="ababa"

**Phase 3** (F positions: 1, 3):
- i=1: word[1..2]="ba" ≠ "ab" ✓
- i=3: word[3..4]="ba" ≠ "ab" ✓

Output: **"ababa"** ✓

---

## Pitfalls & Edge Cases

- **Two T windows conflicting**: same position, different required characters → `""`
- **F window fully forced**: all positions in `[i, i+m-1]` are forced by T windows, and they happen to equal `str2` → impossible → `""`
- **`'a'` if `str2[j] != 'a'` else `'b'`**: ensures we change to a character different from `str2[j]`, and `'a'` is preferred for lexicographic minimality

---

## Key Takeaway

"Smallest word satisfying T=match, F=no-match constraints" → **force T windows, fill 'a', fix F violations by changing rightmost mutable position**:

```
Phase 1: T windows → hard constraints
Phase 2: fill 'a' → minimize lexicographic value
Phase 3: F checks → change last mutable position in violating windows
```

Processing F windows left-to-right ensures cascading fixes are handled correctly.
