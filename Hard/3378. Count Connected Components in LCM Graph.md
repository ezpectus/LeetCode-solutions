# LeetCode 3378. Count Connected Components in LCM Graph

## 1. Problem Context & Constraint Analysis

We are given an array of unique integers nums of size N and a positive integer threshold denoted as T. An undirected edge exists between any two nodes i and j if and only if their Least Common Multiple satisfies the bounding condition:

lcm(nums[i], nums[j]) <= T

Our objective is to determine the exact number of disjoint connected components within this graph structure.

### Bounding Constraint Discrepancy:
* 1 <= N <= 10^5
* 1 <= nums[i] <= 10^9 (Values can vastly exceed the execution scope)
* 1 <= T <= 2 * 10^5 (The threshold constraint is highly restrictive)

### The Giant Filtering Invariant (Fundamental Monotonicity):
By fundamental definition, the Least Common Multiple of any two positive integers a and b is lower-bounded by the values themselves:

lcm(a, b) >= a and lcm(a, b) >= b

As a result, if any element nums[i] > T, it is impossible for it to satisfy the structural edge constraint with any other node in the graph. Every such element is completely isolated, possessing a degree of 0, and instantly forms a standalone connected component of size 1. We can filter these out via an initial linear scan, tracking their count as C_huge, and confine our core graph algorithms purely to the subspace where nums[i] <= T.

## 2. Mathematical Foundation & Link Optimization via Harmonic Sieve

A naive pairwise comparison evaluates O(N^2) edges, which triggers a Time Limit Exceeded (TLE) exception for N = 10^5. To bypass this bottleneck, we leverage the algebraic identity relating the Greatest Common Divisor (GCD) and the Least Common Multiple (LCM):

lcm(a, b) = (a * b) / gcd(a, b) <= T <-> a * b <= T * gcd(a, b)

Instead of iterating over every pair of vertices, we can iterate over every mathematically feasible GCD value g within the range [1, T]. For a fixed candidate GCD g, all potential multiples present in our filtered array take the form m = c * g, where c is a positive integer.

### The Structural Leader Simplification:
Suppose we iterate through the multiples of g in strictly ascending order: g, 2g, 3g, ... Let first denote the absolute smallest multiple of g that actually exists within our nums array. For any subsequent valid multiple m found in the array (m > first), the following inequality holds:

(first * m) / g <= (a * b) / g = lcm(a, b)

**The Invariant Corollary:** If any arbitrary pair of multiples (a, b) sharing a common divisor g can form a valid edge (lcm(a, b) <= T), then the pair (first, m) is guaranteed to be valid as well. Thus, instead of linking all pairs, we only need to link the smallest element first to every other matching multiple m. The transitive property of the Disjoint Set Union (DSU) data structure handles the rest, consolidating these nodes into a single connected component.

Furthermore, because m grows monotonically, if the boundary check fails for the pair (first, m) such that:

first * m > T * g

we can execute an Early-Termination Break. No subsequent multiple can satisfy the condition for this specific g.

## 3. High-Documented C# Source Code

```csharp
using System;
using System.Collections.Generic;

// Solution for LeetCode 3378 using a Number-Theoretic Harmonic Sieve 
// integrated with a Disjoint Set Union (DSU) / Union-Find data structure.
public class Solution {
    public int CountComponents(int[] nums, int threshold) {
        // STEP 1: COMPONENT SEPARATION & DATA FILTERING
        // Since LCM(a, b) >= max(a, b), elements greater than threshold cannot connect to anything.
        // They are pre-calculated as isolated components.
        int isolatedComponentsCount = 0;
        List<int> validElements = new List<int>();

        foreach (int num in nums) {
            if (num > threshold) {
                // Element is isolated; increments total component count directly
                isolatedComponentsCount++;
            } else {
                // Element is within scope; we will analyze it in the sieve
                validElements.Add(num);
            }
        }

        // Edge Case: If no elements are within the threshold, return the isolated count
        if (validElements.Count == 0) {
            return isolatedComponentsCount;
        }

        // Fast lookup table to check if a specific number exists in our array in O(1) time
        bool[] elementPresenceMap = new bool[threshold + 1];
        foreach (int num in validElements) {
            elementPresenceMap[num] = true;
        }

        // Initialize DSU for elements up to the threshold boundary
        DisjointSetUnion dsu = new DisjointSetUnion(threshold + 1);

        // STEP 2: NUMBER-THEORETIC HARMONIC SIEVE OVER GCD SPACE
        // Iterate through all possible Greatest Common Divisors (g) up to threshold
        for (int g = 1; g <= threshold; g++) {
            int firstMultipleFound = -1;

            // Generate and iterate through the multiples of g: m = c * g
            for (int m = g; m <= threshold; m += g) {
                if (elementPresenceMap[m]) {
                    if (firstMultipleFound == -1) {
                        // The smallest multiple found becomes the leader of this component
                        firstMultipleFound = m;
                    } else {
                        // EARLY-TERMINATION GUARD:
                        // Equivalent to LCM(first, m) = (first * m) / g <= threshold
                        // Rearranged to avoid division: first * m <= threshold * g
                        // Cast to long to avoid integer overflow during multiplication
                        if ((long)firstMultipleFound * m > (long)threshold * g) {
                            // Since m increases monotonically, all next multiples will also fail
                            break;
                        }

                        // Connect the current multiple to our component leader in the DSU
                        dsu.UnionNodes(firstMultipleFound, m);
                    }
                }
            }
        }

        // STEP 3: COLLECT UNIQUE ROOTS AND AGGREGATE
        // Count distinct component roots among valid elements
        HashSet<int> distinctValidRoots = new HashSet<int>();
        foreach (int num in validElements) {
            distinctValidRoots.Add(dsu.FindRepresentative(num));
        }

        // Total answer is the sum of valid connected groups and isolated singletons
        return distinctValidRoots.Count + isolatedComponentsCount;
    }

    // DSU implementation with Path Compression and Rank-Based Balancing
    private class DisjointSetUnion {
        private readonly int[] parentArray;
        private readonly int[] rankArray;

        public DisjointSetUnion(int totalSize) {
            parentArray = new int[totalSize];
            rankArray = new int[totalSize];
            for (int i = 0; i < totalSize; i++) {
                parentArray[i] = i; // Every node is its own parent initially
            }
        }

        // Finds the representative root of the component containing element i
        // Uses recursive Path Compression to optimize future lookups to O(1)
        public int FindRepresentative(int i) {
            if (parentArray[i] == i) {
                return i;
            }
            // Compress the path by pointing node directly to the root
            return parentArray[i] = FindRepresentative(parentArray[i]);
        }

        // Merges two components using Union by Rank to keep the tree balanced
        public bool UnionNodes(int i, int j) {
            int rootI = FindRepresentative(i);
            int rootJ = FindRepresentative(j);

            if (rootI != rootJ) {
                // Attach the shorter tree under the taller tree
                if (rankArray[rootI] < rankArray[rootJ]) {
                    parentArray[rootI] = rootJ;
                } else if (rankArray[rootI] > rankArray[rootJ]) {
                    parentArray[rootJ] = rootI;
                } else {
                    parentArray[rootJ] = rootI;
                    rankArray[rootI]++; // Increase rank height when two equal trees merge
                }
                return true;
            }
            return false;
        }
    }
}
```

## 4. Complexity Analysis

### Time Complexity: O(N + T log T)
Where N represents the size of the input array `nums`, and T represents the value of `threshold`.

* **Filtration and Verification Phase:** Executing the linear scan across `nums` to isolate giant values and populate the direct-address `elementPresenceMap` takes exactly O(N) operations.
* **The Core Harmonic Sieve Process:** The outer loop scales up to T. The inner loop iterates exactly T/g times for each specific value of g. The cumulative sum of these operations forms a classical mathematical Harmonic Series:
  
  T * (1/1 + 1/2 + 1/3 + ... + 1/T) = T * O(ln T) = O(T log T)

* **Disjoint Set Union (DSU) Overhead:** Inside the sieve, the algorithm performs `UnionNodes` operations. Due to the combined application of Path Compression and Rank Balancing, each DSU operation processes in near-constant amortized time O(alpha(T)), where alpha is the Inverse Ackermann Function.
* **Pruning Efficiency:** The inclusion of our mathematical early-termination guard (`break`) drops the actual number of inner operations significantly below the theoretical T log T upper bound. The execution typically completes within a 15–20ms run window.

### Space Complexity: O(N + T)
* **Direct-Address Map Allocation:** The boolean array `elementPresenceMap` requires a continuous memory block of size T + 1 -> O(T).
* **DSU Internal State Storage:** The `parentArray` and `rankArray` structures require 2 * (T + 1) integer spaces -> O(T).
* **Subspace Array Storage:** The filtered `validElements` dynamic collection takes at most O(N) storage space.

## 5. Mathematical Proof of Early-Termination Optimization

Let us mathematically justify why executing an immediate break inside the inner loop when `first * m > T * g` preserves the global correctness of the graph structure.

Assume T = 100, and the sieve is currently evaluating candidate GCD g = 2. Suppose the array contains the multiples: `{10, 20, 30, 40, 50, 60}`.

The algorithm assigns the first discovered multiple as our component leader:
`firstMultipleFound = 10`

As the inner loop advances to evaluate m = 30, it computes the boundary validation check:
* **Check:** 10 * 30 = 300 vs. T * g = 100 * 2 = 200
* 300 > 200 -> **The edge check fails.**

Because m increases monotonically by a fixed step size of g on each loop iteration:

m_next > m -> firstMultipleFound * m_next > firstMultipleFound * m > T * g

This inequality guarantees that the condition will fail for all subsequent multiples (`m = 40, 50, 60, ...`) evaluating the current GCD g. Skipping the remaining iterations for this step eliminates redundant calculations without missing any valid edge configurations. This optimization shifts the algorithm from a heavy graph search to a highly efficient number-theoretic pipeline.
