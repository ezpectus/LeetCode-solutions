# 2646. Minimize the Total Price of the Trips
## O(n·m + n) — Path Counting + Max Weight Independent Set on Tree

---

## Problem Statement

- Tree with n nodes, each having a price. Perform all trips (unique tree paths).
- Before trips: choose a set of **non-adjacent** nodes to **halve** their prices.
- Minimize the total price sum across all trips.

---

## Two-Phase Approach

**Phase 1 — Count node visits:** For each trip [start, end], find the unique tree path and increment `cnt[v]` for each visited node v.

**Phase 2 — Choose which nodes to halve (maximum weight independent set on tree):**

The baseline cost = `Σ price[v] * cnt[v]`.

Halving node v **saves** `price[v] * cnt[v] / 2`. To maximize savings, pick a maximum-weight independent set where node v has weight `price[v] * cnt[v] / 2`.

**Tree DP for max-weight independent set:**
```
dp[v][0] = max savings in v's subtree when v is NOT halved
dp[v][1] = max savings in v's subtree when v IS halved

dp[v][0] = Σ max(dp[c][0], dp[c][1])    for each child c
dp[v][1] = (price[v]*cnt[v]/2) + Σ dp[c][0]
```

Answer = `baseline - max(dp[root][0], dp[root][1])`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int n;
    List<int>[] adj;

    public int MinimumTotalPrice(int n, int[][] edges, int[] price, int[][] trips) {
        this.n = n;
        adj = new List<int>[n];
        for (int i = 0; i < n; i++) adj[i] = new List<int>();
        foreach (var e in edges) {
            adj[e[0]].Add(e[1]);
            adj[e[1]].Add(e[0]);
        }

        // Phase 1: count node visits across all trips
        int[] cnt = new int[n];
        foreach (var trip in trips) {
            foreach (int v in GetPath(trip[0], trip[1]))
                cnt[v]++;
        }

        // Phase 2: max weight independent set (tree DP)
        int baseline = 0;
        for (int v = 0; v < n; v++) baseline += price[v] * cnt[v];

        var (notH, halved) = Dp(0, -1, cnt, price);
        return baseline - Math.Max(notH, halved);
    }

    List<int> GetPath(int start, int end) {
        int[] par = new int[n];
        Array.Fill(par, -1);
        bool[] seen = new bool[n];
        var q = new Queue<int>();
        q.Enqueue(start);
        seen[start] = true;

        while (q.Count > 0) {
            int u = q.Dequeue();
            if (u == end) break;
            foreach (int v in adj[u]) {
                if (!seen[v]) {
                    seen[v] = true;
                    par[v] = u;
                    q.Enqueue(v);
                }
            }
        }

        var path = new List<int>();
        for (int cur = end; cur != -1; cur = par[cur])
            path.Add(cur);
        return path;
    }

    // Returns (savings_if_not_halved, savings_if_halved)
    (int, int) Dp(int v, int p, int[] cnt, int[] price) {
        int notH = 0, halved = price[v] * cnt[v] / 2;

        foreach (int u in adj[v]) {
            if (u == p) continue;
            var (cn, ch) = Dp(u, v, cnt, price);
            notH   += Math.Max(cn, ch); // v not halved: child can be either
            halved += cn;               // v halved: child must NOT be halved
        }

        return (notH, halved);
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(m·n + n) | m trips × O(n) BFS each; O(n) tree DP |
| Space Complexity | O(n) | adj list, cnt, DP stack |

n ≤ 50, m ≤ 100 → trivially fast.

## Example Walkthrough

**Example 1:** `n=4, price=[2,2,10,6], trips=[[0,3],[2,1],[2,3]]`

Tree: 0—1—2 and 1—3.

**Phase 1 paths:**
- [0→3]: 0,1,3 → cnt: 0:1, 1:1, 3:1
- [2→1]: 2,1 → cnt: 2:1, 1:1
- [2→3]: 2,1,3 → cnt: 2:1, 1:1, 3:1

Final cnt: 0:1, 1:3, 2:2, 3:2.

baseline = 2·1 + 2·3 + 10·2 + 6·2 = 2+6+20+12 = **40**.

**Phase 2 savings (price·cnt/2):** 0→1, 1→3, 2→10, 3→6.

DP (root=0):
```
Dp(2) → (0, 10)       [leaf, value=10]
Dp(3) → (0, 6)        [leaf, value=6]
Dp(1): notH = max(0,10)+max(0,6)=16; halved = 3+0+0=3 → (16, 3)
Dp(0): notH = max(16,3)=16; halved = 1+16=17 → (16, 17)
```

Max savings = 17. Answer = 40 − 17 = **23** ✓

*(Optimal: halve nodes 0 (save 1), 2 (save 10), 3 (save 6). Total saved = 17.)*

---

**Example 2:** `n=2, price=[2,2], trips=[[0,0]]`

Trip [0→0]: path = [0]. cnt: 0:1, 1:0.

baseline = 2·1 + 2·0 = 2.

Dp(1) → (0, 0). Dp(0): notH=0; halved=1+0=1 → (0,1).

Max savings = 1. Answer = 2 − 1 = **1** ✓

## Pitfalls & Edge Cases

- **start == end:** path is a single node [start]. BFS sets `par[end] = -1` so the path-building loop runs once. ✓

- **prices are even:** `price[v] * cnt[v] / 2` is always an integer. ✓

- **cnt[v] = 0 for some nodes:** halving such a node gives 0 savings — the DP handles this naturally. ✓

- **Adjacency constraint (non-adjacent):** enforced by `halved += cn` (child must be in `not_halved` state when parent is halved). ✓

---

## Key Takeaway

Decompose into:
1. **Counting phase** — how many times is each node used? (BFS paths, O(m·n))
2. **Optimization phase** — which non-adjacent nodes to halve to maximize savings? (Tree DP = Maximum Weight Independent Set on Tree, O(n))

**Pattern recognition:** "Choose subset of non-adjacent nodes on a tree to maximize some property" → always solvable in O(n) with `dp[v][0/1]` = value of subtree when v is in/out of the chosen set.
