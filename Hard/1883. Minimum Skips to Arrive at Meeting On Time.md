# 1883. Minimum Skips to Arrive at Meeting On Time
## O(n²) — DP with Integer Arithmetic (Avoid Floating Point)

---

## Problem Statement

- Travel `n` roads at `speed` km/h. After each road (except the last), must wait until the next integer hour.
- **Skip**: don't wait after a road — continue immediately.
- Return minimum skips to arrive within `hoursBefore` hours, or -1 if impossible.

---

## Core Idea — DP in Units of `1/speed` Hours

**Avoid floating point**: multiply all times by `speed`. In these units:
- Road `i` takes `dist[i]` time units
- One hour = `speed` units
- After road `i` without skip: round up to the nearest multiple of `speed`
- After road `i` with skip: no rounding

**State**: `dp[j]` = minimum time (in units of `1/speed`) after processing the current road using exactly `j` skips.

**Transition** for road `i` with `j` previous skips:
```
t = dp[j] + dist[i]

No skip (i != last):  new_dp[j]   = ceil(t / speed) * speed
Skip:                 new_dp[j+1] = t
Last road (no rest):  new_dp[j]   = t
```

**Answer**: smallest `j` where `dp[j] ≤ hoursBefore * speed`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinSkips(int[] dist, int speed, int hoursBefore) {
        int n = dist.Length;
        long INF = long.MaxValue / 2;
        long[] dp = new long[n + 1];
        Array.Fill(dp, INF);
        dp[0] = 0;

        for (int i = 0; i < n; i++) {
            long[] ndp = new long[n + 1];
            Array.Fill(ndp, INF);
            for (int j = 0; j <= i; j++) {
                if (dp[j] == INF) continue;
                long t = dp[j] + dist[i];

                if (i == n - 1) {
                    // Last road: no rest
                    ndp[j] = Math.Min(ndp[j], t);
                } else {
                    // No skip: round up to next multiple of speed
                    long rounded = (t + speed - 1) / speed * speed;
                    ndp[j] = Math.Min(ndp[j], rounded);
                    // Skip: don't round
                    if (j + 1 <= n)
                        ndp[j+1] = Math.Min(ndp[j+1], t);
                }
            }
            dp = ndp;
        }

        for (int j = 0; j <= n; j++)
            if (dp[j] <= (long)hoursBefore * speed)
                return j;
        return -1;
    }
}
```

## Why Integer Arithmetic is Essential

Naive float DP fails due to precision: `ceil(1.4 + 0.6) = ceil(2.0) = 2` but floating point might give `ceil(1.9999999...) = 2` or `ceil(2.0000001) = 3`. 

By multiplying everything by `speed`, all intermediate values are integers — `ceil(t/speed)*speed` becomes `(t + speed - 1) / speed * speed` using integer division, which is exact.

**Potential overflow**: `dist[i] ≤ 10⁵`, `speed ≤ 10⁶`, `n ≤ 1000`. Max time ≈ `n × max_dist × speed = 1000 × 10⁵ × 10⁶ = 10¹⁴` → needs `long`.

---

## Example Walkthrough

**`dist=[1,3,2], speed=4, hoursBefore=2`**

Work in units of `1/4` hour. `hoursBefore * speed = 8`.

**Road 0** (dist=1):
- j=0: t=1. No skip: ceil(1/4)*4=4 → ndp[0]=4. Skip: ndp[1]=1.
- dp=[4, 1, ∞, ∞]

**Road 1** (dist=3):
- j=0: t=4+3=7. No skip: ceil(7/4)*4=8 → ndp[0]=8. Skip: ndp[1]=7.
- j=1: t=1+3=4. No skip: ceil(4/4)*4=4 → ndp[1]=min(7,4)=4. Skip: ndp[2]=4.
- dp=[8, 4, 4, ∞]

**Road 2** (last, dist=2):
- j=0: t=8+2=10 → ndp[0]=10
- j=1: t=4+2=6 → ndp[1]=6
- j=2: t=4+2=6 → ndp[2]=6
- dp=[10, 6, 6, ∞]

Check: j=0: 10>8 ✗; j=1: 6≤8 ✓ → **1 skip** ✓

## Pitfalls & Edge Cases

- **Float precision** → use integer arithmetic multiplied by `speed`
- **`long` for dp values** → max value ~10¹⁴, overflows `int`
- **`(t + speed - 1) / speed * speed`** → integer ceiling formula
- **Last road**: no rest after it, so just add `dist[n-1]` without rounding

---

## Key Takeaway

"Minimum skips to stay under time budget with mandatory ceiling waits" → **DP on (road, skips_used) with integer arithmetic**:

```
no skip: time = ceil(curr / speed) * speed  (integer: (t+speed-1)/speed*speed)
skip:    time = curr (exact, no rounding)
```

Multiplying by `speed` converts fractional hours to integers — eliminates all floating point issues. This trick works whenever all fractional parts have a common denominator.
