# 1278. Palindrome Partitioning III
## O(n³) — Two-Layer DP: Palindrome Cost + Partition DP

---

## Problem Statement

- Change minimum characters in `s` so it can be split into exactly `k` palindromic substrings.
- Return the minimum total changes.

---

## Core Idea — Two Separate DP Tables

**Layer 1 — Palindrome cost table** `cost[i][j]`:
Minimum changes to make `s[i..j]` a palindrome. Built by interval DP:

```
cost[i][j] = cost[i+1][j-1] + (s[i] != s[j] ? 1 : 0)
```

Compare outer characters: if they differ, 1 change needed; recurse inward. Base cases: single char = 0, two chars = `s[i]!=s[j]?1:0`.

**Layer 2 — Partition DP** `dp[i][p]`:
Minimum changes to split `s[0..i-1]` into exactly `p` palindromes.

```
dp[i][p] = min over j in [p-1, i):  dp[j][p-1] + cost[j][i-1]
```

"The last palindrome is `s[j..i-1]`, and the first `j` characters are optimally split into `p-1` palindromes."

**Base case**: `dp[0][0] = 0` (empty string, 0 parts, 0 cost).

**Answer**: `dp[n][k]`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int PalindromePartition(string s, int k) {
        int n = s.Length;

        // Layer 1: cost[i][j] = min changes to make s[i..j] palindrome
        int[,] cost = new int[n, n];
        for (int len = 2; len <= n; len++) {
            for (int i = 0; i <= n - len; i++) {
                int j = i + len - 1;
                cost[i, j] = cost[i+1, j-1] + (s[i] != s[j] ? 1 : 0);
            }
        }

        // Layer 2: dp[i][p] = min changes for s[0..i-1] into p palindromes
        int[,] dp = new int[n+1, k+1];
        for (int i = 0; i <= n; i++)
            for (int p = 0; p <= k; p++)
                dp[i, p] = int.MaxValue / 2;
        dp[0, 0] = 0;

        for (int i = 1; i <= n; i++) {
            for (int p = 1; p <= Math.Min(i, k); p++) {
                for (int j = p-1; j < i; j++) { // last part: s[j..i-1]
                    if (dp[j, p-1] < int.MaxValue / 2)
                        dp[i, p] = Math.Min(dp[i, p], dp[j, p-1] + cost[j, i-1]);
                }
            }
        }

        return dp[n, k];
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n³)** | cost table: O(n²); partition DP: O(n²k) ≤ O(n³) since k≤n |
| **Space Complexity** | **O(n²)** | cost table dominates |

`n ≤ 100` → at most 10⁶ operations, trivially fast.

---

## Why Interval DP for Palindrome Cost

The recurrence `cost[i][j] = cost[i+1][j-1] + (s[i]!=s[j])` works because:
- To make `s[i..j]` a palindrome, the outer characters `s[i]` and `s[j]` must match — cost 0 or 1.
- The inner substring `s[i+1..j-1]` must also be a palindrome — solved recursively.
- Changes to outer and inner are **independent** (fixing outer doesn't affect inner and vice versa).

**Fill order**: by increasing length, so `cost[i+1][j-1]` (shorter substring) is always ready.

## Example Walkthrough

**Example 1**: `s = "abc"`, `k = 2`

**cost table**:
```
cost[0][0]=0 (a), cost[1][1]=0 (b), cost[2][2]=0 (c)
cost[0][1]: s[0]='a'≠s[1]='b' → cost[1][0]+1 = 0+1 = 1  (ab)
cost[1][2]: s[1]='b'≠s[2]='c' → 0+1 = 1  (bc)
cost[0][2]: s[0]='a'≠s[2]='c' → cost[1][1]+1 = 0+1 = 1  (abc)
```

**dp table** (n=3, k=2):

`dp[0][0]=0`, rest=∞

| i | p | j | dp[j][p-1] + cost[j][i-1] | dp[i][p] |
|---|---|---|--------------------------|----------|
| 1 | 1 | 0 | dp[0][0]+cost[0][0]=0+0=0 | **0** |
| 2 | 1 | 0 | dp[0][0]+cost[0][1]=0+1=1 | **1** |
| 2 | 2 | 1 | dp[1][1]+cost[1][1]=0+0=0 | **0** |
| 3 | 1 | 0 | dp[0][0]+cost[0][2]=0+1=1 | **1** |
| 3 | 2 | 1 | dp[1][1]+cost[1][2]=0+1=1 | **1** |
| 3 | 2 | 2 | dp[2][1]+cost[2][2]=1+0=1 | **1** |

`dp[3][2] = 1` ✓ — split as "ab"|"c", change 1 char in "ab".

## Pitfalls & Edge Cases

- **`cost[i+1][j-1]` when `i+1 > j-1`** (length ≤ 2): for length 2, `i+1=j` so `cost[j][j-1]` is out of bounds conceptually — initialize all `cost[i][j]=0` for `i≥j`, which is automatic with default 0-init
- **`dp[0][0]=0`, all others = ∞** — base case: empty string needs 0 parts, 0 cost
- **`k = n`** — each character is its own palindrome → 0 changes (every single char is a palindrome); `dp[n][n]=0` ✓
- **`j` lower bound = `p-1`** — need at least `p-1` characters before the last part (to form `p-1` non-empty palindromes)

---

## Key Takeaway

"Minimum changes to split string into k palindromes" → **two-layer DP**:

1. **Interval DP** for palindrome cost: `cost[i][j] = cost[i+1][j-1] + (s[i]≠s[j])`
2. **Partition DP** using cost as a lookup: `dp[i][p] = min_j(dp[j][p-1] + cost[j][i-1])`

The separation of concerns is key — precompute all palindrome costs, then solve the partition problem independently. Classic pattern for string partition problems with non-trivial substring costs.
