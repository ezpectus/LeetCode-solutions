## 3123. Find Edges in Shortest Paths

### Problem Statement
You are given a 0-indexed 2D integer array `edges` representing an undirected weighted graph of `n` nodes numbered from `0` to `n - 1`, where `edges[i] = [ai, bi, wi]` indicates an edge between nodes `ai` and `bi` with weight `wi`.

Consider all the shortest paths from node `0` to node `n - 1`. Find a boolean array `answer` where `answer[i]` is `true` if `edges[i]` is part of at least one shortest path. Otherwise, `answer[i]` is `false`.

---

### Core Insight — Bidirectional Dijkstra Verification
An edge `u-v` with weight `w` belongs to at least one shortest path from a source node `S` to a target node `T` if and only if the shortest distance from `S` to `T` equals the distance passing exactly through this edge. 

Since the graph is undirected, the path could cross the edge from `u` to `v` or from `v` to `u`. Therefore, the validation condition is:
`distFromSrc[u] + w + distFromDst[v] == totalShortestPath` 
**OR** `distFromSrc[v] + w + distFromDst[u] == totalShortestPath`

To achieve this in `O(M log N)` time, we execute Dijkstra's algorithm twice:
* **First Pass:** Calculate the shortest distance from node `0` to all other nodes (`distFromSrc`).
* **Second Pass:** Calculate the shortest distance from node `n - 1` to all other nodes (`distFromDst`).

If `distFromSrc[n - 1]` is unreachable (remains infinity), no path exists, and we can immediately return an array of `false`. Otherwise, we iterate through every edge and check if it satisfies the structural match equality using our precomputed distance arrays.

---

### Algorithm
1. Build an adjacency list representing the undirected graph. Each adjacency item should include the neighbor node, weight, and its original index from the `edges` array.
2. Implement a standard Dijkstra function that takes a starting node and returns a long array of minimum distances.
3. Compute `distFromSrc = Dijkstra(0)` and `distFromDst = Dijkstra(n - 1)`.
4. Store the minimum total distance: `totalShortestPath = distFromSrc[n - 1]`.
5. If `totalShortestPath` equals infinity, return the default `false` boolean array.
6. Initialize a boolean array `answer` of size `m`. Loop through each edge `i` containing `(u, v, w)`:
   * If `distFromSrc[u] + w + distFromDst[v] == totalShortestPath`, set `answer[i] = true`.
   * If `distFromSrc[v] + w + distFromDst[u] == totalShortestPath`, set `answer[i] = true`.
7. Return `answer`.

---

### Full Optimal Implementation (C#)
```csharp
using System;
using System.Collections.Generic;

public class Solution {
    public bool[] FindAnswer(int n, int[][] edges) {
        int m = edges.Length;
        bool[] answer = new bool[m];
        
        // Build adjacency list: node -> List of (neighbor, weight, originalEdgeIndex)
        List<(int neighbor, int weight, int index)>[] adj = new List<(int, int, int)>[n];
        for (int i = 0; i < n; i++) {
            adj[i] = new List<(int, int, int)>();
        }
        
        for (int i = 0; i < m; i++) {
            int u = edges[i][0];
            int v = edges[i][1];
            int w = edges[i][2];
            adj[u].Add((v, w, i));
            adj[v].Add((u, w, i));
        }
        
        // Compute shortest paths from both ends
        long[] distFromSrc = Dijkstra(0, n, adj);
        long[] distFromDst = Dijkstra(n - 1, n, adj);
        
        long shortestPath = distFromSrc[n - 1];
        
        // If target is unreachable, no edges can be part of a shortest path
        if (shortestPath >= long.MaxValue / 2) {
            return answer;
        }
        
        // Verify each edge against the bidirectional shortest path metric
        for (int i = 0; i < m; i++) {
            int u = edges[i][0];
            int v = edges[i][1];
            long w = edges[i][2];
            
            if ((distFromSrc[u] + w + distFromDst[v] == shortestPath) || 
                (distFromSrc[v] + w + distFromDst[u] == shortestPath)) {
                answer[i] = true;
            }
        }
        
        return answer;
    }
    
    private long[] Dijkstra(int start, int n, List<(int neighbor, int weight, int index)>[] adj) {
        long[] dist = new long[n];
        Array.Fill(dist, long.MaxValue / 2); // Avoid overflow when adding distances later
        dist[start] = 0;
        
        // Priority Queue stores pairs of (distance, node) sorted by distance ascending
        PriorityQueue<int, long> pq = new PriorityQueue<int, long>();
        pq.Enqueue(start, 0);
        
        while (pq.Count > 0) {
            pq.TryDequeue(out int u, out long d);
            
            if (d > dist[u]) continue;
            
            foreach (var edge in adj[u]) {
                int v = edge.neighbor;
                int w = edge.weight;
                
                if (dist[u] + w < dist[v]) {
                    dist[v] = dist[u] + w;
                    pq.Enqueue(v, dist[v]);
                }
            }
        }
        
        return dist;
    }
}
```
### Complexity
| Metric | Value | Notes |
|---|---|---|
| **Time** | O(M log N) | The complexity is driven by the two executions of Dijkstra's algorithm using a min-priority queue, where `N` is the number of nodes and `M` is the number of edges. |
| **Space** | O(N + M) | The adjacency list consumes O(N + M) space. The priority queue and distance arrays allocate tracking records up to O(N). |

### Pitfalls & Edge Cases
* **Disconnected Graphs:** If node `n - 1` is completely isolated or unlinked from node `0`, `distFromSrc[n - 1]` will hold our initialized infinity value. Make sure you check this upfront to avoid overflow bugs.
* **Large Int Accumulations:** While individual edge weights are relatively small, path sums can accumulate across 50,000 segments. Store your working distance computations in `long` variables rather than standard `int` primitives to avoid precision wrapping.

### Key Takeaway
When you need to discover if a structural element (like a node or edge) sits on a shortest path between two targets, run pathfinding checks from both directions. The target values computed from the source and destination will combine to create an invariant equation that can validate any matching item in O(1) constant time.
