# 2484. Count Palindromic Subsequences

## O(n) — Two-Pass with Fixed Outer Digits

---

## Problem Statement

- Given string `s` of digits, count palindromic subsequences of length 5.
- Return count modulo 10⁹+7.
- **Constraints**: 1 ≤ s.length ≤ 10⁴, s contains only digits

---

## Core Idea — Fix Outer Two Digits, Count Middle

**Palindrome structure**: ABCBA
- Fix outer pair: A and B (positions 0,1 and 3,4)
- Middle C (position 2) can be anything

**Key insight**: For each digit pair (A,B):
- Count how many "AB..." patterns exist going left-to-right
- Count how many "...BA" patterns exist going right-to-left
- Multiply counts at each position to get palindromes

**Example**: For pair (1,0) in "103301"
- "10..." patterns: positions ending at index 1
- "...01" patterns: positions starting from later indices
- Middle can be any character between them

---

## Algorithm Breakdown

### For Each Digit Pair (outer1, outer2)

**Pass 1: Scan Left-to-Right**
```
Track:
  - outer1Count: how many times outer1 appeared so far
  - pairCount: how many (outer1, outer2) pairs seen so far

At each position i:
  leftPairs[i] = pairCount before position i
  
  If s[i] == outer2:
    pairCount += outer1Count  // Found new pair
  If s[i] == outer1:
    outer1Count++
```

**Pass 2: Scan Right-to-Left**
```
Track:
  - outer1Count: how many outer1 from right
  - pairCount: how many (outer2, outer1) pairs from right

At each position i:
  rightPairs = pairCount from right of i
  
  result += leftPairs[i-1] * rightPairs
  
  If s[i] == outer2:
    pairCount += outer1Count
  If s[i] == outer1:
    outer1Count++
```

## Full Implementation (C#)

```csharp
public class Solution {
    private const int MOD = 1_000_000_007;
    
    public int CountPalindromes(string s) {
        int n = s.Length;
        long result = 0;
        
        // Try all digit pairs (outer1, outer2) for positions 0,1 and 3,4
        for (char outer1 = '0'; outer1 <= '9'; outer1++) {
            for (char outer2 = '0'; outer2 <= '9'; outer2++) {
                // leftPairs[i] = count of (outer1, outer2) pairs in s[0..i]
                long[] leftPairs = new long[n];
                
                long pairCount = 0;
                long outer1Count = 0;
                
                // Scan left-to-right to build leftPairs
                for (int i = 0; i < n; i++) {
                    leftPairs[i] = pairCount;
                    
                    char ch = s[i];
                    
                    // If we see outer2, complete pairs with all previous outer1
                    if (ch == outer2) {
                        pairCount += outer1Count;
                    }
                    
                    // Track count of outer1
                    if (ch == outer1) {
                        outer1Count++;
                    }
                }
                
                // Reset for right-to-left scan
                pairCount = 0;
                outer1Count = 0;
                
                // Scan right-to-left to count palindromes
                for (int i = n - 1; i >= 3; i--) {
                    char ch = s[i];
                    
                    // Update counts (process current character)
                    if (ch == outer2) {
                        pairCount += outer1Count;
                    }
                    if (ch == outer1) {
                        outer1Count++;
                    }
                    
                    // rightPairs = pairCount from right of position i
                    // leftPairs[i-1] = pairs to the left of position i
                    // Multiply to get palindromes with middle at positions between
                    result = (result + pairCount * leftPairs[i - 1]) % MOD;
                }
            }
        }
        
        return (int)result;
    }
}
```

| **Metric** | **Value** |

| **Space** | **O(n)** |

100 = 10×10 digit pairs (constant factor).

## Why This Works — Detailed Example

### Example: `s = "103301"`, pair (1,0)

**Pass 1 (left-to-right)**:
```
i=0, s[0]='1': outer1Count=1, pairCount=0, leftPairs[0]=0
i=1, s[1]='0': outer2 seen, pairCount=0+1=1, leftPairs[1]=0
i=2, s[2]='3': leftPairs[2]=1
i=3, s[3]='3': leftPairs[3]=1
i=4, s[4]='0': outer2 seen, pairCount=1+1=2, leftPairs[4]=1
i=5, s[5]='1': outer1Count=2, leftPairs[5]=2
```

**Pass 2 (right-to-left)**:
```
i=5, s[5]='1': outer1Count=1, pairCount=0
i=4, s[4]='0': pairCount=0+1=1
  result += 1 * leftPairs[3] = 1 * 1 = 1
i=3, s[3]='3':
  result += 1 * leftPairs[2] = 1 * 1 = 2
... continue
```

Final result considers all digit pairs.

---

## Key Variables Explained

**leftPairs[i]**: How many (outer1, outer2) pairs exist in substring s[0..i]
- Example: leftPairs[3] = 1 means one "10" pair before index 3

**pairCount** (left-to-right): Running count of (outer1, outer2) pairs
- Increments when we see outer2, by number of outer1's seen before it

**pairCount** (right-to-left): Running count of (outer2, outer1) pairs from right
- Increments when we see outer2, by number of outer1's to the right

**Multiplication**: leftPairs[i-1] × rightPairs
- Left pairs form "AB..."
- Right pairs form "...BA"  
- Middle can be anything at position i
- Together form palindrome ABCBA

---

## Key Takeaway

**Fix outer two digits (A,B), count pairs on left and right**. For each position, multiply left "AB" count by right "BA" count. Sum over all 100 digit pairs. O(n) time with two-pass algorithm.
