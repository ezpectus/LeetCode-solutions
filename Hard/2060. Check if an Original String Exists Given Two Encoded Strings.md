# 2060. Check if an Original String Exists Given Two Encoded Strings
## O(n1 × n2 × maxNum) — DP with Debt/Credit State

---

## Problem Statement

- Encoded strings contain lowercase letters (literal chars) and digits (represent N arbitrary chars).
- Return true if both encoded strings could decode to the same original string.

---

## Core Idea — DP with "Diff" State

`solve(i, j, diff)` = can `s1[i:]` and `s2[j:]` represent the same suffix of some original string, given that s1 has already "promised" `diff` more original characters than s2?

- `diff > 0`: s1 is `diff` chars ahead — s2 still needs to consume `diff` more chars to catch up.
- `diff < 0`: s2 is `|diff|` chars ahead.
- `diff = 0`: both are aligned.

**Transitions**:

| Situation | Action |
|-----------|--------|
| `s1[i]` is digit | Expand to value `v`, advance `i`, `diff += v` (s1 promises more chars) |
| `s2[j]` is digit | Expand to value `v`, advance `j`, `diff -= v` (s2 promises more) |
| Both are letters, `diff==0` | Must match: advance both if `s1[i]==s2[j]` |
| `s1[i]` is letter, `diff < 0` | s2 is ahead: s1's letter fills one of s2's "slots". `diff += 1`, advance `i` |
| `s2[j]` is letter, `diff > 0` | s1 is ahead: s2's letter fills one slot. `diff -= 1`, advance `j` |

**State space**: `i ∈ [0..40]`, `j ∈ [0..40]`, `diff ∈ [-999..999]` → at most `40 × 40 × 2000 ≈ 3.2M` states (bounded because max 3 consecutive digits → max number = 999).

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    string s1, s2;
    int n1, n2;
    Dictionary<(int,int,int),bool> memo = new();

    int[] GetNums(string s, int i) {
        var nums = new List<int>();
        int num = 0;
        while (i < s.Length && char.IsDigit(s[i]))
            nums.Add(num = num*10 + (s[i++]-'0'));
        return nums.ToArray();
    }

    bool Solve(int i, int j, int diff) {
        if (i == n1 && j == n2) return diff == 0;
        var key = (i, j, diff);
        if (memo.TryGetValue(key, out bool res)) return res;

        bool ans = false;

        if (i < n1 && char.IsDigit(s1[i]))
            foreach (int v in GetNums(s1, i))
                if (Solve(i + v.ToString().Length, j, diff + v)) { ans = true; break; }

        if (!ans && j < n2 && char.IsDigit(s2[j]))
            foreach (int v in GetNums(s2, j))
                if (Solve(i, j + v.ToString().Length, diff - v)) { ans = true; break; }

        if (!ans && i < n1 && !char.IsDigit(s1[i]) && j < n2 && !char.IsDigit(s2[j]))
            if (diff == 0) ans = s1[i] == s2[j] && Solve(i+1, j+1, 0);

        if (!ans && i < n1 && !char.IsDigit(s1[i]) && diff < 0)
            ans = Solve(i+1, j, diff+1);

        if (!ans && j < n2 && !char.IsDigit(s2[j]) && diff > 0)
            ans = Solve(i, j+1, diff-1);

        return memo[key] = ans;
    }

    public bool PossiblyEquals(string s1, string s2) {
        this.s1 = s1; this.s2 = s2;
        n1 = s1.Length; n2 = s2.Length;
        return Solve(0, 0, 0);
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n1 × n2 × maxNum)** | maxNum ≤ 999 (at most 3 consecutive digits) |
| **Space** | **O(n1 × n2 × maxNum)** | Memoization table |

With n1,n2 ≤ 40, maxNum = 999: ~1.6M states — fast.

## Example Walkthrough

**`s1="internationalization"` (20 letters), `s2="i18n"`**

- `solve(0,0,0)`: both 'i', diff=0 → match → `solve(1,1,0)`
- `solve(1,1,0)`: s2[1]='1' digit → expand to 18 → `solve(1,3,-18)`
- `solve(1,3,-18)`: s1[1]='n' letter, diff=-18 → `solve(2,3,-17)` → ... → `solve(19,3,0)`
- `solve(19,3,0)`: both 'n', diff=0 → match → `solve(20,4,0)` → both done, diff=0 → **True** ✓

---

## Pitfalls & Edge Cases

- **`diff=0` letter mismatch**: when both sides have letters and diff=0, they MUST match exactly. Return false if different.
- **Digit expansion generates multiple values**: "123" can be 1, 12, or 123 — try all.
- **Don't mix letter-advance with wrong diff**: advance s1's letter only when `diff < 0`; s2's letter only when `diff > 0`.
- **Sign convention**: `diff > 0` = s1 is ahead (gave more original chars than s2 consumed).

---

## Key Takeaway

"Do two encoded strings decode to the same original?" → **DP(i, j, diff) where diff = chars s1 is ahead of s2**:

```
digit → expand, adjust diff
both letters + diff==0 → must match, advance both
letter + debt → advance letter, reduce debt
```

The bounded diff (≤999 due to max 3 consecutive digits) makes the state space manageable.
