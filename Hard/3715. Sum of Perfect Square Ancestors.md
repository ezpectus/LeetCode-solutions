# 3715. Sum of Perfect Square Ancestors

## 1. Idea & Architectural Choice

### 1.1 Core Philosophy
The problem asks for the sum of $t_i$ values across all non-root nodes, where $t_i$ represents the number of ancestors of node $i$ such that $nums[i] \times nums[ancestor]$ is a perfect square.

A fundamental number-theoretic property states that the product of two positive integers $a$ and $b$ is a perfect square if and only if $a$ and $b$ share the exact same **square-free core** (also known as square-free kernel). The square-free core of an integer $x$, denoted as $core(x)$, is obtained by removing all square prime factors from $x$. For instance:
- $core(8) = core(2^3) = 2$
- $core(12) = core(2^2 \times 3) = 3$
- $core(18) = core(2 \times 3^2) = 2$

Since $core(8) == core(18) == 2$, their product $8 \times 18 = 144 = 12^2$ is a perfect square.

### 1.2 Mathematical Reduction & Algorithm Steps
Using this reduction, the condition $nums[i] \times nums[ancestor]$ is a perfect square becomes equivalent to $core(nums[i]) == core(nums[ancestor])$.

1. **Precompute Square-Free Cores**: Compute $core(v)$ for all $v \in [1, \max(nums)]$.
2. **DFS with Ancestor Frequency Tracking**: Traverse the tree rooted at node 0 using Depth-First Search (DFS). Maintain an array/hash map `freq` where `freq[c]` tracks how many current ancestors on the path from root to node $u$ have a square-free core equal to $c$.
3. **Counting & Backtracking**:
   - For node $u$ with core $c = core(nums[u])$, the number of valid ancestors is simply `freq[c]`.
   - Add `freq[c]` to the running answer sum.
   - Increment `freq[c]` by 1 before processing children.
   - Recursively visit all children of node $u$.
   - Decrement `freq[c]` by 1 upon backtracking.

### 1.3 Architectural Choice
- **Alternative 1: Explicit Path Tracking per Node ($O(N^2)$)**:
  For each node, iterate through all its ancestors up to the root and test if the product is a perfect square. This takes $O(N^2)$ time in a skewed tree (e.g., line graph), which results in Time Limit Exceeded for $N = 10^5$.
- **Alternative 2: Heavy-Light Decomposition / Fenwick Trees ($O(N \log^2 N)$)**:
  Unnecessary overhead. DFS backtracking allows $O(1)$ state updates during tree traversal without complex path queries.
- **Selected Choice: Sieve Precomputation + DFS with Frequency Backtracking ($O(N + V)$)**:
  Optimal time complexity $O(N + V)$ where $V = \max(nums[i])$. Precomputing square-free cores takes $O(V \log \log V)$ or $O(V)$, and tree traversal takes $O(N)$.

## 2. Algorithm Design & Mathematical Invariants

### 2.1 Square-Free Core Precomputation
Using a Modified Smallest Prime Factor (SPF) Sieve or iterative reduction:
For every number $x \in [1, 100000]$:
- Divide out all squared prime factors $p^2$.
- The remaining factor is the square-free core $core(x)$.

### 2.2 Variables & Step-by-Step Logic
- `n`: Number of nodes in the tree.
- `edges`: Unweighted edges forming the tree.
- `nums`: Node values.
- `core[v]`: Precomputed square-free core of value $v$.
- `freq[c]`: Array tracking ancestor core counts along the current DFS root-to-node path.
- `total_sum`: Global running accumulator (`long long`) for $t_i$.

#### DFS Invariant:
When visiting node $u$, `freq[c]` contains the exact number of nodes on the simple path from root (node 0) to $parent(u)$ whose $nums$ value has square-free core $c$.

## 3. Full Optimal Implementation (C++)

```cpp
// DFS + Tree Traversal + Number Theory (Square-Free Core)
#include <vector>
#include <algorithm>

class Solution {
private:
    static const int MAX_VAL = 100000;
    std::vector<int> core;
    std::vector<int> freq;
    long long total_sum;

    void precomputeCores() {
        if (!core.empty()) return;
        core.assign(MAX_VAL + 1, 0);
        for (int i = 1; i <= MAX_VAL; ++i) {
            core[i] = i;
        }
        for (int i = 2; i * i <= MAX_VAL; ++i) {
            int sq = i * i;
            for (int j = sq; j <= MAX_VAL; j += sq) {
                while (core[j] % sq == 0) {
                    core[j] /= sq;
                }
            }
        }
    }

    void dfs(int u, int p, const std::vector<std::vector<int>>& adj, const std::vector<int>& nums) {
        int u_core = core[nums[u]];
        
        // Add number of ancestors with matching square-free core
        total_sum += freq[u_core];

        // Include current node in ancestor frequency map for children
        freq[u_core]++;

        for (int v : adj[u]) {
            if (v != p) {
                dfs(v, u, adj, nums);
            }
        }

        // Backtrack
        freq[u_core]--;
    }

public:
    long long sumOfAncestors(int n, std::vector<std::vector<int>>& edges, std::vector<int>& nums) {
        precomputeCores();

        std::vector<std::vector<int>> adj(n);
        for (const auto& edge : edges) {
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]);
        }

        freq.assign(MAX_VAL + 1, 0);
        total_sum = 0;

        dfs(0, -1, adj, nums);

        return total_sum;
    }
};
```

## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The total execution time consists of three main steps:
- **Precomputing Square-Free Cores**: A harmonic-like sieve runs over numbers up to V = max(nums[i]) = 100000. Operations: T_sieve = O(V log log V) or O(V) with a linear sieve.
- **Adjacency List Construction**: Iterates through N - 1 edges to construct the tree representation. Operations: T_graph = O(N).
- **DFS Traversal with O(1) State Updates**: Visits every node exactly once and processes each edge twice. For each node, lookup and update of the `freq` array takes O(1) constant time. Operations: T_dfs = O(N).

**Total Time Complexity:**
T_total = T_sieve + T_graph + T_dfs = O(V log log V + N)

For N = 10^5 and V = 10^5, T_total performs roughly 3 * 10^5 operations, completing in under 10 milliseconds.

### 4.2 Detailed Space Complexity Decomposition
- **core Array**: Precomputed array of size V + 1 = 100001 integers (~400 KB).
- **freq Array**: Frequency tracking array of size V + 1 = 100001 integers (~400 KB).
- **adj Graph**: Adjacency list storing 2 * (N - 1) integers (~1.6 MB for N = 10^5).
- **Call Stack Memory**: DFS recursion stack depth up to O(N) in a line graph (~1.6 MB).

**Total Space Complexity:**
S_total = O(N + V)

Total memory consumed is less than 5 MB, well within standard platform limits (256 MB).

### 4.3 Complexity Summary

| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(N + V log log V) | V = max(nums[i]) = 10^5. Precomputation plus single linear DFS pass. |
| **Space Complexity** | O(N + V) | Stores tree adjacency list, recursion stack, and frequency lookup array. |

## 5. Mathematical Formulations & Foundational Algorithms

### Perfect Square Product Invariant
For any two positive integers a and b, let their prime factorizations be:
a = (p_1^e_1) * (p_2^e_2) * ... * (p_k^e_k)
b = (p_1^f_1) * (p_2^f_2) * ... * (p_k^f_k)

The product a * b = (p_1^(e_1 + f_1)) * (p_2^(e_2 + f_2)) * ... * (p_k^(e_k + f_k)) is a perfect square if and only if (e_i + f_i) mod 2 == 0 for all prime factors p_i.

This condition implies:
e_i mod 2 == f_i mod 2 for all p_i

Therefore, the square-free core of an integer x, defined as:
core(x) = product of all prime factors p_i in x where e_i is odd

must be identical for both a and b:
a * b is a perfect square if and only if core(a) == core(b)

Using this invariant converts a non-linear multiplication condition (a * b = k^2) into an equality check on precomputed integer values (core(a) == core(b)).

## Pitfalls & Edge Cases

* **Integer Overflow in Total Sum:** The sum of t_i values can reach O(N^2) in a line graph where all node values have identical cores. A 32-bit signed integer (`int`) will overflow; using 64-bit `long long` for `total_sum` prevents this.
* **Backtracking Memory Leaks / State Pollution:** Forgetting to decrement `freq[u_core]--` after returning from child nodes pollutes frequency counts across sibling branches. Explicit backtracking guarantees strict root-to-node path invariants.
* **Large Node Values (nums[i] = 10^5):** Precomputing square-free cores up to 10^5 statically or dynamically avoids repeated factorization overhead during DFS.

---

## Key Takeaway

To count valid pairs along tree paths matching a multiplicative number-theoretic property, transform the property into an equivalent equality invariant (such as square-free cores) and maintain path counts using DFS with frequency array backtracking.

**Pattern recognition:** "sum of ancestor pairs where product is a perfect square" -> Square-Free Core Reduction + DFS Tree Traversal + Frequency Map Backtracking
