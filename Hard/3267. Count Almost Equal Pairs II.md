# 3267. Count Almost Equal Pairs II
## O(n · 441) — Generate ≤2-Swap Neighborhood, Count with Seen Map

---

## Problem Statement

Count pairs `(i, j)` with `i < j` such that `nums[i]` and `nums[j]` are "almost equal": one of them can be transformed into the other by at most **2 swap-any-two-digits** operations (on either number).

---

## Core Insight #1 — "1 Swap Each" Is Subsumed by "2 Swaps on One"

The 6 cases for 2 total operations are:
- 0 ops: x = y
- 1 op on x: x' = y
- 1 op on y: x = y'
- 2 ops on x: x'' = y
- 2 ops on y: x = y''
- **1 op on x, 1 op on y:** x' = y' (after 1 swap each)

**Key theorem:** The mixed case (1 swap each) is ALWAYS subsumed by the "2 swaps on one" case.

**Proof:** If applying swap (a,b) to x gives x', and applying swap (c,d) to y gives y', and x' = y':
```
y = y' with swap(c,d) [self-inverse: swapping twice = identity]
  = x' with swap(c,d)
  = (x with swap(a,b)) with swap(c,d)
  = x with 2 swaps: first (a,b), then (c,d)
```
So y ∈ Reach₂(x). ✓

**Conclusion:** x and y are almost equal iff `y ∈ Reach(x)`, where `Reach(x)` = all numbers reachable from x in 0, 1, or 2 digit swaps (on x alone).

---

## Core Insight #2 — Reach(x) Has At Most 463 Distinct Values

Since `nums[i] < 10⁷`, every number has at most 7 digits (padded with leading zeros to exactly 7 digits, since leading zeros are explicitly allowed). Digit positions: 0..6.

| Swap count | Choices | Elements |
|---|---|---|
| 0 | 1 | 1 (x itself) |
| 1 | C(7,2) = 21 | ≤ 21 |
| 2 | 21 × 21 = 441 | ≤ 441 |

Total ≤ **463** distinct numbers per x (fewer in practice due to duplicate digits and collisions).

---

## Core Insight #3 — Count Pairs with a Running "Seen" Map

Process indices left-to-right. Before processing index i, `seen[v]` = number of j < i with `nums[j] = v`.

For each i:
1. Generate `Reach(nums[i])` — a set of ≤ 463 numbers.
2. Count: `count += Σ seen[z] for z in Reach(nums[i])`.
3. Increment `seen[nums[i]]`.

Each pair (i, j) with i > j is counted exactly once (when we reach index i and j is in the `seen` map). Symmetry of Reach guarantees correctness: y ∈ Reach(x) ↔ x ∈ Reach(y).

---

## Full Implementation (C++)

```cpp
class Solution {
public:
    int countPairs(vector<int>& nums) {
        int n = nums.size();

        // Generate all numbers reachable from x in ≤ 2 digit swaps
        // Numbers < 10^7 have ≤ 7 digits; represent with leading zeros
        auto getReach = [](int x) {
            char d[7]; // d[0] = most significant digit
            int tmp = x;
            for (int i = 6; i >= 0; i--) { d[i] = tmp % 10; tmp /= 10; }

            unordered_set<int> reach;
            reach.insert(x); // 0 swaps

            // 1 swap: C(7,2) = 21 pairs
            for (int a = 0; a < 7; a++) {
                for (int b = a + 1; b < 7; b++) {
                    swap(d[a], d[b]);
                    int v = 0;
                    for (int k = 0; k < 7; k++) v = v * 10 + d[k];
                    reach.insert(v);
                    swap(d[a], d[b]); // restore
                }
            }

            // 2 swaps: 21 × 21 = 441 ordered pairs
            for (int a = 0; a < 7; a++) {
                for (int b = a + 1; b < 7; b++) {
                    swap(d[a], d[b]); // apply first swap
                    for (int c = 0; c < 7; c++) {
                        for (int e = c + 1; e < 7; e++) {
                            swap(d[c], d[e]); // apply second swap
                            int v = 0;
                            for (int k = 0; k < 7; k++) v = v * 10 + d[k];
                            reach.insert(v);
                            swap(d[c], d[e]); // restore second
                        }
                    }
                    swap(d[a], d[b]); // restore first
                }
            }

            return reach;
        };

        int count = 0;
        unordered_map<int, int> seen; // value → count of times seen

        for (int i = 0; i < n; i++) {
            auto reach = getReach(nums[i]);
            for (int z : reach)
                count += seen.count(z) ? seen[z] : 0;
            seen[nums[i]]++;
        }

        return count;
    }
};
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Reach generation per element | O(441) | 21×21 two-swap loops |
| Pair counting per element | O(463) | iterate Reach, hash map lookup |
| **Total** | **O(n × 441)** | n=5000 → ≈ 2.3M hash ops |
| Space | O(n + 463) | seen map + reach set per element |

**Seen-map trace:**

| i | nums[i] | Reach hits in seen | count | seen update |
|---|---|---|---|---|

## Why the "Mixed Case" Proof Matters

Without this proof, we'd need a separate check for each pair: "does Reach₁(x) ∩ Reach₁(y) ≠ ∅?" This would require iterating Reach₁(x) (21 elements) and looking each up in a hash set of Reach₁(y) (21 elements) — O(n² × 21) = O(n²). Still manageable for n=5000, but unnecessary. The proof reduces to: just generate Reach(x) (which already includes 2-swap reachability) and check membership.

---

## Pitfalls & Edge Cases

- **Leading zeros are allowed** by problem statement. Treating numbers as 7-digit strings handles "10 with swap(0,1) = 01 = 1" correctly.

- **Order of 2 swaps matters when they share a digit position.** Swapping (0,1) then (0,2) ≠ swapping (0,2) then (0,1) in general. The nested loop explores all 441 ordered pairs, correctly covering all cases.

- **Swap of same position pair (a==b):** Not possible since inner loop has `b > a`. The identity case (0 swaps) is handled separately by `reach.insert(x)`.

- **When first swap = second swap (c=a, e=b):** This undoes the first swap, giving x back (already in reach). Harmless — the set deduplicates it.

- **Duplicate values in nums:** Handled correctly. If nums[i] = nums[j], they're already equal (0 swaps), and x ∈ Reach(x) always. The `seen[nums[i]]` count accumulates correctly.

- **Don't use `seen[z]` if z not in map (map insertion side effect).** Use `seen.count(z) ? seen[z] : 0` or `auto it = seen.find(z); if (it != seen.end()) count += it->second;` to avoid creating 0-entries that pollute the map.

---

## Key Takeaway

This is a **"neighborhood pre-generation + seen-map counting"** pattern. When all pairs can be checked against a precomputed "neighborhood" of fixed size, we bypass the O(n²) pair-iteration by processing elements sequentially and counting past neighbors that land in the current element's neighborhood.

**Pattern recognition:** "Count pairs (i,j) where nums[j] is reachable from nums[i] in ≤ k operations" → if |Reach_k(x)| is small and constant, generate Reach_k for each element and count using a seen-frequency map. For digit swaps with k=2 and 7-digit numbers: |Reach| ≤ 463 operations per element.
