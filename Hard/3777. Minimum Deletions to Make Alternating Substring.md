# 3777. Minimum Deletions to Make Alternating Substring

## 1. Idea & Structural Analysis

### 1.1 Core Idea
An alternating string is defined as a string where no two adjacent characters are identical.

Suppose we have a contiguous substring `s[l..r]`. If there are adjacent identical characters (e.g., "AA" or "BB"), every such adjacent equal pair requires at least one deletion to eliminate the conflict and separate the characters.

Let us define an auxiliary indicator array `eq` of length N:
- `eq[i] = 1` if `i >= 1` and `s[i] == s[i - 1]`
- `eq[i] = 0` otherwise

For any query asking for the minimum deletions to make `s[l..r]` alternating, the required number of deletions is precisely the sum of `eq[i]` over the range `[l + 1, r]`.

### 1.2 Point Updates
When query type 1 flips the character at index `j` (`s[j]` changes from 'A' to 'B' or 'B' to 'A'):
- Only two adjacent equality checks can possibly change:
  1. The equality between `s[j - 1]` and `s[j]` (affecting `eq[j]`).
  2. The equality between `s[j]` and `s[j + 1]` (affecting `eq[j + 1]`).
- All other positions in `eq` remain completely unchanged.

Therefore, each type 1 query translates to at most 2 point update operations on the data structure managing `eq`.

---

## 2. Structural & Architectural Choice

### 2.1 Primary Choice: Fenwick Tree (Binary Indexed Tree)
Since the problem requires:
1. Dynamic point updates (`eq[j]` and `eq[j+1]`).
2. Range sum queries over `[l + 1, r]`.

A **Fenwick Tree (BIT)** is the optimal structure.
- **Time per query:** O(log N)
- **Space complexity:** O(N) auxiliary space with very low constant factors compared to Segment Tree.

### 2.2 Alternative Approaches Comparison
- **Segment Tree:** Technologically viable and achieves the same O(log N) time bounds. However, a Segment Tree incurs 4x memory overhead and higher pointer/node traversal constant factors.
- **Prefix Sums Array:** Computes range queries in O(1) time, but rebuilding the prefix array after each flip takes O(N) time, leading to O(Q * N) overall time complexity, which TLEs for N, Q <= 10^5.

## 3. Full Optimal Implementation (C++)

```cpp
// Fenwick Tree + Range Sum Queries
#include <vector>
#include <string>

class FenwickTree {
private:
    int n;
    std::vector<int> tree;

public:
    FenwickTree(int size) : n(size), tree(size + 1, 0) {}

    void add(int idx, int delta) {
        if (idx <= 0 || idx > n) return;
        for (; idx <= n; idx += idx & -idx) {
            tree[idx] += delta;
        }
    }

    int query(int idx) const {
        if (idx <= 0) return 0;
        int sum = 0;
        for (; idx > 0; idx -= idx & -idx) {
            sum += tree[idx];
        }
        return sum;
    }

    int queryRange(int left, int right) const {
        if (left > right) return 0;
        return query(right) - query(left - 1);
    }
};

class Solution {
public:
    std::vector<int> minDeletions(std::string s, std::vector<std::vector<int>>& queries) {
        int n = s.length();
        FenwickTree bit(n);

        // Build initial eq state in Fenwick Tree
        // eq[i] corresponds to 1-based index (i + 1) in Fenwick Tree
        for (int i = 1; i < n; ++i) {
            if (s[i] == s[i - 1]) {
                bit.add(i + 1, 1);
            }
        }

        std::vector<int> result;
        result.reserve(queries.size());

        for (const auto& q : queries) {
            if (q[0] == 1) {
                int j = q[1];

                // Helper lambda to update eq value at a specific index
                auto update_eq = [&](int idx, int new_val) {
                    if (idx < 1 || idx >= n) return;
                    int current_val = (s[idx] == s[idx - 1]) ? 1 : 0;
                    int diff = new_val - current_val;
                    if (diff != 0) {
                        bit.add(idx + 1, diff);
                    }
                };

                // Store character before flip
                char old_char = s[j];
                char new_char = (old_char == 'A') ? 'B' : 'A';

                // Temporarily evaluate what eq[j] and eq[j+1] will become
                // Check left neighbor j-1
                if (j > 0) {
                    int new_eq_j = (new_char == s[j - 1]) ? 1 : 0;
                    update_eq(j, new_eq_j);
                }

                // Check right neighbor j+1
                if (j < n - 1) {
                    int new_eq_j1 = (s[j + 1] == new_char) ? 1 : 0;
                    update_eq(j + 1, new_eq_j1);
                }

                // Mutate the actual string character
                s[j] = new_char;

            } else {
                int l = q[1];
                int r = q[2];

                if (l == r) {
                    result.push_back(0);
                } else {
                    // Query sum of eq array in range [l + 1, r]
                    // In 1-based Fenwick Tree terms: range [l + 2, r + 1]
                    int deletions = bit.queryRange(l + 2, r + 1);
                    result.push_back(deletions);
                }
            }
        }

        return result;
    }
};
```


## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
Let N be the length of the string s, and Q be the total number of queries.

1. **Initialization Phase**:
   - Iterates through string s of length N to set up initial `eq` states.
   - Inserts entries into the Fenwick Tree.
   - Operations: T_init = O(N * log N).

2. **Query Processing Phase**:
   - For Type 1 Query (Character Flip):
     - Calculates at most 2 point updates.
     - Each Fenwick Tree update takes O(log N) time.
     - Operations per Type 1 query: T_type1 = O(log N).
   - For Type 2 Query (Range Deletions Calculation):
     - Executes 2 prefix sum queries on the Fenwick Tree.
     - Operations per Type 2 query: T_type2 = O(log N).
   - Total operations across Q queries: T_queries = O(Q * log N).

**Total Time Complexity:**
T_total = T_init + T_queries = O((N + Q) * log N)

For N = 10^5 and Q = 10^5, (N + Q) * log2(N) is approximately 2 * 10^5 * 17 = 3.4 * 10^6 operations, executing in under 30 milliseconds.

### 4.2 Detailed Space Complexity Decomposition
- **Fenwick Tree Structure**: Array `tree` of size N + 1 integers (~ 400 KB).
- **Result Array**: Vector of size at most Q integers (~ 400 KB).
- **Original String**: Characters stored in contiguous memory (~ 100 KB).

**Total Space Complexity:**
S_total = O(N + Q)

Auxiliary space consumed is under 2 MB, well within standard platform limits (256 MB).

### 4.3 Complexity Summary

| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O((N + Q) * log N) | N = s.length, Q = queries.length. Dynamic point updates and range queries on Fenwick Tree. |
| **Space Complexity** | O(N + Q) | Space for Fenwick Tree array and output result vector. |

## 5. Mathematical Formulations & Foundational Algorithms

### Reducing Deletions to Adjacent Equality Pairs
Let S = s[l..r] be a substring of length K = r - l + 1.
Define indicator variable eq_i for index i in range l + 1 to r:
- eq_i = 1 if s[i] == s[i - 1]
- eq_i = 0 if s[i] != s[i - 1]

**Claim:** The minimum number of character deletions required to transform S into an alternating string is equal to sum of eq_i for i from l + 1 to r.

**Proof / Logic:**
1. A string S is alternating if and only if no two adjacent characters are equal, which means eq_i = 0 for all i in [l + 1, r].
2. Each contiguous block of identical characters of length L (e.g., "AAAA" where L = 4) contributes L - 1 equal adjacent pairs (eq_i = 1).
3. To eliminate all equal adjacent pairs in a block of length L, we must delete exactly L - 1 characters.
4. Summing across all blocks in the range [l, r], the minimum number of total deletions is:
   Total Deletions = sum over all blocks of (L_block - 1) = sum(eq_i) for i in [l + 1, r].

Thus, the range query on substring s[l..r] reduces directly to range sum query over eq[l + 1 .. r].

## Pitfalls & Edge Cases

* **Single Character Substring (l == r):** Range [l + 1, r] becomes invalid (l + 1 > r). Handled explicitly by returning 0 deletions immediately.
* **Boundary Flips (j = 0 or j = N - 1):** Flipping index 0 has no left neighbor (j - 1 < 0), and flipping N - 1 has no right neighbor (j + 1 >= N). Bound checks `j > 0` and `j < N - 1` safely ignore non-existent neighbors.
* **1-Based Index Mapping in Fenwick Tree:** `eq[i]` corresponds to index `i + 1` in the 1-based Fenwick Tree array. Range `[l + 1, r]` translates to `[l + 2, r + 1]` in Fenwick Tree terms.
* **Redundant Updates:** If a flip does not change the value of `eq[j]` (e.g., flipping 'A' to 'B' when both left and right neighbors are also changed accordingly), the difference `diff` is 0, avoiding unnecessary Fenwick tree updates.

---

## Key Takeaway

When tasked with counting required deletions to maintain alternating patterns under dynamic point updates, convert adjacent equality checks into a binary indicator array `eq` and handle point updates with range sum queries using a Fenwick Tree.

**Pattern recognition:** "minimum deletions to make alternating substring" + "character flip updates" -> Auxiliary Pair Equality Array + Fenwick Tree (BIT)
