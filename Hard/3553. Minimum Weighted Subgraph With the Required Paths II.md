# LeetCode 3553. Minimum Weighted Subgraph With the Required Paths II

## 1. Constraints & Complexity Bottleneck Analysis

- **Number of Nodes ($N$):** $3 \le N \le 10^5$. Since the graph is a tree with exactly $N - 1$ edges, any approach attempting to traverse the paths dynamically for each query using Breadth-First Search (BFS) or Depth-First Search (DFS) will yield an $O(Q \cdot N)$ runtime, leading to an immediate Time Limit Exceeded (TLE) bottleneck.
- **Number of Queries ($Q$):** $1 \le Q \le 10^5$. To handle this volume of queries within a typical 2-second execution limit, each individual query must be resolved in $O(\log N)$ or $O(1)$ time. This dictates that we must decouple tree traversals from the query phase.
- **Path Weight Constraints ($W_i$):** $1 \le W_i \le 10^4$. Cumulative path lengths from the root can reach up to $10^5 \times 10^4 = 10^9$. While the final answer for any individual query is guaranteed to fit within a standard signed 32-bit integer, intermediate pairwise additions (such as $d(a, b) + d(b, c) + d(c, a)$) can reach up to $3 \times 10^9$, which exceeds the max limit of a signed 32-bit integer ($2.14 \times 10^9$). This is the primary overflow trap, requiring selective intermediate calculations to be processed using 64-bit integer types (`long long`) before casting back to the final return type `vector<int>`.

## 2. Algorithm Design & Mathematical Invariants

### The Core Concept: Three-Node Spanning Subtree Invariant

For any three vertices $a$, $b$, and $c$ in an undirected tree, there exists a unique central junction node $p$ (which can coincide with $a$, $b$, or $c$) where the unique simple paths between the three nodes intersect. The minimal connected subgraph containing $a$, $b$, and $c$ is a Y-shaped or linear structure composed of three disjoint paths starting at $p$ and ending at $a$, $b$, and $c$ respectively.

The sum of the pairwise distances can be algebraically represented by splitting the paths via the junction node $p$:

$$d(a, b) = d(a, p) + d(b, p)$$
$$d(b, c) = d(b, p) + d(c, p)$$
$$d(c, a) = d(c, p) + d(a, p)$$

Summing these equations yields:

$$d(a, b) + d(b, c) + d(c, a) = 2 \cdot \left( d(a, p) + d(b, p) + d(c, p) \right)$$

Since the total weight of the minimal spanning subtree is exactly $d(a, p) + d(b, p) + d(c, p)$, we can compute it directly as:

$$\text{Subtree Weight} = \frac{d(a, b) + d(b, c) + d(c, a)}{2}$$

This reduces the problem of finding a minimal subtree to three independent pairwise distance queries on the tree.

### O(log N) Distance Queries via Binary Lifting

To query the distance between any two nodes $u$ and $v$ in logarithmic time, we root the tree at node $0$ and compute:
1. `depth[u]`: The number of edges on the path from root to $u$.
2. `dist[u]`: The sum of edge weights on the path from root to $u$.

The shortest path distance between $u$ and $v$ is then given by the relation:

$$d(u, v) = \text{dist}[u] + \text{dist}[v] - 2 \cdot \text{dist}[\text{LCA}(u, v)]$$

We find the Lowest Common Ancestor ($\text{LCA}(u, v)$) using the Binary Lifting technique. We precompute an ancestor jump table `up[u][i]` which stores the $2^i$-th ancestor of node $u$. This table allows us to lift nodes to equivalent depths and find their lowest common ancestor in $O(\log N)$ steps.

## 3. High-Performance C++ Implementation

```cpp
// Tree DFS + Binary Lifting for LCA + Pairwise Tree Distance
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
private:
    int n;
    int max_log;
    vector<vector<pair<int, int>>> adj;
    vector<vector<int>> up;
    vector<int> depth;
    vector<long long> dist;

    // DFS to initialize depths, parent pointers (up[v][0]), and root distances
    void dfs(int u, int p, int d, long long current_dist) {
        depth[u] = d;
        dist[u] = current_dist;
        up[u][0] = p;

        for (int i = 1; i < max_log; ++i) {
            if (up[u][i - 1] != -1) {
                up[u][i] = up[up[u][i - 1]][i - 1];
            } else {
                up[u][i] = -1;
            }
        }

        for (auto& edge : adj[u]) {
            int v = edge.first;
            int weight = edge.second;
            if (v != p) {
                dfs(v, u, d + 1, current_dist + weight);
            }
        }
    }

    // Binary lifting routine to find the Lowest Common Ancestor
    int getLCA(int u, int v) {
        if (depth[u] < depth[v]) {
            swap(u, v);
        }

        // Lift u to the same depth as v
        int diff = depth[u] - depth[v];
        for (int i = 0; i < max_log; ++i) {
            if ((diff >> i) & 1) {
                u = up[u][i];
            }
        }

        if (u == v) return u;

        // Lift both nodes simultaneously
        for (int i = max_log - 1; i >= 0; --i) {
            if (up[u][i] != up[v][i]) {
                u = up[u][i];
                v = up[v][i];
            }
        }

        return up[u][0];
    }

    // O(log N) Distance Query returning 64-bit integer to prevent addition overflow
    long long getDistance(int u, int v) {
        int lca = getLCA(u, v);
        return dist[u] + dist[v] - 2 * dist[lca];
    }

public:
    vector<int> minimumWeight(vector<vector<int>>& edges, vector<vector<int>>& queries) {
        n = edges.size() + 1;
        max_log = 0;
        while ((1 << max_log) <= n) {
            max_log++;
        }
        max_log++; // Ensure boundary safety

        adj.assign(n, vector<pair<int, int>>());
        for (const auto& edge : edges) {
            int u = edge[0];
            int v = edge[1];
            int w = edge[2];
            adj[u].push_back({v, w});
            adj[v].push_back({u, w});
        }

        up.assign(n, vector<int>(max_log, -1));
        depth.assign(n, 0);
        dist.assign(n, 0LL);

        // Run DFS from root node 0
        dfs(0, -1, 0, 0LL);

        vector<int> ans;
        ans.reserve(queries.size());

        for (const auto& q : queries) {
            int a = q[0];
            int b = q[1];
            int c = q[2];

            long long d_ab = getDistance(a, b);
            long long d_bc = getDistance(b, c);
            long long d_ca = getDistance(c, a);

            // Compute total weight using 64-bit precision to prevent overflow during intermediate addition
            long long min_subtree_weight = (d_ab + d_bc + d_ca) / 2;
            
            // Safely cast to 32-bit signed integer for the final return type
            ans.push_back(static_cast<int>(min_subtree_weight));
        }

        return ans;
    }
};
```

## 4. Comprehensive Complexity & Mathematical Analysis

### Detailed Time Complexity Decomposition

The execution timeline of this system scales with tree size $N$ and queries $Q$:

$$T_{\text{total}} = T_{\text{graph\_build}} + T_{\text{dfs\_preprocessing}} + T_{\text{queries}}$$

#### Graph Adjacency Construction ($T_{\text{graph\_build}}$):
Parsing the $N-1$ edges into an adjacency list is performed in linear time:
$$T_{\text{graph\_build}} = O(N)$$

#### DFS & Binary Lifting Preprocessing ($T_{\text{dfs\_preprocessing}}$):
We traverse the tree using DFS. For each of the $N$ nodes, we fill in a binary lift jump table of size $\log N$:
$$T_{\text{dfs\_preprocessing}} = O(N \log N)$$

#### Binary Lifting LCA Queries ($T_{\text{queries}}$):
We process $Q$ queries. Each query performs exactly three distance evaluations. Each distance evaluation relies on finding the Lowest Common Ancestor using binary lifting, which takes $O(\log N)$ time:
$$T_{\text{queries}} = O(Q \log N)$$

Thus, the overall time complexity is tightly bounded by:
$$\text{Time Complexity} = O((N + Q) \log N)$$

### Detailed Space Complexity Decomposition

The structural memory overhead is dominated by the auxiliary structures required for LCA queries:

#### Jump Table Matrix (`up`):
The binary lifting jump table requires $N \times \log N$ storage:
$$S_{\text{up}} = N \times \log N \times 4 \text{ bytes} \implies O(N \log N)$$

#### Traversal Vectors (`depth`, `dist`, `adj`):
- Graph adjacency list: $2(N - 1) \times 8 \text{ bytes} \approx O(N)$
- Distance values `dist` (64-bit integers): $N \times 8 \text{ bytes} \approx O(N)$
- Depth indices `depth` (32-bit integers): $N \times 4 \text{ bytes} \approx O(N)$

$$\text{Space Complexity} = O(N \log N)$$

## 5. Mathematical Formulations & Foundational Algorithms

### The Three-Point Tree Centrality Invariant

For any three distinct vertices $a$, $b$, and $c$ in a tree, there exists a unique central junction vertex $p$ (which can coincide with one of the vertices) that lies on the intersection of all simple paths connecting the three nodes. The minimal subtree connecting them consists of three disjoint simple paths starting at $p$ and ending at $a$, $b$, and $c$ respectively.

The sum of the pairwise distances can be algebraically represented by splitting the paths via the junction node $p$:

$$d(a, b) = d(a, p) + d(b, p)$$
$$d(b, c) = d(b, p) + d(c, p)$$
$$d(c, a) = d(c, p) + d(a, p)$$

Summing these equations yields:

$$d(a, b) + d(b, c) + d(c, a) = 2 \cdot \left( d(a, p) + d(b, p) + d(c, p) \right)$$

Since the subtree weight is exactly the sum of the disjoint branches $d(a, p) + d(b, p) + d(c, p)$, we divide the pairwise distance sum by 2 to obtain the exact total weight:

$$\text{Subtree Weight} = \frac{d(a, b) + d(b, c) + d(c, a)}{2}$$

This algebraic formulation transforms a dynamic subtree construction problem into three static distance queries on the tree.
