# 2440. Create Components With Same Value
## O(n × d(total)) — Try All Divisors + DFS Validation

---

## Problem Statement

- Delete edges in a tree to create components all with equal sum. Maximize edges deleted (= components - 1).

---

## Core Idea — Enumerate Divisors of Total Sum

If we split into `k` components, each must have sum `total/k`. So `k` must divide `total`, and each component sum = `total/k`.

**DFS validation**: `dfs(node, parent, target)` returns the subtree sum contributed to the parent's component. When a subtree sum hits `target`, we "cut" it (return 0 to parent). If it exceeds `target` at any point → invalid (-1).

**Enumerate `k`** from 2 to n (at most n components). For each `k` where `total % k == 0`, run DFS. The maximum valid `k` gives answer `k-1`.

Number of divisors of `total` ≤ 50×n ≤ 10⁶, but d(total) ≪ n typically. Each DFS is O(n). Total: O(n × d(total)).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int[] nums;
    List<int>[] graph;

    public int ComponentValue(int[] nums, int[][] edges) {
        int n = nums.Length;
        this.nums = nums;
        graph = new List<int>[n];
        for (int i = 0; i < n; i++) graph[i] = new();
        foreach (var e in edges) { graph[e[0]].Add(e[1]); graph[e[1]].Add(e[0]); }

        int total = nums.Sum(), res = 0;
        for (int k = 2; k <= n; k++) {
            if (total % k != 0) continue;
            if (Dfs(0, -1, total/k) == 0) res = k-1;
        }
        return res;
    }

    int Dfs(int node, int parent, int target) {
        int s = nums[node];
        foreach (int nb in graph[node]) {
            if (nb == parent) continue;
            int child = Dfs(nb, node, target);
            if (child == -1) return -1;
            s += child;
        }
        if (s == target) return 0;  // complete component
        if (s > target) return -1;  // exceeded
        return s;
    }
}
```

## Example Walkthrough

**`nums=[6,2,2,2,6], edges=[[0,1],[1,2],[1,3],[3,4]]`**

total=18. Divisors: 2,3,6,9,18.

k=3, target=6:
- Dfs(2, 1, 6): leaf, s=2 < 6 → return 2.
- Dfs(4, 3, 6): leaf, s=6 = 6 → return 0 (cut).
- Dfs(3, 1, 6): s=2+0=2 < 6 → return 2.
- Dfs(1, 0, 6): s=2+2+2=6 = 6 → return 0 (cut).
- Dfs(0, -1, 6): s=6+0=6 = 6 → return 0 ✓

Valid! res = 3-1 = **2** ✓

## Pitfalls & Edge Cases

- **Return 0 when subtree sum = target**: this "detaches" the component and contributes 0 to parent.
- **Return -1 when sum > target**: propagates failure upward immediately.
- **Single node**: n=1, no edges, answer always 0.
- **Iterate k largest first**: could break early since we want max k, but it's O(n) per DFS so iterating all is fine.

---

## Key Takeaway

"Max edges deleted with equal component sums" → **enumerate divisors k of total, DFS checks if tree splits into k equal-sum parts**:

```
for k in 2..n where total%k==0:
    if dfs(root, target=total/k) == 0: res = k-1

dfs: accumulate subtree sum, cut (return 0) when hitting target, fail (-1) if exceeded
```

O(n × d(total)) — efficient given small values constraint.
