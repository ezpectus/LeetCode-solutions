# 2584. Split the Array to Make Coprime Products
## O(n √V) — Last-Occurrence Precompute + Greedy Reach Scan

---

## Problem Statement

- Find the smallest index i such that `product(nums[0..i])` and `product(nums[i+1..n-1])` are coprime.
- Two values are coprime iff `gcd = 1`, i.e., they share no prime factor.

---

## Core Idea — Track "Prime Reach" Left to Right

Two products are coprime iff no prime factor appears in BOTH sides.

For each prime p, define `last[p]` = rightmost index where p divides some element.

A split at i is **valid** iff every prime p seen in `nums[0..i]` has `last[p] ≤ i` (it doesn't "bleed" past the split point).

Algorithm:
1. Precompute `last[p]` for all primes p by scanning the full array.
2. Scan left to right. Maintain `reach` = max last[p] over all primes seen in the prefix.
3. Split at i is valid iff `reach ≤ i` (no seen prime extends past i). Return the first such i.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int FindValidSplit(int[] nums) {
        int n = nums.Length;

        // Precompute last occurrence of each prime factor across all elements
        var last = new Dictionary<int, int>();
        for (int i = 0; i < n; i++) {
            int x = nums[i];
            for (int p = 2; (long)p * p <= x; p++) {
                if (x % p == 0) {
                    last[p] = i; // overwrites → keeps last index
                    while (x % p == 0) x /= p;
                }
            }
            if (x > 1) last[x] = i; // prime remainder
        }

        // Scan left to right, track max "reach" of seen primes
        int reach = 0;
        for (int i = 0; i < n - 1; i++) {
            int x = nums[i];
            for (int p = 2; (long)p * p <= x; p++) {
                if (x % p == 0) {
                    reach = Math.Max(reach, last[p]);
                    while (x % p == 0) x /= p;
                }
            }
            if (x > 1) reach = Math.Max(reach, last[x]);

            if (reach <= i) return i; // all primes in prefix stay in prefix
        }

        return -1;
    }
}
```

## Example Walkthrough

**Example 1:** `nums = [4, 7, 8, 15, 3, 5]`

Step 1 — Precompute `last`:
| num | primes | last[p] updates |
|-----|--------|-----------------|
| 4   | {2}    | 2→0 |
| 7   | {7}    | 7→1 |
| 8   | {2}    | 2→2 |
| 15  | {3,5}  | 3→3, 5→3 |
| 3   | {3}    | 3→4 |
| 5   | {5}    | 5→5 |

`last = {2:2, 7:1, 3:4, 5:5}`

Step 2 — Scan with `reach`:

| i | nums[i] | Prime factors | reach after | reach ≤ i? |
|---|---------|---------------|-------------|-------------|
| 0 | 4       | {2}           | max(0, 2)=**2** | 2≤0? ✗ |
| 1 | 7       | {7}           | max(2, 1)=**2** | 2≤1? ✗ |
| 2 | 8       | {2}           | max(2, 2)=**2** | 2≤2? **✓** → return **2** |

**Output:** 2 ✓

---

**Example 2:** `nums = [4, 7, 15, 8, 3, 5]`

`last = {2:3, 7:1, 3:4, 5:5}`

| i | reach after | reach ≤ i? |
|---|-------------|-------------|
| 0 | 3           | 3≤0? ✗ |
| 1 | 3           | 3≤1? ✗ |
| 2 | 5           | 5≤2? ✗ |
| 3 | 5           | 5≤3? ✗ |
| 4 | 5           | 5≤4? ✗ |

**Output:** -1 ✓

## Pitfalls & Edge Cases

- **`nums[i] = 1`** — no prime factors, nothing updates `reach`. If the whole prefix has only `1`s, `reach = 0 ≤ i` from the start → split at 0 is valid. ✓

- **Single-element prefix coprime check** — if all prime factors of `nums[0]` only appear at index 0, `reach = 0 ≤ 0`, return 0. ✓

- **Scanning to `n-2` only** — split at `n-1` is invalid by definition. The loop `i < n-1` handles this. ✓

- **Initial reach = 0** — correct since before processing any elements, no prime has been seen and the split before index 0 would be trivially coprime. Processing index 0 then either raises reach or confirms validity. ✓

---

## Key Takeaway

"Smallest split where two products are coprime" → "no prime straddles the split point" → for each prime track its rightmost occurrence, then scan with a running max of how far each prime "reaches."

**Pattern recognition:** This is the classic **interval merge / reach** pattern: maintain a `reach` variable that monotonically increases as you scan left. A valid split at i exists when `reach ≤ i` (everything seen so far is fully contained in the prefix). Same pattern as the "jump game" or "no-overlapping intervals" family of problems.
