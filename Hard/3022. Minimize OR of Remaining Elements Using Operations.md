# 3022. Minimize OR of Remaining Elements Using Operations

## O(N) — Bitwise Greedy Masking with Subarray Merging

### Problem Statement
You are given a 0-indexed integer array `nums` and an integer `k`. In one operation, you can pick any index `i` and replace `nums[i]` and `nums[i + 1]` with a single element equal to `nums[i] & nums[i + 1]`.

Return the **minimum possible value** of the bitwise OR of the remaining elements of `nums` after applying at most `k` operations.

---

### Core Idea — Bit-by-Bit Greedy Elimination
Our goal is to minimize the final bitwise OR sum. In binary representations, minimizing a value means we want to turn off ($0$) bits starting from the **most significant bit (MSB)** down to the least significant bit ($2^{29}$ down to $2^0$).

Instead of testing elements globally, we can use a greedy prefix test. Suppose we want to see if we can force a set of target bits to be completely $0$ in the final array. Let's build a `target_zero_mask` representing the bits we want to eliminate.

#### Checking Feasibility via Subarray Merging
To check if a specific `target_zero_mask` can be satisfied within `k` operations:
1. We iterate through `nums`. We track a running bitwise AND accumulator initialized to all bits set (`0x3FFFFFFF`).
2. For each element, we merge it into our accumulator using `currentAnd &= num`.
3. An operation is equivalent to combining adjacent elements. If we merge $X$ elements to achieve a specific bit configuration, we use exactly $X - 1$ operations.
4. We can safely "close" a subarray when the bits specified in `target_zero_mask` are successfully cleared inside `currentAnd`. That is:
   `(currentAnd & target_zero_mask) == 0`
5. If we reach the end of the array and the final trailing block cannot satisfy the mask, it must be merged with the preceding valid block, adding to our operation count.
6. If the total operations required $\le k$, then keeping those target bits turned off is achievable!

---

### Full Optimal Implementation (C#)

```csharp
using System;

public class Solution {
    public int MinOrAfterOperations(int[] nums, int k) {
        int n = nums.Length;
        int answerMask = 0;

        // Iterate from MSB (bit 29) down to LSB (bit 0)
        for (int bit = 29; bit >= 0; bit--) {
            // Test if we can force the current bit to be 0 alongside previously established 0s
            int testTargetZeroMask = answerMask | (1 << bit);
            
            int operationsNeeded = 0;
            // Initialize accumulator with all bits set (30 bits maximum as per constraints)
            int currentAnd = (1 << 30) - 1; 

            for (int i = 0; i < n; i++) {
                currentAnd &= nums[i];

                // Check if the bits specified in our test mask are successfully cleared
                if ((currentAnd & testTargetZeroMask) == 0) {
                    // Reset the tracking accumulator for the next subsegment
                    currentAnd = (1 << 30) - 1;
                } else {
                    // If they aren't cleared, we are forced to merge this element, costing 1 operation
                    operationsNeeded++;
                }
            }

            // If the total operations exceed k, this bit configuration is impossible to clear.
            // Therefore, this bit MUST remain 1 in our final minimized answer.
            if (operationsNeeded > k) {
                // Keep the bit as 0 in our target mask, meaning it stays 1 in the actual result
            } else {
                // If it is possible, we lock this bit as a 0-target for all subsequent iterations
                answerMask = testTargetZeroMask;
            }
        }

        // The answer is the complement of all successfully zeroed bits within our 30-bit frame
        return ((1 << 30) - 1) ^ answerMask;
    }
}
```
### Complexity

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | $O(N)$ | The outer loop runs exactly 30 times (constant relative to $N$). The inner loop scans the array of size $N$ once per bit test, resulting in a strict time complexity of $30 \times N \approx O(N)$. |
| **Space Complexity** | $O(1)$ | No auxiliary arrays or structures are allocated; computation relies entirely on primitive bitwise variables. |

---

### Pitfalls & Edge Cases

* **Bit Width Boundary:** Constraints show $nums[i] < 2^{30}$. Make sure your tracking masks use a full 30-bit limit initialization (`(1 << 30) - 1`) to avoid masking overflows into sign bits.
* **Trailing Block Merges:** If the loop terminates and the last block has an active `currentAnd` that does not clear the mask, the `operationsNeeded++` branch correctly increments to reflect merging it backwards into the final valid segment.

---

### Key Takeaway

> When minimizing an aggregate bitwise OR total through local adjustments, analyze the bits independently from MSB to LSB. Convert the constraint into a validation sequence that counts the minimum number of contiguous subarray joins needed to clear the test bits.

* **Pattern recognition:** "Minimize absolute array bitwise OR sum using adjacent element reduction" -> Greedy Digit Isolation + Subarray Bitwise AND Consolidation.
