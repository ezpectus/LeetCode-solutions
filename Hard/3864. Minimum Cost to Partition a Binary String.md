# 3864. Minimum Cost to Partition a Binary String

## 1. Idea & Structural Analysis

### 1.1 Core Idea & Mathematical Basis
The problem asks for the minimum cost to partition a binary string `s` under a specific splitting rule: a segment can **only** be split into two contiguous segments of equal length if its current length `L` is even. This constraint drastically restricts the search space, transforming the problem into a fixed binary tree traversal over dyadic intervals (similar to a Segment Tree or Divide and Conquer structure).

For any segment covering the index range `[left, right]` with length `L = right - left + 1` and containing `X` sensitive elements ('1's):
1. **Base Choice (No Split):**
   - If `X == 0`, cost = `flatCost`.
   - If `X > 0`, cost = `L * X * encCost`.
2. **Recursive Choice (Split in Half):**
   - Allowed only if `L % 2 == 0`.
   - The segment splits into `[left, mid]` and `[mid + 1, right]`, where `mid = left + (right - left) / 2`.
   - Cost = `solve(left, mid) + solve(mid + 1, right)`.

The optimal answer for `solve(left, right)` is simply `min(cost_no_split, cost_split)` if `L` is even, or `cost_no_split` if `L` is odd.

### 1.2 Step-by-Step Derivation & Case Breakdown
To evaluate the cost of any segment in $O(1)$ time, we precompute a Prefix Sum array `pref` of ones where `pref[i]` stores the number of '1's in `s[0...i-1]`.
For a segment `[left, right]`:
- `X = pref[right + 1] - pref[left]`
- `L = right - left + 1`

Let us break down the evaluation logic for `solve(left, right)`:

- **Case 1: Segment without Sensitive Elements (X == 0)**
  - `cost_no_split = flatCost`
  - If we split a segment that contains zero '1's into two equal halves, each half will also contain zero '1's and thus cost `flatCost` each. Therefore, splitting would cost `2 * flatCost`, which is strictly greater than or equal to `flatCost`.
  - *Optimization / Edge Case:* If `X == 0`, splitting is never optimal, so we can prune immediately and return `flatCost`.

- **Case 2: Segment with Sensitive Elements (X > 0) and Odd Length (L % 2 != 0)**
  - Splitting is forbidden by problem constraints.
  - `return L * X * encCost`.

- **Case 3: Segment with Sensitive Elements (X > 0) and Even Length (L % 2 == 0)**
  - Both choices are valid.
  - `cost_no_split = L * X * encCost`
  - `cost_split = solve(left, mid) + solve(mid + 1, right)`
  - `return min(cost_no_split, cost_split)`

## 2. Structural & Architectural Choice

### 2.1 Primary Choice: Divide & Conquer + Prefix Sum
- **Why it fits constraints:** $N \le 10^5$. The allowed splitting operation strictly divides the interval in half. This creates a recursion tree of depth at most $O(\log N)$. At each level, the subproblems partition the original array. The total number of states visited across all recursive calls is bounded by $O(N)$. Combining this with $O(1)$ prefix sum range queries yields an optimal time complexity without complex memoization data structures (though top-down recursion naturally handles it).

### 2.2 Alternative Approaches & Drawbacks
- **General Dynamic Programming $O(N^2)$:**
  - Standard DP where `dp[i]` is the minimum cost for prefix `s[0...i]` considering arbitrary splits.
  - *Why it fails:* The problem explicitly forbids arbitrary splits. Splits must divide an even-length segment into two **equal** halves. General DP would explore invalid states and take $O(N^2)$ time, resulting in Time Limit Exceeded (TLE).

## 3. Full Optimal Implementation (C++)
```cpp

// Divide and Conquer + Prefix Sum
#include <string>
#include <vector>
#include <algorithm>

class Solution {
private:
    std::vector<int> pref;
    long long enc_cost;
    long long flat_cost;

    // Helper function to query the number of '1's in s[left...right] in O(1)
    inline int getOnesCount(int left, int right) {
        return pref[right + 1] - pref[left];
    }

    long long solve(int left, int right) {
        int L = right - left + 1;
        int X = getOnesCount(left, right);

        // Case 1: No sensitive elements
        if (X == 0) {
            return flat_cost;
        }

        // Base cost without splitting for X > 0
        long long cost_no_split = (long long)L * X * enc_cost;

        // Case 2: Odd length, cannot split further
        if (L % 2 != 0) {
            return cost_no_split;
        }

        // Case 3: Even length with X > 0, compare no-split vs split
        int mid = left + (right - left) / 2;
        long long cost_split = solve(left, mid) + solve(mid + 1, right);

        return std::min(cost_no_split, cost_split);
    }

public:
    long long minCost(std::string s, int encCost, int flatCost) {
        int n = s.length();
        this->enc_cost = encCost;
        this->flat_cost = flatCost;

        // Precompute prefix sums of '1's
        pref.assign(n + 1, 0);
        for (int i = 0; i < n; ++i) {
            pref[i + 1] = pref[i] + (s[i] - '0');
        }

        return solve(0, n - 1);
    }
};
```

## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The overall time complexity is given by:
T_total = T_prefix + T_divide_conquer

1. **Prefix Sum Construction (T_prefix):**
   - Iterating through string s of length N to build pref array takes O(N) operations.
2. **Divide and Conquer Recursion (T_divide_conquer):**
   - In the worst case (e.g., string filled with '1's), the recursion tree resembles that of Merge Sort or Segment Tree construction over an array of size N.
   - At recursion depth d, there are 2^d subproblems of size N / 2^d.
   - Each node in the recursion tree performs O(1) work (prefix sum lookup, arithmetic, min).
   - The total number of nodes in a full binary tree with N leaves is 2N - 1 = O(N).
   - Pruning occurs early whenever X = 0, reducing nodes further.

**Total Time Complexity:**
T_total = O(N) + O(N) = O(N)

### 4.2 Detailed Space Complexity Decomposition
1. **Prefix Sum Array:**
   - Vector pref of size N + 1 storing integers requires O(N) memory.
2. **Recursion Stack:**
   - The recursion halves the segment length at each step, bounding the maximum call stack depth by O(log N).

**Total Space Complexity:**
S_total = O(N) auxiliary space.

### 4.3 Complexity Summary

| Metric | Value | Notes |
|---|---|---|
| **Time Complexity** | O(N) | Total nodes visited in the binary divide-and-conquer tree is bounded by O(N). |
| **Space Complexity** | O(N) | Dominated by the prefix sum array of size N + 1 and O(log N) stack depth. |

## 5. Mathematical Formulations & Foundational Algorithms

### Range Sum Query via Prefix Sums
To compute the number of sensitive elements X in segment [L, R] in O(1) time, we define:
pref[i] = sum_{j=0}^{i-1} s[j]

The count of '1's in range [L, R] is given by:
X = pref[R + 1] - pref[L]

### Dyadic Segment Tree Recursion Bound
Let T(N) be the maximum number of recursive calls for a segment of length N:
T(N) = 2 * T(N / 2) + O(1)

By the Master Theorem (Case 1 where a = 2, b = 2, c = 0):
T(N) = O(N^(log_2 2)) = O(N)

Thus, the total work across all recursive splits is strictly linear in N.

## Pitfalls & Edge Cases

* **Integer Overflow in Cost Calculation:** L * X * encCost can exceed 2^31 - 1 when N = 10^5, X = 10^5, encCost = 10^5 (product up to 10^15). Obviated by casting variables to long long prior to multiplication.
* **Pruning All-Zero Segments:** If X = 0, splitting yields 2 * flatCost >= flatCost. Returning flatCost immediately avoids unnecessary deeper recursive calls on all-zero subsegment branches.
* **Odd Length Segments:** Halving odd lengths is invalid. Handled by checking L % 2 != 0 and prohibiting recursive splitting.

---

## Key Takeaway

When segment partitions are constrained to exact midpoint splits, treat the problem as a fixed binary tree traversal where range properties are queried in O(1) using prefix sums.

**Pattern recognition:** "split segment into two equal halves" + "cost based on range count" -> Divide and Conquer + Prefix Sum
