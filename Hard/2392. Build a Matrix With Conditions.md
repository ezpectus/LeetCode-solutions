# 2392. Build a Matrix With Conditions

## O(k² + E) — Topological Sort + Matrix Construction

---

## Problem Statement

- Given integer `k` and two constraint arrays:
  - `rowConditions[i] = [above, below]` → number `above` must be in a row strictly above number `below`
  - `colConditions[i] = [left, right]` → number `left` must be in a column strictly left of number `right`
- Build k×k matrix containing numbers 1 to k exactly once, with remaining cells = 0
- Return any valid matrix, or empty matrix if impossible

**Constraints**: 2 ≤ k ≤ 400, 1 ≤ conditions.length ≤ 10⁴

---

## Core Idea — Two Independent Topological Sorts

**Key insight**: Row and column orderings are **independent** constraints. We can solve them separately:

1. **Row ordering**: Topological sort on rowConditions → gives valid row positions for each number
2. **Column ordering**: Topological sort on colConditions → gives valid column positions for each number
3. **Matrix construction**: Place each number at (rowPos[num], colPos[num])

**Why this works**: Topological sort produces a linear ordering that respects all directed edges (constraints). If a cycle exists, no valid ordering exists → return empty matrix.

**DAG requirement**: Both constraint graphs must be Directed Acyclic Graphs (DAGs). A cycle means contradictory constraints (e.g., 1 above 2, 2 above 3, 3 above 1).

---

## Algorithm Breakdown

### Phase 1: Topological Sort (Kahn's Algorithm)

**Input**: Adjacency list representation of constraints  
**Output**: Array where `order[i]` = position of number i in the sorted order, or empty if cycle detected

```
TopologicalSort(k, conditions):
  graph = adjacency list (1..k)
  inDegree = array of in-degrees for each node
  
  // Build graph
  for each [u, v] in conditions:
    graph[u].add(v)
    inDegree[v]++
  
  // Kahn's algorithm
  queue = all nodes with inDegree == 0
  order = []
  
  while queue not empty:
    node = dequeue()
    order.append(node)
    
    for each neighbor in graph[node]:
      inDegree[neighbor]--
      if inDegree[neighbor] == 0:
        enqueue(neighbor)
  
  // Check if all nodes processed (no cycle)
  if order.length != k:
    return empty (cycle detected)
  
  // Convert to position mapping
  position = array of size k+1
  for i from 0 to k-1:
    position[order[i]] = i
  
  return position
```

**Complexity**: O(k + E) where E = number of edges (conditions)

### Phase 2: Matrix Construction

```
BuildMatrix(k, rowConditions, colConditions):
  rowPos = TopologicalSort(k, rowConditions)
  colPos = TopologicalSort(k, colConditions)
  
  if rowPos is empty OR colPos is empty:
    return empty matrix (impossible)
  
  matrix = k×k array filled with 0
  
  for num from 1 to k:
    row = rowPos[num]
    col = colPos[num]
    matrix[row][col] = num
  
  return matrix
```

**Complexity**: O(k²) for matrix initialization + O(k) for placement = O(k²)

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[][] BuildMatrix(int k, int[][] rowConditions, int[][] colConditions) {
        // Get topological orderings
        var rowOrder = TopologicalSort(k, rowConditions);
        var colOrder = TopologicalSort(k, colConditions);
        
        // If either has a cycle, impossible
        if (rowOrder.Length == 0 || colOrder.Length == 0) {
            return new int[0][];
        }
        
        // Build matrix
        int[][] matrix = new int[k][];
        for (int i = 0; i < k; i++) {
            matrix[i] = new int[k];
        }
        
        // Place each number at (rowPos[num], colPos[num])
        for (int num = 1; num <= k; num++) {
            int row = rowOrder[num];
            int col = colOrder[num];
            matrix[row][col] = num;
        }
        
        return matrix;
    }
    
    private int[] TopologicalSort(int k, int[][] conditions) {
        // Build adjacency list and in-degree array
        var graph = new List<int>[k + 1];
        int[] inDegree = new int[k + 1];
        
        for (int i = 0; i <= k; i++) {
            graph[i] = new List<int>();
        }
        
        foreach (var edge in conditions) {
            int u = edge[0], v = edge[1];
            graph[u].Add(v);
            inDegree[v]++;
        }
        
        // Kahn's algorithm
        var queue = new Queue<int>();
        for (int i = 1; i <= k; i++) {
            if (inDegree[i] == 0) {
                queue.Enqueue(i);
            }
        }
        
        var order = new List<int>();
        while (queue.Count > 0) {
            int node = queue.Dequeue();
            order.Add(node);
            
            foreach (int neighbor in graph[node]) {
                inDegree[neighbor]--;
                if (inDegree[neighbor] == 0) {
                    queue.Enqueue(neighbor);
                }
            }
        }
        
        // Cycle detection: if not all nodes processed
        if (order.Count != k) {
            return new int[0]; // Cycle exists
        }
        
        // Convert to position mapping: position[num] = index in order
        int[] position = new int[k + 1];
        for (int i = 0; i < k; i++) {
            position[order[i]] = i;
        }
        
        return position;
    }
}
```

## Why This Works — Example Walkthrough

### Example 1: `k = 3, rowConditions = [[1,2],[3,2]], colConditions = [[2,1],[3,2]]`

**Row Topological Sort**:
- Graph: 1→2, 3→2
- In-degrees: [0, 0, 2] (node 1 and 3 have 0, node 2 has 2)
- Queue start: {1, 3}
- Process 1: remove edge 1→2, inDegree[2] = 1
- Process 3: remove edge 3→2, inDegree[2] = 0, enqueue 2
- Process 2: done
- Order: [1, 3, 2] or [3, 1, 2] (both valid)
- **rowOrder**: {1→0, 3→1, 2→2} (if order was [1,3,2])

**Column Topological Sort**:
- Graph: 2→1, 3→2
- In-degrees: [1, 0, 1]
- Queue start: {3}
- Process 3: remove 3→2, inDegree[2] = 0, enqueue 2
- Process 2: remove 2→1, inDegree[1] = 0, enqueue 1
- Process 1: done
- Order: [3, 2, 1]
- **colOrder**: {3→0, 2→1, 1→2}

**Matrix Construction**:
```
Place 1 at (rowOrder[1], colOrder[1]) = (0, 2)
Place 2 at (rowOrder[2], colOrder[2]) = (2, 1)
Place 3 at (rowOrder[3], colOrder[3]) = (1, 0)

Matrix:
[0, 0, 1]
[3, 0, 0]
[0, 2, 0]
```

**Validation**:
- Row: 1 at row 0, 2 at row 2 → 1 above 2 ✓
- Row: 3 at row 1, 2 at row 2 → 3 above 2 ✓
- Col: 2 at col 1, 1 at col 2 → 2 left of 1 ✓
- Col: 3 at col 0, 2 at col 1 → 3 left of 2 ✓

### Example 2: `k = 3, rowConditions = [[1,2],[2,3],[3,1],[2,3]], colConditions = [[2,1]]`

**Row Topological Sort**:
- Graph: 1→2, 2→3, 3→1, 2→3 (duplicate)
- **Cycle detected**: 1→2→3→1
- In-degrees after building graph: [1, 1, 2]
- No node with inDegree = 0 → queue starts empty OR processes incomplete
- Order.Count = 0 or < k → **return empty**

Result: `[]` (empty matrix) ✓

## Pitfalls & Edge Cases

- **Cycle detection**: If topological sort doesn't process all k nodes, a cycle exists → impossible. Must check `order.Count == k`.
- **Duplicate conditions**: `[[1,2], [1,2]]` → graph builder handles naturally (redundant edges don't break algorithm).
- **Self-loops**: Problem states `above != below` and `left != right`, so self-loops impossible by constraint.
- **Disconnected nodes**: If a number has no constraints, it gets inDegree = 0 → processed first, can go anywhere. Valid.
- **Multiple valid answers**: Topological sort may produce different orders depending on tie-breaking (queue order). All are correct.
- **Zero-indexed vs one-indexed**: Numbers are 1 to k, but matrix indices are 0 to k-1. Careful with position mapping.

## Alternative Approach — DFS-Based Topological Sort

Instead of Kahn's (BFS), use DFS with cycle detection:

```csharp
private int[] TopologicalSortDFS(int k, int[][] conditions) {
    var graph = new List<int>[k + 1];
    for (int i = 0; i <= k; i++) graph[i] = new List<int>();
    
    foreach (var edge in conditions) {
        graph[edge[0]].Add(edge[1]);
    }
    
    var visited = new int[k + 1]; // 0=unvisited, 1=visiting, 2=visited
    var order = new List<int>();
    
    for (int i = 1; i <= k; i++) {
        if (!DFS(i, graph, visited, order)) {
            return new int[0]; // Cycle detected
        }
    }
    
    order.Reverse(); // DFS gives reverse topological order
    
    int[] position = new int[k + 1];
    for (int i = 0; i < k; i++) {
        position[order[i]] = i;
    }
    return position;
}

private bool DFS(int node, List<int>[] graph, int[] visited, List<int> order) {
    if (visited[node] == 1) return false; // Cycle: back edge to visiting node
    if (visited[node] == 2) return true;  // Already processed
    
    visited[node] = 1; // Mark as visiting
    
    foreach (int neighbor in graph[node]) {
        if (!DFS(neighbor, graph, visited, order)) return false;
    }
    
    visited[node] = 2; // Mark as visited
    order.Add(node);
    return true;
}
```

**Comparison**: Kahn's is more intuitive for detecting cycles (explicit queue size check). DFS is more compact but requires careful state tracking. Both O(k + E).

## Optimization Notes

### Early Termination

If one topological sort fails (cycle), can return immediately without computing the second:

```csharp
var rowOrder = TopologicalSort(k, rowConditions);
if (rowOrder.Length == 0) return new int[0][]; // Early exit

var colOrder = TopologicalSort(k, colConditions);
if (colOrder.Length == 0) return new int[0][];
```

### Sparse Matrix Representation

If k is very large but few conditions, most positions are flexible. Could use dictionary for sparse storage, but problem requires k×k matrix output anyway.

## Key Takeaway

"Build matrix with ordering constraints" → **Two independent topological sorts**. **Algorithm**:
1. **Topological sort** on rowConditions → row positions
2. **Topological sort** on colConditions → column positions
3. **Cycle check**: if either sort fails (cycle exists) → impossible
4. **Place numbers**: matrix[rowPos[num]][colPos[num]] = num

O(k² + E) where E = total edges. **Critical insight**: row and column orderings are independent → solve separately then combine.

Classic **constraint satisfaction via topological sort** — appears when "ordering constraints on elements" need to be satisfied. Cycle = contradiction = no solution.
