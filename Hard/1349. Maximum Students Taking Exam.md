# 1349. Maximum Students Taking Exam
## O(m · 4ⁿ) — Bitmask DP on Row-by-Row Seat Configurations

---

## Problem Statement

Given an `m × n` grid `seats` where `'.'` is a good seat and `'#'` is broken.

A student can see answers of those sitting **left, right, upper-left, and upper-right** — but not directly in front or behind.

Return the **maximum number of students** that can take the exam without any cheating being possible. Students must be placed only in good seats.

---

## Core Idea — Bitmask DP

Since `n ≤ 8`, each row's seating configuration fits in a **bitmask** of up to 8 bits (at most 256 states per row).

We do **row-by-row DP**: for each row, enumerate all valid bitmask configurations, check they don't cause cheating *within* the row (no two adjacent students), then check they don't conflict with the *previous* row (no diagonal neighbors), and only allow seating in *available* positions.

We keep:
- **`rowMask[i]`** — bitmask of available seats in row `i` (`1 = good seat`)
- **`dp[mask]`** — max students seated so far when current row config is `mask`
- At each new row, build **`ndp`** from all valid `(prev, cur)` pairs

---

## Clean Implementation (C#)

```csharp
public class Solution {
    public int MaxStudents(char[][] seats) {
        int m = seats.Length, n = seats[0].Length;

        // Build available-seat bitmask for each row
        int[] rowMask = new int[m];
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                if (seats[i][j] == '.')
                    rowMask[i] |= (1 << j);

        int states = 1 << n;
        // dp[mask] = max students when current row config is mask
        int[] dp = new int[states];
        Array.Fill(dp, -1);
        dp[0] = 0;

        for (int i = 0; i < m; i++) {
            int[] ndp = new int[states];
            Array.Fill(ndp, -1);

            for (int cur = 0; cur < states; cur++) {
                // Must be subset of available seats
                if ((cur & rowMask[i]) != cur) continue;
                // No two adjacent students in same row (left-right cheating)
                if ((cur & (cur >> 1)) != 0) continue;

                for (int prev = 0; prev < states; prev++) {
                    if (dp[prev] == -1) continue;
                    // No upper-left or upper-right diagonal cheating
                    if ((cur & (prev >> 1)) != 0) continue;   // cur sees upper-left
                    if ((cur & (prev << 1)) != 0) continue;   // cur sees upper-right

                    int count = dp[prev] + PopCount(cur);
                    ndp[cur] = Math.Max(ndp[cur], count);
                }
            }
            dp = ndp;
        }

        return Math.Max(0, dp.Max());
    }

    private int PopCount(int x) {
        int c = 0;
        while (x > 0) { c += x & 1; x >>= 1; }
        return c;
    }
}
```

## Complexity

| Metric | Value | Notes |
|--------|-------|-------|
| **Time Complexity** | O(m · 4ⁿ) | m rows × 2ⁿ current masks × 2ⁿ previous masks |
| **Space Complexity** | O(2ⁿ) | Two DP arrays of size 2ⁿ, rolled over each row |

Practical — for `n ≤ 8` the inner loop runs at most `256 × 256 × 8 ≈ 524K` iterations, well within limits.

---

## Why This Works — Example Walkthrough

**Example 3:** `5×5` grid, output = `10`

Place students in columns 1, 3, 5 of every row. In bitmask terms that's `0b10101 = 21` for each row.

- `(21 & (21 >> 1)) = (10101 & 01010) = 0` ✓ no left-right adjacency
- Diagonal check between identical rows: `(21 & (21 >> 1)) = 0` ✓ upper-left clear; `(21 & (21 << 1)) = 0` ✓ upper-right clear
- `PopCount(21) = 3` students per row × 5 rows = 15... but broken seats reduce this to 10

BFS explores all states level by level — at each row the DP transition finds the globally optimal previous configuration.

**Key insight on the diagonal check:**

When the current row has a student at column `j`, they can cheat from `(prev_row, j-1)` — that's `prev >> 1` shifted into column `j`. And from `(prev_row, j+1)` — that's `prev << 1`. Both directions must be independently cleared.

---

## Pitfalls & Edge Cases

- **Broken seats** — a configuration mask must be a subset of available seats; filter with `(cur & rowMask[i]) != cur`
- **Same-row adjacency** — check `cur & (cur >> 1) != 0` before even touching the previous row
- **Diagonal directions are asymmetric** — upper-left: `cur & (prev >> 1)`; upper-right: `cur & (prev << 1)`; both must be checked separately
- **dp initialized to -1** — unreachable states stay -1 so invalid prev configs are skipped; only `dp[0] = 0` is seeded
- **n = 1** — single column, `states = 2`; mask is either 0 or 1; no adjacency possible, just check availability
- **All seats broken** — all masks fail the availability check; `dp.Max()` returns -1, `Math.Max(0, ...)` returns 0 correctly

All handled correctly by the bitmask validity checks.

---

## Key Takeaway

The small grid size (`n ≤ 8`) is the key hint — it unlocks bitmask DP:

- Represent each row's seating as a **bitmask** of occupied seats
- Use a **dp array** over masks, transitioning row by row
- Three checks per `(cur, prev)` pair: **seats must be valid**, **no left-right neighbors**, **no diagonal neighbors with the row above**
- Two bitwise ops handle all diagonal conflict checking in O(1)
- The DP naturally finds the globally optimal assignment — greedy would fail here because a locally suboptimal row choice can unlock more students in future rows

Clean bitmask DP, not BFS, not greedy — the constraints scream it.
