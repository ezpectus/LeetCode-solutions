## Problem Statement

- Place `n` unit cubes in a corner. A box on top of another requires all 4 vertical sides of the lower box to be adjacent to a wall or another box.
- Minimize the number of boxes touching the floor.

---

## Core Idea â€” Optimal Shape is a Corner Pyramid

The optimal arrangement packs boxes in a corner using a **staircase pyramid** structure. A complete pyramid of height `h`:
- **Total boxes**: `T(h) = h*(h+1)*(h+2)/6` (tetrahedral number)
- **Floor boxes**: `F(h) = h*(h+1)/2` (triangular number)

**Algorithm**:
1. Find the largest `h` such that `T(h) â‰¤ n`.
2. Remaining boxes: `rem = n - T(h)`.
3. Add extra floor boxes one at a time along the pyramid's edge. The `k`-th extra floor box allows `k` additional boxes to be stacked above it. Increment `floor` and subtract capacity until `rem â‰¤ 0`.

**Why each extra floor box adds k capacity**: the staircase edge of a height-`h` pyramid has positions `1, 2, ..., h`. Adding a box at position `k` along the new row creates support for a stack of height `k`.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumBoxes(int n) {
        // Find largest h with T(h) = h*(h+1)*(h+2)/6 <= n
        int h = 0;
        while ((long)(h+1)*(h+2)*(h+3)/6 <= n) h++;

        int floor = h * (h+1) / 2;
        long total = (long)h * (h+1) * (h+2) / 6;

        // Add extra floor boxes until total >= n
        int k = 1;
        while (total < n) {
            floor++;
            total += k;
            k++;
        }

        return floor;
    }
}
```

## Pitfalls & Edge Cases
