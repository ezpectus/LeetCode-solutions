# 3525. Find X Value of Array II
## O((n + Q) · log n · k²) — Segment Tree with Modular Product Context Propagation

---

## Problem Statement

Array `nums`, integers `k` (≤ 5) and queries `[index, value, start, x]`. Each query:
1. (Persistently) update `nums[index] = value`.
2. Remove the prefix `nums[0..start−1]`, leaving `nums[start..n−1]`.
3. Count the ways to remove a suffix (including empty) such that the product of the remaining elements ≡ x (mod k).

The remaining elements after removing a suffix from `nums[start..n−1]` are `nums[start..j]` for some j ∈ [start, n−1]. So the answer is the count of j ∈ [start, n−1] where:

```
(nums[start] × nums[start+1] × … × nums[j]) mod k == x
```

---

## Core Insight — Reduce to Segment Tree with Propagated Modular Context

For a fixed `start`, the consecutive prefix products are:
```
P(start, start) = nums[start] mod k
P(start, j)     = P(start, j-1) × nums[j] mod k
```

Each subsequent product is determined by the previous one and the next element. This is a left-to-right **chain of multiplications** — naturally handled by a segment tree with an "accumulated context" (the product of elements seen so far from the query's left boundary).

**Key observation:** k ≤ 5, so the product mod k has only k possible values. A segment tree node covering [l, r] can store, for every possible incoming context q ∈ {0, …, k−1} and every target x ∈ {0, …, k−1}:
```
cnt[q][x] = # of j ∈ [l, r] such that (q × product(nums[l..j])) mod k == x
```
Plus the node's total product: `prod = product(nums[l..r]) mod k`.

---

## Segment Tree Structure

### Node Definition

```cpp
struct Node {
    int cnt[5][5]; // cnt[q][x]: count j in [l,r] where q*prod(l..j) ≡ x (mod k)
    int prod;      // product(nums[l..r]) mod k
};
```

Space per node: O(k²) = O(25). Total tree: O(n · k²).

### Leaf Node [i, i]

```
prod = nums[i] mod k
cnt[q][x] = 1  if (q × prod) mod k == x
           = 0  otherwise
```

### Merge(Left [l,mid], Right [mid+1,r])

```
prod = (Left.prod × Right.prod) mod k

For each q in {0..k-1}, x in {0..k-1}:
  q' = (q × Left.prod) mod k     // context for the right segment
  cnt[q][x] = Left.cnt[q][x]     // j in [l, mid]
             + Right.cnt[q'][x]   // j in [mid+1, r], with updated context
```

**Why:** For j ∈ [l, mid], the count is directly from the left node with context q. For j ∈ [mid+1, r], the context becomes q × (product of left segment) = q × Left.prod ≡ q' (mod k). ✓

### Range Query [ql, qr] with context q₀ = 1, target x

The query returns (count, product_of_range):

```
query(node[l..r], ql, qr, q):
  if disjoint:       return (0, 1)               // empty range, identity product
  if fully inside:   return (cnt[q][x], prod)    // O(1) lookup
  (lc, lp) = query(left_child, ql, qr, q)        // left half
  q' = (q × lp) mod k                            // updated context for right
  (rc, rp) = query(right_child, ql, qr, q')      // right half
  return (lc + rc, lp × rp mod k)
```

Context `q` encodes "the product of all elements in [ql, …, l-1]" — exactly the accumulated product before the current node. This is threaded through the recursion so each sub-segment sees the correct context. ✓

---

## Algorithm

```
Build segment tree on nums (O(n · k²)).
For each query [index, value, start, x]:
  1. Point-update: nums[index] = value (O(log n · k²)).
  2. Range query [start, n-1] with context q=1, target x (O(log n · k)).
  3. Append count to result.
```

## Full Optimal Implementation (C++)

```cpp
class Solution {
    struct Node{
    // record the all [left, right] prefix product.
    // Example: (left), (left, left+1), ... (left,... right)
    public:
        Node(){ // empty node
            fill(remain, remain+5, 0);
            prod = 1;
        }
        int remain[5]; 
        int prod; 
    };

    class SegmentTree{ // 0->1,2 (2n+1 and 2n+2)
        int k, n;
        vector<Node> nodes;
    public:
        SegmentTree(vector<int>& nums, int k){
            this->k = k;
            this->n = nums.size();
            nodes.resize(4*(this->n));
            build(nums, 0, 0, n-1);
        }
        void update(int cur, int left, int right, int target, int val){
            if(left == right){
                for(int i=0; i<k; ++i) nodes[cur].remain[i] = 0;
                nodes[cur].remain[val] = 1;
                nodes[cur].prod = val;
                return;
            }
            int mid = (left + right) / 2;
            if(target <= mid)
                update(2*cur+1, left, mid, target, val);
            else 
                update(2*cur+2, mid+1, right, target, val);
            merge(cur);
        }
        Node query(int cur, int qL, int qR, int left, int right){
            if(qR < left || qL > right) return Node(); // empty
            if(qL <= left && right <= qR) return nodes[cur];
            int mid = (left + right) / 2;
            Node nLeft = query(2*cur+1, qL, qR, left, mid);
            Node nRight = query(2*cur+2, qL, qR, mid+1, right);
            merge(nLeft, nRight);
            return nLeft;
        }
    private:
        void build(vector<int>& nums, int cur, int left, int right){
            if(left == right){
                nodes[cur].remain[nums[left]] = 1;
                nodes[cur].prod = nums[left];
                return;
            } 
            int mid = (left + right)/2;
            build(nums, 2*cur+1, left, mid);
            build(nums, 2*cur+2, mid+1, right);
            merge(cur);
        }
        void merge(int cur){
            // merge the 2*cur+1 and 2*cur+2 node to cur
            int left = 2*cur+1;
            int right = 2*cur+2;
            nodes[cur].prod = (nodes[left].prod * nodes[right].prod) % k;
            // The part [left ~ mid]'s prefix still include in the [left, right] prefix.
            for(int i=0; i<k; ++i) nodes[cur].remain[i] = nodes[left].remain[i];
            for(int i=0; i<k; ++i){
                int newVal = (i*nodes[left].prod) % k;
                nodes[cur].remain[newVal] += nodes[right].remain[i];
            }
        }
        void merge(Node& n1, Node& n2){
            // merge the n1 and n2 node to n1
            for(int i=0; i<k; ++i){
                int newVal = (i*n1.prod) % k;
                n1.remain[newVal] += n2.remain[i];
            }
            n1.prod = (n1.prod * n2.prod) % k;
        }
    };
public:
    vector<int> resultArray(vector<int>& nums, int k, vector<vector<int>>& queries) {
        int n = nums.size();
        int q = queries.size();
        for(auto& val : nums){
            val %= k;
        }
        for(auto& query : queries){
            query[1] %= k;
        }
        SegmentTree segmentTree(nums, k);
        vector<int> res;
        for(auto& query : queries){
            int ind = query[0];
            int value = query[1];
            int start = query[2];
            int x = query[3];
            // update nums
            segmentTree.update(0, 0, n-1, ind, value);
            res.push_back(segmentTree.query(0, start, n-1, 0, n-1).remain[x]);
        }
        return res;
    }
};
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Build | O(n · k²) | Each of 2n nodes: O(k²) to fill cnt |
| Point update | O(log n · k²) | O(log n) nodes updated, each O(k²) |
| Range query | O(log n · k) | O(log n) node visits, each O(k) for cnt lookup |
| Total | **O((n + Q) · log n · k²)** | n=10⁵, Q=2×10⁴, k=5 → ≈10⁷ ops |
| Space | O(n · k²) | Tree with 4n nodes, each k×k matrix |

### Example 1 — `nums=[1,2,3,4,5]`, `k=3`, queries=`[[2,2,0,2],[3,3,3,0],[0,1,0,1]]`

**Query 0: update nums[2]=2 → nums=[1,2,2,4,5]. Query [0,4], x=2.**

Prefix products mod 3: 1, 2, 4%3=1, 16%3=1, 80%3=2.

| j | Product mod 3 | == 2? |
|---|---|---|
| 0 | 1 | ✗ |
| 1 | 2 | ✓ |
| 2 | 1 | ✗ |
| 3 | 1 | ✗ |
| 4 | 2 | ✓ |

Count = **2** ✓

**Query 1: update nums[3]=3 → nums=[1,2,2,3,5]. Query [3,4], x=0.**

Products from start=3: 3%3=0, (3×5)%3=0.

Both ≡ 0. Count = **2** ✓

**Query 2: update nums[0]=1 → nums=[1,2,2,3,5]. Query [0,4], x=1.**

Products mod 3: 1, 2, 4%3=1, 12%3=0, 60%3=0.

j=0: 1 ✓. j=2: 1 ✓. Count = **2** ✓

### Merge Trace — k=3, segment [1..2] (nums=[2,2])

**Leaf [1]: prod=2. cnt[q][x]:** (q×2)%3==x.
- q=0: 0→(0,0). q=1: 2→(1,2). q=2: 4%3=1→(2,1).

**Leaf [2]: prod=2. cnt[q][x]:** same as leaf [1].

**Merge [1..2]:**
- res.prod = 2×2%3 = 1.
- For q=0: q'=0×2%3=0. cnt[0][x] = leaf1.cnt[0][x] + leaf2.cnt[0][x].
  - leaf1.cnt[0][0]=1, leaf2.cnt[0][0]=1 → res.cnt[0][0]=2. (Both j=1 and j=2 give product 0×prod≡0.)
  - Wait, leaf1.cnt[0][0] = 1 means: 0 × product(nums[1..1]) = 0×2 = 0 ≡ 0 (mod 3). So j=1 gives 0. And j=2: 0 × product(nums[2..2]) = 0×2 = 0. So res.cnt[0][0] = 2. ✓
- For q=1: q'=1×2%3=2. cnt[1][x] = leaf1.cnt[1][x] + leaf2.cnt[2][x].
  - leaf1.cnt[1][2]=1 (j=1 gives 2). leaf2.cnt[2][1]=1 (j=2: 2×2=4≡1). → res.cnt[1][2]=1, res.cnt[1][1]=1.
  - Meaning: with context q=1, j=1 gives product 1×2=2≡2, j=2 gives 1×2×2=4≡1. ✓

### Context Propagation in Query — Example 1, Query 0

Query range [0,4], x=2, initial q=1. nums=[1,2,2,4,5].

Suppose tree has segments [0..2] and [3..4] at some level.

Processing [0..2] first with q=1:
- Returns count=1 (j=1 gives 1×1×2≡2), prod of [0..2] = 1×2×2%3=1.

Update context: q' = 1 × 1 % 3 = 1.

Processing [3..4] with q'=1:
- products(nums[3..3])=4%3=1, products(nums[3..4])=20%3=2.
- cnt[1][2] for this segment: j=4 gives 1×4×5=20≡2. Count=1.

Total = 1 + 1 = 2. ✓

## Pitfalls & Edge Cases

- **Context `q=1` for the initial query call:** q represents "accumulated product before this segment." At the query start, no elements have been processed, so q=1 (multiplicative identity). Passing q=0 would mean "anything times 0 ≡ 0", giving wrong results for all non-zero x. ✓

- **Empty intersection returns `(0, 1)` (product=1):** When a subtree is fully outside the query range, its "product" should be the identity element 1 (so it doesn't affect the context update `nq = q × 1 = q`). Returning 0 would zero out the context and corrupt subsequent segments. ✓

- **`cnt[q][x]` is defined relative to the segment's LEFT boundary:** `cnt[q][x]` counts j where q × product(nums[**l**..j]) ≡ x. When doing a range query and we reach a full-overlap node, the passed `q` already encodes the product from `ql` to `l-1`. So `q × product(nums[l..j])` = `product(nums[ql..j])`, and we count exactly the right j values. ✓

- **Persistent updates (carry over across queries):** The problem says "Only this step [update] persists for the rest of the queries." The segment tree naturally handles this — each update modifies the tree in place and all subsequent queries see the updated values. ✓

- **k ≤ 5 bounds cnt values:** Each `cnt[q][x]` ≤ n = 10⁵, fitting in `int`. The total count per query ≤ n, also fitting in `int` (returned as `vector<int>`). ✓

- **Merge cost is O(k²):** The double loop `for q in {0..k-1}` and `for x in {0..k-1}` is at most 5×5=25 iterations. With k=1, this degenerates to a single counter (trivial). ✓

---

## Key Takeaway

**"Count range [ql, qr] positions where the prefix-product from ql (accumulated left-to-right) hits a target modular residue"** → segment tree where each node stores a k×k matrix `cnt[incoming_context][result]`. Merge propagates the context through the matrix in O(k²). Range queries thread the context through canonical segments in O(log n × k) total.

**Pattern recognition:** "Count subarrays/prefixes satisfying a product or XOR modular condition, with point updates" → the condition produces a finite state machine with k states → segment tree nodes store k-state-to-count matrices → merge = matrix composition over the state transition function. This pattern generalizes to any binary operation on a small finite group/monoid (XOR, AND, OR, modular multiplication with small modulus).
