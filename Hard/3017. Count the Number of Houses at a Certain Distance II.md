# 3017. Count the Number of Houses at a Certain Distance II

## O(N) — Pivot-Based Boundary Line Sweep

### Problem Statement
You are given three positive integers `n`, `x`, and `y`. Houses numbered `1` to `n` are connected sequentially in a line by $n - 1$ streets. An additional street connects house `x` with house `y`.

For each distance $k$ from $1$ to $n$, find the total number of pairs of houses $(house_1, house_2)$ such that the minimum number of streets to travel between them is exactly $k$. Return a 1-indexed array representing these counts.

---

### Core Idea — Pivot-Based Point Mapping
This approach targets specific landmark pivot points inside the graph rather than completely decoupling lines and cycles. By utilizing the absolute cycle midpoint $m = y - \lfloor(y-x)/2\rfloor$, we can calculate boundaries for all pairs where $j > i$:

1. **Right Tail Nodes ($i \ge y$):** These elements cannot effectively optimize their paths to the left using the shortcut, so their distance increments linearly.
2. **Cycle Midpoint Nodes ($m \le i < y$):** These nodes fill their localized intervals directly up to the right boundary.
3. **Cycle Entry Nodes ($x \le i < m$):** Paths are split into three zones based on whether they reach up to the midpoint `to`, drop down from the midpoint back to $y$, or escape into the right tail ($y+1 \dots n$).
4. **Left Tail Nodes ($i < x$):** Paths scale straight up to $m$, loop around using the $x-y$ edge to cover the back half of the cycle, and then scale into the rightmost nodes.

We track all coordinate ranges using a 1-indexed difference array `prefix` to finalize the counting in a single sweep.

---

### Full Optimal Implementation (C++)

```cpp
#include <vector>
#include <algorithm>

using namespace std;
typedef long long int ll;

class Solution {
public:
    vector<long long> countOfPairs(int n, int x, int y) {
        vector<ll> prefix(n + 2, 0);
        
        if (x > y) swap(x, y);
        
        // Base Case: Shortcut does not exist or is redundant
        if (x == y || x + 1 == y) {
            for (int i = 1; i <= n; i++) {
                prefix[1]++;
                prefix[n - i + 1]--;
            }
        }
        else {
            // 1. Process paths starting from right tail nodes
            for (int i = y; i <= n; i++) {
                prefix[1]++;
                prefix[n - i + 1]--;
            }

            int m = y - (y - x) / 2;
            
            // 2. Process paths starting from nodes between midpoint and y
            for (int i = m; i < y; i++) {
                prefix[1]++;
                prefix[n - i + 1]--;
            }

            // 3. Process paths starting from nodes between x and midpoint
            for (int i = x, to = m; i < m; i++, to++) {
                // Interval: i ... to
                prefix[1]++;
                prefix[to - i + 1]--;
                
                if (to + 1 <= y) {
                    // Interval: to+1 ... y via shortcut
                    prefix[(i - x) + 1]++;
                    prefix[(i - x) + (y - to + 1)]--;
                }
                
                if (to + 1 <= n) {
                    // Interval: y+1 ... n via shortcut
                    prefix[(i - x + 1) + 1]++;
                    prefix[(i - x + 1) + n - y + 1]--;
                }
            }

            // 4. Process paths starting from left tail nodes
            for (int i = 1; i < x; i++) {
                // Interval: i ... m
                prefix[1]++;
                prefix[m - i + 1]--;

                if (m + 1 <= y - 1) {
                    // Interval: m+1 ... y-1 via shortcut
                    prefix[(x - i + 1) + 1]++;
                    prefix[(x - i + 1) + (y - m)]--;
                }
            
                // Interval: y ... n via shortcut
                prefix[(x - i) + 1]++;
                prefix[(x - i) + (n - y + 2)]--;
            }
        }
        
        // Collect prefix sums and apply bidirectional factor
        vector<ll> result(n, 0);
        for (int j = 1; j <= n; j++) {
            prefix[j] += prefix[j - 1];
            result[j - 1] = prefix[j] * 2;
        }
        
        return result;
    }
};
```


### Complexity

| Metric | Value | Notes |
| :--- | :--- | :--- |
| **Time Complexity** | $O(N)$ | Iterates through sequential intervals ($1 \dots n$) exactly once. Inside the loop, all range boundary assignments run in $O(1)$ constant time. |
| **Space Complexity** | $O(N)$ | Driven by the size of the `prefix` tracking buffer and the `result` vector. |

---

### Pitfalls & Edge Cases

* **Dynamic Right Bounds (`to++`):** In the third loop, the `to` pointer mirrors the progression of index `i`. This represents the shifting boundary inside the cycle where the direct path and shortcut distances balance out perfectly.
* **Offset Scaling Factors:** When jumping from a left tail node into the shortcut, the baseline offset distance is $(x - i)$. Exiting out of the cycle introduces an additional constant shift that must be carefully tracked inside the `prefix[...]` bracket indices.

---

### Key Takeaway

> If structural components within a graph overlap heavily, find the exact geographic midpoint of the loop overlay. Using the midpoint as a static pivot point allows you to map all path choices to clean interval updates on a difference array.

* **Pattern recognition:** "Find path count distributions across an anchored graph shortcut" -> Midpoint Pivot Inversion + 1D Prefix Range Update.
