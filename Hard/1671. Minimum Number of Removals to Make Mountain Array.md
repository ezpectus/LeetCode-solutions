# 1671. Minimum Number of Removals to Make Mountain Array
## O(nВІ) вҖ” LIS from Left + LIS from Right, Peak Enumeration

---

## Problem Statement

- Given integer array `nums`, return the **minimum number of elements to remove** to make it a mountain array.
- A mountain array has a peak index `i` (not first/last) such that it strictly increases up to `i` and strictly decreases after.

---

## Core Idea вҖ” LIS from Both Sides, Enumerate Peak

**Key insight**: a mountain array is exactly a **bitonic subsequence** вҖ” strictly increasing then strictly decreasing. To maximize what we **keep** (minimizing removals), we want the **longest bitonic subsequence**.

For each candidate peak index `i`:
- `lis[i]` = length of Longest Increasing Subsequence **ending at** `i` (left side of mountain)
- `lds[i]` = length of Longest Increasing Subsequence **ending at** `i` going from right (= longest decreasing subsequence **starting at** `i`)

The longest mountain with peak at `i` = `lis[i] + lds[i] - 1` (subtract 1 to not double-count the peak).

**Valid peak**: `i` must not be the first or last element, and must have both `lis[i] > 1` and `lds[i] > 1` (i.e., the mountain has at least one element on each side).

**Answer**: `n вҲ’ max(lis[i] + lds[i] вҲ’ 1)` over all valid peaks `i`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumMountainRemovals(int[] nums) {
        int n = nums.Length;
        int[] lis = new int[n];
        int[] lds = new int[n];

        // LIS ending at each index (left to right)
        for (int i = 0; i < n; i++) {
            lis[i] = 1;
            for (int j = 0; j < i; j++)
                if (nums[j] < nums[i])
                    lis[i] = Math.Max(lis[i], lis[j] + 1);
        }

        // LIS ending at each index from right (= LDS starting at each index)
        for (int i = n - 1; i >= 0; i--) {
            lds[i] = 1;
            for (int j = n - 1; j > i; j--)
                if (nums[j] < nums[i])
                    lds[i] = Math.Max(lds[i], lds[j] + 1);
        }

        // Find longest valid mountain
        int best = 0;
        for (int i = 1; i < n - 1; i++)
            if (lis[i] > 1 && lds[i] > 1)
                best = Math.Max(best, lis[i] + lds[i] - 1);

        return n - best;
    }
}
```

## Why This Works вҖ” Bitonic Subsequence via LIS

**Why compute LIS from both sides?**

A valid mountain with peak at `i` requires:
- A strictly increasing sequence from some left elements up to `i` вҶ’ longest such = `lis[i]`
- A strictly decreasing sequence from `i` down to some right elements вҶ’ longest such = `lds[i]`

These two are independent given the peak, so the total mountain length with peak at `i` is `lis[i] + lds[i] - 1`.

**Why `lis[i] > 1` and `lds[i] > 1`?**

The peak cannot be the first or last element of the mountain вҖ” it must have at least one element strictly smaller on each side. `lis[i] > 1` guarantees at least one element to the left strictly less than `nums[i]`; `lds[i] > 1` guarantees at least one to the right.

**Why minimize removals = maximize kept elements?**

Elements removed = `n вҲ’ (elements kept)`. The maximum elements we can keep that form a valid mountain = the longest bitonic subsequence = `max(lis[i] + lds[i] - 1)`.

## Pitfalls & Edge Cases

- **Peak validity check `lis[i] > 1 && lds[i] > 1`**: without this, a peak at the boundary with nothing on one side passes incorrectly
- **Don't skip index 0 and n-1** in LIS/LDS computation вҖ” they're valid sequence elements, just not valid peaks
- **Strictly increasing**: equal elements don't count вҖ” use strict `<` not `<=` in comparisons
- **`n` is guaranteed to allow a mountain**: problem states a valid mountain always exists, so `best` will always be вүҘ 3

---

## Key Takeaway

"Minimum removals to form mountain array" вҶ’ **maximum bitonic subsequence** via LIS from both sides:

```
lis[i] = LIS length ending at i (left to right)
lds[i] = LIS length ending at i (right to left)

best = max(lis[i] + lds[i] - 1)  for all valid peaks i
answer = n - best
```

Valid peak: `1 вүӨ i вүӨ n-2`, `lis[i] > 1`, `lds[i] > 1`. Two O(nВІ) passes вҶ’ O(nВІ) total.
