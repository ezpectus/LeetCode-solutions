# 1960. Maximum Product of the Length of Two Palindromic Substrings
## O(n) â€” Manacher's Algorithm + Shrink Propagation + Prefix/Suffix Max

---

## Problem Statement

- Given string `s`, find two **non-intersecting palindromic substrings of odd length**.
- Choose `i â‰¤ j < k â‰¤ l` such that `s[i..j]` and `s[k..l]` are both odd-length palindromes.
- Return the **maximum product** of their lengths.

---

## Core Idea â€” Manacher + Left/Right Best Arrays

**Step 1 â€” Manacher's Algorithm**

Compute `p[i]` = radius of the longest odd palindrome centered at `i`. Palindrome spans `[i-p[i], i+p[i]]` with length `2*p[i]+1`.

**Step 2 â€” Build `left[i]`** = longest odd palindrome length ending **at or before** index `i`.

- For each center `c`, mark: `bestEnd[c + p[c]] = max(..., 2*p[c]+1)`
- **Shrink propagation** (right â†’ left): a palindrome of length `L` ending at `i` â†’ palindrome of length `L-2` ending at `i-1` (peel off outer chars). So: `bestEnd[i-1] = max(bestEnd[i-1], bestEnd[i] - 2)`
- **Prefix max** (left â†’ right): `left[i] = max(left[i], left[i-1])`

**Step 3 â€” Build `right[i]`** = longest odd palindrome length starting **at or after** index `i`.

- For each center `c`, mark: `bestStart[c - p[c]] = max(..., 2*p[c]+1)`
- **Shrink propagation** (left â†’ right): a palindrome of length `L` starting at `i` â†’ palindrome of length `L-2` starting at `i+1`. So: `bestStart[i+1] = max(bestStart[i+1], bestStart[i] - 2)`
- **Suffix max** (right â†’ left): `right[i] = max(right[i], right[i+1])`

**Step 4 â€” Answer**: `max over all j: left[j] Ă— right[j+1]`

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaxProduct(string s) {
        int n = s.Length;
        int[] p = Manacher(s);

        // --- left[i]: longest odd palindrome ending at or before i ---
        long[] bestEnd = new long[n];
        Array.Fill(bestEnd, 1);
        for (int c = 0; c < n; c++)
            bestEnd[c + p[c]] = Math.Max(bestEnd[c + p[c]], 2 * p[c] + 1);
        // Shrink: palindrome ending at i of length L â†’ length L-2 ending at i-1
        for (int i = n - 1; i > 0; i--)
            bestEnd[i - 1] = Math.Max(bestEnd[i - 1], bestEnd[i] - 2);
        // Prefix max
        long[] left = (long[])bestEnd.Clone();
        for (int i = 1; i < n; i++)
            left[i] = Math.Max(left[i], left[i - 1]);

        // --- right[i]: longest odd palindrome starting at or after i ---
        long[] bestStart = new long[n];
        Array.Fill(bestStart, 1);
        for (int c = 0; c < n; c++)
            bestStart[c - p[c]] = Math.Max(bestStart[c - p[c]], 2 * p[c] + 1);
        // Shrink: palindrome starting at i of length L â†’ length L-2 starting at i+1
        for (int i = 0; i < n - 1; i++)
            bestStart[i + 1] = Math.Max(bestStart[i + 1], bestStart[i] - 2);
        // Suffix max
        long[] right = (long[])bestStart.Clone();
        for (int i = n - 2; i >= 0; i--)
            right[i] = Math.Max(right[i], right[i + 1]);

        long ans = 0;
        for (int j = 0; j < n - 1; j++)
            ans = Math.Max(ans, left[j] * right[j + 1]);

        return ans;
    }

    private int[] Manacher(string s) {
        int n = s.Length;
        int[] p = new int[n];
        int c = 0, r = 0;
        for (int i = 0; i < n; i++) {
            int mirror = 2 * c - i;
            if (i < r) p[i] = Math.Min(r - i, p[mirror]);
            while (i - p[i] - 1 >= 0 && i + p[i] + 1 < n && s[i - p[i] - 1] == s[i + p[i] + 1])
                p[i]++;
            if (i + p[i] > r) { c = i; r = i + p[i]; }
        }
        return p;
    }
}
```

## Why This Works

**Why Manacher's Algorithm?**

Computing all palindrome radii naively is O(nÂ˛). Manacher exploits symmetry: if we've found a palindrome reaching position `r` centered at `c`, then for any new center `i < r`, its radius is at least `min(r-i, p[2c-i])`. This amortizes to O(n) total.

**Why shrink propagation for `bestEnd`?**

A palindrome `s[a..b]` of odd length `L` ending at `b` contains a shorter palindrome `s[a+1..b-1]` of length `L-2` ending at `b-1` (remove both outer characters â€” still a palindrome, still odd length). So any position that is the right end of a long palindrome is also the right end of all shorter palindromes of the same character structure. Propagating rightâ†’left: `bestEnd[i-1] = max(bestEnd[i-1], bestEnd[i] - 2)`.

**Why shrink propagation for `bestStart`?**

Symmetrically: palindrome starting at `i` of length `L` â†’ palindrome starting at `i+1` of length `L-2`. Propagate leftâ†’right: `bestStart[i+1] = max(bestStart[i+1], bestStart[i] - 2)`.

**Why prefix/suffix max after shrink propagation?**

`left[i]` should reflect the best palindrome ending **anywhere at or before** `i` â€” not just at `i` itself. The prefix max collapses the array so each position inherits the best from all positions to its left. Symmetric reasoning for `right`.

**The critical bug to avoid**: `right`'s shrink propagation must go **leftâ†’right** (NOT rightâ†’left). A palindrome starting at position `i` shrinks into one starting at `i+1` â€” later positions, not earlier. Using rightâ†’left propagation would incorrectly propagate a palindrome's length to positions where it cannot start.

## Pitfalls & Edge Cases

- **Shrink direction is asymmetric**: `bestEnd` shrinks rightâ†’left; `bestStart` shrinks leftâ†’right â€” swapping these is the critical bug
- **Prefix max for `left`, suffix max for `right`**: without these, positions that benefit from a nearby long palindrome aren't captured
- **Initialize all arrays to 1**: every single character is a valid length-1 palindrome
- **Manacher is for odd palindromes only**: no `#`-padding needed â€” this problem guarantees odd lengths required

## Key Takeaway

"Max product of two non-intersecting odd palindromes" â†’ **Manacher + shrink propagation + prefix/suffix max**:

```
p = Manacher(s)  // odd palindrome radii

// LEFT: longest palindrome ending at or before i
bestEnd[c+p[c]] = max(..., 2p[c]+1)
for i rightâ†’left: bestEnd[i-1] = max(bestEnd[i-1], bestEnd[i]-2)  // shrink
for i leftâ†’right: left[i] = max(left[i], left[i-1])               // prefix max

// RIGHT: longest palindrome starting at or after i  
bestStart[c-p[c]] = max(..., 2p[c]+1)
for i leftâ†’right: bestStart[i+1] = max(bestStart[i+1], bestStart[i]-2)  // shrink
for i rightâ†’left: right[i] = max(right[i], right[i+1])                  // suffix max

answer = max(left[j] * right[j+1])
```

O(n) time, O(n) space.
