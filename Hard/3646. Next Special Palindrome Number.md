# 3646. Next Special Palindrome Number
## O(K log K) — Bitmask Subset Enumeration + Permutation Generation + Binary Search

---

## Problem Statement

Number n (0 ≤ n ≤ 10¹⁵). A number is **special** if it is a palindrome AND every digit k that appears in it appears exactly k times. Return the smallest special number strictly greater than n.

---

## Core Insight 1 — Valid Digit Subsets Have At Most One Odd Digit

If digit k is chosen, it appears exactly k times. A palindrome can have at most one digit with an odd frequency (the center digit in an odd-length palindrome). Since digit k has frequency k:
- k even → appears an even number of times (no constraint on palindrome center). ✓
- k odd → appears an odd number of times (contributes to center).

**Rule:** A valid subset S ⊆ {1,…,9} of chosen digits must have **at most one odd-valued digit.**

Available digits:
- **Odd values:** {1,3,5,7,9} — choose 0 or 1.
- **Even values:** {2,4,6,8} — choose any subset.

This gives at most (1+5) × 2⁴ − 1 = **95 valid non-empty subsets**, all of which can be enumerated efficiently.

---

## Core Insight 2 — Build Palindromes From First Half + Optional Middle

For subset S, total length L = Σ_{k∈S} k. The palindrome structure:
- **First half** (positions 0 to ⌊L/2⌋−1): for each digit k, contributes ⌊k/2⌋ copies.
- **Middle digit** (position ⌊L/2⌋, only if L odd): the unique odd-k digit in S.
- **Second half**: mirror of first half.

Each distinct permutation of the first half generates one distinct special palindrome.

**Example (S = {2,3}):** L=5. First half = [2, 3] (one 2, one 3). Middle = 3.
- Permutation [2,3] → palindrome: 2**3**3**3**2 = "23332" ✓
- Permutation [3,2] → palindrome: 3**2**3**2**3 = "32323" ✓

---

## Core Insight 3 — Total Count Is Small; Precompute All and Binary Search

For L ≤ 18 (covering all special numbers ≤ 10^17), the first half has ≤ 9 digits. The maximum distinct permutations for any single subset is 9!/(2!3!4!) = 1260 (for S = {4,6,8}). Summing across all valid subsets, the total special numbers is in the thousands — easily sorted and binary-searched.

**Why 18 digits is enough:** n ≤ 10^15 has at most 16 digits. All 15-digit special numbers are < 10^15. The smallest 16-digit special number (≈ 2.67 × 10^15) is always the answer if n ≥ all 15-digit specials. We never need specials longer than 17–18 digits.

---

## Algorithm

```
1. For each non-empty bitmask over {1,...,9}:
   a. Check: at most one odd digit selected. Skip if ≥ 2 odd digits.
   b. Build first_half: for digit k, add ⌊k/2⌋ copies.
   c. Identify middle digit (the odd k in S, or -1 if none).
   d. Skip if total length > 18 (won't be needed for n ≤ 10^15).
   e. Generate all permutations of first_half using next_permutation.
   f. For each permutation: form palindrome = first_half + [middle] + reverse(first_half).
   g. Add numeric value to the special numbers list.
2. Sort and deduplicate the list.
3. Binary search for the first element strictly greater than n.
```

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    long long specialPalindrome(long long n) {
        vector<long long> special;

        // Enumerate all non-empty subsets of {1,...,9}
        for (int mask = 1; mask < (1 << 9); mask++) {
            // Check: at most one odd digit
            int odd_digit = -1;
            bool valid = true;
            for (int k = 1; k <= 9; k++) {
                if ((mask >> (k - 1)) & 1) {
                    if (k & 1) {
                        if (odd_digit != -1) { valid = false; break; }
                        odd_digit = k;
                    }
                }
            }
            if (!valid) continue;

            // Build the first-half digit sequence
            vector<int> half;
            for (int k = 1; k <= 9; k++) {
                if ((mask >> (k - 1)) & 1) {
                    int cnt = (k & 1) ? (k - 1) / 2 : k / 2;
                    for (int i = 0; i < cnt; i++) half.push_back(k);
                }
            }

            int total_len = 2 * (int)half.size() + (odd_digit != -1 ? 1 : 0);
            if (total_len > 18) continue; // exceeds long long range or not needed

            // Generate all distinct permutations of the first half
            sort(half.begin(), half.end());
            do {
                string s;
                for (int d : half)  s += char('0' + d);
                if (odd_digit != -1) s += char('0' + odd_digit);
                for (int i = (int)half.size() - 1; i >= 0; i--) s += char('0' + half[i]);
                special.push_back(stoll(s));
            } while (next_permutation(half.begin(), half.end()));
        }

        sort(special.begin(), special.end());
        special.erase(unique(special.begin(), special.end()), special.end());

        // Find the smallest special number strictly greater than n
        return *upper_bound(special.begin(), special.end(), n);
    }
};
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Bitmask enumeration | O(2⁹ × 9) = O(4608) | 512 masks × checking 9 bits |
| Permutations per subset | ≤ 1260 | S={4,6,8}: 9!/(2!3!4!) |
| Total special numbers K | ≤ ~5,000 | All subsets × their permutation counts |
| Sort | O(K log K) | Negligible |
| Binary search | O(log K) | O(1) effectively |
| **Total** | **O(K log K)** | Effectively constant |

### Tracing the Generation for S = {2,3}

- odd_digit = 3. half = [2, 3] (one 2 from k=2, one 3 from k=3).
- total_len = 2×2 + 1 = 5. ✓
- Sorted half: [2, 3]. next_permutation generates [2,3], then [3,2].
  - [2,3]: s = "2" + "3" + "32" = "23332". stoll → 23332. ✓
  - [3,2]: s = "3" + "3" + "23" = "32323". stoll → 32323. ✓ (32323 is a palindrome with 3 appearing 3×, 2 appearing 2×.)

---

### Why n ≤ 10¹⁵ Always Has a 16-Digit or Smaller Answer

**15-digit specials:** All are ≤ 999,999,999,999,999 = 10¹⁵ − 1. Maximum 15-digit special (from S={9,6}): palindrome ≈ 9.99 × 10¹⁴ < 10¹⁵.

**Smallest 16-digit special (from S={2,6,8}):** Length 16, first half = [2,6,6,6,8,8,8,8] (sorted). Palindrome = 2666888888886662 ≈ 2.67 × 10¹⁵.

For n = 10¹⁵ − 1: answer = 2,666,888,888,886,662 ✓ (smallest 16-digit special).

---

### Why Only {2,6,8} Gives Length-16 Specials

For Σk = 16 from distinct digits {1,...,9}:
- Even-only subsets of {2,4,6,8} summing to 16: only **{2,6,8}** = 2+6+8 = 16.
- With one odd digit k: we need even sum = 16−k. Since 16−k is odd for all odd k, and sums of even digits are always even, **no valid subset** exists with an odd digit.

Thus S={2,6,8} is the unique source of all 280 length-16 special numbers.

## Pitfalls & Edge Cases

- **Digit 0 never appears:** Digit 0 appears 0 times — it simply isn't in the number. Our enumeration over {1,...,9} naturally excludes it. ✓

- **No leading zeros:** Every digit is ≥ 1, so the first (and last) digit of any palindrome is ≥ 1. ✓

- **`next_permutation` on sorted input:** Starting from the sorted sequence generates all distinct permutations exactly once, handling repeated digits (e.g., [6,6,6,8,8,8,8] for S={6,8}) without duplicates. ✓

- **`unique` call:** Different subsets S cannot produce the same palindrome (the digit multiset uniquely determines S). `next_permutation` also generates each distinct permutation once. The `unique` call is a safety net with no practical effect. ✓

- **Length limit of 18:** The answer for any n ≤ 10¹⁵ is at most a 16-digit number (≤ 10^17), well within the 18-character cut. `long long` safely holds values up to ~9.2 × 10¹⁸. ✓

- **n = 0:** Smallest special is 1. `upper_bound(0)` points to 1 in the list. ✓

---

## Key Takeaway

**"Special palindrome" = palindrome where each digit k appears exactly k times.** The two conditions together constrain the valid digit subsets to those with at most one odd-valued digit (≤ 95 subsets). For each subset, permuting the first half and mirroring generates all palindromes — a total of only ~5,000 special numbers up to 18 digits.

**Pattern recognition:** When a number's structure is defined by a very small combinatorial space (chosen digit subset + permutation of half), enumerate all candidates, sort once, then answer each query in O(log K). This "precompute + binary search" pattern is ideal when the candidate space is provably small despite the numbers themselves being large.
