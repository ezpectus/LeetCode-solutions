# LeetCode 3455. Shortest Matching Substring

## 1. Constraints & Complexity Bottleneck Analysis
- **String `s` length:** `1 <= s.length <= 10^5`
- **Pattern `p` length:** `2 <= p.length <= 10^5`
- **Character Set:** Lowercase English letters, with exactly two `*` in `p`.

### Bottleneck Analysis
A naive substring matching or backtracking approach to find all combinations of matches for the wildcards would run in $O(|s|^2)$ or worse, which immediately triggers Time Limit Exceeded (TLE) under $|s|, |p| \le 10^5$. 

The core bottleneck is to find all occurrences of the fixed subpatterns in $s$ efficiently and then find a triplet of indices $(i, j, k)$ corresponding to matches of the three segments such that:
- Segment 0 starts at $i$ and ends at $i + |p_0| - 1$.
- Segment 1 starts at $j$ and ends at $j + |p_1| - 1$, with $j \ge i + |p_0|$.
- Segment 2 starts at $k$ and ends at $k + |p_2| - 1$, with $k \ge j + |p_1|$.
- The total window size $(k + |p_2| - i)$ is minimized.

To achieve this in optimal time, we must locate all occurrences of the subpatterns in $O(|s| + |p|)$ and perform the lookup combination in no more than $O(|s| \log |s|)$ steps.

## 2. Algorithm Design & Mathematical Invariants

### 2.1 Core Concept
Since the pattern $p$ contains exactly two asterisks, we can split it into three segments:
- `p0`: Characters before the first `*`
- `p1`: Characters between the first and second `*`
- `p2`: Characters after the second `*`

We use the Knuth-Morris-Pratt (KMP) algorithm to scan string $s$ and record all starting indices where `p0`, `p1`, and `p2` occur. Let these sorted lists of starting indices be `pos0`, `pos1`, and `pos2`.

Once we have these positions, for each candidate starting index `start0` in `pos0`:
1. The end of the first segment is `end0 = start0 + len(p0)`.
2. We need the earliest starting index of `p1` that is greater than or equal to `end0`. We can find this efficiently using binary search (`std::lower_bound`) on `pos1`. Let this be `start1`.
3. The end of the second segment is `end1 = start1 + len(p1)`.
4. We need the earliest starting index of `p2` that is greater than or equal to `end1`. We find this using binary search on `pos2`. Let this be `start2`.
5. If both matches are found, the total matching substring in $s$ spans from `start0` to `start2 + len(p2) - 1`. Its length is `(start2 + len(p2)) - start0`. We minimize this value across all valid choices of `start0`.

### 2.2 Mathematical Invariants of KMP & Prefix Function
The core of the string matching helper is the **Prefix Function** (often denoted as `pi`).
For a string $S$ of length $L$, `pi[i]` is defined as the length of the longest proper prefix of substring `S[0...i]` which is also a suffix of this substring.

#### Mathematical transition of KMP state:
Let `j = pi[i-1]` be the match length before evaluating character `S[i]`.
1. If `S[i] == S[j]`, we can extend the match: `pi[i] = j + 1`.
2. If `S[i] != S[j]`, we must fall back to a smaller prefix that is also a suffix. The next candidate length to check is `j = pi[j-1]`. We repeat this fallback until we find a match or `j` becomes `0`.
3. This state machine guarantees that we never backtrack the pointer in the main text $s$, achieving a linear run time.

### 2.3 Edge Case: Empty Segments
If a segment like `p0` is empty (e.g., when the pattern starts with a `*`), its match can theoretically start at any index. To handle this cleanly and avoid storing all indices from `0` to `|s|`:
- An empty segment matches anywhere with zero cost.
- If `p0` is empty, `start0` can simply be assumed to be any index, but optimally we can anchor our search on the first non-empty segment.
- In our implementation, we represent empty segments explicitly by matching them at dummy positions, or we bypass search steps by setting their match lengths to 0 and aligning them directly with adjacent segment bounds.

## 3. Full Optimal Implementation (C++)

```cpp
// KMP String Matching + Binary Search Lookup
class Solution {
private:
    // Computes the pi array (prefix function) for KMP
    vector<int> computePrefixFunction(const string& pattern) {
        int m = pattern.length();
        vector<int> pi(m, 0);
        for (int i = 1; i < m; ++i) {
            int j = pi[i - 1];
            while (j > 0 && pattern[i] != pattern[j]) {
                j = pi[j - 1];
            }
            if (pattern[i] == pattern[j]) {
                j++;
            }
            pi[i] = j;
        }
        return pi;
    }

    // Finds all starting indices of pattern in text using KMP
    vector<int> kmpSearch(const string& text, const string& pattern) {
        vector<int> occurrences;
        if (pattern.empty()) {
            return occurrences;
        }
        
        vector<int> pi = computePrefixFunction(pattern);
        int n = text.length();
        int m = pattern.length();
        int j = 0; // state pointer for pattern
        
        for (int i = 0; i < n; ++i) {
            while (j > 0 && text[i] != pattern[j]) {
                j = pi[j - 1];
            }
            if (text[i] == pattern[j]) {
                j++;
            }
            if (j == m) {
                occurrences.push_back(i - m + 1);
                j = pi[j - 1]; // Reset state to find overlapping occurrences
            }
        }
        return occurrences;
    }

public:
    int shortestMatchingSubstring(string s, string p) {
        // Find the positions of the two '*' wildcards
        size_t firstStar = p.find('*');
        size_t secondStar = p.find('*', firstStar + 1);

        // Split the pattern into 3 components
        string p0 = p.substr(0, firstStar);
        string p1 = p.substr(firstStar + 1, secondStar - firstStar - 1);
        string p2 = p.substr(secondStar + 1);

        int len0 = p0.length();
        int len1 = p1.length();
        int len2 = p2.length();

        // Special Case: All segments are empty (e.g., "**")
        if (len0 == 0 && len1 == 0 && len2 == 0) {
            return 0;
        }

        // Run KMP to gather occurrences
        vector<int> pos0 = kmpSearch(s, p0);
        vector<int> pos1 = kmpSearch(s, p1);
        vector<int> pos2 = kmpSearch(s, p2);

        int minLen = 1e9;

        // Helper Lambda to evaluate a candidate window starting at a given index
        auto evalWindow = [&](int start0) {
            int end0 = start0 + len0;
            
            // Find valid start for p1
            int start1 = end0;
            if (len1 > 0) {
                auto it1 = lower_bound(pos1.begin(), pos1.end(), end0);
                if (it1 == pos1.end()) return;
                start1 = *it1;
            }
            int end1 = start1 + len1;

            // Find valid start for p2
            int start2 = end1;
            if (len2 > 0) {
                auto it2 = lower_bound(pos2.begin(), pos2.end(), end1);
                if (it2 == pos2.end()) return;
                start2 = *it2;
            }
            int end2 = start2 + len2;

            minLen = min(minLen, end2 - start0);
        };

        // If p0 is non-empty, we must anchor our search from pos0
        if (len0 > 0) {
            for (int start0 : pos0) {
                evalWindow(start0);
            }
        } 
        // If p0 is empty but p1 is non-empty, we anchor from pos1 (start0 is effectively start1)
        else if (len1 > 0) {
            for (int start1 : pos1) {
                // Since p0 is empty, start0 matches implicitly at start1
                evalWindow(start1); 
            }
        } 
        // If both p0 and p1 are empty, we anchor from pos2
        else if (len2 > 0) {
            for (int start2 : pos2) {
                evalWindow(start2);
            }
        }

        return (minLen > (int)s.length()) ? -1 : minLen;
    }
};
```


## 4. Complexity & Mathematical Analysis

### 4.1 Detailed Time Complexity Decomposition
The algorithm's total execution time is computed as:
T_total = T_split + T_kmp + T_search

- **Pattern Splitting (T_split):** Finding asterisks and extracting substrings takes O(p_len) time.
- **KMP Execution (T_kmp):** Finding occurrences for all three segments in s takes O(s_len + len0) + O(s_len + len1) + O(s_len + len2). Since len0 + len1 + len2 = p_len - 2, this step simplifies to:
  T_kmp = O(s_len + p_len)
- **Binary Search Lookup (T_search):** In the worst case, we anchor on one of the lists (e.g., pos0 of size at most s_len). For each starting index, we perform up to two binary searches using std::lower_bound on arrays of size at most s_len. Each binary search takes logarithmic time:
  T_search = O(s_len * log(s_len))

**Total Time Complexity:** O(s_len * log(s_len) + p_len). For s_len, p_len less than or equal to 100,000, this requires roughly 1.7 * 10^6 operations, running well under 25ms.

### 4.2 Detailed Space Complexity Decomposition
- **KMP State Arrays:** The pi tables for prefix functions take space proportional to the pattern segment sizes: O(len0 + len1 + len2) = O(p_len).
- **Match Lists:** The arrays pos0, pos1, and pos2 hold starting indices of matches. In the worst case (e.g., string is "aaaaa" and subpattern is "a"), a list can grow to O(s_len).

**Total Space Complexity:** O(s_len + p_len) auxiliary space.

### 4.3 Complexity Summary
| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(s_len * log(s_len) + p_len) | Extremely efficient sequence matching utilizing KMP and binary search bounds. |
| Space Complexity | O(s_len + p_len) | Used for storing match indices and prefix tables. |

## 5. Mathematical Formulations & Foundational Algorithms
The mathematical foundation of KMP relies on the amortized analysis of the state variable j. Even though there is a nested while loop within kmpSearch:
- In every step of the for loop, j increases by at most 1 (when text[i] == pattern[j]).
- The while loop fallback `j = pi[j-1]` strictly decreases j.
- Since j cannot decrease more times than it has increased, the total number of decrements across the entire execution of the algorithm is bounded by s_len.

Thus, the inner loop runs in amortized O(1) time per character, proving the O(s_len + p_len) complexity bound.

## Pitfalls & Edge Cases
- **Overlapping Wildcards / Empty Strings:** Patterns like "**" or "ab" result in empty segments. We avoid out-of-bounds queries on empty vectors by evaluating only non-empty anchoring segments.
- **Zero-Length Match Window:** If p contains only stars (e.g., "**"), the shortest valid substring matching it is empty, which correctly returns 0.
- **Upper Bound Limits:** Using 1e9 as a default infinity placeholder ensures no overflow issues when subtracting indices or comparing values.
- **Strict Inequalities on Positions:** A segment match start1 must begin at or after the index where the previous segment ended (`end0 = start0 + len0`). Using lower_bound guarantees we respect this strictly non-overlapping constraint.

---

## Key Takeaway
Splitting wildcard-based patterns around their wildcards transforms a global matching problem into localized exact string-matching subproblems. Once occurrences are mapped linearly with KMP, binary search can greedily construct the optimal global window.

**Pattern recognition:** "exactly two '*' characters, find shortest matching substring" -> KMP Segment Decomposition + Binary Search / Two Pointers Sliding Window
