# 2747. Count Zero Request Servers
## O((L + Q) log(L + Q)) — Offline Sweep with Sliding Window

---

## Problem Statement

- `n` servers. Each log `[server_id, time]` = one request.
- For each query `q`: count servers with **zero** requests in `[q-x, q]`.
- Return answers for all queries.

---

## Core Idea — Offline Processing + Two-Pointer Sliding Window

**Offline**: sort queries by value. Process them in order so we can maintain a sliding window `[q-x, q]` that only moves rightward — each log is added/removed at most once.

**Window invariant**: at any query `q`, the window contains all logs with `time ∈ [q-x, q]`. Track distinct active server IDs with a frequency map.

**Algorithm**:
1. Sort logs by time. Sort queries (keep original indices).
2. For each sorted query `(q, original_index)`:
   - **Expand right**: add all logs with `time ≤ q`
   - **Shrink left**: remove all logs with `time < q-x`
   - Answer = `n - distinct_active_servers`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] CountServers(int n, int[][] logs, int x, int[] queries) {
        Array.Sort(logs, (a, b) => a[1] - b[1]);
        int Q = queries.Length;
        int[] idx = Enumerable.Range(0, Q).OrderBy(i => queries[i]).ToArray();
        int[] res = new int[Q];

        var freq = new Dictionary<int, int>();
        int active = 0, left = 0, right = 0;

        foreach (int qi in idx) {
            int lo = queries[qi] - x, hi = queries[qi];

            // Expand right
            while (right < logs.Length && logs[right][1] <= hi) {
                int sid = logs[right][0];
                if (!freq.ContainsKey(sid) || freq[sid] == 0) active++;
                freq[sid] = freq.GetValueOrDefault(sid) + 1;
                right++;
            }

            // Shrink left
            while (left < right && logs[left][1] < lo) {
                int sid = logs[left][0];
                freq[sid]--;
                if (freq[sid] == 0) active--;
                left++;
            }

            res[qi] = n - active;
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O((L+Q) log(L+Q))** | Sort logs + queries; each log added/removed once |
| **Space** | **O(L + Q)** | Frequency map + sorted indices |

---

## Why Offline + Two Pointers

If queries were processed online (random order), for each query we'd need O(L) to scan logs → O(Q×L) total = 10¹⁰, TLE.

Sorting queries lets both pointers (`left`, `right`) move monotonically right → each log enters and exits the window exactly once → O(L) total pointer movement.

---

## Example Walkthrough

**`n=3, logs=[[1,3],[2,6],[1,5]], x=5, queries=[10,11]`**

Sorted logs by time: `[[1,3],[1,5],[2,6]]`
Sorted queries: `q=10 (idx=0), q=11 (idx=1)`

**q=10, window=[5,10]**:
- Expand: add [1,3]? 3≤10 ✓ freq={1:1}, active=1; add [1,5]? ✓ freq={1:2}; add [2,6]? ✓ freq={2:1}, active=2; right=3
- Shrink: logs[0][1]=3 < 5 → remove server 1, freq={1:1}; logs[1][1]=5 ≥ 5 → stop; left=1
- active=2, res[0] = 3-2 = **1** ✓

**q=11, window=[6,11]**:
- Expand: right=3, no more logs
- Shrink: logs[1][1]=5 < 6 → remove server 1, freq={1:0}, active=1; left=2
- active=1, res[1] = 3-1 = **2** ✓

## Pitfalls & Edge Cases

- **Frequency map needed** — multiple logs for the same server; only track distinct active servers
- **`active` decrements only when freq hits 0** — server is only "gone" when all its logs leave the window
- **`left < right` guard** — don't move left past right when window is empty
- **Original query index** — queries are processed out of order; `res[qi]` not `res[sorted_position]`

---

## Key Takeaway

"Count elements absent from sliding time window, per query" → **offline sort + two-pointer with frequency map**:

- Sort queries → both pointers move right only → O(L+Q) pointer ops
- Frequency map → track distinct active IDs
- Answer = `n - active`

Pattern: sliding window over sorted events with multiple occurrences per key → frequency map for "is key present?" in O(1).
