# 3618. Split Array by Prime Indices

## O(n√n) — Prime Check + Single Pass

---

## Problem Statement

- Given integer array `nums`, split into two arrays A and B:
  - **Array A**: elements at **prime indices** (2, 3, 5, 7, 11, ...)
  - **Array B**: elements at **non-prime indices** (0, 1, 4, 6, 8, ...)
- Return `|sum(A) - sum(B)|`.
- **Constraints**: 1 ≤ nums.length ≤ 10⁵, -10⁹ ≤ nums[i] ≤ 10⁹

---

## Core Idea — Prime Index Check + Running Sum Difference

**Key observation**: We don't need to actually build arrays A and B — just track their sum difference.

**Algorithm**:
1. Iterate through indices 0 to n-1
2. If index `i` is prime → add `nums[i]` to sumA
3. Else → add `nums[i]` to sumB
4. Return `|sumA - sumB|`

**Critical detail**: Index primality check, not value primality. `nums = [2,3,4]` → only index 2 is prime, so A = [4].

**Prime check optimization**: For index `i`, check divisibility up to √i. Indices 0 and 1 are not prime by definition.

---

## Algorithm Breakdown

### Phase 1: Prime Check Function

```
IsPrime(n):
  if n < 2: return false
  if n == 2: return true
  if n % 2 == 0: return false
  
  for i from 3 to √n step 2:
    if n % i == 0: return false
  
  return true
```

**Complexity per check**: O(√n)

### Phase 2: Single Pass Summation

```
sumA = 0, sumB = 0

for i from 0 to n-1:
  if IsPrime(i):
    sumA += nums[i]
  else:
    sumB += nums[i]

return |sumA - sumB|
```

**Complexity**: O(n) iterations × O(√n) prime check = O(n√n)

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long SplitArray(int[] nums) {
        long sumA = 0, sumB = 0;
        
        for (int i = 0; i < nums.Length; i++) {
            if (IsPrime(i)) {
                sumA += nums[i];
            } else {
                sumB += nums[i];
            }
        }
        
        return Math.Abs(sumA - sumB);
    }
    
    private bool IsPrime(int n) {
        if (n < 2) return false;
        if (n == 2) return true;
        if (n % 2 == 0) return false;
        
        for (int i = 3; i * i <= n; i += 2) {
            if (n % i == 0) return false;
        }
        
        return true;
    }
}
```

## Why This Works — Example Walkthrough

### Example 1: `nums = [2,3,4]`

**Index analysis**:
- Index 0: not prime (< 2) → B
- Index 1: not prime (< 2) → B
- Index 2: **prime** → A

**Summation**:
- sumA = nums[2] = 4
- sumB = nums[0] + nums[1] = 2 + 3 = 5

**Result**: |4 - 5| = 1 ✓

### Example 2: `nums = [-1,5,7,0]`

**Index analysis**:
- Index 0: not prime → B
- Index 1: not prime → B
- Index 2: **prime** → A
- Index 3: **prime** → A

**Summation**:
- sumA = nums[2] + nums[3] = 7 + 0 = 7
- sumB = nums[0] + nums[1] = -1 + 5 = 4

**Result**: |7 - 4| = 3 ✓

## Optimization — Sieve of Eratosthenes for Large n

If we need to process **multiple arrays** or **very large n**, precompute all primes up to n using Sieve of Eratosthenes:

```csharp
private bool[] SieveOfEratosthenes(int maxN) {
    bool[] isPrime = new bool[maxN + 1];
    Array.Fill(isPrime, true);
    isPrime[0] = isPrime[1] = false;
    
    for (int i = 2; i * i <= maxN; i++) {
        if (isPrime[i]) {
            for (int j = i * i; j <= maxN; j += i) {
                isPrime[j] = false;
            }
        }
    }
    
    return isPrime;
}

public long SplitArray(int[] nums) {
    bool[] isPrime = SieveOfEratosthenes(nums.Length - 1);
    long sumA = 0, sumB = 0;
    
    for (int i = 0; i < nums.Length; i++) {
        if (isPrime[i]) sumA += nums[i];
        else sumB += nums[i];
    }
    
    return Math.Abs(sumA - sumB);
}
```

**Complexity with Sieve**:
- **Time**: O(n log log n) for sieve + O(n) for summation = **O(n log log n)**
- **Space**: O(n) for boolean array

**When to use Sieve**: If n > 10⁴ and you process multiple arrays, or if you want guaranteed O(n log log n).

## Pitfalls & Edge Cases

- **Index vs Value**: Problem asks for **prime indices**, not prime values. `nums = [7,11,13]` → all values are prime, but only index 2 is prime → A = [13].
- **Indices 0 and 1**: By definition, 0 and 1 are **not prime** → always go to array B.
- **Index 2**: First prime index → always goes to array A if it exists.
- **Negative numbers**: `nums[i]` can be negative → use `long` for sums to avoid overflow (sum range: -10¹⁴ to 10¹⁴).
- **Empty arrays**: If no prime indices exist (n < 2), sumA = 0 → return |0 - sumB| = |sumB|.
- **All prime indices**: Rare but possible (e.g., n = 1000 has many primes) → sumB could be 0.

## Key Takeaway

"Split by prime indices" → **Single pass with prime index check**. **Algorithm**:
1. **Prime check**: O(√i) for each index i
2. **Running sums**: add to sumA if prime index, else sumB
3. **Return difference**: |sumA - sumB|

O(n√n) with O(1) space. For repeated queries or large n, use **Sieve of Eratosthenes** for O(n log log n) preprocessing.

**Critical distinction**: Prime **indices** not prime **values** — index 0 and 1 never prime, index 2 always prime.
