# 3742. Maximum Path Score in a Grid
## O(m×n×k) — 3D DP (row, col, cost)

---

## Problem Statement

- Grid `m×n` with values 0, 1, or 2. Move only right or down from `(0,0)` to `(m-1,n-1)`.
- Cell value 0: score +0, cost +0. Value 1: score +1, cost +1. Value 2: score +2, cost +1.
- Total cost must not exceed `k`. Return the maximum achievable score, or -1 if no valid path exists.

---

## Core Idea — 3D DP with Cost as Third Dimension

Standard grid path DP, extended with a cost budget.

**State**: `dp[r][c][cost]` = maximum score reachable at cell `(r, c)` having spent exactly `cost` total.

**Transition**: from `(r, c)` move right to `(r, c+1)` or down to `(r+1, c)`. For the destination cell `(nr, nc)`:
```
addCost  = grid[nr][nc] > 0 ? 1 : 0
addScore = grid[nr][nc]
dp[nr][nc][cost + addCost] = max(..., dp[r][c][cost] + addScore)
```

**Initialization**: `dp[0][0][0] = 0` (start at top-left with zero cost and zero score). All other states initialized to -1 (unreachable sentinel).

**Answer**: `max(dp[m-1][n-1][cost])` for all `cost ∈ [0..k]`. If all are -1, return -1.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxPathScore(int[][] grid, int k) {
        int m = grid.Length, n = grid[0].Length;
        int[,,] dp = new int[m, n, k+1];
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                for (int c = 0; c <= k; c++)
                    dp[i,j,c] = -1;
        dp[0,0,0] = 0;

        for (int r = 0; r < m; r++)
        for (int c = 0; c < n; c++)
        for (int cost = 0; cost <= k; cost++) {
            if (dp[r,c,cost] < 0) continue;
            int score = dp[r,c,cost];

            foreach (var (nr, nc) in new[]{(r, c+1), (r+1, c)}) {
                if (nr >= m || nc >= n) continue;
                int addCost = grid[nr][nc] > 0 ? 1 : 0;
                int newCost = cost + addCost;
                if (newCost > k) continue;
                dp[nr,nc,newCost] = Math.Max(dp[nr,nc,newCost], score + grid[nr][nc]);
            }
        }

        int res = -1;
        for (int cost = 0; cost <= k; cost++)
            if (dp[m-1,n-1,cost] >= 0)
                res = Math.Max(res, dp[m-1,n-1,cost]);
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(m × n × k)** | Three nested loops, O(1) per state |
| **Space** | **O(m × n × k)** | 3D DP table |

With `m, n ≤ 200`, `k ≤ 1000`: 200 × 200 × 1000 = 40M states — fits in memory and runs fast in C#.

## Example Walkthrough

**`grid = [[0,1],[2,0]], k = 1`**

Paths from (0,0) to (1,1):

- Right then down: (0,0)→(0,1)→(1,1). Cost: 0+1+0=1 ≤ 1. Score: 0+1+0=1.
- Down then right: (0,0)→(1,0)→(1,1). Cost: 0+1+0=1 ≤ 1. Score: 0+2+0=2.

DP trace:
- dp[0][0][0] = 0.
- Move right to (0,1): cost+1=1, score+1=1 → dp[0][1][1] = 1.
- Move down to (1,0): cost+1=1, score+2=2 → dp[1][0][1] = 2.
- From (0,1)[cost=1]: move down to (1,1): cost+0=1, score+0=1 → dp[1][1][1] = 1.
- From (1,0)[cost=1]: move right to (1,1): cost+0=1, score+0=2 → dp[1][1][1] = max(1,2) = 2.

Answer: dp[1][1][1] = **2** ✓

## Pitfalls & Edge Cases

- **`-1` sentinel**: only `-1` marks unreachable, not `0`. Score 0 is valid (all-zero path). Check `< 0` to skip unreachable states.
- **Starting cell always free**: `grid[0][0] == 0` per constraints — no cost or score gained at start.
- **`k = 0`**: only paths through all-zero cells are valid.
- **No valid path**: if all `dp[m-1][n-1][cost]` remain -1 → return -1.
- **`int` is enough**: max score ≤ 2 × (m+n) ≤ 800.

---

## Key Takeaway

"Max score path with cost budget" → **3D DP: state = (row, col, cost spent)**:

```
dp[r][c][cost] = max score at (r,c) having spent exactly cost
transition: move right/down, add cell's score and cost
answer: max dp[m-1][n-1][0..k]
```

Same pattern as "shortest path with limited fuel" or "0-1 knapsack on a grid". The cost dimension converts a budget-constrained path problem into a standard DP.
