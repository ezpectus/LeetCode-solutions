# 3380. Maximum Area Rectangle With Point Constraints I
## O(n⁴ × n) — Brute Force All Quadruples

---

## Problem Statement

- Find the maximum area axis-aligned rectangle whose 4 corners are among the given points, with no other point inside or on the border.
- Return -1 if no such rectangle exists.

---

## Core Idea — Enumerate All C(n,4) Quadruples

With `n ≤ 10`, there are at most C(10,4) = 210 quadruples. For each:

1. **Check axis-aligned**: exactly 2 distinct x-values and 2 distinct y-values among the 4 points.
2. **Check corners**: all 4 corners `(minX,minY)`, `(minX,maxY)`, `(maxX,minY)`, `(maxX,maxY)` must be present.
3. **Check no interior/border points**: none of the other `n-4` points lies inside or on the border of the rectangle.
4. **Track maximum area**.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxRectangleArea(int[][] points) {
        int n = points.Length, res = -1;

        for (int a = 0; a < n; a++)
        for (int b = a+1; b < n; b++)
        for (int c = b+1; c < n; c++)
        for (int d = c+1; d < n; d++) {
            int[] xs = {points[a][0],points[b][0],points[c][0],points[d][0]};
            int[] ys = {points[a][1],points[b][1],points[c][1],points[d][1]};
            int minX=xs.Min(), maxX=xs.Max(), minY=ys.Min(), maxY=ys.Max();

            // Must have exactly 2 distinct x and 2 distinct y values
            if (xs.Distinct().Count() != 2 || ys.Distinct().Count() != 2) continue;

            // Check no other point on or inside the rectangle
            bool valid = true;
            foreach (var p in points) {
                bool isCorner = (p[0]==minX||p[0]==maxX) && (p[1]==minY||p[1]==maxY)
                             && (p[0]==minX||p[0]==maxX) && (p[1]==minY||p[1]==maxY);
                // More precisely: corner = one of the 4 known corners
                if ((p[0]==minX&&p[1]==minY)||(p[0]==minX&&p[1]==maxY)||
                    (p[0]==maxX&&p[1]==minY)||(p[0]==maxX&&p[1]==maxY)) continue;
                if (p[0]>=minX && p[0]<=maxX && p[1]>=minY && p[1]<=maxY) {
                    valid = false; break;
                }
            }
            if (valid) res = Math.Max(res, (maxX-minX)*(maxY-minY));
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n⁵)** = **O(10⁵)** | C(n,4) quadruples × n check per quadruple |
| **Space** | **O(1)** | No extra structures needed |

With n=10: 210 × 10 = 2100 operations — trivially fast.

---

## Example Walkthrough

**`points = [[1,1],[1,3],[3,1],[3,3],[1,2],[3,2]]`**

One valid quadruple: [1,3],[1,2],[3,2],[3,3].
- xs = {1,3} (2 distinct ✓), ys = {2,3} (2 distinct ✓).
- Corners: (1,2),(1,3),(3,2),(3,3) — all present ✓.
- Check other points: [1,1]: y=1 < minY=2 → outside ✓. [3,1]: same ✓.
- Area = (3-1)×(3-2) = 2.

**`points = [[1,1],[1,3],[3,1],[3,3],[2,2]]`**

Only valid quadruple for axis-aligned rectangle: [1,1],[1,3],[3,1],[3,3].
- [2,2]: x=2 ∈ [1,3], y=2 ∈ [1,3], not a corner → **invalid**. Return -1 ✓.

---

## Pitfalls & Edge Cases

- **"2 distinct x and y" check**: ensures the 4 points actually form a rectangle, not a line or triangle.
- **Corner check vs. interior check**: the 4 corners themselves are excluded from the "interior/border" check — only other points invalidate the rectangle.
- **`n-4` other points**: even though we pick 4 from the list, we still scan all `n` points for the interior check (corners will just be skipped by the corner condition).
- **Area can be 0**: if minX==maxX or minY==maxY (degenerate rectangle). The "2 distinct" check prevents this.

---

## Key Takeaway

"Max area axis-aligned rectangle from ≤10 points with no interior points" → **brute force C(n,4) quadruples: check axis-aligned + no other point inside/on border**. O(n⁵) = O(100K) for n=10.
