# 2930. Number of Strings Which Can Be Rearranged to Contain Substring
## O(n) ÔÇö Inclusion-Exclusion on Character Count Constraints

---

## Problem Statement

- Given integer `n`, count strings of length `n` over lowercase English alphabet.
- A string is **good** if it can be rearranged to contain `"leet"` as a substring.
- Return the count modulo `10^9 + 7`.

---

## Core Idea ÔÇö Complement Counting via Inclusion-Exclusion

**What does "rearrangeable to contain `leet`" mean?**
A string can be rearranged to contain `"leet"` iff its character multiset contains at least:
- **1├Ś** `'l'`
- **2├Ś** `'e'`
- **1├Ś** `'t'`

So we need: count of length-`n` strings over 26 letters where `count('l') Ôëą 1`, `count('e') Ôëą 2`, `count('t') Ôëą 1`.

**Strategy**: complement counting.

> **Good = Total Ôłĺ Bad**
>
> Bad = strings missing at least one of the three requirements.

Define the "bad" events:
- **A** = strings with `count('l') = 0` (no `l`)
- **B** = strings with `count('e') ÔëĄ 1` (fewer than 2 `e`s)
- **C** = strings with `count('t') = 0` (no `t`)

By inclusion-exclusion:
```
|A Ôł¬ B Ôł¬ C| = |A| + |B| + |C|
             Ôłĺ |AÔłęB| Ôłĺ |AÔłęC| Ôłĺ |BÔłęC|
             + |AÔłęBÔłęC|
```

**Answer** = `26^n Ôłĺ |A Ôł¬ B Ôł¬ C|` (mod `10^9+7`)

---

## Computing Each Term

All terms follow the same pattern: restrict certain characters, count strings over remaining alphabet with possible forced placements.

| Set | Constraint | Count |
|-----|-----------|-------|
| `\|A\|` | no `l` Ôćĺ 25 choices per position | `25^n` |
| `\|C\|` | no `t` Ôćĺ 25 choices per position | `25^n` |
| `\|B\|` | `e` appears 0 or 1 times | `25^n + n├Ś25^(n-1)` |
| `\|AÔłęC\|` | no `l`, no `t` Ôćĺ 24 choices | `24^n` |
| `\|AÔłęB\|` | no `l`, `e` appears 0 or 1 times | `24^n + n├Ś24^(n-1)` |
| `\|BÔłęC\|` | no `t`, `e` appears 0 or 1 times | `24^n + n├Ś24^(n-1)` |
| `\|AÔłęBÔłęC\|` | no `l`, no `t`, `e` 0 or 1 times | `23^n + n├Ś23^(n-1)` |

**Why `|B| = 25^n + n├Ś25^(n-1)`?**

Strings where `e` appears 0 times: `25^n` (pick from 25 non-`e` letters).
Strings where `e` appears exactly 1 time: choose which of `n` positions gets `e` (`n` ways), fill rest with 25 non-`e` letters Ôćĺ `n ├Ś 25^(n-1)`.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    const long MOD = 1_000_000_007;

    public int StringCount(int n) {
        long p26 = Pow(26, n);
        long p25 = Pow(25, n);
        long p24 = Pow(24, n);
        long p23 = Pow(23, n);

        // |A|, |C|
        long A = p25;
        long C = p25;

        // |B| = 0 e's + exactly 1 e
        long B = (p25 + (long)n % MOD * Pow(25, n - 1)) % MOD;

        // |AÔłęC|
        long AC = p24;

        // |AÔłęB|, |BÔłęC|
        long AB = (p24 + (long)n % MOD * Pow(24, n - 1)) % MOD;
        long BC = (p24 + (long)n % MOD * Pow(24, n - 1)) % MOD;

        // |AÔłęBÔłęC|
        long ABC = (p23 + (long)n % MOD * Pow(23, n - 1)) % MOD;

        long bad = ((A + B + C - AB - AC - BC + ABC) % MOD + MOD) % MOD;
        return (int)((p26 - bad + MOD) % MOD);
    }

    private long Pow(long base_, int exp) {
        long result = 1;
        base_ %= MOD;
        while (exp > 0) {
            if ((exp & 1) == 1) result = result * base_ % MOD;
            base_ = base_ * base_ % MOD;
            exp >>= 1;
        }
        return result;
    }
}
```

## Why This Works ÔÇö Inclusion-Exclusion

**Why inclusion-exclusion and not direct DP?**

Direct DP tracking `(count_l, count_e, count_t)` with states `(0/Ôëą1, 0/1/Ôëą2, 0/Ôëą1)` = 2├Ś3├Ś2 = 12 states, iterated over `n` characters, also works in O(12n). But inclusion-exclusion gives a **closed-form O(log n)** solution using only modular exponentiation ÔÇö much cleaner.

**Why does `|B| = 25^n + n┬Ě25^(n-1)` count strings with fewer than 2 e's?**

- Exactly 0 e's: all `n` characters chosen from 25 non-e letters Ôćĺ `25^n`
- Exactly 1 e: `C(n,1)` ways to place the one `e`, remaining `n-1` positions from 25 non-e letters Ôćĺ `n ┬Ě 25^(n-1)`

**Why `+MOD` before final `%MOD`?**

Inclusion-exclusion involves subtraction, which can go negative under modular arithmetic. Adding `MOD` before taking `%` ensures a non-negative result.

## Pitfalls & Edge Cases

- **Negative modular arithmetic**: `(A + B + C Ôłĺ AB Ôłĺ AC Ôłĺ BC + ABC) % MOD` can be negative Ôćĺ always add `MOD` before final `%`
- **`n = 1`, `n = 2`, `n = 3`**: strings shorter than 4 can never contain `leet` Ôćĺ answer is 0; the formula handles this naturally since no string of length < 4 can have `count(l)Ôëą1, count(e)Ôëą2, count(t)Ôëą1`
- **`n % MOD` when multiplying by `n`**: `n` can be up to `10^5`, multiply by `25^(n-1)` mod `10^9+7` ÔÇö cast to `long` and apply `%MOD` to `n` before multiplying
- **Reuse `Pow` calls**: `Pow(25, n-1)` appears multiple times ÔÇö compute once and reuse

## Key Takeaway

"Count strings rearrangeable to contain `leet`" Ôćĺ **complement counting: Total Ôłĺ Bad** where Bad is computed via inclusion-exclusion on three deficit events:

```
A = no 'l'          Ôćĺ 25^n
B = fewer than 2 'e' Ôćĺ 25^n + n┬Ě25^(n-1)
C = no 't'          Ôćĺ 25^n

Bad = |A| + |B| + |C| Ôłĺ |AÔłęB| Ôłĺ |AÔłęC| Ôłĺ |BÔłęC| + |AÔłęBÔłęC|
Answer = 26^n Ôłĺ Bad  (mod 10^9+7)
```

Each intersection reduces the available alphabet by the excluded characters. O(log n) via fast exponentiation.
