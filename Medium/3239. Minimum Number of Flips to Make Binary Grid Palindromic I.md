# 3239. Minimum Number of Flips to Make Binary Grid Palindromic I
## O(m×n) — Compare Two Independent Costs

---

## Problem Statement

- Flip cells (0↔1) to make either **all rows** palindromic OR **all columns** palindromic.
- Return the minimum flips needed.

---

## Core Idea — Two Independent Costs, Take Minimum

Compute both options independently:

**Row cost**: for each row, count mismatched symmetric pairs `(grid[r][c], grid[r][n-1-c])`. Each mismatch requires 1 flip.

**Col cost**: for each column, count mismatched symmetric pairs `(grid[r][c], grid[m-1-r][c])`.

Answer = `min(row_cost, col_cost)`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinFlips(int[][] grid) {
        int m = grid.Length, n = grid[0].Length;

        int rowCost = 0;
        for (int r = 0; r < m; r++)
            for (int c = 0; c < n/2; c++)
                if (grid[r][c] != grid[r][n-1-c]) rowCost++;

        int colCost = 0;
        for (int c = 0; c < n; c++)
            for (int r = 0; r < m/2; r++)
                if (grid[r][c] != grid[m-1-r][c]) colCost++;

        return Math.Min(rowCost, colCost);
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(m×n)** |
| **Space** | **O(1)** |

---

## Key Takeaway

Row palindrome cost and column palindrome cost are independent — just compute both, return the minimum.
