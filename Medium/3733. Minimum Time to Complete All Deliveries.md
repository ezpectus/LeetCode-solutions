# 3733. Minimum Time to Complete All Deliveries

## O(log T) — Binary Search on Time

---

## Problem Statement

- Two drones with delivery counts `d = [d1, d2]` and recharge intervals `r = [r1, r2]`.
- Each delivery takes 1 hour. Only one drone delivers per hour.
- Drone `i` must recharge at hours that are multiples of `ri` (cannot deliver during recharge).
- Return minimum total time to complete all deliveries.
- **Constraints**: 1 ≤ di ≤ 10⁹, 2 ≤ ri ≤ 3×10⁴

---

## Core Idea — Binary Search + Greedy Assignment

**Key observation**: If we can complete in time T, we can also complete in T+1 → monotonic → binary search.

**Critical insight**: At any hour, only one drone can deliver (mutual exclusion). We need to check if we can **schedule** all deliveries within T hours.

**Scheduling constraint**:
- Drone 1 available at hours NOT divisible by r1
- Drone 2 available at hours NOT divisible by r2
- At hours when both available, only one can deliver

**Greedy strategy**: Count maximum deliveries possible in T hours by each drone, then check if we can fit d1+d2 total.

---

## Algorithm Breakdown

### Binary Search on Time

```
left = 1
right = (d1 + d2) * max(r1, r2) // Upper bound

while left < right:
  mid = (left + right) / 2
  if CanSchedule(mid, d1, d2, r1, r2):
    right = mid
  else:
    left = mid + 1

return left
```

### Check Function — Can Schedule All Deliveries?

**For time T, count available slots for each drone**:
```
avail1 = T - floor(T / r1) // Hours drone 1 NOT recharging
avail2 = T - floor(T / r2) // Hours drone 2 NOT recharging
```

**Key insight**: We need to assign deliveries such that:
1. Drone 1 completes d1 deliveries
2. Drone 2 completes d2 deliveries
3. No two deliveries at same hour

**Greedy assignment**:
- Reserve min(d1, avail1) hours for drone 1's deliveries
- Reserve min(d2, avail2) hours for drone 2's deliveries
- Check if assignments can fit without collision

**Hours when both available**: hours not divisible by r1 AND not divisible by r2
```
bothAvail = T - floor(T/r1) - floor(T/r2) + floor(T/lcm(r1,r2))
```

**Check logic**:
- If d1 > avail1 OR d2 > avail2: impossible (drone doesn't have enough hours)
- Otherwise, we need d1+d2 total delivery hours
- Maximum delivery hours = T - (hours both recharge)
- bothRecharge = floor(T / lcm(r1, r2))
- Check: T - bothRecharge >= d1 + d2

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinimumTime(int[] d, int[] r) {
        long d1 = d[0], d2 = d[1];
        long r1 = r[0], r2 = r[1];
        
        // Better upper bound: worst case both drones work sequentially
        // with maximum recharge overhead
        long left = Math.Max(d1, d2);
        long right = d1 + d2 + (d1 / (r1 - 1) + 1) * r1 + (d2 / (r2 - 1) + 1) * r2;
        
        while (left < right) {
            long mid = left + (right - left) / 2;
            
            if (CanSchedule(mid, d1, d2, r1, r2)) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }
        
        return left;
    }
    
    private bool CanSchedule(long T, long d1, long d2, long r1, long r2) {
        // Available hours for each drone
        long avail1 = T - T / r1;
        long avail2 = T - T / r2;
        
        // Check if each drone has enough hours for its deliveries
        if (avail1 < d1 || avail2 < d2) {
            return false;
        }
        
        // Count hours when both drones recharge simultaneously
        long lcm = LCM(r1, r2);
        long bothRecharge = T / lcm;
        
        // Maximum deliveries = hours with at least one drone available
        long maxDeliveries = T - bothRecharge;
        
        return maxDeliveries >= d1 + d2;
    }
    
    private long GCD(long a, long b) {
        while (b != 0) {
            long temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }
    
    private long LCM(long a, long b) {
        return a / GCD(a, b) * b;
    }
}
```

## Example Walkthrough (Fixed)

### Example 1: `d = [3,1], r = [2,3]`

**T = 5**:
- avail1 = 5 - floor(5/2) = 5 - 2 = 3 ≥ 3 ✓
- avail2 = 5 - floor(5/3) = 5 - 1 = 4 ≥ 1 ✓
- LCM(2,3) = 6
- bothRecharge = floor(5/6) = 0
- maxDeliveries = 5 - 0 = 5 ≥ 4 ✓
- Result: CAN schedule ✓

**T = 4**:
- avail1 = 4 - 2 = 2 < 3 ✗
- Result: CANNOT schedule

**Answer**: 5 ✓

Let me trace through the actual schedule:
- Hour 1: drone 1 delivers (both available, assign to drone 1) → d1=2 left
- Hour 2: both recharge
- Hour 3: drone 2 delivers (drone 1 available, drone 2 recharges... wait)

Hmm, hour 3 is NOT divisible by 3, so drone 2 should be available. Let me recalculate:

**Hour-by-hour for T=5**:
- Hour 1: both available → drone 1 delivers
- Hour 2: drone 1 recharges (2%2=0), drone 2 available → drone 2 delivers
- Hour 3: drone 1 available, drone 2 recharges (3%3=0) → drone 1 delivers
- Hour 4: drone 1 recharges (4%2=0), drone 2 available → but drone 2 already finished!
- Hour 5: both available → drone 1 delivers

Total: drone 1 did 3, drone 2 did 1 ✓

So the answer is correct: 5.

---

## Key Takeaway

"Minimum time with periodic recharge constraints" → **Binary search on time**. **Check function**:
1. Each drone has enough available hours (T - T/ri ≥ di)
2. Total delivery slots (excluding both-recharge hours) ≥ d1 + d2

O(log T × log r) time. **Critical insight**: Use LCM to find hours when both drones recharge (wasted hours).
