# 3121. Count the Number of Special Characters II
## O(n) — Last Lowercase / First Uppercase Index Tracking

---

## Problem Statement

- Given string `word` of lowercase and uppercase letters.
- A letter `c` is **special** if:
  1. It appears in both lowercase and uppercase in `word`.
  2. Every lowercase occurrence of `c` comes **before** the first uppercase occurrence of `c`.
- Return the count of special letters.

---

## Core Idea — Track Last Lowercase and First Uppercase per Letter

**Key insight**: for each letter `c` (a–z), the condition simplifies to:

> `lastLower[c] < firstUpper[c]`

i.e. the last time `c` appears lowercase must come before the first time `C` appears uppercase.

If this holds (and both exist), every lowercase `c` is automatically before every uppercase `C`, since the last lowercase is before the first uppercase.

**Algorithm**:
1. Scan `word` left to right, recording:
   - `lastLower[c]` = last index where `c` appears in lowercase (or `-1` if never).
   - `firstUpper[c]` = first index where `c` appears in uppercase (or `n` if never).
2. For each letter `c`: if `lastLower[c] != -1` and `firstUpper[c] != n` and `lastLower[c] < firstUpper[c]` → count it.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int NumberOfSpecialChars(string word) {
        int n = word.Length;
        int[] lastLower  = new int[26];
        int[] firstUpper = new int[26];
        Array.Fill(lastLower,  -1);
        Array.Fill(firstUpper, n);  // sentinel: "never seen"

        for (int i = 0; i < n; i++) {
            char ch = word[i];
            if (char.IsLower(ch))
                lastLower[ch - 'a'] = i;             // keep updating → last occurrence
            else
                firstUpper[ch - 'A'] = Math.Min(firstUpper[ch - 'A'], i); // keep min → first occurrence
        }

        int count = 0;
        for (int c = 0; c < 26; c++)
            if (lastLower[c] != -1 && firstUpper[c] != n && lastLower[c] < firstUpper[c])
                count++;

        return count;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Single pass through word + O(26) check |
| **Space** | **O(1)** | Two fixed-size arrays of length 26 |

---

## Why This Works

**Why track `lastLower` and `firstUpper`?**

The condition "every lowercase `c` before the first uppercase `C`" is equivalent to: the latest lowercase `c` comes before the earliest uppercase `C`. If the last lowercase is before the first uppercase, all lowercases are trivially before all uppercases.

**Why `lastLower[c] < firstUpper[c]` (strict)?**

The same index can't hold both a lowercase and an uppercase character, so `<` and `≤` are equivalent here — but strict `<` is correct since a character is either lowercase or uppercase, never both.

**Why sentinels `-1` and `n`?**

- `lastLower[c] = -1` means no lowercase `c` exists → letter can't be special.
- `firstUpper[c] = n` means no uppercase `C` exists → letter can't be special.
- The check `lastLower[c] < firstUpper[c]` naturally excludes these cases since `-1 < n` is always true — so we also need the explicit existence checks.

## Example Walkthrough

**`word = "aaAbcBC"`**

| i | char | action |
|---|------|--------|
| 0 | a | lastLower[a]=0 |
| 1 | a | lastLower[a]=1 |
| 2 | A | firstUpper[a]=2 |
| 3 | b | lastLower[b]=3 |
| 4 | c | lastLower[c]=4 |
| 5 | B | firstUpper[b]=5 |
| 6 | C | firstUpper[c]=6 |

Results:
- `a`: lastLower=1, firstUpper=2 → 1 < 2 ✓ **special**
- `b`: lastLower=3, firstUpper=5 → 3 < 5 ✓ **special**
- `c`: lastLower=4, firstUpper=6 → 4 < 6 ✓ **special**

Answer: **3** ✓

**`word = "AbBCab"`**

| i | char | action |
|---|------|--------|
| 0 | A | firstUpper[a]=0 |
| 1 | b | lastLower[b]=1 |
| 2 | B | firstUpper[b]=2 |
| 3 | C | firstUpper[c]=3 |
| 4 | a | lastLower[a]=4 |
| 5 | b | lastLower[b]=5 |

Results:
- `a`: lastLower=4, firstUpper=0 → 4 < 0? ✗
- `b`: lastLower=5, firstUpper=2 → 5 < 2? ✗
- `c`: lastLower=-1 → not present ✗

Answer: **0** ✓

---

**`word = "abc"`**

No uppercase letters → all `firstUpper[c] = n = 3` → no letter qualifies.

Answer: **0** ✓

## Pitfalls & Edge Cases

- **Don't confuse Part I and Part II**: Part I (3120) only requires both cases exist. Part II adds the ordering constraint — the key difference is needing `lastLower` (not just any occurrence) vs `firstUpper`
- **`lastLower` updates on every lowercase**: always overwrite — we want the **last** occurrence
- **`firstUpper` takes min**: only update when smaller — we want the **first** occurrence
- **Both conditions must be checked**: if only lowercase exists (no uppercase), `firstUpper = n` and `lastLower < n` is true but not special — the `firstUpper != n` guard is essential

---

## Key Takeaway

"Every lowercase `c` before first uppercase `C`" ↔ `lastLower[c] < firstUpper[c]`:

```
for each char in word:
    if lower: lastLower[c] = i           // keep updating
    if upper: firstUpper[c] = min(...,i) // keep minimum

count c where lastLower[c] != -1
           && firstUpper[c] != n
           && lastLower[c] < firstUpper[c]
```

Single scan + O(26) check → O(n) time, O(1) space.
