# 3964. Minimum Lights to Illuminate a Road
## O(n) â€” Difference Array Coverage + Rightmost-Extension Greedy

---

## Problem Statement

Array `lights` of length `n`, positions `0..n-1` on a road.
- `lights[i] = v > 0`: existing bulb at `i` illuminates `[max(0, i-v), min(n-1, i+v)]`.
- `lights[i] = 0`: no bulb at position `i`.
- Additional bulbs you install at position `j` illuminate `[max(0, j-1), min(n-1, j+1)]` (fixed radius 1).

Return the **minimum number of additional bulbs** to make every position visible.

---

## Core Insight #1 â€” Separate "What's Already Lit" from "What Needs Covering"

Existing bulbs have arbitrary radii up to `n`. Additional bulbs always have radius 1. These two types have nothing in common structurally, so solve them in two clean phases:

**Phase 1:** Compute which positions are already illuminated by existing bulbs.
**Phase 2:** Greedily cover the remaining dark positions using minimum additional radius-1 bulbs.

No interleaving needed â€” the existing coverage is a read-only constraint for Phase 2.

---

## Core Insight #2 â€” Difference Array Makes Phase 1 O(n)

Naive approach: for each existing bulb, mark its range â€” O(n) bulbs Ã— O(n) range = O(nÂ²). Instead use a **difference array**:

```
diff[l]++, diff[r+1]--   for each bulb covering [l, r]
prefix-sum diff â†’ coverage count per position
lit[i] = (coverage[i] > 0)
```

Total: O(n). Max sum = 1.5 Ã— 10Â¹Â² fits in `long long`; actual coverage counts are â‰¤ n, so `int` is fine.

## Core Insight #3 â€” Greedy for Phase 2: Always Place as Far Right as Possible

When scanning left to right and hitting the first uncovered dark position `i`, we **must** place a bulb that covers `i`. Of the three candidate positions:

| Placement `j` | Covers | Rightmost covered | Next uncovered |
|---|---|---|---|
| `i - 1` | `[i-2, i]` | `i` | `i + 1` |
| `i` | `[i-1, i+1]` | `i+1` | `i + 2` |
| `i + 1` | `[i, i+2]` | `i+2` | `i + 3` |

**Placing at `i+1` is always optimal**: it covers `i` (satisfying the must-cover constraint) and extends the furthest to the right, minimizing how many additional bulbs will be needed for remaining positions. Standard exchange argument: any solution that places a bulb at `i` or `i-1` can be shifted right to `i+1` without uncovering any previously covered position and only gaining coverage to the right.

**Algorithm:**
```
i = 0, ans = 0
while i < n:
    if lit[i]:
        i++
    else:
        place at j = min(i+1, n-1)   // rightmost valid position
        ans++
        i = j + 2                     // jump past coverage end (j+1)
```

The jump `i = j + 2` skips positions `j-1` and `j` and `j+1` (all covered by the new bulb), landing at the first potentially uncovered position.

## Full Implementation (C++)

```cpp
class Solution {
public:
    int minLights(vector<int>& lights) {
        int n = lights.size();

        // Phase 1: existing bulb coverage via difference array
        vector<int> diff(n + 1, 0);
        for (int i = 0; i < n; i++) {
            if (lights[i] > 0) {
                int l = max(0, i - lights[i]);
                int r = min(n - 1, i + lights[i]);
                diff[l]++;
                diff[r + 1]--;
            }
        }

        vector<bool> lit(n, false);
        int cur = 0;
        for (int i = 0; i < n; i++) {
            cur += diff[i];
            lit[i] = (cur > 0);
        }

        // Phase 2: greedy scan â€” cover dark positions with minimum radius-1 bulbs
        int ans = 0;
        int i = 0;
        while (i < n) {
            if (lit[i]) {
                i++;
            } else {
                // Must cover position i. Place bulb as far right as possible.
                int j = min(i + 1, n - 1); // placement position
                ans++;
                i = j + 2; // jump past coverage end; loop exits if i >= n
            }
        }

        return ans;
    }
};
```

## Why Greedy = ceil(gap\_length / 3) Per Dark Gap

Each maximal contiguous dark interval `[l, r]` (unlit by existing bulbs) is solved independently by the greedy. Within a gap of length `len = r - l + 1`:
- Bulb at `l+1`: covers `[l, l+2]`. Next uncovered: `l+3`.
- Bulb at `l+4`: covers `[l+3, l+5]`. Next uncovered: `l+6`.
- ...

After `k` bulbs, the gap is covered through `l + 3k - 1`. Done when `l + 3k - 1 â‰¥ r`, i.e.:
```
k = âŒˆ(r - l + 1) / 3âŒ‰ = âŒˆlen / 3âŒ‰
```

This is also a **tight lower bound**: each additional bulb covers at most 3 positions in the dark gap, so `âŒˆlen/3âŒ‰` bulbs are necessary. The greedy achieves it.

| Gap length | Bulbs needed | Example placement |
|---|---|---|
| 1 | 1 | `j = l+1` covers `[l, l+2]` |
| 2 | 1 | `j = l+1` covers `[l, l+2]` |
| 3 | 1 | `j = l+1` covers `[l, l+2]` |
| 4 | 2 | `l+1` then `l+4` |
| 5 | 2 | `l+1` then `l+4` |
| 6 | 2 | `l+1` then `l+4` |
| 7 | 3 | `l+1`, `l+4`, `l+7` |

## Pitfalls & Edge Cases

- **Boundary placement:** When `i = n-1` (last position is dark), `min(i+1, n-1) = n-1 = i`. The bulb at `n-1` covers `[n-2, n-1]`. Jump to `n`. Done. âœ“

- **Very large existing bulb radii:** `lights[i]` can be up to `n`. The difference array handles any radius trivially â€” just clamp `l` and `r` to `[0, n-1]`. No special case needed.

- **Entirely covered road:** All positions lit by existing bulbs. The greedy scan never enters the `else` branch. Returns 0. âœ“

- **Single position (`n=1`):** If `lights[0] > 0`, already lit, return 0. If `lights[0] = 0`, dark: place at `min(1, 0) = 0`, covers `[0, 1â†’0]`. Return 1. âœ“

- **Do NOT use the difference array value itself as a "how many bulbs here" counter for placement.** The diff array only tells us which positions are already lit â€” it says nothing about where to place additional bulbs.

## Key Takeaway

This is a **"fixed-radius interval cover"** problem: given a set of uncovered points on a line, cover them all with minimum intervals of fixed length. The optimal strategy is always to place each interval starting at the leftmost uncovered point â€” the "greedy from the left" pattern. Pairing it with a difference array for O(n) pre-computation of existing coverage makes the whole solution linear.

**Pattern recognition:** "Minimum number of fixed-radius sensors/lights/intervals to cover all uncovered points on a 1D line" â†’ sort/scan left to right, place each coverage unit at `first_uncovered + radius` to extend as far right as possible. Count = `âŒˆgap_length / (2Â·radius + 1)âŒ‰` per dark gap.
