# 3767. Maximize Points After Choosing K Tasks
## O(n log n) — Greedy: Maximize Freely, Then Repair with Minimum Penalty

---

## Problem Statement

- You have n tasks, each completable via technique 1 (earn `technique1[i]`) or technique 2 (earn `technique2[i]`).
- At least k tasks must use technique 1.
- Return the maximum total points achievable.

---

## Core Idea — Start Optimal, Then Satisfy the Constraint Cheaply

**Two observations:**
1. For any task where `technique1[i] ≥ technique2[i]`, technique 1 is at least as good — use it always.
2. For tasks where `technique2[i] > technique1[i]`, technique 2 is strictly better — but we might be forced to use technique 1 for some to meet the k requirement.

**Algorithm:**
1. **Baseline:** assign every task its better technique → `total = Σ max(tech1[i], tech2[i])`
2. **Count free k:** `naturalK` = tasks where `tech1[i] ≥ tech2[i]` (already using technique 1)
3. **If `naturalK ≥ k`:** done, return `total` (constraint already satisfied)
4. **If `naturalK < k`:** must switch `k - naturalK` more tasks from technique 2 to technique 1. Each switch costs `penalty[i] = tech2[i] - tech1[i]`. **Sort penalties ascending and take the cheapest ones** to minimize total loss.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaxPoints(int[] technique1, int[] technique2, int k) {
        int n = technique1.Length;
        long total = 0;
        int naturalK = 0;
        var penalties = new List<int>();

        for (int i = 0; i < n; i++) {
            total += Math.Max(technique1[i], technique2[i]);

            if (technique1[i] >= technique2[i]) {
                naturalK++;
            } else {
                penalties.Add(technique2[i] - technique1[i]); // cost to switch this task to tech1
            }
        }

        if (naturalK >= k) return total;

        // Sort penalties ascending — take the cheapest switches first
        penalties.Sort();

        int need = k - naturalK;
        for (int i = 0; i < need; i++) {
            total -= penalties[i];
        }

        return total;
    }
}
```

| Space Complexity | O(n) | Penalties list |

## Why This Works — Example Walkthrough

**Example 1:** `technique1 = [5,2,10], technique2 = [10,3,8], k = 2`

| i | tech1 | tech2 | Best      | naturalK? | Penalty |
|---|-------|-------|-----------|-----------|---------|
| 0 | 5     | 10    | 10 (tech2)| no        | 10−5=5  |
| 1 | 2     | 3     | 3  (tech2)| no        | 3−2=1   |
| 2 | 10    | 8     | 10 (tech1)| ✓ (=1)    | —       |

total = 10+3+10 = 23, naturalK = 1, need = 2−1 = 1

penalties sorted: [1, 5] → subtract cheapest 1: total = 23 − 1 = **22** ✓

Switching task 1 to technique 1: now use [tech2[0], tech1[1], tech1[2]] = [10, 2, 10] = 22.

---

**Example 2:** `technique1 = [10,20,30], technique2 = [5,15,25], k = 2`

| i | tech1 | tech2 | Best | naturalK? |
|---|-------|-------|------|-----------|
| 0 | 10    | 5     | 10   | ✓         |
| 1 | 20    | 15    | 20   | ✓         |
| 2 | 30    | 25    | 30   | ✓         |

total = 60, naturalK = 3 ≥ k = 2 → return **60** ✓

---

**Example 3:** `technique1 = [1,2,3], technique2 = [4,5,6], k = 0`

naturalK = 0 ≥ k = 0 → return total = 4+5+6 = **15** ✓

## Pitfalls & Edge Cases

- **Overflow** — n ≤ 10⁵, technique values ≤ 10⁵, so max total = 10¹⁰. Must use `long`. ✓

- **k = 0** — naturalK ≥ 0 always, so we always return the unconstrained maximum immediately. ✓

- **k = n** — All tasks must use technique 1. If naturalK = n, return sum of tech1. If naturalK < n, we sort all penalties and subtract all of them (all tasks switch to technique 1).

- **Ties (tech1[i] == tech2[i])** — Counted in naturalK (treated as "free" technique 1 usage). Penalty would be 0 anyway, so this is optimal.

- **All penalties needed** — If `need == penalties.Count`, we subtract all penalties, effectively using technique 1 for every task.

---

## Key Takeaway

Classic **greedy "maximize then fix" pattern**:
1. Solve the unconstrained version (take best per task)
2. Measure the cost of satisfying the constraint (switch k − naturalK tasks to tech1)
3. Greedily apply the cheapest repairs first (sort penalties, take minimum)

**Pattern recognition:** "Minimize cost to satisfy a minimum-count constraint after greedy assignment" → sort the "switch costs" and take the cheapest. This pattern appears in problems like assigning workers, buying items with a minimum-quantity rule, etc.
