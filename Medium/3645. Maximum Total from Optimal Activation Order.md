# 3645. Maximum Total from Optimal Activation Order

## O(n log n) — Greedy by Limit Buckets 

---

## Problem Statement

- You are given two arrays `value` and `limit`, both of length `n`.
- All elements start **inactive**.
- You may activate elements in **any order**.
- **Activation rules**:
  - To activate element `i`, the current active count must satisfy  
    ```
    activeCount < limit[i]
    ```
  - Activating `i` adds `value[i]` to the total.
  - After activation, if the active count becomes `x`,  
    all elements `j` with  
    ```
    limit[j] <= x
    ```  
    become **permanently inactive**.
- Return the **maximum total value** achievable.
- **Constraints**:  
  `1 ≤ n ≤ 10⁵`,  
  `1 ≤ value[i] ≤ 10⁵`,  
  `1 ≤ limit[i] ≤ n`.

---

## Core Idea — Position Constraint Reformulation

Despite the complicated activation/deactivation rules,  
the true mathematical constraint is surprisingly simple:

### ✔ An element with limit `L` can only appear among the first `L` activations.

Reason:

- To activate element `i`, we need  
  `activeCount_before < limit[i]`
- `activeCount_before` equals the number of previously activated elements.
- Therefore, if an element is placed at activation position `k`,  
  it must satisfy:```k ≤ limit[i]```
  
This transforms the problem into:

> **Choose a subset of elements such that each element with limit L  
> can be placed within the first L positions, maximizing total value.**

Deactivation does **not** restrict future activations —  
it only removes elements that were already impossible to activate later.

---

## Greedy Strategy — Bucket by Limit

1. **Group elements by their limit**  
```bucket[L] = all values[i] with limit[i] = L```

2. **Sort each bucket in descending order of value**

3. **From bucket L, you may take at most L elements**  
(because they can occupy positions 1..L)

4. Sum the top `L` values from each bucket.

This greedy is optimal because:

- Elements with small limits are more restrictive.
- Within each limit group, taking the highest values maximizes contribution.
- Groups are independent under the `position ≤ limit` rule.

---

## Algorithm Breakdown

### Phase 1 — Build Buckets
```
For each i:
bucket[limit[i]].append(value[i])
```

### Phase 2 — Sort Each Bucket

```
For each L:
sort bucket[L] in descending order
```

### Phase 3 — Take Best L Values
```
For each L:
take = min(L, bucket[L].size)
add top 'take' values to answer
```

## Full Implementation (C#)

```csharp
using System;
using System.Collections.Generic;

public class Solution {
    public long MaxTotal(int[] value, int[] limit) {
        int n = value.Length;
        if (n == 0) return 0;

        // Map: limit L -> list of values with that limit
        var buckets = new Dictionary<int, List<int>>();

        for (int i = 0; i < n; i++) {
            int L = limit[i];
            if (L <= 0) continue;          // never activatable
            if (L > n + 1) L = n + 1;      // optional clamp
            if (!buckets.TryGetValue(L, out var list)) {
                list = new List<int>();
                buckets[L] = list;
            }
            list.Add(value[i]);
        }

        long ans = 0;
        foreach (var kv in buckets) {
            int L = kv.Key;
            var list = kv.Value;
            list.Sort((a, b) => b.CompareTo(a)); // descending
            int take = Math.Min(L, list.Count);
            for (int i = 0; i < take; i++) ans += list[i];
        }

        return ans;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|------------|-----------|-----------|
| **Time**   | **O(n log n)** | Sorting inside buckets |
| **Space**  | **O(n)**       | Buckets store all values |

## Example Walkthrough

### Input
```
value = [4,1,5,2]
limit = [3,3,2,3]
```

### Buckets
```
limit = 2 → [5]
limit = 3 → [4,1,2]
```

### Take:
- From limit = 2 → take 1 element → **5**
- From limit = 3 → take 3 elements → **4 + 2 + 1**

### Total = **12**

## Key Takeaway

Even though the problem describes complex activation/deactivation behavior,  
the true constraint is simply:

> **An element with limit L can appear only within the first L activations.**

Therefore, grouping by limit and taking the top L values is optimal.
