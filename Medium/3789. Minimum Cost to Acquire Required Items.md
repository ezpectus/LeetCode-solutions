# 3789. Minimum Cost to Acquire Required Items
## O(1) — Three-Point Evaluation on a Piecewise Linear Function

---

## Problem Statement

- Three item types: type 1 (costs `cost1`, contributes to need1), type 2 (costs `cost2`, contributes to need2), type 3 (costs `costBoth`, contributes to both).
- Satisfy `need1` and `need2` at minimum total cost.

---

## Core Idea — Fix the Number of "Both" Items, Minimize Analytically

Let `b` = number of type-3 (both) items purchased. Then:
- We need `max(0, need1 − b)` additional type-1 items
- We need `max(0, need2 − b)` additional type-2 items

Total cost as a function of `b`:
```
f(b) = b × costBoth + max(0, need1−b) × cost1 + max(0, need2−b) × cost2
```

**f(b) is piecewise linear** with breakpoints at `b = need1` and `b = need2`. On each linear segment, f is monotone, so the **minimum is always at one of the three breakpoints**: `b = 0`, `b = need1`, or `b = need2`.

---

## Piecewise Linear Analysis

Assume WLOG need1 ≤ need2. Three segments:

| Region | f(b) simplified | Derivative | Optimal endpoint |
|--------|-----------------|------------|-----------------|
| 0 ≤ b ≤ need1 | b(costBoth − cost1 − cost2) + C | costBoth − cost1 − cost2 | b=0 or b=need1 |
| need1 < b ≤ need2 | b(costBoth − cost2) + C | costBoth − cost2 | b=need1 or b=need2 |
| b > need2 | b × costBoth (wasteful) | +costBoth > 0 | never optimal past need2 |

The minimum of a linear function on a segment is always at one endpoint → **evaluate f at 0, need1, and need2, return the minimum**.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinimumCost(int cost1, int cost2, int costBoth, int need1, int need2) {
        long res = long.MaxValue;

        foreach (long b in new long[] { 0, need1, need2 }) {
            long cost = b * costBoth
                      + Math.Max(0L, need1 - b) * cost1
                      + Math.Max(0L, need2 - b) * cost2;
            res = Math.Min(res, cost);
        }

        return res;
    }
}
```

## Why This Works — Example Walkthrough

**Example 1:** `cost1=3, cost2=2, costBoth=1, need1=3, need2=2`

| b | b × costBoth | max(0, 3−b) × 3 | max(0, 2−b) × 2 | total |
|---|-------------|-----------------|-----------------|-------|
| 0 | 0           | 9               | 4               | **13** |
| 3 | 3           | 0               | 0               | **3** ← min |
| 2 | 2           | 3               | 0               | **5** |

**Output:** 3 ✓ (buy 3 "both" items, covers 3 ≥ need1 and 3 ≥ need2)

---

**Example 2:** `cost1=5, cost2=4, costBoth=15, need1=2, need2=3`

| b | b × 15 | max(0,2−b) × 5 | max(0,3−b) × 4 | total |
|---|--------|----------------|----------------|-------|
| 0 | 0      | 10             | 12             | **22** ← min |
| 2 | 30     | 0              | 4              | **34** |
| 3 | 45     | 0              | 0              | **45** |

**Output:** 22 ✓ (costBoth=15 is too expensive, buy pure type 1 and type 2)

---

**Example 3:** `need1=0, need2=0`

All three candidates evaluate to 0.

**Output:** 0 ✓

---

## Pitfalls & Edge Cases

- **Overflow** — b up to 10⁹, costBoth up to 10⁶ → b × costBoth up to 10¹⁵. Use `long`. ✓

- **need1 = 0 or need2 = 0** — Max handles naturally; max(0, 0−b) = 0 for any b ≥ 0. ✓

- **costBoth < cost1 + cost2** — Buying "both" items is a deal; optimal b = max(need1, need2). Evaluated at b=need2 (or b=need1 if need1 > need2). ✓

- **costBoth > cost1 + cost2** — Never worth buying "both" items; optimal b = 0. Evaluated and selected automatically. ✓

---

## Key Takeaway

When the cost function is **piecewise linear**, the global minimum is always at a breakpoint. Identifying the breakpoints (`b = 0`, `b = need1`, `b = need2`) reduces the problem from an optimization search to **three evaluations**.

**Pattern recognition:** "Choose how many shared items to buy" → parametrize on shared count, recognize the resulting function is piecewise linear, evaluate at all breakpoints.
