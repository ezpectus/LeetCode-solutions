# 2615. Sum of Distances
## O(n) — Group by Value + Prefix Sum per Group

---

## Problem Statement

- You are given a **0-indexed** integer array `nums` (1 ≤ n ≤ 10⁵, 0 ≤ nums[i] ≤ 10⁹).
- For each index `i`, compute `arr[i] = Σ |i - j|` for all `j ≠ i` where `nums[j] == nums[i]`.
- Return `arr`.

---

## Core Idea — Group Indices + Prefix Sum

**Key insight**: Only indices with the **same value** contribute to each other's sum. Group all indices by value, then for each group compute the sum of absolute differences efficiently using a **prefix sum**.

**For a sorted group of indices `[g0, g1, ..., gm-1]`**, the sum of `|gi - gj|` for a fixed `gi` splits into two parts:

```
arr[gi] = (left cost) + (right cost)
         = (gi × i  - prefSum[i])          ← i elements to the left, all smaller
         + (prefSum[m] - prefSum[i+1]) - gi × (m-1-i)   ← (m-1-i) elements to the right
```

where `prefSum[k] = g0 + g1 + ... + g(k-1)`.

**Why this works**:
- For each `gj < gi` (left side): `|gi - gj| = gi - gj` → summing gives `gi × (count left) - Σgj`
- For each `gj > gi` (right side): `|gi - gj| = gj - gi` → summing gives `Σgj - gi × (count right)`
- Prefix sum gives `Σgj` for any range in O(1)

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long[] Distance(int[] nums) {
        int n = nums.Length;
        long[] res = new long[n];
        var groups = new Dictionary<int, List<int>>();

        // Group indices by value
        for (int i = 0; i < n; i++) {
            if (!groups.ContainsKey(nums[i])) groups[nums[i]] = new List<int>();
            groups[nums[i]].Add(i); // indices added in sorted order since we iterate left→right
        }

        foreach (var idxs in groups.Values) {
            int m = idxs.Count;
            if (m == 1) continue; // no other same-value index

            // Build prefix sum of indices in this group
            long[] pref = new long[m + 1];
            for (int i = 0; i < m; i++)
                pref[i + 1] = pref[i] + idxs[i];

            for (int i = 0; i < m; i++) {
                long idx = idxs[i];
                // Left: i elements with smaller index
                long left  = idx * i - pref[i];
                // Right: (m-1-i) elements with larger index
                long right = (pref[m] - pref[i + 1]) - idx * (m - 1 - i);
                res[idx] = left + right;
            }
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Each index visited once for grouping + once for prefix sum pass |
| **Space Complexity** | **O(n)** | Groups dictionary + prefix arrays sum to O(n) total |

**Optimal** — must read every element at least once.

## Formula Breakdown

For group `[g0, g1, ..., gm-1]` sorted ascending, at position `i` (value `gi`):

```
left cost  = Σ_{j<i} (gi - gj) = gi × i  -  pref[i]
right cost = Σ_{j>i} (gj - gi) = (pref[m] - pref[i+1])  -  gi × (m-1-i)

arr[gi]    = left + right
```

This is the same pattern as "sum of distances from a point to all other points on a number line" — a classic prefix sum trick.

## Pitfalls & Edge Cases

- **Singleton groups** — skip; `arr[i] = 0` by default ✓
- **Long overflow** — indices up to 10⁵, group size up to 10⁵ → max sum ≈ 10¹⁰; use `long` for `pref` and `res`
- **Indices already sorted** — since we iterate `nums` left→right, `groups[x].Add(i)` naturally inserts in sorted order; no extra sort needed
- **All same value** — one giant group of size n; prefix sum handles it in O(n) ✓
- **`pref[i]` not `pref[i+1]` for left** — `pref[i]` = sum of first `i` elements = `g0+...+g(i-1)` (exclusive of `gi` itself); double-check indexing

---

## Key Takeaway

"Sum of `|i - j|` over a set of indices" → **sort indices, build prefix sum, split at each position**:

```
Σ|gi - gj| at position i = (gi×i - prefSum[i])  +  (prefSum[m] - prefSum[i+1] - gi×(m-1-i))
```

Group by value first → apply formula independently per group → O(n) total.

Pattern: absolute distance sums on grouped indices → prefix sum trick replaces O(n²) brute force with O(n).
