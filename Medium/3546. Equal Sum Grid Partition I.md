# 3546. Equal Sum Grid Partition I
## O(m×n) — Prefix Row/Column Sums

---

## Problem Statement

- Make one horizontal OR one vertical cut on the grid.
- Both sections must be non-empty and have equal sums.
- Return true if such a cut exists.

---

## Core Idea — Prefix Sums Along Each Axis

**Total sum** must be even for any valid cut to exist (since each half = total/2).

**Horizontal cut** between rows `i` and `i+1`: left section sum = `sum(rows 0..i)`. Valid if it equals `total/2`.

**Vertical cut** between columns `j` and `j+1`: left section sum = `sum(cols 0..j)`. Valid if it equals `total/2`.

Precompute row sums and column sums, then scan with a running prefix sum.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool CanPartitionGrid(int[][] grid) {
        int m = grid.Length, n = grid[0].Length;
        long total = 0;
        long[] rowSum = new long[m];
        long[] colSum = new long[n];

        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++) {
                rowSum[i] += grid[i][j];
                colSum[j] += grid[i][j];
                total     += grid[i][j];
            }

        if (total % 2 != 0) return false;
        long half = total / 2;

        // Horizontal cuts
        long pref = 0;
        for (int i = 0; i < m - 1; i++) {
            pref += rowSum[i];
            if (pref == half) return true;
        }

        // Vertical cuts
        pref = 0;
        for (int j = 0; j < n - 1; j++) {
            pref += colSum[j];
            if (pref == half) return true;
        }

        return false;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(m×n)** | One pass to build sums, O(m+n) to scan |
| **Space** | **O(m+n)** | Row and column sum arrays |

---

## Example Walkthrough

**`grid=[[1,4],[2,3]]`**, total=10, half=5

Row sums: `[5, 5]`. Column sums: `[3, 7]`.

Horizontal: pref after row 0 = 5 == half → **true** ✓

---

## Pitfalls & Edge Cases

- **Odd total**: immediately return false — can't split into two equal integer halves
- **`long` for sums**: `m×n ≤ 10⁵`, `grid[i][j] ≤ 10⁵` → max total = `10¹⁰`, overflows `int`
- **Cut must be non-empty on both sides**: only iterate `i` from `0` to `m-2` (and `j` to `n-2`)

---

## Key Takeaway

"Equal partition by a single straight cut" → **check prefix sums row-by-row and column-by-column**. If any prefix equals `total/2`, a valid cut exists. O(m×n) single pass.
