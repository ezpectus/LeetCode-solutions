# 2861. Maximum Number of Alloys
## O(k × n × log(max)) — Binary Search per Machine

---

## Problem Statement

- `k` machines, each with a recipe (`composition[m][j]` = units of metal `j` needed per alloy).
- All alloys must use the same machine. Stock available; buy extra at cost.
- Return maximum alloys producible within `budget`.

---

## Core Idea — Binary Search on Number of Alloys

**Monotone predicate**: if we can make `x` alloys with machine `m`, we can make `x-1` too → binary search.

**Feasibility check** for machine `m` and `x` alloys:
```
total_cost = sum over j of max(0, composition[m][j]*x - stock[j]) * cost[j]
valid if total_cost <= budget
```

For each of the `k` machines, binary search on `x` in `[0, upper_bound]`. Take the maximum across all machines.

**Upper bound**: `budget / min_cost_per_alloy + max_stock_contribution`. Safely use `10^8 + 1` (given `budget ≤ 10^8` and minimum cost per alloy ≥ 1).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxNumberOfAlloys(int n, int k, int budget,
        IList<IList<int>> composition, IList<int> stock, IList<int> cost) {

        int res = 0;

        for (int m = 0; m < k; m++) {
            long lo = 0, hi = stock.Max() + (long)budget + 1; // stock on hand + max purchasable
            while (lo < hi) {
                long mid = (lo + hi + 1) / 2;
                if (CanMake(m, mid, n, budget, composition, stock, cost))
                    lo = mid;
                else
                    hi = mid - 1;
            }
            res = Math.Max(res, (int)lo);
        }

        return res;
    }

    bool CanMake(int m, long x, int n, int budget,
        IList<IList<int>> composition, IList<int> stock, IList<int> cost) {
        long total = 0;
        for (int j = 0; j < n; j++) {
            long need = (long)composition[m][j] * x;
            long buy = Math.Max(0, need - stock[j]);
            total += buy * cost[j];
            if (total > budget) return false; // early exit
        }
        return true;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(k × n × log(10⁸))** | k machines × binary search × n metals per check |
| **Space** | **O(1)** | No extra space |

With `k,n ≤ 100` and `log(10⁸) ≈ 27`: ~270,000 operations — instant.

## Example Walkthrough

**Example 1**: n=3, k=2, budget=15, `composition=[[1,1,1],[1,1,10]]`, stock=[0,0,0], cost=[1,2,3]

**Machine 0** (recipe [1,1,1]):
- Binary search for max x. Try x=2:
  - Need: 2 of each. Stock: 0. Buy: 2,2,2. Cost: 2×1+2×2+2×3=12 ≤ 15 ✓
- Try x=3: cost=3+6+9=18 > 15 ✗ → max=2

**Machine 1** (recipe [1,1,10]):
- x=1: buy 1+2+10×3=33 > 15 ✗ → max=0

`res = max(2,0) = 2` ✓

## Pitfalls & Edge Cases

- **`long` for intermediate calculations**: `composition[m][j] * x` up to `100 × 10⁸ = 10¹⁰`, overflows `int`
- **Early exit in cost check**: if `total > budget` mid-loop, skip remaining metals
- **Upper bound**: `10⁸ + 1` safely exceeds max possible answer (budget ≤ 10⁸, min cost per alloy ≥ 1)
- **Stock can exceed need**: `max(0, need - stock[j])` ensures we never "buy negative"

---

## Key Takeaway

"Maximize output subject to budget constraint with multiple recipes" → **binary search per recipe + feasibility check**:

```
for each machine: binary search on x
  feasible(x) = sum(max(0, recipe[j]*x - stock[j]) * cost[j]) <= budget
```

Binary search works because cost increases monotonically with `x`. Take max over all machines.
