# 3627. Maximum Median Sum of Subsequences of Size 3

## O(n log n) — Greedy After Sorting Descending

---

## Problem Statement

- Given integer array `nums` where length is divisible by 3.
- Repeatedly select 3 elements, compute their **median** (middle value when sorted), remove them.
- Continue until array is empty.
- Return maximum possible sum of all medians.
- **Constraints**: 1 ≤ nums.length ≤ 5×10⁵, nums.length % 3 == 0, 1 ≤ nums[i] ≤ 10⁹

---

## Core Idea — Greedy Pairing Strategy

**Key observation**: For triple (a, b, c) sorted as a ≤ b ≤ c, median is **b** (middle value).

**Goal**: Maximize sum of medians → use largest possible values as medians.

**Greedy strategy**: After sorting **descending**:
- Two pointers: `left` at largest values, `right` at smallest values
- Form triples: (nums[left], nums[left+1], nums[right])
- When sorted: (nums[right], nums[left+1], nums[left])
- **Median = nums[left+1]** (second-largest in the triple)
- This uses the largest available values as medians

**Why this works**: 
- Pairing two large values with one small value
- Second-largest becomes the median
- Greedy choice: always pick the largest available as median

---

## Algorithm Breakdown

```
Sort nums in descending order
left = 0 (largest)
right = n - 1 (smallest)

while left < right:
  // Form triple: (nums[left], nums[left+1], nums[right])
  // Sorted: (nums[right], nums[left+1], nums[left])
  // Median = nums[left+1]
  medianSum += nums[left + 1]
  left += 2
  right--

return medianSum
```

**Complexity**: O(n log n) for sorting + O(n) for iteration

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumMedianSum(int[] nums) {
        int n = nums.Length;
        Array.Sort(nums);
        Array.Reverse(nums); // Sort descending
        
        long medianSum = 0;
        int left = 0; // Start from largest
        int right = n - 1; // Start from smallest
        
        while (left < right) {
            // Form triple: (nums[left], nums[left+1], nums[right])
            // When sorted: (nums[right], nums[left+1], nums[left])
            // Median = nums[left+1] (second-largest)
            medianSum += nums[left + 1];
            left += 2;
            right--;
        }
        
        return medianSum;
    }
}
```

## Why This Works — Example Walkthrough

### Example 1: `nums = [2,1,3,2,1,3]`

**Sort descending**: [3, 3, 2, 2, 1, 1]

**Iteration 1**:
- left=0, right=5
- Triple: (nums[0], nums[1], nums[5]) = (3, 3, 1)
- Sorted: (1, 3, 3)
- Median = nums[1] = 3
- medianSum = 3
- left=2, right=4

**Iteration 2**:
- left=2, right=4
- Triple: (nums[2], nums[3], nums[4]) = (2, 2, 1)
- Sorted: (1, 2, 2)
- Median = nums[3] = 2
- medianSum = 5

**Result**: 5 ✓

### Example 2: `nums = [1,1,10,10,10,10]`

**Sort descending**: [10, 10, 10, 10, 1, 1]

**Iteration 1**:
- left=0, right=5
- Triple: (10, 10, 1)
- Sorted: (1, 10, 10)
- Median = 10
- medianSum = 10
- left=2, right=4

**Iteration 2**:
- left=2, right=4
- Triple: (10, 10, 1)
- Sorted: (1, 10, 10)
- Median = 10
- medianSum = 20

**Result**: 20 ✓

## Pitfalls & Edge Cases

- **Sorting direction**: Must sort **descending** to use largest values as medians.
- **Pointer management**: `left += 2` (skip two large values per triple), `right--` (use one small value).
- **Triple formation**: (largest, second-largest, smallest) → when sorted, second-largest is median.
- **Array length**: Guaranteed divisible by 3, no remainder handling needed.
- **Overflow**: Use `long` for sum (max: 5×10⁵ × 10⁹ = 5×10¹⁴).

---

## Key Takeaway

"Maximum median sum from size-3 subsequences" → **Greedy after sorting descending**. **Algorithm**:
1. **Sort descending**
2. **Two pointers**: left at largest, right at smallest
3. **Form triples**: (nums[left], nums[left+1], nums[right])
4. **Median = nums[left+1]** (second-largest)
5. **Advance**: left += 2, right--

O(n log n). **Critical insight**: Pair two large values with one small → second-largest becomes median, maximizing sum.
