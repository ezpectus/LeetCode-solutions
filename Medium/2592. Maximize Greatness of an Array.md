# 2592. Maximize Greatness of an Array
## O(n log n) — Sort + Greedy Two Pointers

---

## Problem Statement

- You are given a **0-indexed** integer array `nums` (1 ≤ n ≤ 10⁵), where each `nums[i]` (0 ≤ nums[i] ≤ 10⁹).
- Permute `nums` into any array `perm` of your choosing.
- **Greatness** = number of indices `i` where `perm[i] > nums[i]`.
- Return the **maximum possible greatness**.

---

## Core Idea — Sort + Greedy Two Pointers

**Key insight**: Sort `nums`. Then greedily try to match each element `nums[r]` (acting as `perm[i]`) against the smallest unmatched element `nums[l]` (acting as `nums[i]`) that it strictly beats.

**Why this works?**
- After sorting, `perm` is just a rearrangement of the same sorted array
- We want to maximize the number of elements that "beat" another element
- The greedy strategy: always use the **smallest available value that beats the current target** — never waste a large value on a target it could beat more easily later
- `l` tracks which original position we're trying to beat; `r` scans for something strictly greater

**Two-pointer sweep**:
- Both `l` and `r` start at 0, sweep over sorted array
- If `nums[r] > nums[l]`: valid match found → `res++`, advance `l`
- Always advance `r`
- Final `res` = number of matched pairs

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximizeGreatness(int[] nums) {
        Array.Sort(nums);
        int n = nums.Length;
        int l = 0, res = 0;

        for (int r = 0; r < n; r++) {
            if (nums[r] > nums[l]) {
                res++;
                l++;
            }
        }

        return res;
    }
}
```

## Why This Works — Example Walkthrough

**Example 1**: `nums = [1,3,5,2,1,3,1]` → sorted: `[1,1,1,2,3,3,5]`

| r | nums[r] | l | nums[l] | Match? | res |
|---|---------|---|---------|--------|-----|
| 0 | 1 | 0 | 1 | ❌ | 0 |
| 1 | 1 | 0 | 1 | ❌ | 0 |
| 2 | 1 | 0 | 1 | ❌ | 0 |
| 3 | 2 | 0 | 1 | ✅ | 1 |
| 4 | 3 | 1 | 1 | ✅ | 2 |
| 5 | 3 | 2 | 1 | ✅ | 3 |
| 6 | 5 | 3 | 2 | ✅ | 4 |

Answer: **4** ✓

**Example 2**: `nums = [1,2,3,4]` → sorted: `[1,2,3,4]`

| r | nums[r] | l | nums[l] | Match? | res |
|---|---------|---|---------|--------|-----|
| 0 | 1 | 0 | 1 | ❌ | 0 |
| 1 | 2 | 0 | 1 | ✅ | 1 |
| 2 | 3 | 1 | 2 | ✅ | 2 |
| 3 | 4 | 2 | 3 | ✅ | 3 |

Answer: **3** ✓

## Pitfalls & Edge Cases

- **Strict inequality required** — `perm[i] > nums[i]`, not `>=`; duplicates never count as a match
- **All equal elements** — e.g. `[1,1,1,1]`: `r` advances but `l` never moves → `res = 0` ✓
- **`r` always advances** — even on a miss; `l` only advances on a hit (standard greedy pair matching)
- **Same array for both sides** — `perm` is a permutation of `nums` itself; we're matching elements of sorted `nums` against elements of sorted `nums`, just offset

---

## Key Takeaway

Maximize the number of "beating" pairs from a self-permutation → **sort + greedy two pointers**:

- **Sort** so smallest values are targets (`l`) and we scan for beaters (`r`)
- **Greedy match**: use the smallest available value that strictly beats the current target — never overshoot
- `r` always moves; `l` only moves on a successful match → `res` = total matches = maximum greatness

Pattern: "maximize matched pairs with strict inequality from same array" → sort → greedy scan.
