# 3653. XOR After Range Multiplication Queries I
## O(n × q) — Direct Simulation

---

## Problem Statement

- For each query `[l, r, k, v]`: multiply `nums[idx]` by `v` (mod 10⁹+7) for each `idx = l, l+k, l+2k, ...` up to `r`.
- After all queries, return XOR of all elements.

---

## Core Idea — Simulate Each Query Directly

With `n, q ≤ 1000`, each query updates at most `n/k` elements. Total updates ≤ `q × n = 10⁶` — fast enough.

Simply iterate through each query, update the relevant indices with strided multiplication, then XOR the final array.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int XorAfterQueries(int[] nums, int[][] queries) {
        const long MOD = 1_000_000_007;
        long[] arr = nums.Select(x => (long)x).ToArray();

        foreach (var q in queries)
            for (int idx = q[0]; idx <= q[1]; idx += q[2])
                arr[idx] = arr[idx] * q[3] % MOD;

        int res = 0;
        foreach (long x in arr) res ^= (int)x;
        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × q)** | Each query updates at most n elements; q queries total |
| **Space** | **O(n)** | Working array |

With n=q=1000: at most 10⁶ operations — instant.

---

## Example Walkthrough

**`nums=[2,3,1,5,4], queries=[[1,4,2,3],[0,2,1,2]]`**

Query 1 `[1,4,2,3]`: idx=1,3. nums[1]=3×3=9, nums[3]=5×3=15 → `[2,9,1,15,4]`

Query 2 `[0,2,1,2]`: idx=0,1,2. nums[0]=2×2=4, nums[1]=9×2=18, nums[2]=1×2=2 → `[4,18,2,15,4]`

XOR: 4^18^2^15^4 = **31** ✓

---

## Pitfalls & Edge Cases

- **`long` for intermediate products**: `nums[i]` up to 10⁹, `v` up to 10⁵ → product up to 10¹⁴, overflows `int`. Use `long` with mod.
- **Mod before XOR**: apply `% MOD` at each update step to keep values in `[0, 10⁹+6]`.
- **Stride `k`**: `idx += k` ensures we only hit indices `l, l+k, l+2k, ...` — not every element in `[l,r]`.

---

## Key Takeaway

"Strided range multiplication queries, return final XOR" → **direct simulation**:

```
for each query [l, r, k, v]:
    for idx = l; idx <= r; idx += k:
        nums[idx] = nums[idx] * v % MOD

return XOR of all nums
```

Small constraints (n, q ≤ 1000) make O(n×q) simulation the simplest and sufficient approach.
