# 🏔️ 3913. Sort Vowels by Frequency — Architectural Solution

## Problem Statement

You are given a string `s` consisting of lowercase English characters. 
Rearrange only the vowels in the string so that they appear in **non-increasing order of their frequency**. 

If multiple vowels have the same frequency, order them by the **position of their first occurrence** in `s`.
Return the modified string.

Vowels are defined as: 'a', 'e', 'i', 'o', and 'u'.

### Constraints
- 1 <= s.length <= 10^5
- `s` consists of lowercase English letters.

---

## 💡 Core Idea

The problem requires a custom sort of vowels based on two keys:
1. **Primary Key**: Frequency (higher frequency comes first).
2. **Secondary Key**: First occurrence index in the string (smaller index comes first).

Using an `unordered_map` directly with standard sorting can cause bugs because hash maps do not preserve the insertion or original index order. Since there are always exactly 5 vowels, we can track their metrics using fixed-size arrays of size 26. 

We count frequencies and record the first occurrence index for each vowel in a single pass. Then, we sort the 5 vowel characters using a custom lambda comparator and rebuild the string.

---

## 💻 Implementation (C++)

```cpp
class Solution {
public:
    bool isVowel(char ch) {
        return ch == 'a' || ch == 'e' || ch == 'i' || ch == 'o' || ch == 'u';
    }

    string sortVowels(string s) {
        vector<int> freq(26, 0);
        vector<int> first_occ(26, -1);
        
        for (int i = 0; i < s.length(); i++) {
            if (isVowel(s[i])) {
                freq[s[i] - 'a']++;
                if (first_occ[s[i] - 'a'] == -1) {
                    first_occ[s[i] - 'a'] = i;
                }
            }
        }

        vector<char> vowels = {'a', 'e', 'i', 'o', 'u'};
        sort(vowels.begin(), vowels.end(), [&](char a, char b) {
            if (freq[a - 'a'] != freq[b - 'a']) {
                return freq[a - 'a'] > freq[b - 'a'];
            }
            return first_occ[a - 'a'] < first_occ[b - 'a'];
        });

        string sorted_vowels = "";
        for (char v : vowels) {
            sorted_vowels.append(freq[v - 'a'], v);
        }

        int idx = 0;
        for (int i = 0; i < s.length(); i++) {
            if (isVowel(s[i])) {
                s[i] = sorted_vowels[idx++];
            }
        }
        return s;
    }
};
```

## 📐 Complexity Analysis

| Metric | Value | Justification |
| :--- | :--- | :--- |
| **Time Complexity** | O(n) | **Frequency Scan**: A single loop through `s` takes O(n) time.<br>**Sorting**: We only sort a vector of 5 elements, which takes O(5 log 5) = O(1) constant time.<br>**Rebuilding**: Replacing vowels back into the string takes O(n) time. |
| **Space Complexity** | O(1) | The arrays `freq` and `first_occ` always use a fixed size of 26. The `vowels` array always holds 5 characters. Thus, auxiliary space is constant and independent of string length. |

## ⚠️ Pitfalls & Edge Cases

- **Frequency Ties**: If 'e' and 'o' both appear 2 times, the one that appeared first in the original string must be placed first. Tracking `first_occ[v - 'a']` prevents violating this rule.
- **No Vowels Present**: If the string consists entirely of consonants (e.g., "bcdfgh"), the loop runs safely, `sorted_vowels` remains empty, and the original string is returned unchanged.
- **Fixed Alphabet Optimization**: Using hash maps adds overhead and removes ordering. Since we only care about 5 specific lowercase characters, flat `vector<int>` arrays mapped via `ch - 'a'` provide direct O(1) cache-friendly lookups.

## 🔄 Alternative Approaches

- **Counting Sort / Bucket Sort**: Since frequencies cannot exceed n, you can create an array of buckets where the index represents frequency. However, since there are only 5 vowels total, a standard sort on 5 elements is already as optimal as any bucket sorting strategy.
- **Priority Queue**: You could push all individual vowel instances into a Max-Heap with a custom tracking structure. This would take O(n log n) time due to heap pushes, making it less efficient than the O(n) array tracking approach.
