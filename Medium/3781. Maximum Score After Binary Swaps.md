# 3781. Maximum Score After Binary Swaps
## O(n log n) — Greedy Left-Scan with Max-Heap

---

## Problem Statement

- You are given `nums` (1 ≤ n ≤ 10⁵) and a binary string `s` of the same length.
- Score = sum of `nums[i]` where `s[i] = '1'`.
- Operation: choose index `i` where `s[i] = '0'` and `s[i+1] = '1'`, swap them (moves a '1' one step left).
- Unlimited operations allowed. Return maximum achievable score.

---

## Core Idea — What Can '1's Actually Reach?

Key observation: each operation moves a '1' **one step left** through a '0'. '1's cannot pass each other. After unlimited operations:
- Each '1' can be placed at **any position to its left** (including its original).
- The **relative order** of '1's is always preserved.

So if '1's start at positions `p[0] < p[1] < ... < p[k-1]`, the set of achievable states is any assignment `q[0] < q[1] < ... < q[k-1]` where `q[i] <= p[i]`.

**Goal:** choose k distinct positions (sorted as q[0] < ... < q[k-1], q[i] <= p[i]) to maximize `sum(nums[q[i]])`.

**Greedy insight:** Scan left to right. Maintain a max-heap of all `nums` values seen so far. Each time we reach a `'1'`, greedily extract the maximum value from the heap — that's the best position this '1' can claim from the range `[0, p[i]]`. The heap naturally enforces `q[i] <= p[i]` because we only add positions up to `p[i]` before extracting for '1' `i`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumScore(int[] nums, string s) {
        int n = nums.Length;

        // C# PriorityQueue is min-heap; use negative priority for max-heap behavior
        var heap = new PriorityQueue<int, int>();
        long score = 0;

        for (int i = 0; i < n; i++) {
            // Add current position's value to heap
            heap.Enqueue(nums[i], -nums[i]);

            // When we hit a '1', greedily take the best available value
            if (s[i] == '1') {
                score += heap.Dequeue();
            }
        }

        return score;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n log n) | Each element pushed and popped from heap at most once |
| Space Complexity | O(n) | Heap holds at most n elements |

## Why This Works — Proof of Correctness

**Feasibility:** When we process '1' at position `p[i]`, the heap contains values from positions `0` to `p[i]` only (we add left to right). So the extracted position satisfies `q[i] <= p[i]`. ✓

**Optimality:** Among all valid assignments, the heap always picks the maximum available value. By exchange argument: if any other assignment gives a higher sum, we can swap one of its chosen positions with our max extraction — contradiction.

**Ordering constraint:** We never assign two '1's the same position (each value is extracted once and removed from the heap). The sorted selection of k positions is always a valid increasing sequence that satisfies `q[i] <= p[i]`. ✓

## Example Walkthrough

**Example 1:** `nums = [2,1,5,2,3]`, `s = "01010"`

'1's at positions 1, 3.

| i | nums[i] | Heap after push | s[i] | Action | Score |
|---|---------|-----------------|------|--------|-------|
| 0 | 2       | {2}             | '0'  | —      | 0     |
| 1 | 1       | {2,1}           | '1'  | pop 2  | 2     |
| 2 | 5       | {5,1}           | '0'  | —      | 2     |
| 3 | 2       | {5,2,1}         | '1'  | pop 5  | 7     |
| 4 | 3       | {3,2,1}         | '0'  | —      | 7     |

'1' at pos 1 claimed nums[0]=2 (moved left), '1' at pos 3 claimed nums[2]=5 (moved left).

**Output:** 7 ✓

---

**Example 2:** `nums = [4,7,2,9]`, `s = "0000"`

No '1's encountered → score never increases.

**Output:** 0 ✓

## Pitfalls & Edge Cases

- **No '1's in string** — Heap is never popped, score stays 0.

- **All '1's** — Every position gets extracted in order; since heap always holds the max of everything seen so far, we greedily claim the largest values. Equivalent to selecting all elements (score = total sum).

- **'1' at position 0** — Only nums[0] is in the heap when we extract, so it takes nums[0] (can't go further left).

- **Overflow** — nums[i] up to 10⁹, n up to 10⁵ → max score up to 10¹⁴. Must use `long`.

- **C# max-heap trick** — `PriorityQueue<TElement, TPriority>` is min-heap. Store `-nums[i]` as priority so the element with the highest value dequeues first.

- **'1's block each other** — Handled naturally: each extracted value is removed from the heap, so two '1's never claim the same position.

---

## Key Takeaway

'1' can only move **left**, and '1's preserve **relative order** → each '1' at `p[i]` can claim any position in `[0, p[i]]` not taken by an earlier '1'. This is equivalent to an online scheduling problem: for each '1' (deadline = its position), greedily pick the max-value job available so far.

**Pattern recognition:** "Element can shift left through neutral elements, maximize score" → max-heap left-scan. The heap accumulates candidates and each '1' greedily claims the best one available at its position.

Classic trick: PriorityQueue on a left-to-right scan, extract when a deadline (the '1') is reached.
