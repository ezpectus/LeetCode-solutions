# 3164. Find the Number of Good Pairs II
## O(max1/k × log(max1)) — Frequency Map + Divisor Enumeration

---

## Problem Statement

- Pair `(i, j)` is good if `nums1[i] % (nums2[j] * k) == 0`.
- Return count of good pairs.

---

## Core Idea — Count Multiples of Each Target

For a fixed `nums2[j]`, the required divisor is `target = nums2[j] * k`. We need to count how many elements of `nums1` are divisible by `target`.

**Key optimization**: instead of checking every `nums1[i]` against every `target`, use a frequency map of `nums1` and enumerate **multiples** of `target` up to `max(nums1)`:

```
count(target) = Σ freq1[target], freq1[2×target], freq1[3×target], ...
```

Also use a frequency map of `nums2` to avoid redundant computation for repeated values.

**Total work**: for each distinct `v2 ∈ nums2`, enumerate `max1 / (v2*k)` multiples. Sum across all distinct `v2` values ≈ `O(max1/k × H(max_v2))` where H is harmonic series.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long NumberOfPairs(int[] nums1, int[] nums2, int k) {
        var freq1 = new Dictionary<int, int>();
        int max1 = 0;
        foreach (int x in nums1) {
            freq1[x] = freq1.GetValueOrDefault(x) + 1;
            max1 = Math.Max(max1, x);
        }

        var freq2 = new Dictionary<int, int>();
        foreach (int x in nums2)
            freq2[x] = freq2.GetValueOrDefault(x) + 1;

        long res = 0;
        foreach (var (v2, cnt2) in freq2) {
            long target = (long)v2 * k;
            if (target > max1) continue;
            for (long mult = target; mult <= max1; mult += target)
                if (freq1.TryGetValue((int)mult, out int c1))
                    res += (long)c1 * cnt2;
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(max1/k × log(max_v2))** | Harmonic series of multiples; k in denominator helps |
| **Space** | **O(n + m)** | Two frequency maps |

With `max1 = 10^6`, `k = 1`: ~10^6 × ln(10^6) ≈ 14M operations. With larger `k`: fewer multiples per target.

---

## Example Walkthrough

**`nums1=[1,2,4,12], nums2=[2,4], k=3`**

freq1: {1:1, 2:1, 4:1, 12:1}. max1=12.
freq2: {2:1, 4:1}.

- v2=2, target=6: multiples ≤12: 6,12. freq1[6]=0, freq1[12]=1. Contribution: 1×1=1.
- v2=4, target=12: multiples ≤12: 12. freq1[12]=1. Contribution: 1×1=1.

Total = **2** ✓

---

## Pitfalls & Edge Cases

- **`target > max1`**: skip immediately — no multiple can exist.
- **`long` for target**: `v2 * k` can reach `10^6 × 10^3 = 10^9` → overflows `int`.
- **`long` for result**: up to `10^5 × 10^5 = 10^10` pairs possible → overflows `int`.

---

## Key Takeaway

"Count pairs where a divisible by b" → **frequency map + enumerate multiples**:

```
for each distinct v2 in nums2:
    target = v2 * k
    for mult = target, 2×target, ..., max1:
        res += freq1[mult] × freq2[v2]
```

This avoids O(n×m) brute force by leveraging the divisibility structure. O(max1/k) multiples per target value.
