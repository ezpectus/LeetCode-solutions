# 2641. Cousins in Binary Tree II
## O(n) — BFS Level-by-Level with Sibling Subtraction

---

## Problem Statement

- Two nodes are **cousins** if they have the same depth but **different parents**.
- Replace each node's value with the **sum of all its cousins' values**.
- Return the modified tree root.

---

## Core Idea — BFS: Next-Level Sum minus Sibling Sum

**Key insight**: For any node at depth `d`, its new value = `(sum of all nodes at depth d)` − `(sum of its siblings)`. Process level by level via BFS. For each node in the current level, compute the next level's total sum, then assign each child: `nextLevelSum - siblingSum`.

**Why `nextLevelSum - siblingSum`?**

```
cousinSum(child) = sum of all nodes at child's depth  −  sum of child's siblings
                 = nextLevelSum                        −  (parent.left.val + parent.right.val)
```

**Critical ordering**: compute `nextLevelSum` from **original** values before modifying any children. Both children of the same parent share the same `sibSum`, so siblings correctly cancel each other.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public TreeNode ReplaceValueInTree(TreeNode root) {
        root.val = 0; // root has no cousins
        var q = new Queue<TreeNode>();
        q.Enqueue(root);

        while (q.Count > 0) {
            // Phase 1: sum all children of current level (original next-level values)
            int nextSum = 0;
            foreach (var node in q) {
                if (node.left  != null) nextSum += node.left.val;
                if (node.right != null) nextSum += node.right.val;
            }

            // Phase 2: assign cousin sum = nextSum - siblingSum to each child
            int size = q.Count;
            for (int k = 0; k < size; k++) {
                var node = q.Dequeue();
                int sibSum = (node.left  != null ? node.left.val  : 0)
                           + (node.right != null ? node.right.val : 0);

                if (node.left  != null) { node.left.val  = nextSum - sibSum; q.Enqueue(node.left);  }
                if (node.right != null) { node.right.val = nextSum - sibSum; q.Enqueue(node.right); }
            }
        }

        return root;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Each node visited twice per level (sum pass + assign pass) |
| **Space Complexity** | **O(w)** | w = max width of tree; queue holds at most one full level |

---

## BFS vs DFS

**DFS** requires two passes:
1. Collect `levelSum[depth]` for all depths
2. Re-traverse: assign `levelSum[depth] - sibSum` to each node

**BFS** does it in **one pass** — queue naturally groups nodes by level, so `nextLevelSum` is computed exactly when needed. No extra array, cleaner logic.

---

## Two-Phase Ordering — Why It Matters

```
Current queue: [nodeA, nodeB]  (same level, different parents)

Phase 1 — compute nextSum from ORIGINAL values:
  nextSum = A.left + A.right + B.left + B.right

Phase 2 — update:
  For nodeA: sibSum = A.left + A.right
    A.left.val  = nextSum - sibSum  (= B's children sum = cousin sum) ✓
    A.right.val = nextSum - sibSum  ✓
  For nodeB: sibSum = B.left + B.right
    B.left.val  = nextSum - sibSum  (= A's children sum) ✓
```

If Phase 2 ran during Phase 1, `nextSum` would include already-replaced values → wrong.

---

## Example Walkthrough

**Example 1**: `root = [5,4,9,1,10,null,7]`

**Level 0** → `root.val = 0`

**Process level 0** `[node(5)]`:
- `nextSum = 4 + 9 = 13`
- `sibSum for node(5) = 4+9 = 13`
- `node(4).val = 13-13 = 0` ✓
- `node(9).val = 13-13 = 0` ✓

**Process level 1** `[node(4), node(9)]`:
- `nextSum = 1+10+0+7 = 18`
- `node(4): sibSum = 1+10=11 → left(1).val=7, right(10).val=7` ✓
- `node(9): sibSum = 0+7=7  → right(7).val=18-7=11` ✓

Output: `[0,0,0,7,7,null,11]` ✓

---

## Pitfalls & Edge Cases

- **Compute `nextSum` before modifying** — modifying during the sum pass corrupts results
- **`root.val = 0` upfront** — root has no cousins at any depth
- **Single child** — `sibSum` = that child only; the child still gets `nextSum - sibSum` = sum of all cousins ✓
- **Null children** — safely guarded with null checks; `sibSum` only adds existing children
- **All nodes on same level with same parent** — all get 0 (no cousins), since `nextSum = sibSum` for every parent

---

## Key Takeaway

"Replace each node with cousin sum" → **BFS two-phase per level**:

```
nextSum = Σ children values at current level   (compute FIRST, from originals)
child.val = nextSum - (parent.left + parent.right)   (sibling subtraction)
```

Queue groups nodes by level naturally — no depth array needed. The formula `nextSum - sibSum` is the key: total level sum minus your parent's contribution = exactly your cousin sum.
