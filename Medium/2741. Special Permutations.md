# 2741. Special Permutations
## O(2вЃї Г— nВІ) вЂ” Bitmask DP

---

## Problem Statement

- Permutation is **special** if every adjacent pair satisfies: one divides the other.
- Count special permutations mod 10вЃ№+7.
- `n в‰¤ 14` в†’ bitmask DP is the natural choice.

---

## Core Idea вЂ” Bitmask DP on Subsets

**State**: `dp[mask][i]` = number of valid arrangements using exactly the elements in `mask`, where `nums[i]` is the last element placed.

**Transition**: to extend a state `(mask, i)` by appending element `j` (not yet in `mask`):
```
if nums[i] % nums[j] == 0 or nums[j] % nums[i] == 0:
    dp[mask | (1<<j)][j] += dp[mask][i]
```

**Base case**: `dp[1<<i][i] = 1` for each `i` вЂ” one way to have just element `i` as the only element.

**Answer**: `sum(dp[(1<<n)-1][i])` for all `i` вЂ” sum over all elements that could be last.

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int specialPerm(vector<int>& nums) {
        const int MOD = 1e9 + 7;
        int n = nums.size();
        int full = (1 << n) - 1;
        vector<vector<long long>> dp(1 << n, vector<long long>(n, 0));

        for (int i = 0; i < n; i++)
            dp[1 << i][i] = 1;

        for (int mask = 1; mask <= full; mask++) {
            for (int i = 0; i < n; i++) {
                if (!(mask >> i & 1) || dp[mask][i] == 0) continue;
                for (int j = 0; j < n; j++) {
                    if (mask >> j & 1) continue; // j already used
                    if (nums[i] % nums[j] == 0 || nums[j] % nums[i] == 0)
                        dp[mask | (1 << j)][j] = (dp[mask | (1 << j)][j] + dp[mask][i]) % MOD;
                }
            }
        }

        long long res = 0;
        for (int i = 0; i < n; i++) res = (res + dp[full][i]) % MOD;
        return res;
    }
};
```

## Why Bitmask DP?

A permutation problem where we need to track **which elements have been used** and **what the last element was** в†’ bitmask encodes the used set in O(1) space per state. With `n в‰¤ 14`, the state space is `2^14 Г— 14 = 229376` вЂ” feasible.

Brute force (all permutations): `14! в‰€ 87Г—10вЃ№` вЂ” way too slow.

## Pitfalls & Edge Cases

- **`dp[mask][i] == 0` early skip** вЂ” avoids unnecessary inner loop iterations; important for performance
- **`!(mask >> i & 1)` check** вЂ” skip if `i` not in `mask` (invalid state)
- **`long long` for dp values** вЂ” values accumulate up to `(n-1)!` в‰€ 13! в‰€ 6Г—10вЃ№ before modding; use `long long`

---

## Key Takeaway

"Count permutations with adjacent constraints" в†’ **bitmask DP**:

```
dp[mask][last] = ways to arrange elements in mask ending with last
transition: if compatible(last, next) в†’ dp[mask|(1<<next)][next] += dp[mask][last]
```

`n в‰¤ 14` в†’ `2^14 Г— 14ВІ` states fits comfortably. Pattern appears in TSP, Hamiltonian path, and any problem requiring "arrange all elements with pairwise constraints".
