# 3694. Distinct Points Reachable After Substring Removal

## O(n) — Prefix Vector Difference (Optimal Solution)

---

## Problem Statement

You are given:

- A movement string `s` consisting of `'U'`, `'D'`, `'L'`, `'R'`
- An integer `k`

Each character moves a point on the infinite 2D grid:

- `U`: (x, y+1)
- `D`: (x, y−1)
- `L`: (x−1, y)
- `R`: (x+1, y)

You must:

1. Remove **exactly one** contiguous substring of length `k`
2. Start at `(0,0)`
3. Execute the remaining moves in order

Return the **number of distinct final coordinates** reachable.

---

## Core Idea — Final Position = Total − Removed Segment

Let:

- `pref[i] = movement vector after first i moves`
- `total = pref[n]` — final movement without removal

If we remove substring `s[l..r)` of length `k`:

- Movement removed:
  ```
  removed = pref[r] − pref[l]
  ```
  - Final coordinate becomes:
```
Thus:
finalX = totalX − (prefX[r] − prefX[l])
finalY = totalY − (prefY[r] − prefY[l])
```

We simply compute this for all `n − k + 1` windows.

---

## Why This Works

Removing a substring removes **exactly the movement contributed by that substring**.

Everything before `l` and after `r` stays unchanged.

Therefore:

- No simulation needed
- No sliding window of positions
- No recomputation of paths
- Just prefix sums + vector subtraction

This gives a clean **O(n)** solution.

---

## Algorithm Breakdown

### Phase 1 — Build Prefix Movement Arrays
```
px[i] = total x-movement after i moves
py[i] = total y-movement after i moves
```

### Phase 2 — Compute Total Movement
```
totalX = px[n]
totalY = py[n]
```

### Phase 3 — For Each Window [l, l+k)
```
removedX = px[r] − px[l]
removedY = py[r] − py[l]

finalX = totalX − removedX
finalY = totalY − removedY

insert (finalX, finalY) into a set
```

### Phase 4 — Return size of set

---

## Full Implementation (C#)

```csharp
public class Solution {
    public int DistinctPoints(string s, int k) {
        int n = s.Length;

        // prefix sums: px[i], py[i] = movement after i moves
        int[] px = new int[n + 1];
        int[] py = new int[n + 1];

        for (int i = 0; i < n; i++) {
            px[i + 1] = px[i];
            py[i + 1] = py[i];
            switch (s[i]) {
                case 'U': py[i + 1]++; break;
                case 'D': py[i + 1]--; break;
                case 'L': px[i + 1]--; break;
                case 'R': px[i + 1]++; break;
            }
        }

        int totalX = px[n];
        int totalY = py[n];

        var seen = new HashSet<(int,int)>();

        for (int l = 0; l + k <= n; l++) {
            int r = l + k;

            int removedX = px[r] - px[l];
            int removedY = py[r] - py[l];

            int finalX = totalX - removedX;
            int finalY = totalY - removedY;

            seen.Add((finalX, finalY));
        }

        return seen.Count;
    }
}
```
## Complexity

| **Metric** | **Value** | **Notes** |
|------------|-----------|-----------|
| **Time**   | **O(n)**  | One prefix pass + one window pass |
| **Space**  | **O(n)**  | Prefix arrays |

## Example Walkthrough

### Input
```
s = "LUL"
k = 1
```

### Prefix movement
```
i:  0   1    2    3
s:      L    U    L
px: 0  -1   -1   -2
py: 0   0    1    1
```

### Total movement:
```total = (-2, 1)```

---

## Windows

### Remove s[0] = 'L'
```
removed = (-1, 0)
final = (-2, 1) - (-1, 0) = (-1, 1)
```

### Remove s[1] = 'U'
```
removed = (0, 1)
final = (-2, 1) - (0, 1) = (-2, 0)
```

### Remove s[2] = 'L'
```
removed = (-1, 0)
final = (-1, 1)
```

### Distinct final points
```
(-1, 1), (-2, 0)
```

**Answer = 2**

---

## Key Takeaway

Removing a substring removes **exactly its movement vector**.

Therefore:
```
final = total − (prefix[r] − prefix[l])
```

This yields a clean **O(n)** solution using prefix sums and a hash set.
