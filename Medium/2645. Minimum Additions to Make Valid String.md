# 2645. Minimum Additions to Make Valid String
## O(n) — Greedy: Count Groups

---

## Problem Statement

- Given string `word` containing only `'a'`, `'b'`, `'c'`.
- Insert any characters anywhere to make the string a concatenation of `"abc"` repeated.
- Return the **minimum number of insertions**.

---

## Core Idea — Count "abc" Groups Greedily

**Key insight**: A valid string = `k` copies of `"abc"`. Each character of `word` must fit into one of these copies, preserving order `a → b → c`. A new group starts whenever the sequence "resets" — i.e., when the current character is **≤ the previous character** (we can't continue the current `abc` group forward).

**Formula**:
```
groups * 3 - len(word)
```
- `groups * 3` = total characters in the final valid string
- `len(word)` = already present characters
- Difference = minimum insertions needed

**How to count groups**: scan left to right, increment `groups` each time `word[i] <= word[i-1]`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int AddMinimum(string word) {
        int groups = 1;
        for (int i = 1; i < word.Length; i++)
            if (word[i] <= word[i-1]) groups++;
        return groups * 3 - word.Length;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Single pass over the string |
| **Space Complexity** | **O(1)** | Only one counter |

## Why This Works

Each character in `word` belongs to exactly one `"abc"` group. A group boundary occurs when `word[i] <= word[i-1]` — the current character can't follow the previous one within the same `"abc"` copy (since `a < b < c`).

**Visual**:
```
word = "a a b"
         ↑
       'a' <= 'a' → new group

Groups: [a__] [ab_] → 2 groups × 3 = 6, minus 3 chars = 3 insertions
Result: "abc" + "abc" → insert b,c and c → 3 ✓
```

Every group of size 3 needs exactly `3 - (chars in group)` insertions. Sum over all groups = `groups×3 - total chars`.

---

## Example Walkthrough

**Example 2**: `word = "aaa"`

| i | word[i] | word[i-1] | new group? | groups |
|---|---------|-----------|------------|--------|
| 1 | a | a | ✅ (a ≤ a) | 2 |
| 2 | a | a | ✅ | 3 |

`3 × 3 - 3 = 6` ✓

**Example 1**: `word = "b"`

No loop iterations → `groups = 1`

`1 × 3 - 1 = 2` ✓ (insert `a` before and `c` after)

**`word = "cb"`**:

| i | word[i] | word[i-1] | new group? | groups |
|---|---------|-----------|------------|--------|
| 1 | b | c | ✅ (b ≤ c) | 2 |

`2 × 3 - 2 = 4` ✓ → `"abc"` + `"abc"`, `c` fits in group 1, `b` fits in group 2

---

## Pitfalls & Edge Cases

- **`word[i] == word[i-1]`** — equal chars also trigger a new group (e.g. `"aa"` → can't have two `a`s in one `"abc"`)
- **`word = "abc"`** — no resets → 1 group → `3 - 3 = 0` ✓
- **Single character** — always 1 group → `3 - 1 = 2` ✓
- **Already valid** — any concatenation of `"abc"` → each `"abc"` boundary triggers a reset → correct group count

---

## Key Takeaway

"Minimum insertions to form repeated pattern" → **count pattern groups greedily**:

- New group starts when the sequence can't continue (current ≤ previous)
- Answer = `groups × patternLength - existingLength`

No DP needed — the greedy boundary detection is optimal because each character must belong to exactly one group, and we always assign it to the current group if possible (never beneficial to skip ahead).
