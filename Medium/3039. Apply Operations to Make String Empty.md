# 3039. Apply Operations to Make String Empty
## O(n) — Max Frequency Characters, Ordered by Last Occurrence

---

## Problem Statement

- Each operation: for every letter a-z, remove its **first** occurrence (if present).
- Return the string just before the **last** operation (the one that makes it empty).

---

## Core Idea — Which Characters Survive to the Last Round?

**Key insight**: the number of operations equals `max_freq` = maximum frequency of any character. A character `c` that appears `f` times survives until round `f` (it gets removed once per round). So the characters present in the last round are **exactly those with frequency = max_freq**.

**What remains just before the last operation**: the last occurrence of each max-frequency character (since each round removes the first occurrence, after `max_freq - 1` rounds only the last occurrence of each such character remains).

**Order of characters in the result**: they appear in the order of their **last occurrence** in `s` (leftmost last-occurrence first).

**Why last occurrence order?** After removing all but the final occurrence of each character, the remaining string preserves the relative order of those last occurrences as they appear in the original string.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string LastNonEmptyString(string s) {
        int[] freq = new int[26];
        foreach (char c in s) freq[c - 'a']++;
        int maxFreq = freq.Max();

        // Track last occurrence position for max-freq characters
        int[] lastPos = new int[26];
        Array.Fill(lastPos, -1);
        for (int i = 0; i < s.Length; i++)
            if (freq[s[i] - 'a'] == maxFreq)
                lastPos[s[i] - 'a'] = i;

        // Collect max-freq chars, sort by their last occurrence position
        var chars = Enumerable.Range(0, 26)
            .Where(i => lastPos[i] >= 0)
            .OrderBy(i => lastPos[i])
            .Select(i => (char)('a' + i));

        return new string(chars.ToArray());
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n + 26 log 26)** = **O(n)** | One pass for freq, one pass for last positions |
| **Space** | **O(26)** = **O(1)** | Fixed-size arrays |

## Example Walkthrough

**`s = "aabcbbca"`**

Frequencies: a=3, b=3, c=2. max_freq=3.

Max-freq characters: {a, b}.

Last occurrence of 'a' in `s`: index 6 (`s[6]='a'`).
Last occurrence of 'b' in `s`: index 5 (`s[5]='b'`).

Sorted by last occurrence: b(5), a(6) → **"ba"** ✓

---

## Why This Works — Simulation Trace

`s = "aabcbbca"` (indices 0-7: a,a,b,c,b,b,c,a)

- **Round 1**: remove first a(0), b(2), c(3) → remaining: a(1),b(4),b(5),c(6),a(7) → `"abbca"`
- **Round 2**: remove first a(1), b(4), c(6) → remaining: b(5),a(7) → `"ba"`
- **Round 3** (last): would remove b(5), a(7) → `""`

String just before last operation = **"ba"** ✓.

These are exactly: last occurrence of 'b' (index 5) and last occurrence of 'a' (index 7), in order.

---

## Pitfalls & Edge Cases

- **All characters appear once** (e.g., "abcd"): max_freq=1, all characters are max-freq, their last occurrence IS their only occurrence → result = all characters in original order ✓
- **One character dominates** (e.g., "aaaa"): only 'a' with max_freq=4. Last occurrence = last index. Result = "a".
- **Multiple characters tied at max_freq**: all included, ordered by their last occurrence in s.

---

## Key Takeaway

"String before last operation" → **characters with max frequency, ordered by last occurrence**:

```
max_freq = max(freq[c] for all c)
result = chars with freq == max_freq, sorted by last occurrence in s
```

Each operation removes one copy (front-to-back), so after `max_freq - 1` operations only the last copy of each max-freq character remains.
