# 3387. Maximize Amount After Two Days of Conversions
## O((n+m) × currencies) — DFS for Max Rates + Two-Day Optimization

---

## Problem Statement

- Start with 1.0 of `initialCurrency`. Day 1: convert using `pairs1/rates1`. Day 2: convert using `pairs2/rates2`. Maximize final amount of `initialCurrency`.

---

## Core Idea — Two-Phase DFS

**Phase 1**: From `initialCurrency` with 1.0, find the maximum amount of every reachable currency after day 1. Use DFS/BFS propagating the best (maximum) amount.

**Phase 2**: For each intermediate currency `c` with amount `day1[c]`, find the maximum `initialCurrency` obtainable on day 2 starting from `c`. Multiply `day1[c]` by the day-2 conversion rate back to `initialCurrency`.

**Answer**: max over all intermediate currencies, also comparing to 1.0 (no conversions).

Since the graphs are acyclic (no contradictions per constraints), simple DFS works without Bellman-Ford.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public double MaxAmount(string init, IList<IList<string>> pairs1, double[] rates1,
                                        IList<IList<string>> pairs2, double[] rates2) {
        var day1 = BestRates(pairs1, rates1, init);
        double res = 1.0;

        foreach (var (mid, amt) in day1) {
            var day2 = BestRates(pairs2, rates2, mid);
            if (day2.ContainsKey(init))
                res = Math.Max(res, amt * day2[init]);
        }
        return res;
    }

    Dictionary<string,double> BestRates(IList<IList<string>> pairs, double[] rates, string start) {
        var graph = new Dictionary<string, List<(string, double)>>();
        for (int i = 0; i < pairs.Count; i++) {
            string a = pairs[i][0], b = pairs[i][1];
            if (!graph.ContainsKey(a)) graph[a] = new();
            if (!graph.ContainsKey(b)) graph[b] = new();
            graph[a].Add((b, rates[i]));
            graph[b].Add((a, 1.0/rates[i]));
        }

        var best = new Dictionary<string,double> { [start] = 1.0 };
        var stack = new Stack<string>();
        stack.Push(start);
        while (stack.Count > 0) {
            string curr = stack.Pop();
            if (!graph.ContainsKey(curr)) continue;
            foreach (var (nxt, rate) in graph[curr]) {
                double newAmt = best[curr] * rate;
                if (!best.ContainsKey(nxt) || newAmt > best[nxt]) {
                    best[nxt] = newAmt;
                    stack.Push(nxt);
                }
            }
        }
        return best;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(currencies × (n+m))** | DFS per day + once per intermediate currency |
| **Space** | **O(currencies)** | At most ~20 distinct currencies |

With n,m ≤ 10: trivially fast.

---

## Example Walkthrough

**EUR → (day1) → EUR:1, USD:2, JPY:6 → (day2 from JPY) → JPY:1, USD:4, CHF:20, EUR:120**

Final: `day1[JPY] × day2fromJPY[EUR]` = 6 × 120 = **720** ✓

---

## Pitfalls & Edge Cases

- **Bidirectional edges**: `a→b` at rate `r` implies `b→a` at rate `1/r`.
- **No conversions needed**: always compare with 1.0 (keep initial currency both days).
- **Acyclic graph guaranteed**: DFS with best-value update is sufficient (no Bellman-Ford needed).
- **Intermediate currency not in day2 graph**: skip those (can't convert back).

---

## Key Takeaway

"Max currency after two independent conversion days" → **DFS best-rates from start (day1), then DFS best-rates from each intermediate to find max return (day2)**:

```
day1 = max_amounts_from(initialCurrency, day1_graph)
for mid, amt in day1:
    day2 = max_amounts_from(mid, day2_graph)
    result = max(result, amt × day2[initialCurrency])
```

Small constraints → O(currencies²) outer loop is fine.
