# 3867. Sum of GCD of Formed Pairs
## O(n log n) — Build prefixGcd, Sort, Two-Pointer Pair GCDs

---

## Problem Statement

- Build `prefixGcd[i] = gcd(nums[i], max(nums[0..i]))`.
- Sort `prefixGcd`, pair smallest with largest, second smallest with second largest, etc.
- If n is odd, the middle element is unpaired.
- Return the sum of GCDs of all pairs.

---

## Core Idea — Simulate Exactly As Described

This problem requires no clever insight — follow the spec directly:

1. **Build prefixGcd:** maintain running max as we scan left to right.
2. **Sort** prefixGcd.
3. **Two pointers** (lo=0, hi=n−1): pair and compute GCD, advance both inward.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long GcdSum(int[] nums) {
        int n = nums.Length;
        int[] prefixGcd = new int[n];
        int maxSoFar = 0;

        for (int i = 0; i < n; i++) {
            maxSoFar = Math.Max(maxSoFar, nums[i]);
            prefixGcd[i] = GCD(nums[i], maxSoFar);
        }

        Array.Sort(prefixGcd);

        long res = 0;
        int lo = 0, hi = n - 1;

        while (lo < hi) {
            res += GCD(prefixGcd[lo], prefixGcd[hi]);
            lo++;
            hi--;
        }

        return res;
    }

    private int GCD(int a, int b) {
        while (b != 0) { int t = b; b = a % b; a = t; }
        return a;
    }
}
```

## Example Walkthrough

**Example 1:** `nums = [2, 6, 4]`

| i | nums[i] | maxSoFar | prefixGcd[i] |
|---|---------|----------|--------------|
| 0 | 2       | 2        | gcd(2,2)=2   |
| 1 | 6       | 6        | gcd(6,6)=6   |
| 2 | 4       | 6        | gcd(4,6)=2   |

prefixGcd = [2,6,2] → sorted: [2,2,6]

Two-pointer: lo=0 (val=2), hi=2 (val=6) → gcd(2,6) = **2**. lo=1, hi=1: stop (lo = hi, odd n).

**Output:** 2 ✓

---

**Example 2:** `nums = [3, 6, 2, 8]`

| i | nums[i] | maxSoFar | prefixGcd[i] |
|---|---------|----------|--------------|
| 0 | 3       | 3        | gcd(3,3)=3   |
| 1 | 6       | 6        | gcd(6,6)=6   |
| 2 | 2       | 6        | gcd(2,6)=2   |
| 3 | 8       | 8        | gcd(8,8)=8   |

sorted: [2,3,6,8]

Pairs: gcd(2,8)=**2**, gcd(3,6)=**3**. Sum = 5.

**Output:** 5 ✓

## Pitfalls & Edge Cases

- **`prefixGcd[i]` always divides `maxSoFar`** — since `gcd(nums[i], max)` always divides `max`. Common GCD values are factors of running maxima.

- **Running max in prefixGcd:** `gcd(nums[i], maxSoFar)` where maxSoFar includes nums[i]. Since `gcd(nums[i], nums[i]) = nums[i]` when nums[i] is the new max, new maxima always contribute their own value.

- **Odd n:** two-pointer terminates when `lo == hi`, skipping the middle element automatically. ✓

- **Overflow:** GCD values ≤ 10⁹, sum ≤ n/2 × 10⁹ ≤ 5×10¹³ → use `long`. ✓

## Key Takeaway

Sometimes the problem description is the algorithm. Three steps, each O(n) or O(n log n):

1. **prefixGcd:** running max + GCD per element
2. **Sort:** standard
3. **Pair GCDs:** two-pointer sum

**Pattern:** "Build array, sort, pair endpoints" → construct + sort + two-pointer. No insight needed beyond careful reading.
