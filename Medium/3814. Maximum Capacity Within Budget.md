# 3814. Maximum Capacity Within Budget
## O(n log n) — Sort by Cost + Prefix Max Capacity + Binary Search

---

## Problem Statement

- Given n machines with `costs[i]` and `capacity[i]`.
- Select **at most 2 distinct** machines with total cost **strictly less than** `budget`.
- Return the maximum total capacity achievable.

---

## Core Idea — Fix the "Right" Machine, Find the Best "Left" Machine

Naive two-pointer fails here because capacity isn't monotone with cost — advancing `lo` when a pair is valid can miss a better-capacity left machine.

**Correct approach:**
1. Sort machines by cost.
2. Precompute `prefMaxCap[i]` = max capacity among machines[0..i] (best left partner up to index i).
3. For each machine j (as the "right" / more expensive machine):
   - Maximum valid "left" partner: binary search for largest lo < j with `costs[lo] < budget − costs[j]`.
   - Best pair capacity = `prefMaxCap[lo_max] + capacity[j]`.
4. Also track the best single machine (cost < budget).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxCapacity(int[] costs, int[] capacity, int budget) {
        int n = costs.Length;

        // Pair up and sort by cost
        var machines = new (int cost, int cap)[n];
        for (int i = 0; i < n; i++) machines[i] = (costs[i], capacity[i]);
        Array.Sort(machines, (a, b) => a.cost.CompareTo(b.cost));

        // prefMaxCap[i] = max capacity among machines[0..i]
        int[] prefMaxCap = new int[n];
        prefMaxCap[0] = machines[0].cap;
        for (int i = 1; i < n; i++) {
            prefMaxCap[i] = Math.Max(prefMaxCap[i - 1], machines[i].cap);
        }

        int res = 0;

        for (int j = 0; j < n; j++) {
            if (machines[j].cost >= budget) break; // all subsequent also >= budget

            // Single machine j
            res = Math.Max(res, machines[j].cap);

            if (j == 0) continue; // no left partner possible

            // Binary search: largest lo in [0, j-1] with machines[lo].cost < need
            int need = budget - machines[j].cost;
            int lo = 0, hi = j - 1, loMax = -1;
            while (lo <= hi) {
                int mid = (lo + hi) / 2;
                if (machines[mid].cost < need) { loMax = mid; lo = mid + 1; }
                else hi = mid - 1;
            }

            if (loMax >= 0) {
                res = Math.Max(res, prefMaxCap[loMax] + machines[j].cap);
            }
        }

        return res;
    }
}
```

## Why Naive Two-Pointer Fails

Consider: `costs = [1,2,3]`, `caps = [5,8,4]`, `budget = 5`.

Sorted: (1,5),(2,8),(3,4). Two-pointer: lo=0, hi=2: costs 1+3=4<5 valid → record 5+4=9, lo++. lo=1, hi=2: costs 2+3=5 not < 5, hi--. Done. Returns 9.

**But pair (0,1): costs 1+2=3<5, caps 5+8=13 is better!** The naive two-pointer misses it because advancing lo past it when hi=2 was valid.

The prefix max approach correctly finds: j=1, need=5−2=3, lo_max=0 (cost 1<3). Pair = prefMaxCap[0]+8 = 5+8 = **13**. ✓

## Pitfalls & Edge Cases

- **Naive two-pointer is wrong** — advancing `lo` on a valid pair can skip the best left partner. Always use binary search + prefix max. ✓

- **Strictly less than budget** — binary search condition is `costs[lo] < need` (strict), where `need = budget − costs[j]`, correctly encoding `costs[lo] + costs[j] < budget`. ✓

- **Same cost machines** — multiple machines at the same cost are handled naturally; binary search finds the last valid one, and prefMaxCap tracks the maximum capacity up to that index. ✓

- **All machines cost ≥ budget** — the loop breaks at j=0 and res stays 0. ✓

- **No valid pair, only single** — if no loMax is found for any j, the result comes from the single machine maximum. ✓

---

## Key Takeaway

"Find the best pair satisfying a constraint" doesn't always yield to two pointers. When the objective function (capacity sum) is not monotone with the sorting key (cost), use:

**Sort → Prefix Max → Binary Search per element**

For each "right" element j, binary search finds the valid range of "left" elements, and the prefix max gives the best one in O(1).

**Pattern recognition:** "Select 1 or 2 items from a sorted array, maximize non-monotone value, cost < budget" → sort by cost, prefix max on value, binary search per item.
