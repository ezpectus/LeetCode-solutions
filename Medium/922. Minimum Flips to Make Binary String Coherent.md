# 3922. Minimum Flips to Make Binary String Coherent
## O(n) — Classify All Coherent Strings, Compute Min Hamming Distance

---

## Problem Statement

- A string is "coherent" if it contains neither `"011"` nor `"110"` as a subsequence.
- Return the minimum number of single-character flips to make `s` coherent.

---

## Core Idea — Characterize the Structure of Every Coherent String

Let `T` = total number of `'1'`s in the string.

**"011" forbidden** ⟺ for every `'0'` at position i, the number of `'1'`s **after** i is ≤ 1.
**"110" forbidden** ⟺ for every `'0'` at position i, the number of `'1'`s **before** i is ≤ 1.

For a `'0'` at position i, let `a` = ones before i, `b` = ones after i. Then `a + b = T`. Both conditions require `a ≤ 1` and `b ≤ 1`, so `T = a+b ≤ 2` whenever a `'0'` exists.

**This gives a full classification of coherent strings:**

| T (count of 1's) | Constraint | Coherent strings |
|---|---|---|
| T = 0 | — | All zeros |
| T = n | — | All ones |
| T = 1 | a≤1,b≤1 auto-satisfied | One '1' anywhere, rest 0's |
| T = 2 | every 0 needs a=1,b=1 | Both 1's at positions exactly 0 and n−1; everything between is 0 |
| T ≥ 3, has a 0 | T=a+b≤2 violated | **Never coherent** |

So every coherent string falls into one of 4 patterns:
1. All `'0'`s
2. All `'1'`s
3. Exactly one `'1'`, at any position
4. Exactly two `'1'`s, at positions 0 and n−1 (everything else `'0'`)

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinFlips(string s) {
        int n = s.Length;
        int count1 = 0;
        foreach (char c in s) if (c == '1') count1++;
        int count0 = n - count1;

        // Pattern 1: all zeros
        int cost1 = count1;

        // Pattern 2: all ones
        int cost2 = count0;

        // Pattern 3: exactly one '1' (placed where s already has a '1' if possible)
        int cost3 = (count1 == 0) ? 1 : (count1 - 1);

        // Pattern 4: '1' at position 0 and n-1, rest '0' (needs n >= 2)
        int cost4 = int.MaxValue;
        if (n >= 2) {
            int end1Count = (s[0] == '1' ? 1 : 0) + (s[n-1] == '1' ? 1 : 0);
            cost4 = count1 + 2 - 2 * end1Count;
        }

        return Math.Min(Math.Min(cost1, cost2), Math.Min(cost3, cost4));
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n) | Single pass to count 1's |
| Space Complexity | O(1) | |

---

## Derivation of Each Cost Formula

**Pattern 3 (one '1', rest '0'):** Hamming distance to target `e_j` (1 at position j) = `count1 - [s[j]='1'] + [s[j]='0']`.
- If `count1 ≥ 1`, choose j where `s[j]='1'` → cost = `count1 - 1 + 0 = count1 - 1`.
- If `count1 = 0`, every j has `s[j]='0'` → cost = `0 + 1 = 1`.

**Pattern 4 ('1' at both ends, rest '0'):**
- Let `end1Count` = number of `{s[0], s[n-1]}` that are already `'1'` (0, 1, or 2).
- Middle 1's (positions 1..n-2) that must flip to 0: `count1 - end1Count`.
- Endpoints that must flip to 1: `2 - end1Count`.
- Total: `cost4 = (count1 - end1Count) + (2 - end1Count) = count1 + 2 - 2·end1Count`.

## Example Walkthrough

**Example 1:** `s = "1010"`, n=4, count1=2, count0=2.

- cost1 (all 0) = 2
- cost2 (all 1) = 2
- cost3 = count1−1 = 1
- end1Count = [s[0]='1']+[s[3]='1'] = 1+0 = 1; cost4 = 2+2−2 = 2

min(2,2,1,2) = **1** ✓

**Example 2:** `s = "0110"`, n=4, count1=2.

- cost1=2, cost2=2, cost3=1
- end1Count = [s[0]='0']+[s[3]='0'] → both 0 → end1Count=0; cost4 = 2+2−0 = 4

min(2,2,1,4) = **1** ✓

**Example 3:** `s = "1000"`, n=4, count1=1.

- cost1=1, cost2=3, cost3=count1−1=0
- end1Count = [s[0]='1']+[s[3]='1'] = 1+0=1; cost4 = 1+2−2 = 1

min(1,3,0,1) = **0** ✓

---

## Pitfalls & Edge Cases

- **n = 1:** Pattern 4 doesn't apply (positions 0 and n−1 coincide). Skip it (set to `int.MaxValue`). Patterns 1-3 always cover single-character strings: "0" → cost1=0; "1" → cost2=0. ✓

- **count1 = 0 (all zeros already):** cost1=0, dominates. ✓

- **count1 = n (all ones already):** cost2=0, dominates. ✓

- **Overlap between patterns is fine** — e.g., "11" satisfies both pattern 2 (all ones) and pattern 4 (ones at both ends, 0 zeros between). Taking the min over all formulas naturally handles overlaps. ✓

---

## Key Takeaway

The key step is **fully characterizing the target structure** via the constraint `a + b = T ≤ 2` for any zero's neighbor-counts. Once you know there are only 4 possible "shapes" for a coherent string, the problem reduces to computing 4 simple Hamming-distance formulas and taking the minimum — O(n).

**Pattern recognition:** "Minimum edits to avoid certain subsequences" → first characterize ALL strings avoiding those subsequences (often a small finite family of patterns), then minimize Hamming distance to each pattern.
