# 3844. Longest Almost-Palindromic Substring
## O(n²) — Precomputed Palindrome DP + Outer-Match Depth Table

---

## Problem Statement

- A substring is almost-palindromic if removing exactly one character makes it a palindrome.
- Return the length of the longest almost-palindromic substring of `s` (2 ≤ n ≤ 2500).

---

## Core Idea — Two Precomputed Tables Enable O(1) Per-Pair Check

**Key observation:** For any substring s[l..r], use two pointers from both ends. If at some depth x the pointers first mismatch (at positions lp = l+x, rp = r−x), then s[l..r] is almost-palindromic iff `isPalin[lp+1][rp]` or `isPalin[lp][rp−1]`.

Precompute two tables:

1. **`isPalin[i][j]`** — standard palindrome DP: `isPalin[i][j] = (s[i]==s[j]) && isPalin[i+1][j-1]`.

2. **`inner[i][j]`** — number of outer pairs that match before the first mismatch:
   ```
   inner[i][j] = s[i]==s[j] ? 1 + inner[i+1][j-1] : 0
   ```

With both tables filled in O(n²), each substring pair (l, r) is checked in O(1).

**Three cases for s[l..r]:**
- `lp > rp` (even palindrome) or `lp == rp` (odd palindrome): almost-palindromic ✓ (any palindrome of length ≥ 2 qualifies).
- `lp < rp` (mismatch): check `isPalin[lp+1][rp]` or `isPalin[lp][rp−1]`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int AlmostPalindromic(string s) {
        int n = s.Length;

        // isPalin[i,j] = is s[i..j] a palindrome?
        bool[,] isPalin = new bool[n, n];
        for (int i = 0; i < n; i++) isPalin[i, i] = true;
        for (int i = 0; i + 1 < n; i++) isPalin[i, i+1] = s[i] == s[i+1];
        for (int len = 3; len <= n; len++) {
            for (int i = 0; i + len - 1 < n; i++) {
                int j = i + len - 1;
                isPalin[i, j] = s[i] == s[j] && isPalin[i+1, j-1];
            }
        }

        // inner[i,j] = number of matching outer pairs starting from (i,j)
        int[,] inner = new int[n, n];
        for (int len = 2; len <= n; len++) {
            for (int i = 0; i + len - 1 < n; i++) {
                int j = i + len - 1;
                inner[i, j] = s[i] == s[j] ? 1 + (len > 2 ? inner[i+1, j-1] : 0) : 0;
            }
        }

        int res = 2; // all length-2 substrings are always almost-palindromic
        for (int l = 0; l < n; l++) {
            for (int r = l + 1; r < n; r++) {
                int x  = inner[l, r];
                int lp = l + x, rp = r - x;

                if (lp >= rp) {
                    // Palindrome (all pairs matched or single inner char)
                    res = Math.Max(res, r - l + 1);
                } else {
                    // Mismatch at (lp, rp): try removing lp or rp
                    if (isPalin[lp+1, rp] || isPalin[lp, rp-1])
                        res = Math.Max(res, r - l + 1);
                }
            }
        }

        return res;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n²) | Two precomputation passes + O(1) per pair |
| Space Complexity | O(n²) | Two 2D tables |

For n=2500: ~6.25M entries per table, ~37 MB total. Well within limits.

---

## Why Every Length-2 Substring Is Almost-Palindromic

For any s[i..i+1]: remove either character → single character (palindrome by definition). ✓ So the answer is always ≥ 2.

---

## Example Walkthroughs

**Example 1:** `s = "abca"`

`inner[0][3]`: s[0]='a'==s[3]='a' → 1 + inner[1][2]. s[1]='b'!=s[2]='c' → inner[1][2]=0. So inner[0][3]=1.

For (l=0, r=3): x=1, lp=1, rp=2. Mismatch.
- `isPalin[2][2]` = true (single char). ✓ → length **4** is almost-palindromic.

**Output:** 4 ✓ (remove 'c', get "aba")

---

**Example 2:** `s = "abba"`

`inner[0][3]`: a==a → 1+inner[1][2]. b==b → 1+inner[2][1]=1+0=1. So inner[0][3]=2.

For (l=0, r=3): x=2, lp=2, rp=1. lp > rp → palindrome. ✓ → length **4**.

**Output:** 4 ✓ (remove one 'b', get "aba")

---

**Example 3:** `s = "zzabba"`

For (l=1, r=5) = "zabba": inner[1][5]=0 (z≠a). lp=1, rp=5.
- `isPalin[2][5]`: s[2..5]="abba" → isPalin[2][5] = (a==a) && isPalin[3][4]=(b==b)=T → **true** ✓. Length **5**.

For (l=0, r=5) = "zzabba": inner[0][5]=0 (z≠a). lp=0, rp=5.
- `isPalin[1][5]`: s[1..5]="zabba" → z≠a → false.
- `isPalin[0][4]`: s[0..4]="zzabb" → z≠b → false.
- Not almost-palindromic.

**Output:** 5 ✓

## Pitfalls & Edge Cases

- **`inner` base case:** `inner[i][j] = 0` when `i >= j` (C# zeroes int arrays). The `len > 2` guard prevents accessing `inner[i+1][j-1]` when `j = i+1`. ✓

- **Accessing `isPalin[lp+1][rp]` when `lp+1 == rp`:** single char → always true. ✓

- **Palindromes are almost-palindromic:** for any palindrome of length ≥ 2, you can always remove one char and get a palindrome (remove center for odd, remove one of innermost pair for even). The `lp >= rp` branch handles this. ✓

- **Why naive O(n³) fails:** iterating all (l,r) pairs with the two-pointer check is O(n) per pair = O(n³) total. The `inner` table converts the "find first mismatch depth" step to O(1). ✓

---

## Key Takeaway

"Almost-palindromic" = "palindrome with one allowable mismatch." The two-pointer check has exactly one mismatch point; precomputing its depth via `inner[i][j]` converts the check to O(1) per pair.

**Pattern recognition:** Any "palindrome with at most one exception" problem → precompute both the palindrome DP and the "outer match depth" DP, then check the two candidates at the mismatch in O(1).
