# 2730. Find the Longest Semi-Repetitive Substring
## O(n) — Sliding Window with At-Most-One Pair

---

## Problem Statement

- A string is **semi-repetitive** if it has **at most one** adjacent pair of equal digits.
- Return the length of the longest semi-repetitive substring.

---

## Core Idea — Sliding Window, Shrink on Second Pair

Maintain a window `[left, right]`. Track `pairs` = number of adjacent duplicate pairs in the current window, and `lastPair` = the starting index of the most recent pair found.

When `pairs` reaches 2 (second duplicate found), the window is no longer valid. Shrink `left` to `lastPair + 1` — this removes exactly the previous pair, leaving `pairs = 1`. Update `lastPair` to the new pair's position.

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int longestSemiRepetitiveSubstring(string s) {
        int n = s.size();
        int left = 0, pairs = 0, lastPair = 0, res = 1;

        for (int right = 1; right < n; right++) {
            if (s[right] == s[right-1]) {
                pairs++;
                if (pairs == 2) {
                    left = lastPair + 1; // remove previous pair from window
                    pairs = 1;
                }
                lastPair = right - 1;
            }
            res = max(res, right - left + 1);
        }

        return res;
    }
};
```

## Example Walkthrough

**`s = "52233"`**

| right | s[r] | s[r-1] | pairs | left | lastPair | window | len |
|-------|------|--------|-------|------|----------|--------|-----|
| 1 | 2 | 5 | 0 | 0 | 0 | "52" | 2 |
| 2 | 2 | 2 | 1 | 0 | 1 | "522" | 3 |
| 3 | 3 | 2 | 1 | 0 | 1 | "5223" | **4** |
| 4 | 3 | 3 | 2→1 | 2 | 3 | "233" | 3 |

Max = **4** ✓ (`"5223"`)

---

## Pitfalls & Edge Cases

- **`lastPair` tracks the start index of the pair** — when pair at `(right-1, right)` is found, `lastPair = right - 1`; shrink to `lastPair + 1` removes that pair from the window
- **`pairs = 1` after shrink** — the new pair (just found) is still in window; don't reset to 0
- **Single character** — `res` initialised to 1; loop starts at `right = 1` ✓
- **All same digits `"1111"`** — each step finds a pair; on second pair shrinks left by 1 → window always size 2 ✓

---

## Key Takeaway

"Longest substring with at most one duplicate adjacent pair" → **sliding window with `lastPair` pointer**:

- Track position of the last pair
- On second pair: jump `left` past the previous pair, keep `pairs = 1`
- No need to recount pairs from scratch — `lastPair` gives the exact jump target
