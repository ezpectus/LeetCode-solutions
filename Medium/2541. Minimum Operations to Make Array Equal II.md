# 2541. Minimum Operations to Make Array Equal II  
*O(n) — Greedy Difference Analysis*

---

## Problem Statement

You are given two integer arrays `nums1` and `nums2` of equal length `n`, and an integer `k`.

You can perform the following operation on `nums1` any number of times:

- Choose two indices `i` and `j`
- Increment `nums1[i]` by `k`
- Decrement `nums1[j]` by `k`  
  (i.e., transfer `k` from `j` to `i`)

Return the **minimum number of operations** needed to make `nums1` equal to `nums2`.  
If impossible, return **-1**.

**Examples**:

**Example 1**  
Input: nums1 = [4,3,1,4], nums2 = [1,3,7,1], k = 3  
Output: **2**  
Operations:  
1. i=2, j=0: nums1 becomes [1,3,4,4]  
2. i=2, j=3: nums1 becomes [1,3,7,1]

**Example 2**  
Input: nums1 = [3,8,5,2], nums2 = [2,4,1,6], k = 1  
Output: **-1**  
Impossible

**Constraints**:
- n == nums1.length == nums2.length
- 2 ≤ n ≤ 10⁵
- 0 ≤ nums1[i], nums2[j] ≤ 10⁹
- 0 ≤ k ≤ 10⁵

---

## Core Idea — Difference Analysis + Greedy

**Key observations**:
- Each operation **transfers** exactly `k` units from one position to another
- The **total sum** of `nums1` must equal the total sum of `nums2` (operations preserve sum)
- If sum(nums1) ≠ sum(nums2) → impossible → -1
- For each position i, define diff[i] = nums2[i] - nums1[i]
  - If diff[i] > 0 → we need to **add** diff[i] to position i
  - If diff[i] < 0 → we need to **remove** |diff[i]| from position i
- Since we transfer in multiples of k, diff[i] must be divisible by k for every i
- Positive diffs must be supplied by negative diffs
- Total added = total removed (sum of positives = -sum of negatives)
- Number of operations = total transferred / k = sum of all positive diffs / k

**Algorithm**:
1. Compute total sum of nums1 and nums2
2. If sums differ → return -1
3. For each i: compute diff[i] = nums2[i] - nums1[i]
4. If any diff[i] % k != 0 → impossible → -1
5. Compute total positive diff (sum of max(0, diff[i]))
6. Return total_positive / k

**Why it works**:
- Every transfer moves exactly k units
- All diff[i] must be multiples of k (otherwise impossible)
- Total amount to move = sum of positive diffs (or absolute sum of negative diffs)
- Operations needed = total amount to move / k

## Clean Implementation (C#)

```csharp
public class Solution {
    public long MinOperations(int[] nums1, int[] nums2, int k) {
        if (k == 0) {
            // If k=0, arrays must already be equal
            for (int i = 0; i < nums1.Length; i++) {
                if (nums1[i] != nums2[i]) return -1;
            }
            return 0;
        }

        long totalDiff = 0;
        long positiveSum = 0;

        for (int i = 0; i < nums1.Length; i++) {
            long diff = (long)nums2[i] - nums1[i];

            if (diff % k != 0) return -1;

            totalDiff += diff;

            if (diff > 0) {
                positiveSum += diff;
            }
        }

        // Total sum must remain the same
        if (totalDiff != 0) return -1;

        // Minimum operations = total amount to move / k
        return positiveSum / k;
    }
}
```

## Complexity

| **Metric**            | **Value**     | **Notes**                                      |
|-----------------------|---------------|------------------------------------------------|
| **Time Complexity**   | **O(n)**      | Single linear pass over both arrays to compute sums and differences |
| **Space Complexity**  | **O(1)**      | Only a constant number of variables (sums and counters) — no extra arrays |

**Optimal** — linear time and constant space — perfect and highly efficient for n ≤ 10⁵.

## Why This Works — Example Walkthrough

**Example 1**: nums1 = [4,3,1,4], nums2 = [1,3,7,1], k = 3

**Differences**:  
diff = [1-4, 3-3, 7-1, 1-4] = [-3, 0, 6, -3]

- All differences divisible by 3 → OK
- Total sum change = -3 + 0 + 6 - 3 = **0** → sums preserved → OK
- Positive differences sum = 6 (need to add 6 units in total)
- Minimum operations = 6 / 3 = **2** → correct

**Example 2**: nums1 = [3,8,5,2], nums2 = [2,4,1,6], k = 1

**Differences**:  
diff = [2-3, 4-8, 1-5, 6-2] = [-1, -4, -4, 4]

- All divisible by 1 → OK
- Total sum change = -1 -4 -4 + 4 = **-5** ≠ 0 → sums not preserved → **-1** → correct

**Correct** —  
Each operation transfers **exactly k units** from one position to another, preserving the **total sum** of the array.  
Therefore, the sums of `nums1` and `nums2` **must be equal** initially.  
Every difference `nums2[i] - nums1[i]` must be a **multiple of k** (otherwise impossible to reach).  
The total amount that needs to be transferred (sum of all positive differences) must be divisible by k.  
The **minimum number of operations** is exactly that total amount divided by k.

---

## Key Takeaway

This is a **beautiful greedy difference analysis** problem:

* First check: sum(nums1) must equal sum(nums2) — operations preserve the total sum
* For every position i: diff[i] = nums2[i] - nums1[i] **must be divisible by k**
* If any diff is not divisible by k → impossible → return -1
* Compute total amount to move = sum of all positive diff[i] (or absolute sum of negative diff[i])
* Minimum operations = total_positive / k
* If sums unequal or any diff not divisible → return -1

**Pure, clean, optimal** — O(n) time, O(1) space, extremely fast, very intuitive, and mathematically precise.
