# 3847. Find the Score Difference in a Game
## O(n) — Direct Simulation of Swap Rules

---

## Problem Statement

- Two players; player 1 starts active.
- For each game i, apply rules in order:
  1. If `nums[i]` is odd → swap active/inactive.
  2. If i is a 6th-game index (i = 5, 11, 17, …) → swap active/inactive.
  3. Active player gains `nums[i]` points.
- Return player 1's total minus player 2's total.

---

## Core Idea — Simulate Step by Step

Track `active` (0 = player 1, 1 = player 2). Apply both swap conditions before awarding points.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int ScoreDifference(int[] nums) {
        int[] score = new int[2]; // score[0] = player 1, score[1] = player 2
        int active = 0;           // 0 = player 1 is active

        for (int i = 0; i < nums.Length; i++) {
            // Rule 1: odd value → swap
            if (nums[i] % 2 == 1) active = 1 - active;

            // Rule 2: every 6th game (i = 5, 11, 17, ...) → swap
            if ((i + 1) % 6 == 0) active = 1 - active;

            // Rule 3: active player gains points
            score[active] += nums[i];
        }

        return score[0] - score[1];
    }
}
```

## Example Walkthrough

**Example 2:** `nums = [2, 4, 2, 1, 2, 1]`

| i | nums[i] | Odd swap? | 6th-game swap? | active | scores (p1, p2) |
|---|---------|-----------|----------------|--------|-----------------|
| 0 | 2       | No        | No             | 0(p1)  | (2, 0)          |
| 1 | 4       | No        | No             | 0(p1)  | (6, 0)          |
| 2 | 2       | No        | No             | 0(p1)  | (8, 0)          |
| 3 | 1       | **Yes**→p2 | No             | 1(p2)  | (8, 1)          |
| 4 | 2       | No        | No             | 1(p2)  | (8, 3)          |
| 5 | 1       | **Yes**→p1 | **Yes**→p2   | 1(p2)  | (8, 4)          |

Result: 8 − 4 = **4** ✓

At game 5: odd swap first (→ p1), then 6th-game swap (→ p2). Player 2 gains the point.

---

## Pitfalls & Edge Cases

- **Order matters:** both swaps happen BEFORE scoring. If both swap at the same game, they cancel out (net effect = no change from start of that game's swaps).

- **6th-game check:** `(i+1) % 6 == 0` correctly identifies i=5, 11, 17, … ✓

- **Both swaps in the same game** (game 5 with odd value): two swaps = return to original active player. Example 2 game 5 illustrates this.

---

## Key Takeaway

Pure simulation with two conditional swaps per game, then score assignment. Read order carefully: swap rules apply first, scoring second.
