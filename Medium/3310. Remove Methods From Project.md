# 3310. Remove Methods From Project
## O(n + e) — BFS for Suspicious Set + External Edge Check

---

## Problem Statement

- Method `k` is buggy. All methods reachable from `k` (directly or indirectly) are "suspicious". Remove them only if no non-suspicious method invokes any suspicious one.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public IList<int> RemainingMethods(int n, int k, int[][] invocations) {
        var graph = new List<int>[n];
        for (int i = 0; i < n; i++) graph[i] = new();
        foreach (var e in invocations) graph[e[0]].Add(e[1]);

        // BFS: find all methods reachable from k
        bool[] suspicious = new bool[n];
        var q = new Queue<int>();
        suspicious[k] = true; q.Enqueue(k);
        while (q.Count > 0) {
            int u = q.Dequeue();
            foreach (int v in graph[u])
                if (!suspicious[v]) { suspicious[v] = true; q.Enqueue(v); }
        }

        // If any external method invokes a suspicious one → can't remove
        foreach (var e in invocations)
            if (!suspicious[e[0]] && suspicious[e[1]])
                return Enumerable.Range(0, n).ToList();

        return Enumerable.Range(0, n).Where(i => !suspicious[i]).ToList();
    }
}
```

**Time**: O(n + e) — **Space**: O(n + e)

---

## Key Takeaway

BFS finds suspicious set. One scan of all edges checks for external → suspicious edges. If found → return all n methods unchanged.
