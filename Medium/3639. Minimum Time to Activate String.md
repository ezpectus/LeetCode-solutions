# 3639. Minimum Time to Activate String  
## O(n log n) — Binary Search + Sliding Window Validation

---

## Problem Statement

You are given:

- A string `s` of length `n`
- A permutation array `order[]` of size `n`
- At time `t`, the character at index `order[t]` becomes `'*'`

A substring is **valid** if it contains **at least one `'*'`**.

A string becomes **active** when:

```
#valid_substrings ≥ k
```

Your task is to find the **minimum time `t`** such that the string becomes active.  
If it is impossible for any `t`, return `-1`.

---

## Core Idea — Binary Search on Time + Sliding Window Check

The key observation:

- If the string is active at time `t`, then it is also active at any time `t' > t`
- This means the predicate “active or not” is **monotonic**
- Therefore, we can apply **binary search** on `t`

To evaluate a candidate time `t`, we must compute:

```
Check(t) = number of substrings containing at least one '*'
```

### How to compute Check(t)

1. Build a temporary string where all indices `order[0..t]` are replaced with `'*'`
2. Use a **sliding window** to count substrings that contain at least one star

Sliding window logic:

- Expand `right` until the window contains at least one `'*'`
- For each valid window starting at `left`, all substrings:

```
[left..right], [left..right+1], ..., [left..n-1]
```

are valid

Thus we add:

```
n - right + 1
```

Then move `left` forward and adjust the star count.

This gives an **O(n)** check.

Binary search performs **O(log n)** checks → total **O(n log n)**.

---

## Algorithm

1. Set binary search bounds:
   ```
   left = 0
   right = n
   ```
2. While `left < right`:
   - Compute `mid`
   - If `Check(mid) ≥ k`, move `right = mid`
   - Else move `left = mid + 1`
3. After binary search:
   - If `right == n`, activation is impossible → return `-1`
   - Otherwise return `right`

---


## Full Implementation (C#)

```csharp
public class Solution {
    public int MinTime(string s, int[] order, int k)
    {
        var left = 0;
        var right = order.Length;

        while (left < right)
        {
            var mid = left + (right - left) / 2;

            if (Check(s, order, mid) >= k)
            {
                right = mid;
            }
            else
                left = mid + 1;
        }

        return right == order.Length ? -1 : right;
    }

    private long Check(string s, int[] order, int time)
    {
        var sb = new StringBuilder(s);
        for (var i = 0; i <= time; i++)
        {
            sb[order[i]] = '*';
        }

        var length = s.Length;
        var left = 0;
        var right = 0;
        var cnt = 0L;
        var result = 0L;

        while (left < length)
        {
            while (right < length && cnt < 1)
            {
                if (sb[right] == '*') {
                    cnt++;
                }
                right++;
            }

            if (cnt < 1)
                break;

            result += length - right + 1;

            if (sb[left++] == '*'){
                cnt--;
            }
        }

        return result;
    }
}
```

## Complexity

| Metric | Value |
|--------|--------|
| Time Complexity | **O(n log n)** |
| Space Complexity | **O(n)** |
| Sliding window per check | O(n) |
| Binary search steps | O(log n) |

---

## Pitfalls & Edge Cases

- Must rebuild the string for each check
- Sliding window must correctly handle multiple stars
- Off‑by‑one errors in counting `n - right + 1`
- If no valid `t` exists, return `-1`
- StringBuilder is required for efficient mutation

---

## Example

`s = "abc"`, `order = [1,0,2]`, `k = 2`

At `t = 0`:

```
a*c
Valid substrings = 4 ≥ 2
```

Thus the answer is `0`.

---

## Key Takeaway

Binary search reduces the problem to repeatedly checking whether the string is active at time `t`.  
The sliding window efficiently counts valid substrings in linear time.

This combination yields a clean **O(n log n)** solution.
