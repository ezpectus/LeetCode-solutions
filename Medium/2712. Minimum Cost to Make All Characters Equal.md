# 2712. Minimum Cost to Make All Characters Equal
## O(n) — Greedy on Adjacent Boundaries

---

## Problem Statement

- Binary string `s`. Two operations:
  - Flip `s[0..i]` (prefix), cost = `i+1`
  - Flip `s[i..n-1]` (suffix), cost = `n-i`
- Return minimum total cost to make all characters equal.

---

## Core Idea — Each Boundary Costs Its Minimum Flip

**Key insight**: the final string is either all-0 or all-1. In either case, every adjacent pair `(s[i], s[i+1])` where `s[i] ≠ s[i+1]` represents a "break" that must be resolved. Each such break requires exactly one flip operation that crosses boundary `(i, i+1)`.

**Why exactly one flip per break?**
Any flip operation (prefix or suffix) that crosses boundary `(i, i+1)` inverts both sides of that boundary simultaneously, potentially resolving or creating breaks. The optimal strategy is to treat each break independently: fix it with the cheaper of:
- Flip the left part: prefix flip at `i`, cost = `i+1`
- Flip the right part: suffix flip at `i+1`, cost = `n-i-1`

```
res += min(i+1, n-i-1)  for each i where s[i] != s[i+1]
```

**Why this is optimal?** Each boundary break is independent — a flip that crosses one boundary to fix it doesn't create new breaks (it inverts a contiguous block, maintaining or reducing breaks at other positions). Greedy minimum per break is globally optimal.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinimumCost(string s) {
        int n = s.Length;
        long res = 0;
        for (int i = 0; i < n - 1; i++)
            if (s[i] != s[i+1])
                res += Math.Min(i + 1, n - i - 1);
        return res;
    }
}
```

## Example Walkthrough

**Example 2**: `s = "010101"`, n=6

| i | s[i] | s[i+1] | break? | min(i+1, n-i-1) |
|---|------|--------|--------|-----------------|
| 0 | 0 | 1 | ✓ | min(1, 5) = 1 |
| 1 | 1 | 0 | ✓ | min(2, 4) = 2 |
| 2 | 0 | 1 | ✓ | min(3, 3) = 3 |
| 3 | 1 | 0 | ✓ | min(4, 2) = 2 |
| 4 | 0 | 1 | ✓ | min(5, 1) = 1 |

Total: `1+2+3+2+1 = 9` ✓

## Pitfalls & Edge Cases

- **Already uniform string** — no breaks → loop adds nothing → `res=0` ✓
- **`long` for result** — n up to 10⁵, cost per boundary up to n/2 ≈ 5×10⁴, up to n-1 breaks → max ≈ 10⁵ × 5×10⁴ = 5×10⁹ → overflows `int`, use `long`
- **`n-i-1` not `n-i`** — suffix flip at `i+1` costs `n-(i+1) = n-i-1`, not `n-i`

---

## Key Takeaway

"Minimum flips to equalize binary string" → **count boundary breaks, pay minimum flip cost per break**:

```csharp
if (s[i] != s[i+1]) res += Math.Min(i+1, n-i-1);
```

Each `s[i] ≠ s[i+1]` boundary needs exactly one crossing flip. The cheapest crossing flip is the shorter side. Independent greedy per boundary is globally optimal.
