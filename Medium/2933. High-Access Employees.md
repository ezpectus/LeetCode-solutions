# 2933. High-Access Employees
## O(n log n) — Group by Name, Sort Times, Sliding Window of 3

---

## Problem Statement

- Employee is "high-access" if they accessed the system **3 or more times within a strict one-hour window** (less than 60 minutes between first and third access).
- Return names of all high-access employees.

---

## Core Idea — Convert to Minutes, Check Every Consecutive Triple

**Step 1**: Group access times by employee name.

**Step 2**: Convert each time string `"HHMM"` to minutes: `HH * 60 + MM`.

**Step 3**: Sort each employee's times. Check every consecutive triple `(ts[i], ts[i+1], ts[i+2])`: if `ts[i+2] - ts[i] < 60`, the employee is high-access.

**Why consecutive triples?** If any 3 times fit in a window < 60 min, the earliest and latest of those 3 differ by < 60. By sorting, checking consecutive triples covers all possible windows — any non-consecutive triple would have a larger span.

**Why `< 60` not `<= 60`?** The problem states "exactly one hour of difference is NOT in the same period" → strictly less than 60 minutes.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public IList<string> FindHighAccessEmployees(IList<IList<string>> access_times) {
        var times = new Dictionary<string, List<int>>();

        foreach (var entry in access_times) {
            string name = entry[0];
            int minutes = int.Parse(entry[1][..2]) * 60 + int.Parse(entry[1][2..]);
            if (!times.ContainsKey(name)) times[name] = new List<int>();
            times[name].Add(minutes);
        }

        var res = new List<string>();
        foreach (var (name, ts) in times) {
            ts.Sort();
            for (int i = 0; i + 2 < ts.Count; i++) {
                if (ts[i+2] - ts[i] < 60) {
                    res.Add(name);
                    break;
                }
            }
        }

        return res;
    }
}
```

## Example Walkthrough

**Example 1**: `"a"` has times `[0532, 0549, 0621]` in minutes = `[332, 349, 381]`

Sorted: `[332, 349, 381]`

Triple (0,1,2): `381 - 332 = 49 < 60` ✓ → `"a"` is high-access.

`"b"` has only 2 accesses → can't form a triple → not high-access.

---

## Pitfalls & Edge Cases

- **`< 60` not `<= 60`**: times exactly 60 apart are NOT in the same window
- **`break` after first found triple**: avoid adding the same employee multiple times
- **Parse `"0800"` correctly**: `entry[1][..2]` = "08" = 8, times 60 = 480; `entry[1][2..]` = "00" = 0

---

## Key Takeaway

"3+ accesses within 1 hour" → **group by employee, sort times, check every consecutive triple for span < 60 minutes**. Sorting ensures consecutive triples are the tightest possible windows.
