# 2857. Count Pairs of Points With Distance k
## O(n × k) — Enumerate XOR Splits + Hash Map

---

## Problem Statement

- Distance between `(x1,y1)` and `(x2,y2)` = `(x1 XOR x2) + (y1 XOR y2)`.
- Count pairs `(i,j)` with `i < j` and distance = `k`.
- `k ≤ 100` → small number of splits.

---

## Core Idea — Enumerate All XOR Splits

Distance `k = a + b` where `a = x1 XOR x2` and `b = y1 YOR y2`. For each valid split `(a, b)` with `a + b = k`, there are only `k+1 ≤ 101` such splits.

Process points one by one. For each new point `(x, y)`, check how many **previously seen** points `(x2, y2)` satisfy:
```
x XOR x2 == a  →  x2 = x XOR a
y XOR y2 == b  →  y2 = y XOR b
```

Look up `(x XOR a, y XOR b)` in the hash map for every `a` in `[0, k]`. Add the count. Then insert `(x, y)` into the map.

This processes each pair exactly once (`i < j` guaranteed by the order of insertion).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CountPairs(IList<IList<int>> coordinates, int k) {
        var seen = new Dictionary<(int,int), int>();
        int res = 0;

        foreach (var coord in coordinates) {
            int x = coord[0], y = coord[1];

            // For each split a+b=k, look for matching previous point
            for (int a = 0; a <= k; a++) {
                int b = k - a;
                var key = (x ^ a, y ^ b);
                if (seen.TryGetValue(key, out int cnt))
                    res += cnt;
            }

            // Insert current point
            var cur = (x, y);
            seen[cur] = seen.GetValueOrDefault(cur) + 1;
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × k)** | n points × k+1 splits each |
| **Space** | **O(n)** | Hash map stores at most n distinct points |

With `n ≤ 50000` and `k ≤ 100`: at most 5×10⁶ operations — fast.

---

## Why This Works

For a pair `(i, j)` with `i < j`, we encounter point `j` after point `i`. When processing `j`, point `i` is already in `seen`. We check `(x_j XOR a, y_j XOR b)` — if that equals `(x_i, y_i)`, then `x_j XOR x_i = a` and `y_j XOR y_i = b`, so distance = `a + b = k`. ✓

Every pair is counted exactly once since we look backwards.

---

## Example Walkthrough

**`coordinates=[[1,2],[4,2],[1,3],[5,2]], k=5`**

Processing `(1,2)`: seen={}. Add `(1,2):1`.

Processing `(4,2)`: check splits:
- a=5,b=0: look for `(4^5, 2^0)=(1,2)` → seen has 1 → res=1
- others: no match. Add `(4,2):1`.

Processing `(1,3)`: check splits:
- a=4,b=1: look for `(1^4,3^1)=(5,2)` → not in seen
- a=0,b=5: look for `(1^0,3^5)=(1,6)` → not in seen
- others: no match. Add `(1,3):1`.

Processing `(5,2)`: check splits:
- a=4,b=1: look for `(5^4,2^1)=(1,3)` → seen has 1 → res=2
- others: no match. Add `(5,2):1`.

`res = 2` ✓

---

## Pitfalls & Edge Cases

- **`k=0`**: only split is `a=0,b=0` → look for identical points. For 5 identical points: `C(5,2)=10` ✓
- **Duplicate coordinates**: handled correctly by the frequency count in the map
- **XOR properties**: `x XOR a XOR a = x`, so the lookup `x ^ a` correctly inverts the XOR

---

## Key Takeaway

"Count pairs with XOR-based distance = k" → **enumerate k+1 splits of k, hash map for O(1) lookup**:

```
for each (x,y): for a in [0,k]: res += seen[(x^a, y^(k-a))]
```

The small bound `k ≤ 100` is the key constraint enabling this approach — otherwise O(n×k) would be too slow.
