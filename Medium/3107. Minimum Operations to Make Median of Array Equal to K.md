# 3107. Minimum Operations to Make Median of Array Equal to K
## O(n log n) — Sort + Greedy on Violated Side

---

## Problem Statement

- Median of sorted array = element at index `n/2` (0-indexed). If tied, take the larger.
- Increase or decrease any element by 1 per operation.
- Return minimum operations to make median = `k`.

---

## Core Idea — Fix Only the Problematic Side

Sort `nums`. Median = `nums[mid]` where `mid = n/2`.

**Case 1: `nums[mid] == k`** → already correct, return 0.

**Case 2: `nums[mid] > k`** → need to lower the median.

To make median = k after sorting, we need exactly `mid+1` elements ≤ k. Elements in positions `0..mid` that are **greater than k** must be reduced to k. Reducing them puts k at position mid in the sorted order.

Cost = `Σ (nums[i] - k)` for `i ∈ [0, mid]` where `nums[i] > k`.

**Case 3: `nums[mid] < k`** → need to raise the median.

We need at least `mid+1` elements ≥ k. Elements in positions `mid..n-1` that are **less than k** must be increased to k.

Cost = `Σ (k - nums[i])` for `i ∈ [mid, n-1]` where `nums[i] < k`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinOperationsToMakeMedianK(int[] nums, int k) {
        Array.Sort(nums);
        int n = nums.Length, mid = n / 2;
        long res = 0;

        if (nums[mid] > k) {
            // Reduce elements in 0..mid that exceed k
            for (int i = 0; i <= mid; i++)
                if (nums[i] > k) res += nums[i] - k;
        } else if (nums[mid] < k) {
            // Increase elements in mid..n-1 that are below k
            for (int i = mid; i < n; i++)
                if (nums[i] < k) res += k - nums[i];
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n log n)** — sort dominates |
| **Space** | **O(1)** |

---

## Why This Works

After sorting, the median is fixed at position `mid`. For the median to be `k`:

**Lowering case**: We need `nums[mid]` (in sorted order after changes) to be `k`. The cheapest way: bring all elements in `[0, mid]` that are above `k` down to `k`. This ensures position `mid` is now at most `k`, and since elements beyond `mid` are still ≥ original values > k, the sorted median lands exactly at k. Reducing right-half elements is wasteful — they don't affect the median position.

**Raising case**: Symmetric — bring elements in `[mid, n-1]` below `k` up to `k`. Left-half elements don't affect raising the median.

---

## Example Walkthrough

**`nums=[2,5,6,8,5], k=4`** → sorted: `[2,5,5,6,8]`, mid=2, nums[2]=5 > 4.

Scan indices 0..2: nums[0]=2 ≤ 4 (skip), nums[1]=5 > 4 (cost 1), nums[2]=5 > 4 (cost 1).
Total = **2** ✓. After: [2,4,4,6,8], median=4 ✓.

**`nums=[2,5,6,8,5], k=7`** → sorted: `[2,5,5,6,8]`, mid=2, nums[2]=5 < 7.

Scan indices 2..4: nums[2]=5 < 7 (cost 2), nums[3]=6 < 7 (cost 1), nums[4]=8 ≥ 7 (skip).
Total = **3** ✓. After: [2,5,7,7,8], median=7 ✓.

---

## Pitfalls & Edge Cases

- **Include `mid` itself in both cases**: when `nums[mid] > k`, it must be reduced to `k` (cost `nums[mid]-k`). Similarly for the right case.
- **`long` for result**: `n=2×10^5`, each element up to `10^9` → sum can reach ~`2×10^{14}`, overflows `int`.
- **Even-length arrays**: median is at `n/2` (larger of two middle elements by problem definition). Formula `mid = n/2` handles this correctly.

---

## Key Takeaway

"Min ops to make median = k" → **sort, check which side is wrong, fix only that side**:

```
if nums[mid] > k: reduce all nums[0..mid] that exceed k
if nums[mid] < k: increase all nums[mid..n-1] that are below k
```

Changing the wrong side never helps — elements beyond `mid` don't affect lowering the median, and elements before `mid` don't affect raising it.
