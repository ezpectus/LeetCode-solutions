# 🏔️ 3881. Direction Assignments with Exactly K Visible People — Architectural Solution

## Problem Statement

You are given three integers `n`, `pos`, and `k`. There are `n` people standing in a line indexed from `0` to `n - 1`. Each person independently chooses a direction: 'L' or 'R'.

A person at index `pos` (the observer) sees others as follows:
- A person to the left (`i < pos`) is visible if and only if they choose 'L'.
- A person to the right (`i > pos`) is visible if and only if they choose 'R'.

Return the number of possible direction assignments such that the person at index `pos` sees **exactly** `k` people. Since the answer can be large, return it modulo `10^9 + 7`.

### Constraints
- 1 <= n <= 10^5
- 0 <= pos, k <= n - 1

---

## 💡 Core Idea

At first glance, the position `pos` seems to split the problem into left and right sub-problems. However, a closer look at the rules reveals a beautiful simplification:

1. **The Left Side (`i < pos`)**: Each person has 2 choices ('L' or 'R'). Exactly **1 choice** makes them visible ('L') and **1 choice** makes them invisible ('R').
2. **The Right Side (`i > pos`)**: Each person has 2 choices ('L' or 'R'). Exactly **1 choice** makes them visible ('R') and **1 choice** makes them invisible ('L').

This means that for **every single person** other than the observer, there is always exactly 1 way to be visible and 1 way to be invisible, regardless of whether they stand to the left or right of `pos`.

Therefore, the location of `pos` does not matter at all. The problem simply reduces to:
- There are `n - 1` total people available to be seen.
- We must choose exactly `k` of them to be visible. The number of ways to do this is given by the combination formula `C(n - 1, k)`.
- The person at `pos` themselves can choose either 'L' or 'R' without affecting anyone's visibility, which gives **2 choices**.

The final formula is: `Total Ways = 2 * C(n - 1, k) % (10^9 + 7)`

## 💻 Implementation (C++)

```cpp
#include <vector>

using namespace std;

class Solution {
private:
    long long MOD = 1e9 + 7;

    // Function to calculate (base^exp) % MOD using binary exponentiation
    long long power(long long base, long long exp) {
        long long res = 1;
        base %= MOD;
        while (exp > 0) {
            if (exp % 2 == 1) res = (res * base) % MOD;
            base = (base * base) % MOD;
            exp /= 2;
        }
        return res;
    }

    // Function to calculate modular inverse using Fermat's Little Theorem
    long long modInverse(long long n) {
        return power(n, MOD - 2);
    }

    // Function to compute C(n, k) % MOD
    long long nCr(int n, int k) {
        if (k < 0 || k > n) return 0;
        if (k == 0 || k == n) return 1;

        long long num = 1, den = 1;
        for (int i = 0; i < k; i++) {
            num = (num * (n - i)) % MOD;
            den = (den * (i + 1)) % MOD;
        }
        return (num * modInverse(den)) % MOD;
    }

public:
    int countVisiblePeople(int n, int pos, int k) {
        // Total candidate people to be seen is n - 1
        int totalCandidates = n - 1;

        if (k > totalCandidates) return 0;

        // Calculate combinations C(n - 1, k)
        long long combinations = nCr(totalCandidates, k);

        // Multiply by 2 because the person at 'pos' can choose either 'L' or 'R'
        long long result = (combinations * 2) % MOD;

        return static_cast<int>(result);
    }
};
```

## 📐 Complexity Analysis

| Metric | Value | Justification |
| :--- | :--- | :--- |
| **Time Complexity** | O(k) or O(n) | Computing `C(n - 1, k)` takes a single loop up to `k` iterations. Inside the loop, operations are O(1). The modular inverse takes O(log MOD) time due to binary exponentiation. Thus, total time is bounded by O(n). |
| **Space Complexity** | O(1) | The combination formula is calculated on the fly using a few scalar variables (`num`, `den`, `res`). No arrays or tables are allocated. |

## ⚠️ Pitfalls & Edge Cases

- **k Exceeds Available People**: If `k > n - 1`, it is physically impossible for the observer to see `k` people because there aren't enough people in the line. The code handles this by returning 0 immediately.
- **n = 1**: When `n = 1`, `totalCandidates = 0`. If `k = 0`, `C(0, 0) = 1`, and the result is `1 * 2 = 2` (since the lone person can choose 'L' or 'R'). This matches Example 3 correctly.
- **Modular Division**: You cannot divide directly by `den % MOD`. We must use the modular multiplicative inverse `modInverse(den)` via Fermat's Little Theorem to safely perform division under a prime modulo.

## 🔄 Alternative Approaches

- **Precomputed Factorials**: If this function were called multiple times for queries, we could precompute factorials and their modular inverses up to `n` in O(n) time. This would allow each query to be answered in O(1) constant time using the formula `fact[n-1] * invFact[k] * invFact[n-1-k]`.
- **Dynamic Programming (Pascal's Triangle)**: `C(n, k)` can be computed using a 2D DP table where `dp[i][j] = dp[i-1][j-1] + dp[i-1][j]`. However, this requires O(n^2) time and space, which will trigger a Memory Limit Exceeded (MLE) or Time Limit Exceeded (TLE) error for `n = 10^5`.
