# 2826. Sorting Three Groups
## O(n) — LIS on 3-Valued Array

---

## Problem Statement

- Array with values only `{1, 2, 3}`. Remove minimum elements to make it non-decreasing.

---

## Core Idea — Minimum Removals = n - Longest Non-Decreasing Subsequence

The elements we **keep** form a non-decreasing subsequence. To minimize removals, maximize what we keep → find the **longest non-decreasing subsequence (LNDS)**.

Answer = `n - LNDS`.

**O(n) DP** exploiting that values are only 1, 2, 3:

`dp[v]` = length of the longest non-decreasing subsequence seen so far that ends with value `v`.

For each element `x`: `dp[x] = max(dp[1], dp[2], ..., dp[x]) + 1`

Only 3 values → inner max is O(1) → whole algorithm O(n).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumOperations(IList<int> nums) {
        int[] dp = new int[4]; // dp[1], dp[2], dp[3]
        foreach (int x in nums)
            dp[x] = Math.Max(Math.Max(dp[1], x >= 2 ? dp[2] : 0), x >= 3 ? dp[3] : 0) + 1;
```

Clean version:

```csharp
public class Solution {
    public int MinimumOperations(IList<int> nums) {
        int d1 = 0, d2 = 0, d3 = 0; // LNDS ending with 1, 2, 3

        foreach (int x in nums) {
            if (x == 1) d1 = d1 + 1;
            else if (x == 2) d2 = Math.Max(d1, d2) + 1;
            else d3 = Math.Max(Math.Max(d1, d2), d3) + 1;
        }

        return nums.Count - Math.Max(Math.Max(d1, d2), d3);
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — one pass, O(1) per element |
| **Space** | **O(1)** — only 3 DP variables |

---

## Example Walkthrough

**`nums=[2,1,3,2,1]`**

| x | d1 | d2 | d3 |
|---|----|----|-----|
| 2 | 0 | 1 | 0 |
| 1 | 1 | 1 | 0 |
| 3 | 1 | 1 | 2 |
| 2 | 1 | 2 | 2 |
| 1 | 2 | 2 | 2 |

LNDS = max(2,2,2) = 2. Removals = 5-2 = **3** ✓

---

## Key Takeaway

"Min removals to make array non-decreasing" = `n - LNDS`. With only 3 distinct values, LNDS reduces to 3 variables updated in O(1) per element → O(n) total.
