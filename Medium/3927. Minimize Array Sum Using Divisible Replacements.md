# 3927. Minimize Array Sum Using Divisible Replacements
## O(n + V log V) — Sieve for Smallest Present Divisor

---

## Problem Statement

- Operation: pick a,b with `nums[a] % nums[b] == 0`, replace `nums[a]` with `nums[b]`.
- Repeat any number of times. Minimize the sum.

---

## Core Insight — Transitivity Collapses Chains to a Single Lookup

**Claim:** Each element `x = nums[i]` can be reduced to the **minimum value `w` present in the original array such that `x % w == 0`**. The answer is the sum of these minimums.

**Why chaining doesn't help beyond this:**

Suppose `x` could be reduced via a multi-step chain `x → v₁ → v₂ → ... → w`, where each `vᵢ` is some value that exists in the array at the time of the operation, and each step requires `prev % next == 0`.

- By transitivity of divisibility, if `x % v₁ == 0` and `v₁ % v₂ == 0`, then `x % v₂ == 0` directly.
- Crucially, `b`'s value never needs to be modified for `a`'s operation — any value `w` that ever appears as `nums[b]` for some operation must trace back (transitively) to a value that was **originally present** in the array (since the very first operations can only use original values as `b`).
- So the *set of values ever usable as `b`* is exactly the set of values **originally present** in `nums` (modifying other elements doesn't introduce new "smaller" values usable elsewhere — any value an element becomes is itself bounded below by an original value).

Therefore: for each `x`, the best achievable final value is `min{ w : w ∈ originalValues, x % w == 0 }`. This minimum is well-defined since `x` itself is always in `originalValues` and `x % x == 0`.

---

## Algorithm — Sieve for "Smallest Present Divisor"

1. Mark which values 1..maxVal are present in `nums` (as a boolean array).
2. For each present value `v` (ascending order), iterate over its multiples `v, 2v, 3v, ...` up to maxVal. For each multiple `m`, if `best[m]` hasn't been set yet, set `best[m] = v`.
   - Since we process present values in increasing order, the first time `best[m]` is set, it's set to the **smallest** present divisor of `m`.
3. Answer = `Σ best[nums[i]]`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinArraySum(int[] nums) {
        int maxVal = 0;
        foreach (int x in nums) maxVal = Math.Max(maxVal, x);

        bool[] present = new bool[maxVal + 1];
        foreach (int x in nums) present[x] = true;

        int[] best = new int[maxVal + 1]; // 0 = unset

        for (int v = 1; v <= maxVal; v++) {
            if (!present[v]) continue;
            for (int m = v; m <= maxVal; m += v) {
                if (best[m] == 0) best[m] = v;
            }
        }

        long sum = 0;
        foreach (int x in nums) sum += best[x];

        return sum;
    }
}
```

## Example Walkthrough

**Example 1:** `nums = [3,6,2]`, present = {2,3,6}, maxVal=6.

| v (present, ascending) | Multiples → best update |
|---|---|
| 2 | best[2]=2, best[4]=2, best[6]=2 |
| 3 | best[3]=3, best[6] already set (skip) |
| 6 | best[6] already set (skip) |

best[3]=3, best[6]=2, best[2]=2 → sum = 3+2+2 = **7** ✓

---

**Example 2:** `nums = [4,2,8,3]`, present = {2,3,4,8}, maxVal=8.

| v | Multiples → best update |
|---|---|
| 2 | best[2]=2, best[4]=2, best[6]=2, best[8]=2 |
| 3 | best[3]=3, best[6] already set |
| 4 | best[4],best[8] already set |
| 8 | best[8] already set |

best[4]=2, best[2]=2, best[8]=2, best[3]=3 → sum = 2+2+2+3 = **9** ✓

---

**Example 3:** `nums = [7,5,9]`, present = {5,7,9}, maxVal=9.

No value divides another (5,7,9 pairwise non-divisors). Each `best[x]=x`.

Sum = 5+7+9 = **21** ✓

---

## Pitfalls & Edge Cases

- **Single-element array:** `best[x]=x` (x divides itself), sum = x. No operations possible/needed. ✓

- **All elements equal:** `best[x]=x` for all, sum = n·x. ✓

- **Value 1 present:** if `1 ∈ nums`, then `best[m]=1` for ALL m (since 1 divides everything and is processed first, v=1 covers all multiples). Every element collapses to 1. ✓

- **Iteration order (ascending v)** is essential — guarantees the FIRST divisor found for any `m` is the smallest. ✓

---

## Key Takeaway

The operation's transitivity property ("if a is divisible by b and b is divisible by c, then a is divisible by c") means multi-step chains never beat a direct lookup. This reduces a seemingly complex "operations any number of times" problem to: **for each value, find its smallest present divisor** — a classic sieve pattern (similar to smallest prime factor sieve, but over all present values, not just primes).

**Pattern recognition:** "Repeatedly replace with divisors/multiples present in the set, minimize/maximize result" → check if transitivity collapses the problem to a single sieve-based lookup per element.
