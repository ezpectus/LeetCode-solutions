# 2657. Find the Prefix Common Array of Two Arrays
## O(n) — Frequency Count: Detect When a Number Appears in Both

---

## Problem Statement

- Given two permutations `A` and `B` of `[1..n]`.
- `C[i]` = count of numbers that appear in both `A[0..i]` and `B[0..i]`.
- Return array `C`.

---

## Core Idea — Count Appearances, Fire on Second Hit

Since both arrays are permutations, every number from 1 to n appears **exactly once** in each. A number becomes "common" at index `i` the moment it has been seen in **both** prefixes — i.e., when its total appearance count across both arrays reaches **2**.

Track a frequency array `cnt[1..n]`. At each step:
1. Increment `cnt[A[i]]` — if it hits 2, this number just became common → `common++`
2. Increment `cnt[B[i]]` — same check
3. `C[i] = common`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] FindThePrefixCommonArray(int[] A, int[] B) {
        int n = A.Length;
        int[] cnt = new int[n + 1];
        int[] res = new int[n];
        int common = 0;

        for (int i = 0; i < n; i++) {
            if (++cnt[A[i]] == 2) common++;
            if (++cnt[B[i]] == 2) common++;
            res[i] = common;
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Single pass, O(1) per step |
| **Space Complexity** | **O(n)** | Frequency array of size n+1 |

---

## Why `cnt == 2` is the exact trigger

Each number appears once in A and once in B — so `cnt[v]` can only ever reach 0, 1, or 2. Hitting 2 means "seen in both prefixes" = newly common. No number can be counted twice.

---

## Example Walkthrough

**Example 1**: `A=[1,3,2,4]`, `B=[3,1,2,4]`

| i | A[i] | B[i] | cnt changes | common | C[i] |
|---|------|------|-------------|--------|------|
| 0 | 1 | 3 | cnt[1]=1, cnt[3]=1 | 0 | 0 |
| 1 | 3 | 1 | cnt[3]=2✓, cnt[1]=2✓ | 2 | 2 |
| 2 | 2 | 2 | cnt[2]=1, cnt[2]=2✓ | 3 | 3 |
| 3 | 4 | 4 | cnt[4]=1, cnt[4]=2✓ | 4 | 4 |

Output: `[0,2,3,4]` ✓

---

## Pitfalls & Edge Cases

- **`A[i] == B[i]` at same step** — both increments fire in the same iteration; `cnt` hits 2 on the second increment → correctly counted once ✓
- **Permutation guarantee** — ensures `cnt[v]` never exceeds 2; no extra bounds checks needed

---

## Key Takeaway

"Count elements seen in both prefixes" → **single frequency array, trigger on count = 2**:

```csharp
if (++cnt[A[i]] == 2) common++;
if (++cnt[B[i]] == 2) common++;
```

The permutation property guarantees each value hits exactly 2 across both arrays — making the trigger clean and collision-free.
