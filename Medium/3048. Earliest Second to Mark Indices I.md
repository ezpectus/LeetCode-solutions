# 3048. Earliest Second to Mark Indices I
## O(m log m) — Binary Search + EDF Feasibility Check

---

## Problem Statement

- Each second: decrement any `nums[i]`, OR mark `changeIndices[s]` if `nums[changeIndices[s]-1] == 0`, OR do nothing.
- Return earliest second when all indices can be marked, or -1 if impossible.

---

## Core Idea — Binary Search on Time + Greedy EDF Check

**Binary search**: answer is monotone — if all indices can be marked by time `t`, they can also be marked by `t+1`. Binary search on `t ∈ [1..m]`.

**Feasibility check for time `t`**:
1. For each index `i`, find its **last occurrence** in `changeIndices[0..t-1]` → deadline `d_i`. Last occurrence is optimal: more decrement time available before it.
2. If any index has no occurrence → infeasible.
3. Sort indices by deadline. Apply **EDF (Earliest Deadline First)**:
   - Track `free` = decrement seconds accumulated so far.
   - Between consecutive deadlines: `free += gap` (those seconds are available for decrements).
   - At deadline `d_i`: need `nums[i]` decrement seconds. If `free < nums[i]` → infeasible. Otherwise spend them.

**Why EDF works?** Mark seconds are fixed (one per index at its deadline). The remaining seconds are decrement seconds freely assignable. EDF guarantees: if any valid schedule exists, the EDF schedule also works.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int EarliestSecondToMarkIndices(int[] nums, int[] changeIndices) {
        int n = nums.Length, m = changeIndices.Length;
        int lo = 1, hi = m, res = -1;
        while (lo <= hi) {
            int mid = (lo + hi) / 2;
            if (CanDo(nums, changeIndices, n, mid)) { res = mid; hi = mid - 1; }
            else lo = mid + 1;
        }
        return res;
    }

    bool CanDo(int[] nums, int[] changeIndices, int n, int t) {
        int[] last = new int[n];
        Array.Fill(last, -1);
        for (int s = 0; s < t; s++)
            last[changeIndices[s]-1] = s;

        foreach (int d in last)
            if (d == -1) return false;

        int[] order = Enumerable.Range(0, n).OrderBy(i => last[i]).ToArray();

        int free = 0, prevDeadline = -1;
        foreach (int i in order) {
            int d = last[i];
            free += d - prevDeadline - 1; // gap seconds available for decrements
            if (free < nums[i]) return false;
            free -= nums[i];
            prevDeadline = d;
        }
        return true;
    }
}
```

| **Space** | **O(n)** | Deadline arrays |

With `n, m ≤ 2000`: well within limits.

## Gap Seconds Formula

The deadline `d_i` is the second used for the **mark action** — it counts as 1 spent second but NOT as a free decrement second. So between two consecutive deadlines `prevDeadline` and `d`:

```
free += d - prevDeadline - 1
```

These are the seconds strictly between the two deadlines, all available for decrements.

## Example Walkthrough

**`nums=[1,3], changeIndices=[1,1,1,2,1,1,1], t=6`**

`changeIndices` (1-indexed): s=0→1, s=1→1, s=2→1, s=3→2, s=4→1, s=5→1.

Last occurrences in `[0..5]`: idx 0 → s=5, idx 1 → s=3.

Sorted by deadline: idx 1 (d=3) then idx 0 (d=5).

EDF sweep:
- idx 1 (d=3): free += 3-(-1)-1 = 3. Need nums[1]=3. 3≥3 ✓. free=0. prevDeadline=3.
- idx 0 (d=5): free += 5-3-1 = 1. Need nums[0]=1. 1≥1 ✓. free=0. prevDeadline=5.

Feasible ✓ → answer = **6**.

Checking t=5: last of idx 0 in [0..4] = s=4, last of idx 1 = s=3.
- idx 1 (d=3): free=3, need 3 ✓. free=0.
- idx 0 (d=4): free += 4-3-1=0. Need 1. 0 < 1 → False. ✓

## Pitfalls & Edge Cases

- **Last occurrence recomputed per binary search step**: O(m) scan each time — necessary since `t` changes.
- **`nums[i] = 0`**: needs 0 decrements → always feasible if index appears at all.
- **`gap = d - prevDeadline - 1` can be 0**: adjacent deadlines give no gap seconds. Still correct.
- **Mark second itself not counted as free**: it's used for the mark action, not a decrement.

---

## Key Takeaway

"Earliest time to mark all indices with decrement + mark constraints" → **binary search + EDF feasibility**:

```
last[i] = last occurrence of i in changeIndices[0..t-1]
sort by deadline → EDF check:
    free += gap between deadlines
    if free < nums[i]: infeasible
    free -= nums[i]
```

Last-occurrence assignment maximizes available decrement time. EDF scheduling proves the feasibility check is both necessary and sufficient.
