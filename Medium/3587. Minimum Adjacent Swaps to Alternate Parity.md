# 3587. Minimum Adjacent Swaps to Alternate Parity  
## O(n) — Match Even/Odd Positions + Count Inversions

---

## Problem Statement

You are given:

- An array `nums` of **distinct integers**
- In one operation, you may swap **two adjacent elements**
- A valid arrangement is one where **parity alternates**:
```
even, odd, even, odd, ...
or
odd, even, odd, even, ...
```

You must return:

> The **minimum number of adjacent swaps** needed to transform `nums` into **any** valid alternating-parity arrangement.

If no valid arrangement exists, return `-1`.

---

## Core Idea — Only Two Possible Target Patterns

Let:

- `E` = number of even elements
- `O` = number of odd elements
- `n = nums.length`

A valid alternating array must be either:

### Pattern A (even starts): 
```
even, odd, even, odd, ...
```
### Pattern B (odd starts):
```
odd, even, odd, even, ...
```

For each pattern to be possible:

- Pattern A requires:
```
E == ceil(n/2)
O == floor(n/2)
```
- Pattern B requires:
```
O == ceil(n/2)
E == floor(n/2)
```

If neither pattern is feasible → return `-1`.

---

## Key Insight — Adjacent Swaps = Sum of Distances to Target Positions

We do **not** care about values, only parity.

Let’s extract:
```
evenPositions = indices of even numbers in nums
oddPositions  = indices of odd numbers in nums
```


For a chosen pattern, we know exactly where evens and odds must go.

Example for Pattern A (even starts):
```
even target positions = [0, 2, 4, ...]
odd  target positions = [1, 3, 5, ...]
```

The minimum number of adjacent swaps needed to move elements to these positions is:

> Sum of absolute differences between current positions and target positions.

This works because all elements are distinct and we only care about relative ordering.

---

## Algorithm

1. Count evens and odds.
2. Determine which patterns (A or B) are feasible.
3. For each feasible pattern:
   - Build target positions for evens and odds.
   - Compute:
     ```
     cost = Σ |evenPositions[i] - evenTarget[i]| 
          + Σ |oddPositions[i]  - oddTarget[i]|
     ```
4. Return the minimum cost among feasible patterns.

---

## Full Implementation (C#)

```csharp
public class Solution {
    public int MinSwaps(int[] nums) {
        int n = nums.Length;

        var evenPos = new List<int>();
        var oddPos = new List<int>();

        for (int i = 0; i < n; i++) {
            if ((nums[i] & 1) == 0) evenPos.Add(i);
            else oddPos.Add(i);
        }

        int E = evenPos.Count;
        int O = oddPos.Count;

        long best = long.MaxValue;

        // Pattern A: even at 0,2,4,... (even starts)
        if (E == (n + 1) / 2 && O == n / 2) {
            long cost = 0;
            for (int i = 0; i < E; i++) {
                int target = 2 * i;
                cost += Math.Abs(evenPos[i] - target);
            }
            best = Math.Min(best, cost);
        }

        // Pattern B: odd at 0,2,4,... (odd starts)
        if (O == (n + 1) / 2 && E == n / 2) {
            long cost = 0;
            for (int i = 0; i < O; i++) {
                int target = 2 * i;
                cost += Math.Abs(oddPos[i] - target);
            }
            best = Math.Min(best, cost);
        }

        return best == long.MaxValue ? -1 : (int)best;
    }
}

```
## Complexity

| Metric | Value |
|--------|--------|
| Time Complexity | **O(n)** |
| Space Complexity | **O(n)** |
| Patterns checked | **2** |

---

## Pitfalls & Edge Cases

- You **must not** sum distances for both evens and odds — that double‑counts swaps.  
- You must check **both** patterns (even‑start and odd‑start) if both are feasible.  
- If `|E − O| > 1`, no alternating arrangement is possible → return `-1`.  
- All elements are distinct, so the internal order of evens and odds is fixed.  

---

## Example

### Example 1  
`nums = [2,4,6,5,7]`

Indices:

- evens at `[0,1,2]`  
- odds at `[3,4]`  
- `n = 5`, `E = 3`, `O = 2`

Pattern A (even starts) is feasible:

- even target positions: `0, 2, 4`

Cost:
```
|0 - 0| + |1 - 2| + |2 - 4| = 0 + 1 + 2 = 3
```

Pattern B (odd starts) is **not** feasible (`O != (n+1)/2`).

Final answer: **3**

---

## Key Takeaway

To minimize adjacent swaps:

> You only need to place **one** parity group (evens or odds) into its required alternating positions — the other group automatically fills the remaining slots.

This yields a clean and optimal **O(n)** solution with no complex data structures.
