# 2811. Check if it is Possible to Split Array
## O(n) — Greedy: Adjacent Pair with Sum ≥ m

---

## Problem Statement

- Split array into single elements. Each split produces two "good" subarrays (length 1, OR sum ≥ m).
- Return whether this is possible.

---

## Core Idea — Key Observations

**Observation 1**: `n ≤ 2` → always true. A length-1 array is always good, and a length-2 array can be split into two length-1 arrays directly.

**Observation 2**: For `n > 2`, we must at some point split a subarray of length ≥ 2 into two parts where at least one part has length ≥ 2. The only way a length-2 subarray can be "good" is if its sum ≥ m. And a length-2 subarray is always a pair of **adjacent elements** (since we can only split contiguous subarrays).

**Key insight**: it's possible to reduce the array to singletons **if and only if** there exists at least one adjacent pair `(nums[i], nums[i+1])` with `nums[i] + nums[i+1] ≥ m`. We can always isolate such a pair early, split it, then recursively handle the rest.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool CanSplitArray(IList<int> nums, int m) {
        int n = nums.Count;
        if (n <= 2) return true;
        for (int i = 0; i < n - 1; i++)
            if (nums[i] + nums[i+1] >= m) return true;
        return false;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

---

## Why Adjacent Pairs are Sufficient

Any split eventually produces length-2 subarrays before reaching length-1. A length-2 subarray `[a,b]` must satisfy `a+b ≥ m` to be splittable (it's not length-1, so it needs sum ≥ m). Any length-2 subarray formed during splits is a contiguous pair from the original array. Therefore the necessary and sufficient condition is that at least one adjacent pair in the original array has sum ≥ m — that pair can always be "saved" for a valid split step.

---

## Example Walkthrough

**`nums=[2,3,3,2,3], m=6`**

Adjacent pairs: `(2,3)=5<6`, `(3,3)=6≥6` ✓ → **true**

We split off `[3,3]` at the end and the rest follows.

---

## Key Takeaway

"Can array be recursively split into singletons with sum constraints?" → **check if any adjacent pair sums to ≥ m** (plus trivial base case for n≤2). Deceptively simple once the invariant is identified.
