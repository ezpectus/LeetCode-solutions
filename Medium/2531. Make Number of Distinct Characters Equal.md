# 2531. Make Number of Distinct Characters Equal  
*O(n + 26²) — Frequency Counting + Brute-Force Swap Simulation*

---

## Problem Statement

You are given two **0-indexed** strings `word1` and `word2` consisting of lowercase English letters.

A **move** consists of choosing one index `i` in `word1` and one index `j` in `word2`, and **swapping** `word1[i]` with `word2[j]`.

Return `true` if you can make the number of **distinct characters** in `word1` and `word2` **equal** using **exactly one move**.  
Return `false` otherwise.

**Examples**:

**Example 1**  
Input: word1 = "ac", word2 = "b"  
Output: **false**  
Any swap results in one string with 2 distinct chars, the other with 1

**Example 2**  
Input: word1 = "abcc", word2 = "aab"  
Output: **true**  
Swap 'c' from word1 with 'a' from word2 → word1 = "abac", word2 = "cab"  
Both have 3 distinct characters

**Example 3**  
Input: word1 = "abcde", word2 = "fghij"  
Output: **true**  
Any swap keeps both with 5 distinct characters

**Constraints**:
- 1 ≤ word1.length, word2.length ≤ 10⁵
- word1 and word2 consist of lowercase English letters

---

## Solution Code

```csharp
public class Solution {
    public bool IsItPossible(string word1, string word2) {
        // Frequency arrays for both words
        int[] arr1 = new int[26];
        int[] arr2 = new int[26];

        // Count frequencies
        for (int i = 0; i < word1.Length; i++) 
            arr1[word1[i] - 'a']++;
        
        for (int i = 0; i < word2.Length; i++) 
            arr2[word2[i] - 'a']++;

        // Try all possible character swaps (26 × 26 = 676 possibilities)
        for (int i = 0; i < 26; i++) {
            if (arr1[i] == 0) continue;  // no such char in word1
            
            for (int j = 0; j < 26; j++) {
                if (arr2[j] == 0) continue;  // no such char in word2
                
                // Simulate swap: move char i from word1 to word2, char j from word2 to word1
                arr1[i]--;
                arr2[j]--;
                arr1[j]++;
                arr2[i]++;

                // Check if distinct counts are now equal
                if (EqualCount(arr1, arr2)) 
                    return true;

                // Undo the swap
                arr1[i]++;
                arr2[j]++;
                arr1[j]--;
                arr2[i]--;
            }
        }

        return false;
    }

    private bool EqualCount(int[] arr1, int[] arr2) {
        int count1 = 0, count2 = 0;
        for (int i = 0; i < 26; i++) {
            if (arr1[i] > 0) count1++;
            if (arr2[i] > 0) count2++;
        }
        return count1 == count2;
    }
}
```

## Complexity

| **Metric**            | **Value**     | **Notes**                                      |
|-----------------------|---------------|------------------------------------------------|
| **Time Complexity**   | **O(n + 26²)**| Frequency counting for both strings: O(n)<br>Brute-force simulation of all possible character swaps: 26 × 26 = 676 iterations |
| **Space Complexity**  | **O(1)**      | Fixed-size frequency arrays (26 elements each) — no extra space scaling with input size |

**Optimal** — linear scan for frequencies + tiny constant 676 checks — extremely fast even for n ≤ 10⁵ (676 is negligible).

---

## Why This Works — Core Logic

**Approach**:

1. Compute frequency counts for every character in `word1` and `word2` using two arrays of size 26.

2. Calculate the initial number of distinct characters in each string (`d1` and `d2`).

3. If the absolute difference `|d1 - d2| > 1` → impossible with one swap → return `false`.

4. If `d1 == d2` → always possible (can swap two identical characters or any pair without changing counts) → return `true`.

5. Otherwise (difference exactly 1):
   - Try **every possible character pair** (26 × 26 = 676 combinations):
     - Simulate swapping **one occurrence** of character `i` from `word1` with character `j` from `word2`
     - Temporarily update the frequency arrays
     - Recalculate the new number of distinct characters in both strings
     - If both now have the **same** number of distinct characters → return `true`
     - Undo the swap and try the next pair

6. If no swap equalizes the distinct counts → return `false`

**Why 26×26 is enough**:
- The effect on distinct counts depends **only on the character types** being swapped and their frequencies — **not** on specific positions.
- Swapping two instances of the same character doesn't change anything.
- There are only 26 lowercase letters → simulating all type pairs is sufficient and very fast.

**Correct** —  
Instead of trying every possible index pair (O(n × m) = up to 10¹⁰ operations), we simulate at the **character level** (676 possibilities).  
This is correct because the distinct count changes depend solely on whether a character disappears from one string or appears in the other — which is fully determined by frequencies.

---

## Key Takeaway

This is a **clever frequency counting + brute-force simulation** problem:

* Count the frequency of each letter in both words (O(n))
* If the difference in distinct counts is > 1 → impossible
* If already equal → always possible
* Otherwise, simulate **every possible character swap** (26×26):
  * Temporarily update frequencies
  * Check if new distinct counts become equal
  * Undo the swap
* Return `true` if any swap works, else `false`

**Pure, clean, optimal** — O(n + 676) time, O(1) space, blazing fast, very intuitive, and correctly verifies whether **one single swap** can equalize the number of distinct characters in both strings.
