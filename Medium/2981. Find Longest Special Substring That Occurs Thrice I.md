# 2981. Find Longest Special Substring That Occurs Thrice I
## O(n²) — Run-Length Encoding + Greedy Count

---

## Problem Statement

- Given string `s` of lowercase English letters.
- A **special** string consists of only one repeated character (e.g. `"aaa"`, `"bb"`).
- Return the length of the **longest special substring** that appears **at least 3 times** in `s`, or `-1` if none exists.

---

## Core Idea — Collect Runs, Count Occurrences Greedily

**Key insight**: every special substring is a run of the same character. We can decompose `s` into maximal runs (consecutive same-character blocks), then for each character and each candidate length, count how many times that special substring appears.

**How many times does `"aaa"` (length `L` of char `c`) appear?**

Any run of `c` with length `R` contains exactly `R - L + 1` occurrences of the length-`L` special substring of `c`. Sum this over all runs of `c`.

**Algorithm**:
1. Parse `s` into runs: collect lengths of all maximal runs per character.
2. For each character `c` and each candidate length `L` (from largest to smallest):
   - Count total occurrences = `sum max(0, run - L + 1)` for all runs of `c`.
   - If count ≥ 3 → return `L` (first valid = longest).
3. If nothing found → return `-1`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumLength(string s) {
        // Collect run lengths per character
        var runs = new List<int>[26];
        for (int i = 0; i < 26; i++) runs[i] = new List<int>();

        int i2 = 0;
        while (i2 < s.Length) {
            int j = i2;
            while (j < s.Length && s[j] == s[i2]) j++;
            runs[s[i2] - 'a'].Add(j - i2);
            i2 = j;
        }

        int best = -1;

        for (int c = 0; c < 26; c++) {
            if (runs[c].Count == 0) continue;

            // Try lengths from largest possible down to 1
            int maxRun = runs[c].Max();
            for (int L = maxRun; L >= 1; L--) {
                int count = 0;
                foreach (int r in runs[c])
                    count += Math.Max(0, r - L + 1);
                if (count >= 3) {
                    best = Math.Max(best, L);
                    break; // longest L for this char found
                }
            }
        }

        return best;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | For each char, iterating lengths up to n and runs up to n |
| **Space** | **O(n)** | Storing all run lengths |

Given `n ≤ 50`: effectively constant time in practice.

---

## Why This Works

**Why count `R - L + 1` occurrences per run?**

A run of length `R` for character `c` looks like `ccc...c` (R times). A special substring of length `L` within it can start at positions `0, 1, ..., R - L` — that's `R - L + 1` starting positions, each giving a distinct occurrence. Since different runs are separated by other characters, occurrences never bleed across run boundaries.

**Why iterate lengths from largest down and break on first hit?**

We want the longest `L` for each character that appears ≥ 3 times. Starting from the max run length and decreasing, the first `L` that satisfies the condition is the best for that character. We track the global maximum across all characters.

**Why is this correct?**

Special substrings of character `c` can only come from runs of `c`. Runs are fully independent, and within each run all occurrences of a given length are contiguous and countable analytically — no need for a full substring search.

## Example Walkthrough

**`s = "aaaa"`**

Runs: `a → [4]`

| L | count = 4-L+1 | ≥ 3? |
|---|--------------|------|
| 4 | 1 | ✗ |
| 3 | 2 | ✗ |
| 2 | 3 | ✓ → best = 2, break |

Answer: **2** ✓

---

**`s = "abcdef"`**

Runs: `a→[1], b→[1], c→[1], d→[1], e→[1], f→[1]`

For each char, only run of length 1:
- `L=1`: count = 1 < 3 ✗

No character reaches count ≥ 3 → Answer: **-1** ✓

---

**`s = "abcaba"`**

Runs: `a→[1,1,1], b→[1,1], c→[1]`

Character `a`, runs `[1,1,1]`:
- `L=1`: count = (1-1+1)×3 = 3 ✓ → best = 1, break

Character `b`, runs `[1,1]`:
- `L=1`: count = 2 < 3 ✗

Answer: **1** ✓

## Pitfalls & Edge Cases

- **`R - L + 1` can be negative** if `L > R` — guard with `Math.Max(0, ...)` when summing across all runs
- **Multiple runs of same character**: a single character like `a` may appear in several separated runs — sum contributions from all of them
- **`n ≤ 50` allows brute force too**: even O(n³) substring counting works, but the run-based approach is cleaner and scales to the harder version (n ≤ 5000) without modification
- **Return global max across all characters**: different characters may each contribute a candidate — take the overall maximum

## Key Takeaway

"Longest single-char substring appearing ≥ 3 times" → **collect runs per character, count occurrences analytically**:

```
for each char c:
    for L = maxRun down to 1:
        count = sum(max(0, run - L + 1)) for all runs of c
        if count >= 3: best = max(best, L); break

return best  // or -1
```

A run of length `R` contributes `R - L + 1` occurrences of the length-`L` special substring. Greedy top-down search finds the longest valid `L` per character in O(n) → O(26n) = O(n) total per length → O(n²) overall.
