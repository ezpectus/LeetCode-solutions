# 2530. Maximal Score After Applying K Operations  
*O(n log n + k log n) — Greedy + Max-Heap (Priority Queue)*

---

## Problem Statement

You are given a **0-indexed** integer array `nums` and an integer `k`.  
You start with a score of **0**.

In one operation:
- Choose an index `i` (0 ≤ i < nums.length)
- Add `nums[i]` to your score
- Replace `nums[i]` with `ceil(nums[i] / 3)`

Perform **exactly k operations**.  
Return the **maximum possible score** you can achieve.

**Examples**:

**Example 1**  
Input: nums = [10,10,10,10,10], k = 5  
Output: **50**  
Apply operation to each 10 once → each becomes 4, score = 10×5 = 50

**Example 2**  
Input: nums = [1,10,3,3,3], k = 3  
Output: **17**  
Optimal: take 10 → becomes 4, score=10  
take 4 → becomes 2, score=10+4=14  
take 3 → becomes 1, score=14+3=17

**Constraints**:
- 1 ≤ nums.length, k ≤ 10⁵
- 1 ≤ nums[i] ≤ 10⁹

---

## Core Idea — Greedy: Always Pick the Current Maximum

**Key insight**:
- Each operation adds the **current value** of some element to the score
- Then that element is reduced to `ceil(value / 3)`
- To maximize the score, we should **always pick the largest current value** (greedy choice)
- After picking, the reduced value might still be large → we need to consider it again later
- Use a **max-heap** (priority queue) to always get the current maximum quickly

**Algorithm**:
1. Put all `nums[i]` into a **max-heap** (priority queue)
2. For each of k operations:
   - Pop the largest value `x`
   - Add `x` to the score
   - Push `ceil(x / 3)` back into the heap
3. Return the total score

**Time**: O(n log n + k log n) — building heap O(n), k extractions/insertions O(k log n)  
**Space**: O(n) — heap size up to n

---

## Clean Implementation (C#)

```csharp
public class Solution{
    public long MaxKelements(int[] nums, int k){
        // Max-heap (priority queue) — use negative values for max 
        var pq = new PriorityQueue<long, long>();

        // Add all elements (use negative for max-heap)
        foreach (int num in nums){
            pq.Enqueue(-num, -num); // negative for max-heap
        }

        long score = 0;
        for (int i = 0; i < k; i++){
            // Get max (negative → make positive)
            long maxVal = -pq.Dequeue();
            score += maxVal;

            // Push ceil(maxVal / 3) back (negative)
            long next = (maxVal + 2) / 3; // ceil division
            pq.Enqueue(-next, -next);
        }

        return score;
    }
}
```

| **Metric**            | **Value**              | **Notes**                                                                 |
|-----------------------|------------------------|---------------------------------------------------------------------------|
| **Time Complexity**   | **O(n log n + k log n)** | Building the max-heap: O(n)<br>Performing k operations: each Dequeue + Enqueue = O(log n) → O(k log n) |
| **Space Complexity**  | **O(n)**               | Priority queue (max-heap) stores up to n elements at any time             |

**Optimal** — efficient heap operations make it very fast even for n, k ≤ 10⁵ (heap size stays bounded, log n ≈ 17).

## Why This Works — Example Walkthrough

**Example 2**: nums = [1,10,3,3,3], k = 3

**Initial max-heap** (values negated internally for min-heap simulation in C#):  
[-10, -3, -3, -3, -1]

**Operation 1**:
- Extract max = **10** → score += 10 → now score = **10**
- Compute ceil(10 / 3) = **4** → push 4 back
- Heap now: [-4, -3, -3, -3, -1]

**Operation 2**:
- Extract max = **4** → score += 4 → score = **14**
- Compute ceil(4 / 3) = **2** → push 2 back
- Heap now: [-3, -3, -3, -2, -1]

**Operation 3**:
- Extract max = **3** → score += 3 → score = **17**
- Compute ceil(3 / 3) = **1** → push 1 back
- Heap now: [-3, -3, -2, -1, -1]

**Final score = 17** → correct

**Correct** —  
The greedy strategy works because **always picking the current largest value** maximizes the immediate contribution to the score.  
After adding a value, we reduce it to `ceil(x / 3)` and put it back — this value may still be among the largest in future operations, so we can pick it again if beneficial.  
This approach prioritizes **large contributions early** (when the multiplier is effectively higher due to repeated picks), leading to the optimal total score.

---

## Key Takeaway

This is a **classic greedy + max-heap** problem:

* To maximize the score over k operations, **always select the current largest number** in the array at each step
* After adding it to the score, replace it with `ceil(x / 3)` and continue
* Use a **max-heap** (priority queue) to efficiently retrieve and update the maximum value each time
* Perform exactly k such operations and accumulate the sum

**Pure, clean, optimal** — O(n log n + k log n) time, O(n) space, extremely fast in practice, very intuitive greedy choice, and correctly maximizes the total score by always prioritizing the highest possible contribution at every step.
