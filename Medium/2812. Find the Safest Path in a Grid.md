# 2812. Find the Safest Path in a Grid
## O(n² log n) — Multi-Source BFS + Binary Search

---

## Problem Statement

- Grid with thieves (`1`) and empty cells (`0`). Path from `(0,0)` to `(n-1,n-1)`.
- **Safeness** = minimum Manhattan distance from any cell on path to any thief.
- Return the maximum safeness over all possible paths.

---

## Core Idea — Two Phases

### Phase 1: Multi-Source BFS for Thief Distances

Run BFS starting simultaneously from all thief cells. After BFS, `dist[r][c]` = minimum distance from cell `(r,c)` to any thief. This is the "safeness" of cell `(r,c)` in isolation.

### Phase 2: Binary Search on Safeness + Path Reachability

**Monotone predicate**: if a path with safeness ≥ `s` exists, then one with safeness ≥ `s-1` also exists → binary search.

For threshold `s`: check if there's a path from `(0,0)` to `(n-1,n-1)` where every cell has `dist ≥ s`. Simple BFS/DFS over cells with `dist ≥ s`.

**Search range**: `[0, n]` (max possible distance in n×n grid).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumSafenessFactor(IList<IList<int>> grid) {
        int n = grid.Count;
        int[][] dist = new int[n][];
        for (int i = 0; i < n; i++) { dist[i] = new int[n]; Array.Fill(dist[i], -1); }

        var q = new Queue<(int,int)>();
        for (int r = 0; r < n; r++)
            for (int c = 0; c < n; c++)
                if (grid[r][c] == 1) { dist[r][c] = 0; q.Enqueue((r,c)); }

        int[] dr = {0,0,1,-1}, dc = {1,-1,0,0};
        while (q.Count > 0) {
            var (r,c) = q.Dequeue();
            foreach (var (nr,nc) in new[]{(r+1,c),(r-1,c),(r,c+1),(r,c-1)}) {
                if (nr>=0&&nr<n&&nc>=0&&nc<n&&dist[nr][nc]==-1) {
                    dist[nr][nc]=dist[r][c]+1; q.Enqueue((nr,nc));
                }
            }
        }

        bool CanReach(int s) {
            if (dist[0][0]<s||dist[n-1][n-1]<s) return false;
            bool[][] vis = new bool[n][];
            for (int i=0;i<n;i++) vis[i]=new bool[n];
            var bq = new Queue<(int,int)>();
            bq.Enqueue((0,0)); vis[0][0]=true;
            while (bq.Count>0) {
                var (r,c)=bq.Dequeue();
                if (r==n-1&&c==n-1) return true;
                foreach (var (nr,nc) in new[]{(r+1,c),(r-1,c),(r,c+1),(r,c-1)})
                    if (nr>=0&&nr<n&&nc>=0&&nc<n&&!vis[nr][nc]&&dist[nr][nc]>=s) {
                        vis[nr][nc]=true; bq.Enqueue((nr,nc));
                    }
            }
            return false;
        }

        int lo=0, hi=n;
        while (lo<hi) {
            int mid=(lo+hi+1)/2;
            if (CanReach(mid)) lo=mid; else hi=mid-1;
        }
        return lo;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n² log n)** | BFS: O(n²); binary search: O(log n) × O(n²) reachability |
| **Space** | **O(n²)** | dist array + BFS visited |

---

## Why Multi-Source BFS

Instead of computing distance from each thief separately (O(thieves × n²)), start BFS from all thieves simultaneously. Each cell gets the distance to its nearest thief in a single O(n²) pass. This is the classic "multi-source BFS" pattern.

---

## Example Walkthrough

**`grid=[[0,0,1],[0,0,0],[0,0,0]]`**

After multi-source BFS (thief at `(0,2)`):
```
dist:
2 1 0
3 2 1
4 3 2
```

Binary search:
- `s=3`: path needs dist≥3. Only `(0,0)` has dist=3 but `(2,2)` has dist=2 < 3 → false
- `s=2`: check path where all cells have dist≥2.
  Path: (0,0)→(1,0)→(2,0)→(2,1)→(2,2): dists=[3,3,4,3,2] → min=2≥2 ✓

Answer: `2` ✓

---

## Pitfalls & Edge Cases

- **`(0,0)` or `(n-1,n-1)` is a thief** → `dist=0` → answer is 0
- **Upper bound `hi=n`**: maximum possible distance in an n×n grid is `2(n-1)` but since we start binary search we just use `n` as safe upper bound (any path has safeness ≤ n-1 anyway)
- **`(lo+hi+1)/2`** — ceiling division in binary search to avoid infinite loop when `lo+1==hi` and `canReach(lo)` is true

---

## Key Takeaway

"Maximize minimum value along a path" → **binary search on threshold + reachability check**:

1. **Multi-source BFS** → compute safeness of each cell in O(n²)
2. **Binary search** on safeness value s
3. **BFS/DFS** over cells with safeness ≥ s to check reachability

Multi-source BFS is the key optimization — O(n²) instead of O(thieves × n²).
