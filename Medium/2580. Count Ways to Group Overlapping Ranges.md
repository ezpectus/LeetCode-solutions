# 2580. Count Ways to Group Overlapping Ranges
## O(n log n) — Sort + Greedy Component Counting

---

## Problem Statement

- You are given a **2D integer array** `ranges` where `ranges[i] = [start_i, end_i]` denotes all integers from `start_i` to `end_i` (inclusive).
- Split all ranges into **two (possibly empty) groups** such that any two overlapping ranges must belong to the **same group**.
- Two ranges overlap if they share at least one integer.
- Return the **number of ways** to split modulo 10⁹ + 7.

---

## Core Idea — Count Independent Components → 2^components

**Key insight**: Sort ranges by start. Sweep through and count how many **disconnected components** exist — groups of ranges that are all connected by overlaps. Each component is independent and can go into either group → multiply `2` per component.

**Why 2^components?**
- Overlapping ranges are forced into the same group — they form one indivisible "block"
- Each independent block has exactly **2 choices**: group 1 or group 2
- Blocks are independent of each other → multiply choices → `2^k` total ways

**Sweep logic**:
- Sort by `start`
- Track `max_end` (the furthest right boundary seen so far in the current component)
- If `start > max_end` → current range doesn't overlap with anything before it → **new component**
- Update `max_end = max(max_end, end)` to extend the current component's reach

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CountWays(int[][] ranges) {
        const int MOD = 1_000_000_007;
        Array.Sort(ranges, (a, b) => a[0] - b[0]); // sort by start

        int components = 0;
        int maxEnd = -1;

        foreach (var range in ranges) {
            if (range[0] > maxEnd)
                components++; // new disconnected component
            maxEnd = Math.Max(maxEnd, range[1]);
        }

        // Each component can independently go to group 1 or group 2
        long result = 1;
        for (int i = 0; i < components; i++)
            result = result * 2 % MOD;

        return (int)result;
    }
}
```

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n log n)** | Dominated by sorting; sweep is O(n) |
| **Space Complexity** | **O(1)** | Only tracking `maxEnd` and `components` counter |

**Optimal** — single pass after sorting; total ops ≈ n log n.

## Why This Works — Example Walkthrough

**Example 2**: `ranges = [[1,3],[10,20],[2,5],[4,8]]`

After sorting by start: `[[1,3],[2,5],[4,8],[10,20]]`

| Range | start > maxEnd? | Component | maxEnd |
|-------|----------------|-----------|--------|
| [1,3] | 1 > -1 ✅ | 1 | 3 |
| [2,5] | 2 > 3 ❌ | 1 | 5 |
| [4,8] | 4 > 5 ❌ | 1 | 8 |
| [10,20] | 10 > 8 ✅ | 2 | 20 |

`components = 2` → `2² = 4` ✓

**Example 1**: `ranges = [[6,10],[5,15]]`

After sorting: `[[5,15],[6,10]]`

| Range | start > maxEnd? | Component | maxEnd |
|-------|----------------|-----------|--------|
| [5,15] | 5 > -1 ✅ | 1 | 15 |
| [6,10] | 6 > 15 ❌ | 1 | 15 |

`components = 1` → `2¹ = 2` ✓

## Pitfalls & Edge Cases

- **Initialize `maxEnd = -1`** — since `start_i >= 0`, this guarantees the first range always starts a new component
- **Update `maxEnd` even within a component** — a range like `[1,100]` must extend reach even if `[2,5]` comes next; always take `max(maxEnd, end)`
- **Don't confuse component count with range count** — multiple ranges can be in one component; only gaps between components matter
- **MOD required** — `2^k` can be astronomically large; apply modulo at each multiplication step
- **Sort by start only** — ties in start don't matter; `maxEnd` tracking handles all overlap detection correctly

---

## Key Takeaway

When ranges must be grouped by overlap, **sort + sweep** finds connected components in O(n log n):

- **Sort** by start so overlapping ranges are adjacent
- **Sweep** tracking the furthest right endpoint seen (`maxEnd`)
- **New component** whenever the next range starts beyond `maxEnd`
- **Answer** = `2^(number of components)` — each independent block has 2 placement choices

Classic interval problem pattern: sort → sweep → track boundary → count gaps.
