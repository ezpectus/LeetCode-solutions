# 2545. Sort the Students by Their Kth Score  
*O(m log m) — Custom Sort by k-th Column*

---

## Problem Statement

You are given a **0-indexed** `m × n` integer matrix `score`, where:
- Each **row** represents one student
- `score[i][j]` is the score of the `i`-th student in the `j`-th exam

All values in `score` are **distinct**.

Given an integer `k` (0 ≤ k < n), sort the **rows** of the matrix by the scores in the **k-th exam** (0-indexed) from **highest to lowest**.

Return the matrix after sorting.

**Examples**:

**Example 1**  
Input: score = [[10,6,9,1],[7,5,11,2],[4,8,3,15]], k = 2  
Output: [[7,5,11,2],[10,6,9,1],[4,8,3,15]]  
Explanation:  
- Student 1: 11 in exam 2 (highest) → first  
- Student 0: 9 in exam 2 → second  
- Student 2: 3 in exam 2 → third

**Example 2**  
Input: score = [[3,4],[5,6]], k = 0  
Output: [[5,6],[3,4]]  
- Student 1: 5 in exam 0 → first  
- Student 0: 3 in exam 0 → second

**Constraints**:
- m == score.length
- n == score[i].length
- 1 ≤ m, n ≤ 250
- 1 ≤ score[i][j] ≤ 10⁵
- All values in `score` are **distinct**
- 0 ≤ k < n

---

## Core Idea — Sort Rows by k-th Column (Descending)

**Key insight**:
- We need to sort the **rows** of the matrix based only on the value in column `k`
- Sort in **descending** order (highest score in column k first)
- Since all values are **distinct**, no ties to handle
- In C#, we can sort the 2D array directly using a custom comparer or LINQ

**Steps**:
1. Treat each row as an array (or use indices)
2. Sort the rows using the value at index `k` in **descending** order
3. Return the sorted matrix

**Time**: O(m log m) — sorting m rows  
**Space**: O(1) extra (in-place sort) or O(m) if copying

---

## Clean Implementation (C#)

```csharp
public class Solution {
    public int[][] SortTheStudents(int[][] score, int k) {
        return score
            .OrderByDescending(row => row[k])
            .ToArray();
    }
}
```
## Complexity

| **Metric**            | **Value**     | **Notes**                                      |
|-----------------------|---------------|------------------------------------------------|
| **Time Complexity**   | **O(m log m)**| Sorting m rows based on the k-th column value — each comparison is O(1) |
| **Space Complexity**  | **O(1)**      | In-place sorting with `Array.Sort` (no extra array needed) — LINQ version may use O(m) temporary space |

**Optimal** — sorting dominates the time complexity, very fast and efficient for m ≤ 250 (m log m ≈ 250 × 8 = ~2000 operations).

## Why This Works — Example Walkthrough

**Example 1**: score = [[10,6,9,1],[7,5,11,2],[4,8,3,15]], k = 2

**Column k=2 values** (exam 2 scores):  
- Row 0: 9  
- Row 1: 11  
- Row 2: 3  

**Sort rows descending by column 2**:
- Row 1 (11) → first
- Row 0 (9) → second
- Row 2 (3) → third

**Resulting matrix**:'
```
[7,5,11,2]
[10,6,9,1]
[4,8,3,15]
```

→ matches the expected output

**Correct** —  
Since the problem only requires sorting based on **one specific column** (the k-th exam) in **descending order**, we can directly sort the 2D array (jagged array in C#) using a custom comparer that looks at `row[k]`.  
C#’s `Array.Sort` with a lambda comparer or LINQ `OrderByDescending(row => row[k])` handles this cleanly and efficiently.

---

## Key Takeaway

This is a **straightforward 2D array row sorting** problem:

* Sort the rows of the matrix based **only** on the value in column `k`
* Use **descending** order (highest score in column k comes first)
* In C#:
  - Preferred: `Array.Sort(score, (a, b) => b[k].CompareTo(a[k]));` — in-place, clean, and fast
  - Alternative: LINQ `OrderByDescending(row => row[k]).ToArray();` — readable but creates new array
* Return the sorted matrix

**Pure, clean, optimal** — O(m log m) time, O(1) extra space (in-place sort), extremely fast for m ≤ 250, very simple to implement, and directly solves the problem.
