# 2761. Prime Pairs With Target Sum
## O(n log log n) — Sieve of Eratosthenes + Linear Scan

---

## Problem Statement

- Find all pairs `(x, y)` where `x ≤ y`, `x + y == n`, both prime.
- Return sorted by `x`.

---

## Core Idea — Sieve Then Scan

**Step 1**: Sieve of Eratosthenes up to `n` → O(n log log n).

**Step 2**: For each `x` from `2` to `n/2`: check if both `x` and `n-x` are prime → O(n).

Since `x ≤ y = n-x` implies `x ≤ n/2`, iterating to `n//2` covers all valid pairs exactly once, already sorted by `x`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public IList<IList<int>> FindPrimePairs(int n) {
        bool[] sieve = new bool[n + 1];
        Array.Fill(sieve, true);
        sieve[0] = sieve[1] = false;
        for (int i = 2; (long)i * i <= n; i++)
            if (sieve[i])
                for (int j = i * i; j <= n; j += i)
                    sieve[j] = false;

        var res = new List<IList<int>>();
        for (int x = 2; x <= n / 2; x++)
            if (sieve[x] && sieve[n - x])
                res.Add(new List<int> { x, n - x });

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n log log n)** — sieve dominates |
| **Space** | **O(n)** — sieve array |

---

## Pitfalls & Edge Cases

- **`n = 2`**: loop `x=2` to `n/2=1` → empty ✓
- **`n` odd**: only one pair possible where one number is 2 (e.g. `n=5`: `[2,3]`)
- **`(long)i*i`**: for `i` up to ~1000 no overflow, but good practice

---

## Key Takeaway

"Prime pairs summing to n" → **sieve + two-pointer scan from both ends** (or just scan x to n/2). Classic Goldbach conjecture verification pattern.
