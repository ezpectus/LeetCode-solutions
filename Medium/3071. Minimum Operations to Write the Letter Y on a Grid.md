# 3071. Minimum Operations to Write the Letter Y on a Grid
## O(n²) — Frequency Count + Enumerate 6 Combinations

---

## Problem Statement

- `n×n` grid (`n` odd). The letter Y consists of:
  - Left diagonal: `(i,i)` for `i ∈ [0, n/2]`
  - Right diagonal: `(i, n-1-i)` for `i ∈ [0, n/2]`
  - Vertical stem: `(i, n/2)` for `i ∈ [n/2, n-1]`
- Goal: make all Y-cells the same value and all non-Y-cells the same (different) value.
- Return minimum number of cell changes.

---

## Core Idea — Count Frequencies, Try All 6 (Y_val, non_Y_val) Pairs

**Step 1**: Identify which cells belong to the Y and count their value frequencies.

**Step 2**: For Y-cells: `y_freq[v]` = count of cells with value `v`. For non-Y-cells: `non_freq[v]` similarly.

**Step 3**: Try all 6 ordered pairs `(yv, nv)` where `yv ≠ nv`, `yv, nv ∈ {0, 1, 2}`:
```
cost = (y_total - y_freq[yv]) + (non_total - non_freq[nv])
```
The first term = Y-cells that need to change (those not already `yv`).
The second term = non-Y-cells that need to change.

Return the minimum cost over all 6 pairs.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumOperationsToWriteY(int[][] grid) {
        int n = grid.Length, mid = n / 2;
        int[] yFreq = new int[3], nonFreq = new int[3];

        for (int r = 0; r < n; r++) {
            for (int c = 0; c < n; c++) {
                bool isY = (r <= mid && (c == r || c == n-1-r)) // diagonals
                        || (r >= mid && c == mid);               // vertical stem
                if (isY) yFreq[grid[r][c]]++;
                else     nonFreq[grid[r][c]]++;
            }
        }

        int yTotal = yFreq.Sum(), nonTotal = nonFreq.Sum();
        int res = int.MaxValue;

        for (int yv = 0; yv < 3; yv++)
            for (int nv = 0; nv < 3; nv++)
                if (yv != nv)
                    res = Math.Min(res,
                        (yTotal - yFreq[yv]) + (nonTotal - nonFreq[nv]));

        return res;
    }
}
```

## Y-Cell Detection Formula

For a cell `(r, c)` to be in the Y:
- **Left diagonal**: `r ≤ mid` and `c == r`
- **Right diagonal**: `r ≤ mid` and `c == n-1-r`
- **Vertical stem**: `r ≥ mid` and `c == mid`

Note: the center cell `(mid, mid)` satisfies all three conditions — it's counted once.

---

## Example Walkthrough

**`grid=[[1,2,2],[1,1,0],[0,1,0]]`**, n=3, mid=1.

Y-cells: (0,0)=1, (0,2)=2, (1,1)=1, (2,1)=1. → yFreq=[0,3,1], yTotal=4.
Non-Y-cells: (0,1)=2, (1,0)=1, (1,2)=0, (2,0)=0, (2,2)=0. → nonFreq=[3,1,1], nonTotal=5.

Try (yv=1, nv=0): cost = (4-3) + (5-3) = 1+2 = **3**. ✓

All 6 pairs:
- (0,1): (4-0)+(5-1)=4+4=8
- (0,2): (4-0)+(5-1)=4+4=8
- (1,0): (4-3)+(5-3)=1+2=**3** ✓
- (1,2): (4-3)+(5-1)=1+4=5
- (2,0): (4-1)+(5-3)=3+2=5
- (2,1): (4-1)+(5-1)=3+4=7

Minimum = **3** ✓

## Pitfalls & Edge Cases

- **Center cell `(mid, mid)`** belongs to both diagonals and the stem — count it only ONCE as a Y-cell. The formula `(r <= mid && (c == r || c == n-1-r)) || (r >= mid && c == mid)` correctly handles this since it's a single boolean check.
- **n=3 (smallest case)**: mid=1. Y has 4 cells, non-Y has 5 cells. Formula works correctly.
- **All cells already valid**: if Y-cells are all one value and non-Y-cells are all another, cost=0.

---

## Key Takeaway

"Minimize changes to make Y-cells uniform and non-Y-cells uniform (different values)" → **count frequencies per region, try all 6 valid (Y_val, non_Y_val) pairs**:

```
cost(yv, nv) = (y_total - y_freq[yv]) + (non_total - non_freq[nv])
answer = min over all yv ≠ nv
```

O(n²) for counting + O(1) for the 6-pair enumeration. Clean and direct.
