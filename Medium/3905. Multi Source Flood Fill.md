# 3905. Multi Source Flood Fill
## O(n·m) — Level-by-Level BFS with Max-Color Tiebreak

---

## Problem Statement

- Grid n×m. Multiple colored source cells spread simultaneously to all 4-directional uncolored neighbors each time step.
- If multiple colors reach an uncolored cell in the same step, it takes the **maximum** color value.
- Return the final fully-colored grid.

---

## Core Idea — Process BFS Level by Level, Resolve Ties Per Level

Standard single-queue BFS processes cells one at a time — the first color to "claim" a cell wins, which is WRONG when multiple colors arrive simultaneously.

**Correct approach:** Process the BFS frontier as discrete **levels** (time steps). For each level:
1. From every cell in the current level, propagate to uncolored neighbors.
2. For each candidate uncolored cell, collect the **maximum color** among all colors trying to claim it this step.
3. After processing the entire current level, commit these max-colors to the grid and form the next level.

This guarantees simultaneous arrivals are resolved by max-color before any cell is "locked in."

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[][] ColorGrid(int n, int m, int[][] sources) {
        int[][] grid = new int[n][];
        for (int i = 0; i < n; i++) grid[i] = new int[m];

        var currentLevel = new List<(int r, int c)>();

        foreach (var s in sources) {
            int r = s[0], c = s[1], color = s[2];
            grid[r][c] = color;
            currentLevel.Add((r, c));
        }

        int[] dr = {0, 0, 1, -1};
        int[] dc = {1, -1, 0, 0};

        while (currentLevel.Count > 0) {
            // candidate[(r,c)] = max color trying to claim this cell this step
            var candidates = new Dictionary<(int, int), int>();

            foreach (var (r, c) in currentLevel) {
                int myColor = grid[r][c];
                for (int d = 0; d < 4; d++) {
                    int nr = r + dr[d], nc = c + dc[d];
                    if (nr < 0 || nr >= n || nc < 0 || nc >= m) continue;
                    if (grid[nr][nc] != 0) continue; // already colored

                    var key = (nr, nc);
                    if (!candidates.TryGetValue(key, out int existing) || myColor > existing) {
                        candidates[key] = myColor;
                    }
                }
            }

            var nextLevel = new List<(int r, int c)>();
            foreach (var kv in candidates) {
                var (r, c) = kv.Key;
                grid[r][c] = kv.Value;
                nextLevel.Add((r, c));
            }

            currentLevel = nextLevel;
        }

        return grid;
    }
}
```

| Metric | Value | Notes |

| Space Complexity | O(n·m) | Grid + level lists + candidates dictionary |

## Example Walkthrough

**Example 1:** `n=3, m=3, sources=[[0,0,1],[2,2,2]]`

Initial: grid[0][0]=1, grid[2][2]=2. currentLevel = [(0,0),(2,2)].

**Step 1 (level 1):**
- From (0,0,color=1): neighbors (0,1) and (1,0) → candidates: {(0,1):1, (1,0):1}
- From (2,2,color=2): neighbors (1,2) and (2,1) → candidates: {(1,2):2, (2,1):2}

Commit: grid[0][1]=1, grid[1][0]=1, grid[1][2]=2, grid[2][1]=2.
nextLevel = [(0,1),(1,0),(1,2),(2,1)]

**Step 2 (level 2):**
- From (0,1,color=1): neighbor (0,2) uncolored, neighbor (1,1) uncolored → candidates: {(0,2):1, (1,1):1}
- From (1,0,color=1): neighbor (2,0) uncolored, neighbor (1,1) uncolored → candidates: {(2,0):1, (1,1): max(1,1)=1}
- From (1,2,color=2): neighbor (0,2) uncolored, neighbor (1,1) uncolored, neighbor (2,2) colored → candidates: {(0,2): max(1,2)=2, (1,1): max(1,2)=2}
- From (2,1,color=2): neighbor (2,0) uncolored, neighbor (1,1) uncolored, neighbor (2,2) colored → candidates: {(2,0): max(1,2)=2, (1,1): max(2,2)=2}

Final candidates: (0,2)→2, (1,1)→2, (2,0)→2.

Commit: grid[0][2]=2, grid[1][1]=2, grid[2][0]=2.

**Final grid:**
```
1 1 2
1 2 2
2 2 2
```
✓ Matches expected output.

## Pitfalls & Edge Cases

- **Same-step ties resolved BEFORE committing** — the `candidates` dictionary aggregates ALL colors trying to reach a cell in this step, taking the max, before any are written to `grid`. This prevents order-dependent results. ✓

- **Already-colored cells are skipped** — `if (grid[nr][nc] != 0) continue;` ensures a cell colored in an earlier (closer) step is never overwritten by a later, farther step. ✓

- **Single source (Example 3):** BFS expands in standard rings; no ties ever occur. ✓

- **Multiple sources with same color reaching same cell:** `myColor > existing` is fine; equal values just don't update (any value works since they're equal). ✓

---

## Key Takeaway

"Simultaneous multi-source spread with tie-breaking" → BFS **by levels** (not a flat queue), aggregating all simultaneous arrivals into a temporary map before committing to the grid.

**Pattern recognition:** Whenever "simultaneously" or "at the same time step" appears with a tiebreak rule, naive single-queue BFS is WRONG — you must process the frontier as discrete levels and resolve conflicts before moving cells from "candidate" to "final."
