# 3326. Minimum Division Operations to Make Array Non Decreasing
## O(n + M log log M) — Smallest Prime Factor Sieve + Greedy

---

## Problem Statement

- Divide any element by its greatest proper divisor (any number of times). Minimize operations to make array non-decreasing.

---

## Core Idea — Operation = Replace x with spf(x)

**Greatest proper divisor of x** = `x / spf(x)` where `spf` = smallest prime factor.

**After dividing by greatest proper divisor**: `x / (x/spf(x)) = spf(x)`.

So each operation replaces `x` with its smallest prime factor — the minimum possible value reachable in one operation.

**Greedy right-to-left**: if `nums[i] > nums[i+1]`, reduce `nums[i]` to `spf(nums[i])`. If still greater → impossible (-1).

Only one operation per element is ever needed (we can't do better than spf).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperations(int[] nums) {
        const int MAX = 1_000_001;
        int[] spf = new int[MAX];
        for (int i = 0; i < MAX; i++) spf[i] = i;
        for (int i = 2; (long)i*i < MAX; i++)
            if (spf[i] == i)
                for (int j = i*i; j < MAX; j += i)
                    if (spf[j] == j) spf[j] = i;

        int ops = 0;
        for (int i = nums.Length-2; i >= 0; i--) {
            if (nums[i] > nums[i+1]) {
                if (nums[i] == 1) return -1;
                nums[i] = spf[nums[i]]; // one operation
                if (nums[i] > nums[i+1]) return -1;
                ops++;
            }
        }
        return ops;
    }
}
```

## Why Only One Operation Per Element?

After one operation, `nums[i]` becomes `spf(nums[i])` — the smallest possible value reachable. Doing more operations would require `nums[i]` to first become `spf(nums[i])`, and then `spf(spf(nums[i]))`, but that's a prime whose spf is itself. So the sequence ends at the smallest prime factor. We never need more than 1 operation per element.

---

## Example Walkthrough

**`nums=[25,7]`**: 25 > 7. spf[25]=5. After: nums[0]=5 ≤ 7 ✓. ops=1. Answer: **1** ✓

**`nums=[7,7,6]`**: i=1, nums[1]=7 > nums[2]=6. spf[7]=7 (prime). After: nums[1]=7 > 6. → **-1** ✓

---

## Pitfalls & Edge Cases

- **x=1**: no proper divisors → can't be reduced. If `1 > nums[i+1]` (impossible since all ≥ 1) or if we need to reduce 1 → return -1.
- **Prime numbers**: `spf[p] = p` → operation gives `p/p = 1`... wait, dividing by greatest proper divisor: for prime p, greatest proper divisor = 1, so `p/1 = p`. Operation doesn't change it! So if prime > next element → -1.

Actually: for prime p, spf[p]=p. `x/(x/spf(x)) = spf(x) = p`. So after operation on prime p, it STAYS p. That means primes can't be reduced → if prime p > nums[i+1], return -1.

This is handled naturally: `nums[i] = spf[nums[i]]` = p for prime p, still > nums[i+1] → return -1. ✓

---

## Key Takeaway

"Divide by greatest proper divisor" = "replace with smallest prime factor". Greedy right-to-left: if violation found, apply one operation. If still violated → impossible.
