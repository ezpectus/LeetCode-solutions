# 2602. Minimum Operations to Make All Array Elements Equal
## O((n + m) log n) — Sort + Prefix Sum + Binary Search

---

## Problem Statement

- You are given a **positive integer** array `nums` (1 ≤ n ≤ 10⁵) and a query array `queries` (1 ≤ m ≤ 10⁵).
- For each query `q`: find the minimum number of `±1` operations to make all elements of `nums` equal to `q`.
- Return an array of size `m` with answers for each query.
- Array resets to original state after each query.

---

## Core Idea — Sort + Prefix Sum + Binary Search

**Key insight**: The minimum total cost to make all elements equal to `q` is the **sum of absolute differences** `Σ|nums[i] - q|`. After sorting `nums`, this splits cleanly into two parts at the insertion point of `q`:

- **Left part** (`nums[i] < q`): each contributes `q - nums[i]` → total = `q × cnt_left - sum_left`
- **Right part** (`nums[i] >= q`): each contributes `nums[i] - q` → total = `sum_right - q × cnt_right`

Both parts are computed in O(1) per query using a **prefix sum array** + **binary search** for the split point.

**Prefix sum**:
- `pref[i]` = sum of `nums[0..i-1]` (sorted)
- `pref[n]` = total sum
- `sum_left = pref[idx]`, `sum_right = pref[n] - pref[idx]`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public IList<long> MinOperations(int[] nums, int[] queries) {
        Array.Sort(nums);
        int n = nums.Length;

        // Build prefix sum
        long[] pref = new long[n + 1];
        for (int i = 0; i < n; i++)
            pref[i + 1] = pref[i] + nums[i];

        var res = new List<long>();
        foreach (int q in queries) {
            // First index where nums[idx] >= q
            int idx = LowerBound(nums, q);

            // Left: elements < q → cost = q*idx - pref[idx]
            long left = (long)q * idx - pref[idx];

            // Right: elements >= q → cost = (pref[n] - pref[idx]) - q*(n-idx)
            long right = (pref[n] - pref[idx]) - (long)q * (n - idx);

            res.Add(left + right);
        }

        return res;
    }

    int LowerBound(int[] arr, int target) {
        int lo = 0, hi = arr.Length;
        while (lo < hi) {
            int mid = (lo + hi) / 2;
            if (arr[mid] < target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O((n + m) log n)** | O(n log n) sort + O(n) prefix build + O(m log n) queries |
| **Space Complexity** | **O(n)** | Prefix sum array |

**Optimal** — each query answered in O(log n); can't do better without preprocessing.

---

## Why This Works — Math Breakdown

For sorted `nums` and query `q`, let `idx` = first index where `nums[idx] >= q`:

```
Total cost = Σ(i<idx) (q - nums[i])  +  Σ(i>=idx) (nums[i] - q)
           = q*idx - pref[idx]        +  (pref[n] - pref[idx]) - q*(n-idx)
```

Both terms computed in O(1) after O(n) prefix build. ✓

## Example Walkthrough

**Example 1**: `nums = [3,1,6,8]` → sorted: `[1,3,6,8]`

`pref = [0, 1, 4, 10, 18]`

**Query q=1**:
- `idx = 0` (all elements ≥ 1)
- `left = 1×0 - 0 = 0`
- `right = (18 - 0) - 1×4 = 18 - 4 = 14`
- Total = **14** ✓

**Query q=5**:
- `idx = 2` (nums[0]=1, nums[1]=3 are < 5)
- `left = 5×2 - 4 = 6`
- `right = (18 - 4) - 5×2 = 14 - 10 = 4`
- Total = **10** ✓

**Example 2**: `nums = [2,9,6,3]` → sorted: `[2,3,6,9]`

`pref = [0, 2, 5, 11, 20]`

**Query q=10**:
- `idx = 4` (all elements < 10)
- `left = 10×4 - 20 = 20`
- `right = 0`
- Total = **20** ✓

## Pitfalls & Edge Cases

- **Use `long`** — `nums[i]` up to 10⁹, n up to 10⁵ → max sum ≈ 10¹⁴, overflows `int`
- **`LowerBound` not `UpperBound`** — split at first `>= q`; elements equal to `q` contribute 0 cost and land in the right part (correct either way since `nums[i] - q = 0`)
- **`q` smaller than all elements** — `idx = 0`, `left = 0`, all cost goes to right part ✓
- **`q` larger than all elements** — `idx = n`, `right = 0`, all cost goes to left part ✓
- **Cast to `long` before multiply** — `(long)q * idx` to avoid overflow before the multiplication

---

## Key Takeaway

"Minimum total `|x - q|` for all elements" → **sort + prefix sum + binary search**:

- Sort once → O(n log n)
- Prefix sum once → O(n)
- Per query: binary search for split point → O(log n), then O(1) math with prefix sums

Pattern: sum of absolute differences from a target → sort the array, split at target, apply prefix sums on both sides.
