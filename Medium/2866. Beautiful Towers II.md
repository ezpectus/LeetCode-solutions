# 2866. Beautiful Towers II
## O(n) — Monotonic Stack for Prefix/Suffix Mountain Sums

---

## Problem Statement

Same as Beautiful Towers I but `n ≤ 10⁵` → need O(n), not O(n²).

- Find maximum sum mountain configuration where each `heights[i] ≤ maxHeights[i]`.

---

## Core Idea — Prefix/Suffix Sums with Monotonic Stack

For each index `i` as the **peak**, the best configuration:
- Left of peak: non-decreasing up to `maxHeights[i]` → each tower height = `min(maxHeights[j], maxHeights[peak])` going left
- Right of peak: symmetric

Define:
- `prefix[i]` = maximum sum of a non-decreasing sequence ending at index `i` (treating `i` as peak from the left)
- `suffix[i]` = maximum sum of a non-increasing sequence starting at index `i`

**Answer**: `max(prefix[i] + suffix[i] - maxHeights[i])` for each `i` (subtract peak once since counted in both).

**Computing `prefix[i]` with a monotonic decreasing stack**:

When processing index `i`, pop all indices `j` where `maxHeights[j] >= maxHeights[i]` — those towers get capped to `maxHeights[i]`. The stack maintains indices in **strictly increasing** order of `maxHeights`.

```
if stack is non-empty (top = j):
    prefix[i] = prefix[j] + maxHeights[i] * (i - j)
    // From j+1 to i: all capped to maxHeights[i]
    // prefix[j] already accounts for 0..j optimally
else:
    prefix[i] = maxHeights[i] * (i + 1)
    // All of 0..i capped to maxHeights[i]
```

Compute `suffix` symmetrically right-to-left.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumSumOfHeights(IList<int> maxHeights) {
        int n = maxHeights.Count;
        long[] prefix = new long[n];
        long[] suffix = new long[n];
        var stack = new Stack<int>();

        // Prefix: non-decreasing sequence ending at i
        for (int i = 0; i < n; i++) {
            while (stack.Count > 0 && maxHeights[stack.Peek()] >= maxHeights[i])
                stack.Pop();
            if (stack.Count > 0) {
                int j = stack.Peek();
                prefix[i] = prefix[j] + (long)maxHeights[i] * (i - j);
            } else {
                prefix[i] = (long)maxHeights[i] * (i + 1);
            }
            stack.Push(i);
        }

        stack.Clear();

        // Suffix: non-increasing sequence starting at i
        for (int i = n - 1; i >= 0; i--) {
            while (stack.Count > 0 && maxHeights[stack.Peek()] >= maxHeights[i])
                stack.Pop();
            if (stack.Count > 0) {
                int j = stack.Peek();
                suffix[i] = suffix[j] + (long)maxHeights[i] * (j - i);
            } else {
                suffix[i] = (long)maxHeights[i] * (n - i);
            }
            stack.Push(i);
        }

        long res = 0;
        for (int i = 0; i < n; i++)
            res = Math.Max(res, prefix[i] + suffix[i] - maxHeights[i]);

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Each element pushed/popped from stack at most once |
| **Space** | **O(n)** | prefix, suffix arrays + stack |

---

## Why the Stack Works

`prefix[i]` = sum of the optimal non-decreasing sequence ending at `i`. When `maxHeights[i]` is smaller than some previous tower `maxHeights[j]`, tower `j` (and everything between `j` and `i`) gets capped to `maxHeights[i]`.

The monotonic stack tracks the last position `j` where `maxHeights[j] < maxHeights[i]`. Everything from `j+1` to `i` takes value `maxHeights[i]` — contributing `maxHeights[i] * (i - j)` — plus whatever was optimal before `j`.

## Example Walkthrough

**`maxHeights=[6,5,3,9,2,7]`**, compute prefix:

| i | maxH[i] | stack before | after pop | prefix[i] |
|---|---------|-------------|-----------|-----------|
| 0 | 6 | [] | [] | 6×1=6 |
| 1 | 5 | [0] | [] (6≥5) | 5×2=10 |
| 2 | 3 | [1] | [] (5≥3) | 3×3=9 |
| 3 | 9 | [2] | [2] (3<9) | 9+(9×1)=9+9=18? → prefix[2]+9×(3-2)=9+9=**18** |
| 4 | 2 | [3] | [] (9≥2) | 2×5=10 |
| 5 | 7 | [4] | [4] (2<7) | prefix[4]+7×(5-4)=10+7=**17** |

Suffix similarly computed. Then `prefix[3]+suffix[3]-9 = 18+13-9 = 22` ✓

## Pitfalls & Edge Cases

- **`long` for sums**: `maxHeights[i]` up to 10⁹, n up to 10⁵ → max sum ~10¹⁴
- **Strictly increasing stack** (`>=` pop condition): when heights are equal, pop — ensures the formula `maxHeights[i] * (i - j)` correctly covers the segment
- **`prefix[i] + suffix[i] - maxHeights[i]`**: peak is counted in both prefix and suffix, subtract once

---

## Key Takeaway

"Maximum mountain sum" with O(n) → **monotonic stack prefix/suffix**:

```
prefix[i] = prefix[stack_top] + maxH[i] * (i - stack_top)
suffix[i] = suffix[stack_top] + maxH[i] * (stack_top - i)
answer    = max(prefix[i] + suffix[i] - maxH[i])
```

The stack amortizes the "capping" operation — each tower's contribution is computed once by finding the nearest shorter tower to its left/right.
