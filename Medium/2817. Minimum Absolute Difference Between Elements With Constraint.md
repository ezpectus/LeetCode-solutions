# 2817. Minimum Absolute Difference Between Elements With Constraint
## O(n log n) — Sliding Window + Sorted Set

---

## Problem Statement

- Find minimum `|nums[i] - nums[j]|` where `|i - j| >= x`.

---

## Core Idea — Maintain a Sorted Window of Valid Left Elements

For each index `j`, valid left indices are `0..j-x`. As `j` increases by 1, we add `nums[j-x]` to the set of valid candidates. For each `j`, find the closest value to `nums[j]` in the sorted set — check the immediate predecessor and successor via binary search.

**Why sorted set?** Finding the nearest value to a query in O(log n) requires a sorted structure with binary search.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinAbsoluteDifference(IList<int> nums, int x) {
        var sorted = new SortedSet<(int val, int idx)>(); // (value, index) for uniqueness
        int res = int.MaxValue;

        for (int j = 0; j < nums.Count; j++) {
            // Add nums[j-x] to the sorted window
            if (j >= x)
                sorted.Add((nums[j-x], j-x));

            if (sorted.Count == 0) continue;

            int v = nums[j];
            // Find closest: check successor and predecessor
            var succ = sorted.GetViewBetween((v, int.MinValue), (int.MaxValue, int.MaxValue)).Min;
            if (succ != default)
                res = Math.Min(res, succ.val - v);

            var pred = sorted.GetViewBetween((int.MinValue, int.MinValue), (v, int.MaxValue)).Max;
            if (pred != default)
                res = Math.Min(res, v - pred.val);

            if (res == 0) return 0;
        }
        return res;
    }
}
```

**Simpler C# with `SortedList`** (allows duplicates via index as tiebreaker):

```csharp
public class Solution {
    public int MinAbsoluteDifference(IList<int> nums, int x) {
        // Use sorted array + binary search (simpler for this problem)
        var window = new List<int>();
        int res = int.MaxValue;

        for (int j = 0; j < nums.Count; j++) {
            if (j >= x) {
                int pos = window.BinarySearch(nums[j-x]);
                if (pos < 0) pos = ~pos;
                window.Insert(pos, nums[j-x]);
            }

            if (window.Count == 0) continue;
            int v = nums[j];
            int idx = window.BinarySearch(v);
            if (idx < 0) idx = ~idx;

            if (idx < window.Count) res = Math.Min(res, window[idx] - v);
            if (idx > 0)            res = Math.Min(res, v - window[idx-1]);
            if (res == 0) return 0;
        }
        return res;
    }
}
```

*Note*: `List.Insert` is O(n) — use a proper balanced BST (`SortedSet` with tiebreaker) for true O(n log n). The `List` version is O(n²) worst case but passes given constraints.

## Pitfalls & Edge Cases

- **Duplicates in `nums`**: `SortedSet` requires unique keys — use `(value, index)` pairs as keys
- **`x=0`**: window starts filling immediately from j=0; but `|i-j|≥0` is always true → every element qualifies
- **Early exit**: if `res==0`, return immediately

---

## Key Takeaway

"Min absolute difference with index distance constraint" → **sliding window of left-side values + binary search for nearest neighbor**:

```
for j: add nums[j-x] to sorted set; query closest to nums[j]
```

The sorted set grows monotonically (we never remove) — predecessor/successor search gives the optimal pair in O(log n).
