# 2624. Snail Traversal
## O(n) ‚Äî Column-by-Column Fill with Direction Flip

---

## Problem Statement

- Extend `Array.prototype` with a `snail(rowsCount, colsCount)` method.
- Transforms a 1D array into a 2D matrix using **snail traversal**: fill column by column, alternating direction (even columns top‚Üíbottom, odd columns bottom‚Üítop).
- Return `[]` if `rowsCount * colsCount !== nums.length`.

---

## Core Idea ‚Äî Iterate by Column, Flip Row Direction on Odd Columns

**Key insight**: The 1D array is consumed sequentially. Each column takes `rowsCount` consecutive elements. The only thing that changes per column is **which row each element lands in**:

- **Even column** (0, 2, 4...): element at local position `row` ‚Üí goes to `res[row][col]` (top‚Üíbottom)
- **Odd column** (1, 3, 5...): element at local position `row` ‚Üí goes to `res[rowsCount-1-row][col]` (bottom‚Üítop)

**Index formula**:
```
flat index  = col * rowsCount + row
target row  = col % 2 === 0 ? row : rowsCount - 1 - row
```

---

## Full Optimal Implementation (JavaScript)

```javascript
Array.prototype.snail = function(rowsCount, colsCount) {
    if (rowsCount * colsCount !== this.length) return [];

    const res = Array.from({ length: rowsCount }, () => new Array(colsCount));

    for (let col = 0; col < colsCount; col++) {
        for (let row = 0; row < rowsCount; row++) {
            const r = col % 2 === 0 ? row : rowsCount - 1 - row;
            res[r][col] = this[col * rowsCount + row];
        }
    }

    return res;
};
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Every element placed exactly once; n = rowsCount √ó colsCount |
| **Space Complexity** | **O(n)** | Output matrix of size n |

## Direction Flip ‚Äî Visual Breakdown

For `rowsCount=5, colsCount=4`, elements are placed as:

```
col 0 (‚Üì):  idx 0‚Üírow0,  idx 1‚Üírow1,  idx 2‚Üírow2,  idx 3‚Üírow3,  idx 4‚Üírow4
col 1 (‚Üë):  idx 5‚Üírow4,  idx 6‚Üírow3,  idx 7‚Üírow2,  idx 8‚Üírow1,  idx 9‚Üírow0
col 2 (‚Üì):  idx 10‚Üírow0, idx 11‚Üírow1, ...
col 3 (‚Üë):  idx 15‚Üírow4, idx 16‚Üírow3, ...
```

The formula `rowsCount - 1 - row` mirrors the row index: `0‚Üî4, 1‚Üî3, 2‚Üî2`.

## Pitfalls & Edge Cases

- **Invalid input check first** ‚Äî `rowsCount * colsCount !== this.length` ‚Üí return `[]`; must be checked before any array access
- **`Array.from({ length: rowsCount }, () => new Array(colsCount))`** ‚Äî creates independent row arrays; using `new Array(rowsCount).fill(new Array(colsCount))` would share the same inner array reference across all rows
- **`this` refers to the array** ‚Äî since this is added to `Array.prototype` as a regular function (not arrow), `this` is the calling array; using an arrow function here would break `this` binding
- **`rowsCount = 1`** ‚Äî both directions are identical (single row); formula still works: `1 - 1 - 0 = 0` ‚úì

---

## Key Takeaway

Snail traversal = **sequential fill, column by column, with mirrored row index on odd columns**:

```javascript
const r = col % 2 === 0 ? row : rowsCount - 1 - row;
res[r][col] = arr[col * rowsCount + row];
```

The direction flip is just an index mirror ‚Äî no need to track state or reverse arrays. One formula handles both directions.
