# 2779. Maximum Beauty of an Array After Applying Operation
## O(n log n) — Sort + Sliding Window

---

## Problem Statement

- Each `nums[i]` can be replaced with any value in `[nums[i]-k, nums[i]+k]`.
- **Beauty** = length of longest subsequence of equal elements.
- Return maximum possible beauty.

---

## Core Idea — Interval Overlap as Sliding Window

Each element `nums[i]` can reach any value in `[nums[i]-k, nums[i]+k]`. Two elements can be made equal iff their reachable intervals **overlap**:

```
[nums[i]-k, nums[i]+k] ∩ [nums[j]-k, nums[j]+k] ≠ ∅
⟺ |nums[i] - nums[j]| ≤ 2k
```

So the problem becomes: find the largest group of elements where all pairwise differences ≤ `2k`.

**After sorting**, this is exactly a sliding window: the window `[l, r]` is valid while `nums[r] - nums[l] ≤ 2k`. Order doesn't matter for subsequences — we just need the count.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumBeauty(int[] nums, int k) {
        Array.Sort(nums);
        int l = 0, res = 0;
        for (int r = 0; r < nums.Length; r++) {
            while (nums[r] - nums[l] > 2 * k) l++;
            res = Math.Max(res, r - l + 1);
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n log n)** — sort dominates |
| **Space** | **O(1)** |

---

## Why Subsequence Not Subarray

The problem asks for a **subsequence** of equal elements — order doesn't matter, just count. After sorting, any subset of elements within a window `[l,r]` forms a valid subsequence of the original array. The window size `r-l+1` directly gives the answer.

---

## Example Walkthrough

**`nums=[4,6,1,2], k=2`** → sorted: `[1,2,4,6]`, threshold = `2k=4`

| r | nums[r] | nums[l] | diff | window | size |
|---|---------|---------|------|--------|------|
| 0 | 1 | 1(l=0) | 0≤4 | [1] | 1 |
| 1 | 2 | 1(l=0) | 1≤4 | [1,2] | 2 |
| 2 | 4 | 1(l=0) | 3≤4 | [1,2,4] | **3** |
| 3 | 6 | 1→2(l=1) | 6-1=5>4→l++; 6-2=4≤4 | [2,4,6] | 3 |

Max = **3** ✓

---

## Pitfalls & Edge Cases

- **`2k` not `k`** — both endpoints can move toward each other by `k`, so total reachable overlap span is `2k`
- **`k=0`** — no changes allowed; beauty = max frequency of any element → sliding window with `nums[r]-nums[l]≤0` handles it correctly (only equal elements in window)

---

## Key Takeaway

"Max elements that can all become equal after ±k adjustment" → **sort + sliding window with gap ≤ 2k**:

```
nums[r] - nums[l] ≤ 2k  →  all elements in window can meet at a common value
```

Reducing interval-overlap problems to sorted sliding window is a common pattern.
