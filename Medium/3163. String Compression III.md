# 3163. String Compression III
## O(n) — Greedy Run-Length Encoding Capped at 9

---

## Problem Statement

- Compress `word`: repeatedly take the longest prefix of a single character (at most 9 characters), append `count + char` to result.
- Return the compressed string.

---

## Core Idea — Greedy Runs Capped at 9

Scan through the string, grouping consecutive identical characters. For each run, output chunks of at most 9 characters at a time.

The cap at 9 (not arbitrary) means we never output multi-digit counts — each chunk produces exactly `"Nc"` where `N ∈ {1..9}`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string CompressedString(string word) {
        var sb = new StringBuilder();
        int i = 0, n = word.Length;
        while (i < n) {
            char c = word[i];
            int j = i;
            while (j < n && word[j] == c && j - i < 9) j++;
            sb.Append(j - i).Append(c);
            i = j;
        }
        return sb.ToString();
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — each character processed once |
| **Space** | **O(n)** — output string |

---

## Example Walkthrough

**`word="aaaaaaaaaaaaaabb"`** (14 a's, 2 b's)

- i=0: c='a', run until j-i=9 → j=9. Append "9a". i=9.
- i=9: c='a', run 5 more a's → j=14. Append "5a". i=14.
- i=14: c='b', run 2 b's → j=16. Append "2b". i=16.

Result: **"9a5a2b"** ✓

---

## Pitfalls & Edge Cases

- **Cap at 9**: the inner `while` condition `j - i < 9` (not `<= 9`) means we take at most 9 chars — correct since `j - i` reaches 9 before we exit.
- **Single unique chars**: each gets "1c" prefix — works naturally.
- **Use `StringBuilder`**: avoids O(n²) string concatenation in C#.

---

## Key Takeaway

"Compress with max-9 run-length encoding" → **greedy inner loop capped at `j - i < 9`**. Each iteration consumes 1–9 characters, producing one `"Nc"` token. O(n) total.
