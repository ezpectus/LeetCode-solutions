# LeetCode 3984. Divisible Game

## Problem Description
Alice and Bob play a game on an array `nums`. Alice chooses an integer `k > 1` and a subarray range `[l, r]`. 
* If `nums[i]` is divisible by `k`, Alice gets `+nums[i]` points.
* Otherwise, Bob gets `+nums[i]` points (which counts as `-nums[i]` for the score difference).

Alice wants to maximize the score difference (Alice's score - Bob's score). If there's a tie, she chooses the smallest `k`. Return `(maximum_score_difference * k) % (10^9 + 7)`.

---

## Method Explanation: Kadane's Algorithm with Smart Parameter Sieve

This problem can be broken down into two components:
1. **Subarray Optimization (Kadane's Algorithm):** For a fixed value of `k`, we can transform each element `nums[i]` into a score value: `+nums[i]` if divisible by `k`, and `-nums[i]` otherwise. Finding the optimal `[l, r]` range on this transformed array is equivalent to the Maximum Subarray Sum problem, solvable in $\mathcal{O}(N)$ using Kadane's algorithm.
2. **Search Space Reduction:** The values in `nums` are bounded by $10^6$. Therefore, we only need to test values of `k` that can actually divide at least one number in `nums` (plus the default baseline `k = 2` if no elements match). We can gather all unique divisors using a sieve-like approach or hash set to minimize the number of candidate `k` values processed.

Given $N \le 1000$, running Kadane's algorithm for each unique candidate `k` is highly efficient and easily passes within the time limit.


### Deep Dive: Algorithmic Synergy (Sieve Concept + Kadane's)

To solve this problem efficiently within the strict execution constraints, we combine two fundamental concepts: **Sieve-like Factor Extraction** and **Kadane's Dynamic Programming Algorithm**.

#### 1. The Core Idea of Kadane's Algorithm
Kadane's algorithm is a classic dynamic programming technique designed to find the maximum subarray sum in a 1D array in linear time, $\mathcal{O}(N)$. 
* **The Mechanism:** It iterates through the array, maintaining a `running_sum` of the current subarray. At each element, it makes a greedy choice: either append the current element to the existing subarray or start a brand new subarray from the current element. If the `running_sum` drops below zero, it resets to zero because a negative prefix can never help maximize a subsequent sum.
* **Application Here:** Once a specific value of $k$ is chosen, every number $nums[i]$ becomes a fixed weight ($+nums[i]$ if divisible, $-nums[i]$ if not). Finding Alice's optimal subarray $[l, r]$ matches the exact design pattern of Kadane's algorithm.

#### 2. The Sieve of Eratosthenes Concept (Search Space Reduction)
The standard Sieve of Eratosthenes marks composite numbers by iterating through multiples of primes. In this problem, we adapt the core mathematical property of the sieve: **highly efficient divisor exploration up to a defined numeric limit $M$ ($10^6$)**.
* **The Mechanism:** Instead of testing every integer $k$ sequentially up to $10^6$ (which causes a massive amount of redundant operations), we use trial division bounded by $\sqrt{num}$ to harvest only the *actual existing divisors* present within our specific input array. 
* This drastically cuts down the search space from $10^6$ potential values to a tiny set of unique valid candidates ($U$).

#### Why a Hybrid Approach is Necessary
A naive brute-force solution checking every possible range $[l, r]$ for every possible $k$ would run in $\mathcal{O}(M \cdot N^2)$ time, resulting in an instant Time Limit Exceeded (TLE) exception.

## C++ Source Code

```cpp
#include <vector>
#include <algorithm>
#include <unordered_set>
#include <cmath>

using namespace std;

class Solution {
public:
    int divisibleGame(vector<int>& nums) {
        int n = nums.size();
        long long MOD = 1e9 + 7;

        // Collect all viable candidate values for k
        // k must divide at least one element in nums, or be the minimum fallback (k = 2)
        unordered_set<int> candidates;
        candidates.insert(2); 

        for (int num : nums) {
            for (int d = 2; d * d <= num; ++d) {
                if (num % d == 0) {
                    candidates.insert(d);
                    candidates.insert(num / d);
                }
            }
            if (num > 1) {
                candidates.insert(num);
            }
        }

        long long max_diff = -1e18; 
        int best_k = 2;

        // For each candidate k, run Kadane's algorithm to find the max subarray score
        for (int k : candidates) {
            long long current_max = -1e18;
            long long running_sum = 0;

            for (int i = 0; i < n; ++i) {
                long long weight = (nums[i] % k == 0) ? nums[i] : -nums[i];
                running_sum += weight;
                
                if (running_sum > current_max) {
                    current_max = running_sum;
                }
                if (running_sum < 0) {
                    running_sum = 0;
                }
            }

            // Track the absolute maximum score difference, picking the smaller k on ties
            if (current_max > max_diff) {
                max_diff = current_max;
                best_k = k;
            } else if (current_max == max_diff) {
                best_k = min(best_k, k);
            }
        }

        // Handle modulo arithmetic safely for potentially negative max_diff values
        long long result = (max_diff % MOD) * (best_k % MOD);
        result = (result % MOD + MOD) % MOD;

        return result;
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(N \cdot \sqrt{M} + U \cdot N)$
  * **Divisor Extraction:** Extracting all valid potential values for $k$ takes $\mathcal{O}(N \cdot \sqrt{M})$ time, where $M$ represents the maximum upper bound value of elements inside the sequence ($10^6$).
  * **Kadane Processing Pass:** For each unique factor $U$ identified, the sequence is evaluated via a linear $\mathcal{O}(N)$ Kadane traversal window. Given $N \le 1000$, this updates rapidly.
  * **Total Time:** $\mathcal{O}(N \cdot \sqrt{M} + U \cdot N)$, which comfortably passes execution requirements.

* **Space Complexity:** $\mathcal{O}(U)$
  * **Candidate Allocation:** Extra workspace allocations scale linearly with the total count of unique numeric factors $U$ retained within the deduplicated `candidates` lookup bucket.
