# 3933. Largest Local Values in a Matrix II
## O(nm log n log m) — 2D Sparse Table + 3-Rectangle Decomposition

---

## Problem Statement

- For a non-zero cell (row,col) with `x = matrix[row][col]`, consider all cells within `x` rows AND `x` columns of (row,col) — except the 4 corner cells where BOTH row-distance and column-distance equal exactly `x`.
- (row,col) is a local maximum if no considered cell has a value greater than `x`.
- Count local maximums. `n,m ≤ 200`, values ≤ 200.

---

## Core Idea — Decompose "Square Minus 4 Corners" into 3 Rectangles

Brute force per cell is O(x²), and x can be up to 200 → worst case O(nm·x²) = 200·200·400² ≈ 6.4×10⁹. Too slow.

**Key geometric observation:** The (2x+1)×(2x+1) square centered at (row,col), minus its 4 corner cells, can be exactly covered by **3 non-overlapping rectangles**:

- **A** = all rows `[row-x, row+x]` × middle columns `[col-x+1, col+x-1]` (size `(2x+1)×(2x-1)`)
- **B** = middle rows `[row-x+1, row+x-1]` × leftmost column `{col-x}` (size `(2x-1)×1`)
- **C** = middle rows `[row-x+1, row+x-1]` × rightmost column `{col+x}` (size `(2x-1)×1`)

Check: `A + B + C = (2x+1)(2x-1) + 2(2x-1) = (2x-1)(2x+3) = 4x²+4x-3 = (2x+1)²-4`. ✓ Exactly matches "square minus 4 corners".

For x ≥ 1, **A always contains (row,col)** itself (since `col ∈ [col-x+1, col+x-1]` requires x≥1), so A is never empty after clipping to matrix bounds.

For each cell, compute `max(A, B, C)` (clipped to matrix bounds, skipping empty rectangles) using a **2D sparse table** for O(1) range-max queries. If this max ≤ x, the cell is a local maximum.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CountLocalMaximums(int[][] matrix) {
        int n = matrix.Length, m = matrix[0].Length;
        int LOG_N = (int)Math.Log2(n) + 1;
        int LOG_M = (int)Math.Log2(m) + 1;

        // st[kr][kc][i][j] = max over rows [i, i+2^kr-1], cols [j, j+2^kc-1]
        var st = new int[LOG_N][][][];
        for (int kr = 0; kr < LOG_N; kr++) {
            st[kr] = new int[LOG_M][][];
            for (int kc = 0; kc < LOG_M; kc++) {
                st[kr][kc] = new int[n][];
                for (int i = 0; i < n; i++) st[kr][kc][i] = new int[m];
            }
        }

        for (int i = 0; i < n; i++)
            for (int j = 0; j < m; j++)
                st[0][0][i][j] = matrix[i][j];

        for (int kr = 1; kr < LOG_N; kr++) {
            int len = 1 << (kr - 1);
            for (int i = 0; i + (1 << kr) <= n; i++)
                for (int j = 0; j < m; j++)
                    st[kr][0][i][j] = Math.Max(st[kr-1][0][i][j], st[kr-1][0][i+len][j]);
        }
        for (int kc = 1; kc < LOG_M; kc++) {
            int len = 1 << (kc - 1);
            for (int kr = 0; kr < LOG_N; kr++)
                for (int i = 0; i < n; i++)
                    for (int j = 0; j + (1 << kc) <= m; j++)
                        st[kr][kc][i][j] = Math.Max(st[kr][kc-1][i][j], st[kr][kc-1][i][j+len]);
        }

        int Query(int r1, int c1, int r2, int c2) {
            int kr = (int)Math.Log2(r2 - r1 + 1);
            int kc = (int)Math.Log2(c2 - c1 + 1);
            int a = st[kr][kc][r1][c1];
            int b = st[kr][kc][r2 - (1<<kr) + 1][c1];
            int c = st[kr][kc][r1][c2 - (1<<kc) + 1];
            int d = st[kr][kc][r2 - (1<<kr) + 1][c2 - (1<<kc) + 1];
            return Math.Max(Math.Max(a, b), Math.Max(c, d));
        }

        int count = 0;
        for (int row = 0; row < n; row++) {
            for (int col = 0; col < m; col++) {
                int x = matrix[row][col];
                if (x == 0) continue;

                int best = -1;

                // A: rows [row-x, row+x], cols [col-x+1, col+x-1]
                {
                    int r1 = Math.Max(0, row-x), r2 = Math.Min(n-1, row+x);
                    int c1 = Math.Max(0, col-x+1), c2 = Math.Min(m-1, col+x-1);
                    if (r1 <= r2 && c1 <= c2)
                        best = Math.Max(best, Query(r1, c1, r2, c2));
                }
                // B: rows [row-x+1, row+x-1], col = col-x
                {
                    int r1 = Math.Max(0, row-x+1), r2 = Math.Min(n-1, row+x-1);
                    int c = col - x;
                    if (r1 <= r2 && c >= 0 && c < m)
                        best = Math.Max(best, Query(r1, c, r2, c));
                }
                // C: rows [row-x+1, row+x-1], col = col+x
                {
                    int r1 = Math.Max(0, row-x+1), r2 = Math.Min(n-1, row+x-1);
                    int c = col + x;
                    if (r1 <= r2 && c >= 0 && c < m)
                        best = Math.Max(best, Query(r1, c, r2, c));
                }

                if (best <= x) count++;
            }
        }

        return count;
    }
}
```

## Example Walkthrough

**Example 2:** `matrix = [[1,2],[3,4]]`

**(1,1)=4, x=4:** A = rows[max(0,-3),min(1,5)]×cols[max(0,-3),min(1,5)] = entire 2×2 matrix, max=4. B: col=1-4=-3 invalid, skip. C: col=1+4=5 invalid, skip. best=4 ≤ 4 → **local max**. ✓

**(0,0)=1, x=1:** A = rows[0,1]×cols[max(0,0),min(1,0)]=col 0 → values {1,3}, max=3. B: col=0-1=-1 invalid. C: col=0+1=1, rows[max(0,0),min(1,0)]=[0,0] → matrix[0][1]=2. best=max(3,2)=3 > 1 → not local max.

Similarly (0,1) and (1,0) are dominated by (1,1)=4 in their windows. **Total = 1** ✓

---

**Example 1:** Single non-zero `2` at (3,3) in a 7×7 grid of zeros, x=2.

A = rows[1,5]×cols[2,4] (5×3 region) → contains (3,3)=2, max=2.
B = rows[2,4]×col{1} → all 0, max=0.
C = rows[2,4]×col{5} → all 0, max=0.

best=2 ≤ 2 → **local max**. **Total = 1** ✓

## Why the 3-Rectangle Decomposition Is Disjoint and Complete

- A excludes columns `col-x` and `col+x` entirely (the leftmost/rightmost columns of the square) — so A can never contain a corner.
- B and C cover exactly those two excluded columns, but only for the **middle rows** `[row-x+1, row+x-1]` — excluding the top and bottom rows where the corners live.
- Together, every cell in the square except the 4 corners is covered exactly once.

For x=1, this decomposition gives the familiar "plus sign" shape: A = the column `{col}` for 3 rows (up, center, down), B = left neighbor, C = right neighbor — exactly the 4-directional neighbors + self from Example 3.

---

## Pitfalls & Edge Cases

- **A is never empty for x ≥ 1** — guaranteed since `col ∈ [col-x+1, col+x-1]`. The cell itself is always in A, so `best` is always set to at least `matrix[row][col] = x`. ✓

- **B/C validity check `r1 <= r2`** — for x=1, middle rows = `[row, row]`, a single row, valid. For x≥1 always `r1 <= r2` after clipping IF row is within [0,n-1] (which it always is). But the column `col±x` may be out of bounds — checked separately. ✓

- **2D sparse table query for single row/column** (size-1 ranges in B, C) — `kr` or `kc` = 0, `(1<<0)=1`, formula degenerates correctly to a direct lookup. ✓

- **`x == 0` cells are skipped** — per problem statement, only non-zero cells can be local maximums. ✓

---

## Key Takeaway

"Square minus corners" range-max queries reduce to **3 disjoint rectangle range-max queries**, each answerable in O(1) via a 2D sparse table. The geometric decomposition is the key insight — once found, the rest is standard 2D RMQ machinery.

**Pattern recognition:** Whenever a problem asks for range-max/min over an irregular shape (square with holes, L-shapes, etc.), look for a decomposition into a small constant number of rectangles, then apply 2D sparse table for O(1) per query.
