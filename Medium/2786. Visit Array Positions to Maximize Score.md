# 2786. Visit Array Positions to Maximize Score
## O(n) — DP on Last Parity

---

## Problem Statement

- Start at index 0, can jump to any later index.
- Collect `nums[i]` at each visited position.
- Penalty `-x` when jumping between different parities.
- Return maximum total score.

---

## Core Idea — DP Keyed by Parity

**State**: `dp[p]` = maximum score achievable ending at any element with parity `p` (0=even, 1=odd), considering all elements up to the current position.

**Transition** for element `nums[i]` with parity `p`:
```
dp[p] = max(
    dp[p]     + nums[i],      // came from same parity — no penalty
    dp[1-p]   + nums[i] - x   // came from opposite parity — penalty x
)
```

Take the max of both options, update `dp[p]`.

**Base case**: `dp[nums[0] % 2] = nums[0]`, other = `-∞` (unreachable start).

**Answer**: `max(dp[0], dp[1])`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaxScore(int[] nums, int x) {
        long[] dp = { long.MinValue / 2, long.MinValue / 2 };
        dp[nums[0] % 2] = nums[0];

        for (int i = 1; i < nums.Length; i++) {
            int p = nums[i] % 2;
            long best = Math.Max(dp[p], dp[1-p] - x) + nums[i];
            dp[p] = Math.Max(dp[p], best);
        }

        return Math.Max(dp[0], dp[1]);
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — single pass |
| **Space** | **O(1)** — only two dp values |

---

## Example Walkthrough

**`nums=[2,3,6,1,9,2], x=5`**

Start: `dp[0]=2, dp[1]=-∞`

| i | nums[i] | p | dp[p] | dp[1-p]-x | best | dp after |
|---|---------|---|-------|-----------|------|----------|
| 1 | 3 | 1 | -∞ | 2-5=-3 | -3 | dp[1]=-3 |
| 2 | 6 | 0 | 2 | -3-5=-8 | 2+6=8 | dp[0]=8 |
| 3 | 1 | 1 | -3 | 8-5=3 | 3+1=4 | dp[1]=4 |
| 4 | 9 | 1 | 4 | 8-5=3 | 4+9=13 | dp[1]=13 |
| 5 | 2 | 0 | 8 | 13-5=8 | 8+2=10 | dp[0]=10 |

`max(10, 13) = 13` ✓

---

## Pitfalls & Edge Cases

- **`long.MinValue / 2`** not `long.MinValue` — subtracting `x` from `MinValue` would overflow; divide by 2 gives a safe sentinel
- **Must start at index 0** — initialize only `dp[nums[0]%2]`, not both
- **Can skip elements** — DP naturally handles this; `dp[p]` carries the best score from any previous same-parity element

---

## Key Takeaway

"Max score path with parity-switch penalty" → **DP on last parity state**:

```
dp[p] = max(dp[p], dp[1-p] - x) + nums[i]
```

Only two states needed — the exact previous position doesn't matter, only its parity. Classic "last state" DP compression.
