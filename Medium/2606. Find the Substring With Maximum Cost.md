# 2606. Find the Substring With Maximum Cost
## O(n) — Kadane's Algorithm

---

## Problem Statement

- You are given string `s`, string `chars` of distinct characters, and integer array `vals` of the same length.
- **Value of a character**: if it's in `chars` → use corresponding `vals[i]`; otherwise → its 1-indexed alphabet position (`'a'`=1 ... `'z'`=26).
- **Cost of a substring** = sum of values of its characters (empty string = 0).
- Return the **maximum cost** among all substrings of `s`.

---

## Core Idea — Kadane's Algorithm

**Key insight**: Build a value map for each character, then the problem reduces to **maximum subarray sum** (Kadane's algorithm) on the mapped values.

**Why Kadane's?**
- Each character maps to a fixed integer value (possibly negative)
- We want the contiguous subarray with maximum sum
- The answer is at least 0 (empty substring is always valid)

**Kadane's with floor at 0**:
- `curr` = best sum ending at current position
- If `curr + v < 0` → reset to 0 (start fresh; the empty substring is better)
- `res = max(res, curr)` after each step

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumCostSubstring(string s, string chars, int[] vals) {
        // Build value lookup for all 26 letters
        int[] val = new int[26];
        for (int i = 0; i < 26; i++) val[i] = i + 1; // default: alphabet position
        for (int i = 0; i < chars.Length; i++)
            val[chars[i] - 'a'] = vals[i];            // override with custom value

        int res = 0, curr = 0;
        foreach (char c in s) {
            curr = Math.Max(0, curr + val[c - 'a']); // Kadane's with floor
            res = Math.Max(res, curr);
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Single pass over `s`; value lookup is O(1) via array |
| **Space Complexity** | **O(1)** | Fixed-size array of 26 values |

**Optimal** — must read each character at least once.

---

## Why This Works — Example Walkthrough

**Example 1**: `s = "adaa"`, `chars = "d"`, `vals = [-1000]`

Value map: `a=1, d=-1000` (rest default)

| c | val | curr = max(0, prev+val) | res |
|---|-----|------------------------|-----|
| a | 1 | max(0, 0+1) = 1 | 1 |
| d | -1000 | max(0, 1-1000) = 0 | 1 |
| a | 1 | max(0, 0+1) = 1 | 1 |
| a | 1 | max(0, 1+1) = 2 | **2** |

Returns `2` ✓ (substring `"aa"`)

**Example 2**: `s = "abc"`, `chars = "abc"`, `vals = [-1,-1,-1]`

Value map: `a=-1, b=-1, c=-1`

| c | val | curr | res |
|---|-----|------|-----|
| a | -1 | 0 | 0 |
| b | -1 | 0 | 0 |
| c | -1 | 0 | 0 |

Returns `0` ✓ (empty substring)

---

## Pitfalls & Edge Cases

- **Floor at 0** — `Math.Max(0, curr + v)` handles the empty substring case; without it Kadane's would return negative values when all chars have negative values
- **Override default values correctly** — build the full 26-letter array first with defaults, then override; avoids per-character branching in the main loop
- **`vals[i]` can be negative** — chars with negative values act as "breaks" in the subarray; Kadane's resets automatically
- **All negative values** — result is 0 (empty substring); the `res = 0` initialization handles this

---

## Key Takeaway

"Maximum cost substring" with fixed character values = **maximum subarray sum** = **Kadane's algorithm**:

- Map each character to its integer value once → O(26)
- Run Kadane's with a floor of 0 (empty substring always valid) → O(n)
- `curr = max(0, curr + val[c])` — reset when going negative, track global max

Pattern: any "maximum sum contiguous subarray" problem → Kadane's. The character-value mapping is just preprocessing.
