# 3805. Count Caesar Cipher Pairs
## O(n · m) — Canonical Form + Frequency Count

---

## Problem Statement

- Given `n` strings each of length `m` (n × m ≤ 10⁵).
- Two strings are similar if one can be obtained from the other by cyclically shifting every character by the same amount k times.
- Count pairs (i, j) with i < j where words[i] and words[j] are similar.

---

## Core Idea — Normalize to a Canonical Form

Two strings s and t are similar iff there exists k such that `t[i] = (s[i] + k) % 26` for all i. This means the pairwise difference `(t[i] − s[i]) mod 26` must be constant for all i.

**Canonical form:** normalize each string by subtracting its first character from all characters (mod 26). This makes the first character always 'a' and captures the relative structure.

```
canonical[i] = (s[i] − s[0] + 26) % 26
```

Two strings are similar ⟺ their canonical forms are identical.

**Algorithm:**
1. Compute canonical form of each word.
2. Count frequency of each canonical form.
3. For each canonical form appearing `c` times: add `c*(c-1)/2` pairs.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long CountPairs(string[] words) {
        var freq = new Dictionary<string, int>();

        foreach (var word in words) {
            int shift = word[0] - 'a';
            var sb = new StringBuilder(word.Length);

            foreach (char c in word) {
                sb.Append((char)('a' + (c - 'a' - shift + 26) % 26));
            }

            string canonical = sb.ToString();
            freq[canonical] = freq.GetValueOrDefault(canonical, 0) + 1;
        }

        long res = 0;
        foreach (int count in freq.Values) {
            res += (long)count * (count - 1) / 2;
        }

        return res;
    }
}
```

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n · m) | n × m ≤ 10⁵ total characters processed |
| Space Complexity | O(n · m) | Dictionary stores all canonical strings |

## Why This Works — Example Walkthrough

**Example 1:** `words = ["fusion", "layout"]`

canonical("fusion"): shift by f=5
```
f→a, u→p, s→n, i→d, o→j, n→i  →  "apndji"
```
canonical("layout"): shift by l=11
```
l→a, a→p, y→n, o→d, u→j, t→i  →  "apndji"
```
Same canonical → 1 pair ✓

---

**Example 2:** `words = ["ab","aa","za","aa"]`

| word | shift | canonical |
|------|-------|-----------|
| "ab" | 0 (a=0) | "ab" |
| "aa" | 0       | "aa" |
| "za" | 25 (z=25) | (z−z)=0→a, (a−z+26)%26=1→b → "ab" |
| "aa" | 0       | "aa" |

freq: {"ab": 2, "aa": 2}

Pairs: 2×1/2 + 2×1/2 = 1 + 1 = **2** ✓

---

## Pitfalls & Edge Cases

- **Shift by 0 ≤ k ≤ 25:** since we take mod 26, all 26 shifts are captured. No range issues. ✓

- **`+ 26` before mod:** prevents negative remainders in `(c - 'a' - shift + 26) % 26`. ✓

- **Single character strings:** canonical is always "a". Any two such strings are similar. ✓

- **All same strings:** c strings of same canonical → c*(c-1)/2 pairs. ✓

- **n*m constraint:** ensures total work is O(n*m) ≤ O(10⁵), not O(n) × O(m) with large individual dimensions.

---

## Key Takeaway

When a problem involves "equivalent under some cyclic transformation," the standard technique is to **normalize to a canonical representative** of the equivalence class.

Here: canonical = shift so first character becomes 'a'. All strings in the same similarity class map to the same canonical → count frequencies, use n*(n−1)/2 formula.

**Pattern recognition:** "Count pairs equivalent under some operation" → canonical form + HashMap frequency count → O(n) pairs counted in one final loop.
