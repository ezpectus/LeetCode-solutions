# 3290. Maximum Multiplication Score
## O(n) — DP with 4 States

---

## Problem Statement

- Choose indices `i0 < i1 < i2 < i3` from `b`. Maximize `a[0]*b[i0] + a[1]*b[i1] + a[2]*b[i2] + a[3]*b[i3]`.

---

## Core Idea — DP on How Many a-Elements Used

`dp[i]` = maximum score using exactly `i+1` elements of `a` paired with some `i+1` elements of `b` seen so far.

Process `b` left to right. For each `b[j]`, update `dp` right to left (to avoid reusing `b[j]`):

```
dp[i] = max(dp[i], dp[i-1] + a[i] * b[j])
```

Base: `dp[-1] = 0` (no elements chosen yet).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaxScore(int[] a, int[] b) {
        long[] dp = new long[4];
        Array.Fill(dp, long.MinValue / 2);

        foreach (int bj in b) {
            for (int i = 3; i >= 0; i--) {
                long prev = i == 0 ? 0L : dp[i-1];
                if (prev > long.MinValue / 2)
                    dp[i] = Math.Max(dp[i], prev + (long)a[i] * bj);
            }
        }
        return dp[3];
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(4n) = O(n)** |
| **Space** | **O(1)** — 4-element dp array |

---

## Example Walkthrough

**`a=[3,2,5,6], b=[2,-6,4,-5,-3,2,-7]`**

After processing all b: dp[3] = 3×2 + 2×(-6) + 5×4 + 6×2 = 6-12+20+12 = **26** ✓

---

## Pitfalls & Edge Cases

- **Right-to-left update**: prevents using the same `b[j]` for multiple `a[i]` in one pass.
- **`long.MinValue/2`**: initial "unset" sentinel; avoids overflow when adding.
- **`(long)a[i] * bj`**: both up to 10^5, product up to 10^10 — needs `long`.

---

## Key Takeaway

"Pick 4 ordered indices to maximize weighted sum" → **DP[i] = best score using i+1 a-elements, update right-to-left per b element**. O(4n) = O(n).
