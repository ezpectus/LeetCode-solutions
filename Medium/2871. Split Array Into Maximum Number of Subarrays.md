# 2871. Split Array Into Maximum Number of Subarrays
## O(n) — Greedy AND Accumulation

---

## Problem Statement

- Split array into subarrays, minimize total score (sum of AND of each subarray).
- Among all minimum-score splits, return the maximum number of subarrays.

---

## Core Idea — AND of Whole Array = Minimum Achievable Score

**Key observation**: AND is monotonically non-increasing as you extend a subarray — adding elements can only clear bits, never set them. Therefore:

- The AND of the **entire array** is the smallest possible AND of any single subarray containing all elements.
- Any split's total score ≥ AND of the full array (since the bits set in the total AND must be set in at least one subarray's AND).

So the minimum achievable total score = **AND of entire array**.

**Two cases**:

**Case 1: `total_and > 0`**
Minimum score = `total_and`. Any split can only increase the total score (subarrays' ANDs ≥ full AND, summing multiple gives ≥ full AND). Only one subarray (the whole array) achieves this minimum → return **1**.

**Case 2: `total_and == 0`**
Minimum score = 0. Greedily cut subarrays as soon as their AND hits 0 — this maximizes the count. The greedy works because ending a subarray at the earliest zero-AND point leaves the remaining array with AND still capable of reaching 0 (guaranteed since total AND = 0).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxSubarrays(int[] nums) {
        // Compute total AND
        int totalAnd = ~0; // all bits set
        foreach (int x in nums) totalAnd &= x;

        if (totalAnd > 0) return 1; // can't do better than whole array

        // Greedy: cut whenever running AND hits 0
        int res = 0, curr = ~0;
        foreach (int x in nums) {
            curr &= x;
            if (curr == 0) {
                res++;
                curr = ~0; // reset
            }
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Two linear passes (or one combined) |
| **Space** | **O(1)** | Only counters |

---

## Why Greedy is Optimal When `total_and == 0`

Claim: cutting as soon as AND hits 0 maximizes the number of subarrays.

Proof sketch: if we delay a cut (extend the current subarray past the first zero-AND point), we only merge two valid subarrays into one, reducing the count. The remaining suffix still has AND = 0 (since total AND = 0 and the current subarray's AND contribution is 0), so it can form at least as many subarrays. Cutting early is always weakly better.

## Example Walkthrough

**`nums=[1,0,2,0,1,2]`**

total_and = 1&0&2&0&1&2 = 0 → greedy case.

| i | nums[i] | curr (before &) | curr (after &) | cut? | res |
|---|---------|----------------|----------------|------|-----|
| 0 | 1 | all-1s | 1 | no | 0 |
| 1 | 0 | 1 | 0 | ✓ | 1 |
| 2 | 2 | all-1s | 2 | no | 1 |
| 3 | 0 | 2 | 0 | ✓ | 2 |
| 4 | 1 | all-1s | 1 | no | 2 |
| 5 | 2 | 1 | 0 | ✓ | 3 |

**3** ✓

## Pitfalls & Edge Cases

- **`~0` initializer**: in C#, `~0` = `-1` (all bits set in 32-bit int) — correctly serves as AND identity
- **`total_and > 0` check first**: if minimum score is non-zero, there's no benefit to splitting (any split would have total score ≥ total_and, and equality only holds for the whole array)
- **Last subarray**: the loop always ends with `curr == 0` when `total_and == 0`, so no remainder handling needed

---

## Key Takeaway

"Maximize splits with minimum AND sum" → **AND of full array = minimum score**; if 0, greedily cut when running AND hits 0:

```
if totalAnd > 0: return 1
else: count resets of running AND to 0 → greedy cuts
```

The AND monotonicity property (adding elements only decreases AND) is the key insight enabling both the minimum-score proof and the greedy approach.
