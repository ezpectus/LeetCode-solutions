# 3937. Minimum Operations to Make Array Modulo Alternating I
## O(n·k + k²) — Per-Element Cost Table + Brute Force Over (x, y) Pairs

---

## Problem Statement

- Choose two distinct remainders `x, y ∈ [0, k)`.
- Every even-indexed element must end up `≡ x (mod k)`, every odd-indexed element `≡ y (mod k)`.
- Each operation changes an element by ±1. Minimize total operations.

---

## Core Idea — Decouple Per-Element Cost from the Choice of (x, y)

**Step 1 — Cost to reach a target remainder:**

For `nums[i]` with `rem = nums[i] % k`, the minimum operations to make it `≡ r (mod k)` is the **circular distance** between `rem` and `r` on a cycle of size k:

```
d = ((r - rem) % k + k) % k
cost(i, r) = min(d, k - d)
```

This works because we can move up by `d` steps or down by `k-d` steps (wrapping around the modular circle) — either direction reaches a number with remainder r.

**Step 2 — Aggregate by parity:**

```
sumEven[r] = Σ cost(i, r) for all even i
sumOdd[r]  = Σ cost(i, r) for all odd i
```

**Step 3 — Brute force over (x, y), x ≠ y:**

```
answer = min over x≠y of (sumEven[x] + sumOdd[y])
```

Since `k ≤ 100`, this is only ~10⁴ pairs.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperations(int[] nums, int k) {
        int n = nums.Length;

        long[] sumEven = new long[k];
        long[] sumOdd = new long[k];

        for (int i = 0; i < n; i++) {
            int rem = nums[i] % k;
            for (int r = 0; r < k; r++) {
                int d = ((r - rem) % k + k) % k;
                int cost = Math.Min(d, k - d);
                if (i % 2 == 0) sumEven[r] += cost;
                else            sumOdd[r]  += cost;
            }
        }

        long best = long.MaxValue;
        for (int x = 0; x < k; x++) {
            for (int y = 0; y < k; y++) {
                if (x == y) continue;
                best = Math.Min(best, sumEven[x] + sumOdd[y]);
            }
        }

        return (int)best;
    }
}
```

## Example Walkthrough

**Example 1:** `nums = [1,4,2,8], k = 3`

Remainders: [1,1,2,2] for indices [0,1,2,3].

`cost(rem, r) = min(|r-rem| mod-circular)`:
- rem=1: cost(0)=1, cost(1)=0, cost(2)=1
- rem=2: cost(0)=1, cost(1)=1, cost(2)=0

Even indices (0,2) have rems (1,2):
- sumEven[0]=1+1=2, sumEven[1]=0+1=1, sumEven[2]=1+0=1

Odd indices (1,3) have rems (1,2):
- sumOdd[0]=1+1=2, sumOdd[1]=0+1=1, sumOdd[2]=1+0=1

Best (x≠y): x=1,y=2 → sumEven[1]+sumOdd[2] = 1+1 = **2** ✓

---

**Example 2:** `nums = [1,1,1], k = 3`

All rems = 1. cost(1, 0)=1, cost(1,1)=0, cost(1,2)=1.

Even indices (0,2): sumEven[0]=2, sumEven[1]=0, sumEven[2]=2.
Odd indices (1): sumOdd[0]=1, sumOdd[1]=0, sumOdd[2]=1.

Best (x≠y): x=1,y=0 → 0+1=1, or x=1,y=2 → 0+1=1. **Output = 1** ✓

## Pitfalls & Edge Cases

- **Modulo of negative numbers in C#:** `(r - rem) % k` can be negative when `r < rem`. The `+ k) % k` correction normalizes it to `[0, k)`. ✓

- **x must differ from y** — explicitly skip `x == y` in the brute force; otherwise both parities target the same remainder, violating "two distinct integers x and y". ✓

- **n = 1 (only even index 0 exists):** sumOdd[r] = 0 for all r. Best = `min over x≠y of sumEven[x] + 0`. Since we need x≠y but y has no cost, pick any y≠x — answer = min(sumEven[x]) over x, achieved by picking y arbitrarily different. ✓ (works correctly since sumOdd is all zeros)

- **Result fits in int** — n,k ≤ 100, max cost per element ≤ k/2=50, total ≤ 100*50=5000. `long` used defensively but `int` would suffice.

---

## Key Takeaway

This is a **decoupled optimization**: the per-element cost depends only on (element, target remainder), and the global choice (x,y) only needs to combine two independent aggregates (sumEven, sumOdd). Brute-forcing the small (x,y) space after precomputing aggregates avoids redundant per-element recomputation.

**Pattern recognition:** "Assign one of two global parameters based on parity/category, minimize total cost" → precompute cost aggregates per category per candidate parameter value, then brute-force the small parameter space.
