# 2938. Separate Black and White Balls
## O(n) — Count Ones Before Each Zero

---

## Problem Statement

- Binary string: `1` = black ball, `0` = white ball.
- Move all whites (0s) to the left, all blacks (1s) to the right using adjacent swaps.
- Return minimum number of swaps.

---

## Core Idea — Each Zero Needs to Pass All Ones to Its Left

For every `0` encountered, it must move left past all `1`s that appeared before it. The number of swaps needed for that `0` = number of `1`s seen so far (to its left).

Scan left to right: maintain a counter `ones` of `1`s seen. Each time we see a `0`, add `ones` to the result.

**Why this is optimal?** Each `0` needs exactly `ones_before_it` swaps to reach its final sorted position. Swaps don't interfere with each other's counts since we're counting by relative inversions (0,1 pairs where 0 is to the right).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinimumSteps(string s) {
        long res = 0;
        int ones = 0;
        foreach (char c in s) {
            if (c == '1') ones++;
            else res += ones; // this 0 must pass all 1s to its left
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

---

## Example Walkthrough

**`s="100"`**

| i | c | ones | res |
|---|---|------|-----|
| 0 | 1 | 1 | 0 |
| 1 | 0 | 1 | 1 |
| 2 | 0 | 1 | 2 |

**2** ✓ — first `0` passes 1 one, second `0` also passes 1 one.

---

## Connection to Inversions

The answer = number of (1, 0) pairs where `1` appears before `0` in the string — exactly the count of **inversions** between blacks and whites. Each inversion requires one adjacent swap to resolve.

---

## Pitfalls & Edge Cases

- **`long` for result**: up to `n/2` ones and `n/2` zeros → max result ≈ `(n/2)²/2 = n²/8`. With `n=10⁵`: ~1.25×10⁹ which fits in `int`, but using `long` is safer
- **Already sorted** (`"0111"`): no zeros after ones → `res = 0` ✓

---

## Key Takeaway

"Minimum swaps to sort binary string (0s left, 1s right)" → **count inversions = sum of ones seen before each zero**. One pass, O(1) extra space.
