# 2895. Minimum Processing Time
## O(n log n) — Greedy: Sort Both, Match Earliest Processor to Longest Tasks

---

## Problem Statement

- `n` processors, each with 4 cores. `4n` tasks total, each assigned to a unique core.
- Processor `i` becomes available at `processorTime[i]`. All 4 tasks assigned to it start at that time.
- Finish time for a processor = `processorTime[i] + max(4 assigned tasks)`.
- Minimize the **overall** finish time = max over all processors.

---

## Core Idea — Greedy Pairing by Sorting Both

**Goal**: minimize `max(processorTime[i] + max_task_in_group_i)`.

**Key insight**: To minimize the maximum, pair the **earliest available processor** with the **4 longest tasks**. This is the standard "schedule longest job on earliest machine" greedy.

**Why?** The earliest processor has the smallest `processorTime[i]`. Giving it the 4 longest tasks minimizes the penalty of those tasks — they get added to the smallest base time. If instead we gave long tasks to a later processor, we'd add them to a larger base, potentially increasing the maximum.

**After sorting**:
- `processorTime` ascending → processor `i` gets tasks `[4i, 4i+1, 4i+2, 4i+3]` (0-indexed) from the tasks array
- `tasks` descending → group 0 has the 4 longest, group 1 has the next 4, etc.
- Finish time for processor `i` = `processorTime[i] + tasks[4i]` (only the max matters per group)

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinProcessingTime(IList<int> processorTime, IList<int> tasks) {
        var procs = processorTime.OrderBy(x => x).ToArray();
        var sorted = tasks.OrderByDescending(x => x).ToArray();

        int res = 0;
        for (int i = 0; i < procs.Length; i++)
            res = Math.Max(res, procs[i] + sorted[4 * i]);

        return res;
    }
}
```

## Example Walkthrough

**Example 1**: `processorTime=[8,10], tasks=[2,2,3,1,8,7,4,5]`

Sorted processors: `[8, 10]`
Sorted tasks (desc): `[8, 7, 5, 4, 3, 2, 2, 1]`

- Processor 0 (time=8): tasks `[8,7,5,4]` → finish = 8+8 = **16**
- Processor 1 (time=10): tasks `[3,2,2,1]` → finish = 10+3 = 13

`max(16,13) = 16` ✓

---

## Pitfalls & Edge Cases

- **Only the maximum task per group matters**: `processorTime[i] + tasks[4i]` since tasks are sorted descending, `tasks[4i]` is the largest in its group
- **`tasks.length == 4 * n`** guaranteed: no remainder handling needed
- **All processors available at same time**: reduces to minimizing `max_task_in_group`, which sorting still handles correctly

---

## Key Takeaway

"Assign k tasks to each machine, minimize max finish time" → **sort machines ascending, sort tasks descending, greedily pair i-th earliest machine with i-th group of k largest remaining tasks**:

```
finish[i] = processorTime[i] + tasks[k*i]   (k=4 here)
answer    = max(finish[i])
```

Earliest processor gets longest tasks — reduces their large values by the smallest base time. Classic rearrangement inequality application.
