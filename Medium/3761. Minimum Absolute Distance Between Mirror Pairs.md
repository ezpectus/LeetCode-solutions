# 3761. Minimum Absolute Distance Between Mirror Pairs
## O(n) — Hash Map of Reversed Values

---

## Problem Statement

- Pair `(i, j)` with `i < j` is a mirror pair if `reverse(nums[i]) == nums[j]`.
- Return minimum `|i - j|` across all mirror pairs, or -1 if none.

---

## Core Idea — For Each j, Look Up i Where rev(nums[i]) == nums[j]

When processing index `j`, we want to find the **most recently seen** index `i < j` such that `reverse(nums[i]) == nums[j]`.

Maintain a map `rev_map` where `rev_map[v]` = list of indices seen so far whose `reverse(nums[i]) == v`. When processing `j`:
- Check if `nums[j]` is in `rev_map` → the last entry in `rev_map[nums[j]]` gives the nearest `i`.
- Then store `reverse(nums[j])` mapped to index `j` for future lookups.

**Why the most recent?** Minimum distance = largest `i` less than `j` → last element in the list.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinMirrorPairDistance(int[] nums) {
        int Reverse(int x) {
            int rev = 0;
            while (x > 0) { rev = rev*10 + x%10; x /= 10; }
            return rev;
        }

        var revMap = new Dictionary<int, int>(); // rev(nums[i]) -> most recent index i
        int res = int.MaxValue;

        for (int j = 0; j < nums.Length; j++) {
            int x = nums[j];
            // Find i<j where rev(nums[i]) == x
            if (revMap.TryGetValue(x, out int i))
                res = Math.Min(res, j - i);
            // Store: rev(nums[j]) -> j (overwrite with most recent)
            revMap[Reverse(x)] = j;
        }

        return res == int.MaxValue ? -1 : res;
    }
}
```

**Note**: storing only the most recent index (overwriting) is correct because we always want the nearest `i` for minimum distance.

## Example Walkthrough

**`nums=[12,21,45,33,54]`**

| j | x=nums[j] | rev(x) | revMap before | match? | revMap after |
|---|-----------|--------|---------------|--------|--------------|
| 0 | 12 | 21 | {} | No | {21:0} |
| 1 | 21 | 12 | {21:0} | Yes! i=0, dist=1 | {21:0, 12:1} |
| 2 | 45 | 54 | ... | No | +{54:2} |
| 3 | 33 | 33 | ... | No | +{33:3} |
| 4 | 54 | 45 | {54:2,...} | Yes! i=2, dist=2 | overwrite |

min(1, 2) = **1** ✓

---

## Why Palindromic Numbers Work

For `nums[i] = 33` (palindrome): `reverse(33) = 33`. So `revMap[33] = i`. A future `nums[j] = 33` will find `revMap[33]` and form a valid pair. This handles palindromic values naturally.

---

## Pitfalls & Edge Cases

- **`reverse(120) = 21`** (not 021): integer reversal drops leading zeros. Handle by integer arithmetic or `int.Parse(reversed_string)`.
- **Overwrite in revMap**: storing only the most recent index is correct — we want minimum distance, which comes from the closest previous `i`.
- **`rev(nums[i]) == nums[j]` is NOT symmetric**: `rev(21)=12≠120`, so `[21,120]` has no mirror pair, while `[120,21]` does. The direction matters.

---

## Key Takeaway

"Min distance between mirror pairs" → **hash map of rev(nums[i]) → most recent index; for each j, look up nums[j]**:

```
for j: if nums[j] in revMap: res = min(res, j - revMap[nums[j]])
        revMap[rev(nums[j])] = j   // overwrite with most recent
```

O(n) single pass. The overwrite trick keeps only the closest previous candidate.
