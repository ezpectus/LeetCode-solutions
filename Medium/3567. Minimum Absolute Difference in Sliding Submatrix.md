# 3567. Minimum Absolute Difference in Sliding Submatrix
## O(m × n × k² log k) — Brute Force with Sort

---

## Problem Statement

- For each `k×k` submatrix, find the minimum absolute difference between any two **distinct** values.
- Return the `(m-k+1) × (n-k+1)` result matrix.

---

## Core Idea — Collect, Deduplicate, Sort, Min Adjacent Gap

For each top-left corner `(i,j)`:
1. Collect all `k²` values from the submatrix
2. Deduplicate (only distinct values matter)
3. Sort
4. Minimum diff = minimum gap between consecutive sorted values

If only one distinct value → answer is 0.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[][] MinAbsDiff(int[][] grid, int k) {
        int m = grid.Length, n = grid[0].Length;
        int[][] res = new int[m-k+1][];

        for (int i = 0; i <= m-k; i++) {
            res[i] = new int[n-k+1];
            for (int j = 0; j <= n-k; j++) {
                var vals = new SortedSet<int>();
                for (int di = 0; di < k; di++)
                    for (int dj = 0; dj < k; dj++)
                        vals.Add(grid[i+di][j+dj]);

                if (vals.Count == 1) { res[i][j] = 0; continue; }

                int minDiff = int.MaxValue;
                int prev = vals.Min;
                foreach (int v in vals.Skip(1)) {
                    minDiff = Math.Min(minDiff, v - prev);
                    prev = v;
                }
                res[i][j] = minDiff;
            }
        }
        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O((m-k)(n-k) × k² log k)** | Each submatrix: k² inserts into SortedSet |
| **Space** | **O(k²)** | Set per submatrix |

With m,n≤30 and k≤30: at most 30×30×900×log(900) ≈ 7M ops — trivially fast.

---

## Why Sort + Adjacent Gap

After sorting distinct values `[v₀, v₁, ..., vₜ]`, the minimum absolute difference is always between two consecutive elements. Any non-adjacent pair `|vᵢ - vⱼ|` with `j > i+1` is larger since `vⱼ > vᵢ₊₁ > vᵢ`.

**`SortedSet`** automatically deduplicates and keeps sorted order — two birds with one stone.

---

## Pitfalls & Edge Cases

- **`k=1`**: single element per submatrix → always 0 ✓ (`vals.Count == 1`)
- **All same values** → `vals.Count == 1` → return 0 ✓
- **Negative values**: `SortedSet` sorts correctly; differences are always non-negative since we iterate in order

---

## Key Takeaway

Small constraints (m,n,k≤30) → brute force is fine. Collect k² elements, deduplicate + sort → scan adjacent pairs for minimum gap. `SortedSet` handles both dedup and sorting in one structure.
