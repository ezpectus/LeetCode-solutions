# 3703. Remove K-Balanced Substrings
## O(n) Amortized — Stack with Consecutive Count Compression

---

## Problem Statement

- You are given a string `s` (2 ≤ length ≤ 10⁵) consisting only of '(' and ')'.
- You are given an integer `k` (1 ≤ k ≤ length/2).
- A k-balanced substring is exactly k consecutive '(' followed by k consecutive ')': "(" * k + ")" * k.
- Repeatedly remove all non-overlapping k-balanced substrings until none exist.
- Return the final string after all possible removals.

---

## Core Idea — Stack with Compressed Counts

Key insight: Instead of rescanning the string multiple times, use a stack that tracks consecutive character counts. When we accumulate k consecutive '(' followed by k consecutive ')', remove them immediately. This handles cascading removals in a single pass.

Approach:
- Maintain parallel stacks: one for characters, one for their consecutive counts
- For '(': merge with previous '(' if exists, else push new entry
- For ')': accumulate consecutive ')', then greedily match with '(' groups of size ≥ k
- The matching loop handles cascading: after one removal, check if more removals are possible

This achieves O(n) amortized time by processing each character once.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string RemoveSubstring(string s, int k) {
        var stack = new Stack<char>();
        var count = new Stack<int>(); // count[i] = consecutive count of stack[i]
        
        foreach (char ch in s) {
            if (ch == '(') {
                if (stack.Count > 0 && stack.Peek() == '(') {
                    count.Push(count.Pop() + 1);
                } else {
                    stack.Push('(');
                    count.Push(1);
                }
            } else { // ')'
                int closeCount = 1;
                
                // Merge with existing consecutive ')'
                if (stack.Count > 0 && stack.Peek() == ')') {
                    closeCount += count.Pop();
                    stack.Pop();
                }
                
                // Greedily match with '(' groups
                while (closeCount >= k && stack.Count > 0 && stack.Peek() == '(') {
                    int openCount = count.Pop();
                    stack.Pop();
                    
                    if (openCount >= k) {
                        // Remove k '(' and k ')'
                        openCount -= k;
                        closeCount -= k;
                        
                        // Put back remaining '(' if any
                        if (openCount > 0) {
                            stack.Push('(');
                            count.Push(openCount);
                        }
                    } else {
                        // Not enough '(' to match, restore and break
                        stack.Push('(');
                        count.Push(openCount);
                        break;
                    }
                }
                
                // Add remaining ')' if any
                if (closeCount > 0) {
                    stack.Push(')');
                    count.Push(closeCount);
                }
            }
        }
        
        // Build result from stack
        var res = new StringBuilder();
        var chars = stack.ToArray();
        var counts = count.ToArray();
        
        Array.Reverse(chars);
        Array.Reverse(counts);
        
        for (int i = 0; i < chars.Length; i++) {
            res.Append(chars[i], counts[i]);
        }
        
        return res.ToString();
    }
}
```

Optimal — single pass solution with immediate removal, avoiding O(n²) repeated scans.

## Why This Works — Example Walkthrough

**Example 1:** `s = "(())", k = 1`

Processing step-by-step:
1. '(' → stack: ['('], count: [1]
2. '(' → stack: ['('], count: [2] (merged)
3. ')' → closeCount=1, top is ('(', 2)
   - openCount=2 ≥ k=1 → remove k=1: openCount=1, closeCount=0
   - Remaining: stack: ['('], count: [1]
4. ')' → closeCount=1, top is ('(', 1)
   - openCount=1 ≥ k=1 → remove k=1: openCount=0, closeCount=0
   - Remaining: stack: [], count: []

**Output:** "" ✓

---

**Example 2:** `s = "(()(", k = 1`

1. '(' → stack: ['('], count: [1]
2. '(' → stack: ['('], count: [2]
3. ')' → closeCount=1, top is ('(', 2)
   - openCount=2 ≥ k=1 → remove k=1: openCount=1, closeCount=0
   - stack: ['('], count: [1]
4. '(' → stack: ['('], count: [2]

**Output:** "((" ✓

---

**Example 3:** `s = "((()))()()()", k = 3`

1. Process "(((" → stack: ['('], count: [3]
2. Process ")))" → closeCount accumulates to 3
   - top is ('(', 3), 3 ≥ k=3 → remove 3 and 3 → stack: []
3. Process "()" → stack: ['('], count: [1] then ')' closeCount=1
   - openCount=1 < k=3, can't match → stack: ['(', ')'], count: [1, 1]
4. Process "()" → same pattern
5. Process "()" → same pattern

Final stack builds: "()()()"

**Output:** "()()()" ✓

## Pitfalls & Edge Cases

- **Cascading removals** — The while loop automatically handles cases where removing one k-balanced pattern exposes another. For example, "((()))" with k=1 removes middle "()", then outer "()", then final "()".

- **Stack compression** — By storing counts instead of individual characters, we reduce space and enable O(1) merging of consecutive same characters.

- **Greedy matching** — We match as many k-balanced patterns as possible when processing each ')'. This ensures we don't miss any removals.

- **Time complexity proof** — Each character is pushed onto the stack at most once and popped at most once. The while loop may pop multiple stack entries, but total pops across all iterations is bounded by total pushes = O(n).

- **Edge case: empty result** — Perfectly valid, stack ends up empty (Example 1).

- **Edge case: no removals** — If no k-balanced patterns exist, the stack just builds the original string (e.g., "(((" or "((" with k=1).

- **Large k** — Works efficiently; fewer matches mean faster processing.

## Key Takeaway

This is a **stack-based greedy matching** problem optimized with count compression:
- Compress consecutive identical characters into (char, count) pairs
- For each ')', accumulate its count and greedily match with '(' groups
- The matching loop handles cascading removals automatically
- Achieves O(n) by processing each character exactly once

**Pattern recognition:** "Repeatedly remove substring pattern" + linear structure → stack with immediate removal for O(n) instead of O(n²) repeated scans.

The critical optimization over the naive approach:
- **Naive**: Scan entire string, remove one pattern, repeat → O(n² / k) worst case
- **Optimal**: Stack with counts, match immediately → O(n) amortized

This pattern applies to many bracket/parenthesis matching problems where immediate removal on the stack is possible.
