# 3016. Minimum Number of Pushes to Type Word II
## O(n + 26 log 26) — Greedy: Assign Most Frequent Letters to First Positions

---

## Problem Statement

- 8 keys (2-9), each can hold any number of letters.
- Typing a letter costs `pos` pushes where `pos` = its position on its key (1st letter on key = 1 push, 2nd = 2 pushes, etc.).
- Minimize total pushes to type the given word.

---

## Core Idea — Sort by Frequency, Assign Greedily

**Key insight**: there are 8 keys, so the first 8 most-frequent letters each get "position 1" (1 push each). The next 8 most-frequent get "position 2" (2 pushes each), and so on.

This is optimal by a greedy exchange argument: if a less-frequent letter occupies a cheaper slot than a more-frequent letter, swapping them reduces total cost.

**Algorithm**:
1. Count frequency of each distinct letter in `word`.
2. Sort frequencies descending.
3. Letter at sorted index `i` (0-based) costs `(i / 8) + 1` pushes per occurrence.
4. Total = `Σ freq[i] × (i/8 + 1)`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumPushes(string word) {
        int[] freq = new int[26];
        foreach (char c in word) freq[c - 'a']++;

        Array.Sort(freq);
        // Reverse: process largest first
        int res = 0;
        for (int i = 0; i < 26; i++) {
            int rank = 25 - i;                   // 0-indexed from largest
            int pushes = rank / 8 + 1;           // 1 for first 8, 2 for next 8, etc.
            res += freq[i] * pushes;
        }
        return res;
    }
}
```

Alternatively, sort descending and iterate forward:

```csharp
public class Solution {
    public int MinimumPushes(string word) {
        int[] freq = new int[26];
        foreach (char c in word) freq[c - 'a']++;

        var sorted = freq.OrderByDescending(f => f).ToArray();
        int res = 0;
        for (int i = 0; i < 26; i++)
            res += sorted[i] * (i / 8 + 1);
        return res;
    }
}
```

## Why `(i / 8 + 1)` is the Cost

There are 8 keys (2 through 9). Letters are assigned in order of decreasing frequency:

| Index range (0-based) | Keys used | Position on key | Push count |
|----------------------|-----------|-----------------|------------|
| 0–7 | 2,3,4,5,6,7,8,9 | 1st letter | **1** |
| 8–15 | 2,3,4,5,6,7,8,9 | 2nd letter | **2** |
| 16–23 | 2,3,4,5,6,7,8,9 | 3rd letter | **3** |
| 24–25 | 2,3 | 4th letter | **4** |

Formula: letter at 0-based index `i` → `i / 8 + 1` pushes (integer division).

## Greedy Exchange Argument

Suppose letter A (freq `fA`) is at index `i` (cost `pA = i/8+1`) and letter B (freq `fB`) is at index `j > i` (cost `pB = j/8+1 ≥ pA`), with `fA < fB`.

Current cost contribution: `fA × pA + fB × pB`
After swap: `fA × pB + fB × pA`

Difference = `(fB - fA) × (pA - pB) ≤ 0` (since `fB > fA` and `pA ≤ pB`).

Swapping reduces or maintains cost → always optimal to put the most frequent letters at the cheapest positions. ✓

---

## Pitfalls & Edge Cases

- **Letters not in word**: their frequency is 0, contributing 0 to the total — no special handling needed
- **All same letter** (e.g., "aaaa"): one non-zero freq at index 0 → 1 push per 'a' × count
- **26 distinct letters**: slots 0–7 cost 1 push, 8–15 cost 2, 16–23 cost 3, 24–25 cost 4 — all covered by the formula

---

## Key Takeaway

"Assign letters to keypad to minimize total pushes" → **sort by frequency descending, assign index `i` a cost of `i/8 + 1`**:

```
sort frequencies descending
result = Σᵢ freq[i] × (i/8 + 1)
```

8 keys × multiple positions per key → 8 letters share each "push level". Most frequent letters always go first to minimize weighted sum.
