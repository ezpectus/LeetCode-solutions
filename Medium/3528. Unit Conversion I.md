# 3528. Unit Conversion I
## O(n log MOD) — BFS from Node 0 with Modular Inverse

---

## Problem Statement

- Tree of `n` unit types rooted at 0. Edge `u→v` with factor `f` means 1 unit of `u` = `f` units of `v`. Find units of each type equivalent to 1 unit of type 0.

---

## Core Idea — BFS Propagation with Forward/Reverse Edge Handling

BFS from node 0. Maintain `res[u]` = units of type `u` per 1 unit of type 0.

For each edge from known node `u` to unknown node `v`:
- **Forward edge** (original `u→v`, factor `f`): `res[v] = res[u] × f`
- **Reverse edge** (original `v→u`, factor `f`): 1 unit of `v` = `f` units of `u`, so `res[v] = res[u] / f = res[u] × f⁻¹ mod MOD`

Store both directions in adjacency list, tagged with forward/reverse.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    const long MOD = 1_000_000_007;

    public int[] BaseUnitConversions(int[][] conversions) {
        int n = conversions.Length + 1;
        var graph = new List<(int to, long factor, bool fwd)>[n];
        for (int i = 0; i < n; i++) graph[i] = new();
        foreach (var c in conversions) {
            graph[c[0]].Add((c[1], c[2], true));
            graph[c[1]].Add((c[0], c[2], false));
        }

        long[] res = new long[n];
        res[0] = 1;
        bool[] visited = new bool[n];
        visited[0] = true;
        var q = new Queue<int>();
        q.Enqueue(0);

        while (q.Count > 0) {
            int u = q.Dequeue();
            foreach (var (v, factor, fwd) in graph[u]) {
                if (visited[v]) continue;
                visited[v] = true;
                res[v] = fwd
                    ? res[u] * factor % MOD
                    : res[u] * ModInv(factor) % MOD;
                q.Enqueue(v);
            }
        }
        return Array.ConvertAll(res, x => (int)x);
    }

    long ModInv(long a) => ModPow(a, MOD-2);
    long ModPow(long b, long e) {
        long r = 1; b %= MOD;
        for (; e > 0; e >>= 1) { if ((e&1)==1) r=r*b%MOD; b=b*b%MOD; }
        return r;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n log MOD)** | n BFS steps; modular inverse is O(log MOD) ≈ 30 ops |
| **Space** | **O(n)** | Graph + result array |

## Example Walkthrough

**`conversions=[[0,1,2],[0,2,3],[1,3,4]]`**

BFS from 0: res=[1,0,0,0].
- Edge 0→1 (fwd, f=2): res[1]=1×2=2.
- Edge 0→2 (fwd, f=3): res[2]=1×3=3.
- Edge 1→3 (fwd, f=4): res[3]=2×4=8.

Answer: [1,2,3,8] ✓

---

## Pitfalls & Edge Cases

- **Reverse edges**: the problem guarantees tree + unique conversion path, but conversions may not be listed root-to-leaf. Reverse edge handling with modular inverse covers all cases.
- **`factor` can be up to 10⁹**: intermediate product `res[u] × factor` needs `long` (up to ~10¹⁸ before mod).
- **Fermat's little theorem**: `a⁻¹ ≡ a^(MOD-2) (mod MOD)` works since MOD is prime.

---

## Key Takeaway

"Unit conversions in a tree from root" → **BFS from node 0, multiply by factor for forward edges, multiply by modular inverse for reverse edges**. O(n log MOD).
