# 2849. Determine if a Cell Is Reachable at a Given Time
## O(1) — Chebyshev Distance

---

## Problem Statement

- 8-directional movement (including diagonals). Move exactly `t` steps.
- Can revisit cells. Return true if `(fx,fy)` is reachable in exactly `t` steps.

---

## Core Idea — Chebyshev Distance

With 8-directional movement, the minimum steps to go from `(sx,sy)` to `(fx,fy)` is the **Chebyshev distance**:

```
minDist = max(|fx-sx|, |fy-sy|)
```

Diagonal moves cover both axes simultaneously. After reaching the target in `minDist` steps, we can waste extra steps by oscillating between two adjacent cells — so any `t ≥ minDist` works.

**Special case**: `sx==fx && sy==fy` (same cell). If `t==0` → already there ✓. If `t==1` → must move, can't return in 1 step → **false**. If `t≥2` → move to neighbor and back ✓.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool IsReachableAtTime(int sx, int sy, int fx, int fy, int t) {
        int dx = Math.Abs(fx - sx), dy = Math.Abs(fy - sy);

        if (dx == 0 && dy == 0)
            return t == 0 || t >= 2;

        return t >= Math.Max(dx, dy);
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time / Space** | **O(1)** |

---

## Why Chebyshev Distance

In a grid with 8-directional movement, a diagonal step moves 1 in both x and y simultaneously. So to cover `dx` horizontal and `dy` vertical distance: use `min(dx,dy)` diagonal steps, then `|dx-dy|` straight steps → total = `max(dx,dy)`.

Any extra steps can be spent by moving to an adjacent cell and back (2 steps per "wasted" cycle), so `t ≥ minDist` suffices.

---

## Pitfalls & Edge Cases

- **Same cell, `t=1`**: must move away but can't return in 1 step → **false**
- **Same cell, `t=0`**: already there → **true**
- **Same cell, `t≥2`**: move to neighbor and back → **true**

---

## Key Takeaway

8-directional grid reachability → **Chebyshev distance = `max(|dx|, |dy|)`**. Reachable iff `t ≥ minDist`, with special case for same cell and `t=1`.
