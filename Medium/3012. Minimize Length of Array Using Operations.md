# 3012. Minimize Length of Array Using Operations
## O(n) — Two Cases Based on Whether Min Divides Everything

---

## Problem Statement

- Operation: pick two positive elements, insert their remainder, delete both (net: -1 element).
- Minimize array length.

---

## Core Idea — Two Cases

Let `min_val = min(nums)`.

### Case 1: `min_val` does NOT divide some element

If some `x % min_val != 0`, we can generate a value smaller than `min_val` (since `x % min_val < min_val`). By applying Euclidean-like reductions, we can eventually generate `1` (or the GCD). With `1` in the array, any positive element `v` satisfies `1 % v = 1` — this removes `v` and keeps `1`. We can eliminate all elements one by one → **answer = 1**.

### Case 2: `min_val` divides ALL elements

Every element is `k × min_val` for some integer `k`. Operations on multiples of `min_val` always produce multiples of `min_val` — we can never go below `min_val`. The `min_val` copies are the "atoms".

**Strategy**: use copies of `min_val` to eliminate larger elements:
- `min_val % larger = min_val` (since `larger > min_val`): removes `larger`, keeps `min_val`. Net: -1 element, same count of `min_val`.
- After eliminating all non-`min_val` elements, pair up `min_val` copies: `min_val % min_val = 0`, each pair → one zero.

→ **Answer = `ceil(count(min_val) / 2)`**

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumArrayLength(int[] nums) {
        int minVal = nums.Min();
        
        // Check if minVal divides all elements
        foreach (int x in nums)
            if (x % minVal != 0) return 1;
        
        // Count occurrences of minVal
        int count = nums.Count(x => x == minVal);
        return (count + 1) / 2;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

---

## Why Case 2 Gives `ceil(count_min / 2)`

With all multiples of `min_val`, normalize to q-values: `q[i] = nums[i] / min_val`. Now q-min = 1.

**Eliminating non-1 q-values**: `1 % q = 1` for any `q > 1`. This op removes both 1 and q from the array and adds 1 back → net: loses one non-1 element, count of 1s unchanged.

After eliminating all non-1s: left with `count_min` copies of 1 (= `min_val`).

**Pairing 1s**: `1 % 1 = 0`. Each pair → one zero (dead element). Net: floor(`count_min / 2`) ops, leaving `ceil(count_min / 2)` elements.

---

## Example Walkthrough

**`[5,5,5,10,5]`**: min=5. 10%5=0 ✓, all divisible. count(5)=4. `ceil(4/2)=2` ✓

Execution: `5 % 10 = 5` → [5,5,5,5]. `5%5=0` × 2 → [0,0]. Length=2. ✓

**`[2,3,4]`**: min=2. `3%2=1 ≠ 0` → not all divisible → **1** ✓

Execution: `3%4=3` → [2,3]. `3%2=1` → [1]. Length=1. ✓

---

## Pitfalls & Edge Cases

- **All elements equal to min**: all divisible by min; count = n; answer = `ceil(n/2)`
- **Only one copy of min**: `ceil(1/2) = 1` ✓ (use min to eliminate everyone: `min % x = min`, net -1 per op)
- **Large arrays**: O(n) is needed — avoid O(n²) gcd checking

---

## Key Takeaway

"Minimize array via modulo operations" → **check if min divides everything**:

```
if any x % min != 0: return 1   // can generate smaller value, reduce to 1
else: return ceil(count(min) / 2) // min is irreducible atom, pair copies
```

The `min % larger = min` trick is the key: min_val can eliminate any larger multiple for free (net -1 element, no loss of min_val copies).
