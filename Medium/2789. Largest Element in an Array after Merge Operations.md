# 2789. Largest Element in an Array after Merge Operations
## O(n) — Greedy Right-to-Left Accumulation

---

## Problem Statement

- Merge `nums[i]` into `nums[i+1]` (add them) only if `nums[i] <= nums[i+1]`.
- Return the maximum value achievable in the final array.

---

## Core Idea — Scan Right to Left, Accumulate Greedily

**Key insight**: to maximize the final element, we want to merge as many elements as possible into one. Scanning right-to-left, maintain a running sum `curr` starting from the last element. If `nums[i] <= curr`, we can merge `nums[i]` into the growing sum (since after merging everything to the right, `curr` represents the merged block). Otherwise, start fresh from `nums[i]`.

**Why right-to-left?** The merge operation merges `nums[i]` into `nums[i+1]`. Once we've merged everything from `i+1` to the right into a block of value `curr`, we can merge `nums[i]` into it if `nums[i] <= curr`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaxArrayValue(int[] nums) {
        long curr = nums[^1], res = curr;
        for (int i = nums.Length - 2; i >= 0; i--) {
            curr = nums[i] <= curr ? curr + nums[i] : nums[i];
            res = Math.Max(res, curr);
        }
        return res;
    }
}
```

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

---

## Example Walkthrough

**`nums=[2,3,7,9,3]`**, scan right to left:

| i | nums[i] | curr | condition | new curr |
|---|---------|------|-----------|----------|
| 3 | 9 | 3 | 9>3 → reset | 9 |
| 2 | 7 | 9 | 7≤9 → merge | 16 |
| 1 | 3 | 16 | 3≤16 → merge | 19 |
| 0 | 2 | 19 | 2≤19 → merge | 21 |

Max = **21** ✓

---

## Pitfalls & Edge Cases

- **`long` for curr and res** — values accumulate up to `n × max(nums[i]) = 10⁵ × 10⁶ = 10¹¹`, overflows `int`
- **Single element** — loop doesn't execute, returns `nums[0]` ✓
- **Descending array** (`[5,3,3]`)  — `5 > curr=6`? No, `3≤3` first, then `5≤6` → merges to 11 ✓

---

## Key Takeaway

"Max element after conditional left-absorb merges" → **right-to-left greedy accumulation**:

```
curr = nums[i] <= curr ? curr + nums[i] : nums[i]
```

Scan from right, accumulate while can merge. The greedy is optimal because merging into a larger block only makes future merges easier (larger right side).
