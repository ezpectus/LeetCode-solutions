# 3970. Shortest Path With At Most K Consecutive Identical Characters
## O((n·k + E) · log(n·k)) — Dijkstra on Augmented (Node, Streak) State Space

---

## Problem Statement

Directed weighted graph, `n` nodes, edges `[u, v, w]`. Each node `i` has label `labels[i]`. Find the minimum-weight path from `0` to `n-1` such that the label string along the path has **at most `k` consecutive identical characters**. Return `-1` if no valid path exists.

---

## Core Insight #1 — Plain Node State Is Insufficient

Standard Dijkstra tracks `dist[node]`. That fails here because **whether we can extend a path depends not just on where we are, but on the label streak we've built up arriving there**.

Two paths that both arrive at node `v` with the same cost may have wildly different futures:
- Path A arrives via a node with `labels[prev] == labels[v]` — streak is high, choices are constrained.
- Path B arrives via a node with `labels[prev] != labels[v]` — streak resets to 1, full freedom ahead.

Treating them as the same state would silently discard the path with the lower streak (which has more future options) in favor of whichever arrived cheaper, potentially missing the globally optimal valid route.

---

## Core Insight #2 — The Streak Is the Only Extra Information Needed

When standing at node `u` with a consecutive count of `c`:
- `c` is always the count of character `labels[u]` at the tail of the current path.
- `labels[u]` itself is read from the node — **no need to store the character separately**.

So the full state is:
```
(node, streak)
where streak ∈ [1 .. k] = consecutive count of labels[node] at the end of the path so far
```

Total states: `n × k ≤ 5×10⁴ × 50 = 2.5×10⁶`.

---

## Core Insight #3 — Transition Is a Single Comparison

Moving from state `(u, c)` along edge `(u → v, w)`:

```
if labels[v] == labels[u]:
    new_streak = c + 1
    if new_streak > k: INVALID (prune this edge entirely)
else:
    new_streak = 1      // streak resets; doesn't matter how large c was

new state = (v, new_streak),  cost += w
```

This is complete. The streak always measures the tail run of the current node's character, and a single equality check updates it in O(1).

---

## Algorithm: Dijkstra on Augmented State Space

```
dist[node][streak] = min cost to reach (node, streak)

Init:
  dist[0][1] = 0       // at node 0, streak starts at 1
  all others = ∞

Process states in cost order (min-heap).
Answer: min over c in [1..k] of dist[n-1][c], or -1 if all ∞.
```

---

## Full Implementation (C++)

```cpp
class Solution {
public:
    int shortestPath(int n, vector<vector<int>>& edges, string labels, int k) {
        // Build adjacency list
        vector<vector<pair<int,int>>> adj(n);
        for (auto& e : edges) adj[e[0]].push_back({e[1], e[2]});

        const int INF = INT_MAX / 2;

        // dist[node][streak]: min cost reaching node with streak consecutive labels[node]
        vector<vector<int>> dist(n, vector<int>(k + 1, INF));

        // Min-heap: (cost, node, streak)
        priority_queue<tuple<int,int,int>,
                       vector<tuple<int,int,int>>,
                       greater<>> pq;

        dist[0][1] = 0;
        pq.push({0, 0, 1});

        while (!pq.empty()) {
            auto [cost, u, streak] = pq.top(); pq.pop();

            if (cost > dist[u][streak]) continue; // stale entry

            for (auto& [v, w] : adj[u]) {
                int new_streak;
                if (labels[v] == labels[u]) {
                    new_streak = streak + 1;
                    if (new_streak > k) continue; // constraint violated — prune
                } else {
                    new_streak = 1; // streak resets regardless of how large it was
                }

                int new_cost = cost + w;
                if (new_cost < dist[v][new_streak]) {
                    dist[v][new_streak] = new_cost;
                    pq.push({new_cost, v, new_streak});
                }
            }
        }

        int ans = INF;
        for (int c = 1; c <= k; c++) ans = min(ans, dist[n - 1][c]);
        return ans == INF ? -1 : ans;
    }
};
```

## State Transition Diagram

```
         labels[v] == labels[u]           labels[v] != labels[u]
(u, c) ─────────────────────────► (v, c+1)   if c+1 ≤ k
       ─────────────────────────► (v, 1)      otherwise
         cost += w in both cases
         c+1 > k → edge pruned
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| States (V) | O(n·k) | ≤ 2.5×10⁶ |
| State-graph edges (E) | O(\|edges\|·k) | each edge spawns ≤ k transitions |
| **Dijkstra total** | **O((n·k + \|edges\|·k) · log(n·k))** | ≈ 52M ops for max input |
| Space | O(n·k + \|edges\|) | dist table + heap |

With `n, |edges| ≤ 5×10⁴` and `k ≤ 50`, this is well within limits.


| From state | Edge | labels check | new_streak | valid? | cost |
|---|---|---|---|---|---|
| (0, 1) | 0→1 | 'a'=='a' → streak+1 = 2 > k=1 | — | ✗ pruned | — |
| (0, 1) | 0→2 | 'b'!='a' → reset | 1 | ✓ | 3 |

Only path 0→2 is valid. **Output: 3** ✓

## Pitfalls & Edge Cases

- **Don't store the streak character separately.** `labels[node]` always gives the current-tail character. The streak implicitly refers to `labels[current_node]` — adding a `char` to the state triples its size for zero benefit.

- **Streak resets to 1, not 0.** When entering a node with a different label, the streak becomes 1 (we're now at a node, contributing one character). A streak of 0 would mean "we haven't arrived yet" — not a valid in-path state.

- **Multiple different streaks at the same node can coexist.** `dist[v][1]` and `dist[v][3]` are genuinely independent states. A cheaper `dist[v][3]` does **not** dominate `dist[v][1]` — lower streak = more future flexibility, so both must be tracked.

- **Answer = min over all valid streaks at `n-1`.** The destination node may be reachable with streak 1, 2, ..., up to k — all are valid. Take the minimum.

- **n=1 (start == destination):** Node 0 is both source and target. In this case `dist[0][1] = 0` and the answer is `0` immediately. Handled naturally.

- **Overflow guard:** Edge weights up to `10⁴` and paths up to `n ≤ 5×10⁴` nodes → max total weight `~5×10⁸`, safely within `int`. Using `INF = INT_MAX / 2` prevents overflow in additions.

---

## Key Takeaway

This is the **"constraint-augmented Dijkstra"** pattern: when traversal validity depends on path *history*, identify the minimal suffix of that history needed for future decisions, and make it part of the node state. Here, the streak (count of the current tail character) is that minimal suffix — one integer, capped at `k`, turning an infinite history-dependent problem into a clean finite-state shortest-path problem.

**Pattern recognition:** "Shortest path with a constraint on consecutive same-label steps" → augment node with `streak ∈ [1..k]`, run Dijkstra, complexity scales by factor `k`. Identical technique applies to "at most k consecutive edges of same color", "at most k uphill steps", and similar bounded-run constraints.
