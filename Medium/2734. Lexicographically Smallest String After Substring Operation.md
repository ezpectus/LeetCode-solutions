# 2734. Lexicographically Smallest String After Substring Operation
## O(n) ÔÇö Greedy: Skip Leading 'a's, Decrement Until Next 'a'

---

## Problem Statement

- One operation: pick any non-empty substring, decrement every character by 1 (`'b'Ôćĺ'a'`, `'a'Ôćĺ'z'`).
- Exactly one operation must be performed.
- Return the lexicographically smallest result.

---

## Core Idea ÔÇö Greedy Substring Selection

To minimize lexicographically, we want to:
1. **Not touch leading `'a'`s** ÔÇö decrementing `'a'` wraps to `'z'` which is larger, hurting the result.
2. **Decrement the first non-`'a'` character onwards** ÔÇö this reduces the earliest possible position, which matters most lexicographically.
3. **Stop at the next `'a'`** ÔÇö decrementing an `'a'` turns it to `'z'` which is worse; stop before it.

**Special case**: if the entire string is `'a'`s, we must perform some operation. Best choice: decrement only the last character (`'a'Ôćĺ'z'`) ÔÇö it affects the least significant position.

**Algorithm**:
1. Skip leading `'a'`s (index `i`)
2. If `i == n`: decrement `s[n-1]` to `'z'`, return
3. Decrement `s[i]` while `s[i] != 'a'`, advancing `i`

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    string smallestString(string s) {
        int n = s.size(), i = 0;

        // Skip leading 'a's
        while (i < n && s[i] == 'a') i++;

        // All 'a's: must operate, best = last char only
        if (i == n) { s[n-1] = 'z'; return s; }

        // Decrement non-'a' chars until hitting 'a' or end
        while (i < n && s[i] != 'a') {
            s[i]--;
            i++;
        }

        return s;
    }
};
```

---

## Why Stop at (Original) 'a'?

If we continue past an existing `'a'` and decrement it to `'z'`, we'd increase that character ÔÇö making the string larger at that position. Since we want lexicographically smallest, we must stop before any original `'a'`.

The already-decremented characters that land on `'a'` are fine ÔÇö we just don't continue past pre-existing `'a'`s.

---

## Pitfalls & Edge Cases

- **All `'a'`s** (`"aa"`): forced to operate; decrement only `s[n-1]` Ôćĺ `'z'` affects last position only
- **No leading `'a'`s** (`"leetcode"`): start immediately from `i=0`, decrement until hitting `'a'` or end
- **Single character `'a'`**: must decrement Ôćĺ `'z'` (unavoidable)
- **Single non-`'a'`**: decrement it Ôćĺ done

---

## Key Takeaway

"Lexicographically smallest after one decrement-substring operation" Ôćĺ **greedy: skip leading 'a's, decrement contiguous non-'a' block**:

- `'a'`s at the front: untouchable (decrement wraps to `'z'`)
- First non-`'a'` block: decrement all of it (contiguous block = one substring operation)
- Stop at the next pre-existing `'a'` (decrementing it would give `'z'`)
