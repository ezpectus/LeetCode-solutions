# 3026. Maximum Good Subarray Sum
## O(n) — Prefix Sums + Hash Map of Best Start Prefix

---

## Problem Statement

- Subarray `nums[i..j]` is good if `|nums[i] - nums[j]| == k`.
- Return maximum sum of a good subarray, or 0 if none exist.

---

## Core Idea — Fix End, Look Up Best Start

**Sum of subarray** `nums[i..j]` = `prefix[j+1] - prefix[i]`.

To maximize this for fixed `j`, we want to **minimize `prefix[i]`** over all valid starts `i ≤ j` where `nums[i] ∈ {nums[j]+k, nums[j]-k}`.

**Algorithm**: sweep `j` from left to right, maintaining a hash map `best_prefix[v]` = minimum `prefix[i]` seen so far where `nums[i] == v`. For each `j`:
1. Check if `nums[j]+k` or `nums[j]-k` exists in `best_prefix` → update result candidate.
2. Update `best_prefix[nums[j]] = min(best_prefix[nums[j]], prefix[j])` (current `j` as potential start for future).

**Note**: update `best_prefix` AFTER checking — ensures `i ≤ j` (allows `i == j`, which is a single-element subarray with `|nums[j]-nums[j]|=0 ≠ k` anyway since `k ≥ 1`).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumSubarraySum(int[] nums, int k) {
        int n = nums.Length;
        long[] prefix = new long[n + 1];
        for (int i = 0; i < n; i++) prefix[i+1] = prefix[i] + nums[i];

        var bestPrefix = new Dictionary<int, long>(); // value → min prefix[i]
        long res = long.MinValue;
        bool found = false;

        for (int j = 0; j < n; j++) {
            int v = nums[j];

            // Check both targets: nums[i] = nums[j] ± k
            foreach (int target in new[] { v + k, v - k }) {
                if (bestPrefix.TryGetValue(target, out long minP)) {
                    res = Math.Max(res, prefix[j+1] - minP);
                    found = true;
                }
            }

            // Update best start for value v
            if (!bestPrefix.ContainsKey(v) || prefix[j] < bestPrefix[v])
                bestPrefix[v] = prefix[j];
        }

        return found ? res : 0;
    }
}
```

| **Metric** | **Value** |

| **Space** | **O(n)** — hash map stores at most n distinct values |

## Example Walkthrough

**`nums=[-1,3,2,4,5], k=3`**

prefix = [0,-1,2,4,8,13]

| j | nums[j] | targets | bestPrefix | candidate | res |
|---|---------|---------|-----------|-----------|-----|
| 0 | -1 | {2,-4} | {} | none | -∞ |
|   |   |   | {-1:0} | | |
| 1 | 3 | {6,0} | {-1:0} | none | -∞ |
|   |   |   | {-1:0,3:-1} | | |
| 2 | 2 | {5,-1} | {-1:0,...} | -1→prefix[3]-0=4 | 4 |
|   |   |   | {...,2:2} | | |
| 3 | 4 | {7,1} | ... | none | 4 |
|   |   |   | {...,4:4} | | |
| 4 | 5 | {8,2} | {2:2,...} | 2→prefix[5]-2=13-2=**11** | 11 |

Answer: **11** ✓ (subarray [2,4,5], |2-5|=3=k)

## Pitfalls & Edge Cases

- **`long` for prefix sums**: `nums[i]` up to `10^9`, n up to `10^5` → prefix up to `10^14`, overflows `int`
- **Update AFTER checking**: ensures `i < j` (not i=j), though even `i=j` gives single element with `|v-v|=0 ≠ k` — technically safe either way
- **No good subarray**: `found` flag tracks whether any valid pair was found → return 0 if not
- **Both `v+k` and `v-k`**: both are valid targets since `|nums[i]-nums[j]|=k` means either direction

---

## Key Takeaway

"Max subarray sum with |first-last|=k" → **prefix sums + hash map of best start**:

```
for j: check bestPrefix[nums[j]±k] → update res
        update bestPrefix[nums[j]] = min(prefix[j], ...)
```

Classic pattern: for each end, look up the optimal start in O(1) via hash map keyed by the start's value.
