# 2750. Ways to Split Array Into Good Subarrays
## O(n) — Product of Gaps Between Consecutive Ones

---

## Problem Statement

- Binary array. A **good subarray** contains exactly one `1`.
- Split the entire array into good subarrays.
- Return number of ways mod 10⁹+7.

---

## Core Idea — Independent Choices Between Consecutive Ones

Each good subarray must contain exactly one `1`. So every split must place exactly one `1` per part. The positions of `1`s are fixed — we just decide where to cut between consecutive `1`s.

Between `ones[i-1]` and `ones[i]`, there are `ones[i] - ones[i-1]` possible cut positions (anywhere from right after `ones[i-1]` up to and including right before `ones[i]`). Each gap is an **independent choice**, so multiply all gap sizes.

**Formula**:
```
result = Π (ones[i] - ones[i-1])  for i in 1..len(ones)-1
```

**Edge case**: if no `1` exists → impossible to form good subarrays → return `0`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int NumberOfGoodSubarraySplits(int[] nums) {
        const int MOD = 1_000_000_007;
        int prev = -1;
        long res = 1;
        bool hasOne = false;

        for (int i = 0; i < nums.Length; i++) {
            if (nums[i] == 1) {
                if (prev != -1)
                    res = res * (i - prev) % MOD;
                prev = i;
                hasOne = true;
            }
        }

        return hasOne ? (int)res : 0;
    }
}
```

## Example Walkthrough

**`nums=[0,1,0,0,1]`**: ones at indices `[1, 4]`

Gap between index 1 and 4 = `4-1 = 3` choices (cut after index 1, 2, or 3).

Result = **3** ✓

The 3 cuts: `[0,1] [0,0,1]`, `[0,1,0] [0,1]`, `[0,1,0,0] [1]`.

---

## Pitfalls & Edge Cases

- **No `1`s** → impossible → return `0`
- **Consecutive `1`s** (`[1,1]`): gap = `1-0 = 1` → only one way (must cut right between them) → `1` ✓
- **All `1`s**: every gap = 1 → product = 1 ✓

---

## Key Takeaway

"Count splits where each part has exactly one 1" → **product of gaps between consecutive 1s**. Each gap is an independent binary choice — multiply them all.
