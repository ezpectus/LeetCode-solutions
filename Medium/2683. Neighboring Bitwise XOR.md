# 2683. Neighboring Bitwise XOR
## O(n) — XOR of All Elements Must Equal 0

---

## Problem Statement

- `derived[i] = original[i] ⊕ original[i+1]` (last: `original[n-1] ⊕ original[0]`).
- Given `derived`, determine if a valid binary `original` exists.

---

## Core Idea — XOR of All derived Must Be 0

XOR all equations together:

```
derived[0] ⊕ derived[1] ⊕ ... ⊕ derived[n-1]
= (original[0]⊕original[1]) ⊕ (original[1]⊕original[2]) ⊕ ... ⊕ (original[n-1]⊕original[0])
```

Every `original[i]` appears in exactly **two** terms → `x ⊕ x = 0`. Everything cancels:

```
= 0
```

So `XOR(derived) = 0` is a **necessary** condition. It's also **sufficient**: if it holds, we can always construct a valid `original` — fix `original[0] = 0`, then derive each subsequent element as `original[i+1] = original[i] ⊕ derived[i]`, and the circular condition is automatically satisfied.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool DoesValidArrayExist(int[] derived) {
        int xor = 0;
        foreach (int x in derived) xor ^= x;
        return xor == 0;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

---

## Why Sufficiency Holds

If `XOR(derived) = 0`, construct `original` greedily:
- Set `original[0] = 0`
- For `i` in `[0, n-2]`: `original[i+1] = original[i] ⊕ derived[i]`

The last condition `derived[n-1] = original[n-1] ⊕ original[0]` is automatically satisfied precisely when `XOR(derived) = 0` (the cancellation proof above runs in reverse). Starting with `original[0] = 1` also works — both yield valid arrays when the condition holds.

---

## Pitfalls & Edge Cases

- **n = 1**: `derived[0] = original[0] ⊕ original[0] = 0` always → valid only if `derived[0] = 0`; the XOR check gives `derived[0] == 0` ✓
- **All zeros**: `XOR = 0` → valid (`original` = all zeros) ✓
- **All ones**: `XOR = n % 2`; valid iff `n` is even ✓

---

## Key Takeaway

"Does a circular XOR system have a solution?" → **XOR of all equations = 0**. Each variable appears in exactly two equations → all cancel. This reduces an O(n) system to a single O(n) accumulation check.
