# 2746. Decremental String Concatenation
## O(n × 26²) — DP on (First, Last) Character Pair

---

## Problem Statement

- `join(x, y)` = concatenate `x+y`, but remove one char if `x[-1] == y[0]`.
- At each step, prepend or append the next word to the current string.
- Minimize the final string length.

---

## Core Idea — DP State = (First Char, Last Char)

When joining the current string with the next word, only two things matter:
- The **first** character of the current string (used if we prepend the next word)
- The **last** character of the current string (used if we append the next word)

The internal characters are irrelevant. So the DP state is `(first, last)` → minimum length achievable with that pair.

**At most 26×26 = 676 states** at any time.

**Transitions** for each state `(sf, sl)` with length `slen`, adding word `w` with `wf=w[0], wl=w[-1]`:

```
Option 1 — append: join(prev, w)
  new first = sf,  new last = wl
  saves 1 if sl == wf  (last of prev == first of w)

Option 2 — prepend: join(w, prev)
  new first = wf,  new last = sl
  saves 1 if wl == sf  (last of w == first of prev)
```

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimizeConcatenatedLength(string[] words) {
        // dp[(first,last)] = min length
        var dp = new Dictionary<(char,char), int>();
        dp[(words[0][0], words[0][^1])] = words[0].Length;

        for (int i = 1; i < words.Length; i++) {
            string w = words[i];
            char wf = w[0], wl = w[^1];
            int wlen = w.Length;
            var ndp = new Dictionary<(char,char), int>();

            foreach (var ((sf, sl), slen) in dp) {
                // Append: join(prev, w)
                int v1 = slen + wlen - (sl == wf ? 1 : 0);
                var k1 = (sf, wl);
                if (!ndp.TryGetValue(k1, out int cur1) || v1 < cur1)
                    ndp[k1] = v1;

                // Prepend: join(w, prev)
                int v2 = slen + wlen - (wl == sf ? 1 : 0);
                var k2 = (wf, sl);
                if (!ndp.TryGetValue(k2, out int cur2) || v2 < cur2)
                    ndp[k2] = v2;
            }
            dp = ndp;
        }

        return dp.Values.Min();
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × 26²)** | n words × at most 676 states × 2 transitions |
| **Space** | **O(26²)** | DP map with at most 676 entries |

---

## Why Only First and Last Matter

The join operation only looks at `x[-1]` and `y[0]`. After joining, the new string's "active" endpoints are its own first and last characters. Everything in between is irrelevant for future joins.

This is the key insight that compresses an exponential search (2^n orderings) into a polynomial DP.

---

## Example Walkthrough

**`words=["aa","ab","bc"]`**

**After "aa"**: `dp = {('a','a'): 2}`

**Add "ab"** (wf='a', wl='b', wlen=2):
- Append: sl='a'==wf='a' → save 1, v=2+2-1=3, key=('a','b')
- Prepend: wl='b'≠sf='a' → v=2+2-0=4, key=('a','a')
- `dp = {('a','b'):3, ('a','a'):4}`

**Add "bc"** (wf='b', wl='c', wlen=2):
- From ('a','b'): Append: sl='b'==wf='b' → 3+2-1=4, key=('a','c') ✓
- From ('a','b'): Prepend: wl='c'≠sf='a' → 3+2=5, key=('b','b')
- From ('a','a'): Append: sl='a'≠wf='b' → 4+2=6, key=('a','c') → worse
- ...

Min = **4** ✓

---

## Pitfalls & Edge Cases

- **Same word appended/prepended can give different keys** — both options are always tried; take minimum per key
- **Single word** — no operations needed; `dp` initialized directly from `words[0]`
- **`w[^1]` in C#** — index from end syntax; equivalent to `w[w.Length-1]`

---

## Key Takeaway

"Minimize length of iterative join (with optional char deletion at junction)" → **DP on (first, last) char pair**:

- State space: at most 26² = 676 states
- Transition: 2 options per state per word (append/prepend)
- Only endpoints matter; internal content is irrelevant

Pattern: when an operation only looks at the boundary of a structure, compress DP state to just the boundary.
