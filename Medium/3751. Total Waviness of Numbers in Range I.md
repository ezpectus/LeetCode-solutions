# 3751. Total Waviness of Numbers in Range I
## O(n · D) — Brute Force Per-Digit Check

---

## Problem Statement

- Given a range `[num1, num2]` (1 ≤ num1 ≤ num2 ≤ 10⁵).
- Waviness of a number = count of interior digits that are strictly greater than both neighbors (peak) or strictly less than both neighbors (valley). First and last digits never count.
- Numbers with fewer than 3 digits have waviness 0.
- Return the total waviness summed over all numbers in the range.

---

## Core Idea — Brute Force is Optimal Here

With num2 ≤ 10⁵, the range has at most 100,000 numbers. Each has at most 5 digits. Total work = 100,000 × 5 = **500,000 ops** — brute force is trivially fast and direct.

For each number: convert to string, check each interior digit (positions 1 to len−2), count peaks and valleys.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int TotalWaviness(int num1, int num2) {
        int total = 0;

        for (int x = num1; x <= num2; x++) {
            total += Waviness(x);
        }

        return total;
    }

    private int Waviness(int x) {
        string s = x.ToString();
        int n = s.Length;

        if (n < 3) return 0;

        int count = 0;

        for (int i = 1; i < n - 1; i++) {
            int prev = s[i - 1] - '0';
            int curr = s[i]     - '0';
            int next = s[i + 1] - '0';

            if (curr > prev && curr > next) count++; // peak
            if (curr < prev && curr < next) count++; // valley
        }

        return count;
    }
}
```

---

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n · D) | n = num2 − num1 + 1 ≤ 10⁵, D ≤ 5 digits |
| Space Complexity | O(D) | Temporary string per number |

Max ops ≈ 500,000 — well within limits.

## Why This Works — Example Walkthrough

**Example 1:** `[120, 130]`

| x   | Digits    | Interior | Peak/Valley? | Waviness |
|-----|-----------|----------|--------------|----------|
| 120 | [1, 2, 0] | 2        | 2>1 and 2>0  | 1        |
| 121 | [1, 2, 1] | 2        | 2>1 and 2>1  | 1        |
| 122 | [1, 2, 2] | 2        | 2>1 not 2>2  | 0        |
| 123 | [1, 2, 3] | 2        | 2>1 not 2>3  | 0        |
| ... | ...       | ...      | ...          | 0        |
| 130 | [1, 3, 0] | 3        | 3>1 and 3>0  | 1        |

**Output:** 1+1+0+...+0+1 = **3** ✓

---

**Example 2:** `[198, 202]`

| x   | Digits    | Interior | Reason              | Waviness |
|-----|-----------|----------|---------------------|----------|
| 198 | [1, 9, 8] | 9        | 9>1 and 9>8 → peak  | 1        |
| 199 | [1, 9, 9] | 9        | 9>1 but 9==9 → no   | 0        |
| 200 | [2, 0, 0] | 0        | 0<2 but 0==0 → no   | 0        |
| 201 | [2, 0, 1] | 0        | 0<2 and 0<1 → valley| 1        |
| 202 | [2, 0, 2] | 0        | 0<2 and 0<2 → valley| 1        |

**Output:** 1+0+0+1+1 = **3** ✓

---

**Example 3:** `num1 = num2 = 4848`

Digits: [4, 8, 4, 8]
- pos 1 (8): 8>4 and 8>4 → **peak** ✓
- pos 2 (4): 4<8 and 4<8 → **valley** ✓

**Output:** **2** ✓

## Pitfalls & Edge Cases

- **Strictly** greater/less — `122` has middle digit `2` which is greater than the left neighbor but **equal** to the right → not a peak. Strict inequality is required.

- **< 3 digit numbers** — 1 and 2 digit numbers always return 0. No interior positions exist.

- **1-indexed constraints** — num2 can equal exactly 10⁵ = 100000 (6 digits), which has interior digits at positions 1, 2, 3, 4. The max waviness for a 6-digit number is 4.

- **Both conditions are independent** — A digit can't be both a peak and a valley simultaneously. Using two separate `if` (not `else if`) is fine since the conditions are mutually exclusive.

---

## Key Takeaway

When the constraint ceiling is small (n ≤ 10⁵, D ≤ 5), brute force iteration is the cleanest and correct approach. No digit DP, no prefix sums needed.

**Pattern recognition:** "Sum a property over a range of small integers" → iterate. Always check: range_size × work_per_element ≤ ~10⁷? If yes, brute force.
