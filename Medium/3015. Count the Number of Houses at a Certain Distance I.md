# 3015. Count the Number of Houses at a Certain Distance I
## O(n²) — BFS from Every Node

---

## Problem Statement

- Houses 1..n connected as a line (i — i+1). Extra edge between x and y.
- For each distance k, count ordered pairs (a,b) with shortest path = k.

---

## Core Idea — BFS from Each House

With `n ≤ 100`, run BFS from every house to compute all pairwise shortest distances. For each distance found, increment `result[distance-1]`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] CountOfPairs(int n, int x, int y) {
        var adj = new List<int>[n+1];
        for (int i = 0; i <= n; i++) adj[i] = new List<int>();
        for (int i = 1; i < n; i++) { adj[i].Add(i+1); adj[i+1].Add(i); }
        if (x != y) { adj[x].Add(y); adj[y].Add(x); }

        int[] res = new int[n];

        for (int src = 1; src <= n; src++) {
            int[] dist = new int[n+1];
            Array.Fill(dist, -1);
            dist[src] = 0;
            var q = new Queue<int>();
            q.Enqueue(src);
            while (q.Count > 0) {
                int u = q.Dequeue();
                foreach (int v in adj[u]) {
                    if (dist[v] == -1) { dist[v] = dist[u]+1; q.Enqueue(v); }
                }
            }
            for (int dst = 1; dst <= n; dst++)
                if (dst != src && dist[dst] > 0 && dist[dst] <= n)
                    res[dist[dst]-1]++;
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n²)** — n BFS runs, each O(n) |
| **Space** | **O(n)** |

n≤100 → at most 10,000 operations — instant.

---

## Key Takeaway

Small n (≤100) → BFS from every node. Build graph with line edges plus extra (x,y) edge. Count pairs by distance. Ordered pairs → both (a,b) and (b,a) counted → naturally handled since BFS from every src counts all directed pairs.
