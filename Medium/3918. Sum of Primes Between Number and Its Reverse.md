# 3918. Sum of Primes Between Number and Its Reverse
## O(n√n) — Reverse Digits + Sum Primes in Range

---

## Problem Statement

- Compute `r` = reverse of n's digits.
- Sum all primes in `[min(n,r), max(n,r)]` inclusive.
- Constraints: 1 ≤ n ≤ 1000.

---

## Core Idea — Direct Computation

1. Reverse the digits of n to get r.
2. For each integer in `[min(n,r), max(n,r)]`, check primality (trial division up to √x) and sum if prime.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int SumOfPrimesInRange(int n) {
        int r = Reverse(n);
        int lo = Math.Min(n, r), hi = Math.Max(n, r);

        int sum = 0;
        for (int x = lo; x <= hi; x++) {
            if (IsPrime(x)) sum += x;
        }

        return sum;
    }

    private int Reverse(int n) {
        int r = 0;
        while (n > 0) {
            r = r * 10 + n % 10;
            n /= 10;
        }
        return r;
    }

    private bool IsPrime(int x) {
        if (x < 2) return false;
        for (int i = 2; (long)i * i <= x; i++)
            if (x % i == 0) return false;
        return true;
    }
}
```

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O((hi-lo)·√hi) | hi ≤ 1000 (max reverse of 1000 is small but reverse of 999 = 999, etc.) |
| Space Complexity | O(1) | |

## Example Walkthrough

**Example 1:** `n = 13` → r = 31. Range [13, 31].

Primes: 13, 17, 19, 23, 29, 31 → sum = 132 ✓

**Example 3:** `n = 8` → r = 8. Range [8, 8]. 8 is not prime → sum = 0 ✓

---

## Pitfalls & Edge Cases

- **Trailing zeros:** e.g. n=10 → reverse digit-by-digit gives r=01=1 (leading zero dropped naturally by integer arithmetic). ✓
- **n = r (palindrome):** range collapses to a single value. ✓
- **n = 1:** r = 1. Range [1,1]. 1 is not prime → sum = 0. ✓

---

## Key Takeaway

Straightforward simulation: reverse digits via mod/div loop, then trial-division primality check over a small range (≤1000). No optimization needed given tiny constraints.
