# 3979. Maximum Valid Pair Sum
## O(n) — Rolling Prefix Maximum

---

## Problem Statement

Find the maximum `nums[i] + nums[j]` over all pairs with `0 ≤ i < j < n` and `j - i ≥ k`.

---

## Core Insight

For a fixed `j`, the valid indices are `i ∈ [0, j-k]`. To maximize `nums[i] + nums[j]`, we want the **maximum** `nums[i]` in that range. As `j` increases by 1, the valid range for `i` expands by one element (index `j-k` joins the eligible set). So we maintain a **running prefix maximum** over `nums[0..j-k]`.

```
For each j from k to n-1:
    prefMax = max(nums[0], ..., nums[j-k])
    ans = max(ans, prefMax + nums[j])
```

---

## Full Implementation (C++)

```cpp
class Solution {
public:
    int maxValidPairSum(vector<int>& nums, int k) {
        int n = nums.size();
        int ans = INT_MIN;
        int prefMax = nums[0]; // max of nums[0..j-k], starting with j=k so j-k=0

        for (int j = k; j < n; j++) {
            // nums[j-k] just became eligible; expand prefix max
            prefMax = max(prefMax, nums[j - k]);
            ans = max(ans, prefMax + nums[j]);
        }

        return ans;
    }
};
```

## Complexity

| Metric | Value |
|---|---|
| Time | O(n) — one pass |
| Space | O(1) |

## Example Walkthrough

### Example 1 — `nums = [1,3,5,2,8], k = 2`

| j | j-k | nums[j-k] | prefMax | nums[j] | sum |
|---|---|---|---|---|---|
| 2 | 0 | 1 | 1 | 5 | 6 |
| 3 | 1 | 3 | 3 | 2 | 5 |
| 4 | 2 | 5 | 5 | 8 | **13** |

**Output: 13** ✓

### Example 2 — `nums = [5,1,9], k = 1`

| j | prefMax | nums[j] | sum |
|---|---|---|---|
| 1 | 5 | 1 | 6 |
| 2 | 5 | 9 | **14** |

**Output: 14** ✓

---

## Key Takeaway

"Maximum `nums[i] + nums[j]` with `i ≤ j - k`" → fix `j`, maximize over a growing prefix of `i` values → rolling prefix maximum updated one step per `j`. Classic sliding-window-max simplification when the window only grows (no eviction needed).
