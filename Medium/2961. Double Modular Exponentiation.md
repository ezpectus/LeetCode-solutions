# 2961. Double Modular Exponentiation
## O(n log b + n log c) ‚Äî Direct Fast Exponentiation per Entry

---

## Problem Statement

- Given 2D array `variables` where `variables[i] = [a, b, c, m]` and integer `target`.
- Index `i` is **good** if `((a^b % 10)^c) % m == target`.
- Return all good indices in any order.

---

## Core Idea ‚Äî Two-Step Modular Exponentiation

The formula involves two separate exponentiations:

1. **Inner**: compute `a^b % 10`
2. **Outer**: compute `(result)^c % m`

Both are straightforward applications of **fast modular exponentiation** (binary exponentiation). For each entry, evaluate the formula directly and check against `target`.

**Why modular exponentiation and not just `Math.Pow`?**
`a^b` can be astronomically large even for small inputs (`a=1000, b=1000`). Modular exponentiation keeps values bounded at every step, computing `x^n % mod` in O(log n) multiplications.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public IList<int> GetGoodIndices(int[][] variables, int target) {
        var result = new List<int>();

        for (int i = 0; i < variables.Length; i++) {
            int a = variables[i][0], b = variables[i][1],
                c = variables[i][2], m = variables[i][3];

            int inner = ModPow(a, b, 10);   // a^b % 10
            int outer = ModPow(inner, c, m); // (a^b % 10)^c % m

            if (outer == target)
                result.Add(i);
        }

        return result;
    }

    private int ModPow(long base_, int exp, int mod) {
        long result = 1;
        base_ %= mod;
        while (exp > 0) {
            if ((exp & 1) == 1) result = result * base_ % mod;
            base_ = base_ * base_ % mod;
            exp >>= 1;
        }
        return (int)result;
    }
}
```

## Why This Works ‚Äî Binary Exponentiation

**How does `ModPow(base, exp, mod)` work?**

Uses the identity:
```
x^n = x^(n/2) * x^(n/2)        if n is even
x^n = x^(n/2) * x^(n/2) * x   if n is odd
```

At each step:
- If the current bit of `exp` is 1 ‚Üí multiply result by current `base_`
- Square `base_` and shift `exp` right

This halves the exponent each iteration ‚Üí O(log exp) steps. Taking `% mod` at every multiplication keeps values ‚â§ `mod¬≤` ‚Äî no overflow as long as we use `long` for intermediate products.

**Why is it correct to compute `a^b % 10` first, then raise to `c` mod `m`?**

The formula is literally `((a^b % 10)^c) % m` ‚Äî the inner result is used as the base for the outer exponentiation. We compute them sequentially, respecting the parenthesization exactly.

**Special case: `mod = 1`**

Any number `% 1 = 0`. If `m = 1`, the outer result is always 0. Our `ModPow` handles this: `base_ %= 1` ‚Üí `base_ = 0`, so `result` stays 0. Only valid if `target = 0`.

## Pitfalls & Edge Cases

- **Use `long` for intermediate products**: `base_ * base_` with `base_` up to `mod-1 = 999` ‚Üí up to `998001`, fits in `int`, but `result * base_` up to `999 * 999 = 998001` ‚Äî safe with `int`, but using `long` is defensive best practice
- **`mod = 1` edge case**: `ModPow` returns 0 correctly since `base_ %= 1 = 0`
- **`exp = 0`**: `x^0 = 1` for any `x` ‚Äî the while loop doesn't execute, `result = 1` is returned correctly
- **Don't use `Math.Pow`**: floating-point precision errors make it unreliable for large integers; always use integer `ModPow`

## Key Takeaway

"Evaluate `((a^b % 10)^c) % m`" ‚Üí **two sequential modular exponentiations**:

```
inner = ModPow(a, b, 10)
outer = ModPow(inner, c, m)
if outer == target: add index

ModPow(base, exp, mod):
    result = 1
    base %= mod
    while exp > 0:
        if exp & 1: result = result * base % mod
        base = base * base % mod
        exp >>= 1
    return result
```

Binary exponentiation keeps values bounded and runs in O(log exp) per call ‚Üí O(n log max(b,c)) total.
