# 2721. Execute Asynchronous Functions in Parallel
## Promise.all Polyfill — Counter + Pre-allocated Results Array

---

## Problem Statement

- Run all async functions in parallel.
- Resolve with array of results (in original order) when all resolve.
- Reject immediately with the first rejection reason.

---

## Core Idea — Counter + Index-Preserved Results

Wrap everything in a `new Promise`. Launch all functions immediately (parallel execution). Track:
- `results[i]` — store each resolved value at its original index
- `resolved` counter — when it hits `functions.length`, all done → resolve
- `.catch(reject)` — first rejection propagates immediately

**Why pre-allocated array with index?** Promises resolve in arbitrary order. Using `results[i] = val` preserves original order regardless of which promise finishes first.

---

## Full Implementation (JavaScript)

```javascript
var promiseAll = function(functions) {
    return new Promise((resolve, reject) => {
        const results = new Array(functions.length);
        let resolved = 0;

        functions.forEach((fn, i) => {
            fn()
                .then(val => {
                    results[i] = val;
                    if (++resolved === functions.length) resolve(results);
                })
                .catch(reject); // first rejection wins
        });
    });
};
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(max(t_i))** | All run in parallel; total time = slowest promise |
| **Space** | **O(n)** | Results array |

---

## Why `.catch(reject)` Works for "First Rejection"

Once `reject` is called, the outer Promise is settled — subsequent calls to `resolve` or `reject` are **no-ops** in JS (a Promise can only be settled once). So whichever promise rejects first triggers `reject`, and all later resolutions/rejections are silently ignored. ✓

---

## Pitfalls & Edge Cases

- **`results[i] = val` not `results.push(val)`** — push loses ordering; index assignment preserves it
- **`functions.length === 0`** — the counter never increments → `resolve` never called. In practice the constraint says `length ≥ 1`, but `Promise.all([])` resolves with `[]` — add `if (!functions.length) resolve([])` for robustness
- **Multiple rejections** — only the first matters; subsequent rejections call `reject` again but are ignored by the Promise spec

---

## Key Takeaway

`Promise.all` polyfill = **counter + index-keyed results array**:

```javascript
results[i] = val
if (++resolved === n) resolve(results)  // all done
.catch(reject)                          // first rejection wins
```

The index assignment is the key detail — ensures correct order regardless of resolution timing.
