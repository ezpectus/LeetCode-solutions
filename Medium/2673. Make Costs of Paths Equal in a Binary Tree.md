# 2673. Make Costs of Paths Equal in a Binary Tree
## O(n) — Bottom-Up Greedy: Equalize Siblings, Propagate Max

---

## Problem Statement

- Perfect binary tree, nodes 1..n (1-indexed). Node `i` has children `2i` and `2i+1`.
- `cost[i]` = cost of node `i+1` (0-indexed array).
- Increment any node's cost any number of times.
- Return minimum total increments so all root-to-leaf paths have equal sum.

---

## Core Idea — Bottom-Up Sibling Equalization

**Key insight**: we can only **increment** (never decrement). So the target path sum ≥ current maximum path sum. To minimize increments, process bottom-up: at each internal node, equalize its two children's subtree path sums to the **maximum** of the two. The cost of equalization = `|left - right|` increments. Then propagate `max(left, right)` upward as the subtree's contribution.

**Why greedy is optimal?**
- We must make both children equal (they share the same parent prefix).
- We can only raise the smaller one to match the larger — never lower the larger.
- Doing this bottom-up ensures each level is handled independently and optimally.

**In-place**: add `max(left, right)` into `cost[i]` (the parent) to accumulate the subtree path sum bottom-up.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinIncrements(int n, int[] cost) {
        int res = 0;
        // Process internal nodes bottom-up (last internal = n/2)
        for (int i = n / 2; i >= 1; i--) {
            int left  = cost[2*i - 1]; // node 2i   → 0-indexed: 2i-1
            int right = cost[2*i];     // node 2i+1 → 0-indexed: 2i
            res += Math.Abs(left - right);
            cost[i - 1] += Math.Max(left, right); // propagate max to parent
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — one pass over all internal nodes |
| **Space** | **O(1)** — in-place, no extra arrays |

## Example Walkthrough

**Example 1**: `n=7, cost=[1,5,2,2,3,3,1]` (nodes 1-7)

```
Tree:          1(cost=1)
             /          \
          2(5)          3(2)
         /    \        /    \
       4(2)  5(3)   6(3)  7(1)
```

**i=3** (node 3, children 6 and 7): `left=3, right=1` → `res+=2`, `cost[2] += 3` → `cost[2]=5`
**i=2** (node 2, children 4 and 5): `left=2, right=3` → `res+=1`, `cost[1] += 3` → `cost[1]=8`
**i=1** (node 1, children 2 and 3): `left=8, right=5` → `res+=3`, done

Total: `2 + 1 + 3 = 6` ✓

---

## Pitfalls & Edge Cases

- **0-indexed vs 1-indexed**: node `i` (1-indexed) → `cost[i-1]`; children `2i` and `2i+1` → `cost[2i-1]` and `cost[2i]`
- **Perfect binary tree guaranteed** — no null children checks needed; every internal node has exactly 2 children
- **n=3** (minimal tree, 1 root + 2 leaves) → one iteration at `i=1`

---

## Key Takeaway

"Equalize all root-to-leaf paths with minimum increments" in perfect binary tree → **bottom-up greedy**:

```
res += |left - right|        // cost to equalize siblings
parent += max(left, right)   // propagate the larger path sum up
```

Each level is handled independently. Propagating the max ensures the parent sees the correct accumulated path cost from below.
