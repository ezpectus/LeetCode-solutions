# 3275. K-th Nearest Obstacle Queries
## O(q log k) — Max-Heap of Size k

---

## Problem Statement

- After each obstacle added, find the k-th smallest Manhattan distance from origin. Return -1 if fewer than k obstacles exist.

---

## Core Idea — Max-Heap of Size k

Maintain a max-heap of the k smallest distances seen so far. After each insertion:
- If heap size > k: remove the maximum (it's no longer in the top-k).
- If heap size == k: the top = k-th smallest distance.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] ResultsArray(int[][] queries, int k) {
        // Max-heap: largest element at top
        var heap = new PriorityQueue<int, int>(Comparer<int>.Create((a, b) => b - a));
        int[] res = new int[queries.Length];

        for (int i = 0; i < queries.Length; i++) {
            int d = Math.Abs(queries[i][0]) + Math.Abs(queries[i][1]);
            heap.Enqueue(d, d);
            if (heap.Count > k) heap.Dequeue(); // remove largest
            res[i] = heap.Count == k ? heap.Peek() : -1;
        }
        return res;
    }
}
```

## Key Takeaway

k-th smallest → max-heap of size k. Top = k-th smallest. O(q log k).
