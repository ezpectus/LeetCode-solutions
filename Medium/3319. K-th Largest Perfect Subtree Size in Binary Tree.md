# 3319. K-th Largest Perfect Subtree Size in Binary Tree
## O(n log n) — DFS Returns Perfect Height + Sort

---

## Problem Statement

- Find the k-th largest size among all perfect binary subtrees. Return -1 if fewer than k exist.
- A perfect binary tree: all leaves same level, every parent has 2 children.

---

## Core Idea — DFS Returning Height of Perfect Subtree

`Dfs(node)` returns the height of the perfect subtree rooted at `node`, or -1 if not perfect.

- Null node → height 0 (base case, perfect).
- Left and right subtrees must both be perfect AND equal height.
- If perfect: record size `2^(h+1) - 1`, return `h+1`.
- If not: return -1.

Collect all sizes, sort descending, return k-th.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    List<int> sizes = new();

    int Dfs(TreeNode node) {
        if (node == null) return 0;
        int l = Dfs(node.left), r = Dfs(node.right);
        if (l == -1 || r == -1 || l != r) return -1;
        sizes.Add((1 << (l+1)) - 1);
        return l + 1;
    }

    public int KthLargestPerfectSubtree(TreeNode root, int k) {
        Dfs(root);
        sizes.Sort((a, b) => b - a);
        return k <= sizes.Count ? sizes[k-1] : -1;
    }
}
```

**Time**: O(n log n) — O(n) DFS + O(n log n) sort. **Space**: O(n).

---

## Key Takeaway

Perfect subtree check: both children must be perfect AND same height. `2^(h+1)-1` gives node count for height `h`.
