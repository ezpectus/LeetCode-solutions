# 3776. Minimum Moves to Balance Circular Array
## O(n log n) — Greedy: Route to Deficit From Closest Sources First

---

## Problem Statement

- Circular array `balance`, at most one negative element.
- One move = one unit travels one hop (left or right) between adjacent elements.
- Make all balances ≥ 0. Return minimum total moves, or −1 if impossible.

---

## Core Idea — Each Unit Takes Its Shortest Path to the Deficit

Since there's at most one negative element at position `neg` with value `d < 0`, we need to route `|d|` units from surrounding positive elements to position `neg`.

**Key observations:**
1. If total sum < 0: impossible (return −1).
2. In a circular array, a unit at position `p` can reach `neg` via clockwise (distance `(neg−p+n) % n`) or counter-clockwise (distance `n − clockwise`). Its optimal cost is `min(cw, n−cw)`.
3. Units from different sources travel independently (no blocking) — each takes its shortest path.
4. To minimize total cost = sum of (amount × distance): take from the **closest** sources first (fractional knapsack greedy).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinMoves(int[] balance) {
        int n = balance.Length;
        int negIdx = -1;
        long total = 0;

        for (int i = 0; i < n; i++) {
            total += balance[i];
            if (balance[i] < 0) negIdx = i;
        }

        if (negIdx == -1) return 0;  // no negative element
        if (total < 0) return -1;    // sum negative → impossible

        long deficit = -(long)balance[negIdx];

        // Build (distance, supply) pairs for all positive elements
        var sources = new List<(int dist, int supply)>();
        for (int i = 0; i < n; i++) {
            if (balance[i] <= 0) continue;
            int cw = (negIdx - i + n) % n;           // clockwise hops from i to neg
            int dist = Math.Min(cw, n - cw);          // shortest path
            sources.Add((dist, balance[i]));
        }

        // Greedy: take from closest sources first
        sources.Sort((a, b) => a.dist.CompareTo(b.dist));

        long cost = 0;
        long remaining = deficit;

        foreach (var (dist, supply) in sources) {
            if (remaining <= 0) break;
            long take = Math.Min((long)supply, remaining);
            cost += take * dist;
            remaining -= take;
        }

        return cost;
    }
}
```

| Space Complexity | O(n) | Sources list |

## Why the Greedy Is Optimal — Exchange Argument

We want: minimize `Σ (x_i × d_i)` subject to `Σ x_i = deficit`, `0 ≤ x_i ≤ supply_i`.

This is a **linear cost assignment** problem. Suppose we take some amount from a farther source when a closer source still has capacity. Swapping — take more from closer, less from farther — reduces total cost without violating constraints. Therefore taking from the closest sources first is always optimal. ✓

## Example Walkthrough

**Example 1:** `balance = [5, 1, −4]`, n=3, negIdx=2, deficit=4

| i | balance[i] | Clockwise to 2 | Min dist |
|---|------------|----------------|----------|
| 0 | 5          | (2−0+3)%3 = 2  | min(2,1)=**1** |
| 1 | 1          | (2−1+3)%3 = 1  | min(1,2)=**1** |

Sources sorted: [(1,5), (1,1)]

- Take 1 from supply=1, dist=1: cost=1, remaining=3
- Take 3 from supply=5, dist=1: cost=1+3=4, remaining=0

**Output:** 4 ✓

(Unit from element 1 goes 1 hop right; 3 units from element 0 go 1 hop left via counter-clockwise)

---

**Example 2:** `balance = [1, 2, −5, 2]`, n=4, negIdx=2, deficit=5

| i | balance[i] | Clockwise to 2 | Min dist |
|---|------------|----------------|----------|
| 0 | 1          | (2−0+4)%4 = 2  | min(2,2)=**2** |
| 1 | 2          | (2−1+4)%4 = 1  | min(1,3)=**1** |
| 3 | 2          | (2−3+4)%4 = 3  | min(3,1)=**1** |

Sources sorted: [(1,2), (1,2), (2,1)]

- Take 2, dist=1: cost=2, remaining=3
- Take 2, dist=1: cost=4, remaining=1
- Take 1, dist=2: cost=6, remaining=0

**Output:** 6 ✓

**Example 3:** `balance = [−3, 2]`

total = −3 + 2 = −1 < 0 → **−1** ✓

## Pitfalls & Edge Cases

- **total < 0:** The combined supply of positive elements is less than the deficit. Impossible. ✓

- **No negative element:** Return 0 immediately. ✓

- **n = 2:** Two elements, one negative. The only other element is at distance 1. If it has enough, cost = deficit × 1. Otherwise −1. ✓

- **Overflow:** max deficit = 10⁹, max distance = n/2 = 5×10⁴, max total units = 10⁵ × 10⁹. Wait, actually deficit ≤ 10⁹ and max distance ≈ 5×10⁴ so cost ≤ 5×10¹³ → must use `long`. ✓

- **Element exactly at zero:** `balance[i] = 0` has no supply, skipped by `balance[i] <= 0` check. ✓

- **Multiple elements at same distance:** Their order relative to each other doesn't matter (same unit cost). ✓

---

## Key Takeaway

One negative element = one "sink". Problem reduces to: **minimum cost flow from multiple sources to a single sink on a circle**, where each unit takes its shortest path.

Greedy is optimal: sort sources by distance, take from closest first (exchange argument). No DP, no complex flow algorithms needed.

**Pattern recognition:** "Route units to a fixed destination, minimize total hop count" → sort by distance, take greedily. Circular distance = `min(cw, n−cw)`.
