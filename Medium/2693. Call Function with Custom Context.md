# 2693. Call Function with Custom Context
## Polyfill for `Function.call` ‚Äî Temporary Method Injection

---

## Problem Statement

- Implement `Function.prototype.callPolyfill(context, ...args)` without using `Function.call`.
- `context` becomes `this` inside the function. Additional `args` are passed as arguments.

---

## Core Idea ‚Äî Temporarily Attach Function as a Method

In JavaScript, when you call `obj.method()`, `this` inside `method` is `obj`. We exploit this:

1. Attach `this` (the function) as a property on `context`
2. Call it via `context[key](...args)` ‚Äî now `this` is `context`
3. Delete the temporary property to leave `context` unchanged

**Why `Symbol()`?** A Symbol creates a guaranteed-unique key that won't collide with any existing property on `context`, no matter what keys it already has.

---

## Full Implementation (JavaScript)

```javascript
Function.prototype.callPolyfill = function(context, ...args) {
    const sym = Symbol();   // unique key ‚Äî no collision risk
    context[sym] = this;    // attach function as method on context
    const res = context[sym](...args);  // call: this === context ‚úì
    delete context[sym];    // clean up
    return res;
};
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(1)** setup + cost of `fn` itself |
| **Space** | **O(1)** |

---

## Why Not Just Use a String Key?

```javascript
context['_fn'] = this; // RISKY ‚Äî context might already have '_fn'
```

Any hardcoded string key risks overwriting an existing property. `Symbol()` is always unique ‚Äî even two `Symbol()` calls in the same file produce different symbols. Safe regardless of `context`'s contents.

---

## How `this` Binding Works Here

```javascript
context[sym] = this;        // this = the function being polyfilled
context[sym](...args);      // method call ‚Üí this inside fn = context
```

JavaScript's method call rule: `obj.method()` ‚Üí `this === obj`. By temporarily making the function a method of `context`, we get the right `this` binding for free.

---

## Pitfalls & Edge Cases

- **`delete` after call** ‚Äî must clean up; if `fn` throws, the property leaks. Production code would use `try/finally`
- **Arrow functions** ‚Äî arrow functions ignore `this` binding entirely; this polyfill (like the real `call`) can't override their `this`. But the problem guarantees regular functions
- **`context` is the first arg, rest are `...args`** ‚Äî rest parameters cleanly separate them; no need to slice `arguments`

---

## Key Takeaway

"Set custom `this` without using `call`" ‚Üí **temporary method injection**:

```javascript
const sym = Symbol();
context[sym] = fn;
const res = context[sym](...args);  // this === context
delete context[sym];
```

This is exactly how JavaScript engines implement `call`/`apply` under the hood conceptually ‚Äî make it a method, invoke it, clean up.
