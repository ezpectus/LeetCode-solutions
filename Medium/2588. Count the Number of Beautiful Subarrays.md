# 2588. Count the Number of Beautiful Subarrays
## O(n) — XOR Prefix Hashing

---

## Problem Statement

- You are given a **0-indexed** integer array `nums` (1 ≤ n ≤ 10⁵), where each `nums[i]` (0 ≤ nums[i] ≤ 10⁶).
- In one operation: pick two indices `i`, `j` and a bit position `k` where both `nums[i]` and `nums[j]` have bit `k` set → subtract `2^k` from both.
- A subarray is **beautiful** if all its elements can be reduced to 0 by any number of such operations.
- Return the **count of beautiful subarrays**.

---

## Core Idea — XOR Prefix Hashing

**Key insight**: A subarray can be reduced to all zeros if and only if its **XOR equals 0**. This is because:

- Each operation cancels one bit from **two** elements simultaneously — it's equivalent to XOR-cancellation
- For all elements to reach 0, every bit position must appear an **even number of times** across the subarray
- XOR of the entire subarray equals 0 ⟺ every bit appears an even number of times ✓

**Reduction to prefix XOR**:

Define `prefix[i] = nums[0] ^ nums[1] ^ ... ^ nums[i-1]` (with `prefix[0] = 0`).

Then XOR of subarray `[l..r]` = `prefix[r+1] ^ prefix[l]`.

This equals 0 iff `prefix[r+1] == prefix[l]` — so we just count **pairs of equal prefix XOR values**.

**Hash map counting**:
- Maintain a frequency map of prefix XOR values seen so far
- For each new prefix XOR value, the number of new beautiful subarrays ending here = `count[prefix]` (how many times this prefix XOR was seen before)
- Add that to the answer, then increment `count[prefix]`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long BeautifulSubarrays(int[] nums) {
        var count = new Dictionary<int, int>();
        count[0] = 1; // empty prefix

        int prefix = 0;
        long ans = 0;

        foreach (int x in nums) {
            prefix ^= x;
            if (count.TryGetValue(prefix, out int freq))
                ans += freq;
            count[prefix] = count.GetValueOrDefault(prefix, 0) + 1;
        }

        return ans;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Single pass; each dictionary lookup/insert is O(1) average |
| **Space Complexity** | **O(n)** | Dictionary stores at most n+1 distinct prefix XOR values |

**Optimal** — cannot do better than O(n) since we must read all elements.

---

## Why This Works — Proof & Intuition

**Why XOR = 0 means beautiful?**

Each operation subtracts `2^k` from exactly two elements. In terms of bits, this **clears bit k in both**. So every operation is a paired bit cancellation.

For ALL elements to reach 0, every bit position `k` must be clearable:
- Bit `k` is clearable iff there's an **even count** of elements that have bit `k` set (so they can be paired up)
- "Every bit appears even times" ⟺ XOR of all elements = 0

**Why prefix XOR works?**

`XOR(l..r) = prefix[r+1] ^ prefix[l]`

This equals 0 iff `prefix[r+1] == prefix[l]`. So counting beautiful subarrays = counting pairs `(l, r+1)` with equal prefix XOR.

## Example Walkthrough

**Example 1**: `nums = [4, 3, 1, 2, 4]`

| Step | x | prefix (XOR) | count[prefix] before | ans |
|------|---|--------------|----------------------|-----|
| init | — | 0 | {0:1} | 0 |
| 0 | 4 | 4 | 0 | 0 |
| 1 | 3 | 7 | 0 | 0 |
| 2 | 1 | 6 | 0 | 0 |
| 3 | 2 | 4 | 1 | **1** |
| 4 | 4 | 0 | 1 | **2** |

- At step 3: prefix = 4, seen before at step 0 → subarray `[3,1,2]` has XOR=0 ✓
- At step 4: prefix = 0, seen before at init → subarray `[4,3,1,2,4]` has XOR=0 ✓

Answer: **2** ✓

**Example 2**: `nums = [1, 10, 4]`

| Step | x | prefix | count[prefix] before | ans |
|------|---|--------|----------------------|-----|
| init | — | 0 | {0:1} | 0 |
| 0 | 1 | 1 | 0 | 0 |
| 1 | 10 | 11 | 0 | 0 |
| 2 | 4 | 15 | 0 | 0 |

No prefix repeats → Answer: **0** ✓

## Pitfalls & Edge Cases

- **Initialize `count[0] = 1`** — the empty prefix (before index 0) must be seeded; without it, subarrays starting at index 0 are never counted
- **Return type is `long`** — with n = 10⁵ elements all equal to 0, every subarray is beautiful → up to n*(n+1)/2 ≈ 5×10⁹ subarrays, overflows `int`
- **All zeros** — `nums = [0,0,0]`: every subarray has XOR=0 → all 6 subarrays are beautiful; the prefix XOR stays 0 throughout, so `count[0]` grows as 1,2,3 giving 1+2+3=6 ✓
- **Single element 0** — `nums = [0]`: prefix after = 0, count[0] was 1 → ans = 1 ✓ (the single-element subarray [0] is beautiful)
- **XOR range** — `nums[i] ≤ 10⁶ < 2²⁰`, so prefix XOR fits in 20 bits; could use array of size 2²⁰ instead of dictionary for slightly faster constant, but dictionary is cleaner

---

## Key Takeaway

**"Reduce all elements to 0 by cancelling pairs of bits"** = **"XOR of the subarray equals 0"**.

Once you see that, it becomes a classic **prefix XOR + hash map** problem:

- Build prefix XOR on the fly
- Count pairs of equal prefix XOR values using a frequency map
- Each match at index `r` contributes `count[prefix[r]]` new beautiful subarrays ending at `r`

Pattern: subarray property that depends on XOR → prefix XOR → count equal pairs → O(n) with a hash map.
