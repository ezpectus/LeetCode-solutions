# 3081. Replace Question Marks in String to Minimize Its Value
## O(n log 26) — Greedy Min-Heap + Sorted Assignment

---

## Problem Statement

- Replace each `?` with a lowercase letter to minimize the string's "value".
- `cost(i)` = number of times `s[i]` appeared before index `i`. Value = sum of all costs.
- Among all minimum-value solutions, return the lexicographically smallest.

---

## Core Idea — Two Phases

**Phase 1: Choose which characters to assign to `?` positions**

`cost(i)` for character `c` at position `i` = frequency of `c` in `s[0..i-1]`. So each time we add a character, its cost equals its current frequency. To minimize total cost, we should always assign the character with the **current lowest frequency** (greedy, like Huffman).

Use a min-heap of `(frequency, char)` for all 26 letters. For each `?`, pop the minimum-frequency character, record it, push back with frequency+1.

**Phase 2: Assign chosen characters to `?` positions lexicographically**

Once we have the multiset of characters to place, sort them and assign in order left-to-right. This minimizes the lexicographic value of the result since smaller characters go to earlier positions.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string MinimizeStringValue(string s) {
        int[] freq = new int[26];
        int qCount = 0;
        foreach (char c in s) {
            if (c == '?') qCount++;
            else freq[c - 'a']++;
        }

        // Min-heap: (frequency, char_index)
        var heap = new PriorityQueue<(int freq, int ci), (int, int)>();
        for (int i = 0; i < 26; i++)
            heap.Enqueue((freq[i], i), (freq[i], i));

        // Phase 1: greedily pick qCount chars with min frequency
        char[] chosen = new char[qCount];
        for (int q = 0; q < qCount; q++) {
            var (f, ci) = heap.Dequeue();
            chosen[q] = (char)('a' + ci);
            heap.Enqueue((f+1, ci), (f+1, ci));
        }

        // Phase 2: sort chosen, assign left-to-right to '?' positions
        Array.Sort(chosen);
        var res = s.ToCharArray();
        int idx = 0;
        for (int i = 0; i < res.Length; i++)
            if (res[i] == '?')
                res[i] = chosen[idx++];

        return new string(res);
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n + q log 26)** ≈ **O(n)** | q = number of `?`s; heap has fixed size 26 |
| **Space** | **O(n)** | Result array + chosen array |

The heap always has exactly 26 elements, so heap operations are effectively O(1).

---

## Why Two Separate Phases?

**Phase 1** determines the optimal *multiset* of characters. Using the min-frequency character at each step minimizes total `cost` because each assignment's cost equals the character's current frequency — always choosing the minimum minimizes the sum.

**Phase 2** determines the optimal *placement*. Given the multiset, placing characters in sorted order left-to-right minimizes lexicographic value. Smaller characters earlier = lexicographically smaller string, without changing the total cost (cost only depends on *which* characters are assigned, not *where* relative to non-`?` characters).

---

## Example Walkthrough

**`s = "a?a?"`**

Initial freq: a=2. q_count=2.

Heap: {(0,'b'), (0,'c'), ..., (2,'a'), ...}

- Pick 1: (0,'b') → chosen=['b'], push (1,'b').
- Pick 2: (0,'c') → chosen=['b','c'], push (1,'c').

Chosen sorted: ['b','c'].

Assign: position 1 (first `?`) → 'b', position 3 (second `?`) → 'c'.

Result: "abac" ✓

---

## Pitfalls & Edge Cases

- **No `?` in string**: qCount=0, chosen is empty, return s unchanged.
- **All `?`**: assign 'a','b','c',... repeatedly — perfectly balanced.
- **PriorityQueue tie-breaking**: `(freq, char_index)` as composite key ensures that when frequencies are equal, the lexicographically smaller char is chosen first. This is important for Phase 1 to produce the optimal multiset.
- **Sort in Phase 2**: even if Phase 1 chose optimally, the order of placement matters for lexicographic minimality — always sort before assigning.

---

## Key Takeaway

"Minimize sum of 'repeated prefix counts' by filling `?`s" → **two-phase greedy**:

```
Phase 1: use min-heap to greedily pick chars with lowest current frequency
Phase 2: sort chosen chars, assign left-to-right to '?' positions
```

Phase 1 minimizes the total value (cost). Phase 2 ensures lexicographic minimality among all equal-value solutions.
