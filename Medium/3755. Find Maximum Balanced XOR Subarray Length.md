# 3755. Find Maximum Balanced XOR Subarray Length
## O(n) — Prefix XOR + Prefix Balance with HashMap

---

## Problem Statement

- You are given an integer array `nums` (1 ≤ n ≤ 10⁵, 0 ≤ nums[i] ≤ 10⁹).
- Find the longest subarray where:
  1. XOR of all elements = 0
  2. Equal count of even and odd numbers
- Return 0 if no such subarray exists.

---

## Core Idea — Two Prefix Conditions Collapsed into One Key

Both conditions can be expressed as prefix equalities:

**XOR = 0:** `prefixXor[r+1] == prefixXor[l]`
(prefix XOR at both endpoints are equal ⟺ XOR of subarray is 0)

**Equal odd/even count:** Define `balance[i]` = (odd count) − (even count) in prefix [0..i−1].
Then `balance[r+1] == balance[l]` ⟺ subarray [l, r] has equal odds and evens.

**Combined:** We need `(prefixXor[i], balance[i])` to match a previously seen pair. Store the **earliest** prefix index where each `(xor, balance)` pair appeared. When the same pair appears again at index `i`, the subarray length is `i − firstSeen`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxBalancedSubarray(int[] nums) {
        int n = nums.Length;
        var first = new Dictionary<(int, int), int>();

        // Prefix index 0: before any element, xor=0, balance=0
        first[(0, 0)] = 0;

        int xorVal = 0, balance = 0;
        int res = 0;

        for (int i = 0; i < n; i++) {
            xorVal ^= nums[i];
            balance += (nums[i] & 1) == 1 ? 1 : -1; // odd → +1, even → -1

            var key = (xorVal, balance);

            if (first.TryGetValue(key, out int j)) {
                // Subarray nums[j..i] satisfies both conditions
                res = Math.Max(res, i + 1 - j);
            } else {
                first[key] = i + 1; // Store prefix index after element i
            }
        }

        return res;
    }
}
```

## Why This Works — Example Walkthrough

**Example 1:** `nums = [3,1,3,2,0]`

| Prefix idx | Element | xorVal | balance | key      | Action         |
|------------|---------|--------|---------|----------|----------------|
| 0          | —       | 0      | 0       | (0,0)    | store → 0      |
| 1          | 3 (odd) | 3      | +1      | (3,1)    | store → 1      |
| 2          | 1 (odd) | 2      | +2      | (2,2)    | store → 2      |
| 3          | 3 (odd) | 1      | +3      | (1,3)    | store → 3      |
| 4          | 2 (even)| 3      | +2      | (3,2)    | store → 4      |
| 5          | 0 (even)| 3      | +1      | **(3,1)**| **seen at 1!** length = 5−1 = **4** |

Subarray `nums[1..4] = [1,3,2,0]`: XOR = 1^3^2^0 = 0, odds: {1,3}, evens: {2,0} ✓

**Output:** 4 ✓

Full array XOR = 0, 4 odds and 4 evens ✓

**Example 3:** `nums = [0]`

## Pitfalls & Edge Cases

- **Subarray must be non-trivial** — The initial `(0,0)` at prefix index 0 is the base case. A repeated `(0,0)` at index `i` means the entire subarray `[0..i-1]` is valid with length `i`.

- **Balance can go negative** — When there are more evens than odds, balance is negative. The HashMap handles this naturally.

- **Only store the earliest occurrence** — We want maximum length, so we skip updating `first[key]` if it already exists. Earlier = longer subarray when seen again.

- **Even length guaranteed** — If balance = 0, count_odd = count_even, so subarray length must be even. This is automatically enforced by the balance constraint.

- **nums[i] = 0** — Zero is even. `(0 & 1) == 0` → balance decrements. Its XOR contribution is 0 (no effect). Single `[0]` has XOR=0 but balance=−1, correctly excluded.

---

## Key Takeaway

Two independent subarray conditions → one combined prefix key:

| Condition | Prefix form |
|-----------|-------------|
| XOR = 0   | `prefixXor[r+1] == prefixXor[l]` |
| equal odd/even | `balance[r+1] == balance[l]` |

Pair them as `(prefixXor, balance)` and use a HashMap to find the earliest matching prefix in O(1).

**Pattern recognition:** Any time you have multiple "prefix must match" conditions on a subarray, combine them into a single tuple key and apply the classic "first seen" HashMap trick. Extends naturally to 3+ conditions as additional tuple elements.
