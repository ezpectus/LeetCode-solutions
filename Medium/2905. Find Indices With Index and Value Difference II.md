# 2905. Find Indices With Index and Value Difference II
## O(n) — Track Running Min/Max in Valid Left Window

---

## Problem Statement

- Find `i, j` such that `|i-j| ≥ indexDifference` and `|nums[i]-nums[j]| ≥ valueDifference`.
- Return `[i, j]` or `[-1, -1]`.

---

## Core Idea — Sweep j, Maintain Min/Max for Valid i's

For a fixed `j`, valid `i` values satisfy `i ≤ j - indexDifference`. As `j` increases, the set of valid `i`s grows by one element at a time (we add index `j - indexDifference`).

**To maximize `|nums[i] - nums[j]|`**, the best `i` candidate is either the **minimum** or **maximum** value seen so far in the valid window. Tracking both gives us the best possible match for any `j`.

**Algorithm**:
- `min_idx`, `max_idx`: indices of running min and max in valid window
- For each `j`: update window by adding new valid `i = j - indexDifference`; check both candidates

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] FindIndices(int[] nums, int indexDifference, int valueDifference) {
        int n = nums.Length;
        int minIdx = 0, maxIdx = 0;

        for (int j = 0; j < n; j++) {
            int i = j - indexDifference;
            if (i >= 0) {
                if (nums[i] < nums[minIdx]) minIdx = i;
                if (nums[i] > nums[maxIdx]) maxIdx = i;

                if (Math.Abs(nums[j] - nums[minIdx]) >= valueDifference)
                    return new[] { minIdx, j };
                if (Math.Abs(nums[j] - nums[maxIdx]) >= valueDifference)
                    return new[] { maxIdx, j };
            }
        }

        return new[] { -1, -1 };
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Single pass, O(1) per step |
| **Space** | **O(1)** | Only two index trackers |

---

## Why Min and Max are Sufficient

For fixed `j`, `|nums[i] - nums[j]|` is maximized when `nums[i]` is as far from `nums[j]` as possible — i.e., at the extreme (min or max) of the valid window. Any other `i` gives a smaller or equal absolute difference. So checking only `minIdx` and `maxIdx` covers all possible solutions.

---

## Example Walkthrough

**`nums=[5,1,4,1], indexDifference=2, valueDifference=4`**

| j | i=j-2 | Update min/max | Check min(idx=0,val=5) | Check max(idx=0,val=5) |
|---|-------|---------------|----------------------|----------------------|
| 0 | -1 | — | — | — |
| 1 | -1 | — | — | — |
| 2 | 0 | min=0(5),max=0(5) | |5-4|=1 < 4 | |5-4|=1 < 4 |
| 3 | 1 | min=1(1),max=0(5) | |1-1|=0 < 4 | |5-1|=4 ≥ 4 ✓ → **[0,3]** |

---

## Pitfalls & Edge Cases

- **`indexDifference = 0`**: `i = j`, `min_idx = max_idx = 0` initially; at `j=0`, `i=0` is valid → checks `|nums[0]-nums[0]| = 0 ≥ valueDifference` (true if `valueDifference = 0`)
- **`min_idx` and `max_idx` initialized to 0**: safe because `i=0` becomes valid at `j=indexDifference`; before that, `i<0` so we skip the update/check
- **Both candidates needed**: sometimes min gives the answer, sometimes max; checking both handles all cases

---

## Key Takeaway

"Find pair with index gap ≥ d and value gap ≥ v" → **sweep j, maintain running min/max index for valid i window**. Min/max exhaustively covers all possible maximizers of `|nums[i]-nums[j]|`. O(n) single pass.
