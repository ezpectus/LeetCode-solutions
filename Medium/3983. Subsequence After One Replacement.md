# LeetCode 3983. Subsequence After One Replacement

## Problem Description
You are given two strings `s` and `t` consisting of lowercase English letters. You may choose at most **one index** in `s` and replace the character at that index with any lowercase English letter.

Return `true` if it is possible to make `s` a subsequence of `t`; otherwise, return `false`.

---

## Method Explanation: Greedy Two-Pointers with Single Replacement Lookahead

To find if `s` can be a subsequence of `t` with at most one character replacement, we must match characters of `s` inside `t` while maintaining a strict left-to-right relative ordering of indices inside `t`.

We can maintain a pointer `sIdx` for `s` and iterate through `t` using `tIdx`. At each step, we check if we can match `s[sIdx]` with `t[tIdx]`:
1. **Exact Match:** If `s[sIdx] == t[tIdx]`, we successfully match the character without using any operations. We advance `sIdx`.
2. **Mismatch with Available Replacement:** If they don't match, we look ahead. If the remaining suffix of `s` (excluding the current mismatched character) can be matched as a perfect subsequence in the remaining suffix of `t`, it means we can safely use our single replacement on the current character. 

To implement the lookahead efficiently without nesting loops, we can precompute a matching array or track the maximum suffix matches greedily. Alternatively, since we only have **one** replacement, we can track two states dynamically: `matched_without_replacement` and `matched_with_replacement`.

## C++ Source Code

```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    bool canMakeSubsequence(string s, string t) {
        int sLen = s.length();
        int tLen = t.length();
        
        if (sLen > tLen) return false;

        // s_with_rep: length of prefix of s matched using exactly 1 replacement so far
        // s_no_rep: length of prefix of s matched strictly without any replacements
        int s_with_rep = 0;
        int s_no_rep = 0;

        for (int i = 0; i < tLen; ++i) {
            char t_char = t[i];

            // 1. Update the state that already used a replacement
            if (s_with_rep < sLen && s[s_with_rep] == t_char) {
                s_with_rep++;
            }

            // 2. A state that didn't use a replacement can transition to s_with_rep 
            // by spending its 1 replacement right now (matching any t_char)
            if (s_no_rep < sLen) {
                // If we use the replacement on s[s_no_rep], it matches t_char, 
                // so the new length with replacement becomes s_no_rep + 1
                s_with_rep = max(s_with_rep, s_no_rep + 1);
            }

            // 3. Update the clean state (no replacements used yet)
            if (s_no_rep < sLen && s[s_no_rep] == t_char) {
                s_no_rep++;
            }
        }

        // If either path successfully matched all characters of s, return true
        return (s_no_rep == sLen || s_with_rep == sLen);
    }
};
```

## Complexity Analysis

* **Time Complexity:** $\mathcal{O}(|t|)$
  * **Single Linear Pass:** The algorithm iterates through the target string `t` exactly once from start to finish.
  * **Constant Time State Transitions:** Inside the loop, all conditional updates, index increments, and maximum value computations occur in strict $\mathcal{O}(1)$ time.
  * **Total Time:** $\mathcal{O}(|t|)$, which scales linearly and operates efficiently well within the $10^5$ character constraints.

* **Space Complexity:** $\mathcal{O}(1)$
  * **Constant Auxiliary Allocation:** The solution does not allocate any additional arrays, hash maps, or dynamic data structures.
  * **Minimal Tracking Variables:** Storage requirements are bounded by a small number of primitive integer scalar fields (`s_with_rep`, `s_no_rep`) used to hold state counters.
