# 3899. Angles of a Triangle
## O(1) — Law of Cosines + Triangle Inequality

---

## Problem Statement

- Given three side lengths, determine if a valid triangle (positive area) can be formed.
- If yes, return the three internal angles (degrees), sorted ascending.
- Otherwise return an empty array.

---

## Core Idea — Triangle Inequality + Law of Cosines

**Validity check (strict triangle inequality):**
```
a + b > c, a + c > b, b + c > a
```
Equivalent to: `max(a,b,c) < sum of the other two`.

**Law of Cosines** — angle opposite side `a`:
```
cos(A) = (b² + c² - a²) / (2bc)
A = arccos(cos(A))
```

Compute all three angles this way, convert radians → degrees, sort ascending.

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    vector<double> internalAngles(vector<int>& sides) {
        double a = sides[0], b = sides[1], c = sides[2];

        // Triangle inequality (strict, for positive area)
        if (a + b <= c || a + c <= b || b + c <= a) return {};

        auto angle = [](double opp, double adj1, double adj2) {
            double cosA = (adj1*adj1 + adj2*adj2 - opp*opp) / (2*adj1*adj2);
            // Clamp for floating-point safety
            cosA = max(-1.0, min(1.0, cosA));
            return acos(cosA) * 180.0 / M_PI;
        };

        vector<double> angles = {
            angle(a, b, c), // angle opposite side a
            angle(b, a, c), // angle opposite side b
            angle(c, a, b)  // angle opposite side c
        };

        sort(angles.begin(), angles.end());
        return angles;
    }
};
```

## Example Walkthrough

**Example 1:** `sides = [3,4,5]`

Triangle check: 3+4=7>5 ✓, 3+5=8>4 ✓, 4+5=9>3 ✓. Valid.

- Angle opposite 3: cos = (16+25-9)/(2·4·5) = 32/40 = 0.8 → arccos(0.8) ≈ 36.87°
- Angle opposite 4: cos = (9+25-16)/(2·3·5) = 18/30 = 0.6 → arccos(0.6) ≈ 53.13°
- Angle opposite 5: cos = (9+16-25)/(2·3·4) = 0/24 = 0 → arccos(0) = 90°

Sorted: [36.870, 53.130, 90.000] ✓

**Example 2:** `sides = [2,4,2]`

2+2=4 not > 4 (degenerate/zero area) → return **[]** ✓

---

## Pitfalls & Edge Cases

- **Strict inequality** — `a+b == c` is a degenerate (zero-area) "triangle" — must return []. ✓
- **Clamping cosA** — floating-point errors near ±1 can cause `acos` to return NaN; clamp to [-1,1]. ✓
- **Sum of angles** — should sum to ≈180°, useful for sanity-checking. ✓

---

## Key Takeaway

Classic geometry formula application: triangle inequality for validity, Law of Cosines for angles. Always clamp `cos` values before `acos` to avoid domain errors from floating-point precision.
