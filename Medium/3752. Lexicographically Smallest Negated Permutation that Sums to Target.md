# 3752. Lexicographically Smallest Negated Permutation that Sums to Target

## O(n log n) — Greedy Construction

---

## Problem Statement

- Given `n` and `target`.
- Return lexicographically smallest array of size `n` where:
  - Sum = `target`
  - Absolute values form permutation [1, 2, ..., n]
- **Constraints**: 1 ≤ n ≤ 10⁵, -10¹⁰ ≤ target ≤ 10¹⁰

---

## Core Idea — Greedy Sign Assignment from Largest to Smallest

**Key insight**: Build array by processing numbers from **largest to smallest** (n down to 1), greedily choosing sign for each.

**For each number `i` (from n down to 1)**:
- Check if we can make `i` negative while still reaching target with remaining numbers
- If `SumUpTo(i-1) - i >= target`: can afford to make `i` negative
  - Add `-i` to result
  - Update target += i (since we used -i instead of +i)
- Else: must use `i` positive
  - Add `i` to result
  - Update target -= i

**After processing all**: check if target == 0 (exact match).

**Finally**: Sort result to get lexicographically smallest.

---

## Algorithm

```
S = n(n+1)/2
if |target| > S: return [] (impossible)

result = []

for i from n down to 1:
  // Can we use -i and still reach target with [1..i-1]?
  if SumUpTo(i-1) - i >= target:
    result.add(-i)
    target += i
  else:
    result.add(i)
    target -= i

if target != 0: return [] (didn't reach exact target)

sort(result) // Lex-smallest order
return result
```

## Full Implementation (C#)

```csharp
public class Solution {
    public int[] LexSmallestNegatedPerm(int n, long target) {
        long SumUpTo(long x) {
            return x * (x + 1) / 2;
        }
        
        long S = SumUpTo(n);
        
        // Check if target reachable
        if (target < -S || target > S) {
            return new int[0];
        }
        
        var list = new List<int>();
        
        // Process from largest to smallest
        for (int i = n; i >= 1; i--) {
            // Can we afford to make i negative?
            if (SumUpTo(i - 1) - i >= target) {
                target += i;
                list.Add(-i);
            } else {
                target -= i;
                list.Add(i);
            }
        }
        
        // Check if we reached exact target
        if (target != 0) {
            return new int[0];
        }
        
        // Sort for lexicographically smallest
        list.Sort();
        
        return list.ToArray();
    }
}
```

## Why This Works

**Greedy choice correctness**:
- At each step (processing number `i`), we decide: use `+i` or `-i`?
- If using `-i`, we need remaining numbers [1..i-1] to sum to `(target + i)`
- Max sum from [1..i-1] = SumUpTo(i-1)
- Min sum from [1..i-1] = -SumUpTo(i-1)
- We can use `-i` if: `SumUpTo(i-1) - i >= target` (after adjustment)
  - This means: max achievable with [1..i-1] is ≥ what we need

**Lexicographical order**:
- After greedy assignment, we have correct signs
- Sorting puts negatives first (smaller), then positives
- Within each group, sorted by absolute value → lex-smallest

## Example Walkthrough

### Example 1: `n = 3, target = 0`

**S = 6**, target valid ✓

**i = 3**:
- SumUpTo(2) - 3 = 3 - 3 = 0 >= 0? Yes ✓
- Use `-3`, target = 0 + 3 = 3
- list = [-3]

**i = 2**:
- SumUpTo(1) - 2 = 1 - 2 = -1 >= 3? No ✗
- Use `+2`, target = 3 - 2 = 1
- list = [-3, 2]

**i = 1**:
- SumUpTo(0) - 1 = 0 - 1 = -1 >= 1? No ✗
- Use `+1`, target = 1 - 1 = 0 ✓
- list = [-3, 2, 1]

**target == 0** ✓

**Sort**: [-3, 1, 2] ✓

### Example 2: `n = 1, target = 10^10`

**S = 1**, target = 10^10

10^10 > 1 → return [] ✓

## Key Takeaway

**Greedy from largest to smallest with feasibility check**. For each number, check if using negative sign still allows reaching target with smaller numbers. Finally sort for lex-order.

O(n log n) time, O(n) space. **Critical insight**: Processing largest-to-smallest allows greedy decision with simple feasibility check.
