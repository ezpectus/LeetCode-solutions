# 2597. The Number of Beautiful Subsets  
*O(2ⁿ) — Backtracking with Constraint Check*

---

## Problem Statement

You are given an array `nums` of positive integers and a positive integer `k`.

A **beautiful subset** is a **non-empty** subset of `nums` that does **not contain** two integers with an **absolute difference** equal to `k`.

Return the number of beautiful subsets of the array `nums`.

**Examples**:

**Example 1**  
Input: nums = [2,4,6], k = 2  
Output: **4**  
Beautiful subsets: [2], [4], [6], [2,6]  
([2,4], [4,6], [2,4,6] are invalid — difference 2)

**Example 2**  
Input: nums = [1], k = 1  
Output: **1**  
Only [1]

**Constraints**:
- 1 ≤ nums.length ≤ 18
- 1 ≤ nums[i], k ≤ 1000

---

## Solution Code

```csharp
public class Solution {
    private int BackTrackingSubsets(int[] nums, int k, int idx, List<int> cur, HashSet<int> visited) { 
        int len = nums.Length;
        if (idx == len) return cur.Count > 0 ? 1 : 0;

        int cnt = 0;
        // skip current:
        cnt += BackTrackingSubsets(nums, k, idx + 1, cur, visited);

        // include current idx:
        int curItem = nums[idx];
        if (!visited.Contains(curItem - k) && !visited.Contains(curItem + k)) {
            cur.Add(curItem);
            visited.Add(curItem);
            cnt += BackTrackingSubsets(nums, k, idx + 1, cur, visited);
            cur.RemoveAt(cur.Count - 1);
            visited.Remove(curItem);
        }

        return cnt;
    }

    public int BeautifulSubsets(int[] nums, int k) {
        Array.Sort(nums);
        int totalSubCnt = BackTrackingSubsets(nums, k, 0, new List<int>(), new HashSet<int>());
        return totalSubCnt;
    }
}
```

## Complexity

| **Metric**            | **Value**     | **Notes**                                      |
|-----------------------|---------------|------------------------------------------------|
| **Time Complexity**   | **O(2ⁿ)**     | At each of the n positions we make 2 choices: include the element or skip it<br>With n ≤ 18 → maximum 2¹⁸ = 262,144 recursive calls |
| **Space Complexity**  | **O(n)**      | Recursion stack depth is O(n)<br>List `cur` and HashSet `visited` use O(n) in the worst case (when subset is large) |

**Optimal** — exponential time complexity is completely acceptable and very fast in practice for n ≤ 18 (runs in milliseconds).

---

## Why This Works — Core Logic

**Approach**:

* Sort `nums` first (helps with early pruning in some cases, though not strictly required for correctness)

* Use **backtracking** to build all possible subsets:

  * At each index `idx`, we have two choices:
    - **Skip** the current element → recurse to `idx + 1`
    - **Take** the current element `curItem = nums[idx]` — but only if valid

  * To check if we can take `curItem`:
    - Look if `curItem - k` or `curItem + k` is already present in the current subset
    - Use `HashSet visited` for O(1) lookup
    - If either value exists → cannot take (would violate the difference constraint) → skip this branch
    - If safe → add `curItem` to current subset (`cur`) and to `visited` → recurse → backtrack (remove it)

* Base case: when `idx == len` (end of array reached)
  - If current subset is **non-empty** → count it as 1 beautiful subset
  - Empty subset is **not counted** (checked via `cur.Count > 0`)

**Correctness** —  
We systematically explore **all possible subsets** (2ⁿ total possibilities).  
The constraint is checked **immediately** when adding a number — if adding it would create a forbidden pair (difference k), we **prune** that branch early.  
The `HashSet visited` keeps track of numbers already chosen in the current path.  
Empty subset is excluded naturally by the condition `cur.Count > 0`.

---

## Key Takeaway

This is a **classic backtracking + constraint pruning** problem:

* Sort the array (optional but helpful)
* Use recursion to decide for each element: **take** or **skip**
* When taking an element x:
  * Check if x−k or x+k is already in the current subset (fast lookup with HashSet)
  * If violation → skip this choice
  * Else → add x to subset and visited → recurse → backtrack (remove)
* At the end of the array → count non-empty valid subsets

**Pure, clean, optimal** — O(2ⁿ) time, O(n) space, extremely fast for n ≤ 18, elegant backtracking with early pruning, and correctly counts all beautiful subsets without forbidden differences.
