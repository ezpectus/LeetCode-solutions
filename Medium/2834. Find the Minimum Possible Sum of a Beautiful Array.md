# 2834. Find the Minimum Possible Sum of a Beautiful Array
## O(1) — Math: Take 1..target//2, then target, target+1, ...

---

## Problem Statement

- `n` distinct positive integers, no two summing to `target`.
- Minimize the sum, return mod 10⁹+7.
- `n, target` up to 10⁹ → must be O(1).

---

## Core Idea — Two Non-Conflicting Ranges

**Which numbers conflict?** `x` and `y` conflict if `x + y == target` and `x ≠ y`. The conflicting pairs are `(1, target-1), (2, target-2), ..., (target//2-1, target//2+1)`.

**What about `target//2`?** Its only "partner" would be itself: `target//2 + target//2 = target`. But we can only have one copy (distinct elements), so `target//2` is always **safe** to include alone.

**Safe prefix**: `1, 2, ..., target//2` — these `target//2` numbers can all coexist (we always take the smaller of each conflicting pair, and `target//2` has no conflict partner).

After `target//2` numbers, skip `target//2+1 .. target-1` (each conflicts with someone in `1..target//2-1`), then resume from `target` onward (no conflicts: `target + x > target` for any `x ≥ 1`).

**Case 1**: `n ≤ target//2` → take `1..n`:
```
sum = n(n+1)/2
```

**Case 2**: `n > target//2` → take `1..target//2`, then `target..target+(n-half-1)`:
```
half = target // 2
rest = n - half
sum = half(half+1)/2  +  rest × target  +  rest(rest-1)/2
```

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    const long MOD = 1_000_000_007;
    // Modular inverse of 2: 500000004 (since 2 × 500000004 ≡ 1 mod 10⁹+7)
    const long INV2 = 500000004;

    public int MinimumPossibleSum(int n, int target) {
        long h = target / 2;  // NOT (target-1)/2 !
        long N = n;

        if (N <= h) {
            // sum = N*(N+1)/2
            return (int)(N % MOD * ((N + 1) % MOD) % MOD * INV2 % MOD);
        }

        long rest = N - h;
        // sum1 = h*(h+1)/2
        long sum1 = h % MOD * ((h + 1) % MOD) % MOD * INV2 % MOD;
        // sum2 = rest * target
        long sum2 = rest % MOD * (target % MOD) % MOD;
        // sum3 = rest*(rest-1)/2
        long sum3 = rest % MOD * ((rest - 1) % MOD) % MOD * INV2 % MOD;

        return (int)((sum1 + sum2 + sum3) % MOD);
    }
}
```

## Critical Bug: `target//2` vs `(target-1)//2`

For `target=6`:
- `(target-1)//2 = 2` → would only take `{1,2}`, skipping `3`
- `target//2 = 3` → correctly takes `{1,2,3}`

Why is `3` safe when `target=6`? Because `3+3=6` would require TWO copies of `3`, but distinct elements means we only have one → no conflict. So `target//2` (when `target` is even) has **no valid conflict partner** and is always safe.

---

## Example Walkthrough

**`n=16, target=6`**: `half = 6//2 = 3`

- Take `{1,2,3}`: sum = 6
- Need `rest = 16-3 = 13` more, starting from `target=6`:  
  `6+7+...+18` = 13 terms = `13×(6+18)/2 = 156`
- Total = **162** ✓

---

## Key Takeaway

"Minimum sum of n beautiful integers avoiding pairs summing to target" → take `1..target//2` (safe prefix), then `target, target+1, ...`. Use `target//2` not `(target-1)//2` — the midpoint `target//2` is safe since it can only conflict with itself (impossible with distinct elements).
