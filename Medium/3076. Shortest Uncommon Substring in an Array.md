# 3076. Shortest Uncommon Substring in an Array
## O(n² × L³) — Brute Force All Substrings

---

## Problem Statement

- For each `arr[i]`, find the shortest substring that does not appear in any other `arr[j]`.
- If tie in length, return lexicographically smallest.
- If none exists, return `""`.

---

## Core Idea — Try All Substrings by Length Then Lex Order

With `n ≤ 100` and `len ≤ 20`, brute force is feasible. For each string `arr[i]`:
1. Enumerate all substrings, grouped by **increasing length**.
2. Within each length group, sort **lexicographically**.
3. For each candidate substring, check if it appears in any other `arr[j]` (using `Contains`).
4. First candidate that's unique → answer. If none found after all lengths → `""`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string[] ShortestSubstrings(string[] arr) {
        int n = arr.Length;
        string[] res = new string[n];

        for (int i = 0; i < n; i++) {
            string s = arr[i];
            bool found = false;

            for (int len = 1; len <= s.Length && !found; len++) {
                // Collect all substrings of this length, sorted lex
                var candidates = new SortedSet<string>();
                for (int start = 0; start <= s.Length - len; start++)
                    candidates.Add(s.Substring(start, len));

                foreach (string sub in candidates) {
                    bool unique = true;
                    for (int j = 0; j < n; j++) {
                        if (j != i && arr[j].Contains(sub)) {
                            unique = false; break;
                        }
                    }
                    if (unique) {
                        res[i] = sub;
                        found = true;
                        break;
                    }
                }
            }

            if (!found) res[i] = "";
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n² × L³)** | n strings × L² substrings × n×L `Contains` check |
| **Space** | **O(L²)** | Candidate set per string |

With `n=100`, `L=20`: ~100 × 400 × 100 × 20 = 80M operations — acceptable.

## Example Walkthrough

**`arr=["cab","ad","bad","c"]`**

For `"cab"` (i=0):
- Length 1: "a", "b", "c". Check each against "ad","bad","c".
  - "a": in "ad" and "bad" → not unique.
  - "b": in "bad" → not unique.
  - "c": in "c" → not unique.
- Length 2: "ab", "ca". Sorted: ["ab","ca"].
  - "ab": in "ad"? No. "bad"? No. "c"? No. → **unique!** → res[0]="ab".

For `"ad"` (i=1):
- Length 1: "a","d". "a"∈"cab","bad". "d"∈"bad". None unique.
- Length 2: "ad". "ad"∈"bad". Not unique.
- No unique substring → res[1]="".

---

## Pitfalls & Edge Cases

- **`SortedSet`** deduplicates AND sorts: handles duplicate substrings (e.g., "aa" in "aab") and ensures lex-smallest is tried first.
- **`s.Contains(sub)` for substring check**: O(L) built-in, clean and correct.
- **Same string in arr**: problem guarantees checking only `j ≠ i`, handled by the `j != i` condition.
- **Length 1 never unique**: possible when every single character appears in some other string.

---

## Key Takeaway

"Shortest unique substring per string in small array" → **brute force: try all substrings by ascending length, check uniqueness against all others**:

```
for each arr[i]:
    for length 1..len(arr[i]):
        for each sub of this length (sorted lex):
            if sub not in any arr[j], j≠i: answer is sub
```

Small constraints (n≤100, L≤20) make O(n²L³) brute force viable. `SortedSet` ensures lex-smallest candidate is tried first within each length group.
