# 3412. Find Mirror Score of a String
## O(n) — Stack per Character

---

## Problem Statement

- Mirror of letter at position `c` in alphabet = letter at position `25-c` (a↔z, b↔y, etc.).
- For each index `i`, find the closest unmarked `j < i` where `s[j]` is the mirror of `s[i]`. Mark both, add `i-j` to score.
- Return total score.

---

## Core Idea — Stack per Character for O(1) Mirror Lookup

Maintain one stack per letter (26 stacks). Each stack holds indices of **unmarked** occurrences of that letter seen so far.

For each index `i` with character `c`:
- Compute `mirror = 25 - c`.
- If `stacks[mirror]` is non-empty: pop the **top** (closest unmarked j with s[j]=mirror of c). Add `i - j` to score. (Both i and j are now marked — just don't push i.)
- If empty: push `i` onto `stacks[c]` (waiting for its mirror).

The stack gives us the **closest** (most recent) unmarked index automatically via LIFO order.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long CalculateScore(string s) {
        var stacks = new Stack<int>[26];
        for (int i = 0; i < 26; i++) stacks[i] = new();
        long score = 0;

        for (int i = 0; i < s.Length; i++) {
            int c = s[i] - 'a';
            int mirror = 25 - c;
            if (stacks[mirror].Count > 0) {
                score += i - stacks[mirror].Pop();
            } else {
                stacks[c].Push(i);
            }
        }
        return score;
    }
}
```

| **Metric** | **Value** |

| **Space** | **O(n)** — stacks hold at most n indices total |

## Example Walkthrough

**`s = "aczzx"`**

| i | c | mirror | stacks[mirror] | action | score |
|---|---|--------|----------------|--------|-------|
| 0 | a | z | empty | push 0 to stacks[a] | 0 |
| 1 | c | x | empty | push 1 to stacks[c] | 0 |
| 2 | z | a | [0] | pop 0, score += 2-0=2 | 2 |
| 3 | z | a | empty | push 3 to stacks[z] | 2 |
| 4 | x | c | [1] | pop 1, score += 4-1=3 | 5 |

Answer: **5** ✓

---

## Pitfalls & Edge Cases

- **LIFO = closest**: the stack naturally gives the most recently seen (closest) unmarked index. No need to search.
- **`long` for score**: `n=10^5`, max single contribution = n-1 ≈ 10^5. Max total score ≈ n²/2 = 5×10^9 → overflows `int`, use `long`.
- **Self-mirror**: 'm' mirrors 'n' (25-12=13), not itself. No letter mirrors itself (`25-c = c` only if `c=12.5`, impossible). So we always look at a different stack.

---

## Key Takeaway

"Find closest unmarked mirror match for each char" → **one stack per letter, pop from mirror's stack when found, push to own stack otherwise**. O(n) with O(1) amortized per character.
