# 3784. Minimum Deletion Cost to Make All Characters Equal
## O(n) — Maximize Kept Cost: Total − max(charCost[c])

---

## Problem Statement

- Given string `s` of length n and cost array `cost` (1 ≤ cost[i] ≤ 10⁹).
- Delete characters so the resulting non-empty string has all equal characters.
- Return the minimum total deletion cost.

---

## Core Idea — One Invariant Collapses the Problem

We must pick exactly one target character `c` to keep. For every character in `s`:
- If it's `c`: keep it (deleting it only increases cost needlessly).
- If it's not `c`: must delete it.

So: **cost to make all-c = total − sum of costs of all `c` characters**.

To **minimize** deletion cost → **maximize** the sum of costs for the kept character.

Answer = `total − max over c of charCost[c]`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinCost(string s, int[] cost) {
        long[] charCost = new long[26];
        long total = 0;

        for (int i = 0; i < s.Length; i++) {
            charCost[s[i] - 'a'] += cost[i];
            total += cost[i];
        }

        long maxKept = 0;
        foreach (long c in charCost) {
            maxKept = Math.Max(maxKept, c);
        }

        return total - maxKept;
    }
}
```

---

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n) | Single pass + 26-element scan |
| Space Complexity | O(1) | Fixed 26-char cost buckets |

---

## Why Keeping ALL Occurrences Is Always Optimal

Suppose we target character `c`. Could it ever help to delete some occurrence of `c` itself?

No. Deleting `s[i] = c` costs `cost[i] > 0` and leaves the string still valid (all remaining chars are `c`). So deleting a `c` character strictly increases cost with no benefit. We always keep every occurrence of the target character.

---

## Example Walkthrough

**Example 1:** `s = "aabaac"`, `cost = [1,2,3,4,1,10]`

| char | indices | charCost |
|------|---------|----------|
| a    | 0,1,3,4 | 1+2+4+1=8 |
| b    | 2       | 3         |
| c    | 5       | 10        |

total = 21, maxKept = 10 (keep 'c')

Answer = 21 − 10 = **11** ✓

Delete all non-'c': indices 0,1,2,3,4 → cost 1+2+3+4+1=11.

---

**Example 2:** `s = "abc"`, `cost = [10,5,8]`

| char | charCost |
|------|----------|
| a    | 10       |
| b    | 5        |
| c    | 8        |

total = 23, maxKept = 10 (keep 'a')

Answer = 23 − 10 = **13** ✓

---

**Example 3:** `s = "zzzzz"`, `cost = [67,67,67,67,67]`

charCost[z] = 335 = total, maxKept = 335.

Answer = 335 − 335 = **0** ✓ (already all equal, delete nothing)

---

## Pitfalls & Edge Cases

- **Overflow** — n ≤ 10⁵ and cost[i] ≤ 10⁹ → total ≤ 10¹⁴. Use `long`. ✓

- **All same character** — charCost[c] = total, answer = 0. ✓

- **Single character** — If n=1, any choice keeps the only element, cost = 0. charCost[s[0]] = cost[0] = total, answer = 0. ✓

- **Only one occurrence of best character** — No problem; charCost correctly sums to just that one cost.

---

## Key Takeaway

The problem looks like it might need DP or greedy interval processing, but it collapses to a single formula:

**Answer = total − max(charCost[c] for c in 'a'..'z')**

because: pick the character whose kept occurrences are worth the most, delete everything else.

**Pattern recognition:** "Delete characters to make string uniform, minimize cost" → the cost structure is symmetric: we pay for everything we don't keep. So maximize what we keep (= maximize kept character's total cost).
