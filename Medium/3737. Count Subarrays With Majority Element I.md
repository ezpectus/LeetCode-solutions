# 3737. Count Subarrays With Majority Element I
## O(n²) — Prefix Balance with Strict Majority Check

---

## Problem Statement

- You are given an integer array `nums` (1 ≤ n ≤ 1000) and an integer `target`.
- Return the number of subarrays where `target` is the majority element.
- The majority element appears **strictly more than half** the time in the subarray.
- A subarray of length L needs target count > L/2, i.e. at least ⌊L/2⌋ + 1 occurrences.

---

## Core Idea — Balance Array + Prefix Sums

Key insight: Replace each element with +1 if it equals `target`, -1 otherwise. A subarray has `target` as its majority element if and only if the sum of its transformed values is **strictly positive** (count of target > count of non-target).

Steps:
- Build a balance array: `bal[i] = 1` if `nums[i] == target`, else `-1`
- For each subarray `[l, r]`, compute the prefix sum difference `prefix[r+1] - prefix[l]`
- Count all pairs `(l, r)` where this difference > 0

With n ≤ 1000, O(n²) brute force over all subarrays is fast enough.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int CountMajoritySubarrays(int[] nums, int target) {
        int n = nums.Length;
        int res = 0;

        for (int l = 0; l < n; l++) {
            int balance = 0;

            for (int r = l; r < n; r++) {
                // +1 if target, -1 otherwise
                balance += nums[r] == target ? 1 : -1;

                // Strictly positive balance means target is majority
                if (balance > 0) res++;
            }
        }

        return res;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n²) | Two nested loops over all subarrays |
| Space Complexity | O(1) | Only a running balance variable |

Optimal for n ≤ 1000 — total operations ≈ 500,000 which is well within limits.

## Why This Works — Example Walkthrough

**Example 1:** `nums = [1,2,2,3], target = 2`

Balance values: `[-1, +1, +1, -1]`

Fixing each left endpoint:

| l | r | Subarray | Balance | > 0? |
|---|---|----------|---------|------|
| 0 | 0 | [1]      | -1      | ✗    |
| 0 | 1 | [1,2]    | 0       | ✗    |
| 0 | 2 | [1,2,2]  | +1      | ✓    |
| 0 | 3 | [1,2,2,3]| 0       | ✗    |
| 1 | 1 | [2]      | +1      | ✓    |
| 1 | 2 | [2,2]    | +2      | ✓    |
| 1 | 3 | [2,2,3]  | +1      | ✓    |
| 2 | 2 | [2]      | +1      | ✓    |
| 2 | 3 | [2,3]    | 0       | ✗    |
| 3 | 3 | [3]      | -1      | ✗    |

Count = 5 ✓

---

**Example 2:** `nums = [1,1,1,1], target = 1`

All elements are target → balance always positive → every subarray counts.

Total subarrays = n*(n+1)/2 = 4*5/2 = 10 ✓

---

**Example 3:** `nums = [1,2,3], target = 4`

Target never appears → balance always negative or zero → res = 0 ✓

## Pitfalls & Edge Cases

- **Strict majority** — Balance must be **strictly > 0**, not ≥ 0. A subarray like `[2, 3]` with target=2 has balance = 0, meaning equal counts — target is NOT the majority.

- **Target not in array** — All balances are -1, never positive → res stays 0. Handled naturally.

- **All elements equal target** — Every subarray has fully positive balance → result is n*(n+1)/2.

- **Single element subarrays** — `[target]` has balance +1 > 0 → counted. `[other]` has balance -1 → not counted. Handled correctly.

- **Resetting balance** — The inner variable `balance` is reset to 0 at each new `l` by declaring it inside the outer loop. Don't carry balance across different starting points.

---

## Key Takeaway

Classic **balance/majority encoding** trick:
- Replace target → +1, non-target → -1
- Subarray sum > 0 ⟺ target appears more than half the time
- Reduces "majority element" condition to a simple positivity check

**Pattern recognition:** "Count subarrays where element appears more than half the time" → balance encoding with ±1, then count subarrays with positive sum. For larger n, this extends to O(n) using prefix sum frequency maps.

The n ≤ 1000 constraint is the hint that O(n²) is intended here — the harder version of this problem (larger n) would require the prefix sum + HashMap approach.
