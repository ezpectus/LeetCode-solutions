# 2718. Sum of Matrix After Queries
## O(q) — Reverse Processing with Seen Sets

---

## Problem Statement

- Start with n×n zero matrix.
- Queries: set entire row or column to a value (later queries overwrite earlier ones).
- Return the final sum.

---

## Core Idea — Process Queries in Reverse

**Key insight**: later queries overwrite earlier ones. If we process queries **from last to first**, the first time we see a row/column index is its **final effective write**. Earlier writes to the same row/column can be ignored.

**Contribution of a row query** (first time we see row `r` from the right):
- The entire row gets value `val`.
- But some columns have already been set by later column queries — those cells will be overwritten.
- So row `r` effectively contributes `val × (n - |seen_cols|)` cells.

**Symmetrically for column queries**: contributes `val × (n - |seen_rows|)` cells.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MatrixSumQueries(int n, int[][] queries) {
        var seenRow = new HashSet<int>();
        var seenCol = new HashSet<int>();
        long res = 0;

        for (int i = queries.Length - 1; i >= 0; i--) {
            int type = queries[i][0], idx = queries[i][1], val = queries[i][2];
            if (type == 0) { // row
                if (seenRow.Add(idx))           // first time = final write
                    res += (long)val * (n - seenCol.Count);
            } else {         // col
                if (seenCol.Add(idx))
                    res += (long)val * (n - seenRow.Count);
            }
        }
        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(q)** | One pass over queries, O(1) HashSet ops |
| **Space** | **O(n)** | At most n rows + n cols in seen sets |

---

## Why `n - seenCol.Count` (not `n - seenRow.Count`) for Row Queries

When we process a row query at position `i` (from the right), the cells in this row that will **not** be overwritten are exactly those in columns that have **no later column query**. The columns that DO have a later column query are already in `seenCol` — those cells belong to the column, not the row. So the row contributes only to the `n - seenCol.Count` "free" columns.

```
Row query at index r, value v:
  |seenCol| columns are already "owned" by later column queries → skip
  (n - |seenCol|) columns are free → row fills them with v
  contribution = v × (n - |seenCol|)
```

## Example Walkthrough

**Example 1**: `n=3, queries=[[0,0,1],[1,2,2],[0,2,3],[1,0,4]]`

Processing reversed: `[1,0,4]`, `[0,2,3]`, `[1,2,2]`, `[0,0,1]`

| query | type | idx | val | seenRow | seenCol | contribution |
|-------|------|-----|-----|---------|---------|-------------|
| [1,0,4] | col | 0 | 4 | {} | {0} | 4×(3-0)=**12** |
| [0,2,3] | row | 2 | 3 | {2} | {0} | 3×(3-1)=**6** |
| [1,2,2] | col | 2 | 2 | {2} | {0,2} | 2×(3-1)=**4** |
| [0,0,1] | row | 0 | 1 | {2,0} | {0,2} | 1×(3-2)=**1** |

Total: `12+6+4+1 = 23` ✓

## Pitfalls & Edge Cases

- **`long` for result** — `val` up to 10⁵, `n` up to 10⁴, up to 5×10⁴ queries → max sum ≈ 10⁵ × 10⁴ × n = 10¹³, overflows `int`
- **`HashSet.Add` returns bool** — in C#, `seenRow.Add(idx)` returns `true` if newly added → clean one-liner for "first time seen" check
- **Order matters**: must subtract `seenCol.Count` for row queries and `seenRow.Count` for col queries — not vice versa

---

## Key Takeaway

"Last write wins + sum of final matrix" → **process in reverse, first-seen = final write**:

- Row query contributes `val × (n - columns already finalized)`
- Column query contributes `val × (n - rows already finalized)`
- Track finalized rows/cols with HashSets

Avoids simulating the entire n×n matrix (n up to 10⁴ → 10⁸ cells would TLE).
