# 3447. Assign Elements to Groups with Constraints
## O(M log M) — Sieve-Style Multiples Marking

---

## Problem Statement

- For each group `g`, find the smallest-index element `e` such that `g % e == 0`. Return -1 if none.

---

## Core Idea — Mark Multiples (Sieve-Like)

For each element `e` at index `j`, every group value that is a multiple of `e` can be served by `j`. Iterate through all multiples of `e` up to `maxG` and update `best[mult] = min(best[mult], j)`.

After processing all elements, `best[g]` holds the smallest index among all elements that divide `g`.

Total work: `Σ maxG/e` for each distinct `e` = O(M log M) by harmonic series (where M = max group value).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] AssignElements(int[] groups, int[] elements) {
        int maxG = groups.Max();

        // For each element VALUE, keep only the smallest index
        int[] firstIdx = new int[maxG + 2];
        Array.Fill(firstIdx, int.MaxValue);
        for (int j = 0; j < elements.Length; j++)
            if (elements[j] <= maxG)
                firstIdx[elements[j]] = Math.Min(firstIdx[elements[j]], j);

        // Sieve: iterate over distinct values only
        int[] best = new int[maxG + 1];
        Array.Fill(best, int.MaxValue);
        for (int e = 1; e <= maxG; e++) {
            if (firstIdx[e] == int.MaxValue) continue;
            for (int mult = e; mult <= maxG; mult += e)
                best[mult] = Math.Min(best[mult], firstIdx[e]);
        }

        return groups.Select(g => best[g] == int.MaxValue ? -1 : best[g]).ToArray();
    }
}
```

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(M log M + n)** | M = max group value ≤ 10⁵; harmonic series for sieve |
| **Space** | **O(M)** | `best` array |

## Example Walkthrough

**`groups=[8,4,3,2,4], elements=[4,2]`**

M=8. Process j=0 (e=4): multiples 4,8 → best[4]=0, best[8]=0.
Process j=1 (e=2): multiples 2,4,6,8 → best[2]=1, best[4]=min(0,1)=0, best[6]=1, best[8]=min(0,1)=0.

Results: groups[0]=8→best[8]=0, groups[1]=4→0, groups[2]=3→∞→-1, groups[3]=2→1, groups[4]=4→0.

Answer: **[0,0,-1,1,0]** ✓

---

## Pitfalls & Edge Cases

- **Smallest index**: processing elements in order and using `min` ensures the smallest-index element wins.
- **`e > maxG`**: skip elements larger than max group value — they can't divide any group.
- **Duplicate elements**: naturally handled — whichever index comes first wins via the `min` check.

---

## Key Takeaway

"Assign smallest-index divisor to each group value" → **sieve: for each element, mark all multiples with min(current index)**. O(M log M) — same pattern as Sieve of Eratosthenes.
