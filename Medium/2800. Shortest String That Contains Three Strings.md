# 2800. Shortest String That Contains Three Strings
## O(1) — Try All 6 Permutations + Greedy Merge

---

## Problem Statement

- Find the shortest string containing `a`, `b`, `c` as substrings.
- If tie in length, return lexicographically smallest.

---

## Core Idea — Enumerate All Merge Orders

Only **6 permutations** of `{a, b, c}` exist. For each, greedily merge left-to-right: overlap the suffix of the left string with the prefix of the right string as much as possible. Pick the best result.

**`merge(s, t)`**: find the longest suffix of `s` that equals a prefix of `t`, then append the remainder of `t`.

```
merge("abc", "bca"):
  suffix "bc" == prefix "bc" of "bca" → "abc" + "a" = "abca"
```

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    string Merge(string s, string t) {
        if (s.Contains(t)) return s; // t already in s — no need to append anything
        for (int i = Math.Min(s.Length, t.Length); i > 0; i--)
            if (s.EndsWith(t.Substring(0, i)))
                return s + t.Substring(i);
        return s + t;
    }

    public string MinimumString(string a, string b, string c) {
        string[] strs = { a, b, c };
        string res = null;
        int[] perm = { 0, 1, 2 };

        foreach (var p in new[] {
            new[]{0,1,2}, new[]{0,2,1}, new[]{1,0,2},
            new[]{1,2,0}, new[]{2,0,1}, new[]{2,1,0} }) {
            string merged = Merge(Merge(strs[p[0]], strs[p[1]]), strs[p[2]]);
            if (res == null || merged.Length < res.Length ||
               (merged.Length == res.Length && string.Compare(merged, res) < 0))
                res = merged;
        }
        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(L²)** — L = max string length, for suffix matching per merge |
| **Space** | **O(L)** |

6 permutations × 2 merges × O(L) each = O(L) total, trivially fast for L≤100.

---

## Why All 6 Permutations

The optimal superstring might interleave the strings in any order. Since we have 3 strings, 3! = 6 orderings cover all possibilities. Greedy merge per ordering is optimal for that ordering — but different orderings can give different results, so we must try all.

---

## Pitfalls & Edge Cases

- **Substring containment**: if `t` is already a substring of `s`, must return `s` immediately — the suffix-overlap loop won't catch this case since overlap at `len(t)` would only match if `t` is a suffix of `s`, not a general substring. **Always check `s.Contains(t)` first.**
- **Tie-breaking**: compare both length and lexicographic order; update `res` only when strictly better

---

## Key Takeaway

"Shortest superstring of 3 strings" → **try all 6 permutations + greedy suffix-prefix merge**. With only 3 strings, brute force over orderings is optimal and trivial.
