# 2910. Minimum Number of Groups to Create a Valid Assignment
## O(min_freq × V) — Enumerate Minimum Group Size k

---

## Problem Statement

- Each box contains balls of one value only.
- All boxes must have sizes in `{k, k+1}` for some `k` (smallest and largest box differ by at most 1).
- Minimize total number of boxes.

---

## Core Idea — Try Each Minimum Group Size k

For a given `k` (minimum group size), each value with frequency `f` must be split into groups of exactly `k` or `k+1`. Enumerate `k` from 1 to `min_frequency` and find the valid `k` giving the fewest total groups.

**For frequency `f` with group size `k`**:
We need `a*k + b*(k+1) = f` with `a, b ≥ 0`. To **minimize groups** `a+b`, maximize `b` (use more k+1 groups, fewer total).

**Key observation**: `b ≡ f (mod k)` must hold (since `b*(k+1) ≡ b (mod k)` and we need `f ≡ 0 (mod k)` after subtracting `b*(k+1)`). So the smallest valid `b = f % k`. The largest valid `b`:

```
base = f % k
upper = f // (k+1)            // max possible b
if base > upper: impossible   // not enough room for even the smallest valid b
max_b = base + k * ((upper - base) // k)   // largest b ≡ base (mod k) ≤ upper
```

Minimum groups for this `f`:
```
total = a + max_b  where  a = (f - max_b*(k+1)) / k
```

**Answer**: minimum total groups over all valid `k`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinGroupsForValidAssignment(int[] balls) {
        var freq = new Dictionary<int,int>();
        foreach (int x in balls)
            freq[x] = freq.GetValueOrDefault(x) + 1;

        var freqs = freq.Values.ToArray();
        int minFreq = freqs.Min();
        int best = int.MaxValue;

        for (int k = 1; k <= minFreq; k++) {
            int total = 0;
            bool valid = true;

            foreach (int f in freqs) {
                int b_base = f % k;         // smallest valid b
                int upper  = f / (k + 1);   // max possible b

                if (b_base > upper) { valid = false; break; }

                int max_b = b_base + k * ((upper - b_base) / k);
                int a = (f - max_b * (k + 1)) / k;
                total += a + max_b;
            }

            if (valid) best = Math.Min(best, total);
        }

        return best;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(min_freq × V)** | V = number of distinct values; min_freq ≤ n |
| **Space** | **O(V)** | Frequency map |

In practice `min_freq` is small for typical inputs. Worst case when all balls have same value: V=1, min_freq=n, but inner loop is trivial → O(n).

## Why Maximize b (Groups of k+1)?

Total groups = `a + b = (f - b*(k+1))/k + b`.

As a function of `b`:
```
total = f/k - b*(k+1)/k + b = f/k + b*(1 - (k+1)/k) = f/k - b/k
```

Total **decreases** as `b` increases → maximize `b` to minimize total groups. ✓

## Pitfalls & Edge Cases

- **`k` only goes up to `min_freq`**: if `k > min_freq`, some frequency `f < k` can't form even one group → always invalid
- **`base > upper` → invalid**: the smallest valid `b` already exceeds the maximum possible `b`
- **Integer division throughout**: `(upper - base) / k` uses floor division to find how many full `k`-steps fit

---

## Key Takeaway

"Minimum boxes where all box sizes in {k, k+1}" → **enumerate k, greedily maximize groups of size k+1 per frequency**:

```
base = f % k          // smallest valid b (congruence requirement)
max_b = base + k * ((f/(k+1) - base) / k)    // largest valid b
groups(f,k) = a + max_b  where a = (f - max_b*(k+1))/k
```

Validity: `f%k ≤ f/(k+1)`. Optimality: maximizing `b` minimizes total groups.
