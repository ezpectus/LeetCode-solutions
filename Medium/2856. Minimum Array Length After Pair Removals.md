# 2856. Minimum Array Length After Pair Removals
## O(n) — Greedy: Max Frequency Determines the Bottleneck

---

## Problem Statement

- Remove pairs `(i, j)` where `nums[i] < nums[j]`. Repeat to minimize array length.
- Array is sorted in non-decreasing order.

---

## Core Idea — Maximum Frequency is the Bottleneck

Each removal requires two **distinct** values (`nums[i] < nums[j]`). The most frequent element is the bottleneck — it can only be removed by pairing with a different element.

**Two cases**:

**Case 1**: `maxFreq > n/2` (dominant element)  
The dominant element appears `maxFreq` times. All other elements combined = `n - maxFreq`. Each "other" element can pair with one dominant → we eliminate `n - maxFreq` dominant elements. Remaining = `maxFreq - (n - maxFreq) = 2*maxFreq - n`.

**Case 2**: `maxFreq ≤ n/2` (balanced)  
No single element dominates. We can always find pairs to remove. Pairs removed = `⌊n/2⌋`. Remaining = `n - 2*⌊n/2⌋ = n % 2`.

**Combined formula**:
```
result = max(2 * maxFreq - n, n % 2)
```

This works because:
- When `maxFreq > n/2`: `2*maxFreq - n > 0 ≥ n%2` (the dominant case applies)
- When `maxFreq ≤ n/2`: `2*maxFreq - n ≤ 0`, so `max(...) = n % 2` (balanced case applies)

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinLengthAfterRemovals(IList<int> nums) {
        int n = nums.Count;
        // Find max frequency (array is sorted, so count consecutive runs)
        int maxFreq = 1, currFreq = 1;
        for (int i = 1; i < n; i++) {
            if (nums[i] == nums[i-1]) currFreq++;
            else currFreq = 1;
            maxFreq = Math.Max(maxFreq, currFreq);
        }
        return Math.Max(2 * maxFreq - n, n % 2);
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Single pass to find max frequency |
| **Space** | **O(1)** | Only counters |

We exploit the sorted order to count frequencies in O(n) without a hash map.

## Example Walkthrough

**Example 4**: `nums=[2,3,4,4,4]`, n=5

- Frequencies: `{2:1, 3:1, 4:3}` → `maxFreq=3`
- `2*3 - 5 = 1`, `5%2 = 1`
- `max(1, 1) = 1` ✓

The three 4s can only pair with the two other elements (2 and 3). One 4 has no partner → 1 left.

**Example 3**: `nums=[10⁹, 10⁹]`, n=2

- `maxFreq=2`, `2*2-2=2`, `2%2=0`
- `max(2, 0) = 2` ✓

Both elements are equal → can't form a valid pair → neither removed.

---

## Pitfalls & Edge Cases

- **All same elements**: `maxFreq = n`, result = `max(2n-n, n%2) = max(n, n%2) = n` ✓
- **Already minimal**: `n%2` handles the odd-length balanced case correctly
- **Using sorted property**: count max freq in O(n) via consecutive run counting instead of a hash map

---

## Key Takeaway

"Minimum remaining after removing pairs with distinct values" → **max frequency is the bottleneck**:

```
result = max(2 * maxFreq - n,  n % 2)
              ↑                  ↑
         dominant case      balanced case
```

If the most common element occupies more than half the array, it limits how many pairs we can form. Otherwise, every element can find a partner (modulo one leftover for odd n).
