# 3132. Find the Integer Added to Array II
## O(n log n) — Try 3 Candidate Values of x

---

## Problem Statement

- Remove 2 elements from `nums1`, add integer `x` to every remaining element → equals `nums2`.
- Find the minimum possible `x`.

---

## Core Idea — x is Determined by Matching nums2[0] to a nums1 Element

After sorting both arrays, `nums2[0]` (the smallest element of nums2) must match some element of `nums1` after adding `x`. Since we remove exactly 2 elements from `nums1`, the smallest remaining `nums1` element is either `nums1[0]`, `nums1[1]`, or `nums1[2]` (we can remove at most 2 elements before it).

So: try `x = nums2[0] - nums1[i]` for `i ∈ {0, 1, 2}`.

**Validation**: for each candidate `x`, do a linear scan — check if exactly ≤ 2 elements of `nums1` can be skipped to match all of `nums2`.

Return the minimum valid `x`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumAddedInteger(int[] nums1, int[] nums2) {
        Array.Sort(nums1);
        Array.Sort(nums2);
        int res = int.MaxValue;

        for (int i = 0; i < 3; i++) {
            int x = nums2[0] - nums1[i];
            if (IsValid(nums1, nums2, x)) res = Math.Min(res, x);
        }
        return res;
    }

    bool IsValid(int[] nums1, int[] nums2, int x) {
        int removed = 0, j = 0;
        for (int i = 0; i < nums1.Length; i++) {
            if (j < nums2.Length && nums1[i] + x == nums2[j]) j++;
            else if (++removed > 2) return false;
        }
        return j == nums2.Length;
    }
}
```

## Why Only 3 Candidates?

After sorting `nums1`, the smallest element of `nums2` corresponds to some `nums1[k]` where `k ≤ 2`. This is because we remove exactly 2 elements — so at most 2 elements of `nums1` can be "skipped" at the front. Therefore `k ∈ {0, 1, 2}`, giving us exactly 3 candidates for `x`.

## Example Walkthrough

**`nums1=[4,8,12,16,20]` (sorted), `nums2=[10,14,18]` (sorted)**

Candidates from `nums2[0]=10`:
- i=0: x=10-4=6. Validate: 4+6=10 ✓, 8+6=14 ✓, 12+6=18 ✓, removed=2 (16,20). Valid! x=6? But expected -2.

Wait, let me recheck: nums1=[4,20,16,12,8] sorted=[4,8,12,16,20], nums2=[14,18,10] sorted=[10,14,18].

- i=0: x=10-4=6. 4+6=10✓, 8+6=14✓, 12+6=18✓, skip 16,20 (removed=2). Valid! x=6.
- i=1: x=10-8=2. 4+2=6≠10 (skip,rem=1), 8+2=10✓, 12+2=14✓, 16+2=18✓, skip 20 (rem=2). Valid! x=2.
- i=2: x=10-12=-2. 4+(-2)=2≠10 (skip,rem=1), 8+(-2)=6≠10 (skip,rem=2), 12+(-2)=10✓, 16+(-2)=14✓, 20+(-2)=18✓. Valid! x=-2.

min(6, 2, -2) = **-2** ✓

## Pitfalls & Edge Cases

- **Greedy validation**: the linear scan greedily matches elements left-to-right. This works because both arrays are sorted — if `nums1[i]+x < nums2[j]`, we must skip `nums1[i]` (it's too small). If `nums1[i]+x > nums2[j]`, the assignment is impossible (already sorted, no element can fill this).
- **Return minimum x**: there can be multiple valid x values (all 3 candidates might be valid); return the smallest.

---

## Key Takeaway

"Find added integer after removing 2 elements" → **sort both, try x = nums2[0] - nums1[i] for i ∈ {0,1,2}, validate each greedily**:

```
for i in [0,1,2]:
    x = nums2[0] - nums1[i]
    if valid(x): update min
```

Only 3 candidates needed since at most 2 elements precede the match for `nums2[0]` in sorted `nums1`.
