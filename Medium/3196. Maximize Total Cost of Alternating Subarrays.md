# 3196. Maximize Total Cost of Alternating Subarrays
## O(n) — DP on Sign State

---

## Problem Statement

- Split `nums` into subarrays. Cost of subarray `[l..r]` = `nums[l] - nums[l+1] + nums[l+2] - ...` (alternating signs starting with +).
- Maximize total cost across all subarrays.

---

## Core Idea

Within any subarray, signs alternate: first element positive, second negative, etc. When a new subarray starts, its first element is always positive.

**DP states**:
- `dp_plus` = max total cost where the current element was added with **+** sign.
- `dp_minus` = max total cost where the current element was added with **-** sign.

**Transitions** for element `nums[i]`:
- Add with `+`: either start a new subarray here (best of `dp_plus`, `dp_minus` + `nums[i]`), or this is the first element — always `+`.
- Add with `-`: must continue the current subarray after a `+` element: `dp_plus - nums[i]`.

```
new_plus  = max(dp_plus, dp_minus) + nums[i]
new_minus = dp_plus - nums[i]
```

**Answer**: `max(dp_plus, dp_minus)` after processing all elements.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumTotalCost(int[] nums) {
        long dpPlus = nums[0], dpMinus = long.MinValue / 2;

        for (int i = 1; i < nums.Length; i++) {
            long newPlus  = Math.Max(dpPlus, dpMinus) + nums[i];
            long newMinus = dpPlus - nums[i];
            dpPlus  = newPlus;
            dpMinus = newMinus;
        }

        return Math.Max(dpPlus, dpMinus);
    }
}
```

## Example Walkthrough

**`nums=[1,-2,3,4]`**

| i | nums[i] | dp_plus | dp_minus |
|---|---------|---------|----------|
| — | — | 1 | -∞ |
| 1 | -2 | max(1,-∞)+(-2)=-1 | 1-(-2)=3 |
| 2 | 3 | max(-1,3)+3=6 | -1-3=-4 |
| 3 | 4 | max(6,-4)+4=10 | 6-4=2 |

Answer: max(10,2) = **10** ✓

---

## Pitfalls & Edge Cases

- **`dp_minus` starts at `-∞`**: you can't start a subarray with a negative sign — first element is always positive.
- **`long` for values**: `n=10^5`, values up to `10^9` → sum up to `10^{14}`, overflows `int`.
- **Single element**: loop doesn't run, return `dp_plus = nums[0]` ✓.

---

## Key Takeaway

"Max cost of alternating-sign subarrays" → **DP on sign state `(+, -)`**:

```
dp_plus  = max(dp_plus, dp_minus) + nums[i]   // start new or continue as +
dp_minus = dp_plus_prev - nums[i]             // continue as - (only after +)
```

O(n) with O(1) space. The sign alternation constraint is captured by only allowing `-` after `+`.
