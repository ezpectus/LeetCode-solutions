# 2906. Construct Product Matrix
## O(n×m) — Flatten to 1D + Prefix/Suffix Products

---

## Problem Statement

- For each cell `(i,j)`, compute the product of all other cells modulo 12345.
- Return the result matrix.

---

## Core Idea — 1D Prefix/Suffix Product (Classic Pattern)

**Step 1**: Flatten the 2D grid into a 1D array. Cell `(i,j)` maps to index `k = i*m + j`. This reduces the problem to the classic "product of all except self" on a 1D array.

**Step 2**: Compute prefix and suffix products:
- `prefix[k]` = product of `flat[0..k-1]`
- `suffix[k]` = product of `flat[k+1..total-1]`

**Step 3**: `result[k] = prefix[k] * suffix[k]` — product of everything except `flat[k]`.

**Modular arithmetic**: apply `% 12345` at each step to avoid overflow. Note `grid[i][j]` up to `10⁹` → always take mod when adding to flat.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[][] ConstructProductMatrix(int[][] grid) {
        const int MOD = 12345;
        int n = grid.Length, m = grid[0].Length, total = n * m;

        // Flatten
        int[] flat = new int[total];
        for (int i = 0; i < n; i++)
            for (int j = 0; j < m; j++)
                flat[i*m+j] = grid[i][j] % MOD;

        // Prefix products
        int[] prefix = new int[total];
        prefix[0] = 1;
        for (int k = 1; k < total; k++)
            prefix[k] = (int)((long)prefix[k-1] * flat[k-1] % MOD);

        // Suffix products
        int[] suffix = new int[total];
        suffix[total-1] = 1;
        for (int k = total-2; k >= 0; k--)
            suffix[k] = (int)((long)suffix[k+1] * flat[k+1] % MOD);

        // Build result
        int[][] res = new int[n][];
        for (int i = 0; i < n; i++) {
            res[i] = new int[m];
            for (int j = 0; j < m; j++) {
                int k = i*m+j;
                res[i][j] = (int)((long)prefix[k] * suffix[k] % MOD);
            }
        }
        return res;
    }
}
```

## Why Flatten to 1D

The 2D structure is irrelevant — we need the product of all elements except one. The 1D prefix/suffix pattern works identically regardless of how elements are arranged, since multiplication is commutative. Flattening simplifies the index arithmetic.

---

## Example Walkthrough

**`grid=[[1,2],[3,4]]`** → `flat=[1,2,3,4]`, MOD=12345

```
prefix = [1, 1, 2, 6]        (prefix[k] = product of flat[0..k-1])
suffix = [24, 12, 4, 1]      (suffix[k] = product of flat[k+1..])

result:
k=0: 1 * 24 = 24  → (0,0)
k=1: 1 * 12 = 12  → (0,1)
k=2: 2 *  4 =  8  → (1,0)
k=3: 6 *  1 =  6  → (1,1)
```

Output: `[[24,12],[8,6]]` ✓

---

## Pitfalls & Edge Cases

- **MOD = 12345** (not 10⁹+7) — smaller modulus means `prefix[k] * suffix[k]` ≤ 12344² ≈ 1.5×10⁸ which fits in `int`, but use `long` intermediate to be safe
- **`flat[k] = grid[i][j] % MOD`** — reduce input immediately; `grid[i][j]` up to 10⁹ but we only care mod 12345
- **`prefix[0] = 1`**, `suffix[total-1] = 1`** — boundary base cases (empty product = 1)

---

## Key Takeaway

"Product of all except self in 2D matrix" → **flatten to 1D, apply classic prefix/suffix product**:

```
result[k] = prefix[k] * suffix[k]
prefix[k] = product of elements before k
suffix[k] = product of elements after k
```

The 2D→1D flattening is the key insight — reduces a 2D problem to a well-known 1D pattern in O(n×m) with no division needed.
