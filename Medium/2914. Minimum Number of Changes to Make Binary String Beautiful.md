# 2914. Minimum Number of Changes to Make Binary String Beautiful
## O(n) ‚Äî Greedy Pair Scanning

---

## Problem Statement

- Given a 0-indexed binary string `s` of even length.
- A string is **beautiful** if it can be partitioned into substrings where each substring has even length and contains only `'0'`s or only `'1'`s.
- You can change any character to `'0'` or `'1'`.
- Return the **minimum number of changes** to make `s` beautiful.

---

## Core Idea ‚Äî Scan Adjacent Pairs

**Key insight**: since every substring in the partition must have even length, we can think of the string as fixed non-overlapping pairs: `(s[0],s[1]), (s[2],s[3]), ...`

Any valid beautiful string must have each of these pairs be uniform ‚Äî both `'0'` or both `'1'`. If a pair is already uniform ‚Üí cost 0. If a pair is mixed (`"01"` or `"10"`) ‚Üí cost 1 (change one character).

**Why pairs?** Any even-length partition of an even-length string can be reduced to pairs without loss ‚Äî a uniform block of length 4 is just two uniform pairs. The pair grid `(0,1),(2,3),...` is the minimal even-length tiling and captures all constraints.

**Greedy algorithm**:
- Walk `s` in steps of 2
- For each pair `(s[i], s[i+1])`: if they differ ‚Üí increment answer

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinChanges(string s) {
        int changes = 0;
        for (int i = 0; i < s.Length; i += 2)
            if (s[i] != s[i + 1])
                changes++;
        return changes;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Single pass through string, step 2 |
| **Space** | **O(1)** | No extra storage |

---

## Why This Works ‚Äî The Pair Argument

Any beautiful string must be partitionable into even-length uniform blocks. The key observation:

- Every even-length uniform block covers **complete pairs** from our fixed `(0,1),(2,3),...` grid
- Therefore each pair in the grid must itself be uniform (all within one block or split across two ‚Äî but split across two blocks means both sides are uniform anyway, so the pair is still uniform)
- Fixing a mixed pair costs exactly **1 change** (change either character to match the other)
- Pairs are **independent** ‚Äî fixing one never affects another

So the answer is simply the count of mixed pairs.

## Pitfalls & Edge Cases

- **Don't try to build the partition explicitly** ‚Äî you only need to count, not reconstruct
- **Step by 2, not 1** ‚Äî scanning every character and comparing neighbors is wrong; the pair grid is fixed at even indices
- **Already beautiful strings** ‚Äî all pairs uniform ‚Üí answer is 0, no special case needed

---

## Key Takeaway

"Partition into even-length uniform substrings" ‚Üí **fix the pair grid `(0,1),(2,3),...` and count mixed pairs**:

```
for i in 0, 2, 4, ..., n-2:
    if s[i] != s[i+1]: changes++
```

Even-length uniform blocks always align to the fixed pair grid ‚Üí pairs are independent ‚Üí one pass, O(n) time, O(1) space.
