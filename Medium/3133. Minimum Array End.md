# 3133. Minimum Array End
## O(log n + log x) — Spread (n-1) Into Free Bit Positions of x

---

## Problem Statement

- Build array of `n` strictly increasing positive integers where `AND` of all = `x`.
- Return minimum possible last element.

---

## Core Idea — Fixed Bits and Free Bits

For AND of all elements to equal `x`, every element must have all bits of `x` set (the bits of `x` are "locked"). The bits NOT in `x` (zero-bits of `x`) are "free" — they can vary between elements.

The `n` smallest distinct elements all having `x`'s bits set are obtained by:
- Starting with `x` itself (free bits all 0).
- Then "counting up" using only the free bit positions.

The values are: `x`, `x OR f1`, `x OR f2`, ..., `x OR f(n-1)` where `f0=0, f1, f2, ...` are consecutive values spread into the free bit positions of `x`.

**Answer**: take `n-1`, spread its binary representation into the free bit positions of `x`.

**Spreading**: iterate through bit positions 0, 1, 2, .... For each bit position where `x` has a 0 (free bit), place the corresponding bit of `(n-1)` there.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinEnd(int n, int x) {
        long result = x;
        long remaining = n - 1L;
        int bit = 0;

        while (remaining > 0) {
            if ((((long)x >> bit) & 1) == 0) {  
                if ((remaining & 1) == 1) result |= (1L << bit);
                remaining >>= 1;
            }
            bit++;
        }

        return result;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(log n + log x)** | Iterate through bits of x and n-1; at most 60 bits total |
| **Space** | **O(1)** |  |

---

## Why This Gives the Minimum Last Element

The elements of our array are `x | spread(0), x | spread(1), ..., x | spread(n-1)` where `spread(k)` maps integer `k` into the free bit positions of `x`. These are strictly increasing (since free bits increment), all have `x`'s bits set (AND = x), and `spread(n-1)` is the minimum possible last element because we can't have the last element smaller while still having `n` elements.

---

## Example Walkthrough

**`n=3, x=4 (binary: 100)`**

Free bit positions (where `x` has 0): positions 0, 1, 3, 4, 5, ...

`n-1 = 2 = binary 10`.

| bit | x bit | action |
|-----|-------|--------|
| 0 | 0 (free) | bit 0 of remaining(2)=0 → don't set. remaining=1. |
| 1 | 0 (free) | bit 0 of remaining(1)=1 → set bit 1. remaining=0. Done. |

Result = 4 | (1 << 1) = 4 | 2 = **6** ✓

**`n=2, x=7 (binary: 111)`**

Free bits: positions 3, 4, 5, ...

`n-1 = 1 = binary 1`.

| bit | x bit | action |
|-----|-------|--------|
| 0 | 1 (locked) | skip |
| 1 | 1 (locked) | skip |
| 2 | 1 (locked) | skip |
| 3 | 0 (free) | bit 0 of remaining(1)=1 → set bit 3. remaining=0. Done. |

Result = 7 | 8 = **15** ✓

---

## Pitfalls & Edge Cases

- **`n=1`**: `n-1=0`, loop doesn't execute, return `x` directly ✓.
- **`long` for result and remaining**: `n` up to 10⁸, `x` up to 10⁸; final answer can exceed 10⁸ → use `long`. `n-1` should also be `long` to avoid overflow in bit shifts.
- **`remaining >>= 1` consumes bits of (n-1)**: loop terminates when all bits of `n-1` are placed.

---

## Key Takeaway

"Minimum last element of n-length AND-x array" → **spread binary `(n-1)` into free bit positions of x**:

```
for bit = 0, 1, 2, ...:
    if x has 0 at this bit:
        place next bit of (n-1) here in result
        consume that bit from (n-1)
```

The "free bits" act as a counter; we need `n-1` steps, so the final state of the counter is `n-1` spread across those free positions.
