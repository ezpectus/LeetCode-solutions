# 1344. Angle Between Hands of a Clock — Architectural Clock Hand Position Calculation  
*O(1) — Optimal Direct Mathematical Formula*

---

## Problem Statement

Given `hour` (1–12) and `minutes` (0–59).

Return **smaller angle** (in degrees) between hour and minute hands.

Accept within **10⁻⁵** accuracy.

---

## Core Idea — Direct Position Calculation

**Key insight**:
- Minute hand: **6° per minute** → `6 * minutes`
- Hour hand: **30° per hour** (360/12) + **0.5° per minute** (30/60) → `30 * hour + 0.5 * minutes`
- Difference → `|hour_pos - min_pos|`
- Smaller angle → `min(diff, 360 - diff)`

**No loops** — pure math.

---

##  Implementation (C#)

```csharp
public class Solution {
    public double AngleClock(int hour, int minutes) {
        hour %= 12;
        double minAngle = 6.0 * minutes;
        double hourAngle = 30.0 * hour + 0.5 * minutes;
        double diff = Math.Abs(hourAngle - minAngle);

        return Math.Min(diff, 360.0 - diff);
    }
}
```

## Why This Works — Example Walkthrough

**Example 1**: `hour=12`, `minutes=30`

- Minute hand: `6 × 30 = 180°`
- Hour hand: `30 × 0 + 0.5 × 30 = 15°`
- Difference: `|15 - 180| = 165°`
- Smaller angle: `min(165, 360 - 165) = 165°`

**Correct**.

**Example 2**: `hour=3`, `minutes=30`

- Minute hand: `180°`
- Hour hand: `30 × 3 + 0.5 × 30 = 105°`
- Difference: `|105 - 180| = 75°`
- Smaller angle: `75°`

**Correct**.

**Example 3**: `hour=3`, `minutes=15`

- Minute hand: `6 × 15 = 90°`
- Hour hand: `30 × 3 + 0.5 × 15 = 97.5°`
- Difference: `|97.5 - 90| = 7.5°`
- Smaller angle: `7.5°`

**Correct**.

## Pitfalls & Edge Cases

- **hour=12** → correctly treated as 0 (hour %= 12)
- **minutes=0** → hour hand at exact hour position
- **Large difference** → always return smaller angle (≤180°)
- **Floating point precision** → `double` provides sufficient accuracy

All handled perfectly.
