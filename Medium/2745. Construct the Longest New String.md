# 2745. Construct the Longest New String
## O(1) — Greedy: Alternate AA/BB, Insert All AB

---

## Problem Statement

- Concatenate some `"AA"`, `"BB"`, `"AB"` strings without creating `"AAA"` or `"BBB"`.
- Return the maximum total length.

---

## Core Idea — Structure of a Valid String

**`"AB"` strings are always safe**: `AB` contributes one A and one B, never creates `AA` or `BB` on its own. It can be inserted anywhere without breaking constraints.

**`"AA"` and `"BB"` must alternate**: placing two `"AA"` in a row gives `...AAAA...` → contains `"AAA"`. Similarly for `"BB"`. So `"AA"` and `"BB"` must strictly alternate: `AABB AABB...` or `BBAABB...`.

**How many AA/BB blocks can we use?**

Let `m = min(x, y)`. We can always alternate `m` copies of `"AA"` and `m` copies of `"BB"`.

If `x ≠ y` (unequal counts): the larger side gets **one extra** block at the beginning or end:
```
BB AA BB AA BB  (y=3, x=2: one extra BB on the end)
```
Total AA/BB blocks = `2m + 1`.

If `x == y`: exactly `2m` blocks, no extra (adding one more would require a match from the other side).

**Formula**:
```
blocks = 2 * min(x,y) + (x != y ? 1 : 0) + z
length = blocks * 2
```

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int LongestString(int x, int y, int z) {
        int m = Math.Min(x, y);
        int blocks = 2 * m + (x != y ? 1 : 0) + z;
        return blocks * 2;
    }
}
```

## Example Walkthrough

**Example 1**: `x=2, y=5, z=1`

`m=2, x≠y` → extra=1. Blocks = 4+1+1 = 6. Length = **12** ✓
String: `BB AA BB AA BB AB` → "BBAABBAABBAB" (no AAA, no BBB) ✓

**Example 2**: `x=3, y=2, z=2`

`m=2, x≠y` → extra=1. Blocks = 4+1+2 = 7. Length = **14** ✓
String: `AB AB AA BB AA BB AA` → "ABABAABBAABBAA" ✓

---

## Pitfalls & Edge Cases

- **`x == y`**: no extra AA/BB block — adding one more of either would need a balancing block from the other side which we don't have
- **All AB (`x=0, y=0`)**: `m=0`, no extra, just `z` blocks → length = `2z` ✓
- **`z=0`**: just alternating AA/BB, formula still correct

---

## Key Takeaway

"Max length without AAA/BBB" → **count alternating AA/BB pairs + all AB**:

```
length = (2 * min(x,y) + (x≠y ? 1 : 0) + z) * 2
```

`AB` is always insertable. `AA` and `BB` alternate with the smaller side determining the pair count; the larger side gets one bonus block at an end.
