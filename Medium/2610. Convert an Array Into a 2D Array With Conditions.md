# 2610. Convert an Array Into a 2D Array With Conditions
## O(n) — Frequency Map Row Assignment

---

## Problem Statement

- You are given an integer array `nums` (1 ≤ n ≤ 200, 1 ≤ `nums[i]` ≤ n).
- Build a 2D array using all elements of `nums` where **each row contains distinct integers**.
- **Minimize the number of rows**.
- Return any valid result.

---

## Core Idea — Frequency Map Row Assignment

**Key insight**: The minimum number of rows needed equals the **maximum frequency** of any element. The `k`-th occurrence of a value must go into row `k` (0-indexed) — it can't share a row with any earlier occurrence.

**Greedy assignment**:
- Track `freq[x]` = how many times `x` has been seen so far
- When processing `x` for the `freq[x]`-th time (0-indexed count before increment), place it in `res[freq[x]]`
- If `freq[x] == len(res)` → we need a new row, append it first

This guarantees each row gets exactly the first, second, third... occurrence of every value — no row ever has a duplicate.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public IList<IList<int>> FindMatrix(int[] nums) {
        var freq = new Dictionary<int, int>();
        var res = new List<IList<int>>();

        foreach (int x in nums) {
            int cnt = freq.GetValueOrDefault(x, 0);
            if (cnt == res.Count) res.Add(new List<int>()); // need a new row
            res[cnt].Add(x);
            freq[x] = cnt + 1;
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Single pass; dictionary ops are O(1) average |
| **Space Complexity** | **O(n)** | `freq` map + output rows together hold all n elements |

**Optimal** — must read every element at least once.

---

## Why This Works — Example Walkthrough

**Example 1**: `nums = [1,3,4,1,2,3,1]`

| x | cnt (before) | row | freq after | res |
|---|-------------|-----|------------|-----|
| 1 | 0 | new → 0 | {1:1} | [[1]] |
| 3 | 0 | 0 | {1:1,3:1} | [[1,3]] |
| 4 | 0 | 0 | {…,4:1} | [[1,3,4]] |
| 1 | 1 | new → 1 | {1:2} | [[1,3,4],[1]] |
| 2 | 0 | 0 | {…,2:1} | [[1,3,4,2],[1]] |
| 3 | 1 | 1 | {3:2} | [[1,3,4,2],[1,3]] |
| 1 | 2 | new → 2 | {1:3} | [[1,3,4,2],[1,3],[1]] |

Returns `[[1,3,4,2],[1,3],[1]]` ✓

---

## Pitfalls & Edge Cases

- **`cnt == res.Count` check** — when `cnt` equals the current number of rows, we're placing the first element into a brand new row; this is the only time we `Add` a new row
- **All same elements** — e.g. `[1,1,1,1]`: each occurrence goes into a new row → 4 rows of `[1]` each ✓
- **All distinct** — `cnt` always 0 → only one row ever created ✓
- **Order preserved per row** — elements are appended in the order they appear in `nums`; problem says any valid answer is accepted

---

## Key Takeaway

Minimum rows to place elements with no duplicates per row = **max frequency** of any element. Assign greedily:

- `freq[x]`-th occurrence (0-indexed) always goes to row index `freq[x]`
- New row added on demand when `cnt == res.Count`
- Single pass, no sorting needed

Pattern: "distribute elements into groups with no duplicates, minimize groups" → frequency map, each occurrence index determines its group.
