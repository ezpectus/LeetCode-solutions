# 2731. Movement of Robots
## O(n log n) — Collision = Pass-Through + Prefix Sum for Pairwise Distances

---

## Problem Statement

- Robots on a number line, each moving L or R at speed 1. Collisions reverse both directions.
- Return sum of all pairwise distances after `d` seconds, mod 10⁹+7.

---

## Core Idea — Two Key Insights

### Insight 1: Collision = Pass-Through (Phantom Robots)

When two robots collide and reverse, the result is **identical** to them passing through each other — the set of final positions is the same. This is a classic trick: instead of simulating collisions, just move every robot independently for `d` seconds:

```
final_pos[i] = nums[i] + d   if s[i] == 'R'
final_pos[i] = nums[i] - d   if s[i] == 'L'
```

Then sort the positions (since robots are indistinguishable after collisions).

### Insight 2: Sum of Pairwise Distances on Sorted Array via Prefix Sum

For a sorted array `pos[0..n-1]`, the sum of all `|pos[j] - pos[i]|` for `i < j`:

```
For each j: it contributes pos[j] to j pairs (all i < j) and subtracts pos[i] from each
= sum over j of: pos[j] * j - (pos[0] + pos[1] + ... + pos[j-1])
= sum over j of: pos[j] * j - prefSum[j]
```

One pass, O(n).

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int sumDistance(vector<int>& nums, string s, int d) {
        const int MOD = 1e9 + 7;
        int n = nums.size();
        vector<long long> pos(n);

        for (int i = 0; i < n; i++)
            pos[i] = (long long)nums[i] + (s[i] == 'R' ? d : -d);

        sort(pos.begin(), pos.end());

        long long res = 0, prefSum = 0;
        for (int i = 0; i < n; i++) {
            res = (res + pos[i] % MOD * i % MOD - prefSum % MOD + MOD) % MOD;
            prefSum += pos[i];
        }

        return (int)res;
    }
};
```

## Why Collision = Pass-Through Works

Two robots `A(pos=1, R)` and `B(pos=3, L)` collide at `pos=2` after 1 second, reverse.
Result: `A` at 2 going L, `B` at 2 going R → after 1 more second: `A` at 1, `B` at 3.

Two robots passing through: `A` continues R → pos=3, `B` continues L → pos=1.
Same multiset `{1, 3}` ✓

The robots are **indistinguishable** — we only care about the positions, not which robot ends up where.

---

## Prefix Sum Formula Derivation

For sorted `pos = [p₀, p₁, ..., p_{n-1}]`:

```
Σ_{i<j} (pⱼ - pᵢ)  [sorted so pⱼ ≥ pᵢ, no abs needed]

= Σⱼ [ pⱼ·j - (p₀+p₁+...+p_{j-1}) ]
= Σⱼ [ pⱼ·j - prefSum[j] ]
```

At index `j`: `pⱼ` is larger than all `j` previous elements → contributes `+pⱼ` to `j` pairs. Sum of smaller elements = `prefSum[j]`.

---

## Example Walkthrough

**Example 1**: `nums=[-2,0,2], s="RLL", d=3`

```
pos = [-2+3, 0-3, 2-3] = [1, -3, -1]
sorted: [-3, -1, 1]

i=0: res += (-3)*0 - 0 = 0;  prefSum = -3
i=1: res += (-1)*1 - (-3) = -1+3 = 2;  prefSum = -4
i=2: res += 1*2 - (-4) = 2+4 = 6;  total = 8
```

Output: `8` ✓

## Pitfalls & Edge Cases

- **`long long` for positions** — `nums[i]` up to 2×10⁹, `d` up to 10⁹ → sum up to 3×10⁹ overflows `int`
- **MOD arithmetic carefully** — `prefSum` can be negative (negative positions); when computing `pos[i]*i - prefSum`, add MOD before taking `%` to avoid negative results
- **`prefSum` itself not modded** — accumulate as raw `long long`, only apply MOD when contributing to `res`; modding `prefSum` early causes incorrect subtraction

---

## Key Takeaway

Two tricks combined:
1. **Collision = pass-through** → ignore collisions, compute final positions independently, sort
2. **Pairwise distance sum on sorted array** = `Σⱼ (pos[j]·j - prefSum[j])` → O(n) instead of O(n²)

Both tricks are reusable patterns that appear frequently in competitive programming.
