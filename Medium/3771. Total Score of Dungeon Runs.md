# 3771. Total Score of Dungeon Runs
## O(n log n) — Contribution Counting + Binary Search on Prefix Sums

---

## Problem Statement

- You have n rooms. Entering room i from a run starting at j costs `damage[i]` health and earns 1 point if `hp − sum(damage[j..i]) ≥ requirement[i]`.
- `score(j)` = total points starting from room j through n.
- Return `score(1) + score(2) + ... + score(n)`.

---

## Core Idea — Count Each Room's Contribution Separately

Instead of simulating each starting room, flip the perspective:

**How many starting rooms j ≤ i earn a point in room i?**

For room i, starting at j earns a point iff:
```
hp − (cumDamage[i] − cumDamage[j−1]) ≥ requirement[i]
  ⟺ cumDamage[j−1] ≥ cumDamage[i] − hp + requirement[i]
  ⟺ cumDamage[j−1] ≥ threshold[i]
```

where `threshold[i] = cumDamage[i] − hp + requirement[i]`.

Since all `damage[i] > 0`, `cumDamage` is **strictly increasing**. So counting how many values in `cumDamage[0..i−1]` are ≥ `threshold[i]` is a single **binary search** per room.

**Total = sum over i of count(k ∈ [0, i−1] : cumDamage[k] ≥ threshold[i])**

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long TotalScore(int hp, int[] damage, int[] requirement) {
        int n = damage.Length;
        long[] cum = new long[n + 1];

        for (int i = 0; i < n; i++) {
            cum[i + 1] = cum[i] + damage[i];
        }

        long total = 0;

        for (int i = 1; i <= n; i++) {
            long threshold = cum[i] - hp + requirement[i - 1];

            // Count k in [0, i-1] where cum[k] >= threshold
            // Binary search: find first index lo where cum[lo] >= threshold
            int lo = 0, hi = i;
            while (lo < hi) {
                int mid = (lo + hi) / 2;
                if (cum[mid] >= threshold) hi = mid;
                else lo = mid + 1;
            }

            total += i - lo; // valid starting rooms: k = lo, lo+1, ..., i-1
        }

        return total;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n log n) | Binary search over [0, i-1] for each of n rooms |
| Space Complexity | O(n) | Prefix sum array |

---

## Why This Works — Derivation

For starting room `j` (1-indexed), entering rooms j through i gives remaining HP:
```
hp − damage[j] − damage[j+1] − ... − damage[i]
= hp − (cumDamage[i] − cumDamage[j−1])
```

Room i earns a point iff this ≥ requirement[i], i.e.:
```
cumDamage[j−1] ≥ cumDamage[i] − hp + requirement[i] = threshold[i]
```

Since k = j−1 ranges over {0, 1, ..., i−1} as j ranges over {1, ..., i}, and `cumDamage` is strictly increasing (all damage ≥ 1), we binary search for the first valid k.

Count of valid j's for room i = i − (first k where cum[k] ≥ threshold[i]).

## Example Walkthrough

**Example 1:** `hp = 11, damage = [3,6,7], requirement = [4,2,5]`

Prefix sums: cum = [0, 3, 9, 16]

| i | cum[i] | req[i] | threshold | Binary search in cum[0..i-1] | count |
|---|--------|--------|-----------|------------------------------|-------|
| 1 | 3      | 4      | 3−11+4=−4  | cum[0]=0 ≥ −4 → lo=0        | 1−0=1 |
| 2 | 9      | 2      | 9−11+2=0   | cum[0]=0 ≥ 0 → lo=0         | 2−0=2 |
| 3 | 16     | 5      | 16−11+5=10 | cum[2]=9 < 10 → lo=3        | 3−3=0 |

Total = 1 + 2 + 0 = **3** ✓

Verification:
- score(1): room 1 → 11−3=8≥4 ✓, room 2 → 8−6=2≥2 ✓, room 3 → 2−7=−5<5 ✗ → 2
- score(2): room 2 → 11−6=5≥2 ✓, room 3 → 5−7=−2<5 ✗ → 1
- score(3): room 3 → 11−7=4<5 ✗ → 0
- Total: 2+1+0 = 3 ✓

**Example 2:** `hp = 2, damage = [10000, 1], requirement = [1, 1]`

cum = [0, 10000, 10001]

| i | threshold        | Search in cum[0..i-1]             | count |
|---|------------------|-----------------------------------|-------|
| 1 | 10000−2+1=9999   | cum[0]=0 < 9999 → lo=1           | 1−1=0 |

## Pitfalls & Edge Cases

- **Overflow** — cum can reach n × max_damage = 10⁵ × 10⁴ = 10⁹. threshold = cum[i] − hp + req ≈ 2×10⁹. Total ≈ n(n+1)/2 ≈ 5×10⁹. All must be `long`. ✓

- **Negative threshold** — When `hp` is large, threshold[i] < 0 ≤ cum[0]. The binary search finds lo=0, giving count = i (all starting rooms valid for that room). ✓

- **No valid rooms** — When threshold[i] > cum[i−1], lo = i, count = 0. ✓

- **Strict increase of cum** — Since all damage[i] ≥ 1, cum is strictly increasing. This guarantees binary search correctness (unique sorted order). ✓

---

## Key Takeaway

The critical reframe: instead of "for each starting room, count points earned," ask **"for each room i, how many starting rooms earn a point here?"**

This decouples the double loop into independent binary searches, reducing O(n²) brute force to O(n log n).

**Pattern recognition:** "Sum over (j, i) pairs satisfying a linear condition involving prefix sums" → fix one dimension, binary search the other. Works when the prefix sum array is monotone (guaranteed here since all damage > 0).
