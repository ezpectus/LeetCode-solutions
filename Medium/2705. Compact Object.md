# 2705. Compact Object
## Recursive DFS — Filter Falsy Values at Every Level

---

## Problem Statement

- Remove all keys/elements with **falsy** values from an object or array, recursively.
- Falsy in JS: `false`, `0`, `""`, `null`, `undefined`, `NaN`.
- Arrays stay arrays, objects stay objects. Empty arrays `[]` are kept (they're truthy).

---

## Core Idea — Recursive Type Dispatch

Three cases based on the type of `obj`:

1. **Array** → `map` each element through `compactObject`, then `filter(Boolean)` to remove falsy results
2. **Object** (non-null) → recurse on each value; include key only if the compacted value is truthy
3. **Primitive** → return as-is; the **caller** decides whether to keep it based on truthiness

The key insight: for objects, an empty array `[]` is truthy so it's kept (example 3: `[0]` → `[]` stays). A primitive `0` or `null` is falsy so the caller drops it.

---

## Full Implementation (JavaScript)

```javascript
var compactObject = function(obj) {
    if (Array.isArray(obj))
        return obj.map(compactObject).filter(Boolean);

    if (typeof obj === 'object' && obj !== null) {
        const res = {};
        for (const [k, v] of Object.entries(obj)) {
            const compact = compactObject(v);
            if (compact) res[k] = compact;
        }
        return res;
    }

    return obj; // primitive — caller checks truthiness
};
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | n = total nodes in the JSON tree |
| **Space** | **O(d)** | d = max nesting depth (recursion stack) |

---

## Why `Array.isArray` Must Come First

`typeof [] === 'object'` — arrays pass the object check. Always check `Array.isArray` before the generic object branch, otherwise arrays get treated as plain objects and lose their array structure.

---

## Example Walkthrough

**Example 3**: `[null, 0, 5, [0], [false, 16]]`

```
map each element through compactObject:
  null   → null        (primitive)
  0      → 0           (primitive)
  5      → 5           (primitive)
  [0]    → [0].map→[0].filter(Boolean) = []
  [false,16] → [false,16].map→[false,16].filter(Boolean) = [16]

Results: [null, 0, 5, [], [16]]
filter(Boolean):
  null → falsy ✗
  0    → falsy ✗
  5    → truthy ✓
  []   → truthy ✓  (empty array is truthy!)
  [16] → truthy ✓

Output: [5, [], [16]] ✓
```

---

## Pitfalls & Edge Cases

- **`null` check in object branch** — `typeof null === 'object'` is a JS quirk; must explicitly check `obj !== null` before treating as object
- **Empty array `[]` is truthy** — `Boolean([]) === true`; keep it even if all its elements were removed
- **Empty object `{}` is truthy** — similarly kept after all falsy keys removed
- **`0` as array index value** — falsy, correctly filtered out by `filter(Boolean)`
- **Nested arrays inside objects** — handled naturally by recursion: value gets compacted, then checked for truthiness

---

## Key Takeaway

"Recursively remove falsy values from JSON structure" → **DFS with type dispatch**:

```
Array   → map(compact) then filter(Boolean)
Object  → recurse values, keep only truthy results
Primitive → return as-is, caller filters
```

The `filter(Boolean)` / `if (compact)` check at the **caller level** (not inside the recursive call) is the clean pattern — each level decides what to keep from its children.
