# 3067. Count Pairs of Connectable Servers in a Weighted Tree Network
## O(n²) — Per-Center DFS Branch Counting

---

## Problem Statement

- Weighted tree of `n` servers. Two servers `a,b` are connectable through `c` if:
  - `a ≠ c`, `b ≠ c`, `a < b`
  - `dist(c,a) % signalSpeed == 0` and `dist(c,b) % signalSpeed == 0`
  - Paths `c→a` and `c→b` don't share edges (i.e., `a` and `b` are in different subtrees of `c`)
- Return `count[i]` = number of such pairs through server `i`.

---

## Core Idea — Fix Center, Count Valid Nodes Per Branch

For each center `c`, its removal splits the tree into branches (one per neighbor). A valid pair `(a, b)` must have `a` in one branch and `b` in a different branch (to avoid shared edges).

**Algorithm for center `c`**:
1. For each neighbor `nb` of `c`: DFS into that subtree, counting nodes at distance divisible by `signalSpeed` from `c`.
2. The number of valid pairs = accumulate: `total_so_far × current_branch_count` (each node in current branch can pair with each node seen in previous branches).
3. Update `total_so_far` by adding `current_branch_count`.

This avoids double-counting: by processing branches sequentially and multiplying by cumulative count.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int signalSpeed;
    List<(int to, int weight)>[] graph;

    public int[] CountPairsOfConnectableServers(int[][] edges, int signalSpeed) {
        this.signalSpeed = signalSpeed;
        int n = edges.Length + 1;
        graph = new List<(int,int)>[n];
        for (int i = 0; i < n; i++) graph[i] = new();
        foreach (var e in edges) {
            graph[e[0]].Add((e[1], e[2]));
            graph[e[1]].Add((e[0], e[2]));
        }

        int[] result = new int[n];
        for (int c = 0; c < n; c++) {
            int totalSoFar = 0;
            foreach (var (nb, w) in graph[c]) {
                int branch = CountValid(nb, c, w);
                result[c] += totalSoFar * branch;
                totalSoFar += branch;
            }
        }
        return result;
    }

    int CountValid(int cur, int parent, long dist) {
        int res = dist % signalSpeed == 0 ? 1 : 0;
        foreach (var (nb, w) in graph[cur])
            if (nb != parent)
                res += CountValid(nb, cur, dist + w);
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | For each of n centers: O(n) DFS total |
| **Space** | **O(n)** | Adjacency list + recursion stack depth O(n) |

With `n ≤ 1000`: at most 10⁶ operations — instant.

---

## Why Sequential Branch Multiplication Avoids Double-Counting

Suppose branches have counts `b₁, b₂, b₃`. Valid pairs:
- Between branch 1 and 2: `b₁ × b₂`
- Between branch 1 and 3: `b₁ × b₃`
- Between branch 2 and 3: `b₂ × b₃`

Total = `b₁(b₂+b₃) + b₂b₃`.

Sequential approach:
- After branch 1: total_so_far=b₁, result+=0×b₁=0
- After branch 2: result+=b₁×b₂, total_so_far=b₁+b₂
- After branch 3: result+=(b₁+b₂)×b₃=b₁b₃+b₂b₃, total_so_far=b₁+b₂+b₃

Total = b₁b₂+b₁b₃+b₂b₃ ✓

## Example Walkthrough

**`edges=[[0,6,3],[6,5,3],[0,3,1],[3,2,7],[3,1,6],[3,4,2]], signalSpeed=3`**

For center `c=0` (neighbors: 6 with w=3, 3 with w=1):

- Branch via 6 (dist=3): 3%3=0 → node 6 valid (+1). Then 5 from 6 (dist=6): 6%3=0 → node 5 valid (+1). Branch=2.
- total_so_far = 0 → result[0] += 0×2 = 0. total_so_far=2.
- Branch via 3 (dist=1): 1%3≠0 → node 3 not valid. Then node 2 (dist=8): 8%3≠0. Node 1 (dist=7): 7%3≠0. Node 4 (dist=3): 3%3=0 ✓. Branch=1.
- result[0] += 2×1 = **2**. total_so_far=3.

`count[0] = 2` ✓ (pairs: (4,5) and (4,6))

## Pitfalls & Edge Cases

- **`dist` type**: use `long` to avoid overflow when summing weights (`n≤1000`, weights up to `10⁶` → max dist ≈ `10⁹`, fits in `int` but safer with `long`)
- **Center node itself**: center `c` is not counted as a valid endpoint (distance 0: 0%signalSpeed=0 might accidentally count `c` — but we start DFS from neighbors with `dist = w ≥ 1`)
- **Leaf nodes**: a center with only one neighbor → no pairs possible → result stays 0 ✓

---

## Key Takeaway

"Pairs of nodes connectable through a center in a weighted tree" → **for each center, DFS per branch counting valid distances, multiply branch counts sequentially**:

```
for c in all nodes:
    total = 0
    for each branch from c:
        branch_count = DFS counting nodes with dist%speed==0
        result[c] += total * branch_count
        total += branch_count
```

O(n²) — acceptable for n≤1000. The sequential multiply avoids double-counting cross-branch pairs.
