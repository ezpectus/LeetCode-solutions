# 2875. Minimum Size Subarray in Infinite Array
## O(n) — Decompose Target + Sliding Window on Doubled Array

---

## Problem Statement

- Infinite array = `nums` repeated forever.
- Find shortest subarray with sum exactly `target`.
- Return length or -1 if impossible.

---

## Core Idea — Decompose Target into Full Cycles + Remainder

Let `total = sum(nums)`. Any subarray of the infinite array with sum `target` consists of:
- Some **full cycles** of `nums` (each contributing `total`)
- A **contiguous piece** within one or two adjacent copies of `nums` (contributing the remainder)

Decompose: `target = k * total + remainder` where `k = target // total`, `remainder = target % total`.

**Full cycles** contribute `k * n` to the length (fixed, unavoidable).

**Remainder**: if `remainder == 0`, answer is `k * n`. Otherwise, find the shortest subarray in `nums + nums` (two copies concatenated) with sum exactly `remainder`. Add `k * n` to that length.

**Why `nums + nums`?** A contiguous remainder piece spans at most two adjacent copies of `nums` — anything longer would include a full cycle and contradict minimality.

**Sliding window** on the doubled array finds the shortest subarray with sum = `remainder` in O(n).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinSizeSubarray(int[] nums, int target) {
        int n = nums.Length;
        long total = 0;
        foreach (int x in nums) total += x;

        long k = target / total;
        long remainder = target % total;
        int baseLen = (int)(k * n);

        if (remainder == 0) return baseLen;

        int res = int.MaxValue, currSum = 0, left = 0;
        for (int right = 0; right < 2 * n; right++) {
            currSum += nums[right % n];
            while (currSum > remainder) {
                currSum -= nums[left % n];
                left++;
            }
            if (currSum == remainder)
                res = Math.Min(res, right - left + 1);
        }

        return res == int.MaxValue ? -1 : baseLen + res;
    }
}
```

| **Metric** | **Value** | **Notes** |

| **Time** | **O(n)** | Sliding window over 2n elements |
| **Space** | **O(1)** | No extra arrays (use `% n` indexing) |

## Example Walkthrough

**`nums=[1,2,3], target=5`**

`total=6, k=0, remainder=5, baseLen=0`

Sliding window on `[1,2,3,1,2,3]` for sum=5:
- right=0: sum=1
- right=1: sum=3
- right=2: sum=6 > 5 → shrink: sum=5, left=1 → window [1,2]=2 ✓
- Continue: find no shorter window

`res = 0 + 2 = 2` ✓

**`nums=[2,4,6,8], target=3`**

`total=20, k=0, remainder=3`

Sliding window: all elements are ≥ 2, can't reach sum=3 with first element=2 and others even larger → `res = ∞` → **-1** ✓

## Pitfalls & Edge Cases

- **`remainder == 0`**: exactly `k` full cycles, no remainder subarray needed → return `k * n` directly
- **`target < total`**: `k=0`, search for `remainder = target` in doubled array
- **`target > total` but `remainder` unreachable**: e.g., all elements even but remainder is odd → return -1
- **`long` for `total`**: `nums[i]` up to 10⁵, n up to 10⁵ → total up to 10¹⁰ → use `long` for `total`; but `target ≤ 10⁹` so `k` and `remainder` fit in `int`

---

## Key Takeaway

"Shortest subarray in infinite array with given sum" → **decompose into full cycles + remainder**:

```
k = target / total           → contributes k*n to length
remainder = target % total   → find shortest subarray with this sum in nums+nums
answer = k*n + min_subarray_length(remainder)
```

The two-copy trick (`nums+nums`) handles wraparound without explicit modular arithmetic in the window. Sliding window finds the minimum in O(n).
