# 3302. Find the Lexicographically Smallest Valid Sequence
## O(n + m) — Suffix Match Array + Greedy Forward Scan

---

## Problem Statement

- Find the lexicographically smallest sequence of `m` indices from `word1` such that the selected characters are "almost equal" to `word2` (differ in at most 1 position).

---

## Core Idea — Precompute Suffix Matchability + Greedy Wildcard

**Precompute `suf_start[i]`**: the leftmost index `j` in `word1` from which `word2[i:]` can be matched as a subsequence. Computed right-to-left in O(n+m).

**Greedy forward scan**: for each position `i` in `word1`, try to match `word2[k]` exactly. If it doesn't match exactly, try using `i` as the "wildcard" position (the one allowed change). We can use `i` as wildcard for `word2[k]` iff:
- `word2[k+1:]` (the remaining suffix) can still be matched from `word1[i+1:]`.
- Condition: `k+1 == m` (nothing left) OR `suf_start[k+1] != n AND suf_start[k+1] > i`.

After placing the wildcard, greedily match remaining `word2[k+1:]` exactly.

**Why lex smallest?** We match from left to right taking the earliest valid positions. Using a wildcard as early as possible (at the leftmost mismatch) gives the smallest indices.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] ValidSequence(string word1, string word2) {
        int n = word1.Length, m = word2.Length;

        // suf_start[i] = leftmost word1 index where word2[i:] can be matched
        int[] sufStart = new int[m+1];
        Array.Fill(sufStart, n);
        int j = n-1;
        for (int i = m-1; i >= 0; i--) {
            while (j >= 0 && word1[j] != word2[i]) j--;
            if (j >= 0) { sufStart[i] = j; j--; }
        }

        int[] res = new int[m];
        int k = 0;
        for (int i = 0; i < n && k < m; i++) {
            if (word1[i] == word2[k]) {
                res[k++] = i;
            } else {
                bool canWild = (k+1 == m) || (sufStart[k+1] != n && sufStart[k+1] > i);
                if (canWild) {
                    res[k++] = i;
                    int j2 = i+1;
                    while (k < m) {
                        while (j2 < n && word1[j2] != word2[k]) j2++;
                        if (j2 == n) return Array.Empty<int>();
                        res[k++] = j2++;
                    }
                    return res;
                }
            }
        }
        return k == m ? res : Array.Empty<int>();
    }
}
```

## Example Walkthrough

**`word1="bacdc", word2="abc"`**

`suf_start`: 
- i=2 ('c'): scan right-to-left, word1[4]='c' → suf_start[2]=4.
- i=1 ('b'): word1[3]='c',word1[2]='d',word1[1]='a',word1[0]='b' → suf_start[1]=0. Wait: j starts at 3. j=3: 'c'≠'b', j=2:'d'≠'b', j=1:'a'≠'b', j=0:'b'='b' → suf_start[1]=0.
- i=0 ('a'): from j=-1: can't find → suf_start[0]=n=5.

Greedy scan:
- i=0,'b'≠'a'. canWild: suf_start[1]=0, but 0 > 0? No. Skip.
- i=1,'a'='a'. res[0]=1, k=1.
- i=2,'c'≠'b'. canWild: suf_start[2]=4 > 2 ✓. Use wildcard. res[1]=2, k=2.
  Match 'c' from i=3: word1[3]='d'≠'c', word1[4]='c' → res[2]=4.

Result: **[1,2,4]** ✓

## Pitfalls & Edge Cases

- **`suf_start[i] != n`**: explicitly check the suffix is actually matchable (n means "impossible").
- **`suf_start[k+1] > i`**: remaining suffix must start AFTER the wildcard position.
- **No wildcard used**: if all m chars match exactly, return as-is (k==m after the loop).
- **Wildcard only once**: once the wildcard branch is taken, we return immediately after exact-matching the rest.

---

## Key Takeaway

"Lex smallest subsequence almost equal to word2" → **precompute suffix matchability + greedy: exact match when possible, use wildcard at first opportunity where suffix can still be completed**:

```
suf_start[i] = leftmost j where word2[i:] matches word1[j:]
for each i: exact match OR wildcard if suf_start[k+1] > i
```

O(n+m). The suffix array makes the wildcard feasibility check O(1).
