# 3760. Maximum Substrings With Distinct Start
## O(n) — Greedy First Occurrence Counting

---

## Problem Statement

- You are given a string `s` (1 ≤ n ≤ 10⁵) of lowercase English letters.
- Split `s` into the maximum number of substrings such that no two substrings start with the same character.
- Return the maximum number of such substrings.

---

## Core Idea — Count Distinct First Characters

Key insight: Each substring must start with a **unique** character. We want to maximize the number of substrings, so greedily we should start a new substring at every position where the character hasn't been used as a starting character yet.

The answer is simply the number of **distinct characters** in `s`.

Why? For any distinct character `c` in `s`:
- We can always assign the first occurrence of `c` as the start of its own substring
- Everything between two consecutive "new character" positions just gets absorbed into the previous substring
- This greedy split is always valid and always achieves exactly `distinct_count` substrings

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxDistinct(string s) {
        bool[] seen = new bool[26];
        int res = 0;

        foreach (char c in s) {
            int idx = c - 'a';
            if (!seen[idx]) {
                seen[idx] = true;
                res++;
            }
        }

        return res;
    }
}
```

---

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n) | Single pass through the string |
| Space Complexity | O(1) | Fixed-size array of 26 booleans |

Optimal — one pass, constant space.

---

## Why This Works — Example Walkthrough

**Example 1:** `s = "abab"`

- 'a' → new → seen = {a}, res = 1
- 'b' → new → seen = {a,b}, res = 2
- 'a' → already seen, skip
- 'b' → already seen, skip

Split: `"a" | "bab"` — starts with 'a', 'b' ✓

**Output:** 2 ✓

---

**Example 2:** `s = "abcd"`

- 'a' → new → res = 1
- 'b' → new → res = 2
- 'c' → new → res = 3
- 'd' → new → res = 4

Split: `"a" | "b" | "c" | "d"` ✓

**Output:** 4 ✓

---

**Example 3:** `s = "aaaa"`

- 'a' → new → res = 1
- 'a' → seen, skip
- 'a' → seen, skip
- 'a' → seen, skip

Only one distinct starting character possible.

**Output:** 1 ✓

---

## Pitfalls & Edge Cases

- **Single character string** — Only one distinct character → res = 1. Handled trivially.

- **All same characters** — Like "aaaa" → res = 1. The entire string is one substring.

- **All distinct characters** — Like "abcd" → res = n. Every character starts its own single-character substring.

- **Order matters for splits, not for count** — We only care about how many distinct starting characters exist, not where exactly we split. The greedy "absorb everything until next new character" always works.

- **Max possible answer** — At most 26 (number of lowercase letters), regardless of string length.

---

## Key Takeaway

This is a **greedy counting** problem in disguise:
- "Maximum substrings with distinct starts" = "count distinct characters in s"
- No DP, no complex splitting needed — just count unique characters

**Pattern recognition:** When maximizing the number of non-overlapping segments with a uniqueness constraint on the first element, greedily assign each new unique element as a new segment start. The answer collapses to simply counting distinct values.

Simpler than it looks — the key is realizing that the splitting boundaries don't matter, only the count of distinct starting characters does.
