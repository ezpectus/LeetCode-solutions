# 2997. Minimum Number of Operations to Make Array XOR Equal to K
## O(n) — XOR All Elements, Count Differing Bits

---

## Problem Statement

- Flip individual bits in array elements (one flip = one operation).
- Make XOR of all elements equal to `k`.
- Return minimum operations.

---

## Core Idea — One Flip per Differing Bit

**Step 1**: Compute `xor = nums[0] ^ nums[1] ^ ... ^ nums[n-1]` (current XOR of array).

**Step 2**: Compare `xor` with `k` bit by bit. Each bit position where they differ requires exactly one flip operation to fix.

**Answer**: `popcount(xor ^ k)` — the number of set bits in `xor XOR k`.

**Why?** Flipping bit `b` in any element toggles bit `b` in the total XOR. Each differing bit between current XOR and target `k` needs exactly one toggle → one operation. Bits that already match need no changes.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperations(int[] nums, int k) {
        int xor = 0;
        foreach (int x in nums) xor ^= x;
        return System.Numerics.BitOperations.PopCount((uint)(xor ^ k));
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

---

## Example Walkthrough

**`nums=[2,1,3,4], k=1`**

`xor = 2^1^3^4 = 4` (binary: `100`)
`k = 1` (binary: `001`)
`xor ^ k = 4^1 = 5` (binary: `101`) → 2 set bits → **2 operations** ✓

---

## Key Takeaway

"Min bit flips to make total XOR equal k" → **`popcount(current_XOR ^ k)`**. Each bit in `current_XOR ^ k` represents one mismatch that costs exactly one flip to fix.
