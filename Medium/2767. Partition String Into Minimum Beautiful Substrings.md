# 2767. Partition String Into Minimum Beautiful Substrings
## O(n²) — Precompute Powers of 5 + DP

---

## Problem Statement

- **Beautiful substring**: no leading zeros, binary representation of a power of 5 (`5⁰=1, 5¹=5, 5²=25, ...`).
- Partition `s` into minimum number of beautiful substrings, or return -1 if impossible.
- `s.length ≤ 15`.

---

## Core Idea — Precompute + Prefix DP

**Step 1**: Precompute all powers of 5 whose binary representation fits in 15 bits:
`{1, 5, 25, 125, 625, 3125, 15625}` → binary: `{"1", "101", "11001", ...}`. Store in a HashSet.

**Step 2**: Standard partition DP.

`dp[i]` = minimum parts to partition `s[0..i-1]`.

```
dp[0] = 0  (empty string)
dp[i] = min over j < i where:
  - s[j] == '1'  (no leading zero)
  - s[j..i-1] ∈ powers
  of dp[j] + 1
```

**Answer**: `dp[n]`, or -1 if it stays ∞.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumBeautifulSubstrings(string s) {
        int n = s.Length;
        var powers = new HashSet<string>();
        for (long p = 1; p <= (1 << 15); p *= 5)
            powers.Add(Convert.ToString(p, 2));

        int[] dp = new int[n + 1];
        Array.Fill(dp, int.MaxValue / 2);
        dp[0] = 0;

        for (int i = 1; i <= n; i++) {
            if (s[i-1] == '0') continue; // end can't be leading-zero endpoint
            for (int j = 0; j < i; j++) {
                if (s[j] == '0') continue; // leading zero
                if (powers.Contains(s.Substring(j, i-j)) && dp[j] < int.MaxValue/2)
                    dp[i] = Math.Min(dp[i], dp[j] + 1);
            }
        }

        return dp[n] >= int.MaxValue/2 ? -1 : dp[n];
    }
}
```

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | n≤15 → at most 225 iterations |
| **Space** | **O(n)** | dp array + small powers set |

---

## Powers of 5 in Binary (up to 15 bits)

| Power | Value | Binary |
|-------|-------|--------|
| 5⁰ | 1 | `1` |
| 5¹ | 5 | `101` |
| 5² | 25 | `11001` |
| 5³ | 125 | `1111101` |
| 5⁴ | 625 | `1001110001` |
| 5⁵ | 3125 | `110000110101` |
| 5⁶ | 15625 | `11110100001001` |

Only 7 valid values → very small HashSet.

---

## Pitfalls & Edge Cases

- **Leading zero check**: skip any `j` where `s[j]=='0'`; skip `i` endpoint where `s[i-1]=='0'` (optimization — any substring ending there and starting with 0 is invalid)
- **`s = "0"`**: `dp[0]=0`, but `s[0]='0'` → loop skips → `dp[1]` stays ∞ → return -1 ✓
- **`int.MaxValue/2`** not `int.MaxValue` — avoid overflow when doing `dp[j]+1`

---

## Key Takeaway

"Minimum partition into valid substrings" → **precompute valid strings + prefix DP**:

```
dp[i] = min(dp[j]+1)  for j where s[j..i-1] is valid
```

With only 7 valid binary representations of powers of 5 up to length 15, the HashSet lookup is essentially O(1). Standard word-break DP pattern.
