# 3746. Minimum String Length After Balanced Removals
## O(n) — Count Invariant: |countA − countB|

---

## Problem Statement

- You are given string `s` of 'a' and 'b' (1 ≤ n ≤ 10⁵).
- Repeatedly remove any contiguous substring where `#a == #b`.
- Return the minimum possible length after any number of removals.

---

## Core Idea — The Difference Is Invariant

Key insight: every removal removes exactly equal numbers of 'a' and 'b'. Therefore **the difference `countA − countB` never changes** across any sequence of operations.

This means:
- If we removed `k` pairs total, remaining = `(countA − k)` a's + `(countB − k)` b's = `(countA + countB) − 2k`
- Minimum length = `|countA − countB|` achieved at `k = min(countA, countB)`

**Why `min(countA, countB)` is always achievable:** As long as both characters exist, we can always find a contiguous balanced substring to remove. For example, scan from any 'a' rightward until we find a 'b' (or vice versa) — that substring either is already balanced or contains a balanced sub-portion. This greedy argument guarantees we can always make progress until one character is fully exhausted.

The answer is simply `Math.Abs(countA - countB)`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinLengthAfterRemovals(string s) {
        int countA = 0, countB = 0;

        foreach (char c in s) {
            if (c == 'a') countA++;
            else countB++;
        }

        return Math.Abs(countA - countB);
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n) | Single pass to count characters |
| Space Complexity | O(1) | Two integer counters |

## Why This Works — Invariant Proof

**Lower bound:** any removal decreases `countA` and `countB` by the same amount `d` (since the removed substring has `#a == #b == d`). So `countA − countB` is preserved. The final string has at least `|countA − countB|` characters.

**Upper bound (achievability):** We prove we can always pair up `min(countA, countB)` of each character. While both 'a' and 'b' exist in the string, we can find two positions i < j with `s[i] ≠ s[j]` — the substring `s[i..j]` has at least one 'a' and one 'b'. The prefix sum of `(+1 for 'a', -1 for 'b')` starts at 0 and returns to 0 somewhere in `[i, j]` (by the intermediate value theorem on integers), giving us a balanced substring to remove. Repeat until one character is exhausted.

## Example Walkthrough

**Example 1:** `s = "aabbab"` → countA=3, countB=3 → |3−3| = **0** ✓

Remove "aabbab" directly (3a, 3b = balanced). 

**Example 2:** `s = "aaaa"` → countA=4, countB=0 → |4−0| = **4** ✓

No 'b' exists → no balanced substring possible → entire string remains.

**Example 3:** `s = "aaabb"` → countA=3, countB=2 → |3−2| = **1** ✓

Remove "ab" → "aab" → remove "ab" → "a". One 'a' leftover = excess. 

---

## Pitfalls & Edge Cases

- **Looks like a stack/simulation problem** — The repeated removal framing tempts complex solutions. Resist: the invariant is the entire insight.

- **Order of removals doesn't matter** — No matter which balanced substrings we remove and in what order, the achievable minimum is always `|countA - countB|`.

- **All same character** — Like "aaaa": countB=0, answer=4. No removals possible. ✓

- **Already balanced** — If countA == countB, always returns 0 regardless of arrangement (even "bababa..." can be fully removed). ✓

---

## Key Takeaway

Spot the invariant: each operation removes equal a's and b's → `countA − countB` is constant throughout. Minimum length = `|countA − countB|`.

**Pattern recognition:** When an operation "removes equal amounts of two things," the difference between those two things is invariant. Minimum remaining = that invariant's absolute value.

Classic example of a problem that looks like it needs simulation but collapses to a one-liner once you identify the right mathematical property.
