# 3007. Maximum Number That Sum of the Prices Is Less Than or Equal to K
## O(log²(k/x)) — Binary Search + Bit Count Formula

---

## Problem Statement

- Price of `num` = count of set bits at positions `x, 2x, 3x, ...` (1-indexed from LSB).
- Accumulated price of `num` = sum of prices of `1, 2, ..., num`.
- Return largest `num` with accumulated price ≤ `k`.

---

## Core Idea — Binary Search on Answer + O(log n) Accumulated Price

**Monotone property**: accumulated price is non-decreasing → binary search on `num`.

**Computing accumulated price of `[1..num]`**:
For each relevant bit position `p = x-1, 2x-1, 3x-1, ...` (0-indexed), count how many numbers in `[1..num]` have that bit set using the standard formula:

```
count_with_bit_b(n) = (n // 2^(b+1)) * 2^b + max(0, n % 2^(b+1) - 2^b + 1)
```

This formula counts numbers in `[0..n]` with bit `b` set in O(1). Sum across all relevant bit positions.

**Binary search bounds**: `lo=1`, `hi=10^18` (k ≤ 10^15, x ≥ 1 → answer ≤ ~10^18).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long FindMaximumNumber(long k, int x) {
        long lo = 1, hi = (long)2e15;
        while (lo < hi) {
            long mid = (lo + hi + 1) / 2;
            if (AccumulatedPrice(mid, x) <= k) lo = mid;
            else hi = mid - 1;
        }
        return lo;
    }

    long AccumulatedPrice(long num, int x) {
        long total = 0;
        int bit = x - 1; // 0-indexed
        while (bit < 63) {
            long cycle = 1L << (bit + 1);
            long full  = num / cycle;
            long rem   = num % cycle;
            total += full * (1L << bit) + Math.Max(0, rem - (1L << bit) + 1);
            bit += x;
        }
        return total;
    }
}
```

## Bit Count Formula Explained

How many numbers in `[0..n]` have bit `b` (0-indexed) set?

Bits cycle with period `2^(b+1)`. In each full cycle:
- First `2^b` numbers have bit `b` = 0
- Next `2^b` numbers have bit `b` = 1

```
full_cycles = n // 2^(b+1)       → each contributes 2^b set bits
remainder   = n % 2^(b+1)        → contributes max(0, rem - 2^b + 1) set bits
```

Total = `full_cycles * 2^b + max(0, remainder - 2^b + 1)`

## Pitfalls & Edge Cases

- **`hi = 2e15`**: safe upper bound since `k ≤ 10^15` and with `x=1` (every bit counts), answer ≤ ~2×10^15. Using `2e18` causes serialization overflow in LeetCode's judge
- **`bit < 63`**: avoid 64-bit overflow in `1L << bit`
- **`1L << bit`**: use `long` literal to prevent 32-bit overflow in shifts ≥ 32
- **Binary search upper bound**: use `(lo + hi + 1) / 2` (round up) to avoid infinite loop when `lo = hi - 1`

---

## Key Takeaway

"Largest n with accumulated bit-position price ≤ k" → **binary search on n + O(log n) price computation**:

```
accumulated_price(n) = sum over bit positions p in {x-1, 2x-1, ...} of count_with_bit(n, p)
count_with_bit(n, b) = (n // 2^(b+1)) * 2^b + max(0, n % 2^(b+1) - 2^b + 1)
```

The cyclic nature of bit patterns enables O(1) counting per bit position.
