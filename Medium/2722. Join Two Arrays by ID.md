# 2722. Join Two Arrays by ID
## O(n log n) — Map Merge + Sort

---

## Problem Statement

- Merge two arrays of objects by `id` field.
- If same `id` in both: merge properties, `arr2` values override `arr1`.
- Return sorted by `id` ascending.

---

## Core Idea — Map + Spread Operator

Build a `Map<id, object>`. Insert `arr1` first, then for each `arr2` object merge with spread:

```javascript
map.set(obj.id, { ...map.get(obj.id), ...obj })
```

`{ ...existing, ...obj }` merges both objects — keys from `obj` (arr2) override keys from `existing` (arr1). If `map.get(obj.id)` is `undefined` (id only in arr2), spread of `undefined` is a no-op in JS → just `{ ...obj }` ✓.

Finally spread the Map values into an array and sort by `id`.

---

## Full Implementation (JavaScript)

```javascript
var join = function(arr1, arr2) {
    const map = new Map();
    for (const obj of arr1)
        map.set(obj.id, obj);
    for (const obj of arr2)
        map.set(obj.id, { ...map.get(obj.id), ...obj });
    return [...map.values()].sort((a, b) => a.id - b.id);
};
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O((n+m) log(n+m))** | Linear merge + sort of unique ids |
| **Space** | **O(n+m)** | Map stores all unique objects |

---

## Why `{ ...map.get(obj.id), ...obj }` Works

Spread order matters — later spreads override earlier ones:
```javascript
{ ...{x:1, y:2}, ...{x:10} }  →  { x:10, y:2 }
```
arr2's `obj` is spread last → its values win. Note: this is **shallow merge** — nested objects are replaced entirely, not deep-merged (example 3: `b:{c:84}` replaces `b:{b:94}`).

**`undefined` spread trick**: if `id` only exists in arr2, `map.get(obj.id)` returns `undefined`. In JS, `{ ...undefined }` = `{}` — safe no-op, so `{ ...undefined, ...obj }` = `{ ...obj }` ✓

---

## Pitfalls & Edge Cases

- **Shallow merge only** — nested objects from arr2 fully replace arr1's nested objects; no deep merge
- **Sort is necessary** — Map preserves insertion order, but arr1 and arr2 may not be sorted, and merged result needs ascending id order
- **`id` field itself** — always present in both objects; spread keeps the last one (`arr2`'s id = arr1's id anyway since they're equal)

---

## Key Takeaway

"Merge arrays by key, second overrides first" → **Map + spread**:

```javascript
map.set(id, { ...map.get(id), ...newObj })  // arr2 wins on conflicts
```

`{ ...undefined }` = `{}` handles the "id only in arr2" case cleanly without any null checks.
