# 🏔️ 3891. Minimum Increase to Maximize Special Indices — Architectural Solution

## Problem Statement

You are given an integer array `nums` of length `n`. An index `i` (0 < i < n - 1) is called **special** (a peak) if:
`nums[i] > nums[i - 1]` and `nums[i] > nums[i + 1]`

You can perform operations to increase any `nums[i]` by 1. Your goal is to:
1. Maximize the number of special indices.
2. Minimize the total number of operations required to achieve that maximum.

Return the minimum total number of operations required.

### Constraints
- 3 <= n <= 10^5
- 1 <= nums[i] <= 10^9

---

## 💡 Core Idea

To maximize the number of special indices (peaks), we need to place them as densely as possible. Since two peaks cannot be adjacent or separated by zero elements, the optimal placement depends entirely on the parity of the array length `n`:

1. **Odd Length (`n % 2 != 0`)**: 
   The maximum number of peaks achievable is exactly `(n - 1) / 2`. There is only one way to achieve this maximum: every odd index must become a peak (`1, 3, 5, ..., n - 2`). The pattern is completely fixed, so we just iterate and sum up the costs.

2. **Even Length (`n % 2 == 0`)**: 
   The maximum number of peaks achievable is `(n - 2) / 2`. Because the array has an extra element, we have the flexibility to "skip" exactly one position. We can either:
   - Start at index 1 and choose to jump by 3 instead of 2 exactly once.
   - Start at index 2 instead of 1 (which counts as using the skip immediately).
   
   We use **Dynamic Programming (DP)** with a state tracking whether the single allowable skip has already been used (`skip` boolean flag).

---

## 💻 Implementation (C++)

```cpp
class Solution {
public:
    long long r(vector<int> &nums, int pos, bool skip, vector<vector<long long>> &dp) {
        int n = nums.size();
        if (pos >= n - 1) return 0;
        if (dp[skip][pos] != -1) return dp[skip][pos];

        // Calculate operations needed to make nums[pos] greater than both neighbors
        long long ans = max(0LL, max(nums[pos - 1], nums[pos + 1]) + 1 - nums[pos]);
        
        if (skip) {
            // Skip already used, must proceed with the regular step of 2
            return dp[skip][pos] = ans + r(nums, pos + 2, skip, dp);
        }

        // Option 1: Use the skip right now (jump by 3 to shift parity)
        dp[true][pos]  = ans + r(nums, pos + 3, true, dp);
        // Option 2: Do not use the skip yet (jump by 2)
        dp[false][pos] = ans + r(nums, pos + 2, false, dp);

        return min(dp[0][pos], dp[1][pos]);
    }
    
    long long minIncrease(vector<int>& nums) {
        int n = nums.size();
        
        // Odd length: fixed pattern at odd indices
        if (n % 2) {
            long long ans = 0;
            for (int i = 1; i < n - 1; i += 2) {
                ans += max(0LL, max(nums[i-1], nums[i+1]) + 1 - nums[i]);
            }
            return ans;                    
        }
        
        // Even length: DP with exactly one allowed skip
        vector<vector<long long>> dp(2, vector<long long>(n, -1));
        
        // We can either start at index 1 (no skip used yet) or start at index 2 (skip used)
        return min(r(nums, 1, false, dp), r(nums, 2, true, dp));
    }
};
```

## 📐 Complexity Analysis

| Metric | Value | Justification |
| :--- | :--- | :--- |
| **Time Complexity** | O(n) | **Odd length**: Single loop traversing the array steps by 2, taking O(n) time.<br>**Even length**: The DP table has a size of 2 * n. Each state is computed exactly once in O(1) time, resulting in O(n) total time. |
| **Space Complexity** | O(n) | **DP Table**: Requires a 2 * n grid of long long values.<br>**Recursion Stack**: The maximum depth of the recursion tree is n / 2, resulting in O(n) auxiliary space. |

## ⚠️ Pitfalls & Edge Cases

- **Integer Overflow**: The elements can be up to 10^9, and adding up operations over an array of size 10^5 can easily overflow a standard 32-bit integer. The DP table, recursion return types, and accumulators must use `long long`.
- **Already Valid Peaks**: If `nums[i]` is already strictly greater than both its neighbors, `max(0, ...)` correctly registers a cost of 0 operations instead of subtracting from the answer.
- **Out of Bounds**: The base case `if (pos >= n - 1)` catches the pointers safely before they look up non-existent neighbors at `pos + 1`.

## 🔄 Alternative Approaches

- **Iterative DP (Tabulation)**: Instead of top-down recursion with memoization, the same states can be filled bottom-up from `n-1` down to `0`. It would keep the same O(n) time and space complexity but eliminate recursion stack overhead.
- **Prefix and Suffix Arrays**: You could precalculate the cost of greedily choosing peaks from the left up to index `i` (using the index 1 pattern) and from the right down to `i` (using the index 2 pattern). The answer for even lengths would be the minimum combination of `prefix[i] + suffix[i+3]` across all possible split points.
