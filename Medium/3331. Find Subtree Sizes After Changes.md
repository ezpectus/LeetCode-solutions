# 3331. Find Subtree Sizes After Changes
## O(n) — DFS with Ancestor Character Tracking

---

## Problem Statement

- Each node `x` gets reparented to its **closest ancestor** with the same character. Compute all subtree sizes in the resulting tree.

---

## Core Idea — DFS with `last_ancestor[char]` Map

During DFS, maintain `last_ancestor[c]` = the most recently seen ancestor with character `c` (i.e., the deepest ancestor on the current root-to-node path with character `c`).

**When processing child `child` of node**:
- Before recursing into `child`, `last_ancestor[s[child]]` is the closest ancestor of `child` with the same character → that's `child`'s new parent.
- After DFS of `child` completes: add `ans[child]` to `ans[new_parent]` instead of `ans[old_parent]`.

**Restore `last_ancestor[c]`** when backtracking from `node`.

`ans` starts as all 1s (each node counts itself). Children add their subtree sizes to their (new) parent.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int[] ans, parent;
    string s;
    List<int>[] children;
    Dictionary<char,int> lastAnc = new();

    public int[] FindSubtreeSizes(int[] parent, string s) {
        int n = s.Length;
        this.parent = parent; this.s = s;
        ans = new int[n]; Array.Fill(ans, 1);
        children = new List<int>[n];
        for (int i = 0; i < n; i++) children[i] = new();
        for (int i = 1; i < n; i++) children[parent[i]].Add(i);
        Dfs(0);
        return ans;
    }

    void Dfs(int node) {
        char c = s[node];
        int old = lastAnc.ContainsKey(c) ? lastAnc[c] : -1;
        lastAnc[c] = node;

        foreach (int child in children[node]) {
            char cc = s[child];
            int newPar = lastAnc.ContainsKey(cc) ? lastAnc[cc] : parent[child];
            Dfs(child);
            ans[newPar] += ans[child];
        }

        if (old == -1) lastAnc.Remove(c);
        else lastAnc[c] = old;
    }
}
```

## Example Walkthrough

**`parent=[-1,0,0,1,1,1], s="abaabc"`**

DFS from 0 ('a'): lastAnc={a:0}.

Child 1 ('b'): newPar=parent[1]=0 (no 'b' ancestor). lastAnc={a:0,b:1}. 
  Child 3 ('a'): newPar=lastAnc['a']=0 (not 1!). Dfs(3). ans[0]+=ans[3]=1. → ans[0]=2.
  Child 4 ('b'): newPar=lastAnc['b']=1. Dfs(4). ans[1]+=1. → ans[1]=2.
  Child 5 ('c'): newPar=parent[5]=1. Dfs(5). ans[1]+=1. → ans[1]=3.
Back from 1: ans[0]+=ans[1]=3. → ans[0]=5.

Child 2 ('a'): newPar=lastAnc['a']=0. Dfs(2). ans[0]+=1. → ans[0]=6.

Final: ans=[6,3,1,1,1,1] ✓

---

## Pitfalls & Edge Cases

- **When `last_ancestor[s[child]]` doesn't exist**: use `parent[child]` (no reparenting needed).
- **Restore map on backtrack**: use old value (-1 means remove entry).
- **Node 0 (root)**: `parent[0] = -1`, but root is never reparented.

---

## Key Takeaway

"Reparent to closest ancestor with same char, compute subtree sizes" → **DFS tracking `last_ancestor[char]`. When processing child, its new parent = `last_ancestor[s[child]]`. Add child's subtree to new parent**:

```
Dfs(node):
    save and set last_ancestor[s[node]] = node
    for child: newPar = last_ancestor[s[child]] ?? parent[child]
               Dfs(child); ans[newPar] += ans[child]
    restore last_ancestor[s[node]]
```

O(n) — each node processed once, map operations O(26) = O(1).
