# 2662. Minimum Cost of a Path With Special Roads
## O(E log V) — Dijkstra on Implicit Graph

---

## Problem Statement

- Move in 2D space. Cost between any two points = Manhattan distance.
- Special roads: directed edge `(x1,y1) → (x2,y2)` with cost `costi` (may be cheaper than Manhattan).
- Return minimum cost from `start` to `target`.

---

## Core Idea — Dijkstra with Implicit Edges

**Key insight**: the only points worth stopping at are `start`, `target`, and the **endpoints of special roads** (`x2,y2`). Between any two "interesting" points, the cost is Manhattan distance (we can always walk directly). Special roads add cheaper shortcuts.

From any current position `(x, y)` with cost `d`:
- **Direct to target**: `d + manhattan(x,y, tx,ty)`
- **Use special road `i`**: walk to its start `(x1,y1)` + road cost = `d + manhattan(x,y,x1,y1) + costi`, arriving at `(x2,y2)`

Run Dijkstra — no need to enumerate all intermediate points, just expand greedily from the cheapest known node.

**Why only `x2,y2` endpoints?** The start of a special road `(x1,y1)` is never worth "storing" as a node — we always pay Manhattan to reach it on the fly. The end `(x2,y2)` is where we "land" and might take another road from, so it becomes a real graph node.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumCost(int[] start, int[] target, int[][] specialRoads) {
        int tx = target[0], ty = target[1];
        var dist = new Dictionary<(int,int), int>();
        var pq = new PriorityQueue<(int x, int y), int>();

        int sx = start[0], sy = start[1];
        dist[(sx, sy)] = 0;
        pq.Enqueue((sx, sy), 0);

        int Manhattan(int x1, int y1, int x2, int y2)
            => Math.Abs(x2-x1) + Math.Abs(y2-y1);

        while (pq.Count > 0) {
            var (x, y) = pq.Dequeue();
            int d = dist[(x, y)];

            // Skip stale entries
            if (pq.TryPeek(out _, out int top) && d > top) { /* already outdated */ }

            // Option 1: go directly to target
            int toTarget = d + Manhattan(x, y, tx, ty);
            if (!dist.TryGetValue((tx, ty), out int curT) || toTarget < curT) {
                dist[(tx, ty)] = toTarget;
                pq.Enqueue((tx, ty), toTarget);
            }

            // Option 2: walk to start of each special road, then take it
            foreach (var r in specialRoads) {
                int nd = d + Manhattan(x, y, r[0], r[1]) + r[4];
                var dest = (r[2], r[3]);
                if (!dist.TryGetValue(dest, out int curD) || nd < curD) {
                    dist[dest] = nd;
                    pq.Enqueue(dest, nd);
                }
            }
        }

        return dist[(tx, ty)];
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(E log V)** | V = specialRoads.Length + 2; E = V² implicit edges |
| **Space Complexity** | **O(V)** | dist map + priority queue |

With `V ≤ 202` nodes and `E ≤ 202²` edges — trivially fast.

## Example Walkthrough

**Example 1**: `start=[1,1], target=[4,5], roads=[[1,2,3,3,2],[3,4,4,5,1]]`

```
Start: dist[(1,1)]=0

Expand (1,1):
  → target(4,5): 0 + |4-1|+|5-1| = 7
  → road0: walk (1,1)→(1,2)=1, + cost=2, land (3,3): dist[(3,3)]=3
  → road1: walk (1,1)→(3,4)=5, + cost=1, land (4,5): dist[(4,5)]=6

Expand (3,3) [cost=3]:
  → target(4,5): 3 + |4-3|+|5-3| = 6
  → road1: walk (3,3)→(3,4)=1, + cost=1, land (4,5): 3+1+1=5 ✓

dist[(4,5)] = 5
```

Output: `5` ✓

---

## Pitfalls & Edge Cases

- **Direct path may be optimal** — always try `manhattan(start, target)` as a candidate (example 2: answer = 7, no roads used)
- **Special road start `(x1,y1)` is not a stored node** — we always compute `manhattan(curr, x1,y1)` on the fly; only `(x2,y2)` is stored as a reachable node
- **Stale entries in PQ** — C# `PriorityQueue` doesn't support decrease-key; skip processed nodes via dist check
- **Road may be worse than walking** — `costi` could exceed `manhattan(x1,y1, x2,y2)`; Dijkstra naturally ignores worse paths

---

## Key Takeaway

"Minimum cost path with optional shortcuts in continuous space" → **Dijkstra on a small implicit graph**:

- Nodes = special road endpoints + start + target
- Edge weight from `(x,y)` to road `i` endpoint = `manhattan(x,y, x1,y1) + costi`
- Between any two nodes: Manhattan distance (free to walk anywhere)

No need to build the full graph explicitly — expand edges on-the-fly from the priority queue.
