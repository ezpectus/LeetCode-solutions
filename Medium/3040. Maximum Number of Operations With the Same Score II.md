# 3040. Maximum Number of Operations With the Same Score II
## O(n²) — Try 3 Target Scores × Interval DP

---

## Problem Statement

- Each operation: delete first two, last two, or first+last elements. Score = their sum.
- All operations must have the **same** score.
- Maximize the number of operations.

---

## Core Idea — Try 3 Possible Initial Scores, Solve Each with Interval DP

The score of the **first** operation must be one of:
- `nums[0] + nums[1]` (first two)
- `nums[n-1] + nums[n-2]` (last two)
- `nums[0] + nums[n-1]` (first and last)

Once the target score is fixed, use **interval DP** on the remaining subarray `nums[l..r]`.

**State**: `dp(l, r)` = maximum operations achievable on subarray `nums[l..r]` with the fixed target score.

**Transition**:
```
dp(l, r) = max of:
  1 + dp(l+2, r)     if nums[l] + nums[l+1] == target  (delete first two)
  1 + dp(l, r-2)     if nums[r-1] + nums[r] == target  (delete last two)
  1 + dp(l+1, r-1)   if nums[l] + nums[r] == target    (delete first and last)
```

**Base case**: if `r - l < 1` (fewer than 2 elements) → 0 operations possible.

Run this DP 3 times (once per candidate score) and return the maximum.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int[] nums;
    int n;
    int[,] memo;
    int target;

    public int MaxOperations(int[] nums) {
        this.nums = nums;
        n = nums.Length;

        int best = 0;
        int[] candidates = {
            nums[0] + nums[1],
            nums[0] + nums[n-1],
            nums[n-1] + nums[n-2]
        };

        foreach (int t in candidates.Distinct()) {
            target = t;
            memo = new int[n, n];
            for (int i = 0; i < n; i++)
                for (int j = 0; j < n; j++)
                    memo[i,j] = -1;
            best = Math.Max(best, Dp(0, n-1));
        }
        return best;
    }

    int Dp(int l, int r) {
        if (r - l < 1) return 0;
        if (memo[l, r] != -1) return memo[l, r];

        int res = 0;
        if (nums[l] + nums[l+1] == target) res = Math.Max(res, 1 + Dp(l+2, r));
        if (nums[r-1] + nums[r] == target) res = Math.Max(res, 1 + Dp(l, r-2));
        if (nums[l] + nums[r] == target)   res = Math.Max(res, 1 + Dp(l+1, r-1));

        return memo[l, r] = res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | 3 DP runs × O(n²) states, each O(1) |
| **Space** | **O(n²)** | Memoization table |

With `n ≤ 2000`: 3 × 4,000,000 = 12M states — fast.

---

## Why Only 3 Candidate Scores?

The **first operation** must use one of three pairs: (first,first+1), (last-1,last), or (first,last). These are the only options before any deletion happens. Once the score of the first operation is fixed, all subsequent operations must match it.

Trying all three candidates covers every possible optimal sequence.

---

## Example Walkthrough

**`nums=[3,2,1,2,3,4], n=6`**

Candidates: 3+2=5, 3+4=7, 3+4=7 → try scores {5, 7}.

**target=5**:
- `dp(0,5)`: nums[0]+nums[1]=5 ✓ → 1+dp(2,5)
  - `dp(2,5)`: nums[2]+nums[5]=1+4=5 ✓ → 1+dp(3,4)
    - `dp(3,4)`: nums[3]+nums[4]=2+3=5 ✓ → 1+dp(4,3)=1+0=1
  - dp(2,5)=2
- dp(0,5)=3

**target=7**: smaller result (can verify).

Answer: **3** ✓

## Pitfalls & Edge Cases

- **Deduplicate candidates**: if `nums[0]+nums[1] == nums[0]+nums[n-1]`, run DP only once for that score (use `Distinct()`)
- **`r - l < 1`**: base case when fewer than 2 elements remain (l > r or l == r → 0 ops)
- **`memo[l,r] = -1`** initialization: 0 is a valid result (e.g., no valid operation possible from this state), so use -1 as "not computed" sentinel
- **All operations optional**: if no operation matches the target, `dp` returns 0 — correctly handled

---

## Key Takeaway

"Max operations with same score (3 deletion choices)" → **enumerate 3 initial scores + interval DP**:

```
for target in {first_two_sum, last_two_sum, first_last_sum}:
    dp(l, r) = max of 3 choices if they match target
answer = max over all targets
```

O(n²) interval DP, run 3 times. The candidate score enumeration is the key insight — we don't need to try all possible sums.
