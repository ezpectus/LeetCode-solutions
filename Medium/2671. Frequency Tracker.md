# 2671. Frequency Tracker
## O(1) per operation — Two Hash Maps

---

## Problem Statement

Design a structure supporting:
- `Add(number)` — add one occurrence
- `DeleteOne(number)` — remove one occurrence (no-op if absent)
- `HasFrequency(frequency)` — is there any number occurring exactly `frequency` times?

All operations must be O(1).

---

## Core Idea — Maintain Two Maps in Sync

- `cnt[number]` = current count of `number`
- `freqCnt[f]` = how many distinct numbers currently have count exactly `f`

On every `Add`/`DeleteOne`: decrement `freqCnt[old]`, update `cnt[number]`, increment `freqCnt[new]`.

`HasFrequency(f)` = `freqCnt[f] > 0` — O(1).

---

## Full Optimal Implementation (C#)

```csharp
public class FrequencyTracker {
    Dictionary<int,int> cnt     = new(); // number → count
    Dictionary<int,int> freqCnt = new(); // freq → # numbers with that freq

    public void Add(int number) {
        int old = cnt.GetValueOrDefault(number, 0);
        if (old > 0) freqCnt[old]--;
        cnt[number] = old + 1;
        freqCnt[old + 1] = freqCnt.GetValueOrDefault(old + 1, 0) + 1;
    }

    public void DeleteOne(int number) {
        int old = cnt.GetValueOrDefault(number, 0);
        if (old == 0) return;
        freqCnt[old]--;
        cnt[number] = old - 1;
        if (old - 1 > 0)
            freqCnt[old - 1] = freqCnt.GetValueOrDefault(old - 1, 0) + 1;
    }

    public bool HasFrequency(int frequency) =>
        freqCnt.GetValueOrDefault(frequency, 0) > 0;
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Add / DeleteOne / HasFrequency** | **O(1)** |
| **Space** | **O(n)** — n distinct numbers added |

---

## Pitfalls & Edge Cases

- **`DeleteOne` when count = 0** — early return, don't touch `freqCnt`
- **Don't add to `freqCnt[0]`** — count 0 means "not present"; only track counts ≥ 1 in `freqCnt`
- **`freqCnt[old]` can go negative momentarily** — safe since we always check `> 0` in `HasFrequency`, never read negative as meaningful

---

## Key Takeaway

"O(1) frequency-of-frequency queries" → **two maps kept in sync**: whenever a count changes from `old` to `new`, update both `freqCnt[old]--` and `freqCnt[new]++`. Classic design pattern for frequency-tracking structures.
