# 3207. Maximum Points After Enemy Battles
## O(n) — Greedy: Absorb All Non-Minimum Enemies, Farm Minimum

---

## Problem Statement

- Op1 (costs energy, gains point): defeat unmarked enemy `i` if `currentEnergy >= enemyEnergies[i]`. Costs `enemyEnergies[i]` energy, gains 1 point.
- Op2 (gains energy, needs ≥1 point): absorb unmarked enemy `i`. Gains `enemyEnergies[i]` energy, marks enemy.
- Maximize total points.

---

## Core Idea — Two-Phase Greedy

**Phase 1**: Get the first point by defeating the minimum-energy enemy (if `currentEnergy >= min`). This unlocks Op2.

**Phase 2**: Use Op2 to absorb every other enemy (gains all their energy). Then spend all accumulated energy on Op1 against the minimum enemy repeatedly.

**Why absorb all others first?** Each absorbed enemy increases your energy pool, which translates directly into more Op1 uses on the minimum enemy. Absorbing is strictly better than defeating non-minimum enemies (each defeat costs energy and gains only 1 point, while absorbing gains energy for free).

**Why farm the minimum?** Op1 costs `enemyEnergies[i]` energy per point. To maximize points, always pick the cheapest enemy = minimum energy.

**Formula**:
```
total_energy = currentEnergy + sum(all enemies except minimum)
points = total_energy / min(enemyEnergies)
```

If `currentEnergy < min`: return 0 (can't even get the first point).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumPoints(int[] enemyEnergies, int currentEnergy) {
        long mn = enemyEnergies.Min();
        long total = enemyEnergies.Sum(x => (long)x);

        if (currentEnergy < mn) return 0;

        // Absorb all non-minimum enemies to maximize energy pool
        long energy = currentEnergy + (total - mn);
        return energy / mn;
    }
}
```

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** |

---

## Example Walkthrough

**`enemyEnergies=[3,2,2], currentEnergy=2`**

min=2, total=7. currentEnergy=2 ≥ 2 ✓.

energy = 2 + (7-2) = 7. points = 7/2 = **3** ✓

**`enemyEnergies=[2], currentEnergy=10`**

min=2, total=2. energy = 10 + 0 = 10. points = 10/2 = **5** ✓

---

## Pitfalls & Edge Cases

- **`currentEnergy < min`**: can't defeat anyone → return 0 immediately.
- **Single enemy**: absorb nothing, just farm it with starting energy.
- **`long`**: `n=10^5`, energies up to `10^9` → sum up to `10^{14}`, overflows `int`.

---

## Key Takeaway

"Max points from enemy battles" → **absorb all non-minimum enemies, farm minimum**:

```
if currentEnergy < min: return 0
energy = currentEnergy + (total - min)
return energy / min
```

Absorbing maximizes energy pool for free; farming the minimum maximizes points per energy spent.
