# 2865. Beautiful Towers I
## O(n²) — Try Every Peak, Accumulate Min Outward

---

## Problem Statement

- Remove bricks to form a "mountain": non-decreasing to a peak, then non-increasing.
- Multiple consecutive towers can share the same peak height.
- Return maximum possible sum of heights.

---

## Core Idea — Fix Peak, Greedy Outward Min

For each index `peak`, assume `heights[peak]` is the maximum (peak value). Going left and right from the peak, each tower's height is capped by the minimum seen so far — we can only decrease (or keep equal) as we move away.

**Left sweep** (from `peak-1` down to 0):
```
curr = heights[peak]
curr = min(curr, heights[i])   // can't increase as we go left
total += curr
```

**Right sweep** (from `peak+1` up to n-1): same with `min`.

Try all `n` peaks, return the maximum total.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumSumOfHeights(int[] heights) {
        int n = heights.Length;
        long res = 0;

        for (int peak = 0; peak < n; peak++) {
            long total = heights[peak];

            // Left side: heights must be non-increasing going left
            long curr = heights[peak];
            for (int i = peak - 1; i >= 0; i--) {
                curr = Math.Min(curr, heights[i]);
                total += curr;
            }

            // Right side: heights must be non-increasing going right
            curr = heights[peak];
            for (int i = peak + 1; i < n; i++) {
                curr = Math.Min(curr, heights[i]);
                total += curr;
            }

            res = Math.Max(res, total);
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | n peaks × O(n) sweep each |
| **Space** | **O(1)** | Only running totals |

With `n ≤ 1000`: at most 10⁶ operations — fine. For larger `n`, use monotonic stack (O(n)).

---

## Example Walkthrough

**`heights=[6,5,3,9,2,7]`**, peak=3 (value=9):

Left sweep (indices 2,1,0):
- i=2: curr=min(9,3)=3, total+=3
- i=1: curr=min(3,5)=3, total+=3
- i=0: curr=min(3,6)=3, total+=3

Right sweep (indices 4,5):
- i=4: curr=min(9,2)=2, total+=2
- i=5: curr=min(2,7)=2, total+=2

Total = 9+3+3+3+2+2 = **22** ✓

---

## Pitfalls & Edge Cases

- **`long` for total**: heights up to 10⁹, n up to 10³ → max sum ~10¹² overflows `int`
- **Single element**: loop bodies don't execute → returns `heights[0]` ✓
- **All equal heights**: every peak gives same sum, all valid mountain shapes
- **Peak at boundary**: one sweep is empty, still works correctly

---

## Key Takeaway

"Maximum mountain-shaped sum" → **try each peak, greedily clamp outward with running min**:

```
going left from peak:  curr = min(curr, heights[i]) at each step
going right from peak: same
```

The running min enforces the non-increasing constraint away from peak. O(n²) for n≤1000; use monotonic stack for O(n) with larger inputs.
