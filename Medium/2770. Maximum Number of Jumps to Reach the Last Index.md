# 2770. Maximum Number of Jumps to Reach the Last Index
## O(n²) — DP: Maximum Jumps to Each Index

---

## Problem Statement

- From index `i` jump to any `j > i` if `|nums[j] - nums[i]| ≤ target`.
- Return the maximum number of jumps to reach index `n-1`, or `-1` if impossible.

---

## Core Idea — DP on Reachability

**State**: `dp[j]` = maximum jumps to reach index `j` from index `0`. Initialize all to `-1` (unreachable), except `dp[0] = 0`.

**Transition**: for each `j`, try every previous index `i`:
```
if dp[i] != -1 and |nums[j] - nums[i]| <= target:
    dp[j] = max(dp[j], dp[i] + 1)
```

**Answer**: `dp[n-1]`.

The `-1` sentinel is essential — it propagates "unreachability" correctly. Only reachable states (`dp[i] != -1`) can extend to `j`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumJumps(int[] nums, int target) {
        int n = nums.Length;
        int[] dp = new int[n];
        Array.Fill(dp, -1);
        dp[0] = 0;

        for (int j = 1; j < n; j++)
            for (int i = 0; i < j; i++)
                if (dp[i] != -1 && Math.Abs(nums[j] - nums[i]) <= target)
                    dp[j] = Math.Max(dp[j], dp[i] + 1);

        return dp[n-1];
    }
}
```

| **Metric** | **Value** | **Notes** |

| **Time** | **O(n²)** | Double loop; n ≤ 1000 → 10⁶ ops |
| **Space** | **O(n)** | dp array |

## Example Walkthrough

**`nums=[1,3,6,4,1,2], target=2`**

| j | valid i's (|nums[j]-nums[i]|≤2) | dp[j] |
|---|--------------------------------|-------|
| 1 | i=0: |3-1|=2≤2 ✓ → dp[0]+1=1 | **1** |
| 2 | i=0: |6-1|=5✗; i=1: |6-3|=3✗ | **-1** |
| 3 | i=0: |4-1|=3✗; i=1: |4-3|=1✓ dp[1]+1=2; i=2: unreachable | **2** |
| 4 | i=3: |1-4|=3✗; others fail | **-1** |
| 5 | i=3: |2-4|=2✓ dp[3]+1=3; i=4: unreachable | **3** |

`dp[5] = 3` ✓

---

## Pitfalls & Edge Cases

- **`dp[i] != -1` guard** — never extend from an unreachable state; otherwise `-1 + 1 = 0` would falsely mark `j` as reachable with 0 jumps
- **`target = 0`** — only `nums[j] == nums[i]` jumps are valid; if no such pair exists on path to `n-1` → returns `-1` ✓
- **`n = 2`** — one iteration: check if index 0 → index 1 is valid

---

## Key Takeaway

"Maximum steps path with value constraint" → **O(n²) DP**:

```
dp[j] = max over valid i of dp[i] + 1
```

Use `-1` as "unreachable" sentinel — never extend from it. Standard pattern for longest/maximum path problems where not all transitions are valid.
