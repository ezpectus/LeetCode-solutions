# 3843. First Element with Unique Frequency
## O(n) — Frequency Count + Frequency-of-Frequency Map

---

## Problem Statement

- Given `nums` (1 ≤ n ≤ 10⁵, 1 ≤ nums[i] ≤ 10⁵).
- Return the first element (by leftmost position) whose frequency among all elements is unique — no other element appears the same number of times.
- Return −1 if no such element exists.

---

## Core Idea — Two Maps + Single Scan

1. `freq[x]` = how many times element x appears in nums.
2. `freqCount[f]` = how many distinct elements have frequency f.

A frequency is "unique" iff `freqCount[freq[x]] == 1`.

Scan nums left to right; for each new (first occurrence) element x: if its frequency is unique, return x.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int FirstUniqueFreq(int[] nums) {
        // Step 1: count element frequencies
        var freq = new Dictionary<int, int>();
        foreach (int x in nums) {
            freq[x] = freq.GetValueOrDefault(x, 0) + 1;
        }

        // Step 2: count how many elements share each frequency
        var freqCount = new Dictionary<int, int>();
        foreach (int f in freq.Values) {
            freqCount[f] = freqCount.GetValueOrDefault(f, 0) + 1;
        }

        // Step 3: scan left to right, first element with unique frequency
        var seen = new HashSet<int>();
        foreach (int x in nums) {
            if (seen.Add(x) && freqCount[freq[x]] == 1) return x;
        }

        return -1;
    }
}
```

---

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n) | Three linear passes |
| Space Complexity | O(n) | Two dictionaries + seen set |

---

## Example Walkthrough

**Example 1:** `nums = [20, 10, 30, 30]`

freq: 20→1, 10→1, 30→2
freqCount: {1:2, 2:1} ← two elements with freq 1, one with freq 2

Scan: 20 (new, freq=1, freqCount[1]=2 → not unique), 10 (new, freq=1, not unique), 30 (new, freq=2, freqCount[2]=1 → **unique**) → **return 30** ✓

---

**Example 2:** `nums = [20, 20, 10, 30, 30, 30]`

freq: 20→2, 10→1, 30→3
freqCount: {2:1, 1:1, 3:1} ← all frequencies are unique

Scan: 20 (new, freq=2, freqCount[2]=1 → **unique**) → **return 20** ✓

---

**Example 3:** `nums = [10, 10, 20, 20]`

freq: 10→2, 20→2
freqCount: {2:2} ← two elements share frequency 2

Scan: 10 (new, freqCount[2]=2 → not unique), 20 (new, freqCount[2]=2 → not unique) → **return −1** ✓

---

## Pitfalls & Edge Cases

- **Duplicate positions:** when scanning, skip repeated occurrences of the same element using a `seen` set. We only check each value once (at its first occurrence). ✓

- **All same frequency:** freqCount has one entry with count = number of distinct elements. No element has a unique frequency. ✓

- **All distinct elements:** each has frequency 1. freqCount = {1: n}. Only unique if n = 1. ✓

- **Return first by position:** the scan order (left to right) with `seen.Add(x)` ensures we check each value at its first occurrence, giving leftmost-position semantics. ✓

---

## Key Takeaway

"Find element with a unique property" → compute the property for all, count occurrences of each property value (freqCount), then scan for the first element where its property count = 1.

This two-level frequency counting pattern (freq of elements → freq of frequencies) is the standard approach for problems asking about "unique" frequencies or occurrences.
