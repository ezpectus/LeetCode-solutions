# 2640. Find the Score of All Prefixes of an Array
## O(n) ‚Äî Running Max + Running Sum

---

## Problem Statement

- Define `conver[i] = nums[i] + max(nums[0..i])`.
- Define score of prefix `[0..i]` = `Œ£ conver[j]` for `j` in `[0..i]`.
- Return an array `res` where `res[i]` = score of prefix `nums[0..i]`.

---

## Core Idea ‚Äî Two Running Variables

**Key insight**: Both the prefix maximum and the prefix score can be maintained incrementally in a single pass:

```
currMax  = max(nums[0..i])         ‚Üí updated each step
conver_i = nums[i] + currMax       ‚Üí current conversion value
runSum  += conver_i                ‚Üí cumulative score
res[i]   = runSum
```

No need to recompute prefix max or resum the conversion array from scratch at each `i` ‚Äî both grow by exactly one element per step.

**Why runSum works?**

```
res[i] = Œ£_{j=0}^{i} conver[j]
       = Œ£_{j=0}^{i-1} conver[j]  +  conver[i]
       = res[i-1]                  +  (nums[i] + currMax)
```

Each step just adds the new conversion value to the previous total.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long[] FindPrefixScore(int[] nums) {
        int n = nums.Length;
        long[] res = new long[n];
        int currMax = 0;
        long runSum = 0;

        for (int i = 0; i < n; i++) {
            currMax = Math.Max(currMax, nums[i]);   // running prefix max
            runSum += nums[i] + currMax;            // conver[i] = nums[i] + max(0..i)
            res[i] = runSum;
        }

        return res;
    }
}
```

## Pitfalls & Edge Cases

- **`long` for `runSum` and `res`** ‚Äî `nums[i]` up to 10‚Åπ, `currMax` up to 10‚Åπ, `conver[i]` up to 2√ó10‚Åπ per step, summed over 10‚Åµ elements ‚Üí max `runSum` ‚âà 2√ó10¬π‚Å¥ ‚Üí overflows `int`, use `long`
- **`currMax` can stay `int`** ‚Äî it's just the max of `nums[i]` values which fit in `int` (‚â§ 10‚Åπ)
- **Non-decreasing `runSum`** ‚Äî since all `nums[i] ‚â• 1` and `currMax ‚â• 1`, `conver[i] ‚â• 2` always, so `runSum` strictly increases ‚úì

---

## Key Takeaway

"Score of all prefixes" ‚Üí **two running variables, one pass**:

- `currMax` = running prefix maximum, updated with `Math.Max`
- `runSum` = running prefix score, incremented by `nums[i] + currMax` each step

Pattern: any problem asking for prefix aggregates (max, sum, product) ‚Üí maintain incrementally rather than recomputing from scratch.
