# 3532. Path Existence Queries in a Graph I
## O(n + q) — Contiguous Group IDs on Sorted Array

---

## Problem Statement

- Nodes `i` and `j` connected iff `|nums[i]-nums[j]| ≤ maxDiff`. `nums` is sorted.
- For each query `(u,v)`, determine if `u` and `v` are in the same connected component.

---

## Core Idea — Connected Components Are Contiguous Ranges

Since `nums` is sorted, `|nums[i]-nums[j]| ≤ maxDiff` for all pairs in a component iff consecutive pairs satisfy the condition (triangle property on sorted array). Two nodes `i` and `i+1` are directly connected iff `nums[i+1]-nums[i] ≤ maxDiff`.

Assign group IDs: scan left to right, increment group when consecutive difference exceeds `maxDiff`. Two nodes are connected iff they have the same group ID.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool[] PathExistenceQueries(int n, int[] nums, int maxDiff, int[][] queries) {
        int[] group = new int[n];
        for (int i = 1; i < n; i++)
            group[i] = nums[i]-nums[i-1] <= maxDiff ? group[i-1] : group[i-1]+1;

        return queries.Select(q => group[q[0]] == group[q[1]]).ToArray();
    }
}
```

**Time**: O(n + q) — **Space**: O(n).

---

## Example Walkthrough

**`nums=[2,5,6,8], maxDiff=2`**

- i=1: |5-2|=3>2 → group[1]=1.
- i=2: |6-5|=1≤2 → group[2]=1.
- i=3: |8-6|=2≤2 → group[3]=1.

groups=[0,1,1,1].

- Query [1,3]: group[1]=group[3]=1 → **true** ✓
- Query [0,1]: group[0]=0 ≠ group[1]=1 → **false** ✓

---

## Key Takeaway

Sorted array + distance threshold → connected components = contiguous ranges separated by gaps > maxDiff. Assign incremental group IDs in one pass. O(1) per query.
