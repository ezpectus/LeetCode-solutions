# 3478. Choose K Elements With Maximum Sum
## O(n log n + n log k) — Sort + Min-Heap of Size k

---

## Problem Statement

- For each `i`, find all `j` where `nums1[j] < nums1[i]`, pick at most `k` values from `nums2[j]` to maximize sum.

---

## Core Idea — Sort by nums1, Process Groups, Maintain Top-k Heap

Sort indices by `nums1` value. Process in ascending order. For a group of equal `nums1` values, all elements in that group see the **same set of eligible `j`s** (all previous groups with strictly smaller `nums1`).

**Algorithm**:
1. Sort indices by `nums1[i]`.
2. Sweep left to right in groups of equal `nums1` values.
3. **Before** processing a group: record `heap_sum` as the answer for all indices in this group (they can use all previously added nums2 values).
4. **After** recording: add all `nums2[order[i]]` from this group to the min-heap. If heap size exceeds `k`, remove the minimum (keeping top-k largest).

The min-heap maintains the top-k largest `nums2` values seen so far, with `heap_sum` tracking their sum.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long[] FindMaxSum(int[] nums1, int[] nums2, int k) {
        int n = nums1.Length;
        long[] ans = new long[n];

        int[] order = Enumerable.Range(0, n).OrderBy(i => nums1[i]).ToArray();

        var heap = new PriorityQueue<int, int>(); // min-heap
        long heapSum = 0;
        int i = 0;

        while (i < n) {
            int j = i;
            // Find group with same nums1 value
            while (j < n && nums1[order[j]] == nums1[order[i]]) j++;

            // Record answer for all in this group (heap contains only strictly smaller nums1)
            for (int idx = i; idx < j; idx++)
                ans[order[idx]] = heapSum;

            // Add this group's nums2 values to heap
            for (int idx = i; idx < j; idx++) {
                int val = nums2[order[idx]];
                heap.Enqueue(val, val);
                heapSum += val;
                if (heap.Count > k) {
                    heapSum -= heap.Dequeue();
                }
            }

            i = j;
        }

        return ans;
    }
}
```

## Example Walkthrough

**`nums1=[4,2,1,5,3], nums2=[10,20,30,40,50], k=2`**

Sorted by nums1: indices [2,1,4,0,3] → nums1 values [1,2,3,4,5].

- Group nums1=1 (idx=2): heap empty → ans[2]=0. Add nums2[2]=30. Heap=[30], sum=30.
- Group nums1=2 (idx=1): ans[1]=30. Add nums2[1]=20. Heap=[20,30], sum=50.
- Group nums1=3 (idx=4): ans[4]=50. Add nums2[4]=50. Heap=[20,30,50]→pop 20. Heap=[30,50], sum=80.
- Group nums1=4 (idx=0): ans[0]=80. Add nums2[0]=10. Heap=[10,30,50]→pop 10. Heap=[30,50], sum=80.
- Group nums1=5 (idx=3): ans[3]=80. Add nums2[3]=40. Heap=[30,40,50]→pop 30. sum=90 (but k=2 exceeded after add).

Wait: at group nums1=5, ans[3]=80 ✓.

Final: ans=[80,30,0,80,50] ✓

## Pitfalls & Edge Cases

- **Group processing**: must record answers for the entire group BEFORE adding their nums2 values — elements with equal nums1 cannot use each other's nums2.
- **`long` for sum**: up to n×10⁶ = 10¹¹ → overflows `int`.
- **Min-heap for top-k**: when heap size exceeds k, remove the minimum to keep the k largest.

---

## Key Takeaway

"For each i, sum of top-k nums2 where nums1[j] < nums1[i]" → **sort by nums1, process groups of equal values, maintain top-k min-heap. Record answer BEFORE adding current group to heap**:

```
sort by nums1
for each group of equal nums1:
    ans[i] = heap_sum (from previous groups only)
    add this group's nums2 to heap, maintain size ≤ k
```
