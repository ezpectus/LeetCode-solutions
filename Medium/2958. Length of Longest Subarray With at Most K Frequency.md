# 2958. Length of Longest Subarray With at Most K Frequency
## O(n) — Sliding Window with Frequency Map

---

## Problem Statement

- Given integer array `nums` and integer `k`.
- A **good** array has every element appearing at most `k` times.
- Return the length of the **longest good subarray** of `nums`.

---

## Core Idea — Sliding Window

**Key insight**: we want the longest contiguous subarray where no element has frequency > `k`. This is a classic sliding window — expand the right boundary greedily, shrink the left boundary only when a violation occurs.

**Invariant**: the window `[left, right]` is always a good subarray.

**When `nums[right]` causes a violation** (its frequency exceeds `k`):
- Shrink from the left by advancing `left` until `freq[nums[right]] ≤ k`.
- Each step: decrement `freq[nums[left]]`, increment `left`.

**At every step** record `right - left + 1` as a candidate answer.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxSubarrayLength(int[] nums, int k) {
        Dictionary<int, int> freq = new();
        int left = 0, best = 0;

        for (int right = 0; right < nums.Length; right++) {
            freq[nums[right]] = freq.GetValueOrDefault(nums[right]) + 1;

            while (freq[nums[right]] > k) {
                freq[nums[left]]--;
                left++;
            }

            best = Math.Max(best, right - left + 1);
        }

        return best;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Each element enters and leaves the window at most once |
| **Space** | **O(n)** | Frequency map holds at most `n` distinct keys |

---

## Why This Works — Sliding Window Correctness

**Why only shrink when `freq[nums[right]] > k`?**

Adding `nums[right]` to the window can only violate the constraint for `nums[right]` itself — all other elements in `[left, right-1]` were already valid. So we only need to check the newly added element.

**Why shrink from the left until the violation clears?**

The leftmost elements are the "oldest" in the window. To minimally reduce the count of `nums[right]`, we remove elements from the left until one occurrence of `nums[right]` has been evicted. This is optimal — removing from anywhere else would shrink the window unnecessarily.

**Why is the window always valid after the while loop?**

We stop shrinking exactly when `freq[nums[right]] == k`, restoring the good-array property. All other elements remained ≤ k throughout (since we only modified `nums[right]`'s count by adding it).

**Why is this O(n)?**

`left` only moves forward — each index is added to the window once (via `right`) and removed at most once (via `left`). Total operations: O(2n) = O(n).

## Pitfalls & Edge Cases

- **Check only `nums[right]` in the while loop**: only the newly added element can cause a violation — no need to scan all frequencies after each expansion
- **`GetValueOrDefault`**: elements not yet seen have frequency 0 — use default-safe dictionary access to avoid `KeyNotFoundException`
- **`k = nums.Length`**: entire array is always valid → answer is `n`, handled naturally (window never shrinks)
- **All same elements**: window grows until frequency hits `k`, then slides — correctly returns `k`

---

## Key Takeaway

"Longest subarray where every element appears at most `k` times" → **sliding window with frequency map**:

```
left = 0
for right in 0..n-1:
    freq[nums[right]]++
    while freq[nums[right]] > k:
        freq[nums[left]]--
        left++
    best = max(best, right - left + 1)
return best
```

Only the newly added element can cause a violation → shrink left until it's resolved → each element enters/exits window once → O(n) time, O(n) space.
