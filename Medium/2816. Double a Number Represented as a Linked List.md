# 2816. Double a Number Represented as a Linked List
## O(n) — Reverse, Double with Carry, Reverse Back

---

## Problem Statement

- Linked list represents a number (most significant digit first).
- Return the list after doubling the number.

---

## Core Idea — Reverse to Process Least Significant First

Doubling requires propagating carry from least significant to most significant digit. The list is stored MSB-first, so we reverse it, apply the doubling digit by digit, then reverse back.

**Steps**:
1. Reverse the list (now LSB-first)
2. Traverse: `val = node.val * 2 + carry`, set `node.val = val % 10`, `carry = val / 10`
3. Reverse back
4. If carry remains, prepend a new node

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public ListNode DoubleIt(ListNode head) {
        // Step 1: Reverse
        ListNode prev = null, curr = head;
        while (curr != null) {
            var next = curr.next;
            curr.next = prev;
            prev = curr;
            curr = next;
        }

        // Step 2: Double with carry (prev is now the reversed head)
        int carry = 0;
        ListNode newHead = null;
        curr = prev;
        while (curr != null) {
            int val = curr.val * 2 + carry;
            carry = val / 10;
            curr.val = val % 10;
            // Reverse back while processing
            var next = curr.next;
            curr.next = newHead;
            newHead = curr;
            curr = next;
        }

        // Step 3: Handle remaining carry
        if (carry > 0)
            newHead = new ListNode(carry, newHead);

        return newHead;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — two passes (reverse + double) |
| **Space** | **O(1)** — in-place |

---

## Alternative — No Reverse Needed

Since carry can only be 0 or 1 (doubling a digit ≤ 9 gives ≤ 18, carry ≤ 1), we can use a forward pass with a look-ahead:

```csharp
// If current node * 2 >= 10, next node needs +1 carry
// So: if curr.val * 2 + (next generates carry?) ...
// Simpler: just check if next.val >= 5 (5*2=10 → carry=1)
```

```csharp
// Forward pass approach:
if (head.val >= 5) head = new ListNode(0, head); // potential leading carry
var node = head;
while (node != null) {
    node.val *= 2;
    if (node.next != null && node.next.val >= 5) node.val++;
    node.val %= 10;
    node = node.next;
}
return head.val == 0 ? head.next : head; // remove leading zero if no carry
```

Both approaches are O(n). The reverse approach is more general; the forward look-ahead is more elegant for this specific problem.

---

## Pitfalls & Edge Cases

- **All 9s** (`999 → 1998`): carry propagates all the way → new head node needed
- **Single node** (`5 → 10`): carry=1, new head created ✓
- **`head.val == 0`**: number is 0, doubling gives 0 ✓

---

## Key Takeaway

"Double a number in a linked list (MSB-first)" → **reverse + digit-by-digit doubling + reverse** OR **forward pass with look-ahead carry detection** (`next.val >= 5`). Both O(n).
