# 2850. Minimum Moves to Spread Stones Over Grid
## O(k!) — Enumerate Assignments via Permutations

---

## Problem Statement

- 3×3 grid with exactly 9 stones total. Move stones one step (4-directional) at a time.
- Return minimum moves to place exactly one stone per cell.

---

## Core Idea — Match Extras to Empties

**Sources**: cells with `grid[r][c] > 1` contribute `grid[r][c]-1` extra stones (each extra needs to move somewhere).  
**Targets**: empty cells (`grid[r][c] == 0`) each need one stone.

Number of extras = number of targets (both equal `9 - filled_cells` effectively).

**Enumerate all assignments**: try all ways to assign sources to targets (permutations). Cost of one assignment = sum of Manhattan distances. Take the minimum.

**Why permutations?** Since the grid is 3×3, at most 8 extras/targets (degenerate: one cell has 9). In practice, very few cells — typically 2-4 — making `k!` small (≤ 8! = 40320 at most, usually much less).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumMoves(int[][] grid) {
        var sources = new List<(int,int)>();
        var targets = new List<(int,int)>();

        for (int r = 0; r < 3; r++)
            for (int c = 0; c < 3; c++) {
                if (grid[r][c] == 0) targets.Add((r,c));
                else for (int k = 1; k < grid[r][c]; k++) sources.Add((r,c));
            }

        int res = int.MaxValue;
        // Try all permutations of targets assignments
        var idx = Enumerable.Range(0, targets.Count).ToArray();
        do {
            int cost = 0;
            for (int i = 0; i < sources.Count; i++)
                cost += Math.Abs(sources[i].Item1 - targets[idx[i]].Item1)
                      + Math.Abs(sources[i].Item2 - targets[idx[i]].Item2);
            res = Math.Min(res, cost);
        } while (NextPermutation(idx));

        return res;
    }

    bool NextPermutation(int[] arr) {
        int i = arr.Length - 2;
        while (i >= 0 && arr[i] >= arr[i+1]) i--;
        if (i < 0) return false;
        int j = arr.Length - 1;
        while (arr[j] <= arr[i]) j--;
        (arr[i], arr[j]) = (arr[j], arr[i]);
        Array.Reverse(arr, i+1, arr.Length-i-1);
        return true;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(k! × k)** | k = number of extras ≤ 8; k! ≤ 40320 |
| **Space** | **O(k)** | Source/target lists |

In practice k is small (2-4 for most inputs) → essentially O(1).

---

## Example Walkthrough

**`grid=[[1,3,0],[1,0,0],[1,0,3]]`**

- Sources: `(0,1),(0,1)` (3-1=2 extras) and `(2,2),(2,2)` (3-1=2 extras) → 4 sources
- Targets: `(0,2),(1,1),(1,2),(2,1)` → 4 targets

Best assignment (by intuition): `(0,1)→(0,2)`, `(0,1)→(1,1)`, `(2,2)→(1,2)`, `(2,2)→(2,1)`
Cost = 1 + 1 + 1 + 1 = **4** ✓

---

## Pitfalls & Edge Cases

- **Multiple stones at same cell**: each extra stone is an independent source (duplicate coordinates in sources list)
- **`res = int.MaxValue`**: safe since there's always at least one valid assignment
- **Grid already solved**: no extras, no targets → `res = 0` ✓

---

## Key Takeaway

"Minimum transport cost matching sources to targets in small grid" → **enumerate all assignments via permutations + Manhattan distance**. Works here because 3×3 grid bounds the problem size tightly.

For larger grids, use **minimum cost bipartite matching** (Hungarian algorithm or min-cost flow).
