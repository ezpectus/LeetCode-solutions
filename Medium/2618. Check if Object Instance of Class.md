# 2618. Check if Object Instance of Class
## O(d) — Manual Prototype Chain Walk

---

## Problem Statement

- Write a function that checks if `obj` is an instance of `classFunction` or any of its superclasses.
- An object is considered an instance if it has access to the class's methods (i.e., the class prototype appears in its prototype chain).
- **Cannot use `instanceof`** — it returns `false` for primitives like `5` vs `Number`.
- No constraints on input types — handle `null`, `undefined`, primitives, functions, etc.

---

## Core Idea — Walk the Prototype Chain Manually

**Key insight**: `instanceof` checks the prototype chain under the hood — but it fails for **primitives** because `5` is not an object and has no prototype chain directly. The fix: wrap the value with `Object(obj)` first, which boxes primitives (`5` → `Number{5}`, `'str'` → `String{'str'}`) without affecting objects or class instances.

**Algorithm**:
1. Guard: if `obj` is `null`/`undefined` or `classFunction` is not a valid function → return `false`
2. Start at `proto = Object.getPrototypeOf(Object(obj))`
3. Walk up: if `proto === classFunction.prototype` → `true`
4. If `proto === null` (reached top of chain) → `false`

**Why `Object(obj)` works for primitives**:
```
Object(5)         → Number{5}   → prototype chain includes Number.prototype
Object('str')     → String{...} → prototype chain includes String.prototype
Object(new Dog()) → new Dog()   → unchanged, works as before
```

---

## Full Optimal Implementation (JavaScript)

```javascript
var checkIfInstanceOf = function(obj, classFunction) {
    // Guard: null/undefined obj or invalid classFunction
    if (obj === null || obj === undefined || 
        classFunction === null || classFunction === undefined || 
        typeof classFunction !== 'function') 
        return false;

    // Walk prototype chain (Object() boxes primitives)
    let proto = Object.getPrototypeOf(Object(obj));
    while (proto !== null) {
        if (proto === classFunction.prototype) return true;
        proto = Object.getPrototypeOf(proto);
    }
    return false;
};
```

## Prototype Chain — How It Works

Every JavaScript object has an internal `[[Prototype]]` link. `Object.getPrototypeOf(x)` reads it. The chain for `new Dog()` (where `Dog extends Animal`):

```
new Dog()
  → Dog.prototype
    → Animal.prototype
      → Object.prototype
        → null
```

Checking `instanceof Animal` means: does `Animal.prototype` appear anywhere in this chain? → Yes → `true`.

For primitive `5` (boxed to `Number{5}`):
```
Number{5}
  → Number.prototype
    → Object.prototype
      → null
```

`Number.prototype` is in the chain → `checkIfInstanceOf(5, Number) = true` ✓

## Example Walkthrough

**Example 3**: `checkIfInstanceOf(Date, Date)` → `false`

`Date` is a function/constructor. `Object(Date)` = `Date` itself (functions are objects).
```
Object.getPrototypeOf(Date) = Function.prototype  (not Date.prototype)
Object.getPrototypeOf(Function.prototype) = Object.prototype
Object.getPrototypeOf(Object.prototype) = null
```
`Date.prototype` never appears → `false` ✓

**Example 4**: `checkIfInstanceOf(5, Number)` → `true`

`Object(5)` = `Number{5}` (boxed primitive).
```
Object.getPrototypeOf(Number{5}) = Number.prototype  ← match! → true ✓
```

---

## Pitfalls & Edge Cases

- **`obj = null` or `obj = undefined`** — `Object(null)` = `{}` (empty object), `Object(undefined)` = `{}` → their prototype chain never matches anything useful; safer to guard upfront and return `false`
- **`classFunction` not a function** — `classFunction.prototype` would throw or be `undefined`; guard with `typeof classFunction !== 'function'`
- **`classFunction = null/undefined`** — same, must guard before accessing `.prototype`
- **Don't use `instanceof` directly** — fails for primitives: `5 instanceof Number` = `false`, but `checkIfInstanceOf(5, Number)` should be `true`
- **`Object(obj)` vs `new Object(obj)`** — both work, but `Object(obj)` is preferred (idiomatic)

---

## Key Takeaway

Manual `instanceof` reimplementation for all types (including primitives):

```javascript
let proto = Object.getPrototypeOf(Object(obj)); // Object() boxes primitives
while (proto !== null) {
    if (proto === classFunction.prototype) return true;
    proto = Object.getPrototypeOf(proto);
}
return false;
```

- `Object(x)` boxes primitives so they have a traversable prototype chain
- Walking with `Object.getPrototypeOf` replicates exactly what `instanceof` does internally
- Difference from `instanceof`: handles primitives correctly
