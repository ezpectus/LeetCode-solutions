# 2896. Apply Operations to Make Two Strings Equal
## O(n) — DP on Diff Positions

---

## Problem Statement

- Two binary strings `s1`, `s2`. Operations on `s1`:
  - Flip `s1[i]` and `s1[j]` (any two positions): cost `x`
  - Flip `s1[i]` and `s1[i+1]` (adjacent): cost `1`
- Return minimum cost to make `s1 == s2`, or -1 if impossible.

---

## Core Idea — Reduce to Pairing Problem on Diff Positions

**Step 1**: XOR `s1` and `s2` — collect positions `diffs[]` where they differ. These are the positions we must flip an **odd** number of times.

**Step 2**: If `|diffs|` is odd → impossible (-1). Both operations flip exactly 2 positions → parity preserved.

**Step 3**: We must pair up diff positions. For each pair `(d[i], d[j])`, the cost is:
- Move a "flip token" from `d[i]` to `d[j]` using adjacent ops: cost = `|d[j] - d[i]|`
- Use op1 directly: cost = `x`
- Best: `min(|d[j] - d[i]|, x)`

For minimum cost on a sorted line, **consecutive pairing is optimal**. But we have a choice at each step: pair `d[i]` with `d[i-1]`, or "defer" it (use op1 for a cost of `x/2` per position).

### DP Formulation

`dp[i]` = 2× (minimum cost to resolve `diffs[0..i]`), working in doubled units to avoid fractions when `x` is odd.

**Transition**:
```
// Option 1: pair d[i] with d[i-1] (consecutive pairing)
opt1 = dp[i-2] + min(d[i]-d[i-1], x) * 2

// Option 2: use op1 for d[i] independently (costs x/2 per position)
opt2 = dp[i-1] + x

dp[i] = min(opt1, opt2)
```

**Answer**: `dp[k-1] / 2` where `k = |diffs|`.

---

## Why x/2 in Option 2?

Op1 costs `x` and fixes **two** positions simultaneously. If we decide to use op1 for `d[i]`, we're "spending" `x/2` for `d[i]` and `x/2` for some other position. Multiplying through by 2 avoids fractions: we track `2×cost`, so `x/2` becomes `x` in the doubled representation.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperations(string s1, string s2, int x) {
        var diffs = new List<int>();
        for (int i = 0; i < s1.Length; i++)
            if (s1[i] != s2[i]) diffs.Add(i);

        if (diffs.Count % 2 != 0) return -1;
        if (diffs.Count == 0) return 0;

        int k = diffs.Count;
        int[] dp = new int[k]; // stores 2× actual cost
        Array.Fill(dp, int.MaxValue / 2);

        for (int i = 0; i < k; i++) {
            // Option 1: pair diffs[i] with diffs[i-1]
            if (i >= 1) {
                int pairCost = Math.Min(diffs[i] - diffs[i-1], x) * 2;
                int prev2 = i >= 2 ? dp[i-2] : 0;
                dp[i] = Math.Min(dp[i], prev2 + pairCost);
            }

            // Option 2: use op1 for diffs[i] (costs x/2 → x in doubled units)
            int prev1 = i >= 1 ? dp[i-1] : 0;
            dp[i] = Math.Min(dp[i], prev1 + x);
        }

        return dp[k-1] / 2;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | One pass to collect diffs, one pass DP |
| **Space** | **O(k)** | k = number of diff positions ≤ n |

## Example Walkthrough

**`s1="1100011000", s2="0101001010", x=2`**

`diffs = [0, 1, 4, 6, 8, 9]` (positions where they differ)

Working in 2× units (dp stores 2×cost):

| i | d[i] | opt1 (pair with prev) | opt2 (op1) | dp[i] |
|---|------|----------------------|------------|-------|
| 0 | 0 | — | 0+2=2 | 2 |
| 1 | 1 | 0+min(1,2)×2=2 | 2+2=4 | **2** |
| 2 | 4 | 2+min(3,2)×2=6 | 2+2=4 | **4** |
| 3 | 6 | 2+min(2,2)×2=6 | 4+2=6 | **6** |
| 4 | 8 | 4+min(2,2)×2=8 | 6+2=8 | **8** |
| 5 | 9 | 6+min(1,2)×2=8 | 8+2=10 | **8** |

Answer: `dp[5] / 2 = 8/2 = 4` ✓

---

## Pitfalls & Edge Cases

- **Odd count of diffs** → -1 immediately
- **Empty diffs** → 0 (already equal)
- **`x` is odd** → work in 2× units, divide at end; result is always integer since `k` is even (each op1 covers 2 positions)
- **`dp[i] = int.MaxValue/2` init** → prevent overflow when adding to `int.MaxValue`

---

## Key Takeaway

"Minimum cost to fix binary string via adjacent/arbitrary flips" → **collect diff positions, DP on consecutive pairing**:

```
dp[i] = min(
    dp[i-2] + min(d[i]-d[i-1], x) * 2,   // pair consecutive
    dp[i-1] + x                             // use op1 for this position (x/2 × 2)
)
answer = dp[k-1] / 2
```

Work in 2× units to handle fractional `x/2` cleanly with integer arithmetic.
