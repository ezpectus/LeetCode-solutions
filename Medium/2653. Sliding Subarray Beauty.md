# 2653. Sliding Subarray Beauty
## O(n × 50) — Sliding Window + Frequency Array

---

## Problem Statement

- For each subarray of size `k`, find its **beauty** = the `x`-th smallest **negative** integer, or `0` if fewer than `x` negatives exist.
- Return the array of beauties for all `n - k + 1` subarrays.
- Constraints: `n ≤ 10⁵`, `-50 ≤ nums[i] ≤ 50`.

---

## Core Idea — Sliding Window + Fixed-Size Frequency Array

**Key insight**: `nums[i] ∈ [-50, 50]` — only 101 distinct values. Instead of sorting the window each time, maintain a **frequency array** of size 101 (offset by +50 so index `i` = value `i-50`). Slide the window: add the new right element, remove the old left element, then scan the frequency array from index 0 (value -50) upward to find the x-th negative in O(50).

**Why O(50) per window?**
Only values `[-50, -1]` (indices 0-49) are negative. Scanning at most 50 entries to find the x-th negative is a constant-time operation regardless of `k` or `n`.

**Offset trick**:
```
nums[i] = v  →  freq[v + 50]++
index i in freq  →  value = i - 50
```

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] GetSubarrayBeauty(int[] nums, int k, int x) {
        int n = nums.Length;
        int[] freq = new int[101]; // index i → value (i - 50)
        int[] res = new int[n - k + 1];

        for (int i = 0; i < n; i++) {
            freq[nums[i] + 50]++;                          // add right element
            if (i >= k) freq[nums[i - k] + 50]--;         // remove left element

            if (i >= k - 1) {
                res[i - k + 1] = XthSmallestNeg(freq, x);
            }
        }

        return res;
    }

    int XthSmallestNeg(int[] freq, int x) {
        int cnt = 0;
        for (int i = 0; i < 50; i++) { // i=0..49 → values -50..-1
            cnt += freq[i];
            if (cnt >= x) return i - 50;
        }
        return 0; // fewer than x negatives
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n × 50)** = O(n) | n windows × O(50) scan; 50 is a constant |
| **Space Complexity** | **O(101)** = O(1) | Fixed-size frequency array |

**Optimal** — O(n) amortized. Sorting per window would be O(n × k log k).

## Frequency Array — Visual

```
Values:  -50  -49  ...  -1   0   1   ...  50
Index:     0    1  ...  49  50  51   ...  100
```

When scanning for x-th smallest negative: only indices `0..49` matter.

## Example Walkthrough

**Example 1**: `nums=[1,-1,-3,-2,3]`, `k=3`, `x=2`

**Window [1,-1,-3]** (i=2):
```
freq: ...[1](-50=0)...[1](-3=47)...[1](-1=49)...[1](+1=51)...
Scan: i=47 → cnt=1, i=49 → cnt=2 ≥ x=2 → return 49-50 = -1 ✓
```

**Window [-1,-3,-2]** (i=3):
```
freq: ...[1](-3=47)...[1](-2=48)...[1](-1=49)...
Scan: i=47 → cnt=1, i=48 → cnt=2 ≥ 2 → return 48-50 = -2 ✓
```

**Window [-3,-2,3]** (i=4):
```
freq: ...[1](-3=47)...[1](-2=48)...[1](+3=53)...
Scan: i=47 → cnt=1, i=48 → cnt=2 ≥ 2 → return -2 ✓
```

Output: `[-1,-2,-2]` ✓

## Pitfalls & Edge Cases

- **Offset**: always `nums[i] + 50` for index, `i - 50` for value — easy to forget the `-50` when reading back
- **Remove before or after?** — remove left element when `i >= k` (not `i >= k-1`); the window is full at `i = k-1`, and the element to remove is `nums[i-k]`
- **Return 0 if < x negatives** — the scan loop naturally falls through to `return 0` if fewer than `x` negative entries are found
- **Non-negative values** — indices 50-100 are never scanned in `XthSmallestNeg`; they're tracked in `freq` but ignored for beauty computation

---

## Key Takeaway

Sliding window + order statistics → **frequency array when value range is small**:

- Constant-range values (`[-50, 50]`) → O(1) space frequency array
- Add/remove in O(1), query x-th smallest in O(range) = O(100) = O(1)
- Replaces sorting (O(k log k) per window) with O(range) scan

Pattern: "x-th smallest in sliding window" with small value range → frequency array + prefix count scan. If range were large, would need a sorted structure (e.g. two heaps or order-statistics tree).
