# 2711. Difference of Number of Distinct Values on Diagonals
## O(m × n) — Two-Pass Per Diagonal

---

## Problem Statement

- For each cell `(r,c)`: compute `|leftAbove - rightBelow|` where:
  - `leftAbove` = count of **distinct** values on the same diagonal, strictly above-left
  - `rightBelow` = count of **distinct** values on the same diagonal, strictly below-right

---

## Core Idea — Process Each Diagonal Twice

All cells on the same diagonal share `r - c = const`. For each diagonal:

**Pass 1 (left→right)**: walk top-left to bottom-right, track a running `HashSet`. At cell `i`, the set contains all values from cells `0..i-1` → that's `leftAbove[i]`. Store `len(set)` in `ans[r][c]`.

**Pass 2 (right→left)**: walk bottom-right to top-left. At cell `i`, the set contains all values from cells `i+1..end` → that's `rightBelow[i]`. Compute `abs(ans[r][c] - len(set))` → final answer.

**Diagonals to process**: start from each cell in the top row `(0, c)` and each cell in the left column `(r, 0)` — covers all diagonals exactly once.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[][] DifferenceOfDistinctValues(int[][] grid) {
        int m = grid.Length, n = grid[0].Length;
        int[][] ans = new int[m][];
        for (int i = 0; i < m; i++) ans[i] = new int[n];

        void ProcessDiagonal(int r0, int c0) {
            var cells = new List<(int r, int c)>();
            for (int r = r0, c = c0; r < m && c < n; r++, c++)
                cells.Add((r, c));

            // Pass 1: leftAbove
            var seen = new HashSet<int>();
            foreach (var (r, c) in cells) {
                ans[r][c] = seen.Count;
                seen.Add(grid[r][c]);
            }

            // Pass 2: rightBelow
            seen.Clear();
            for (int i = cells.Count - 1; i >= 0; i--) {
                var (r, c) = cells[i];
                ans[r][c] = Math.Abs(ans[r][c] - seen.Count);
                seen.Add(grid[r][c]);
            }
        }

        for (int c = 0; c < n; c++) ProcessDiagonal(0, c);
        for (int r = 1; r < m; r++) ProcessDiagonal(r, 0);

        return ans;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(m × n)** | Each cell visited exactly twice (once per pass) |
| **Space Complexity** | **O(min(m,n))** | HashSet per diagonal, max size = diagonal length |

---

## Example Walkthrough

**Main diagonal** `(0,0)→(1,1)→(2,2)` with values `[1,1,1]`:

Pass 1 (leftAbove):
- `(0,0)`: seen=`{}` → `ans=0`, add 1 → seen=`{1}`
- `(1,1)`: seen=`{1}` → `ans=1`, add 1 → seen=`{1}`
- `(2,2)`: seen=`{1}` → `ans=1`, add 1 → seen=`{1}`

Pass 2 (rightBelow):
- `(2,2)`: seen=`{}` → `ans=|1-0|=1`, add 1
- `(1,1)`: seen=`{1}` → `ans=|1-1|=0`, add 1
- `(0,0)`: seen=`{1}` → `ans=|0-1|=1`

Results: `ans[0][0]=1, ans[1][1]=0, ans[2][2]=1` ✓

---

## Pitfalls & Edge Cases

- **Start from both top row AND left column** (but not `(0,0)` twice) — `r=1..m-1` for the left column avoids double-processing the top-left corner
- **`ans[r][c]` used as temp storage** — first pass writes `leftAbove`, second pass overwrites with final `|leftAbove - rightBelow|`; this is safe since we process each cell exactly once per pass
- **Single-cell diagonals** — both passes run once, `ans=|0-0|=0` ✓

---

## Key Takeaway

"Diagonal prefix/suffix distinct counts" → **two-pass per diagonal with a HashSet**:

```
Pass 1 (→): ans[r][c] = |seen before|     (leftAbove)
Pass 2 (←): ans[r][c] = |leftAbove - |seen after||  (final)
```

Same pattern as prefix/suffix arrays but with a set for distinct counting instead of a sum.
