# 2870. Minimum Number of Operations to Make Array Empty
## O(n) — Greedy: ceil(freq / 3) Per Value

---

## Problem Statement

- Delete elements in groups of 2 or 3 (same value only).
- Return minimum operations to empty the array, or -1 if impossible.

---

## Core Idea — Handle Each Value Independently

Values don't interact across different elements, so handle each unique value separately. For a value with frequency `f`:

- If `f == 1`: impossible — can't form a group of 2 or 3 → return -1.
- Otherwise: minimize number of groups of 2 and 3 that sum to `f`.

**Why `ceil(f / 3)`?**

We want to partition `f` into groups of 2 and 3, minimizing count. Groups of 3 are more efficient (remove 3 in 1 op vs 3 in 1.5 ops). Use as many 3s as possible:

| f | optimal partition | ops |
|---|-------------------|-----|
| 2 | 2 | 1 |
| 3 | 3 | 1 |
| 4 | 2+2 | 2 |
| 5 | 3+2 | 2 |
| 6 | 3+3 | 2 |
| 7 | 3+2+2 | 3 |

Pattern: `ceil(f / 3)`. This works because:
- `f = 3k` → k groups of 3
- `f = 3k+1` → `(k-1)` groups of 3 + 2 groups of 2 = k+1 ops = `ceil((3k+1)/3)` ✓
- `f = 3k+2` → k groups of 3 + 1 group of 2 = k+1 ops = `ceil((3k+2)/3)` ✓

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperations(int[] nums) {
        var freq = new Dictionary<int, int>();
        foreach (int x in nums)
            freq[x] = freq.GetValueOrDefault(x) + 1;

        int res = 0;
        foreach (int f in freq.Values) {
            if (f == 1) return -1;
            res += (f + 2) / 3; // ceil(f/3) using integer arithmetic
        }
        return res;
    }
}
```

## Example Walkthrough

**`nums=[2,3,3,2,2,4,2,3,4]`**

Frequencies: `{2:4, 3:3, 4:2}`

- `f=4`: `ceil(4/3)=2` ops (2+2)
- `f=3`: `ceil(3/3)=1` op (3)
- `f=2`: `ceil(2/3)=1` op (2)

Total = 2+1+1 = **4** ✓

---

## Pitfalls & Edge Cases

- **`f == 1`**: only this case is impossible — no group of 2 or 3 can use exactly 1 element
- **`ceil(f/3)` in integer arithmetic**: `(f + 2) / 3` avoids floating point
- **`f == 2` special case not needed**: `ceil(2/3) = 1` ✓ via formula

---

## Key Takeaway

"Minimum groups of 2 or 3 to partition f elements" → **`ceil(f/3)`** — greedy use of triples, remainder handled by pairs. Only `f=1` is impossible.
