# 3020. Find the Maximum Number of Elements in Subset
## O(n log(max_val)) — Greedy Chain Squaring per Starting Value

---

## Problem Statement

- Find the largest subset arrangeable as `[x, x², x⁴, ..., x^(k/2), x^k, x^(k/2), ..., x², x]` where `k` is a power of 2 (or just `[x]` for length 1).
- The pattern is a palindrome with the center element appearing once and all others appearing twice.

---

## Core Idea — Fix Starting Value, Square Until Chain Breaks

For each distinct value `x` in the array, try to build the longest chain `x → x² → x⁴ → ...`:
- Each level `x^(2^i)` must appear **at least twice** (once on each side of the palindrome).
- The **center** (highest power reached) needs only **one** occurrence.
- Length = `2 × (number of levels with freq ≥ 2) + (1 if center exists)`.

**Special case — `x = 1`**: Since `1^(2^m) = 1` for all `m`, the chain is all 1s. Any odd number of 1s forms a valid pattern. So take `freq[1]` if odd, else `freq[1] - 1`.

**Overflow guard**: since `nums[i] ≤ 10⁹`, squaring past `~31622` gives values larger than `10⁹` which can't appear in the array. Once we compute `cur * cur > 10⁹`, we can break.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaximumLength(int[] nums) {
        var freq = new Dictionary<long, int>();
        foreach (int x in nums)
            freq[x] = freq.GetValueOrDefault(x) + 1;

        int res = 1;

        // Special case: x = 1
        if (freq.ContainsKey(1)) {
            int c = freq[1];
            res = Math.Max(res, c % 2 == 1 ? c : c - 1);
        }

        foreach (long x in freq.Keys) {
            if (x == 1) continue;
            int length = 0;
            long cur = x;

            while (freq.ContainsKey(cur) && freq[cur] >= 2) {
                length += 2;
                if (cur > 31622) { cur = cur * cur; break; } // next will overflow
                cur = cur * cur;
            }

            // Pattern must have odd length → center is mandatory.
            // If center value exists: add it. Otherwise: remove outermost pair (length-1).
            if (freq.ContainsKey(cur) && freq[cur] >= 1) length++;
            else if (length > 0) length--; // drop one from the pair → still odd length

            res = Math.Max(res, length);
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n log(max_val))** | n distinct values × at most log₂(log₂(10⁹)) ≈ 5 squarings per chain |
| **Space** | **O(n)** | Frequency map |

At most ~5 squarings per chain (since squaring doubles the exponent and 2^5=32 > log₂(10⁹)≈30).

---

## Pattern Structure Explained

For `k = 2^m`:

| m | k | Pattern | Length |
|---|---|---------|--------|
| 0 | 1 | `[x]` | 1 |
| 1 | 2 | `[x, x², x]` | 3 |
| 2 | 4 | `[x, x², x⁴, x², x]` | 5 |
| 3 | 8 | `[x, x², x⁴, x⁸, x⁴, x², x]` | 7 |

Length is always odd: `2m + 1`. Each element except center appears exactly twice.

## Example Walkthrough

**`nums = [5,4,1,2,2]`**

freq: {5:1, 4:1, 1:1, 2:2}

- x=1: count=1, odd → res = max(1,1) = 1
- x=2: cur=2, freq[2]=2 ≥ 2 → length=2, cur=4. freq[4]=1 < 2 → stop while loop. freq[4] exists → length+=1 = 3.
- x=4: cur=4, freq[4]=1 < 2 → stop. freq[4] exists → length=0+1=1.
- x=5: cur=5, freq[5]=1 < 2 → stop. freq[5] exists → length=1.

res = **3** ✓ → subset {2,2,4} arranged as [2,4,2]

---

**`nums = [2,2,4,4,16,16,256]`**

freq: {2:2, 4:2, 16:2, 256:1}

x=2: cur=2→freq=2≥2, length=2, cur=4. cur=4→freq=2≥2, length=4, cur=16. cur=16→freq=2≥2, length=6, cur=256. cur=256→freq=1<2, stop. freq[256] exists → length=7.

res = **7** ✓ → [2,4,16,256,16,4,2]

## Pitfalls & Edge Cases

- **`x = 1` special case**: chain never grows (1²=1), so any count of 1s is valid but length must be odd (palindrome center = 1). Take `freq[1] - (freq[1] % 2 == 0 ? 1 : 0)`.
- **Overflow**: `cur > 31622` means `cur² > 10⁹`. After breaking, check if the overflowed `cur` exists in freq (it won't, since `nums[i] ≤ 10⁹`) → no center added. Use `long` to avoid overflow on the multiplication itself.
- **Starting from small values only**: each distinct value in freq is tried as a chain start. Values that are `x²` for some other `x` will also be tried as starts, giving shorter results (dominated by the longer chain starting from `x`).
- **Minimum answer = 1**: always exists since we can pick any single element.

---

## Key Takeaway

"Largest palindrome-like subset with powers-of-square pattern" → **for each starting value x, greedily chain x → x² → x⁴ → ... while freq ≥ 2; add 1 for center**:

```
for x in freq (x ≠ 1):
    length = 0
    cur = x
    while freq[cur] >= 2: length += 2; cur = cur²
    if cur in freq: length += 1
    res = max(res, length)

for x = 1: res = max(res, freq[1] if odd else freq[1]-1)
```

Chain length is at most ~5 steps (since squaring grows exponentially), making the algorithm effectively O(n).
