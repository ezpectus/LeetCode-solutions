# 2567. Minimum Score by Changing Two Elements  
*O(n log n) — Sorting + Greedy Window*

---

## Problem Statement

Given an integer array `nums` of length at least 3.

The **score** of the array is defined as:
- **low score** = minimum absolute difference between any two elements  
- **high score** = maximum absolute difference between any two elements  
- **score** = low score + high score

You can **change exactly two elements** to **any values** (positive or negative integers allowed).

Return the **minimum possible score** after exactly two changes.

**Examples**:

**Example 1**  
Input: nums = [1,4,7,8,5]  
Output: **3**  
Change 1 and 4 to 6 → [6,6,7,8,5]  
low = 0 (|6-6|), high = 3 (|8-5|), score = **3**

**Example 2**  
Input: nums = [1,4,3]  
Output: **0**  
Change 4 and 3 to 1 → [1,1,1]  
low = 0, high = 0, score = **0**

**Constraints**:
- 3 ≤ nums.length ≤ 10⁵
- 1 ≤ nums[i] ≤ 10⁹

---

## Core Idea — Sort + Minimize (max - min) After Removing Two Extremes

**Key insight**:
- After changing **exactly two elements**, the final array will have some **minimum** and **maximum**.
- The score = (final max - final min) + (smallest pairwise diff)
- But smallest pairwise diff is **hard** to control → usually 0 is achievable by making two elements equal.
- In optimal, we can **always** make low score = 0 by changing two elements to the same value (or matching existing ones).
- Thus, the problem reduces to **minimizing the high score** (final max - final min) after changing two elements.

**To minimize (max - min)** after changing two values:
- Sort the array: nums[0] ≤ nums[1] ≤ ... ≤ nums[n-1]
- The final min and max will be among the remaining n-2 elements after removing/changing two.
- Possible ways to choose the remaining range:
  1. Remove two smallest → final range: nums[2] to nums[n-1] → diff = nums[n-1] - nums[2]
  2. Remove two largest → final range: nums[0] to nums[n-3] → diff = nums[n-3] - nums[0]
  3. Remove one smallest and one largest → final range: nums[1] to nums[n-2] → diff = nums[n-2] - nums[1]

- The **minimum** among these three differences is the answer (since we can always achieve low score = 0).

**Time**: O(n log n) for sorting  
**Space**: O(1) extra (in-place sort)

---

## Clean Implementation (C#)

```csharp
public class Solution {
    public int MinimizeSum(int[] nums) {
        int n = nums.Length;
        Array.Sort(nums);
        
        // Three possible ways to choose the remaining range after changing two elements:
        // 1. Remove two smallest → range: nums[2] ... nums[n-1]
        // 2. Remove two largest → range: nums[0] ... nums[n-3]
        // 3. Remove one smallest and one largest → range: nums[1] ... nums[n-2]
        
        int opt1 = nums[n-1] - nums[2];
        int opt2 = nums[n-3] - nums[0];
        int opt3 = nums[n-2] - nums[1];
        
        return Math.Min(Math.Min(opt1, opt2), opt3);
    }
}
```

| **Space Complexity**  | **O(1)**      | In-place sorting (Array.Sort modifies the input array)<br>No extra space proportional to n is used |

**Optimal** — sorting is necessary in the general case to find extremes efficiently. Very fast for n ≤ 10⁵.

## Why This Works — Example Walkthrough

**Example 1**: nums = [1,4,7,8,5]

**After sorting**: [1, 4, 5, 7, 8]

**Three possible ranges after changing/removing two elements**:

1. Remove two smallest (1 and 4) → remaining [5, 7, 8] → max - min = 8 - 5 = **3**
2. Remove two largest (7 and 8) → remaining [1, 4, 5] → max - min = 5 - 1 = **4**
3. Remove one smallest + one largest (1 and 8) → remaining [4, 5, 7] → max - min = 7 - 4 = **3**

**Minimum among the three** = **3** → correct answer

**Example 2**: nums = [1,4,3]

**After sorting**: [1, 3, 4]

1. Remove two smallest (1,3) → remaining [4] → max - min = **0**
2. Remove two largest (3,4) → remaining [1] → max - min = **0**
3. Remove one smallest + one largest (1,4) → remaining [3] → max - min = **0**

**Minimum** = **0** → correct

**Correct** —  
After changing **exactly two elements**, the final **minimum** and **maximum** of the array will come from the **remaining original values** (the two changed elements can be set to anything, including matching existing ones).  
To minimize (final max - final min), we should **exclude the two most extreme values** in a way that shrinks the range as much as possible.  
The three cases cover all meaningful ways of removing two outliers:
- Both from the left (two smallest)
- Both from the right (two largest)
- One from each end

We can **always** achieve **low score = 0** (minimum pairwise difference) by setting the two changed values appropriately (e.g., make them equal to some remaining value or to each other).  
Therefore, the total score is minimized by minimizing just the high score (max - min) among the three options.

---

## Key Takeaway

This is a **beautiful greedy sorting + case analysis** problem:

* Sort the array in non-decreasing order
* After changing **two elements**, the final min and max will be among the **original values** (except the two changed ones)
* To minimize (final max - final min), consider the three ways to exclude two extreme values:
  1. Exclude two smallest → range becomes nums[2] to nums[n-1]
  2. Exclude two largest → range becomes nums[0] to nums[n-3]
  3. Exclude one smallest and one largest → range becomes nums[1] to nums[n-2]
* Take the **minimum** of these three differences (nums[right] - nums[left])
* We can **always** make the low score (minimum pairwise difference) = 0

**Pure, clean, optimal** — O(n log n) time, O(1) extra space, extremely fast, very intuitive, and correctly minimizes the range (high score) after two changes.
