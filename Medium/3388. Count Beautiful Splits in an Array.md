# 3388. Count Beautiful Splits in an Array
## O(nÂ²) â€” LCP Table via DP + O(nÂ²) Enumeration

---

## Problem Statement

- Split `nums` into `nums1`, `nums2`, `nums3` (all non-empty, in order).
- **Beautiful** if: `nums1` is a prefix of `nums2` OR `nums2` is a prefix of `nums3`.
- Count all beautiful splits.

---

## Core Idea â€” Precompute LCP Table, Then Enumerate Splits

**LCP table**: `lcp[i][j]` = length of the longest common prefix of `nums[i:]` and `nums[j:]`.

Computed bottom-up in O(nÂ²):
```
lcp[i][j] = lcp[i+1][j+1] + 1   if nums[i] == nums[j]
           = 0                    otherwise
```

**Split enumeration**: split points `i` (end of nums1) and `j` (end of nums2), where `1 â‰¤ i < j < n`.

- `len1 = i`, `len2 = j-i`, `len3 = n-j`
- **Condition 1** (nums1 is prefix of nums2): `len1 â‰¤ len2` AND `lcp[0][i] â‰¥ len1`
- **Condition 2** (nums2 is prefix of nums3): `len2 â‰¤ len3` AND `lcp[i][j] â‰¥ len2`

Count all `(i,j)` pairs where condition 1 OR condition 2 holds.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int BeautifulSplits(int[] nums) {
        int n = nums.Length;

        // lcp[i][j] = LCP length of nums[i:] and nums[j:]
        int[,] lcp = new int[n+1, n+1];
        for (int i = n-1; i >= 0; i--)
            for (int j = n-1; j >= 0; j--)
                lcp[i,j] = nums[i]==nums[j] ? lcp[i+1,j+1]+1 : 0;

        int res = 0;
        for (int i = 1; i < n-1; i++) {
            for (int j = i+1; j < n; j++) {
                int len1 = i, len2 = j-i, len3 = n-j;
                bool cond1 = len1 <= len2 && lcp[0,i] >= len1;
                bool cond2 = len2 <= len3 && lcp[i,j] >= len2;
                if (cond1 || cond2) res++;
            }
        }
        return res;
    }
}
```

## Why Bottom-Up LCP DP Works

The LCP of `nums[i:]` and `nums[j:]` satisfies:
- If `nums[i] == nums[j]`: LCP = 1 + LCP(`nums[i+1:]`, `nums[j+1:]`) = `lcp[i+1][j+1] + 1`.
- If `nums[i] != nums[j]`: LCP = 0.

Processing `i` and `j` from `n-1` down to `0` (right to left) ensures `lcp[i+1][j+1]` is already computed when we need it.

## Pitfalls & Edge Cases

- **Both conditions must be OR**: a split is counted once even if both conditions hold simultaneously.
- **`lcp[i][j]` bounds**: accessing `lcp[i+1][j+1]` at `i=n-1` or `j=n-1` uses the `n+1`-sized table with `lcp[n][*] = 0` as base case.
- **Non-empty subarrays**: `i` starts at 1 (nums1 â‰¥ 1 element), `j` goes up to `n-1` (nums3 â‰¥ 1 element).

## Key Takeaway

"Count splits where one part is prefix of another" â†’ **precompute LCP table bottom-up in O(nÂ²), then O(1) per split pair**:

```
lcp[i][j] = lcp[i+1][j+1]+1 if nums[i]==nums[j], else 0
for i, j: check lcp[0][i] >= len1 or lcp[i][j] >= len2
```

The O(nÂ²) LCP precomputation converts O(n) prefix comparisons into O(1) lookups.
