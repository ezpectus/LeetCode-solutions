# 3035. Maximum Palindromes After Operations
## O(n log n + total_chars) — Count Global Pairs, Greedy Fill Shortest Words

---

## Problem Statement

- Swap any characters freely between any words (any number of times).
- Maximize the number of words that are palindromes.

---

## Core Idea — Global Pair Pool + Greedy by Length

**Key observation**: since we can swap any character anywhere, the positions don't matter — only the global multiset of characters matters. We can redistribute freely.

**Palindrome requirement**: a word of length `L` needs `floor(L/2)` pairs of identical characters (the middle character of an odd-length word can be anything).

So the problem reduces to:
1. Count total **pairs** available: `pairs = Σ floor(freq[c] / 2)` for each character `c`.
2. Each word of length `L` needs `floor(L/2)` pairs to become a palindrome.
3. **Greedy**: sort words by length ascending, fill shortest words first (they need fewest pairs). Count how many words can be satisfied.

**Why greedy works**: satisfying shorter words uses fewer pairs, leaving more pairs for additional words. Skipping a shorter word to satisfy a longer one never helps.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxPalindromesAfterOperations(string[] words) {
        // Count global character frequencies
        int[] freq = new int[26];
        foreach (string w in words)
            foreach (char c in w)
                freq[c - 'a']++;

        // Total pairs available
        int pairs = 0;
        foreach (int f in freq) pairs += f / 2;

        // Sort word lengths ascending
        int[] lengths = words.Select(w => w.Length).OrderBy(x => x).ToArray();

        int res = 0;
        foreach (int len in lengths) {
            int needed = len / 2;
            if (pairs >= needed) {
                pairs -= needed;
                res++;
            } else {
                break; // can't satisfy this word or any longer one
            }
        }

        return res;
    }
}
```

## Why Character Positions Don't Matter

Since we can swap any character `words[i][x]` with any `words[j][y]`, and there's no cost, we effectively have a global pool of characters to redistribute freely. The only constraint is we must distribute them back into words of the same lengths.

**Palindrome structure for a word of length L**:
- Needs `floor(L/2)` pairs to fill symmetric positions.
- Odd-length words get one "free" middle character (can be any singleton).
- So: word of length L is achievable as palindrome iff we can assign `floor(L/2)` pairs to it.

## Pitfalls & Edge Cases

- **`break` after first failure**: once pairs are insufficient for the current (shortest remaining) length, no longer word can be satisfied either → stop immediately
- **Odd-length words**: the middle character costs no pair — only `floor(len/2)` pairs needed, not `ceil`
- **All words length 1**: each needs 0 pairs → all can be palindromes → result = n
- **Single character pairs**: a character appearing twice contributes 1 pair; appearing 3 times still just 1 pair (the odd one is unused but could be a middle character)

---

## Key Takeaway

"Max palindromes by free character swaps" → **global pair count + greedy fill by ascending length**:

```
pairs = Σ freq[c] / 2   (total character pairs available globally)
for length in sorted_lengths:
    if pairs >= length/2: pairs -= length/2; count++
    else: break
```

Characters are fungible across words → reduces to a pure counting problem. Greedy by length is optimal since shorter words are cheaper to palindromize.
