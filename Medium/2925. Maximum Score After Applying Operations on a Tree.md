# 2925. Maximum Score After Applying Operations on a Tree
## O(n) — DFS with Subtree Sum / "Sacrifice One per Path"

---

## Problem Statement

- Undirected tree with `n` nodes, rooted at node `0`.
- Each node `i` has value `values[i]`.
- Operation: pick node `i`, add `values[i]` to score, set `values[i] = 0`.
- A tree is **healthy** if the sum of values on the path from root to **every** leaf is non-zero.
- Return the **maximum score** after any number of operations keeping the tree healthy.

---

## Core Idea — "Sacrifice" One Node Per Root-to-Leaf Path

**Key insight**: the healthy condition requires that on every root-to-leaf path, at least one node keeps its original value (non-zero). So for every path from root to leaf, we must **leave at least one node untouched**.

We want to **collect as much as possible**, which means we want to **sacrifice as little as possible** — i.e., leave behind the minimum value node on each path.

This leads to the reformulation:

> **Score = totalSum − minimumSacrifice**
>
> where `minimumSacrifice` = minimum total values we must leave behind to keep the tree healthy.

**What must be left behind?** For each root-to-leaf path, exactly one node's value must remain. But paths share prefixes — leaving one ancestor node "covers" all paths through it.

**DFS subproblem**: define `Loss(node)` = minimum values we must sacrifice in the subtree rooted at `node` to keep all paths through it healthy.

For a **leaf** node: we must leave the leaf itself → `Loss(leaf) = values[leaf]`

For an **internal** node with children `c1, c2, ..., ck`:
- **Option A**: sacrifice the current node itself (leave `values[node]` behind). This covers ALL paths through node at once → `Loss = values[node]`
- **Option B**: sacrifice independently in each child's subtree → `Loss = sum of Loss(ci) for all children`
- Take the minimum: `Loss(node) = min(values[node], sum(Loss(children)))`

**Answer**: `totalSum − Loss(root)`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaximumScoreAfterOperations(int[][] edges, int[] values) {
        int n = values.Length;
        List<int>[] adj = new List<int>[n];
        for (int i = 0; i < n; i++) adj[i] = new List<int>();
        foreach (var e in edges) {
            adj[e[0]].Add(e[1]);
            adj[e[1]].Add(e[0]);
        }

        long totalSum = values.Sum(v => (long)v);
        long minLoss = DFS(0, -1, adj, values);
        return totalSum - minLoss;
    }

    private long DFS(int node, int parent, List<int>[] adj, int[] values) {
        // Leaf node — must sacrifice itself
        if (adj[node].Count == 1 && node != 0)
            return values[node];

        long childrenLoss = 0;
        foreach (int child in adj[node]) {
            if (child == parent) continue;
            childrenLoss += DFS(child, node, adj, values);
        }

        // Option A: sacrifice this node (covers all paths through it)
        // Option B: sacrifice independently in each child subtree
        return Math.Min(values[node], childrenLoss);
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Single DFS traversal, each node visited once |
| **Space** | **O(n)** | Adjacency list + recursion stack depth O(n) worst case |

---

## Why This Works — The Sacrifice Argument

**Why does one untouched node per path suffice?**

The healthy condition is: for every leaf, the path root→leaf has sum > 0. Since all `values[i] ≥ 1`, leaving any single node untouched on a path guarantees sum ≥ 1 > 0.

**Why `min(values[node], sum(Loss(children)))`?**

- If we **sacrifice node** itself: cost is `values[node]`, and all paths through this node are covered — we can collect everything in all subtrees below.
- If we **don't sacrifice node**: each child subtree must independently ensure its paths are covered, costing `Loss(c1) + Loss(c2) + ...`
- We pick whichever option is cheaper.

**Why is this globally optimal?**

The subproblems are independent given the tree structure — the loss in one subtree doesn't affect another. DFS bottom-up correctly propagates the minimum sacrifice up the tree.

## Example Walkthrough

**`edges=[[0,1],[0,2],[0,3],[2,4],[4,5]], values=[5,2,5,2,1,1]`**

Tree structure:
```
        0(5)
       /|\ 
    1(2) 2(5) 3(2)
         |
        4(1)
         |
        5(1)
```

DFS bottom-up:

| Node | Type | Loss computation | Loss |
|------|------|-----------------|------|
| 1 | leaf | `values[1] = 2` | **2** |
| 3 | leaf | `values[3] = 2` | **2** |
| 5 | leaf | `values[5] = 1` | **1** |
| 4 | internal | `min(values[4]=1, Loss[5]=1) = min(1,1)` | **1** |
| 2 | internal | `min(values[2]=5, Loss[4]=1) = min(5,1)` | **1** |
| 0 | root | `min(values[0]=5, Loss[1]+Loss[2]+Loss[3]=2+1+2=5) = min(5,5)` | **5** |

totalSum = 5+2+5+2+1+1 = **16**
Answer = 16 − 5 = **11** ✓

        /     \
      1(10)   2(9)
      / \     / \
   3(7) 4(4) 5(3) 6(5)

DFS bottom-up:

| Node | Loss computation | Loss |
|------|-----------------|------|
| 3 | leaf → `7` | **7** |

| 5 | leaf → `3` | **3** |

| 2 | `min(9, 3+5) = min(9,8)` | **8** |

totalSum = 20+10+9+7+4+3+5 = **58**
Answer = 58 − 18 = **40** ✓

## Pitfalls & Edge Cases

- **Leaf detection**: `adj[node].Count == 1 && node != 0` — root with one child has degree 1 but is not a leaf
- **Use `long`** — values up to `10^9`, summed over `n = 2×10^4` → up to `2×10^13`, overflows `int`
- **Parent tracking in DFS**: tree is given as undirected adjacency list — pass `parent` to avoid going back up
- **Option A vs B**: even if `values[node]` is large, it might still be cheaper than the sum of all children's losses (e.g., a node with many expensive-to-cover children)

---

## Key Takeaway

"Keep at least one node per root-to-leaf path" → **minimize sacrifice, maximize collection**:

```
totalSum − minLoss(root)

Loss(leaf)     = values[leaf]
Loss(internal) = min(values[node], sum(Loss(children)))
```

Sacrifice the cheapest option at each node: either the node itself (covers all paths below at once) or independently cover each child subtree. DFS bottom-up in O(n).
