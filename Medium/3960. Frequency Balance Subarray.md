# 3960. Frequency Balance Subarray
## O(n²) — Sliding Window with Frequency-of-Frequency Map

---

## Problem Statement

- Subarray is "frequency balanced" iff:
  - Exactly 1 distinct value: always valid.
  - 2+ distinct values: all must appear either `f` or `2f` times (for some f > 0), AND both `f` AND `2f` must appear as actual frequencies.
- Return the length of the longest such subarray.

---

## Core Insight — Valid iff (1 distinct value) OR (2 distinct frequencies where max = 2×min)

For a subarray to satisfy the constraint (2+ distinct values):
- The set of distinct frequency values must be exactly `{f, 2f}` — two distinct values where one is double the other.
- A frequency set of size 1 (all values have the same count) is INVALID for 2+ distinct values (needs both f and 2f, but only f is present).
- A frequency set of size 3+ is clearly invalid.

So: valid iff `distinct_values == 1` OR `distinct_freq_count == 2 AND max_freq == 2 × min_freq`.

---

## Implementation — Track Frequency-of-Frequency Map Incrementally

For each left endpoint `l`, extend right endpoint `r`:
- Maintain `freq[val]` (occurrence count of each distinct value).
- Maintain `freqCount[f]` (number of distinct values with frequency f).

When extending to `r`:
1. Decrement `freqCount[oldFreq]`, increment `freqCount[newFreq]` → O(1) update.
2. Check validity using `freqCount.Count` and its (at most 2) keys → O(1).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int GetLength(int[] nums) {
        int n = nums.Length;
        int best = 1;

        for (int l = 0; l < n; l++) {
            var freq = new Dictionary<int, int>();
            var freqCount = new Dictionary<int, int>(); // freq_value → how many vals have it

            for (int r = l; r < n; r++) {
                int v = nums[r];
                freq.TryGetValue(v, out int oldF);
                int newF = oldF + 1;
                freq[v] = newF;

                if (oldF > 0) {
                    freqCount[oldF]--;
                    if (freqCount[oldF] == 0) freqCount.Remove(oldF);
                }
                freqCount.TryGetValue(newF, out int cur);
                freqCount[newF] = cur + 1;

                bool valid = false;
                int distinctVals = freq.Count;
                int distinctFreqs = freqCount.Count;

                if (distinctVals == 1) {
                    valid = true;
                } else if (distinctFreqs == 2) {
                    int f1 = 0, f2 = 0;
                    foreach (int f in freqCount.Keys) {
                        if (f1 == 0) f1 = f;
                        else f2 = f;
                    }
                    int lo = Math.Min(f1, f2), hi = Math.Max(f1, f2);
                    valid = hi == 2 * lo;
                }

                if (valid) best = Math.Max(best, r - l + 1);
            }
        }

        return best;
    }
}
```

## Example Walkthrough

**Example 1:** `nums = [1,2,2,1,2,3,3,3]`

l=1 (num=2), extending right:

| r | subarray | freq | freqCount | valid? |
|---|----------|------|-----------|--------|
| 1 | [2] | {2:1} | {1:1} | 1 distinct val ✓ |
| 2 | [2,2] | {2:2} | {2:1} | 1 distinct val ✓ |
| 3 | [2,2,1] | {2:2,1:1} | {2:1,1:1} | 2 freqs, 2=2×1 ✓ len=3 |
| 4 | [2,2,1,2] | {2:3,1:1} | {3:1,1:1} | 2 freqs, 3≠2×1 ✗ |
| 5 | [2,2,1,2,3] | {2:3,1:1,3:1} | {3:1,1:2} | 2 freqs, 3≠2×1 ✗ |

Hmm, that doesn't find length 5 with l=1. Let me check l=1 r=5: subarray [2,2,1,2,3] freq={2:3,1:1,3:1}, freqCount={3:1,1:2}. 3≠2×1. Not valid.

Let me try l=1, r=5 is [2,2,1,2,3]. l=2?

Actually checking [2,1,2,3,3] = indices 1..5 (0-based). Let me redo with l=1:

r=5 (num=3): subarray [2,2,1,2,3] → as above, invalid.

Try l=2 (start at index 2=second 2):
- r=2: [2] ✓
- r=3: [2,1] freq{2:1,1:1} freqCount{1:2} → 2 distinct vals, 1 distinct freq → invalid.
- r=4: [2,1,2] freq{2:2,1:1} freqCount{2:1,1:1} → 2 freqs, 2=2×1 ✓ len=3.
- r=5: [2,1,2,3] freq{2:2,1:1,3:1} freqCount{2:1,1:2} → max=2,min=1, 2=2×1 ✓ len=4.
- r=6: [2,1,2,3,3] freq{2:2,1:1,3:2} freqCount{2:2,1:1} → max=2,min=1, 2=2×1 ✓ len=**5** ✓

**Output:** 5 ✓

**Example 3:** `nums = [1,2,3,4]`

Any 2+ element subarray: all distinct values appear exactly once → freqCount={1:k}. For 2+ distinct vals, distinctFreqs=1 → invalid. **Output: 1** ✓

---

## Pitfalls & Edge Cases

- **distinctFreqs=1 with distinctVals>1:** all elements appear the same number of times but we need BOTH f and 2f — invalid. ✓

- **k=0 (max = n*(n-1)/2):** since k doesn't bound the cost here (this problem has no k!), note the solution just checks structure. ✓

- **Iterating freqCount.Keys when size=2:** the `foreach` body runs exactly twice — O(1). ✓

---

## Key Takeaway

Maintain a "frequency of frequencies" map incrementally during O(n²) sliding window. The validity check reduces to: does the frequency multiset have ≤ 2 distinct values with one being double the other? Each step costs O(1).
