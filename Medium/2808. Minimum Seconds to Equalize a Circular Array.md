# 2808. Minimum Seconds to Equalize a Circular Array
## O(n) — Max Circular Gap Between Occurrences ÷ 2

---

## Problem Statement

- Each second, every element can copy from its left or right neighbor.
- Return minimum seconds to make all elements equal.

---

## Core Idea — Value Spreads from Its Positions

Each value can "spread" outward from all its positions simultaneously. In `t` seconds, a value at position `p` covers positions `[p-t, p+t]` (circular). To cover the entire array, we need `t` large enough that every gap between consecutive occurrences of that value is bridged.

**For a value with occurrences at positions `p₀, p₁, ..., pₖ₋₁`** (sorted, circular):
- The maximum gap between consecutive occurrences determines the hardest region to fill
- To fill a gap of size `g`, the value needs to spread from both ends: `t = g // 2`
- (If `g` is odd, one side has `⌊g/2⌋` and the other `⌈g/2⌉` — `t = g // 2` suffices since adjacent indices can share)

**Best value to spread** = the one with the smallest maximum gap.

**Answer**: `min over all values of (max_circular_gap // 2)`

**Edge case**: single occurrence → the gap "wraps around" = `n`, so `n // 2`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumSeconds(IList<int> nums) {
        int n = nums.Count;
        var pos = new Dictionary<int, List<int>>();
        for (int i = 0; i < n; i++) {
            if (!pos.ContainsKey(nums[i])) pos[nums[i]] = new List<int>();
            pos[nums[i]].Add(i);
        }

        int res = n;
        foreach (var indices in pos.Values) {
            int m = indices.Count;
            int maxGap = 0;
            for (int i = 0; i < m; i++) {
                int nxt = indices[(i+1) % m];
                int cur = indices[i];
                int gap = (nxt - cur + n) % n;
                if (gap == 0) gap = n; // single element or full wrap
                maxGap = Math.Max(maxGap, gap);
            }
            res = Math.Min(res, maxGap / 2);
        }
        return res;
    }
}
```

---

## Example Walkthrough

**`nums=[2,1,3,3,2]`**, n=5

- `pos[2] = [0,4]`: gaps = `(4-0)%5=4`, `(0-4+5)%5=1` → max=4 → **4//2=2**
- `pos[1] = [1]`: single → gap=5 → **5//2=2**
- `pos[3] = [2,3]`: gaps = `(3-2)%5=1`, `(2-3+5)%5=4` → max=4 → **4//2=2**

`min(2,2,2) = 2` ✓

---

## Pitfalls & Edge Cases

- **Single occurrence**: `(i+1)%m == i` when `m=1`, so `(nxt-cur)%n = 0` → must set `gap = n`
- **All same value**: max gap between consecutive identical elements = 1 (or 0 if adjacent) → answer 0 ✓
- **Circular gap**: `(nxt - cur + n) % n` handles wrap-around (e.g. index 4 → index 0)

---

## Key Takeaway

"Min time for one value to fill array by spreading" = `max_circular_gap // 2`. The answer is the minimum of this over all values — we choose the value with the smallest hardest-to-fill gap.
