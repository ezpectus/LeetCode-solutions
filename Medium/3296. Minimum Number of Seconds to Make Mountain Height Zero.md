# 3296. Minimum Number of Seconds to Make Mountain Height Zero
## O(n log(max_T)) — Binary Search on Time

---

## Problem Statement

- Workers work in parallel. Worker `i` reducing height by `x` costs `workerTimes[i] * x*(x+1)/2` seconds.
- Return the minimum time T such that all workers together reduce total height by `mountainHeight`.

---

## Core Idea — Binary Search on Answer T

**Monotonicity**: if all workers together can reduce total height ≥ `mountainHeight` in time T, they can also do it in any T' > T. So the predicate `feasible(T)` is monotone → **binary search**.

**Given time T, how much can worker i reduce?**

Worker `i` reducing by `x` costs:
```
workerTimes[i] * 1 + workerTimes[i] * 2 + ... + workerTimes[i] * x
= workerTimes[i] * x*(x+1)/2
```

So we need: `w * x*(x+1)/2 ≤ T` → `x*(x+1) ≤ 2T/w`

Solving the quadratic `x² + x - 2T/w ≤ 0`:
```
x = floor( (-1 + sqrt(1 + 8T/w)) / 2 )
```

**Feasibility check**: sum of `maxReduction(w, T)` over all workers ≥ `mountainHeight`.

**Search bounds**:
- `lo = 1`
- `hi = max(workerTimes) * mountainHeight*(mountainHeight+1)/2` — worst case: slowest worker does everything alone

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinNumberOfSeconds(int mountainHeight, int[] workerTimes) {
        long lo = 1;
        long hi = (long)workerTimes.Max() * mountainHeight * (mountainHeight + 1) / 2;

        while (lo < hi) {
            long mid = (lo + hi) / 2;
            if (Feasible(mid, mountainHeight, workerTimes))
                hi = mid;
            else
                lo = mid + 1;
        }
        return lo;
    }

    bool Feasible(long T, int mountainHeight, int[] workerTimes) {
        long total = 0;
        foreach (int w in workerTimes) {
            // x*(x+1)/2 <= T/w  →  x = floor((-1 + sqrt(1 + 8T/w)) / 2)
            long x = (long)((-1 + Math.Sqrt(1 + 8.0 * T / w)) / 2);
            total += x;
            if (total >= mountainHeight) return true; // early exit
        }
        return total >= mountainHeight;
    }
}
```

## Deriving the Formula

Worker `i` needs `w * x*(x+1)/2 ≤ T` seconds to reduce by `x`.

Solve for max `x`:
```
x*(x+1) ≤ 2T/w
x² + x - 2T/w ≤ 0

Quadratic formula: x = (-1 + sqrt(1 + 8T/w)) / 2

Take floor since x must be integer.
```

**Careful with floating point**: `sqrt` may have precision issues for large values. If needed, verify `x+1` doesn't also satisfy the constraint:

```csharp
while ((x+1)*(x+2)/2 * w <= T) x++;  // optional correction
```

## Example Walkthrough

**Example 1**: `mountainHeight=4, workerTimes=[2,1,1]`, binary search finds T=3:

| Worker | w | x = floor((-1+sqrt(1+8×3/w))/2) | reduction |
|--------|---|----------------------------------|-----------|
| 0 | 2 | floor((-1+sqrt(13))/2)=floor(1.30)=1 | 1 |
| 1 | 1 | floor((-1+sqrt(25))/2)=floor(2.0)=2 | 2 |
| 2 | 1 | 2 | 2 |

Total reduction = 1+2+2 = 5 ≥ 4 ✓

At T=2:
- Worker 0: floor((-1+sqrt(9))/2)=1
- Worker 1: floor((-1+sqrt(17))/2)=1
- Worker 2: 1

Total = 3 < 4 ✗ → T=3 is indeed the minimum.

## Pitfalls & Edge Cases

- **`long` throughout** — `workerTimes[i]` up to 10⁶, `mountainHeight` up to 10⁵ → `hi` up to ~5×10¹⁵, overflows `int` and even `long` if not careful; compute `hi` as `(long)max * h * (h+1) / 2`
- **Floating point in `sqrt`** — for very large T/w, `sqrt` precision may give `x` that's 1 off; add a while-loop correction or use integer Newton's method if needed
- **Early exit in feasibility** — once `total >= mountainHeight`, return immediately; avoids iterating all workers unnecessarily
- **Single worker** — binary search still works; hi = that worker doing everything

---

## Key Takeaway

"Minimum time for parallel workers with quadratic cost" → **binary search on T + closed-form max-work per worker**:

```
feasible(T): sum over workers of floor((-1 + sqrt(1 + 8T/w)) / 2) >= mountainHeight
```

Deriving the per-worker formula from `w*x*(x+1)/2 ≤ T` via quadratic formula is the key step. Binary search handles the outer optimization cleanly.
