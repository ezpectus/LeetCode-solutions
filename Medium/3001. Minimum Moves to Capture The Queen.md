# 3001. Minimum Moves to Capture The Queen
## O(1) — Check 1-Move Attacks, Otherwise 2

---

## Problem Statement

- 8×8 chessboard. White rook at `(a,b)`, white bishop at `(c,d)`, black queen at `(e,f)`.
- Return minimum moves to capture the queen (moving only white pieces).

---

## Core Idea — Answer is Always 1 or 2

**Answer is 1** if either piece can directly attack the queen without being blocked.

**Answer is 2** otherwise — we can always reposition a piece in 1 move to attack on the next (e.g., move rook to the queen's row, then capture).

### 1-Move Conditions

**Rook attacks queen** (same row or column, no blocker):
- Same row (`a == e`): rook attacks unless bishop is on the same row **between** them
  → blocked if `c == a AND min(b,f) < d < max(b,f)`
- Same col (`b == f`): rook attacks unless bishop is on the same col **between** them
  → blocked if `d == b AND min(a,e) < c < max(a,e)`

**Bishop attacks queen** (same diagonal, no blocker):
- Main diagonal (`c-d == e-f`): bishop attacks unless rook is on the same diagonal between them
  → blocked if `a-b == c-d AND min(c,e) < a < max(c,e)`
- Anti-diagonal (`c+d == e+f`): blocked if `a+b == c+d AND min(c,e) < a < max(c,e)`

If any condition holds without being blocked → return **1**. Otherwise → return **2**.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinMovesToCaptureTheQueen(int a, int b, int c, int d, int e, int f) {
        // Rook attacks: same row, bishop not blocking
        if (a == e && !(c == a && Math.Min(b,f) < d && d < Math.Max(b,f))) return 1;
        // Rook attacks: same col, bishop not blocking
        if (b == f && !(d == b && Math.Min(a,e) < c && c < Math.Max(a,e))) return 1;
        // Bishop attacks: main diagonal, rook not blocking
        if (c-d == e-f && !(a-b == c-d && Math.Min(c,e) < a && a < Math.Max(c,e))) return 1;
        // Bishop attacks: anti-diagonal, rook not blocking
        if (c+d == e+f && !(a+b == c+d && Math.Min(c,e) < a && a < Math.Max(c,e))) return 1;
        return 2;
    }
}
```

---

## Why Answer is Never More Than 2

The rook can always reach the queen in at most 2 moves:
- Move rook to queen's row (1 move), then capture (1 move).

Even if the bishop blocks, we can choose the other approach (rook to queen's column first).

---

## Example Walkthrough

**Example 2**: `a=5,b=3, c=3,d=4, e=5,f=2`

Rook (5,3) and queen (5,2): same row? `a==e → 5==5` ✓
- Is bishop blocking? Bishop at (3,4). Is `c==a`? `3==5`? No → not blocking.
→ Rook attacks → **1** ✓

---

## Pitfalls & Edge Cases

- **Blocker must be strictly between**: use strict `<` not `≤`; a piece at the exact position of rook or queen doesn't block
- **Check rook blocker for bishop (and vice versa)**: the OTHER piece is the potential blocker, not any piece
- **Both diagonals**: bishop attacks on either main or anti-diagonal — check both

---

## Key Takeaway

"Minimum chess moves to capture with rook+bishop" → **check 4 direct attack conditions with blocker validation; otherwise 2**:

```
if piece aligns with queen AND no blocker between them: return 1
else: return 2
```

The 2-move bound is guaranteed by the rook's flexibility on an 8×8 board.
