# 3091. Apply Operations to Make Sum of Array Greater Than or Equal to k
## O(k) — Enumerate Increment Count, Compute Min Duplications

---

## Problem Statement

- Start with `nums = [1]`. Operations: increment any element by 1, OR duplicate any element.
- Minimize total operations to reach sum ≥ `k`.

---

## Core Idea — Fix Number of Increments, Compute Required Duplications

**Strategy**: increment the single element `i` times (to value `v = 1+i`), then duplicate it enough times to reach sum ≥ `k`.

- After `i` increments: value `v = 1 + i`.
- Duplications needed: `ceil(k / v) - 1` (start with 1 copy, need `ceil(k/v)` total copies).
- Total operations: `i + ceil(k/v) - 1`.

Enumerate `i` from `0` to `k-1` and return the minimum.

**Why only enumerate increments?** The optimal strategy is to increment first, then duplicate repeatedly — duplicating before incrementing is never better (duplicating a small value then incrementing both is worse than incrementing once then duplicating).

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int minOperations(int k) {
        if (k == 1) return 0;
        int res = k - 1; // worst case: k-1 increments, no duplications
        for (int i = 0; i < k; i++) {
            int v = 1 + i;
            int dups = (k + v - 1) / v - 1; // ceil(k/v) - 1
            res = min(res, i + dups);
        }
        return res;
    }
};
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(k)** | Enumerate all possible increment counts |
| **Space** | **O(1)** |  |

With `k ≤ 10⁵`: 10⁵ iterations — instant. Can also be optimized to O(√k) by noting the optimal `i ≈ √k - 1`.

## Example Walkthrough

**`k=11`**

| i | v=1+i | dups=ceil(11/v)-1 | total=i+dups |
|---|-------|-------------------|--------------|
| 0 | 1 | 10 | 10 |
| 1 | 2 | 5 | 6 |
| 2 | 3 | 3 | 5 |
| **3** | **4** | **2** | **5** ✓ |
| 4 | 5 | 2 | 6 |
| 5 | 6 | 1 | 6 |
| 10 | 11 | 0 | 10 |

Minimum = **5** ✓ (increment 3 times to get 4, duplicate twice: [4,4,4], sum=12≥11)

---

## Pitfalls & Edge Cases

- **`k=1`**: sum already ≥ 1 → return 0 immediately.
- **`ceil(k/v) - 1`** using integer arithmetic: `(k + v - 1) / v - 1`.
- **Worst case** initialization `res = k-1`: just increment `k-1` times to reach `[k]`, sum=k≥k. Always a valid baseline.

---

## Key Takeaway

"Min ops to reach sum ≥ k via increment + duplicate" → **enumerate increment count `i`, compute `ceil(k/(1+i)) - 1` duplications, minimize total**:

```
for i in [0, k):
    v = 1 + i
    res = min(res, i + ceil(k/v) - 1)
```

The optimal split is near `i ≈ √k - 1` (AM-GM minimizes `i + k/v = i + k/(1+i)`), giving O(√k) answer, but O(k) enumeration is simple and fast enough.
