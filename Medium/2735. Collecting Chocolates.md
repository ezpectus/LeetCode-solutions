# 2735. Collecting Chocolates
## O(n²) — Enumerate Rotation Count + Running Minimum

---

## Problem Statement

- `n` chocolates, type `i` costs `nums[i]`. One operation (cost `x`): shift all types by 1 cyclically.
- After `k` operations, type `i` can be collected at any position it has visited: `i, i-1, i-2, ..., i-k` (mod n).
- Collect one chocolate of each type with minimum total cost.

---

## Core Idea — Enumerate Number of Rotations

For `k` rotations (cost `k × x`), type `i` has been at positions `i, (i+1)%n, ..., (i+k)%n`. The cheapest we can buy type `i` is:

```
minCost[i] = min(nums[i], nums[(i+1)%n], ..., nums[(i+k)%n])
```

**Total cost for k rotations** = `k × x + Σ minCost[i]`

We try all `k` from 0 to `n-1` (at `k = n` everyone's back to start, no point going further). For efficiency, maintain `minCosts[i]` incrementally — at each step `k`, update `minCosts[i] = min(minCosts[i], nums[(i+k)%n])`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinCost(int[] nums, int x) {
        int n = nums.Length;
        long[] minCosts = nums.Select(v => (long)v).ToArray();
        long res = minCosts.Sum(); // k=0

        for (int k = 1; k < n; k++) {
            for (int i = 0; i < n; i++)
                minCosts[i] = Math.Min(minCosts[i], nums[(i + k) % n]);
            res = Math.Min(res, (long)k * x + minCosts.Sum());
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | n rotations × n updates + n sum |
| **Space** | **O(n)** | minCosts array |

n ≤ 1000 → 10⁶ ops, trivially fast.

## Example Walkthrough

**`nums=[20,1,15], x=5`**

**k=0**: costs=[20,1,15], total=36
**k=1**: 
- `minCosts[0]=min(20,nums[1])=min(20,1)=1`
- `minCosts[1]=min(1,nums[2])=min(1,15)=1`
- `minCosts[2]=min(15,nums[0])=min(15,20)=15`
- total = 1×5 + (1+1+15) = 5+17 = 22

**k=2**:
- `minCosts[0]=min(1,nums[2])=min(1,15)=1`
- `minCosts[1]=min(1,nums[0])=min(1,20)=1`
- `minCosts[2]=min(15,nums[1])=min(15,1)=1`
- total = 2×5 + (1+1+1) = 10+3 = **13** ✓

Result: `min(36, 22, 13) = 13` ✓

## Pitfalls & Edge Cases

- **`long` for result** — `nums[i]` up to 10⁹, `n` up to 1000, `x` up to 10⁹ → `k*x` up to ~10¹², `sum` up to ~10¹² → use `long`
- **`k` only goes to `n-1`** — at `k=n` the cycle completes; further rotations repeat already-seen positions
- **Incremental `minCosts` update** — don't recompute from scratch each `k`; just add one new position per element per rotation

---

## Key Takeaway

"Minimum cost with cyclic rotation choices" → **enumerate rotation count, maintain running minimums**:

```
for k in 0..n-1:
    minCosts[i] = min(minCosts[i], nums[(i+k)%n])
    candidate = k*x + sum(minCosts)
```

Each additional rotation expands the "window" of available prices for each type. O(n²) is optimal here since we must consider all rotation counts.
