# 3096. Minimum Levels to Gain More Points
## O(n) — Prefix Score Comparison

---

## Problem Statement

- Binary array `possible`. Clearing level `i` gives +1 if `possible[i]=1`, -1 if `possible[i]=0`.
- Alice plays levels `0..k-1`, Bob plays `k..n-1`. Both play at least 1 level.
- Find minimum `k` where Alice's score > Bob's score.

---

## Core Idea — Track Running Scores

**Score for k levels**: `ones - zeros = ones - (k - ones) = 2*ones - k`.

**Total score**: `2*total_ones - n` (fixed, split between Alice and Bob).

So `bob_score = total_score - alice_score`.

Alice wins when `alice_score > total_score - alice_score`, i.e., `2*alice_score > total_score`.

Sweep `k` from 1 to `n-1`, compute Alice's score using a running `ones` counter. Return the first `k` where Alice wins.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumLevels(int[] possible) {
        int n = possible.Length;
        int totalOnes = possible.Sum();
        int totalScore = 2 * totalOnes - n;

        int aliceOnes = 0;
        for (int k = 1; k < n; k++) {
            aliceOnes += possible[k-1];
            int aliceScore = 2 * aliceOnes - k;
            if (aliceScore > totalScore - aliceScore) return k;
        }
        return -1;
    }
}
```

## Example Walkthrough

**`possible=[1,1,1,1,1]`**, total_ones=5, total_score=10-5=5.

| k | alice_ones | alice_score=2*ones-k | bob_score=5-alice | alice>bob? |
|---|------------|---------------------|-------------------|-----------|
| 1 | 1 | 2-1=1 | 4 | No |
| 2 | 2 | 4-2=2 | 3 | No |
| 3 | 3 | 6-3=3 | 2 | **Yes** → return 3 |

Answer: **3** ✓

---

## Key Takeaway

"Min prefix where Alice score > Bob score" → **running prefix score vs. fixed total**:

```
alice_score = 2*alice_ones - k
bob_score   = total_score - alice_score
win if alice_score > bob_score ↔ 2*alice_score > total_score
```

O(n) single pass with running counter. No extra space needed.
