# 3488. Closest Equal Element Queries
## O((n + q) log n) — Group Positions by Value + Binary Search

---

## Problem Statement

- Circular array `nums`. For each query index `q`, find the minimum circular distance to any other index with the same value.
- Return -1 if no other index has the same value.

---

## Core Idea — Group Positions, Binary Search Neighbors

**Preprocessing**: group positions by value — `positions[v]` = sorted list of all indices where `nums[index] == v`.

**Per query `q`**: value `v = nums[q]`.
- If only 1 occurrence: return -1 (no "other" index with same value).
- Otherwise: binary search for `q` in `positions[v]`. Check neighbors (previous, same, next in sorted list with wrap-around). Skip `q` itself. Compute minimum circular distance: `min(|a-b|, n-|a-b|)`.

**Why only check 3 neighbors?** The sorted position list gives the closest indices linearly. The circular wrap can make the "last" element in the list close to the "first", but the two nearest elements in the sorted list (plus handling wrap-around) cover all cases.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public IList<int> SolveQueries(int[] nums, int[] queries) {
        int n = nums.Length;
        var positions = new Dictionary<int, List<int>>();
        for (int i = 0; i < n; i++) {
            if (!positions.ContainsKey(nums[i])) positions[nums[i]] = new();
            positions[nums[i]].Add(i);
        }

        var res = new int[queries.Length];
        for (int qi = 0; qi < queries.Length; qi++) {
            int idx = queries[qi];
            int v = nums[idx];
            var pos = positions[v];

            if (pos.Count == 1) { res[qi] = -1; continue; }

            // Binary search for idx
            int j = pos.BinarySearch(idx);
            int best = int.MaxValue;

            for (int k = -1; k <= 1; k++) {
                int ki = ((j + k) % pos.Count + pos.Count) % pos.Count;
                int other = pos[ki];
                if (other == idx) continue;
                int d = Math.Abs(other - idx);
                d = Math.Min(d, n - d);
                best = Math.Min(best, d);
            }

            res[qi] = best == int.MaxValue ? -1 : best;
        }
        return res;
    }
}
```

## Example Walkthrough

**`nums=[1,3,1,4,1,3,2], queries=[0,3,5]`**, n=7.

positions: {1:[0,2,4], 3:[1,5], 4:[3], 2:[6]}

- q=0: v=1, pos=[0,2,4]. j=bisect(0)=0. Neighbors: k=-1→idx=2(pos[2]=4), k=0→pos[0]=0(skip=same), k=1→pos[1]=2. Distances: |4-0|=4→min(4,3)=3, |2-0|=2. best=**2** ✓.

- q=3: v=4, pos=[3]. Count=1 → **-1** ✓.

- q=5: v=3, pos=[1,5]. j=1. Neighbors: k=-1→pos[0]=1, k=0→pos[1]=5(skip), k=1→pos[0]=1(wrapped). |1-5|=4→min(4,3)=3. best=**3** ✓.

## Pitfalls & Edge Cases

- **Wrap-around in position list**: use `((j+k) % count + count) % count` for negative modulo in C#.
- **Skip `q` itself**: `other == idx` check.
- **Only 2 occurrences**: both neighbors in the list are the other one — handles correctly.
- **Circular distance**: `min(|a-b|, n-|a-b|)`.

---

## Key Takeaway

"Min circular distance to same-value element" → **group positions by value, binary search + check 3 neighbors**:

```
pos = sorted positions of value v
j = binary search for q in pos
for k in [-1, 0, 1]: check circular dist to pos[(j+k) % len(pos)]
```

O(log n) per query after O(n) preprocessing.
