# 2658. Maximum Number of Fish in a Grid
## O(m × n) — DFS Connected Components

---

## Problem Statement

- Grid cells: `0` = land, `> 0` = water with that many fish.
- Fisher can move between adjacent water cells freely, collecting all fish.
- Return the maximum fish collectible from any single connected water component.

---

## Core Idea — DFS on Connected Water Components

This is a standard **connected components** problem: find the water component with the maximum sum. Run DFS from each unvisited water cell, summing all fish in the component. Zero out visited cells in-place to avoid revisiting (no separate `visited` array needed).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    int m, n;
    int[][] grid;

    public int FindMaxFish(int[][] grid) {
        this.grid = grid;
        m = grid.Length; n = grid[0].Length;
        int res = 0;
        for (int r = 0; r < m; r++)
            for (int c = 0; c < n; c++)
                if (grid[r][c] > 0)
                    res = Math.Max(res, Dfs(r, c));
        return res;
    }

    int Dfs(int r, int c) {
        if (r < 0 || r >= m || c < 0 || c >= n || grid[r][c] == 0) return 0;
        int fish = grid[r][c];
        grid[r][c] = 0; // mark visited
        return fish + Dfs(r+1,c) + Dfs(r-1,c) + Dfs(r,c+1) + Dfs(r,c-1);
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(m × n)** | Each cell visited at most once |
| **Space Complexity** | **O(m × n)** | Recursion stack depth in worst case (all water) |

---

## Pitfalls & Edge Cases

- **In-place zeroing** avoids a `visited` array — safe here since grid values ≥ 0 and `0` already means land/visited
- **All land** → `max` of all DFS calls returns 0 ✓ (`Math.Max(0, ...)` never updates from 0)
- **Single cell** → DFS returns `grid[r][c]` immediately ✓

---

## Key Takeaway

"Max sum connected component in grid" → **DFS with in-place visited marking**, return component sum. Classic island problem variant.
