# 2919. Minimum Increment Operations to Make Array Beautiful
## O(n) — Greedy DP on Last 3 Positions

---

## Problem Statement

- Given a 0-indexed integer array `nums` of length `n`, and integer `k`.
- You can **increment any element by 1** any number of times.
- An array is **beautiful** if every subarray of size ≥ 3 has its maximum element ≥ `k`.
- Return the **minimum number of increment operations** to make `nums` beautiful.

---

## Core Idea — DP Over Which Element in Each Window Gets Boosted

**Key insight**: every subarray of size ≥ 3 must have max ≥ `k`. This means there must be **no 3 consecutive elements all below `k`**. Equivalently, among every 3 consecutive positions at least one must have value ≥ `k`.

So the constraint reduces to: **in every window of 3, at least one element ≥ k**.

This means between any two "covered" positions (elements ≥ k), the gap can be at most 2. We must choose a set of positions to boost to ≥ k (at minimum cost), with no 3 consecutive positions all uncovered.

**DP formulation**:

> `dp[i]` = minimum total cost to satisfy all windows up to index `i`, **given that index `i` is the last "covered" position**.

Cost to cover position `i`:
```
cost(i) = max(0, k - nums[i])
```

**Transition**: the previous covered position must be `i-1`, `i-2`, or `i-3` (gap ≤ 2). For `i < 3` no window has yet formed, so no prior cover is required → `best = 0`:
```
dp[i] = cost(i) + (i < 3 ? 0 : min(dp[i-1], dp[i-2], dp[i-3]))
```

**Answer**: `min(dp[n-1], dp[n-2], dp[n-3])` — the last covered position must be within the last 3.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MinIncrementOperations(int[] nums, int k) {
        int n = nums.Length;
        long[] dp = new long[n];

        for (int i = 0; i < n; i++) {
            long cost = Math.Max(0L, k - nums[i]);
            long best = i < 3 ? 0 : Math.Min(dp[i-1], Math.Min(dp[i-2], dp[i-3]));
            dp[i] = cost + best;
        }

        return Math.Min(dp[n-1], Math.Min(dp[n-2], dp[n-3]));
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n)** | Single pass, O(1) work per element |
| **Space** | **O(n)** | dp array — reducible to O(1) with 3 rolling variables |

## Why This Works — The Window Constraint

**Why does "every subarray of size ≥ 3 has max ≥ k" reduce to "no 3 consecutive elements all < k"?**

If any 3 consecutive elements are all < k, the subarray formed by those 3 elements has max < k → not beautiful. Conversely, if every group of 3 consecutive elements contains at least one ≥ k, then any longer subarray also contains at least one ≥ k. So the constraint is exactly: no 3 consecutive elements all below `k`.

**Why gap ≤ 2 between covered positions?**

If two covered positions are at indices `i` and `j` with `j - i ≥ 4`, then positions `i+1, i+2, i+3` are all uncovered → 3 consecutive elements below k → violation.

**Why `best = 0` for `i < 3`?**

For the first 3 positions no complete window of size 3 exists yet, so there's no constraint forcing a prior covered position. Chaining off earlier dp values would artificially inflate cost.

**Why `min(dp[n-1], dp[n-2], dp[n-3])` as answer?**

The last covered position must be at index `n-1`, `n-2`, or `n-3` — any earlier and the final 3 elements would be uncovered.

## Example Walkthrough

**`nums = [2,3,0,0,2], k = 4`**

Cost per position: `[2, 1, 4, 4, 2]`

| i | cost | best | dp[i] |
|---|------|------|-------|
| 0 | 2 | 0 | **2** |
| 1 | 1 | 0 | **1** |
| 2 | 4 | 0 | **4** |
| 3 | 4 | min(2,1,4)=1 | **5** |
| 4 | 2 | min(1,4,5)=1 | **3** |

Answer: `min(dp[4], dp[3], dp[2]) = min(3, 5, 4)` → **3** ✓

---

**`nums = [0,1,3,3], k = 5`**

Cost per position: `[5, 4, 2, 2]`

| i | cost | best | dp[i] |
|---|------|------|-------|
| 0 | 5 | 0 | **5** |
| 1 | 4 | 0 | **4** |
| 2 | 2 | 0 | **2** |
| 3 | 2 | min(5,4,2)=2 | **4** |

Answer: `min(dp[3], dp[2], dp[1]) = min(4, 2, 4)` → **2** ✓

---

**`nums = [1,1,2], k = 1`**

All elements already ≥ 1 → costs = `[0, 0, 0]`, dp = `[0, 0, 0]`

Answer: `min(0, 0, 0)` → **0** ✓

---

## Pitfalls & Edge Cases

- **`best = 0` for `i < 3`** — no window has formed yet, chaining off earlier dp values inflates the cost unnecessarily
- **Use `long` not `int`** — costs up to `k = 10^9`, summed over `n = 10^5` → up to `10^14`, overflows `int`
- **Answer is `min` of last 3 dp values** — the rightmost covered position can be any of `n-1`, `n-2`, `n-3`
- **Elements already ≥ k** cost 0 and are free covers — `Math.Max(0L, k - nums[i])` handles this

---

## Key Takeaway

"Every subarray of size ≥ 3 has max ≥ k" → **no 3 consecutive elements all < k** → **dp[i] = cost to cover i + min cost of any of the 3 previous covered positions**:

```
dp[i] = max(0, k - nums[i]) + (i < 3 ? 0 : min(dp[i-1], dp[i-2], dp[i-3]))
answer = min(dp[n-1], dp[n-2], dp[n-3])
```

Gap between covered positions ≤ 2 → sliding min over 3 predecessors → O(n) time, O(1) rolling space possible.
