# 3820. Pythagorean Distance Nodes in a Tree
## O(n) — Three BFS from x, y, z + Per-Node Triplet Check

---

## Problem Statement

- Given an undirected tree with n nodes (4 ≤ n ≤ 10⁵) and three distinct target nodes x, y, z.
- For each node u: dx = dist(u, x), dy = dist(u, y), dz = dist(u, z).
- Node u is **special** if (dx, dy, dz) form a Pythagorean triplet (sorted as a ≤ b ≤ c: a² + b² = c²).
- Return the count of special nodes.

---

## Core Idea — Three Independent BFS + Per-Node Check

Simple decomposition:
1. BFS from x → distance array `dx[]`
2. BFS from y → distance array `dy[]`
3. BFS from z → distance array `dz[]`
4. For each node u: sort (dx[u], dy[u], dz[u]) and check Pythagorean condition

**Why BFS over DFS:** BFS naturally computes shortest (and only) path distances in an unweighted tree in O(n).

**Pythagorean check:** Note that (0, c, c) is valid since 0² + c² = c². This handles the case where u = x, y, or z.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int SpecialNodes(int n, int[][] edges, int x, int y, int z) {
        var adj = new List<int>[n];
        for (int i = 0; i < n; i++) adj[i] = new List<int>();

        foreach (var e in edges) {
            adj[e[0]].Add(e[1]);
            adj[e[1]].Add(e[0]);
        }

        int[] dx = BFS(adj, n, x);
        int[] dy = BFS(adj, n, y);
        int[] dz = BFS(adj, n, z);

        int res = 0;
        for (int u = 0; u < n; u++) {
            if (IsPythagorean(dx[u], dy[u], dz[u])) res++;
        }

        return res;
    }

    private int[] BFS(List<int>[] adj, int n, int src) {
        int[] dist = new int[n];
        Array.Fill(dist, -1);
        dist[src] = 0;

        var queue = new Queue<int>();
        queue.Enqueue(src);

        while (queue.Count > 0) {
            int u = queue.Dequeue();
            foreach (int v in adj[u]) {
                if (dist[v] == -1) {
                    dist[v] = dist[u] + 1;
                    queue.Enqueue(v);
                }
            }
        }

        return dist;
    }

    private bool IsPythagorean(int a, int b, int c) {
        // Sort the three values
        if (a > b) (a, b) = (b, a);
        if (b > c) (b, c) = (c, b);
        if (a > b) (a, b) = (b, a);

        return (long)a * a + (long)b * b == (long)c * c;
    }
}
```

## Pitfalls & Edge Cases

- **Integer overflow** — Max distance = n−1 ≤ 99,999. Max distance² ≈ 10¹⁰, overflows `int`. Use `long` for the Pythagorean check. ✓

- **(0, c, c) is a valid triplet** — When u = x, dx = 0, and if dy = dz = c: 0² + c² = c². Correctly handled by the general check.

- **Sorting three values** — Three comparison swaps (bubble sort on 3 elements) is cleaner than `Array.Sort` for fixed-size inputs. Can also use `Math.Min/Max` tricks.

- **Tree is connected** — Guaranteed by problem, so BFS from any node reaches all n nodes. No need to check for unreachable nodes (dist = -1 never remains).

---

## Key Takeaway

The tree structure is almost irrelevant — it just determines distances. Once you have the three distance arrays via BFS, this reduces to a pure arithmetic check per node.

**Pattern recognition:** "Per-node property involving distances to k fixed nodes" → k BFS runs + O(n) per-node check. For k = 3 this is O(n) total, regardless of what the property is.

The Pythagorean check is the mathematical core; the tree is just the distance metric.
