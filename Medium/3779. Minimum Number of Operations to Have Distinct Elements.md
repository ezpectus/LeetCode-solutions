# 3779. Minimum Number of Operations to Have Distinct Elements
## O(n) — Suffix Max of Prev-Occurrence Array

---

## Problem Statement

- You are given array `nums` (1 ≤ n ≤ 10⁵, 1 ≤ nums[i] ≤ 10⁵).
- One operation removes the first 3 elements (or all remaining if fewer than 3).
- Stop when the array is empty or has all distinct elements.
- Return the number of operations required.

---

## Core Idea — Check Each Candidate Suffix Efficiently

After `k` operations the remaining array is `nums[3k..n-1]`. We want the smallest `k` such that this suffix has no duplicates.

**Key observation:** `nums[i..n-1]` has no duplicates iff for every `j ∈ [i, n-1]`, the previous occurrence of `nums[j]` (call it `prev[j]`) is strictly before `i`.

**Formally:** suffix is dup-free iff `max(prev[j] for j in [i, n-1]) < i`.

Define `maxPrev[i] = max(prev[j] for j = i to n-1)` (suffix maximum of the `prev` array).

Then: answer = smallest `k ≥ 0` such that `maxPrev[3k] < 3k` (or `3k ≥ n` for empty array).

This requires:
1. Compute `prev[j]` for all j → O(n)
2. Compute suffix max `maxPrev` → O(n)
3. Scan k = 0, 1, 2, ... → O(n/3)

**Total: O(n)**

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperations(int[] nums) {
        int n = nums.Length;

        // prev[i] = last index j < i where nums[j] == nums[i], or -1 if none
        int[] prev = new int[n];
        var lastSeen = new Dictionary<int, int>();

        for (int i = 0; i < n; i++) {
            prev[i] = lastSeen.TryGetValue(nums[i], out int p) ? p : -1;
            lastSeen[nums[i]] = i;
        }

        // maxPrev[i] = max(prev[j] for j in [i, n-1])
        int[] maxPrev = new int[n + 1];
        maxPrev[n] = -1;
        for (int i = n - 1; i >= 0; i--) {
            maxPrev[i] = Math.Max(prev[i], maxPrev[i + 1]);
        }

        // Find smallest k where suffix nums[3k..n-1] is dup-free
        for (int k = 0; ; k++) {
            int start = 3 * k;
            if (start >= n) return k;         // empty suffix
            if (maxPrev[start] < start) return k; // no duplicate in suffix
        }
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(n) | One pass for prev, one for maxPrev, one scan of k |
| Space Complexity | O(n) | prev and maxPrev arrays + dictionary |

## Why `maxPrev[i] < i` Detects No Duplicates

A duplicate exists in `nums[i..n-1]` iff there exists some `j ∈ [i, n-1]` where `prev[j] ≥ i` — meaning the previous occurrence of `nums[j]` is also within our window `[i, n-1]`.

The **maximum** of all `prev[j]` over this range gives the "closest" any value has been repeated. If even the maximum is still less than `i`, every value's previous occurrence is outside our window — no duplicates. ✓

## Example Walkthrough

**Example 1:** `nums = [3, 8, 3, 6, 5, 8]`

Prev computation:
| j | nums[j] | prev[j] |
|---|---------|---------|
| 0 | 3       | −1      |
| 1 | 8       | −1      |
| 2 | 3       | **0**   |
| 3 | 6       | −1      |
| 4 | 5       | −1      |
| 5 | 8       | **1**   |

Suffix maxPrev (right to left):
| i | maxPrev[i]                  |
|---|-----------------------------|
| 6 | −1                          |
| 5 | max(1, −1) = **1**          |
| 4 | max(−1, 1) = **1**          |
| 3 | max(−1, 1) = **1**          |
| 2 | max(0, 1) = **1**           |
| 1 | max(−1, 1) = **1**          |
| 0 | max(−1, 1) = **1**          |

Scan:
- k=0: start=0, maxPrev[0]=1 ≥ 0 → duplicate exists
- k=1: start=3, maxPrev[3]=1 < 3 → dup-free! ✓

**Output:** 1 ✓

---

**Example 2:** `nums = [2, 2]`

prev = [−1, 0], maxPrev = [0, 0, −1]

- k=0: start=0, maxPrev[0]=0 ≥ 0 → duplicate
- k=1: start=3 ≥ n=2 → empty suffix

**Output:** 1 ✓

---

**Example 3:** `nums = [4, 3, 5, 1, 2]`

All prev = −1, maxPrev[0] = −1 < 0 → dup-free immediately.

**Output:** 0 ✓

## Pitfalls & Edge Cases

- **Empty suffix:** `3k ≥ n` → return `k`. An empty array has no duplicates by definition. ✓

- **All distinct from the start:** `maxPrev[0] = -1 < 0` → return 0 immediately. ✓

- **All duplicates:** e.g., `[1,1,1,1,1,1]`. prev = [−1,0,1,2,3,4], maxPrev[0]=4, maxPrev[3]=4 ≥ 3, maxPrev[6]=−1. Returns k=2 (removes all 6 elements in 2 ops). ✓

- **The for loop terminates:** since every 3 operations we consume at least 1 element (or terminate early), we'll always hit `3k ≥ n` in at most `⌈n/3⌉` steps. ✓

---

## Key Takeaway

The problem asks for the minimum k such that `nums[3k..n-1]` is dup-free. The condition "suffix is dup-free" is equivalent to a single comparison: `max_prev_in_suffix < suffix_start`.

By precomputing the **suffix maximum of prev-occurrence indices**, each candidate `k` is checked in O(1), giving O(n) total.

**Pattern recognition:** "Minimum prefix to remove so a suffix satisfies a property involving no-repeat" → `prev` array + suffix max. This technique generalizes to any "no duplicate in window" check when the boundary moves monotonically.
