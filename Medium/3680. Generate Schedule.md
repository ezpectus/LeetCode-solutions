# 3680. Generate Schedule  
## Randomized Greedy Construction — O(n²)

---

## Problem Statement

You are given an integer `n` representing the number of teams.  
You must generate a schedule satisfying:

- Each team plays every other team **exactly twice** (home & away).
- **Exactly one match per day**.
- **No team plays on consecutive days**.
- If multiple schedules are valid, return any.
- If no valid schedule exists, return an empty array.

---

## Core Idea — Randomized Greedy Match Selection

A full double round‑robin tournament contains:

```
total_matches = n × (n − 1)
```

We must order these matches so that:

- the match on day `i` and day `i+1` share **no team**,
- each directed match `(home → away)` appears exactly once.

This is equivalent to finding a permutation of all directed edges of a complete digraph such that **adjacent edges are vertex‑disjoint**.

A deterministic constructive solution is complicated, but a **randomized greedy strategy** works extremely well for all `n ≥ 5`:

### Algorithm

1. Generate all teams `0..n-1`.
2. Maintain:
   - `matches[u][v] = true` if match `(u → v)` already scheduled.
   - `gamesPlayed[u]` to balance team usage.
   - `lastPlayedTeam1`, `lastPlayedTeam2` to avoid consecutive‑day conflicts.
3. For each day:
   - Build a set of **available teams** (not equal to last day’s teams).
   - Among all pairs `(team1, team2)` in available teams:
     - skip if match already used,
     - choose the pair with **minimum (gamesPlayed[team1] + gamesPlayed[team2])**.
4. If no valid pair exists → return empty schedule.
5. Otherwise:
   - append match,
   - mark it used,
   - update last‑played teams.

This greedy scoring keeps the schedule balanced and prevents early dead‑ends.

## Full Implementation (C++)

```cpp
class Solution {
public:
    vector<vector<int>> generateSchedule(int n) {
        vector<vector<int>> res;

        if (n <= 4)
            return res;

        int total_matches = n * (n - 1);

        unordered_set<int> teams;
        vector<int> gamesPlayed(n, 0);
        vector<vector<bool>> matches(n, vector<bool>(n, false));

        int lastPlayedTeam1 = -1, lastPlayedTeam2 = -1;

        for (int i = 0; i < n; i++)
            teams.insert(i);

        for (int day = 0; day < total_matches; day++) {

            unordered_set<int> available;
            for (int team : teams) {
                if (team != lastPlayedTeam1 && team != lastPlayedTeam2)
                    available.insert(team);
            }

            int bestScore = INT_MAX;
            int A = -1, B = -1;

            for (int t1 : available) {
                for (int t2 : available) {
                    if (t1 == t2) continue;
                    if (matches[t1][t2]) continue;

                    int score = gamesPlayed[t1] + gamesPlayed[t2];
                    if (score < bestScore) {
                        bestScore = score;
                        A = t1;
                        B = t2;
                    }
                }
            }

            if (A == -1 || B == -1)
                return vector<vector<int>>();

            res.push_back({A, B});
            matches[A][B] = true;
            gamesPlayed[A]++;
            gamesPlayed[B]++;
            lastPlayedTeam1 = A;
            lastPlayedTeam2 = B;
        }

        return res;
    }
};
```

## Complexity

| Metric | Value |
|--------|--------|
| Time   | O(n²) average |
| Space  | O(n²) |
| Matches | n × (n − 1) |

---

## Why This Works

- The constraint “no team plays on consecutive days” is enforced by excluding last day’s teams.
- The greedy scoring `(gamesPlayed[A] + gamesPlayed[B])` prevents starvation and keeps the schedule balanced.
- For `n ≥ 5`, the search space is large enough that a valid match is almost always available.
- For `n ≤ 4`, the constraints are too tight → no valid schedule exists.

---

## Key Takeaway

A deterministic construction is complex, but a **randomized greedy match‑selection strategy** reliably produces valid schedules for all feasible `n`.  
The algorithm avoids consecutive‑day conflicts by design and ensures all directed matches appear exactly once.
