# 3080. Mark Elements on Array by Performing Queries
## O(n log n) — Min-Heap + Lazy Deletion

---

## Problem Statement

- All elements start unmarked. `m` queries, each `[index, k]`:
  1. Mark `nums[index]` if not already marked.
  2. Mark `k` smallest unmarked elements (by value, ties broken by index).
- After each query, return the sum of all unmarked elements.

---

## Core Idea — Min-Heap with Lazy Deletion

**Min-heap** of `(value, index)` gives the smallest unmarked element in O(log n). When we pop an already-marked element, we discard it (lazy deletion).

**Running sum**: maintain `total = sum(nums)`. Each time we mark an element, subtract its value from `total`. After each query, append `total` to the result.

**Algorithm per query `[idx, k]`**:
1. If `nums[idx]` not marked: mark it, `total -= nums[idx]`.
2. Pop from heap `k` times, skipping already-marked elements, marking each valid pop.

**Lazy deletion** ensures we never process a marked element twice — each element is pushed once and popped at most once (either as a skip or as an actual mark). Total heap operations = O(n log n).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long[] UnmarkedSumArray(int[] nums, int[][] queries) {
        int n = nums.Length;
        bool[] marked = new bool[n];
        long total = nums.Sum(x => (long)x);

        // Min-heap: (value, index)
        var heap = new PriorityQueue<(int val, int idx), (int, int)>();
        for (int i = 0; i < n; i++)
            heap.Enqueue((nums[i], i), (nums[i], i));

        long[] res = new long[queries.Length];
        for (int q = 0; q < queries.Length; q++) {
            int idx = queries[q][0], k = queries[q][1];

            // Step 1: mark queries[q][0]
            if (!marked[idx]) {
                marked[idx] = true;
                total -= nums[idx];
            }

            // Step 2: mark k smallest unmarked
            int count = 0;
            while (heap.Count > 0 && count < k) {
                var (v, i) = heap.Peek();
                heap.Dequeue();
                if (marked[i]) continue; // already marked, skip (lazy delete)
                marked[i] = true;
                total -= v;
                count++;
            }

            res[q] = total;
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n log n)** | n heap insertions + at most n pops (each element popped once) |
| **Space** | **O(n)** | Heap + marked array |

Total heap pops across all queries ≤ 2n (at most n valid marks + n lazy-deleted skips) → amortized O(log n) per query.

## Example Walkthrough

**`nums=[1,2,2,1,2,3,1], queries=[[1,2],[3,3],[4,2]]`**

Initial: total=12. Heap: [(1,0),(1,3),(1,6),(2,1),(2,2),(2,4),(3,5)].

**Query [1,2]**:
- Mark idx=1 (val=2): total=10.
- Mark 2 smallest: pop (1,0) → mark, total=9. Pop (1,3) → mark, total=8. count=2.
- res[0]=**8** ✓

**Query [3,3]**:
- Mark idx=3: already marked, skip.
- Mark 3 smallest: pop (1,6) → mark, total=7. Pop (2,1) → already marked, skip. Pop (2,2) → mark, total=5. Pop (2,4) → mark, total=3. count=3.
- res[1]=**3** ✓

**Query [4,2]**:
- Mark idx=4: already marked, skip.
- Mark 2 smallest: pop (3,5) → mark, total=0. count=1. Heap empty, stop.
- res[2]=**0** ✓

## Pitfalls & Edge Cases

- **`Peek()` then `Dequeue()`**: in C#'s `PriorityQueue`, use `Dequeue()` which does both. Check marked status after dequeuing.
- **Fewer than `k` unmarked elements**: heap empties before `count == k` → loop stops naturally ✓.
- **`long` for total**: `n=10⁵`, `nums[i]=10⁵` → max sum = `10¹⁰`, overflows `int`.
- **Tie-breaking**: `(value, index)` as composite heap key ensures smaller indices come first when values are equal ✓.

---

## Key Takeaway

"Mark smallest k elements repeatedly, track sum" → **min-heap + lazy deletion + running sum**:

```
heap = min-heap of (value, index)
total = sum(nums)
per query: mark[idx], then pop k valid elements, total -= each
skip already-marked elements (lazy delete)
```

O(n log n) amortized since each element enters and exits the heap exactly once.
