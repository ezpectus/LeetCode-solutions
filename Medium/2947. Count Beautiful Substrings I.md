# 2947. Count Beautiful Substrings I
## O(n²) — Brute Force All Substrings

---

## Problem Statement

- Beautiful substring: `vowels == consonants` AND `(vowels * consonants) % k == 0`.
- Return count of beautiful substrings.
- `n ≤ 1000` → O(n²) is fine.

---

## Core Idea — Check Every Substring

Fix start `i`, extend end `j`, maintain running vowel/consonant counts. Check both conditions at each step.

**Simplification**: since `vowels == consonants = v`, the condition `(v * v) % k == 0`. So we need `v² % k == 0`, which means `v` must contain all prime factors of `k` with at least half the required multiplicity.

But since n is small, just check directly.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int BeautifulSubstrings(string s, int k) {
        var vowels = new HashSet<char> { 'a','e','i','o','u' };
        int n = s.Length, res = 0;

        for (int i = 0; i < n; i++) {
            int v = 0, c = 0;
            for (int j = i; j < n; j++) {
                if (vowels.Contains(s[j])) v++;
                else c++;
                if (v == c && v * c % k == 0) res++;
            }
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n²)** — n≤1000 → 500K iterations |
| **Space** | **O(1)** |

---

## Key Observations

- `vowels == consonants` → substring length must be even
- `vowels * consonants = v²` (since `v = c`) → need `v² % k == 0`
- For very large `n`, a smarter approach using prefix sums and hash maps can achieve O(n) — see problem 2949 (Beautiful Substrings II)

---

## Key Takeaway

Two-pointer brute force: fix left, extend right, maintain running counts. Both conditions checked in O(1) per step. O(n²) total — sufficient for n≤1000.
