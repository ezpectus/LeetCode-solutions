# 2915. Length of the Longest Subsequence That Sums to Target
## O(n × target) — 1D Knapsack DP

---

## Problem Statement

- Given a 0-indexed integer array `nums` and an integer `target`.
- Return the **length of the longest subsequence** of `nums` that sums to `target`.
- If no such subsequence exists, return `-1`.
- A subsequence is formed by deleting some (or no) elements without changing order.

---

## Core Idea — 0/1 Knapsack DP

**Key insight**: this is a classic 0/1 knapsack — each element is either included or excluded. Instead of tracking if a sum is *reachable*, we track the **maximum number of elements** used to reach each sum.

Define:
> `dp[s]` = maximum number of elements from `nums` that sum exactly to `s`, or `-∞` if sum `s` is unreachable.

**Transition**: for each `num` in `nums`, iterate sums from `target` down to `num` (backwards to avoid reusing the same element):
```
dp[s] = max(dp[s], dp[s - num] + 1)   if dp[s - num] != -∞
```

**Answer**: `dp[target]` if reachable, else `-1`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int LengthOfLongestSubsequence(IList<int> nums, int target) {
        int[] dp = new int[target + 1];
        Array.Fill(dp, int.MinValue);
        dp[0] = 0;

        foreach (int num in nums) {
            for (int s = target; s >= num; s--) {
                if (dp[s - num] != int.MinValue)
                    dp[s] = Math.Max(dp[s], dp[s - num] + 1);
            }
        }

        return dp[target] < 0 ? -1 : dp[target];
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × target)** | For each of n elements, iterate up to target sums |
| **Space** | **O(target)** | 1D dp array of size target+1 |

---

## Why This Works — Knapsack Argument

**Why not a boolean reachability array?** We need the *longest* subsequence, not just any. So instead of `bool[] reachable`, we store the best (maximum) element count per reachable sum.

**Why iterate backwards?** We iterate `s` from `target` down to `num` so each element is used **at most once**. If we iterated forwards, `dp[s - num]` might already reflect the current element being added, allowing it to be picked multiple times (unbounded knapsack — wrong here).

**Why initialize to `int.MinValue`?** We need to distinguish "unreachable" from "reachable with 0 elements". Only `dp[0] = 0` is valid at the start (empty subsequence sums to 0). All other sums are unreachable.

**The transition**:
- `dp[s]` — best count to reach sum `s` without using current `num`
- `dp[s - num] + 1` — best count to reach `s - num`, then add `num`
- Take the max of both options

## Example Walkthrough

**`nums = [1,2,3,4,5], target = 9`**

Initial: `dp[0]=0`, all others `-∞`

After `num=1`: `dp[1]=1`

After `num=2`: `dp[2]=1, dp[3]=2`

After `num=3`: `dp[3]=max(2,1)=2, dp[4]=2, dp[5]=3, dp[6]=3`

After `num=4`: `dp[4]=max(2,1)=2, dp[5]=max(3,2)=3, dp[6]=max(3,3)=3, dp[7]=3, dp[8]=4, dp[9]=4`

After `num=5`: `dp[9]=max(4, dp[4]+1)=max(4,3)=4`... wait, `dp[4]=2` → `dp[9]=max(4,3)=4`

`dp[9] = 3` → **3** ✓

---

**`nums = [4,1,3,2,1,5], target = 7`**

Tracing key steps — after processing all elements:
`dp[7] = 4` (subsequence `[1,3,2,1]`) → **4** ✓

---

**`nums = [1,1,5,4,5], target = 3`**

No combination of elements sums to 3 → `dp[3]` stays `-∞` → return **-1** ✓

## Pitfalls & Edge Cases

- **Initialize unreachable as `int.MinValue`, not `-1`** — using `-1` would make `dp[s - num] + 1 = 0`, silently treating unreachable states as reachable with 0 elements
- **Backwards iteration is mandatory** — forward iteration turns this into unbounded knapsack (elements reused), giving wrong answers
- **Guard `dp[s - num] != int.MinValue`** before adding 1 — without this check you get integer overflow on `int.MinValue + 1`
- **Return -1 if `dp[target] < 0`** — covers the unreachable case cleanly

---

## Key Takeaway

"Longest subsequence summing to target" → **0/1 knapsack where dp[s] = max elements to reach sum s**:

```
dp[0] = 0, dp[1..target] = -∞
for each num:
    for s = target down to num:
        if dp[s - num] != -∞:
            dp[s] = max(dp[s], dp[s - num] + 1)
return dp[target] < 0 ? -1 : dp[target]
```

Backwards sweep ensures each element used once → O(n × target) time, O(target) space.
