# 2807. Insert Greatest Common Divisors in Linked List
## O(n log m) — Single Pass with Node Insertion

---

## Problem Statement

- Between every pair of adjacent nodes, insert a new node with their GCD.
- Return the modified list.

---

## Core Idea — Walk and Insert

For each node `curr` with a next node: create a new GCD node between them, then advance `curr` by two steps (skip the newly inserted node).

```
curr → next  becomes  curr → gcd_node → next
then advance: curr = gcd_node.next  (= original next)
```

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public ListNode InsertGreatestCommonDivisors(ListNode head) {
        var curr = head;
        while (curr?.next != null) {
            int g = Gcd(curr.val, curr.next.val);
            curr.next = new ListNode(g, curr.next);
            curr = curr.next.next; // skip inserted node
        }
        return head;
    }

    int Gcd(int a, int b) => b == 0 ? a : Gcd(b, a % b);
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n log m)** | n pairs × O(log m) GCD; m = max node value |
| **Space** | **O(1)** | No extra space beyond new nodes |

---

## Pitfalls & Edge Cases

- **Skip the inserted node** — `curr = curr.next.next` not `curr.next`; otherwise we'd compute GCD of the inserted node with the original next
- **Single node** — `curr.next == null` → loop doesn't execute → return head ✓

---

## Key Takeaway

Linked list insertion between nodes: insert new node, then jump two steps forward to skip it. Standard pattern.
