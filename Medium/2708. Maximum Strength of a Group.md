# 2708. Maximum Strength of a Group
## O(n log n) — Greedy: Take All Positives + Pairs of Negatives

---

## Problem Statement

- Choose a non-empty subset of `nums`, return the maximum product.
- `nums[i] ∈ [-9, 9]`, `n ≤ 13`.

---

## Core Idea — Greedy by Sign

To maximize a product:
- **Positive numbers**: always include — they can only increase the product.
- **Negative numbers**: include in **pairs** (negative × negative = positive). Take pairs with largest absolute values first (sorted ascending = most negative pairs first).
- **Zeros**: never include unless the only alternative is a negative product (zero > negative).
- **Unpaired single negative**: discard (makes product negative).

**Three cases for the final result**:
1. We have at least one positive or one pair of negatives → multiply them all together.
2. We have only zeros and/or a single unpaired negative → return `0` if a zero exists, else the single (least-negative) number.
3. Single element array with one negative → return that negative (non-empty group required).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaxStrength(int[] nums) {
        var pos = nums.Where(x => x > 0).ToList();
        var neg = nums.Where(x => x < 0).OrderBy(x => x).ToList(); // ascending
        bool hasZero = nums.Contains(0);

        long result = 1;
        bool hasProduct = false;

        // Take all positives
        foreach (int x in pos) { result *= x; hasProduct = true; }

        // Take pairs of negatives (most negative = largest abs value first)
        for (int i = 0; i + 1 < neg.Count; i += 2) {
            result *= (long)neg[i] * neg[i+1];
            hasProduct = true;
        }

        if (!hasProduct) {
            // Only unpaired negatives and/or zeros
            if (hasZero) return 0;
            return neg.Last(); // least negative (single element)
        }

        return result;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n log n)** | Sorting negatives; n ≤ 13 so practically O(1) |
| **Space Complexity** | **O(n)** | Separated pos/neg lists |

Since n ≤ 13, even brute-force over all 2^13 = 8192 subsets would work — but greedy is cleaner.

---

## Why This Greedy is Optimal

- **Including a positive** multiplies result by x > 1 → always beneficial.
- **Including a negative pair** multiplies result by x*y > 0 (both negative → product positive) → beneficial.
- **Including a single unpaired negative** makes result negative → never beneficial if we already have a positive product.
- **Including zero** makes result 0 → only beneficial if all alternatives are negative.

The pairing of negatives by largest absolute value (most negative first) maximizes the contribution of each pair — standard greedy for maximizing products.

## Example Walkthrough

**Example 1**: `nums = [3,-1,-5,2,5,-9]`

```
pos = [3,2,5]     → product = 30
neg = [-9,-5,-1]  → sorted ascending: [-9,-5,-1]
  pair (-9,-5): product *= 45  → 30*45 = 1350
  unpaired -1: discard
result = 1350 ✓
```

**Example 2**: `nums = [-4,-5,-4]`

```
pos = []
neg = [-5,-4,-4]  → sorted: [-5,-4,-4]
  pair (-5,-4): product *= 20  → 20
  unpaired -4: discard
result = 20 ✓
```

## Edge Cases

| Input | Reason | Output |
|-------|--------|--------|
| `[-1]` | Single negative, no zero → return it | `-1` |
| `[0]` | Only zero → return 0 | `0` |
| `[0,-1]` | Zero beats single negative | `0` |
| `[0,-1,-2]` | Pair (-1,-2)=2, has product | `2` |

---

## Key Takeaway

"Maximum product subset" → **greedy by sign**:
- All positives in ✓
- Negative pairs in (sorted by abs desc) ✓
- Zeros and unpaired negatives out ✗ (unless forced)

The only tricky part is the "no valid product" edge case: when only zeros and/or a single negative exist, zero wins over a negative product.
