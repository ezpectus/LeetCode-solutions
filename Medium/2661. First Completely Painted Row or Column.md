# 2661. First Completely Painted Row or Column
## O(m × n) — Inverted Position Map + Row/Column Counters

---

## Problem Statement

- `arr` is a permutation of `[1, m×n]`. Paint `mat` cells in the order given by `arr`.
- Return the first index `i` where any full row or full column is painted.

---

## Core Idea — Invert Matrix + Count

**Preprocess**: build `pos[v] = (r, c)` — invert the matrix so we can look up any value's position in O(1).

**Simulate**: maintain `rowCnt[r]` and `colCnt[c]` — how many cells painted in each row/column. When `rowCnt[r] == n` or `colCnt[c] == m`, that row/column is complete → return current index.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int FirstCompleteIndex(int[] arr, int[][] mat) {
        int m = mat.Length, n = mat[0].Length;
        int[] posR = new int[m * n + 1];
        int[] posC = new int[m * n + 1];

        for (int r = 0; r < m; r++)
            for (int c = 0; c < n; c++) {
                posR[mat[r][c]] = r;
                posC[mat[r][c]] = c;
            }

        int[] rowCnt = new int[m];
        int[] colCnt = new int[n];

        for (int i = 0; i < arr.Length; i++) {
            int r = posR[arr[i]], c = posC[arr[i]];
            if (++rowCnt[r] == n || ++colCnt[c] == m)
                return i;
        }
        return -1;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(m × n)** | One pass to build pos, one pass over arr |
| **Space Complexity** | **O(m × n)** | pos arrays + row/col counters |

---

## Pitfalls & Edge Cases

- **Two separate arrays `posR`/`posC`** instead of a tuple — avoids struct allocation, cleaner in C#
- **`-1` never reached** — problem guarantees arr is a permutation of all values, so a row/col always completes

---

## Key Takeaway

"First complete row/column" → **invert matrix for O(1) lookup + per-row/column counters**. Fire when counter hits the required size.
