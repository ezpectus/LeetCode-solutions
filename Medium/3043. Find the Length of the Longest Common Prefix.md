# 3043. Find the Length of the Longest Common Prefix
## O((n+m) × D) — HashSet of Prefixes from arr1, Scan from arr2

---

## Problem Statement

- Find the maximum length of any common prefix between any `x ∈ arr1` and `y ∈ arr2`.
- A prefix of an integer = any number formed by its leading digits.
- Return 0 if no common prefix exists.

---

## Core Idea — Store All Numeric Prefixes of arr1, Look Up from arr2

**Step 1**: For every number in `arr1`, generate all its numeric prefixes (repeatedly divide by 10) and store in a HashSet.

**Step 2**: For every number `y` in `arr2`, try `y` itself, then `y//10`, then `y//100`, ... until we find a match in the HashSet. The longest match found across all `y` is the answer.

**Why this works**: A numeric prefix of `y` that also appears as a prefix of some `x ∈ arr1` means `x` and `y` share that prefix. We stop as soon as we find the first (longest) match for each `y` since we're dividing from the full number downward.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int LongestCommonPrefix(int[] arr1, int[] arr2) {
        var prefixes = new HashSet<int>();

        // Store all numeric prefixes of arr1 elements
        foreach (int x in arr1) {
            int cur = x;
            while (cur > 0) {
                prefixes.Add(cur);
                cur /= 10;
            }
        }

        int res = 0;
        // For each arr2 element, strip from the right until we find a prefix match
        foreach (int y in arr2) {
            int cur = y;
            while (cur > 0) {
                if (prefixes.Contains(cur)) {
                    res = Math.Max(res, cur.ToString().Length);
                    break; // longest match for this y found
                }
                cur /= 10;
            }
        }

        return res;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O((n+m) × D)** | D = max digits ≈ 9 (values up to 10⁸); each number generates at most D prefixes |
| **Space** | **O(n × D)** | HashSet stores at most n×D prefixes |

Effectively O(n+m) since D is constant (max 9 digits for values ≤ 10⁸).

---

## Example Walkthrough

**`arr1=[1,10,100], arr2=[1000]`**

Prefixes of arr1:
- 1 → {1}
- 10 → {10, 1}
- 100 → {100, 10, 1}
- HashSet = {1, 10, 100}

Processing arr2[0]=1000:
- 1000: not in set
- 100: **in set!** → len("100")=3 → res=3

Answer: **3** ✓

---

## Pitfalls & Edge Cases

- **`cur.ToString().Length`**: digit count of the matching prefix — this gives the prefix length directly
- **`break` after first match**: since we divide from largest to smallest, first match = longest prefix for this `y`
- **No common prefix**: `res` stays 0, returned correctly
- **Single-digit numbers**: while loop runs once (cur=x, then cur=0 → loop ends), correctly adding just `x` to prefixes

---

## Key Takeaway

"Longest common numeric prefix between two arrays" → **HashSet all prefixes of arr1, for each arr2 element strip from right until match**:

```
prefixes = {all truncations of arr1[i] by dividing by 10}
for y in arr2: strip y from right until y in prefixes → update res
```

O(n×D + m×D) ≈ O(n+m) since D≤9 is constant. Avoids O(n×m) pairwise comparison.
