# 2740. Find the Value of the Partition
## O(n log n) вҖ” Sort + Min Adjacent Difference

---

## Problem Statement

- Split `nums` into two non-empty arrays `nums1` and `nums2`.
- Minimize `|max(nums1) - min(nums2)|`.

---

## Core Idea вҖ” Sort, Then Min Gap Between Neighbors

**Key insight**: after sorting, the optimal partition is always a **prefix/suffix split** вҖ” `nums1 = nums[0..i]` and `nums2 = nums[i+1..n-1]` for some `i`. The value becomes `nums[i+1] - nums[i]` (no absolute value needed since sorted).

**Why no other partition is better?**

For any partition, `max(nums1) вүҘ` some element and `min(nums2) вүӨ` some other element. If `max(nums1)` and `min(nums2)` are not adjacent in the sorted order, there's a gap element between them that belongs to one of the arrays and makes the difference larger. The tightest possible gap is always between two consecutive elements in sorted order.

**Answer**: `min(nums[i+1] - nums[i])` for all `i`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int FindValueOfPartition(int[] nums) {
        Array.Sort(nums);
        int res = int.MaxValue;
        for (int i = 0; i < nums.Length - 1; i++)
            res = Math.Min(res, nums[i+1] - nums[i]);
        return res;
    }
}
```

## Why Prefix/Suffix Split is Always Optimal

After sorting: `aвӮҖ вүӨ aвӮҒ вүӨ ... вүӨ aвӮҷвӮӢвӮҒ`

For split at position `i`: `max(nums1) = aбөў`, `min(nums2) = aбөўвӮҠвӮҒ` вҶ’ value = `aбөўвӮҠвӮҒ - aбөў`.

For any non-contiguous partition: suppose `max(nums1) = aбөў` and `min(nums2) = aвұј` with `j > i+1`. Then `aбөўвӮҠвӮҒ` is in one of the arrays вҖ” if in `nums1` then `max(nums1) вүҘ aбөўвӮҠвӮҒ > aбөў` contradiction; if in `nums2` then `min(nums2) вүӨ aбөўвӮҠвӮҒ < aвұј` вҶ’ value = `aвұј - aбөў вүҘ aбөўвӮҠвӮҒ - aбөў`. Never better than adjacent split.

## Example Walkthrough

**`nums=[100,1,10]`** вҶ’ sorted: `[1,10,100]`

Adjacent differences: `10-1=9`, `100-10=90` вҶ’ min = **9** вң“

Split: `nums1=[1,10]`, `nums2=[100]` вҶ’ `|10-1|`... wait, `max(nums1)=10`, `min(nums2)=100`... no.

Actually split at `i=1`: `nums1=[1,10]`, `nums2=[100]` вҶ’ value = `100-10=90`.
Split at `i=0`: `nums1=[1]`, `nums2=[10,100]` вҶ’ value = `10-1=9` вң“

---

## Key Takeaway

"Minimize |max(A) - min(B)| partition" вҶ’ **sort + minimum adjacent difference**. The optimal split always puts a contiguous prefix in one array and the rest in the other вҖ” any non-adjacent split is provably worse.
