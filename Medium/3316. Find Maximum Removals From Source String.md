# 3316. Find Maximum Removals From Source String
## O(n × m) — DP on (source position, pattern matched)

---

## Problem Statement

- Remove characters at `targetIndices` from `source` (one by one), but `pattern` must remain a subsequence after each removal. Maximize removals.

---

## Core Idea — DP Tracking Pattern Progress

`dp[j]` = maximum removals achievable when we've matched exactly `j` characters of `pattern` from `source[0..i-1]`.

For each `source[i]`:
- **Remove it** (if `i ∈ targetIndices`): `dp[j] = max(dp[j], dp[j] + 1)` — stay at same pattern position, gain 1 removal.
- **Match it** (if `source[i] == pattern[j]`): `dp[j+1] = max(dp[j+1], dp[j])` — advance pattern, no removal.

These are mutually exclusive choices for each character — use a copy `new_dp` to avoid using the same character twice.

**Answer**: `dp[m]` = max removals while having matched all of `pattern`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxRemovals(string source, string pattern, int[] targetIndices) {
        int n = source.Length, m = pattern.Length;
        var targetSet = new HashSet<int>(targetIndices);
        const int NEG_INF = int.MinValue / 2;

        int[] dp = new int[m+1];
        Array.Fill(dp, NEG_INF);
        dp[0] = 0;

        for (int i = 0; i < n; i++) {
            bool isTarget = targetSet.Contains(i);
            int[] newDp = (int[])dp.Clone();

            for (int j = 0; j <= m; j++) {
                if (dp[j] == NEG_INF) continue;
                // Match source[i] to pattern[j]
                if (j < m && source[i] == pattern[j])
                    newDp[j+1] = Math.Max(newDp[j+1], dp[j]);
                // Remove source[i] (gain 1, stay at j)
                if (isTarget)
                    newDp[j] = Math.Max(newDp[j], dp[j] + 1);
            }
            dp = newDp;
        }
        return dp[m];
    }
}
```

## Example Walkthrough

**`source="bcda", pattern="d", targetIndices=[0,3]`**

- i=0 'b' (target): match? 'b'≠'d'. Remove: dp[0]=0→1.
- i=1 'c': nothing matches, not target. dp unchanged.
- i=2 'd': match pattern[0]='d': dp[1]=max(NEG_INF, dp[0])=1.
- i=3 'a' (target): remove: dp[1]=max(1, dp[1]+1)=2.

dp[1]=**2** ✓

---

## Pitfalls & Edge Cases

- **Clone dp before updates**: each source char can be used for AT MOST ONE action (match or remove), not both.
- **`NEG_INF` sentinel**: states not yet reachable shouldn't propagate.
- **Remove adds +1 to same state**: removing a target char that doesn't help pattern matching still gains a removal count.

---

## Key Takeaway

"Max removals while preserving pattern as subsequence" → **DP(pattern index) + copy per source char**:

```
for each source[i]:
    newDp = copy(dp)
    if match: newDp[j+1] = max(..., dp[j])
    if target: newDp[j] = max(..., dp[j]+1)
    dp = newDp
answer = dp[m]
```

O(n×m). Copy prevents reusing the same character.
