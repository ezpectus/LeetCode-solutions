# 3896. Minimum Operations to Transform Array into Alternating Prime
## O(n + L log log L) — Sieve of Eratosthenes + Per-Element Increment

---

## Problem Statement

- Even indices must be prime; odd indices must be non-prime.
- Only increment is allowed. Return minimum total increments.

---

## Core Idea — Sieve + Forward Search

For each element:
- **Even index:** find the next prime ≥ nums[i]. Cost = next_prime − nums[i].
- **Odd index:** find the next non-prime ≥ nums[i]. Cost = next_non_prime − nums[i].

**Gap bounds:**
- **Next prime gap:** ≤ 148 for numbers up to 10⁶ (much less for ≤ 10⁵). Safe sieve up to 2×10⁵.
- **Next non-prime:** for any odd prime p > 2, p+1 is even and composite. For p=2, next non-prime is 4. So gap ≤ 2.

Precompute primality with a sieve, then use a short while-loop per element.

---

## Full Optimal Implementation (C++)

```cpp
class Solution {
public:
    int minOperations(vector<int>& nums) {
        const int LIMIT = 200005;
        vector<bool> is_prime(LIMIT, true);
        is_prime[0] = is_prime[1] = false;

        for (int i = 2; i < LIMIT; i++) {
            if (is_prime[i]) {
                for (long long j = (long long)i * i; j < LIMIT; j += i)
                    is_prime[j] = false;
            }
        }

        int ops = 0;
        for (int i = 0; i < (int)nums.size(); i++) {
            int v = nums[i];
            if (i % 2 == 0) {
                // Even index: need prime
                while (!is_prime[v]) v++;
            } else {
                // Odd index: need non-prime (1 is non-prime; 4, 6, 8, 9, ... )
                while (is_prime[v]) v++;
            }
            ops += v - nums[i];
        }

        return ops;
    }
};
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(L log log L + n) | L=2×10⁵ sieve + while-loops nearly O(1) per element |
| Space Complexity | O(L) | Boolean sieve |

## Example Walkthrough

**Example 1:** `nums = [1, 2, 3, 4]`

| i | nums[i] | Role | Search | Cost |
|---|---------|------|--------|------|
| 0 | 1       | prime | 1→2 (prime) | 1 |
| 1 | 2       | non-prime | 2(prime)→3(prime)→4(non-prime) | 2 |
| 2 | 3       | prime | 3 (prime) | 0 |
| 3 | 4       | non-prime | 4 (non-prime) | 0 |

**Total = 3** ✓

---

## Pitfalls & Edge Cases

- **1 is non-prime** (`is_prime[1] = false`): at odd index, nums[i]=1 costs 0. ✓
- **nums[i] = 2 at odd index:** 2 is prime → 3 (prime) → 4 (non-prime). Cost = 2. ✓
- **LIMIT safety:** nums[i] ≤ 10⁵. Next prime after 10⁵ = 100003 << 200005. ✓
- **Non-prime gap:** For p > 2 prime, p+1 is always even and composite. Max non-prime gap is 2 (only at p=2). ✓

---

## Key Takeaway

Sieve + forward search is the standard pattern for "find next prime/non-prime from x." The while-loop is effectively O(1) per element since gaps are tiny. Total cost dominated by the O(L log log L) sieve.
