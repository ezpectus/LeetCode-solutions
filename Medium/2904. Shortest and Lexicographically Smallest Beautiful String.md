# 2904. Shortest and Lexicographically Smallest Beautiful String
## O(n²) — Try All Substrings Starting at Each Index

---

## Problem Statement

- Beautiful substring: exactly `k` ones.
- Find the shortest such substring; if tie in length, return lexicographically smallest.
- Return empty string if none exists.

---

## Core Idea — Fix Start, Extend to First k-th One

For each starting index `i`, extend rightward until we accumulate exactly `k` ones — that gives the **shortest** beautiful substring starting at `i`. Compare with current best: prefer shorter, then lexicographically smaller.

**Why stop at first k ones?** Any further extension would add more characters (making the substring longer) without reducing the one-count — not beneficial for minimizing length.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string ShortestBeautifulSubstring(string s, int k) {
        int n = s.Length;
        string best = "";

        for (int i = 0; i < n; i++) {
            int ones = 0;
            for (int j = i; j < n; j++) {
                ones += s[j] - '0';
                if (ones == k) {
                    string candidate = s.Substring(i, j - i + 1);
                    if (best.Length == 0
                        || candidate.Length < best.Length
                        || (candidate.Length == best.Length && candidate.CompareTo(best) < 0))
                        best = candidate;
                    break; // shortest from i found, no need to extend further
                }
            }
        }

        return best;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n²)** | n starting indices × at most n extensions |
| **Space** | **O(n)** | Storing best candidate substring |

With `n ≤ 100`: at most 10,000 iterations — trivially fast.

## Pitfalls & Edge Cases

- **`break` after finding k ones**: essential for getting shortest from each start position
- **`s.Length == 0` initial check**: handled by `best.Length == 0` condition
- **`CompareTo` for lexicographic comparison**: returns negative if `candidate < best`
- **No beautiful substring**: returns `""` (initial value of `best`)

---

## Key Takeaway

"Shortest then lexicographically smallest substring with exactly k ones" → **fix start index, extend to first k-th one, compare candidates**:

```
for each i: extend j until ones==k, compare s[i..j] with best
```

`break` after reaching `k` ensures we get the shortest possible substring starting at `i`. O(n²) is fine for n≤100.
