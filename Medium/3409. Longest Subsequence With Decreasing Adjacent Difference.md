# 3409. Longest Subsequence With Decreasing Adjacent Difference
## O(n × V) — DP with Suffix Max, V = max value = 300

---

## Problem Statement

- Find the longest subsequence where absolute differences between consecutive elements are **non-increasing**.
- `|seq[1]-seq[0]| >= |seq[2]-seq[1]| >= ... >= |seq[m]-seq[m-1]|`

---

## Core Idea — DP on (Last Value, Last Difference) with Suffix Max Optimization

**State**: `dp[v][d]` = maximum length of a valid subsequence ending with value `v` and last absolute difference `d`.

**Transition**: when appending `x` after some `v` with last diff `d`:
- New difference `nd = |x - v|`
- Must satisfy `nd <= d` (non-increasing)
- `dp[x][nd] = max over all v, d>=nd of (dp[v][d]) + 1`

**Key insight**: for a fixed `x` and `nd`, the only valid previous values are `v = x + nd` or `v = x - nd`. So instead of iterating all `v`:

```
best = max(suf[x+nd][nd], suf[x-nd][nd])   // two lookups!
dp[x][nd] = best + 1
```

where `suf[v][nd] = max(dp[v][nd], dp[v][nd+1], ..., dp[v][300])` is the suffix max.

After processing each `x`, update `suf[x]` by scanning right to left: `suf[x][d] = max(dp[x][d], suf[x][d+1])`.

**Why two values of v?** `|x - v| = nd` means `v = x ± nd`. These are the only candidates.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int LongestSubsequence(int[] nums) {
        const int V = 301;
        int[,] dp  = new int[V, V]; // dp[val][diff]
        int[,] suf = new int[V, V]; // suf[val][diff] = max(dp[val][diff..300])

        int res = 1;
        foreach (int x in nums) {
            int[] newDp = new int[V];

            for (int nd = 0; nd < V; nd++) {
                int best = 0;
                int v1 = x + nd, v2 = x - nd;
                if (v1 < V) best = Math.Max(best, suf[v1, nd]);
                if (nd > 0 && v2 > 0) best = Math.Max(best, suf[v2, nd]);
                if (nd == 0) best = Math.Max(best, suf[x, 0]); // same value
                newDp[nd] = best + 1;
                res = Math.Max(res, newDp[nd]);
            }

            // Update dp[x] and recompute suf[x]
            for (int nd = 0; nd < V; nd++)
                dp[x, nd] = Math.Max(dp[x, nd], newDp[nd]);

            suf[x, V-1] = dp[x, V-1];
            for (int d = V-2; d >= 0; d--)
                suf[x, d] = Math.Max(dp[x, d], suf[x, d+1]);
        }

        return res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × V)** | n elements × V=300 diffs per element |
| **Space** | **O(V²)** | dp and suf arrays, 300×300 each |

With n=10⁴, V=300: 3M operations — very fast.

The naive approach iterating all (v, d) pairs would be O(n × V²) = 900M. The two-candidate trick reduces it to O(n × V).

## Example Walkthrough

**`nums = [16, 6, 3]`**

Process x=16: all new_dp[nd]=1 (first element). suf[16] updated.

Process x=6:
- nd=10: v1=16, v2=-4. suf[16][10]=1. new_dp[10]=2.
- nd=0: suf[6][0]=0. new_dp[0]=1.

Process x=3:
- nd=3: v1=6, v2=0. suf[6][3]=? suf[6][3] = max over d>=3 of dp[6][d] = 2 (from nd=10 → suf[6][3]=2? No: suf[6][d] = max dp[6][d..300]. dp[6][10]=2, so suf[6][3]=2). new_dp[3] = 2+1 = 3.

Answer: **3** ✓

## Pitfalls & Edge Cases

- **`nd = 0` (same value)**: both `v = x+0 = x` and `v = x-0 = x` are the same, so check `suf[x][0]` once. Avoid double-counting with `nd > 0` guard.
- **Boundary checks**: `v1 = x+nd < V` and `v2 = x-nd > 0` (values are 1-indexed up to 300).
- **dp accumulates across elements**: same value `x` can appear multiple times; `dp[x][nd]` stores the best seen so far.
- **Suffix max must be recomputed after each x update**: suffix max is scanned right to left after updating `dp[x]`.

---

## Key Takeaway

"Longest subsequence with non-increasing adjacent differences" → **DP(last\_value, last\_diff) + suffix max for O(1) transition**:

```
for each x:
    for nd in [0..V]:
        v = x ± nd  (two candidates only)
        new_dp[nd] = suf[v][nd] + 1
    update dp[x], recompute suf[x]
```

The critical insight: `|x - v| = nd` has only two solutions for `v`. This reduces O(V²) per element to O(V).
