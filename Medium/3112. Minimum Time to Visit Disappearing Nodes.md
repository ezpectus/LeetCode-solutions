# 3112. Minimum Time to Visit Disappearing Nodes
## O((V + E) log V) — Dijkstra with Disappearance Constraint

---

## Problem Statement

- Weighted undirected graph. Node `i` disappears at time `disappear[i]` — can only be visited if arrival time < `disappear[i]`.
- Find minimum time to reach each node from node 0. Return -1 if unreachable.

---

## Core Idea — Standard Dijkstra with One Extra Condition

Dijkstra works directly. The only modification: when relaxing edge `(u, v, w)`, only update `dist[v]` if the new arrival time `dist[u] + w < disappear[v]`. Nodes we arrive at "on time or after" they disappear are treated as unreachable.

Note: condition is **strictly less than** (`< disappear[v]`) because if you arrive exactly when a node disappears, you can't visit it (as shown in Example 3).

**Starting node**: `dist[0] = 0`. Check that `0 < disappear[0]`; if not, node 0 itself is inaccessible from the start (but constraints guarantee disappear[i] ≥ 1 and we start at t=0, so this is always valid).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] MinimumTime(int n, int[][] edges, int[] disappear) {
        var graph = new List<(int to, int w)>[n];
        for (int i = 0; i < n; i++) graph[i] = new();
        foreach (var e in edges) {
            graph[e[0]].Add((e[1], e[2]));
            graph[e[1]].Add((e[0], e[2]));
        }

        int[] dist = new int[n];
        Array.Fill(dist, int.MaxValue);
        dist[0] = 0;

        var pq = new PriorityQueue<(int d, int u), int>();
        pq.Enqueue((0, 0), 0);

        while (pq.Count > 0) {
            var (d, u) = pq.Dequeue();
            if (d > dist[u]) continue;
            foreach (var (v, w) in graph[u]) {
                int nd = d + w;
                if (nd < dist[v] && nd < disappear[v]) {
                    dist[v] = nd;
                    pq.Enqueue((nd, v), nd);
                }
            }
        }

        return dist.Select(d => d == int.MaxValue ? -1 : d).ToArray();
    }
}
```

## Example Walkthrough

**`edges=[[0,1,2],[1,2,1],[0,2,4]], disappear=[1,1,5]`**

Start: dist=[0,∞,∞]. Push (0,0).

Pop (0,0): relax (0,1,w=2): nd=2. Check 2 < disappear[1]=1? No → skip. Relax (0,2,w=4): nd=4. 4 < 5 ✓ → dist[2]=4.

Pop (4,2): relax (2,1,w=1): nd=5. 5 < disappear[1]=1? No → skip. Relax (2,0,w=4): nd=8, 8>dist[0]=0 → skip.

dist=[0,∞,4] → **[0,-1,4]** ✓

## Pitfalls & Edge Cases

- **Strict inequality `nd < disappear[v]`**: arrival exactly at `disappear[v]` = node already gone. Use `<` not `<=`.
- **Node 0**: `dist[0]=0`, and since `disappear[0] >= 1`, node 0 is always reachable.
- **Multiple edges**: handled naturally by Dijkstra — shortest path is found regardless.
- **Disconnected graph**: unreachable nodes keep `dist = int.MaxValue` → mapped to -1.

---

## Key Takeaway

"Dijkstra with node disappearance" → **standard Dijkstra + one extra condition on relaxation**:

```
if nd < dist[v] AND nd < disappear[v]:
    update dist[v]
```

The disappearance constraint simply restricts which edges can be "used" to update distances — nodes that would be reached too late are treated as walls.
