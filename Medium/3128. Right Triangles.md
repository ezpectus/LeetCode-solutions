# 3128. Right Triangles
## O(m × n) — Row/Column Count Precomputation

---

## Problem Statement

- Given a 2D boolean matrix `grid`.
- A **right triangle** = 3 cells all with value `1`, where one cell shares its row with another and its column with the third (the right angle vertex).
- Return the total number of such right triangles.

---

## Core Idea — Fix the Right-Angle Vertex

**Key insight**: a right triangle has exactly one **right-angle vertex** — the cell that is both in the same row as one other `1` and the same column as the third `1`. For cell `(i,j)` with value `1`:

- Number of other `1`s in its row = `rowCount[i] - 1`
- Number of other `1`s in its column = `colCount[j] - 1`

If `(i,j)` is the right-angle vertex, the number of right triangles it forms = `(rowCount[i] - 1) × (colCount[j] - 1)`.

Summing over all `1`-cells gives the total count (each triangle is counted exactly once — at its right-angle vertex).

**Algorithm**:
1. Precompute `rowCount[i]` = number of `1`s in row `i`.
2. Precompute `colCount[j]` = number of `1`s in column `j`.
3. For each `(i,j)` where `grid[i][j] == 1`: add `(rowCount[i] - 1) × (colCount[j] - 1)`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long NumberOfRightTriangles(int[][] grid) {
        int m = grid.Length, n = grid[0].Length;

        int[] rowCount = new int[m];
        int[] colCount = new int[n];

        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                if (grid[i][j] == 1) { rowCount[i]++; colCount[j]++; }

        long total = 0;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                if (grid[i][j] == 1)
                    total += (long)(rowCount[i] - 1) * (colCount[j] - 1);

        return total;
    }
}
```

| **Space** | **O(m + n)** | Row and column count arrays |

## Why This Works — Right-Angle Vertex Counting

**Why does each triangle get counted exactly once?**

A right triangle has exactly one right-angle vertex (the cell at the "corner"). The other two cells extend along the same row and same column from this vertex. When we fix `(i,j)` as the right-angle vertex:
- We pick any of the `rowCount[i] - 1` other `1`s in row `i` as the horizontal leg endpoint.
- We pick any of the `colCount[j] - 1` other `1`s in column `j` as the vertical leg endpoint.

Each combination gives a distinct valid right triangle, and no triangle is double-counted because a triangle has exactly one right-angle vertex.

**Why subtract 1 from each count?**

`rowCount[i]` includes `(i,j)` itself — we need the count of *other* `1`s in the row. Same for `colCount[j]`.

**Why `long`?**

`rowCount[i]` and `colCount[j]` can each be up to `1000`, so their product is up to `10^6`. Summed over up to `10^6` cells, total can reach `10^12` — overflows `int`.

## Pitfalls & Edge Cases

- **Use `long` for the product and accumulator**: row/col counts up to 1000, product up to 10^6, total up to 10^12
- **Don't triple-count**: fixing the right-angle vertex guarantees each triangle counted exactly once — no need for deduplication
- **Cells with value 0 contribute nothing**: only iterate (or accumulate) over `1`-cells
- **`rowCount[i] - 1 = 0` or `colCount[j] - 1 = 0`**: no triangle possible from this vertex — the multiplication naturally yields 0

---

## Key Takeaway

"Count right triangles in a binary grid" → **fix the right-angle vertex, multiply row and column neighbor counts**:

```
rowCount[i] = # of 1s in row i
colCount[j] = # of 1s in column j

for each (i,j) with grid[i][j] == 1:
    total += (rowCount[i] - 1) * (colCount[j] - 1)
```

Each `1`-cell is the right-angle vertex of exactly `(rowCount[i]-1) × (colCount[j]-1)` right triangles. Two passes over the grid → O(mn) time, O(m+n) space.
