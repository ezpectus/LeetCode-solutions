# 2694. Event Emitter
## Map + Closure Unsubscribe

---

## Problem Statement

Design an `EventEmitter` with:
- `subscribe(eventName, callback)` ‚Üí returns `{ unsubscribe() }` object
- `emit(eventName, args?)` ‚Üí calls all subscribed callbacks in order, returns array of results
- Multiple subscribers per event, preserved in subscription order

---

## Core Idea ‚Äî Map of Callback Arrays + Closure for Unsubscribe

- `this.events: Map<string, Function[]>` stores callbacks per event name.
- `subscribe` pushes the callback into the array and returns an object whose `unsubscribe` closes over the **callback reference** to filter it out later.
- `emit` maps over the callbacks array, spreading `args` into each call.

**Why closure for unsubscribe?** The returned object captures `eventName` and `callback` in its closure ‚Äî no index needed. When called, it filters the exact function reference out of the array.

---

## Full Implementation (JavaScript)

```javascript
class EventEmitter {
    constructor() {
        this.events = new Map(); // eventName ‚Üí [callbacks]
    }

    subscribe(eventName, callback) {
        if (!this.events.has(eventName))
            this.events.set(eventName, []);
        this.events.get(eventName).push(callback);

        return {
            unsubscribe: () => {
                const cbs = this.events.get(eventName);
                this.events.set(eventName, cbs.filter(cb => cb !== callback));
                return undefined;
            }
        };
    }

    emit(eventName, args = []) {
        if (!this.events.has(eventName)) return [];
        return this.events.get(eventName).map(cb => cb(...args));
    }
}
```

---

## Complexity

| **Method** | **Time** | **Notes** |
|---|---|---|
| `subscribe` | O(1) | Array push |
| `emit` | O(k) | k = number of subscribers |
| `unsubscribe` | O(k) | `filter` over subscriber array |

---

## Why Reference Equality Works for Unsubscribe

The problem guarantees: *"no callbacks passed to subscribe are referentially identical"*. So filtering by `cb !== callback` always removes exactly the one subscriber registered in that `subscribe` call ‚Äî no risk of removing a different callback with the same body.

---

## Example Walkthrough

**Example 4**: subscribe `x=>x+1` (sub1), subscribe `x=>x+2` (sub2), unsubscribe sub1, emit `[5]`

```
After subscribes: events["firstEvent"] = [x=>x+1, x=>x+2]
sub1.unsubscribe():
  filter(cb => cb !== (x=>x+1)) ‚Üí [x=>x+2]
emit([5]): [x=>x+2].map(cb => cb(5)) = [7] ‚úì
```

---

## Pitfalls & Edge Cases

- **`args = []` default** ‚Äî `emit` with no second argument still spreads `[]`, so `cb(...[])` = `cb()` ‚úì
- **Event not yet registered in `emit`** ‚Äî `Map.has` check returns `[]` early
- **`unsubscribe` returns `undefined`** ‚Äî explicit `return undefined` satisfies the spec
- **Double unsubscribe** ‚Äî `filter` on an already-removed callback is a safe no-op

---

## Key Takeaway

Classic pub/sub in JS: **Map of arrays + closure unsubscribe**:

```javascript
this.events = new Map();           // event ‚Üí [callbacks]
subscribe  ‚Üí push + return {unsubscribe: () => filter}
emit       ‚Üí map callbacks, spread args, collect results
```

The closure captures the exact callback reference ‚Äî no need to track indices or generate IDs.
