# 2948. Make Lexicographically Smallest Array by Swapping Elements
## O(n log n) — Sort, Group by Connectivity, Assign Sorted Values

---

## Problem Statement

- Swap `nums[i]` and `nums[j]` if `|nums[i] - nums[j]| ≤ limit`, any number of times.
- Return the lexicographically smallest achievable array.

---

## Core Idea — Transitivity Creates Groups

**Key insight**: swapping is transitive. If `|a-b| ≤ limit` and `|b-c| ≤ limit`, then `a`, `b`, `c` can all be arranged among each other's positions. So elements form **groups** where any two can be swapped (possibly indirectly).

**How to find groups efficiently**: sort elements by value. In sorted order, consecutive elements that differ by ≤ `limit` belong to the same group (they can be directly swapped; by transitivity, the whole chain is one group). A group boundary occurs when consecutive sorted elements differ by > `limit`.

**Once groups are found**: within each group, we can place values in any order at the group's original positions. For lexicographic minimality, assign the **smallest values to the smallest indices**.

**Algorithm**:
1. Sort elements by value (keeping original indices).
2. Scan sorted list: group consecutive elements where adjacent difference ≤ `limit`.
3. For each group: collect original indices, sort them. Assign sorted values to sorted indices.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] LexicographicallySmallestArray(int[] nums, int limit) {
        int n = nums.Length;
        var sorted = nums.Select((v,i) => (v,i)).OrderBy(x => x.v).ToArray();
        int[] res = new int[n];

        int i = 0;
        while (i < n) {
            int j = i + 1;
            while (j < n && sorted[j].v - sorted[j-1].v <= limit) j++;

            // Group: sorted[i..j-1]
            var indices = sorted[i..j].Select(x => x.i).OrderBy(x => x).ToArray();
            var values  = sorted[i..j].Select(x => x.v).ToArray(); // already sorted

            for (int k = 0; k < indices.Length; k++)
                res[indices[k]] = values[k];

            i = j;
        }

        return res;
    }
}
```

## Example Walkthrough

**`nums=[1,7,6,18,2,1], limit=3`**

Sorted by value: `[(1,0),(1,5),(2,4),(6,2),(7,1),(18,3)]`

Group boundaries (consecutive diff > 3):
- `(1,0)→(1,5)`: diff=0 ≤ 3 → same group
- `(1,5)→(2,4)`: diff=1 ≤ 3 → same group
- `(2,4)→(6,2)`: diff=4 > 3 → **boundary**
- `(6,2)→(7,1)`: diff=1 ≤ 3 → same group
- `(7,1)→(18,3)`: diff=11 > 3 → **boundary**

Groups:
- Group 1: values=[1,1,2], original indices=[0,5,4] → sorted indices=[0,4,5] → assign [1,1,2]
  → `res[0]=1, res[4]=1, res[5]=2`
- Group 2: values=[6,7], original indices=[2,1] → sorted indices=[1,2] → assign [6,7]
  → `res[1]=6, res[2]=7`
- Group 3: values=[18], original indices=[3] → `res[3]=18`

Result: `[1,6,7,18,1,2]` ✓

---

## Pitfalls & Edge Cases

- **Transitivity via sorted order**: elements that can't swap directly (diff > limit) might still belong to different groups even if they could swap via intermediate elements. The sorted consecutive-diff check correctly identifies all reachable groups.
- **Single-element groups**: no swaps possible for that element → stays in place ✓
- **Duplicate values**: `diff=0 ≤ limit` always → duplicates always in the same group ✓

---

## Key Takeaway

"Minimum lexicographic array under transitive swaps within distance limit" → **sort by value, group consecutive elements with diff ≤ limit, assign sorted values to sorted indices within each group**:

```
sorted pairs → scan for group boundaries → for each group:
    sort original indices, assign smallest values to smallest positions
```

Sorting ensures minimum values go to earliest positions — optimal for lexicographic order.
