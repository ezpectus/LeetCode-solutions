# 2934. Minimum Operations to Maximize Last Elements in Arrays
## O(n) — Try Two Cases for Last Element

---

## Problem Statement

- Swap `nums1[i]` and `nums2[i]` at any index.
- Goal: `nums1[n-1] = max(nums1)` and `nums2[n-1] = max(nums2)`.
- Minimize total swaps, or return -1 if impossible.

---

## Core Idea — Two Cases for the Last Index

The last element `(nums1[n-1], nums2[n-1])` can either be **kept as-is** or **swapped**. This gives us two possible ceilings `(m1, m2)` that all other elements must respect.

**For each case**, iterate through positions `0..n-2`:
- If `nums1[i] ≤ m1` and `nums2[i] ≤ m2`: fine, no swap needed
- If `nums2[i] ≤ m1` and `nums1[i] ≤ m2`: swap needed (+1 op)
- Otherwise: impossible for this case → infinity

Take the minimum of both cases. If both are infinity → -1.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinOperations(int[] nums1, int[] nums2) {
        int n = nums1.Length;

        int Count(int m1, int m2, int extra) {
            int ops = extra;
            for (int i = 0; i < n - 1; i++) {
                int a = nums1[i], b = nums2[i];
                if (a <= m1 && b <= m2) continue;
                if (b <= m1 && a <= m2) { ops++; continue; }
                return int.MaxValue; // impossible
            }
            return ops;
        }

        int res1 = Count(nums1[n-1], nums2[n-1], 0); // no swap at last
        int res2 = Count(nums2[n-1], nums1[n-1], 1); // swap at last

        int ans = Math.Min(res1, res2);
        return ans == int.MaxValue ? -1 : ans;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — two passes of length n |
| **Space** | **O(1)** |

---

## Example Walkthrough

**`nums1=[1,2,7], nums2=[4,5,3]`**

**Case 1** (no swap at last): m1=7, m2=3
- i=0: 1≤7, 4>3 → try swap: 4≤7,1≤3 ✓ → ops=1
- i=1: 2≤7, 5>3 → try swap: 5≤7,2≤3 ✓ → ops=2
- Total=2

**Case 2** (swap at last): m1=3, m2=7, extra=1
- i=0: 1≤3, 4≤7 ✓
- i=1: 2≤3, 5≤7 ✓
- Total=1

`min(2,1) = 1` ✓

---

## Pitfalls & Edge Cases

- **`int.MaxValue` for impossible**: use it as sentinel, but avoid adding to it (overflow). Here we just return early or take `Math.Min` safely
- **Both cases try ALL possibilities**: there are exactly 2 choices for the last element → exhaustive 2-case check is complete

---

## Key Takeaway

"Minimize swaps so last elements are row-wise maxima" → **fix last element (2 choices), greedily check each prior position**:

```
for each i: no-swap OK? → 0 ops; swap OK? → 1 op; neither? → impossible
```

Two cases × O(n) scan = O(n) total.
