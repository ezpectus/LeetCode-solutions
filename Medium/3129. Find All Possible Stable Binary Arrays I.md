# 3129. Find All Possible Stable Binary Arrays I
## O(zero × one) — 3D DP with Run-Length Subtraction

---

## Problem Statement

- Build a binary array with exactly `zero` zeros and `one` ones.
- **Stable**: every consecutive run of identical digits has length ≤ `limit`.
- Return the count of stable arrays modulo 10⁹+7.

---

## Core Idea — DP on (zeros used, ones used, last digit) + Overcount Subtraction

**State**: `dp[i][j][d]` = number of valid stable arrays using exactly `i` zeros and `j` ones, ending with digit `d` (0 or 1).

**Transition — appending a 0** to form `dp[i][j][0]`:

```
dp[i][j][0] = dp[i-1][j][0] + dp[i-1][j][1]   ← append 0 to any valid (i-1, j) array
            - dp[i-limit-1][j][1]               ← subtract: run of limit+1 zeros at end
```

**Why subtract `dp[i-limit-1][j][1]`?**

The uncorrected sum counts arrays where the last `limit+1` elements are all zeros — invalid. These arrays look like:

```
[... ends with 1 | 0 repeated (limit+1) times]
      ↑
 dp[i-limit-1][j][1]   ← the part before the forbidden run
```

Subtract these to restore correctness. Symmetrically for digit 1.

**Base cases** (no mixing yet — only one digit type):

```
dp[i][0][0] = 1  for i in [1, limit]   (pure zero arrays of valid length)
dp[0][j][1] = 1  for j in [1, limit]   (pure one arrays of valid length)
```

These replace the sentinel trick. Explicitly setting base cases avoids the subtle double-counting bug that a `dp[0][0][0]=dp[0][0][1]=1` sentinel causes.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    const int MOD = 1_000_000_007;

    public int NumberOfStableArrays(int zero, int one, int limit) {
        long[,,] dp = new long[zero + 1, one + 1, 2];

        // Base: pure-zero arrays (valid only up to length limit)
        for (int i = 1; i <= Math.Min(zero, limit); i++)
            dp[i, 0, 0] = 1;

        // Base: pure-one arrays (valid only up to length limit)
        for (int j = 1; j <= Math.Min(one, limit); j++)
            dp[0, j, 1] = 1;

        for (int i = 0; i <= zero; i++) {
            for (int j = 0; j <= one; j++) {
                if (i == 0 || j == 0) continue; 

                // Append 0
                dp[i, j, 0] = (dp[i-1, j, 0] + dp[i-1, j, 1]) % MOD;
                if (i - limit - 1 >= 0)
                    dp[i, j, 0] = (dp[i, j, 0] - dp[i-limit-1, j, 1] + MOD) % MOD;

                // Append 1
                dp[i, j, 1] = (dp[i, j-1, 0] + dp[i, j-1, 1]) % MOD;
                if (j - limit - 1 >= 0)
                    dp[i, j, 1] = (dp[i, j, 1] - dp[i, j-limit-1, 0] + MOD) % MOD;
            }
        }

        return (int)((dp[zero, one, 0] + dp[zero, one, 1]) % MOD);
    }
}
```

| **Space Complexity** | **O(zero × one)** | 3D DP table (zero+1)×(one+1)×2 |

Constraints: zero, one ≤ 200 → ≤ 40K cells total.

## Why Explicit Base Cases (not a Sentinel)

A natural attempt is to use `dp[0][0][0] = dp[0][0][1] = 1` (empty array). This fails because:

```
dp[1][0][0] = dp[0][0][0] + dp[0][0][1] = 1 + 1 = 2  ← WRONG, should be 1
```

The empty array has **one** representation, not two. Setting both sentinel values to 1 double-counts it in every first-step transition.

Using only `dp[0][0][0] = 1` avoids double-counting the transition but then `dp[0][0][1] = 0`, which makes the subtraction for pure-zero runs fail:

```
dp[2][0][0] subtraction: dp[2-limit-1][0][1] = dp[0][0][1] = 0  ← should subtract 1
```

So `[0,0]` with `limit=1` is incorrectly counted as valid.

**Correct fix**: explicit base cases. `dp[i][0][0] = 1` for `i ∈ [1, limit]` directly encodes that pure-zero arrays are valid only up to length `limit`. No sentinel needed.

## Example Walkthrough

**Failing case**: `zero=2, one=1, limit=1`

Valid arrays: only `[0,1,0]`. Arrays `[0,0,1]` and `[1,0,0]` both have a run of 2 zeros > limit=1.

Base cases: `dp[1][0][0]=1` (just `[0]`), `dp[0][1][1]=1` (just `[1]`)

| (i,j) | dp[i][j][0] | dp[i][j][1] | array(s) |
|-------|-------------|-------------|----------|
| (1,1) | dp[0][1][0]+dp[0][1][1]=0+1=1 | dp[1][0][0]+dp[1][0][1]=1+0=1 | [1,0] and [0,1] |
| (2,0) | dp[1][0][0]+dp[1][0][1]-dp[0][0][1]=1+0-0=**0** | — | [0,0] correctly removed |
| (2,1) | dp[1][1][0]+dp[1][1][1]-dp[0][1][1]=1+1-1=**1** | dp[2][0][0]+dp[2][0][1]=0+0=**0** | only [0,1,0] ✓ |

Answer: `dp[2][1][0]+dp[2][1][1] = 1+0 = 1` ✓

---

## Pitfalls & Edge Cases

- **Explicit base cases for pure-digit arrays** — not a sentinel at `(0,0)`; see above
- **Subtraction only when `i-limit-1 >= 0`** — if negative, the forbidden run can't exist yet
- **Always `+MOD` before `%MOD`** — subtraction can go negative; `(val - sub + MOD) % MOD`
- **`limit >= zero`** — subtraction condition never triggers for zeros; `dp[i][0][0] = 1` for all `i ∈ [1,zero]` ✓

---

## Key Takeaway

"Count binary arrays with bounded runs" → **DP on (count0, count1, lastDigit) + run-length subtraction**:

```
dp[i][j][0] = (dp[i-1][j][0] + dp[i-1][j][1]) - dp[i-limit-1][j][1]
```

- Sum counts all extensions, subtraction removes forbidden `limit+1` runs
- Base cases encode single-digit arrays directly (`dp[i][0][0]=1` for `i≤limit`)
- Do **not** use a `(0,0)` sentinel — it causes subtle double-counting in the first transitions
