# 2829. Determine the Minimum Sum of a k-avoiding Array
## O(n) — Greedy: Take Smallest Valid Numbers

---

## Problem Statement

- Build an array of `n` distinct positive integers where no two elements sum to `k`.
- Minimize the total sum.

---

## Core Idea — Take 1, 2, 3, ... Skipping Conflicts

Greedily take the smallest available positive integer. Skip `x` only if `k - x` is already chosen (since `x + (k-x) = k` would violate the constraint).

```
for x = 1, 2, 3, ...:
    if (k - x) not in chosen:
        take x
```

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumSum(int n, int k) {
        var chosen = new HashSet<int>();
        int res = 0, x = 1;
        while (chosen.Count < n) {
            if (!chosen.Contains(k - x)) {
                chosen.Add(x);
                res += x;
            }
            x++;
        }
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n)** — at most ~2n iterations |
| **Space** | **O(n)** |

## Example Walkthrough

**`n=5, k=4`**:

| x | k-x | k-x in chosen? | take? | chosen |
|---|-----|----------------|-------|--------|
| 1 | 3 | No | ✓ | {1} |
| 2 | 2 | No | ✓ | {1,2} |
| 3 | 1 | Yes ✗ | skip | {1,2} |
| 4 | 0 | No | ✓ | {1,2,4} |
| 5 | -1 | No | ✓ | {1,2,4,5} |
| 6 | -2 | No | ✓ | {1,2,4,5,6} |

Sum = 1+2+4+5+6 = **18** ✓

---

## Key Takeaway

"Minimum sum distinct array avoiding pairs summing to k" → **greedy smallest-first, skip if complement already taken**. Simple and optimal since taking smaller numbers always minimizes sum.
