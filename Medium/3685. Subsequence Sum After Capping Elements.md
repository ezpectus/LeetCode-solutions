# 3685. Subsequence Sum After Capping Elements

## O(n × k) — Incremental DP with Frequency Tracking

---

## Problem Statement

- Given array `nums` of size `n` and positive integer `k`.
- **Capped array by x**: replace each `nums[i]` with `min(nums[i], x)`.
- For each `x` from 1 to `n`, check if subsequence from capped array can sum to exactly `k`.
- Return boolean array `answer` of size `n`.
- **Constraints**: 1 ≤ n ≤ 4000, 1 ≤ nums[i] ≤ n, 1 ≤ k ≤ 4000

---

## Core Idea — Incremental DP Processing Values in Order

**Key insight**: Process unique values in sorted order, incrementally building achievable sums.

**Strategy**:
1. Group nums by value with frequencies
2. Process values 1 to n in order
3. For each x, track:
   - `visited`: sums achievable from values < x
   - `tmp`: new sums when adding value x with its frequency
4. Once k is found, all future x values are valid

---

## Algorithm

```
freq = count frequency of each value in nums
keys = sorted unique values
visited = {0} // Achievable sums so far
findValid = -1 // First x where k is achievable

For x from 1 to n:
  if k already found:
    answer[x-1] = true
    continue
  
  // If x is a value in nums, add it with frequency
  if x in keys:
    tmp = new sums from adding x (1 to freq[x] times)
    combine with visited sums
    
    if k in tmp:
      findValid = x-1
    else:
      merge tmp into visited
  
  // Check if k achievable with remaining elements (all capped to x)
  for count from 1 to restLen:
    if x*count == k or visited.contains(k - x*count):
      answer[x-1] = true
      break

return answer
```

## Full Implementation (C#)

```csharp
public class Solution {
    public bool[] SubsequenceSumAfterCapping(int[] nums, int k) {
        int n = nums.Length;
        bool[] res = new bool[n];
        
        HashSet<int> visited = new();
        Dictionary<int, int> freq = new();
        foreach(int cur in nums) {
            freq.TryAdd(cur, 0);
            freq[cur]++;
        }
        
        List<int> keys = freq.Keys.ToList();
        keys.Sort();
        int restLen = n, idx = 0;
        int findValid = -1;
        
        HashSet<int> tmp = new();
        
        for(int x = 1; x <= n; x++) {
            if(findValid >= 0) {
                res[x-1] = true;
                continue;
            }
            
            // Process value x if it exists in nums
            if(idx < keys.Count && keys[idx] == x) {
                int cnt = freq[x];
                
                // Add sums using x (1 to cnt times)
                for(int m = 1; m <= cnt; m++) {
                    int start = x * m;
                    if(start > k) break;
                    
                    tmp.Add(start);
                    
                    // Combine with previous sums
                    foreach(int pre in visited) {
                        if(pre + start <= k)
                            tmp.Add(pre + start);
                    }
                }
                
                if(tmp.Contains(k)) {
                    findValid = x - 1;
                } else {
                    restLen -= cnt;
                    idx++;
                    visited.UnionWith(tmp);
                    tmp.Clear();
                }
            }
            
            // Check with remaining elements (all capped to x)
            if(restLen > 0) {
                for(int i = 1; i <= restLen; i++) {
                    int start = x * i;
                    if(start > k) break;
                    
                    if(start == k || visited.Contains(k - start)) {
                        res[x-1] = true;
                        break;
                    }
                }
            }
            
            if(findValid >= 0)
                res[x-1] = true;
        }
        
        return res;
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(n × k × U)** where U = unique values |
| **Space** | **O(k)** |

---

## Why This Works

**Incremental construction**:
- `visited`: tracks all achievable sums from processed values
- When processing value x:
  - Add new sums from using x (with its frequency)
  - Check if remaining elements (capped to x) can complete sum to k

**Early termination**:
- Once k found at some x, all larger x also valid (more flexibility)

**Remaining elements optimization**:
- Elements not yet processed are all > x, so capped to x
- Can use up to `restLen` copies of x
- Check if `x*i` or `visited + x*i` equals k

---

## Key Takeaway

**Process values incrementally in sorted order**. Track achievable sums, add new value with frequency, check if k reachable. Once found, propagate to all larger x. O(n × k × U) time.
