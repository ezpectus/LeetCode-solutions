# 3097. Shortest Subarray With OR at Least K II
## O(n × 32) — Sliding Window with Bit Frequency Array

---

## Problem Statement

- Find shortest subarray whose bitwise OR ≥ `k`.
- Return -1 if none exists.

---

## Core Idea — Sliding Window with Bit Count Tracking

OR is **monotone**: adding elements to a window can only set more bits (OR never decreases as window grows). This makes sliding window valid.

**Challenge**: when shrinking the window from the left, how do we efficiently recompute OR? OR doesn't support "undo" directly. **Solution**: maintain `bit_count[i]` = number of elements in the current window that have bit `i` set. Then `window_OR = OR of bits where bit_count[i] > 0`.

**Removing** `nums[left]`: decrement `bit_count[i]` for each set bit in `nums[left]`. The window OR updates automatically.

**Algorithm**:
1. Expand right: add `nums[right]` by incrementing `bit_count`.
2. While `window_OR >= k`: record length, shrink from left.
3. Return minimum length found, or -1.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumSubarrayLength(int[] nums, int k) {
        int n = nums.Length;
        int[] bitCount = new int[32];
        int res = int.MaxValue, left = 0;

        int WindowOr() {
            int or = 0;
            for (int i = 0; i < 32; i++)
                if (bitCount[i] > 0) or |= 1 << i;
            return or;
        }

        for (int right = 0; right < n; right++) {
            for (int i = 0; i < 32; i++)
                if ((nums[right] >> i & 1) == 1)
                    bitCount[i]++;

            while (WindowOr() >= k && left <= right) {
                res = Math.Min(res, right - left + 1);
                for (int i = 0; i < 32; i++)
                    if ((nums[left] >> i & 1) == 1)
                        bitCount[i]--;
                left++;
            }
        }

        return res == int.MaxValue ? -1 : res;
    }
}
```

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × 32)** = **O(n)** | Each element added/removed once; 32 bits per operation |
| **Space** | **O(32)** = **O(1)** | Fixed-size bit count array |

---

## Why Sliding Window Works for OR

OR is monotonically non-decreasing as we extend a window rightward. If `OR(left..right) >= k`, then `OR(left..right+1) >= k` too. Conversely, OR only decreases when we shrink from the left (by removing elements). This monotone property enables the two-pointer approach.

**Bit count array** is the key enabler: without it, we'd need to recompute OR from scratch on each shrink (O(n) per step). With it, each add/remove is O(32).

---

## Example Walkthrough

**`nums=[2,1,8], k=10`**

bit_count initially all zeros.

- right=0, nums[0]=2 (bit 1): bit_count[1]=1. OR=2 < 10.
- right=1, nums[1]=1 (bit 0): bit_count[0]=1. OR=3 < 10.
- right=2, nums[2]=8 (bit 3): bit_count[3]=1. OR=11 ≥ 10.
  - res=min(∞,3)=3. Remove nums[0]=2: bit_count[1]=0. OR=9 < 10. Stop shrinking.

Answer: **3** ✓

---

## Pitfalls & Edge Cases

- **`k = 0`**: any single element's OR ≥ 0 → answer is always 1 (as long as array non-empty). The while loop triggers immediately for every right.
- **No valid subarray**: `OR` of entire array < `k` → `res` stays `int.MaxValue` → return -1.
- **32-bit integers**: `nums[i] ≤ 10⁹ < 2³⁰`, so 30 bits suffice, but 32 is safe.

---

## Key Takeaway

"Shortest subarray with OR ≥ k" → **sliding window + bit frequency array for O(32) shrink**:

```
bit_count[i] = #elements in window with bit i set
window_OR    = bits where bit_count[i] > 0

expand right: bit_count[i]++ for set bits in nums[right]
shrink left:  bit_count[i]-- for set bits in nums[left], then left++
```

OR monotonicity enables sliding window; bit counts enable O(1) OR recomputation on shrink.
