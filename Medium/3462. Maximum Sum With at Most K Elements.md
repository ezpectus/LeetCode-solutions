# 3462. Maximum Sum With at Most K Elements
## O(n × m log m + N log N) — Per-Row Top Selection + Global Sort

---

## Problem Statement

- From row `i`, take at most `limits[i]` elements. Total taken ≤ k. Maximize sum.

---

## Core Idea — Greedy: Take Best Per Row, Then Best Overall

For each row `i`, the optimal strategy is to take the `limits[i]` largest values (those give maximum contribution if selected). Collect all these candidates into one pool, sort descending, take top `k`.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long MaxSum(int[][] grid, int[] limits, int k) {
        var candidates = new List<int>();
        for (int i = 0; i < grid.Length; i++) {
            var top = grid[i].OrderByDescending(x => x).Take(limits[i]);
            candidates.AddRange(top);
        }
        return candidates.OrderByDescending(x => x).Take(k).Sum(x => (long)x);
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time** | **O(n × m log m + N log N)** | N = Σ limits[i] ≤ n×m |
| **Space** | **O(N)** | Candidate list |

---

## Example Walkthrough

**`grid=[[5,3,7],[8,2,6]], limits=[2,2], k=3`**

Row 0 top 2: [7,5]. Row 1 top 2: [8,6]. Candidates: [7,5,8,6].
Sorted: [8,7,6,5]. Top 3: 8+7+6 = **21** ✓

---

## Key Takeaway

Per-row limit → take top `limits[i]` from each row. Global k limit → take top k from the merged pool. Two sorts, pure greedy.
