# 3122. Minimum Number of Operations to Satisfy Conditions
## O(m × n × 10) — Column-wise DP with 10-digit Value Tracking

---

## Problem Statement

- Given `m × n` grid where `grid[i][j] ∈ [0,9]`.
- Operations: change any cell to any non-negative integer.
- Requirements after operations:
  - Each column must be **uniform** (all cells in a column equal).
  - Adjacent columns must have **different** values.
- Return the **minimum number of operations**.

---

## Core Idea — Column DP with Best-Two Tracking

**Key observations**:
1. Since all values must be `0–9` (values > 9 would always cost more to change to than some value in `0–9`), each column gets assigned one digit `0–9`.
2. Cost of assigning digit `d` to column `j` = number of cells in column `j` that are NOT `d` = `m − count[j][d]`.
3. Adjacent columns must differ → this is a **1D DP** across columns.

**DP formulation**:

> `dp[j][d]` = minimum total operations to assign values to columns `0..j` such that column `j` is assigned digit `d`.

**Transition**:
```
dp[j][d] = cost(j, d) + min over all d' ≠ d of dp[j-1][d']
```

To compute `min over d' ≠ d of dp[j-1][d']` efficiently for all `d` simultaneously, track only the **two smallest** values in `dp[j-1]` (and which digit achieves the minimum). For digit `d`:
- If `d ≠ argmin1`: use `min1`
- If `d == argmin1`: use `min2`

This reduces the inner loop from O(10) to O(1), keeping the whole solution at O(n × 10).

**Answer**: `min over all d of dp[n-1][d]`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MinimumOperations(int[][] grid) {
        int m = grid.Length, n = grid[0].Length;

        // count[j][d] = how many rows in column j already have value d
        int[][] count = new int[n][];
        for (int j = 0; j < n; j++) {
            count[j] = new int[10];
            for (int i = 0; i < m; i++)
                count[j][grid[i][j]]++;
        }

        // cost[j][d] = operations to make column j all d = m - count[j][d]
        // dp[d] = min ops for columns 0..current, with current column = d
        int[] dp = new int[10];
        for (int d = 0; d < 10; d++)
            dp[d] = m - count[0][d];

        for (int j = 1; j < n; j++) {
            // Find best two values in dp (min1, min2) and which digit gives min1
            int min1 = int.MaxValue, min2 = int.MaxValue, bestD = -1;
            for (int d = 0; d < 10; d++) {
                if (dp[d] < min1) { min2 = min1; min1 = dp[d]; bestD = d; }
                else if (dp[d] < min2) min2 = dp[d];
            }

            int[] newDp = new int[10];
            for (int d = 0; d < 10; d++) {
                int prevBest = (d == bestD) ? min2 : min1;
                newDp[d] = (m - count[j][d]) + prevBest;
            }
            dp = newDp;
        }

        return dp.Min();
    }
}
```

| **Space** | **O(n × 10)** | Count array; dp is O(10) rolling |

## Why This Works

**Why only digits 0–9?**

All original values are in `[0,9]`. For any column, the cheapest assignment is always some digit already present in that column (or any digit if all values differ). Digits > 9 never appear, and assigning any digit > 9 to a column costs `m` (change everything) — always at least as expensive as the best digit in `[0,9]`. So the optimal assignment uses only digits `0–9`.

**Why column-uniform is free to enforce?**

We count the cost to make a column uniform as `m − count[j][d]` — the number of cells that disagree with the chosen digit `d`. We're allowed to change any cell, so we simply pick the cheapest digit for each column.

**Why track top-2 minimums?**

The transition `dp[j][d] = cost(j,d) + min_{d'≠d}(dp[j-1][d'])` requires the minimum of `dp[j-1]` excluding index `d`. By tracking the global minimum (`min1` at index `bestD`) and second minimum (`min2`):
- If `d ≠ bestD`: the minimum over all `d' ≠ d` is still `min1`.
- If `d == bestD`: the minimum over all `d' ≠ d` is `min2`.

This gives O(1) per transition instead of O(10).

## Example Walkthrough

**`grid = [[1,1,1],[0,0,0]]`** (m=2, n=3)

Column counts:

| col | 0s | 1s |
|-----|----|----|
| 0 | 1 | 1 |
| 1 | 1 | 1 |
| 2 | 1 | 1 |

**Initial dp** (column 0): cost = `2 - count[0][d]`
- dp[0]=1, dp[1]=1, dp[2..9]=2

**Column 1**: min1=1 (bestD=0 or 1, say 0), min2=1
- newDp[0] = cost(1,0) + min2 = 1 + 1 = 2 (can't reuse bestD=0)
- newDp[1] = cost(1,1) + min1 = 1 + 1 = 2
- newDp[2..9] = 2 + 1 = 3

dp = [2,2,3,3,3,3,3,3,3,3]

**Column 2**: min1=2 (bestD=0), min2=2
- newDp[0] = cost(2,0) + min2 = 1 + 2 = 3
- newDp[1] = cost(2,1) + min1 = 1 + 2 = 3
- newDp[2..9] = 2 + 2 = 4

min(dp) = **3** ✓

**`grid = [[1,0,2],[1,0,2]]`** (m=2, n=3)

Column counts: col0={1:2}, col1={0:2}, col2={2:2}

Initial dp: dp[1]=0, all others=2

Column 1: min1=0 (bestD=1), min2=2
- newDp[0] = cost(1,0)=0 + min1=0 = **0** (d=0 ≠ bestD=1 → use min1)
- newDp[1] = cost(1,1)=2 + min2=2 = 4
- others = 2+0=2

dp[0]=0, dp[1]=4, rest=2

Column 2: min1=0 (bestD=0), min2=2
- newDp[2] = cost(2,2)=0 + min1=0 = **0** (d=2 ≠ bestD=0)
- newDp[0] = cost(2,0)=2 + min2=2 = 4

min(dp) = **0** ✓

dp[1]=2, dp[2]=2, dp[3]=2, others=3

## Pitfalls & Edge Cases

- **Only one column**: no adjacent constraint → just find the most frequent value in that column; `min(dp)` handles this naturally
- **All cells already valid**: cost = 0 → dp correctly returns 0
- **Top-2 tracking when min1 = min2**: works fine — the `else if` ensures `min2` captures the second distinct minimum value (even if it's the same number)
- **`bestD` is the index of `min1`, not min2**: when `d == bestD`, fall back to `min2`; otherwise use `min1`

---

## Key Takeaway

"Uniform columns, adjacent columns differ, minimize changes" → **column DP over digits 0–9 with top-2 min tracking**:

```
dp[d] = cost to make column 0 all-d

for each column j from 1 to n-1:
    find min1, min2, bestD from dp
    newDp[d] = cost(j, d) + (d == bestD ? min2 : min1)
    dp = newDp

answer = min(dp)
```

Cost per column = `m − count[j][d]`. Top-2 tracking makes each column transition O(10) instead of O(100) → O(mn) total.
