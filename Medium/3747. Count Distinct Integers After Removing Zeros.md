# 3747. Count Distinct Integers After Removing Zeros
## O(log n) — Digit DP: Count Zero-Free Integers in [1, n]

---

## Problem Statement

- For every integer `x` from 1 to `n`, write down `strip_zeros(x)` (remove all '0' digits from decimal representation).
- Return the number of **distinct** values written down.
- 1 ≤ n ≤ 10¹⁵.

---

## Core Insight — The Result Set Is Exactly the Zero-Free Integers in [1, n]

**Claim:** `{strip_zeros(x) : 1 ≤ x ≤ n}` = `{y : 1 ≤ y ≤ n, y has no '0' digit}`.

**Proof of both directions:**

**⊇ (every zero-free y ≤ n appears):** For any zero-free integer y ≤ n, we have `strip_zeros(y) = y` (no zeros to remove). Since y ∈ [1, n], it appears in the output. ✓

**⊆ (every result is a zero-free integer ≤ n):**
- `strip_zeros(x)` has no '0' digits by definition. ✓
- `strip_zeros(x) ≤ x ≤ n`: removing digits from a number always yields a smaller or equal value (fewer decimal digits → strictly smaller; same digits → equal). So all results are ≤ n. ✓

**Conclusion:** The answer is simply the count of zero-free integers in [1, n].

---

## Algorithm — Digit DP to Count Zero-Free Integers ≤ n

A zero-free integer has every decimal digit in {1, 2, ..., 9}. For k-digit zero-free integers, all k digits are from {1,...,9} → exactly **9ᵏ** such numbers.

To count zero-free integers ≤ n (with d = len(n) digits):

1. Add `9¹ + 9² + ... + 9^(d-1)` for all shorter lengths.
2. For d-digit zero-free integers ≤ n: standard digit DP while "tight":
   - At each position `i` with digit `di = n[i]`:
     - If `di = 0`: no valid choice (need 1..di-1 = empty), stop.
     - Else: add `(di - 1) × 9^(d-1-i)` for free choices below `di`.
     - Continue tight with `di`. If at last digit, add 1 for n itself.

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public long CountDistinct(long n) {
        string s = n.ToString();
        int len = s.Length;
        long res = 0;

        // Count all zero-free integers with 1 to len-1 digits
        for (int k = 1; k < len; k++) {
            res += Pow9(k);
        }

        // Digit DP for len-digit zero-free integers ≤ n
        for (int i = 0; i < len; i++) {
            int di = s[i] - '0';

            if (di == 0) break; // No valid digit 1..di-1, can't continue tight either

            // Choose digit 1..di-1 at position i, then 9^(remaining) free choices
            res += (long)(di - 1) * Pow9(len - 1 - i);

            // If at last position and n is fully non-zero: count n itself
            if (i == len - 1) res++;
        }

        return res;
    }

    private long Pow9(int exp) {
        long result = 1;
        for (int i = 0; i < exp; i++) result *= 9;
        return result;
    }
}
```

## Complexity

| Metric | Value | Notes |
|---|---|---|
| Time Complexity | O(log n) | d = number of digits in n ≤ 16 for n ≤ 10¹⁵ |
| Space Complexity | O(log n) | String representation of n |

Essentially O(1) since d ≤ 16.

## Example Walkthrough

**Example 1:** `n = 10`

Digits of 10: `"10"`, len = 2.

- 1-digit zero-free: 9¹ = **9**
- 2-digit DP:
  - i=0: di=1 → res += (1-1) × 9¹ = 0. Continue tight.
  - i=1: di=0 → **break** (zero digit, no valid choice).

**res = 9** ✓

---

**Example 2:** `n = 3`

Digits: `"3"`, len = 1.

- No shorter lengths.
- 1-digit DP:
  - i=0: di=3 → res += (3-1) × 9⁰ = 2. Last digit: res += 1.

**res = 3** ✓ (integers 1, 2, 3)

---

**n = 20:**

- 1-digit: 9
- 2-digit DP:
  - i=0: di=2 → res += 1 × 9 = 9.
  - i=1: di=0 → break.
- res = 18

Manual check: 1–9 (nine 1-digit) + 11,12,...,19 (nine 2-digit zero-free) = 18 ✓

---

**n = 1,000,000,000,000,000 (10¹⁵):**

- len = 16
- Shorter lengths: 9 + 81 + ... + 9¹⁵ = (9¹⁶ - 9) / 8
- DP: i=0, di=1 → res += 0. i=1, di=0 → break.
- Total = (9¹⁶ - 9) / 8 ≈ 2.3 × 10¹⁴

---

## Pitfalls & Edge Cases

- **n = 1**: len=1, no shorter. DP: di=1, res += 0, last digit res+=1. Answer: 1 ✓

- **di = 0 in the middle**: e.g. n=103. At i=1 (digit '0'), break immediately — any number with '0' at this position is not zero-free.

- **n with no zeros**: e.g. n=999. DP runs all the way through, counting 8×81 + 8×9 + 8×1 + 1 = 729 two-digit-and-under zero-free + 729 = 999... wait: 9 + 81 + 729 = 819 total.

  Manual: 9¹ + 9² + 9³ = 9 + 81 + 729 = 819. DP adds: 8×81 + 8×9 + 8×1 + 1 = 648 + 72 + 8 + 1 = 729 = 9³. So total = 9 + 81 + 729 = 819 ✓

- **Overflow**: 9¹⁵ ≈ 2×10¹⁴, sum of all terms ≤ (9¹⁶−9)/8 ≈ 2.3×10¹⁵. Fits in `long` (max ~9.2×10¹⁸). ✓

## Key Takeaway

The problem disguises itself as a "simulation of removals" but reduces instantly to **counting zero-free integers in [1, n]** via two observations:

1. `strip_zeros(x) ≤ x` always (removing digits shrinks the number)
2. Every zero-free y ≤ n produces itself via `strip_zeros(y) = y`

These two facts together mean the result set equals exactly the zero-free integers in [1, n].

**Pattern recognition:** "Count integers in [1, n] with a digit constraint" → digit DP in O(log n). The key is seeing through the problem's framing to the underlying counting question.
