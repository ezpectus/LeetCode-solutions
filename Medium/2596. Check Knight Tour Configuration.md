# 2596. Check Knight Tour Configuration
## O(n²) — Position Lookup + Step Validation

---

## Problem Statement

- You are given an **n×n** integer matrix `grid` (3 ≤ n ≤ 7) with distinct values from `[0, n²-1]`.
- `grid[row][col]` = the step number when the knight visited cell `(row, col)`.
- A **valid configuration**: knight starts at `(0,0)` and visits every cell exactly once via legal knight moves.
- Return `true` if `grid` is a valid knight tour, `false` otherwise.

---

## Core Idea — Position Lookup + Step Validation

**Key insight**: Invert the grid into a `pos[]` array where `pos[k] = (row, col)` of step `k`. Then walk steps `0 → 1 → 2 → ...` and verify each consecutive pair is a valid knight move.

**A valid knight move** between two cells: the absolute row difference and absolute column difference must be exactly `{1, 2}` (one is 1, the other is 2 — in any order).

**Two checks needed**:
1. Step 0 must be at `(0, 0)` — knight starts top-left
2. For every step `k` → `k+1`: `{|Δrow|, |Δcol|} == {1, 2}`

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public bool CheckValidGrid(int[][] grid) {
        int n = grid.Length;
        var pos = new (int r, int c)[n * n];

        // Invert: pos[step] = (row, col)
        for (int r = 0; r < n; r++)
            for (int c = 0; c < n; c++)
                pos[grid[r][c]] = (r, c);

        // Must start at top-left
        if (pos[0] != (0, 0)) return false;

        // Validate each consecutive knight move
        for (int k = 1; k < n * n; k++) {
            int dr = Math.Abs(pos[k].r - pos[k-1].r);
            int dc = Math.Abs(pos[k].c - pos[k-1].c);
            if (!((dr == 1 && dc == 2) || (dr == 2 && dc == 1)))
                return false;
        }

        return true;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n²)** | One pass to build `pos[]`, one pass to validate n²-1 moves |
| **Space Complexity** | **O(n²)** | `pos[]` array of size n² |

**Optimal** — must read all n² cells at minimum.

## Why This Works — Example Walkthrough

**Example 2**: `grid = [[0,3,6],[5,8,1],[2,7,4]]`

Build `pos[]`:

| step | pos |
|------|-----|
| 0 | (0,0) |
| 1 | (1,2) |
| 2 | (2,0) |
| 3 | (0,1) |
| 4 | (2,2) |
| 5 | (1,0) |
| 6 | (0,2) |
| 7 | (2,1) |
| 8 | (1,1) |

Check step 7 → 8: `(2,1)` → `(1,1)` → `dr=1, dc=0` → `{1,0} ≠ {1,2}` ❌

Returns `false` ✓

---

## Pitfalls & Edge Cases

- **Check start position first** — `pos[0] != (0,0)` must be an early return; don't assume it
- **Order of `{dr,dc}` doesn't matter** — `(dr==1 && dc==2) || (dr==2 && dc==1)` covers both orientations of a knight move
- **Grid already guarantees all values unique and in range** — no need to validate that; just invert and check moves
- **n=3 minimum** — smallest valid board; a complete knight tour on 3×3 is actually impossible, so any 3×3 input will return `false`

---

## Key Takeaway

When a grid encodes a **sequence by value** (cell value = step number), **invert it** first:

- Build `pos[step] = (row, col)` in one O(n²) pass
- Then validate the sequence linearly instead of searching for each next step

Pattern: grid stores order → invert to get position-by-order → validate consecutive pairs.
