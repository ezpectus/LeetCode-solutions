# 2684. Maximum Number of Moves in a Grid
## O(m × n) — Column-by-Column DP

---

## Problem Statement

- Start at any cell in column 0. Move right to `(r-1, c+1)`, `(r, c+1)`, or `(r+1, c+1)` only if the destination value is **strictly greater**.
- Return the maximum number of moves possible.

---

## Core Idea — DP Column by Column

**State**: `dp[r]` = maximum moves to reach row `r` in the **current** column. Initially all 0 (every cell in column 0 is reachable with 0 moves). Cells that become unreachable are marked `-1`.

**Transition**: for each cell `(r, col)`, check the three possible predecessors in column `col-1`:

```
for dr in {-1, 0, +1}:
    pr = r + dr   (predecessor row)
    if dp[pr] >= 0 and grid[r][col] > grid[pr][col-1]:
        ndp[r] = max(ndp[r], dp[pr] + 1)
```

**Early termination**: if an entire column becomes unreachable (all `-1`), stop — no further moves possible.

**Answer**: `max(dp)` after processing all columns (or at the last reachable column).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int MaxMoves(int[][] grid) {
        int m = grid.Length, n = grid[0].Length;
        int[] dp = new int[m]; // all 0: column 0 fully reachable

        for (int col = 1; col < n; col++) {
            int[] ndp = new int[m];
            Array.Fill(ndp, -1);
            bool any = false;

            for (int r = 0; r < m; r++) {
                for (int dr = -1; dr <= 1; dr++) {
                    int pr = r + dr;
                    if (pr >= 0 && pr < m && dp[pr] >= 0
                        && grid[r][col] > grid[pr][col-1]) {
                        ndp[r] = Math.Max(ndp[r], dp[pr] + 1);
                        any = true;
                    }
                }
            }

            if (!any) break;
            dp = ndp;
        }

        return dp.Max();
    }
}
```

|---|---|---|
| **Time Complexity** | **O(m × n)** | Each cell processed once with 3 predecessor checks |
| **Space Complexity** | **O(m)** | Only two DP arrays of size m at a time |

## Example Walkthrough

**Example 1**: `grid = [[2,4,3,5],[5,4,9,3],[3,4,2,11],[10,9,13,15]]`

**col=0**: `dp = [0,0,0,0]`

**col=1** (values 4,4,4,9):
- `r=0`: pr∈{0,1}, grid[0][1]=4 > grid[0][0]=2 ✓ → ndp[0]=1; 4>5? ✗
- `r=1`: 4>2✓→1, 4>5✗, 4>3✓→1; ndp[1]=1
- `r=2`: 4>5✗, 4>3✓→1, 4>10✗; ndp[2]=1
- `r=3`: 4>3✓→1, 9>10✗; ndp[3]=1

`dp = [1,1,1,1]`

**col=2** (values 3,9,2,13):
- `r=1`: 9>4✓→2; ndp[1]=2
- `r=3`: 13>4✓→2, 13>9✓→2; ndp[3]=2

`dp = [-1,2,-1,2]`

**col=3** (values 5,3,11,15):
- `r=2`: 11>9✓(pr=1,dp=2)→3; ndp[2]=3
- `r=3`: 15>2✗, 15>13✓(pr=3,dp=2)→3; ndp[3]=3

`dp = [-1,-1,3,3]` → `max = 3` ✓

## Pitfalls & Edge Cases

- **`-1` for unreachable** — don't use `0` as "unreachable" since `0` means "reachable from column 0"; use `-1` sentinel
- **Starting column is fully reachable** — initialize `dp = [0,0,...,0]`, not `-1`
- **`max(dp)` at the end** — not `max(ndp)`; if the last processed column has all `-1`, `dp` still holds values from the previous column (due to early break)
- **Early break** — if no cell in `ndp` is reachable, no point continuing; saves time on sparse grids

---

## Key Takeaway

"Maximum steps moving strictly right with value constraint" → **column DP with reachability tracking**:

- `dp[r]` = max moves to reach current column at row `r`
- `-1` = unreachable
- Transition: 3 predecessors per cell, take max of valid ones + 1
- Process left → right; answer = max over final dp array
