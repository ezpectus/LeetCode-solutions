# 2575. Find the Divisibility Array of a String
## O(n) — Rolling Modulo on Prefix Value

---

## Problem Statement

- You are given a **0-indexed** string `word` of length `n` (1 ≤ n ≤ 10⁵) consisting of digits, and a positive integer `m` (1 ≤ m ≤ 10⁹).
- The **divisibility array** `div` has length `n` such that:
  - `div[i] = 1` if the numeric value of `word[0..i]` is divisible by `m`
  - `div[i] = 0` otherwise
- Return the divisibility array of `word`.

---

## Core Idea — Rolling Modulo

**Key insight**: the numeric value of `word[0..i]` is too large to store directly (up to 10¹⁰⁵). But we only need its **remainder mod m**.

When we extend the prefix by one digit `d`:

```
value(word[0..i]) = value(word[0..i-1]) * 10 + d
```

So:

```
rem[i] = (rem[i-1] * 10 + d) % m
```

This is a standard **rolling modulo** — O(1) per character, O(n) total. No big integer arithmetic needed.

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public int[] DivisibilityArray(string word, int m) {
        int n = word.Length;
        int[] div = new int[n];
        long rem = 0;

        for (int i = 0; i < n; i++) {
            rem = (rem * 10 + (word[i] - '0')) % m;
            div[i] = rem == 0 ? 1 : 0;
        }

        return div;
    }
}
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Single pass, one mod operation per character |
| **Space Complexity** | **O(n)** | Output array only; O(1) extra |

**Optimal** — a single linear pass is the theoretical minimum; you must read every character at least once.

---

## Why This Works — Example Walkthrough

**Example 1**: `word = "998244353"`, `m = 3`

| i | digit | rem = (rem * 10 + d) % 3 | div[i] |
|---|-------|--------------------------|--------|
| 0 | 9 | (0 * 10 + 9) % 3 = **0** | 1 |
| 1 | 9 | (0 * 10 + 9) % 3 = **0** | 1 |
| 2 | 8 | (0 * 10 + 8) % 3 = **2** | 0 |
| 3 | 2 | (2 * 10 + 2) % 3 = **1** | 0 |
| 4 | 4 | (1 * 10 + 4) % 3 = **2** | 0 |
| 5 | 4 | (2 * 10 + 4) % 3 = **0** | 1 |
| 6 | 3 | (0 * 10 + 3) % 3 = **0** | 1 |
| 7 | 5 | (0 * 10 + 5) % 3 = **2** | 0 |
| 8 | 3 | (2 * 10 + 3) % 3 = **2** | 0 |

Output: `[1,1,0,0,0,1,1,0,0]` ✓

**Example 2**: `word = "1010"`, `m = 10`

| i | digit | rem | div[i] |
|---|-------|-----|--------|
| 0 | 1 | 1 | 0 |
| 1 | 0 | 10 % 10 = **0** | 1 |
| 2 | 1 | 1 | 0 |
| 3 | 0 | 10 % 10 = **0** | 1 |

Output: `[0,1,0,1]` ✓

**Key insight on why rem resets naturally**: when `rem == 0` after some prefix, the next step starts from `(0 * 10 + d) % m = d % m` — the rolling formula handles this automatically with no special casing.

---

## Pitfalls & Edge Cases

- **Use `long` for `rem`** — `rem` can be up to `m - 1 ≤ 10⁹ - 1`; multiplying by 10 gives up to ~10¹⁰ which overflows `int` but fits in `long`
- **`word[i] - '0'`** — standard char-to-digit conversion; don't forget this, `word[i]` is a char not an int
- **m = 1** — every prefix is divisible by 1; `rem` is always 0; output is all 1s — handled correctly with no special case
- **Leading zeros in word** — e.g. `word = "001"`, `m = 1`; the rolling formula treats this correctly as numeric values 0, 0, 1
- **Single character** — n = 1; one iteration, returns `[1]` if `word[0] == '0'` or digit divisible by m, else `[0]`

---

## Key Takeaway

This problem is a direct application of **modular arithmetic on prefix values**:

- You can't store the actual number (up to 10¹⁰⁵ digits)
- But `(prev * 10 + digit) % m` gives the running remainder in O(1)
- `div[i] = 1` iff that remainder is 0

One pass, one variable, zero extra space — as simple as it gets. The only trap is using `long` to avoid overflow in the intermediate `rem * 10` step.
