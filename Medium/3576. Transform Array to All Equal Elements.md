# 3576. Transform Array to All Equal Elements  
## O(n) — Greedy Adjacent Flips Towards Target Value

---

## Problem Statement

You are given:

- An integer array `nums` of length `n`
- Each `nums[i]` is either `1` or `-1`
- An integer `k`

You may perform the following operation **at most k times**:

- Choose an index `i` with `0 ≤ i < n - 1`
- Multiply both `nums[i]` and `nums[i + 1]` by `-1`

You may choose the same index multiple times.

Your task:

> Determine whether it is possible to make **all elements of the array equal** (all `1` or all `-1`) using at most `k` operations.

Return `true` if possible, otherwise `false`.

---

## Core Idea — Greedy Fix from Left to Right for Each Target

We want all elements to become either:

- all `1`, or
- all `-1`

We can try both targets independently and see if either is achievable within `k` operations.

### Key observation

Consider trying to make the array **all equal to some target value `T`** (`T = 1` or `T = -1`):

- We process the array from **left to right**
- At position `i` (for `0 ≤ i < n - 1`):
  - If `nums[i] == T`, do nothing
  - If `nums[i] != T`, we **must** flip at index `i`:
    - This operation flips `nums[i]` and `nums[i+1]`
    - After flipping, `nums[i]` becomes correct (`T`)
- We continue until `i = n - 2`
- At the end, we check if `nums[n - 1] == T`
  - If yes → we successfully transformed the whole array into `T`
  - If no → it is **impossible** to make all elements equal to `T` using any sequence of operations

This greedy is correct because:

- Once we leave index `i`, we never touch it again (operations only affect `i` and `i+1`)
- So the only chance to fix `nums[i]` is when we are at `i`
- If `nums[i] != T`, we are forced to flip at `i` to fix it

We do this process twice:

- Once with `T = 1`
- Once with `T = -1`

For each target, we count the number of flips (operations).  
If the transformation is possible and the number of operations ≤ `k`, then the answer is `true`.

If neither target is achievable within `k` operations, the answer is `false`.

---

## Algorithm

1. Define a helper function `OpsToMake(nums, T)`:
   - Make a copy of `nums`
   - Set `ops = 0`
   - For `i` from `0` to `n - 2`:
     - If `a[i] != T`:
       - Flip `a[i]` and `a[i+1]` (multiply by `-1`)
       - Increment `ops`
   - After the loop:
     - If `a[n - 1] != T`, return `∞` (impossible)
     - Else return `ops`
2. Compute:
   - `ops1 = OpsToMake(nums, 1)`
   - `ops2 = OpsToMake(nums, -1)`
3. Let `best = min(ops1, ops2)`
4. If `best ≤ k`, return `true`, else `false`

---

## Full Implementation (C#)

```csharp
public class Solution {
    public bool CanMakeEqual(int[] nums, int k) {
        long opsToAllOne = OpsToMake(nums, 1);
        long opsToAllMinusOne = OpsToMake(nums, -1);

        long best = Math.Min(opsToAllOne, opsToAllMinusOne);
        return best <= k;
    }

    private long OpsToMake(int[] nums, int target) {
        int n = nums.Length;
        int[] a = new int[n];
        Array.Copy(nums, a, n);

        long ops = 0;

        for (int i = 0; i < n - 1; i++) {
            if (a[i] != target) {
                // flip a[i] and a[i+1]
                a[i] = -a[i];
                a[i + 1] = -a[i + 1];
                ops++;
            }
        }

        if (a[n - 1] != target)
            return long.MaxValue; // impossible for this target

        return ops;
    }
}
```

## Complexity

- Time Complexity	O(n) (two passes over the array)
- Space Complexity	O(n) (copy of the array per simulation)


## Pitfalls & Edge Cases
- You must simulate separately for target 1 and target -1
- Greedy is left-to-right; once you pass index i, you never touch it again
- If the last element cannot be made equal to the target, that target is impossible
- k is just a cap: we compare the minimal operations with k
- Using long.MaxValue (or any large sentinel) to represent “impossible” for a target is convenient

## Example
```
Example 1
nums = [1, -1, 1, -1, 1], k = 3

Try target 1:
i = 0: a[0] = 1 → ok

i = 1: a[1] = -1 ≠ 1 → flip at 1
a = [1, 1, -1, -1, 1], ops = 1

i = 2: a[2] = -1 ≠ 1 → flip at 2
a = [1, 1, 1, 1, 1], ops = 2
i = 3: a[3] = 1 → ok

last element a[4] = 1 = target → possible with 2 ops
2 ≤ k (=3) → true.
```
# Key Takeaway
The operation only affects adjacent pairs, so a left-to-right greedy is natural:

- Fix each position as you go
- If it doesn’t match the target, you are forced to flip at that index
- Check feasibility for both targets (1 and -1)
- Compare minimal required operations with k
This yields a clean O(n) solution that matches all constraints and examples.
