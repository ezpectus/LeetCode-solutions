# 3698. Split Array With Minimum Difference

## O(n) — Prefix Sum with Validity Check

---

## Problem Statement

- Given integer array `nums`.
- Split into two subarrays: `left` (strictly increasing) and `right` (strictly decreasing).
- Return minimum |sum(left) - sum(right)|, or -1 if no valid split.
- **Constraints**: 2 ≤ nums.length ≤ 10⁵, 1 ≤ nums[i] ≤ 10⁵

---

## Core Idea — Precompute Valid Prefixes and Suffixes

**Key observations**:
1. Left part must be strictly increasing from start
2. Right part must be strictly decreasing to end
3. Valid split at index i means:
   - nums[0..i] is strictly increasing
   - nums[i+1..n-1] is strictly decreasing

**Strategy**:
- Precompute `leftSum[i]` = sum of valid increasing prefix ending at i
- Precompute `rightSum[i]` = sum of valid decreasing suffix starting at i
- Try all split points where both parts valid

---

## Algorithm

```
1. Build leftSum array:
   - leftSum[0] = nums[0]
   - For i from 1 to n-1:
     - If nums[i] > nums[i-1]: leftSum[i] = leftSum[i-1] + nums[i]
     - Else: leftSum[i] = -1 (invalid, break)

2. Build rightSum array (right-to-left):
   - rightSum[n-1] = nums[n-1]
   - For i from n-2 down to 0:
     - If nums[i] > nums[i+1]: rightSum[i] = rightSum[i+1] + nums[i]
     - Else: rightSum[i] = -1 (invalid, break)

3. Find minimum difference:
   - For each i from 0 to n-1:
     - left = leftSum[i]
     - right = rightSum[i+1] (if exists)
     - If both valid: update result with |left - right|

4. Return result or -1 if no valid split
```

---

## Full Implementation (C#)

```csharp
public class Solution {
    public long SplitArray(int[] nums) {
        int n = nums.Length;
        long[] leftSum = new long[n];
        long[] rightSum = new long[n];
        
        // Initialize as invalid
        for (int i = 0; i < n; i++) {
            leftSum[i] = -1;
            rightSum[i] = -1;
        }
        
        // Build leftSum (strictly increasing from start)
        leftSum[0] = nums[0];
        for (int i = 1; i < n; i++) {
            if (nums[i] > nums[i - 1]) {
                leftSum[i] = leftSum[i - 1] + nums[i];
            } else {
                break; // No longer increasing, stop
            }
        }
        
        // Build rightSum (strictly decreasing from end)
        rightSum[n - 1] = nums[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            if (nums[i] > nums[i + 1]) {
                rightSum[i] = rightSum[i + 1] + nums[i];
            } else {
                break; // No longer decreasing, stop
            }
        }
        
        // Find minimum difference
        long result = long.MaxValue;
        
        for (int i = 0; i < n; i++) {
            long left = leftSum[i];
            long right = (i == n - 1) ? 0 : rightSum[i + 1];
            
            // If left part invalid, no point continuing
            if (left < 0) break;
            
            // If right part valid, calculate difference
            if (right > 0) {
                result = Math.Min(result, Math.Abs(left - right));
            }
        }
        
        return result == long.MaxValue ? -1 : result;
    }
}
```

## Example Walkthrough

### Example 1: `nums = [1,3,2]`

**leftSum**:
- i=0: leftSum[0] = 1
- i=1: 3 > 1 → leftSum[1] = 1 + 3 = 4
- i=2: 2 < 3 → break, leftSum[2] = -1

**rightSum**:
- i=2: rightSum[2] = 2
- i=1: 3 > 2 → rightSum[1] = 2 + 3 = 5
- i=0: 1 < 3 → break, rightSum[0] = -1

**Split attempts**:
- i=0: left=1, right=rightSum[1]=5 → |1-5|=4
- i=1: left=4, right=rightSum[2]=2 → |4-2|=2 ✓
- i=2: left=-1 → invalid

**Result**: 2 ✓

### Example 2: `nums = [1,2,4,3]`

**leftSum**:
- [1, 3, 7, -1] (stops at index 2 because 4>3 but 3<4 breaks)

Actually wait, let me recalculate:
- i=0: leftSum[0] = 1
- i=1: 2 > 1 → leftSum[1] = 3
- i=2: 4 > 2 → leftSum[2] = 7
- i=3: 3 < 4 → break, leftSum[3] = -1

**rightSum**:
- i=3: rightSum[3] = 3
- i=2: 4 > 3 → rightSum[2] = 7
- i=1: 2 < 4 → break, rightSum[1] = -1
- i=0: rightSum[0] = -1

**Split attempts**:
- i=0: left=1, right=rightSum[1]=-1 → invalid
- i=1: left=3, right=rightSum[2]=7 → |3-7|=4 ✓
- i=2: left=7, right=rightSum[3]=3 → |7-3|=4 ✓

**Result**: 4 ✓

### Example 3: `nums = [3,1,2]`

**leftSum**:
- i=0: leftSum[0] = 3
- i=1: 1 < 3 → break, leftSum[1] = -1

**rightSum**:
- i=2: rightSum[2] = 2
- i=1: 1 < 2 → break, rightSum[1] = -1

**Split attempts**:
- i=0: left=3, right=rightSum[1]=-1 → invalid
- i=1: left=-1 → invalid

**Result**: -1 ✓

## Key Insights

**Why early break in left/right loops?**
- Once sequence is no longer increasing/decreasing, no point continuing
- All subsequent indices will be invalid for that direction

**Why `right > 0` check?**
- rightSum[i] = -1 means invalid
- rightSum[i] = 0 means empty (when i = n-1)
- We need rightSum > 0 for valid split

**Edge case**: When i = n-1
- Left = entire array (must be increasing)
- Right = empty (set to 0, but then right > 0 fails, so no valid split here)

---

## Key Takeaway

**Precompute valid increasing prefix sums and decreasing suffix sums**. Try all split points where both parts valid. O(n) time with two-pass preprocessing.
