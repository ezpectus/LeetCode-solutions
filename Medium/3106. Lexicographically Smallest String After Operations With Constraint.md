# 3106. Lexicographically Smallest String After Operations With Constraint
## O(n) — Greedy Left-to-Right

---

## Problem Statement

- `distance(s1, s2)` = sum of minimum cyclic distances between corresponding characters.
- Find lexicographically smallest `t` such that `distance(s, t) ≤ k`.

---

## Core Idea — Greedy: Minimize Each Character Left to Right

To minimize lexicographically, change earlier characters first. For each position `i`:

1. Compute minimum cyclic distance from `s[i]` to `'a'`: `d = min(ord(s[i])-ord('a'), 26-(ord(s[i])-ord('a')))`.
2. If `d ≤ k`: set `t[i] = 'a'`, spend `d` from budget. Continue to next position.
3. If `d > k`: we can't reach `'a'`. Spend all remaining `k` to move `s[i]` as close to `'a'` as possible. Stop (no budget left for later positions).

**Why greedy works?** Earlier positions contribute more to lexicographic order. Spending budget on position `i` to get `'a'` is always at least as good as spending it on a later position (where 'a' is also the target but contributes less).

**Moving toward 'a'**: if we have `k` budget and can't reach 'a', move `k` steps in the direction that gets closer to 'a'. Since we're going toward 'a' (smaller character), this is `chr(ord(s[i]) - k)` (move down the alphabet by k).

---

## Full Optimal Implementation (C#)

```csharp
public class Solution {
    public string GetSmallestString(string s, int k) {
        char[] res = s.ToCharArray();
        for (int i = 0; i < s.Length && k > 0; i++) {
            int dist = Math.Min(s[i] - 'a', 26 - (s[i] - 'a'));
            if (dist <= k) {
                k -= dist;
                res[i] = 'a';
            } else {
                res[i] = (char)(s[i] - k);
                k = 0;
            }
        }
        return new string(res);
    }
}
```

## Cyclic Distance to 'a'

Characters are arranged in a cycle: a-b-c-...-z-a. Minimum distance from `c` to `'a'`:
- Going left (decreasing): `ord(c) - ord('a')` steps.
- Going right (via z): `26 - (ord(c) - ord('a'))` steps.
- Minimum of both.

For `'z'`: left distance = 25, right distance = 1 → d=1.
For `'n'`: left = 13, right = 13 → d=13.

## Example Walkthrough

**`s="zbbz", k=3`**

| i | s[i] | d to 'a' | k before | action | k after |
|---|------|----------|----------|--------|---------|
| 0 | 'z' | min(25,1)=1 | 3 | set 'a', spend 1 | 2 |
| 1 | 'b' | min(1,25)=1 | 2 | set 'a', spend 1 | 1 |
| 2 | 'b' | 1 | 1 | set 'a', spend 1 | 0 |
| 3 | 'z' | 1 | 0 | no budget, keep | 0 |

Result: **"aaaz"** ✓

**`s="xaxcd", k=4`**

| i | s[i] | d | k | action | k' |
|---|------|---|---|--------|----|
| 0 | 'x' | min(23,3)=3 | 4 | set 'a', spend 3 | 1 |
| 1 | 'a' | 0 | 1 | set 'a', spend 0 | 1 |
| 2 | 'x' | 3 | 1 | can't reach 'a'. move left by 1: 'x'-1='w' | 0 |
| 3,4 | | | 0 | no budget, keep | |

Result: **"aawcd"** ✓

## Pitfalls & Edge Cases

- **`k=0`**: loop body never executes (due to `k > 0` condition) → return `s` unchanged ✓
- **Moving toward 'a' when budget insufficient**: subtract `k` from `ord(s[i])` to move left. This is correct because `'a'` is to the left of most characters, and we've already established `d > k` means the shorter path is leftward (otherwise d would equal the rightward distance and be ≤ k).
- **Special case 'a' itself**: d=0, spend 0, keep 'a' ✓

---

## Key Takeaway

"Lexicographically smallest string within cyclic distance budget" → **greedy left-to-right: spend budget on each character to reach 'a'; if can't, move as far as possible and stop**:

```
for each i:
    d = min_cyclic_dist(s[i], 'a')
    if d <= k: set 'a', k -= d
    else: set s[i]-k, k=0, break
```

O(n), no lookahead needed. Greedily minimize earlier characters first.
