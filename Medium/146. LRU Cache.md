# 146. LRU Cache  
*O(1) average time for get & put — HashMap + Doubly Linked List*

---

## Problem Statement

Design a **Least Recently Used (LRU) Cache** that supports the following operations in **O(1)** average time complexity:

- `LRUCache(int capacity)` — Initialize the cache with positive size `capacity`.
- `int get(int key)` — Return the value of the key if it exists, otherwise return `-1`.
- `void put(int key, int value)` — 
  - Update the value of the key if the key exists.
  - Otherwise, add the key-value pair to the cache.
  - If the number of keys exceeds the capacity from this operation, **evict** the **least recently used** key.

**Example**:

```text
LRUCache lRUCache = new LRUCache(2);
lRUCache.put(1, 1);   // cache is {1=1}
lRUCache.put(2, 2);   // cache is {1=1, 2=2}
lRUCache.get(1);      // return 1
lRUCache.put(3, 3);   // LRU key was 2, evicts key 2, cache is {1=1, 3=3}
lRUCache.get(2);      // returns -1 (not found)
lRUCache.put(4, 4);   // LRU key was 1, evicts key 1, cache is {4=4, 3=3}
lRUCache.get(1);      // return -1 (not found)
lRUCache.get(3);      // return 3
lRUCache.get(4);      // return 4
```

## Constraints:

- 1 ≤ capacity ≤ 3000
- 0 ≤ key ≤ 10⁴
- 0 ≤ value ≤ 10⁵
- At most 2×10⁵ calls will be made to get and put


## Core Idea — HashMap + Doubly Linked List
Why this combination?

- We need O(1) access to check if a key exists and get its value → HashMap
- We need to know which key was least recently used (to evict) and most recently used (to move to front) → Doubly Linked List (fast removal and insertion at both ends)
- HashMap stores key → node mapping for O(1) access to the node
- Doubly Linked List stores the actual order of usage (head = most recent, tail = least recent)

## Operations:

get(key):
- If key not in map → return -1
- Else: get node → move node to front (most recently used) → return value

put(key, value):
If key exists:
- Update value in node
- Move node to front

## Else:
If cache full (size == capacity):
- Remove tail node (least recently used)
- Remove from map

- Create new node with key-value
- Add to map
- Insert node at front



## Doubly Linked List allows:

- Remove any node in O(1) (with direct reference from map)
- Move node to front in O(1)
- Add new node at front in O(1)
- Remove tail in O(1)


## Clean Implementation (C#)
```cpp
public class LRUCache {
    private readonly int capacity;
    private readonly Dictionary<int, LinkedListNode<CacheItem>> cacheMap;
    private readonly LinkedList<CacheItem> cacheList;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        cacheMap = new Dictionary<int, LinkedListNode<CacheItem>>(capacity);
        cacheList = new LinkedList<CacheItem>();
    }
    
    public int Get(int key) {
        if (cacheMap.TryGetValue(key, out var node)) {
            // Move to front (most recently used)
            cacheList.Remove(node);
            cacheList.AddFirst(node);
            return node.Value.Value;
        }
        return -1;
    }
    
    public void Put(int key, int value) {
        if (cacheMap.TryGetValue(key, out var node)) {
            // Update value and move to front
            node.Value.Value = value;
            cacheList.Remove(node);
            cacheList.AddFirst(node);
        } else {
            // Cache full? Evict LRU (tail)
            if (cacheMap.Count >= capacity) {
                var lastNode = cacheList.Last;
                cacheMap.Remove(lastNode.Value.Key);
                cacheList.RemoveLast();
            }

            // Add new item at front
            var newItem = new CacheItem(key, value);
            var newNode = new LinkedListNode<CacheItem>(newItem);
            cacheMap.Add(key, newNode);
            cacheList.AddFirst(newNode);
        }
    }

    private class CacheItem {
        public int Key { get; }
        public int Value { get; set; }

        public CacheItem(int key, int value) {
            Key = key;
            Value = value;
        }
    }
}
```

## Complexity

| **Metric**            | **Value**              | **Notes**                                                                 |
|-----------------------|------------------------|---------------------------------------------------------------------------|
| **Time Complexity**   | **O(1)** average per operation | All core operations (`get`, `put`, move-to-front, evict tail) are constant time on average:<br>• HashMap lookups, inserts, removes: O(1) average<br>• Doubly linked list remove/insert at known position: O(1) |
| **Space Complexity**  | **O(capacity)**        | HashMap stores up to `capacity` key-node mappings<br>Doubly linked list stores up to `capacity` nodes<br>Each node holds a `CacheItem` (key + value) |

**Optimal** — fully satisfies the strict requirement of **O(1) average time** for both `get` and `put` operations.

## Why This Works — Step-by-Step Explanation

**Data Structures Used**:

1. `Dictionary<int, LinkedListNode<CacheItem>> cacheMap`  
   → Provides **O(1)** access to the node containing a given key  
   → Allows instant check if key exists and instant move-to-front

2. `LinkedList<CacheItem> cacheList`  
   → Maintains the **exact order of usage**  
   → Front = **most recently used** (MRU)  
   → Back = **least recently used** (LRU)  
   → Doubly linked list allows O(1) removal from any position (if we have the node reference) and O(1) insert at front

3. `CacheItem` inner class  
   → Stores `Key` (immutable) and `Value` (mutable for updates)

**get(key)** — Return value if exists, else -1:

- Lookup key in `cacheMap` → O(1)
- If not found → return -1
- If found:
  - Get the node
  - Remove node from its current position in list → O(1) (doubly linked list)
  - Move node to front (most recently used) → `AddFirst(node)` → O(1)
  - Return the value stored in node

**put(key, value)** — Update or insert key-value pair:

- Case 1: Key already exists (in cacheMap)
  - Get the node → O(1)
  - Update the value in the node
  - Move node to front (remove from current position → add to front) → O(1)

- Case 2: Key does not exist (new entry)
  - If cache is full (`cacheMap.Count >= capacity`):
    - Get the tail node (LRU) → `cacheList.Last` → O(1)
    - Remove key from `cacheMap` using tail's key → O(1)
    - Remove tail from list → `RemoveLast()` → O(1)
  - Create new `CacheItem(key, value)`
  - Create new `LinkedListNode` with this item
  - Add to `cacheMap` → O(1)
  - Add node to front of list → `AddFirst(node)` → O(1)

**Correctness** —  
The doubly linked list always keeps the **exact order of recent usage**:  
- Every `get` or `put` that touches a key moves it to the front → marks it as most recently used  
- When cache is full and we need to insert, we always evict the **tail** (least recently used)  
- HashMap gives us **instant access** to the node of any key → no need to search the list
