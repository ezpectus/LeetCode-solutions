# 2631. Group By
## O(n) — Prototype Extension with Object Accumulator

---

## Problem Statement

- Extend `Array.prototype` with a `groupBy(fn)` method.
- `fn` takes each array element and returns a **string key**.
- Returns an object where each key maps to an array of all elements that produced that key, **in original order**.
- Do not use `_.groupBy`.

---

## Core Idea — Object Accumulator + Lazy Key Init

**Key insight**: Iterate the array, compute the key for each item via `fn(item)`, and accumulate into a plain object. Initialize each key's bucket on first encounter.

```
for each item:
    key = fn(item)
    if key not in res → res[key] = []
    res[key].push(item)
```

This is the standard **grouping/bucketing** pattern — one of the most fundamental operations in data transformation.

---

## Full Optimal Implementation (JavaScript)

```javascript
Array.prototype.groupBy = function(fn) {
    const res = {};
    for (const item of this) {
        const key = fn(item);
        if (!res[key]) res[key] = [];
        res[key].push(item);
    }
    return res;
};
```

---

## Complexity

| **Metric** | **Value** | **Notes** |
|---|---|---|
| **Time Complexity** | **O(n)** | Single pass; `fn` called once per element |
| **Space Complexity** | **O(n)** | Output object holds all n elements across buckets |

---

## Alternative: `reduce` Version

```javascript
Array.prototype.groupBy = function(fn) {
    return this.reduce((res, item) => {
        const key = fn(item);
        (res[key] ??= []).push(item);
        return res;
    }, {});
};
```

Uses **nullish coalescing assignment** (`??=`): `res[key] ??= []` initializes the bucket only if it's `null` or `undefined` — equivalent to `if (!res[key]) res[key] = []` but more concise.

Both versions are O(n) — pick whichever reads cleaner to you.

## Example Walkthrough

**Example 3**: `[1..10].groupBy(n => String(n > 5))`

| item | key | bucket |
|------|-----|--------|
| 1 | `"false"` | `[1]` |
| 2 | `"false"` | `[1,2]` |
| ... | ... | ... |
| 5 | `"false"` | `[1,2,3,4,5]` |
| 6 | `"true"` | `[6]` |
| ... | ... | ... |
| 10 | `"true"` | `[6,7,8,9,10]` |

Output: `{ "false": [1,2,3,4,5], "true": [6,7,8,9,10] }` ✓

---

## Pitfalls & Edge Cases

- **Must use `function`, not arrow** — `Array.prototype.groupBy = function(fn)` uses `this` to refer to the calling array; an arrow function would capture `this` from the outer scope instead
- **`!res[key]` vs `!(key in res)`** — `!res[key]` is falsy for `0`, `""`, `false` as values, but since the values are arrays (always truthy once initialized), `!res[key]` correctly distinguishes "bucket exists" from "no bucket yet". If values could be falsy primitives, use `!(key in res)` instead
- **Key order not guaranteed** — JS objects maintain insertion order for string keys in modern engines, but the problem explicitly says "any order of keys is acceptable"
- **Empty array** — loop body never executes, returns `{}` ✓
- **`fn` must return string** — per constraints; if it returned objects, they'd all stringify to `"[object Object]"` and collide

---

## Key Takeaway

`groupBy` = **single-pass accumulator with lazy bucket initialization**:

```javascript
const key = fn(item);
if (!res[key]) res[key] = [];
res[key].push(item);
```

Or with modern JS: `(res[key] ??= []).push(item)`.

Pattern: any "group items by some property" task → iterate once, accumulate into object/Map by computed key. This is the building block of countless data transformation pipelines.
