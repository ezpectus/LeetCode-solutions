# 3543. Maximum Weighted K-Edge Path  
## O(k × n × t + k × m × t) — 3D DP Over (Edges, Node, Weight)

---

## Problem Statement

You are given:

- A **Directed Acyclic Graph (DAG)** with `n` nodes labeled `0..n-1`
- A list of directed weighted edges `edges[i] = [u, v, w]`
- Two integers:
  - `k` — the required number of edges in the path  
  - `t` — a strict upper bound on the total path weight

Your task:

> Find the **maximum possible total weight** of any path that uses **exactly k edges** and has total weight **strictly less than t**.  
> If no such path exists, return **-1**.

Constraints:

- `1 ≤ n ≤ 300`
- `0 ≤ m ≤ 300`
- `0 ≤ k ≤ 300`
- `1 ≤ t ≤ 600`
- `1 ≤ w ≤ 10`
- Graph is guaranteed to be a **DAG**

---

## Core Idea — Why 2D DP Fails and 3D DP Is Required

A natural idea is:

```
dp[e][v] = maximum weight to reach node v using exactly e edges
```

This is **incorrect**.

### Why 2D DP fails

Because a **heavier partial path** may block a **lighter partial path** that leads to a valid solution.

Example:

```
0 -> 2 (3)
1 -> 2 (9)
2 -> 3 (5)
t = 11
k = 2
```

- dp[1][2] becomes `9` (the maximum)
- but `9 + 5 = 14 ≥ 11` → invalid  
- the valid path is `3 + 5 = 8 < 11`, but it gets overwritten  
- final answer incorrectly becomes `-1` instead of `8`

### Correct approach: track **all reachable weights**, not just the maximum

We need:

```
dp[e][v][w] = true  
    if there exists a path ending at node v
    using exactly e edges
    with total weight w (< t)
```

This ensures:

- lighter paths are preserved
- all valid weight combinations are explored
- the final maximum weight < t is correctly computed

---

## DP Definition

```
dp[e][v][w] = reachable?
```

Where:

- `e` = number of edges used (0..k)
- `v` = ending node (0..n-1)
- `w` = total weight (< t)

### Initialization

A path of 0 edges and weight 0 is possible at **every node**:

```
dp[0][v][0] = true
```

This matches the problem statement:  
**paths may start at any node**.

### Transition

For each reachable state:

```
dp[e][u][w] = true
```

and each outgoing edge:

```
u -> v with weight wt
```

we update:

```
if w + wt < t:
    dp[e+1][v][w + wt] = true
```

### Final Answer

Among all nodes `v` and all weights `w < t`:

```
answer = max w such that dp[k][v][w] = true
```

If no such state exists → return `-1`.

## Full Correct Implementation (C++)

```cpp
class Solution {
public:
    int maxWeight(int n, vector<vector<int>>& edges, int k, int t) {
        // dp[e][v][w] = reachable?
        static bool dp[301][301][601];

        // Clear DP
        for (int e = 0; e <= k; e++)
            for (int v = 0; v < n; v++)
                for (int w = 0; w < t; w++)
                    dp[e][v][w] = false;

        // 0 edges → weight 0 at every node
        for (int v = 0; v < n; v++)
            dp[0][v][0] = true;

        // adjacency list
        vector<vector<pair<int,int>>> g(n);
        for (auto &e : edges) {
            int u = e[0], v = e[1], w = e[2];
            g[u].push_back({v, w});
        }

        // DP transitions
        for (int e = 0; e < k; e++) {
            for (int u = 0; u < n; u++) {
                for (int w = 0; w < t; w++) {
                    if (!dp[e][u][w]) continue;

                    for (auto &[v, wt] : g[u]) {
                        int nw = w + wt;
                        if (nw < t) {
                            dp[e + 1][v][nw] = true;
                        }
                    }
                }
            }
        }

        // Find best weight < t for exactly k edges
        int ans = -1;
        for (int v = 0; v < n; v++)
            for (int w = 0; w < t; w++)
                if (dp[k][v][w])
                    ans = max(ans, w);

        return ans;
    }
};
```

This fits comfortably within limits.

## Example Walkthrough (The Failing Case)

Input:

```
n = 4
edges = [[0,1,4],[0,2,3],[1,2,9],[2,3,5],[0,3,5]]
k = 2
t = 11
```

### Step 1 — dp[1][*][*]

From node 0:

- 0 → 1 (4)
- 0 → 2 (3)
- 0 → 3 (5)

From node 1:

- 1 → 2 (9)

From node 2:

- 2 → 3 (5)

So dp[1][2] contains **both**:

- weight 3  
- weight 9  

### Step 2 — dp[2][*][*]

From weight 3 at node 2:

```
3 + 5 = 8 < 11 → valid
```

From weight 9 at node 2:

```
9 + 5 = 14 ≥ 11 → invalid
```

Thus:

```
dp[2][3][8] = true
```

Final answer: **8**

---

## Pitfalls & Edge Cases

- **You must track all weights**, not only the maximum.
- **Paths may start at any node** → dp[0][v][0] = true.
- **Strict inequality**: weight must be `< t`, not `≤ t`.
- **k = 0** → answer is 0 if `0 < t`, else -1.
- **DAG guarantee** simplifies transitions (no cycles).
- **Weight dimension is small (≤ 600)** → DP is feasible.

---

## Key Takeaway

 This problem is a classic example where:

> **Maximizing too early destroys valid states.**

 When future transitions depend on weight constraints,  
you must preserve **all reachable weights**, not just the best one.

The correct solution is a clean, efficient **3D DP over (edges, node, weight)**.
