# 3665. Twisted Mirror Path Count

## O(m × n) — DP with Direction State

---

## Problem Statement

- Given m×n binary grid: 0 = empty, 1 = mirror.
- Robot starts at (0,0), goal (m-1, n-1).
- Can move right or down. **Mirrors reflect**:
  - Right into mirror → turned down, moves below mirror
  - Down into mirror → turned right, moves right of mirror
- Chained reflections continue until non-mirror or out of bounds.
- Count unique valid paths modulo 10⁹+7.
- **Constraints**: 2 ≤ m, n ≤ 500, grid[0][0] = grid[m-1][n-1] = 0

---

## Core Idea — DP with Direction Tracking

**Key observation**: Need to track **direction** robot is moving when entering each cell, since mirrors reflect based on entry direction.

**State**: `dp[r][c][dir]` = number of ways to reach (r,c) while moving in direction `dir`
- `dir = 0`: moving right
- `dir = 1`: moving down

**Transition**: From (r,c) with direction `dir`:
1. Calculate next position based on `dir`
2. If next is mirror, apply reflection (possibly chained)
3. Add paths to final position with final direction

**Base case**: `dp[0][0][0] = 1` and `dp[0][0][1] = 1` (start can go either direction)

Actually, better base: start at (0,0), can move right or down from there.

---

## Algorithm Breakdown

### Reflection Logic

```
Move(r, c, dir):
  while true:
    if dir == 0: // Moving right
      next_r, next_c = r, c + 1
    else: // Moving down
      next_r, next_c = r + 1, c
    
    if out of bounds:
      return null (invalid)
    
    if grid[next_r][next_c] == 0: // Empty cell
      return (next_r, next_c, dir)
    
    // Mirror: reflect
    if dir == 0: // Was moving right
      dir = 1 // Turn down
      r, c = next_r, next_c // Continue from mirror
    else: // Was moving down
      dir = 0 // Turn right
      r, c = next_r, next_c
```

### DP Iteration

```
dp[0][0][0] = 1 // Can start moving right
dp[0][0][1] = 1 // Can start moving down

For each (r, c) in topological order:
  For each dir:
    if dp[r][c][dir] == 0: continue
    
    result = Move(r, c, dir)
    if result != null:
      (nr, nc, ndir) = result
      dp[nr][nc][ndir] += dp[r][c][dir]

return dp[m-1][n-1][0] + dp[m-1][n-1][1]
```

## Full Implementation (C#)

```csharp
public class Solution {
    private const int MOD = 1_000_000_007;
    
    public int UniquePaths(int[][] grid) {
        int m = grid.Length;
        int n = grid[0].Length;
        
        // dp[r][c][dir] = ways to reach (r,c) arriving from direction dir
        // dir: 0 = arrived from left (was moving right), 1 = arrived from top (was moving down)
        long[,,] dp = new long[m, n, 2];
        
        // Base case: start at (0,0)
        // We consider it as if we can "arrive" at start from either direction
        dp[0, 0, 0] = 1; // Consider as arriving from left
        dp[0, 0, 1] = 1; // Consider as arriving from top
        
        // But actually, we need to process moves FROM (0,0)
        // Let's use a different approach: track states we can transition FROM
        
        var memo = new Dictionary<(int r, int c, int dir), long>();
        
        long CountPaths(int r, int c, int dir) {
            if (r == m - 1 && c == n - 1) {
                return 1;
            }
            
            var key = (r, c, dir);
            if (memo.ContainsKey(key)) {
                return memo[key];
            }
            
            long count = 0;
            
            // Try moving in both directions from current cell
            for (int nextDir = 0; nextDir < 2; nextDir++) {
                var result = Move(grid, r, c, nextDir, m, n);
                if (result != null) {
                    count = (count + CountPaths(result.Value.r, result.Value.c, result.Value.dir)) % MOD;
                }
            }
            
            memo[key] = count;
            return count;
        }
        
        // Start from (0,0), can move right or down
        long total = 0;
        
        // Try moving right from start
        var rightMove = Move(grid, 0, 0, 0, m, n);
        if (rightMove != null) {
            total = (total + CountPaths(rightMove.Value.r, rightMove.Value.c, rightMove.Value.dir)) % MOD;
        }
        
        // Try moving down from start
        var downMove = Move(grid, 0, 0, 1, m, n);
        if (downMove != null) {
            total = (total + CountPaths(downMove.Value.r, downMove.Value.c, downMove.Value.dir)) % MOD;
        }
        
        return (int)total;
    }
    
    private (int r, int c, int dir)? Move(int[][] grid, int r, int c, int dir, int m, int n) {
        while (true) {
            // Calculate next position based on current direction
            int nr = dir == 1 ? r + 1 : r; // down
            int nc = dir == 0 ? c + 1 : c; // right
            
            // Check bounds
            if (nr >= m || nc >= n) {
                return null;
            }
            
            // Empty cell: stop here
            if (grid[nr][nc] == 0) {
                return (nr, nc, dir);
            }
            
            // Mirror: reflect and continue
            r = nr;
            c = nc;
            dir = 1 - dir; // Flip direction
        }
    }
}
```

## Complexity

| **Metric** | **Value** |
|---|---|
| **Time** | **O(m × n × k)** where k = max reflection chain length |
| **Space** | **O(m × n)** |

Worst case k ≈ m+n (diagonal mirror chain), so O(m × n × (m+n)).

## Example Walkthrough

### Example 1: `grid = [[0,1,0],[0,0,1],[1,0,0]]`

**DP iteration**:

**(0,0) → right**:
- Try move right to (0,1)
- (0,1) is mirror → reflect down → move to (1,1)
- `dp[1][1][1] += dp[0][0][0]` → dp[1][1][1] = 1

**(0,0) → down**:
- Try move down to (1,0)
- (1,0) is empty → stop
- `dp[1][0][1] += dp[0][0][1]` → dp[1][0][1] = 1

**(1,0) → down**:
- Try move down to (2,0)
- (2,0) is mirror → reflect right → move to (2,1)
- `dp[2][1][0] += dp[1][0][1]` → dp[2][1][0] = 1

**(1,1) → right**:
- Try move right to (1,2)
- (1,2) is mirror → reflect down → move to (2,2)
- `dp[2][2][1] += dp[1][1][0]` = ...

(continue for all states)

**Result**: 5 paths ✓

## Key Takeaway

**DP with direction state tracking**. Simulate mirror reflections with chained loops. State: (position, direction). Transitions handle reflections until reaching non-mirror cell. O(m × n × chain_length) time.
