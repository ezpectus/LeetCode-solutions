# 🏔️ 3904. Smallest Stable Index II — Architectural Solution

## Problem Statement

You are given an integer array `nums` of length `n` and an integer `k`.
For each index `i`, we define its **instability score** as:
`max(nums[0..i]) - min(nums[i..n - 1])`

An index `i` is called **stable** if its instability score is less than or equal to `k`.
Return the **smallest stable index**. If no such index exists, return `-1`.

### Constraints
- 1 <= n <= 10^5
- 0 <= nums[i] <= 10^9
- 0 <= k <= 10^9

---

## 💡 Core Idea

A brute-force approach that calculates the maximum of the prefix and the minimum of the suffix for each index `i` independently would take O(n^2) time, which is too slow for `n = 10^5`. 

To optimize this to O(n) time, we can precompute the boundary values:
1. **Suffix Minimums**: We can construct a `suffixMin` array by scanning from right to left. `suffixMin[i]` will store the minimum value from index `i` to `n - 1`.
2. **Prefix Maximums**: Instead of storing this in an array, we can maintain a running variable `currentMax` as we iterate from left to right.

By combining the running `currentMax` and the precomputed `suffixMin[i]`, we can evaluate the stability condition for each index `i` in O(1) constant time. Returning the first index that satisfies the condition guarantees that we find the smallest stable index.

---

## 💻 Implementation (C++)

```cpp
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    int firstStableIndex(vector<int>& nums, int k) {
        int n = nums.size();
        if (n == 0) return -1;

        // Step 1: Precompute suffix minimums from right to left
        vector<int> suffixMin(n);
        suffixMin[n - 1] = nums[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            suffixMin[i] = min(nums[i], suffixMin[i + 1]);
        }

        // Step 2: Track prefix maximums from left to right and find the first stable index
        int currentMax = nums[0];
        for (int i = 0; i < n; i++) {
            currentMax = max(currentMax, nums[i]);
            
            // Instability score calculation
            if (currentMax - suffixMin[i] <= k) {
                return i; // Return immediately since we look for the smallest index
            }
        }

        return -1;
    }
};
```

## 📐 Complexity Analysis

| Metric | Value | Justification |
| :--- | :--- | :--- |
| **Time Complexity** | O(n) | **Suffix Pass**: Populating the `suffixMin` array takes a single backward pass of O(n) steps.<br>**Prefix Pass**: Finding the index takes a single forward pass of at most O(n) steps. All internal operations inside the loops are basic arithmetic updates taking O(1) time. |
| **Space Complexity** | O(n) | **Suffix Array**: An auxiliary array of size `n` is allocated to store the suffix minimums. The prefix maximums are maintained using a single scalar variable (`currentMax`), requiring O(1) extra space. |

## ⚠️ Pitfalls & Edge Cases

- **Single Element Arrays**: If `n = 1`, the loop runs once for `i = 0`. `currentMax` equals `nums[0]`, and `suffixMin[0]` equals `nums[0]`. The score is `nums[0] - nums[0] = 0`. Since `0 <= k`, index `0` is always returned. This matches Example 3 correctly.
- **Overlapping Bounds at Index i**: Notice that `nums[i]` is included in both the prefix slice `nums[0..i]` and the suffix slice `nums[i..n-1]`. The logic accounts for this naturally by processing `nums[i]` in both checks.
- **No Overflow during Subtraction**: Since `0 <= nums[i] <= 10^9`, the maximum possible value for `currentMax` is 10^9 and the minimum possible value for `suffixMin` is 0. The maximum difference is 10^9, which safely fits within a standard 32-bit signed integer (`int`).

## 🔄 Alternative Approaches

- **Two Pointer / Monotonic Stack**: Using a monotonic stack can optimize space complexity down to O(1) auxiliary space by tracking changes dynamically. However, since the problem requires range minimums starting exactly from index `i` to the end, a precomputed suffix array remains the cleanest and most readable implementation strategy.
- **Segment Tree / Monotonic Queue**: A Segment Tree could perform range max and range min queries dynamically. This approach would require O(n) space and increase the runtime lookup to O(n log n). It is over-engineered for this scenario because the windows always stretch to the absolute boundaries (`0` and `n-1`).
